# Ad-Hoc Struct Unions as Allocation-Free Tagged Sums — Field-Collapse Algebra and the Default-Poison Typestate

[REPRESENTATION_ECONOMICS]:
- A struct ad-hoc owner is a value-type tagged sum: one private 1-based `int _valueIndex` plus the minimal field set the member algebra demands, transported entirely on the stack — the discriminant is the tag, the typed fields are the payload arena, and no allocation, no boxing, and no vtable touch the value across its lifetime. This is the only union form whose dispatch is a jump table on an integer rather than a runtime type-test ladder, so the per-dispatch cost is one bounds-free `switch` over a dense `int` versus N sequential `is`-pattern checks.
- The backing-field set is computed, not one-per-member: a private shared `object? _obj` field is emitted once and absorbs every reference-type member when two or more *stateful, non-type-parameter, non-duplicate* reference members coexist; below that threshold each reference member keeps its own typed field. The collapse predicate is exact — reference-typed, not a type parameter, duplicate-counter at most one, and not stateless — counted at two — so adding a third reference case can flip the entire family's storage shape from typed fields to one boxed slot, a silent representation change that alters nothing observable but the memory diagram.
- Struct members never join `_obj`: each stateful value-type member retains a dedicated typed field so its bits live inline with no box, and the owner's total footprint is the discriminant plus the largest concurrently-live field cluster — value members cost their own width always, reference members cost one shared pointer once the collapse fires. The owner therefore sizes to its widest *struct* member plus one pointer plus the tag, not to the sum of all payloads, because `[StructLayout(LayoutKind.Auto)]` is emitted whenever no layout is user-declared, handing the runtime permission to overlap-pack and reorder the heterogeneous fields and the tag without the padding holes a hand-declared sequential layout would freeze in.
- `UseSingleBackingField` forces the `_obj` collapse for the whole family regardless of count, boxing every struct member at construction and making the object slot the sole storage — the representation degrades to a uniform boxed cell, trading the inline-value-type win for one fixed field shape. Choose it only when the family is reference-dominated and the residual struct members are rare; otherwise it pays a box on the hot value cases to save one field declaration.
- Stateless members carry *no field at all*: `T1IsStateless = true` stores only the discriminant, the accessor returns `default(T)`, intra-case equality is constant-true, and the case occupies exactly the tag value — an empty marker struct as a stateless member is a zero-width alternative whose entire runtime identity is its index, the densest possible "this case is present, it has no data" encoding and strictly cheaper than a nullable reference marker because it has no null channel to mis-handle.

[DEFAULT_POISON_AS_TYPESTATE]:
- `default(struct-union)` is not the first case — it is index 0, a state the constructor set cannot produce, and every value-bearing surface treats it as poison: `Value`, `ToString`, `GetHashCode`, `Equals`, both `Switch` forms, and `Map` each emit a leading `case 0` arm that throws `InvalidOperationException` naming the struct type and instructing initialization. The zero index is the language's unavoidable default-struct hole reified as a runtime-detectable invalid state, so a struct ad-hoc union is a value type whose `default` is definitionally uninhabited yet syntactically constructible.
- The poison is invisible to the nullable flow analysis the language otherwise relies on: a `default(T)` struct union satisfies `[NotNull]`, flows through non-nullable parameters, and lives in arrays and uninitialized fields with zero diagnostic, because the type *is* a non-null value type — the invalid state hides inside a fully-inhabited static type. The hole the compiler closes for reference absence (the `?` channel) has no analogue here; the discriminant carries a state the type system cannot name, so the only proof of validity is a runtime probe, never a compile-time annotation.
- The analyzer catches the literal sources — bare `default(T)`, `new T()` with no factory, an undeclared required field — but the poison escapes through every indirection the analyzer cannot see: `new T[n]` zero-fills its elements, `default(TGeneric)` flowing through an unconstrained type parameter erases the concrete union, and a struct union nested as a field inside another uninitialized struct inherits the zero-state transitively. Each escape route ends at the same runtime throw, which means the *crash site is the first observation* — hashing the value into a dictionary, logging it, or comparing it is itself the fault, far from the array allocation or generic call that minted the poison.
- `Is{Member}` is the one generated member that is total over the poisoned state and never throws: it is `this._valueIndex == k`, a pure discriminant compare that reads `0` as "not this case" rather than as an error. Every rehydration, pooling, array-scan, and deserialization seam that might hold a default-constructed value guards with an `Is`-probe *before* touching any value-bearing member — the probe is the typestate test the language refuses to express, and a fold over the `Is`-probes reconstructs "is this value initialized at all" as the disjunction across all member indices.

```csharp
[Union<Anchor, Drift>(T2IsStateless = true)]
public readonly partial struct Cursor;

public readonly record struct Drift;

// A scan that may encounter zero-filled array slots guards on the throw-free probe,
// never on Value/Switch, which would fault on the poisoned default.
public static Seq<Anchor> Anchors(ReadOnlySpan<Cursor> window)
{
    var live = Seq<Anchor>.Empty;
    foreach (var c in window)               // span kernel: the named statement exemption
        live = (c.IsAnchor, c.IsDrift) switch
        {
            (true, _) => live.Add(c.AsAnchor),
            _ => live,                       // index 0 and Drift both fall here, no throw
        };
    return live;
}
```

[THROW_SET_AS_DISCRIMINANT_NET]:
- The invalid-index throws split by surface and the split is exact: `Value`, both `Switch` action and func forms, and the per-member `As{Member}` mismatch all throw `IndexOutOfRangeException` for an impossible discriminant, while `Map` and `MapPartially` throw `ArgumentOutOfRangeException`. The cut is value-projection-versus-map, not per-method, because the eager-arm map surface reasons about its argument array while the value surface reasons about the field index — two different "unexpected" framings, two exception types, one underlying corruption.
- `As{Member}` on the wrong live case is a distinct failure from the poisoned default: a *correctly-initialized* union asked for the wrong member throws `InvalidOperationException` naming the requested type, the cast-assertion conversion operators do the same, and only the impossible-index path reaches `IndexOutOfRangeException`. The taxonomy is therefore three-tier — `InvalidOperationException` for poisoned-default and wrong-case-projection, `IndexOutOfRangeException` for corrupted-discriminant on value/switch surfaces, `ArgumentOutOfRangeException` for the same corruption on map surfaces — and the exception type alone localizes which invariant broke without a stack walk.
- The throw set is the representation's last line of defense precisely because the discriminant is a raw `int` with no enum domain restriction: nothing in the type prevents a corrupted or deserialized value carrying an out-of-range index, so the generated `_ =>`/`default:` arm exists to convert silent memory corruption into a named exception at the first dispatch rather than a wrong-arm execution. An intact value never reaches it; the arm is the backstop for binary corruption, struct-tearing under torn reads, or a reflection-built instance bypassing the constructors.

[EQUALITY_OVER_TAGGED_BITS]:
- Generated equality is discriminant-gated then field-dispatched: `_valueIndex` inequality short-circuits to false before any field is read, so cross-case comparison never touches a payload, and only same-index values reach the per-member equality arm. The hash mirrors the gate — it switches on the index and hashes only the active member's field — but deliberately *omits the discriminant from the mix*, so two values in different cases can collide in hash while remaining unequal, an asymmetry that is correct for hashing (equality is still exact) but means index-disjoint cases are not hash-separated.
- Reference-type member equality is null-safe by construction, not by `==`: the emitted arm is `this.field is null ? other.field is null : field.Equals(other)`, so a stateless-derived null reference and a genuinely-null payload compare structurally without a NullReferenceException, and the shared-`_obj` collapse adds an index re-check (`this._valueIndex == other._valueIndex &&`) before the field compare because one object field now serves several cases and the index alone disambiguates which member's `Equals` semantics apply.
- String members carry the family's `DefaultStringComparison` directly into both the equality `Equals(other, comparison)` arm and the hash, so a union over string members is case-insensitive-equal by the library default until pinned to `StringComparison.Ordinal` at the owner — the comparison policy is a representation-level field property, not a call-site decision, and two textually-distinct strings can be union-equal silently. Type-parameter members route through `EqualityComparer<T>.Default` for both equality and hash, so a generic member inherits whatever equality the closed instantiation supplies, computed without boxing the value type.
- `==`/`!=` are the generic-math `IEqualityOperators<TSelf, TSelf, bool>` implementation, not hand-written operators, so a struct ad-hoc union is usable as a generic-math-constrained operand and its equality composes into any algorithm written against the operator interface — the value-type sum participates in constraint-based numeric and comparison generics the same way a primitive does, with the discriminant-gated structural equality underneath.

[STRUCT_VERSUS_CLASS_REPRESENTATION_TRADE]:
- The struct-versus-class ad-hoc choice is a trade of *failure channels*, not safety: the struct buys allocation-free stack transport and a jump-table dispatch at the cost of a poisoned `default` that hides from nullability analysis; the class buys an ordinary `null` reference channel the language already polices and a runtime-type dispatch at the cost of one heap allocation per value and reference-identity `Equals(object)` for a ref-struct owner. Pick the struct where the surrounding code already polices initialization (admission boundaries, pooled buffers, span pipelines) and the class where the code already polices null — the union does not remove the absence discipline, it relocates it to whichever channel the owner kind exposes.
- A ref-struct ad-hoc owner is the extreme of the value form: it is legal, `Equals(object?)` hard-returns false (a ref struct can never be boxed to compare), the equatable and operator interfaces are skipped entirely, and `allows ref struct` anti-constraints on the owner's *own* type parameters are rejected — the representation is stack-bound, equality-less, and exists to carry one of several stack-only payloads through a single dispatch without ever escaping to the heap, the densest possible "sum of spans" carrier.
- The class form's representation is uniform — one reference, runtime-type dispatch — and pays per-value allocation, but its `default` is an honest `null` that the entire language already guards, so it carries no poison typestate; the cost asymmetry is that the struct's invalid state is *unnameable but free to construct* while the class's invalid state is *nameable and the same as every other reference's*. Where a union must live in large arrays or flow through hot value-type code, the struct's allocation win dominates; where it must round-trip through reflection, serialization, or any path that zero-fills, the class's honest null is worth the allocation because it cannot mint a poison the analyzer misses.
