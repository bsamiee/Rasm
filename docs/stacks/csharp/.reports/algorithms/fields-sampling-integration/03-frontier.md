# Fields, Sampling, and Integration — Frontier

[SLICE_SAMPLER_MECHANICS]:
- `UnivariateSliceSampler` implements Neal stepping-out with no expansion budget: the bracket `[low, high]` expands by `Scale` in each direction until the log-density falls below the slice level, then shrinks by retraction on rejection. There is no cap on bracket expansion — a nearly-flat log-density causes unbounded bracket growth proportional to `Scale` per iteration. `Scale` must be calibrated to the tail slope of the field-derived log-density, not left at the constructor default.
- `UnivariateSliceSampler` runs `Burn(burnInterval)` synchronously in its constructor body; `AcceptRate` computed before any `Sample()` call returns zero, not the burn-phase acceptance rate. `Scale` is mutable post-construction, allowing adaptive scale tuning between `Sample()` calls without restarting the chain. A scale increase that widens the bracket can cause an immediate rejection cycle if the density has already tightened around the current location.

[MCMC_DIAGNOSTICS_SURFACE]:
- `MCMCDiagnostics.ACF<T>` computes autocorrelation at a single lag by materializing the series to an array and calling `Correlation.Pearson` on shifted windows — `O(N)` per lag, `O(N * maxLag)` for a lag ladder. For a full autocorrelation spectrum, `Correlation.Auto(double[])` uses `Fourier.Forward`/`Fourier.Inverse` with `AsymmetricScaling` and zero-padding to `CeilingToPowerOfTwo * 2`, delivering `O(N log N)`. The two are not composable: `MCMCDiagnostics.ACF` accepts `IEnumerable<T>` with a projection delegate; `Correlation.Auto` requires `double[]` directly.
- `MCMCDiagnostics.EffectiveSize<T>` uses only lag-1 autocorrelation via `(1 - ρ₁)/(1 + ρ₁) * N`. This is the first-order approximation; the full Geyer monotone sequence estimator sums positive initial sequence terms across all positive lags. The lag-1 approximation overestimates effective sample size when the chain has long-range positive autocorrelation — a common case for HMC chains with large step sizes.
- `RejectionSampler<T>.Sample` throws `ArgumentException` at runtime when `pdfQ(val) < pdfP(val)` for any sampled point. The proposal envelope must dominate — this is a design-time contract violation that surfaces as a runtime exception with no diagnostic about which point violated the bound. A field-derived target density modified after construction (truncated SDF-based density, clamped kernel) must re-verify the envelope condition after every modification.
- `MetropolisHastingsSampler<T>.Burn` computes the log Hastings ratio and accepts deterministically when it equals exactly `0.0`. Floating-point equality to `0.0` fires only when all four log-density values cancel perfectly; in practice, for floating-point field log-densities, `Bernoulli.Sample(RandomSource, Math.Exp(ratio))` handles all acceptance, and the deterministic branch is dead code.

[QUANTILE_ESTIMATION_PIPELINE]:
- `SortedArrayStatistics.Quantile(data, tau)` hardcodes R-8 (approximately median-unbiased): `h = (N + 1/3) * tau + 1/3`, interpolating between adjacent order statistics. For sampling acceptance gates based on empirical quantile thresholds, R-8 is appropriate; for strict compatibility with a specific statistical software's quantile definition, use `QuantileCustom(data, tau, QuantileDefinition)`. Named aliases in the enum include `R7 = Excel`, `R8 = Median`, `R6 = Weibull = Nist = SPSS`, `R5 = Hazen`.
- `QuantileCustom(data, tau, a, b, c, d)` follows the Mathematica four-parameter form directly: `h = a + (N + b)*tau`. This is the correct route for algorithmic compatibility with Mathematica's `Quantile[data, tau, {a,b,c,d}]`.
- Extension methods such as `IEnumerable<double>.QuantileCustomFunc(QuantileDefinition)` materialize the data to a sorted array once and return a `Func<double, double>` that queries without re-sorting. For field-sampled distributions requiring many quantile probes (adaptive binning, percentile-based normalization), this amortizes the `O(N log N)` sort cost across all calls.

[FOURIER_FIELD_TRANSFORM_LAW]:
- `Fourier.ForwardMultiDim(Complex[] samples, int[] dimensions, FourierOptions)` performs a multi-dimensional DFT over a flat row-major array. The `FourierOptions` enum controls convention: `Default = SymmetricScaling` (`1/√N` in both directions), `AsymmetricScaling` (no forward scaling, `1/N` inverse), `NoScaling | InverseExponent`. For field power spectrum computation, the forward transform under `AsymmetricScaling` followed by pointwise `|X[k]|²` is the standard convolution-theorem path; using `SymmetricScaling` introduces an incorrect `1/N` factor.
- `Fourier.Forward2D(Matrix<Complex> samples, FourierOptions)` extracts the matrix to a flat row-major `Complex[]` via `Storage.AsArray()` when the matrix is column-major (`FortranArrayStorage`), requiring an internal copy. For 2D FFT over row-major dense matrices, `Forward2D(Complex[] samplesRowWise, int rows, int columns, FourierOptions)` avoids the copy.
- `Correlation.Auto(double[] x)` zero-mean-centers the data before the FFT and normalizes by the zero-lag variance, so the autocorrelation at lag zero is always `1.0` by construction — not the empirical variance. Callers expecting unnormalized autocorrelation must multiply by the sample variance. The method zero-pads to `CeilingToPowerOfTwo * 2`, guaranteeing linear (acyclic) convolution with no wrap-around aliasing.

[SIMPSONRULE_AND_QUADRATIC_SPLINE_GAP]:
- `SimpsonRule.IntegrateComposite` requires `numberOfPartitions` to be even — passing an odd value throws `ArgumentException` with no automatic rounding. In an adaptive field integration loop that bisects sub-intervals, any bisection from an even count preserves evenness; an initial odd-count call is a hard error with no recovery.
- `LogLinear` stores `log(y)` in a `LinearSpline` and exponentiates at evaluation time. `Differentiate(t)` is `Interpolate(t) * _spline.Differentiate(t)` — chain rule through the log-space spline. `Differentiate2(t)` composes the linear spline's piecewise-constant second derivative (zero within each segment) with the first derivative, yielding `(logSlope)² * Interpolate(t)` within each segment — a piecewise-constant second derivative in log-space. The `InterpolateSorted` factory parallelizes the `log` computation via `CommonParallel.For` with 4096-element chunks.

[COMBINATORICS_AND_SAMPLING_WITHOUT_REPLACEMENT]:
- `Combinatorics.GenerateCombination(n, k, randomSource)` uses two paths based on density: when `k * 3 < n` (sparse), it fills a `HashSet<int>` with repeated sampling; when `k * 3 >= n` (dense), it generates a full permutation and takes the first `k`. The sparse path has expected cost `O(k * n/(n-k))` due to collision retries; near-half selections (`k ≈ n/2`) trigger the dense path at `O(n)`.
- `Combinatorics.SelectPermutationInplace<T>` implements Fisher-Yates (Durstenfeld) in-place shuffle. Passing a seeded `MersenneTwister` produces a deterministic, serializable permutation sequence for reproducible field sampling traversal order. The LINQ `SelectPermutation` variant copies and yields lazily, avoiding full materialization when only a prefix of the permutation is needed.
- `Combinatorics.GenerateVariation(BigInteger n, int k, randomSource)` samples `k` distinct indices from `[0, n)` using `RandomExtensions.NextFullRangeInt64` chained via `BigInteger` arithmetic. For field sampling from large discrete spaces (voxel grids beyond `int.MaxValue`), this is the only correct path; the `int`-overloaded `GenerateVariation(int n, k)` silently overflows when `n > int.MaxValue`.

[RUNNING_STATISTICS_PARALLEL_MERGE]:
- `RunningStatistics.Combine(a, b)` implements Chan's parallel online algorithm for combined mean, variance, skewness, and kurtosis, with the fourth-moment formula accumulating the delta and cross-terms exactly. `operator+` is sugar over `Combine`. The type is `[DataContract]` serializable with seven `[DataMember]` fields, enabling distributed field sampling pipelines where each worker serializes a local `RunningStatistics` and the aggregator calls `Combine` on all partial results.
- `RunningStatistics.Push(double value)` propagates `NaN`: `if (value < _min || double.IsNaN(value))` unconditionally sets `_min = value`. A single `NaN` in the field sample stream permanently poisons `Minimum`, `Maximum`, and all moment estimates. There is no `Reset()` method — recovery requires constructing a new instance.

[DISTANCE_METRIC_SURFACE_AND_GAPS]:
- `Distance.SSD(double[] a, double[] b)` dispatches to `LinearAlgebraControl.Provider.SubtractArrays` and `DotProduct` — BLAS-backed when a native provider is loaded. `Distance.Euclidean` wraps `Math.Sqrt(SSD(a, b))` without scaled-accumulation overflow protection; for field comparison vectors with components above `√(Double.MaxValue / N)`, the intermediate squared sum overflows silently.
- `Distance.Canberra(double[] a, double[] b)` computes `Σ|a-b|/(|a|+|b|)` with a scalar loop. The denominator is zero when both `a[i] == 0` and `b[i] == 0`; the implementation returns `NaN` for that term (IEEE `0/0`), which propagates into the sum. For sparse field vectors with matching zero components, the denominator must be guarded.
- `Distance.Cosine(double[] a, double[] b)` calls `LinearAlgebraControl.Provider.DotProduct` three times. The formula `1 - dot/sqrt(a·a * b·b)` does not handle zero or near-zero vectors — `sqrt(0 * ...)` produces `NaN`. `Distance.Pearson` delegates to `Correlation.Pearson`, which uses a two-pass Welford-style computation and returns `NaN` for constant series, making it the safer route for field sample correlation where any dimension may be constant.

[AKIMA_AND_MAKIMA_WEIGHT_DETAIL]:
- `CubicSpline.InterpolateAkimaSorted` assigns zero weight to both neighbors when `|Δm_{i+1} - Δm_{i-1}| = 0` and `|Δm_i - Δm_{i-2}| = 0` simultaneously. The fallback in this degenerate case is a harmonic mean of neighboring slopes weighted by interval widths: `((x[k+1]-x[k])*m[k-1] + (x[k]-x[k-1])*m[k]) / (x[k+1]-x[k-1])`. This produces a linearly interpolated tangent, not a zero tangent — callers expecting zero gradient at inflection points of smooth fields should use `InterpolatePchipSorted` instead.
- `CubicSpline.InterpolateMakimaSorted` computes weights `|Δm_{l+2} - Δm_{l+1}| + 0.5 * |Δm_{l+2} + Δm_{l+1}|`. When the total weight is zero (all four surrounding differences vanish), the fallback is `0.5 * (m[n+3] + m[n])` — the arithmetic mean of the extended-boundary slopes. This prevents the zero-weight artifact that Akima produces at constant-slope regions, at the cost of a boundary-slope dependency in interior nodes.

[TENSOR_OPERATION_INJECTION_SURFACE]:
- `TensorPrimitives.SoftMax(ReadOnlySpan<float>, Span<float>)` directly calls `Exp` over the input span, then divides by the sum. It does not subtract the maximum before exponentiation — numerically unstable for inputs above ~88 for `float`, where `Exp` overflows to `Infinity`. The numerically stable log-sum-exp-stabilized softmax requires manual pre-subtraction of `TensorPrimitives.Max(x)` before calling `Exp`. For field-derived energy distributions with unscaled magnitudes, the naive `SoftMax` call produces `NaN` or `Infinity` without error.
- `TensorPrimitives.IUnaryOperator<TInput, TOutput>` is `internal`. The public custom-operator path is `TensorOperation.IUnaryOperation_Tensor<T, TResult>`, whose `TensorOperation.Invoke` dispatch loop iterates over `FlattenedLength` using `AdjustToNextIndex` stride arithmetic and is scalar-only at the dispatch level. For SIMD-accelerated custom field kernels, the span overload body must call `TensorPrimitives.*` primitives directly rather than relying on `TensorOperation.Invoke` to vectorize the loop.
