# Absence and Sentinels — Frontier

[RETRY_AS_TRANSIENT_ABSENCE_PROBE]:
- `retryUntil(schedule, ma, e => e.HasCode(terminalCode))` is the correct surface for re-probing a transiently absent capability: the predicate stops retry when the error is terminal and allows retry on transient codes. `retryWhile` is the complement — retry continues while the predicate holds. Both accept a `Schedule` that bounds retry duration and controls inter-probe delay; without a schedule, retry is unbounded.
- Each retry iteration wraps its attempt in `BracketFail()`, which calls `WithEnvFail((EnvIO env) => env.LocalResources)`. `LocalResources` creates a new `Resources` tracker while inheriting the outer `CancellationToken`. Each attempt therefore gets an isolated resource scope — acquired-and-failed resources are released on that attempt's failure — while outer cancellation still terminates the loop.
- `retryUntil` without a schedule recurses via `Catch(BracketFail(), e => !predicate(e) ? RetryUntil(predicate) : IO.fail(e))`. This is trampolined recursion on the `IO` data structure, not stack recursion; the `IO` interpreter handles the chain iteratively and the stack does not grow per attempt.

[BATCH_ABSENCE_PARTITION]:
- `FinExtensions.Partition<F, A>()` operates on `K<F, Fin<A>>` for any `F : Foldable<F>` and yields `(Seq<Error> Fails, Seq<A> Succs)` in a single fold. When a boundary sweeps a collection of capability probes, `Seq<Fin<CapabilityResult>>` folded through `Partition()` extracts both typed failures and usable results without short-circuiting and without discarding either side.
- `partitionFallible<F, M, A>(K<F, K<M, A>> fma) where F : Foldable<F>, M : Monad<M>, Fallible<M>` sequences each `K<M, A>` in the foldable, catches failures from `Fallible<M>`, and accumulates `(Seq<Error> Fails, Seq<A> Succs)` inside `K<M, ...>`. Errors are accumulated monoidally; the outer monad is never failed by individual inner failures.
- For `Option`-typed probe results, `OptionExtensions.Somes<A>(Seq<Option<A>> self)` returns the unwrapped present values, dropping absent results without diagnostic content. When the `None` case warrants a log or metric, use `Choose` with a labeled `None` branch instead of `Somes`.

[SAFEHANDLE_SETHANDLEASINVALID_SENTINEL]:
- `SafeHandle.SetHandleAsInvalid()` sets the `IsClosed` bit via `Interlocked.Or` and suppresses the finalizer; it does not call `ReleaseHandle`. This marks the handle as closed-but-not-released: the OS has already reclaimed the resource. The projection law must test `handle.IsInvalid || handle.IsClosed` together before constructing `Some(handle)`, because `IsClosed` is a BCL field check (always accurate) while `IsInvalid` is abstract and may inspect only the handle value without checking the closed bit.
- After `SetHandleAsInvalid`, `DangerousAddRef` throws `ObjectDisposedException` because bit 0 is set. The projection site that calls `DangerousAddRef` must occur before any `SetHandleAsInvalid` call or after an `IsInvalid || IsClosed` guard; there is no safe retry path after either flag is set.

[CANCELASYNC_CALLBACK_SCHEDULING_DIVERGENCE]:
- `CancellationTokenSource.Cancel()` invokes `ExecuteCallbackHandlers` synchronously on the calling thread. `CancellationTokenSource.CancelAsync()` schedules `ExecuteCallbackHandlers` via `Task.Factory.StartNew(..., TaskScheduler.Default)` when callbacks exist; when no callbacks exist, it returns `Task.CompletedTask`.
- A seam that transitions `Atom<CapabilityState>` via a callback registered with `UnsafeRegister` will have that transition run synchronously on the `Cancel()` caller's thread but asynchronously on a thread-pool thread when triggered by `CancelAsync()`. The `Atom.SwapMaybe` CAS is safe from either thread, but code that reads `atom.Value` immediately after calling `CancelAsync()` before `await`ing the returned task may observe the pre-cancelled capability state. Either `await CancelAsync()` before reading the cell, or use `Cancel()` when synchronous observation is required.
- `CancelAsync()` returns `Task.FromException(new ObjectDisposedException(...))` when the source is already disposed. Awaiting it without a catch propagates the disposal exception. `Cancel()` on a disposed source throws synchronously.

[LOCAL_CANCEL_RESOURCE_SCOPE_ASYMMETRY]:
- `EnvIO.LocalCancel` inherits the outer `Resources` tracker and allocates a new `CancellationTokenSource`. Any resources acquired inside a `LocalCancel` scope are tracked in the shared outer `Resources` and survive the inner timeout. `EnvIO.LocalCancelWithTimeout(TimeSpan)` has the same behavior, with the source auto-cancelling after the timeout.
- `IO.Timeout(TimeSpan)` creates a scoped environment with a timed source. Resources acquired inside the timeout window that are not explicitly released before expiry remain in the outer `Resources` tracking if the outer context is not also `BracketFail`-isolated. The pattern for a bounded-lifetime probe that must not leak acquired resources on timeout is `ma.BracketFail().Timeout(timeout)`, not `ma.Timeout(timeout).BracketFail()`. `BracketFail` wrapping `Timeout` means the bracket sees the timeout as a failure and releases resources; `Timeout` wrapping `BracketFail` means the timeout fires within the already-isolated resource scope.

[OPTION_BIBIND_AS_DUAL_PATH_ABSENCE_ROUTER]:
- `Option<A>.BiBind<B>(Func<A, Option<B>> Some, Func<Option<B>> None)` routes both the present and absent cases to new `Option<B>` values, keeping the pipeline monadic. The `None` branch receives no argument; it is a thunk. `BiBind` is denser than `Match` followed by re-wrapping because it stays inside the monadic context, enabling further `Bind` and `Map` chaining without an intermediate re-entry.
- `BiBind` does not call `Check.NullReturn` on either branch. If the `Some` branch returns `Some(null)` for a reference-type `B`, the result is an `Option<B>` with `isSome = true` and a null interior — a `Some`-state option with a null value that defeats the boundary guarantee. A `null` return from the `None` branch produces `default(Option<B>)`, which is `None`, silently dropping the absent case without diagnostic. Neither branch may return null.

[FOLDABLE_HASHMAP_HEALTH_DERIVATION]:
- `HashMap<Key>` (the static class) implements `Foldable<HashMap<Key>>`. `Fold` is available as `K<HashMap<Key>, V>.Fold<S>(initialState, (v, s) => ...)` via `FoldableExtensions`. Code that uses `.AsIterable().Fold(...)` or LINQ `.Aggregate(...)` instead of the `Foldable` trait bypasses the trait machinery and is not polymorphic — it cannot be lifted into an `IO` fold that checks cancellation per element.
- For a health aggregate over `AtomHashMap<CapabilityKey, CapabilityState>`, the vocabulary-complete fold requires projecting every key in the vocabulary: `smartEnum.Values.Map(key => (key, registry.Find(key).IfNone(CapabilityState.Unknown)))`. `AtomHashMap.Find(key)` returns `Option<V>`. Keys absent from the map represent capabilities not yet probed; they must map to a defined state for the health fold to produce a complete picture.
