# [PY_DATA_API_RHINO3DM]

`rhino3dm` is the OpenNURBS Python binding for reading, writing, and constructing Rhino `.3dm` model data without Rhino installed. `File3dm` is the document owner over typed component tables; `GeometryBase` roots a `CommonObject`/`ModelComponent` hierarchy (`Curve`, `Surface`, `Brep`, `Mesh`, `SubD`, `Extrusion`, `PointCloud`, annotation, instance, light); value-type primitives (`Point3d`, `Vector3d`, `Transform`, `BoundingBox`, `Plane`) and the `Intersection` static kernel round out the geometry surface. `Encode`/`Decode` is the JSON-dict bridge shared with Rhino.Compute and the rhino3dm.js binding, so this binding is the host-neutral 3dm IO and lightweight geometry construction layer beneath the Rhino-MCP and bridge owners.

[IMPLEMENTATION_LAW]:
- `File3dm` is the single document owner; geometry enters and leaves only through typed component tables (`Objects`/`Layers`/`Materials`/`InstanceDefinitions`/...), never as loose globals; `f.Objects.Add(geometry, attributes)` is the polymorphic add over the typed `Add*` shortcuts.
- `f.Write(path, version=0)` selects the 3dm archive version (`0` = latest); use `ReadArchiveVersion` to inspect an unknown file before reading, and `FromByteArray(buffer)`/`Encode()`/`Decode(str)` for in-memory transport.
- Geometry construction uses static `Create*` factories that return `None` on invalid input; check the result rather than assuming success, since this binding signals geometric failure by null, not exception.
- pybind11 dispatches overloads at the C++ layer (`Create(*args, **kwargs)`, `Translation(*args, **kwargs)`); pass the documented argument shape per overload and read `__doc__` for the concrete per-overload signatures.
- `Encode()`/`Decode()` round-trip any `CommonObject` (and `File3dm`) through a JSON-compatible structure; this is the supported serialization bridge to and from Rhino.Compute and the rhino3dm.js/WebAssembly binding — never a hand-rolled 3dm parser.
- Object identity in tables is index- and GUID-based (`FindId(guid)`); `ObjectAttributes` carries layer, material, color/material source, name, mode, and visibility, and is passed alongside geometry on `Add*`.
- User strings (`SetUserString`/`GetUserStrings`) attach arbitrary key/value metadata to geometry and persist through 3dm round-trips; the document `Strings` table holds document-scoped pairs.
- The `Intersection` static kernel and the `Curve`/`Surface` `Is*`/`TryGet*` predicate families (with explicit `tolerance`) own geometric classification and intersection — never re-derive line/plane/sphere intersection or curve-shape detection by hand.
- `DracoCompression` compresses `Mesh` and other geometry to a compact byte/base64 form; use it for mesh transport rather than re-encoding vertex arrays manually.
- `File3dm.Settings -> File3dmSettings` carries the document unit system: `File3dmSettings.ModelUnitSystem`/`PageUnitSystem` are `UnitSystem` enum members (default `Millimeters`; `UnitSystem` enumerates `Meters`/`Millimeters`/`Centimeters`/`Feet`/`Inches`/...), and `File3dmSettings.ModelAbsoluteTolerance` the model tolerance — read `f.Settings.ModelUnitSystem.name` for the declared unit rather than assuming a hardcoded unit.
- `Mesh` element shapes are plain tuples/structs, not wrapper objects: `Mesh.Faces[i]` is a 4-int tuple (a triangle repeats the third index, so `face[3] != face[2]` is the quad probe), a `MeshVertexColorList` element is a plain `(R, G, B, A)` int tuple (NOT a `Color` object with `.R`/`.G`/`.B`/`.A`), and a `MeshNormalList` element is a `Vector3f` with `.X`/`.Y`/`.Z` float accessors.

## [03]-[LOCAL_ADMISSION]

[STACK_LAW]:
- `rhino3dm` <-> Rhino.Compute / rhino3dm.js: `Encode()`/`Decode()` JSON dicts are the host-neutral wire shape; the same `File3dm`/`GeometryBase` objects cross to the compute service and the browser binding without re-serialization.
- `rhino3dm` <-> data mesh codec: a `rhino3dm.Mesh` exposes `Vertices`/`Faces`/`Normals`/`TextureCoordinates`/`VertexColors` lists that map to the in-memory triangulation the `MeshPayload`/`trimesh`/`meshio` owners exchange; geometry crosses as vertex/face arrays, and Draco bytes are the compressed transport form.
- `rhino3dm` -> Rhino-MCP / bridge: this binding is the headless `.3dm` IO and lightweight construction layer; the live Rhino host (RhinoCommon over MCP) and the certificate-backed bridge own runtime evaluation, so rhino3dm never substitutes for the host where a full Rhino kernel is required (e.g. boolean solids, advanced filleting) — it constructs primitives, Breps-from-primitives, NURBS, meshes, and SubD control nets, and reads/writes the document.

[RAIL_LAW]:
- Package: `rhino3dm`
- Owns: OpenNURBS `.3dm` read/write and byte/JSON round-trip, document component tables, NURBS curve/surface/Brep/Mesh/SubD/Extrusion construction and evaluation, the `Intersection` static kernel, geometry value types and transforms, annotation/dimension/leader/hatch geometry, instance definitions/references, object attributes and user strings, render content, meshing parameters, Draco compression
- Accept: `File3dm` as the document owner with typed table mutation, the polymorphic `Objects.Add(geometry, attrs)`, static `Create*` factories with null-result checks, `Encode`/`Decode` and `FromByteArray` for the serialization/transport bridge, `ObjectAttributes` alongside geometry on add, user strings for round-trip metadata, the `Intersection` kernel and `Curve`/`Surface` `Is*`/`TryGet*` predicates, Draco for mesh transport
- Reject: loose geometry outside the document tables, assuming non-null factory results, hand-rolled 3dm archive parsing where `Read`/`Encode`/`Decode` apply, manual vertex re-encoding where Draco applies, hand-derived intersections/shape-detection where the `Intersection` kernel and `Is*`/`TryGet*` predicates resolve, substituting this binding for a live Rhino kernel on operations OpenNURBS-without-Rhino cannot perform
