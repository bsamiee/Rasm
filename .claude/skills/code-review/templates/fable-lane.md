# [FABLE_LANE]

Fixer-lane dispatch form for Claude lanes: one Agent-tool call per lane, the whole contract on one prompt — the Agent surface carries no developer channel, so the filled lane law rides verbatim first and the task instance follows it in the same message. Lane law transfers by composition, never restatement; this template owns only what codex transport mechanics do not.

## [01]-[DISPATCH]

- One `Agent` call per lane, concurrent lanes launched in a single message. No channel split exists to manage and no spawn gate to arm.
- Skill loads ride the lane: the task block names the territory's required skills, composed whole — no register projection fills for a Claude lane.
- A fable lane self-mines: both lane-law mining charges run as the lane's own read pass before its first edit, because a lane under a running workflow launches no nested subagents; the rosters stay candidate data under the lane law's capability mandate.
- Report custody: the lane writes `<round-dir>/lane-<letter>-report.json` itself as its final act and returns only the path plus one status line — report bodies never ride the return.
- Health is the Agent tool's own return: no stderr banner, no liveness probe, no rollout audit. A lane returning prose instead of the path, or returning with no report file on disk, is the failure — relaunch once fresh; a second failure routes to a closer.

## [02]-[TASK_MESSAGE]

```markdown template
<task>
Disposition every finding in <round-dir>/lane-<letter>-brief.md (ids and structured rows: <round-dir>/lane-<letter>.json). Your write territory is exactly these files:
- <sliced-file-1>
- <sliced-file-2>

Load <required-skills: docgen for any prose territory; skill-writer added when the territory is a skill bundle>, then run both lane-law mining charges yourself over <doctrine-or-catalog-paths> and the territory files before your first edit; judge the rosters under your capability mandate and land only what qualifies.
Write <round-dir>/lane-<letter>-report.json as your final act and return only its path plus one status line.
</task>
```
