# Spectral-Eigen Frontier

[CHOLESKY_FACTOR_STORAGE_TRUTH]:
- `ManagedLinearAlgebraProvider.CholeskyFactor(double[])` writes `a[i*order+j]` for `j > i` at each column `i`. In column-major layout (`DenseColumnMajorMatrixStorage`), index `i*order+j` maps to row `j`, column `i` — the lower triangle. The zeroing pass `a[k*order+i] = 0.0` for `k > i` clears the upper triangle. `Cholesky<T>.Factor` is lower-triangular L, confirmed by `DoCholeskySolve`'s forward-substitution pass reading `a[num3 * orderA + i]` (column `num3`, row `i`) in column order.
- `Cholesky.Solve(B)` dispatches to `DoCholeskySolve`, which performs `L·y = B` (forward) then `Lᵀ·x = y` (backward) in one pass — it returns `(L·Lᵀ)⁻¹·B`. Any congruence step that requires only `L⁻¹·A` must use `chol.Factor.LU()`, not `Cholesky.Solve`. The correct two-step sequence: (1) `Y = chol.Factor.LU().Solve(A)` — solves `L·Y = A`, `Y = L⁻¹·A`; (2) `Z = chol.Factor.LU().Solve(Y.Transpose()).Transpose()` — `Z = L⁻¹·A·L⁻ᵀ`. Each `.LU()` call on the lower-triangular factor allocates an n×n PLU struct; `.Transpose()` allocates an n×n copy. Three n×n allocations per congruence. Back-transform `X = L⁻ᵀ·Y`: `chol.Factor.Transpose().LU().Solve(evd.EigenVectors)` is correct — upper-triangular backward substitution.

[NONSYMMETRIC_CONVERGENCE_ASYMMETRY]:
- The real double path (`NonsymmetricReduceHessenberToRealSchur(double[], ...)`) has a bounded outer iteration count: `if (num10 >= 30 * order) throw new NonConvergenceException()`. The counter `num10` increments per outer step and resets only on a 2×2 block deflation. The ceiling is `30n` total steps across all sub-problems; a pathological n=100 matrix can exhaust 3000 steps and throw.
- The complex path (`NonsymmetricReduceHessenberToRealSchur(Complex[], ...)`) has no outer iteration bound. The `while (num >= 0)` loop terminates only on deflation. Exceptional shift strategies fire at internal counters 10 and 20 but impose no ceiling check. A complex matrix whose Schur form fails to deflate through the shift heuristics runs indefinitely in the managed path with no exception.
- The three convergence behaviors: symmetric and Hermitian paths throw `NonConvergenceException` at 1000 steps per eigenvalue; the real nonsymmetric path throws `NonConvergenceException` at `30n` total outer steps; the complex nonsymmetric path may not terminate. A receipt that catches `NonConvergenceException` and emits `ExhaustionFault` handles symmetric and real nonsymmetric paths. The complex nonsymmetric path requires a post-hoc Frobenius residual guard as the only detection mechanism — the receipt must include `‖A·V − V·D‖_F` unconditionally for complex nonsymmetric inputs.

[RANK_THRESHOLD_MAGNITUDESQUARED_HAZARD]:
- `ComplexExtensions.Norm(Complex c)` returns `c.MagnitudeSquared()` — squared modulus, not magnitude. `Evd.Rank` calls `EigenValues[i].AlmostEqual(Complex.Zero)`, which expands to `a.Norm().AlmostEqualNorm(0.0, a.NormOfDifference(Complex.Zero), DefaultDoubleAccuracy)` where both `Norm` and `NormOfDifference` use `MagnitudeSquared`. The comparison reduces to `|λ|² < DefaultDoubleAccuracy ≈ 1.11 × 10⁻¹⁵`, equivalent to `|λ| < sqrt(1.11 × 10⁻¹⁵) ≈ 3.33 × 10⁻⁸`. An eigenvalue at magnitude `3.4 × 10⁻⁸` clears the threshold; one at `3.2 × 10⁻⁸` is classified zero. For problems with spectral radius ∼ 10⁶, eigenvalues at `10⁻⁸` that are numerically significant are misclassified.
- `Determinant` uses the same `AlmostEqual` call: if any eigenvalue has `|λ| < 3.33 × 10⁻⁸`, the determinant short-circuits to exactly `0.0` before accumulating the remaining product. Never expose `evd.Rank` or `evd.IsFullRank` to domain logic; recompute rank as `n − count(i : |λᵢ| < ε_rank × max|λⱼ|)` with a caller-supplied spectral-radius-relative `ε_rank`.

[NONSYMMETRIC_EVD_SOLVE_THROWS]:
- `DenseEvd.Solve(Matrix<double>, Matrix<double>)` throws `ArgumentException("Matrix must be symmetric.")` when `!IsSymmetric`. The spectral-decomposition solve path `V·Λ⁻¹·Vᵀ·b` exists only for the symmetric case. For nonsymmetric Evd, `evd.Solve(b)` is not a warning — it is a hard throw. A receipt architecture must not expose `evd.Solve` for nonsymmetric inputs.

[QR_THIN_ALLOCATION_AND_Q_SHAPE]:
- `DenseQR.Create(matrix, QRMethod.Thin)` routes through `LinearAlgebraControl.Provider.ThinQRFactor`, which stores Q in-place in the n×k input array and writes R to a separate k×k array. The resulting `qr.Q` is exactly n×k. `GramSchmidt<T>` hardcodes `QRMethod.Full` in `base(q, rFull, QRMethod.Full)` — calling `.GramSchmidt()` on any matrix produces an n×n Q regardless of input shape. For LOBPCG with k ≪ n, `QR(QRMethod.Thin).Q` avoids the n×n allocation that `GramSchmidt()` always incurs.
- `QR<T>.R` is lazy: `_lazyR = new Lazy<Matrix<T>>(FullR.UpperTriangle)`. For `QRMethod.Thin`, `FullR` is already k×k; `.UpperTriangle()` allocates a k×k copy on first access. Since LOBPCG orthonormalization uses only Q, R is never materialized unless the near-dependence check accesses `qr.R.Diagonal()`.

[SPARSE_DENSE_MULTIPLY_NOT_DENSIFYING]:
- `SparseMatrix.DoMultiply(Matrix<double> other, Matrix<double> result)` does not densify when `other.Storage` is `DenseColumnMajorMatrixStorage<double>`. The verified code path reads sparse `RowPointers` and `ColumnIndices` directly against the dense backing array `data`, computing each result entry as a sparse inner product: `result.At(num, num4, sum)` where the sum runs over `values[num7] * data[num5 + columnIndices[num7]]`. Cost is O(nnz × k) for a sparse n×n matrix times a dense n×k block. The densification fallback (via `DenseVector` column extraction) activates only for user-defined storage types not matching the recognized patterns.
- For LOBPCG operator application: `A.Multiply(W)` where A is `SparseMatrix<double>` and W is a `DenseMatrix` n×3k block is O(nnz × 3k) with no intermediate densification. The result is a `DenseMatrix` (constructed via `Matrix<double>.Build.SameAs(A, W, ...)`). The subsequent Rayleigh-Ritz `W.TransposeThisAndMultiply(A_W)` is then a dense-by-dense multiply.

[MAPSINDEXEDINPLACE_FOR_RESYMMETRIZATION]:
- `DenseColumnMajorMatrixStorage.MapIndexedInplace(f, zeros)` iterates all n×n elements via `CommonParallel.For(0, ColumnCount, ...)` with direct array access through the captured `Data` field. For re-symmetrization `S[i,j] = (S[i,j] + S[j,i]) / 2.0`, `s.MapIndexedInplace((i, j, v) => (v + s[j, i]) / 2.0, Zeros.Include)` is in-place with no allocations — `s[j, i]` accesses `Values[i * n + j]` directly. This is the minimum-allocation form; `(s + s.Transpose()) * 0.5` allocates three matrices.
- `Zeros.Include` is required for correctness: the default `Zeros.AllowSkip` may leave zero-valued off-diagonal entries unaveraged when the opposite triangle entry is nonzero, producing an asymmetric result that defeats the re-symmetrization.

[APPEND_ZERO_COPY_VARIANT]:
- `Matrix<T>.Append(Matrix<T> right, Matrix<T> result)` (the in-place overload) calls `CopySubMatrixToUnchecked` for both blocks into the pre-allocated `result.Storage`. No intermediate allocation occurs when `result` is a pre-allocated `DenseMatrix` of the correct shape. For LOBPCG subspace construction `[X | A_X | P]`, pre-allocating a single n×3k workspace avoids the chain of intermediate n×2k and n×3k allocations from chained `X.Append(A_X).Append(P)`.

[FILTER_KERNEL_VECTORIZED_HEAT_RECIPE]:
- `TensorPrimitives.Exp<double>(ReadOnlySpan<double>, Span<double>)` compiles because `double : IExponentialFunctions<double>`. Heat filter weight vector `w[i] = exp(-t × λᵢ)`: (1) allocate `double[] weights = new double[n]`, (2) `weights[i] = eigenValues[i].Real` (scalar projection loop, LLVM-autovectorized on .NET 10), (3) `TensorPrimitives.Multiply<double>(weights, -t, weights)` in-place, (4) `TensorPrimitives.Exp<double>(weights, weights)` in-place. Steps 3 and 4 dispatch through AVX/SSE on supported hardware. Step 2 is the only non-SIMD step due to the `Complex → double` projection.
- Zero-mode exclusion combined with the heat filter requires a separate scalar mask pass after the vectorized exp: `weights.MapIndexedInplace((i, _, w) => eigenValues[i].Real < ε_zero ? 0.0 : w, Zeros.Include)`. Vectorized exp plus one scalar mask pass is faster than n scalar `Math.Exp` calls for large n.

[FIN_V5_CONSTRUCTION_SURFACE]:
- In LanguageExt 5, `Fin<A>` is an abstract class with two sealed nested cases: `Fin<A>.Fail(Error Error)` and `Fin<A>.Succ(A Value)`. `FinSucc<A>` and `FinFail<A>` are `[Obsolete]`. The active factory surface is `Fin.Succ(value)` and `Fin.Fail<A>(error)`.
- `Try<A>` is `record Try<A>(Func<Fin<A>> runTry) : K<Try, A>`. The `Run()` extension wraps `ma.As().runTry()` in a try/catch that converts any `Exception` to `Fin.Fail<A>(ex)` via `Fin.Fail<A>(Exception)`. The eigensolver boundary form is `new Try<EigenReceipt>(() => Fin.Succ(BuildReceipt(matrix.Evd(sym)))).Run()`, which returns `Fin<EigenReceipt>`.
- `Error.New(Exception)` in v5 sets `IsExceptional = true` and wraps the exception as `Error.Inner`. To distinguish `NonConvergenceException` from `ArgumentException` without inspecting exception type at use sites, wrap the Evd call in a pre-catch that converts each specific exception to a tagged error with a domain-specific code via `Error.New(int code, string message, Error inner)` before re-returning `Fin.Fail`. Domain code values `ExhaustionCode`, `IndefiniteBCode`, and `ShapeCode` survive the `Fin.Fail` carrier as first-class discriminants.

[ISSYMMETRIC_EXACT_EQUALITY_HAZARD]:
- `DenseMatrix.IsSymmetric()` uses exact bit equality: `_values[j * n + i] != _values[i * n + j]`. After any floating-point matrix operation the check returns `false` for entries differing by even one ULP, routing accumulated matrices through the nonsymmetric Hessenberg path unconditionally. After `MapIndexedInplace((i, j, v) => (v + s[j, i]) / 2.0)`, the [i,j] and [j,i] entries both become `(a + b) / 2` — bitwise identical under IEEE 754 since addition is commutative in floating-point. Passing `Symmetricity.Symmetric` after this re-symmetrization is both safe and correct.

[EXHAUSTION_PARTIAL_EIGENVECTOR_STATE]:
- When `SymmetricDiagonalize` throws `NonConvergenceException` at iteration `num3 >= 1000` for eigenvalue index `j`, the `matrixEv` array has been partially updated through all QR sweeps up to `j`. Eigenvectors for converged indices `< j` are complete and orthonormal. Eigenvectors for indices `>= j` are intermediate Givens-rotation products — numerically meaningless. The exception does not identify `j`. The diagnostic receipt for exhaustion cannot determine the failing index through the public API; the pre-call spectral radius estimate in the receipt payload is the only available context.

[REORTHOGONALIZATION_PROTOCOL_EXACT_COST]:
- Single-pass MGS for an n×k block costs O(2nk²) flops; a second pass costs another O(2nk²). For LOBPCG with k = 8 and n = 10000, single pass is O(1.28 × 10⁶) flops — identical to Householder QR at O(2nk² − 2k³/3). The advantage of conditional double-pass MGS over single-pass Householder is the second pass is only invoked when the R diagonal has entries below the floor. Since `DenseGramSchmidt.Create` unconditionally clones the input, each pass allocates a new n×k copy regardless.
- Accessing `DenseGramSchmidt.R.Diagonal()` (for the near-dependence check) is lazy and O(k) for a k×k dense R. The check `gs.R.Diagonal().Any(v => Math.Abs(v) < floor)` is correct and allocation-free until the condition fires.

[SPECTRAL_FILTER_CLOSED_UNION_DISPATCH_SHAPE]:
- A spectral filter is fully specified by a per-eigenvalue `Invoke(double lambda)` method plus zero-mode policy and bandwidth floor evidence. Heat, wave, biharmonic, diffusion, commute-time, and identity collapse into a closed union with a virtual `Invoke` on each case. The composed filter `f₁ ∘ f₂` is a `Product(FilterCase, FilterCase)` union case whose `Invoke(λ) = f₁.Invoke(λ) × f₂.Invoke(λ)` — two virtual dispatches per eigenvalue. For n ∼ 500 eigenvalues the dispatch cost is negligible; inlining through a pattern match would violate the closed-union invariant by requiring the dispatch site to enumerate all cases.
- The `Product` case is not a recursive union of unions. `Identity`'s `Invoke(λ) = 1.0` is the monoid unit; `Product(f, Identity)` folds to `f` by the monoid law. The delegate-bundle form (storing a raw `Func<double, double>`) is open and belongs only at the receipt boundary where the weight vector is materialized, not in the filter algebra.
