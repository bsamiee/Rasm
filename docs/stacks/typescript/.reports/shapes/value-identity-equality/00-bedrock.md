# Value Identity And Equality Bedrock

[STRUCTURAL_FUSION]:
- `Data.struct(record)`, `Data.tuple(...elems)`, `Data.array(elems)` reshape native objects/arrays into values carrying `[Equal.symbol]` and `[Hash.symbol]` by construction; the declaration-chain `const x = {...}` then a hand-written comparator collapses to one call that fuses value, structural equality, and hash.
- `class X extends Data.Class<{...}>` and `class X extends Data.TaggedClass("X")<{...}>` are simultaneously a type, a value-constructor (`new X({...})`), and an `Equal`/`Hash` instance; no `type X = ...` and no separate equality function ever follow the class.
- `Data.TaggedClass(tag)` strips `_tag` from the constructor argument (`P extends "_tag" ? never : P`) and stamps it on the instance — the discriminant is supplied once at declaration, never passed per-construction, and is recoverable from the value for downstream `Match`.
- `Data.taggedEnum<Union>()` returns the per-tag constructor map fused with `$is(tag)` (a refinement guard `(u) => u is Extract<...>`) and `$match` (curried and data-first overloads), so the union owner emits its own narrowing and exhaustive dispatch with zero sibling guards.
- `Data.case<A>()` and `Data.tagged<A>(tag)` build a `Case.Constructor` whose argument is `VoidIfEmpty<A>` — an empty-field case constructs with no argument, not `{}`, eliminating the empty-object literal at every call site.
- `Data.struct` is `Object.assign(Object.create(StructuralPrototype), fields)` and `Data.unsafeStruct` is `Object.setPrototypeOf(existing, StructuralPrototype)` — the latter restamps an already-built object in place rather than copying, the one entry that grants structural identity to a foreign-constructed record without reallocating it.

[STRUCTURAL_COMPARISON_ALGEBRA]:
- Struct/class equality (`StructuralPrototype[Equal.symbol]`) keys off `Object.keys(this)` versus `Object.keys(that)`: a length mismatch fails fast, then every self-key must satisfy `key in that && Equal.equals(this[key], that[key])` — comparison is over OWN ENUMERABLE keys only, so a key absent from one side breaks equality even when the present values would match.
- The same prototype recursion re-enters the full `Equal.equals` gate per field, so nesting `Data` values inside `Data` values yields deep structural equality with no per-level comparator; a plain object nested inside a `Data.struct` degrades that branch to reference equality.
- The `selfKeys.length !== thatKeys.length` guard makes a superset `that` unequal — structural equality is bijective on the key set, not a self-subset check; a `that` carrying extra keys is admitted only when key-counts still match by coincidence.
- A non-enumerable or prototype-inherited field is invisible to this comparison and to `Hash.structure` alike (both walk `Object.keys`), so two values differing only on a non-enumerable property compare equal and hash equal — the symmetry is intact, but identity silently ignores anything not own-enumerable.
- `ArrayProto[Equal.symbol]` guards `Array.isArray(that) && this.length === that.length` then `this.every((v, i) => Equal.equals(v, that[i]))` — positional and length-sensitive, a strictly different algorithm from the key-set struct comparison; a tuple is never equal to a struct even when fields align, because the two prototypes never share a comparison body.

[DISPATCH_GATE]:
- `Equal.equals(self, that)` runs a fixed gate sequence before any owner code: reference identity (`self === that`) wins first; a `typeof` mismatch fails immediately; only the `object`/`function` arm with both operands non-null and both `Equal.isEqual` reaches owner logic — a `Data` value compared against any non-`Equal` operand never enters the structural branch and falls through to `false`.
- The owner-logic arm is `Hash.hash(self) === Hash.hash(that) && self[Equal.symbol](that)` — the hash is a precondition consulted BEFORE `[Equal.symbol]`, so a hash that disagrees with equality short-circuits to inequality without ever invoking the comparator; the equal-values-hash-equal law is not advisory but the literal gate that runs first.
- Hash equality is necessary, never sufficient: collisions are expected so a hashed bucket still calls `[Equal.symbol]`, but a fast wrong `Equal` is a correctness bug a passing hash will not surface.
- `Date` and `URL` are recognized inline after the `Equal`-pair branch fails: two `Date`s compare by `getTime()` with `t1 === t2 || (Number.isNaN(t1) && Number.isNaN(t2))` (NaN admitted equal), two `URL`s by `href` — these native carriers get value identity without a `Data` wrapper, and a `Date` field nested in a `Data.struct` recurses into this arm rather than degrading to reference.
- A mixed pair where exactly one operand is `Equal` skips the owner arm entirely; identity fusion is required on BOTH operands, never inferred from one — feeding a constructed value against a plain literal of identical shape is the membership miss no field-level test surfaces.

[PROTOTYPE_TIER]:
- Three prototypes assign the identity symbols and the consumer reads whichever the value's prototype chain carries: `StructuralPrototype` (`[Hash.symbol] = Hash.structure`, `[Equal.symbol]` = `Object.keys` comparison) backs `Data.struct`/`Data.case`/`Data.Class`/`Schema.Class`; `ArrayProto` (`Hash.array`, index-wise `Equal`) backs `Data.tuple`/`Data.array`; `EffectPrototype` (`Hash.random`, `[Equal.symbol] = this === that`) backs every `Effect`/service/`Stream` carrier.
- `EffectPrototype`'s `[Equal.symbol]` is reference identity by design — an Effect-bearing value placed in a `HashSet` deduplicates by reference, never by content, so a service handle or a `Layer` is never content-collapsed; the structural-versus-reference choice is fixed at the prototype, not selectable at the comparison site.
- `Schema.Class` resolves to `data_.Class`, which is `internal.Structural` whose `.prototype` is exactly `StructuralPrototype`; the schema class therefore inherits the same `Object.keys` equality and `Hash.structure` hashing as a bare `Data.struct` — the codec wrapper adds no identity layer, the structural tier is the single source for both.

[HASH_OWNER]:
- `Hash.hash(self)` dispatches to `self[Hash.symbol]()` when present and falls back to structural/primitive hashing otherwise; `Hash.structure(o)` is `structureKeys(o, Object.keys(o))` (all own enumerable keys) while `Hash.structureKeys(o, keys)` hashes only a named `ReadonlyArray<keyof A>` subset — identity that must ignore derived or volatile fields uses the keyed form.
- `Hash.string` is djb2 reversed (seed `5381`, `h = h * 33 ^ charCodeAt(--i)`), `Hash.combine(b)(self)` is `self * 53 ^ b` (NON-commutative — order of combination is load-bearing), `Hash.array` folds `combine` over elements from a `6151` seed (index order changes the hash), and `Hash.optimize` (`n & 0xbfffffff | n >>> 1 & 0x40000000`) clamps every result into the SMI range so hashes stay V8-fast integers.
- `Hash.structureKeys(o, keys)` seeds `12289` and XOR-accumulates `string(key) combined-with hash(o[key])` per key — XOR makes the key-value contributions ORDER-INDEPENDENT, so two records with the same key set in different insertion orders hash identically, matching the order-independence of the `Object.keys` equality and keeping the gate coherent.
- `Hash.number` returns `0` for `NaN` and `Infinity`, so two distinct non-finite numeric fields collide in the hash and fall to the comparator; `Hash.hash` routes `bigint` through `string(self.toString(10))` and `Date` through its ISO string, so identity-bearing numeric edge values hash deterministically rather than via reference.
- The `Equal`/`Hash` contract is co-constraining and the failure is supplying a `[Hash.symbol]` that disagrees with the `[Equal.symbol]`: equal values must hash equal or the value corrupts `HashMap`/`HashSet` buckets, and the disagreement is the only break the type system cannot catch.

[HASH_MEMOIZATION]:
- `Hash.cached(self, h)` installs `[Hash.symbol]` as a NON-ENUMERABLE own property via `Object.defineProperty`, so the memo survives every `Object.keys` walk used by structural equality and `Hash.structure` — caching the hash does not pollute the key set the equality compares, the first `Hash.hash` pays, and every repeat reads the frozen value.
- Mutating a `Data` value's fields after its first `Hash.hash` desyncs the cache: the memoized non-enumerable `[Hash.symbol]` keeps returning the pre-mutation hash while the structural equality reads the new fields, so the gate `Hash.hash === Hash.hash` can fail on values the comparator would call equal — immutability is the precondition the memo assumes, and `unsafeStruct` over a later-mutated object is the trap.
- `Hash.random(self)` memoizes one `Math.floor(Math.random() * Number.MAX_SAFE_INTEGER)` integer per object reference in a global `WeakMap`, so a non-`Hash` object and every `EffectPrototype` carrier gets a STABLE per-reference hash for its lifetime — reference identity with a fast hash bucket, never content identity, and the `WeakMap` keying means the hash dies with the reference rather than leaking.
- A custom `[Hash.symbol]` must call `Hash.cached(this, computed)` to match the prototype's amortization; returning an uncached value recomputes per `Hash.hash` call and, because `HashMap` hashes the key on every `get`/`set`, turns each lookup into a full re-hash of the key's structure.
- `Hash.structureKeys`'s XOR accumulation and the `Object.keys` equality share order-independence over the key set while `Hash.array`'s `combine` fold and the index-wise tuple equality share order-sensitivity over positions — each prototype pairs a hash whose algebraic symmetry exactly matches its comparator's, so the equal-implies-hash-equal gate holds for free per tier and a hand-rolled `[Hash.symbol]` that breaks the symmetry (XOR for a positional value, or fold for a keyed one) re-introduces the scatter the prototypes avoid by construction.

[STRUCTURAL_REGION]:
- `structuralRegion(body, tester)` flips a global `structuralRegionState.enabled` for the dynamic extent of `body`, restored in a `finally` — inside it `Hash.hash` returns `0` for every input (collapsing all bucket routing) and `Equal.equals` bypasses the owner `[Equal.symbol]` to recurse structurally over plain arrays and `Object.prototype`-rooted plain objects, with the optional `tester` consulted at every leaf mismatch.
- The region is the seam custom test matchers use to compare raw decoded shapes and `Data` values uniformly without each operand carrying `Equal`; interior domain code never enters it, so the all-zero hash and plain-object recursion are a test-only equality regime, never a production identity path.
- Within the region the plain-object arm requires `Object.getPrototypeOf` to be exactly `Object.prototype` on both sides before key-walking — a class instance or `Data` value still routes through its own `[Equal.symbol]` first, so the region widens equality for naked records without overriding owner-defined identity.

[EQUIVALENCE_OWNER]:
- `Equivalence<A>` is `(self: A, that: A) => boolean`; `Equivalence.make(f)` is the only constructor, `Equivalence.strict<A>()` wraps `===`, and `string`/`number`/`boolean`/`bigint`/`symbol`/`Date` are ready leaf instances — a hand-rolled `(a, b) => a.x === b.x` is the spelling these delete.
- `Equivalence.struct({k: Equivalence<...>})` derives a record equivalence keyed by field; `Equivalence.tuple(...elems)` derives a positional one; `Equivalence.array(item)` lifts one element equivalence to `ReadonlyArray` — the field-level instance is the single source the aggregate reads.
- `Equivalence.mapInput(f)` contravariantly retargets an equivalence onto a projection (`mapInput(Equivalence.number, (p) => p.priority)`), collapsing "compare by extracted field" to one combinator instead of a custom closure.
- `Equivalence.product` pairs two instances into `Equivalence<readonly [A, B]>`, `all` lifts a homogeneous collection to `Equivalence<ReadonlyArray<A>>`, and `productMany` prepends one head instance to a tail collection — the heterogeneous-tuple, list, and head-plus-rest shapes each have a named combinator rather than a manual zip.
- `Equivalence` is a `Semigroup` but is NOT exposed as a `Monoid`: it has `combine`/`combineMany`/`combineAll` and the always-true equivalence is its mathematical identity, yet no `empty` and no `reverse` are exported — the dual of an equivalence is itself (symmetry), and the identity element is rarely the intended default, so the surface withholds them while `Order` exposes both.
- `Equivalence.combine(a, b)` is the meet (logical AND with short-circuit): the later equivalence is consulted only where the earlier already returned `true`, so layered `mapInput`-projected equivalences form an AND-chain whose evaluation order is a cost decision (cheapest discriminating field first) and whose result is order-independent — associativity holds even though evaluation short-circuits, and precedence is declaration order.
- `Equivalence.combineAll([])` over an empty collection is the everything-equal instance (the unexposed identity surfacing through the variadic fold), so a dynamically assembled multi-field equality with zero selected fields collapses every value to one equivalence class — the degenerate-key-list trap the analogous `Order.empty` makes explicit but the equivalence fold reaches silently.

[ORDER_OWNER]:
- `Order<A>` returns `-1 | 0 | 1`; `Order.make(compare)` constructs, `string`/`number`/`boolean`/`bigint`/`Date` are leaves, and the derived comparators `lessThan`/`greaterThan`/`lessThanOrEqualTo`/`greaterThanOrEqualTo`/`min`/`max`/`clamp`/`between` are read off one `Order` — none re-implements the comparison.
- `Order.mapInput(O, f)` retargets onto a projected field and `Order.reverse(O)` is `make((self, that) => O(that, self))` — argument-swap, not codomain sign-flip; descending-by-field is `reverse(mapInput(Order.number, f))`, never a custom `(a, b) => b.x - a.x`, and reversing one arm of a `combine` chain flips that key's direction alone.
- `Order.combine`/`combineMany`/`combineAll` fold the `Ordering` codomain by the left-most-non-zero rule (`out !== 0 ? out : next`) inlined per combinator — the first key with a non-zero result wins outright and every later key is consulted only across the tie it leaves; `combineMany` short-circuits the moment a key returns non-zero, so combine order is the precedence declaration and the fold is associative under grouping yet sequence-sensitive in precedence.
- `Order.struct({...})` and `Order.tuple(...)`/`Order.array(O)` derive the same lexicographic order from a field/positional declaration, with `array` falling back to length when all compared elements tie.
- `Order.empty<A>()` is `make(() => 0)` — the constant-tie neutral element seeding `combineAll` over a variadic key list, so a dynamically assembled key sequence degrades to a stable no-op total tie rather than throwing.
- An `Order` is strictly finer than an `Equivalence`: two values comparing `0` are order-equivalent but may differ on fields the `Order` ignores; deriving equality from a partial `Order` (`O(a,b) === 0`) silently merges distinct values and hard-codes the coarser merge into both halves.

[EXTREMUM_MONOID]:
- `Bounded<A>` is the `Order` extended with bounds: `{ compare: Order<A>, maxBound: A, minBound: A }` — the bound pair is exactly the identity element a value-selecting fold needs, declared once on the owner alongside its comparator.
- `Semigroup.min(O)`/`Semigroup.max(O)` derive a value-combining `Semigroup<A>` straight from an `Order<A>` — `combine` selects the smaller/larger of two values, so a folded reduction picks the extremum with no `Math.min` reach and no explicit accumulator seed; the unbounded form has no `empty` because an empty fold has no extremum to return.
- `Monoid.min(B)`/`Monoid.max(B)` lift a `Bounded<A>` into a value-selecting `Monoid<A>` whose `empty` is the OPPOSITE bound — `min`'s identity is `maxBound`, `max`'s identity is `minBound` — so `combineAll([])` returns the bound that loses to every real value, making extremum-of-empty total instead of partial.
- The extremum monoid composes structurally: `Monoid.struct({ ...rows })` and `Monoid.tuple(...rows)` build a per-field extremum whose `empty` is the field-wise struct/tuple of each row's `empty`, so a record of `Monoid.max(Bounded)` columns folds a collection into a per-field componentwise maximum in one `combineAll`.
- `Order.min(O)`/`Order.max(O)` and `Semigroup.min(O)`/`Semigroup.max(O)` disagree on tie retention: the `Order` helpers keep the FIRST argument when two values compare `0` (`O(self, that) < 1 ? self : that`), while the `Semigroup` constructors keep the LAST minimum/maximum (`O(self, that) === -1 ? self : that`) — folding the same `Order` through the two surfaces over equal-comparing distinct values returns different representatives.
- The representative choice is observable whenever the `Order` is coarser than the value's structural `Equal` — two values tying under an `Order.mapInput` projection differ on the ignored fields, so first-vs-last retention selects which incidental payload survives the reduction; an extremum fold over a projected order is non-deterministic in payload unless the retention surface is chosen deliberately.

[DUAL_COHERENCE]:
- `reverse` is one dual functor coherent across every layer: `Order.reverse` swaps `compare`'s arguments, `Bounded.reverse` BOTH reverses the `compare` AND swaps `maxBound ↔ minBound`, and `Monoid.reverse`/`Semigroup.reverse` swap `combine`'s argument order — so a single reversed `Bounded` turns `Monoid.max` into a correct `Monoid.min` with its identity element flipped in lockstep, no field edited by hand.
- Reversing only the `Order` inside a `Bounded` without swapping its bounds breaks the monoid laws: `Monoid.min` over a hand-built `{ compare: reverse(O), maxBound, minBound }` returns the original maximum but seeds `combineAll([])` with the wrong identity, so the empty fold disagrees with every non-empty fold — `Bounded.reverse` is the only law-preserving inversion.

[OBLIGATION_LATTICE]:
- The `Equal` interface `extends Hash.Hash`, so `[Equal.symbol]` cannot be declared without `[Hash.symbol]` — the equal-values-hash-equal co-constraint is a structural type obligation, not a runtime convention; a value carrying a custom `Equal` is statically forced to also carry a `Hash`, and the only freedom left to break is supplying a `Hash` that disagrees with the `Equal`.
- The equality half bridges to the combinator surface but the ordering half does not: `Equal.equivalence<A>()` lifts `[Equal.symbol]` into the `Equivalence` every default-identity verb reaches for, yet an `Order` does not bridge to `Equal` and `Equal` does not derive an `Order` — so the comparison and the equality halves of identity are independently sourced and only the owner keeps them coherent.

[SCHEMA_FUSION]:
- `class X extends Schema.Class<X>("X")({...fields})` extends `Data.Class` internally, so a schema class is at once a codec, a static type, a validating `make`/`new` constructor, and a structural `Equal`/`Hash` instance — the deepest single-source owner; `Schema.TaggedClass`/`Schema.TaggedError` add the `_tag` and (for errors) `YieldableError`.
- `Schema.Data(schema)` lifts a struct/array schema so its decoded output is a `Data` value carrying `[Equal.symbol]`/`[Hash.symbol]`, making boundary-admitted records `HashSet`/`HashMap`-key-correct by construction — the decode and the structural-identity stamping fuse, so post-decode interior code drops every value into a hashed structure without a comparator.
- `Schema.equivalence(schema)` reads the same per-member declaration that the codec reads, so adding a field to the schema extends both the wire codec and the derived equality in one edit — the equality is never restated beside the schema, and the one identity surface this propagation does not reach is the `Order` half.

[SCHEMA_EQUIVALENCE_DERIVATION]:
- `Schema.equivalence(schema)` derives `Equivalence<A>` by walking the AST and honoring an `equivalence` annotation hook per node: the hook short-circuits the structural arm at exactly the node it sits on and wins over the default walk for that subtree alone, so a custom identity is injected at one node while siblings keep deriving structurally; a `Transformation` derives from its `to` (decoded) side, never the encoded one.
- A bare `path: ReadonlyArray<PropertyKey>` threads the descent — `path.concat(ps.name)` at each property, `path.concat(i)` at each tuple position — so a `Never`-node throw names the precise field route to the unsupported node, making the only fatal derivation point self-locating.
- Annotation arity tracks node `_tag` at the hook call site: a `Declaration` hook is a function of N type-parameter equivalences (`EquivalenceAnnotation<A, TypeParameters> = (...equivalences) => Equivalence<A>`), a `Refinement` hook of exactly one base (`from`) equivalence, every other node's hook nullary — supplying the wrong arity is a structural mismatch the hook signature surfaces, and a generic owner's derived equality is itself parameterized by its members' equalities.
- `TypeLiteral` derives a record equivalence over `propertySignatures` (each compared at `path.concat(ps.name)`) and `indexSignatures`; both operands must satisfy `Predicate.isRecord` or the equivalence returns `false`. An optional property (`isOptional: true`) makes presence load-bearing: `aHas !== bHas` returns `false`, both-omit skips the field, both-carry runs the field equivalence — optionality fuses presence-parity into the derived identity. Index signatures partition keys by encoded parameter kind (`AST.getEncodedParameter` then `AST.isSymbolKeyword` splits string-keyed from symbol-keyed), each demanding equal key-count per kind before comparing values.
- `TupleType` derives positional equivalence with a length guard and three zones — fixed `elements` by index, a `rest` head applied across the middle span, and post-rest `tail` elements aligned from the end — so a schema tuple with a rest element compares head-fixed, middle-uniform, and tail-fixed positions under distinct element equivalences in one derived instance.
- `Union` builds `ParseResult.getSearchTree(ast.types, true)` and routes both operands by their discriminant literal into candidate members, then requires both to satisfy the SAME member's refinement (`ParseResult.is({ ast })`) before its equivalence is consulted — two values of different union members are never equal even when their shared fields match, and a non-discriminable member falls into the `otherwise` bucket and is tried against every operand.
- `Suspend` wraps its child in `memoizeThunk(() => go(ast.f(), path))` and returns `(a, b) => get()(a, b)` — a recursive schema (a tree, a nested-list owner) derives a lazily-resolved equivalence whose thunk fires once and caches, so the self-referential node never re-walks per comparison and a cyclic owner produces a terminating, memoized identity; the thunk is keyed by the `Suspend` node instance, not by value depth, so deep structures compare by the same cached element equivalence at every level — the comparator is built once, applied to arbitrary depth.

[SCHEMA_DERIVATION_FALLBACK]:
- Absent a hook, a broad set of leaf nodes collapses to `Equal.equals`: `Declaration`, `Literal`, `StringKeyword`, `TemplateLiteral`, `UniqueSymbol`, `SymbolKeyword`, `UnknownKeyword`, `AnyKeyword`, `NumberKeyword`, `BooleanKeyword`, `BigIntKeyword`, `UndefinedKeyword`, `VoidKeyword`, `Enums`, `ObjectKeyword` — a bare `Declaration` without an `equivalence` annotation does NOT throw, it degrades to reference/structural `Equal.equals`, so an opaque carrier silently compares by identity until the annotation is supplied.
- `NeverKeyword` is the SOLE throwing node across the entire walk — the failure surfaces at `Schema.equivalence` call time, not at compare time, so a degenerate field is caught when the equivalence is built rather than when two values meet; every other node either folds to a default or recurses. Derivation safety is therefore a property of the type's inhabitedness, checked once at build time: every well-formed owner short of a `Never`-decoded field yields a terminating equivalence, so the single failure mode is a degenerate type, not deep nesting or cycles.
- `UnknownKeyword`/`AnyKeyword` folding to `Equal.equals` is the latent trap: an `unknown`-typed field admitted past the boundary deduplicates and tests membership by reference, so a value owner that leaks `Schema.Unknown` into an identity-bearing field defeats every `Array.dedupe`/`HashSet` consumer downstream — the field must decode to a concrete shape for content identity to hold.
- The refinement predicate is invisible to the derived equivalence: a `Schema.Number.pipe(Schema.int())` compares as a plain `number` because the walk takes the base, so two refined values are equal iff their bases are equal — the predicate is a decode-time admission gate, never an identity lever, and narrowing the admitted set leaves the grain untouched.
- The same `to`-side selection that picks the decoded shape is uniform across `arbitrary` and `pretty` derivations (`getDescription` walks `ast.to`), so a `Schema.transform`'s decoded type is the canonical subject of every derived capability — equality, generation, and rendering agree on which side of the transform they describe.
- A `Schema.Class` presents as a transformation into a `TypeLiteral`, so the derived `Equivalence<A>` and the class's intrinsic constructed-value `Equal` answer the same field-by-field question by two paths — the derived instance is a free-standing value consumable where a consumer wants a supplied comparator (`Array.dedupeWith`/`SortedSet` keying), while `[Equal.symbol]` serves the comparator-free `HashSet`/`Array.dedupe` surface; one owner, two identity delivery channels for two consumer shapes.

[ORDER_DERIVATION_GAP]:
- No `Schema.order` and no `Order` annotation exist — `Equivalence` derives automatically from the AST, `Order` does NOT; a sorted-structure key over a schema-defined owner is composed by hand from `Order.struct({...})`/`Order.mapInput` over the member set, never lifted from the schema, so the codec, the auto-derived equivalence, and the manual `Order` are two automatic reads plus one hand-composed read of the same declaration.
- The asymmetry is a coherence hazard: a field added to the schema widens the codec and the derived equivalence silently and correctly, but the hand-composed `Order` does not see it — the new field is invisible to sort precedence and sorted-membership until the `Order.struct` is extended, so the one place the owner does not self-propagate is exactly the place a missed edit merges distinct keys.
- The discipline that closes the gap: build the `Order` by `Order.struct` over the literal `keyof typeof` of the schema's field record, anchored with `satisfies { readonly [K in keyof Fields]: Order<...> }`, so a missing field surfaces as a key error rather than a silent omission — the manual `Order` is anchored to the same symbol set the schema declares, recovering the compile-time propagation the derivation does not give for free.

[TAGGED_UNION_IDENTITY]:
- `Data.taggedEnum<Union>()` returns one owner fusing four surfaces the declaration-chain would otherwise spell separately: the per-tag constructor map (each `Case.Constructor<Extract<A, {_tag: Tag}>, "_tag">`), the `$is` refinement guard, the `$match` exhaustive dispatcher, and — through every case being a `Data.tagged` value — structural `Equal`/`Hash` per instance; no sibling guard, comparator, or `type X = ...` follows the union.
- Each case constructor is `tagged(tag)` = `args => Object.assign(Object.create(StructuralPrototype), args)` then `_tag` stamped, so a union member is a `StructuralPrototype`-rooted value with `Object.keys` equality and `Hash.structure` by construction — the identity is not added to the union, it is what the union members are; a field-empty case constructs with no argument (`VoidIfEmpty` collapses `{}` to `void`, runtime takes the `args === undefined` arm) yet still carries full structural identity.
- The union owner is a `Proxy` whose `get(tag)` returns `Predicate.isTagged` for `"$is"`, the `taggedMatch` thunk for `"$match"`, and `tagged(tag)` for any other key — the constructor map is not enumerated, it is synthesized per accessed tag, so adding a union member is one type-level arm with zero new runtime surface.
- `Data.taggedEnum<Z extends TaggedEnum.WithGenerics<N>>()` (N = 1..4) parameterizes the union at the type level only: `WithGenerics` carries `numberOfGenerics` plus the `A`/`B`/`C`/`D` slots, `Kind<Z, A, B, ...>` instantiates `Z["taggedEnum"]`, and `Args`/`Value` derive each case's constructor argument and result from the instantiated `Kind` — the runtime `Proxy` is identical to the monomorphic form and erases every generic, so a `Shape<number>` member and a `Shape<string>` member share the same `StructuralPrototype`, the same `Object.keys` equality, and the same `Hash.structure`; two members of the same tag are `Equal.equals` exactly when their decoded payloads are, regardless of the phantom type parameter.
- `GenericMatchers<Z>` supplies the generic `$is`/`$match` whose `$match` second overload returns `(self: Kind<Z, A, B, C, D>) => Unify<...>` — the type arguments are recovered at the match site from the matched value, never threaded through construction; the generic `$is` carries the richer narrowing overload `<T extends Kind<Z, any, any, any, any>>(u: T) => u is T & {_tag: Tag}` which INTERSECTS the matched tag onto the input type rather than collapsing to `Extract<Kind<Z>, {_tag: Tag}>`, preserving the inferred type arguments the bare `unknown` overload would discard.
- `$is(tag)` is `Predicate.isTagged(tag)`, a `_tag`-ONLY structural guard inspecting the discriminant alone, never the payload, so a value of the right tag carrying a malformed payload narrows positively — it answers membership-in-the-arm, not full-value validity; `isTagged` carries both the curried `(tag) => (self) => …` and data-first `(self, tag) => …` overloads, so the guard is a filter predicate for `Array.filter`/`Array.partition` and a direct test in one place without a wrapper lambda.
- `$match` dispatches by `cases[value._tag](value)` — a pure discriminant-indexed table read, so exhaustiveness is enforced by the `Cases & {[K in Exclude<keyof Cases, A["_tag"]>]: never}` constraint (an extra arm types as `never`, a missing arm is a missing key the checker flags); the identity discriminant `_tag` is simultaneously the dispatch key `$match` indexes, the narrowing key `$is` tests, and an own-enumerable field the `StructuralPrototype` equality compares — one symbol drives dispatch, refinement, and identity, so cross-tag values are never `Equal.equals` and never reach the same `$match` arm, keeping equality scope and dispatch scope the same partition.

[BRAND_OWNER]:
- `Brand.nominal<T>()` is a runtime identity passthrough returning a `Brand.Constructor` — it adds a phantom `[BrandTypeId]` tag at the type level and performs no validation; it exists only to distinguish two values of the same base type by meaning (an identifier vs. a count, both `number`).
- `Brand.refined<T>(predicate, onFailure)` (or the single-arg `(unbranded) => Option<BrandErrors>` form) attaches a runtime check; the constructor throws on the call form, `.option` returns `Option`, `.either` returns `Either<T, BrandErrors>`, and `.is` is a type guard — one constructor exposes throw, optional, validated, and refinement modalities by member, not by name suffix.
- `Brand.all(...constructors)` intersects multiple brands into one constructor (output is `UnionToIntersection` of their phantom key records), enforced by `EnsureCommonBase` so every combined brand must share the unbranded base type (a mismatch surfaces as the literal-type error `"ERROR: All brands should have the same base type"`); `Brand.unbranded(value)` recovers the raw value.
- `Schema.brand(name, annotations)` accumulates brands as an array annotation on the AST (`[...existing, name]`), so chained `Schema.brand` calls stack distinct phantom tags on one schema; it introduces no refinement of its own, deferring validation to the schema it brands (e.g. `Schema.Number.pipe(Schema.int(), Schema.brand("Id"))`), and yields a `BrandSchema` whose `make(unbranded)` constructs the branded value through the schema, fusing decode-time validation with the nominal tag.
- Branding is identity-transparent: a branded schema's derived equivalence passes straight to the underlying type, so two same-base values carrying different brands compare equal under the derived equivalence even though their phantom tags differ at the type level — the nominal distinction lives only in the static type, never in the runtime equality, and the brand disappears at runtime.

[COLLISION_DELETION]:
- A brand with no runtime predicate and no real nominal collision is deleted — the primitive stands; `Brand.nominal` earns its place only when two same-typed values are genuinely interchangeable at the type level and must not be.
- `Brand.refined` earns its place only by carrying a predicate the primitive cannot; a refined brand whose predicate is vacuous degrades to `nominal` and the deletion rule applies again.
- A brand row in an identity policy table with no real nominal collision and no refinement is the deleted layer: the primitive leaf stands as the bare `Eq.string`/`Order.number`, and `Brand.all` earns its place in a row only when two same-based fields of the owner (two `number` identifiers, a count and an index) are genuinely interchangeable and must not be — the table is where that interchangeability is forbidden once for every consumer that reads it.

```typescript
import { Brand, Data, Equivalence as Eq, Order, Schema as S } from 'effect';

type Code = string & Brand.Brand<'Code'>;
const Code = Brand.refined<Code>(
    (raw) => raw.length === 4,
    (raw) => Brand.error(`expected 4 chars, got ${raw.length}`),
);

class Entry extends S.Class<Entry>('Entry')({
    code: S.String.pipe(S.brand('Code')),
    rank: S.Number,
    note: S.String,
}) {}

const byIdentity: Eq.Equivalence<Entry> = Eq.mapInput(Eq.string, (e: Entry) => e.code);
const ranked: Order.Order<Entry> = Order.combine(
    Order.reverse(Order.mapInput(Order.number, (e: Entry) => e.rank)),
    Order.mapInput(Order.string, (e: Entry) => e.code),
);
const decoded: Eq.Equivalence<Entry> = S.equivalence(Entry);

const a = new Entry({ code: Code('AB12'), rank: 3, note: '<note-a>' });
const b = new Entry({ code: Code('AB12'), rank: 9, note: '<note-b>' });
const sameValue = byIdentity(a, b);
const tie = ranked(a, b);
const fullEq = decoded(a, b);
```

[IDENTITY_POLICY_TABLE]:
- The per-field identity decision (which fields are identity, which incidental, which descending) is one `satisfies`-anchored record of instances the owner declares once, not N scattered `mapInput` closures and N comparators at use sites: a `Record<keyof Owner, Equivalence<...>>` for equality, the parallel `Record<keyof Owner, Order<...>>` for ordering, and a `ReadonlyArray<keyof Owner>` for the hash subset — three reads of one declared field-key vocabulary.
- `Struct.getEquivalence(table)` is the alias of `Equivalence.struct` and `Struct.getOrder(table)` the alias of `Order.struct`; each FOLDS its field-keyed table into one aggregate instance, inferring the owner type from the record (`{ readonly [K in keyof R]: [R[K]] extends [Equivalence<infer A>] ? A : never }`), so the table is the single source and the aggregate is derived, never restated beside it. The struct fold projects per key, so the rows hold bare leaf instances (`Order.number`, `Eq.string`) and the projection is the fold's job, collapsing the N closures the manual spelling spreads across use sites.
- A field present in the owner but absent from the policy table is a missing key the aggregate's inferred type silently narrows away; anchoring the table with `satisfies { readonly [K in keyof Owner]: Equivalence<Owner[K]> }` converts the omission into a key error at declaration, so adding an owner field forces a table row. The struct fold drops a field from the aggregate only by dropping its row, so "compared / ignored" is row presence — an incidental field is excluded by OMISSION, never by a negative filter, the same decision `Hash.structureKeys`'s key list encodes for the hash half.
- `Struct.keys(o)` returns `Array<keyof T & string>` and `Struct.entries(o)` returns `Array<[keyof R & string, R[...]]>`, both excluding symbol keys at the type level — the hash subset and the table key set are drawn from this string-key vocabulary, so a symbol-keyed field is invisible to a `Struct.keys`-derived policy exactly as it is to `Object.keys` structural equality. The three policy surfaces consume the field-key vocabulary at two arities (record KEYS for the equivalence/order tables, a value ARRAY for the hash), so a renamed field breaks all three at the same `keyof` site rather than passing one and silently degrading the others.
- Reversing one ordering row is `Order.reverse(leaf)` inside the table cell, so descending-by-one-field-ascending-by-another is two cells, never a sign-flipping comparator; the equality table has no `reverse` because equivalence is its own dual, so the ordering table carries direction the equality table cannot and the two tables diverge exactly on the axis ordering owns.

[RUNTIME_SELECTED_POLICY]:
- A multi-key precedence order whose key sequence is RUNTIME-selected (a sort spec, a user-chosen column list) folds through `Order.combineAll` over a `ReadonlyArray<Order<Owner>>` assembled by indexed access from the table; `Order.empty<Owner>()` is the fold unit, so an empty selected-key list degrades to a stable total tie rather than throwing — the variadic-key sort builder is total.
- The matching field-scoped equality for the same runtime-selected key list folds through `Equivalence.combineAll`; with no exposed identity element, the empty list collapses every value to one class (everything-equal) where the order's empty leaves order untouched — the two folds have OPPOSITE empty-list semantics, so a dynamically built identity policy with zero selected keys is a silent merge for equality and a no-op for ordering.
- The variadic `combineAll` builders and the static `Struct.getEquivalence`/`getOrder` tables are the SAME fold at two binding times: the static table fixes the key set at declaration (the checker chases an owner-field addition into a missing row), the variadic builder fixes it at runtime — the choice is whether the identity grain is a compile-time owner property or a runtime-selected policy value, never a hand-written comparator either way.

[NULLARY_BRIDGE]:
- `Equal.equivalence<Owner>()` is NULLARY and wraps the owner's structural `[Equal.symbol]` — it is the FULL-value identity the `StructuralPrototype` already carries, the maximal table where every field is a row; the projected `Struct.getEquivalence(subset)` is the deliberate coarsening that drops incidental rows, so one owner exposes maximal identity (the nullary bridge) and intentional identity (the field-subset table) as two values, neither restating the owner.
- The structural `[Equal.symbol]` and a `Struct.getEquivalence` over the SAME full key set answer the same question by two routes — the prototype compares `Object.keys`, the table folds per-key leaves — so the table is only worth declaring when it is STRICTLY coarser (omits a field) or STRICTLY finer (a leaf that the structural recursion would compare by reference, e.g. an opaque field given an explicit row); a table equal to the structural identity is the nullary bridge spelled long.
- The nullary `Equal.equivalence` bridge, the auto-derived `Schema.equivalence`, and the hand-built `Struct.getOrder` table are the three identity-delivery channels off one owner, and the only one that does NOT self-propagate on a field addition is the ordering table — so `satisfies` over the schema's literal field record is the single discipline that makes all three break loudly together, closing the one gap where a forgotten field would merge distinct values under a sort while equality and hashing already saw it.

[STACKED_NOMINAL_POLICY]:
- A field's identity is a type-level decision orthogonal to its runtime comparison: a table row can be typed `Equivalence<string & Brand<"Code">>` to forbid comparing a code against a raw string, while the runtime instance is still `Eq.string` because branding is identity-transparent — the nominal wall lives in the row's type parameter, the comparison in its erased body.
- `Equivalence<in A>` and `Order<in A>` are CONTRAVARIANT, so a coarse leaf instance over the base (`Eq.string`, `Order.number`) is assignable to a row cell typed over the branded narrowing (`Eq.Equivalence<Code>`) — the brand wall is declared in the cell's type parameter and the leaf flows in unwidened, and the same contravariance lets a folded `Struct.getEquivalence` over a `string`-keyed record satisfy a target keyed by the branded subtype with no cast.
- The brand intersection narrows the table's INPUT type without changing its fold — `Brand.all(IsCode, IsSlot)` produces one constructor whose output flows into a row whose leaf instance never inspects the brand, so two values carrying different brand stacks over equal bases compare equal at runtime yet are non-interchangeable at the type level.

```typescript
import { Brand, Equal, Equivalence as Eq, Hash, Order, Struct } from 'effect';

type Code = string & Brand.Brand<'Code'>;
const Code = Brand.refined<Code>(
    (raw) => raw.length === 4,
    (raw) => Brand.error(`expected 4 chars, got ${raw.length}`),
);
type Row = { readonly code: Code; readonly weight: number; readonly span: number; readonly note: string };

const Identity = { code: Eq.string, weight: Eq.number } satisfies { readonly [K in 'code' | 'weight']: Eq.Equivalence<Row[K]> };
const Ranking = { weight: Order.reverse(Order.number), code: Order.string } satisfies { readonly [K in 'code' | 'weight']: Order.Order<Row[K]> };
const idKeys = ['code', 'weight'] as const satisfies ReadonlyArray<keyof Row>;

const sameId: Eq.Equivalence<Pick<Row, 'code' | 'weight'>> = Struct.getEquivalence(Identity);
const ranked: Order.Order<Pick<Row, 'code' | 'weight'>> = Struct.getOrder(Ranking);
const idHash = (r: Row): number => Hash.structureKeys(r, idKeys);
const fullId: Eq.Equivalence<Row> = Equal.equivalence<Row>();

const selected: ReadonlyArray<keyof Row> = idKeys;
const dynamicId: Eq.Equivalence<Row> = Eq.combineAll(selected.map((k) => Eq.mapInput(Eq.string, (r: Row) => String(r[k]))));
const dynamicOrder: Order.Order<Row> = Order.combineAll(selected.map((k) => Order.mapInput(Order.string, (r: Row) => String(r[k]))));
```

[COLLECTION_CONSUMPTION]:
- Identity instances are policy values consumed by data structures and combinators, not predicates re-passed per call, and the structure family bifurcates on HOW element identity reaches it: value-carried-identity structures (`HashSet`, `HashMap`, `MutableHashSet`, `MutableHashMap`) read the stored value's `[Equal.symbol]`/`[Hash.symbol]` directly and expose NO `getEquivalence`, while `SortedSet.empty`/`fromIterable`/`make`, `SortedMap.empty`/`make`, and `RedBlackTree.empty`/`make` capture an explicit `Order<K>` at construction as the membership-and-key policy and every subsequent `add`/`has`/`remove` reads that captured policy.
- Hashed structures locate a value by `Hash.hash` (bucket) then disambiguate within the bucket by `Equal.equals`; sorted structures locate by `Order` comparison alone, so two keys that compare `0` collide as one entry — a coarse `Order.mapInput` projection silently merges keys the hashed twin would keep distinct, and sorted-structure cardinality is a direct readout of the injected `Order`'s discriminating power.
- `Array.dedupe`, `Array.contains`, `Array.union`, `Array.intersection`, `Array.difference` consume `Equal.equivalence()` (the `Equivalence` backed by `Equal.equals`); the `*With` siblings (`dedupeWith`, `containsWith`, `unionWith`, `intersectionWith`, `differenceWith`) accept an explicit `(self, that) => boolean` — the bare verb is the default-identity collapse, the suffixed verb the override seam, never a parallel function family.
- A bucket-correct dedup over content-equal-but-reference-distinct values requires the operands to be `Data`/`Schema.Class` fused; feeding `dedupe` plain literals returns them unchanged because `Equal.equals` degrades to reference identity for non-`Equal` operands — the fusion is the precondition the default resolution assumes, and a raw object never dedupes by content.
- `Array.dedupeAdjacent`/`dedupeAdjacentWith` collapse only runs of equal neighbors, distinct from `dedupe`'s whole-array uniqueness — adjacency dedup over a sorted array is the streaming-cheap path, full dedup the order-agnostic path, a complexity decision not a style one.
- `Array.sort<B>(O: Order<B>)` constrains the input to `A extends B`, so a coarser `Order` (declared over a supertype or a single projected field) sorts a richer element — the `Order` need not name the full value, and `Order.mapInput(Order.number, project)` is the canonical coarse comparator the constraint admits; `Array.sort`/`sortBy`/`sortWith`, `SortedSet`, `SortedMap`, `RedBlackTree`, and `Order.min`/`max`/`clamp`/`between` all read one declared `Order`, deleting every per-call-site comparator across sorting, sorted membership, and bounded selection.
- Binary `SortedSet.union`/`intersection`/`difference`/`isSubset` take the second operand as a bare `Iterable<A>`, so the receiver's `Order` governs the result and a second set built under a different `Order` contributes elements re-judged by the receiver's policy — mismatched-`Order` set algebra is a membership trap the types do not catch.

[CONTAINER_LIFTING]:
- Eager linear containers expose `getEquivalence` to LIFT an element `Equivalence` into a whole-container one: `Array.getEquivalence(eq)`, `Chunk.getEquivalence(eq)`, `List.getEquivalence(eq)`, `Option.getEquivalence(eq)`, `Record.getEquivalence(eq)`, `Tuple.getEquivalence(...eqs)` — the lifted instance is consumable wherever a `getEquivalence`-taking container is itself an element, composing container-of-container identity by nesting the lift; a container without `getEquivalence` cannot be compared by a derived instance and must rely on its element's `[Equal.symbol]` or its captured `Order`.
- The lift arity tracks the container's algebra: `Tuple.getEquivalence(...isEquivalents)` is variadic-positional, `Either.getEquivalence({left, right})` takes a two-channel record (one `Equivalence` per side, not a single element instance), and `SortedSet.getEquivalence<A>()` is NULLARY — set equality there means equal membership under each set's own captured `Order`, layering the structure's identity above the element's already-fixed identity.
- `Equal.equivalence<A>()` is the single NULLARY bridge that turns a value's per-instance structural identity into the element `Equivalence` every `getEquivalence` demands, so `List.getEquivalence(Equal.equivalence<Shape>())` compares lists of values by deep structural content with no hand-written element comparator; the bridge composes recursively — `Array.getEquivalence(Option.getEquivalence(Equal.equivalence<Shape>()))` is the identity for `ReadonlyArray<Option<Shape>>`, each layer a named combinator reading the layer below, the only leaf the one nullary call.
- Feeding `HashSet.fromIterable` `Data`/union members works with no bridge at all — the structure reads `[Equal.symbol]` off the value — while feeding a list-equivalence-keyed dedup needs the bridge, so the same owner flows into a hashed set comparator-free and into a `List.getEquivalence` comparison through exactly one nullary call; `Equal.equivalence` never derives an `Order` and no container exposes a `getOrder`, so a value's ordering for `SortedSet`/`RedBlackTree` is composed by hand from `Order.struct`/`Order.mapInput`, the one identity surface the owner does not self-supply.

```typescript
import { Array as Arr, Data, Equal, Equivalence as Eq, HashSet, List, Order, RedBlackTree, SortedSet } from 'effect';
import type { TaggedEnum } from 'effect/Data';

type Shape<A> = Data.TaggedEnum<{
    readonly Point: { readonly at: A };
    readonly Span: { readonly lo: A; readonly hi: A };
    readonly Void: object;
}>;
interface ShapeDef extends TaggedEnum.WithGenerics<1> {
    readonly taggedEnum: Shape<this['A']>;
}
const Shape = Data.taggedEnum<ShapeDef>();

const at = Shape.Point({ at: 3 });
const same = Shape.Point({ at: 3 });
const span = Shape.Span({ lo: 1, hi: 9 });
const empty = Shape.Void();

const distinct: number = HashSet.size(HashSet.fromIterable([at, same, span, empty]));
const points: ReadonlyArray<Shape<number>> = [at, same, span].filter(Shape.$is('Point'));
const reach: number = Shape.$match(span, { Point: (p) => p.at, Span: (s) => s.hi - s.lo, Void: () => 0 });

const item: Eq.Equivalence<Shape<number>> = Equal.equivalence<Shape<number>>();
const sameRun: Eq.Equivalence<List.List<Shape<number>>> = List.getEquivalence(item);
const runEqual: boolean = sameRun(List.make(at, span), List.make(same, span));
const ranked: Order.Order<Shape<number>> = Order.mapInput(Order.number, (s) => (s._tag === 'Span' ? s.hi : s._tag === 'Point' ? s.at : 0));
const indexed: RedBlackTree.RedBlackTree<Shape<number>, Shape<number>> = RedBlackTree.fromIterable(
    Arr.map([at, span], (n) => [n, n] as const),
    ranked,
);
const ordered: SortedSet.SortedSet<Shape<number>> = SortedSet.fromIterable(ranked)([at, span]);
```

[PROJECTED_COARSENING]:
- `mapInput` over a chosen field is the one lever that turns full-value identity into field-scoped identity for any consumer: `Equivalence.mapInput`/`Order.mapInput` retarget contravariantly onto a projection, so `HashSet` membership stays content-deep while a `RedBlackTree` keyed by `Order.mapInput(Order.string, byField)` indexes by one field — the same value owner feeds two structures with two deliberately different identity grains.
- A field-projected `Equivalence` fed to `Array.dedupeWith` keeps the first occurrence per projected key (occurrence order preserved), turning dedup into a keyed-first-wins reduction without a grouping pass; the same projection fed to `Array.groupBy` partitions every occurrence by that key — projection grain is shared, retention policy differs.
- `Order.combine(primary, secondary)` is consumed identically by `Array.sortBy` (multiple `Order`s, lexicographic) and by a single `RedBlackTree`/`SortedSet` `Order`, so multi-key sort precedence and multi-key tree indexing are one declared comparator — sort order and tree traversal order stay in lockstep, and adding a tiebreak is one `combine` arm every consumer inherits.
- `SortedSet.getEquivalence<A>()` yields `Equivalence<SortedSet<A>>` for set-of-sets equality; equal sets here mean same membership under each set's own `Order`, layering the structure's identity above the element's.

[KEYED_GRAIN]:
- A keyed identity policy reaches the consumer as two coupled instances that must share a grain: `HashSet`/`HashMap` route by `Hash.hash` then settle by `Equal.equals`, so a hash narrower than its equality (`Hash.structureKeys` over fewer fields than the equality compares) over-collides into longer buckets — correct, slower — while a hash wider than its equality (hashing an incidental field the equality ignores) scatters equal values across buckets and reports them absent, the membership failure no test of the equality alone exposes.
- The over-collide-versus-scatter asymmetry makes the safe default a hash that is never wider than its equality; deriving both from one `Hash.structureKeys`/`Equivalence` over the same declared field set is the single read that keeps a `HashSet` membership-correct regardless of which incidental fields the value also carries, and because the `satisfies`-anchored equivalence table and the `keyof typeof` hash array index the same literal field-key set, the equal-implies-hash-equal gate holds by anchor rather than by audit — a field promoted from incidental to identity is one row plus one array entry, both forced by the same `keyof` constraint.
- `Array.dedupe` and `HashSet` membership inherit the latent equality bug, not the hash bug — a fast-wrong custom `Equal` produces duplicates `dedupe` cannot remove and a hashed lookup that misses, with a passing `Hash` masking the cause, so identity correctness is proven at the owner and never re-judged at a dedup or membership call site.
- The `getEquivalence`-bearing-versus-not bifurcation and the `$is` payload-blindness compose into the membership hazard: `HashSet`/`SortedSet` judge a value by full structural identity or captured `Order`, but `$is` admits a malformed-payload member of the right tag — so a value narrowed by `$is` is arm-correct yet may differ structurally from a canonical member, and only construction through the case constructor (never an ad-hoc `{_tag, ...}` literal) guarantees the `StructuralPrototype` the hashed structures require.

[ERROR_VALUE_IDENTITY]:
- `YieldableError.prototype` is assigned `StructuralCommitPrototype` (`CommitPrototype` spread then `StructuralPrototype`), so `Data.Error`/`Data.TaggedError`/`Schema.TaggedError` instances carry STRUCTURAL equality and `Hash.structure` — two failures of the same tag with equal fields are `Equal.equals`, and a typed-failure channel deduplicates by content like any other `Data` value.
- The error constructor `Object.assign(this, args)` makes every supplied field own-enumerable, so `Object.keys` (hence both equality and hash) sees the domain fields; the original args are ALSO stashed under a non-enumerable `plainArgsSymbol` purely for `toJSON`, invisible to identity — `cause` and `message` participate in equality only when passed as args, while native `stack` stays non-enumerable and is correctly excluded.
- `Schema.TaggedError` routes through `data_.Error` (not `data_.Class`), so an error class and a data class diverge in their base yet converge on `StructuralPrototype`-equivalent identity; the `_tag` is an own-enumerable field there too, making cross-tag errors unequal by the key-set comparison the same way a tagged data union discriminates.

```typescript
import { Data, Equal, Hash, HashSet } from 'effect';

class Marker extends Data.TaggedError('Marker')<{ readonly code: number; readonly note: string }> {}

const seen: HashSet.HashSet<Marker> = HashSet.fromIterable([
    new Marker({ code: 1, note: '<a>' }),
    new Marker({ code: 1, note: '<a>' }),
    new Marker({ code: 2, note: '<b>' }),
]);
const distinct: number = HashSet.size(seen);

const probe = new Marker({ code: 1, note: '<a>' });
const routed: boolean = HashSet.has(seen, probe);

const drifted = new Marker({ code: 1, note: '<a>' });
Object.defineProperty(drifted, Hash.symbol, { value: () => Hash.hash('<wrong>'), enumerable: false });
```

[INTERACTION_LAWS]:
- The structural `Equal` on a `Data`/`Schema.Class` value compares every own key, so two instances differing only in an incidental field are unequal there; `Equivalence.mapInput` over the identity fields is the narrower, intentional equality — the two coexist and answer different questions on the same value, while `Schema.equivalence` derives a third equality from the decoded type, all three reading the one schema's member set so a field added to the owner widens every identity surface at once with no sibling edit.
- `Order.combine` and `Equivalence.combine` are the same monoidal layering at different codomains (`-1|0|1` vs `boolean`); a multi-key `Order` and the matching multi-field `Equivalence` are derivable from the identical projection set, keeping sort precedence and equality scope in lockstep — but the codomain fold and the value-selecting monoid are also the same algebra at two altitudes: `Order.combine` folds the `Ordering` codomain to decide WHICH key breaks the tie, then `Monoid.max(Bounded)` folds that resolved `Order` to decide WHICH value wins, so a tiebreak added as a `combine` arm reshapes the extremum a `combineAll` selects with zero fold-site edits.
- The dual coherence and the bound-swap fuse: `Monoid.reverse(Monoid.max(bound))` and `Monoid.min(Bounded.reverse(bound))` are the same monoid by two routes — reversing the fold or reversing the bounded order both flip max to min with the identity element tracked, so the inversion is expressible at whichever layer the call site already holds, never by editing `maxBound`/`minBound` literals.
- The obligation lattice and the semilattice boundary together fix where each identity surface self-propagates: `Equal extends Hash` forces equality and hashing to be declared as one obligation, `Equivalence.combine` is the law-total meet with no exposed identity, and `Order` alone carries both `empty` and `reverse` — so widening an owner's field set propagates to its derived `Equivalence` and structural `Equal` automatically while the value-selecting `Bounded` and the keying `Order` are the two surfaces a new field must be hand-threaded into before the extremum fold and the sorted key see it.
- The hash-gate ordering and the `Hash.cached` memo compose into the mutation hazard: equality runs `Hash.hash === Hash.hash` first against frozen memos, so a post-hash field mutation makes the full gate report unequal while the comparator alone still reports equal — the two diverge precisely because the gate trusts the cache the comparator ignores, and only an immutable owner keeps them in lockstep.
- The prototype-tier dispatch and the structural region are orthogonal switches on the same gate: the prototype fixes which `[Equal.symbol]`/`[Hash.symbol]` a value carries permanently, while the region temporarily neutralizes hash routing and reroutes only naked-record/array operands — owner identity from the prototype always wins inside the region, so the region widens equality for un-fused shapes without ever softening a `Data` value's declared identity.
- The hook-first walk and the `to`/`from` crossing rules compose: a `Schema.transform` whose `to` side is a `Schema.declare` carrying an `equivalence` annotation resolves the annotation at the declaration node after the transformation selects the `to`-side, so a decoded opaque value gets exactly the identity its declaration supplies while a refinement layered over it leaves that identity untouched — annotation, transformation, and refinement each act at a distinct node and never collide; `Union` discriminant routing and `TypeLiteral` presence-parity together make the derived equivalence reject cross-member and presence-mismatched values that a naive `mapInput` over shared fields would accept, so the derived instance is strictly finer than a projected field equivalence and `mapInput` over identity fields is the deliberate coarsening that re-admits cross-member equality when the domain wants it.
- One value owner exposes membership grains selected entirely by which identity instance the consumer reads, never by a per-site comparator: `Array.dedupe` (deep, via `Equal.equivalence()`) keeps distinct-by-content values, `Array.dedupeWith(_, byKey)` keeps one per projected key, `HashSet.fromIterable` keeps distinct-by-deep-content, and a `RedBlackTree`/`SortedSet` keyed by a tiebroken `Order` collapses two values only where that `Order` returns `0` — three questions (full-value identity, field identity, ordering-with-tiebreak) on one owner, kept coherent because all three project the owner's declared members, so a field added to the owner reshapes every membership, dedup, and indexing site that reads them at compile time.
- The generic union's runtime-erased identity and the nullary `Equal.equivalence` bridge fuse into provider-free container identity: because every `Shape<A>` instantiation shares the one `StructuralPrototype`, `Equal.equivalence<Shape<A>>()` is the same lift for every `A`, and a `HashSet<Shape<A>>` deduplicates by content while a `List.getEquivalence(Equal.equivalence<Shape<A>>())` compares runs — two identity delivery channels off one owner, one comparator-free and one through a single nullary call.
- Per-member identity policy is declared at the owner, not the comparison site: `Hash.structureKeys` and `Equivalence`/`Order` built from `mapInput` over chosen fields encode "these fields are identity, those are incidental" once, so every equality, ordering, hash, and set-membership decision reads the same key policy, and the recursive `Suspend` memoization with the `NeverKeyword`-throws boundary bracket the derivation's totality — derivation safety is a property of the type's inhabitedness, checked once at `Schema.equivalence` time, not a property of nesting depth or cycles.
