# Decorator Stack Equilibrium And Reconciliation Calculus

# Decorator Equilibrium Invariant Catalog

- Equilibrium invariants are closed predicates on decorator-admitted evidence that must hold across federation contexts after promotion landing and boot freeze — not restatements of individual certificate closure gates.
- **I1 fingerprint parity** — `stack_fingerprint_after` on production adapter, law-matrix collected item, and certificate row agree on unwrap depth, `__type_params__`, and synthesis family — mismatch is equilibrium violation before domain logic is suspect.
- **I2 lattice-certificate sync** — every seam decorator transition proof lattice `transition_id` referencing a promoted owner carries matching promotion certificate concurrency calculus `certificate_id` with synchronized `stack_version` — partial bump on one artifact alone violates equilibrium.
- **I3 compile oracle alignment** — `__pydantic_core_schema__` tag multiset, `TypeAdapter.core_schema`, and `msgspec.inspect.type_info` agree with admitted decorator modes after first-touch — stale generic graphs violate equilibrium under concurrency.
- **I4 altitude separation** — domain modules do not import ingress-only decorator owners; interior folds do not invoke `@validate_call` or ingress `TypeAdapter` without declared `re_ingress_boundary` row — altitude bleed is equilibrium defect, not style drift.
- **I5 duplicate-law collapse** — wire scalar normalization exists on exactly one altitude per invariant — ingress `@field_validator(mode="before")` mirrored on canonical `@model_validator` without divergence policy row violates equilibrium.
- **I6 trust posture consistency** — `.raw_function`, `model_construct`, and semi-trusted replay paths match composition-root trust rows for current compile fingerprint and `schema_version` — trust without re-pin after graph evolution violates equilibrium.
- **I7 boot immutability** — post-freeze `TypeAdapter`, `Decoder`, and `BeartypeConf` constants are read-dominant; post-boot rebinding without certificate is equilibrium violation under parallel first-touch.
- **I8 metamorphic codec totality** — polymorphic arms with egress serializers admit encode-decode-revalidate-metamaterialize receipts per arm — partial arm coverage violates equilibrium on wire handoff edges.
- Invariant law — each invariant carries `equilibrium_invariant_id`, `default_reconciliation_row`, `proof_layers_to_replay`, and `liable_owner_module` — undocumented invariant blocks merge.

# Violation Signal Taxonomy

- Violation signals classify decorator-evidence drift through a closed vocabulary — each signal maps to one equilibrium invariant and one default reconciliation row from the decision lattice below.
- **D1 fingerprint skew** — production adapter unwrap depth or signature frozenset diverges from law-matrix collected item or certificate `stack_fingerprint_after` — invariant I1; liable owner is seam adapter or test harness owner per attribution table.
- **D2 lattice-certificate desync** — `lattice_row_ids` on certificate reference owners whose transition proof uses stale fingerprint — invariant I2; liable owner is promotion certificate module.
- **D3 compile oracle stale** — first-touch or dependency pin leaves tag multiset diverging from promoted baseline without documented delta receipt — invariant I3; liable owner is generic owner module or composition root adapter constant.
- **D4 altitude bleed** — domain import of ingress decorator owner or interior `model_validate` on `model_dump` output — invariant I4; liable owner is violating domain module; restoration is seam repair or quarantine, not domain validator edit.
- **D5 duplicate-law** — identical wire normalization on ingress and canonical altitudes — invariant I5; liable owner is promotion unit that landed partial mirror; restoration is thin projection row.
- **D6 trust stale** — trusted replay path accepts payloads compiled under obsolete graph — invariant I6; liable owner is composition-root trust table; restoration is `trust_repin` certificate before replay resume.
- **D7 boot mutable adapter** — module-level `TypeAdapter` rebinding after warm-graph freeze — invariant I7; liable owner is composition root; restoration is worker restart with certificate sync.
- **D8 metamorphic gap** — polymorphic arm lacks encode-decode receipt after serializer promotion — invariant I8; liable owner is egress projection adapter.
- **D9 propagation partial** — `fan_out_contexts` consumer lacks updated certificate or lattice row after vocabulary-backed decorator promotion — cross-cuts I2 and federation law from promotion certificate concurrency calculus; signal carries obligated context ids in fault payload.
- **D10 lock-order deadlock** — parallel first-touch on shared generic memo without `acquire_order_rank` row — invariant I7 under free-threaded boot; liable owner is root lock-order registry.
- Signal law — each signal carries `signal_id`, `equilibrium_invariant_id`, `default_reconciliation_row`, `choreography_phases`, and `liable_owner_module` — free-form error strings without signal id block reconciliation routing.

# Reconciliation Decision Lattice

- Restoration events classify through a finite chooser lattice — no ad hoc multi-context decorator patches without row binding.
- **Rollback-first row** — D2, D9 on active promotion unit — full atomic certificate and lattice revert per promotion certificate atomic sequencing law; no forward-fix on consuming context until vocabulary or warm-graph unit completes.
- **Thin projection row** — D5 duplicate-law — collapse secondary altitude to projection mirror in same promotion unit; canonical owner retains rule; ingress thins to wire-only normalization.
- **Seam repair row** — D1, D4 at foreign pair — adapter unwrap chain, altitude import graph, or remap table patch on liable seam module; domain interiors excluded from repair edit set.
- **Compile refresh row** — D3, D7 — parametrized first-touch replay, warm-graph re-registration, or process restart per boot immutability law; interior domain validators not edited during refresh.
- **Trust re-pin row** — D6 — composition root updates trust rows with schema version and encoder identity before semi-trusted replay resumes — domain interior does not construct local bypasses.
- **Quarantine row** — repeated D4 or D10 within grace window — root isolates edge or context; handoffs return tagged `EquilibriumFault` without partial canonical emission.
- **Re-sync row** — D8 or receipt fold lag on decorator-induced fault kinds — metamorphic codec replay across obligated contexts; no bypass through undecorated dump helpers.
- **Forward-fix row** — permitted only when rollback orphans production stores — root documents exemption with migration hop certificate and sunset date; default remains rollback-first.
- Lattice law — each row names liable owner, affected context ids, restoration morphism sequence, and invariant restored — undocumented restoration path is merge blocker.

# Restoration Choreography On Decorator Evidence

- Multi-context decorator restoration executes as typed choreography — not parallel domain edits across contexts.
- **Phase E0 — signal freeze** — composition root pins `certificate_id`, violating `signal_id`, and obligated context set; no new handoffs on affected decorator edges until choreography completes or quarantine declared.
- **Phase E1 — certificate consolidate** — when D2 or D9 — liability owner lands synchronized certificate and lattice bump before seam or compile repair; fingerprint receipts regenerate at liable module.
- **Phase E2 — edge repair** — seam adapter unwrap realignment, altitude import remediation, or thin projection on duplicate-law — domain modules in all contexts excluded from E2 edit set.
- **Phase E3 — proof replay** — `proof_layers` from promotion certificate concurrency calculus re-run in order from failed layer; static checker before compile oracle before unwrap fingerprint before seam lattice before metamorphic codec — conflated attribution fails harness.
- **Phase E4 — handoff resume** — root clears quarantine; canonical handoffs restart at materialization exit per ingress-to-materialization handoff resume rule — ingress carriers forbidden at resume boundary.
- **Phase E5 — witness commit** — reconciliation witness row updates beside liable owner; `fan_out_contexts` import revised row in same commit as E4 resume.
- Choreography parallelism — E1 and E2 serialize per liability assignment; E3 may parallelize per context only after shared certificate and warm-graph nodes pass.

# Compose-Chain Certificate Serialization

- Multi-hop decorator admission across seam edges serializes as one compose-chain certificate — distinct from single-owner promotion certificate concurrency calculus certificates when policy stacks, ingress compile, materialize, and egress projection chain load-bearing.
- Compose-chain row shape: `chain_id`, `ordered_certificate_ids`, `ordered_lattice_row_ids`, `unwrap_depth_budget`, `phase_attribution_vector`, `metamorphic_arm_ids`, `liability_head_module`.
- `ordered_certificate_ids` lists certificates in handoff order — ingress compile certificate precedes materialize certificate precedes egress serializer certificate; inverting order without `depends_on_certificate_ids` edges is planning defect.
- `unwrap_depth_budget` caps cumulative policy wrapper depth across chain — exceeding budget triggers compose-stability foreign-key review for OTEL cardinality and lock-order extensions.
- `phase_attribution_vector` maps each chain link to decoration phase — import, class-body, first-touch, call-time — conflated phase in one chain digest fails compose-chain closure.
- Audit replay deserializes chain from frozen JSON beside root seam map — pytest node ids and CI run numbers are not chain keys.
- Chain version bumps atomically with head `certificate_id` — partial chain update without head bump is synchronization defect indexed `chain_without_head_sync`.
