# [PATTERNS]

Pattern law binding two or more language branches. A row admitted here binds every branch it names; a single-branch law routes to that branch's stack doctrine instead.

## [01]-[ROWS]

| [INDEX] | [LAW]                                                                                                           | [BINDS]             |
| :-----: | :-------------------------------------------------------------------------------------------------------------- | :------------------ |
|  [01]   | a derived artifact keys on the content hash of its source; cache validity is key equality, never path or mtime  | C#, Python, tooling |
|  [02]   | root and anchor discovery walks upward to a sentinel file, never a fixed `parents[N]` depth or a cwd assumption | Python, C#, tooling |
|  [03]   | an operational rail returns one typed envelope; failure rides the envelope, never sentinel values in data rows  | all branches        |
|  [04]   | a wire token admits only its exact emission spelling; a tolerant parse re-emitting normalized forks the key     | all branches        |
|  [05]   | a multi-field hash preimage length-frames every variable-width field and count-frames every adjacent collection | all branches        |
|  [06]   | a content key's preimage covers every identity-bearing member; a member outside it is declared derived on site  | all branches        |

C# spells the content key `XxHash128` on the wire, and cross-runtime peers compare wire tokens byte-wise — the emitting owner's spelling is the admission grammar. Row [01] splits slot from address: an evidence record's identity slot carries the pre-run source key the hit test compares, and a produced output's content address is a separate derived fact — minted into the slot it silently defeats keyed elision. Row [01] also demands canonical bytes: a source with no canonical byte form — a live handle, a callable, a non-deterministic serialization — joins the key as environment-scoped identity and demotes admission to forced-live, never trusted to elide. Row [05] rejects separator-joined concatenation outright: a separator character inside one value shifts two distinct field splits onto one digest, and fixed-width elements never identify the boundary between collections. Row [05] extends beyond digests to composite ids: a composite identity rides a canonical codec — framed canonical bytes or canonical JSON — never a hand-rolled join or quote scheme injective on one ambiguity axis, and an array-bearing key frames shape beside canonicalized dtype and layout bytes. Row [06] makes the key and the record one identity: a stored member the preimage omits is a split-brain whose stated re-word or re-order semantics is false unless the declaration names it derived-or-annotation. Row [06] reads inputs as identity: any input whose value shifts the produced output — a toolchain generation, a credential's content, a consumed template's digest — is identity-bearing wherever it lives and joins the preimage, never only the record's stored fields. Row [06] bounds the preimage in both directions: an input that cannot shift a produced success's bytes — an execution policy, a timeout, a retry budget — stays outside the content preimage, because policy keys when work fails, never what a success produces, and admitting it forks one content identity across policy motion. Row [06] excludes display labels on the same ground: a human-facing label cannot shift a success's bytes, so admitting it forks one content identity across renames. A security identity — a credential fingerprint, trust material — rides a cryptographic digest; a speed hash keys caches and elision, never trust.
