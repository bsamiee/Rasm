# Spectral Field Operators as Multiplier Algebra

[OPERATOR_IS_A_SPECTRAL_MULTIPLIER_NOT_A_STENCIL]:
- A translation-invariant field operator on a periodic grid — gradient, divergence, Laplacian, Helmholtz inverse, fractional power, Gaussian smoothing, Hockney free-space convolution — is one diagonal multiplication in the frequency carrier bracketed by a forward and an inverse transform. The finite-difference stencil family and the spectral family are not two algorithms to choose between per operator: every constant-coefficient differential and convolutional operator collapses to the single form `inverse(symbol(k) ⊙ forward(field))`, and the operator's entire identity is its symbol — a pure function of the wavenumber vector. The collapse trigger is three or more sibling operators (`Gradient`, `Laplacian`, `Smooth`) each carrying its own forward-multiply-inverse triple; the derived form is one operator surface parameterized by a symbol value, and a new operator is a new symbol row, never a new code path.
- The symbol is a policy value carrying its own behavior, not a flag the apply loop re-derives: the first-derivative symbol is `i·k`, the Laplacian symbol is `−|k|²`, the screened-Poisson inverse symbol is `1/(|k|² + κ²)`, the fractional-Laplacian symbol is `|k|^{2s}`, and the heat-smoothing symbol is `exp(−t|k|²)`. Each is a closure over the wavenumber that the operator owner stores once; composing two operators is multiplying their symbols pointwise before a single inverse transform, so a gradient-then-smooth is one fused multiplier pass, not two bracketed round trips. The design records which symbol produced a field as evidence, because the same field under two symbols is two different operators and a downstream gate attributes a spurious mode to the symbol, not to the data.
- The inverse of a self-adjoint operator is the reciprocal of its real symbol, but the reciprocal is singular exactly where the symbol vanishes — the zero wavenumber for the bare Laplacian inverse — so the operator inverse is a partial function whose domain fault is the constant mode. The screened variant `1/(|k|²+κ²)` is total because the screening shifts the symbol off zero; the bare Poisson inverse owns a typed regularization policy that either zeros the constant mode (mean-free projection) or pins it to a supplied integral, and the receipt names which, because the two choices give fields differing by a constant the caller cannot otherwise recover.

[WAVENUMBER_VECTOR_IS_A_DERIVED_OWNER]:
- The frequency carrier is not the natural index — it is the split-spectrum ordering where the first half holds the ascending positive wavenumbers through the Nyquist mode and the second half holds the descending negatives, with the per-bin spacing scaled by the sample rate over the length. This ordering is a derived fact of the transform convention, computed once at grid construction from the length and the domain extent, and carried as the operator's wavenumber owner; hand-indexing `k = i` over the raw buffer applies the symbol to a wrong frequency for every bin past the half-length and silently produces a field that is the operator applied to an aliased symbol.
- The wavenumber owner derives every dimension's axis from one length-and-extent pair: the physical wavenumber is the FFT-ordered bin index times `2π/L`, so a separable multi-dimensional symbol like `−(k_x²+k_y²+k_z²)` is the outer combination of per-axis wavenumber vectors, each a one-dimensional derived owner, never a flattened literal table. The Nyquist mode is the structural hazard: at an even length the highest positive and lowest negative wavenumbers collapse onto one real bin, and the first-derivative symbol `i·k` must zero that mode rather than assign it the ambiguous `±k_Nyquist`, because an odd-derivative multiplier at Nyquist is not representable on the real-packed spectrum and leaks a checkerboard mode. The wavenumber owner therefore carries the Nyquist index as a known position and the symbol family zeroes odd-order multipliers there as a definition-time law, not a runtime guard the apply loop forgets.

```csharp
readonly record struct WaveAxis(int Length, double Extent)
{
    public double[] K() // FFT-ordered physical wavenumbers; Nyquist owned, not guessed
    {
        var dk = 2.0 * Math.PI / Extent;
        var half = (Length >> 1) + 1;
        return Generate.LinearRangeMap(0.0, 1.0, Length - 1.0, i =>
            (i < half ? i : i - Length) * dk);
    }
    public int Nyquist => Length.IsEven() ? Length >> 1 : -1; // -1 ⇒ no exact Nyquist mode
}

static Complex[] Apply(Complex[] field, WaveAxis ax, Func<double, Complex> symbol)
{
    var k = ax.K();
    Fourier.Forward(field, FourierOptions.AsymmetricScaling);       // see scaling law below
    var n = ax.Nyquist;
    var driven = field.Select((c, i) => c * (i == n ? Complex.Zero : symbol(k[i]))).ToArray();
    Fourier.Inverse(driven, FourierOptions.AsymmetricScaling);
    return driven;
}
```

[SCALING_CONVENTION_IS_THE_SILENT_OPERATOR_BUG]:
- The transform options enum carries three orthogonal bits — an inverse-exponent sign, a forward-scaling suppression, and a no-scaling-at-all — and the default is symmetric `1/√N` in each direction. A spectral multiplier brackets a forward and an inverse, so under the symmetric default the operator carries a net `1/N` that exactly cancels only because the pair multiplies to `1/N · N`; the trap is that the pair cancels for a round trip but the operator is not a round trip — a symbol that is not unit-modulus changes the field magnitude, and the symmetric convention silently scales the operator output by `1/√N` relative to the asymmetric convention that suppresses forward scaling entirely. The asymmetric option — forward unscaled, inverse `1/N` — is the convention under which `forward·symbol·inverse` reproduces the continuous operator with no spurious size factor, and it is the only admissible option for a multiplier operator. The default is correct for a power spectrum and wrong for an operator; selecting it is the most common silent error in the surface.
- The inverse-exponent bit and the no-scaling bit have named composite aliases, and the real-packed transform path rejects the inverse-exponent bit outright with a not-supported throw, so an operator built over the real-packed half-spectrum can never request the conjugate convention and must keep the forward sign canonical. The scaling choice is therefore not a free parameter on the operator — it is fixed to the asymmetric convention by the operator's own definition, encoded once in the operator owner, never re-passed at each call where a caller might default it back to symmetric.

[REAL_FIELD_HALF_SPECTRUM_IS_THE_DENSE_CARRIER]:
- A real-valued field has a conjugate-symmetric spectrum, so half the complex bins are redundant and the real-packed transform stores the non-redundant half in place: the buffer must be the field length plus two for an even length, or plus one for an odd length, and the transform packs the real and imaginary parts of the half-spectrum interleaved into that extended real buffer with no separate complex allocation. The operator over a real field is therefore a multiplier over the packed half-spectrum at half the storage and half the multiply count of the full complex path, and the symbol must be evaluated only over the non-negative wavenumbers — the conjugate half is reconstructed by the inverse-real transform and never touched. An operator that allocates a full complex buffer for a real field doubles memory and applies the symbol redundantly to bins the inverse discards.
- The packed buffer length is a definition-time admissibility law, not a runtime check the caller satisfies by luck: the real transform throws when the buffer is shorter than the parity-dependent packed length, so the field grid owner sizes its backing store to the packed length at construction and the operator never sees an undersized buffer. The packed layout couples the real and imaginary parts of each half-spectrum bin into adjacent slots, so the symbol multiply over the packed buffer is a per-bin complex multiply walking the buffer two slots at a time, with the zero-frequency and Nyquist bins purely real and owning the buffer's first and last real slots respectively — a separable multi-dimensional operator over a real field applies the one-dimensional real transform along the contiguous axis and full complex transforms along the strided axes, so the carrier shape is mixed and the apply walk must respect it.

[POWER_OF_TWO_GRID_IS_A_HARD_PERFORMANCE_CONTRACT]:
- The transform provider dispatches on length: a power-of-two length runs the radix-2 decimation in place at `N log N`, and every other length — including every prime — silently routes through a Bluestein chirp-convolution that pads to the next power of two above twice the length minus one and runs three radix-2 transforms plus a precomputed chirp sequence allocation. The cost gap is not a constant factor the caller can ignore: a prime-length field grid pays roughly three power-of-two transforms over a buffer more than twice its size, so a grid sized to a round decimal count is an order of magnitude slower per operator application than the same field padded to the enclosing power of two. The grid length is therefore a power-of-two policy on the field owner, not a free dimension, and a field whose natural extent is not a power of two is zero-padded or resampled to one at construction, with the padding recorded so the operator's spectral derivative is interpreted on the padded period, not the physical one.
- The Bluestein path carries a chirp-sequence overflow ceiling: the chirp index square overflows the integer carrier above a length near the square root of the signed integer max, so a non-power-of-two grid beyond that threshold is not merely slow but unrepresentable through the managed provider. This is a second reason the field grid owner pins the length to a power of two — the slow path has a hard upper bound the fast path does not, so the only length policy that scales is the power-of-two one, and the design states it as a law of the grid, not a tuning hint.

[WINDOWING_AND_DEALIASING_AS_SPECTRAL_POLICY]:
- A field on a non-periodic support transformed as if periodic injects a discontinuity at the wrap seam whose spectral signature is a broadband leakage tail that the operator's symbol then amplifies — the Laplacian symbol weights the high wavenumbers by `|k|²`, so seam leakage at the Nyquist scale dominates the operator output. The window family carries both a symmetric and a periodic variant of each taper, and the distinction is load-bearing: the symmetric window divides its phase by the length minus one and is correct for a finite measurement, while the periodic window divides by the length and is the only variant consistent with the discrete-transform period assumption a spectral operator makes. Applying a symmetric window before a spectral derivative introduces a one-sample asymmetry that biases the lowest wavenumber bins; the periodic variant is the operator-consistent taper, and the choice between them is a policy row keyed on whether the field is genuinely periodic or a windowed measurement, never a default.
- A nonlinear field product evaluated on the grid — an advection term, a quadratic reaction — generates wavenumbers above the grid Nyquist that fold back as aliased low modes the linear operator cannot distinguish from signal. The dealiasing law is a spectral truncation policy: the two-thirds rule zeros the top third of the wavenumber band before the inverse transform of a product, and the smooth-filter alternative multiplies by an exponential cutoff symbol that decays the marginal modes rather than truncating them sharply. Dealiasing is therefore a second multiplier composed into the operator's symbol — a real mask over the wavenumber owner — so a dealiased spectral derivative is the product of the derivative symbol and the truncation symbol, evaluated in one pass, and the receipt records the cutoff fraction because the truncation changes the represented bandwidth and a downstream resolution claim depends on it.

[SEPARABLE_AND_FUSED_MULTIDIMENSIONAL_APPLY]:
- A separable symbol over a multi-dimensional field is the outer sum or product of per-axis one-dimensional symbols, so the multi-dimensional operator is one multi-dimensional transform, one separable multiplier walk over the flat row-major spectrum, and one inverse — never a sequence of per-axis bracketed transforms. The multi-dimensional transform over a flat row-major buffer takes the dimension extents as an integer vector and transforms every axis in place, so the spectrum is a flat array whose multi-index decomposes into per-axis wavenumber lookups; the apply walk strides the flat buffer computing each bin's symbol as the separable combination of its per-axis wavenumbers, and the tensor-primitive fused multiply-add accumulates the separable Laplacian symbol `−(k_x²+k_y²+k_z²)` over contiguous spans without materializing a full symbol grid.
- The zero-copy bridge from a column-major matrix field to the row-major transform kernel is a real performance seam: the dense column-major storage exposes its backing array directly with swapped dimension extents for the two-dimensional transform, so a matrix-shaped field transforms with no copy, while requesting a row-major view of the same storage forces a copy. The operator over a matrix field therefore takes the matrix overload that consumes the column-major backing directly, and the wavenumber owner swaps its axis order to match the swapped extents — a mismatch between the storage order the kernel assumes and the axis order the wavenumber owner builds applies the wrong axis's symbol to each bin and produces a transposed operator with no error.

[ROUNDTRIP_AND_HERMITIAN_EVIDENCE_AS_TYPED_RECEIPT]:
- A spectral operator owes two pieces of evidence no scalar return carries: the imaginary residual of the inverse transform of a real-symbol operator applied to a real field, which must be at machine zero and whose excess diagnoses a broken Hermitian symmetry from a mis-zeroed Nyquist mode or an odd-symbol leak; and the discarded high-wavenumber energy fraction, which is the share of the spectral power above the dealiasing cutoff and quantifies how much the truncation removed. The first is a correctness receipt — a nonzero imaginary residual means the operator is not self-consistent on the real field and the result is inadmissible — and the second is a resolution receipt the downstream gate reads to decide whether the grid resolves the field. Collapsing both into one operator-evidence carrier alongside the scaling convention used and the Nyquist-handling discriminant gives the consumer one value to pattern-match for admissibility, replacing a silent complex output whose imaginary part the caller must remember to inspect.
- The energy-conservation identity is a free cross-check the operator can record: the spectral power summed over wavenumbers equals the field energy summed over the grid up to the scaling convention, so an operator that preserves or scales energy by a known symbol-dependent factor can verify itself by comparing the two sums, and a mismatch beyond tolerance flags either a scaling-convention error or an aliasing fold. The receipt carries the spatial energy, the spectral energy, and their ratio against the expected symbol-derived factor, so a wrong scaling option surfaces as a ratio off by the grid length rather than as a field that is silently the right shape at the wrong magnitude.
