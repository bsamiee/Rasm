# Execution Standard

The demand level every writing and review stage of a workflow holds, independent of the project and the artifact kind — real code, a design corpus, configuration, durable prose all take the same aggression. Project artifacts (a quality doctrine, capability catalogs, an architecture law) sharpen this standard where they exist; their absence never lowers it. This reference owns the stage-prompt CONTENT bar; topology, dataflow, and handoff integrity are the patterns reference.

## [01]-[STANCE]

Every substantive stage — author, critique, red-team, fixer alike — holds the work hostile:

- The artifact under review was authored by ANOTHER engineer and is naive, shallow, or illusory until it survives a real attack; the burden of proof is on the work, never the reviewer. Dense, confident, idiom-fluent output is the PRIME suspect for hollowness — a name promising capability the body omits, decorative density carrying nothing, a stub dressed as finished work.
- Naivety is a defect on two orthogonal axes, both intolerable. COVERAGE — the owner models a thin slice of its concept: the obvious three fields where the domain carries fifteen, a two-case family for a twenty-case domain. APPROACH — enumerated hardcoded instances standing where one parameterized, algorithmic owner generates the space: a fixed roster of variants is seed DATA feeding one generator over named parameters, never the mechanism itself.
- Every enumerated checklist, collapse-signal list, and attack list in a stage prompt is a FLOOR hunted past, never the complete set.
- NO CHURN: an edit requires a named violated law or invariant plus the concrete case that breaks it; a clean verdict earned by an attack that finds nothing is a first-class result, proven by adding nothing — and a reviewer forced to edit invents defects.

The stance is SUBSTANCE, not register. The burden of proof on the work, both naivety axes, and the cold pass bind every stage regardless of which model executes it, but the PHRASING is model-shaped. A Claude-executed stage carries this hostile register verbatim — Claude reads the intensifiers as calibration. A codex-executed stage (a codex work or review lane) carries the same demands de-conflicted and neutral: the intensifier stacks and hostile-stance paragraphs that sharpen a Claude lane make a codex lane over-use tools and probe out of territory. A workflow routing a stage to codex builds that stage's prompt codex-shaped and never hands codex the native Claude-register builder verbatim (codex-lanes reference, the prompt-shape law).

## [02]-[WRITER_BAR]

The implement stage authors ground-up, never patches:

- New capability is woven into the owning shape as if it had always existed — a case, row, field, operation, or policy value INSIDE an existing owner before any new surface appears; the surface narrows by absorption, never omission. Shims, compat aliases, migration layers, and append-beside are rejected; the API breaks when the collapse improves the system.
- Capability is conserved absolutely: densify, never delete; zero current consumers never lowers the bar, because the owner is shaped for the demand the domain implies. Anticipate the family — the proof of a correct shape is the diff of the next feature: one declaration inside the owner, every consumer untouched or loudly broken.
- Material depth: every admitted dependency is mined to the operator depth it ships; logic below that depth re-derives line-by-line what a deeper composed form states once, and a hand-rolled reimplementation of shipped capability is a defect. A cited external member is verified against its source of truth before it is written; an unverifiable member is never authored.
- Variation lives in input shape, policy values, and table rows — one polymorphic entrypoint per concern discriminating on the value itself, never name-suffix siblings, boolean knobs, or parallel exports. Anything that can vary is a parameterized value, never a hardcode.
- Where the project carries a quality doctrine, the writer reads it AT SOURCE in full and treats it as the floor to push past; where none exists, this register still binds in the language's own strongest idiom.

## [03]-[REVIEWER_BAR]

Critique and red-team are REBUILDERS with the writer's full authority, never annotators:

- COLD PASS first: the reviewer derives its own defect list from the artifact on disk before consulting any prior claim; producer output is strictly a refutation target (handoff law: patterns reference). Scope rows bound where the reviewer looks first — never what it may fix.
- Beyond fixing: a reviewer that only repairs findings underperforms its mandate. Types, constants, classes, and functions sharing a discriminant COLLAPSE into fundamentally stronger owners; thin owners EXTEND to the full domain their concept carries; a fundamentally stronger design, once seen, is BUILT — never defended against. The work ends objectively denser and more capable than the prior stage left it, or the reviewer proves the strongest form is already present by finding nothing.
- Distinct objectives per stage, never two identical reviews. CRITIQUE is predicate-positive: the clause-by-clause conformance and capability-completeness audit — verify each required law holds, cite the clause, repair every miss in place. RED-TEAM is predicate-negative: the pre-mortem — counterfactual on the core owner/algebra/dispatch, diff-of-the-next-feature, long-tail and failure-mode attack (empty/singular/plural/stream/malformed/concurrent/cancelled/partial-failure), boundary and ownership integrity, sprawl and phantom members — then one full cold re-review of every conformance dimension by name.
- Residuals drain forward, never pool: every defect found is fixed at its root in the same pass regardless of scope; deferral is reserved for territory a LIVE sibling owns, and deferred rows drain at the nearest subsequent stage — a terminal mega-reconcile is the priced-in failure of every stage before it.

## [04]-[MAPPING_LANES]

The burden-reduction contract: writers write, cheap lanes read.

- Every substantive writing stage is preceded by cheap-model mapping lanes that absorb the deep reading — full-file reads of the territory, dependency-surface enumeration to operator depth, seam and consumer tracing across the blast radius, current-state census, defect hunting — so the writer arrives with the territory pre-mapped instead of spending its context on reconnaissance.
- The map PRODUCT is a consumer-scoped contract, never a dump: the full dependency-catalog roster of every stratum the territory composes, each classified used / underutilized (unexploited depth in exact members) / unused; a location map anchoring each concern to its owning file and line; seam and ripple endpoints both ends named; entries scoped to their consuming stage so each executor opens a clean slice. Cross-territory evidence rides surgical probes at named symbols, never full foreign reads, under the same verification duty.
- Where a run fans concurrent writers over one corpus, one ideation stage between mapping and writing converts the map into a shared cross-territory worklist: each idea an owner, an admitted member or domain concept, a disk anchor, and why; cross-territory ideas name both ends and their seam; stand-alone ideas are marked. Every writer reads the WHOLE worklist, so concurrent lanes converge instead of drifting into local optima. Ideas are ambition and information, never a prescription or a ceiling — writers spot-verify each anchor, build the strongest form the idea points toward, and still run their own cold pass; a second-pass cull deletes any idea whose anchor fails or that disk already realizes.
- Maps carry INFORMATION, never prescriptions: anchored facts, verified member spellings, seam endpoints, capability the concept admits but nothing exploits. A map entry that tells the writer what to build — a fence sketch, a prescribed shape, a pre-ruled design — anchors and weakens the writer and is the lane's defect; findings name the law and the defect, never the resulting code.
- Delivery is part of the contract: a map produced but not routed is spend without effect. The consuming stage's prompt names the exact report paths, orders the read (grounding maps before defect lists), and instructs full-read, spot-verify, and exceed — the writer never relitigates what a map settled (it verifies anchors, never re-derives the census) and never treats the map as a ceiling.
- One read is never delegated: the writer consumes its quality doctrine at source itself — doctrine conformance cannot ride a cheap lane's summary. Everything else — inventory, seams, member verification, staleness — is mapped for it.

## [05]-[DURABLE_LEARNING]

The loop that converts run findings into standing law, active wherever the repo carries a maintenance-law corpus (a laws router such as `docs/laws/README.md` that owns the corpus admission law):

- Every substantive stage reads the corpus at source — it stays small by its own admission law, so the read is cheap — and an edit touching a topology surface lands the obligated counterparts in the same pass.
- Writer and reviewer schemas carry a required-but-usually-empty `harvest` attestation: nominations of generalizable lessons only — a reusable collapse pattern, an unnamed naivety class, a review rule that catches the defect class before review, a hard-won coupling — each citing the exact existing clause it hardens, or proving absence across the searched surfaces. A stage-local fix never nominates.
- ONE terminal doctrine stage per run adjudicates the pooled nominations refutation-first against the live surfaces under the corpus admission law — harden an existing clause, then extend an existing file, then mint a new file — with the repo's durable-prose skills loaded before any edit. Landing nothing is a first-class verdict; a landed row states law and carries no run provenance; a rule a formatter, analyzer, or gate can enforce routes to that gate owner, never to reviewer prose. The lander prompt ORDERS a read of the corpus router and never restates its routing or bar — a workflow that inlines the routing forks it from the owner and drifts on the next landing.
- Terminal residual drains run as a fixpoint loop: one serial closer per round takes the full pooled residual set, verifies every row against live disk before acting (rows a later stage already resolved are culled with proof), fixes the rest at root, and re-feeds the still-open remainder; a round cap plus a no-shrinkage progress gate bound the loop, and the genuinely blocked remainder — each row naming its blocker and owner — rides the run return.
