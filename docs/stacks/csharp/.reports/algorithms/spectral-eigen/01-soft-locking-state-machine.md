# Soft-Locking Convergence State Machine

[ITERATION_STATE_CARRIER]:
- The block iteration is one state-threaded fold over a single immutable carrier, not a mutating loop. The carrier holds the full eigenvector block `X` (all `k` columns, converged and active alike), the previous-direction block `P`, the per-column residual norms, the active boolean mask, the smallest residual norm seen so far, and the iteration index. Each step is a pure transition `Carrier → Carrier`; the driver is `Schedule.recurs(maxIter)` folding the transition under an `Eff`, with early exit when the active count reaches zero.
- The mask is the discriminant, not a flag tail: `activeMask[j] = residualNorm[j] > tol` recomputed each step and AND-folded into the prior mask so a converged column never re-activates — `mask = mask.Zip(residualNorms, (m, r) => m && r > tol)`. The active count `mask.Count(identity)` is the only quantity that drives block-width contraction; no parallel `bool isLocked` field exists because the discriminant is recoverable from the residual vector itself.
- `X` stays full-width across every iteration; only the *working* triple `[Xₐ | Rₐ | Pₐ]` contracts, where the `ₐ` blocks select columns through `mask`. Active-column extraction is `Matrix<T>.Build.DenseOfColumnVectors(block.EnumerateColumns().Where((_, j) => mask[j]))` — never `RemoveColumn` in a loop, which reallocates the backing array `p` times and is O(n·k²) of pure copying. The full `X` participates in Rayleigh-Ritz substitution at step end so locked columns keep refining; only their residual and direction computation is skipped.
[RESIDUAL_GATE_AND_TERMINATION]:
- Per-column residual is `rⱼ = A·xⱼ − λⱼ·xⱼ` (standard) or `A·xⱼ − λⱼ·B·xⱼ` (generalized), assembled blockwise as `R = A.Multiply(X) − X.Multiply(Λ_diag)` with `Λ_diag = Matrix<T>.Build.DenseDiagonal(k, k, j => ritzValues[j])`, then the column norm `R.Column(j).L2Norm()`. The block form computes all `k` residuals in two matrix multiplies; the per-column `L2Norm` then projects the convergence vector.
- The default tolerance is *absolute* in the column 2-norm: `tol = sqrt(εmach)·n` — scaled by problem dimension, not by eigenvalue magnitude or operator norm. A relative criterion `‖rⱼ‖ / (|λⱼ|·‖xⱼ‖ + ‖A·xⱼ‖)` is the correct domain override for spectra spanning many orders of magnitude, supplied as a policy value on the request, never hardcoded. The gate compares `residualNorms.AbsoluteMaximum()` against `tol`: termination is the *worst* active column clearing the bar, not the mean.
- The aggregate `residualNorm = residualNorms.Sum() / k` is carried only as restart telemetry, distinct from the per-column termination test. Termination has three sealed outcomes: `Converged` (active count reaches zero), `Exhausted` (iteration index reaches the ceiling with active columns remaining — the surviving columns are valid Ritz pairs at their last residual, recorded with that residual as evidence, not discarded), and `Stalled` (the orthonormalization factorization fails before convergence). Each is a receipt case, not an exception.
[P_DEGENERACY_RESTART_FOLD]:
- As `xⁱ` and `xⁱ⁻¹` approach linear dependence near convergence, the previous-direction block `P` (which spans the same subspace as `[Xⁱ | Xⁱ⁻¹]` without the precision loss of holding `Xⁱ⁻¹` directly) becomes ill-conditioned and the `[Xₐ | Rₐ | Pₐ]` Gram pencil loses rank. The transition catches the dense generalized-eigensolve failure on the `3k`-wide pencil and *folds to a narrower pencil*: drop `P` entirely and re-solve on `[Xₐ | Rₐ]` (`2k`-wide). This is the same Rayleigh-Ritz with `Pₐ` columns elided — one expression, two arities, dispatched on the prior solve's outcome.

```csharp
// One Rayleigh-Ritz body, two arities: the 3k pencil, folding to the leading 2k principal
// block of the same Gram pair when the wide pencil loses rank.
static (Vector<double> Values, Matrix<double> Vectors) RayleighRitz(
    Matrix<double> gramA, Matrix<double> gramB, int narrow) =>
    new Try<(Vector<double>, Matrix<double>)>(() =>
    {
        var e = Reduce(gramA, gramB).Evd(Symmetricity.Symmetric);
        return Fin.Succ((Vector<double>.Build.DenseOfEnumerable(
            e.EigenValues.Select(c => c.Real)), e.EigenVectors));
    }).Run()
      .IfFail(_ => RayleighRitz(gramA.SubMatrix(0, narrow, 0, narrow),   // drop Pₐ: same solve,
                                gramB.SubMatrix(0, narrow, 0, narrow), narrow)); // 2k arity
```

- A *forced* restart fires independent of factorization failure: when the aggregate residual rebounds past `2^restartControl · smallestResidualNorm` (the smallest aggregate residual ever observed, threaded in the carrier; `restartControl` defaults to `20`, so the trigger is a `2²⁰ ≈ 10⁶` rebound), the stale `P` is poisoning the search direction and is discarded for the next step. `smallestResidualNorm` updates as `Math.Min(prior, residualNorm)` each step — a monotone carrier field whose only purpose is this rebound predicate. The two restart paths (Gram-rank failure, residual rebound) collapse to one `dropP` boolean on the carrier consumed by the next transition's subspace assembler.
- On the first iteration `P` is empty by construction, so the subspace is `[Xₐ | Rₐ]` unconditionally — the same `2k` path the restart fold reaches, proving the narrow form is not a special case but the base of the recurrence. The `dropP`-true and iteration-zero branches share one assembler; the only difference is whether `Pₐ` columns are appended.
[B_ORTHONORMALIZATION_AND_RECONSTRUCTION]:
- Each working block is B-orthonormalized by the Cholesky route `V ← V·(Vᴴ·B·V)^{-1/2}`: form the Gram `Vᴴ·B·V` via `V.TransposeThisAndMultiply(B.Multiply(V))`, factor `chol = gram.Cholesky()`, and apply `V·R⁻¹` where `R = Lᵀ` is the upper factor. Because `V·R⁻¹ = (R⁻ᵀ·Vᵀ)ᵀ = (L⁻¹·Vᵀ)ᵀ`, the realization solves through the *lower* factor: `chol.Factor.LU().Solve(V.Transpose()).Transpose()`. Routing through `chol.Factor.Transpose().LU().Solve(V.Transpose()).Transpose()` instead yields `V·L⁻¹` — the wrong half-factor, whose result is not B-orthonormal (`(V·L⁻¹)ᵀ·B·(V·L⁻¹) = L⁻ᵀ·G·L⁻¹ ≠ I`), and the defect is silent because the shapes agree. A Cholesky throw signals the working block is rank-deficient — for `Rₐ` this means every active residual collapsed (true convergence reached mid-step); for `Pₐ` it is the degeneracy that triggers the `P`-drop restart. The two failures route to different receipt outcomes despite the identical exception type, tagged at the catch site before propagation.
- The Rayleigh-Ritz pencil partitions the stacked Ritz eigenvector by block width to reconstruct the next `X` and `P`. With the top `k` Ritz pairs selected, the eigenvector matrix `C` splits row-wise as `C_X = C[0:k]`, `C_R = C[k:k+p]`, `C_P = C[k+p:]` where `p` is the surviving active count. The new direction is `P⁺ = Rₐ·C_R + Pₐ·C_P` — the search-space component *orthogonal to the current X span* — and the new block is `X⁺ = X·C_X + P⁺`. `P⁺` is reused directly as next-iteration `Pₐ` after its own B-orthonormalization; it is never recomputed from `Xⁱ⁻¹`, which is the precision-preserving substitution that defines the method.
- Hard locking (freezing converged columns as immutable constraints and orthogonalizing the active block against them) is cheaper but caps accuracy: a column locked at residual `tol` never refines further, so its error floor is `tol` permanently. Soft locking keeps converged columns in the Rayleigh-Ritz substitution, so their accuracy improves as the active subspace sharpens — the converged eigenpairs ride the later iterations' refinement for free. The receipt records which discipline produced each pair because the per-column error floor differs by construction: hard-locked pairs carry `tol` as a hard ceiling; soft-locked pairs carry their final post-substitution residual.
[FROZEN_CONSTRAINT_DEFLATION]:
- An externally supplied invariant subspace `Y` (already-known eigenvectors, or the nullspace constant `1/√n` vector for a graph Laplacian) deflates by B-orthogonal projection applied to *every* working block before each Rayleigh-Ritz: `V ← V − Y·(Yᴴ·B·V)`. This is `V − Y.Multiply(Y.TransposeThisAndMultiply(B.Multiply(V)))` for the generalized case, collapsing to `V − Y·(Yᵀ·V)` when `B = I`. The projection is idempotent only if `Y` is itself B-orthonormal, so `Y` is B-orthonormalized once at setup and the per-step projection assumes that invariant.
- Constraint deflation and soft locking compose but are distinct axes: constraints are *external* known directions the solver must avoid (search outside their span); soft locking is *internal* convergence the solver discovered. Conflating them — re-projecting converged internal columns as if external constraints — re-introduces the hard-locking accuracy ceiling by forbidding their refinement. The carrier therefore holds `Y` (frozen constraints, projected out of `R` and `P`) separately from `mask` (soft locks, retained in the substitution).
- For a clustered near-zero spectrum the constant nullspace vector seeded as a constraint removes the trivial `λ₀ = 0` mode from the search, so the active block converges on the spectral gap above it rather than wasting iterations resolving the known zero — the iteration count drops by exactly the number of frozen-constraint columns since each removes one direction the `2k`/`3k` pencil would otherwise spend rank on.
