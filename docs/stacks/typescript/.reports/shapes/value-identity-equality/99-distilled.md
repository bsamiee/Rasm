# Value Identity And Equality Distilled

[STRUCTURAL_FUSION_OWNER]:
- `class X extends Data.Class<{...}>` / `Data.TaggedClass(tag)<{...}>` is simultaneously type, value-constructor (`new X({...})`), and `Equal`/`Hash` instance — no `type X = ...` and no separate comparator ever follows; `Schema.Class<X>("X")({...})` extends `data_.Class` internally, fusing codec, static type, validating `make`/`new`, and the same `StructuralPrototype` identity into the deepest single-source owner.
- `Data.struct`/`Data.tuple`/`Data.array` reshape native objects/arrays into `[Equal.symbol]`+`[Hash.symbol]` carriers by construction; `Data.unsafeStruct` is `Object.setPrototypeOf(existing, StructuralPrototype)` — the one entry that grants structural identity to a foreign-constructed record in place without reallocating it.
- `Data.case<A>()`/`Data.tagged<A>(tag)` build a constructor whose argument is `VoidIfEmpty<A>`, so an empty-field case constructs with no argument, not `{}` — `Data.TaggedClass(tag)` strips `_tag` from the constructor argument (`P extends "_tag" ? never : P`) and stamps it once at declaration, recoverable from the value for downstream `Match`.
- Three prototypes fix the identity tier permanently: `StructuralPrototype` (`Object.keys` equality + `Hash.structure`) backs `Data.struct`/`Data.case`/`Data.Class`/`Schema.Class`; `ArrayProto` (index-wise `Equal` + `Hash.array`) backs `Data.tuple`/`Data.array`; `EffectPrototype` (`this === that` + `Hash.random`) backs every `Effect`/service/`Stream` — the structural-versus-reference choice is the prototype, never selectable at the comparison site, so a `Layer` in a `HashSet` deduplicates by reference for its lifetime and is never content-collapsed.
- `Schema.Data(schema)` lifts a struct/array schema so its decoded output carries `[Equal.symbol]`/`[Hash.symbol]`, making boundary-admitted records `HashSet`/`HashMap`-key-correct by construction — decode and structural-identity stamping fuse, so interior code drops every value into a hashed structure with no comparator.

[THE_DISPATCH_GATE]:
- `Equal.equals(self, that)` runs a fixed sequence before any owner code: `self === that` wins first; a `typeof` mismatch fails immediately; only the `object`/`function` arm with both operands non-null and both `Equal.isEqual` reaches owner logic — a `Data` value against any non-`Equal` operand never enters the structural branch and falls to `false`, so identity fusion is required on BOTH operands, never inferred from one.
- The owner-logic arm is `Hash.hash(self) === Hash.hash(that) && self[Equal.symbol](that)` — the hash is a precondition consulted BEFORE the comparator, so a hash disagreeing with equality short-circuits to inequality without invoking it; equal-values-hash-equal is the literal gate that runs first, not advisory. Hash equality is necessary, never sufficient — collisions still call the comparator, so a fast-wrong `Equal` is a correctness bug a passing hash will not surface.
- `Date` (by `getTime()`, NaN admitted equal via `Number.isNaN(t1) && Number.isNaN(t2)`) and `URL` (by `href`) are recognized inline after the `Equal`-pair branch fails — these native carriers get value identity without a `Data` wrapper, and a `Date` field nested in a `Data.struct` recurses into this arm rather than degrading to reference.
- Struct equality keys off `Object.keys(this)` vs `Object.keys(that)` (own enumerable only): `selfKeys.length !== thatKeys.length` fails fast then each self-key must satisfy `key in that && Equal.equals(this[key], that[key])` — bijective on the key set, so a superset `that` is unequal and a non-enumerable/prototype-inherited field is invisible to both equality and `Hash.structure`. `ArrayProto` is a strictly different algorithm — `Array.isArray(that) && length` then positional `every` — so a tuple is never equal to a struct even when fields align, the two prototypes never sharing a comparison body.
- The co-constraint is a STRUCTURAL TYPE OBLIGATION, not a runtime convention: the `Equal` interface `extends Hash.Hash`, so `[Equal.symbol]` cannot be declared without `[Hash.symbol]` — a value carrying a custom `Equal` is statically forced to carry a `Hash`, and the only break the type system cannot catch is supplying a `Hash` that disagrees with the `Equal`. The equality half bridges to the combinator surface (`Equal.equivalence<A>()` lifts `[Equal.symbol]` into the `Equivalence` every default-identity verb reaches for) but the ordering half does NOT: `Order` does not bridge to `Equal` and `Equal` derives no `Order`, so the comparison and equality halves of identity are independently sourced and only the owner keeps them coherent — the mechanistic root of every "ordering does not self-propagate" hazard downstream.

[HASH_OWNER_AND_MEMOIZATION_HAZARD]:
- `Hash.string` is reversed djb2 (`5381`, `h*33 ^ charCodeAt(--i)`); `Hash.combine(b)(self)` is `self*53 ^ b` (NON-commutative, order load-bearing); `Hash.array` folds `combine` from a `6151` seed (index-sensitive); `Hash.structureKeys(o, keys)` seeds `12289` and XOR-accumulates `string(key) combined-with hash(o[key])` per named subset key (XOR makes contributions ORDER-INDEPENDENT, matching `Object.keys` equality); `Hash.optimize` (`n & 0xbfffffff | n >>> 1 & 0x40000000`) clamps into the V8 SMI range — each prototype pairs a hash whose algebraic symmetry exactly matches its comparator's (XOR/order-independent for keyed, fold/order-sensitive for positional), so equal-implies-hash-equal holds for free per tier and a hand-rolled `[Hash.symbol]` swapping the symmetry re-introduces scatter.
- `Hash.number` returns `0` for `NaN`/`Infinity` (distinct non-finite fields collide, fall to the comparator); `bigint` routes through `toString(10)`, `Date` through ISO string — identity-bearing numeric edge values hash deterministically rather than by reference.
- `Hash.cached(self, h)` installs `[Hash.symbol]` as a NON-ENUMERABLE own property, surviving every `Object.keys` walk — the first `Hash.hash` pays, repeats read the frozen value; a custom `[Hash.symbol]` MUST call `Hash.cached` or every `HashMap` `get`/`set` re-hashes the key's full structure.
- Mutating a `Data` value's fields after first `Hash.hash` desyncs the memo: the frozen `[Hash.symbol]` returns the pre-mutation hash while structural equality reads new fields, so `Hash.hash === Hash.hash` fails on values the comparator calls equal — but reflexivity survives because the `compareBoth` gate's `self === that` precedes the hash precondition, so a value still compares EQUAL to itself; only the distinct-copy comparison surfaces the hazard. Immutability is the precondition the memo assumes; `unsafeStruct` over a later-mutated object is the trap.
- `Hash.random(self)` memoizes one integer per object reference in a global `WeakMap`, so every non-`Hash` object and `EffectPrototype` carrier gets a STABLE per-reference hash dying with the reference — reference identity with a fast bucket, never content.

[TAGGED_UNION_AS_ONE_OWNER]:
- `Data.taggedEnum<Union>()` fuses four surfaces the declaration-chain would spell separately: the per-tag constructor map, the `$is` refinement guard, the `$match` exhaustive dispatcher, and structural `Equal`/`Hash` per instance — the owner is a `Proxy` whose `get(tag)` returns `Predicate.isTagged` for `"$is"`, the `taggedMatch` thunk for `"$match"`, and `tagged(tag)` otherwise, so adding a member is one type-level arm with zero new runtime surface; identity is not added to the union, it is what the members ARE.
- `_tag` is one symbol driving three scopes coherently: the dispatch key `$match` indexes (`cases[value._tag](value)`, exhaustiveness via `Cases & {[K in Exclude<...>]: never}`), the narrowing key `$is` tests, and an own-enumerable field the `StructuralPrototype` equality compares — so cross-tag values are never `Equal.equals` and never reach the same `$match` arm, keeping equality scope and dispatch scope the same partition.
- `$is(tag)` is `Predicate.isTagged(tag)` — a `_tag`-ONLY guard inspecting the discriminant, never the payload — so a right-tag value with a malformed payload narrows positively: it answers membership-in-the-arm, not full-value validity; carries both curried and data-first overloads for `Array.filter`/`partition` without a wrapper lambda.
- `Data.taggedEnum<Z extends WithGenerics<N>>()` (N=1..4) parameterizes at the type level ONLY — the runtime `Proxy` erases every generic, so `Shape<number>` and `Shape<string>` members share one `StructuralPrototype`, one `Object.keys` equality, one `Hash.structure`; two same-tag members are `Equal.equals` exactly when payloads are, the phantom irrelevant. The generic `$is` carries `<T extends Kind<Z, any, any, any, any>>(u: T) => u is T & {_tag: Tag}`, INTERSECTING the matched tag onto the input rather than collapsing to `Extract`, preserving inferred type arguments the bare overload would discard.

[EQUIVALENCE_AND_ORDER_INSTANCE_ALGEBRA]:
- `Equivalence.make(f)` is `(self, that) => self === that || f(self, that)` and `Order.make(c)` is `(self, that) => self === that ? 0 : c(self, that)` — the reference gate is the constructor's outer layer, not a body the author writes, so EVERY combinator (`combine`/`mapInput`/`product`/`tuple`/`array`/`struct`/`reverse`/`empty`) returns through `make` and the gate is multiplicatively present: a `struct` of five `mapInput` leaves is six nested gates, the aggregate firing first on reference-identical operands and collapsing the fold before any field reads, inner gates re-testing sub-values so partial structural sharing skips a shared field while comparing the rest.
- The gate runs the body on zero inputs for `f(x,x)`, so it discharges reflexivity/antisymmetry-corner structurally and the body owns ONLY the distinct-pair comparison — a hand-rolled `self === that ? 0 : ...` head is dead code the constructor already short-circuited, and a NON-reflexive body (returns `false`/non-`0` for some `f(x,x)`) is masked until two structurally-equal-reference-distinct copies meet; a property test feeding `f(x,x)` proves nothing about the body.
- The construction regime decides whether the gate is load-bearing: `Equivalence` leaves are `strict()` = bare `===` (no `make`, no second body to short-circuit) while `Order` leaves are `make((self, that) => self < that ? -1 : 1)` — a body that never returns `0`, so the ONLY path to an order tie is the gate; the gate is exactly `===`, under-eager on `NaN` (`O(n,n)` skips the gate, body returns `1`, so `NaN > NaN` and antisymmetry fails on the diagonal — `NaN` is a `SortedSet` structural sink) and over-eager on signed zero (`+0 === -0`, so a `1/x` distinction is unreachable for that pair).
- `mapInput(O, f)`'s gate is on UN-projected operands, so `f` never runs on `self === that`: a coarse projection built to merge many distinct values reports `true` on `x === x` via reference not projection — finest exactly where coarseness is least needed, the discriminating power observable only across distinct references.

```typescript
import { Equivalence as Eq, Order } from 'effect';

type Span = { readonly lo: number; readonly hi: number };

const byWidth: Order.Order<Span> = Order.mapInput(Order.number, (s: Span) => s.hi - s.lo);

// reject: the reflexive arm is dead — make's outer gate returned 0 before this body runs
const handRolled: Order.Order<Span> = Order.make((self, that) =>
    self === that ? 0 : Order.number(self.hi - self.lo, that.hi - that.lo),
);
```

[MONOIDAL_PRECEDENCE_ONE_FOLD_TWO_CODOMAINS]:
- `combine` is one left-biased fold differing only in the codomain's absorbing element: order is `out !== 0 ? out : that(a1, a2)`, equality is `self(x, y) && that(x, y)` — `0` and `true` play the identical "left inconclusive, defer right" role, so a multi-key `Order` and the matching multi-field `Equivalence` derive from the IDENTICAL projection sequence; adding a tiebreak key and adding an identity field is one `combine` arm at both surfaces.
- Declaration order IS evaluation order IS a cost decision, but resolves oppositely per codomain: the `Order` exits when a key DISAGREES (`out !== 0`, so highest-cardinality key leftmost), the `Equivalence` exits when a field DISAGREES (`!eq`, so most-volatile field leftmost) — never one field-priority order shared blindly. Reordering preserves the partition into classes but not the representative (which key breaks the tie, which payload survives an extremum), so a cost reorder is verdict-safe for membership and representative-unsafe — licensed only when no later combinator reads the resolved tiebreaker.
- `Order.reverse(O)` is `make((self, that) => O(that, self))` — argument swap, not codomain sign-flip — and because each arm carries its own instance, reversing ONE arm flips that key's direction alone while preserving rank: descending-by-rank-then-ascending-by-code is two cells, never `(a, b) => b.rank - a.rank || a.code.localeCompare`. The equality half has no `reverse` (conjunction is symmetric, equivalence is its own dual) and withholds `empty` while `Order` publishes both — the order arm sequence encodes precedence AND direction, the equality sequence only membership, the one axis ordering owns and equality lacks; a hand-written `||`-chain over subtractions/`localeCompare` restates the leftmost-non-zero fold, hard-codes each direction, and truncates the codomain to whatever `-` returns rather than `-1 | 0 | 1`, the spelling `combineAll` over projected leaves deletes.
- The empty fold diverges structurally: `Order.combineAll([])` seeds `Order.empty<A>()` = `make(() => 0)` (a STABLE total tie) while `Equivalence.combineAll([])` seeds the unexposed `isAlwaysEquivalent` (the EVERYTHING-EQUAL instance, withheld because it is a correctness hazard not a default) — so a runtime-selected key list with zero keys is a benign no-op for sorting and a silent total merge for equality, and the static `Struct.getOrder`/`getEquivalence` table versus the runtime variadic `combineAll` over an assembled `ReadonlyArray<Order<A>>` are the SAME fold at two binding times deciding whether the identity grain is a compile-time owner property (the checker chases a field addition into a missing row) or a runtime policy value.
- `Order.combine` and the value-selecting `Monoid.max(Bounded)` are the same left-biased algebra at two altitudes: the codomain fold decides WHICH key breaks the tie, the value fold reads that verdict to decide WHICH value wins, so a tiebreak appended as one `combine` arm reshapes the extremum with zero fold-site edits; `Monoid.min/max(B)`'s `empty` is the OPPOSITE bound (min's identity is `maxBound`) so extremum-of-empty is total, the order/equality asymmetry surfacing one level higher because the equality `Semigroup` carries no analogous upward lift. `Order.min/max(O)` keep the FIRST argument on a `0` tie (`O(self, that) < 1 ? self : that`) while `Semigroup.min/max(O)` keep the LAST (`=== -1 ? self : that`), so folding one `Order` through the two surfaces over equal-comparing distinct values returns different representatives — observable whenever the `Order` is coarser than structural `Equal`, an extremum fold over a projected order non-deterministic in payload unless the retention surface is chosen deliberately. `Bounded.reverse` is the only law-preserving inversion: it reverses `compare` AND swaps `maxBound ↔ minBound` in lockstep, so reversing only the inner `Order` without swapping bounds seeds `combineAll([])` with the wrong identity and the empty fold disagrees with every non-empty one.

```typescript
import { Equivalence as Eq, Order } from 'effect';

type Row = { readonly tier: number; readonly code: string; readonly stamp: number };

// one projection sequence, two codomains — adding a key/field is one combine arm at both;
// reverse owns the one descending cell, the codomain stays -1 | 0 | 1
const precedence: Order.Order<Row> = Order.combineAll([
    Order.reverse(Order.mapInput(Order.number, (r: Row) => r.tier)),
    Order.mapInput(Order.string, (r: Row) => r.code),
]);
const sameClass: Eq.Equivalence<Row> = Eq.combineAll([
    Eq.mapInput(Eq.number, (r: Row) => r.tier),
    Eq.mapInput(Eq.string, (r: Row) => r.code),
]);
```

[POSITIONAL_ARITY_BLINDNESS]:
- One private `all(collection)` fold backs every positional aggregate except one — it walks `Math.min(x.length, y.length)` positions and returns the prefix verdict (`true`/`0`) the moment the shorter operand exhausts, surplus tail never read. `tuple(...elements)` is literally `all(elements)`, inheriting the truncation, so a tuple comparator admits any prefix-agreeing operand; for equality this is a prefix-equal trap, for order a prefix-tie that merges a sequence with its tied prefixes into one sorted entry.
- `array(item)` is the SOLE positional aggregate not routing through `all`: equality `array` opens with `self.length !== that.length` early-`false`, order `array` walks `Math.min` then closes with `number(aLen, bLen)` — lexicographic with length as terminal tiebreak. The split is the algebra each shape models: a tuple is a product of fixed slots whose arity the type presumes discharged, an array a list whose length is part of its value, so the array fold admits length and the tuple fold cannot.
- `Equivalence.struct` reads arity from `Object.keys(fields)` — the DECLARED instance record captured at construction, never the operands — the third regime: struct fixes arity at the instance, tuple presumes it from the type and truncates at runtime, array reads it from the values. One concept, three answers to "same shape", none recoverable by inspecting operands; folding through the struct surface is the arity-safe collapse, breaking loudly on a renamed/dropped key at the `keyof`-anchored declaration where the positional tuple silently re-admits prefixes.
- The truncating equality is a `HashSet` membership defect: a stored short tuple and a probe extending it compare `true` under `tuple`/`all`, yet `Hash.array` folds over ALL elements so the surplus shifts the bucket and `Hash.hash === Hash.hash` fails before the comparator — the equal-implies-hash-equal law broken by a length-blind equality paired with a length-sensitive hash. A `HashSet` of variable-length sequences MUST key on `Equivalence.array`, never `tuple`; `Order.array`'s `number(aLen, bLen)` tiebreak makes a `SortedSet` key TOTAL over unequal lengths while `Order.tuple` ties them at the prefix into one entry, so a structure's cardinality reads directly off which form its captured comparator was. `productMany`'s blindness is to the ZERO-arity edge instead: `self(x[0], y[0])` dereferences `x[0]` to `undefined` on a wrongly-admitted empty array.
- The hand-built combinator and the schema derivation impose OPPOSITE arity policies behind one surface name: `Equivalence.tuple` is `all` (no length read), while the `Schema.equivalence` `TupleType` arm opens with `len !== b.length` returning `false` then partitions into three zones — fixed `elements` by index, a `rest` head across the middle span, post-rest `tail` aligned from the end — so deriving from a `Schema.Tuple` yields length-SENSITIVE identity and composing from positional instances yields length-BLIND, the same `Math.min` meaning truncation in `all` and pure zone-partition after the guard. Composing a sort key by hand from `Order.tuple` and an equality by derivation from the schema yields two surfaces disagreeing on arity for one owner — the coherence hazard mirroring the ordering-derivation gap, the one surface that does not self-propagate.

[SCHEMA_DERIVATION_HOOK_AND_TOTALITY]:
- `Schema.equivalence(schema)` walks the AST consulting each node's `equivalence` hook FIRST via `getAnnotation(key)` = `hasOwnProperty.call(ast.annotations, key)` — an OWN-property read on exactly this node — so a hook is a total replacement returning immediately before the structural `_tag` switch, strictly node-local: an outer hook wins its whole subtree, an inner one wins only what its parent delegates down, and cross-node conflict-freedom is STRUCTURAL — the only "conflict" is nesting, and nesting is total dominance, never precedence to author.
- A SECOND `_tag` switch inside the `isSome` branch sets the hook's call shape: a `Declaration` hook is N-ary `hook(...typeParameters.map(tp => go(tp, path)))` over fully-derived parameter equivalences, a `Refinement` hook is unary `hook(go(ast.from, path))` over the base, every other node nullary `hook()` — the arity is the literal call the walk emits keyed on the node, so a generic owner's identity is itself parameterized by its members' identities. The typed-arity WALL lives ONLY on the parameterized `Schema.declare(typeParameters, options, annotations)` (`EquivalenceAnnotation<A, TypeParameters>` tuple-coupled to `typeParameters`); the universal `.annotations()` erases it to `(..._: any)`, so a generic owner needing parameter-keyed identity MUST go through `declare`.
- `NeverKeyword` is the SOLE throwing node — at `Schema.equivalence` BUILD time, embedding `at path: ${formatPath(path)}` (the bracketed route the structural descent `path.concat`ed) and `schema (${ast._tag})` — so a degenerate field is caught when the comparator is built, not when values meet, and derivation safety is a property of the type's inhabitedness checked once: every well-formed owner short of a `Never`-decoded field yields a terminating comparator regardless of nesting or cycles. The boundary is LOOSER than Arbitrary's (which `absurd`s a bare `Declaration` too) — comparing needs a value in hand, generating needs a recipe.
- `Suspend` is doubly memoized — the node's own `this.f = memoizeThunk(f)` plus the walk's `memoizeThunk(() => go(ast.f(), path))` returning `(a, b) => get()(a, b)` — keyed by NODE instance not value depth, so the comparator is a build-time property of the cycle's single suspension point applied at constant cost to unbounded depth. The deferred build INVERTS the totality boundary for a suspended subtree: a `Suspend` wrapping an uninhabited node reports total at build and throws on the first comparison forcing the thunk — the one place the `NeverKeyword` guard slips past construction.
- Absent a hook a broad leaf set folds to `Equal.equals` (`Declaration`, all keyword nodes, `Enums`, `ObjectKeyword`, an empty `TypeLiteral`) — the latent reference trap the throw does NOT catch: `UnknownKeyword`/`AnyKeyword`/a hookless `Declaration` build successfully yet compare by REFERENCE, defeating every `Array.dedupe`/`HashSet` downstream, so the build's silence on a degrade node is the signal to supply the hook. `Refinement` is the canonical injection seam — the unary hook receives `go(ast.from)` so a custom identity (tolerance-banded number, case-folded string) is a strictly-finer/coarser WRAPPER on the base, the one node where the hook modifies its child rather than replacing the subtree; the predicate is otherwise invisible (a decode-time admission gate, never an identity lever), so two refined values are equal iff their bases are.
- The structural arms make the derived instance STRICTLY FINER than a `mapInput` over shared fields: `TypeLiteral` fuses presence-parity into identity — an optional property (`isOptional: true`) returns `false` on `aHas !== bHas`, both-omit skips, both-carry runs the field equivalence — while `Union` routes both operands by discriminant through `ParseResult.getSearchTree` and requires the SAME member's refinement before its equivalence is consulted, so cross-member values are never equal even when shared fields match; a non-discriminable member falls into the `otherwise` bucket tried against every operand, and index signatures partition keys by encoded parameter kind demanding equal key-count per kind. A `Transformation` derives from its `to` (decoded) side uniformly with `arbitrary`/`pretty`, so the decoded type is the canonical subject of every derived capability — and `mapInput` over identity fields is the deliberate COARSENING that re-admits the cross-member and presence-mismatched equality the derived instance rejects.

```typescript
import { Equivalence as Eq, ParseResult, Schema as S } from 'effect';

// N-ary Declaration hook: receives each fully-derived element equivalence, returns the container's —
// arity checked because this is the parameterized declare channel, not the erased .annotations()
const Boxed = <A, I>(item: S.Schema<A, I>): S.declare<ReadonlyArray<A>, ReadonlyArray<I>, readonly [S.Schema<A, I>]> =>
    S.declare(
        [item],
        {
            decode: (it) => (input, opts) => ParseResult.decodeUnknown(S.Array(it))(input, opts),
            encode: (it) => (input, opts) => ParseResult.encodeUnknown(S.Array(it))(input, opts),
        },
        { equivalence: (elem: Eq.Equivalence<A>) => Eq.make((x, y) => x.length === y.length && x.every((v, i) => elem(v, y[i]))) },
    );

const eq: Eq.Equivalence<ReadonlyArray<string>> = S.equivalence(Boxed(S.String));
```

[KEYED_IDENTITY_POLICY_TABLE]:
- The per-field identity decision is one `satisfies`-anchored record the owner declares once, not N scattered closures: `Struct.getEquivalence(table)` (alias of `Equivalence.struct`) and `Struct.getOrder(table)` FOLD a field-keyed record of bare leaf instances into one aggregate, inferring the owner type, the projection being the fold's job — so rows hold `Order.number`/`Eq.string`, not `mapInput` closures, collapsing the N closures the manual spelling spreads across use sites. The parallel `ReadonlyArray<keyof Owner>` drives the hash via `Hash.structureKeys`.
- A field present in the owner but absent from the table is silently narrowed away; `satisfies { readonly [K in keyof Owner]: Equivalence<Owner[K]> }` converts the omission into a key error so adding a field forces a row — "compared / ignored" is row PRESENCE, an incidental field excluded by OMISSION never a negative filter, the same decision `Hash.structureKeys`'s key list encodes for the hash half. Because the `satisfies`-anchored equivalence table and the `keyof typeof` hash array index the same literal key set, equal-implies-hash-equal holds by anchor rather than audit — a field promoted to identity is one row plus one array entry forced at the same `keyof` site.
- Reversing one ordering row is `Order.reverse(leaf)` inside the cell — descending-by-one-field is two cells, never a sign-flipping comparator — and the equality table has no `reverse` because equivalence is its own dual, so the two tables diverge exactly on the axis ordering owns. `Struct.keys`/`entries` exclude symbol keys at the type level, so a symbol-keyed field is invisible to a `Struct.keys`-derived policy exactly as to `Object.keys` equality; a renamed field breaks all three surfaces at the same `keyof` site rather than passing one and silently degrading the others.
- `Equal.equivalence<Owner>()` is the NULLARY bridge wrapping the owner's structural `[Equal.symbol]` — the maximal table where every field is a row, returning `equals` itself (not a `make`-wrapped instance) so it inherits the `compareBoth` gate that precedes the hash precondition, never a doubled layer. The projected `Struct.getEquivalence(subset)` is the deliberate coarsening; a table worth declaring is STRICTLY coarser (omits a field) or STRICTLY finer (an explicit row for an opaque field the structural recursion would compare by reference), a table equal to the structural identity being the nullary bridge spelled long.
- No `Schema.order` and no `Order` annotation exist — `Equivalence` derives automatically from the AST, `Order` does NOT — so the three delivery channels off one owner are the nullary bridge, the auto-derived `Schema.equivalence`, and a `Order.struct`/`Order.mapInput` composed BY HAND (two automatic reads plus one manual read of the same declaration). A field added to the schema widens the codec and derived equivalence silently and correctly, but the hand-composed `Order` does not see it — invisible to sort precedence and sorted-membership until `Order.struct` is extended, the one place the owner does not self-propagate exactly where a missed edit merges distinct keys under a sort while equality and hashing already saw it. `satisfies` over the schema's literal `keyof typeof` field record is the single discipline making all three break loudly together, surfacing the omission as a key error at declaration rather than a silent merge.

```typescript
import { Equal, Equivalence as Eq, Hash, Order, Struct } from 'effect';

type Row = { readonly code: string; readonly weight: number; readonly span: number; readonly note: string };

const Identity = { code: Eq.string, weight: Eq.number } satisfies { readonly [K in 'code' | 'weight']: Eq.Equivalence<Row[K]> };
const Ranking = { weight: Order.reverse(Order.number), code: Order.string } satisfies { readonly [K in 'code' | 'weight']: Order.Order<Row[K]> };
const idKeys = ['code', 'weight'] as const satisfies ReadonlyArray<keyof Row>;

const sameId: Eq.Equivalence<Pick<Row, 'code' | 'weight'>> = Struct.getEquivalence(Identity);
const ranked: Order.Order<Pick<Row, 'code' | 'weight'>> = Struct.getOrder(Ranking);
const idHash = (r: Row): number => Hash.structureKeys(r, idKeys);
const fullId: Eq.Equivalence<Row> = Equal.equivalence<Row>();
```

[HAMT_PHYSICAL_LAYER]:
- `SIZE = 5`, `MASK = 31`, `hashFragment(shift, hash) = hash >>> shift & MASK` slices one 5-bit window per level (low first), so a bucket route is a base-32 numeral of the hash and the hash's job is to spread LOW bits; `toBitmap(frag) = 1 << frag`, `fromBitmap(mask, bit) = popcount(mask & (bit - 1))` recovers the dense index — a 32-way branch costs one mask integer plus a packed array sized to live entries. Routing is purely structural and never consults `Equal.equals`: `IndexedNode` descent is `popcount` + array index per level, the comparator fired exactly once at the terminal leaf.
- The bedrock gate's hash precondition is discharged STRUCTURALLY for the single-leaf case: a `LeafNode` carries its insertion `hash` but `getHash`'s leaf arm does NOT re-check `hash === node.hash` (the path already proved fragments matched), re-asserting it only inside `CollisionNode`. A fast-wrong `Equal` is a MEMBERSHIP bug confined to leaf/collision arms; a fast-wrong `Hash` is a ROUTING bug — the probe descends a different fragment path and lands on `EmptyNode`/`none()` without the comparator running, so the value is unreachable rather than miscompared. The bedrock scatter-vs-over-collide pair is exactly these: a hash WIDER than its equality scatters equal values across buckets (correctness break, reports absent), NARROWER packs distinct keys into one `CollisionNode` paying a linear `Equal` scan per touch (complexity break) — the safe grain is a hash exactly as discriminating as the equality.
- Seven `hashFragment` windows address the full 32 bits — six clean 5-bit branches over `0,5,...,25` plus a degenerate 4-way window at `shift = 30` reading only the top two bits — and `Hash.optimize` preserves bit 31 (duplicating into 30), not a non-negativity clamp, so every bit is genuine routing entropy via unsigned `>>>`; discriminating power in bits 30-31 fragments four-way from depth seven alone while the same power in bits 0-29 branches 32-way, so spread the entropy LOW. `mergeLeavesInner` extends the path only while fragments AGREE and builds a two-child `IndexedNode` at the divergence bit (children ordered `subH1 < subH2 ? [n1, n2] : [n2, n1]`), so trie depth is the shared fragment prefix plus one and two `HashMap`s built by different insertion sequences are node-level structurally equal; the ONLY `CollisionNode` source is full-32-bit-hash equality (`h1 === h2`), so collisions are a hash-space property fixed at `optimize` time, never a depth the trie descends out of, near-certain past tens of thousands of keys by the birthday bound — survived by a discriminant-first comparator that fails on the first field, never one walking every key before the discriminating one.
- The trie swaps representation by OCCUPANCY not key count: `IndexedNode` promotes to dense `ArrayNode` at `MAX_INDEX_NODE = 16` and packs back at `MIN_ARRAY_NODE = 8` (hysteresis against thrashing), and `ArrayNode` lookup is `children[hashFragment(...)]` — no `popcount`, a single direct read — so a hot dense level is FASTER per step than a sparse one, inverting "fuller is slower"; removal continuously re-flattens (a two-child `IndexedNode` collapses to its bare sibling leaf, a single-survivor `CollisionNode` to a `LeafNode`) so depth tracks LIVE contention, never historical maximum. The `structuralRegion` all-zero hash collapses every key to fragment `0` at every level, degenerating the trie to one linear-`Equal` bucket — but `has` re-hashes the PROBE to `0` so a region lookup against an out-of-region trie mis-routes to `EmptyNode`, which is why interior domain code never enters it.

```typescript
import { Data, Hash, HashSet } from 'effect';

class Leaf extends Data.Class<{ readonly slot: number; readonly tag: string; readonly payload: string }> {}

// entropy LOW where the trie branches 32 ways: slot drives routing, tag breaks within-collision ties —
// reject a hash concentrating entropy in bits 30-31 or one narrower than the Equal grain
const route = (l: Leaf): number => Hash.combine(Hash.hash(l.tag))(Hash.hash(l.slot));

const dense: HashSet.HashSet<Leaf> = HashSet.fromIterable(
    Array.from({ length: 24 }, (_, i) => new Leaf({ slot: i, tag: '<t>', payload: `<p-${i}>` })),
);
const present: boolean = HashSet.has(dense, new Leaf({ slot: 7, tag: '<t>', payload: '<p-7>' }));
const routed: number = route(new Leaf({ slot: 7, tag: '<t>', payload: '<p-7>' }));
```

[ORDER_AS_CAPTURED_MEMBERSHIP_POLICY]:
- `RedBlackTree.empty(ord)`/`SortedSet.empty(O)`/`SortedMap.empty(ord)` write the comparator into a private `_ord` once via `makeImpl`/`fromTree`, recovered by `getOrder` and re-threaded through every `makeImpl(self._ord, newRoot)` — there is no `add(self, value, order)` overload, the policy is the receiver's permanent membership-and-key law, the construction-time analogue of `DECLARATION_COLLAPSE`. The one place a second `Order` enters is `SortedSet.map(self, O, f)`/`flatMap` where a NEW image is born and the source order is explicitly abandoned because projected values no longer inhabit its domain.
- `findFirst` descends by the captured comparator (`d <= 0` left, `d > 0` right) but settles the hit by a SEPARATE `Equal.equals` — the `Order` navigates, structural `Equal` accepts — silently lossy where the captured `Order` is coarser than `Equal`: a probe comparing `0` against a node yet structurally unequal routes LEFT past it and lands on `none()`, reporting absent a value the tree contains on an unvisited branch; `removeFirst` inherits the same route-past so a non-equal sibling on the wrong branch is unfound and unremovable. The hashed twin "route by hash, settle by Equal" has a sorted twin "route by Order, settle by Equal", but the sorted router is a TOTAL ORDER whose coarseness MERGES where the hash merely collides.
- The merge fires at the conjunction of order-tie AND structural-equality: `SortedSet.add`/`SortedMap.set` gate on `RBT.has`, so a value whose `Order` returns `0` is NOT auto-merged unless it also passes `Equal.equals` — but where the `Order` is coarser than `Equal` AND routing reaches the tied node, `set` does `insert(removeFirst(tree, key), key, value)`, evicting the structurally-equal incumbent (last-write-wins, the evicted payload silently gone), so a sorted map's cardinality is a readout of its comparator's discriminating power, not the distinct values fed it. The bare `RedBlackTree` is a MULTIMAP — `insert` appends a fresh node with NO equal-key replacement — and `findAll` (full in-order traversal, immune to route-past) is the only retrieval honoring multiplicity; the set/map wrapper imposes single-occupancy by consulting `has`, so dropping to the raw tree for `findAll`/`getAt` forfeits dedup.
- Rebalancing trusts the captured `Order` to be a consistent total order over the FULL inhabited domain (`NaN` keys included): a transitivity violation corrupts the search invariant silently — a key present in iteration yet unreachable by lookup — and a raw-subtraction comparator producing `NaN` makes `out !== 0` true on a tie the instant it is `combine`d, poisoning routing. Range walkers (`between`, `greaterThanEqual`, `forEach*`) descend by `cmp` ALONE and NEVER consult `Equal`, so range membership (order-position) and point membership (route-then-settle) disagree — `between` answers "which stored keys occupy this order-interval", and `getAt(i)` indexes by in-order RANK so a coarse merging `Order` has fewer ranks than distinct values fed it, returning each merge class's surviving representative.
- Binary `union`/`intersection`/`difference` are typed `(self: SortedSet<A>, that: Iterable<A>) => SortedSet<A>` — the type ALREADY erased `that`'s captured comparator, so the second operand is re-admitted through the RECEIVER's `add`/`has`, two elements `that` kept distinct collapse under `self`'s coarser policy, and `union(a, b) !== union(b, a)` in CARDINALITY not merely iteration order whenever the comparators diverge — the membership trap no per-element comparison surfaces, the reason a sorted-set algebra is lawful only when every operand was built under one shared `Order`. A tree's own `[Equal.symbol]` compares in-order materializations and NEVER `this._ord` vs `that._ord`, and its `Hash` XOR-folds the materialized pairs order-independently, so two trees built under DIFFERENT policies are `Equal.equals` and hash-equal whenever their sequences coincide — the captured `Order` shapes the materialization then is erased from identity; `SortedSet.getEquivalence()` is the weaker `(a, b) => isSubset(a, b) && isSubset(b, a)` (mutual containment re-keying each through the other), diverging from the order-blind structural `Equal` exactly when the two sets merge different elements.

```typescript
import { Order, SortedMap } from 'effect';

type Entry = { readonly id: string; readonly version: number; readonly body: string };

// the captured Order keys on id ALONE: two versions are Order-tied (=== 0) and Equal-distinct,
// so set evicts-and-replaces — last write wins, the prior body silently dropped
const byId: Order.Order<string> = Order.string;
const store: SortedMap.SortedMap<string, Entry> = SortedMap.fromIterable(byId)([
    ['<x>', { id: '<x>', version: 1, body: '<v1>' }],
    ['<x>', { id: '<x>', version: 2, body: '<v2>' }],
]);
const survivors: number = SortedMap.size(store);
const kept: Order.Order<string> = SortedMap.getOrder(store);
```

[CONSUMER_GRAIN_AND_CONTAINER_LIFTING]:
- The structure family bifurcates on HOW element identity reaches it: value-carried-identity structures (`HashSet`, `HashMap`, `MutableHashSet`, `MutableHashMap`) read the stored value's `[Equal.symbol]`/`[Hash.symbol]` directly and expose NO `getEquivalence`, while `SortedSet`/`SortedMap`/`RedBlackTree` capture an explicit `Order<K>` at construction. `Array.dedupe`/`contains`/`union`/`intersection`/`difference` consume `Equal.equivalence()` (default identity); the `*With` siblings accept an explicit `(self, that) => boolean` — the bare verb is the collapse, the suffixed verb the override seam, never a parallel family. `Array.sort<B>(O)` constrains `A extends B` so a coarser `Order` over a projected field sorts a richer element, and `dedupeAdjacent` collapses only equal runs (streaming-cheap over a sorted array) distinct from `dedupe`'s whole-array uniqueness — a complexity decision not a style one.
- Eager linear containers expose `getEquivalence` to LIFT an element instance into a whole-container one: `Array`/`Chunk`/`List`/`Option`/`Record.getEquivalence(eq)` are unary, `Tuple.getEquivalence(...eqs)` variadic-positional, `Either.getEquivalence({left, right})` a two-channel record, `SortedSet.getEquivalence<A>()` NULLARY (set equality is equal membership under each set's own captured `Order`) — the lift arity tracks the container's algebra and composes by nesting. `Equal.equivalence<A>()` is the single nullary bridge turning per-instance structural identity into the element `Equivalence` every `getEquivalence` demands, composing recursively (`Array.getEquivalence(Option.getEquivalence(Equal.equivalence<Shape>()))`), the only leaf the one nullary call.
- Hashed structures locate by `Hash.hash` (bucket) then disambiguate by `Equal.equals`; sorted structures locate by `Order` alone so two keys comparing `0` collide as one entry — a coarse `Order.mapInput` silently merges keys the hashed twin keeps distinct, and sorted cardinality is a direct readout of the injected `Order`'s discriminating power. `mapInput` is the one lever turning full-value identity into field-scoped identity for any consumer, so one owner feeds a content-deep `HashSet` and a one-field-keyed `RedBlackTree` simultaneously; a field-projected `Equivalence` fed to `Array.dedupeWith` keeps first-per-key (occurrence order preserved, a keyed reduction without a grouping pass) while the same projection fed to `Array.groupBy` partitions every occurrence — projection grain shared, retention policy differs.
- The membership hazard composes from two facts: a bucket-correct dedup over content-equal-reference-distinct values requires `Data`/`Schema.Class` fusion (feeding `dedupe` plain literals returns them unchanged because `Equal.equals` degrades to reference for non-`Equal` operands), and `$is`-narrowing admits a malformed-payload member of the right tag — so a value narrowed by `$is` is arm-correct yet may differ structurally from a canonical member, and only construction through the case constructor (never an ad-hoc `{_tag, ...}` literal) guarantees the `StructuralPrototype` the hashed structures require. `Array.dedupe`/`HashSet` membership inherit the latent EQUALITY bug not the hash bug — a fast-wrong `Equal` produces duplicates `dedupe` cannot remove with a passing `Hash` masking the cause — so identity correctness is proven at the owner and never re-judged at a call site.

```typescript
import { Array as Arr, Data, Equal, Equivalence as Eq, HashSet, List, Order, RedBlackTree, SortedSet } from 'effect';
import type { TaggedEnum } from 'effect/Data';

type Shape<A> = Data.TaggedEnum<{
    readonly Point: { readonly at: A };
    readonly Span: { readonly lo: A; readonly hi: A };
    readonly Void: object;
}>;
interface ShapeDef extends TaggedEnum.WithGenerics<1> {
    readonly taggedEnum: Shape<this['A']>;
}
const Shape = Data.taggedEnum<ShapeDef>();

const at = Shape.Point({ at: 3 });
const span = Shape.Span({ lo: 1, hi: 9 });
const distinct: number = HashSet.size(HashSet.fromIterable([at, Shape.Point({ at: 3 }), span, Shape.Void()]));
const points: ReadonlyArray<Shape<number>> = [at, span].filter(Shape.$is('Point'));

const item: Eq.Equivalence<Shape<number>> = Equal.equivalence<Shape<number>>();
const sameRun: Eq.Equivalence<List.List<Shape<number>>> = List.getEquivalence(item);
const ranked: Order.Order<Shape<number>> = Order.mapInput(Order.number, (s) => (s._tag === 'Span' ? s.hi : s._tag === 'Point' ? s.at : 0));
const indexed: RedBlackTree.RedBlackTree<Shape<number>, Shape<number>> = RedBlackTree.fromIterable(Arr.map([at, span], (n) => [n, n] as const), ranked);
const ordered: SortedSet.SortedSet<Shape<number>> = SortedSet.fromIterable(ranked)([at, span]);
```

[TRANSPARENT_BRAND_WALL]:
- `Brand.nominal<T>()` is a runtime passthrough adding a phantom `[BrandTypeId]` tag with no validation — earning its place only when two same-typed values are genuinely interchangeable and must not be; `Brand.refined<T>(predicate, onFailure)` attaches a runtime check exposing throw / `.option` / `.either` / `.is` off ONE constructor — never a `makeCode`/`tryCode`/`isCode`/`validateCode` family, the reject being a sibling-factory restating one member by name. `Brand.all(...constructors)` intersects via `UnionToIntersection` of phantom records, enforced by `EnsureCommonBase` (a mismatch is the literal-type error `"ERROR: All brands should have the same base type"`); `Schema.brand(name)` accumulates a `NonEmptyReadonlyArray<string | symbol>` array annotation, stacking distinct tags on one `BrandSchema` whose `make` fuses decode-time validation with the tag. A brand with no predicate and no real nominal collision is deleted — the primitive leaf stands as bare `Eq.string`/`Order.number`.
- Branding is identity-transparent on two opposite variances: the wall stands on `Brand<in out K>` INVARIANCE (`{ readonly [BrandTypeId]: { [k in K]: K } }`, two distinct keys neither assignable) while `Equivalence<in A>`/`Order<in A>` are CONTRAVARIANT (a leaf over the wide base flows INTO a cell typed over the branded subtype unwidened, no cast, because a comparator over the supertype serves the subtype) — the wall is precisely the gap between these variances, living in the invariant intersected conjunct: `string & Brand<K>` is assignable iff bases unify AND keys are identical. `mapInput`'s `<B, A>(f: (b: B) => A) => Equivalence<B>` erases the brand by construction — the projection's INPUT is the branded narrowing the cell preserves, its OUTPUT the base the leaf compares — so the author chooses erasure depth by the projection's annotated return type.
- The schema brand is the canonical hookless refinement: `Schema.brand`/`fromBrand` build a `Refinement` whose `from` is the base, and the walk's `Refinement` arm is `go(ast.from, path)` — descending straight to the base, brand contributing nothing — so a triple-branded schema's runtime identity is byte-identical to the bare base, the three phantom tags existing only in `Schema.Type<S> & Brand<B1> & Brand<B2> & Brand<B3>`. Transparency is not a derivation choice but a consequence of the brand occupying the `BrandAnnotationId` key while identity reads `EquivalenceAnnotationId` — the two keys never collide, and a brand wanting runtime-distinct identity would attach the equivalence key alongside, abandoning transparency deliberately.
- The wall guards ASSIGNMENT not MEMBERSHIP: a branded value in a `HashSet`/`HashMap`/sorted key is located by the BASE's `Hash`/`Equal`/`Order` blind to the phantom, so two brand stacks over one base hash to one bucket, compare equal, dedup to one, and test present — the brand protects the construction-site assignment, never the membership the structure decides. `UnionToIntersection` makes the wall TIGHTER as brands stack (each conjunct shrinks the admitted source set) while the comparison stays FLAT (every conjunct erases to the leaf) — the type-level admission set and the runtime equality class move in OPPOSITE directions. The wall is one-directional in time: `.is` re-admits by PREDICATE re-stamping the type-level tag on a value that never carried it at runtime, so once two differently-stacked values erase into one comparison no constructor surface recovers the distinction — a value laundered via `Brand.unbranded` then re-branded under a different key is membership-indistinguishable. A branded sort key surfaces the wall AND the ordering-derivation gap at the one hand-composed `Order.struct`, closed by `satisfies { readonly [K in keyof Owner]: Order.Order<Owner[K]> }` over the literal field record — the wall is declared once in the table anchor and inherited by every aggregate the table folds into, checkable exactly where the field-key vocabulary is anchored.

import { Brand, Equivalence as Eq, HashSet } from 'effect';

type Code = string & Brand.Brand<'Code'>;
type Slot = string & Brand.Brand<'Slot'>;
const Code = Brand.nominal<Code>();
const Slot = Brand.nominal<Slot>();

// contravariance ADMITS the coarse base leaf; invariance REJECTS cross-brand at the same seam
const byCode: Eq.Equivalence<Code> = Eq.string;
// @ts-expect-error two distinct brand keys are invariant — Equivalence<Slot> is not Equivalence<Code>
const crossed: Eq.Equivalence<Code> = Eq.string as Eq.Equivalence<Slot>;

// the wall guards the assignment seam; membership is decided by the erased base

const crossBrandPresent: boolean = HashSet.has(codes, Slot('AB12') as unknown as Code);
