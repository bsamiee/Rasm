# Collections and Traversal — Distillate

[COLLECTION_CHOICE_AND_CONVERSION]:
- `Seq<A>` over `Lst<A>` everywhere except positional mutation: `Lst` is a position-indexed AVL tree (descent compares the index against `Left.Count`) giving the collection set's only O(log n) mid-sequence insert; outside that regime `Seq`'s start-offset structural sharing wins on `Cons`/`Tail`. `Lst` implements no `Natural` arrow — reaching `Seq` at the HKT layer is `Foldable.ToSeq`, a full O(n) tree fold — and the `Iterable` trait's `CoNatural<Lst, Iterable>` arrow materializes Iterable→Lst, not the reverse, so `Lst` never enters `Iterable`-consuming combinators for free.
- `Arr<A>` is read-only-array territory: O(1) read, full copy per modification — read-intensive, short-lived, never incrementally grown; its `AsSpan(start, count)` overloads are zero-copy slices into the backing array.
- `HashMap`/`HashSet` (CHAMP, O(1) amortized, unordered) versus `Map`/`Set` (AVL, O(log n), key-ordered with min/max/range) is the ordered-access decision; `BiMap<A,B>` is two AVL trees for a symmetric bijection — the owner when bidirectional lookup is the domain constraint, not a pair of maps; `SeqEmpty` implicitly converts to every collection type — the universal empty at call sites, never an explicitly constructed empty.
- Conversion arrows are outbound-only and asymmetric: `Arr` and `Iterator` supply `Natural` arrows to `Seq`/`Lst`/`Set`/`HashSet`/`Iterable`; `Seq` supplies none, so `where F : Natural<F, Seq>` is satisfiable by `Arr`, `Iterable`, `IterableNE`, `Iterator` but never by `Seq` itself — `Seq`-typed inputs need a direct branch; no X→Iterable arrow exists anywhere — only explicit `AsIterable()`/`createRange`; `NaturalMono<F,G>` registers one F→G arrow at both `Natural` and `CoNatural` dispatch positions and is not an isomorphism — only `NaturalIso` carries both directions.
- `createRange(ReadOnlySpan<A>)` exists for every collection: construction copies (`span.ToArray()`) but a strict sequence then supports zero-copy `AsSpan()` — the lowest-allocation creation path from in-memory data; `toSeq(IEnumerable<A>)` takes the strict path eagerly for `IList`/`ICollection`/array inputs and builds `SeqLazy` otherwise; `Iterable` is the admission bridge when a caller-owned `IEnumerable` must enter trait-polymorphic combinators — never a substitute for `Seq` where evaluation count matters, and its span constructor copies as well.

[SEQ_STRICTNESS_AND_LIFETIME]:
- Four internals own the cost model — strict (array + `start` offset), lazy (cache-once per element), concat (deferred segment tree), empty — and `.Strict()` forces any of them into the strict form.
- Lazy-forcing traps: `Count` (including the `Foldable` count/length forms — the HKT spelling is not exempt), `AsSpan()`, and `Add` all force complete evaluation of a lazy backing; `Cons` does not — incremental build from a lazy source is prepend-then-reverse, never repeated `Add`.
- There is no zero-copy span path for a lazy sequence; `SeqStrict.AsSpan()` is zero-copy over `(data, start, count)` — `AsSpan` on a sequence of unknown internal type inside a hot path is a hidden full evaluation.
- Concat segment trees: `Count` is O(segments) and segment-by-segment enumeration is allocation-free — `Fold` and `TraverseM` over a concat are cheap — but any contiguity demand copies all segments, and repeated `+` without a terminal `.Strict()` re-pays O(n) on every enumeration.
- `consDisallowed` protects offset sharing: prepend is O(1) amortized while front capacity exists, but once a derived view (`Tail`) is taken over the shared buffer, further `Cons` allocates a new block — the flag is what makes head/tail recursive descent structurally safe.
- `Seq.generate` builds a lazy sequence (a range-map pipeline, forced per element on first access) — apply `.Strict()` when all elements will be read; `unfold` overloads are fully lazy generators: the tuple form never emits the seed, the state-only form emits the seed first, and multi-state overloads up to four components avoid tuple construction in the step.
- `SeqLoan<A>` wraps a pool-rented array with a finalizer safety net and exposes folds/`Iter`/`Take`/`Skip` but no `Map`/`Filter`/`Bind` — consume in one pass, promote via `ToSeq()`, dispose; holding a loan beyond its rent scope is use-after-return, and `Dispose` during a concurrent read is undefined.

[FOLDABLE_KERNEL_AND_DERIVED_AGGREGATION]:
- The ~50-method aggregation surface derives from exactly two abstract methods — `FoldWhile` and `FoldBackWhile`; every other member is a `static virtual` default expressed as one call into them, so a new foldable costs two bodies and inherits the full numeric, search, decomposition, and conversion vocabulary.
- `Fold` is `FoldWhile` with a constant-true predicate; `FoldUntil` is `FoldWhile` plus predicate negation — any combinator pair differing only by polarity is one kernel plus `not`, never a second traversal.
- The predicate receives the `(State, Value)` pair and is checked before the step runs: the halting element is observed but never consumed, so early exit can hinge on accumulated state (halt when a running sum crosses a budget) — strictly stronger than element-only guards.
- Search and existence are sentinel-state folds, not loops: `Find` folds `Option<A>` and halts on `IsNone` failing; `Head` is the same shape; `IsEmpty` seeds `true` and flips on the first element; `Exists`/`ForAll` are the `||`/`&&` mirrors over until/while — O(1)-on-match search over any structure with no concrete `Find` method.
- `At` threads `(Index, Option<A>)` through the same machinery — forward via `FoldWhile`, from-end via `FoldBackWhile` — so on an unoverridden foldable it is an O(n) early-exit scan, never random access; the sequence override is O(1) after a bounds guard, while `Set`/`Map` keep the default fold because AVL nodes carry no positional index at the trait surface.
- The override lattice is exactly the asymptotic wins: a concrete type overrides `Count`/`IsEmpty`/`Head`/`Last`/`At` where its layout makes them O(1), plus the two primitives — overriding a derivation that already reduces to the primitives adds surface without behavior and is the rejected form; default `Count` is a full left fold that diverges on infinite structures and forces lazy backings.
- `FoldBackWhile` over enumerable backings reverses then forward-iterates — right folds pay a reversal materialization unless the backing walks from the end natively; on strict backings `FoldBack` is a plain reverse loop with O(1) stack at any length, and `Rev().Fold` is its stack-cost equivalent when the accumulator must build in reverse order.
- `FoldM` rides the pure fold with no monadic kernel: fold into the continuation `Func<S, K<M, S>>` seeded with `pure`, then apply the initial state —

```csharp
static K<M, S> FoldM<A, M, S>(Func<A, Func<S, K<M, S>>> f, S seed, K<T, A> ta) where M : Monad<M> =>
    Fold<A, Func<S, K<M, S>>>(a => bind => s => Monad.bind(f(a)(s), bind), Monad.pure<M, S>, ta)(seed);
```

- `FoldWhileM`/`FoldUntilM` guards degrade to element-only because the state is now an effectful continuation, not an inspectable value — a state-dependent monadic halt must live inside the bound effect; `FoldBackMaybe` halts on `None` with an element-first signature, deciding exit on the element alone.
- Derived numerics are single-pass by construction: `Average` carries `(Count, Total)` in the accumulator with the count as a value of the numeric carrier itself, divided once at the end — no length read, no second traversal on lazy backings; the mapped `Average<A,B>` aggregates a projected metric without an intermediate collection; `Sum`/`Product` ride generic-math identities.
- Explicit-`Ord` `Min`/`Max` carry a null guard the `IComparable` forms omit (a custom total order over a reference type whose default comparison would throw); `Set`/`Map` override only the `IComparable` forms to the tree's O(log n) extremes — the `Ord`-parameterized spelling silently falls back to a full fold; seeded `Min`/`Max` fold to plain `A` from a caller-supplied floor, trading the absence rail for a known identity.
- Every static entry ships curried `Func<A, Func<S, S>>` and uncurried `Func<S, A, S>` over one body: partial application of the per-element step wants the curried form — choosing uncurried and then partially applying reintroduces the lambda the curried form avoids; `scan`/`scanBack` emit every intermediate state including the seed when downstream needs running totals.

[APPLICATIVE_VS_MONADIC_SEQUENCING]:
- Applicative `Traverse` and monadic `TraverseM` are one fold skeleton — `fold(add, Pure(Empty), ta).Map(Kind)` — diverging in exactly one closure: the applicative arm lifts `(bs, b) => bs.Add(b)` through `Applicative.lift`, the monadic arm binds `state.Bind(bs => f(value).Bind(b => Pure(bs.Add(b))))`; accumulate-versus-abort is the applicative-versus-monad sequencing law on one accumulator, not two algorithms.
- `lift` evaluates both operands unconditionally so every failure reaches the result; `Bind` on a failed left never invokes the right so the first failure aborts. Where every failure must be reported, monadic traversal is wrong — not slower, wrong: the lost failures never appear; where later effects must not run, applicative traversal is wrong — it runs them.
- Traversal is `sequence ∘ map`: `Sequence` is `traverse(identity)`, `SequenceM` is `traverseM(identity)` — the failure-collection policy at the layer swap is selected purely by the inner functor's constraint (`Applicative` accumulates, `Monad` short-circuits), with no branch in the caller.
- `TraverseM`'s trait-level default body delegates to the applicative `traverse`: a structure that does not override it inherits accumulate semantics under a monadic name — assuming abort-on-first-failure from an arbitrary foldable kind is a latent correctness bug.
- A carrier owning both an accumulating `Apply` and a short-circuiting `Bind` flips policy silently: a traversal that needs every error must stay on the applicative path, and one `Bind`/query block over the same values reverts to first-failure with no warning — the sequencing operator chosen is the policy.
- Over an effect carrier the dichotomy is also concurrency, mechanically: `Apply` launches both operands before awaiting either and its async join awaits both, while `Bind` is structurally sequential because the right side is a function of the left's produced value; empty or failed operands collapse at node construction and skip the launch entirely. Traversal choice is therefore policy on two axes at once — `TraverseM` over logically independent effects both serializes them and changes failure semantics.
- Short-circuit stops the bind chain, never the source walk already performed: a lazy backing is forced element-by-element up to the abort point regardless of which sequencing is chosen — the reclaim is downstream effect execution, not upstream structure evaluation.
- `.As()` after `Traverse`/`TraverseM` is mandatory: the trait hands back the abstract kind and the terminal `Map(Kind)` plus the call-site `.As()` recover the concrete carrier — without it the result is unusable in concrete-typed position.

[EFFECTFUL_TRAVERSAL_VARIANTS]:
- `PartitionFallible` is the dual of `TraverseM`: every effect runs sequentially with no short-circuit, folding to `K<M, (Seq<Error> Fails, Seq<A> Succs)>`; the typed-error variant accumulates `Seq<E>`; instances span `Seq`/`Iterable`/`Lst`/`HashSet`/`Set`/`IEnumerable` of effects — the form when all effects must run and all outcomes must be collected.
- `Arr` pre-allocates the exact-`Count` backing before its traverse fold writes into it; `Seq` starts empty and appends per element — known-length input traverses with less allocation through `Arr`.
- `Iterable.TraverseIO` is the only async-aware traversal, awaiting each step when the source is asynchronous; `Seq` has no async traversal because it is always synchronously evaluated.
- Discard sequencing is `lift` with a projection — forward action keeps the second value, back-action the first — and both effects still run because it is applicative: the correct primitive for run-for-effect-keep-the-other without a `Bind` that falsely implies data dependence; every action overload ships a memoised twin that defers and caches the right operand, so a reused combinator never reconstructs or re-runs the inner effect.
- `Actions` requires a non-empty carrier: `IterableNE(Head, Tail)` makes the zero-actions case inexpressible at the type level, then folds head-through-tail with `Action`, yielding only the final value — no runtime empty-sequence guard exists because the type forbids the input.
- Non-empty admission paths: `IEnumerable<A>` → `Option<IterableNE<A>>` (empty is `None`, never a throw); `IAsyncEnumerable<A>` → `IO<IterableNE<A>>` (emptiness check deferred to execution); `Seq<A>` → a single `Head.Map` over its existing decomposition — the allocation-minimal admission.

[CHOOSE_PROJECTION_ACCESS]:
- `Choose` is atomic single-pass filter-map — an `Option`-returning step whose `None` results are discarded — replacing two-pass `Filter(p).Map(f)` and its repeated guard; the indexed overload passes the position alongside the value, and the keyed-map form carries the key through.
- `Arr` and `Lst` own no `Choose` instance: they reach choose semantics only through the `MonoidK`+`Monad` constrained trait path — the absence is structural, not an oversight to wrap around.
- `Partition` is a single pass over the source, and the async-aware partition fully materializes both partitions under the ambient cancellation token before returning — never lazy; the indexed `Map` overload (`Func<A, int, B>`) eliminates zip-with-index entirely for position-aware projection.
- The indexer throws on out-of-range with no safe public alternative on the struct: safe access is `HeadOrNone()` plus structural `Tail` decomposition, and `Case` decomposes every collection into `null` / singleton / head-tail object shapes for direct switch-expression matching without `IsEmpty` guards.
- `Zip` stops at the shorter sequence and no padding or zip-longest overload exists — equalize lengths from domain knowledge first; the iterator-level `Zip` pairs same-typed iterators only, with no projection variant.

[SEGMENT_DECOMPOSITION_AND_DIFF]:
- `tails` is O(n) — each suffix an O(1) offset slice — while `inits` is O(n²) (one `Take` per prefix plus append cost): the asymmetry is structural, so prefix-family algorithms over long sequences are the regime to redesign; the module-level `tails` excludes the trailing empty segment where the instance `Tails` property includes it — an off-by-one between two spellings of the same decomposition.
- `span` splits at the first failing element into `(prefix, suffix)` in one pass with no allocation beyond the two slices.
- `Patch` is an edit-list groupoid over ordered sequences — minimal `diff`, `apply`, composition via `append`, inversion via `inverse`, with `applicable`/`composable` preconditions — the primitive for versioned or diffable sequence state, displacing ad-hoc before/after comparison.
- `Compositions` is a skewed binary tree over a monoid: `Skip` and `TakeComposed` O(log k), `Composed` O(log n), `Take` O(k log k) — build once at O(n), then prefix-composition queries cost O(log k) each versus O(n) per flat fold, the running-window regime; only `Combine`/`Singleton` construction guarantees the balance invariant that `wellFormed` validates — direct tree manipulation silently corrupts `Skip`/`Take` results.

[COMBINE_CHOOSE_MEMO]:
- `SemigroupK.Combine` concatenates and always consumes both sides — append for sequences, left-biased union for keyed maps, union for sets; `Choice.Choose` selects the first non-empty and never consumes the right: merge versus fallback are distinct operators over the same type and are not interchangeable at a call site that needs one policy.
- The lazy fallback wraps the alternative in `memoK` through `Choose`'s memoised overload — constructed only if the left is empty, evaluated at most once.
- `Memo` defers carrier construction until first `Value` behind a CAS state gate (construction is thread-safe; usage of a stateful underlying effect is not); applicative `Apply` over a memoised inner collection reads it once and iterates per function, where without it each application re-enumerates a lazy source per function; `Reset` re-acquires only when an acquire function was supplied — pre-loaded memos no-op; a memoised natural transformation stays unmaterialized until first access, so an empty left function set never constructs the right operand at all.

[CHAMP_TRIE_NODE_MECHANICS]:
- Each node carries two disjoint 32-bit bitmaps — entry map and node map — over the same 32 slots: a slot is an inline pair or a child descent, never both, stored densely in parallel arrays; the bitmaps replace the tag, so no sentinel cells and no per-slot waste.
- Position is popcount over the bitmap prefix (a branchless SWAR Hamming weight — no table, no loop): a lookup is one popcount per level plus one final key equality, not O(log n) comparisons.
- A level consumes five hash bits against the 32-bit bitmap width; useful depth stops at six levels, and two keys identical in their low 30 bits fall to a flat collision node with linear scan — the cost model hinges on low-30-bit dispersion, not the full hash range; a slot split builds single-child wrapper nodes as deep as the shared fragment prefix, so a near-colliding pair can chain six nodes for two entries.
- Iteration order is structural — inline entries first, then children in bitmap order: identical contents enumerate identically regardless of insertion order, but the order is hash-derived and never stable across key-type or hash changes.
- Transient batch build mutates node arrays in place under a private flag, so allocation tracks the final shape rather than N intermediate shapes — but the top-level handle is still rebuilt per effective step and no-ops return the receiver: the transient wins element-by-element construction from a source and wins nothing on an already-materialized structure.
- Bulk set algebra is replay, never node merge: union, intersection, difference, and append enumerate one operand and replay into a fresh transient at O((|L|+|R|)·levels) with zero structural sharing between operands and result — repeated whole-map merges of large maps multiply the per-element constant by the full union cardinality, and a domain needing merge sharing must model the merge differently; the full heterogeneous union takes left-only, right-only, and matched resolvers, and difference by pair collection is key-only — right-hand values are ignored.
- Left bias is replay mechanics, not policy surface: try-add insertion keeps the existing entry on collision, so `+` and resolver-less union are left-biased — right bias exists only as the explicit matched resolver `(_, _, r) => r`, and a call site omitting it silently keeps the wrong side.
- Change tracking threads a parallel transient through the same pass: the change map records only inserts reporting actual mutation — idempotent re-adds contribute nothing — so one walk yields both the structure and its minimal delta; the ledger holds one net morphism per key (added, mapped-to, mapped-from, mapped-both, removed) composed over a burst, never the operation sequence, and `Snapshot()` resets it by rebuilding with an empty change map.

[AVL_ORDERED_TREE_MECHANICS]:
- Nodes are immutable with a cached height byte and subtree count; insert rebuilds every node on the comparison path — O(log n) fresh nodes plus up to two rotation nodes — with no transient and no in-place write: bulk ordered-map construction cannot amortize the way the trie does.
- The count-and-height payload is the price of ordered access: a range query positions at the lower bound in O(log n) and yields in-order lazily on demand — no eager materialization, no filter over a full conversion — and positional descent compares the target index against left-subtree counts.
- Rotation re-roots while preserving the in-order key sequence; sharing survives only in the untouched sibling subtrees of the rebuilt path; converting the unordered trie to the ordered tree is a full O(n log n) re-insert, never a cast — the two structures share no backing.
- `Skip` on the ordered map and set walks O(n): the tree shape buys no structural prefix sharing for positional skip.

[EQUALITY_DISPATCH_AT_NODES]:
- The two-parameter keyed forms bind a default comparator that resolves a trait-equality instance by reflection once at static initialization and falls back to boxing the framework default; the three-parameter forms bind an explicit comparator whose static `Equals`/`GetHashCode` dispatch directly with no reflection and no boxing — every operation pays one hash per level and one equality at the final match, so custom key types in hot paths take the explicit form.
- Composite tuple keys route through structural tuple equality under the default comparator, boxing each component: a composite-key map under churn binds an explicit comparator over the tuple shape, changing the allocation profile of the whole workload; composite constructors exist for two- to four-component keys with explicit-comparator scoped variants.
- `Contains` is `Exists` with the resolved equality — a first-match early-exit search fold sharing `Find`'s machinery — and its explicit-equality overload removes the same reflection-and-boxing the default pays at node boundaries.

[ITERATOR_RECUR_LIFECYCLE]:
- The lazy iterator is a Cons/Nil pair over four backings — lazily realized head behind a spin-lock CAS, eager value, live enumerator, thunked tail; `Clone` re-anchors at the current position so the chain behind it becomes collectible — without it a long traversal retains the entire linked history; `Split` is stronger, severing the backward chain so prior references terminate there; `Map`, `Bind`, and `AsEnumerable` call `Clone`, never `Split` — chain-breaking over infinite generators is the caller's responsibility.
- There is no finalizer safety net — unlike the rented-array loan — so an undisposed iterator over a non-rewindable enumerator (cursor, reader) leaks the handle for the GC lifetime.
- `Recur` over collection monads is breadth-first: each generation collects every loop continuation into a swap buffer and fans out while done values yield immediately — observable whenever one step returns multiple loop and done values, and O(1) stack per generation; the iterable form runs as an async generator threading the ambient cancellation token; arms are written through the implicit `Loop`/`Pure` conversions into the step result.
