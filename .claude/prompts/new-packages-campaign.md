[SESSION_ORCHESTRATION_DISCIPLINE] The mechanics of running a session, distinct from the campaign law:
- Commit by explicit pathspec only: `git commit -m <msg> -- <enumerated planning paths>` of the gate's files, never `git add -A` and never a foreign path. Agents and the operator edit the working tree concurrently — a bare commit sweeps their staged writes into the campaign commit. The orchestrator is the sole committer and the sole ledger writer; workflow agents author content and return it, and never run git. Before any commit, `git status` and confirm every staged path is a campaign planning file; operator territory (`libs/csharp/Rasm.Vectors` and the other `libs/csharp/Rasm.*` source packages, `docs/stacks/*` operator edits, `tools/`, `tests/`, `Directory.*`, every `packages.lock.json`, `.api` operator edits) is never staged.
- Bound concurrent heavy workflows: the server-side Opus throttle trips around three concurrent heavy workflows. Keep two or fewer; prefer resuming a near-done run from cache (`Workflow({scriptPath, resumeFromRunId})`, cached prefix instant) over launching new, and `TaskStop` a still-running workflow before resuming it. Cohesion and synthesis of the governing docs stays orchestrator-owned and single-writer — a workflow that edits the same governing files in parallel lanes reintroduces the incoherence the work removes.
- Model policy: Sonnet for exploration, decompile reads, currency dossiers, and throwaway-harness authoring; Opus for every authoring, red-team, merge, alignment, cross-cutting, and ideation lane (producer → red-team → merge, never a single agent for ambitious synthesis). Never Haiku. Do not over-granularize; assign by veracity need.
- `loc <path>` and `tree <path>` take exactly one path argument — no flags, no multiple paths. In workflow scripts guard against `args` drop: prefer inline `script` over `scriptPath`+`args`, pass file lists as real JSON arrays, and verify `args` is populated before using it for phase selection (workflow globals are runtime state, separate from shell, Nix, aliases, and PATH).
- [WARNING — interactive probes] Never run an unattended probe that raises an OS dialog: a macOS keychain read (`SecItem*` / Security.framework), DPAPI, a biometric or credential-store read, a signal-to-self (SIGHUP), or `docker login` all block on a prompt the session cannot answer. These routes are tier-3, proven by decompile member-shape only (`campaign-method.md` `[SPIKE_RESOLUTION]`). Provisioning is clean and additive: public container images only, operator containers and volumes sacrosanct — never stopped or destroyed.
