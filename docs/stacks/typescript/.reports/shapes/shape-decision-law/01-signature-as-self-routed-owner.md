# Signature As Self-Routed Owner

[SIGNATURE_IS_A_ROUTED_CONCEPT]:
- The concept signature is not a record of advisory flags read by hand — it is a concept with its own five-axis tuple, and that tuple routes it to the same owner the routing law assigns any concept: structural-value identity (two signatures with the same axes are the same routing), N-tagged cardinality keyed on the dominant `identity` axis, fixed payload, no codec (the signature never crosses a wire), no deep-equality demand beyond reference. The route is `Data.taggedEnum`, and the chooser eats its own output — the decision surface is an instance of the thing it decides.
- The dominant axis becomes the signature's injected `_tag`, so the signature owner is `Data.taggedEnum<{ Nominal: {...}; Reference: {...}; Structural: {...} }>`: each member carries only the secondary axes its tier reads, the discriminant is the regime, and the generated `$match` is the chooser body. Spelling the signature as a flat `{ identity: "Nominal" | "Reference" | "Structural"; wire?: boolean; cardinality?: "one" | "many" }` record is the reject — the flat form cannot narrow, so every arm re-reads every axis and the optionality lies (a `Nominal` value still types `cardinality` as present-but-undefined).
- The signature owner's correctness proof is the diff of a new regime: a fourth identity tier lands as one `Data.taggedEnum` member, and every `$match` site fails to compile until its case record gains the arm. The chooser breaks loudly on an unrouted signature — the absorbed-growth guarantee applied to the decision law, not the data law it serves.

```typescript
type Signature = Data.TaggedEnum<{
    Nominal: { readonly wire: boolean }
    Reference: {}
    Structural: { readonly cardinality: 'one' | 'many'; readonly wire: boolean }
}>
const Signature = Data.taggedEnum<Signature>()
const a = Signature.Nominal({ wire: true }) // _tag injected, args omit it; Reference() takes no args (VoidIfEmpty)
```

[GENERATED_MATCHER_IS_THE_CHOOSER_BODY]:
- `Data.taggedEnum<T>()` ships `$match` as a paired surface — data-last `(cases) => (value: A) => R` and data-first `(value, cases) => R` — and the pair is exactly the reusable-vs-inlined chooser split: the data-last form yields `(s: Signature) => Tier`, the corpus-wide chooser applied at every routing site, and the data-first form resolves one signature in place. The case record rejects an excess arm structurally (`[K in Exclude<keyof Cases, A["_tag"]>]: never`) and demands every tag, so the chooser is total by construction with no exhaustiveness tail.
- The generated `$match` and `Match.type<Signature>().pipe(Match.tagsExhaustive(...))` are not interchangeable at the chooser tier: `$match` is the owner's own member, resolves in one hop, and demands no second import or `.pipe` chain, so a chooser keyed on the signature owner's own `_tag` uses `$match` and a chooser folding a foreign or computed union uses `Match`. Reaching for `Match.type` over the signature's own owner is the wrapper the owner already deletes — the matcher is on the owner.
- `$is(tag)` returns a `u is Extract<A, {_tag: tag}>` guard, so a partial chooser that fast-paths one regime before the full fold (`reference` short-circuits to the table tier) narrows by the owner's own guard rather than a hand-written `_tag === "Reference"` comparison — the guard the owner generates is the narrowing predicate, and the string comparison is the symbol the program already knows restated.

```typescript
const chooseOwner: (s: Signature) => string = Signature.$match({
    Nominal: (s) => (s.wire ? 'Schema.brand' : 'Brand.refined'),
    Reference: () => 'as-const-table',
    Structural: (s) => routeStructural(s), // arm delegates to a sub-chooser, never re-enters chooseOwner
})
```

[ARM_IS_A_SUB_CHOOSER_OVER_THE_NARROWED_SIGNATURE]:
- A structural arm receives a value narrowed to `Extract<Signature, {_tag: "Structural"}>` — `{ cardinality; wire }` with the irrelevant nominal/reference axes erased — and that residual is a *fresh* sub-signature whose dominant axis is whichever remaining axis splits its routing tree, not the regime the parent already consumed. The recursion descends one level of the axis tree per arm: the parent dispatches on `identity`, the structural sub-chooser dispatches on the `(cardinality, wire)` product, and a deeper owner-family sub-tier (the identity-tier ladder inside `cardinality: one`) would descend again. Each level re-runs the chooser over a strictly smaller axis set, terminating when one tier remains.
- The composition of the two `Match` creators is the load-bearing mechanism the dogfood unlocks: the outer chooser is built on `Match.type`/data-last `$match` (`Pr = never`), whose terminal is a *reusable function* the whole corpus shares; each inner sub-chooser is built on `Match.value`/data-first `$match` (`Pr = the narrowed member`), whose terminal is a *resolved tier* returned inside the arm. The sub-chooser's `Match.orElse` therefore returns the tier directly rather than a `(s) => Tier` thunk the arm would otherwise have to apply — the `[Pr] extends [never]` branch in `orElse`'s own signature is what makes the resolved-value vs function-value distinction a type the arm body inherits, so a misbuilt sub-chooser (one created with `Match.type`) fails at the arm because its terminal hands back a function where a tier was demanded.

```typescript
const routeStructural: (s: Extract<Signature, { _tag: 'Structural' }>) => string = (s) =>
    Match.value(s).pipe(
        Match.whenAnd({ cardinality: 'many' }, { wire: true }, () => 'Schema.Union<TaggedClass>'),
        Match.whenAnd({ cardinality: 'many' }, { wire: false }, () => 'Data.taggedEnum'),
        Match.orElse(() => 'Schema.Class'), // Pr = the narrowed member, so orElse resolves to the tier, not a function
    )
```

[PR_SLOT_IS_THE_COMPOSITION_CONTRACT]:
- The `Matcher<I, F, R, A, Pr, Ret>` `Pr` slot is the type-level record of which chooser form is in hand, and it governs composition mechanically: `Match.exhaustive`, `Match.orElse`, and `Match.orElseAbsurd` are all typed `[Pr] extends [never] ? (input: I) => Unify<...> : Unify<...>`, so a `Pr = never` matcher terminates to a *function* and a `Pr = I` matcher terminates to a *resolved value*. A chooser built with the wrong creator surfaces the mismatch at the terminal, not the call site — applying a resolved value as a function, or returning a function where a tier was demanded, fails at the `pipe` close.
- The reusable chooser is declared once at `Match.type` (or data-last `$match`) and stays a value the whole corpus shares; each arm's sub-chooser is a fresh `Match.value` because its input is bound at the arm. Mixing the regimes — a `Match.type` sub-chooser inside an arm — forces the arm to apply the returned function to its own narrowed input, a needless hop that `Match.value` collapses. The `Pr` slot makes "reusable spine, resolved arms" a checked invariant rather than a convention.
- `Match.withReturnType<Tier>()` pins `Ret` before any arm is written and must be the first pipe step, so the outer chooser's codomain is declared once and every arm — including every inner sub-chooser's terminal — is checked against the same tier shape. An inner sub-chooser whose `orElse` returns a different tier shape than the outer arms is caught at the inner terminal because both inherit the pinned `Ret`. The reject is an untyped recursive chooser whose arms drift in return shape and surface the divergence only at the routing site downstream.

[GENERIC_SIGNATURE_DEFEATS_PER_ARM_NARROWING]:
- A signature owner parameterized over the routed concept's own type — `Data.taggedEnum<WithGenerics<1>>` whose members carry the concept's element type in slot `A` — generates `$match` through `GenericMatchers`, where the case args are `Extract<TaggedEnum.Kind<Z, A>, {_tag: Tag}>` with `A` re-bound per `$match` call. The narrowing still holds per arm, but the *tier* an arm returns can no longer depend on `A`, because the data-last generic `$match` resolves `A` only at application and the arm body sees `A = unknown`. A chooser whose output tier varies with the routed concept's element type cannot be a generic-signature `$match` arm — it is a second dispatch over the element, downstream of the regime dispatch.
- The trap is silent in shape and loud only in the tier: the generic `$match` compiles, every arm is exhaustive, and the routing by regime is correct — but an arm that tried to read the element to refine the tier sees `unknown` and collapses to the widest tier. The signal is an arm whose body inspects slot `A`; that inspection is the second axis (the element's own signature) leaking into the regime chooser, and it re-routes to a nested chooser over the element run *after* the regime arm resolves, never inside it.
- The generic signature owner is correct exactly when the routed tier is a pure function of the regime axes and independent of the element — which is the normal case, because the owner family is chosen by identity-and-cardinality, not by what the value contains. Reaching for a generic signature to thread the element into the regime decision is the over-parameterization the monomorphic `Data.taggedEnum` signature avoids: the element's signature is its own chooser run, joined to the regime chooser by sequence, not fused into one generic dispatch.

[SELF_ROUTING_FAILURE_SURFACES]:
- The signature owner inherits the `UntaggedChildren` guard: a member that pre-declares a `_tag` field (a hand-migrated signature that already stamped its regime as a literal field) resolves the whole `Data.taggedEnum<T>` to the string-literal error type `"It looks like you're trying to create a tagged enum, but one or more of its members already has a `_tag` property."` — not a usable owner. The regime is the injected discriminant, never a field the signature carries; stamping it twice is the self-routing analogue of passing `_tag` to a `TaggedClass` constructor.
- A signature shaped as a flat record fails the discriminator narrowing exactly as any mis-shaped union does: `Match.discriminators("identity")` and the generated `$match` both narrow each arm by `Extract<R, Record<D, P>>`, and a flat record carrying `identity` as a free literal-union field narrows every arm to `never`, so each arm body cannot read the secondary axes it routes on. The flat-signature defect is detected at the arm body's field access, not at the chooser declaration — the arm compiles empty and fails the moment it reads `cardinality`. The fix is structural: the signature must be the tagged union before the chooser can dispatch it, which is the dominant-axis-as-`_tag` law made enforceable.
- The self-routing law forbids a second signature owner the same way the single-source law forbids a parallel schema for one concept: a subset of concepts routed through a sibling `Signature2` owner splits the decision surface, so a regime declared on one owner is structurally invisible to the other's `$match`, and a concept whose axes the sibling does not model is mis-routed with no diagnostic. The signature is one owner across the whole corpus precisely because it is itself a routed concept — duplicating it is the parallel-shape defect the routing law deletes, now applied to the chooser's own input rather than a domain value.
- The dogfood closes the loop: the signature owner is chosen by running the routing law on the signature's own five axes, and the chooser that consumes it is the signature owner's own `$match` — so the decision layer carries no artifact the data layer does not, and reshaping the routing law (a new regime, a new tier) is one declaration inside the signature owner that re-flows every chooser arm at compile time. A routing law expressed as an `if`-ladder beside a flat record is the loose spelling this collapses: the ladder cannot break loudly on an unrouted signature, cannot narrow its arms, and is a second surface drifting from the owner it claims to route.
