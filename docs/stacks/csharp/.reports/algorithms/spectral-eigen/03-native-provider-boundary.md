# Native Provider Boundary

[BINDING_CONTRACT_AND_SWAP_POINT]:
- The single mutable seam between managed determinism and native LAPACK is one process-static field on the linear-algebra control surface, read through a double-checked `lock` and lazily initialized to the managed instance. Every dense eigensolve resolves `Provider.EigenDecomp(...)` against whatever value occupies that field at call time; there is no per-call override, no instance scoping, no ambient context — the provider is global mutable state for the lifetime of the process.
- The binding contract any substitute must satisfy is one method on the generic provider interface: `void EigenDecomp(bool isSymmetric, int order, T[] matrix, T[] matrixEv, Complex[] vectorEv, T[] matrixD)`. The parameter order is load-bearing and the carrier shapes are fixed: `matrix`, `matrixEv`, `matrixD` are length `order²` column-major, while `vectorEv` is always `System.Numerics.Complex[]` of length `order` regardless of `T`. A `float` or single-precision-complex native provider must up-convert its eigenvalues to double-precision `Complex` at this boundary — a precision asymmetry baked into the contract, not the algorithm.
- The dense factory is the only surface that touches `Provider.EigenDecomp`; the user-storage factory runs its own in-class tridiagonalize/diagonalize/Hessenberg/Schur pipeline and never consults the provider. Routing an eigensolve through user storage is the provider-immune escape hatch: it pins the managed numerical regime irrespective of which native binary the host has installed or which environment variable is set.
- Native providers are not present in the core assembly — they are separate assemblies resolved reflectively by assembly-qualified type name through a lazy `ProviderProbe` keyed under `ExecutionAndPublication`. The probe calls `Type.GetType(name)` then `Activator.CreateInstance` cast to a creator interface; a present sibling assembly on the probe path is sufficient for the swap to fire, no explicit opt-in required. The contract a native provider must expose is exactly the provider interface plus a creator factory at the well-known type name — there is no separate native-handle or disposable-provider abstraction.

[SELECTION_PRECEDENCE_AS_POLICY]:
- Default provider selection is a four-way switch on the uppercased value of one environment variable, evaluated once on first field access: the recognized keys route to MKL, CUDA, or OpenBLAS native paths, and any other value falls through to a probe chain that tries MKL, then OpenBLAS, then CUDA, then managed. An ambient environment variable on the host therefore silently selects native LAPACK before any application code runs, and the selection is cached — a later mutation of the variable has no effect once the field is populated.
- An application-context boolean switch disables all native probing ahead of the environment variable; setting it before first field access forces the managed path unconditionally. Three further per-provider disable switches gate the individual probes. This is the deterministic-pin knob: the choice between "host decides" and "library decides" is itself a policy value resolved at definition time of the provider field, not a runtime branch in the solver.
- The thread-degree knob clamps to a closed interval and, on assignment, re-verifies every active provider rather than passing a thread count into any native call — the native side reads the degree indirectly. Single-thread and multi-thread verbs flip the degree plus a thread-safe-RNG flag together and re-verify. Managed-path parallelism is separately gated by two internal order/element thresholds below which the solve runs serially; the eigensolve helpers themselves are serial regardless.

[DETERMINISM_AND_CONTRACT_DIVERGENCE]:
- The managed path is classic Handbook/EISPACK lineage — Householder tridiagonalization then QL iteration for the symmetric case, orthogonal-Hessenberg reduction then real-Schur QR for the nonsymmetric case — with no workspace-query protocol, no info-return convention, and no LAPACK driver naming anywhere. A native provider substitutes a backward-stable LAPACK driver behind the same interface, and the two paths diverge on five contract axes that no carrier records.
- Ordering divergence: the managed symmetric path applies an explicit ascending selection sort as its tail, swapping eigenvector columns in lockstep with eigenvalue entries; the managed nonsymmetric path emits eigenvalues in bottom-up Schur deflation order with no sort. A native symmetric driver also returns ascending values but a native nonsymmetric driver's ordering is its own deflation path. Any consumer relying on index correspondence between two solves of nearby operators is relying on provider-stable ordering that the contract does not guarantee.
- Sign and normalization divergence: eigenvectors are unique only up to sign (real) or unimodular phase (complex), and the managed Householder accumulation fixes a sign convention different from a native driver's largest-component-positive convention. Both bases are valid; their columns differ by sign or phase. A projection that assumes a fixed sign, or a regression baseline asserting eigenvector equality, breaks the instant the provider swaps even though both answers are correct.
- Convergence-ceiling divergence: the managed symmetric QL throws a non-convergence signal at a fixed per-eigenvalue iteration count; the managed real-Schur QR throws at a ceiling proportional to order that resets on every deflation, so it bounds consecutive non-deflating steps since the last deflation, not total work. A native driver returns a nonzero info code on the same pathology rather than throwing — the error rail changes shape across the boundary from exception to status code, and a `Try`-based receipt that catches the managed exception sees nothing when a native provider sets an info code instead.
- The exceptional-shift escape in the managed real-Schur QR substitutes fixed literal constants into the shift state when the quadratic estimate degenerates. Two operators differing below the convergence band can therefore return bit-identical eigenvalues under the managed path — a determinism feature for regression baselines and a precision trap for any equality assertion tighter than the band — and this exact-determinism property is itself provider-specific: a native driver's exceptional-shift heuristic produces different last bits.

[BLOCK_DIAGONAL_PACKING_INVARIANT]:
- The block-diagonal carrier encodes a complex-conjugate pair as a 2×2 real-Schur block, and the packing is the same convention any native provider must emit for the dense consumer to read it: the real part lands on the diagonal at both pair indices, the positive imaginary part lands on the super-diagonal entry and the negative imaginary part on the corresponding sub-diagonal entry. A real eigenvalue writes only its diagonal entry with zero off-diagonals around its index. The eigenvalue vector and the block-diagonal carry the same imaginary information in two layouts, and a native provider that packs the off-diagonals with a different sign or transpose desynchronizes every consumer that reads rotation evidence from the block structure rather than the eigenvalue vector.

[RECEIPT_CONSEQUENCES]:
- The receipt must carry the provider identity as first-class evidence alongside the route variant, because the five-axis contract divergence means the same logical eigendecomposition has different sign, ordering, and last-bit content under different providers, and a downstream comparison of two receipts is only meaningful when both were produced by the same provider. Provider identity is to the eigen-receipt what symmetry class is to the route: a discriminant that changes the meaning of every other field.
- Pinning the managed provider through the application-context disable switch before first field access is the correct definition-time policy for any proof receipt that must reproduce across continuous-integration, host-bridge, and headless runs, because those environments differ in installed native binaries and ambient environment variables; relying on the probe chain delegates numerical reproducibility to host configuration the receipt cannot see. The pin is a one-time policy assertion at process start, not a per-solve parameter, matching the process-static shape of the provider field it controls.
- The error rail must absorb both shapes: the managed boundary wraps a thrown non-convergence signal, but a native boundary that returns a nonzero info code reports the failure in-band, so a receipt taking only the throwing path silently mislabels a native non-convergence as success with meaningless vectors. Residual evidence — the block Frobenius norm of the eigen-equation defect — is the only provider-agnostic correctness detector and is mandatory precisely because it is the one signal both rails surface identically.
