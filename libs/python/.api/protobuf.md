# [PY_BRANCH_API_PROTOBUF]

`protobuf` supplies the `Message` base class, descriptor types, and serialization/deserialization functions for Protocol Buffer messages generated by `protoc`. It provides the binary wire format (`SerializeToString`/`ParseFromString`), the `proto` module for functional encode/decode, and `json_format`/`text_format` converters. All generated `_pb2.py` message classes derive from `google.protobuf.message.Message`.

## [4]-[IMPLEMENTATION_LAW]

[WIRE_TOPOLOGY]:
- generated message classes are auto-registered in the default `DescriptorPool`; `symbol_database.Default()` resolves by full type name
- binary format: `SerializeToString` always produces the canonical wire encoding; `ParseFromString` resets the message before decoding
- unknown fields are preserved by default in proto3; `DiscardUnknownFields()` removes them explicitly
- `HasField` applies only to message-typed fields and proto3 optional/oneof fields; scalar proto3 fields without `optional` do not support presence
- `proto.parse` is the functional alternative to `ParseFromString`; it returns a new instance and does not mutate state

[LOCAL_ADMISSION]:
- Generated `_pb2.py` files are the only source of concrete message classes; never instantiate `Message` directly.
- Use `json_format.ParseDict` at boundary intake; use `MessageToDict` at boundary emit with `preserving_proto_field_name=True` to keep snake_case keys.
- `proto.serialize_length_prefixed`/`parse_length_prefixed` are the streaming frame helpers for length-delimited streams.

[RAIL_LAW]:
- Package: `protobuf`
- Owns: message serialization, descriptor schema, JSON/text format converters, and streaming frame helpers
- Accept: generated `_pb2.py` messages, `proto.serialize`/`parse` functional API, `json_format.ParseDict` at intake, `MessageToDict` at emit
- Reject: hand-rolled binary encoding, direct `Message` instantiation, `ParseFromString` on messages not first cleared
