## [00] ORIENTATION — the disposition this ledger exists to inform

Both folders live in the SAME assembly `Rasm` (the KERNEL). `Rasm.csproj` globally imports `Rasm.Domain` (line 38), so `Vectors`/`Analysis`/`Geometry` see every `Domain` type unqualified — Domain is the internal substrate glue. The strata inside the kernel is real: `Vectors` (math) + `Domain` (Rhino-normalization + Context + Stats + Validation + ContentHash) are the FLOOR; `Analysis` is the OPERATION LAYER built entirely on top of both (`using Rasm.Vectors;` in 8 of 14 Analysis files; `GeometryKernel`/`Context`/`Op`/`Fault`/`Stat` from Domain in all 14).

**The single most important finding, stated once up front so no downstream agent misreads the "Analysis slated DELETE" instruction:** Analysis is NOT dead, thin, or naive. It is a 2709-LOC world-class polymorphic geometry-analysis engine, and its substrate (`AnalysisQuery` + `Operation<TGeometry,TOut>` + `Analyze.Run/Query/Scope/In/From` + `Env`) is THE PUBLIC ENTRY POINT that the entire downstream platform binds to:
- `Rasm.Grasshopper.Components` (`Binding.cs`/`Component.cs`/`Bridge.cs`): the whole GH output pipeline (`OutputBinding.Of<TOut>(…, AnalysisQuery query, …)` → `Analyze.Query<object,TOut>` → `Operation.Apply` → `.Run(env: new Env(...))`) is the Analysis runtime. `TopologyProjection`/`GeometryProjection`/`Fault.Unsupported` are consumed by name in `Output.Drain`.
- `apps/grasshopper/Radyab/Components/*` (e.g. `ExtractPoints.cs`): every output is `n.MeshPointSpatial(Points.*)`, `n.Measure(Measure.*)`, `n.Bounds(Bounds.*)` — direct `AnalysisQuery` factory calls.
- `Rasm.Rhino/Commands/Context.cs`: `Scope = Analyze.In(context: domain)` exposed as `public Analyze.Scope Scope`; `UI/Overlay.cs`: `Analyze.Run<object,BoundingBox>(query: AnalysisQuery.Bounds(Bounds.AxisAligned), …)`.

Deleting the Analysis SOURCE is fine (the campaign lands no source); deleting the CAPABILITY without an explicit ruled home is a zero-loss-gate violation. The greenfield `Geometry/` robust-core does NOT replace it (see [E] below). This ledger's core deliverable is the capability→home map + GAP flags so the Architect can rule `disposition` correctly.
