# [PY_DATA_RULINGS]

Per-folder decision registry for the interchange plane — the settled rulings agents re-litigate for lack of a home. Each row is one decision with the one why guarding its re-litigation: a package admission or rejection, an owner-choice discriminant, a collapse refusal naming the lost guarantee, a structure retirement, or a working law. Append-mostly; a row earns its seat while its why stays homeless, dies only when fact and why both live at one durable surface, and a decision spanning folders homes at the branch tier. `(none)` marks a section awaiting its first ruling.

## [01]-[PACKAGES]

- `arro3-*` admits as the slim-carrier fast path with its split predicate named — a leg needing only IPC or compute kernels on an already-Arrow operand rides arro3, and any acero, dataset, or flight surface keeps `pyarrow` the owner; re-opens only if arro3 grows an engine surface.

## [02]-[SHAPE]

- Transport-band transforms — IPC compression, a federated plan re-serialize, re-framing — never reach the identity fold: a codec is a transport or egress parameter and a gate hands original bytes onward untouched, so wire volume drops while every content key and foreign-plan validation stays unchanged.
- Versioned and transactional store owners emit only the snapshot-identity `ContentKey` as the durable receipt — the version-control engine (branch-merge policy, retention orchestration, table-protocol governance, the reuse ledger) is the C# `Rasm.Persistence` at-rest owner's, `lakehouse` and `virtual` each declining their half; a data-side owner holding any of that state is the rejected move, routed to Persistence; re-opens only on a version decision no Persistence wire carries.
- Store span-kind law: outbound store and network I/O legs open spans `kind=SpanKind.CLIENT`, local-path legs stay `INTERNAL`, and sibling I/O legs carry one span each — folded multi-leg spans and INTERNAL outbound legs drop the client-boundary evidence a distributed trace joins on.

## [03]-[COLLAPSE]

- Every overlapping second library enters as a bounded fast-path under a named split predicate, never a parallel owner — the predicate decides which surface serves a call (format coverage, kernel availability, backend identity), and a per-page arbitrary choice between the two surfaces is the rejected form; a fast-path member never counts as owning the concern it accelerates.
