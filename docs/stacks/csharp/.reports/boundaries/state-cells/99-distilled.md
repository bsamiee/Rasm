# State Cells

[CELL_PRIMITIVE_LADDER]:
- Single-cell CAS (`Atom<A>`) is the floor: it coordinates one location, validates after the transform inside the spin, fires its change event once on success, rejects by returning the unchanged snapshot, and allocates nothing on the reference-payload success path — correct whenever the entire invariant fits in one value.
- Transactional `Ref<T>` is the multi-cell ceiling, earned only when two or more cells must transition as a unit: two independent atoms cannot be swapped together without a window where one has committed and the other has not. Using a transaction for a single cell pays the snapshot-and-replay tax — global-map capture at entry, whole-body re-run per conflict, per-attempt notifications — for atomicity a CAS already provides.
- Both are value-replacement primitives, never critical sections: neither expresses "these reads and writes are mutually exclusive as a block." `Lock.EnterScope()` is earned only for that block-exclusion invariant, and its `Lock.Scope` is a ref struct — unboxable, unstorable, compile-time-rejected across `await` — so the scope cannot outlive the frame; async-spanning exclusion takes `SemaphoreSlim.WaitAsync`/`Release`. Defaulting to a lock where a cell suffices is the over-serialization failure.
- The gate-primitive recursion split is a deliberate choice, not an accident: `Lock` is re-entrant per thread (`IsHeldByCurrentThread` is the assertion surface) and silently permits an initializer that re-enters the guarding layer; `SemaphoreSlim(1,1)` hard-deadlocks on same-thread re-entry — observable and diagnosable. Select by whether recursion should be permitted or surfaced.
- A `Ref`'s process-global registry slot is reclaimed only by its finalizer: the cell is collectible but its slot outlives it by finalization latency, so minting refs per operation accumulates slots faster than reclamation under load — coordinated cells are long-lived spine fixtures, never per-request ephemera.

[SINGLE_CELL_CAS_LAW]:
- The swap transform replays per contention iteration, so any side effect inside it repeats: effects ride the return value and execute once after the successful swap, outside the loop. A swap body that runs acquire or release effects double-fires on CAS retry — the same law that makes a registry sweep inside a map swap double-release on contention.
- Constraint placement is a two-surface law: relational transition guards (legal predecessor-to-successor edges) live in the swap body — `curr => curr.IsValidPredecessor(next) ? next : curr`, or `SwapMaybe` returning `None` — rejecting silently with the unchanged snapshot; absolute domain invariants (a count never negative, a deadline never past) live in the cell validator and fault on violation. A `Ref` validator failure throws a validation exception distinct from the conflict signal — it escapes the retry loop, terminal and never re-snapshotted — and must convert to a typed rail outcome at the seam. Collapsing the two surfaces either swallows an invariant breach as a silent no-op or escalates an ordinary illegal edge into a thrown fault.
- The validator receives only the candidate value, runs on every swap, and a rejected initial value makes construction return `None`. A gate cell's validator must be `_ => true` with the guard in the swap body: a validator that rejects the steady state freezes the cell permanently, because no later transition — including the release path — can pass it.
- Rejection is observable only at the call site: a failing guard or validator leaves the value unchanged, throws nothing, fires nothing; a `SwapMaybe` transition returning `None` is the explicit rejection signal, while the method itself still returns the raw `A`. "Value unchanged" is the rejection contract of the single-cell primitive, never an error condition.
- The metadata-carrying form (`Atom<M,A>`) captures `M` immutably at construction and its change event delivers only the new `A`: a handler needing both reads them as one immutable payload published through a single slot, never the metadata from a second unfenced field after the event fires.

[TOKEN_GATED_SINGLETON_GATE]:
- The canonical once-only gate is winner-take-CAS: `Interlocked.CompareExchange(ref guard, 1, 0) == 0` names exactly one winner; the `Atom<Option<Guid>>` spelling — swap from `None` to the token, `None` on the already-open branch — is the same CAS with rejection observable at the loser's call site.
- Acquire-inside-swap is the double-acquisition hazard: two threads observing absence both run the factory and the loser's handle leaks past the CAS. The two-step protocol eliminates it — gate-CAS to a pending sentinel, acquire outside the loop, commit-CAS the real handle — keeping the atom as the concurrency boundary while no resource is ever constructed inside the spin; the only alternative is structured disposal of the loser's handle when the retry observes a concurrent win.
- Non-re-entrant initialization holds the gate through the full side-effect sequence and signals completion by a second cell transition, never by releasing the gate early; an initializer that calls back into the guarding layer deadlocks on any single-permit gate.
- The init cell is three-state — `Initializing`, `Initialized`, `InitializationFailed` — because a boolean admits the window where a second caller observes done while the winner's side effects still run. Waiters take a process-static `TaskCompletionSource<InitState>` allocated before the gate is reachable, never lazily on first contention; the winner calls `TrySetResult` only after committing the `Initialized` swap so the wake rides the barrier that published the state; waiters read the atom first and await only on `Initializing` — both interleavings then read a settled value.
- Re-open semantics replace the entire `(CancellationTokenSource, CancellationToken)` pair per open transition: `TryReset()` returns false once cancelled — including a linked source whose parent fired — and a successful reset does not sever the parent link, so parent cancellation still propagates afterward. Reset is never a substitute for a fresh pair.

[SESSION_LIFECYCLE_AND_SPINE]:
- Lifecycle vocabulary: `Created`, `Opening`, `Open`, `Draining`, `Closed`, `Faulted`. Draining fences new admissions while keeping mid-flight work alive, and rejects with typed evidence — the rejecting state, the caller's correlation id, a timestamp from the injected clock; a rejection typed as generic cancellation loses the boundary context that lets callers distinguish retriable from terminal. Post-close access guards use `ObjectDisposedException.ThrowIf(condition, typeof(TCell))` in statics and factories — a compile-time-constant type with no virtual dispatch.
- Drain-with-count correctness is three coordinated cells — state, in-flight count, drain deadline — inside one `Serialisable` transaction: the drain transition reads the count and writes the state atomically, so any concurrent in-flight increment conflicts with the drain's read and one side retries against the other's committed value. Collapsing these cells into one boolean reintroduces the torn-read window the transaction exists to close.
- `Faulted` is escaped only by constructing a fresh `Open` instance, never by resetting the faulted one, and fault re-entry decides replace-versus-reuse of correlation state as explicit policy, never as a consequence of allocation order. The mid-flight correlation cell (`ConcurrentDictionary<CorrelationId, TaskCompletionSource<T>>`) is swept fully before a successor accepts work: remove each entry before faulting it — preventing double-fault under concurrent drain-plus-replace — and fault with `TrySetException` carrying boundary identity, not `TrySetCanceled`, because callers treat cancelled as retriable and faulted as not. Drain seams reachable by concurrent shutdown complete through the idempotent try-form (false when already completed), and a faulted completion propagates the fault to drain awaiters rather than a clean finish.
- One process-static spine record owns `(CancellationTokenSource, TimeProvider, ports, configuration)`; a reboot constructs the new record only after the old reaches fully-drained-and-disposed, and two live spines sharing any mutable cell have undefined merge semantics. Ownership of shared cells is allocation-tracked: a scope record marks which cells it created as ownership bits and its CAS-guarded idempotent dispose releases only those — borrowed cells are skipped by construction, and exactly one owner ever disposes a shared cancellation source.
- Linked sources derive from the root via the span overload of `CreateLinkedTokenSource` and are disposed on every path — success, failure, timeout — because each parent registration holds a delegate reference back to the child; undisposed children accumulate as garbage on the parent until it fires. Registrations detach in the owning cell's dispose path, never fire-and-forget; the unsafe-register form that skips execution-context capture is admitted only where the callback owns no ambient state.
- Teardown synchrony is a choice carried by the cancel verb: the synchronous form runs all registered callbacks inline before returning — the seam when later teardown requires callbacks complete; the async form dispatches callback execution to the pool and the await suspends rather than stalls. Timeout-constructed sources start their timer at construction, not first use.
- The clock enters as an injected `TimeProvider` field of the spine record — never closure-captured, never ambient: the same fake provider threaded into the spine and every timeout source makes one test-time advance synchronously fire all outstanding timeouts, giving deterministic drain-on-grace-expiry and failed-on-setup-timeout transitions. An adaptive poll cell is `(ITimer, Atom<TimeSpan>)` with the atom's change handler driving `ITimer.Change` — rescheduling without reallocation — and `Change(TimeSpan.Zero, Timeout.InfiniteTimeSpan)` is the immediate-fire drain trigger for timer-driven cells.

[MEMO_REGISTRY_CELLS]:
- The `Lazy<T>` mode split is the memo-cell decision: `ExecutionAndPublication` caches the factory's first exception permanently — every later read rethrows without retrying — unsuitable for transient failure modes; `PublicationOnly` retries until success but races factories concurrently and silently discards losers without disposal — safe only for non-resource factories or self-cleaning handles. Recursion splits identically: `ExecutionAndPublication` throws on re-entrant access (non-re-entrant lock); `PublicationOnly` silently re-enters the factory. A resource-acquiring factory with transient faults takes `Atom<Option<T>>` with explicit retry policy instead of either mode.
- `AtomHashMap.FindOrAdd` runs its factory inside the CAS spin — at least once per concurrent first-insert per key — so a resource-acquiring factory leaks the loser's handle; the once-only surface is `TryAdd(key, new Lazy<V>(factory))` then reading `.Value`. The same law governs `ConcurrentDictionary`: both `AddOrUpdate` factories may run multiple times under contention and must be pure, the `TArg`-threading overload deletes the closure allocation, and resource factories wrap in `Lazy` identically.
- Per-key-lock memoization (the `Func` memo extensions) is the exactly-once-per-key surface — a per-key sync object serializes factory execution; the weak-dict form adds weak-reference eviction, the unsafe form caches forever. `LazyInitializer.EnsureInitialized` covers the residue: the three-field form for `[ThreadStatic]`-backed per-thread caches needing a first-init lock, the single-field CAS form for process-static reference targets.
- Insertion surfaces discriminate on absence semantics: `SwapKey(K, Func<V,V>)` is a silent no-op on absent keys — conditional insertion takes the `Option`-receiving overload; `FindOrMaybeAdd` with a factory inserts only on `Some` and the factory must produce a stable insertion decision under replay; the value-direct form given a pre-computed `None` is a pure read-probe where absence is a valid terminal state. A conditional swap that decides "no change" returns the original map reference — reference identity is the no-op detector that skips the CAS and the change event; a structurally equal new map still publishes.
- Allocation rides the operation, not just the subscription: `FindOrAdd` always takes the change-logged variant — unconditional diff allocation even unsubscribed; `SetItem` switches on subscription and, on subscribed maps, pre-allocates the diff before the has-changed gate even for idempotent writes. The change event itself fires only post-CAS and only when the trie actually changed.

[CACHE_KEY_IDENTITY_AND_SWEEP]:
- A memo key encodes every dimension that affects the output — content hash plus full configuration hash; one missing dimension produces stale hits. Native-library identity is the same law at three fields minimum — absolute path, file length, last-write timestamp — because in-place file replacement is invisible to path-keyed caches.
- A token-gated native cell stores the handle beside its fingerprint and holds it until process exit or explicit teardown — freeing while exported function pointers remain referenced is an access violation on the next call; the non-throwing try-load populates the handle without exceptions. Two cells wrapping one library share the fingerprint check through a name-keyed map whose insert validates consistency, surfacing a version conflict as a typed failure at the second load attempt instead of undefined behavior from two incompatible loads.
- Registry keys are stable across regeneration: `nameof`, source-generated type identity, and versioned discriminants survive recompilation; runtime hash codes, iteration indexes, and assembly load order do not. Ghost entries from a changed type graph are swept with `Filter` during the drain phase — after in-flight operations referencing old keys complete, before `Closed`; a sweep during `Open` races concurrent inserts, so the predicate checks entry validity, never key existence.

[MVCC_ENGINE_AND_ISOLATION]:
- Conflict is version equality, never value equality: every coordinated cell carries a monotonic version, a transaction snapshots `(value, version)` per touched cell, and commit succeeds only while live versions still match — two transactions writing structurally identical values still conflict.
- The engine is snapshot-plus-one-CAS: entry captures the global cell trie into a flow-local transaction object — free under persistent-trie structural sharing, which is a silent precondition: a payload that defeats sharing converts every snapshot into a deep-copy tax; body reads and writes hit the snapshot while recording read and write sets; commit validates and applies inside the global map's own swap closure, so validate-plus-apply is atomic against other committers — nothing interleaves between the read-set check and the write application.
- Version increments are commit events: a body reading its own just-written cell sees the in-transaction value at the snapshot version and can never infer commitment from any version observable inside the body.
- The write check is unconditional across isolation levels; isolation tunes only the read set. Under `Snapshot`, a written cell whose version moved aborts the transaction, but a merely-read cell is untracked — a concurrent commit between the read and the decision leaves the read torn with no abort. `Serialisable` adds read-set validation: every read-not-written cell must still hold its snapshot version at commit. Any permit decision of the form "the set I observed did not change while I decided" requires `Serialisable`; written cells are fully serialized regardless.
- The entrypoint trap: the serially-named helper dispatches snapshot isolation — behaviorally identical to the snapshot-named one — and only the explicitly parameterized `atomic(op, Isolation.Serialisable)` validates reads. Permit-decision sites spell the isolation argument explicitly; the read protection the name appears to promise otherwise never runs, invisibly to review.

[TRANSACTION_DISCIPLINE_AND_NOTIFICATION]:
- The ambient transaction rides an async-local slot: it flows across awaits within one logical flow and survives resumption on a different worker, so the no-transaction probe is a flow probe that throws when absent — useless as a conditional-entry test; entry is decided structurally. Out-of-transaction ref writes throw immediately — there is no implicit wrapping.
- Nesting joins: an inner coordination inside an outer transaction shares the snapshot and the abort fate — composition-safe, but inner writes are not durable until the outer commit and any inner external effect replays on every outer retry.
- The body replays per conflict and an effect-shaped body's IO subtree re-evaluates per retry: bodies confine to coordinated-cell reads and writes, and the committed values flow into one post-commit continuation that runs the host-visible effect exactly once.
- Commit nests two retry loops with distinct replay triggers: the outer loop re-runs the body per conflict; the inner commit CAS on the global trie re-runs validation, write application, version increment, and commute rebase per inner contention without re-entering the body. Validators and commute functions must be pure and idempotent — both surfaces replay, on different triggers.
- Notification is an inverted-guarantee pair: a transactional cell's change event fires once per outer attempt, in attempt teardown, from the attempt-local change set — including aborted attempts — signaling intent, not commitment, decoupled from inner CAS retries; the single-cell atom fires exactly once, post-CAS, past the barrier, off the committed value. Host-visible effects — timers, metrics, marshaling, waiter wakes — therefore ride the coordination's return value in a post-settle continuation or a single-cell event, never a transactional cell's subscription, which is reserved for idempotent retry-tolerant observers.

[COMMUTE_REBASE]:
- A commute applies its function to the snapshot immediately for the body's benefit, then re-applies it to the latest committed value at commit: the committed result is the function rebased onto whichever base won the race, the body's read after the commute sees the tentative value, and the caller's return is read from the rebased committed state — the value received can differ from the value the body computed.
- The rebase is the concurrency unlock: commit-time application reads the live value, so a commute never conflicts on its own cell — two concurrent commutes both commit where two plain writes force one to spin. The price is commutativity under unspecified order, and a non-commutative function does not fail loudly — it degrades silently to last-application-wins. Increment, set-union, and max are correct commutes; "set to a value derived from a stale read" is silently wrong.
- A permit decision never reads a commuted cell: the in-body value is un-rebased and the committing value is rebased onto a base the decision never saw. Permits use the swap form, whose committed value equals the body value under the version check; commute is strictly blind accumulation no decision conditions on.
- A commute still increments the version and runs the cell validator on the rebased result: an accumulator with a capacity ceiling faults at commit even when the violation was produced by the rebase — higher concurrency never buys past an invariant, it only defers the check to the rebased value.

[BOXED_CARRIER_PUBLICATION]:
- The single-cell primitive publishes any `A` through one `volatile object` slot via a generated `Box<A>` whose accessor delegates are IL-emitted once per closed `A`: reference payloads pass through identity casts — zero allocation, zero indirection; struct payloads copy into one immutable heap box per published value. Where IL emission is unavailable the delegates degrade to closures with identical publication semantics at higher call cost — correctness survives restricted runtimes, only unbox-elision throughput is lost.
- Box immutability is the tear-freedom proof: a multi-word struct — a `(state, count, deadline)` triple — cannot be observed half-written, because the writer fully constructs the box before the compare-exchange and the slot only ever holds a complete box reference. The atomic unit is the reference swap, not the field write; the classic torn `volatile struct` field is structurally impossible here.
- No explicit fence call appears because the field qualifier carries the pairing: the volatile store publishes with release order, the volatile load acquires, the compare-exchange on every transition is a full barrier, and the box payload — fully written before the publishing store — is transitively visible to any thread observing the new reference.
- Box allocation is a load metric, not a constant: one box per committed publish plus one per lost CAS race, because the transform re-boxes per losing iteration, and equal values still allocate distinct boxes. A high-frequency struct cell whose box count grows under contention converts to a pre-boxed reference-wrapped carrier whose stable references are swapped, not re-boxed — the per-race allocation disappears.

[CAS_VISIBILITY_AND_WIDTH]:
- A successful compare-exchange is a full two-way barrier: prior stores retire before it, later loads cannot float above it, and each spin iteration re-reads the slot with full ordering — a loser's retry observes the winner's committed value, never a cached duplicate. The validator runs inside the loop before the swap, and a rejection publishes nothing — invisible to every slot reader; the post-loop event handler always reads a fully-published value.
- The map cell raises the ladder one level: an immutable trie in a volatile field, every mutation a whole-reference CAS. A point read is one acquire load plus an index — wait-free, lock-free, linearizable against concurrent writers; bulk readers capture the trie once and query a frozen consistent snapshot, while re-reading the field per query can observe a newer map mid-iteration. A ref read outside any transaction is the same acquire load of the global registry trie — the cheapest coherent read in the family; the transactional path is reserved for multi-cell commit.
- A waiter-wake for a settled transition fires past the same barrier that committed it, so the resumed waiter observes the published value — wake and value ordered by one barrier, not two; a wake driven from a per-attempt coordination event can release a waiter against a value that never committed. Parked initialization waiters complete only after the winning swap publishes.
- Narrow-width CAS — the `byte`, `sbyte`, `short`, `ushort` overloads — packs a small state machine into one byte and four such cells into one word; width is a memory-density decision and negative sentinel states force the signed widths. Co-residence is double-edged: independent high-frequency cells sharing a cache line ping-pong it between cores — false sharing serializes logically independent transitions — so pad those onto separate lines and pack together only cells that transition under one logical operation, the same cells a multi-ref transaction would coordinate anyway.
- The flag-word asymmetry is load-bearing: an atomic bit-or or bit-and sets or clears a property in one fence-free instruction with no spin and returns the prior word, but reading current flags requires an acquire-ordered read — `Volatile.Read` or a self-compare-exchange — because a plain load is incoherent on weakly-ordered hardware. A seqlock-style optimistic reader of a versioned payload reads the version with acquire ordering, reads the payload, re-reads the version and restarts if it moved — only acquire ordering on both version reads pins the payload load between them.

[UNFENCED_THREE_STATE_TEAR]:
- The three-state lazy memo cell — load progress in a plain `int` (unloaded, loading, loaded), payload in a plain typed field — carries the publication hole boxed carriers close: the winner's payload-then-state stores are unfenced, the fast path's loaded-check is a plain load that bypasses the claiming CAS's barrier entirely, and on weakly-ordered cores a reader can observe the loaded state with the pre-initialization default payload. The hazard is invisible on strongly-ordered hardware and passes every test bench there.
- The fault path reverts to unloaded and rethrows rather than caching, but the adjacent trap is the payload carrier: transient-failure retry semantics need an absence-backed carrier, never one that permanently caches a thrown initialization fault.
- Reset is the same tear in reverse — default-payload-then-state as two unfenced stores after its claiming CAS — and concurrent reset-versus-read is unsafe in general; concurrent waiters spin on the loading state rather than parking. Replacement beats mutation: clone a fresh cell from the same factory and swap it through an outer single-cell slot atomically.
- The fix space has two closes: an acquire load on the progress flag, pinning the dependent payload load beneath it, or migration to the boxed-carrier primitive whose slot-plus-CAS publishes state and payload as one indivisible reference — strictly safer because it deletes the two-field split the fence pair has to bridge.
