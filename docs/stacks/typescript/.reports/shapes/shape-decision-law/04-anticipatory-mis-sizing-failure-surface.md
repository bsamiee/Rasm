# Anticipatory Mis-Sizing Failure Surface

[SIZING_PROOF_IS_THE_NEXT_DIFF_PARTITION]:
- Correct sizing is not asserted, it is discharged: the proof obligation is the conceivable next-feature diff, and the obligation is met iff that diff sorts into exactly one of three outcome classes the owner's type already fixes — a within-family re-route that touches one declaration and no consumer, a cross-family break that fails every consumer at compile time, or the silent corruption class that is forbidden outright. A shape whose next-feature diff lands an edit in two of the three classes at once is mis-sized, and the partition is computable at declaration time from the owner's `Type`/`Encoded`/`Context` triple before any call site exists.
- The three classes are mutually exclusive by construction because each is gated by a different type-level mechanism: within-family zero-churn is the union-widening of `Schema.Union(...members)` (the new member's `Type` joins `Members[number]` and the prior members' types are unchanged), cross-family loud break is the constructor-shape change every `new`/`make`/`decode` site is checked against, and silent corruption is the case where the owner's runtime semantics shift while its static type is stable — the only class with no compile-time witness, hence the class the conservatism gradient exists to exclude.
- The obligation is discharged per axis, not per concept: the five-axis signature is read, and for each axis the agent asks which outcome class a flip of *that* axis produces against the chosen owner. A correct owner answers "within-family" or "loud cross-family" for every axis except the one irrecoverable axis, which it answers by having read that axis most conservatively at the first declaration. An owner that answers "silent" for any axis is rejected before it is written.

[WITHIN_FAMILY_RE_ROUTE_IS_A_TYPE_WIDENING]:
- The within-family class is decidable by one mechanical test the agent runs against the owner's posterior type: does the flip leave every prior member's `Type` and every prior call site's required argument shape unchanged. A flip passes iff its diff is purely additive at the type level — `Schema.Union`'s `Schema.Type<Members[number]>` widening to absorb one appended `TaggedClass`, or a new `Schema.decodeUnknown` site added to a `Schema.Class` whose codec was always present. The test is the proof: a non-zero differential on any prior position means the flip is not within-family regardless of how small the source diff looks.
- The within-family widening is itself split into two consumer sets the same one-declaration diff hits differently, and conflating them is the sizing misread: appending a union member widens the *producing* surface (a new `new`/`make` becomes available) with zero edit, while every *consuming* `Match.tagsExhaustive`/`$match` site fails to compile until its case record gains the arm. A diff that is zero-churn end-to-end — both producers and consumers untouched — is the signal the new case was not actually modeled (it routed to a default arm and is silently mishandled); the correct within-family widening is zero on producers and loud-and-total on consumers, and the consumer breaks are the exact inventory of sites that must reconsider.

```typescript
class PaintCircle extends Schema.TaggedClass<PaintCircle>('PaintCircle')('Circle', { r: Schema.Positive }) {}
class PaintRect extends Schema.TaggedClass<PaintRect>('PaintRect')('Rect', { w: Schema.Positive, h: Schema.Positive }) {}
const Shape = Schema.Union(PaintCircle, PaintRect) // the next variant is one appended arg; Type widens over Members[number]
const area: (s: Schema.Schema.Type<typeof Shape>) => number = Match.type<Schema.Schema.Type<typeof Shape>>().pipe(
    Match.tagsExhaustive({ Circle: (c) => Math.PI * c.r * c.r, Rect: (r) => r.w * r.h }),
) // appending a third TaggedClass to Shape fails this record's required-key check loudly; no prior arm edits, producers untouched
```

[CROSS_FAMILY_BREAK_IS_THE_DESIGN_GOAL]:
- Every axis flip that is neither of the two within-family widenings crosses an owner family, and the crossing manifests as a constructor-or-decode-shape change the checker propagates to every consumer — which is the intended outcome, not a regression. The fan-out is bounded and total: the broken sites are the complete inventory of code that depended on the prior shape, surfaced at once, and an unbroken site is the proof that site never touched the changed axis. The cost asymmetry is the law: a cross-family flip caught at declaration time is a re-route of one owner; the same flip discovered after consumers bind is the identical fan-out plus the hazard that some sites silently still type-check against the stale shape.
- The break is loud exactly when the flip alters the owner's `Type`, `Encoded`, or constructor argument — a structural-value owner gaining a discriminant (`Schema.Class` → `Schema.Union` of `TaggedClass`) changes the value's static type so every site reading a now-removed top-level field fails; a behavior table gaining equality (`as const` → `Data.struct`) changes the read from `Table[k]` indexed access to `Equal.equals` membership so every `keyof typeof` read fails. The flip that does *not* break loudly is the dangerous one, and the next clause names it.

[IDENTITY_REGIME_IS_THE_LONE_SILENT_AXIS]:
- Identity regime is the single axis whose flip can be silent, because `Equal`/`Hash` semantics are baked into the constructor at declaration and the static `Type` is unchanged when the regime shifts: a `Schema.Class` value and a hypothetical reference cell of the same fields present the identical `Schema.Schema.Type`, so a flip from structural to reference identity leaves every `Equal.equals` call still type-checking while its *runtime answer* inverts. There is no compile-time witness — the broken comparison sites do not fail to compile, they return the wrong boolean — which makes identity regime the lone irrecoverable axis and the sole setter of the conservatism gradient.
- The irrecoverability is structural: `Schema.equivalence(schema)` derives the comparator from the schema's `ast` at the moment the schema is declared, so the comparator cannot be re-derived by `extend` or indexed access after the fact — a regime flip forces a hand-migration of every comparison and every `HashMap`/`HashSet` keyed on the value, and that migration has no type error to drive it. Because the migration is undriven, the conservatism rule is absolute: when reference identity is *conceivable*, the structural-value owner is the wrong first route, and the value is shaped at the reference tier from the start even though every consumer in hand wants structural comparison.
- The silent-corruption hazard recurs one tier down inside the wire-variant fork: a `Schema.TaggedStruct` and a `Schema.TaggedClass` member of the same union present identical decoded `Type`, so a union member used as a `HashMap` key compiles whether it carries `Equal`/`Hash` or not — but a `TaggedStruct` key dedupes by reference and corrupts the map with no diagnostic. The identity axis is therefore read not only at the top-level owner but at every member of a union, and a union mixing the two tiers is the fused-concept signal because its members answer the identity axis differently while sharing one `Type`.

```typescript
class Money extends Schema.Class<Money>('Money')({ amount: Schema.BigDecimal, currency: Schema.String }) {}
const sameValue = (a: Money, b: Money): boolean => Equal.equals(a, b) // structural: bakes Equal/Hash into the Class ctor
const ledger = HashMap.empty<Money, number>().pipe(HashMap.set(new Money({ amount: BigDecimal.fromBigInt(1n), currency: 'X' }), 1))
// reject: a regime flip to reference identity leaves `sameValue`'s TYPE intact while its RUNTIME answer inverts —
// `ledger` would silently dedupe by reference; no `Equal.equals` site fails to compile, so the migration is undriven.
const reused: Equivalence.Equivalence<Money> = Schema.equivalence(Money) // comparator derived from the ast at declaration, never re-derivable post-hoc
```

[FAN_OUT_COST_IS_THE_OWNER_TYPE_DIFFERENTIAL]:
- The fan-out of a late flip is not estimated, it is read off the differential between the owner's prior and posterior `Type`: the broken-site count equals the count of consumer positions whose required type changed, and the within-family widenings have a zero differential on prior members by construction. The asymmetry the conservatism gradient encodes is that the differential is *recoverable and loud* for every axis except identity regime — a cardinality, payload-timing, codec, or constraint flip changes a type the checker chases — while the identity differential is *invisible* because the type is stable across the regime change. The gradient is therefore steepest on the one axis whose differential the checker cannot compute.
- The payload-timing axis sits one notch below identity on the gradient: a flip from per-call optionality to per-channel divergence re-routes a flat `Schema.Class` to a `Model.Class`, and the differential is loud because every projection a consumer reads (`X.insert`, `X.json`) either appears or its field set changes — but the flip is recoverable, because the field becomes a `Model.Field` config inside the same identifier and the consumers break at the projection boundary. The within-family path exists for this axis too: a field already declared with a `Model.Field` per-variant config absorbs a new channel as one alphabet member with zero consumer edit, the persistence analogue of the union-member widening.
- The codec axis has the smallest differential of the recoverable axes and a split fan-out: a `Schema.Class`'s async `Schema.decodeUnknown` is present from declaration so a none→wire flip on the async path is zero-churn, but the *synchronous* extraction surface (`decodeUnknownSync`, typed `Schema<A, I, never>`) is unavailable the moment a `filterEffect` lifts the owner's `Context` off `never` — so a service-consulting refinement added later breaks exactly the sync-decode sites and no others. The broken set is the precise inventory of sites that assumed pure decode, and a surviving sync site is one the new requirement never reached.

```typescript
class Profile extends Schema.Class<Profile>('Profile')({ id: Schema.UUID, bio: Schema.String }) {}
const wire: (u: unknown) => Effect.Effect<Profile, ParseResult.ParseError> = Schema.decodeUnknown(Profile) // present from declaration: none→wire is zero-churn
const sync: (u: unknown) => Profile = Schema.decodeUnknownSync(Profile) // typed Schema<A, I, never>: survives only while Context stays never
// a `filterEffect` lifting Profile's Context to a service breaks `sync` at compile time and leaves `wire` intact —
// the broken sync sites are the exact inventory of code that assumed pure decode; the conservatism cost is read off that set.
```

[OVER_SHAPING_COST_IS_ONE_UNUSED_ARM]:
- The conservatism gradient is asymmetric by design and the asymmetry is what licenses early over-shaping: the cost of sizing too wide is one unused union arm, one unread variant key, or one structural-equality instance no consumer compares — a single dead capability the demotion read strips at definition time with a loud signal. The cost of sizing too narrow on the irrecoverable axis is an undriven hand-migration across every comparison site. Because the two costs sit on opposite sides of the loud/silent boundary, the rule is not the symmetric "size for what you see": it is to read identity regime as if the widest conceivable comparison need is already real, and read every recoverable axis as if only the instance is real, because each recoverable axis carries its own loud break to drive its widening when it actually grows.
- The over-shaping cost is bounded and self-correcting precisely because it is loud, and this reduces the entire sizing decision to a single conservatism move: over-shaping on a recoverable axis is reversible at definition time (the dead-capability signal fires before consumers bind), while under-shaping on the irrecoverable axis is reversible at no time without an undriven migration. The gradient therefore collapses the five-axis read to one asymmetric posture — pessimistic on identity regime alone, instance-sized on the four recoverable axes — and any owner that inverts this posture (instance-sized identity, pessimistic elsewhere) has spent its conservatism budget on the four axes that did not need it and starved the one that did.

[CHECKABILITY_IS_THE_TOTAL_PARTITION]:
- The proof obligation is checkable rather than aspirational because the three outcome classes are total over the axis-flip space: every conceivable flip of every axis against any owner lands in exactly one class, and the classification is decidable from the owner's declared type alone. Within-family is decidable by asking whether the flip widens a union member or adds a decode site without changing prior types; cross-family is decidable by asking whether the flip changes the constructor or decoded `Type`; silent is decidable by asking whether the flip changes runtime `Equal`/`Hash` while leaving `Type` stable — and only the identity axis answers yes to the third question. The obligation is met iff the third class is empty for the chosen owner, and the conservative reading of the identity axis is the unique move that empties it.
- The checkability extends to the owner chooser itself: because the partition is a pure function of (axis, owner-tier), it is the same closed dispatch the routing law is, and a fourth outcome class appearing — an axis flip that is neither within-family, nor loud cross-family, nor identity-silent — is the signal a new owner family entered the vocabulary and the partition must gain an arm. The partition breaking on an unclassifiable flip is the absorbed-growth guarantee applied to the sizing proof: the obligation cannot be silently unmet, because an unclassified flip surfaces as a gap in a total dispatch rather than a runtime corruption.
