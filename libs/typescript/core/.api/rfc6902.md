# [rfc6902] — RFC 6902 JSON Patch + RFC 6901 Pointer: the `EntityEdit` diff/apply engine under a Schema-typed `Operation` union

`rfc6902` is the standards-exact JSON Patch (RFC 6902) and JSON Pointer (RFC 6901) engine `wire/codec/patch` composes as the `EntityEdit` egress/apply core. It carries three concerns: a closed `Operation` union discriminated on `op` (`add`/`remove`/`replace`/`move`/`copy`/`test`), a value-returning apply engine (per-op appliers plus `applyPatch` that returns one result slot per op — errors are VALUES in the return array, never thrown), and a structural diff engine (`createPatch`/`diffAny` with a Levenshtein array differ and a user-supplied `VoidableDiff` hook). The library is weakly typed at its edge (`any` on every payload), so the boundary law is fixed: `wire/codec/patch` wraps the `Operation` union in an `effect/Schema.Union`, decodes the C#-authored `JsonPatchDocument` shape (`Rasm.Persistence/Version`) once, dispatches per-op through `effect/Match.discriminatorsExhaustive("op")`, and folds the apply-result slots into the `effect/Effect` typed error channel — rfc6902 stays the pure engine underneath, never the boundary contract.

## [02]-[PUBLIC_TYPES]

[PUBLIC_TYPE_SCOPE]: the `Operation` union — the closed RFC 6902 vocabulary
- rail: boundaries/codec
- Six operation shapes discriminated on the `op` literal form the `Operation` union; `Patch` is `Operation[]`. This is DATA the `wire` owner lifts into an `effect/Schema.Union` of six `Schema.Struct`s (parse-not-validate) so the C#-minted `JsonPatchDocument` decodes once and every downstream fold is total on `op` — never a positional array read of an untyped operation.

| [INDEX] | [SYMBOL]                                                        | [TYPE_FAMILY]     | [CONSUMER / BOUNDARY]                                              |
| :-----: | :-------------------------------------------------------------- | :---------------- | :---------------------------------------------------------------- |
|  [01]   | `Operation` = `Add\|Remove\|Replace\|Move\|Copy\|TestOperation` | discriminated union | `wire/codec/patch` — mirror as `Schema.Union` keyed on `op`; `Match.discriminatorsExhaustive("op")` dispatch |
|  [02]   | `AddOperation` / `ReplaceOperation` / `TestOperation`           | value-bearing op  | `{ op, path, value }` — the `path`/`value` carriers; `TestOperation` doubles as the OCC precondition |
|  [03]   | `RemoveOperation`                                               | path-only op      | `{ op, path }` — the sole destructive-without-value shape         |
|  [04]   | `MoveOperation` / `CopyOperation`                               | relocation op     | `{ op, from, path }` — the two `from`-bearing shapes; `move` forbids `from` as a prefix of `path` |
|  [05]   | `Patch` = `Operation[]`                                         | patch document    | the `EntityEdit` payload; encodes field-for-field to the C# `JsonPatchDocument` array |
|  [06]   | `isDestructive(op): boolean`                                   | op classifier     | `wire/codec/patch` — gate rollback-safety; `remove`/`replace`/`move` mutate-or-drop existing state |
|  [07]   | `Diff` / `VoidableDiff`                                         | diff hook contract | `(input, output, ptr: Pointer) => Operation[] \| void` — the parameterized custom-diff seam `createPatch` accepts |

[PUBLIC_TYPE_SCOPE]: apply faults + `Options` — the value-rail failures
- rail: rails-and-effects
- The three fault classes extend `Error` but are RETURNED, not thrown: `applyPatch` yields `(fault \| null)[]` (one slot per op) and each per-op applier yields `fault \| null`. `wire/codec/patch` maps the three classes 1:1 onto a `Data.TaggedError` family so patch application rides the codec ROP rail; a raw `throw` never appears.

| [INDEX] | [SYMBOL]                                             | [TYPE_FAMILY]   | [CONSUMER / BOUNDARY]                                          |
| :-----: | :--------------------------------------------------- | :-------------- | :------------------------------------------------------------- |
|  [01]   | `MissingError { path }`                              | tagged fault    | `→ Data.TaggedError("PatchMissing")` — target location absent  |
|  [02]   | `TestError { actual, expected }`                     | tagged fault    | `→ Data.TaggedError("PatchTestFailed")` — the OCC precondition miss; carries both sides for the drift report |
|  [03]   | `InvalidOperationError { operation }`               | tagged fault    | `→ Data.TaggedError("PatchInvalid")` — `op` outside the closed set (a drift signal against `contract/drift`) |
|  [04]   | `Options { implicitArrayCreation?: boolean }`        | apply policy    | one policy value threaded through every applier; `true` lets `add "/tag/-"` mint a leaf array |

[PUBLIC_TYPE_SCOPE]: `Pointer` — the RFC 6901 traversal value
- rail: boundaries/codec
- `Pointer` is the reusable JSON Pointer — token list, escape/unescape, and a single `evaluate` that returns the `{ parent, key, value }` triple every applier resolves against. `wire/codec/patch` reuses it rather than re-implementing `~0`/`~1` token escaping or path traversal.

| [INDEX] | [SYMBOL]                                                    | [TYPE_FAMILY]   | [CONSUMER / BOUNDARY]                                          |
| :-----: | :--------------------------------------------------------- | :-------------- | :------------------------------------------------------------- |
|  [01]   | `Pointer { tokens: string[] }`                             | pointer value   | RFC 6901 path as decoded tokens; immutable `add`/`parent` derive children |
|  [02]   | `Pointer.fromJSON(path)` / `.toString()`                  | codec           | escaped-string ↔ token-list crossing; the `path`/`from` field decode |
|  [03]   | `Pointer.evaluate(object): PointerEvaluation`             | resolve         | `{ parent, key, value }` — the empty-pointer case returns `{ parent: null, key: "", value: object }` |
|  [04]   | `Pointer.get` / `.set` / `.push` / `.add` / `.parent`     | traversal       | leaf read/write and immutable-shallow navigation             |
|  [05]   | `escapeToken` / `unescapeToken` / `PointerEvaluation`     | token codec     | `~1`→`/` then `~0`→`~` ordered unescape; the escaping `add`/`test` paths cross verbatim |

## [03]-[ENTRYPOINTS]

[ENTRYPOINT_SCOPE]: egress — compute the minimal patch + OCC test-guards
- rail: boundaries/egress
- `createPatch(base, next)` is the `EntityEdit` egress: the minimal `Operation[]` transforming `base` into `next`, encoded to the C# `JsonPatchDocument`. `createTests` captures a pre-image as `test` ops so the egress carries its own optimistic-concurrency proof. The `diff?` argument is the ONE parameterized reconciliation seam — a domain differ (content-key array match) with `diffAny` fallback, never a per-shape differ roster.

| [INDEX] | [SURFACE]                                                                 | [ENTRY_FAMILY] | [CONSUMER / BOUNDARY]                                     |
| :-----: | :------------------------------------------------------------------------ | :------------- | :-------------------------------------------------------- |
|  [01]   | `createPatch(input, output, diff?: VoidableDiff): Operation[]`            | diff / egress  | `wire/codec/patch` — the `EntityEdit` minimal patch; `diff` supplies content-key array reconciliation |
|  [02]   | `createTests(input, patch: Operation[]): TestOperation[]`                 | precondition   | emit `test` ops guarding a base pre-image; the CAS proof the `state/causal` OCC append verifies |
|  [03]   | `diffAny(input, output, ptr, diff?)` / `diffObjects` / `diffArrays<T>`    | differ         | the type-dispatched cores; `diffArrays` is the Levenshtein sequence differ `createPatch` falls back to |
|  [04]   | `subtract(minuend, subtrahend)` / `intersection(objects)`                | key algebra    | JSON-semantic own-key set ops (undefined-dropping) the object differ builds on |

[ENTRYPOINT_SCOPE]: apply — the value-rail patch application
- rail: rails-and-effects
- `applyPatch` applies a whole document and returns one result slot per op; the per-op appliers apply a single operation. rfc6902 MUTATES the target in place, so `wire/codec/patch` clones (`util.clone` or an `effect/Data` structural copy) before applying to hold the decode-once immutability law, then folds the result slots into the `Effect` channel.

| [INDEX] | [SURFACE]                                                                              | [ENTRY_FAMILY] | [CONSUMER / BOUNDARY]                                     |
| :-----: | :------------------------------------------------------------------------------------- | :------------- | :-------------------------------------------------------- |
|  [01]   | `applyPatch(object, patch, options?): (MissingError\|TestError\|InvalidOperationError\|null)[]` | apply-all | one slot per op; a non-`null` slot folds to a tagged fault |
|  [02]   | `apply(object, operation, options?): MissingError\|InvalidOperationError\|TestError\|null` | apply-one   | the `op`-switch applier; the single-operation entry `Match` mirrors |
|  [03]   | `add` / `remove` / `replace` / `move` / `copy`: `(object, op, options?) => MissingError\|null` | per-op mutate | the five mutating appliers; `move`/`copy` resolve two pointers |
|  [04]   | `test(object, op, options?): TestError\|null`                                          | precondition   | the non-mutating equality guard; the applier form of the OCC check |

[ENTRYPOINT_SCOPE]: `util` — structural helpers
- rail: algorithms
- Value helpers the apply/diff cores share; `clone` is the pre-apply defensive copy site.

| [INDEX] | [SURFACE]                                             | [ENTRY_FAMILY] | [CONSUMER / BOUNDARY]                                     |
| :-----: | :--------------------------------------------------- | :------------- | :-------------------------------------------------------- |
|  [01]   | `clone<T>(source: T): T`                             | deep copy      | pre-apply immutability guard before a mutating `applyPatch` |
|  [02]   | `objectType(v)` / `hasOwnProperty(v)`               | reflection     | the array-vs-object-vs-primitive dispatch the differ keys on |

## [04]-[IMPLEMENTATION_LAW]

[PATCH_TOPOLOGY]:
- rfc6902 is the pure engine, `effect/Schema` is the boundary: the `Operation` union and `Options` are `any`-typed at the library edge, so `wire/codec/patch` owns a `Schema.Union` of six `Schema.TaggedStruct`s keyed on `op` (with `path`/`from` as `Schema.String` and `value` as the entity's decoded shape). The C# `JsonPatchDocument` decodes ONCE through that Schema; downstream code never touches a raw rfc6902 operation. This is the same decode-once law every `wire/codec` page holds.
- errors are values, folded to a rail: `applyPatch` returns `(fault\|null)[]` — no `throw`. The owner zips the slots against the patch, maps `MissingError`/`TestError`/`InvalidOperationError` to a `Data.TaggedError` family (`PatchMissing`/`PatchTestFailed`/`PatchInvalid`), and lifts the first non-`null` slot into the `Effect` error channel. `TestError` carries `{ actual, expected }` for the OCC drift report.
- diff is parameterized, not enumerated: `createPatch(base, next, diff?)` takes ONE `VoidableDiff` hook that owns every domain-specific reconciliation (content-key array match, id-stable object diff), returning `void` to fall back to the built-in `diffAny`. A new entity family is a hook arm, never a new differ function — the array/object/primitive split already lives inside `diffAny`.
- mutation is fenced by a clone: rfc6902 applies in place; the owner clones the target (`util.clone` or `effect/Data` structural copy) before `applyPatch`, so the decoded value stays immutable and the apply produces a new value on the rail.

[STACKS_WITH]:
- `effect` (`.api/effect.md`): the `Operation` union becomes a `Schema.Union`; `Match.discriminatorsExhaustive("op")` dispatches per-op with exhaustiveness; `Data.TaggedError` carries the three apply faults; `Schema.encode` re-serializes `createPatch` output to the C# `JsonPatchDocument`. `Effect.map`/`Effect.fromNullable` fold the result slots.
- `wire/codec` siblings + `contract/drift` (`wire/.api/`): the wire folder is multi-codec and `codec/patch` is the JSON-Patch arm — `@bufbuild/protobuf` owns the descriptor-typed proto families (`ElementGraph`, `FaultDetail`, `GeometryPayload`, the RPC suite), `cbor-x` the canonical `SnapshotHeader` CBOR, `@msgpack/msgpack` the `CrdtOpWire`/`OpLog` MessagePack union, this the `JsonPatchDocument` egress; a `codec` page picks ONE codec by the C# mint format and a proto page never reaches for JSON-Patch. An `InvalidOperationError` (op outside the closed six) is a schema-drift signal — surface it through `contract/drift` as a typed verdict, not a decode crash; the descriptor-level drift gate over the proto families is `@bufbuild/protobuf`'s reflection altitude, not this engine's. `Pointer` reuses the same RFC 6901 notion the graph/version codecs address by path.
- `state/causal` + `store/journal` (`state/.api/`, `store/.api/`): the `createTests` pre-image `test` ops ARE the optimistic-concurrency guard — the `EntityEdit` egress ships its own base-state proof, and the `store/journal` OCC append (version-vector checked) rejects a stale patch before applying. The patch is self-guarding across the C# `Rasm.Persistence/Version` seam.
- `kernel/identity` (`kernel`): a patch document is content-addressable — hash the encoded `Operation[]` through the one `XxHash128` seed-zero mint for the `EntityEdit` content key; rfc6902 never mints its own identity.

[LOCAL_ADMISSION]:
- Decode inbound patches through the `Schema.Union` mirror of `Operation`; never pass a raw C#-minted array into `applyPatch` untyped.
- Fold `applyPatch` result slots into the `Effect` channel through the `Data.TaggedError` map; never inspect the returned `Error` instances imperatively or re-`throw` them.
- Supply the `VoidableDiff` hook for domain-aware reconciliation; never post-process `diffAny` output or maintain a per-shape differ.
- Clone before apply; never mutate a decoded value in place.
- Emit `createTests` guards on the egress where the C# side runs an OCC append; never ship an unconditional patch across the version seam.

[RAIL_LAW]:
- Package: `rfc6902`
- Owns: the RFC 6902 `Operation` union + `Patch`, the value-returning apply engine (`applyPatch` + per-op appliers + the three fault classes), the structural diff engine (`createPatch`/`createTests`/`diffAny`/`diffArrays`/`diffObjects` + key-set algebra), and the RFC 6901 `Pointer`
- Accept: rfc6902 as the pure engine behind a `Schema.Union` boundary, `createPatch` for `EntityEdit` egress, `createTests` for OCC precondition proofs, the `VoidableDiff` hook for domain reconciliation, result-slot folding onto a `Data.TaggedError` rail, `clone` before a mutating apply
- Reject: a raw `applyPatch`/`createPatch` on undecoded `any`, thrown apply faults or imperative `Error` inspection, a per-shape differ roster where one `VoidableDiff` hook fits, an in-place mutation of a decoded value, a hand-rolled JSON Pointer traversal or a second content-address notion for a patch document
