# Shape Decision Law

[OWNER_SELECTION]:
- Five discriminants decide the owner before any attribute is written: admission (does raw material cross a trust boundary here), identity regime (key, structural, case, or reference), variant arity (one shape vs N alternatives), payload timing (case data fixed at declaration vs constructed per occurrence), and openness (closed vocabulary vs foreign extension). Every misplaced shape traces to one mis-answered discriminant.
- A scalar carrying an invariant, normalization, or non-default equality earns the keyed value object, and the owner publishes domain semantics, never the raw key. The generated key member defaults to a private field erased at key-typed use sites through implicit conversion — frictionless evidence erasure is the type's own design, so code operating on the raw key is operating on erased evidence and domain semantics propagate only on the owner type; a public key property is an opt-in, not the default.
- N fields forming one concept under structural identity earn the complex value object. The sum-encoded-as-product test rejects it: when any field is meaningful only under some discriminant value, the concept is a union regardless of field count.
- A bounded vocabulary whose case data is fixed at declaration earns the keyed smart enum when identity must cross a wire, and the keyless form when the vocabulary exists purely for process-local behavior selection — keyless generates no serialization without an object factory, so wire identity is a declared exception, never an ambient capability. Items are singletons; any per-occurrence payload disqualifies the shape and pushes to a union.
- Singleton-vs-instance is the sharpest smart-enum/union discriminant: a smart enum is a fixed set of named instances dispatched by identity, a union a fixed set of shapes dispatched by case. Vocabulary that starts sprouting constructor arguments at use sites has already become a union.
- Closed alternatives with per-occurrence payload and shared base semantics earn the regular union; one value ranging over two to five unrelated existing types with no shared invariant earns the ad-hoc form. The regular/ad-hoc split is relatedness of cases — do the cases carry meaning outside this union — never arity or convenience. Value object vs ad-hoc union over the same data resolves on invariant presence: the ad-hoc form is structural pass-through and validates nothing.
- Duplicate member types in an ad-hoc union are a deliberate modeling tool: the same representation under two names yields semantically distinct slots whose only ingress is the named factories and whose equality discriminates the slot before the payload — equal payloads under different names are unequal values, and the state a wrapper pair or boolean tag would have smuggled in lives in the discriminator:

```csharp
[Union<string, string>(T1Name = "Draft", T2Name = "Published")]
public partial class Copy;

static bool Promoted(Copy before, Copy after) =>
    before.IsDraft && after.IsPublished && before.AsDraft == after.AsPublished;
```

- An ad-hoc union whose members are call modalities, declared with dispatch generation deleted, turns a modality into a stored value that can be queued, logged, and replayed — capabilities an overload set structurally lacks because the modality dies at overload resolution; the same deletion makes arm-style consumption inexpressible on transport-only carriers interpreted elsewhere, so non-enumerable consumption is a type fact rather than a convention.
- Interior product data with no invariant, no admission, and no identity beyond structure stays a plain record or readonly record struct. Generated attributes on such a type buy analyzer ceremony — partial split, private constructor, readonly enforcement, comparer demands — plus a factory that decides nothing.
- A type owns exactly one shape: duplicate or mixed family attributes are rejected outright, and an ad-hoc union requires at least two members — degenerate one-member unions cannot exist.

[BOUNDARY_FACING_SELECTION]:
- A wire contract earns a protocol DTO record only when its topology diverges from the domain owner — different field graph, versioned members, protocol naming. When the wire shape is one scalar, an object factory deletes the DTO entirely and the owner carries the protocol projection as a declared aspect; a complex value object plus a string-valued factory collapses a structural concept to a parse/format micro-grammar — one textual representation owned by the type instead of a field-wise DTO plus mapper pair.
- Wire capability defaults on for keyed owners — the key representation is the contract across JSON and binary stacks unless severed — so the separation decision is explicit in both directions: severing hands the wire to a protocol DTO; leaving the default deletes the DTO. The decision is made at the owner declaration, never discovered at the edge.
- Neither union flavor is wire-capable alone: no type discriminator is emitted, so polymorphic serialization does not exist by construction. A closed alternative set whose cases must round-trip a boundary either admits a single-scalar grammar through an object factory, re-shapes as a keyed owner where the key is the discriminator, or accepts an edge DTO — a union whose cases cannot fold into one grammar is interior-only by construction, and that constraint runs backward into owner selection.
- A language enum remains the correct owner in exactly three places, all boundaries: mirroring a wire schema a foreign protocol owns (converted to vocabulary at admission), bit-flag interop where a foreign ABI owns the bit layout, and ordinal array indexing inside measured kernels. Its disqualifying property is open admission — any integral cast produces an undefined value — so an enum that travels past the boundary smuggles unvalidated input inward; conversion to the closed vocabulary at the admission seam is the re-closing move.
- Combinable capability sets split by where per-flag behavior lives: a flags enum has nowhere to put it, so flag-test chains re-derive policy at every consumer, while vocabulary items held in a frozen or immutable set carry behavior as columns — membership is set algebra, policy is a fold over member items.
- Re-validation cost makes churn rate a placement discriminant: a value rewritten per frame or per solver iteration does not belong in a validated owner — accumulation lives in a plain record or struct, and admission happens once at the seam where the result becomes domain material. Admitting inside a hot loop pays the invariant on every step for evidence nothing consumes mid-loop; the decision variable is reads-of-evidence per write, not type aesthetics. Generated owners delete the external update surface structurally — constructors private, initializers forced private, members forced required and read-only — so member rebinding and `with`-style cloning are inexpressible outside the owner even on struct owners, and every derived state is a new admission.

[SHAPE_SPAM_INVERSIONS]:
- Three or more sibling types sharing fields for one concept collapse into one closed family; the union case list replaces the type list and analyzer-enforced closure prevents regrowth.
- The nullable payload bag — one record, N nullable fields, a kind discriminator — encodes a sum as a product: every consumer re-derives the discriminant–field correspondence the type system already knew, every read is a null gamble, and a new kind extends the bag silently. The union's exhaustive dispatch deletes every null check and breaks every call site when a case is added — the failure mode inverts from silent to loud.
- The enum-plus-dictionary pair splits one vocabulary across two half-owners: the language enum admits undefined values by cast, the dictionary lookup is a partial function, and the two drift independently. A smart enum closes admission with unknown keys typed as errors and stores the policy as constructor state or behavior columns in the same declaration — one owner, total dispatch, no drift surface.
- Owner-per-protocol duplication — a domain type shadowed by a JSON type, a persistence type, and a binding type — collapses into one owner plus object factories partitioned by framework; the partition is analyzer-policed (overlapping framework claims rejected, persistence and binding routes singular), which is what makes the collapse safe by construction.
- A generated owner that validates nothing, applies default comparer policy, and never crosses a boundary decides nothing — the record was already correct. The inverse is equally spam: an anemic record sitting at a boundary scatters its missing validation across every call site as ad-hoc checks.
- A two-case ad-hoc union whose cases are outcome markers rather than domain payloads duplicates an existing result rail; the union form earns existence only when both cases are domain material in their own right.

[MANUAL_OWNER_EXEMPTIONS]:
- Generated families fix an expression-problem stance: the case axis is closed by analyzer law while the operation axis stays open — extension members attach receiver-owned operations to a closed owner from outside, dispatching internally over the generated exhaustive forms. The manual-owner exemption is therefore narrower than "extension point implies manual owner": it triggers on case extension alone, never on operation extension. An extension point foreign code must populate with new cases is an interface or abstract hierarchy by construction — forcing a generated owner there produces a closed family plus an escape hatch, which is two owners.
- Generic case payloads inside a regular union are rejected: either lift the type parameter to the union itself — attribute positions bind to the declaring type's own parameters across all keyed attributes, computing generic owners from one declaration, with key-position parameters required `notnull`, unreferenced parameters flagged, and `allows ref struct` parameters rejected on ad-hoc unions — or write the hierarchy manually with hand-rolled closure (private constructors, sealed cases) mirroring the analyzer's rules.
- An owner must be a class or struct; a concept that must surface as an interface — for variance, for foreign implementation, for host contracts — is outside the generated families entirely and takes the manual route with the closure disciplines applied by hand.
- Ad-hoc arity caps at five members; past it the concept either has shared semantics (regular union) or is not one concept. The cap is a hard generator bound, not a style threshold.
- Runtime-sourced vocabulary (configuration, database rows) cannot be smart-enum items — items are compile-time static fields. The owner becomes a keyed value object plus a frozen registry admitted once at startup; the registry is the manual analogue of the generated lookup.
- Per-item behavior needing generic type parameters cannot ride behavior columns; derived item types carry it through ordinary virtual dispatch inside the closed item hierarchy. One private generic case class closed at several type arguments yields type-indexed items: distinct items carry distinct compile-time type payloads while the vocabulary stays non-generic — the route for per-item type evidence without genericizing the enum or storing runtime type objects.
- Vocabulary graphs — items referencing sibling items, as in transition tables — take lazily evaluated list-valued constructor columns: eager sibling references resolve to null for items not yet initialized in declaration order, and the lazy column defers resolution to first read. Edges between items are a column pressure, not a union pressure.
- Ref-struct unions are declared, not parameterized: the union type itself may be a ref struct (equality generation suppressed; dispatch still emitted with stack-only state and results), but span-carrying alternatives exist only as concrete declarations.
- Smart enums may extend a foreign base class — the generated constructor threads key first, own columns second, base parameters last — so host-mandated base types are not a manual-owner pressure.
- Owners nest anywhere, but every enclosing type must itself be partial — a non-partial container is a generation-blocking compile error, not a degraded mode.

[CHANGE_DETONATION]:
- Each owner kind carries a fixed compile-break signature, and owner selection is selection of where change detonates: a new union case lands at every exhaustive dispatch site (the generated dispatch signature changes, so every consumer fails until it answers, with arms bound by named argument so arm order is irrelevant and case renames are loud); a new smart-enum item with behavior columns lands at the item declaration itself, because the generated constructor demands every column before the item compiles; a new complex-value-object member lands at every factory call; a key-type migration on a keyed owner lands only at conversion, serialization, and persistence seams, because interior consumers hold the owner and never the key — key encapsulation converts a pervasive break into a boundary break.
- Growth absorption is tunable to totality: a vocabulary whose behavior lives entirely in columns and whose dispatch generation is deleted absorbs item growth with zero consumer breaks — the owner is the only file that changes — while the same vocabulary with generated dispatch pushes every addition outward to all call sites. The setting is a declared stance on who pays for growth.
- Arm names and the dispatch state-parameter name are source contract, not style: arms bind by named argument, so renaming a case — or changing the owner-configured state-parameter name — is a deliberate source-breaking act with the same gravity as deleting a method.
- Invariant tightening on a value object is the one change with zero compile signal: the factory narrows silently, and values persisted under the old invariant now fail rehydration. The validation-bypassing rehydration constructor exists exactly for this seam — stored data outlives invariant drift — so an invariant change is a data-migration decision with a code rider, and the proof obligation moves entirely to tests because the type system stays quiet.

[COMPOSITION_AND_NESTING]:
- The owner kinds are closed under nesting: a keyed owner is a structural owner's member, any owner is a union case's payload, a vocabulary item carries another owner as an instance column, and a frozen table keys on a tuple of owners. Every member of a generated owner is either a primitive the language closes or another owner that closes itself, so a domain graph assembled entirely from owners has one provable totality story rather than a patchwork of validated and unvalidated seams.
- The enclosing owner inherits its members' capabilities by structural composition, never re-declaration: the generated factory initializes through already-admitted member instances and generated equality delegates to each member's own equality, so admission, comparer policy, and default-safety arrive free at the outer layer. The outer factory validates only the cross-member relations no single member can see — re-validating a nested member reads as defensive but duplicates a proof the type already carries, widens the outer factory with checks that cannot fail, and couples the outer invariant to the inner rule so that tightening the inner rule silently changes outer factory behavior.
- Admission composes depth-first with a monotonic trust gradient: constructing the outer owner forces every inner owner through its own factory first, so an aggregate of N nested owners admits raw material at exactly N seams, each owning its slice of the invariant. Once a value is the outer owner it carries the conjunction of every inner proof; interior code that re-checks any layer is operating below the gradient.
- Flattening a nested aggregate into one wide structural owner with leaf primitives as direct members is the inverse of collapse: it discards every inner owner's admission, comparer, and default-safety and re-scatters those invariants as ad-hoc checks across the outer factory. The flattening pressure usually traces to a wire shape, and the correct move is one object factory projecting the aggregate to its wire grammar while the interior keeps the composed spine.
- An owner keyed by another generated owner reaches its full derived surface only across an assembly boundary, because a generator cannot observe another generator's output in the same compilation. The same-compilation alternative composes at the boundary instead of in the declaration: the outer owner carries the raw key type, a separate owner validates that key's vocabulary, and one admission expression runs the inner admission then the outer — every derived surface intact at the price of one extra admission hop, a priced trade against an unpriced, undiagnosed capability downgrade.
- Conversion direction composes into a typed gradient: owner-to-key is implicit (erasing evidence is free), key-to-owner is explicit (creating evidence validates), throw-capable downward conversions are segregated under an explicitly unsafe name, and ad-hoc members keep the same polarity (ingress implicit, egress explicit). Projecting an aggregate down to its innermost primitive is therefore a chain of implicit erasures and reconstructing it upward a chain of explicit admissions, uniform across structural and union layers — a reconstruction spelled without the explicit casts does not compile.
- One accessibility knob governs the whole ingress surface — constructors, conversion operators, and factory methods together — token-gating everything to a hand-written factory on the owner. The defaults encode primary use per kind: value objects start private (admission-first by default), unions start public (structural pass-through) and require explicit tightening. Under nesting the knob composes into one funnel: the outer owner can hand-write the sole ingress while each member independently token-gates its own, so the aggregate's admission funnel is private at every layer except the deliberately exposed verb — no layer leaks a public constructor through which interior code could synthesize an inner owner past its invariant.

[DEFAULT_SAFETY_LATTICE]:
- Default-hostile owners mark themselves through an interface whose presence the analyzer enforces at default-initialized variables; because an owner can never be a compile-time constant, optional parameters of owner type are inexpressible — absence enters as an option-typed parameter or an overload, never `= default`.
- Default-hostility propagates as an interface fact, not a local declaration: the enclosing struct's generation walks the full interface set of every member transitively, so one default-hostile member at any nesting depth poisons the entire product upward through the closure, and the computed state overrides any requested permission. There is no mixed lattice — an outer struct cannot allow a default an inner member forbids — so a nested aggregate either stays struct-and-default-safe down its whole spine or takes the class form at the first default-hostile member.
- A struct that legitimately has a default opts in explicitly and names the default through a declared instance property — a canonical named instance replaces an anonymous bit pattern, and the name documents why default is legal.
- Struct/class selection on value objects is an absence-semantics decision: a class admits null at the factory edge with declared null-yielding policies absorbing degenerate input into null instead of an error; a struct trades away absence for layout and inherits the default lattice above.
- A struct ad-hoc union's discriminator reserves zero for uninitialized, and a default instance throws from dispatch, projection, and the typed accessors — at use, not at creation. Expression-level default use is analyzer-flagged, but array allocation and collection growth materialize defaults invisibly, and the breach inherits down the nesting: a struct owner hydrated from pooled or array storage materializes the zero pattern of every nested struct member at once, so the liveness gate is placed once at the outermost storage seam for the whole spine, never per layer — or every layer takes the class form and the question dissolves into the null channel.

[IDENTITY_REGIME_STACKING]:
- Identity has three regimes and the owner choice fixes the regime: key identity (keyed value objects and smart enums — equality, hash, and ordering all delegate to the key), structural identity (complex value objects and ad-hoc unions — member-wise under declared comparer policy), and case identity (regular unions — the generator emits no equality for the hierarchy; record cases get value equality, class cases reference equality, and a mixed hierarchy silently mixes regimes per case, so case-declaration form is an identity decision).
- The regimes do not blend across nesting — they stack, each owner owning exactly its own level: structural identity at an outer layer dispatches into key identity at an inner layer through the member's own equality, and case identity compares runtime case first, deferring to payload equality only within the matching case. No regime conflict is possible because no owner reaches through a member to its internals.
- The case-insensitive string default is the one regime fact that leaks silently across layers: string comparison on structural owners and union members defaults to ordinal-ignore-case — two values differing only in case are equal and collide in hash containers — and the stance is declared per owner and never inherits. An ordinal-sensitive outer layer over an inner string-keyed member that left the default yields an aggregate whose layers disagree on what equal means — unequal at the outer comparison yet colliding at the inner hash. The sensitivity stance must be declared at every string-bearing layer, with the policy value derived from one declared accessor type so the per-layer declarations cannot drift: repetition of the declaration is mandatory, divergence of the policy is the defect.
- The equality-and-ordering pairing obligation — comparers declared together or hashing and sorting disagree on identity — does not telescope: an under-specified inner member lets hashing and sorting disagree at that layer alone while every outer layer stays consistent, and the inconsistency is invisible until exactly that member participates in a sorted container. Identity-without-order is a legal stance only when declared at the layer that holds it; no outer owner can grant or repair it.

[DISPATCH_GRANULARITY_GROWTH]:
- Dispatch granularity over a nested hierarchy is declared at the owner: stop-at overloads treat an intermediate union as a single arm without descending, while an intermediate node carrying its own union attribute generates its own dispatch hiding the root's — so the static type of a reference selects granularity, branch-typed references dispatching over branch cases and root-typed references over root arms. Node-owned dispatch and root-level stop-at overloads are complementary granularities over the same tree; nested arm naming is itself declared policy, with leaf-name mode turning cross-branch name collisions into compile errors.
- Exhaustiveness composes multiplicatively under nested dispatch, and the compile-break sets of the layers are disjoint because arms bind by named argument at distinct dispatch surfaces: adding a root case breaks every root-granularity site, adding a branch case breaks every branch-granularity site, and neither sweep touches the other's consumers. Choosing a consumer's dispatch granularity is therefore choosing which growth axis it is coupled to — branch-granularity consumers are immune to sibling-branch growth, root-granularity consumers immune to within-branch growth.
- Generated dispatch admits stack-only state and results at every nesting level — state threads first-position with static-lambda discipline — so a span can thread an outer dispatch whose arm re-dispatches an inner union with the same span, capture-free the whole way down: a catamorphism over a nested owner graph with zero allocation.
- A recursive owner folds two non-interfering ways on one declaration — case payloads may hold the union type itself — and the placement decision is which change-propagation behavior the operation's correctness wants: case-owned recursion through overridden members never changes a call site (a new case cannot compile until it answers), while dispatch-route recursion breaks every call site on growth.

[BEHAVIOR_PLACEMENT_TABLES]:
- Behavior placement follows ownership of the key that selects it: call-site-varying behavior rides generated dispatch; behavior intrinsic to the case with a uniform signature rides case-owned members — behavior columns on vocabulary items, abstract-member overrides on union cases; behavior keyed by something the family does not own rides a table. With dispatch generation deleted, case-owned members become the only consumption form and the family grows fully owner-locally; partial-coverage dispatch forms exist only when the owner opts in, so non-exhaustive consumption of a family that must always be totally handled is inexpressible at the type, not discouraged at call sites.
- Frozen tables earn existence in exactly three places: cross-product keys (case-by-case correspondences no single family owns), externally sourced policy admitted at startup, and keys outside any generated family. Everything else is a generated projection, a behavior column, or an instance property — secondary correspondences derive from the primary family instead of riding a parallel dictionary that can drift.
- A frozen table keyed by generated vocabulary regains the closure the dictionary form lost: totality is provable at composition time by comparing table cardinality against the product of the vocabularies' item counts, converting the partial-function risk of lookup tables into a one-time startup invariant instead of a per-call gamble. Raw-primitive keys forfeit the proof — nothing enumerates their domain:

```csharp
[SmartEnum<string>]
public sealed partial class Region
{
    public static readonly Region East = new("east");
    public static readonly Region West = new("west");
}

[SmartEnum<string>]
public sealed partial class Tier
{
    public static readonly Tier Basic = new("basic");
    public static readonly Tier Premium = new("premium");
}

public static class Rates
{
    private static readonly FrozenDictionary<(Region, Tier), decimal> Table =
        new Dictionary<(Region, Tier), decimal>
        {
            [(Region.East, Tier.Basic)] = 0.10m,
            [(Region.East, Tier.Premium)] = 0.25m,
            [(Region.West, Tier.Basic)] = 0.12m,
            [(Region.West, Tier.Premium)] = 0.28m,
        }.ToFrozenDictionary() switch
        {
            { Count: var n } table when n == Region.Items.Count * Tier.Items.Count => table,
            _ => throw new InvalidOperationException("<rate-table-incomplete>"),
        };

    public static decimal Of(Region region, Tier tier) => Table[(region, tier)];
}
```

- Cross-product keys ride value tuples for positional pairs and graduate to a readonly record struct once slots deserve names; both hash structurally through the items' generated equality, so vocabulary values are table-key-safe by construction and never need a registered comparer. Frozen construction specializes the implementation to the key population at build time and is paid once at static initialization — the read-optimized trade is the declared reason a policy table is a static singleton and never rebuilt per scope; string-keyed tables additionally expose span-keyed alternate lookup, deleting the allocation on parse-shaped probe paths.

[OPERATOR_AXIS_LATTICE]:
- The six per-owner operator axes are a per-operation grant set, never an "is it a number" toggle, and the enabled set is a statement of the quantity's group structure: the axes that are on are exactly the binary operations under which the quantity is closed. A like-quantity owner grants addition and subtraction and holds multiply and divide at none, because a like-quantity product changes dimension — and no axis can express a dimension-changing operation, since every generated operator is homogeneous owner-op-owner or key-typed owner-op-key returning the owner. Cross-dimension operators are hand-written against the foreign result type; they are the part of the quantity's algebra that leaves the type:

```csharp
[ValueObject<decimal>(
    AdditionOperators = OperatorsGeneration.DefaultWithKeyTypeOverloads,
    SubtractionOperators = OperatorsGeneration.DefaultWithKeyTypeOverloads,
    MultiplyOperators = OperatorsGeneration.None,
    DivisionOperators = OperatorsGeneration.None)]
public readonly partial struct Meters;

[ValueObject<decimal>(MultiplyOperators = OperatorsGeneration.None)]
public readonly partial struct SquareMeters;

static SquareMeters Area(Meters width, Meters height) =>
    SquareMeters.Create((decimal)width * (decimal)height);

static TQuantity Total<TQuantity>(IEnumerable<TQuantity> parts, TQuantity zero)
    where TQuantity : IAdditionOperators<TQuantity, TQuantity, TQuantity> =>
    parts.Aggregate(zero, static (acc, next) => acc + next);
```

- Each granted axis declares the corresponding generic-math interface, with key-typed overloads adding the heterogeneous implementations — comparison and arithmetic against the raw key without converting either side, keeping the owner in play at use sites that would otherwise cast out.
- Every granted arithmetic axis emits both the unchecked operator and its checked counterpart when the key exposes both, so the owner inherits the platform's two-mode arithmetic contract whole: overflow-trapping is a property of the call site's checked context, never a second owner type and never a flag on the value. The duality is what makes a quantity owner a sound substitute for its key under checked-disciplined accumulation — an evidence wrapper that lost the checked mode would silently downgrade every checked algorithm it flows through; the generated pairing forecloses that downgrade.
- An axis is silently inert under two independent conditions, and both read as capability the type does not have: generation per axis is conditional on the key type itself implementing the interface or exposing the operator method, so an axis declared over a non-arithmetic key emits nothing; and the whole operator surface is downstream of admission, so factory suppression forces every arithmetic axis to none regardless of declared values, because the operator bodies re-admit their results through the factory. A no-effect axis declaration is a latent lie worth deleting.
- Factory suppression cascades through every derived capability, proving none is free-standing: it strips the type converter, the factory marker interface, parse interfaces, arithmetic operators, key-to-owner conversion, and all serializer converters (unless an object factory carries serialization independently) — every boundary capability derives from admission.
- The comparison and equality-comparison axes form one monotone chain, not two free knobs: comparison strength can only pull equality strength up to match, the read value of the equality axis is the join of the two settings, and the only route to suppressing equality operators is suppressing the owner's whole equality stance — which forces both axes to none as a single move. "Ordered but not equatable" is inexpressible by construction; identity and order are revoked together or not at all. The inverse stance — identity without order — is declared by skipping comparability with operators off and is then enforced at every sort and range expression; a surrogate identifier declares its ordering meaningless this way, and the stance is enforced by the type system rather than documented.
- Key choice decides whether order is operator-expressible: a vocabulary with intrinsic order earns generated comparison operators only when its key type carries the operator set — numeric-keyed rank ladders reach relational form plus heterogeneous key-typed overloads, while string-coded vocabularies never can — so a vocabulary whose consumers dispatch by range needs the numeric key even when a string code exists, and the code becomes a column, not the key. Items enumeration order is declaration order while key order is comparer order — presentation sequence and semantic rank are distinct axes, and conflating them couples display to identity; range dispatch over an ordered key is total over future items where arm-per-item dispatch re-opens at every addition, because threshold semantics belong to the key and case semantics to the arms.
- Interface synthesis is capability-conditional, which makes capability subtraction meaningful: parse interfaces appear only when the key is parsable (skipping the string-parse interface forcibly skips the span-parse interface that inherits it); formatting appears only over a formattable key; comparability appears when the key compares — or when a key comparer accessor is declared over a non-comparable key, so ordering is grantable by declaration, not only inherited from the key.
- The grantable axes are exactly the binary additive and multiplicative families plus the two relational interfaces; there is no axis for unary negation, modulus, increment, or the aggregate number interface bundling parsing, radix, and identity constants. The partiality is the correctness property, not a generator gap: a dimensioned quantity advertising full number-hood would invite unit constants, absolute values, and radix operations that are nonsense on it, and the absent identity constants are the type-level enforcement that admitted values have no unadmitted siblings — not even algebraic units.
- A generic algorithm names the exact algebra it requires — the addition interface for a sum, the comparison interface for a clamp, both for a running min-with-accumulation — and the constraint solver checks the intersection against what the owner grants: a quantity used outside its group is a compile error at the call, never a representation bug at runtime. An algorithm written against the aggregate number constraint excludes every owner by construction; relaxing it to the exact binary families it uses is the owner-compatible spelling, with every seed and bound entering as an admitted parameter.
- Reading an owner's generated axis set against its hand-written operators recovers the full algebraic signature of the quantity: the generated axes carry the operations that are closed and homogeneous, the manual operators carry the operations that cross dimension or arity, and the axes held at none declare the operations the quantity forbids. The declaration is the algebra, recoverable without reading a single method body.

[TYPED_AND_METADATA_PLANES]:
- Every keyed owner implements one static-abstract factory contract, and keyed smart enums layer a vocabulary contract (static items, lookup, and try-lookup) above it, so one generic constraint owns boundary plumbing across every owner kind: admission sweeps, cache hydration, and configuration readers are written once against the contract, and re-deciding an owner's kind later never touches the generic plumbing. Marker interfaces carry kind evidence for non-generic infrastructure:

```csharp
static IReadOnlyList<TOwner> Admitted<TOwner>(IEnumerable<string> raw, IFormatProvider? culture)
    where TOwner : IObjectFactory<TOwner, string, ValidationError> =>
    [.. raw.Select(value => (Error: TOwner.Validate(value, culture, out var owner), Owner: owner))
           .Where(static slot => slot.Error is null)
           .Select(static slot => slot.Owner!)];
```

- The owner taxonomy itself is reified as a closed family — keyed smart enum, keyed value object, complex value object, keyless smart enum, ad-hoc union, regular union — with its own exhaustive state-threaded dispatch, resolvable from a runtime type. The metadata family describes itself with the machinery it describes: the shape-taxonomy union and its keyed sub-union each publish a self-metadata entry enumerating their own case types, so the introspection layer is its own first instance and a fold over shape kinds is total against a compiler-checked case list — a new shape kind lands as a compile break at every metadata fold rather than a silently skipped instance.
- The keyed sub-union's two arms are deliberately asymmetric in nullability, and the asymmetry is the shape-level encoding of construction semantics: the vocabulary arm's from-key conversion delegate, conversion expression, and validating key extractor are non-nullable because a vocabulary always resolves a key to a member, while the value-object arm makes all three nullable because a factory-cut owner nulls them out — "members are resolved, not reconstructed" versus "construction was cut" is readable from the arm's type shape alone, no flag. The value-object arm alone carries a constructor-bypass conversion expression for persistence materialization of already-admitted truth; metadata consumers selecting a conversion route therefore filter on delegate presence, never on metadata kind — kind says what the owner is, presence says which routes survived its configuration.
- Keyed metadata carries the key projection both as a compiled delegate and as a lambda expression tree, so expression-translating consumers compose key access without the owner exposing a public key member — private-key owners stay query-translatable. The conversion-route record a binder or serializer reads is computed by fusing the keyed metadata's key-typed conversion with the owner's object-factory row list, conditional on type identity: the keyed expressions fold in only when the key type equals the selected factory's value type, so an owner whose alternate factory admits a different value type yields a route that constructs through the alternate value, never the declared key. Resolution composes in a fixed order — keyed lookup, the caller's metadata filter, a last-match fold over factory rows under the caller's factory filter, then fallback to the key-based expression only when no row matched — and the resolved record carries both a constructor-shaped and a method-shaped lambda so a translator chooses the form its target understands from one resolved value.
- Runtime resolution finds the static metadata property by its explicit-interface, fully-qualified, global-prefixed name walking up the base chain — the resolution contract is the generator's private member name, not a convention: a hand-rolled public static property of the same simple name is never discovered, so no foreign type can impersonate a generated owner to the metadata plane, and a derived case instance resolves to its owning family through the same base walk that makes leaf-only knowledge sufficient.
- The erased admission route published beside the typed one — a boxed key-to-item try-get with an object-typed error — carries the same typed validation error boxed, not a separate failure vocabulary: reflection-bound infrastructure that cannot name the key type still surfaces the identical fault family, so error handling written against the typed and erased planes converges on one vocabulary.
- The deferred item table is a static-initialization ordering guard, not an optimization: forcing items eagerly inside the metadata constructor would observe partially initialized static fields during the family's own class construction, so the lazy defers the read to first introspection touch and any infrastructure folding items is safe to call from an unrelated static constructor. Each item row carries the source field-name identifier captured at generation, so discriminators and labels derive from the published identifier and a case rename breaks compilation instead of corrupting decode.
- Shape-polymorphic dispatch that must make key-type mismatch unrepresentable routes through the strongly-typed static-abstract plane; the weakly-typed metadata bridge pattern-matches the boxed key and falls through to the default on mismatch, degenerating a wrong-typed key into zero-value validation on struct keys and the null path on class keys — the weak route trades type safety for shape-erased reflection and is licensed only where the caller structurally cannot name the types.
- Reflection-driven hydration of an aggregate runs real admission per layer by bridging from each runtime type to its static factory contract through the constrained invoker — the fully dynamic edge is not a validation bypass, and an aggregate reconstructed from storage or wire passes every nested owner's actual validation. The metadata namespace is declared unstable across releases: metadata dispatch belongs to infrastructure adapters only — domain code already holds the typed owner, and a domain-layer appearance of the metadata lookup marks a missing typed surface, not a clever shortcut.

[GENERATION_SUBSTRATE]:
- One owner declaration fans into independent incremental provider streams — core body, comparison, equality and arithmetic operators, parsing, span parsing, formatting, serializer surfaces — each gated by a predicate over the same extracted state and each emitting into the same partial. Aspects compose by addition of independent generators, never by one generator branching on flags: an owner that opts out of an aspect produces zero work in that aspect's stream rather than a skipped branch in a shared body. The partial split is the reading contract this substrate produces: the user-declared partial holds only domain members, generated surfaces land in the generated partial, and primary constructors are rejected because the generator owns the private constructor — construction routes are never user-improvisable.
- Every aspect stream is one parameterized five-stage skeleton — predicate filter, collect, distinct-by-type reduction, order-independent set comparer, re-expansion combined with global options into source output — where the predicate is the only aspect-specific line. The generation decision is data, not control flow: a boolean fold over the owner's own policy values, evaluated once per owner per aspect.
- The extraction step is the firewall between the compilation's mutable symbol graph and the cache: candidate transforms read symbol data into frozen value models — strings, enums, flat capability booleans, immutable member arrays — and no symbol, syntax node, or location survives downstream, because symbol identity churns per keystroke and one retained symbol would bust the cache on every edit. The contract cuts both ways: anything the equality does not read cannot trigger regeneration, so everything emission depends on must be in the model — the equality definition is the regeneration specification.
- Cache granularity is the owner per aspect: flipping one policy knob invalidates that owner's whole output for that aspect, never a fragment, while two owners with identical policy differing only in name share a cached emission shape. Capability detection — which interfaces the key implements, which operators it declares — happens once at extraction into flat model fields, so emission is a pure function of the frozen model and never re-queries the type system.
- The collected batch is compared as a set, not a sequence: an order-independent comparer (mutual containment, count-only hash) makes the batch cache-stable under file-order and declaration-order permutation, so adding one owner invalidates only its own downstream slice. The set comparer's precondition is guaranteed by an upstream distinct keyed on fully-qualified type name — the two comparers are deliberately different relations on the same models, identity answering "same owner" for dedup (multi-partial declarations collapse to one) and structure answering "changed owner" for the cache.
- Discovery is two-phase — a purely syntactic gate rejects the overwhelming majority of nodes before any semantic model is touched, keeping per-keystroke cost proportional to changed declarations rather than compilation size — and every stage is wrapped in a fault barrier that converts exceptions into located diagnostics or a logged non-candidacy, rethrowing only cancellation. One malformed declaration degrades to non-generation of one owner with a diagnostic at its location; no stage can throw across the incremental boundary, so the pipeline is total by construction and fault-isolated per owner.
- Emission writes into leased pooled builders with a bounded pool that releases oversized buffers, and each emit threads the host cancellation token — steady-state regeneration reuses buffers warmed by the whole compilation and abandons long generations the instant the host cancels. Empty emission is suppressed structurally: a file is emitted only when the body wrote past the prelude, with a logged warning otherwise — a misconfigured aspect is observable as a warning, never as an empty artifact in the compilation.
- Domain surfaces — the owner body, admission triad, operators, diagnostics — register as source output and participate in semantic analysis, because downstream code must bind against the generated factory contract. Serializer codecs register as implementation source output: emitted late, invisible to semantic analysis and the editor's view of the type, never feeding another generator — terminal surfaces buy incremental throughput by leaving the analysis graph. One wire-facing owner therefore has its domain and codec surfaces generated through channels with different visibility and different recompute triggers.
- Codec generation is conditioned on the conjunction of the declared flag and the presence of the framework's integration assembly in the compilation — the serializer streams derive from the metadata-references provider, so a flag without the assembly produces the round-trip obligation diagnostics but no codec, and adding the reference retroactively materializes converters with no source edit.
- Deterministic hint names encode the owner's full position — namespace, containing-type chain with per-level generic arity, owner name and arity, then the aspect suffix — so same-named owners in different containers or at different arities never collide, and one owner's aspects land as separate files under a shared key: regenerating one aspect rewrites one file while sibling aspects' providers never recompute.
- The structural preconditions of constructive unrevalidatability are error descriptors raised from the generation pipeline itself, before any body is generated: non-partial owners, reachable constructors, public initializers, mutable state, nullable keys, stacked shape attributes. The generated body never defends against these states because they cannot reach it — the no-double-validation guarantees are enforced ahead of generation, not assumed inside it. The same holds for contract obligations (the exact validate signature, the outbound projection when a wire is faced, the rehydration constructor when trusted rehydration is claimed) and for the wire partition (duplicate serialization, binding, or persistence claims are distinct errors), so a downstream codec can never face an ambiguous admission source.
- A self-referential constraint gates the whole aspect before it runs: the validation-error type bound on an owner is checked for satisfying its own static-create contract at definition time, so a fault-lattice member that fails to re-implement creation for its precise case fails the constraint and the owner never generates — the aspect that produces admission is itself guarded by an aspect over the fault type.
- The substrate instruments its own cache contract: an opt-in emission counter watermarks generated files, and under the value-model discipline a file regenerates only when its model changed — a counter climbing on edits to unrelated declarations is direct evidence that some model over-reads, a retained symbol or location breaking the value-only equality. The build-property file logger is its complement: the counter proves the cache is churning, the log proves why a specific type fell out of the pipeline, with the rejected-state reason recorded only if the logger was enabled before the event — generation drift is diagnosable from build output without decompiling generated sources.
