# boundaries — bedrock (platform + backend stack capture)

[HTTP_API_SERVING]: (`@effect/platform` — schema-first contract layer)
- `HttpApi.make(identifier)` → `HttpApi<Id, never, HttpApiDecodeError>`; build by `.add(group)`, `.prefix(path)`, `.middleware(TagClass)`, `.annotate(tag, val)`, `.annotateContext(ctx)`. The top-level `E` channel seeds `HttpApiDecodeError` (request decode failures are always in the union).
- `HttpApiGroup.make(identifier, options?)` with `options.topLevel: true|false` (topLevel mounts endpoints at root, not under group prefix). Build by `.add(endpoint)`, `.prefix`, `.middleware`, `.annotate`. `HttpApiGroup.HttpApiGroup.ToService<Id, Groups>` is the handler-context requirement the builder satisfies.
- `HttpApiEndpoint` verb constructors: `HttpApiEndpoint.get(name, path)` / `.post` / `.put` / `.patch` / `.del` / `.head` / `.options`, plus generic `HttpApiEndpoint.make(method)`. Chain (all dual): `.setPath(Schema)`, `.setPayload(Schema)`, `.setHeaders(Schema)`, `.setUrlParams(Schema)`, `.addSuccess(Schema, {status?})`, `.addError(Schema, {status?})`, `.prefix`, `.middleware`, `.annotate`. Path params and payload are Schema-typed; the handler signature is fully derived from this chain.
- `HttpApiSchema` owns status/encoding annotations: `HttpApiSchema.annotations({status, ...})`, status helpers `Created` (201), `Accepted` (202), `NoContent`/`Empty(status)` (→ `Schema.Void`), `asEmpty`. Status read-back: `getStatusSuccess(schema)`, `getStatusError(schema)`. Encoding: `withEncoding`, `Text({...})`, `Uint8Array({...})`. Multipart: `HttpApiSchema.Multipart(schema, opts)` and `MultipartStream(schema, opts)` mark a payload as multipart form. `param(name, schema)` builds path params. Error status defaults derive from `Schema.TaggedError` annotations.
- `HttpApiError` ships canonical typed HTTP errors with baked statuses: `HttpApiDecodeError` (with `Issue` AST), `BadRequest` (400), `Unauthorized` (401), `Forbidden` (403), `NotFound` (404), `MethodNotAllowed` (405), `NotAcceptable` (406), `RequestTimeout` (408), `Conflict` (409), `Gone` (410), `InternalServerError` (500), `NotImplemented` (501), `ServiceUnavailable` (503). Add via `.addError(HttpApiError.NotFound)`.
- `HttpApiSecurity`: `HttpApiSecurity.bearer` (→ `Redacted`), `apiKey({key, in: "header"|"query"|"cookie"})` (→ `Redacted`), `basic` (→ `Credentials {username,password}`). `.annotate`/`.annotateContext` attach OpenAPI security metadata. `HttpApiBuilder.securityDecode(security)` extracts the credential from request; `securitySetCookie(apiKey, value, opts)` sets the cookie response-side.
- `HttpApiMiddleware.Tag<Self>()(name, options)` defines middleware; `options.provides: Context.Tag` declares the service injected into downstream handlers (e.g. `CurrentUser`); `options.security: Record<string, HttpApiSecurity>` upgrades it to a security middleware (`TagClassSecurity`) whose implementation receives the decoded credential and returns the `provides` service; `options.failure: Schema` types the rejection. Attach at api/group/endpoint via `.middleware(Tag)`.
- `HttpApiBuilder` is the implementation seam: `HttpApiBuilder.api(api)` → `Layer<HttpApi.Api, ...>`. `HttpApiBuilder.group(api, groupName, build)` implements a group — `build` receives `handlers` and chains `handlers.handle(endpointName, handler)`; returns `Layer<ApiGroup<Id,Name>, ...>`. `HttpApiBuilder.handler(api, group, name, fn)` is the standalone handler form. `HttpApiBuilder.serve(middleware?)` → `Layer` requiring `HttpServer`. `HttpApiBuilder.toWebHandler(layer, opts?)` produces a `(Request) => Promise<Response>` for serverless/edge. CORS: `HttpApiBuilder.middlewareCors(opts?)`. OpenAPI mount: `middlewareOpenApi(opts?)`. App-level middleware: `HttpApiBuilder.middleware(fn, {withContext?})` (withContext:true keeps handler-context requirements).
- `HttpApiClient.make(api, {baseUrl?, transformClient?})` derives a fully-typed client `Effect` from the SAME `HttpApi` — methods are `client.<group>.<endpoint>(payload)` returning `Effect<Success, Error>`. `HttpApiClient.group(api, {group, ...})` and `.endpoint(api, {group, endpoint})` derive partial clients. Requires an `HttpClient` in context.
- OpenAPI derivation: `OpenApi.fromApi(api, opts?)` → the OpenAPI 3.1 JSON document. Doc UIs: `HttpApiScalar.layer(opts?)` / `layerCdn` / `layerHttpLayerRouter({path})` (Scalar UI); `HttpApiSwagger.layer(opts?)` (Swagger UI). Both serve from the derived spec automatically.
- `HttpServer.serve(httpApp, middleware?)` / `serveEffect`; `HttpServer.layerContext`, `formatAddress`, `logAddress`, `withLogAddress(layer)`. Node wiring: `NodeHttpServer.layer(() => createServer(), {port})`, `layerConfig(..., Config.Wrap)`, `layerTest` (random port + paired `HttpClient`+`HttpServer` for integration). The `HttpApiBuilder.serve()` layer composes onto `NodeHttpServer.layer`.

[HTTP_ROUTER]: (lower-level, when HttpApi's contract-first model is too rigid)
- `HttpRouter.empty` + `.get(path, handler)` / `.post` / `.put` / `.patch` / `.del` / `.all`, `.mount(prefix, router)`, `.mountApp(prefix, httpApp)`, `.use(middleware)`, `.concat(router)`, `.prefix`, `.catchAll`, `.provideService`. `HttpRouter.Default` is the canonical injectable router Tag (class); `HttpRouter.Default.serve(middleware?)` wires it to `HttpServer`. Path-param schema decode: `HttpRouter.schemaPathParams(Schema)`, `schemaParams(Schema)`. `toHttpApp` lowers a router to an `HttpApp`. Use HttpRouter for raw/streaming/non-schema routes, websocket upgrades, or mounting foreign apps; use HttpApi for everything with a declared contract + OpenAPI.
- `HttpServerResponse`: `.text`, `.json`, `.schemaJson(Schema)`, `.html`, `.empty`, `.raw`, `.file(path)`, `.stream`, `.formData`, `.uint8Array`; chain `.setHeader`/`.setStatus`/`.setCookie`. `HttpServerRequest`: `.schemaBodyJson(Schema)`, `.schemaBodyForm(Schema)` (multipart-aware), `.schemaHeaders(Schema)`, `.schemaCookies(Schema)`, `.multipart` (→ `Stream` of parts), `.arrayBuffer`, `.text`, `.json`, `.upgrade` (websocket), `ParsedSearchParams` reference. `HttpServerRespondable` lets a typed error/value render itself.

[HTTP_CLIENT]: (`@effect/platform/HttpClient`)
- `HttpClient` service Tag; default impl via `FetchHttpClient.layer` (browser/edge — `Fetch`/`RequestInit` tags override `globalThis.fetch`/init) or `NodeHttpClient.layer` / `NodeHttpClient.layerUndici` (Node). Combinators (all return a transformed `HttpClient`): `.mapRequest(f)`, `.mapRequestEffect(f)`, `.retryTransient({schedule, times?})` (retries only transient transport errors), `.followRedirects(n?)`, `.filterStatusOk` (non-2xx → `ResponseError`), `.filterStatus(pred)`, `.transformResponse(f)`, `.tapRequest(f)`, `.catchTag`, `.withTracerPropagation(bool)` (toggles W3C `traceparent`+b3 injection — ON by default), `.withTracerDisabledWhen(pred)`.
- `HttpClientRequest.get(url)` / `.post` / `.put` / `.patch` / `.del` / `.head` / `.options`, `.make(method)(url)`. Body: `.bodyJson(value)` (Effect), `.bodyText`, `.bodyFormData`, `.bodyUrlParams`, `.schemaBodyJson(Schema)(req, value)` (Schema-encode body). Headers: `.setHeader`/`.setHeaders`, `.bearerToken`, `.acceptJson`, `.appendUrlParam`, `.prependUrl`.
- `HttpClientResponse` decode: `.schemaBodyJson(Schema)(response)` → `Effect<A, ParseError|ResponseError>`, `.schemaJson(Schema)`, `.schemaNoBody`, `.schemaHeaders(Schema)`, `.matchStatus({...})` (status-keyed dispatch), `.filterStatus(pred)`.

[PLATFORM_CAPABILITIES]: (`@effect/platform` services, Node-provided)
- `FileSystem.FileSystem`: `readFile`/`readFileString`, `writeFile`/`writeFileString`, `makeDirectory`, `remove`, `rename`, `copy`, `exists`, `stat`, `access`, `open` (scoped `File` handle), `readDirectory`, `watch` (→ `Stream<WatchEvent>`), `makeTempFile`/`makeTempDirectory`/`makeTempDirectoryScoped`, `truncate`, `symlink`, `realPath`, `stream` (→ byte `Stream`), `sink` (→ `Sink`), `utimes`, `chmod`, `chown`. All return `Effect<_, PlatformError>`.
- `Path.Path`: `join`, `resolve`, `relative`, `dirname`, `basename`, `extname`, `parse`, `format`, `normalize`, `isAbsolute`, `fromFileUrl`, `toFileUrl`, `sep`.
- `Command` (process exec): `Command.make(cmd, ...args)`; configure with `Command.env(record)`, `workingDirectory(path)`, `stdin`/`stdout`/`stderr` (`"inherit"|"pipe"`), `feed(input)`, `runInShell(bool|shell)`, `pipeTo(otherCommand)` (process pipelines). Execute: `Command.string(cmd)` (→ stdout string), `lines`, `stream` (byte `Stream`), `streamLines`, `exitCode`, `start` (→ `CommandExecutor.Process` with `.exitCode`/`.stdout`/`.stderr`/`.stdin`/`.kill`/`.pid`/`.isRunning`). Runs against `CommandExecutor` service.
- `Worker`: `Worker.makePool(opts)` / `makePoolLayer(tag, opts)` (raw I/O pool), `makePoolSerialized(opts)` / `makePoolSerializedLayer(tag, opts)` — serialized pools key on `Schema.TaggedRequest` so each request type declares its own success/error/payload Schema, multiplexed over the pool. `makeManager`/`layerManager` (needs `PlatformWorker`), `Spawner` tag. Runner side: `WorkerRunner.make(process)` / `layer(process)` (raw), `makeSerialized(schema, handlers)` / `layerSerialized(schema, handlers)` — handlers map each `TaggedRequest` tag to an `Effect`/`Stream`. `WorkerRunner.launch(layer)` boots the runner; `CloseLatch` reference governs shutdown. Node: `NodeWorker.layerManager` (spawn side) + `NodeWorkerRunner.layer` (inside worker). Browser: `BrowserWorker.layer(spawn)` / `layerPlatform`.
- `Socket.Socket` Tag; `toChannel`/`toChannelString`/`toChannelMap`/`makeChannel` bridge socket↔Effect `Channel`. Node: `NodeSocket.layerWebSocket(url)`. Browser: `BrowserSocket.layerWebSocket(url, opts)` / `layerWebSocketConstructor`.
- `Terminal.Terminal`: `readLine`, `readInput` (→ `Stream<UserInput>`), `display(text)`, `columns`. Node: `NodeTerminal.layer`.
- `KeyValueStore.KeyValueStore`: `get`/`set`/`remove`/`clear`/`size`/`has`/`modify`/`getUint8Array`. Layers: `layerMemory`, `layerFileSystem(dir)` (Node: `NodeKeyValueStore.layerFileSystem`). `KeyValueStore.layerSchema(Schema, idKey)` and `store.forSchema(Schema)` give a Schema-validated KVS (`SchemaStore` with typed `get`/`set`). `prefix(store, str)` namespaces. Browser: `BrowserKeyValueStore.layerLocalStorage` / `layerSessionStorage`.
- `PlatformLogger.toFile(path, opts?)` — composes an Effect `Logger` to a file sink (requires `FileSystem`).
- `PlatformError` taxonomy: union of `BadArgument {module, method, description}` and `SystemError {reason, module, method, ...}` where `reason ∈ {AlreadyExists, BadResource, Busy, InvalidData, NotFound, PermissionDenied, TimedOut, UnexpectedEof, Unknown, WouldBlock, WriteZero}` and `module ∈ {Clipboard, Command, FileSystem, KeyValueStore, Path, Stream, Terminal}`. Both are `Schema.TaggedError`. `TypeIdError(typeId, tag)` is the constructor factory for custom platform errors.
- Runtime entry: `NodeRuntime.runMain(effect, {disableErrorReporting?, disablePrettyLogger?, teardown?})` (Node) / `BrowserRuntime.runMain` (browser). `NodeContext.layer` bundles `Path + CommandExecutor + Terminal + WorkerManager + FileSystem`.

[CLUSTER]: (`@effect/cluster` — 3.19+; `ShardManager` module REMOVED, replaced by `RunnerStorage`+`RunnerHealth`)
- `Sharding` owns entity/singleton registration, routing, entity-client factory. `Sharding.makeClient(entity)` → `(entityId) => RpcClient`. `registerEntity(entity, handlers, {maxIdleTime?, concurrency?, mailboxCapacity?, defectRetryPolicy?})`, `registerSingleton(name, run, {shardGroup?})`. Provided by `Sharding.layer` over `ShardingConfig | Runners | MessageStorage | RunnerStorage | RunnerHealth`.
- `Entity.make(type, protocol: ReadonlyArray<Rpc.Any>)` — protocol is bare RPCs, not a pre-grouped container. Behavior: `entity.toLayer(build)` where `build` returns a tag-keyed handler record `{[Rpc._tag]: (request) => Rpc.Result}`; handler reads `CurrentAddress`/`entityId` from context. Mailbox form: `entity.toLayerMailbox(build)` with `(mailbox, replier)`. `entity.client ≡ Sharding.makeClient(entity)`. `EntityId`/`EntityAddress`/`EntityType` branded; `ShardId.make(group, id)` is interned (`===`-safe).
- `Singleton.make(name, run, {shardGroup?})` → `Layer`; exactly one runner cluster-wide executes `run`, failover re-acquires.
- `MessageStorage` (at-least-once + ack tracking; `requestIdForPrimaryKey` dedup → effective exactly-once). Backends: `layerNoop`, `layerMemory`, SQL via `SqlMessageStorage.layer` (needs `SqlClient | ShardingConfig`). RPCs annotated `ClusterSchema.Persisted` survive restart.
- Runners: `Runners` (inter-runner transport: `ping`/`send`/`notify`; `layerRpc`). `RunnerStorage` (ShardManager replacement; shard assignment via DB advisory locks — breaks under pgBouncer txn pooling, opt out `shardLockDisableAdvisory`): `layerMemory`, `SqlRunnerStorage.layer`. `RunnerHealth`: `layerNoop`, `layerPing`, `layerK8s`.
- Node JOIN entrypoints (moved INTO `@effect/cluster`): `NodeClusterSocket.layer(opts?)` and `NodeClusterHttp.layer(opts)` — `opts.clientOnly` toggles client-only vs full runner; bundle transport + `ShardingConfig.layerFromEnv` + serialization. `NodeClusterShardManagerHttp` is the separate shard-manager-node module. (Bun mirrors: `BunClusterSocket`/`BunClusterHttp`.)
- `ShardingConfig` (Tag, `layerFromEnv` for env-driven): `runnerAddress`, `shardsPerGroup=300` (replaces flat `numberOfShards`; total = `shardsPerGroup × |availableShardGroups|`), `availableShardGroups`/`assignedShardGroups`, `entityMailboxCapacity=4096`, lock intervals.
- `ClusterSchema` annotation keys: `Persisted` (default false), `Uninterruptible`, `ShardGroup`, `ClientTracingEnabled`.
- CRON IS owned natively: `ClusterCron.make({name, cron: Cron.Cron, execute, shardGroup?, skipIfOlderThan?})` → `Layer` — merges `Singleton` (schedules) + a `Persisted`+`Uninterruptible` entity (at-most-once per tick). Takes `effect/Cron`. This is the canonical distributed-cron primitive; `Singleton + Schedule.cron` works but `ClusterCron` is the owner.

[WORKFLOW]: (`@effect/workflow` — durable execution)
- `Workflow.make({name, payload, idempotencyKey, success?, error?, suspendedRetrySchedule?})` — `idempotencyKey: (payload) => string` is the DURABLE IDENTITY (collapses retries onto one execution). Returns an OBJECT: `.execute(payload, {discard?})` → `Effect<Success, Error, WorkflowEngine>`, `.poll`, `.interrupt`, `.resume`, `.executionId(payload)`, `.withCompensation`. Body: `workflow.toLayer((payload, executionId) => Effect)`.
- `Activity.make({name, success?, error?, execute, interruptRetryPolicy?})` — `execute` is an Effect VALUE; runs EXACTLY ONCE per execution (memoized by `(executionId, name, attempt)` via `WorkflowEngine.activityExecute`); replay returns persisted result. Failure-retry is a SEPARATE combinator `Activity.retry({times})` piped on; `interruptRetryPolicy` is orthogonal (governs retry on interruption/rebalance). `Activity.CurrentAttempt` reference. `Activity.raceAll(name, activities)`.
- `DurableClock.sleep({name, duration, inMemoryThreshold?})` — ≤threshold in-memory; longer SUSPENDS on a `DurableDeferred` registered via `WorkflowEngine.scheduleClock`, survives process death.
- `DurableDeferred.make(name, {success?, error?})`; `DurableDeferred.await(self)` (suspends durably); `.token(self)` mints an opaque token to persist/transmit; external process completes via `DurableDeferred.done/succeed/fail(self, {token, exit})`. Token serializes via `TokenParsed.FromString`.
- `Workflow.withCompensation((value, cause) => Effect<void, never>)` — saga undo via `Scope.addFinalizerExit` on the workflow scope; runs in reverse on failure; survives suspend/resume. TOP-LEVEL ONLY — compensation registered inside an `Activity.execute` body SILENTLY never fires.
- `WorkflowEngine` Tag (`execute`/`poll`/`interrupt`/`resume`/`activityExecute`/`deferredResult`/`scheduleClock`); test layer `WorkflowEngine.layerMemory`. Cluster seam: `ClusterWorkflowEngine.layer` (sole export) → `Layer<WorkflowEngine, never, Sharding | MessageStorage>` — each workflow registers as a cluster entity (handlers `run`/`activity`/`deferred`/`resume`), shard-routed by executionId, MessageStorage persists results. Provide the workflow `toLayer` + engine layer together.
- Jobs/queues/crons: Workflow does NOT own cron (no `schedule:` field). Recurrence = `ClusterCron.make` (cluster owner) or `Singleton + Schedule.cron + workflow.execute` with period-derived `idempotencyKey`. In-flow delay = `DurableClock.sleep`. Queues = cluster `MessageStorage` + entity mailboxes.

[SQL]: (`@effect/sql` + `@effect/sql-pg`)
- `SqlClient` Tag is the callable `sql` tagged template; interpolated values are parameter-bound, `sql(identifier)` escapes, nested `Fragment`s compose. Members: `.safe` (untransformed copy), `.withoutTransforms()`, `.reserve: Effect<Connection, SqlError, Scope>`, `.withTransaction`, `.reactive`.
- `sql` helpers: `sql.in(values)` / `sql.in(column, values)` (two-arity; column form → `col IN (...)`), `sql.insert(record | record[])` (chain `.returning("*")`), `sql.update(record, omit?)`, `sql.updateValues(records, alias)`, `sql.and(clauses)` / `sql.or(clauses)`, `sql.unsafe(sql, params?)`, `sql.literal(sql)` (inlined, no binding), `sql.csv(values)` / `sql.csv(prefix, values)`, `sql.onDialect({...})`. Pg-specific: `sql.json(value)`.
- `SqlResolver` (request batching/caching over `RequestResolver`): `.ordered(name, {Request, Result, execute})` → positional map, can fail `ResultLengthMismatch`; `.grouped(name, {Request, RequestGroupKey, Result, ResultGroupKey, execute})` → per-request `Array<A>`; `.findById(name, {Id, Result, ResultId, execute})` → `Option<A>`; `.void(name, {Request, execute})` → side-effect batch.
- `SqlSchema` (schema-validated queries; `ParseError` always in error channel): `.findAll(opts)` → `ReadonlyArray<A>`, `.findOne(opts)` → `Option<A>`, `.single(opts)` → `A` (empty → `NoSuchElementException`), `.void(opts)`. Options `{Request, Result, execute}`.
- Transactions: `sql.withTransaction(effect)` → `Effect<A, E | SqlError, R>`; nesting uses automatic SAVEPOINTs (inner rollback to savepoint, outer intact).
- `PgClient` extends `SqlClient`: `PgClient.layer(config)`, `layerConfig(Config.Wrap<config>)` (env-driven), `make(config)`/`fromPool`. Config: `url`(Redacted)/`host`/`port`/`database`/`username`/`password`/`ssl`/`maxConnections`/`types` + transforms `transformQueryNames`/`transformResultNames` (camel↔snake via `effect/String` camelToSnake/snakeToCamel — but NOT applied inside `sql.unsafe`/`sql.literal`). Pg members: `listen(channel)` → `Stream<string, SqlError>` (LISTEN), `notify(channel, payload)` (NOTIFY), `json(value)`.
- Migrator: files `{id}_{name}.ts` each `export default Effect<_, _, SqlClient>`. `PgMigrator.layer({loader: PgMigrator.fromFileSystem(dir), schemaDirectory?, table?})` (default table `effect_sql_migrations`); loaders `fromFileSystem`/`fromGlob`/`fromRecord`; PgMigrator adds `dumpSchema`. Requires `FileSystem | Path | CommandExecutor`.
- Errors: `SqlError {cause?, message?}`, `ResultLengthMismatch {expected, actual}` — both via `TypeIdError` (not `Data.TaggedError`).

[RPC]: (`@effect/rpc`)
- `RpcGroup.make(...rpcs)` — idiomatically `class FooRpcs extends RpcGroup.make(Rpc.make("X", {...}), ...) {}`. `Rpc.make(tag, {payload?, success?, error?, stream?, primaryKey?})` — defaults success=`Schema.Void`, error=`Schema.Never`, payload=`Schema.Void`; payload accepts a fields record or full Schema; `stream:true` makes success a `Stream<A,E>`. `Rpc.fromTaggedRequest(schema)` alt constructor.
- Group: `.toLayer(handlers | Effect<handlers>)` (the implementation seam — handlers can pull app services), `.middleware(Tag)` (group-wide), `.merge(...groups)`, `.accessHandler(tag)`. Per-procedure `Rpc.middleware(tag)`.
- `RpcSerialization` layers: `layerJson` (framed transport), `layerNdjson` (self-framing), `layerMsgPack` (binary), `layerJsonRpc`/`layerNdJsonRpc`.
- Server (`RpcServer`): handlers via `RpcGroup.toLayer`; protocol layers `layerProtocolHttp({path})` / `layerProtocolHttpRouter({path})` (needs `RpcSerialization` + `HttpLayerRouter`), `layerProtocolWebsocket`/`layerProtocolWebsocketRouter`, `layerProtocolSocketServer`, `layerProtocolWorkerRunner` (needs `WorkerRunner.PlatformRunner`, NOT serialization — uses structured clone), `layerProtocolStdio`. Adapters `RpcServer.toHttpApp`/`toHttpAppWebsocket`/`toWebHandler`.
- Client (`RpcClient`): `RpcClient.make(group, {flatten?, spanPrefix?})` → typed client (`flatten:true` → direct callables). Protocol client layers `layerProtocolHttp({url})` (pair `FetchHttpClient.layer` + serialization), `layerProtocolSocket({retryTransientErrors?})`, `layerProtocolWorker({size|minSize/maxSize, concurrency?})`.
- Middleware: `RpcMiddleware.Tag<Self>()(id, {wrap?, optional?, failure?, provides?, requiredForClient?})`; `provides` injects a service downstream; client impl `RpcMiddleware.layerClient(tag, service)`.
- RPC vs HttpApi: RPC = internal Effect↔Effect typed calls over any transport (HTTP/WS/Socket/Worker/Stdio) with first-class `Stream`/bidirectional + zero schema duplication. HttpApi = externally-contracted REST with fixed method+path+status + generated OpenAPI/Swagger for non-Effect consumers.

[CLI]: (`@effect/cli`)
- `Command.make(name, config?, handler?)` — `config` is a record of `Options`/`Args`; handler receives one object keyed by config keys. `Command.withSubcommands([...])` (adds `subcommand: Option`). `Command.run(self, {name, version, summary?})` → `(argv) => Effect<void, ..., FileSystem | Path | Terminal>` — auto-wires `--help`/`--version`/`--completions <bash|zsh|fish>`/`--wizard`/`--log-level`. `Command.prompt(name, Prompt, handler)` is the Prompt→Command path. `.provide`/`.provideEffect`/`.provideSync` accept `(parsedConfig) => Layer/Effect`.
- `Options` (positional `name` first): `text`/`integer`/`float`/`boolean(name,{negationNames?})`/`choice(name, string[])`/`date`/`file(name,{exists?})`/`directory`/`redacted`/`keyValueMap`/`fileContent`/`fileSchema`. Combinators (dual): `withDefault`, `optional` (→`Option`), `withFallbackConfig(Config)` (bridges `effect/Config` env), `withFallbackPrompt`, `withAlias`, `withSchema(Schema)`, `repeated`/`atLeast(n)`/`atMost`/`between(min,max)` (min-arity selects Array vs NonEmptyArray).
- `Args` (config object, NO positional name; asymmetric with Options): `text({name?})`/`integer`/`file({name?, exists?})`/`directory`/`path`/`choice([label,value][], {name?})`/`redacted`. Same dual combinators + `withFallbackConfig`/`withSchema`.
- `Prompt` (single options object): `text`/`password`(→`Redacted`)/`integer`/`confirm`(→`boolean`)/`toggle`/`select({message, choices})`/`multiSelect`/`list`/`date`/`file`. `Prompt.run(self)` → `Effect<_, QuitException, FileSystem | Terminal>`. `--wizard` auto-derives an interactive walk of all Options/Args (zero wiring).
- `CliConfig.layer({isCaseSensitive?, autoCorrectLimit?, finalCheckBuiltIn?, showBuiltIns?})` — note `finalCheckBuiltIn` default false means `--help` after an invalid line is NOT honored unless set true. Runtime: `cli(process.argv).pipe(Effect.provide(NodeContext.layer), NodeRuntime.runMain)`.

[AI]: (`@effect/ai` + `@effect/ai-openai` + `@effect/ai-google` — naming SETTLED: `LanguageModel`/`Tool`/`Toolkit`, the `Ai`-prefixed names are GONE)
- `LanguageModel.generateText({prompt, toolkit?, toolChoice?, concurrency?})` → `Effect<GenerateTextResponse<Tools>, _, LanguageModel>`. `LanguageModel.generateObject({prompt, schema: Schema, objectName?, toolkit?})` → response adds typed `.value: A` (structured output). `LanguageModel.streamText(options)` → `Stream<Response.StreamPart<Tools>>` (NOT Effect); parts `type`-discriminated: `text-delta`/`tool-call`/`tool-result`/`reasoning`/`finish` (finish carries `finishReason`+`usage`).
- `GenerateTextResponse` getters: `content`, `text`, `reasoning`/`reasoningText`, `toolCalls`, `toolResults`, `finishReason`, `usage: Response.Usage`. `Usage` (TokenUsage): `inputTokens`/`outputTokens`/`totalTokens`/`reasoningTokens`/`cachedInputTokens`.

Pass the handler-bound toolkit to `generateText`/`streamText` via `toolkit` option.
- Provider seam (`Model`): `OpenAiLanguageModel.model(id, config?)` and `GoogleLanguageModel.model(id, config?)` both return `AiModel.Model<provider, LanguageModel, Client>`, which IS simultaneously a `Layer` AND `Effect<Layer>`. - Clients: `OpenAiClient.layer({apiKey?: Redacted, apiUrl?, organizationId?, projectId?})` / `layerConfig(...)` → `Layer<OpenAiClient, _, HttpClient>`. `GoogleClient.layer({apiKey?: Redacted, apiUrl?})` / `layerConfig` → `Layer<GoogleClient, _, HttpClient>` (modules are `GoogleClient`/`GoogleLanguageModel` — NO `GenerativeLanguage`/`GoogleAiClient`). OpenAI `Config.strict` defaults true (forces structured-outputs JSON-schema; set false for schemas exceeding OpenAI's strict subset).
- Embeddings: `OpenAiEmbeddingModel.model(id, {mode})`; layers `layerBatched({model, config?})` / `layerDataLoader({model, config})` → `Layer<EmbeddingModel, never, OpenAiClient>`; service `EmbeddingModel.EmbeddingModel`, batch op `embedMany(string[])`.

[OBSERVABILITY]: (`@effect/opentelemetry` — TWO independent paths)
- SDK-bridge: `NodeSdk.layer(evaluate)` → `Layer<Resource>`. Config: `spanProcessor?` (e.g. `BatchSpanProcessor(OTLPTraceExporter)`), `metricReader?` (`PeriodicExportingMetricReader(OTLPMetricExporter)`), `logRecordProcessor?` (`BatchLogRecordProcessor(OTLPLogExporter)`), `resource: {serviceName, serviceVersion?, attributes?}`. effect `Metric` exports through the OTel `metricReader` bridge. Installs OTel global SDK.
- Pure-Effect path: `Otlp.layer({baseUrl, resource?, headers?, maxBatchSize?})` → `Layer<never, never, HttpClient | OtlpSerialization>` — wires tracer+metrics+logs over OTLP HTTP, NO OTel SDK. Sub-modules `OtlpTracer`/`OtlpMetrics`/`OtlpLogger`/`OtlpResource`. Choose ONE path (running both double-exports). `Otlp` is lighter for pure-Effect services; `NodeSdk` only for non-Effect OTel auto-instrumentation interop.
- Span propagation IS automatic+bidirectional: client (`HttpClient.withTracerPropagation(bool)`, on by default) injects W3C `traceparent`+b3; server (`HttpApp` tracer middleware) extracts inbound via `HttpTraceContext.fromHeaders` (traceparent → b3-single → x-b3 multi). NOTE: server does NOT write trace headers back on the response — add manually via `HttpTraceContext.toHeaders(span)` if a RUM/downstream needs response-side `traceparent`.

Further Considerations
- The `HttpApi` ↔ `RpcGroup` duality is the central backend decision: both are schema-first and both derive a typed client from one definition, but `HttpApi` bakes HTTP semantics (method/path/status/OpenAPI) into the contract while `RpcGroup` is transport-agnostic with first-class `Stream`. A doctrine rule worth codifying: external/public surface → `HttpApi`; internal service mesh + streaming → `RpcGroup`; and the cluster `Entity` protocol is literally `ReadonlyArray<Rpc.Any>`, so the same RPC definitions flow from edge (HttpApi/RPC) to durable actors (Entity) to workflows (each workflow registers as an Entity) — one Schema vocabulary spans all four layers.
- The serialization/transport matrix has one sharp asymmetry to encode: `RpcServer.layerProtocolWorkerRunner` and `Worker.makePoolSerialized` use structured-clone (no `RpcSerialization`), whereas every network transport (HTTP/WS/Socket/Stdio) requires an explicit `RpcSerialization` layer (`layerNdjson` for frameless, `layerMsgPack` for binary density). Layering serialization onto a Worker transport is inert; omitting it from a network transport fails to wire.
- `@effect/experimental/Persistence/ResultPersistence` is the unifying backend for THREE seemingly-separate experimental owners — `PersistedCache`, durable workflow state (via cluster `MessageStorage`'s SQL backend, conceptually parallel), and `RateLimiter/Redis` — all key on `Schema.TaggedRequest` and select an in-memory/KVS/Redis/SQL backend by layer choice alone. A doctrine that treats "persisted keyed-result store" as one concept with swappable backing (not three bespoke stores) collapses cache + rate-limit + idempotency persistence onto a single `ResultPersistence` surface.
