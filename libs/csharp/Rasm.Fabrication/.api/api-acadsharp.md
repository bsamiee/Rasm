# [RASM_FABRICATION_API_ACADSHARP]

`ACadSharp` (3.6.35, MIT, pure-managed AnyCPU IL — no native asset, no RID burden) reads and writes AutoCAD DXF (ASCII and binary) and DWG (AC1014 through AC1032); the fabrication folder consumes only its READ surface to admit external 2D part profiles into the canonical `Loop` vocabulary at the `Ingress/profile#PROFILE_IMPORT` boundary. At the consumer floor (`net10.0` binds the package's highest compatible `lib/net9.0` asset — the `net9.0` dependency group is EMPTY, so the read path drags zero transitive package weight; the netstandard/net48 `System.Memory`/`System.Text.Encoding.CodePages` deps never reach this consumer) the surface splits across four namespaces the boundary `using`s explicitly: the reader facades and the notification/config rail live in `ACadSharp.IO`, the document root and tables in `ACadSharp`/`ACadSharp.Tables`, every profile entity in `ACadSharp.Entities`, and the coordinate/transform primitives in `CSMath`. The reader contract is the `ICadReader : IDisposable` interface (`Read()`/`ReadHeader()` plus `OnNotification`/`OnProgress` events) behind the static `DxfReader.Read`/`DwgReader.Read` facades and the `new DxfReader(filename, notification)`/`.Read()` instance form, all returning a `CadDocument` model root whose `Entities` (`=> ModelSpace.Entities`), `ModelSpace`, and `BlockRecords` carry the drawing geometry. Closed 2D profiles arrive as `LwPolyline` (an `IPolyline` whose `Vertex` carries a `CSMath.XY Location` plus a `double Bulge` — the tangent of one quarter the arc's included angle), `Polyline2D` (`Polyline<Vertex2D> : IPolyline`), `Line`, `Arc`/`Circle` (`Circle : Entity, ICurve`; `Arc : Circle`), and `Spline` entities, and nested-block profiles arrive through `Insert` block references; the boundary tessellates each bulge/arc/circle/spline span to `Loop` vertices through the ACadSharp-owned `Arc.CreateFromBulge`/`PolygonalVertexes`/`Spline.PolygonalVertexes` curve sampler at the owner's chord-precision knob, never a hand-rolled bulge trigonometry or a hand-rolled NURBS de Boor, and flattens each `Insert` through the package-owned `Insert.Explode()`/`Insert.GetTransform()` rather than a hand-built OCS-to-WCS matrix. The read is RESILIENT by default (`CadReaderConfiguration.Failsafe = true`): recoverable corruption rides the `OnNotification` event as a structured `(NotificationType, Message, Exception)` record and the read completes with the recoverable entities, so a hard reader throw is the unrecoverable-structural case the boundary lowers ONCE to `GeometryFault.DegenerateInput` — the warning stream is the resilient rail, the throw is the floor.

## [04]-[RATIFIED]

- [READER_RAIL] Both facades implement `ICadReader : IDisposable` (`Read()`/`ReadHeader()` plus the `OnNotification`/`OnProgress` events), so the canonical boundary call is the static `DxfReader.Read(path, notification)` (and `DwgReader.Read(path, notification)` for DWG) with the disposable `new DxfReader(path, notification)` + `using`/`.Read()` form available when `OnNotification`/`OnProgress` must be wired or the handle disposed deterministically. The read is RESILIENT by default: `CadReaderConfiguration.Failsafe` is `true`, so recoverable corruption (an unknown entity, a malformed sub-record) is captured into a `NotificationEventArgs(NotificationType, Message, Exception)` and routed to `OnNotification` while the read completes with the recoverable entities — the `Failsafe` config plus the notification event ARE the resilient rail, not an afterthought. A hard reader throw is the unrecoverable-structural case (an unreadable/missing file, a non-DXF/DWG stream); the boundary wraps the call so that throw lowers to `GeometryFault.DegenerateInput` ONCE at admission, and a non-default `Failsafe=false` config is the strict opt-in that promotes every recoverable defect to a throw the same lowering catches. The reader exception never escapes the boundary, and a subscribed `OnNotification` sink lets the boundary fold the warning stream (e.g. count `NotificationType.Error` rows) into the admission receipt rather than discarding it.
- [VERTEX_ACCESS] `LwPolyline : Entity, IPolyline` exposes `Vertices: List<Vertex>` whose `Vertex` carries `Location` (`CSMath.XY` with `.X`/`.Y` doubles, `Convert<XYZ>()` lifting to 3D) and `Bulge` (`double`, the tangent of one quarter the included angle, sign = arc direction) plus `StartWidth`/`EndWidth`; closure is `LwPolyline.IsClosed` (a `Flags.HasFlag(LwPolylineFlags.Closed)` projection, `Closed = 1`), and `LwPolyline.Elevation: double` plus `Normal: XYZ` seat the 2D profile on its OCS Z-plane. `Polyline2D : Polyline<Vertex2D> : IPolyline` exposes `Vertices: SeqendCollection<Vertex2D>` (a `CadObjectCollection<Vertex2D>` subclass: `IEnumerable<Vertex2D>` with `Count`/`this[int]`, NOT a `List<T>` — a consumer enumerates it through `toSeq`, never a `List`-only member), with `IsClosed` projected off `Flags`; `Vertex2D : ACadSharp.Entities.Vertex` carries `Location: XYZ` as a PLAIN property (there is no `Pt(XYZ)` overload — read `.Location` directly) and a `Bulge: double`. `Arc : Circle` inherits `Center` (`XYZ`) and `Radius` (`double`) and adds `StartAngle`/`EndAngle`/`Sweep` (radians) plus `GetEndVertices(out XYZ start, out XYZ end)` for the two endpoints without sampling; `Circle : Entity, ICurve` carries `Center`/`Radius`/`Normal`/`Thickness`. ACadSharp SHIPS the bulge-to-arc factory the README names — `Arc.CreateFromBulge(XY p1, XY p2, double bulge)` (with `Arc.GetCenter(start, end, bulge)` for the center directly and `Arc.GetCenter(start, end, bulge, out radius)` for center+radius without minting) — and the curve sampler `Arc.PolygonalVertexes(int precision)` (an override) / `Circle.PolygonalVertexes(int precision)` (the virtual base sampler) returning `List<XYZ>` in object-coordinate space, so the boundary tessellates every bulge/arc/circle span through the package-owned sampler at one `precision` knob, never a hand-rolled bulge trigonometry. A zero `Bulge` is a straight segment kept as the raw `Location`; a non-zero `Bulge` mints the arc through `CreateFromBulge` and samples it.
- [SPLINE_SAMPLER] `Spline : Entity` ships a native NURBS tessellator, so the boundary samples the spline through the package surface and never a hand-rolled in-folder de Boor or control-polygon walk. `Spline.PolygonalVertexes(int precision)` returns the chord-sampled `List<XYZ>` in WCS at the `precision` knob — the same chord-precision knob the `Arc`/`Circle` samplers carry — and `Spline.TryPolygonalVertexes(int precision, out List<XYZ>)` is the non-throwing probe that returns `false` on an unevaluable spline (the boundary lowers a `false` to `GeometryFault.DegenerateInput`). The control-point/knot data (`ControlPoints: List<XYZ>`, `Knots: List<double>`, `Weights: List<double>`, `Degree: int = 3`, `FitPoints: List<XYZ>`) plus `IsClosed` (`Flags.HasFlag(SplineFlags.Closed)`) and `IsPeriodic` (`SplineFlags`/`SplineFlags1` projection) are read-side properties; a spline authored from `FitPoints` only is rebuilt through `UpdateFromFitPoints(uint iterationLimit = 255)` before sampling. `PointOnSpline(double t)`/`TryPointOnSpline(double t, out XYZ)` evaluate a single parametric point when a non-uniform sampling is needed.
- [BLOCK_TRAVERSAL] `CadDocument.Entities` is exactly `ModelSpace.Entities` and does NOT flatten nested blocks: an `Insert` entity references its resolved block directly through `Insert.Block` (a live `ACadSharp.Tables.BlockRecord`, the DXF code-2 name resolved), whose geometry is `BlockRecord.Entities` (a `CadObjectCollection<Entity>`), and carries its own placement transform that must compose explicitly. The placement is `InsertPoint: XYZ` (block origin in WCS), `XScale`/`YScale`/`ZScale: double` (per-axis scale), `Rotation: double` (radians about `Normal`), and `Normal: XYZ` (the OCS axis); an arrayed `Insert` additionally carries `ColumnCount`/`RowCount: ushort` (default 1) and `ColumnSpacing`/`RowSpacing: double`. ACadSharp SHIPS the block flattener: `Insert.Explode(): IEnumerable<Entity>` resolves `Block.Entities` AND applies the composed placement transform to each child in one call, and `Insert.GetTransform(): CSMath.Transform` / `Insert.ApplyTransform(Transform)` expose that affine directly (`Transform` carries `Matrix: Matrix4`, `Translation: XYZ`, `Scale: XYZ`, `EulerRotation: XYZ`). The boundary admits ONLY the top-level `ModelSpace.Entities` closed profiles this campaign; the nested-block flattening arm composes `Insert.Explode()` (the transformed child entities) and recurses `Admit` over the exploded entities — reusing the package-owned transform composer rather than hand-building the OCS-to-WCS matrix — and an arrayed `Insert` emits one transformed child set per `Column`/`Row` cell on the same arm. The package owns the transform; a hand-rolled `Insert` matrix is the rejected form.
- [CONFIG_KNOBS] The config overload (`DxfReader.Read(path, DxfReaderConfiguration, notification)` / `DwgReader.Read(path, DwgReaderConfiguration, notification)`) is the boundary's resilience and metadata switch. `CadReaderConfiguration` (the shared base) carries `Failsafe: bool = true` (resilient read; `false` promotes recoverable defects to throws), `KeepUnknownEntities: bool`, and `KeepUnknownNonGraphicalObjects: bool` (admit-vs-drop policy for entity/object kinds the reader does not model — left default they are dropped before they ever reach `Admit`). `DxfReaderConfiguration` adds `ClearCache: bool` and `CreateDefaults: bool`; `DwgReaderConfiguration` adds `CrcCheck: bool` and `ReadSummaryInfo: bool` (gate the summary-info block `DwgReader.ReadSummaryInfo()` reads). The boundary's default path passes no config (taking `Failsafe=true`); a strict-ingest mode passes `Failsafe=false` to fail fast on any recoverable defect, and a metadata-aware DWG path enables `ReadSummaryInfo`.
