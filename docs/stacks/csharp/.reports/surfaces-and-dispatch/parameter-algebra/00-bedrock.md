# Parameter Algebra

[POLICY_VALUES]:
- A smart-enum row is the canonical policy value form: the row carries constructor-injected behavior delegates alongside its key, and the generated `Switch<TState, TResult>(TState state, Func<TState, CaseN, TResult> onCaseN, ...)` threads state through each case without the caller re-deriving which case was selected. `SwitchMapMethodsGeneration.DefaultWithPartialOverloads` generates separately named `SwitchPartially`/`MapPartially` members with two distinct signatures: the action form is `SwitchPartially(Action<TOwner>? @default = null, Action? caseN = null, ...)` where `@default` is optional and receives the owner when no case handler matches; the func form is `SwitchPartially<TResult>(Func<TOwner, TResult> @default, Func<TResult>? caseN = null, ...)` where `@default` is required and is invoked with the owner for unhandled cases; `MapPartially<TResult>(TResult @default, Argument<TResult> caseN = default, ...)` where `@default` is a required plain value that is returned directly for unhandled cases, and per-case optionality travels as `Argument<TResult>.IsSet` — when `IsSet` is `false` the case is unhandled and `@default` is returned, when `true` the stored `Value` is returned. `[UnionSwitchMapOverloadAttribute(StopAt = new[] { typeof(FailureCase) })]` generates an additive `Switch` overload that accepts a single `Action<FailureCase>` (or `Func<FailureCase, T>`) without descending into its derived leaf cases; the full exhaustive overload is always generated alongside it. This is the correct shape when a dispatch site delegates handling of an entire sub-hierarchy to one handler without knowing its leaf structure.
- `[SmartEnum<TKey>]` generates a `Key` property typed `TKey` and implements `IParsable<T>` and `ISpanParsable<T>` when `TKey` supports them; the parsed key is the row's identity, not a string literal requiring a second match pass. `SkipIParsable = true` forces `SkipISpanParsable = true` because `ISpanParsable<T>` inherits `IParsable<T>`; the coupling is one-way — setting `SkipISpanParsable = true` alone does not force `SkipIParsable`.
- Keyless `[SmartEnum]` generates the same `Switch`/`Map` shape with `SwitchMapStateParameterName` defaulting to `"state"`; the name is overridable on the attribute so generated parameter names remain consistent with codebase vocabulary without touching generated code. `EqualityComparisonOperators` on the keyless form produces no key-overload variant because there is no key member.
- Stateless union members in ad-hoc `[Union<T1,...>]` and `[AdHocUnionAttribute]` forms store only the discriminator index with no payload; generated accessors return `default(T)` for stateless cases. Using a struct for the stateless marker type makes the accessor value structurally non-null. `T1IsStateless = true` implies `T1IsNullableReferenceType = true` automatically for reference-type members. A stateless case carries no information beyond the discriminant — the correct shape for policy entries with no configuration data (`None`, `Default`, or `Absent`). Wrapping a stateless case in `Option<T>` adds a second optionality layer on top of a discriminant that already models absence; `Option<T>` belongs on parameters where the entire union is absent, not inside a union that already encodes absence as a case.
- `NestedUnionParameterNameGeneration.Simple` on a `[Union]` changes generated `Switch` parameter names from the fully-qualified `failureNotFound` form to the leaf-only `notFound` form. When two nested families share a leaf name, the generated code fails to compile; `Default` is the safe choice whenever collision is possible.
- `IEqualityComparerAccessor<TKey>` declares `static abstract IEqualityComparer<TKey> EqualityComparer { get; }`. The generator calls `TAccessor.EqualityComparer` at the type level when resolving the comparer for a smart enum's generated `GetHashCode` and `Equals`. `ComparerAccessors.StringOrdinal`, `StringOrdinalIgnoreCase`, and culture variants implement both `IEqualityComparerAccessor<string>` and `IComparerAccessor<string>`. A custom comparer accessor is the mechanism for changing hash and equality semantics without mutating the key type. `[KeyMemberEqualityComparerAttribute<TAccessor, TKey>]` and `[KeyMemberComparerAttribute<TAccessor, TKey>]` carry two type parameters (accessor first, key second) and are `allow-multiple = false`; only one comparer accessor governs the key member. A custom accessor must be consistent with the `FrozenDictionary` comparer used for any frozen policy table keyed on the same smart enum — ordinal on the smart enum and ordinal-ignore-case on the frozen table produces silent lookup divergence.
- Ad-hoc unions (`[Union<T1,T2>]`) through `[Union<T1,T2,T3,T4,T5>]` inherit from `UnionAttributeBase`, which exposes both `ConversionFromValue` (default `Implicit`) and `ConversionToValue` (default `Explicit`). Regular `[Union]` (non-generic, inheriting `Attribute` directly) exposes only `ConversionFromValue` and has no `ConstructorAccessModifier` property. The asymmetry is intentional: ad-hoc unions are structurally typed composites where extracting a member case back to its naked type is an explicit operation, whereas regular unions rely on `Switch`/`Map` for case consumption. Setting `ConversionToValue = ConversionOperatorsGeneration.None` forces all member extraction through generated accessors, eliminating ambiguous implicit downcast paths. The generic forms support up to five type parameters; beyond five the correct shape is a regular `[Union]` with nested case records. The `T1Name` property changes the name of all generated members regarding `T1`: `Switch`/`Map` parameter names for that slot, and the generated `IsT1`/`AsT1` accessor names are renamed to `Is{T1Name}`/`As{T1Name}` — callers relying on the default accessor names must update every reference site when `T1Name` is set.
- `UnionAttributeBase.DefaultStringComparison` defaults to `StringComparison.OrdinalIgnoreCase` and governs the generated `GetHashCode` and `Equals` for string-typed members in ad-hoc unions — the generator emits `value?.GetHashCode(global::System.StringComparison.X)` inside the `_valueIndex` switch for string slots. It does not govern JSON discriminators or `ToString`/parsing paths on ad-hoc unions, which have no wire discriminator surface. Setting it to `StringComparison.Ordinal` changes structural equality semantics for string members throughout all generated `Switch`-arm hash paths.
- `UnionAttributeBase.ConstructorAccessModifier` (available on all `UnionAttributeBase` subclasses — the generic `[Union<T1,...>]` and `[AdHocUnionAttribute]` forms; absent on regular non-generic `[Union]`) defaults to `UnionConstructorAccessModifier.Public`; `ValueObjectAttributeBase.ConstructorAccessModifier` defaults to `AccessModifier.Private`. This asymmetry is the architecture signal: union case construction is open by default; value-object construction is always factory-gated. Setting `ConstructorAccessModifier = Private` on an ad-hoc union privatizes both the constructor and the implicit conversion operators together as one attribute property change — the constraint is not split across two settings.
- `UnionAttributeBase.UseSingleBackingField = true` forces an `object` backing field for all member cases, including struct cases; struct cases are boxed on storage and unboxed on access. This is a memory-reduction trade-off when cases have significantly unequal sizes. Stateless struct cases already store only the discriminator index with zero payload, making `UseSingleBackingField` irrelevant for those slots.
- `FactoryMethodGeneration` on `UnionAttributeBase` controls factory method generation for ad-hoc union members: `Default` detects triggers (type-parameter members, `interface` or `object` members, duplicate member types); `None` suppresses generation globally; `Always` generates for every member. `FactoryMethodGeneration = None` combined with `ConstructorAccessModifier = Private` produces a type with no public construction path; the custom factory must be authored manually.
- `ConversionOperatorsGeneration.Implicit` (default for union-to-value) lets a policy case convert implicitly to the union type at call sites; `Explicit` forces a cast. Setting `ConstructorAccessModifier = Private` suppresses both the constructor and the implicit conversion operator together, requiring factory methods — the correct shape when the factory carries admission logic beyond naked construction.
- `UnionAttributeBase.SkipEqualityComparison = true` suppresses `IEquatable<T>`, `Equals`, `GetHashCode`, and all comparison operators. This is the correct form for unions used as discriminated command or request types where structural equality is meaningless — equality generation is wasted code at that use site. The suppression makes the `UseSingleBackingField` choice irrelevant to correctness, since the backing-field layout would otherwise affect `GetHashCode` bucket distribution.

[OPTIONAL_CONTEXT]:
- `Option<T>` is the single admitted form for optional parameters carrying absence semantics. `Option<A>` implements `K<Option, A>` and exposes `IsSome`, `IsNone`, and a `Case` property returning `object?` that enables `{ IsSome: true, Case: T value }` property-pattern destructuring without triggering the `Match`-in-expression analyzer prohibition.
- `Option<A>` carries implicit conversions from `A?` (reference type: null maps to `None`) and from `Fail<Unit>` or `Unit` (both map to `None`), making `None` the Prelude constant for both the absence sentinel and the option initializer. The `|` operator on two `Option<A>` values delegates to `Choose` via the `Choice<Option>` trait — a left-biased result returning the first `Some`, but both operands are eagerly evaluated before the operator runs; `Combine` (the `Semigroup<Option<A>>` implementation) is equivalent. Short-circuit evaluation requires `||`: `Option<A>` defines `operator true` (`IsSome`) and `operator |` (left-biased choice), which together satisfy C#'s `||` user-defined short-circuit contract — `optA || optB` skips evaluation of `optB` when `optA.IsSome`. `Option<A>` defines no `operator &`, so `&&` does not compile. Guard expressions that need short-circuit absence-testing chain with `||`; those that require all-present use `Bind`/`Map` or property-pattern destructuring.
- `Option<A>.operator true` returns `IsSome`; `operator false` returns `IsNone`. These are the two hooks C# requires to compile `||` for a user-defined type alongside `operator |`; `&&` additionally requires `operator &`, which `Option<A>` does not provide.
- `Option<T>` as an optional parameter default requires `= default` at the declaration and `IfNone(canonical)` at use; `Option<TVo>` where `TVo` is a `[ValueObject]` or `[SmartEnum]` avoids the CS1736 constraint that forbids a non-const default. The canonical fallback comes from the policy owner — the smart-enum static property or value-object factory — so the default is derived, not re-stated at the call site.
- A runtime context parameter that is optional but carries identity enters as one record, not as a flag tail. A nullable record parameter `T? ctx = null` is the boundary-only form; inside the domain it projects to `Option<T>` at the admission site so interior code never branches on null.

[POLICY_TABLE_FORM]:
- `FrozenDictionary.Create<TKey, TValue>(ReadOnlySpan<KeyValuePair<TKey, TValue>> source)` and `Create<TKey, TValue>(IEqualityComparer<TKey>?, ReadOnlySpan<...>)` are span-based construction overloads. A collection expression at the call site (`[new("k", v), ...]`) binds to the `ReadOnlySpan<T>` parameter with no heap allocation for the intermediate collection.
- `FrozenDictionary<TKey, TValue>` exposes `GetValueRefOrNullRef(TKey key)` returning `ref readonly TValue`; callers test for absence via `Unsafe.IsNullRef(in result)` without boxing the value. `AlternateLookup<TAlternateKey>` (where `TAlternateKey : allows ref struct`) enables `ReadOnlySpan<char>` keying against a string-keyed table with no string allocation; `TryGetAlternateLookup` returns `false` when the comparer does not implement `IAlternateEqualityComparer<TAlternateKey, TKey>`. Combined with `AlternateLookup`, a policy-table lookup on a `ReadOnlySpan<char>` key is allocation-free end-to-end.
- `TryGetAlternateLookup<TAlternateKey>` succeeds only when the dictionary's `Comparer` implements `IAlternateEqualityComparer<TAlternateKey, TKey>`. For string-keyed frozen dictionaries, the only accepted alternate key type is `ReadOnlySpan<char>`; no other alternate key type is accepted for `string` keys regardless of comparer capabilities. `GetAlternateLookup<TAlternateKey>()` throws `InvalidOperationException` when the constraint is unsatisfied. `IAlternateEqualityComparer<TAlternate, T>` requires `bool Equals(TAlternate, T)`, `int GetHashCode(TAlternate)`, and `T Create(TAlternate)` — the `Create` method is used internally for operations that need to produce a canonical key from the alternate. `FrozenSet<T>.AlternateLookup<TAlternate>` exposes `TryGetValue(TAlternate, out T)` rather than a `ref readonly` path; `FrozenDictionary<K,V>.AlternateLookup<TAlternate>` exposes only the indexer, `ContainsKey`, and `TryGetValue` — `GetValueRefOrNullRef` exists solely on the main-key surface and is not carried through the alternate-key path. Span-keyed dictionary lookups do not share the ref-return optimization regardless of the lookup form used.
- A frozen policy table is the implementation surface for bounded vocabularies with non-delegate behavior: the smart-enum row owns behavior via delegate fields; the frozen table owns behavior via value structs or delegates stored as map values. The table is a single static field; a lookup returns the complete policy value and the caller never reconstructs behavior from key combinations.
- `FrozenDictionary<TKey, TValue>.Keys` returns `ImmutableArray<TKey>`, not `IEnumerable<TKey>`. Iterating via `foreach` over the `ImmutableArray<TKey>` never boxes and never allocates; the explicit-interface `IDictionary<TKey, TValue>.Keys` allocates a wrapper. When enumerating policy-table keys for validation or bulk projection, `Keys` (not the `IDictionary` interface) is the zero-allocation path. The same applies to `FrozenSet<T>` via `Items`.
- `Thinktecture.Collections.ProjectionEqualityComparer<T, TItem>` constructs an `IEqualityComparer<T>` from a `Func<T, TItem> selector` and an optional inner `IEqualityComparer<TItem>`. A `FrozenDictionary<RequestType, PolicyRow>` keyed on a union-case type uses `ProjectionEqualityComparer<RequestType, Kind>` where `Kind` is the discriminant, so two requests with the same kind map to the same policy row regardless of payload differences. The comparer is the policy definition, not infrastructure: it declares which dimension of the request type governs policy selection.
- `StringKeyedObjectComparer<T>.Ordinal`, `OrdinalIgnoreCase`, and the culture variants provide `IEqualityComparer<T>` for smart-enum or value-object types keyed on `string`. Using `StringKeyedObjectComparer<T>.Ordinal` as the comparer for a `FrozenDictionary<T, PolicyRow>` is the correct form: it delegates to `StringComparer.Ordinal`, preserving consistency between lookup semantics and enum identity.

[ARITY_COLLAPSE]:
- `params ReadOnlySpan<T>` is the zero-allocation arity collapse form. The compiler emits an inline array or stack-allocated span at the call site — no `IEnumerable<T>` heap object. When a `params ReadOnlySpan<T>` overload coexists with a `params T[]` overload, the compiler prefers the span form for both expanded calls and collection expressions without requiring `[OverloadResolutionPriority]`; the array overload is therefore redundant in a greenfield context and should be removed rather than annotated. A `params` parameter that re-describes what the receiver already discriminates is the prohibited form: if the entrypoint dispatches on the case of a union input, adding a parallel `params` array carrying case-correlated overrides is arity smuggling.
- Collection expressions (`[a, b, c]`) bind to any `params` target type at the call site without specifying the collection kind; the compiler selects the best collection builder. A `params ReadOnlySpan<T>` parameter with a collection expression call site produces no heap allocation when the count is small enough for an inline array.
- `Argument<T>` is a `readonly ref struct where T : allows ref struct` carrying `Value: T` and `IsSet: bool`; the implicit conversion from `T` sets `IsSet = true` and a default `Argument<T>` has `IsSet = false`. It cannot be boxed, stored in a field, or held across an `await` boundary — use is strictly within a synchronous call frame. As a `ref struct` with an `allows ref struct` constraint it is valid as a generic type argument only to parameters that also declare `allows ref struct`. The generator's canonical consumer of `Argument<T>` is the `MapPartially`/`SwitchPartially` per-case optional handler surface: each optional case slot is typed `Argument<TResult>` (or its action equivalent), carrying the caller's supplied value when `IsSet` is `true` and signaling omission when `IsSet` is `false` so the fallback `@default` is returned without re-invoking any case logic. For bespoke factory overloads authored alongside `[ObjectFactoryAttribute<T>]`, `Argument<T>` is the correct form when the factory must distinguish "caller supplied a value that equals the domain default" from "caller omitted the argument." The generated `ValidateFactoryArguments` partial takes `ref T` for each declared field, not `Argument<T>`; expecting `Argument<T>` in a generated `IObjectFactory` interface produces a compile-time mismatch against the `static abstract Validate(TValue?, IFormatProvider?, out T?)` signature.

[DEFAULTS_AND_DERIVATION]:
- Generated value-object factory defaults derive from the owning type, not from the call site. `AllowDefaultStructs = false` (default) prohibits `default(T)` and parameterless construction of struct value objects and generates no `Empty` static member. `AllowDefaultStructs = true` permits the struct-zero state and emits the `static readonly T Empty = default;` field named by `DefaultInstancePropertyName` (default `"Empty"`). Callers using `Option<TVo> = default` receive `None` at the call site while the domain default lives on the type only when `AllowDefaultStructs = true` enables it.
- `SkipFactoryMethods = true` suppresses `Create`, `Validate`, `TryCreate`, the `TypeConverter`, both `IObjectFactory<T>` interfaces, the implicit conversion from key type to value object, and all arithmetic operator generation (permanently), and forces `SkipIParsable = true` and `SkipISpanParsable = true`. An `[ObjectFactoryAttribute<T>]` on the same type restores serialization and EF Core routes but does NOT restore arithmetic operators or `IParsable`/`ISpanParsable`. `SkipIComparable` is independent and is never affected by `SkipFactoryMethods`; suppressing the comparable path requires `SkipIComparable = true` explicitly. Setting `SkipFactoryMethods` requires a manually authored `[ObjectFactoryAttribute<T>]`-marked factory, which is the correct shape when admission logic spans multiple sources or requires `Fin<T>` rail integration.
- `IObjectFactory<TValue>` (one type parameter) is a marker interface with no members. `IObjectFactory<T, TValue, out TValidationError>` (three type parameters) inherits from it and adds `static abstract TValidationError? Validate(TValue?, IFormatProvider?, out T?)`. The `static abstract` method receives the raw `TValue?` — not `Argument<TValue>`.
- `[ObjectFactoryAttribute<T>]` (allow-multiple) registers a custom factory keyed on `T`; `UseForSerialization` directs the specified serialization frameworks to it. `UseWithEntityFramework = true` routes EF Core value converters through the factory. Multiple attributes on one owner register independent conversion paths for different source types with no shared global policy. `HasCorrespondingConstructor` (bool, default `false`) controls whether the generator emits a direct-constructor expression as a fast path for conversion from a trusted source such as EF Core materialization; when `true`, values arriving from a validated store bypass re-validation while values from wire sources still pass through `Validate`. A type can declare `[ObjectFactoryAttribute<Guid>](UseForSerialization = SerializationFrameworks.SystemTextJson)` for JSON and `[ObjectFactoryAttribute<string>](UseForSerialization = SerializationFrameworks.MessagePack)` for binary with different admission validators for each; `UseWithEntityFramework` and `UseForModelBinding` are always per-attribute, so EF Core can use a different factory than the JSON deserializer on the same type.
- `IValidationError<T>` is a `static abstract` interface member: `static abstract T Create(string message)`. A custom error type implements this interface; the generated factory invokes `TCustom.Create(message)` at validation failure time. The policy of which error type to produce lives on the domain type declaration, not at every factory call site. `[ValidationErrorAttribute<TError>]` requires `TError : class, IValidationError<TError>` and registers at the declaring type level, propagating `TError` as the error type for all generated factory methods on that type with no per-factory override; when two value objects in a hierarchy need different error types, each declares its own attribute. The default `ValidationError` type carries only a `Message: string` property; a custom type replaces `ValidationError` entirely in generated factory signatures.
- `ObjectFactoryAttribute<T>` declares `where T : allows ref struct`. Applying `[ObjectFactoryAttribute<ReadOnlySpan<char>>]` on a value object produces a factory seam accepting `ReadOnlySpan<char>` directly, enabling zero-allocation admission at parsing boundaries without an intermediate `string` allocation. `UseForSerialization = SerializationFrameworks.SystemTextJson` routes the `Utf8JsonReader`-based deserializer through the same span factory for zero-copy JSON parsing. `IConvertible<out T>` (where `T : allows ref struct`) enables the reverse path: `ToValue()` returning `ReadOnlySpan<char>` without allocation. The generated implicit conversion operator does not use this interface; callers invoke `ToValue()` explicitly.
- `ValueObjectAttribute<TKey>.UnsafeConversionToKeyMemberType` (default `Explicit`) generates an `explicit operator TKey(TObj obj)` that can throw `InvalidOperationException` when the value object wraps an unset state (`AllowDefaultStructs = true`). Setting it to `None` removes this operator; callers use `ToValue()` or `.Key` explicitly. The safe implicit operator (`ConversionToKeyMemberType = Implicit`) never throws and is the only conversion generated when `AllowDefaultStructs = false`.
- `NullInFactoryMethodsYieldsNull = true` changes the factory contract so `Create(null)` returns `null` instead of a validation error. Setting `EmptyStringInFactoryMethodsYieldsNull = true` automatically sets `NullInFactoryMethodsYieldsNull = true`; the reverse is not guaranteed. Setting `EmptyStringInFactoryMethodsYieldsNull = true` alone is the only way to make both null and empty yield `null`.
- `ValueObjectAttribute<TKey>.SkipKeyMember = true` suppresses generation of the key field/property. The generator reads `KeyMemberName` to locate the manually-declared member, which must be a property with an `init` setter when `KeyMemberKind = MemberKind.Property` — the generated constructor assigns `this.KeyMemberName = value`, requiring a writable accessor. For fields (`MemberKind.Field`), a `readonly` field is accepted because the generated constructor may initialize `readonly` members.

[COMPOSITION_TIME_POLICY]:
- `Schedule` operator semantics are three distinct combinators sharing no behavior: `|` (`Union`) runs both schedules in parallel and emits the minimum duration at each step until the longer stream exhausts; `&` (`Intersect`) emits the maximum duration at each step until the shorter stream exhausts; `+` (`Combine`) concatenates sequentially — the first schedule's durations run to exhaustion, then the second's continue. Conflating `+` (sequential) with `|` (parallel minimum) produces schedules that run more iterations than intended.
- `ScheduleTransformer + ScheduleTransformer` composes two transformers left-to-right via `f + g = x => g.Apply(f.Apply(x))`; the composition is not commutative. `NoDelayOnFirst + jitter(0.1)` removes the first delay then adds jitter to all remaining steps; `jitter(0.1) + NoDelayOnFirst` adds jitter first and then replaces the first jittered duration with zero. Transformer composition order is the stacking order for policy mutation. A `ScheduleTransformer` stored in a `static readonly Schedule` field triggers the implicit conversion `ScheduleTransformer → Schedule.Forever | transformer` — the infinite `Forever` base is invisible at the declaration site; a field declared as `ScheduleTransformer` converts nothing.
- `Schedule.recurs(n)` is a `ScheduleTransformer` that limits iteration count; it is the only form that should appear in a stored policy when the intent is a retry count. `Schedule.repeat(int n)` is also a `ScheduleTransformer`; its implicit conversion produces `Schedule.Forever | repeat(n)` — an infinite base repeated `n` times, not a finite iteration multiplier on the left operand. Combining a finite schedule with `+ Schedule.repeat(2)` appends an infinite `Forever`-backed stream that the left schedule never reaches when the left side is itself infinite; for finite left schedules, it concatenates a repeated-forever tail. The `|` operator between a `Schedule` and a `ScheduleTransformer` applies the transformer — `Schedule.exponential(100*ms) | Schedule.recurs(3)` applies `recurs(3)` to cap the exponential at three iterations. `Schedule.NoDelayOnFirst` and `Schedule.RepeatForever` are static `ScheduleTransformer` fields, not `Schedule` values. `Schedule.jitter(factor, seed)` and `Schedule.jitter(minRandom, maxRandom, seed)` are `ScheduleTransformer`s accepting `Option<int>` for the seed, where `default(Option<int>)` means non-deterministic — the seeded-vs-random distinction travels as presence or absence in the option, not as a boolean knob.
- `Schedule.upto`, `fixedInterval`, `windowed`, `secondOfMinute`, `minuteOfHour`, `hourOfDay`, and `dayOfWeek` all accept a `Func<DateTime> currentTimeFn` parameter; the default uses `DateTime.UtcNow`. The policy value carries its own time source rather than reading a global — `static readonly Schedule CronLike = Schedule.minuteOfHour(0, () => _clock.UtcNow)` binds the clock at construction, not at each retry step. `Schedule.fixedInterval` suppresses delay (emits zero) when the action overruns the interval, but the next window starts from the completion time — the load-leveling form. `Schedule.windowed` always emits the residual time to the next absolute boundary regardless of action duration — the cron-alignment form. Using `windowed` when the intent is load-leveling causes delayed actions to catch up immediately then run at fixed boundary times, producing burst behavior at boundary crossings.
- `CatchM<E, M, A>` is the named catch policy value: `Match: Func<E, bool>` selects the error; `Action: Func<E, K<M, A>>` provides recovery. The `|` operator on `K<F, A>` accepts a `CatchM` on the right, composing recovery policy as a linear chain without nesting; the first matching catch consumes the error, so a broad predicate placed before a narrow one shadows the narrower catch silently. The `|` operator is defined in `FallibleExtensions` constrained to `where F : Fallible<E, F>` — any monad implementing `Fallible<E, M>` inherits it, making a `static readonly CatchM<Error, Eff<RT>, A>` named catch policy directly applicable via `|` without monad-specific wiring.
- `exceptional<M, A>(Func<Exceptional, K<M, A>>)` and `expectedOf<E, M, A>(Func<E, K<M, A>>)` are the two orthogonal catch axes on the `Error` type hierarchy. `Expected` errors (`IsExpected = true`) and `Exceptional` errors (`IsExceptional = true`) are separate classes of failure; a policy that handles domain errors and one that handles infrastructure faults are two separate `static readonly CatchM` values, each applied to the computation in sequence (`comp | expectedPolicy | exceptionalPolicy`) — no operator composes two `CatchM` values into one; the chain always grows on the computation. `catchOf<E, M, A>(Func<E, K<M, A>> Fail)` calls `e.IsType<E>()` recursively across an aggregate, then folds matching sub-errors through `Fail` via `ForAllM` — a plain `@catch` with `e => e is E` matches only the top-level error. `catchOfFold` and `exceptionalOfFold` are the monoid-fold variants: `ForAllM` is fail-fast on first handler failure; `FoldM` continues processing sub-errors after individual handler failures — the policy decision is aggregate fail-fast vs. collect-all. `Error.Filter<E>()` returns sub-errors of type `E`; when none match it returns `Errors.None` (a `ManyErrors` with an empty `Seq<Error>`). `ManyErrors.IsEmpty` is an authored override (`public override bool IsEmpty => Errors.IsEmpty`) — `IsEmpty` is always available; `error.IsType<MyError>()` is the correct shape when the intent is to test before filtering, not because `IsEmpty` is absent.
- `bracketIO<M, A, B, C>(Acq, Use, Catch, Fin)` admits a `Catch: Func<Error, IO<C>>` parameter — a raw function, not a `CatchM`. A named `static readonly CatchM` cannot be passed directly; the bridge is a `static readonly Func<Error, IO<C>>` that the `CatchM`'s `Action` delegates to — predicate-selection lives in the `CatchM`, recovery body is shared by reference.
- `retryWhile(Schedule, K<M, A>, Func<Error, bool>)` and `retryUntil(Schedule, K<M, A>, Func<Error, bool>)` add a predicate gate to the schedule-driven retry loop; the schedule governs spacing and count, the predicate governs the error domain that admits retries. `repeatWhile` and `repeatUntil` accept `Func<A, bool>` — the predicate receives the bound value, not an error. Using `repeatWhile` when the intent is to abort on certain errors keeps repeating until the schedule expires. The correct split: schedule-driven retry for error recovery; schedule-driven repeat with value predicate for polling — never conflated on one call site. Both the schedule and the predicate are named fields on a policy record, not flags passed at call sites.
- `Schedule.Bind(Func<Duration, Schedule> f)` fans each emitted duration into a sub-schedule derived from it, then concatenates all sub-schedules. A duration-parameterized policy value is expressed as a C# query expression: `from d in baseSchedule from d2 in Schedule.spaced(d * factor) select d + d2`. `Schedule.SelectMany(Func<Duration, Schedule> bind, Func<Duration, Duration, Duration> project)` supplies both the outer and inner duration to the projection, enabling capping sub-schedule output to the outer duration. `ScheduleTransformer` has no `SelectMany`, `Select`, or `Bind`; only `Schedule` supports LINQ query expressions. `Schedule | Schedule.spaced(cap)` clamps maximum delay per step; `Bind(d => Schedule.spaced(d.Min(cap)))` clamps by transforming each duration before producing the sub-schedule — the bind form preserves duration-level provenance and is load-bearing when the policy derives the next sub-schedule from the previous duration.
- `Schedule.Interleave(Schedule b)` zips both streams into pairs and emits both durations from each pair; the emitted count is `2 × min(a.Length, b.Length)`, terminating when the shorter stream exhausts. Using `Interleave` where union was intended doubles the retry count silently. `Schedule.maxDelay(Duration max)` is a `ScheduleTransformer` capping per-emission magnitude; `Schedule.maxCumulativeDelay(Duration max)` terminates when the running sum reaches `max` — these are orthogonal caps applied together as `| maxDelay(2*s) | maxCumulativeDelay(30*s)`.
- A schedule is composed once into a `static readonly` field and passed as a single `Schedule` argument to `retry` or `repeat`; no parameters inside the retry loop re-specify the backoff. A default schedule lives on the owning smart-enum row or policy record; callers that accept the default pass the field, and callers that need a different schedule pass their own composed value.

[POLICY_VALUE_THREADING_FAILURE_SURFACES]:
- `SmartEnumAttribute<TKey>.ComparisonOperators` coerces `EqualityComparisonOperators` upward: when `ComparisonOperators` is set to `DefaultWithKeyTypeOverloads`, `EqualityComparisonOperators` is automatically promoted to the same level because comparison operators require equality operators. The reverse also holds: `ComparisonOperators` is forced to `None` when `EqualityComparisonOperators` is `None`. Setting the two independently produces a silent coercion; `ComparisonOperators` is the single authority.
- `SmartEnumAttribute<TKey>.DisableSpanBasedJsonConversion = false` (default) causes the generated JSON converter to use a `ReadOnlySpan<char>`-keyed path during deserialization, avoiding a string allocation per JSON token. The flag has no effect when `KeyMemberType` is not `string`. Keeping this flag at its default across all string-keyed smart enums is the zero-configuration allocation reduction.
- `[MemberEqualityComparerAttribute<T, TMember>]` (where `T : IEqualityComparerAccessor<TMember>`, accessor type first) applies to a property or field of a `[ComplexValueObject]` and overrides the equality comparer for that specific member in `GetHashCode` and `Equals`. Multiple declarations on different members of the same complex value object are independent — each member carries its own comparer. This is the per-field equality policy: a complex value object with a `string` field requiring ordinal comparison and a numeric field requiring exact equality declares both without a shared global policy.

[EXTENSION_BLOCKS]:
- Extension blocks (`extension(ReceiverType r) { ... }`) declare the receiver type once; all members inside share the receiver without repeating `this T` on each. Methods, properties, and static operators are admissible in extension blocks; indexers are rejected with CS9282 under the current compiler — only methods, properties, and operators form the admitted surface.
- Extension blocks intersect with parameter algebra at the policy surface: a frozen table or smart enum can acquire computed access members — projections that derive secondary policy values from the stored primary — without polluting the owner type with unrelated helpers. The block keeps those projections co-located with the vocabulary that governs them.

[ANTI_PATTERNS]:
- A boolean parameter selecting between two bodies is a policy inlined as a knob. The collapsed form uses a smart-enum row or union case: the case carries its own body via a constructor-injected delegate; the caller passes the pre-constructed case, and no `if` or `switch` re-derives the body at the dispatch site.
- A nullable flag tail — `T? optA = null, T? optB = null, bool doX = false` — fragments one optional context into multiple parameters. The collapsed form is one `Option<ContextRecord>` carrying all optional fields; absence of the record means "use defaults"; presence carries the override bundle. `Argument<T>` is the alternative when the factory must distinguish "caller provided this specific value" from "caller omitted, use domain default."
- A `params T[]` overload alongside a semantically identical `params ReadOnlySpan<T>` overload is dead surface: the compiler prefers span unconditionally for both expanded calls and collection expressions without any `[OverloadResolutionPriority]` annotation. The array overload should be removed.
