# Carrier-Polymorphic Event-Seam Transformer

[CARRIER_POLYMORPHIC_SPINE]:
- A seam capsule is written exactly once against `K<M, A>` constrained `where M : MonadUnliftIO<M>`, never against the concrete effect type. The two arrows that make every host-thread, retry, and fault concern carrier-agnostic are `mapIO<M, A, B>(Func<IO<A>, IO<B>>, K<M, A>)` and `toIO<M, A>(K<M, A>) : K<M, IO<A>>`. `mapIO` reaches through the runtime-transformer stack, applies an `IO`-level transform to the innermost `IO`, and re-wraps ‚Äî so a single `io => io.Post()`, `io => io.Repeat(schedule)`, or `io => io.Bracket()` becomes a transformer that holds for the bare effect and for any reader/state/writer/fin transformer layered above it without one rewrite per layer.
- The trait the capsule binds is structural, not nominal: `MonadUnliftIO<M>` supplies `MapIO`, `ToIO`, and `BracketIO<A, B, C>(K<M,A> Acq, Func<A,IO<C>> Use, Func<Error,IO<C>> Catch, Func<A,IO<B>> Fin)`; `Fallible<Error, M>` supplies `Catch<A>(K<M,A>, Func<Error,bool>, Func<Error,K<M,A>>)`; `Final<M>` supplies `Finally<X, A>(K<M,A>, K<M,X>)`. A seam that needs attach + marshal + retry + fault-fence + ordered teardown is the intersection of these four traits ‚Äî declare the constraint set once and the whole composition is available generically, so the capsule never names `IO` in its body.
- The marshal transform `io => io.Post()` lifts through `MonadUnliftIO<M>.PostIO` as `MapIO(ma, io => io.Post())`. The captured `SynchronizationContext` must already live in the environment when the carrier runs; `mapIO` cannot inject it after the fact because the context read happens at the wrapped `IO`'s evaluation, downstream of where the carrier's environment was sealed. The constraint is structural: any `M` reachable from `IO` carries the same host-thread affinity once `EnvIO` holds the context, with no per-`M` marshaling code.
- `toIO` projects the inner `IO` out of the carrier as a value (`K<M, IO<A>>`), which is the move that lets a seam fork its host-event delivery onto a dedicated thread and re-bind the awaited result back into the same `M` ‚Äî `Fork`/`Await` operate on the extracted `IO`, and the surrounding `M`-shaped pipeline never learns that a thread boundary was crossed.

[ONE_TRANSFORMER_FOR_THE_WHOLE_SEAM]:
- Attach, detach, marshal, retry, and fault-fence are not five wiring sites; they are one effect transformer composed by arrow juxtaposition. The bracket arrow owns acquire-project-release as a unit, the catch arrow owns the typed fault rail, the finally arrow owns idempotent teardown, and the schedule-driven retry arrow owns reactivation cadence ‚Äî composing them yields a single `K<M, A> -> K<M, A>` function that any concern threads through, replacing the enumerated detach-then-dispose sequencing the unstructured form repeats at every seam.
- Ordered teardown is encoded in the bracket's `Fin` arm, not inferred from scope-collection iteration order, which is hash-trie order and therefore undefined. The `Fin` arm runs a sequential `IO` chain ‚Äî detach the subscription token first, then release the handle ‚Äî so the order is a property of the composed effect, not of the disposal walk. A racing callback can never touch a handle mid-teardown because the detach effect sequences before the dispose effect inside one bracket finalizer.
- The fault-fence is the `Catch` arrow keyed on a predicate over the error rail, placed once around the marshaled body. Host invoke primitives swallow exceptions thrown inside a posted delegate; the marshal transform's internal capture-and-rethrow restores the throw at the awaiting fiber, and the surrounding `Catch` then routes it to the seam's typed failure. Marshaled work that is not threaded through this composed transformer has no ambient propagation path and silently drops faults ‚Äî the rule is that no marshaled effect leaves the capsule except through the composed transformer.
- Reactivation builds a fresh transformer instance rather than re-running attach against a retained one. The bracket's child resource scope is constructed per activation; the subscription set is the scope, so deactivation that releases the scope destroys the set atomically, and the next activation's bracket constructs a new scope. There is no append-to-retained path, which is the structural cure for cross-cycle subscription accumulation.

[SCHEDULE_AS_REACTIVATION_POLICY_ALGEBRA]:
- Retry and repeat cadence is a `Schedule` value, not enumerated attempt logic. `Schedule` is a lazy `Iterable<Duration>` stream with a closed algebra: `|` is union (the composed schedule runs until the later of the two completes ‚Äî the longer envelope), `&` is intersect (until the earlier completes ‚Äî the shorter envelope), `+` is sequential append. Base rows are `exponential(seed, factor)`, `fibonacci(seed)`, `spaced(space)`, `fixedInterval(interval)`; `ScheduleTransformer` rows ‚Äî `recurs(n)`, `maxDelay(cap)`, `jitter(factor)` ‚Äî compose into a base via the same `|` and `&` operators, so a full reactivation policy is one expression and never an attempt counter plus a delay formula.

```csharp
var reactivation = Schedule.exponential(20 * ms) | Schedule.recurs(6) | Schedule.maxDelay(2 * sec) | Schedule.jitter(0.25);
K<M, A> Activate<M>(K<M, A> attach) where M : MonadUnliftIO<M> =>
    MonadUnliftIO.mapIO(io => io.Bracket().Retry(reactivation), attach);
```

- The transformer arity is load-bearing: `recurs(int)` returns a `ScheduleTransformer`, while `exponential`/`fibonacci`/`spaced` return a `Schedule`. The transformer is a `Schedule -> Schedule` arrow, so `exponential(seed) | recurs(6)` reads as "the exponential delay sequence, capped to six iterations," and `| maxDelay(cap)` reads as "with each delay clamped." The policy is therefore a derived value built from one primary cadence and secondary clamps, never a flag set whose combinations the retry loop re-derives.
- A union of a host-availability probe schedule with a bounded retry schedule expresses "keep probing on the host's recovery cadence, but give up after the bounded envelope" as a single `Schedule` ‚Äî the longer-of-two semantics of `|` is the precise tool when one stream is open-ended polling and the other is a hard ceiling. Intersect (`&`) is the tool when the seam must honor the stricter of two cadences, such as a frame-rate ceiling intersected with a back-off floor.
- `Schedule.Run()` is the only place the duration stream materializes; it is consumed by the carrier's `Repeat`/`Retry` arrow, which threads accumulator state across iterations without re-entering attach. Because the schedule is a pure value, the same reactivation policy is reused across every seam instance and is unit-testable in isolation by enumerating `Run()` ‚Äî no live host is required to prove the cadence.

[ENVIRONMENT_OWNERSHIP_AS_THE_REAL_DISCRIMINANT]:
- The four local-scope constructors are an ownership lattice keyed on two `Own` bits ‚Äî bit 1 (caller allocated the cancellation source) and bit 2 (caller allocated the resources) ‚Äî not four interchangeable scoping helpers. `Local` sets both bits (full isolation, both disposed on release); `LocalResources` sets only bit 2 (fresh resources, inherited source and context); `LocalCancel` sets only bit 1 (fresh cancellation, inherited resources and context); `LocalSyncContext` sets neither (re-reads ambient context, parent source and resources survive). The carrier transform selects the constructor by which lifetime the seam must own, and the bits make the choice mechanical rather than a judgment call.
- A subscription that depends on a parent-allocated handle must be detachable inside a child scope without invalidating the handle ‚Äî this is exactly `LocalCancel`: bit 2 unset means `DisposeResources` skips the parent resources, so the parent handle outlives the child cancellation scope and the detach effect can run against it safely. `LocalResources` is the inverse choice, for a subscription set that must be torn down on child completion while the parent cancellation token continues to gate the outer loop.
- `local<A>(K<IO, A>)` and `uninterruptible<A>(IO<A>)` are the two Prelude entrypoints the seam transformer composes for scope and finalization-safety. `uninterruptible` constructs an environment with `CancellationToken.None` and a fresh source ‚Äî the body's cancellation-observing primitives receive the non-cancellable token and resources are shared with the parent, so a teardown sequence wrapped in `uninterruptible` over a bracket completes its detach-and-dispose even under parent cancellation. The composition `uninterruptible(bracket(detachThenDispose))` is the canonical finalization shape; the finalizer itself must terminate in finite time because nothing can interrupt it.
- The context-bearing reads `IO.syncContext`, `IO.token`, and `IO.source` are effects evaluated at the composition root on the correct host thread and injected into `EnvIO` once. The constructors `Local` and `LocalSyncContext` re-read `SynchronizationContext.Current` at access time and are property getters, so invoking them on a background thread captures the wrong context ‚Äî the seam transformer must seal the environment on the host thread before any fork, and the carrier then propagates the sealed context to every layer via `mapIO` without re-reading.

[LIFTING_THE_HOST_EVENT_INTO_THE_CARRIER]:
- A host event that fires on a dedicated thread enters the carrier through `liftVAsync`/`liftAsync` over a reusable zero-allocation awaiter, not through a per-event task allocation. The awaiter's reset-before-wait discipline and version token are owned inside the lifted effect; the carrier sees only `K<M, bool>` and never the underlying value-task-source mechanics. `mapIO(io => io.Post())` then marshals that lifted event back onto the host thread when the consumer must run there, and `toIO` extracts it for forked off-thread consumption ‚Äî the same lifted event, two carrier-level destinations, zero rewrites.
- The marshal arrow is viable only off the high-frequency path. Each `Post()` allocates a wait handle and performs multiple kernel transitions, so per-frame or per-event marshaling collapses throughput; the carrier instead writes the lifted event into a channel state cell and marshals one batch drain per tick. The transformer that performs this is `mapIO(io => io.Repeat(tickSchedule))` over a body that drains the channel ‚Äî one composed value owns both the cadence (the schedule) and the marshal point (the drain), replacing a hand-written tick loop with inline post-per-item.
- The structural guard against host-thread deadlock is that the seam crosses to the host only from a fiber that does not itself hold the host thread. The marshal arrow uses fire-and-queue posting with a wait-handle rendezvous; a synchronous send from a fiber already occupying the host context is a self-deadlock the type system cannot see. The capsule encodes the guard by sealing the host context at the root and forking blocking work off it before any marshal, so the marshaling fiber is always distinct from the host fiber by construction.
