# Declaration-site collapse

[LEVERS_ARE_POSITIONS]:
- Controlling rule: each literal-preservation lever is one law — keep the inferred type at its narrowest — instantiated once per syntactic position against the single widening default that position faces, never an independent fuser; "which levers does this declaration need" reduces to "which positions here face a real pressure". The widening alphabet is fixed, one per position: a value literal widens to its primitive, an array literal to `Array<element>` losing tuple shape, an object literal to a structural record losing per-property literals and `readonly`, an inferred type parameter to its constraint, an annotated binding `: T` to exactly `T`.
- Compose: levers on disjoint positions of one declaration reorder freely; levers touching one position are ordered narrow→check→withdraw — a NARROWING lever (`as const`, `<const T>`, rest capture) writes a narrow type, a CHECKING lever (`satisfies`) consumes it to prove conformance, a WITHDRAWING lever (`NoInfer`) consumes the inference graph to elect a source. Commutativity is a position-disjointness test, not a per-operator fact.
- Reject: a lever pinning a position no pressure threatens (precautionary-operator), and two levers pinning the same position (redundant) — stacking `as const` onto an argument already bound to a `<const T>` parameter buys nothing the parameter inferred. A position whose narrowness is inherited from an upstream pin (`Tags<D, R>`, a `pickLiteral` sub-union, a `keyof` over an `as const` table) carries NO lever; its absence in the verified signature is the proof the narrowness was inherited.
- Boundary: every lever is fully erased and shares ONE reach edge — the source seam, the program point where a value is authored — and stops there uniformly. A `: T` annotation feeding any lever defeats it (widens before the lever runs); narrowness is the inferred type's, never the value's, so a deep-`readonly` type sits over a structurally mutable object and survives no re-binding through a widened alias. A vocabulary that must reject `unknown` or mutation pairs the contract with a runtime owner past the seam; decode owns that seam, no lever's narrowness survives it.
- Advanced: the lattice is the lever-reach ledger and the collision ledger fused on one axis — position keyed against pressure, listed in pipe order so a declaration reads top-to-bottom and a lever at row N consumes narrowness produced below it; a multi-lever declaration is correct iff each lever's row matches a real pressure at its position and no two share a row. A widened head propagates a widened union the whole pipeline length, so a single missing lever at position zero is a failure surfaced only at the terminal clause, far from its cause.

```typescript
const grades = {
    bronze: { rate: 1, label: 'entry' },
    gold: { rate: 3, label: 'apex' },
} as const satisfies Record<string, { readonly rate: number; readonly label: 'entry' | 'apex' }>
const weight = <const G extends keyof typeof grades>(g: G): (typeof grades)[G]['rate'] => grades[g].rate
```

[SATISFIES_TOTALITY]:
- Controlling rule: `satisfies T` is the only clause proving THREE properties of a literal at once — COVERAGE (every required member of `T` present, a missing one a compile error), EXACT-TYPING (each member at its own narrow type), EXACTNESS (no member beyond `T`) — where `: T` proves coverage and exactness only by WIDENING to exactly `T` and `as const` alone proves narrowness with NO coverage. `as const satisfies Record<K, Row>` carries the chain-collapse AND the totality guarantee; the composition is non-commutative — `as const` deep-freezes to literals first, then `satisfies` checks the frozen type against the contract.
- Reject: `: Record<K, Row>` on a literal-bearing table (widen-then-lose, non-recoverable: once `cap: 1` erased to `number` no later read restores it); a finite-key table asserted `as const` with no `satisfies` (silent incompleteness, discovered only when dispatch indexes the absent row); a `?? fallback`/`undefined` guard on an indexed read into a proven-total table (dead-totality-check). A genuinely sparse table declares `Partial<Record<K, Row>>` — indexed access yields `Row | undefined`, forcing the absent row into a compile-visible arm rather than a silent gap.
- Boundary: the coverage proof is a closed fact over the AUTHORED key set and erases, carrying nothing about whether a key materializing at runtime is a member of `K`. The arriving-membership half is a literal-union codec (`Schema.keyof(Owner)`, `Schema.Literal(...members)`) admitting a foreign key from `unknown`; coverage at the source plus membership at the boundary compose into total indexed access by construction — neither half alone discharges the missing-row branch.
- Advanced: coverage and exactness fail with two distinct diagnostics at two positions (`Property '<k>' is missing` versus `Object literal may only specify known properties`), so over-cover and under-cover repairs are unambiguous. A tagged matcher carries its own generated coverage proof as a handler-record parameter type; a trailing `satisfies` there is redundant, but the hand-authored lever proves what the generated fence cannot — that every covered row KEEPS its literal value, the conjunction a behavior-column dispatch reading data rows depends on and a fold over handler functions does not.

```typescript
const codes = { ok: 200, gone: 410, teapot: 418 } as const
const explain = {
    ok: '<accepted>',
    gone: '<vanished>',
    teapot: '<refused>',
} as const satisfies Record<keyof typeof codes, string>
const render = <const K extends keyof typeof codes>(k: K): readonly [number, string] => [codes[k], explain[k]]
```

[CONST_PARAMETER_PULL]:
- Controlling rule: `<const T>` migrates the literal-preservation assertion from the call expression into the signature, so the assertion is paid once at declaration and amortizes across N call sites — an entrypoint with `K` consumers deletes `K` outward `as const` assertions, the lever's value growing with fan-in rather than per call. The pull is recursive to leaf literals and is the precondition for the entrypoint's own return-type derivation: a result branching on `[T] extends [ReadonlyArray<...>]`, indexing `keyof T`, or reading `T[K]['field']` resolves against the widened constraint the moment `const` drops.
- Accept: the modifier seeds multiple slots of the returned carrier — `Match.value<const I>(i): Matcher<I, Without<never>, I, never, I>` threads the union into the `Input`, `RemainingApplied` remainder, and `Provided` slots at once, so one inferred union fixes what the matcher accepts, what it has left to consume, and what value it folds; the remainder seeded narrow is the precondition for the terminal `never`-proof reaching zero. `<const T extends C>` stacks two orthogonal forces — `C` the shape floor, `const` the literal ceiling against widening — never substituting; a `NonEmptyReadonlyArray` constraint admits a widened array, the `const` preserves the arity any positional projection depends on.
- Reject: `<const T>` on a parameter whose result type is invariant under widening (precautionary, pins what nothing consumes); a bare constraint where the return type indexes positions (missing pull, derives against the widened constraint and collapses). The modifier's presence is positive evidence the parameter is value-sourced and inferred fresh; its absence is evidence the type is a named explicit argument or already-narrow upstream — `Match.value<const I>` carries it (union from a runtime value), `Match.type<I>` omits it structurally (`Provided = never`, no value to widen).
- Boundary: the deepest reach drives a modality-polymorphic entrypoint — `all: <const Arg extends Iterable<F<any>> | Record<string, F<any>>>` resolves the heterogeneous-tuple positional projection only because `const` kept tuple and unbounded-array apart; drop it and `keyof Arg` collapses to `number`, the projection dissolves to the homogeneous fallback, and the naive form splits into `allTuple`/`allRecord` siblings. The omission failure is silent and surfaces at the consumer's call far from the missing modifier — the defect is the signature, the symptom is the call site's lost literals.
- Advanced: `<const T>` is mandatory exactly when a downstream conditional matches on a literal VALUE, noise when it tests key PRESENCE — `partialWith<const Options extends { readonly exact: true }>` branches on the literal `true`, while `optionalWith`'s faces resolve through `Types.Has<Options, 'exact'>` (a `keyof Options` probe) and carry no `const`. The modifier composes with a literal default on the same parameter (`<const Preload extends boolean = false>`), the only spelling pinning both the omitted-argument and the passed-argument case to a literal, so a return conditional `Preload extends true ? never : E` elides a channel the call selected — collapsing a boolean-knob-plus-two-bodies into one signature.

[REST_TUPLE_AND_NOINFER]:
- Controlling rule: a rest parameter `<Literals extends ReadonlyArray<LiteralValue>>(...literals)` preserves each literal in tuple position with NO `const` modifier — each spread slot is inferred independently against the tuple element, a literal-preservation mechanism distinct from `<const T>`; a single array argument collapses to the element type and breaks the tag extraction. `Schema.Literal`, `Match.is`, `Match.tag`, `Schema.pickLiteral`, and `Schema.transformLiterals` all spell their vocabulary as spread literals.
- Compose: rest capture and `<const T>` compose without redundancy only at different nesting depths — `Schema.transformLiterals<const A extends Members<readonly [from, to]>>(...pairs)` pins the OUTER tuple via the rest position (each `[from, to]` pair in its own slot) and the INNER pair via `<const A>`, the two literals a single rest slot would still let widen; stack them at the same depth and one is redundant. The depth a lever pins, not its name, decides whether a second lever composes or collides.
- Reject: the array-plus-cast `Schema.Literal(...(['<a>', '<b>'] as const))` — the spread already carries the literals, so the surrounding array and its `as const` are dead surface; `f(['<a>', '<b>'])` widens to the element type while `f('<a>', '<b>')` preserves the tuple.
- Boundary: `NoInfer<A> = [A][A extends any ? 0 : never]` is a no-op on the type and a barrier on the inference graph — it withdraws a position from the candidate set so an upstream position is the sole inference source, deciding WHICH argument is the truth, not WHAT the truth is; it sits at the pipe's far end and belongs only on the secondary position (default, fallback, instrumentation). Placed on the shape-defining position it inverts the source of truth, leaving the type with no inference source.
- Advanced: the redundancy law is proven in the verified signature — `Match.when<R, const P extends PatternPrimitive<R>>` carries `const P` (an inline pattern object that widens without it), while `Match.tag<R, P extends Tags<'_tag', R> & string>` and `Match.discriminator(field)<R, ...>` carry NO `const` because `Tags<'_tag', R>` reads the already-narrow `_tag` literal union out of the upstream-pinned `R`, so widening is structurally impossible; an `as const` on a tag literal passed to either buys nothing the constraint enforces. `NoInfer` is the mechanism that lets one owner carry both a shape and a same-shaped default without the default leaking into the shape's inference.

[SELF_GENERIC_ENFORCER]:
- Controlling rule: the duality collapse is a contract the factory forces, not a convention — `Schema.Class<Self = never>` gates its real return behind `[Self] extends [never] ? MissingSelfGeneric<...> : Class<...>`, so omitting the self-reference resolves the base to a template-literal `string` subtype (`` `Missing \`Self\` generic - use \`class Self extends Class<Self>()({ ... })\`` ``) that no `extends` clause accepts, the error text spelling the exact corrected re-declaration. The collapse is enforced from the return position, the one place a non-inferred call-site generic is observable; this is the chain link whose ABSENCE is forbidden where the other levers' deletions are merely voluntary.
- Accept: the `[Self] extends [never]` tuple-wrap is load-bearing, not cosmetic — a naked `Self extends never` is DISTRIBUTIVE, mapping over the empty union to yield `never` outright and never reaching the diagnostic branch; the wrap suppresses distribution so `never` (the sole resolution of an un-supplied, never-inferred parameter) is compared whole and hits the true branch. The wall's teeth are syntactic: `class X extends "string-literal"` is ill-formed because a heritage clause requires a construct-signature-bearing expression a `string` lacks — unparseable, not merely type-mismatched.
- Reject: a forward-declared `type Self = { ... }` to feed the factory's own `Self` (the class IS its own type once `<Self>` threads the name; the alias cannot even reference the class's own getters, undeclared when it is read); a `suspend` added beside a `Schema.Class<Owner>` that already names itself (apparent-cycle misread — the heritage reads the erased NAME, not the TDZ binding, so the type cycle closes for free with no value-level back-edge).
- Boundary: enforcement is total against the `never` default (the failure that occurs by accident) but `[any] extends [never]` resolves to the both-branches union, so a deliberate `Factory<any>(...)` types the return `Diagnostic | Owner` and the wall partially dissolves — the collapse is enforced against omission, leaving the sanctioned `any`-escape open because no type-level guard distinguishes a mistake from an override. `Self` decides what `this`, the constructor return, and `.make`'s `InstanceType` are typed against but carries no runtime artifact, which is why the enforcer must live in the type system: a forgotten `<Self>` builds identical values, only the type is refused.
- Advanced: the enforcer is one polymorphic type whose arity reads off the factory family. Three structurally distinct templates exist across families:
    - `MissingSelfGeneric<Usage, Params>` (schema): parameterizes name AND argument prefix; `'"Tag", '` for tagged, `'"Tag", SuccessSchema, FailureSchema, '` for request — `Params` length tracks the family's curried second-call arity, an empty `= ""` signaling a field-block-only second call.
    - `MissingSelfGeneric<Params extends string = "">` (variant model): hardcodes `Class`, the tell of a one-factory family where `${Usage}` would parameterize what cannot vary.
    - parameterless (service): inlines the whole `` `...Effect.Service<Self>()...` `` because its corrected spelling is shape-distinct.
- The `[Self] extends [never]` guard sits at different curry depths, deciding where the diagnostic surfaces: the schema family wraps the INNERMOST return so the template-literal text lands on the heritage expression, the service family wraps the SECOND function so `Effect.Service<Self>()` itself types as `string` and `(key, make)` fails earlier as not-callable. Every inheritance verb re-runs a fresh `[Child] extends [never]` guard with a DOTTED `${Usage}` (`"Base.extend"`, `"Base.transformOrFail"`) locating the omission to one method call; the parent's `Self` does NOT satisfy the child's guard — it migrates into the child's `Inherited` slot threading the parent's typed surface through the edge while the child's own `Self` pins its this-type.

```typescript
import { Schema } from 'effect'
class Owner extends Schema.Class<Owner>('Owner')({
    code: Schema.String,
    span: Schema.Number.pipe(Schema.positive()),
}) {
    get widened(): Owner {
        return Owner.make({ code: this.code, span: this.span * 2 })
    }
}
class Extended extends Owner.extend<Extended>('Extended')({ weight: Schema.Number }) {}
const admit = Schema.decodeUnknown(Extended)
type ExtendedWire = Schema.Schema.Encoded<typeof Extended>
```

[NO_INSTANTIATION_OWNER]:
- Controlling rule: the class-as-both-type-and-value owner satisfies `Class<...> extends Schema<Self, I, R>`, so the class value IS a `Schema` and the constructor is the ONE extra surface layered atop it — every producer keyed off `Schema<A, I, R>` (`decodeUnknown`, `decode`, `validate`, `encode`, `is`, `asserts`, `typeSchema`) consumes the class through its `Schema` face with no `new` in reach. A boundary-fed concept is referenced by exactly two spellings — `Schema.Schema.Type<typeof Owner>` (erased decoded type) and `Schema.decodeUnknown(Owner)` (boundary producer) — and a `new Owner(...)` after either is a third reference to a value the two already fixed.
- Accept: the producer surface is a lattice with two axes — the INPUT FACE (`unknown` at the wire, `I` in wire shape, `A` in memory, parts `C`) crossed with the OUTPUT CHANNEL (`Effect` rail, `Either`, `Option`, throw, `u is A` refinement). The input-face axis is the seam selector: `decodeUnknown`/`decode` cross the boundary running the `I → A` transform, `validate`/`is`/`asserts` read the Type face WITHOUT it, `new`/`make` constructs from parts that were never wire nor `A`. The output-channel axis is the requirement gate — sync and `Option`/`Either` cells constrain `R = never`, so an owner with effectful transformation fields is reachable from `unknown` only through the `Effect`-channel cell.
- Reject: `new Owner(decoded)` after a decode (the decode already minted the instance; the `new` re-runs every field refinement, two validation passes for one value, and a construction-only default can even fire differently); routing interior known parts through `validate(Owner)` (the symmetric over-reach — loses the constructor's `withConstructorDefault` injection and pays a parse traversal). The dead link does not error — it silently doubles work; its deepest cost is a `new` reached for a wire value bypassing the `R` the decode would have demanded, the silently un-discharged transformation traced only by the missing `R` in the surrounding requirement set.
- Boundary: the law reduces to one question about the value's provenance, with three terminal answers and no overlap — from-the-wire → `decodeUnknown` (the only cell that transforms), in-memory-untrusted-`A` → `validate` (re-check, no transform), interior-known-parts → `new`/`make` (the genuine construction). The link is dead per-call-site, not per-owner: a `Schema.Class` whose values sometimes arrive from the wire and sometimes are built interior carries both live producers, the same `new` live at an assembly site and dead at a site that already decoded.
- Advanced: the two apex owners forbid instantiation by OPPOSITE mechanisms. `Effect.Tag(id)<Self, Type>()` resolves to `Context.TagClass` typed `new (_: never): TagClassShape` — a `never` constructor parameter makes `new Self()` uncallable outright. `Effect.Service<Self>()(key, make)` resolves to `Class` typed `new (_: MakeService<Make>): MakeService<Make> & { readonly _tag: Key }` — INSTANTIABLE with the service shape, not `never`-walled; its non-instantiation is enforced upstream by `Service.AllowedType<Key, Make>` constraining the make-body's `Service.ProhibitedType` keys, fencing a re-exposed constructor as a field rather than barring `new` at the constructor. Both fuse type and Tag, the Service additionally a `Default` Layer, in one heritage call. `encode(Owner): (a: A) => Effect<I, ParseError, R>` proves the lattice is two-directional while the constructor is single-directional — it produces `A` from parts only, so every cell producing `I` (the wire-bound direction) has no constructor twin, and a value between a `decodeUnknown`/`encode` round-trip is the dead link on either side.

```typescript
import { Effect } from 'effect'
class Clock extends Effect.Service<Clock>()('Clock', {
    sync: () => ({ now: () => 0 }),
    accessors: true,
}) {}
const elapsed = Effect.gen(function* () {
    const clock = yield* Clock
    return clock.now()
})
const wired = Effect.provide(elapsed, Clock.Default)
```

[DECLARATION_MERGING_FUSION]:
- Controlling rule: declaration merging is the one lever that grows an owner's surface with ZERO new symbols — a `namespace X` augments the value side (statics, seeds, smart constructors) and an `interface X` widens the type side, both resolving through the single name a consumer already imports, so symbol count is invariant under surface growth. It attaches to a CLASS name because `class X extends Schema.Class<X>('X')({...}) {}` pre-occupies both namespaces; a `const X = Schema.Struct({...})` binds only the value namespace, so a following `namespace X` is a redeclaration error and `interface X` has no instance type to merge into.
- Accept: the merge is namespace-ADDITIVE, never overwriting — a namespace member sharing a name with a generated static merges then fails the static-side `extends` check, a name collision surfacing as a static incompatibility, not a silent override; the addable names are the COMPLEMENT of the factory's generated static set. The two directions split on expressibility: a getter/method (a value) goes in the class body, the `interface X` merge is reserved for type-only widening a body cannot express (a phantom-branded slot, a host-injected field) — routing a computable getter through the interface leaves it un-implemented at runtime.
- Reject: a companion `const XStatics = { seed, of }` (value-side merge written as a second symbol, the two-hop resolution the merge collapses); a mirror `interface XShape` restating instance fields (the class already IS the type); a `namespace X` placed ABOVE its `class X` line (order-inverted seed defect — the eager heritage has not built the value, so `new X(...)` reads a TDZ binding); a `type X = typeof X` minted only to re-export a class-family owner (the value `export { X }` already carries the type across the second namespace).
- Boundary: per-family free-static surfaces — `Schema.Class` reserves `make`/`fields`/`extend`/`transformOrFail`/`transformOrFailFrom`/`annotations`/`ast`/`identifier`; `Data.Class`/`Data.TaggedClass` reserve NOTHING (the cleanest merge slot, paying for no factory statics); `Effect.Service` reserves `Default`/`make`/`use`/`key` plus every `accessors: true` method name, walled at BOTH the static-merge seam and the shape-definition seam by `Service.AllowedType`. `Data.taggedEnum` has NO merge target — a value record bound by destructuring, its statics grow through the record key set — so a family later needing author-attached seeds is re-shaped to a `Schema.Union` of mergeable `Schema.TaggedClass` owners, the merge requirement retroactively forcing the family form. The union itself is NOT that merge target: `class Self extends Schema.Union(...) {}` is rejected because the call returns a `Schema` value carrying no construct signature, the same heritage-clause wall `[SELF_GENERIC_ENFORCER]` names for a non-constructor RHS — so a union's derived comparator, codec, and dispatch ride free top-level `const`s keyed off `typeof U.Type`, never statics, and the companion-namespace rescue is twice-walled (a value `namespace`/`enum` beside the union additionally fails `TS1294` under `erasableSyntaxOnly`).
- Advanced: with `accessors: true` the service collision surface is VALUE-DEPENDENT — generated statics are a mapped type over the service's method keys, so adding a method named `seed` retroactively occupies the `seed` slot, the free names shrinking as the service grows; a merged static is safest named for a concept disjoint from any method (`Live`, `Test`, `Mock` layer seeds). Under `verbatimModuleSyntax` the value `export { X }` carries both namespaces, but `export type { ... }` is FORCED the moment a name is genuinely type-only (an extracted `Schema.Schema.Type<typeof X>`, an opaque interface, a foreign type) and a foreign type is admitted through `import type { ... }` — the one `Order.Order<A>` a `SortedSet` seed requires, the minimal pair the dual-namespace identity cannot fuse.

```typescript
import { Schema } from 'effect'
class Tier extends Schema.Class<Tier>('Tier')({
    rank: Schema.Literal('low', 'mid', 'high'),
    cap: Schema.Number.pipe(Schema.positive()),
}) {
    get headroom(): number {
        return this.cap - this.rank.length
    }
}
namespace Tier {
    export const floor = new Tier({ rank: 'low', cap: 1 })
}
type TierWire = Schema.Schema.Encoded<typeof Tier>
export { Tier }
export type { TierWire }
```

[EVALUATION_TEMPORAL_SURFACE]:
- Controlling rule: the one-statement collapse is a claim in two times — the type-level fusion holds at every program point (types are timeless), the value-level fusion holds only at and forward from the owner's eager heritage line, so a collapse the checker accepts can be a TDZ throw the runtime rejects. The collapse-impossibility test asks one question of the owner's value side: at the program point the heritage expression evaluates, is every binding it reads already past its TDZ — a `yes` admits one statement, a `no` splits into a minimal pair whose second point is a deferral, never a restatement. Every literal-preservation lever is fully erased and evaluation-neutral; the temporal surface is exactly the value-bearing-owner subset (`Schema.Class`, `Data.TaggedClass`, `Effect.Service`, the `Data.taggedEnum` record).
- Accept: the universal collapse spelling `class Self extends Factory<Self>(...) {}` is the only form simultaneously hoisted, TDZ-protected, eagerly evaluated, and type-level self-referential — the heritage reads the erased NAME `Self` (no value-level back-edge) while building codec, equality, and constructor at the line. The eager-read alphabet is the alphabet of factory argument shapes: a field-record reads each field schema VALUE, a `dependencies: [Other.Default]` array reads each `Layer` element, while `sync`/`effect`/`scoped` make-body slots are deferred — so one Service declaration carries two opposite timings, a forward reference fatal in `dependencies` and harmless in the body where the requirement flows through `R` at provision.
- Reject: a `var Owner` hoist to defeat a forward reference (silent-corruption defect — `var` reads `undefined` rather than throwing, converting a loud TDZ failure into a corruption propagated into a codec/layer/effect far from cause); a `suspend` around an acyclic owner (precautionary-deferral — the eager read never throws for an acyclic owner, paying a per-traversal allocation for a free source move). The repair for an acyclic forward reference is reordering ALONE (a topological-sort violation, the referenced owner moves above its referent); the repair for a genuine cycle is the thunk and only the thunk (no source order satisfies a back-edge).
- Boundary: the deferral thunk is one polymorphic mechanism instantiated per value-bearing owner family, selected by which domain carries the cycle — `Schema.suspend` for a codec cycle (lives inside the recursive FIELD, the decode walk's entry), `Layer.suspend` for a requirement cycle (wraps the whole recursive LAYER, the build's entry), `Effect.suspend` for a computation cycle (wraps the recursive EFFECT, the run's entry). Cycle genuineness is decided per-SLOT, not per-owner: a Service's `make` body `yield* Other` is a deferred non-pair, its `dependencies: [Other.Default]` is an eager module-init read — the same logical dependency a deferred non-pair through the body and an eager ordering constraint through the array.
- Advanced: the minimal-pair count is the SUM of two orthogonal axes — the TEMPORAL axis counts genuine value-level back-edges (each a `suspend` thunk, surviving any source order), the ERASURE axis counts type-only derivations forced under their own `export type` marker (each surviving any evaluation order) — so an owner's total second-point count is `back-edge count + type-only-export count` computed from two graphs read separately, every statement beyond the sum a re-grown chain link. The TDZ collapse-impossibility test runs once per back-edge: at the point the heritage reads each binding, a still-`undefined` reference is the back-edge the temporal axis counts and the erasure axis is read independently off the `export type` markers — the two cardinalities never share a statement.

[LEVER_REACH_LEDGER]:
- `as const`: deep-`readonly` literals and tuple shape; no validation, decode, equality, or nominal identity; vocabulary owner only — the frozen-looking value is a plain mutable object.
- `satisfies`: coverage + exact-typing + exactness without widening; fully erased, literal-in-source only; coverage discharges only against a closed contract key set, never an open `Record<string, Row>`.
- `<const T>`: pulls literal/tuple inference to the call site so consumers omit `as const`; erased, inference-shaping for that one parameter only; mandatory under a value-matching conditional, noise under a key-presence probe.
- rest-tuple capture: preserves each literal in spread position with no modifier; a single array argument collapses to the element type; composes with `<const T>` only at a different nesting depth.
- `NoInfer<A>`: removes a position from the inference candidate set so a secondary value conforms without defining; no runtime check, ordering-only, never on the shape-defining position.
- `Schema.Class<Self>`: fuses type, codec, `Equal`/`Hash`, fields, constructor, `.extend` inheritance; consumes `unknown` at the boundary; `<Self>` enforced by `MissingSelfGeneric`, the constructor a sibling the `Schema` face never needs.
- `Data.TaggedClass` / `Data.taggedEnum`: tagged constructor plus structural `Equal`/`Hash`, no codec — interior identity only, no boundary admission; `Data.taggedEnum` is a destructured record with no merge target.
- `Effect.Tag` / `Effect.Service`: apex no-instantiation owners by opposite mechanisms — `Effect.Tag` resolves to `Context.TagClass` typed `new (_: never)`, genuinely uninstantiable; `Effect.Service.Class` is typed `new (_: MakeService<Make>)`, instantiable with the service shape and fenced only by `Service.AllowedType`/`ProhibitedType` on the make body, never by a `never` constructor; both fuse type and Tag, the Service additionally a `Default` Layer.
- `Schema.suspend` / `Layer.suspend` / `Effect.suspend`: one deferral mechanism per domain — codec / requirement / computation cycle; admitted only on a genuine value-level back-edge, the precautionary thunk on an acyclic owner a per-traversal allocation for a free reorder.
- declaration merging / dual export: `namespace`/`interface` merge into the value/type sides of one name; order is class-then-namespace; `verbatimModuleSyntax` forces `export type` for type-only derivations and `import type` for foreign types.
