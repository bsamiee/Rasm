# Receipts and Rails: Distilled

[RECEIPT_SHAPE_AND_ALGORITHM_SPECIFIC_EVIDENCE]:
- Every decomposition carries algorithm-specific typed evidence — `Cholesky<T>`: `Factor` + `Determinant`/`DeterminantLn`; `LU<T>`: `L`/`U`/`P` with pivot-sign-corrected `Determinant` (pivots private); `Svd<T>`: `S`/`U`/`VT`/`Rank`/`L2Norm`/`ConditionNumber`; `Evd<T>`: `EigenValues`/`EigenVectors`/`D`/`Rank`/`Determinant`/`IsSymmetric` — and no common receipt abstraction exists without losing evidence: a receipt family is concrete sub-shapes, and at ≥3 parallel sub-shapes it closes into one `[Union]` owner whose state-threaded `Switch` owns every evidence-extraction path with per-arm provenance; a receipt-projection ladder recurring across ≥3 consumers collapses to one `Func<TReceipt, TEvidence>` table keyed on the bounded status vocabulary.
- Algorithm-owned counters never collapse into generic fields: `MinimizationWithLineSearchResult.TotalLineSearchIterations` and `IterationsWithNonTrivialLineSearch` are BFGS-owned with no trust-region analog — folding them into a generic `int TotalIterations` destroys evidence the algorithm alone produces.
- Width law: past eight distinct fields the receipt splits into a required-evidence record plus `Option<TSubReceipt>` materialized only when its gate holds — `NonlinearMinimizationResult.Covariance`/`Correlation`/`StandardErrors` are all `null` when `DegreeOfFreedom < 1`, so the statistical sub-receipt gates on `DegreeOfFreedom >= 1` and embeds its own conservation check (`StandardErrors.Count == Point.Count`).
- Covariance trap pair: `EvaluateCovariance` computes `Hessian.PseudoInverse() * Value / DegreeOfFreedom` via SVD pseudoinverse, so a near-rank-deficient Jacobian yields credible-looking but inflated standard errors on degenerate directions, and the Hessian SVD rank is not exposed — recomputing `Hessian.Svd(false).Rank` post-construction is the only path to effective parameter count; `EvaluateCovariance` also re-evaluates the objective at construction, firing any side effects — the receipt seals immediately after the minimizer returns, before external state the objective reads can change.
- Fitting receipts store both `GoodnessOfFit.RSquared` (squared Pearson correlation — explicitly not R²) and `CoefficientOfDetermination` (1 − SS_res/SS_tot) under unambiguous names — they coincide only for simple linear regression with intercept — plus the `degreesOfFreedom` passed to `StandardError` (N − dof normalizer) versus `PopulationStandardError` (N, no Bessel correction).

[HONESTY_MARKERS_AS_NAMED_POLICY_FIELDS]:
- Controlling rule: every implicit algorithm choice that shaped a result is stored as a named policy field on the receipt — behavior-carrying enumerations are stored, never re-dispatched at call sites, because a consumer that switch-dispatches on them re-derives behavior and breaks re-admission; omitting the field makes the algorithm path ambiguous whenever one receipt shape serves multiple regimes.
- The marker ledger: `Symmetricity` records which case was SUPPLIED, not inferred — `Symmetric`/`Hermitian` bypass the library's symmetry check and contract that the caller verified the precondition, `Unknown` means the library checked; `QRMethod` `Thin` discards the orthogonal complement, so downstream full-column-space reconstruction must reject `Thin` receipts; `computeVectors: false` leaves `U`/`VT` as identity-shaped stubs — `bool VectorsComputed` or consumers silently read stubs; `DirectRegressionMethod` carries distinct stability guarantees (`NormalEquations` squares the condition number, `QR` stable for full-rank overdetermined, `Svd` handles rank-deficiency) that re-admission into an uncertainty quantifier must know.
- Spectral convention markers: `FourierOptions` persists as the full bitfield `int` — it encodes exponent direction and scaling as orthogonal facts whose conventions differ by exactly `sqrt(N)` or `N` — with `(SampleRate, Length, Options)` co-equal evidence beside the transform output (`data.Length` is not N for packed real transforms; no frequency axis is reconstructible without all three); even-length transforms flag the Nyquist-bin ambiguity (the edge bin is simultaneously most-positive and most-negative frequency); `UsesBluesteinLongPath` records `N > 46341`, where the phase-accumulation path switches to double-precision index multiplication and round-trip residual checks become path-sensitive.
- Statistics provenance markers: a normalizer enum `{Population, Sample}` travels with every stored deviation — `TensorPrimitives.StdDev<T>` is population-N while `DescriptiveStatistics`/`RunningStatistics` variance is Bessel-corrected N−1, and unmarked mixing silently corrupts every downstream confidence computation; `DescriptiveStatistics(increasedAccuracy: true)` switches to `decimal` internals but stores no flag on the instance — `bool UsesDecimalArithmetic` is captured at construction, the only moment the fact exists; `RunningStatistics` versus `MovingStatistics` is global-history versus window semantics on an API-compatible surface — the producing type is a recorded fact.
- Tolerance-regime and solver markers: `bool UsesDefaultTolerances` distinguishes a caller who set all three Levenberg–Marquardt tolerances to `1e-15` intentionally from one who accepted defaults — indistinguishable from the stored doubles alone; a non-unit preconditioner changes what "residual" means, so the receipt records the preconditioner type and its policy constants (`FillLevel`, `DropTolerance`, `PivotTolerance`) plus `bool PivotingEnabled` (`PivotTolerance = 0.0` means unconditional diagonal pivoting — a qualitatively different evidence state), and the fill outcome is inferred post-factorization through a separate sparse norm or count because the triangle and pivot accessors are private; a sparse direct solve records `DssMatrixStructure` (symmetric versus nonsymmetric selects the fill-reducing ordering) and `DssMatrixType` (positive-definite versus indefinite selects Cholesky versus LDLT) — and only the generic `ISparseSolverProvider<T>` overload carries `Solve`; the non-generic interface is lifecycle-only, so receipt factories operate against the generic surface.

[TOLERANCE_AS_NAMED_REPLAYABLE_POLICY]:
- The named ledger — `DoublePrecision = 2^-53`, `PositiveDoublePrecision`, runtime-measured `MachineEpsilon`, and the private `DefaultDoubleAccuracy = DoublePrecision * 10` behind parameterless `AlmostEqual` — replaces inline literals; the anti-pattern is provable when a literal equals a multiple of `DoublePrecision` for a concrete platform.
- `EpsilonOf` is the true ULP via bit-increment arithmetic, not `value * DoublePrecision`: the two diverge at binade boundaries (`EpsilonOf(2.5)` = 2^-51 versus `2.5 * DoublePrecision` = 2.78e-16), so a validity predicate built on the multiply formula errs by orders of magnitude at large magnitudes and is wrongly tight near binades.
- Two ranks, two tolerances, never interchangeable: `Svd<T>.Rank` thresholds at `S.Maximum().EpsilonOf() * max(m, n)` (the `Complex32` variant casts the dimension to `float`, widening the threshold), while `Evd<T>.Rank` uses `AlmostEqual` at `DefaultDoubleAccuracy`; receipt fields carry algorithm-qualified threshold names — a bare `int Rank` hides the tolerance policy entirely — and when domain policy differs from either hardcoded tolerance the receipt records `LibraryRank`, `DomainRank`, and the policy constant so re-admission verifies the discrepancy envelope.
- Replay ambiguity is the recurring defect: `AlmostEqualNormRelative` silently falls back to an absolute comparison when both magnitudes drop below `DoublePrecision`, and when one argument is exactly zero it compares the non-zero magnitude absolutely — three branches behind one boolean — so a receipt storing only `IsRelative: true` plus the scalar cannot determine which branch resolved a small-magnitude residual — pre-compute the residual diff and store it beside the threshold (`AlmostEqualNorm`, which all overloads delegate to, compares a pre-computed diff); `ResidualStopCriterion` scales its threshold by `‖b‖∞` each iteration, so recording it as an absolute threshold mis-replays the convergence decision, and `MinimumIterationsBelowMaximum` hysteresis is co-stored; the divergence window length (constructor-floored at 3) decides opposite verdicts for the same relative increase — a 3-step transient spike triggers where a 10-step window would not.
- Levenberg–Marquardt's three tolerances serve distinct criteria — gradient (stationarity), step (parameter change), function (residual norm) — and a receipt storing one scalar "tolerance" conflates them; the five-parameter constructor (`initialMu` plus three tolerances plus `maximumIterations`) is the minimal policy surface, all five stored with their roles.
- Derivative-step evidence: the `StepSize` setter rounds to the nearest power-of-two double, so the receipt reads the property back after assignment — the stored value is the actual h, not the requested one; `BaseStepSize = sqrt(eps)` regardless of stencil width, effective step `(1 + |x|)`-scaled for relative step types; `Evaluations` is a cumulative instance counter, so a single-evaluation receipt snapshots before/after and asserts `delta == Points`; `Center` plus `StepType` is the minimum evidence for first-order versus second-order accuracy — `Center = 0` forward differences lose the cancellation that makes centered stencils second-order.
- Custom value types implement `IPrecisionSupport<T>` (`Norm`, `NormOfDifference`) to unlock the named-tolerance comparison surface — hand-rolled `|a − b| < tol` bypasses the ledger; a squared-norm comparison (dot of the residual with itself against a squared threshold) keeps exact ring arithmetic where `sqrt` would round.

[TERMINAL_STATUS_PARTITION]:
- `IterationStatus` is six cases and the minimum receipt partition is `Converged | StoppedWithoutConvergence | (Diverged | Failure | Cancelled)`: budget-exhausted is diagnostic-but-unusable, not failure — mapping it to `Fin.Fail` destroys the caller's ability to retry with a relaxed criterion or different preconditioner, and coarser partitions hide pathological conditioning behind an innocuous timeout.
- `Iterator<T>.DetermineStatus` returns the FIRST non-`Continue` status in registration order, not the most severe; the default criteria list orders failure, divergence, iteration-count, then residual (`1e-12`), so a solver simultaneously triggering failure and convergence classifies as `Failure` — the receipt captures the full stop-criterion list in insertion order so re-admission can replay the priority logic and prove the terminal status was not an order-dependent artifact; the `Iterator<T>` overload is the receipt factory's entry because it exposes the returned status and `iterator.Status` as co-equal evidence.
- `DelegateStopCriterion<T>` receives `(iteration, solution, source, residual)` per step, so a receipt embeds its snapshot policy directly — every k-th residual norm into a preallocated trail with no heap pressure — and the trail length against total step count becomes a conservation invariant.
- `ExitCondition` carries a load-bearing sub-family split — success-adjacent (`WeakWolfeCriteria`, `StrongWolfeCriteria`, `RelativeGradient`, `AbsoluteGradient`) versus stagnation (`ExceedIterations`, `LackOfProgress`) — and a recorded sub-family marker is more discriminating than the raw twelve-case value; `None` is structural incompleteness the factory rejects with a typed failure, and `ManuallyStopped` is the other non-algorithmic case.

[CONSERVATION_PREDICATES_AND_DEGENERACY_COUNTERS]:
- Re-admission replays O(fields) identities against the stored payload and never re-runs the O(n³) computation; every derivable identity is derived at construction and recorded so the replay needs no recomputation — `Nullity = n − Rank` is stored beside `Rank` because no decomposition exposes it.
- Singular-value order is asserted, never assumed and never restored: `S[0] >= S[1] >= ... >= 0` at construction and again at re-admission, because `L2Norm = |S[0]|` and `ConditionNumber = |S[0]|/|S[last]|` are correct only under descending order — a predicate that treats the order as ascending corrupts every downstream rank threshold, condition number, and pseudoinverse.
- Success can be structural rather than boolean: Cholesky construction throws on non-SPD input and exposes no `IsPositiveDefinite`, so the receipt records success as existence of the `Factor` field; a domain-applied diagonal shift before factorization is degeneracy evidence recorded as a named `DiagonalShift` — log-determinant and condition readings downstream must account for it.
- LU's public surface is already sign-corrected — `Determinant` folds pivot parity internally — and parity re-verification at re-admission runs through the public `Permutation.Determinant`, not private pivot arrays; permutation-array identity across cached and recomputed factorizations asserts via `TensorPrimitives.HammingDistance == 0` — unconstrained, equality-based, no floating-point ambiguity — the canonical re-admission assertion for permutation preservation.
- Hierarchical statistics receipts assert the merge identity `combined.Count == a.Count + b.Count` at construction — violation means a dropped sample or misinitialized sub-accumulator; the pure static `Combine` is CAS-retry-safe as an atomic-swap function; weighted accumulators carry both `Variance` (Bessel-corrected reliability-weight estimator) and `PopulationVariance` (no correction) under unambiguous names plus `TotalWeight` and `EffectiveSampleSize` (`w1²/w2`) so the correction is auditable — inverting the two makes every downstream confidence interval wrong with no signal.
- Degeneracy counters are the only signal separating a clean run from one that silently degraded: clamped-eigenvalue count beside the ORIGINAL eigenvalue vector (mutating the stored vector destroys the original-versus-mollified distinction), `DegenerateElementCount + WellConditionedElementCount == TotalElementCount` in assembly receipts, fallback-path evaluation counts — each a named `int` on the owning receipt, each a replayable identity.

[FACTORY_DUALITY_AND_THE_KEYED_READMISSION_GATE]:
- The construct-side `Create` factory and the deserialize-side re-admission factory are two halves of one codec law running the SAME conservation predicates from one private validation routine; the receipt constructor is private and `Create` returns `Fin<TReceipt>`; the asymmetric form — `Create` asserting invariants while the deserializer trusts bytes — lets a tampered receipt enter through the wire that could never enter through construction.
- A single-keyed generated owner serializes as its bare key and deserializes through the same `Validate(key, formatProvider, out item)` factory that admits a live value — decode IS re-validation in one call, so a receipt field typed as a generated owner cannot deserialize into an invalid state; a hand-edited persisted rank violating its bounds fails at decode, before any cross-field predicate; the bare-primitive field is the rejected form because it deserializes unconditionally and defers validation one omittable hop.
- `AllowDefaultStructs` is the one switch that breaks the law — `default(T)` and field-by-field struct materialization both bypass the factory, admitting an unvalidated zero tolerance, step size, or singular value — while `SkipKeyMember` and `KeyMemberName` retarget the wire field without touching the round-trip-through-factory law; `ObjectFactoryAttribute(..., UseForSerialization: ...)` steers the wire path to a distinct serialization-only admission factory, and the sibling `UseWithEntityFramework`/`UseForModelBinding` flags make the same owner the column and binding codec with no parallel DTO; the threaded `IFormatProvider` pins culture as recorded evidence — thread-default culture parsing is host-nondeterministic.
- Independent conservation checks compose applicatively so one decode reports the FULL violation set — `Validation<Error, TReceipt>` accumulates across `Apply` while its `Bind` short-circuits, so monadic sequencing is reserved for checks that consume an earlier check's success value; the deserializer yields the same `Validation` carrier the in-process factory uses, converting to the terminal `Fin` rail in exactly one boundary hop — the accumulation lives entirely on the `Validation` side and the `Fin` rail never re-accumulates, with `SuccessSpan()` projecting the admitted payload's fields without re-boxing; a nullable-receipt-plus-error-list signature re-introduces the split rail the carrier collapses.
- Composite receipts nest in three tiers — the outer solve receipt owns the outcome and conservation checks, the middle factorization receipt owns factor evidence and structure/type markers, the inner iterative receipt owns the status and residual trail — and re-admission replays each tier's identities independently without re-executing any tier.

{

[NUMERIC_WIRE_SHAPE]:
- Matrix and vector evidence serializes through the storage contract, never the wrapper, and the storage CASE persists as a named discriminant: a sparse factor round-trips as `(values, columnIndices, rowPointers, rows, cols)` with the skeleton invariants `rowPointers.Length == rows + 1` and `columnIndices.Length == values.Length` checked before any numeric predicate; flattening a sparse factor to dense inflates the payload O(nnz) → O(n²) AND erases the sparsity structure that is itself evidence; a flattened value array without its index structure has discarded re-admission capability, not merely bytes.
- Spectra persist in produced order and the descending invariant is re-checked on decode, never restored — a re-sort is data corruption masquerading as normalization because every stored index-0-derived fact becomes inconsistent with the re-ordered vector.
- Status and policy enumerations persist as smart-enum keys round-tripping through the keyed factory, so an out-of-vocabulary wire value (corrupted status code, unknown method tag) is rejected at decode rather than aliased to a default — the budget-exhausted case persists as a distinct retryable key where a raw integer lets corruption alias it into failure; one keyed value reconstitutes the full behavior row including the case's algorithm-branch delegates, where a prose-string persistence sits one unrecoverable hop from the vocabulary; a `[Flags]` convention enum persists as its integer bitfield with bitwise-equality re-admission — decomposing it into booleans or name lists splits one atomic discriminant into fields that drift.
- The codec itself is a definition-time aspect selected by one attribute bitfield over the framework set (`SystemTextJson`, `NewtonsoftJson`, `MessagePack`, with `Json`, `All`, `None` as named combinations) — a binary persistence boundary selects `MessagePack`, a text/HTTP edge `SystemTextJson`, and `All` emits dead generated surface unless the receipt genuinely traverses multiple transports; the generator emits one converter per selected framework, and a hand-rolled converter beside a generated owner is the anti-pattern the bitfield deletes.

[FAILURE_RAIL_AT_THE_BOUNDARY_AND_ON_THE_WIRE]:
- Exceptions convert exactly once at the owning boundary and never propagate inward; the hierarchy has two roots — iterative non-convergence (branching into numerical breakdown) and native-provider failure (invalid-parameter, memory-allocation, singular-U) — plus a separate optimization family, and the breakdown child is caught BEFORE its non-convergence parent: a parent-first ladder silently swallows the more specific signal; two arms, two distinct typed failures.
- `Error.New(code, message)` is `Expected` (code-keyed, non-exceptional) for domain preconditions detected before the library call; `Error.New(ex)` is `Exceptional` for library internals — collapsing both hides whether the error originated in a domain predicate or library code; `Error.New(domainCode, ex.Message, Error.New(ex))` chains both so dispatch keeps the code and diagnosis keeps the exception via `Inner`; flattening in either direction discards one half.
- Independent violation sets accumulate via `Error.Many`/`operator +` in left-to-right diagnostic order — a short-circuiting factory cannot distinguish one violation from several — and re-admission isolates expected domain errors from an aggregate with `Filter<E>()` plus `HasCode`, never by pattern-matching the aggregate graph.
- Typed payload extraction is mandatory where it exists: the evaluation-failure exception's `ObjectiveFunction` property is the library's only typed post-failure evidence (point, gradient, Hessian at the failing evaluation) and is captured with its `IsGradientSupported`/`IsHessianSupported` flags — an unchecked Hessian read silently yields null; the singular-U index lives only in the message string, so the boundary preserves message plus exception identity and maps to a singular-matrix domain error distinct from generic breakdown — the zero-pivot sparse status likewise maps to the singularity case, not a blanket solver failure, because collapsing them discards the definiteness signal.
- Only `Expected` is wire-faithful: its equality is by `Code` ALONE — message is non-load-bearing display text a codec may drop or localize — so every exceptional error converts to an expected error with a domain code BEFORE serialization, the persisted failure surface is a bounded code vocabulary, `HasCode` dispatch on a round-tripped error is sound and message-substring matching is not; an aggregate serializes as its ordered constituent sequence only when every member is expected — each exceptional member is downgraded first or the wire form is partially lossy.
- Library preconditions gate as `Expected` before invocation — a packed real transform gated on its convention bitfield returns a typed domain failure on violation, where the ungated call promotes the same precondition to an `Exceptional` throw.

[SOLVE_LIFECYCLE_COMPOSITION]:
- The measured-solve lifecycle is one composed effect whose order is fixed by the algebra, read inside-out as release < attempt-policy < recovery < deadline < commit: resource bracket innermost, schedule-driven retry wrapping the bracket, the code-keyed catch table wrapping the retry (recovery runs after the budget is spent), cancellation/timeout wrapping the catch (deadline dominates recovery), and atomic receipt accumulation as a sibling node sequenced after success — catch inside retry recovers per-attempt and defeats the schedule; accumulation inside the bracket disposes the receipt with the workspace.
- Retry and repeat are the same fold over the schedule iterator with one load-bearing difference: retry wraps each attempt in `BracketFail()` (release on the failing branch only, so a succeeding solve EXPORTS its acquired handles — a live factorization can be handed to the continuation), repeat wraps each iteration in `Bracket()` (unconditional release before the next iteration sees the value — polling probes, never resource-returning solves); `PrependZero` makes the first attempt immediate so a count transformer of n produces n attempts.
- Schedule exhaustion returns only the LAST captured error; the full failure trajectory requires an external accumulator (an atomic error sequence swapped inside a catch arm) because the combinator discards intermediates; the retry predicate is an UNTIL over the converted domain error — the exact seam where a budget-exceeded code stays retryable and a singular-matrix code terminates.
- The three-argument bracket desugars to `Acq.Bind(x => Use(x).Catch(C).Finally(F(x)))`: the catch arm runs INSIDE the resource scope before release, so an indefinite-matrix recovery (diagonal-shift fallback) still sees the live workspace and the release still fires after recovery — and any resource the recovery itself acquires must be bracketed in turn or a second failure leaks it; zero-argument `Bracket()` re-roots a fresh local-resources scope with unconditional release, `BracketFail()` re-roots with failure-only release — retry/repeat select between exactly these two.
- `Timeout` is a structural interpreter node, not a sleep race — a timed-out solve releases its bracketed workspace through the normal finalizer path; `Local()` installs an independent child cancellation source (parent cancellation still cascades) for per-solve deadlines that spare batch siblings; a fork returns a `(Cancel, Await)` handle pair where cancel is itself a composable effect, parent-acquired resources are SHARED into the fork (either side may release them), and a fork under an already-cancelled ambient token throws at launch so dead work never schedules.
- Catch arms keyed on domain error codes chain left-to-right with first-match-wins, so arm ORDER is policy — most specific code before broadest predicate, mirroring the breakdown-before-non-convergence ladder — and each arm constructs its own provenance with no shared outer handler.

```csharp
static Eff<RT, TReceipt> MeasuredSolve(MatrixHandle a, VectorHandle b) =>
    liftIO(IO.lift(static () => Workspace.Rent(a.Rows))
        .Bracket(
            Use:   ws  => Decompose(a, ws).Bind(f => SolveAndProve(f, b, ws)),
            Catch: err => err.HasCode(Codes.Indefinite)
                              ? ShiftedSolve(a.WithShift(Policy.Shift), b)
                              : IO.fail<TReceipt>(err),
            Fin:   ws  => IO.lift(() => ws.Return())));
```

[SCHEDULE_POLICY_LATTICE]:
- A schedule is a value-typed duration series with a lattice algebra, so backoff is one auditable datum, never call-site delay arithmetic; between two SCHEDULES `|` (union) emits the MIN duration and runs to the LONGER, `&` (intersect) emits the MAX and runs to the SHORTER, `+` appends — but beside a TRANSFORMER the same symbols are plain application (`recurs(n) | exponential(s)` ≡ `recurs(n) & exponential(s)`), so the operand-kind pair, never the symbol, determines semantics, and a policy review that reads the operator without both operand kinds misjudges whether the policy is bounded at all.
- The unbounded defect lives entirely on the schedule-schedule side: exponential backoff derives from the infinite series, a union of two infinites never terminates, and a bound must arrive as a transformer — count cap or cumulative-delay budget — or as an intersect against a genuinely finite series.
- Transformers are first-class over schedules and compose by the same operators: per-step delay clamp, cumulative-delay halt, reset-after rewind, bounded jitter, and decorrelated jitter (each delay derives from the previous REALIZED delay, not the nominal one) — a contention-aware retry for a shared solver pool is one value: exponential base, cumulative budget, decorrelation.
- The count transformer means ATTEMPTS while the repeat transformer means replay-the-whole-schedule — three attempts replayed twice is six attempts in two bursts — and conflating them silently halves or doubles the budget; base series are derived, not enumerated (exponential is one map over the infinite series), so a custom backoff law is a single map, never a hand-built duration array.

[RECEIPT_ACCUMULATION_AND_CACHE_CELLS]:
- The atomic cell's change event fires only after a real CAS commit, and a validator-rejected swap returns the PRIOR value with no event — byte-identical to a successful identity swap — so a conservation invariant encoded as the atom validator is unlearnable from the return value; the only ambiguity-free rail is the maybe-swap with the invariant checked inside the function plus an external `Fin` on the factory result, because a silently retained receipt map is data loss with no error channel; validated atom construction returns a bare option with no failure reason, converted to a typed failure at the construction site.
- The IO-lifted swap variants make accumulation a bind-sequenced pipeline node ordered against the bracket and the retry boundary rather than out-of-band; the swap function re-invokes on every CAS retry, so it must be the pure merge — a closure over a mutable buffer corrupts under contention, and the pure statistics-merge combinator is directly usable as the swap function for moment accumulation.
- Cache admission is conditional: the maybe-add lookup suppresses insertion when the factory returns none, so a failed receipt construction is never cached — the unconditional variant permanently caches a first-failure construct and may invoke the factory multiple times under contention, so only pure, infallible factories pass it; keyed removal via the option-to-option swap is atomic within the CAS loop, so a racing reader sees the old receipt or the absent slot, never a partial write; the change event delivers a patch structure (before-map, after-map, per-key change discriminants), not per-key triples.
- Key design owns staleness: a key derived from a recyclable native handle must encode content version — content hash, change counter, or a (handle, generation) pair — because pointer reuse produces cache hits that silently return the wrong receipt; CAS safety does not extend across handle recycling, and the identity token is the cache owner's responsibility.

[AMBIENT_CONTROL_PROVENANCE]:
- One process-wide static surface holds the linear-algebra, Fourier, and sparse-solver provider slots, the degree of parallelism, two parallelization thresholds (order 64, elements 300), the task scheduler, and the RNG and parameter-validation flags; none is an argument to any call — every algorithm reads it at execution instant, so a receipt recording input, tolerances, and verdict but not this surface has captured the question and the answer while discarding the machine that produced it.
- The slots are lazily-initialized, double-check-locked, mutable statics: first touch runs default resolution — a per-family environment variable through an uppercased switch, evaluated once and cached so later mutation is inert, with unrecognized values falling through a fixed reflective probe chain where a native sibling assembly merely PRESENT on the probe path fires the swap with no opt-in, and application-context switches (disable probing, disable all native, disable one backend) taking precedence over the environment — after which any code may swap a slot silently and globally; the same matrix factored moments apart can route through two backends with no local diff, so the receipt snapshots provider identity at construction, never at re-admission.
- The try-use request's boolean return is the ONLY adoption signal — a throwing use-call and a false-returning try-call both leave the prior provider active silently — so the factory captures the boolean or re-reads the provider tag after the request; the provider's stable low-cardinality `ToString` tag (`"Managed"` for in-house; native backends report their own) is the densest provenance anchor: one field distinguishes the in-house factorization from a native one (numerically distinct algorithms producing distinct factors for one input), cheaper than storing the factor and more discriminating than storing the matrix; the tag also gates WHICH conservation predicates are replayable — in-house tolerance arithmetic is auditable against the named ledger, native internals are opaque results.
- Mutating the parallelism degree re-runs provider initialization as a side effect (the degree clamps on assignment; the single/multi-thread verbs flip degree and RNG flag together), coupling the knob to the slot — both snapshot atomically at solve construction, because a parallelism change between recording the two can re-verify and in principle re-resolve the provider under a half-built receipt.
- Parallelism is a reproducibility cliff: the multiply branches on a size-sum threshold, and above it partial sums accumulate in a runtime-partitioned tree whose topology varies run-to-run, so low-order bits differ; the Frobenius norm rides that multiply, so condition numbers, residual norms, and relative thresholds inherit the nondeterminism; the two-level partition fold changes sums, dots, and moment merges with partition count, so a recorded value is correct for one core count only and bit-comparison on a different host falsely flags tampering; merged moment identities hold only to the floating-point merge envelope, so the receipt carries the merge fan-out to make the envelope computable.
- The bit-versus-envelope verdict is derived, never enumerated: provider tag plus degree plus threshold form one primary correspondence, and a pure function of that triple decides whether two receipts admit bit equality or only span-and-envelope equality — provider-mismatched eigenvector comparison is a category error short-circuited to span equivalence (backends agree up to sign and span, and choose different bases within degenerate eigenspaces) — so no consumer re-derives the reproducibility rule from raw fields.
- The ambient static is the wrong owner for solve policy — mutable, shared, read at an instant no call controls; structural environment access threads the control snapshot through the effect runtime as a trait the solve reads via ask, so the value the receipt seals is provably the value the solve consumed; a scoped override pins a snapshot for one sub-computation and restores on exit — lexically bounded and re-entrant where two writes to the static would race — and the sealed-snapshot-equals-scoping-snapshot equality is a structural invariant; inside a retry the snapshot re-reads per attempt — lifting the read into the interpreter via `localIO` over a `MonadUnliftIO` runtime sequences it relative to the bracketed acquisition and the retry boundary — because a snapshot captured once outside the loop records first-attempt policy on a last-attempt receipt and the provenance lies whenever re-verification fired between attempts.

static K<RT, TReceipt> MeasuredSolve<RT>(MatrixHandle a, VectorHandle b)
    where RT : Has<RT, NumericControl>, Readable<RT, RT>, Monad<RT> =>
    Readable.asksM<RT, RT, TReceipt>(_ =>
        Has<RT, NumericControl>.Ask.Bind(ctrl =>
            Decompose(a, ctrl).Bind(f =>
                Prove(f, b, ctrl).Map(ev => Receipt.Seal(ev, ctrl.Snapshot)))));
