# [TS_CORE_API_RFC6902]

`rfc6902` mints the RFC 6902 JSON-Patch and RFC 6901 JSON-Pointer engine `interchange/format` composes as the `EntityEdit` apply/egress core: a closed `Operation` union on `op`, a value-returning apply engine whose faults are return slots not throws, and a Levenshtein diff engine taking one custom-diff hook. Weakly `any`-typed at its edge, it stays the pure engine beneath the `effect/Schema.Union` boundary `interchange/format` owns.

## [01]-[PACKAGE_SURFACE]

[PACKAGE_SURFACE]: `rfc6902`
- package: `rfc6902` (MIT)
- module: dual CJS/ESM, per-module `.d.ts`; subpaths `rfc6902`, `/diff`, `/patch`, `/pointer`, `/util`
- runtime: universal — zero deps, no host binding; `interchange/format` runs it on every lane
- rail: the RFC 6902/6901 engine behind the `interchange/format` `Schema.Union` boundary

[PUBLIC_TYPE_SCOPE]: the `Operation` union — the closed RFC 6902 vocabulary

| [INDEX] | [SYMBOL]                                                        | [TYPE_FAMILY]       | [CAPABILITY]                    |
| :-----: | :-------------------------------------------------------------- | :------------------ | :------------------------------ |
|  [01]   | `Operation` = `Add\|Remove\|Replace\|Move\|Copy\|TestOperation` | discriminated union | closed six-op vocabulary        |
|  [02]   | `AddOperation` / `ReplaceOperation` / `TestOperation`           | value-bearing op    | `path` + `value` carriers       |
|  [03]   | `RemoveOperation`                                               | path-only op        | sole destructive-without-value  |
|  [04]   | `MoveOperation` / `CopyOperation`                               | relocation op       | `from` + `path` relocation      |
|  [05]   | `Patch` = `Operation[]`                                         | patch document      | the `EntityEdit` egress payload |

|  [07]   | `Diff` / `VoidableDiff`                                         | diff hook contract  | the custom-diff seam            |

| [INDEX] | [SYMBOL]                                              | [TYPE_FAMILY] | [CAPABILITY]                    |
| :-----: | :---------------------------------------------------- | :------------ | :------------------------------ |
|  [01]   | `Pointer { tokens: string[] }`                        | pointer value | decoded RFC 6901 path           |

## [03]-[ENTRYPOINTS]

[ENTRYPOINT_SCOPE]: egress — the minimal patch + OCC test-guards

`createPatch` is the `EntityEdit` egress encoded to the C# `JsonPatchDocument`, and `createTests` captures a pre-image as `test` ops so the egress ships its own optimistic-concurrency proof; the `diff?` argument is the one parameterized reconciliation seam, with `diffArrays` the Levenshtein sequence differ it falls back to.

| [INDEX] | [SURFACE]                                                              | [ENTRY_FAMILY] |
| :-----: | :--------------------------------------------------------------------- | :------------- |
|  [01]   | `createPatch(input, output, diff?: VoidableDiff): Operation[]`         | diff / egress  |
|  [02]   | `createTests(input, patch: Operation[]): TestOperation[]`              | precondition   |
|  [03]   | `diffAny(input, output, ptr, diff?)` / `diffObjects` / `diffArrays<T>` | differ         |
|  [04]   | `subtract(minuend, subtrahend)` / `intersection(objects)`              | key algebra    |

[ENTRYPOINT_SCOPE]: apply — the value-rail patch application

`applyPatch` applies a whole document returning one `fault\|null` slot per op; the five mutators share `(object, op, options?) => MissingError\|null`. rfc6902 mutates in place, so `interchange/format` clones before applying and folds the slots into the `Effect` channel.

| [INDEX] | [SURFACE]                                              | [ENTRY_FAMILY] | [CONSUMER_BOUNDARY]                                        |
| :-----: | :----------------------------------------------------- | :------------- | :--------------------------------------------------------- |
|  [01]   | `applyPatch(object, patch, options?): (fault\|null)[]` | apply-all      | one slot per op; a non-`null` slot folds to a tagged fault |
|  [02]   | `apply(object, operation, options?): fault\|null`      | apply-one      | the `op`-switch applier the single-op `Match` mirrors      |
|  [03]   | `add`/`remove`/`replace`/`move`/`copy`                 | per-op mutate  | five mutating appliers; `move`/`copy` resolve two pointers |
|  [04]   | `test(object, op, options?): TestError\|null`          | precondition   | non-mutating equality guard; applier form of the OCC check |

[ENTRYPOINT_SCOPE]: `util` — structural helpers

Value helpers the apply/diff cores share; `clone` is the pre-apply defensive-copy site.

| [INDEX] | [SURFACE]                             | [ENTRY_FAMILY] | [CONSUMER_BOUNDARY]                                          |
| :-----: | :------------------------------------ | :------------- | :----------------------------------------------------------- |
|  [01]   | `clone<T>(source: T): T`              | deep copy      | pre-apply immutability guard before a mutating `applyPatch`  |
|  [02]   | `objectType(v)` / `hasOwnProperty(v)` | reflection     | the array-vs-object-vs-primitive dispatch the differ keys on |

## [04]-[IMPLEMENTATION_LAW]

[TOPOLOGY]:
- rfc6902 is the pure engine and `effect/Schema` the boundary: the `Operation` union and `Options` are `any`-typed at the edge, so `interchange/format` owns a `Schema.Union` of six `Schema.TaggedStruct`s keyed on `op`, decodes the C# `JsonPatchDocument` once, and downstream code never touches a raw operation.
- Faults are values folded to a rail: `applyPatch` returns `(fault\|null)[]`, and `interchange/format` maps `MissingError`/`TestError`/`InvalidOperationError` onto a `Data.TaggedError` family (`PatchMissing`/`PatchTestFailed`/`PatchInvalid`), lifting the first non-`null` slot into the `Effect` error channel with `TestError`'s `{ actual, expected }` as the OCC drift report.
- Diff is parameterized: `createPatch(base, next, diff?)` takes one `VoidableDiff` hook owning every domain reconciliation, returning `void` to fall back to `diffAny`; a new entity family is a hook arm, and the array/object/primitive split already lives inside `diffAny`.
- Mutation is fenced by a clone: `interchange/format` clones the target before `applyPatch`, so the decoded value stays immutable and apply yields a new value on the rail.

[STACKING]:
- `effect` (`.api/effect.md`): the `Operation` union becomes a `Schema.Union` dispatched by `Match.discriminatorsExhaustive("op")`, `Data.TaggedError` carries the three faults, `Schema.encode` re-serializes `createPatch` output to the C# `JsonPatchDocument`, and `Effect.map`/`Effect.fromNullable` fold the result slots.
- `interchange/codec` peers + `interchange/contract` (`core/.api/`): `interchange/format` owns the `JsonPatchDocument` egress arm of the multi-codec plane, and a `codec` page selects one codec by the C# mint format. An `InvalidOperationError` (op outside the closed six) surfaces through `interchange/contract` as a typed drift verdict, not a decode crash, and `Pointer` reuses the RFC 6901 path notion the graph/version codecs address.
- `core/state/causal` + `data/journal/append`: the `createTests` pre-image `test` ops are the optimistic-concurrency guard the version-vector append verifies before applying, so the `EntityEdit` egress self-guards across the C# `Rasm.Persistence/Version` seam.
- `core/value/contentKey`: a patch document is content-addressable through the one `XxHash128` seed-zero mint over the encoded `Operation[]`; rfc6902 never mints its own identity.

[LOCAL_ADMISSION]:
- Decode inbound patches through the `Schema.Union` mirror of `Operation`, and clone before a mutating apply.
- Emit `createTests` guards on any egress the C# side answers with an OCC append.
- Supply one `VoidableDiff` hook for domain reconciliation, folding the `applyPatch` slots through the `Data.TaggedError` map.

[RAIL_LAW]:
- Package: `rfc6902`
- Owns: the `Operation` union + `Patch`, the value-returning apply engine (`applyPatch` + per-op appliers + the three fault classes), the structural diff engine (`createPatch`/`createTests`/`diffAny`/`diffArrays`/`diffObjects` + key-set algebra), and the RFC 6901 `Pointer`
- Accept: rfc6902 as the pure engine behind a `Schema.Union` boundary, `createPatch` for `EntityEdit` egress, `createTests` for OCC proofs, the `VoidableDiff` hook for domain reconciliation, result-slot folding onto a `Data.TaggedError` rail, `clone` before a mutating apply
- Reject: a raw `applyPatch`/`createPatch` on undecoded `any`, thrown or imperatively-inspected faults, a per-shape differ roster where one `VoidableDiff` hook fits, in-place mutation of a decoded value, a hand-rolled patch applier or JSON Pointer traversal, a second content-address for a patch document
