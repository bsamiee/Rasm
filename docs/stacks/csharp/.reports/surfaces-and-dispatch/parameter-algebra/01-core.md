# Parameter Algebra

[POLICY_VALUES]:
- A smart-enum row is the canonical policy value form: the row carries constructor-injected behavior delegates alongside its key, and the generated `Switch<TState, TResult>(TState state, Func<TState, CaseN, TResult> onCaseN, ...)` threads state through each case without the caller re-deriving which case was selected. `SwitchMapMethodsGeneration.DefaultWithPartialOverloads` generates an additional overload accepting an `otherwise: Func<TState, TResult>` fallback for unhandled cases, delivering a default without spreading it across call sites.
- `[SmartEnum<TKey>]` generates a `Key` property typed `TKey` and implements `IParsable<T>` and `ISpanParsable<T>` when `TKey` supports them; the parsed key is the row's identity, not a string literal requiring a second match pass. `SkipIParsable = true` suppresses both interfaces together because `ISpanParsable<T>` inherits from `IParsable<T>` — suppressing one forces the other.
- Keyless `[SmartEnum]` generates the same `Switch`/`Map` shape with `SwitchMapStateParameterName` defaulting to `"state"`; the name is overridable on the attribute so generated parameter names remain consistent with codebase vocabulary without touching generated code. `EqualityComparisonOperators` on the keyless form produces no key-overload variant because there is no key member.
- Stateless union members in ad-hoc or regular `[Union]` forms store only the discriminator index with no payload; generated accessors return `default(T)` for stateless cases. Using a struct for the stateless marker type makes the accessor value structurally non-null. `T1IsStateless = true` implies `T1IsNullableReferenceType = true` automatically for reference-type members.
- `NestedUnionParameterNameGeneration.Simple` on a `[Union]` changes generated `Switch` parameter names from the fully-qualified `failureNotFound` form to the leaf-only `notFound` form. When two nested families share a leaf name, the generated code fails to compile; `Default` is the safe choice whenever collision is possible.
- `ConversionOperatorsGeneration.Implicit` (default for union-to-value) lets a policy case convert implicitly to the union type at call sites; `Explicit` forces a cast. Setting `ConstructorAccessModifier = Private` suppresses both the constructor and the implicit conversion operator together, requiring factory methods — the correct shape when the factory carries admission logic beyond naked construction.

[OPTIONAL_CONTEXT]:
- `Option<T>` is the single admitted form for optional parameters carrying absence semantics. `Option<A>` in LE5 implements `K<Option, A>` and exposes `IsSome`, `IsNone`, and a `Case` property returning `object?` that enables `{ IsSome: true, Case: T value }` property-pattern destructuring without triggering the `Match`-in-expression analyzer prohibition.
- `Option<A>` carries implicit conversions from `A?` (reference type: null maps to `None`) and from `Fail<Unit>` or `Unit` (both map to `None`), making `None` the Prelude constant for both the absence sentinel and the option initializer. The `|` operator on two `Option<A>` values returns the left if it is `Some`, the right otherwise — a direct fallback chain without a `Match` arm.
- `Option<T>` as an optional parameter default requires `= default` at the declaration and `IfNone(canonical)` at use; `Option<TVo>` where `TVo` is a `[ValueObject]` or `[SmartEnum]` avoids the CS1736 constraint that forbids a non-const default. The canonical fallback comes from the policy owner — the smart-enum static property or value-object factory — so the default is derived, not re-stated at the call site.
- A runtime context parameter that is optional but carries identity enters as one record, not as a flag tail. A nullable record parameter `T? ctx = null` is the boundary-only form; inside the domain it projects to `Option<T>` at the admission site so interior code never branches on null.

[POLICY_TABLE_FORM]:
- `FrozenDictionary.Create<TKey, TValue>(ReadOnlySpan<KeyValuePair<TKey, TValue>> source)` and `Create<TKey, TValue>(IEqualityComparer<TKey>?, ReadOnlySpan<...>)` are the .NET 10 span-based construction overloads. A collection expression at the call site (`[new("k", v), ...]`) binds to the `ReadOnlySpan<T>` parameter with no heap allocation for the intermediate collection.
- `FrozenDictionary<TKey, TValue>` exposes a `GetValueRefOrNullRef(TKey key)` method returning `ref readonly TValue`; callers test for absence via `Unsafe.IsNullRef(in result)` without boxing the value. `AlternateLookup<TAlternateKey>` (where `TAlternateKey : allows ref struct`) enables `ReadOnlySpan<char>` keying against a string-keyed table with no string allocation; `TryGetAlternateLookup` returns `false` when the comparer does not implement `IAlternateEqualityComparer<TAlternateKey, TKey>`.
- A frozen policy table is the `POLICY_VALUES` implementation for bounded vocabularies with non-delegate behavior: the smart-enum row owns behavior via delegate fields; the frozen table owns behavior via value structs or delegates stored as map values. The table is a single static field; a lookup returns the complete policy value and the caller never reconstructs behavior from key combinations.
- `FrozenDictionary<TKey, TValue>.Keys` returns `ImmutableArray<TKey>`, not `IEnumerable<TKey>`. Iterating via `foreach` over the `ImmutableArray<TKey>` never boxes and never allocates; the explicit-interface `IDictionary<TKey, TValue>.Keys` allocates a wrapper. When enumerating policy-table keys for validation or bulk projection, `Keys` (not the `IDictionary` interface) is the zero-allocation path. The same applies to `FrozenSet<T>` via `Items`.

[ARITY_COLLAPSE]:
- `params ReadOnlySpan<T>` (C# 13) is the zero-allocation arity collapse form. The compiler emits an inline array or stack-allocated span at the call site — no `IEnumerable<T>` heap object. When a `params T[]` overload coexists with a `params ReadOnlySpan<T>` overload, `[OverloadResolutionPriority(1)]` on the span overload directs the compiler to prefer it, making the array overload redundant in a greenfield context.
- Collection expressions (`[a, b, c]`) bind to any `params` target type at the call site without specifying the collection kind; the compiler selects the best collection builder. A `params ReadOnlySpan<T>` parameter with a collection expression call site produces no heap allocation when the count is small enough for an inline array.
- A `params` parameter that re-describes what the receiver already discriminates is the prohibited form. If the entrypoint dispatches on the case of a union input, adding a parallel `params` array carrying case-correlated overrides is arity smuggling: the same case selection is expressed twice, and the discriminant must be recoverable from the input value itself.
- `Argument<T>` is a `readonly ref struct` carrying `Value: T` and `IsSet: bool`; the implicit conversion from `T` sets `IsSet = true` and a default `Argument<T>` has `IsSet = false`. This enables optional parameter algebra at factory sites where the absence of a value must be distinguished from a provided value that happens to equal the domain default — the custom factory's `ValidateFactoryArguments` partial receives `ref Argument<T>` for each field so validation can inspect whether the caller explicitly supplied it.

[DEFAULTS_AND_DERIVATION]:
- Generated value-object factory defaults derive from the owning type, not from the call site. `AllowDefaultStructs = false` (default) prohibits `default(T)` and parameterless construction of struct value objects; the generated `Empty` static property is the only named sentinel for the struct-zero case. `DefaultInstancePropertyName` renames it. Callers using `Option<TVo> = default` receive `None` at the call site while the domain default lives on the type.
- `SkipFactoryMethods = true` suppresses `Create`, `Validate`, `TryCreate`, the `TypeConverter`, both `IObjectFactory<T>` interfaces, and the implicit conversion from key type to value object, and forces `SkipIParsable = true`. Setting this requires a manually authored `[ObjectFactoryAttribute<T>]`-marked factory, which is the correct shape when admission logic spans multiple sources or requires `Fin<T>` rail integration.
- `[ObjectFactoryAttribute<T>]` (allow-multiple) registers a custom factory keyed on `T`; `UseForSerialization` directs the specified serialization frameworks to it. `UseWithEntityFramework = true` routes EF Core value converters through the factory. Multiple attributes on one owner register independent conversion paths for different source types with no shared global policy.
- `IValidationError<T>` is a `static abstract` interface member: `static abstract T Create(string message)`. A custom error type implements this interface; the generated factory invokes `TCustom.Create(message)` at validation failure time. The policy of which error type to produce lives on the domain type declaration, not at every factory call site.

[COMPOSITION_TIME_POLICY]:
- `Schedule` is a composable policy value: `|` takes the union (minimum duration, length of the longer stream); `&` takes the intersection (maximum duration, length of the shorter stream); `+` concatenates two schedules sequentially. A schedule is composed once into a `static readonly` field and passed as a single `Schedule` argument to `retry` or `repeat` — no parameters inside the retry loop re-specify the backoff.
- `Schedule.recurs(n)` is a `ScheduleTransformer` that limits iteration count. `Schedule.NoDelayOnFirst` and `Schedule.RepeatForever` are static `ScheduleTransformer` fields, not `Schedule` values. Storing a bare transformer in a `static readonly Schedule` field triggers the implicit conversion `ScheduleTransformer → Schedule`, which uses `Schedule.Forever` as the base — the infinite base is invisible at the declaration site.
- `Schedule.jitter(factor, seed)` and `Schedule.jitter(minRandom, maxRandom, seed)` are `ScheduleTransformer`s accepting `Option<int>` for the seed, where `default(Option<int>)` means non-deterministic. The seeded-vs-random distinction travels as presence or absence in the option, not as a boolean knob.
- `CatchM<E, M, A>` is the named catch policy value: `Match: Func<E, bool>` selects the error; `Action: Func<E, K<M, A>>` provides recovery. The `|` operator on `K<F, A>` accepts a `CatchM` on the right, composing recovery policy as a linear chain without nesting. Named catches produced by `@catch(error, handler)` or `@catch(predicate, handler)` can be extracted to `static readonly` fields and applied across any `K<Fallible<E,M>, A>` computation.
- `bracketIO<M, A, B, C>(Acq, Use, Catch, Fin)` admits a `Catch: Func<Error, IO<C>>` policy alongside `Use` and `Fin`. The three function parameters together form the resource policy: acquire, use, catch errors from the acquisition, and finalize. The two-parameter `bracketIO(Use, Fin)` omits the catch, so the recovery policy is a named optional rather than a flag.

[EXTENSION_BLOCKS]:
- C# 14 extension blocks (`extension(ReceiverType r) { ... }`) are a receiver-owned surface addition distinct from classic `this`-parameter extension methods. The block declares the receiver type once; all members inside share the receiver without repeating `this T` on each. Properties, indexers, and operators are admissible in extension blocks — not just methods — so a full receiver-owned surface including computed properties over a policy type can be expressed without modifying the original type.
- Extension blocks intersect with parameter algebra at the policy surface: a frozen table or smart enum can acquire computed access members — projections that derive secondary policy values from the stored primary — without polluting the owner type with unrelated helpers. The block keeps those projections co-located with the vocabulary that governs them.

[ANTI_PATTERNS]:
- A boolean parameter selecting between two bodies is a policy inlined as a knob. The collapsed form uses a smart-enum row or union case: the case carries its own body via a constructor-injected delegate; the caller passes the pre-constructed case, and no `if` or `switch` re-derives the body at the dispatch site.
- A nullable flag tail — `T? optA = null, T? optB = null, bool doX = false` — fragments one optional context into multiple parameters. The collapsed form is one `Option<ContextRecord>` carrying all optional fields; absence of the record means "use defaults"; presence carries the override bundle. `Argument<T>` is the alternative when the factory must distinguish "caller provided this specific value" from "caller omitted, use domain default."
- A `params T[]` overload alongside a semantically identical `params ReadOnlySpan<T>` overload without `[OverloadResolutionPriority(1)]` on the span form means the compiler prefers the array overload for collection expressions, producing a heap allocation. The span overload must carry priority `1`.
- Re-stating a default at every call site — `op.Exec(schedule: Schedule.recurs(3) | Schedule.exponential(10 * ms))` — is a non-derived default. The schedule belongs on the owning smart-enum row or policy record as a `static readonly` field; callers that accept the default pass the field, and callers that need a different schedule pass their own composed value.
