# [BIM_OBJECT_MODEL]

Rasm.Bim host-neutral BIM object model: one `BimElement` element vocabulary, one `ElementSet` query algebra, one `Classification` axis, and one host-neutral `BimAssembly` spatial-hierarchy owner, all projected from and round-tripped through the `Exchange/interchange#IMPORT_RAIL` `IfcSemanticModel` graph and the kernel `Rasm` geometry — never a per-element-type class family, never a host-bound model. The page owns the `BimElement` typed-element family discriminated by an `IfcClass` row, the `ElementSet` set-algebraic query fold over element predicates, the `Classification` standard-systems axis (Uniclass/OmniClass/IfcClassification mapped onto one keyed vocabulary), and the `BimAssembly` host-neutral spatial-structure tree — composing the `Exchange/interchange#IMPORT_RAIL` `IfcSemanticModel` projection, the kernel `Rasm` geometry, and the `Exchange/interchange#FORMAT_AXIS` `InterchangeFormat` vocabulary as settled vocabulary; the page is HOST-NEUTRAL and carries no TS_PROJECTION.

## [2]-[ELEMENT_MODEL]

- Owner: `BimElement` the single host-neutral element record carrying the `IfcClass` discriminant, the stable `GlobalId`, the kernel-geometry handle, and the property/quantity/material/type bindings projected from the `IfcSemanticModel` graph; `IfcClass` `[SmartEnum<string>]` the closed buildingSMART entity-class vocabulary (IfcWall · IfcSlab · IfcColumn · IfcBeam · IfcDoor · IfcWindow · IfcSpace · IfcBuildingElementProxy · the IFC4.3 infrastructure classes IfcBridge/IfcRoad/IfcRailway/IfcPavement) keyed on the IFC entity-type string; `BimModel` the element-collection owner that wraps the projected element graph with the schema/model-view header.
- Cases: `IfcClass` rows enumerate the closed buildingSMART element-class set keyed on entity-type string (a new element class is one row, never a `WallElement`/`SlabElement` type); `BimElement` is one record discriminated by the `IfcClass` column, never a per-class subtype family.
- Entry: `public static Fin<BimModel> Project(IfcSemanticModel semantic, ClockPolicy clocks)` projects the in-process IFC semantic graph into the host-neutral `BimModel` element collection — `Fin<T>` aborts on an unmapped entity class or a missing spatial host; the geometry handle binds to the kernel `Rasm` geometry by reference, never re-tessellating.
- Packages: GeometryGymIFC_Core, Thinktecture.Runtime.Extensions, LanguageExt.Core, NodaTime, Rasm (project), BCL inbox
- Growth: a new element class is one `IfcClass` row; a new element binding (property/quantity/material/type) is one column on `BimElement` projected from the existing `IfcSemanticModel` row family; a new IFC4.3 infrastructure class is one `IfcClass` row riding the same projection; zero new surface, never a per-element-class type.
- Boundary: `BimElement` is ONE record discriminated by `IfcClass` row data, never a `WallElement`/`SlabElement`/`ColumnElement` class family — the deleted form mirroring the no-per-material-type law; the element model is HOST-NEUTRAL — the geometry handle is the kernel `Rasm` geometry consumed by reference and a RhinoCommon `Brep`/`Mesh` field on `BimElement` is the named seam violation; the `IfcSemanticModel` graph is owned at `Exchange/interchange#IMPORT_RAIL` and consumed here as settled vocabulary — Bim re-mints no semantic-graph shape; the element-class vocabulary is the closed `IfcClass` SmartEnum and a raw entity-type string leaking across a public signature is the named defect; property/quantity/material/type bindings project from the existing `IfcSemanticModel.PropertyRow`/`QuantityRow`/`MaterialRow`/`TypeRow` families, never a second property model.

## [3]-[ELEMENT_SET]

- Owner: `ElementSet` the set-algebraic query fold over a `BimModel` element collection, owning union/intersection/difference/filter as a closed algebra over `ElementPredicate` rather than a `GetWalls`/`GetByLevel`/`GetByMaterial` operation family; `ElementPredicate` `[Union]` the closed predicate family (by-class, by-property, by-classification, by-spatial-container, by-type) each carrying its discriminant payload; `ElementQuery` the composed query record folding predicates into one set-algebraic expression.
- Cases: `ElementPredicate` union arms — `ByClass(IfcClass)` · `ByProperty(string setName, string name, string value)` · `ByClassification(Classification system, string code)` · `BySpatialContainer(string globalId)` · `ByType(string typeGlobalId)`; a new predicate is one union arm, never a sibling `Get*` entrypoint.
- Entry: `public static ElementSet Query(BimModel model, ElementQuery query)` folds the composed predicate expression over the element collection into a result set — total, pure, no rail; set-algebra combinators (`Union`/`Intersect`/`Except`/`Where`) compose `ElementSet` values so a complex selection is one expression, never an imperative accumulation loop.
- Packages: Thinktecture.Runtime.Extensions, LanguageExt.Core, Rasm (project), BCL inbox
- Growth: a new query dimension is one `ElementPredicate` union arm folded by the same set algebra; a new set combinator is one member on the closed algebra; zero new surface, never a `Get<Dimension>` operation family.
- Boundary: there is ONE polymorphic query surface (`ElementSet.Query` discriminating on the `ElementQuery` predicate expression) and a `GetWalls`/`GetByLevel`/`GetByMaterial`/`FindBy<Key>` family is the deleted form per the no-operation-family law; the query is a fold over a closed `ElementPredicate` union, never an imperative filter loop with mutable accumulation; the result is an immutable `ElementSet` composing through the set algebra, never a `List<BimElement>` mutated in place; the classification predicate consumes the `Classification` axis as settled vocabulary, never re-deriving a classification mapping.

## [4]-[CLASSIFICATION]

- Owner: `Classification` `[SmartEnum<string>]` the standard classification-systems axis (Uniclass2015 · OmniClass · MasterFormat · Uniformat · IfcClassification) keyed on the system identifier, each row carrying the system's title and the table-prefix policy; `ClassificationCode` the value-object code-within-a-system (the `Pr_20_31_xx` Uniclass / `21-xx xx xx` OmniClass code shape) carrying its parent-code derivation; `ClassificationRef` the binding record tying a `ClassificationCode` to a `BimElement` GlobalId, projected from and round-tripped through `IfcRelAssociatesClassification`/`IfcClassificationReference`.
- Cases: `Classification` rows enumerate the admitted standard systems keyed on system id; a new classification standard is one row carrying its code-shape and table policy.
- Entry: `public static Fin<ClassificationRef> Classify(BimElement element, Classification system, string code)` validates the code against the system's code shape and binds it to the element — `Fin<T>` aborts on a code-shape mismatch; classification projection from the `IfcSemanticModel` reads `IfcRelAssociatesClassification` at ingest.
- Packages: GeometryGymIFC_Core, Thinktecture.Runtime.Extensions, LanguageExt.Core, Rasm (project), BCL inbox
- Growth: a new classification standard is one `Classification` row carrying its code-shape and table-prefix policy; a new code-derivation rule is one column on `ClassificationCode`; zero new surface, never a per-system classifier type.
- Boundary: the classification systems are ONE keyed axis (`Classification` SmartEnum) and a per-system `UniclassClassifier`/`OmniClassClassifier` type is the deleted form; the code shape is the row's policy validated once at `Classify`, never a per-call regex; the classification binding round-trips through the `IfcRelAssociatesClassification`/`IfcClassificationReference` IFC entities owned at the GeometryGym surface (`Exchange/interchange#FORMAT_AXIS` packages), consumed here as settled vocabulary; the classification predicate on `ElementSet` reads this axis, never re-minting a classification mapping.

## [5]-[ASSEMBLY]

- Owner: `BimAssembly` the host-neutral spatial-structure tree mirroring the IFC spatial hierarchy (project → site → building → storey → space → element) projected from the `IfcSemanticModel.SpatialNode` family; `SpatialContainer` the tree-node record carrying the spatial-element class, the GlobalId, and the contained-element set; `AssemblyRel` `[Union]` the closed decomposition-relationship family (aggregates · nests · contained-in-spatial-structure · voids · connects) mirroring the IFC `IfcRelAggregates`/`IfcRelNests`/`IfcRelContainedInSpatialStructure`/`IfcRelVoidsElement`/`IfcRelConnectsElements` relationships.
- Cases: `AssemblyRel` union arms — `Aggregates(string wholeId, Seq<string> partIds)` · `Nests(string hostId, Seq<string> nestedIds)` · `ContainedIn(string containerId, Seq<string> elementIds)` · `Voids(string elementId, string openingId)` · `Connects(string firstId, string secondId)`; a new relationship is one union arm.
- Entry: `public static Fin<BimAssembly> Assemble(IfcSemanticModel semantic)` folds the spatial hierarchy and the decomposition relationships into the host-neutral assembly tree — `Fin<T>` aborts on a dangling spatial reference; the tree is traversable by GlobalId and queryable through the `ElementSet` spatial-container predicate.
- Packages: GeometryGymIFC_Core, Thinktecture.Runtime.Extensions, LanguageExt.Core, Rasm (project), BCL inbox
- Growth: a new spatial-structure level is one `SpatialContainer` projection from an existing `IfcSemanticModel.SpatialNode` row; a new decomposition relationship is one `AssemblyRel` union arm; zero new surface, never a per-relationship type.
- Boundary: the assembly is HOST-NEUTRAL — it mirrors the IFC spatial hierarchy as a portable tree and a RhinoCommon `Layer`/`InstanceDefinition` binding is the named seam violation; the spatial hierarchy projects from the `IfcSemanticModel.SpatialNode` family owned at `Exchange/interchange#IMPORT_RAIL`, consumed here as settled vocabulary; the decomposition relationships are a closed `AssemblyRel` union mirroring the IFC `IfcRel*` entities, never a per-relationship class; the assembly is the host-neutral counterpart to the Rhino-native `csharp:Rasm.Rhino/Exchange` block-and-layer capture — the two coexist, neither gutted, meeting only at the universal IFC semantic contract.
