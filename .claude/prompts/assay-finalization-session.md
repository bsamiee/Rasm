## [2] DOCTRINE (non-negotiable — every change is judged against this)

- **One singular advanced polymorphic shape per concept.** Unified rails. NO proliferation of strings/types/constants/literals — push everything into `StrEnum`/`SmartEnum`-style payload enums, `msgspec` tagged unions, `pydantic` validators, advanced typing, and stdlib (`functools`/`itertools`/`operator`/`annotationlib`). A new string lands only in an enum member or a catalog row. New shape for an existing concept = rejected; extend a tagged-union variant / enum member / data row instead.
- **Inherent power, not flags.** Improve tools by deepening their internal behavior, resilience, and integration — NOT by adding agent-facing knobs. Agents must be unable to mis-invoke them or feed wrong inputs/outputs; the tool self-corrects, self-describes, auto-routes. Minimal agent surface, maximal internal value.
- **Deep external-lib leverage.** Use the installed libraries at their full power (verify APIs against `.venv/.../site-packages` source, never training data): `msgspec` (enc/dec hooks, `defstruct`, `Meta`, `convert`, `inspect`), `expression` (`map2`/`Block.collect`/`partition`/`sequence`/`pipe`), `pydantic`/`pydantic-settings` (computed fields, validators), `anyio` (memory streams, `TaskGroup.start`, `to_thread`/`to_process`), `beartype.vale.Is[]`, `opentelemetry` Links, `fsspec`/`universal-pathlib`, `psutil`, `stamina`, `structlog`, `cyclopts`, `tree-sitter`, plus CLI tools `ast-grep`/`rg`/`fd`. Adding a library/tool is justified ONLY by real LOC↓ or new capability — reject grab-bag with a reason.
- **No spam, no flat code, no low-quality patterns.** Functional/ROP throughout: `match`/folds/comprehensions/`Result` rails; NO `if`/`for`/`while`/`try` in domain logic (`try` only at marked boundary seams). No single-call thin helpers, no parallel near-duplicate factories, no dead vestiges. Density is concept count, not byte count; never delete functionality to hit a number.
- **Deep, multi-pass improvement.** Line-by-line scrutiny; reduce stringiness, fragile logic, latent/hidden bugs; collapse residual surface; root-shrink suppression clusters (`type:ignore`/`# noqa`) rather than spreading them.
- **Resilience FIRST, then optimize.** Order is mandatory: (a) prove correctness + parity + resilience (bug-free, no fragile logic, every command works), THEN (b) systematically optimize. Never optimize on top of a fragile base.
- **Agent-first output.** One JSON `Envelope` per invocation (NDJSON per fire for automation), `_emit` the sole stdout writer, structlog/diagnostics to stderr. Rich, perfectly-structured agent returns; surface artifact pointers when relevant; **NEVER truncate silently** — a capped inline list must carry `truncated=true` AND an on-disk artifact with the full set. Verb names encode cost/mutability (`fix` mutates, `report` never does, `full` is the expensive parity rail) so an agent predicts safety from the verb alone.
- **Blazing fast.** `code` and every hot path must approach Rust-level speed — without sacrifices, unusual workarounds, or dropped capability. Profile; pick the fastest correct primitive the ecosystem offers.
