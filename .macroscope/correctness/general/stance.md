# [REVIEW_STANCE]

Rasm is a design-first polyglot monorepo: the reviewable surface is a planning corpus — markdown pages whose embedded code fences are the work product — plus workflow and tooling JavaScript, Python operators, and configuration. Fence code is production source judged at full depth; simplifying a fence into a sketch is never a valid suggestion. Repo law rides `CLAUDE.md`, `AGENTS.md`, the root `README.md`, and `libs/.planning/campaign-method.md`; every diff is judged against those files and the `docs/stacks/<language>/` doctrine they route to, never against generic community convention. Doctrine is the floor, never the ceiling: a conformant-but-weak form is a finding whenever a stronger form exists.

## [01]-[STANCE]

- Presume defects until the pass finds none; dense, confident code is the prime suspect, and a clean-reading page is unproven, not clean.
- Verify every claim against the real domain — a geometry invariant, a schema contract, a package's actual API — never by ratifying prose, naming, or a comment's assertion. A claimed absence of a member verifies at the same bar as a positive citation: a false no-such-member claim steers every future rebuild into the inferior spelling.
- Settled corpus law outranks reviewer intuition: a finding contradicting a contract the owning page rules — a factory's declared parameters, a ruled composition form, a seam's admission placement — is wrong until that page is read and the claim verified against its stated contract; re-litigating a ruled design is a false positive, not caution. Pinned-newest provider surfaces are corpus law, so a released-library semantics claim — a member's return type, a pool's self-heal mechanics, a roster's existence — verifies against the owning doctrine page before flagging.
- Every finding names the file, the anchor, the defect, and the stronger form demanded — precise enough that a fixer agent lands the repair without re-deriving the analysis.
- Demand root-cause depth: when a finding admits a literal patch and a deeper collapse, the collapse is the demanded form.
- Cross-surface coupling is standing duty: `docs/laws/topology.md` maps which surfaces obligate which counterparts, and a diff editing a listed SURFACE without its obligated counterparts in the same change is a finding naming the missing counterpart. `docs/laws/patterns.md` and `docs/laws/scars.md` are standing review law.
- Suppressions and gate bypasses are the finding, never the mechanics: a true positive is architecture pressure to fix the shape, a false positive is rule pressure to refine the rule, and a suppression directive is neither. Mechanical shape law is gate-owned — `tools/biome` GritQL rows for TypeScript, `tools/cs-analyzer` rules for C# — so a finding restating what a promoted gate row enforces is noise, and a recurring mechanical anti-pattern demands the gate row.
- Grade across the full severity scale; a review where every finding lands at one severity has averaged, not graded, and a pass that finds nothing after genuine attack states that as its verdict, never padded with manufactured findings.

## [02]-[ARCHITECTURE_LAW]

- One polymorphic entry per rail, discriminating on input shape. Parallel entry points, modality-named siblings, `get`/`getMany`/`getById` families, and direction-named twins of one correspondence are defects; demand absorption into the one owner, never a new sibling.
- Judge every surface from its consumers: a parameter a policy value or input shape reconstructs is knob spam; configuration ceremony pushed onto callers, or orchestration a consumer must hand-write, is a defect. Demand internalized lifecycle, routing, and policy with zero capability sacrifice.
- A long imperative body is a defect wherever a denser expression-shaped form exists: demand fold, combinator, table-driven, or generator forms over hand-rolled loops and branching, and demand surfaces and object types collapse into fewer, denser, richer owners.
- Domain logic rides typed result/effect rails; raw exceptions in domain flow, or dual error paradigms crossing one boundary, are defects.
- Judge composition idiomatically: an interface earns its existence through dispatch or substitution, and generated or source-gen surfaces run at intended power — decorative abstraction is a finding.
- No compat shims, obsolete aliases, backwards-compat wrappers, or migration helpers anywhere: aggressive API breaking with every call site updated in the same change is the sanctioned path, and a preserved stale surface is the finding.
- Admission happens once at the boundary: a typed field's evidence is settled at its factory, so demanding re-validation of what the type already carries is the defect, never the finding; the inverse is the code-side defect — a defensive arm re-testing what the closed case set makes structurally true, an always-true predicate, or an interior re-validation of what admission already discharged deletes, the structural fact stated only where load-bearing.
- Dispatch is closed-world by default; an extensibility hook, open registry, or plugin seam is a defect unless extension must happen outside Rasm and a present consumer exists. A decorative operator overload without a proven algebra (monoid, semigroup, lattice) is rejected.
- A domain owner models its full domain — every attribute, sub-kind, state, relationship, invariant, and operation the domain carries; a missing axis is a defect, not thrift, and zero current consumers never lowers the bar.
- A correct owner shape proves itself in the diff of the next feature: the next case, dimension, or modality lands as one declaration inside the owner with every consumer untouched or loudly broken; sizing for the instance in hand and widening after call sites exist is the rejected order.
- An index is never spelled: pairwise, windowed, and batched traversal ride the language's adjacency operators over `range(len)` indexing, adjacency grouping requires the stream already ordered by its key, and co-iteration proves equal length at the seam rather than truncating on the shorter operand.
- Native recursion is legal only to bounded data depth; input-scaled or adversarial depth forfeits the recursive form for a streaming frontier fold, and raising the interpreter or stack ceiling is never the repair.

## [03]-[DEFECT_CLASSES]

Proven classes worth hunting at root-cause depth: required-field threading dropped by signature migrations, finiteness admission before range checks, identity preimages omitting output-affecting inputs, match/dispatch totality, frozen-payload discipline — mutable defaults, unsorted iteration reaching egress — per-stream timestamp re-basing, resource custody on every exit path, archive and metadata bomb gates, deterministic egress ordering. A multi-field hash preimage length-frames every variable-width field and count-frames every adjacent collection; separator-joined concatenation is rejected outright. A derived artifact keys on the content hash of its source, never path or mtime. Root discovery walks upward to a sentinel file, never a fixed parent depth. A wire token admits only its exact emission spelling; a tolerant parse re-emitting a normalized form forks the key. Execution policy and display labels stay outside a content preimage — a timeout or retry budget changes when work fails, never the bytes a success produces, and a label inside the preimage is a collision defect; a security identity takes a cryptographic digest, never a speed hash, and a composite id rides a canonical codec, never a hand-rolled join or quote scheme injective on one ambiguity axis. End-state depth guides the demanded form: a guard fixed at one site signals the same guard missing at every sibling site — demand the shared owner, never the local check; a defect in one arm of a family is evidence about every arm — demand the family fix; N siblings re-spelling one lifecycle machine over a shared rail — mint, custody hand-off, multi-product harvest, failure release, entry guard — demand that machinery as generic members on the rail owner with the siblings collapsed to one-line compositions, hand-maintained congruence invariants becoming structural inside the one generic builder, repeated dispose-on-failure blocks collapsing to one extension on the rail's result type, and a guard present on some family entries but absent on siblings is the latent defect the absorption fixes; evidence read at use-time belongs at construction — demand mint-time binding: an aggregate owning a correlating mint factory admits only through it, since a direct construction over complete-looking parts erases the correlation the factory binds, a derived value keeps one authority, so a configuration surface re-asserting what a construction factory already derives silently overwrites the mint, a hand-maintained mirror of a roster another owner carries — a slot enum shadowing a case list, hand-numbered keys shadowing host ordinals, per-case traits re-spelled beside the owning row — derives from that roster, and an evidence or receipt parameter every call site hardwires to one value is false evidence — the field derives from the run fact it reports or deletes; an invalid state becomes unspellable rather than guarded at each use; two paths producing one product converge on one seam, and the unifying diff deletes the dead hop; a missing check on a protocol surface signals the incomplete fold — demand the whole admission; a provider-behavior claim — including what a member's internals strip, drop, retain, or default — verifies against the installed distribution's actual code path, never assumption; canonicalization lands at intake so every consumer reads one regime. A residual gated against a zero ceiling is non-negative by construction — absolute deviation from target — or a degenerate input passes the clamp. A clamp over a caller defect launders it into false evidence: typed admission precedes normalization, and a shared evidence fold defines its empty-input value explicitly. A builder that drops a record on a failed reference lookup launders the dangling reference past its own integrity pass — the miss materializes as typed evidence the pass consumes, never a silent drop. A comment naming a root or anchor the code does not use marks a live defect — the owning root resolves once and threads as a parameter, never ambient reads. A dispatch table indexed by foreign material derives its membership admission from the table itself, never an enumerated reject-list; an allowed-set beside hand-spelled per-member writes is two enumerations of one roster — one delegate-column table owns both, the gate deriving from `ContainsKey` and the writes folding over its rows. Recursive expansion over untrusted sources carries container-identity cycle rejection and a node budget on the expand arm. A seam raising a typed fault spells the fault whole as the primary argument with context on the note channel, and multiple raise sites collapse onto one reconstruction owner. Failure-diagnostic side channels beside an empty or failed primary result ride the typed fault or receipt on that branch, never the success branch alone — success-only evidence binding discards the explanation where it decides. Evidence and telemetry labels ride domain-owned prefixes, never a sibling domain's. A filter defined by negation silently admits new states; the surviving set is a named positive constant. A cache splits identity slot from content address — the identity slot carries the pre-run source key the hit test compares, the produced output's content address is a separate derived fact, and minting the output address into the identity slot silently defeats keyed elision; a source with no canonical byte form (live handle, callable, non-deterministic serialization) joins the key as environment-scoped identity and demotes admission to forced-live. A parent-ancestry or dependency-multiset key folds as an ordered incremental hash over sorted parent keys — an XOR or additive fold self-cancels duplicate identical-content parents. A body needing several quantities from one sequence folds them in one pass into a typed frozen seed; a running mean or variance carried as fold state cannot combine associatively across partial folds — derived statistics project at read from raw accumulators.

## [04]-[DO_NOT_FLAG]

Deliberate house shapes that violate generic best practice — flagging them is a false positive:

- Aggressive API breaks with every call site updated in the same change: the sanctioned rename path.
- Dense single-expression bodies and heavy polymorphic dispatch: the bar, not obfuscation.
- Absent defensive guards inside domain logic: admission-once boundaries, not missing error handling.
- Sparse 1-2 line agent-facing comments: comment-law compliance, not missing documentation.
- A large file owning one full concern: sanctioned; file-size budgets do not exist here.
- Bracketed uppercase section dividers (`# --- [LABEL] ---`): source structure, not commented-out code.
- `TODO`/`FIXME` as source comment markers: sanctioned; the ban applies only to durable markdown.
- Rhino-rich host capture coexisting with host-neutral semantics: architecture, never duplication.
- Explicit match/dispatch arms over merged ternaries, and named intermediates on result rails: deliberate.
