# BCL Data Replacement Surfaces

[FROZEN_COLLECTIONS]:
- `FrozenDictionary<TKey,TValue>` and `FrozenSet<T>` are read-only after construction. Build them with `ToFrozenDictionary` or `ToFrozenSet` from any compatible `IEnumerable<T>`; the build step performs one-time analysis that may allocate several internal arrays, after which all lookups are branch-minimal and fully thread-safe.
- The concrete runtime type is selected by key type, count, and comparer: string keys with `StringComparer.Ordinal` on large sets produce `OrdinalStringFrozenDictionary_LeftJustifiedSubstring<T>` (substring-hashing to minimize comparisons); small or integer-keyed sets collapse to `WithFullValues<,,>` or `WithOptionalValues<,,>`; two-entry dictionaries may reduce to direct value tables. Callers see only `FrozenDictionary<TKey,TValue>` — the concrete type is an implementation detail whose selection encodes the lookup-performance guarantee.
- `GetAlternateLookup<ReadOnlySpan<char>>()` returns a `FrozenDictionary<TKey,TValue>.AlternateLookup<ReadOnlySpan<char>>` that accepts spans for `TryGetValue`, `ContainsKey`, and the indexer without allocating a temporary `string`. The underlying comparer must support the alternate key type — `StringComparer.Ordinal` and `OrdinalIgnoreCase` qualify. `FrozenSet<T>.GetAlternateLookup<ReadOnlySpan<char>>()` provides span-based `Contains` under the same comparer constraint.
- The same `GetAlternateLookup<ReadOnlySpan<char>>()` API is available on mutable `Dictionary<string,TValue>`. The mutable alternate lookup additionally exposes `TryAdd`, `Remove`, and the indexer setter, enabling fully span-keyed mutation with no string allocation on either the probe or the write path.
- Frozen construction is not thread-safe; `ToFrozenDictionary` must complete before any concurrent reader observes the result. The rejected form is a `static readonly Dictionary<,>` lookup cache: it requires `ContainsKey` / `TryGetValue` defensive pairs, offers no span-keyed lookup, and leaks a mutable internal reference.

[IMMUTABLE_COLLECTIONS]:
- `ImmutableArray<T>` is a value type wrapping a single array reference. `default(ImmutableArray<T>)` is a distinct state — `.IsDefault == true` — that is not equivalent to `ImmutableArray<T>.Empty`. Accessing `.IsEmpty`, `.Length`, or any element on a default instance throws `NullReferenceException`. The safe guard is `.IsDefaultOrEmpty`, which returns `true` for both states without throwing; `ImmutableCollectionsMarshal.AsArray(default(ImmutableArray<T>))` returns `null`, confirming the distinction.
- `ImmutableArray.CreateBuilder<T>(initialCapacity: int)` pre-allocates the backing array. When `builder.Count == builder.Capacity` exactly, `MoveToImmutable()` transfers ownership of the array without copying — zero allocation. When counts differ, `MoveToImmutable()` throws `InvalidOperationException`; `ToImmutable()` always copies and is safe at any count. The exact-capacity path is the correct form for pipelines that know output size upfront.
- `ImmutableArray.Create(ReadOnlySpan<T>)` and `ImmutableArray.Create(Span<T>)` each allocate exactly one array from the span and wrap it — the correct creation path for span-to-immutable pipelines without an intermediate `List<T>`.
- `ImmutableArray<T>.AsSpan()` returns a `ReadOnlySpan<T>` over the backing array in O(1) with no copy. `ImmutableArray<T>.GetEnumerator()` returns a value-type struct enumerator, so `foreach` over `ImmutableArray<T>` never boxes. The rejected form is routing through `ImmutableCollectionsMarshal.AsArray(imm)!.AsSpan()` — that backdoor exposes the mutable backing array and bypasses the immutability guarantee, whereas `imm.AsSpan()` produces a `ReadOnlySpan<T>` with no such risk.
- `ImmutableList<T>` is a 2-3 finger tree: indexed access is O(log n). `ImmutableArray<T>` is O(1) random access. Defensive copies of `IEnumerable<T>` for read-only exposure use `ImmutableArray<T>`, not `ImmutableList<T>`.
- `ImmutableDictionary.SetItem` returns a new dictionary with one key replaced; `SetItems` batches replacements into one structural copy. Multi-step construction uses the builder pattern — `.ToBuilder()` → mutate → `.ToImmutable()` — to avoid N intermediate dictionary allocations.
- `ImmutableSortedDictionary<TKey,TValue>` maintains key order via a sorted AVL tree: use it when sorted key enumeration is load-bearing. It is not a frozen-lookup replacement; lookups are O(log n) versus O(1) for `FrozenDictionary`.

[SPAN_AND_MEMORY]:
- `CollectionsMarshal.AsSpan(List<T>)` returns a `Span<T>` over the list's internal array covering `[0..Count)`. Any `Add` that triggers a resize invalidates the span; pin the list size before calling. Use for in-place bulk transforms — normalize, scale, encode — that must avoid a copy.
- `CollectionsMarshal.SetCount(List<T>, int)` sets the list's `Count` without zero-initializing new elements. The pattern — `SetCount(list, n)` then write via `AsSpan` — pre-allocates a known-length output and fills it through the span; the list then exposes exactly `n` elements. The rejected forms are `new T[n]`, `Enumerable.Repeat(default, n)`, and anything else that zero-initializes a buffer before overwriting it.
- `CollectionsMarshal.GetValueRefOrNullRef(dict, key)` returns a `ref TValue` into the dictionary's internal storage with no copy. Validate with `Unsafe.IsNullRef(ref result)` before use. `GetValueRefOrAddDefault(dict, key, out bool existed)` atomically returns a ref, inserting `default(TValue)` on absence — one lookup for accumulation loops that both probe and write, replacing the `TryGetValue` + `Add` double-lookup pattern.
- `MemoryMarshal.AsBytes<T>(Span<T>)` reinterprets a `Span<T>` of any unmanaged blittable type as `Span<byte>` in O(1) with no copy. `MemoryMarshal.Cast<TFrom,TTo>` reinterprets between unmanaged types. Both replace `Buffer.BlockCopy` or `Array.Copy` for struct-to-byte or float-to-int reinterpretation.
- `MemoryMarshal.Write<T>(Span<byte>, T)` and `MemoryMarshal.Read<T>(ReadOnlySpan<byte>)` serialize and deserialize unmanaged structs without `BinaryWriter` or `BinaryReader` allocation. Use over `stackalloc` buffers for fixed-format binary protocols.
- `MemoryMarshal.GetReference<T>(Span<T>)` returns a `ref T` to the first element without bounds checking — valid only inside `unsafe` blocks for pointer arithmetic. Use span indexers for safe element access.
- `MemoryMarshal.TryGetArray(ReadOnlyMemory<T>, out ArraySegment<T>)` extracts the backing array and offset when the `Memory<T>` was created from an array. Returns `false` for `MemoryManager<T>`-backed memory such as pinned native buffers; required before calling any API that accepts `ArraySegment<T>` only.
- `ArrayBufferWriter<T>` implements `IBufferWriter<T>`: `GetSpan(sizeHint)` returns a writable span, `Advance(count)` records written bytes, and `WrittenSpan` or `WrittenMemory` exposes the written region without additional allocation. `ArrayBufferWriter<T>` does not implement `IDisposable` — no `using` statement needed or valid. Use as the `IBufferWriter<byte>` target for `Utf8JsonWriter`, `PipeWriter`, and protocol encoders.

[STRING_AND_SPAN_FORMATTING]:
- `string.Create<TState>(int length, TState state, SpanAction<char,TState> action)` allocates exactly one string of the specified length, calls the action with the backing span still mutable, then seals it. The `static` lambda constraint on the action eliminates closure allocations. Use for any fixed-length string construction that would otherwise require `StringBuilder` or string concatenation.
- `Span<char>.TryWrite($"...", out int charsWritten)` is the `MemoryExtensions` interpolated string handler extension. It accepts any `ISpanFormattable` argument, composing multiple formatted values into a `stackalloc` buffer in one pass with no intermediate string allocation. Returns `false` when the buffer is too small; callers must handle that path.
- `ISpanFormattable` — signature `TryFormat(Span<char> dest, out int charsWritten, ReadOnlySpan<char> format, IFormatProvider? provider)` — is the correct interface for domain types that appear in formatted output. All BCL numeric types (`int`, `double`, `Guid`, `DateTime`, `DateTimeOffset`, `TimeSpan`) implement it. Domain value types implement it to participate in `TryWrite` composition without allocation.
- `ISpanParsable<T>` — `T.Parse(ReadOnlySpan<char>)` and `T.TryParse(ReadOnlySpan<char>, out T)` — is the symmetric read surface. All BCL numeric types implement it. Domain types that parse from sliced spans avoid the `string` intermediate that `int.Parse(string)` requires.
- `DefaultInterpolatedStringHandler` (namespace `System.Runtime.CompilerServices`) is the struct the compiler lowers interpolated strings to. Direct instantiation via `AppendLiteral`, `AppendFormatted`, and `ToStringAndClear` builds custom interpolation pipelines — structured logging, tracing, protocol framing — that want format-value composition without boxing or `params object[]`.

[SEARCH_VALUES]:
- `SearchValues<char>` created via `SearchValues.Create(ReadOnlySpan<char>)` compiles the character set into a SIMD-accelerated bitmap or lookup table depending on character range. ASCII-range sets use a 128-bit bitfield; mixed-range sets use a 256-entry lookup. All `MemoryExtensions` methods — `IndexOfAny`, `IndexOfAnyExcept`, `ContainsAny`, `LastIndexOfAny` — accept `SearchValues<char>` or `SearchValues<byte>` as direct replacements for their `char[]` or `byte[]` overloads.
- `SearchValues<string>` created via `SearchValues.Create(string[], StringComparison)` compiles the set with the Teddy algorithm — a SIMD-vectorized Aho-Corasick variant. `StringComparison` is mandatory; no default overload exists. Only `Ordinal` and `OrdinalIgnoreCase` are supported; culture-sensitive comparisons are not. The concrete runtime type name encodes the algorithm variant selected by set size and prefix structure (e.g., `AsciiStringSearchValuesTeddyNonBucketizedN3`). `MemoryExtensions.IndexOfAny(ReadOnlySpan<char>, SearchValues<string>)` returns the start position of the first matched string, not the matched string itself; a subsequent slice comparison is required when the matched value matters.
- `SearchValues<byte>` follows the same API as the `char` variant over `ReadOnlySpan<byte>`. Use for binary protocol scanning — delimiter bytes, sentinel sequences.
- Construct `SearchValues` instances as `static readonly` fields; the construction cost — compilation of the search structure — is non-trivial and must not be paid per call.

[SYSTEM_IO_HASHING]:
- `System.IO.Hashing` provides `XxHash32`, `XxHash64`, `XxHash3`, `XxHash128`, `Crc32`, and `Crc64`, all extending `NonCryptographicHashAlgorithm`. Static one-shot forms are the standard path: `XxHash64.HashToUInt64(ReadOnlySpan<byte>)`, `XxHash3.HashToUInt64(ReadOnlySpan<byte>)`, `Crc32.HashToUInt32(ReadOnlySpan<byte>)`, `Crc64.HashToUInt64(ReadOnlySpan<byte>)`, and `XxHash128.HashToUInt128(ReadOnlySpan<byte>)`. All accept an optional `seed` parameter. Every algorithm also exposes `Hash(ReadOnlySpan<byte>)` returning `byte[]` for callers that need raw bytes.
- `XxHash3` and `XxHash64` each produce 8-byte output; `XxHash128` produces a 16-byte `UInt128`. The `XxHash128.HashToUInt128` one-shot is the correct form when the full 128-bit value is consumed as a `UInt128` rather than a `byte[]`.
- Streaming form: instantiate directly (no `using` — `NonCryptographicHashAlgorithm` does not implement `IDisposable`), call `Append(ReadOnlySpan<byte>)` one or more times, then call `GetCurrentHash(Span<byte>)` (non-destructive) or the span overload of `GetHashAndReset(Span<byte>)` (destructive, fills destination without allocation). The zero-argument `GetHashAndReset()` allocates a `byte[]`; prefer the span overload when the hash is consumed immediately. `Reset()` returns the instance to its initial seeded state without reallocation.
- `XxHash3` outperforms `XxHash64` on modern SIMD hardware at equivalent 8-byte output size; prefer it for new non-cryptographic fingerprinting. `HashCode.Combine` is appropriate for in-process structural equality; `System.IO.Hashing` targets cross-process or persistent-session stable hashes — cache keys, file fingerprints, content-addressable storage. Neither is appropriate for security-sensitive contexts.

[SYSTEM_TEXT_JSON_SOURCE_GENERATION]:
- `[JsonSerializable(typeof(T))]` on a `partial class Ctx : JsonSerializerContext` generates compile-time metadata; `Ctx.Default.T` is the typed `JsonTypeInfo<T>` that `JsonSerializer.Serialize` and `Deserialize` accept directly — no reflection, no runtime type discovery.
- `[JsonSourceGenerationOptions(...)]` on the context class sets options at generation time: `PropertyNamingPolicy`, `WriteIndented`, `UseStringEnumConverter`, `DefaultIgnoreCondition`, `UnmappedMemberHandling`. These compile into the generated metadata and override `JsonSerializerOptions` defaults when using `Ctx.Default` directly.
- `JsonNamingPolicy.SnakeCaseLower`, `SnakeCaseUpper`, `KebabCaseLower`, and `KebabCaseUpper` are first-class statics — no hand-rolled converter or `string.Replace` is needed for any of the four standard identifier-casing schemes.
- `JsonTypeInfoResolver.Combine(ctx1, ctx2, ...)` chains resolvers; the first resolver returning non-null for a type wins. Merge a source-generated context for owned types with `DefaultJsonTypeInfoResolver` for foreign types that require reflection fallback.
- `JsonSerializerContext.WithAddedModifier(Action<JsonTypeInfo>)` returns a new resolver wrapping the context with a runtime modifier. The modifier receives the generated `JsonTypeInfo` and can remove or conditionally suppress properties via `info.Properties.Remove(prop)` or `prop.ShouldSerialize = (obj, val) => false`. This is the correct path for runtime property filtering on source-generated contracts — not a hand-rolled custom `JsonConverter`.
- `JsonSerializerOptions.MakeReadOnly()` with no argument throws `InvalidOperationException` when no `TypeInfoResolver` was assigned. `MakeReadOnly(populateMissingResolver: true)` installs `DefaultJsonTypeInfoResolver` automatically. In production, assign `TypeInfoResolver` explicitly before sealing.
- `JsonUnmappedMemberHandling.Disallow` — set via `[JsonUnmappedMemberHandling(JsonUnmappedMemberHandling.Disallow)]` on the type or `UnmappedMemberHandling` in options — causes deserialization to throw on unrecognized JSON properties. Use on strict protocol types to surface schema drift rather than silently discarding unknown fields.
- `Utf8JsonWriter` over `IBufferWriter<byte>` writes UTF-8 JSON without intermediate string allocation. `Utf8JsonReader` over `ReadOnlySpan<byte>` reads without string allocation. `JsonSerializer.Serialize(string)` allocates a UTF-16 string intermediate and is the rejected form for hot paths.
- `JsonNode`, `JsonObject`, and `JsonArray` support mutation after parse: `node["key"] = value`, `array.Add(element)`. Use for transformations where the input schema is partially unknown; use typed `JsonSerializer` for owned, schema-stable types.

[RUNE_AND_TEXT_SEGMENTATION]:
- `System.Text.Rune` represents a single Unicode scalar value — codepoints in `[U+0000, U+D7FF] ∪ [U+E000, U+10FFFF]`. `Rune.TryCreate(int, out Rune)` returns `false` for lone surrogates and out-of-range values; the constructor throws `ArgumentOutOfRangeException` on invalid input. All character-level predicates — `IsLetter`, `IsDigit`, `IsWhiteSpace`, `IsControl`, `IsPunctuation`, `GetUnicodeCategory` — operate on the full Unicode codepoint. `char` predicates produce meaningless results on the high or low surrogate half of a supplementary-plane character; `Rune` predicates do not.
- `Rune.DecodeFromUtf16(ReadOnlySpan<char>, out Rune, out int charsConsumed)` and `Rune.DecodeFromUtf8(ReadOnlySpan<byte>, out Rune, out int bytesConsumed)` advance a cursor without any intermediate string. Return `OperationStatus.Done`, `NeedMoreData`, or `InvalidData`. These are the correct primitives for custom parsers that must handle arbitrary Unicode input.
- `Rune.EncodeToUtf8(Span<byte>)` writes 1–4 bytes and returns the count. Paired with `DecodeFromUtf8`, this is the zero-alloc UTF-8 transcoding loop: read from a source span, write to a destination span, and advance both cursors by the consumed and written counts.
- `string.EnumerateRunes()` returns a `StringRuneEnumerator` (`ref struct`) for forward-only codepoint traversal over a string without allocation. Use in `foreach`; do not store the enumerator. Use for codepoint-level analysis — letter count, category histogram, normalization check — over strings that may contain supplementary-plane characters.
- `Rune.ToUpperInvariant` and `Rune.ToLowerInvariant` perform codepoint-level case mapping without culture-sensitive rules, safe for identifiers and protocol keywords. They do not handle context-dependent mappings such as Turkish dotted-I; use `CultureInfo`-qualified APIs for locale-aware text.
- `System.Globalization.StringInfo.LengthInTextElements` and `StringInfo.GetTextElementEnumerator(string)` operate at the grapheme cluster level, grouping base characters with combining marks. A string with `N` code units and `M` combining marks reports `LengthInTextElements == N - M`; `LengthInTextElements != string.Length` whenever combining characters, surrogate pairs, or variation selectors are present. Use when the unit of user-perceived character is load-bearing — cursor movement, display-width estimation, text truncation.

[TENSOR_PRIMITIVES_SMELL_ROW]:
- `System.Numerics.Tensors.TensorPrimitives` exposes static generic methods over `ReadOnlySpan<T>` and `Span<T>` with constraints that range from `where T : INumber<T>` for arithmetic and reductions to narrower interfaces for float-specific operations. One method covers `float`, `double`, `int`, `Half`, `decimal`, and any custom `INumber<T>` implementation.
- Element-wise operations — `Add`, `Multiply`, `Sqrt`, `Sin`, `Exp`, `Log2`, `Pow`, `Atan2` — replace scalar loops and vectorize via `Vector128<T>`, `Vector256<T>`, or `Vector512<T>` depending on hardware. `FusedMultiplyAdd(a, b, c, dest)` computes `a*b+c` with a single rounding error, replacing `a*b` then `+c`.
- Reductions — `Sum`, `Product`, `SumOfSquares`, `SumOfMagnitudes`, `Max`, `Min`, `Norm` (L2), `Dot` — replace hand-rolled accumulation loops. `IndexOfMax`, `IndexOfMin`, and `IndexOfMaxMagnitude` replace manual argmax tracking.
- `CosineSimilarity`, `Dot`, and `Norm` are the three distance and similarity primitives for dense vector comparison. `Lerp(a, b, t, dest)` performs element-wise linear interpolation, replacing `a + (b-a)*t` loops.
- `ConvertChecked`, `ConvertTruncating`, and `ConvertSaturating` perform type-converting element-wise copies between span types. `ConvertTruncating` truncates toward zero; `ConvertSaturating` clamps to the destination type's range — both replace manual `for` loops that cast element by element.
- `SoftMax` and `Sigmoid` apply directly to float spans, replacing manual exp-then-normalize loops.
- The signal: any `for` loop over a numeric array whose body consists solely of arithmetic, math function calls, or type conversions is a `TensorPrimitives` candidate. Numeric algorithm law governs the implementation; this row identifies the data-shape signal that triggers the lookup.
