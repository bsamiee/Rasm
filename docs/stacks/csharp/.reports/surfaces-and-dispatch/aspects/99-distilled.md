# Aspects — Distilled

[WEAVE_TAXONOMY]:
- Definition-time aspects are properties of the type — admission, identity, dispatch, serialization, grammar — declared by attribute, emitted by the generator as per-concern partial files, present at every call site. No runtime proxy, no IL rewriting, no container interception: the converter, comparer, parser, and dispatch surfaces live in the type itself. Generated files target disjoint member names; a collision is a duplicate-member compile error, never a silent override — partial merging only unions disjoint member sets.
- Composition-time aspects are properties of a specific call site — retry, recovery, resource lifetime, timeout, marshaling — attached as operators and transformers in the pipeline; the type carries no trace of the policy, and the same computation takes different policies at different sites. The classification test: a concern present at every use of the type weaves at definition; a concern that varies per call site composes at the site.

[CONSTRUCTION_ADVICE_CHAIN]:
- The generated factory threads advice points in a fixed, generator-owned order: reference/absence guards, error-slot initialization, the mutating factory hook, construction gated on a null error, the constructor hook, post-init. The author opts into slots; reordering is not available. Absence guards run BEFORE the factory hook — when the owner's absence policy yields null, the hook never fires, so null/blank rejection written inside the hook is dead code; absence is owner-knob policy, not advice.
- The factory hook (`ValidateFactoryArguments`) passes every argument by `ref`, leading with a `ref` typed-error slot: it is a normalizer and a validator in one weave — write through the `ref` to canonicalize (trim, case, clamp) before storage; write the slot to reject without throwing. A hook that throws breaks the admission contract — the rail bridge expects a returned error. Detection is by exact name and static non-generic shape, no attribute: renaming silently un-weaves the advice, and declaring it generic makes it invisible to the generator. The format provider is consumed by the generated parse layer BEFORE the hook — culture-dependent admission belongs on the owner's parsable surface, not in the hook.
- The constructor hook (`ValidateConstructorArguments`) fires on every construction path — including deserialization, store materialization, and the trusted corresponding-constructor fast path that bypasses the factory entirely. It carries `ref` power but no error slot: it signals by throwing, because rehydration has no result rail. The provenance split is the design lever: expensive admission checks in the constructor hook tax every store read; invariant repair only in the factory hook leaves store values unrepaired. The fully defended owner centralizes the rewrite in one private static called from both — soft-fail on admission, hard-assert on rehydration.
- Declaring the factory hook with a non-void return rewires a second weave point: the returned value threads into post-init as a typed side channel — computation done once during validation is handed forward with no field, no closure, no second pass. Post-init is an instance member, the only advice that observes the live instance, and never runs on the rejection branch. On complex owners every assignable member is a `ref` parameter, per-member absence guards precede the hook in declaration order, and a cross-member invariant must tolerate any member already short-circuited to its absence default.

```csharp
public sealed partial class Handle   // [ValueObject<string>] owner
{
    static partial void ValidateConstructorArguments(ref string value) =>
        value = value.Trim();                                  // every path, incl. store reads

    private static partial int ValidateFactoryArguments(       // non-void => typed channel
        ref ValidationError? validationError, ref string value)
    {
        value = value.ToLowerInvariant();                      // normalize before construct
        var checksum = Hash(value);                            // computed once at admission
        validationError = checksum is 0 ? Err("empty") : null;
        return checksum;
    }

    partial void FactoryPostInit(int checksum) => _index = checksum;  // sees channel + instance
}
```

[PARTIAL_MEMBER_SEAMS]:
- The generated/hand-written seam is the partial member, and each kind carries its own law: partial constructors put the initializer (`: this`/`: base`) only on the implementing part; partial properties keep the definition part exactly auto-property-shaped with accessor sets matching across parts; partial events forbid initializers on both parts and make the implementing part mandatory. Extended partial methods — non-void return, `out` parameters, or extended modifiers — each independently require an access modifier; the three diagnostics do not subsume one another.
- The silent-drop law is the trap: a legacy implicit-void partial method without an access modifier may omit its implementation and is deleted at every call site without diagnostic. A generator advice point that requires guaranteed execution or a return value must be an extended partial method — the access modifier is what converts "optional advice" into "mandatory seam."

[IDENTITY_AND_PROTOCOL_ATTACHMENT]:
- Identity attaches through comparer-accessor types named at the member: one accessor type satisfies both the equality and ordering contracts, so a single definition-time choice fixes `==`, hashing, and ordering together; per-member accessors on complex owners override the type-wide default field by field, and the type-wide string-comparison default covers all string members without per-member annotation. An ignored member is excluded from equality, hashing, serialization, and dispatch simultaneously — and must be immutable, or two "equal" instances become observably different.
- Protocol surfaces gate on capability plus skip flags, and the skips are a derivation, not independent knobs: suppressing parsable forces span-parsable off (the BCL inheritance makes the cascade); suppressing factory methods cascades across five aspects at once — type converter, factory interfaces, key conversion, both parsable surfaces, and all arithmetic operator generation, permanently (a later custom factory restores serialization and store routes, never operators or parsing). Ordering operators drag equality operators up to their level; the equality-suppression flag dominates both. One setting in each pair is the authority.
- Serialization weaves are reference-gated: the owner's framework flags are a ceiling the compilation's actual references intersect — converters emit only for frameworks whose assemblies are referenced, adding a reference later weaves its converter with no owner edit, and absent companions skip silently. The foreign-factory attribute reconciles the typed error rail with throwing host boundaries at one generated seam: the type-converter projection turns a returned error into the host's expected exception — the domain never sees the exception, the host never sees the rail.

[HKT_OPERATOR_SURFACES]:
- Constrained operators over carrier-polymorphic values exist only through extension blocks: `extension<E, F, A>(K<F, A> _) where F : Fallible<E, F>` defines `operator |` directly on the carrier with trait constraints — operator parameters need not include the declaring type. One block defines an entire aspect surface (the unlift family: local, uninterruptible, post, timeout, bracket, repeat/retry variants) as receiver syntax. Any local aspect introducing a constrained operator on a carrier must take this form; there is no other mechanism.

[COMPOSITION_TIME_MECHANICS]:
- The schedule algebra is three distinct binary operations: `|` is union (minimum delay per step, runs to the longer); `&` is intersect (maximum delay per step, runs to the shorter); `+` is sequential append on schedules — but function composition on transformers, left-to-right and non-commutative: zero-first-then-jitter and jitter-then-zero-first are different policies. A transformer stored in a `Schedule`-typed field silently grafts an invisible infinite base through the implicit conversion; store transformers as transformers and apply them at the site. The recur-limit transformer caps iterations; the repeat transformer appends a forever-backed tail — they are not interchangeable.
- Time-anchored builders carry their own clock as a constructor argument — the policy value reads no global, and deterministic tests inject the clock. Window semantics are policy choices encoded in the builder: fixed-interval guarantees a gap from completion (load-leveling, no deficit accumulation); windowed aligns to boundaries anchored at the schedule's own first run, not clock-epoch multiples. Wall-clock-elapsed termination and cumulative-emitted-delay termination diverge whenever action time varies — they are different bounds, composed together when both matter, alongside the per-step delay clamp.
- Zero-first-delay composition is idempotence-aware: the predicate-gated retry forms prepend a zero delay unconditionally, so composing the no-delay-on-first transformer onto a schedule already passed to them yields a double-zero prefix, not an extra retry. Decorrelated jitter approximately doubles the effective step count — its purpose is anti-herd dispersion among parallel instances, never delay expansion. Interleaving alternates single steps from two schedules; interspersal inserts a whole sub-schedule between steps — categorically different. Only `Schedule` is a monad (duration-derived sub-schedules via query expressions); transformers have no bind.
- Retry and repeat are antipodal and type-compatible — the compiler cannot catch the swap: retry re-invokes on failure and predicates on the error; repeat re-invokes on success and predicates on the value. Polling is schedule-driven repeat with a value predicate; recovery is schedule-driven retry with an error predicate; both live as named fields on a policy owner, never flags at call sites.

[RECOVERY_AND_CLEANUP]:
- A named catch is a predicate-handler policy value composed with `|` as handler WRAPPING, left-associative: inner handlers see errors before outer ones, so a broad predicate placed before a narrow one shadows it with no diagnostic — stacking order is precedence order. Selector axes are orthogonal: value-equality match, predicate match, and code match (the code form disambiguates siblings a subtype catch cannot; errors should carry non-zero codes because the code overload has no message fallback). The subtype catch recurses into aggregates; its fail-fast traversal and the fold variant's accumulate-across-matches are constraint-enforced — the fold form does not compile without the monoidal carrier. A predicate-free total catch positioned last guarantees no escape.
- The finalizer is a transparent pass-through, not a barrier: it runs on both exits via a real `finally` and re-propagates the error, so a catch after it still fires — position relative to the finalizer decides only whether cleanup precedes or follows the handler. The unconditional-alternative operator form (right operand a computation, pure value, or error) recovers on any failure without inspection and must not be confused with predicate-guarded catch; the monoid identity element surfaces as a distinguishable none-error that catch predicates can test.
- Retry wraps one atomic unit: retrying a computation-with-inner-catch retries the recovered pipeline; catching around an outer retry additionally sees exhaustion (the last error propagates unchanged when the schedule runs out). The two orders are two policies.

[RESOURCE_AND_SCOPE_ASPECTS]:
- Resource lifetime is an ambient-tracker aspect, not a lexical block: acquisition registers the resource in the effect environment and returns the naked value; release happens at the next bracket boundary, an explicit release, or scope teardown — acquire and release are separate aspect points, so a deep bind-chain acquisition is freed by an outer bracket that never saw the acquire site. The zero-argument bracket frees everything acquired inside on either exit; the fail-only bracket frees only on failure and deliberately lets success-path acquisitions escape to the continuation — selecting fail-only where full was meant leaks every successful acquisition. The three-argument acquire/use/release form is a manual finally with no ambient tracking; the four-argument form's catch arm covers acquisition errors only.
- Iteration combinators bracket each pass: per-iteration resources are freed before the next attempt fires, which yields the hard rule that a resource acquired inside a retried or repeated computation cannot be returned out of it. Forking allocates a child resource set and a linked cancellation source, disposing both when the fork completes — structured concurrency as one woven aspect; parent-acquired resources are visible-but-shared inside the fork, fork-acquired resources die with it.
- Scope aspects are surgical about what they sever: the local scope takes a fresh linked cancellation source but the SAME resource set, and a cancellation raised inside propagates out — containing it requires composing the catch outside the local boundary. The uninterruptible scope severs only the token path; error rails propagate normally. Marshaling posts to the synchronization context captured when the environment was constructed — not the current thread's context at execution — and runs inline when none was captured. Timeout and explicit cancellation surface as distinct coded errors; retry predicates that must treat them differently compare codes. The insulation aspect reifies the error rail into a value-level result inside the effect, so both paths are inspectable as values — recovery combinators applied outside see failures as bound values, not rail continuations.
- Environment scoping at dispatch boundaries is a two-form pair: narrowing maps a broad runtime to the arm's narrower one so the arm stays ignorant of the outer capabilities; widening lifts a runtime-agnostic effect into the runtime-aware sequence. A timeout or deadline as an entrypoint parameter is the policy-smuggling anti-pattern — the bound is an effect-layer aspect injected after dispatch, and the signature never grows a token tail for it.

[STACKING_ORDER]:
- Definition-time aspects stack in generator emission order, fixed; composition-time aspects stack in author-written operator order, and the same two aspects in two orders are two policies: retry around a bracket re-runs acquire-use-release per attempt; a bracket around a retry acquires once and retries only the use. Recovery and cleanup are orthogonal axes — a recovered error still runs the finalizer.
- Generated dispatch is atomic to every wrapper: the dispatch and all case behavior collapse to a value before any composition-time transformer runs, so per-case retry, timeout, or recovery is not expressible by wrapping the dispatch. Per-case policy requires each arm to produce an effectful carrier wrapped with its own policy before the dispatch assembles them — the granularity of a composition-time aspect is fixed by where the carrier is introduced, inside the arm or around the whole.
- The two weaving layers meet at exactly one seam: the admission boundary where a generated factory's typed error lifts into the effect carrier. Below it every aspect is definition-time and generator-ordered; above it every aspect is composition-time and author-ordered. Policy pushed across the seam in either direction — retry inside a construction hook, normalization inside a pipeline combinator — gains hidden control flow and stops being recoverable from the declaration.

[INTERCEPTORS_AND_INITIALIZERS]:
- Interceptors are a generator-only weave with hard bounds: explicit MSBuild namespace opt-in or the weave is wholly suppressed; only the checksum-based location API (regenerated every compilation, never cached — version drift is a compile error, which is the safety); only invocation expressions (no constructors, accessors, or delegate invocations); one interceptor per location; the interceptor must be an ordinary member method in a non-generic type — extension members and lowered block members are disqualified by shape. The companion query API returns the resolved interceptor for a call site, closing the generation-verification loop inside one compilation pass.
- Module initializers are the last-resort unconditional registration aspect: static, parameterless, void, non-generic, in a non-generic type. Multi-initializer ordering follows IL emission order — an implementation artifact, not a guarantee — so order-dependent registrations consolidate into one initializer with explicit calls.
