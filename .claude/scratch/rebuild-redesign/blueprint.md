## [01]-[ARCHITECTURE_OVERVIEW]

Run shape (one invocation, N waves, M packages):

```text
Plan (sonnet/low) ── emits pages[wave, dependency order] + deletePages + riders + acceptance + upstreamMissing
  └─ for (wave of WAVES)                          ← the ONE inter-wave barrier (real: cross-wave content moves)
       package lanes (parallel, disjoint index docs)
         Discover  pool(batches)  opus/max  READ-ONLY → worklist + grounding dossier (scratch)
         Build     per-batch pipeline, dependency order:
                     B0.impl → B0.crit → B0.rt
                               B1.impl → B1.crit → B1.rt        (B[k].impl awaits B[k-1].impl;
                                         B2.impl → …             reviews chain in the same order)
       delete executor (brief deletions, verify-then-delete)
       TAIL (fable, serialized, one per wave): applies indexRows + serial riders, audits rider receipts
       health gate: any failed page → halt before dependent waves (landed waves intact)
  └─ ACCEPTANCE (opus, read-only, fail-open): brief dry-run traces + generic symbol-resolution sweep
```

Phases: `meta.phases` declares the literal four — `Plan`, `Discover`, `Build`, `Close`. Runtime `phase()` calls use `'W'+k+' Discover'` / `'W'+k+' Build'` and `'Close'` — `Plan` + 2×N + 1, structural, never spam. The standalone Critique/Redteam display phases die; the wave index rides agent labels (not cache-relevant).

## [08]-[WRITE_PARTITION]

- LANE = PACKAGE (`pkgOf`, the path before `/.planning/`). Cross-package lanes run fully concurrent — their index docs and page sets are disjoint.
- WITHIN a lane, dependency order is law: batch K's implement awaits batch K-1's implement (a consumer batch authors against landed foundation fences — the NURBS-spine divergence dies here); review pairs chain in the same order behind their implements. Steady-state concurrency per lane is 3 agents (impl/crit/rt of successive batches), all write-ordered.
- SHARED CENTRAL FILES — the owning-package `ARCHITECTURE.md`/`README.md` and the central manifests (`Directory.Packages.props`, `.csproj`, `pyproject.toml`, `pnpm-workspace.yaml`) — have exactly ONE writer: the per-wave TAIL agent. Page agents report `indexRows`; the tail applies them once, deduped, section-grammar preserved. This replaces the current "keep the owning-folder index docs truthful" per-agent instruction, which races ≥3 concurrent writers today.
- UNIQUELY-OWNED cross-file work stays in-pass under WRITE_FULLY: seam counterparts, consumer sites, absorb-then-delete (atomic in the absorber's implement agent, cross-package included). Anchored surgical edits with re-read-and-reapply on conflict remain the discipline for the residual case of two lanes touching one uniquely-owned seam file.
- A later batch's review fixing divergence against an earlier page conforms the CONSUMER to the landed foundation fence (the correct direction — the foundation authored first is law); a genuine foundation-side defect reports through `indexRows`-style tail routing only when the foundation page's chain has closed — otherwise the review edits it directly (its chain ordering guarantees the earlier redteam completed).
- Read-during-write across successive batches (B2.impl reading a B0 page while B0.rt refines it) is tolerated: it is strictly narrower than today's fully-concurrent review pool, and the reader's own critique/redteam re-verify against final state.
