# Monad Transformer Stack Algebra And Capability-By-Order Law

[ONE_INNER_BIND_DELEGATION]:
- Every transformer's `Bind` is written exactly once, purely in terms of the immediately-inner monad's `Bind` тАФ no layer re-implements the layers below it. `ReaderT<Env,M>.Bind` is `new(env => M.Bind(runReader(env), x => f(x).runReader(env)))`; `StateT<S,M>.Bind` is `new(s => M.Bind(runState(s), x => f(x.Value).runState(x.State)))`; `EitherT`/`OptionT`/`FinT` are `new(M.Bind(runInner, x => x.Match(Right/Some/Succ: y => f(y).runInner, Left/None/Fail: short-circuit)))`. A deep stack composes by one-level delegation: `ReaderT<Env, StateT<S, Eff>>.Bind` invokes `StateT.Bind` as its `M.Bind`, which invokes `Eff.Bind` тАФ the spine is a chain of single hops, never a flattened reimplementation. The consequence is that adding a transformer is additive at zero semantic cost to the layers below, and the cost of a stack is exactly the sum of one `Bind` allocation per layer per step.
- `ReaderT` carries `Func<Env, K<M, A>>`; `StateT` carries `Func<S, K<M, (A Value, S State)>>` тАФ note the *named-tuple* inner shape, not a positional pair, so the state thread is `x.State` not `x.Item2`. The state carrier is the only one whose inner bound value is not `A` but `(A, S)`; this asymmetry is the root cause of the capability law below тАФ a monad whose `K<M,_>` payload is a product cannot be unlifted to a bare `IO<A>`.

[CAPABILITY_BY_STACK_ORDER]:
- The entire composition-time IO-effect surface тАФ `Fork`, `Await`, `Timeout`, `Bracket`, `Repeat`, `Retry`, `Fold`, `Post`, `Local`, `Uninterruptible` тАФ is declared once as `static virtual` defaults on `MonadUnliftIO<M>`, each a single line `MapIO(ma, io => io.<Op>(...))`, and `MapIO` itself is `ToIO(ma).Bind(io => M.LiftIO(f(io)))`. The whole surface therefore hinges on one `static abstract K<M, IO<A>> ToIO<A>(K<M,A>)`. A transformer earns the entire algebra by supplying `ToIO`; a transformer that cannot project its layer back into a bare `IO<A>` forfeits all of it.
- `ReaderT` supplies `ToIO` (its payload is `K<M,A>`, projectable) and so propagates the unlift surface; `StateT` supplies only `MonadIO` (`LiftIO`), never `MonadUnliftIO`, because its payload `K<M,(A,S)>` cannot collapse to `IO<A>` without discarding the state thread. The load-bearing rule: stack order is a capability decision, not a cosmetic one. A layer that needs `Retry`, `Bracket`, `Fork`, or `Timeout` must sit on the IO-capable side of any state-shaped or product-shaped layer тАФ `ReaderT<Env, StateT<S, Eff>>` keeps unlift reachable through `ReaderT`'s pass-through only if the op targets a sub-computation that itself bottoms out at `Eff`; a `StateT` placed *outside* the IO boundary makes the outer effect un-bracketable. Order changes what the stack can do, not merely what it means.
- Unlift is a runtime contract, not a compile-time one. `MonadUnliftIO.toIO` resolves to `M.ToIOMaybe(ma)` and `mapIO` to `M.MapIOMaybe` тАФ the `Maybe.*` shadow surface is the escape hatch the cascade actually calls, and a stack whose innermost monad never overrides `LiftIO(IO<A>)` type-checks and then throws a wrapped exceptional at run time on first lift. A stack design must guarantee an IO base; the constraint solver will not.

[LIFT_CASCADE_VS_PURE]:
- Lifting an inner action one layer is `MonadT.lift(ma)` тЖТ `MTran.Lift(ma)`, a direct type-class dispatch with no `Pure` and no allocation beyond the wrapper. Each layer's `Lift` wraps the immediately-inner `K<M,A>` in its own constructor: `ReaderT.Lift(ma) = new(_ => ma)` (constant function, env ignored), `StateT.Lift(ma) = new(s => M.Map(v => (v, s), ma))` (pairs with unchanged state). Lifting across N layers is N nested `Lift` calls, each ascending one level.
- Lifting a raw `IO`/`Eff` action up a stack is `LiftIO`, never `Lift`-of-`Pure`. `ReaderT.LiftIO(io) = new(_ => M.LiftIOMaybe(io))` тАФ each layer delegates to the next inner layer's `LiftIOMaybe`, cascading until a layer is the IO base and overrides it. The distinction is decisive: `Lift(M.Pure(x))` injects a pure value at the *current* layer with no effect semantics, while `LiftIO` threads a real effect through every intermediate layer to the IO floor. Conflating them yields a value-shaped placeholder where an effect was intended, silently.

[HETEROGENEOUS_FRAGMENT_BIND_ABSORPTION]:
- Each transformer overloads `Bind`/`SelectMany` per inner-fragment kind so a `from`-comprehension over a stack mixes layer-local effects with no explicit `lift`. `ReaderT` absorbs `Func<A, Ask<Env,B>>`, `Func<A, IO<B>>`, `Func<A, Pure<B>>`, and `Func<A, K<M,B>>`; `StateT` absorbs `Func<A, Gets<S,B>>`, `Func<A, Put<S>>`, `Func<A, Modify<S>>`, `Func<A, IO<B>>`, `Func<A, K<IO,B>>`, and `Func<A, K<M,B>>`. Inside one comprehension, `get`, `put`, `modify`, `ask`, a bare `IO` step, and a `K<M,_>` inner step each bind through the overload that matches their shape тАФ the transformer is the dispatch surface that erases the lift ceremony at the query boundary.
- The collapse rule: a stack pipeline never interleaves manual `StateT.lift`/`ReaderT.liftIO` calls between domain steps. A `Gets`/`Put`/`Modify`/`Ask`/`IO` fragment is yielded directly into the `from` chain and the overloaded `SelectMany` resolves the lift by argument type. A pipeline peppered with explicit `lift` of single-layer fragments is the rejected form тАФ it re-describes a lift the overload already performs.

[CAPABILITY_CONSTRAINT_OVER_CONCRETE_STACK]:
- The `Stateful<M,S>` and `Readable<M,Env>` trait modules expose `get`/`gets`/`getsM`/`put`/`modify`/`modifyM`/`state`/`local` and `ask`/`asks`/`asksM`/`local`/`with` as carrier-polymorphic free functions returning `K<M,A>`, constrained only on the capability interface, never on the concrete transformer. An algorithm bodied against `where M : Stateful<M,S>, Readable<M,Env>, MonadUnliftIO<M>, Monad<M>` reads state, reads environment, and retries effects without naming `ReaderT`, `StateT`, or their nesting order. The concrete stack is chosen once at the run edge; the algorithm is invariant under reordering as long as the capability set is satisfiable.
- This is the polymorphic-collapse terminal form for stacked effects: instead of one algorithm per concrete stack (`ReaderT<Env, StateT<S, Eff>>`, `StateT<S, ReaderT<Env, Eff>>`, тАж), one body quantified over the capability conjunction. The cost surfaces as the capability law above тАФ `MonadUnliftIO` in the constraint set is unsatisfiable by a stack whose IO-capable layer is buried under a state-shaped layer, so the constraint that compiles also asserts a legal ordering. The constraint set is simultaneously the abstraction and the stack-order proof obligation.

[STACK_LOCAL_PROJECTION_COMBINATORS]:
- `ReaderT<Env,M>.With<Env1>(Func<Env1,Env>)` returns `ReaderT<Env1,M,A>` тАФ a *covariant environment narrowing* that changes the reader's environment type, distinct from `Local(Func<Env,Env>)` which preserves the type and rewrites the value. `With` is `new(env1 => runReader(f(env1)))`: it embeds an inner reader expecting a sub-environment into an outer reader carrying a richer one, with a projection function and no capability interface. A sub-pipeline written against a narrow `Env1` composes into a wide-`Env` parent structurally тАФ the wide-to-narrow projection is the whole seam, no `Has`-style marker interface required.
- `MapM` and `MapT` are the two orthogonal hoists. `MapM<B>(Func<K<M,A>, K<M,B>>)` transforms the inner monad's *bound value* while preserving the transformer layer and the monad type; `MapT<M1,B>(Func<K<M,тАж>, K<M1,тАж>>)` transforms the *monad type itself* (`M` to `M1`) while preserving the transformer layer. `StateT.MapT` constrains `M1 : Monad<M1>, Choice<M1>` and `ReaderT.MapM` constrains `M1 : Monad<M1>, Alternative<M1>` тАФ the target carrier must supply the alternative/choice structure the hoist relies on. `MapT` is the mechanism for swapping the effect base under a fixed reader/state layer without unwinding the stack; `MapM` is the mechanism for transforming the carried effect without touching either the layer or the base type.

```csharp
static K<M, Report> Pipeline<M>(Input raw)
    where M : Stateful<M, Budget>, Readable<M, Config>, MonadUnliftIO<M>, Monad<M> =>
    from cfg in Readable.asks<M, Config, Threshold>(static c => c.Limit)
    from spent in Stateful.get<M, Budget>()
    from _     in Stateful.modify<M, Budget>(b => b.Charge(raw.Cost))
    from step  in MonadUnliftIO.mapIO<M, Raw, Raw>(
                      io => io.Timeout(TimeSpan.FromSeconds(2)).Retry(Backoff),
                      Acquire(raw))            // Timeout+Retry only resolve while M stays IO-unliftable
    from done  in Readable.local<M, Config, Report>(
                      c => c.Sandboxed(),       // env override scoped to this sub-effect only
                      Finalize(step, cfg, spent))
    select done;
```
