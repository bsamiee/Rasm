# Wire-Contracts: Core Findings

[SOURCE_GEN_CONTRACT_COVERAGE]:
- `[JsonSerializable(typeof(T), GenerationMode = JsonSourceGenerationMode.Metadata)]` on a `JsonSerializerContext` subclass generates compile-time `JsonTypeInfo<T>` instances; all `JsonSerializer` overloads accept `JsonTypeInfo<T>` directly, eliminating every reflection call at the serialization boundary. `Metadata` mode generates full contract metadata — property shapes, constructor resolver, polymorphism — but defers fast-path serialization; `Serialization` mode generates fast-path write code at the cost of `Populate`-mode deserialization support. Only `Default` (both) gives both properties simultaneously.
- `[JsonSourceGenerationOptions]` on the context mirrors every `JsonSerializerOptions` property that affects code generation. Setting `RespectNullableAnnotations = true` and `RespectRequiredConstructorParameters = true` at this attribute propagates those constraints into generated code, enforcing nullability and required-field law at compile time rather than in ad-hoc post-deserialization guards.
- Types not covered by any `[JsonSerializable]` declaration require a `JsonConverter<T>` registered via `JsonSerializerOptions.Converters` or a custom `IJsonTypeInfoResolver` returned by `JsonTypeInfoResolver.Combine(...)` and added to `JsonSerializerOptions.TypeInfoResolverChain`. The chain is evaluated left-to-right; the first resolver returning non-null wins. This is the insertion point for attributed payload records whose types are assembled at runtime or come from assemblies not under the generator's sight.

[RESOLVER_CHAIN_AND_MODIFIERS]:
- `JsonTypeInfoResolver.WithAddedModifier(IJsonTypeInfoResolver, Action<JsonTypeInfo>)` wraps a resolver and post-processes every `JsonTypeInfo` it produces. Modifiers are the correct attachment point for cross-cutting wire-contract policy — renaming properties for a protocol version, injecting redaction converters on sensitive fields via `JsonPropertyInfo.CustomConverter`, or forcing `IsRequired = true` on envelope fields — without coupling that policy to any domain type. Chaining `WithAddedModifier` calls stacks the `Action<JsonTypeInfo>` invocations in declaration order.
- `JsonSerializerOptions.TypeInfoResolverChain` exposes the live resolver list as a mutable `IList<IJsonTypeInfoResolver>`. Inserting a source-generated context at position 0 and a `DefaultJsonTypeInfoResolver` with modifiers at position 1 gives source-gen coverage for known types with modifier-shaped reflection fallback for unknowns, without splitting options instances per call site.
- `JsonTypeInfo.CreateJsonTypeInfo<T>(JsonSerializerOptions)` and `JsonTypeInfo.CreateJsonPropertyInfo(Type, string)` build contract metadata programmatically for types the source generator cannot see. The resulting `JsonTypeInfo` is mutable until `MakeReadOnly()` is called; properties added via `Properties.Add(...)` participate in all subsequent serialization exactly as generated ones do. This is the sanctioned path for attributed payload records covering types from foreign assemblies or dynamic shapes.

[UNMAPPED_MEMBERS_AND_CONTRACT_STRICTNESS]:
- `JsonSerializerOptions.UnmappedMemberHandling = JsonUnmappedMemberHandling.Disallow` causes deserialization to throw on any JSON property without a corresponding .NET member. At the type level, `[JsonUnmappedMemberHandling(JsonUnmappedMemberHandling.Disallow)]` overrides the global setting. The correct wire-contract posture for versioned envelopes at an admission boundary is `Disallow` on the inner envelope type — detecting schema drift as a failure — and `Skip` on outer wrapper or extension bags — tolerating forward-compatible noise.
- `JsonSerializerOptions.AllowDuplicateProperties` controls whether duplicate JSON keys throw or silently take last-write-wins. Set it explicitly to `false` on any `JsonSerializerOptions` used at admission to reject malformed input that a lenient parser would silently corrupt.
- `JsonSerializerOptions.AllowOutOfOrderMetadataProperties` relaxes the constraint that `$type`, `$id`, and `$values` metadata properties must precede regular properties. Set it only when the remote protocol requires it; leave it `false` at owned boundaries.
- `JsonIgnoreCondition.WhenReading` ignores a property only during deserialization, not serialization. Use it on write-only computed fields that must appear in outbound payloads but must not be fed back into the domain on inbound deserialization — a precise asymmetric contract shape.

[POLYMORPHIC_WIRE_CONTRACTS]:
- `[JsonPolymorphic(TypeDiscriminatorPropertyName = "$kind", IgnoreUnrecognizedTypeDiscriminators = false)]` with explicit `[JsonDerivedType(typeof(CaseA), "a")]` annotations is the attributed-record form of a closed-hierarchy wire contract. The discriminator property name must be protocol-stable across versions — changing it is a wire-breaking change with no migration path in `JsonPolymorphicAttribute`. The domain `[Union]` hierarchy and the wire hierarchy must stay structurally parallel but decoupled: a `JsonConverter<TDomain>` at the boundary translates between the domain union and its attributed wire projection, so neither carries the other's schema.
- `JsonPolymorphicAttribute.UnknownDerivedTypeHandling = JsonUnknownDerivedTypeHandling.FailSerialization` is the safe default: attempting to serialize an undeclared runtime subtype throws, preventing silent data loss. At the admission boundary, `IgnoreUnrecognizedTypeDiscriminators = false` (the default) causes deserialization of an unknown discriminator to throw. Both behaviors make schema-contract drift visible immediately; set `FallBackToBaseType` or `IgnoreUnrecognizedTypeDiscriminators = true` only when the protocol explicitly specifies forward-compatible fallback semantics.
- When `JsonPolymorphicAttribute` and `JsonTypeInfoResolver.WithAddedModifier` are combined, the modifier can mutate `JsonTypeInfo.PolymorphismOptions.DerivedTypes` at runtime to register additional cases discovered from probe receipts or capability metadata, without touching any domain type. This is the extension point for capability-gated wire cases that must not appear in the base contract.

[STREAMING_AND_PIPELINE_SEAM]:
- `JsonSerializer.DeserializeAsyncEnumerable<T>(PipeReader, JsonTypeInfo<T>, bool topLevelValues, CancellationToken)` is the sanctioned streaming-deserialization path over `System.IO.Pipelines`. When `topLevelValues = true`, it deserializes a sequence of whitespace-separated top-level JSON values (NDJSON semantics); when `false`, it deserializes elements of a root-level JSON array. The `JsonTypeInfo<T>` overload is AoT-safe; the `JsonSerializerOptions` overload is not. Wire seams that stream event sequences use the `topLevelValues = true` path with a source-generated `JsonTypeInfo<T>` to stay AoT-clean throughout.
- `ChannelReader<T>.ReadAllAsync(CancellationToken)` returns `IAsyncEnumerable<T>`, composing directly with `await foreach` and async LINQ combinators. A `Channel<TWire, TDomain>` shape — where the write side accepts deserialized wire messages and the read side emits projected domain events — is the canonical seam marker: it decouples the network-read loop from the domain consumer, absorbs backpressure via `BoundedChannelOptions`, and makes the concurrency contract explicit in the type.
- `Channel.CreateBounded<T>(BoundedChannelOptions, Action<T> itemDropped)` surfaces a drop-notification callback when `FullMode` is `DropNewest`, `DropOldest`, or `DropWrite`. The callback is the correct site for emitting a drop-receipt into a diagnostic channel — a typed evidence record with the dropped item's identity, timestamp, and drop reason — keeping the channel itself stateless and the observability surface separate.

[GENERATED_CONVERTERS_FOR_DOMAIN_VOCABULARY]:
- Thinktecture-generated `[SmartEnum<TKey>]` and `[ValueObject<T>]` types produce `JsonConverter<T>` implementations controlled by `SerializationFrameworks` on the declaring attribute. Setting `SerializationFrameworks = SerializationFrameworks.SystemTextJson` generates a converter registered via the generated `JsonSerializerContext`'s internal `Converters` list. The generated converter reads and writes only the key field; no domain behavior leaks into the wire shape. `DisableSpanBasedJsonConversion = false` on the `SmartEnum` metadata preserves span-based zero-allocation reads for throughput-critical boundaries.
- `ObjectFactoryAttribute.UseForSerialization` on a factory method on a `[ValueObject]` or `[SmartEnum]` type directs the generator to use that factory as the `Read` implementation of the generated converter. This makes admission logic — range checks, sentinel rejection, context validation — part of the deserialization path rather than a separate post-deserialization step, keeping the factory as the single admission point for both the runtime and wire code paths.

[TEMPORAL_VOCABULARY_AT_BOUNDARY]:
- `TimeProvider.GetUtcNow()` returns `DateTimeOffset` with `.Offset == TimeSpan.Zero`; `GetLocalNow()` returns local time using `TimeProvider.LocalTimeZone`. Timestamps in persisted or transmitted wire envelopes must use `DateTimeOffset` — never `DateTime` — with the UTC offset made explicit. `DateTime.MinValue` and `DateTimeOffset.MinValue` must not appear in any serialized field; the sentinel projection at the host boundary converts them to `None` before they reach any serialized shape. Absent timestamps are represented by the absence of the property via `[JsonIgnore(Condition = JsonIgnoreCondition.WhenWritingNull)]` on a `DateTimeOffset?` field, never by `MinValue`.
- `TimeProvider` is not itself serializable; it is a service injected at composition time. The boundary projects `GetUtcNow()` into an attributed record field at the moment of capture. The receipt field name, format (ISO 8601 via `DateTimeOffset`), and JSON property name are fixed at definition time and never vary across codec versions.

[FAILURE_CAPTURE_AT_MARSHAL_BOUNDARY]:
- `IO<A>.Catch(Func<Error, bool>, Func<Error, K<IO, A>>)` is the composable predicate-based error handler. At a wire marshal boundary — where network I/O, codec throws, and timeout exceptions arrive as untyped `Exception` — the pattern is `liftIO(Func<Task<A>>)` (wrapping `Exception` into `Error.IsExceptional`) followed by `.Catch(e => e.IsExceptional, e => IO.Fail<A>(translateToExpected(e)))` to project exceptional errors into typed expected errors before they escape the seam. The result travels the `Fin<A>` rail inward; the raw `Exception` does not.
- `Error.New(string, Exception)` creates an expected error carrying the original exception as context with an overridden message. The exception is preserved for diagnostic extraction via `Error.HasException<T>()` and `Error.Inner` but the public message is sanitized before any error crosses an export boundary. `Error.Many(Seq<Error>)` aggregates multiple field-level failures into one composite error at the validation layer, distinct from the single boundary-crossing error that wraps an I/O failure.

[WIRE_CONTRACT_STABILITY_AS_BREAKING_CHANGE_CONTRACT]:
- `JsonTypeInfo<T>` is the cross-layer serialization contract surface. A change to a property's `JsonPropertyInfo.Name`, `IsRequired`, `CustomConverter`, or its omission from `Properties` is a wire-breaking change regardless of whether the domain type changes. Receipt shapes — typed records carrying correlation, timestamp, discriminant, and policy fields — must treat their `JsonPropertyInfo` configuration as a versioned contract. Adding a new property with `[JsonIgnore(Condition = JsonIgnoreCondition.WhenWritingDefault)]` is forward-compatible; removing a property or changing its JSON name is not.
- Snapshot envelopes encode the schema version field as a `[SmartEnum<int>]` or `[SmartEnum<string>]` value object (not a raw integer) so that the version token participates in the same generated converter path as any other keyed domain value, and unknown version values are rejected at the converter boundary rather than silently propagated.
- `JsonSerializerOptions.MakeReadOnly()` (no-arg, requiring an explicit resolver) and `MakeReadOnly(populateMissingResolver: true)` (falls back to reflection) seal the options instance, preventing mutation after first use. A shared static `JsonSerializerOptions` for a wire lane must be sealed at initialization time; options that are shared but mutable cause non-deterministic behavior when modifiers are added concurrently. The canonical form is a `static readonly` field with `MakeReadOnly()` called in the field initializer expression.
