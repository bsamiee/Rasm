# Type-Level Static-Abstract Dispatch Over The Generated Owner Family

[CRTP_CONSTRAINT_IS_THE_OPEN_DISPATCH_OVER_A_CLOSED_FAMILY]:
- Every generated keyed owner implements `IObjectFactory<TOwner, TValue, TError>` whose sole member is `static abstract TError? Validate(TValue?, IFormatProvider?, out TOwner? item)`, self-constrained by `where TOwner : IObjectFactory<TOwner, TValue, TError>`. The recursive self-bound is the mechanism that turns a static-virtual member into a dispatch target a generic caller can invoke: a method `M<TOwner, TValue, TError>(...) where TOwner : IObjectFactory<TOwner, TValue, TError>` calls `TOwner.Validate(...)` and the JIT resolves the call to the concrete owner's generated body with no instance, no virtual table, no reflection. This is the inverted form of the case-level `Switch`: `Switch` is closed over the cases of one owner; the constrained generic is open over the unbounded family of all owners and closed at each instantiation. One boundary entrypoint thereby admits every present and future generated owner without an arm per owner.
- The admission outcome is a nullable `TError?`, not a thrown exception and not a `bool`: a non-null error is the rejection, a null error with a non-null `out item` is the success. The two-channel return is the type-level analogue of a `Validation<E,T>` cell that a single rail bridge lifts once, so the generic admission point produces a rail cell uniformly for every owner the constraint admits, and the rail choice — `Fin` for abort-first, accumulating `Validation` for independent owners combined under an applicative join — is made once at the bridge, never per owner. The error carries no message getter on the constraint itself; the generic bridge reaches the text through the owner-chosen `TError`'s own surface, which is why the bridge is parameterized on `TError` rather than on a fixed error class.
- `IValidationError<TError>` carries its own `static abstract TError Create(string message)`, so the error type is itself a constrained static-abstract surface. A generic admission helper that must synthesize an error — a wrapping codec, a boundary that adds context — produces it through `TError.Create(...)` rather than constructing a fixed error class, keeping the error vocabulary owner-chosen while the helper stays owner-agnostic. The triad `IObjectFactory` / `IValidationError` / the constrained helper is a closed contract set with no concrete type named anywhere in the boundary code.

```csharp
// One boundary admits every generated owner through the static-virtual Validate;
// TError.Create keeps the failure vocabulary owner-defined while the bridge stays generic.
// allows ref struct on TValue threads a ReadOnlySpan<char> key with no string materialization.
static Fin<TOwner> Admit<TOwner, TValue, TError>(TValue value, IFormatProvider? culture = null)
    where TOwner : IObjectFactory<TOwner, TValue, TError>
    where TValue : notnull, allows ref struct
    where TError : class, IValidationError<TError> =>
    TOwner.Validate(value, culture, out var item) is { } error
        ? FinFail<TOwner>(Error.New(error.ToString() ?? "rejected"))
        : FinSucc(item!);
```

[SPAN_KEYED_ADMISSION_IS_A_SEPARATE_STATIC_OVERLOAD_NOT_A_CONVERSION]:
- A dedicated generic helper `Validate<TOwner, TError>(ReadOnlySpan<char> key, IFormatProvider?, out TOwner?)` exists with `where TOwner : IObjectFactory<TOwner, ReadOnlySpan<char>, TError>` — the span is the `TValue` of the constraint, not a string coerced at the edge. Because the constraint's `TValue` carries `allows ref struct`, a `ReadOnlySpan<char>` is a first-class admission key, and the owner generates a `Validate(ReadOnlySpan<char>, ...)` body whose parse consumes the span directly. Protocol text decodes into the closed owner vocabulary with zero heap traffic, and the generic entrypoint is `AggressiveInlining`-marked so the static-virtual call plus the parse fold into the caller. The string-keyed and span-keyed admissions are two constraint instantiations of one contract, never a `ToString` bridge between them.
- The same `allows ref struct` posture propagates to `IObjectFactory<TValue>` (the non-generic marker base) and to `IConvertible<T>.ToValue()`, so a ref-struct key admits inbound and a ref-struct projection leaves outbound through the same family without ever boxing into `object`. This is the type-system statement that the boundary is allocation-transparent end to end: the discriminant is the static type of the key, and the key may be a stack-only span on both legs.

[CONVERTIBLE_IS_THE_OUTBOUND_STATIC_PROJECTION_MIRRORING_VALIDATE]:
- `IKeyedObject<TKey> : IConvertible<TKey>` gives every keyed owner an instance `TKey ToValue()` — the outbound projection that mirrors inbound `Validate`. The pair is the complete boundary codec expressed as two constrained surfaces: admit through the owner's static `Validate`, project through the owner's instance `ToValue`, and no per-owner adapter sits between the wire and the domain. A generic serializer constrained on both interfaces round-trips any owner: read raw → `TOwner.Validate` → domain value → mutate in the domain → `value.ToValue()` → raw, and the serializer names no concrete owner.
- The generic `TypeConverter` shipped over this contract is the canonical proof that one parameterized type serves the whole family: its `ConvertFrom` body is `TOwner.Validate(raw, culture, out var item)` and its `ConvertTo` body is `val.ToValue()` (or the key's own converter applied to `val.ToValue()` when the destination is foreign). The converter is instantiated once per owner by the generator but written once against the constraints — a single deep surface owning the conversion concern for an unbounded owner set, which is the inverse of one shallow converter class per owner.

[COMPARER_ACCESSORS_BIND_DISPATCH_EQUALITY_AT_THE_TYPE_LEVEL]:
- `IEqualityComparerAccessor<TKey>` and `IComparerAccessor<TKey>` expose `static abstract IEqualityComparer<TKey> EqualityComparer` and `static abstract IComparer<TKey> Comparer`; `[KeyMemberEqualityComparer<TAccessor, TKey>]` and `[KeyMemberComparer<TAccessor, TKey>]` bind an owner's key semantics to one accessor at definition time. The consequence for dispatch tables is structural: a frozen table keyed by the owner, a `HashSet` of owners, and the owner's own `Equals` all resolve their comparer through `TAccessor.EqualityComparer`, so case-insensitive or culture-specific key dispatch is correct by the accessor choice rather than by remembering to pass a comparer at each table construction. A generic dispatch index `Build<TOwner, TKey, TAccessor>(...) where TAccessor : IEqualityComparerAccessor<TKey>` reads `TAccessor.EqualityComparer` to construct a frozen table whose comparer provably matches the owner's equality — the single source the owner already obeys.
- The accessor being a static-abstract member, not a constructor argument or an instance field, is what makes the comparer a compile-time-resolved constant per owner: the frozen table's strategy analysis sees a known comparer at construction and selects its internal layout accordingly, and no call site can supply a divergent comparer that would silently corrupt lookup.

[CONVERSION_POSTURE_DECIDES_WHETHER_OWNER_KEYED_AND_RAW_KEYED_TABLES_INTEROPERATE]:
- `ConversionToKeyMemberType` and `ConversionFromKeyMemberType` independently select `None`/`Implicit`/`Explicit`. `Implicit` out is what lets an owner be passed where the raw key is expected — a `FrozenDictionary<TKey, V>` probe, a span-lookup argument — with no `.ToValue()` ceremony, so owner-valued code reads against a raw-key table directly. `Explicit` in forces every inbound raw key through a visible cast, so admission is never silent. `None` on both severs the owner from its key and routes all construction through the validated factory, the strictest posture; it also makes the owner unusable as a raw-key-table key, which is the deliberate statement that this owner never participates in raw-key dispatch.
- `OperatorsGeneration.DefaultWithKeyTypeOverloads` emits relational and equality operators in mixed owner-vs-key arity, so a range guard or relational dispatch arm compares an owner against a raw key with no conversion at the comparison site. The operator posture and the comparer accessor must agree on the same `TKey` equality and ordering for owner-keyed and raw-key-keyed tables to join correctly; the accessor is the single arbiter that guarantees it.

[FACTORY_METHOD_GENERATION_IS_AN_EXPLICIT_OVERRIDE_OF_THE_CONSTRUCTION_TRIGGER]:
- `FactoryMethodGeneration` has three states: `Default` auto-generates factory methods only when a member triggers it (a type-parameter member, an interface member, an `object` member, a duplicate simple type), `None` suppresses factory generation even when a trigger is present, `Always` forces the factory surface uniformly regardless of triggers. The override matters for dispatch uniformity: `Always` makes every member of an ad-hoc union construct through a named factory method, eliminating the mixed construction surface where some members convert implicitly and others need a factory — a uniform construction vocabulary is itself dispatchable, where a mixed one is not. `None` is the statement that the owner is constructed only through the validated `Validate` path, never through a bare factory, the tightest admission posture for a security-sensitive or invariant-heavy owner.
- The same axis on a multi-member ad-hoc union resolves the rule that any single triggering member forces factory-only construction for all members: setting `Always` makes the intent explicit at the declaration rather than leaving it as an emergent consequence of one member's shape, so adding a triggering member later does not silently flip the whole union's construction surface.

[METADATA_OWNER_IS_A_GENERATED_DISPATCHABLE_DESCRIPTION_OF_THE_CONSTRUCTION_SURFACE]:
- `IObjectFactoryOwner` exposes `static abstract IReadOnlyList<ObjectFactoryMetadata> ObjectFactories`, and `IMetadataOwner` exposes `static abstract Metadata Metadata` where `Metadata` is itself a closed `[Union]` over the owner kinds. A boundary that must enumerate every admission front door of an unknown owner — a generic codec choosing which value type to deserialize from, a generic binder discovering the bindable key type — reads the static-abstract metadata list and dispatches over it with the metadata union's own generated `Switch`, never reflecting over constructors. The construction surface is thus reified as a statically-typed, dispatchable vocabulary: the set of `[ObjectFactory<TValue>]`-declared admission paths becomes data a generic boundary folds over, and each path's boundary-routing flags (`UseForSerialization`, `UseWithEntityFramework`, `UseForModelBinding`, `HasCorrespondingConstructor`) are fields the boundary reads to pick the path matching its own concern.
- Multiple `[ObjectFactory<TValue>]` on one owner is arity-and-shape collapse applied to construction: several `FromString`/`FromInt`/`FromTuple` factories become one attribute-directed family of static-abstract `Validate` overloads keyed on the input's static type. A generic admission helper need not know which value types an owner admits — it enumerates `ObjectFactories`, matches its raw input's type to a metadata entry, and invokes that path. The discriminant is the metadata-described value type, and no mode flag re-describes the input.

[STATIC_VIRTUAL_DISPATCH_DOES_NOT_COMPOSE_THROUGH_EXPRESSION_TREES_OR_OPEN_DELEGATES]:
- A static-abstract interface member resolved through a generic constraint has no instance receiver and no addressable method until the type parameter is substituted, so it cannot be captured as a method group into an open `Func<...>`, cannot be referenced inside an `Expression<Func<...>>`, and cannot be the target of a runtime-built delegate over an unresolved `TOwner`. A dispatch architecture that admits owners generically and then needs the admission as a first-class delegate — a registry of admit functions keyed by type, a deferred predicate fed to a query translator — must close the generic at a concrete owner first (`Admit<ConcreteOwner, …>`) and capture that closed instantiation, or fall back to an instance-level surface. The constraint is the cost of static-virtual resolution: maximal inlining and zero allocation in the closed-at-compile-time path, no first-class function in the open path.
- The boundary between the two is the integration pressure: type-level static-abstract dispatch is the form for boundaries whose owner set is resolved at compile time at the call site (a typed deserializer field, a strongly-typed binder), and instance-level or metadata-list dispatch is the form for boundaries whose owner is discovered at runtime (a polymorphic codec over a heterogeneous payload). Choosing the static-virtual form where the owner is runtime-discovered forces a per-type closed generic method that reflection must invoke — reintroducing the reflection the contract was meant to remove. The metadata-owner surface exists precisely to keep the runtime-discovered case reflection-free by dispatching over the generated `Metadata` union instead.
