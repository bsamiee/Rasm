# BCL System-Seam Replacements

[HANDLE-BASED FILE IO]:
- `File.OpenHandle(path, mode, access, share, options, preallocationSize)` returns `SafeFileHandle` without constructing a `FileStream`; all subsequent IO goes through `RandomAccess` static methods against the handle. The handle carries `IsAsync` state set at open time, which determines whether async scatter/gather overlaps are kernel-dispatched or thread-pool-emulated — opening with `FileOptions.Asynchronous` is the only way to guarantee true OS-async on Windows and io_uring dispatch on Linux.
- `RandomAccess` is a fully offset-addressed, stateless surface: `Read(handle, Span<byte>, long fileOffset)` / `Write(handle, ReadOnlySpan<byte>, long fileOffset)` for single-buffer, `Read(handle, IReadOnlyList<Memory<byte>>, long)` / `Write(handle, IReadOnlyList<ReadOnlyMemory<byte>>, long)` for vectored scatter/gather. Async variants return `ValueTask<int>` or `ValueTask<long>`. Position state is entirely caller-owned — no seek, no lock, safe for concurrent readers on distinct offsets.
- `RandomAccess.GetLength(handle)` and `SetLength(handle, long)` / `FlushToDisk(handle)` complete the handle surface. Reject any shape that opens a `FileStream` solely to call `.Length`, `.SetLength()`, or `.Flush(true)`.
- `FileStreamOptions` (options object ctor) is the only non-obsolete way to construct a `FileStream` when you must use a stream interface. It consolidates `mode`, `access`, `share`, `options`, `bufferSize`, and `preallocationSize`. The old `FileStream(path, mode, access, share, bufferSize, useAsync)` overload chain is the rejected form.
- `MemoryMarshal.Cast<TFrom, TTo>(Span<TFrom>)` reinterprets the binary content of a span in place — zero copy, no new allocation. The complementary `MemoryMarshal.Read<T>(ReadOnlySpan<byte>)` and `TryRead<T>` read unmanaged blobs as structs, and `Write<T>` / `TryWrite<T>` serialize structs to byte spans — eliminating `BinaryReader`/`BinaryWriter` for fixed-layout binary formats.

[PATH ARITHMETIC]:
- `Path.Join(ReadOnlySpan<char>, ReadOnlySpan<char>[, ...])` is the allocation-minimizing alternative to `Path.Combine`; unlike `Combine`, it never discards left segments when a right segment looks rooted, which is usually the correct semantic for programmatic path construction. Four span overloads exist; `params ReadOnlySpan<string?>` is the variadic form.
- `Path.Exists(string?)` unifies `File.Exists` and `Directory.Exists` into one predicate; it returns `true` for any filesystem entry (file, directory, symlink target). The prior bifurcated check is the smell being replaced.
- `Path.GetRelativePath(relativeTo, path)` computes the relative path string; useful for logging and serialization without string slicing. `Path.IsPathFullyQualified`, `EndsInDirectorySeparator`, `TrimEndingDirectorySeparator` round out the non-allocating path predicate family — all available on `ReadOnlySpan<char>` overloads, avoiding round-trips through string.
- `Path.GetDirectoryName(ReadOnlySpan<char>)`, `GetFileName`, `GetExtension`, `GetFileNameWithoutExtension`, `GetPathRoot` all accept and return `ReadOnlySpan<char>`, slicing the input with no heap allocation. The `string?` overloads exist for boundary cases only.
- `File.ReadLinesAsync(path, CancellationToken)` returns `IAsyncEnumerable<string>` — integrates directly into `await foreach` and `AsyncEnumerable` LINQ without intermediate `Task<string[]>` buffering. Reject patterns that `ReadAllLines` then iterate or that open a `StreamReader` to read lines.
- `File.AppendAllBytes(string, ReadOnlySpan<byte>)` and `AppendAllBytesAsync(string, ReadOnlyMemory<byte>, CancellationToken)` are new additions covering the binary-append use case without stream construction.

[ONE-SHOT CRYPTOGRAPHY]:
- `CryptographicOperations.HashData(HashAlgorithmName, ReadOnlySpan<byte>, Span<byte>)` dispatches to any registered hash algorithm by name with a single call that allocates nothing. The `TryHashData` variant writes into a pre-sized span. `HmacData` and `TryHmacData` follow the identical shape with an additional key span. Reject patterns that construct `HMACSHA256`/`SHA256` instances, call `ComputeHash`, and discard the instance.
- `SHA256.HashData`, `SHA512.HashData`, `SHA3_256.HashData`, `SHA3_512.HashData` exist as type-specific one-shots. `SHA3_256.IsSupported` / `SHA3_512.IsSupported` / `SHAKE128.IsSupported` are static properties that reflect OS/hardware capability at runtime; absence of a guard before use is a latent `PlatformNotSupportedException`.
- `HashDataAsync(HashAlgorithmName, Stream, Memory<byte>, CancellationToken)` returning `ValueTask<int>` and the `Task<byte[]>` variant on each concrete type handle streaming hash without ever exposing `ICryptoTransform`. This is the one-shot async path for streamed content.
- `CryptographicOperations.FixedTimeEquals(ReadOnlySpan<byte>, ReadOnlySpan<byte>)` is the only correct constant-time comparison for secrets; a manual loop, `SequenceEqual`, or `==` is a timing oracle regardless of branch-prediction behavior.
- `RandomNumberGenerator.Fill(Span<byte>)` fills a caller-owned buffer without allocation. `GetBytes(int count)` allocates. `GetInt32(fromInclusive, toExclusive)` produces a uniformly distributed random integer using rejection sampling internally. `GetItems<T>(ReadOnlySpan<T>, Span<T>)` samples with replacement and `Shuffle<T>(Span<T>)` shuffles in place — both cryptographically uniform.
- `AesGcm` now requires a `tagSizeInBytes` argument in construction — the old constructor that omits it is gone. Nonce sizes are constrained to `KeySizes NonceByteSizes`; tag sizes constrained to `TagByteSizes`. `ChaCha20Poly1305` fixes nonce at 12 bytes and tag at 16.
- `Base64Url` (`System.Buffers.Text`) encodes/decodes URL-safe base64 directly between `ReadOnlySpan<byte>` ↔ `Span<byte>` and `ReadOnlySpan<char>` ↔ `Span<byte>` without any padding ambiguity. `IsValid(ReadOnlySpan<byte>)` / `IsValid(ReadOnlySpan<char>)` check well-formedness with an optional `out int decodedLength` variant. Reject any hand-rolled base64url mapper or `Convert.ToBase64String` + string-replace chain.

[GENERATED REGEX]:
- `[GeneratedRegex(pattern, options, cultureName)]` targets `static partial` methods returning `Regex` or `partial` properties of `Regex` type (`AttributeTargets.Method | AttributeTargets.Property`). The generator emits a `Regex`-derived type with state-machine match logic inlined as compiled C# — no `Regex` constructor call, no `RegexOptions.Compiled` JIT cost, no per-call lock on the cache.
- `Regex.EnumerateMatches(ReadOnlySpan<char>, pattern)` returns a `ValueMatchEnumerator` (ref struct, stack-allocated) that enumerates `ValueMatch` structs carrying `Index` and `Length` — no `Match` heap allocation, no `MatchCollection`. This is the correct path for scan-and-count or scan-and-slice workflows on spans.
- `Regex.Count(ReadOnlySpan<char>, pattern)` is the terminal aggregation; `IsMatch(ReadOnlySpan<char>, pattern)` is the presence test. Both accept `ReadOnlySpan<char>` directly, removing the `input.AsSpan()` → `string` round-trip.
- `[StringSyntax("Regex")]` marks `string` parameters with the regex syntax identifier; IDEs provide colorization and analysis without any runtime cost. Use it on any method that accepts a pattern literal.
- The `CompositeFormat` class (`System.Text`) pre-parses `{0}` / `{1:format}` templates once at construction: `CompositeFormat.Parse(template)` returns an instance; `string.Format(provider, compositeFormat, args)` accepts it. `MinimumArgumentCount` is inspectable. This replaces runtime `string.Format(string, ...)` when the template is fixed but arguments vary, separating parse-time failure from call-time failure.

[UTF-8 TEXT PIPELINES]:
- `"literal"u8` suffix creates a `ReadOnlySpan<byte>` constant in the assembly's read-only data segment with no runtime allocation and no encoding call. The span is UTF-8 encoded at compile time. This is the correct origin for all fixed UTF-8 payloads — reject `Encoding.UTF8.GetBytes("...")` for string literals.
- `Utf8.TryWrite(Span<byte> destination, ref TryWriteInterpolatedStringHandler handler, out int bytesWritten)` formats an interpolated string directly as UTF-8 bytes with no intermediate string allocation. The handler type is compiler-lowered from `$"..."` syntax. The overload accepting `IFormatProvider` forwards culture context into each `IUtf8SpanFormattable` hole.
- `Utf8.FromUtf16(ReadOnlySpan<char>, Span<byte>, out int charsRead, out int bytesWritten, bool replaceInvalidSequences, bool isFinalBlock)` returns `OperationStatus` for chunked conversion; `ToUtf16` is the reverse. These are the correct primitives for protocol transcoding — reject `Encoding.UTF8.GetBytes(chars)` for non-trivial volumes.
- `Encoding.TryGetBytes(ReadOnlySpan<char>, Span<byte>, out int bytesWritten)` and `TryGetChars(ReadOnlySpan<byte>, Span<char>, out int charsWritten)` are the non-throwing span variants on all `Encoding` instances. Prefer over `GetBytes(ReadOnlySpan<char>, Span<byte>)` when destination sizing is uncertain; they return false rather than throwing `ArgumentException`.
- `IUtf8SpanFormattable.TryFormat(Span<byte>, out int bytesWritten, ReadOnlySpan<char> format, IFormatProvider?)` is the UTF-8 output interface all numeric primitives implement (via `INumber<T>` hierarchy). `IUtf8SpanParsable<TSelf>.Parse(ReadOnlySpan<byte>, IFormatProvider?)` / `TryParse` is the symmetric input interface. Generic code parameterized on `where T : IUtf8SpanFormattable` or `IUtf8SpanParsable<T>` handles any conforming type — all BCL numeric types, `Guid`, `DateTimeOffset`, `TimeSpan`, `Version`.

[GENERIC MATH AND PARSING INTERFACES]:
- `INumber<TSelf>` subsumes `ISpanParsable<TSelf>`, `ISpanFormattable`, `IUtf8SpanParsable<TSelf>`, `IUtf8SpanFormattable`, all arithmetic operator interfaces, and `IComparable<TSelf>` — any generic constrained to `INumber<T>` gets the full parse/format/arithmetic surface. There is no reason to constrain separately to `ISpanParsable<T>` if the domain also requires arithmetic.
- `INumberBase<TSelf>.CreateChecked<TOther>(TOther)`, `CreateSaturating<TOther>`, `CreateTruncating<TOther>` are `static virtual` default-interface methods for type-preserving numeric conversion — replacing per-type cast or `Convert.ToInt32(value)` chains. `CreateChecked` throws on overflow; `CreateSaturating` clamps; `CreateTruncating` wraps.
- `ISpanParsable<TSelf>.TryParse(ReadOnlySpan<char>, IFormatProvider?, out TSelf)` is the canonical parse entry for span input. Constraining a generic to `ISpanParsable<T>` replaces per-type `int.TryParse(span, ...)`, `double.TryParse(span, ...)`, etc. with one generic call.
- `IParsable<TSelf>` is the string-level variant; `IUtf8SpanParsable<TSelf>` is the UTF-8 byte level. The three interfaces form a hierarchy (`IUtf8SpanParsable` is independent, not derived from `ISpanParsable`), so UTF-8 byte paths require separate generic constraints.
- `MemoryExtensions.Split<T>(ReadOnlySpan<T>, T)` / `SplitAny<T>(ReadOnlySpan<T>, SearchValues<T>)` return `SpanSplitEnumerator<T>` — a ref struct enumerator yielding `Range` values. The int-returning `Split(ReadOnlySpan<char>, Span<Range>, char, StringSplitOptions)` overload writes ranges into a caller-supplied buffer and returns the count. Both are allocation-free tokenizers replacing `string.Split()` for parsing over spans.
- `SearchValues<T>.Create(params ReadOnlySpan<T>)` produces a pre-built membership set used by `ContainsAny`, `IndexOfAny`, `IndexOfAnyExcept`, `ContainsAnyExcept` — all on `ReadOnlySpan<T>`. `SearchValues<string>.Create(ReadOnlySpan<string>, StringComparison)` handles multi-string search. Construct once, store as `static readonly`; the internal representation selects the optimal search strategy (bitmap, SIMD Teddy, Aho-Corasick) based on the value set at construction time.

[CHANNELS AS PRODUCER-CONSUMER SEAM]:
- `Channel.CreateBounded<T>(BoundedChannelOptions)` / `CreateUnbounded<T>()` are the sole constructors; `BoundedChannelOptions` carries `Capacity` and `FullMode` (`Wait`, `DropNewest`, `DropOldest`, `DropWrite`). The `itemDropped: Action<T>` overload on `CreateBounded` receives dropped items for logging or metrics — the caller does not need to poll.
- `ChannelWriter<T>.TryWrite(T)` is synchronous and non-blocking; `WriteAsync(T, CancellationToken)` returns `ValueTask` and respects backpressure when the channel is full and `FullMode = Wait`. Calling `Complete(Exception?)` signals end-of-data; downstream `ReadAllAsync` terminates naturally.
- `ChannelReader<T>.ReadAllAsync(CancellationToken)` returns `IAsyncEnumerable<T>` — composable with `System.Linq.AsyncEnumerable` operators. `TryRead(out T)` is the non-blocking pull; `WaitToReadAsync` is the async sentinel. The channel is the correct BCL replacement for `BlockingCollection<T>` and for manual `Task`/`SemaphoreSlim` producer-consumer loops.

[LINQ ADDITIONS]:
- `Enumerable.CountBy<TSource, TKey>(source, keySelector, comparer?)` returns `IEnumerable<KeyValuePair<TKey, int>>` — a streaming frequency map without materializing a `Dictionary`. `AggregateBy<TSource, TKey, TAccumulate>(source, keySelector, seed|seedSelector, func, comparer?)` generalizes to arbitrary per-key accumulators. Reject `GroupBy(...).Select(g => (g.Key, g.Count()))` when these suffice.
- `Enumerable.Index<TSource>(source)` yields `(int Index, TSource Item)` tuples — the canonical replacement for manual counter variables in `foreach`.
- `Enumerable.LeftJoin` / `RightJoin` are new relational join overloads with nullable inner/outer result selectors; they replace the left-join pattern of `GroupJoin` + `SelectMany` + `DefaultIfEmpty`.
- `System.Linq.AsyncEnumerable` is now part of the BCL (no external package required); it provides the full synchronous LINQ surface transposed to `IAsyncEnumerable<T>` — `AggregateBy`, `CountBy`, `WhereAwait`, `SelectAwait`, `OrderByAwait`, etc. The two-callback overloads (`Func<T, CancellationToken, ValueTask<TResult>>`) are preferred over the simpler `Func<T, TResult>` overloads when the selector itself performs async IO.

[DICTIONARY ALTERNATE LOOKUP]:
- `Dictionary<TKey, TValue>.GetAlternateLookup<TAlternateKey>()` returns `AlternateLookup<TAlternateKey>` (a value type) when the dictionary's comparer implements `IAlternateEqualityComparer<TAlternateKey, TKey>`; otherwise throws. `TryGetAlternateLookup<TAlternateKey>(out AlternateLookup<TAlternateKey>)` is the safe non-throwing form.
- `AlternateLookup<TAlternateKey>` exposes `TryGetValue(TAlternateKey, out TValue)`, `TryGetValue(TAlternateKey, out TKey, out TValue)`, `ContainsKey`, `TryAdd`, `Remove` — all keyed by the alternate type. The `where TAlternateKey : allows ref struct` constraint means `ReadOnlySpan<char>` is a valid alternate key against a `Dictionary<string, T>` when the comparer is `StringComparer.Ordinal`-family; no string allocation occurs on lookup.
- `HashSet<T>` carries the same `GetAlternateLookup<TAlternate>` / `TryGetAlternateLookup` surface. `IAlternateEqualityComparer<TAlternate, T>` requires implementing `bool Equals(TAlternate, T)`, `int GetHashCode(TAlternate)`, and `T Create(TAlternate)` — the `Create` method enables `TryAdd` to materialize the canonical key type from an alternate.

[ARGUMENT VALIDATION]:
- `ArgumentNullException.ThrowIfNull(obj, paramName?)` and `ArgumentException.ThrowIfNullOrEmpty` / `ThrowIfNullOrWhiteSpace` carry `[CallerArgumentExpression]` on the first parameter, so the exception message names the expression automatically. `ObjectDisposedException.ThrowIf(bool, object|Type)` completes the pattern for disposed-state guards.
- `ArgumentOutOfRangeException.ThrowIfZero<T>`, `ThrowIfNegative<T>`, `ThrowIfNegativeOrZero<T>`, `ThrowIfGreaterThan<T>`, `ThrowIfGreaterThanOrEqual<T>`, `ThrowIfLessThan<T>`, `ThrowIfLessThanOrEqual<T>`, `ThrowIfEqual<T>`, `ThrowIfNotEqual<T>` — all generic, all `[CallerArgumentExpression]`-attributed. The numeric variants constrain `where T : INumberBase<T>`, making them applicable to all BCL numeric types and any `INumber<T>` implementation. Guard chains that construct these exceptions manually are the rejected form.
