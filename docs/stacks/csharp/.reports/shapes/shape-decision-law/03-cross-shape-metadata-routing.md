# Cross-Shape Metadata Routing

[SELF_DESCRIBING_CLOSURE]:
- The metadata family describes itself with the machinery it describes: the shape-taxonomy union and its keyed sub-union each publish a self-metadata entry enumerating their own case types, so the introspection layer is its own first instance and a fold over shape kinds is total against a compiler-checked case list — generic infrastructure that discriminates owner kind rides exhaustive dispatch whose arms are the taxonomy, and a new shape kind lands as a compile break at every metadata fold rather than a silently skipped instance.
- The keyed sub-union's two arms are deliberately asymmetric in nullability, and the asymmetry is the shape-level encoding of construction semantics: the vocabulary arm's from-key conversion delegate, conversion expression, and validating key extractor are non-nullable because a vocabulary always resolves a key to a member, while the value-object arm makes all three nullable because a factory-cut owner nulls them out. A fold reading the from-key route treats the vocabulary arm as total and the value-object arm as partial — "members are resolved, not reconstructed" versus "construction was cut" is readable from the arm's type shape alone, no flag.
- The value-object arm alone carries a constructor-bypass conversion expression for persistence materialization of already-admitted truth; the vocabulary arm has no such slot because its rehydration is always a validating lookup. Metadata consumers selecting a conversion route therefore filter on delegate presence, never on metadata kind — kind says what the owner is, presence says which routes survived its configuration.

[CONVERSION_ROUTE_FUSION]:
- The conversion-route record a binder or serializer reads is computed by fusing two independent sources — the keyed metadata's key-typed conversion and the owner's object-factory row list — and the fusion is conditional on type identity: the keyed conversion expressions fold in only when the key type equals the selected factory's value type; otherwise the route carries only the factory's own expressions. An owner whose alternate factory admits a different value type than its key yields a route that constructs through the alternate value, not the key — the binding shape follows the matched value type, never the declared key.
- Resolution composes in a fixed order: keyed lookup, the caller's metadata filter, a last-match fold over the factory rows under the caller's factory filter, then fallback to the key-based expression only when no row matched. The resolved record carries both a constructor-shaped and a method-shaped lambda so a translator chooses the form its target understands — compiled constructor expression where a method call is opaque to query translation, delegate for in-memory — from one resolved value.

[DISCOVERY_CONTRACT]:
- Runtime resolution finds the static metadata property by its explicit-interface, fully-qualified, global-prefixed name walking up the base chain — the resolution contract is the generator's private member name, not a convention: a hand-rolled public static property of the same simple name is never discovered, so no foreign type can impersonate a generated owner to the metadata plane, and a derived case instance resolves to its owning family through the same base walk that makes leaf-only knowledge sufficient.
- The erased admission route published beside the typed one — a boxed key-to-item try-get with an object-typed error — carries the same typed validation error boxed, not a separate failure vocabulary: reflection-bound infrastructure that cannot name the key type still surfaces the identical fault family, so error handling written against the typed plane and the erased plane converges on one vocabulary.
- The deferred item table is a static-initialization ordering guard, not an optimization: forcing items eagerly inside the metadata constructor would observe partially initialized static fields during the family's own class construction, so the lazy defers the read to first introspection touch and any infrastructure folding items is safe to call from an unrelated static constructor. Each item row carries the source field-name identifier captured at generation, so discriminators and labels derive from the published identifier and a case rename breaks compilation instead of corrupting decode.

[ROUTING_LAW]:
- Shape-polymorphic dispatch that must make key-type mismatch unrepresentable routes through the strongly-typed static-abstract factory plane; the weakly-typed metadata bridge pattern-matches the boxed key and falls through to the default on mismatch, degenerating a wrong-typed key into zero-value validation on struct keys and the null path on class keys — the weak route trades type safety for shape-erased reflection and is licensed only where the caller structurally cannot name the types.
- Reflection-driven hydration of an aggregate runs real admission per layer by bridging from each runtime type to its static factory contract through the constrained invoker — the fully dynamic edge is not a validation bypass, and an aggregate reconstructed from storage or wire passes every nested owner's actual `Validate`. Domain code never reaches for any of this: it holds the typed owner, and a domain-layer appearance of the metadata lookup marks a missing typed surface, not a clever shortcut.
