# Absence and Sentinels ŌĆö Deepening

[SAFEHANDLE_DANGEROUSADDREF_SEMANTICS]:
- `DangerousAddRef(ref bool success)` never sets `success = false` in the non-exceptional path. It throws `ObjectDisposedException` if `_state` has the closed bit set (`(_state & 1) != 0`), and only reaches `success = true` after a successful CAS loop. The canonical caller pattern initializes `bool success = false` before the call so that a thrown exception leaves `success` unset, enabling a `finally { if (success) handle.DangerousRelease(); }` guard. Treating `success == false` post-call as a silent absence signal is wrong; in the throwing path the `finally` block is the gate, not a branch on `success`.
- `DangerousAddRef` also atomically increments the reference count field (`value = state + 4`), so the guard is not re-entrant-safe per handle instance. Projecting the handle to `Option<T>` before any ref-counted access and holding only the `Option` avoids the hazard entirely: the option is the scoped reference, the raw handle is never stored past the projection point.

[ATOM_CHANGE_EVENT_SUBSCRIPTION_IDENTITY]:
- `Atom<A>.Change` is a plain `delegate void AtomChangedEvent<in A>(A value)` event, not an observable or a disposable subscription. Deregistration requires `atom.Change -= handler` where `handler` is the exact same delegate instance used for `+=`. Lambda expressions do not have stable reference identity ŌĆö a new lambda at the `-=` site creates a delegate that compares unequal to the original, silently leaving the handler registered and firing indefinitely. The seam must capture the handler as a named field or local of type `AtomChangedEvent<A>` and store it for symmetric detachment.
- `AtomHashMap<K, V>.Change` fires `HashMapPatch<K, V>` events only when `this.Change != null` at mutation time; if no listener is attached, the internally tracked change log is never built (`RemoveWithLog`/`AddWithLog`/`SetItemWithLog` are not called, `Change<V>.None` is used instead). Subscribing to `Change` after a mutation never retroactively fires; the patch log is ephemeral. A seam that derives capability state from key transitions must subscribe before any write.

[OPTION_BIMAP_NULL_DIVERGENCE]:
- `Option<A>.BiMap<B>(Func<A, B> Some, Func<Unit, B> None)` passes its result through `Check.NullReturn`, which throws `ResultIsNullException` if either branch returns null. `Option<A>.BiMap<B>(Func<A, B> Some, Func<B> None)` does not call `Check.NullReturn` ŌĆö if `None()` returns null, the result is `default(Option<B>)`, silently wrapping null in a struct that appears to be `None`. The two overloads differ only in the `None` parameter type but diverge on null safety: prefer the `Func<B>` overload and ensure the factory never returns null, or use `BiBind` which stays on the `Option` rail and never touches raw references.

[OPTION_TRAVERSE_EFFECT_SKIPPING]:
- `Option<A>.Traverse<F, B>(Func<A, K<F, B>> f)` when the option is `None` returns `F.Pure(Option<B>.None)` ŌĆö the effectful computation `f` is never called. This is the standard Traversable law, but the boundary implication is non-obvious: a seam that uses Traverse to run a side-effecting validation or I/O lift over an optional value will silently skip the effect when the option is absent. The effect is structural, not conditional ŌĆö if the effect must always run for auditing or logging, it cannot be embedded in the traversal; it must be sequenced outside.

[OPTION_SEMIGROUP_FIRST_SOME]:
- `Option<A>.Combine(Option<A> rhs)` returns `this` if `IsSome`, otherwise `rhs`. This is the first-Some semigroup ŌĆö the `|` operator (`lhs.Choose(rhs)`) implements the same semantics. A chain of absence fallbacks (`primary | secondary | tertiary`) evaluates left-to-right and short-circuits on the first present value. The `Monoid<Option<A>>.Empty` is `None`. When chaining fallback absence sources at a seam boundary, the semigroup is the correct surface rather than nested `IfNone` calls.
- `Seq.Choose<A, B>(Func<A, Option<B>> selector)` is the absence-filtering projection: it applies the selector and retains only `Some` values, unwrapping them. It is the correct primitive for projecting a heterogeneous collection through a partial-domain function, replacing any sequence of `Where`+`Select` that re-applies the predicate.

[ATOMHASHMAP_SWAPKEY_OPTION_SEMANTICS]:
- `AtomHashMap<K, V>.SwapKey(K key, Func<Option<V>, Option<V>> swap)` atomically applies a swap that sees `None` when the key is absent and `Some(current)` when present. Returning `None` from the swap when the key existed removes the key; returning `Some(newValue)` when the key was absent adds it. This is the correct primitive for a capability registry where each key represents a capability that may materialize or disappear ŌĆö a single `SwapKey` call handles all four transitions (absentŌåÆabsent, absentŌåÆpresent, presentŌåÆabsent, presentŌåÆpresent) without an auxiliary read-then-write.
- The `Change` event firing for `SwapKey` is conditional on the `this.Change != null` check at the time of the CAS loop body, not at the time of the outer call. A subscriber added between the check and the `AnnounceChange` call will not receive the event for that specific mutation ŌĆö the event is associated with the in-flight CAS, not the current subscriber set.

[OPTION_NATURAL_TO_FIN_COST]:
- `Option` implements `Natural<Option, Fin>` via `ToFin()` with no argument, which maps `None` to `Fin.Fail<A>(Errors.None)`. `Errors.None` is `ManyErrors(Seq.empty<Error>)`, a `ManyErrors` with code `-2000000006` and an empty error sequence. `Error.HasCode(int code)` on `ManyErrors` delegates to `Errors.Exists(e => e.Code == code)` ŌĆö an empty sequence always returns `false`. So `Fin.Fail(Errors.None)` produces a failure that matches no code and no type predicate, making it impossible to discriminate at recovery sites. The `Natural.Transform` path is therefore unsafe for absence-to-failure projection at a seam; always use `ToFin(Error.New(code, message))` with a non-zero domain code.

[CANCELLATION_NONE_TOKEN_REGISTRATION]:
- `CancellationToken.None` has `_source == null`. Both `Register` and `UnsafeRegister` short-circuit by returning `default(CancellationTokenRegistration)` when `_source` is null ŌĆö the callback is never registered and never called. A seam that conditionally registers a callback on a token that may be `None` (the default parameter) must treat a `None` token as a "never-cancelled" signal, not as an absent token that requires special handling. The boundary contract: callers that pass `CancellationToken.None` explicitly signal indefinite lifetime, and the seam must operate without a cancellation gate for that duration.
- Registering on an already-cancelled token (`IsCancellationRequested == true`) invokes the callback synchronously before `Register` returns, unless `ExecutionContext.Capture()` schedules it. `UnsafeRegister` skips the `ExecutionContext` capture so the synchronous invocation occurs directly on the calling thread. At a seam that transitions a capability cell on cancellation, a `UnsafeRegister` callback that calls `Atom.SwapMaybe` must be reentrant relative to the registering thread.

[TERMINAL_BUILT_IN_CODES]:
- `Errors.SourceClosed` (`-2000000013`, `IsExpected=true`) and `Errors.Closed` (`-2000000004`, tuple-created implicit `Expected`) are distinct error singletons. `SourceClosed` is the correct terminal receipt for a streaming or channel seam that has permanently exhausted its source ŌĆö it is distinguishable by `error.HasCode(-2000000013)` or `error.IsType<Expected>()`. `Closed` is for general close states. Using these built-in singletons rather than custom domain codes avoids collision with the framework's own recovery machinery (`@catch` predicates in `Eff` pipelines).
- `Errors.TimedOut` (`-2000000002`) is the terminal receipt for a time-bounded absence detection window that expired. When `EnvIO.LocalCancelWithTimeout(TimeSpan)` creates a scoped environment and the timeout fires, the resulting `Eff` failure carries this code. A recovery handler that distinguishes a timed-out probe from a permanently-failed capability must match against `Errors.TimedOut` and transition the capability cell to `Degraded`, not `Unavailable` ŌĆö a timeout is evidence of transient absence, not confirmed unavailability.

[OPTION_GUARD_UNAVAILABILITY]:
- `Prelude.guard<F>(bool flag)` is constrained to `where F : MonoidK<F>, Applicative<F>`. `Option` implements `Alternative<Option>` and `Choice<Option>` but not `MonoidK<Option>`. Therefore `guard<Option>(condition)` does not compile; the guard pattern is not available for the `Option` monad. Absence filtering inside an `Option` query expression must use `Option<A>.Filter(pred)` or `Option<A>.Where(pred)` instead. `Where` is identical to `Filter` ŌĆö both return `None` when the predicate is false on a `Some` value, leaving `None` unchanged. Neither has a named error carrier; they silently absorb the rejected case.
- For typed-error absence guards over `Eff` or `Fin`, `guard(flag, Error.New(code, message))` is correct ŌĆö the error-bearing overload returns `Guard<Error, Unit>` which binds into `Eff` query expressions via `SelectMany`. The absence is therefore named and recoverable, not silently swallowed.

[LOCK_SCOPE_ABSENCE_INTERACTION]:
- `System.Threading.Lock.EnterScope()` returns a `ref struct Scope` whose `Dispose()` calls `Exit`. Because `Scope` is a `ref struct`, it cannot be stored in an `Option<Scope>` (ref structs are excluded from generic type arguments). A pattern that conditionally acquires a lock based on a domain-absent state cannot store the scope in a carrier; the acquisition and its scope must be structurally within the same stack frame. The boundary law: lock-guarded operations at absence seams must be synchronous and stack-framed; an async or effect-pipelined operation that needs conditional locking must instead use `Lock.TryEnter(timeout)` returning `bool` and map the `bool` to a domain absence signal.
- `Lock.TryEnter()` (zero-timeout) returns `false` immediately if the lock is contended. This is a boolean absence signal that must be projected to a typed result ŌĆö `false` means "lock currently held" not "lock does not exist" ŌĆö and the caller must handle the contention case explicitly, not retry in a tight loop.

[CHANNEL_SEAM_ABSENCE_PROJECTION]:
- `ChannelReader<T>.ReadAsync(CancellationToken)` throws `ChannelClosedException` when the channel is completed and the internal queue is empty. At the seam boundary this must be captured via `Error.New(exception)` and mapped to `Errors.SourceClosed` or a domain-specific terminal receipt. The thrown exception is `InvalidOperationException`-derived and its `Message` is not stable ŌĆö do not match on string content. `ChannelReader<T>.TryRead(out T)` returns `false` for both "no item available" and "channel closed and empty" ŌĆö the absence cause cannot be distinguished from `TryRead` alone; check `Completion.IsCompleted` to distinguish transient absence from terminal absence.
- `ChannelWriter<T>.TryComplete(Exception? error)` propagates the exception into `ChannelReader<T>.Completion` and surfaces it from `ReadAsync` as a `ChannelClosedException` whose `InnerException` carries the original. A seam that signals structured failure through the channel must embed the typed `Error` into the exception: `writer.TryComplete(new Exception(error.Message))`. The typed `Error` is lost through this path; terminal receipts that must survive the channel must be encoded in the element type, not in the completion signal.

[ATOMHASHMAP_CAPABILITY_REGISTRY_PATTERN]:
- A capability registry keyed by a bounded vocabulary can be modeled as `AtomHashMap<CapabilityKey, CapabilityState>` where `CapabilityKey` is a `[SmartEnum<K>]`-generated key and `CapabilityState` is a closed union. `SwapKey(key, prev => Some(next))` transitions a single capability atomically. The health fold is `map.Fold(aggregate, (acc, pair) => acc.Incorporate(pair.Key, pair.Value))` over an `Aggregate` union ŌĆö the fold is total because the `SmartEnum` vocabulary is closed and finite. Missing keys (capabilities not yet probed) are absence in the registry; the fold must handle `Find(key).IfNone(CapabilityState.Unknown)` for each vocabulary member to produce a complete health picture, rather than folding only over present keys.
