# [PY_ARTIFACTS_API_IMAGECODECS]

`imagecodecs` is the native channel-codec substrate the layered-raster egress owner composes BENEATH its container writers: a single flat family of ~87 `<codec>_encode`/`<codec>_decode`/`<codec>_check`/`<codec>_version` quadruples over `numpy`-shaped contiguous buffers, each backed by a vendored C core (libdeflate, zlib-ng, the bundled `imcd` PackBits/LZW/delta kernels, libtiff predictors, libjpeg-turbo, OpenJPEG, libjxl, LERC, c-blosc2, …). For the `export/layered#LAYERED` PSD/PSB/TIFF plane it owns exactly the bytes-level channel transforms the Photoshop and TIFF file formats define and the container owners (`psdtags`/`tifffile`/`PhotoshopAPI`/`psd-tools`) do NOT re-implement: `none` (PSD compression method 0 — raw), `packbits` (method 1 — PackBits RLE per scanline), `zlib`/`deflate` (method 2 — ZIP), and the `delta`+`floatpred`+`bitorder`+`packints`+`lzw` predictor/repacking codecs the libtiff and PSD "ZIP with prediction" (method 3) paths layer before the deflate pass. It never authors a container or a layer record — `psdtags` owns the PSD `TiffImageSourceData`/`PsdLayer`/`PsdChannel` graph, `tifffile` owns the TIFF directory and extratags, `PhotoshopAPI`/`psd-tools` own the native PSD/PSB document — `imagecodecs` is the one place the per-channel byte sequence is compressed and decompressed. The codec is selected by name through one capability-discriminated boundary: each backend exposes a `<CODEC>.available` flag (the same native-build capability-detection shape the media `media/filtergraph#FILTER` owner routes on), so a build missing a codec routes to its substitute or raises `DelayedImportError` at the boundary, never mid-write. It stacks the artifacts compression band as the RASTER-CHANNEL discriminant: `imagecodecs` owns image/layer channel compression (PSD/PSB/TIFF), while `brotli` owns transport/WOFF2, `zstandard` owns archival containers, `lz4` owns hot-path blocks, `py7zr`/`stream-zip` own archive formats, and the universal `numcodecs` owns array-chunk codecs in the chunked-array store — one byte-codec owner per concern, never a duplicate deflate/PackBits owner across the boundary.

## [03]-[ENTRYPOINTS]

[ENTRYPOINT_SCOPE]: the PSD/PSB/TIFF channel codecs the layered-egress owner composes, plus the generic dispatch
- rail: compression

The PSD/PSB file format compresses each layer channel by a per-image method code; TIFF compresses each strip/tile and may pre-transform with a predictor. These rows are the exact byte transforms those format codes name. `out: int` preallocates an output buffer of that many bytes (the bounded-output path — size it from the channel's `height * rowbytes` worst case so a decode never reallocates); `out: bytearray|memoryview|NDArray` writes into a caller-owned destination. `data: Buffer` is any contiguous buffer (`bytes`, a `numpy` array, a `memoryview`); the predictor codecs (`delta`/`floatpred`) operate on a typed `NDArray` and return one whose dtype/shape match the input but whose values are an encoded byte sequence, NOT meaningful numbers.

## [04]-[IMPLEMENTATION_LAW]

[COMPRESSION_IMAGECODECS]:
- import: `import imagecodecs` at boundary scope only; module-level import is banned by the manifest import policy (the egress page's `lazy import` proxy reifies the native core on first channel-codec use, exactly as it `lazy import`s `psdtags`/`tifffile`).
- one codec, one quadruple — never an instance: a codec is `<name>_encode` / `<name>_decode` / `<name>_check` / `<name>_version`, selected by name through the channel-codec dispatch table, never a constructed `Codec` object and never a per-codec wrapper class. The PSD compression-method code maps to the codec name directly: method `0 -> none`, `1 -> packbits` (per-row `axis=`), `2 -> zlib`/`deflate`, `3 -> delta`+`deflate(raw=True)` (predictor then raw-deflate) — drive this map as a `frozendict[PsdCompression, codec]` discriminant, never an `if/elif` chain at the channel-write site.
- capability-detection at the boundary, not assumption: probe `<CODEC>.available` before routing a channel to a codec the build may lack, mirroring the `media/filtergraph#FILTER` native-vs-substitute routing — a `False` flag (or the `DelayedImportError` an absent-core `<codec>_encode` raises) routes to the substitute arm (`none` raw store, or a different admitted codec), the codec choice recorded on the channel receipt so a downstream reader knows the achieved compression. Never call an `<codec>_*` function whose `<CODEC>.available` was not asserted in a build whose codec set is not fixed.
- predictor-then-compressor ordering is the codec topology, not two rails: the TIFF/PSD "ZIP with prediction" path is `delta_encode`/`floatpred_encode` (raise compressibility) THEN `deflate_encode(raw=True)` (compress the predicted bytes); chain in that order and invert on decode (`deflate_decode(raw=True)` then `delta_decode`). `floatpred` for float channels, `delta` for integer channels — never compress raw gradient channel bytes where a predictor precedes the compressor.
- bounded output via `out=int`: for a decode whose output size is known (a PSD channel is `height * rowbytes`), pass `out=<nbytes>` so the codec writes into a single right-sized allocation and never grows a buffer; a hostile or highly-compressible channel cannot exhaust memory because the destination is fixed. The bit-exact lossless codecs (`packbits`/`zlib`/`deflate`/`lzw`/`delta`/`floatpred`/`bitorder`/`packints`/`none`) round-trip byte-identical — the PSD/TIFF channel contract — so the channel receipt records the codec name and the input/output byte lengths, never a precision-loss parameter.
- buffer in, buffer out: the byte codecs take and return contiguous `bytes`/`bytearray`/`memoryview`; the predictor and image codecs take and return `numpy`-shaped contiguous `NDArray`. Pass a channel through as a contiguous array (the `psdtags.PsdChannel.data` 2-D array slice is already contiguous); a discontiguous view must be `.copy()`-d at the boundary, never handed to a codec that does reject or silently copy it.
- evidence: each channel-codec call captures the codec name, the PSD compression-method code, the predictor (when chained), the `level` (deflate family), the input/output byte lengths and the achieved ratio, and the linked-core version (`<codec>_version()`) as a channel-codec receipt folded onto the layer-export span.
- boundary: `imagecodecs` owns the per-channel byte transform ONLY — `psdtags` owns the PSD `TiffImageSourceData`/`PsdLayer`/`PsdChannel` graph and `tifffile` the TIFF directory/extratags; `PhotoshopAPI`/`psd-tools` own the native PSD/PSB document structure; the codec never authors a layer record, a channel header, a container, or an identity. Failures surface as the per-codec `<Codec>Error` or `DelayedImportError`, never a raw native return code; live UI and re-rasterization stay outside this package.

## [05]-[INTEGRATION_LAW]

[STACKING] — the catalog stacks BOTH `.api` tiers: the shared/universal `libs/python/.api/*` rails ON TOP OF this folder package, never the folder set alone.

- container seam (folder `psdtags`/`tifffile`/`photoshopapi`/`psd-tools` `.api`): `imagecodecs` is the bytes layer the layered-egress `_tiff`/`_psd` arms reach for AFTER the container owner has built the layer/channel structure. The `psdtags.PsdChannel(channel_id, data=<2-D uint8 array>)` records hold raw channel arrays; when the format/profile selects a compressed channel, the arm runs `data` through `packbits_encode(channel, axis=0)` (method 1) or `delta_encode(channel, axis=-1)` + `deflate_encode(..., raw=True)` (method 3) before the container serializes it, and the inverse on read. `tifffile` already calls into `imagecodecs` internally for its strip/tile codecs — this catalog is the page-level codec the PSD/PSB native-author arm (`PhotoshopAPI`/`psd-tools`) and any raw-channel path compose directly; the container owner authors the structure, `imagecodecs` authors the compressed channel bytes — they meet at the channel byte array, never re-implementing each other.
- numeric seam (universal `numpy` `.api`): the codec boundary is `NDArray[uint8] -> bytes` (channel encode) and `bytes -> NDArray[uint8]` (channel decode); the channel is always a contiguous `numpy` 2-D array sliced from the placed RGBA composite (`_rgba_array` in `export/layered`), and the predictor codecs (`delta`/`floatpred`) preserve `numpy` dtype/shape while reinterpreting the values as encoded bytes. Size the `out=` preallocation from `channel.nbytes`; the universal `numpy` rail owns the array, `imagecodecs` owns its compressed serialization.
- model seam (universal `msgspec`/`pydantic` `.api`): the channel-codec choice is a typed field, never a stringly value — model the PSD compression method as a closed `PsdCompression` `IntEnum` (`RAW=0`/`RLE=1`/`ZIP=2`/`ZIP_PREDICTION=3`) on the layer-policy struct, and the `frozendict[PsdCompression, str]` maps each member to the `imagecodecs` codec name the arm dispatches on; `msgspec` decodes the policy and the codec name is a validated tag, never a free-form string reaching `imread(codec=...)`.
- contract seam (universal `beartype` `.api`): annotate the channel-codec rail `bytes | numpy.ndarray -> bytes | numpy.ndarray` so a non-buffer input is rejected at the contract, not deep in the Cython extension; the `<CODEC>.available` capability flag is the boundary predicate the contract reads before dispatch, so an unbuildable codec route fails the contract with the codec name rather than a `DelayedImportError` stack.
- resilience seam (universal `stamina` `.api`): a codec `encode`/`decode` is CPU-pure and deterministic — never wrap it in `@retry`; a `<Codec>Error` (corrupt channel, undersized `out`) or `DelayedImportError` (absent core) surfaces immediately as a terminal fault, never a transient retry trigger. Retry belongs only on the IO around the produced container bytes (artifact-store write owned by `core/plan`), never on the byte transform.
- observability seam (universal `structlog`+`opentelemetry-api` `.api`): record the per-channel codec receipt — codec name, PSD method code, predictor, `level`, input/output byte lengths, achieved ratio, `<codec>_version()` core version — as a structured event on the layer-export span; the build census `version()` rides the run's startup span once so a downstream reader knows exactly which native cores (libdeflate/zlib-ng/libtiff versions) compressed each channel.
- concurrency seam (universal `anyio` `.api`): the channel-codec passes are CPU-bound native calls that release the GIL — a many-channel encode/decode batch crosses `anyio.to_thread.run_sync` bounded by a `CapacityLimiter` (the `export/layered` `_THREAD_GATE` shape), never blocking the event loop on a large channel; for a whole-document PSD/PSB author the heavier `_offloaded` `to_process` lane the page already runs for the libvips composite carries the channel-codec passes alongside the container author in one worker crossing.
- artifacts-rail boundary: `imagecodecs` owns image/layer CHANNEL compression (PSD/PSB/TIFF) — the raster-channel arm of the artifacts compression band. The sibling band owners stay disjoint: `brotli` owns transport (`Content-Encoding: br`) and WOFF2, `zstandard` owns archival containers, `lz4` owns hot-path block compression, `py7zr`/`stream-zip` own archive/ZIP container formats, and the universal `numcodecs` owns array-CHUNK codecs in the chunked-array store. Route a PSD/TIFF channel through `imagecodecs`, a transport payload through `brotli`, an archive through `zstandard`/`py7zr`, an array chunk through `numcodecs` — one byte-codec owner per concern, never a duplicate `deflate`/`PackBits` owner across the boundary (the OpenRaster `ORA` arm's PNG members route through `stream-zip`'s `ZIP_AUTO`/`zlib.crc32`, NOT through this package, because that is container-ZIP, not channel compression).

## [06]-[LOCAL_ADMISSION]

[RAIL_LAW]:
- Package: `imagecodecs`
- Owns: the native per-channel byte-codec substrate for the `export/layered` PSD/PSB/TIFF egress plane — the `none`/`packbits`/`zlib`/`deflate`/`zlibng`/`lzw` channel compressors, the `delta`/`floatpred`/`bitorder`/`packints` TIFF/PSD predictors and sample-repacking codecs, the `<CODEC>.available` build-capability probes, and the polymorphic `imread`/`imwrite`/`imagefileext` codec-name dispatch over the full image-format set
