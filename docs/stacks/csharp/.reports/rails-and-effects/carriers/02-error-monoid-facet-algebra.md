# Error Monoid And Facet Algebra

[MONOID_CARRIER]:
- The failure type is itself the aggregate: a single error and a thousand errors share one type, so a rail's failure slot never widens to a sequence-of-errors to carry accumulation, and a function returning the failure carrier already returns the aggregate without a container change.
- Combine maintains a flat-depth invariant at every append: two collections concatenate their inner sequences, a leaf appended to a collection cons-prepends or appends into the same sequence, two leaves seed a fresh two-element collection — the result is always a leaf or a flat collection, never a tree, so a fold over an accumulated batch is always a single linear pass.
- The flat invariant is what makes the aggregate's one-level membership scan total: code- and type-membership over a combine-built aggregate quantifies over direct members with no recursion to unwind, because combine never nests an aggregate inside an aggregate. Causal chains attached as inner errors live outside that scan — membership predicates do not walk the inner-error projection, so a buried cause is found only by explicit inner traversal.
- The accumulating validation carrier's gather is precisely this monoid: its failure-side constraint threads the identity and combine through the applicative, so pinning the failure type to the error monoid makes applicative accumulation produce one flat aggregate with no bespoke accumulator. The gather is the monoid; the monoid is the type.

[FACET_LATTICE]:
- The expected/exceptional facets are independent abstract booleans; every concrete leaf sets exactly one true, and the aggregate derives both structurally — exceptional if any member is, expected only if every member is. One escaped exception inside a batch of expected validation failures contaminates the whole batch to exceptional: code that routes on the aggregate facet must split the sequence by per-member facet before routing, never trust the aggregate flag to mean the batch is homogeneous.
- Equality on the expected leaf is full structural record equality over message, code, and inner error — the code is not a privileged equality key. Identity-keyed recovery therefore breaks under message templating or localization; recovery that must survive message change keys on the code through the code-matching predicate surface, which is a separate axis from equality.
- The bottom case is exceptional, not expected: a filtered-away comprehension step or a default-initialized rail value surfaces as a code-carrying exceptional bottom rather than a silent empty. A recovery that catches expected failures deliberately lets bottom escape — bottom is a logic defect, not a domain outcome, and the recoverable partition excludes it by construction.

[CONSTRUCTION_DISPATCH]:
- Construction is one polymorphic factory family discriminating on argument shape, reinforced by implicit conversions from a code-message tuple and from an exception — tuples and exceptions are the failure literals at call sites, and the discriminant is recoverable from the argument with no kind flag and no sibling named constructors.
- Exception admission is a canonicalizing dispatch, not a blind wrap: task-cancellation and timeout exceptions collapse to pre-built expected singletons with reserved codes, so cancellation and timeout arrive on the rail as expected, catchable, code-stable values — matched by one code constant rather than several exception types, and never contaminated into the exceptional partition. An aggregate exception explodes through the same lift per inner exception and re-collapses zero/one/many, so a single failure never wears a collection wrapper.
- The aggregate renders its message at construction as the full member-message array — it stringifies to the complete failure set, not a count — and owns a reserved aggregate code distinct from any member's, while code-membership quantifies existentially over members: one predicate can target either the fact of aggregation or the presence of a specific buried failure.
- The heterogeneous inbound funnel admits the carrier itself, the wrapped exception forms, the rail-exception type, a raw exception, a bare string, and an option of the carrier, falling back to the bottom sentinel for anything else — one entry point collapses every foreign failure shape with no per-shape branch at the call site.

[ONE_WAY_EXCEPTIONAL_BRIDGE]:
- The exceptional leaf's live exception is deliberately erased by deserialization or default construction: the rebuilt value still answers exceptional and keeps message and code, but the stack trace, the exception type, and typed-exception membership tests are gone. The bridge carries a throw inward as a value and does not carry it back across a wire intact — exceptions do not leak across application boundaries; only their sanitized projection does.
- Within one process the round trip is lossless: leaves project to wrapped error-exceptions preserving code, message, and inner; the aggregate projects to an aggregate exception over member projections; and admission recognizes the wrapped forms and unwraps them back to the original carrier. A throw/catch hop through foreign code that only speaks exceptions does not degrade a structured aggregate into a flat string, and a thrown aggregate surfaces every accumulated rejection, not only the first. Re-raising routes through dispatch-info so the original stack survives the rail.

[PARAMETERIZED_FAILURE_RAIL]:
- The fallible trait is two-parameter over the failure type and the carrier; a closed domain rejection union instantiates as the failure type so the rail carries a precise discriminated rejection rather than a stringly message, and the generic raise dispatches on the carrier's own failure type with no conversion at the call site. The pinned raise helper takes the monoidal error specifically; a boundary that admits the domain union internally and surfaces the monoidal error externally crosses exactly once, at the trait-instantiation seam, and never re-describes the failure inside the rail.
- Reserved-band law: the runtime owns a contiguous band of large-magnitude negative codes for its structural failure states, each a pre-built singleton. A domain code inside that band is indistinguishable from the runtime singleton under code-keyed recovery — the band is an exclusion zone for domain error vocabularies.
