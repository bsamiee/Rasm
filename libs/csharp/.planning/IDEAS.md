# [CSHARP_IDEAS]

The C# branch forward pool — the cross-package and multi-process topologies the desktop/host suite owns in concert, the cross-package laws that keep one owner per concept, and the folder-tagged deepening horizons. Each concept rides axis rows on the named owning page clusters and never adds a parallel surface; owner state is read at the owning package `ARCHITECTURE.md` `[OWNER_REGISTRY]`, never marked here. Where a concept has a web or science consumer, the cross-language leg is a Tier-0 seam (`libs/.planning/region-map/seam-splits.md`), referenced as such and never restated. The host-modality row (`AppHost/host-profiles#PROFILE_AXIS`, `Persistence/store-profiles#PLACEMENT_MATRIX`, `AppUi/surface-hosts#HOST_AXIS`) is the one dual-mandate switch — universal-standalone and first-class rhino/gh2 — never a fork; the topology vocabulary (in-process, paired, companion, sidecar, hub, service, web-fed) lives inline on each concept group.
