# Entrypoint Law â€” Edge Cases

[CANCELLATION_SCOPE_OPERATORS_ON_ENTRYPOINTS]:
- Three distinct scope operators govern cancellation at entrypoint boundaries. `localIO<M,A>(K<M,A>)` creates a new `CancellationToken` derived from the parent context's token, so a regional `IO.cancel` inside the scope does not propagate to computations outside the scope â€” but the local cancel still raises an exception that propagates upward from the local scope, requiring a catch. `uninterruptible<M,A>(K<M,A>)` ignores all cancellation requests â€” including parent `CancellationToken` cancellation â€” by substituting `CancellationToken.None` for the duration; the computation runs to completion. The entrypoint design choice: `localIO` for fan-out dispatch arms that should not cancel siblings, `uninterruptible` for cleanup arms inside `bracketIO` whose interruption would leave resources open. Both accept `K<M,A>` where `M : MonadUnliftIO<M>`, making them carrier-polymorphic.
- `Prelude.cancel` is a static `IO<Unit>` field that raises cancellation into the current context. Placing it as a dispatch arm in a `Switch` branch aborts the entire computation at that case â€” it is a structured abort that propagates as an exception through `|` catch chains. The resulting error code maps from `TaskCanceledException`. When a `retry` loop wraps a computation containing `cancel`, the cancellation error propagates through `retry` without triggering another attempt; `retry` does not swallow `TaskCanceledException`.
- `timeout(TimeSpan, K<M,A>)` is available for any `MonadIO` carrier, not just `IO`. It internally forks the computation, awaits the fork, and cancels it if the duration expires â€” delivering `Error.New(TimeoutException)` on expiry. For entrypoints that must bound execution time, `timeout` is the correct aspect form rather than a `CancellationTokenSource` parameter on the entrypoint signature, which would be the input-redescribing anti-pattern.

[FORK_AND_PARALLEL_COMPOSITION_AT_ENTRYPOINTS]:
- `fork<M,A>(K<M,A>, Option<TimeSpan>)` (`Prelude`, carrier-polymorphic where `M : MonadUnliftIO<M>`) queues the computation on the thread-pool and returns `K<M, ForkIO<A>>`. The `ForkIO<A>` handle has `.Await` (blocks until completion and returns the result or failure) and `.Cancel` (sends a cancellation signal). When a timeout is supplied, the forked operation is automatically cancelled after that duration and `.Await` delivers `TimeoutException`-mapped `Error`. A forked computation that throws is not swallowed â€” `.Await` delivers the error through the normal `IO` error rail.
- `awaitAll` accepts multiple overloads: `params ForkIO<A>[]`, `Seq<ForkIO<A>>`, `Seq<IO<A>>`, and the generic carrier forms `params K<M,A>[]` and `Seq<K<M,A>>` where `M : MonadUnliftIO<M>`. `awaitAny` has analogous overloads. The generic `params K<M,A>[]` form for `awaitAll` is the dense pattern for fanning out to N independent suboperations without explicit `fork` handle management. For guaranteed concurrent execution rather than potential sequential execution, explicit `fork` + `awaitAll(Seq<ForkIO<A>>)` is required.
- `Applicative.zip` over `K<F,A>` tuples composes two independent applicative effects. The tuple overloads go to arity 5: `zip<F,A,B,C,D,E>((K<F,A>, K<F,B>, K<F,C>, K<F,D>, K<F,E>))`. This is the idiomatic form for parallel-independent entrypoint fan-out without introducing `fork` handles: `zip((readX, readY, readZ)).Map((x,y,z) => process(x,y,z))`. Unlike `>>` sequential bind, `zip` does not impose an ordering requirement on the IO effects â€” the runtime may execute them concurrently; for guaranteed concurrency, explicit `fork` + `awaitAll` is required.

[KEYLESS_SMART_ENUM_AS_ENTRYPOINT_DISCRIMINANT]:
- `[SmartEnum]` (no type parameter) is the keyless variant. The generator emits `Switch`/`Map` dispatch methods and an `.Items` collection but no key property, no `Get(key)`, no `TryGet(key, out item)`, and no `IObjectFactory` interfaces. When a module uses a keyless smart enum as a case discriminant, the dispatch surface is structural: the caller holds an instance and calls `Switch(state, case1: ..., case2: ...)`. There is no admission path from an external key â€” the module owns the only construction paths via the static fields. This is the correct form for internally-discriminated behavior rows where the external caller receives an item by reference, not by string key.
- `SwitchMapMethodsGeneration.None` on `[Union]` or `[SmartEnum<TKey>]` suppresses `Switch` and `Map` generation entirely. Setting `None` is the correct form when dispatch is entirely inside extension blocks that implement a custom dispatch algebra over the union's public properties â€” the generated `Switch` methods would create a secondary, less-restrictive dispatch surface that bypasses the extension block's policy enforcement. `MapMethods` has the same `SwitchMapMethodsGeneration` type and can be suppressed independently of `SwitchMethods`.

[BRACKETIO_CATCH_PARAM_PHASE_PRECISION]:
- The four-argument `bracketIO<M,A,B,C>(Acq, Use, Catch: Func<Error, IO<C>>, Finally)` documentation states: "Function called when an `Error` is raised within the `Acq` computation." An error in the `Use` arm propagates through the `Finally` arm directly; the `Catch` arm is not invoked for `Use` failures. This is architecturally significant: entrypoints that acquire a resource through a fallible acquisition step use `Catch` to reclassify or log the acquisition error while still guaranteeing `Finally` cleanup.
- The three-argument `bracketIO<M,A,B,C>(Acq, Use, Finally)` form: `Finally` returns `IO<A>` where `A` is the final result type, not `IO<Unit>`. The `Finally` arm can therefore contribute a derived value rather than being forced to return `Unit` â€” relevant when cleanup has a meaningful output (e.g., confirming a commit was acknowledged before the connection closed).

[SEALED_UNION_REMAINING_EXTRACTION_SURFACES]:
- When `ConversionToValue = None` removes explicit downcast operators from an ad-hoc union, the only case-extraction paths are `Switch`/`Map` dispatch and C# type-testing pattern matching (`value is CaseType c`). The generator does not emit named `.IsX`/`.AsX` properties; the discriminator is the internal `_valueIndex` field, not a named accessor. Pattern matching against the union's declared member types works because pattern testing uses type comparison, not conversion operators.
- For a `params ReadOnlySpan<T>`-based entrypoint on a regular `[Union]` where `T` is the union base type, removing `ConversionFromValue` operators forces callers to pass already-constructed union instances rather than implicitly-upcast case types. This is architecturally desirable for a sealed command pipeline: the factory method that constructs the union is the only admission path, and the span parameter collects a pre-validated sequence of commands for batch dispatch without any per-element implicit widening.

[PARAMS_COLLECTION_GENERIC_SPAN_HEAP_FALLBACK_DEPTH]:
- When a `params ReadOnlySpan<T>` parameter appears in a generic method and `T` is unconstrained, the compiler emits a heap-allocated `T[]` and passes it as a span when the call site provides a collection expression with a non-statically-known count. Adding `where T : allows ref struct` narrows the type to ref-struct-safe types only; for non-ref-struct domain types, the correct mitigation is a concrete typed `params ReadOnlySpan<KnownCmd>` rather than a generic parameter. The `allows ref struct` anti-constraint is available on .NET 10; a parameter typed as `T where T : allows ref struct` can receive `ReadOnlySpan<char>` directly but cannot be stored on the heap.
- `[OverloadResolutionPriority(-1)]` applied to a `T[]`-accepting overload that exists for binary compatibility downgrades it relative to the span overload without deletion. The priority is evaluated before normal tiebreaking, so the span overload wins at all call sites that could call either â€” regardless of whether the overload is in the same type or a base type.

[MODULE_ENTRYPOINT_INTERNALS_PRIVACY_FAILURE_MODES]:
- A `private` member in an extension block on `RequestUnion` inside a `static class` is callable from another extension block on `Result` inside the same static class. This creates an unexpected coupling surface between dispatch helpers for different receiver types. The mitigation is to declare shared helpers as `file`-scoped (`file static`) at top level or as `private static` methods directly on the containing static class, not inside any extension block.
- `internal sealed partial class` on a generated `[Union]` type restricts the type to the assembly and prevents external subclassing. When `ConstructorAccessModifier = Internal`, case constructors are accessible to the entire assembly â€” not just an enclosing type. When the desired scope is the enclosing type only, declare the union as a `private partial class` nested inside the service class and use `ConstructorAccessModifier = Private` with explicit factory methods on the service.
- `[UnionSwitchMapOverload(StopAt = [typeof(AbstractCase)])]` generates a coarse overload that delivers `AbstractCase` instances; callers that need leaf-level dispatch use the outer full-coverage overload. When `AbstractCase` is itself a `[Union]`-derived abstract partial class, its own generated `Switch` provides the second level of dispatch. This two-tier dispatch surface is the correct design for a command hierarchy where some handlers care about the specific case and others treat the category as atomic â€” both overloads are exhaustive at their respective levels.
