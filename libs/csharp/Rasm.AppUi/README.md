# [RASM_APPUI]

`Rasm.AppUi` is the product UI engine: one shell mounted on every admitted host substrate — Rhino panels, Rhino modals, GH2 companion windows, standalone desktop, sidecar shells, and headless proof surfaces — through one seven-case `SurfaceHost` axis. The package owns shell, navigation, screens, commands, live data, tables, inspectors, charts, offscreen visuals, theme, typography, icons, dialogs, input, motion, accessibility, localization, and evidence as one UI rail.

The package has zero consumers; implementation is full-capability with no holding back. Eighteen finalized planning pages are decision-complete blueprints transcribed verbatim — never re-designed downstream. AppHost ports, Persistence queries, and Compute receipts arrive as settled vocabulary; Rhino and Grasshopper host behavior crosses only as seam delegates. The web-browser surface stays a designed-only growth case whose command keys and evidence cases serialize for the TS layer today.

## [2]-[SCOPE]

- One mount transaction per surface; one scheduler boundary; one command table; one change-set engine; one token catalogue; one evidence union.
- Public vocabulary is product vocabulary; Avalonia, ReactiveUI, SkiaSharp, LiveCharts, Dock, and host types stay internal unless a boundary forces a value.
- Viewport overlays and native scene composition stay with Rasm.Rhino and Rasm.Grasshopper display conduits; AppUi owns retained composition and offscreen raster only.
- New capability lands as a row, case, or policy value inside a budgeted owner — the charter density bar is binding.
