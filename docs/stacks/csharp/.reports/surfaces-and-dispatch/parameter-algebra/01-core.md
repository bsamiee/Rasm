# Parameter Algebra

[POLICY_VALUES]:
- A smart enum row is the canonical policy value form: the row carries constructor-injected behavior delegates alongside its key, and the generated `Switch<TState, TResult>(TState state, Func<TState, CaseN, TResult> onCaseN, ...)` threads state through the row without the caller re-deriving which case was selected. `SwitchMapMethodsGeneration.DefaultWithPartialOverloads` further generates a partial-coverage overload accepting an `otherwise` fallback, which admits a default without spreading it across call sites.
- `[SmartEnum<TKey>]` generates a `Key` property typed `TKey` and implements `IParsable<T>` and `ISpanParsable<T>` when `TKey` supports them; the parsed key is the identity of the policy row, not a string literal that must be re-matched later. `SkipIParsable = true` suppresses both span and parsable interfaces together because `ISpanParsable<T>` inherits from `IParsable<T>` â€” suppressing one forces the other.
- `[SmartEnum]` (keyless) generates the same `Switch`/`Map` shape with `SwitchMapStateParameterName` defaulting to `"state"`; the state parameter name is overridable on the attribute so generated parameter names remain consistent with a codebase vocabulary without touching generated code. The `EqualityComparisonOperators` setting on the keyless form produces no key-overload variant because there is no key member.
- Stateless union members in `[Union<T1,T2>]` and `[Union]` ad-hoc forms store only the discriminator index, no payload; the generated accessors return `default(T)` for stateless cases. Use a struct for the stateless marker type so the accessor value cannot be null-dereferenced. The `T1IsStateless` flag implies `T1IsNullableReferenceType` for reference-type members automatically.
- `NestedUnionParameterNameGeneration.Simple` on a `[Union]` changes generated `Switch` parameter names from the fully-qualified `failureNotFound` form to the leaf-only `notFound` form. Collision when two nested families share a leaf name causes a compile error in the generated code â€” `Default` is the safe choice whenever two sub-unions could produce identical leaf names.
- `ConversionOperatorsGeneration.Implicit` (default for union-to-value) lets a policy case implicitly convert to the union type at call sites; `Explicit` forces an explicit cast. Setting `ConstructorAccessModifier = Private` on the union suppresses both the constructor and the implicit conversion operator, mandating factory methods. This is the correct shape when the factory carries admission logic beyond naked construction.

[OPTIONAL_CONTEXT]:
- `Option<T>` is the single admitted form for optional parameters that carry absence semantics. `Option<A>` in LE5 implements `K<Option, A>` and participates in higher-kinded abstractions; it exposes `IsSome`, `IsNone`, and a `Case` property returning `object?` that enables `{ IsSome: true, Case: T value }` property-pattern destructuring without triggering the `Match`-in-expression analyzer prohibition.
- `Option<A>` carries implicit conversions from `A?` (reference type: null maps to `None`) and from `Fail<Unit>` / `Unit` (both map to `None`), making `None` the Prelude constant for both the absence sentinel and the option initializer. The `|` operator on two `Option<A>` values returns the left if it is `Some`, the right otherwise â€” a direct fallback chain without a Match arm.
- `Option<T>` as an optional parameter default requires `= default` (not `= default(Option<T>)`) at the declaration and `IfNone(canonical)` at use; `Option<TVo>` where `TVo` is a `[ValueObject]` or `[SmartEnum]` avoids the CS1736 constraint that forbids a non-const default. The canonical value comes from the policy owner â€” the smart enum static property or value-object factory â€” so the default is derived, not declared again at the call site.
- A runtime context parameter that is optional but carries identity should enter as one record, not as a flag tail. A nullable record parameter `T? ctx = null` is the boundary-only form; inside the domain it projects to `Option<T>` at the admission site so interior code never branches on null.

[POLICY_TABLE_FORM]:
- `FrozenDictionary<TKey, TValue>.Create(params ReadOnlySpan<KeyValuePair<TKey, TValue>> source)` is the .NET 10 API for constructing a frozen table from a stack-allocated span at a static field initializer. The `params ReadOnlySpan<T>` overload (C# 13 feature `IDS_FeatureParamsCollections`) accepts a collection expression at the call site with no heap allocation for the intermediate collection. The second overload `Create(IEqualityComparer<TKey>?, params ReadOnlySpan<...>)` selects the comparer without changing the calling convention.
- `FrozenDictionary<TKey, TValue>` exposes a `ref readonly TValue this[TKey key]` indexer and a `GetValueRefOrNullRef(TKey key)` method returning `ref readonly TValue`; callers check `Unsafe.IsNullRef(in result)` to distinguish presence from absence without boxing the value. `AlternateLookup<TAlternateKey>` (where `TAlternateKey : allows ref struct`) enables `ReadOnlySpan<char>` keying against a string-keyed table with no string allocation.
- A frozen policy table is the `POLICY_VALUES` implementation for bounded vocabularies with non-delegate behavior: the smart-enum row owns behavior via delegate fields; the frozen table owns behavior via value structs or delegates stored as map values. The table is a single static field; looking up a key returns the complete policy value â€” the caller never reconstructs behavior from key combinations.
- `FrozenSet<T>` produces the same freeze-once/read-many guarantees for membership vocabularies; its `AlternateLookup<TAlternate>` (where `TAlternate : allows ref struct`) permits `ReadOnlySpan<char>` membership tests against a `FrozenSet<string>` with the same zero-allocation pattern as the dictionary form.

[ARITY_COLLAPSE]:
- `params ReadOnlySpan<T>` (C# 13) is the zero-allocation arity collapse form. The compiler emits an inline array or stack-allocated span at the call site â€” no `IEnumerable<T>` heap object. When an existing `params T[]` overload coexists with a `params ReadOnlySpan<T>` overload, `[OverloadResolutionPriority(1)]` on the span overload directs the compiler to prefer it, eliminating the `T[]` overload as a required presence. `OverloadResolutionPriorityAttribute` is a C# 13 / .NET 9+ BCL type (`System.Runtime.CompilerServices`).
- Collection expressions (`[a, b, c]`) bind to any `params` target type at the call site without specifying the collection kind; the compiler selects the best collection builder. A `params ReadOnlySpan<T>` parameter with a collection expression call site produces no heap allocation when the count is small enough for an inline array. The Thinktecture `Schedule.TimeSeries(params Duration[] durations)` overload is the example of the old form; a `params ReadOnlySpan<Duration>` overload with `[OverloadResolutionPriority(1)]` would be the replacement.
- A `params` parameter that re-describes what the receiver already discriminates is the prohibited form. If the entrypoint already dispatches on the case of a union input, adding a parallel `params` array carrying case-correlated overrides is arity smuggling: the same case selection is expressed twice. The discriminant must be recoverable from the input value itself.
- `Argument<T>` from Thinktecture is a `readonly ref struct` that carries a `Value: T` and `IsSet: bool`; the implicit conversion from `T` sets `IsSet = true` and a default `Argument<T>` has `IsSet = false`. This enables optional parameter algebra at factory sites where the absence of a value (not provided by caller) must be distinguished from a provided value that happens to be the domain default â€” the generated `ValidateFactoryArguments` partial receives `ref Argument<T>` for each field so validation can inspect whether the caller explicitly supplied it.

[DEFAULTS_AND_DERIVATION]:
- Generated value-object factory defaults derive from the owning type, not from the call site. `AllowDefaultStructs = false` (default) prohibits `default(T)` and parameterless construction of struct value objects; the generated `Empty` static property is the named sentinel for the struct-zero case and is the only way to obtain it. `DefaultInstancePropertyName` renames it. Callers using `Option<TVo> = default` receive `None` at the call site while the domain default lives on the type.
- `SkipFactoryMethods = true` suppresses `Create`, `Validate`, `TryCreate`, the `TypeConverter`, the `IObjectFactory<T>` interface, and the implicit conversion from key type to value object. Setting this forces callers to use a manually authored `[ObjectFactoryAttribute<T>]`-marked factory, which is the correct shape when admission logic spans multiple sources or requires `Fin<T>` rail integration.
- `[ObjectFactoryAttribute<T>]` (allow-multiple) registers a custom factory keyed on `T`; `UseForSerialization` directs the specified serialization frameworks to the custom factory. `UseWithEntityFramework = true` routes EF Core value converters through the factory. Multiple attributes on one owner register independent conversion paths for different source types.
- `IValidationError<T>` is a `static abstract` interface member: `static abstract T Create(string message)`. A custom error type implements this interface; the generated factory invokes `TCustom.Create(message)` at validation failure time. This means the policy of which error type to produce lives on the domain type declaration, not at every factory call site.

[COMPOSITION_TIME_POLICY]:
- `Schedule` is a composable policy value: `|` takes the union (minimum duration, length of the longer stream); `&` takes the intersection (maximum duration, length of the shorter stream); `+` concatenates two schedules in sequence. A schedule is composed once into a `static readonly` field and passed as a single `Schedule` argument to `retry` or `repeat` â€” no parameters inside the retry loop re-specify the backoff.
- `Schedule.recurs(n)` returns a `ScheduleTransformer`, not a `Schedule`; combining with `|` against a `Schedule` base (e.g., `Schedule.exponential(10*ms)`) produces a `Schedule`. A `ScheduleTransformer` applied to a schedule via `|` or `&` returns a new schedule: the transformer is a higher-order policy value that modifies any base schedule rather than being a schedule itself.
- `Schedule.jitter(factor, seed)` and `Schedule.jitter(minRandom, maxRandom, seed)` are `ScheduleTransformer`s. Both accept `Option<int>` for the seed, with `default(Option<int>)` meaning non-deterministic. The seed parameter is itself an `Option<T>` parameter carrying absence, not a nullable `int?` â€” the seeded-vs-random distinction travels as presence/absence in the option, not as a boolean knob.
- `CatchM<E, M, A>` is the named catch policy value: `Match: Func<E, bool>` selects the error; `Action: Func<E, K<M, A>>` provides the recovery. The `|` operator on `K<F, A>` accepts a `CatchM` on the right, composing recovery policy as a linear chain without nesting. Named catches via `@catch(error, handler)` or `@catch(predicate, handler)` produce `CatchM` values that can be extracted to `static readonly` fields and applied across any `K<Fallible<E,M>, A>` computation.
- `bracketIO<M, A, B, C>(Acq, Use, Catch, Fin)` overload admits a `Catch: Func<Error, IO<C>>` policy alongside `Use` and `Fin`. The three function parameters together are the resource policy: acquire, use-with-recovery, and finalize. The `Catch` is optional at the call site â€” the two-parameter `Bracket(Use, Fin)` omits it â€” so the recovery policy is a named optional rather than a flag.

[EXTENSION_BLOCKS]:
- C# 14 extension blocks (`extension(ReceiverType r) { ... }` â€” `IDS_FeatureExtensions`, requires `LangVersion = 14`) are a receiver-owned surface addition distinct from classic `this`-parameter extension methods. The extension block declares the receiver type and its parameter name once; all members inside share the same receiver without repeating `this T` on each. Properties, indexers, and operators are admissible in extension blocks, not just methods, so a full receiver-owned surface â€” including computed properties over a policy type â€” can be expressed without modifying the original type.
- The compiler models extension block members internally as `IsExtensionBlockMember()` distinct from `IsExtensionMethod`; overload resolution checks `IsExtensionBlockMember` when reducing extension candidates against a receiver type. A method inside an extension block and a classic extension method over the same receiver type can coexist; priority is resolved by the standard overload resolution rules plus `[OverloadResolutionPriority]`.
- Extension blocks intersect with parameter algebra at the policy surface: a frozen table or smart enum can acquire computed access members â€” projections that derive secondary policy values from the stored primary â€” without polluting the owner type with unrelated helpers. The block keeps those projections co-located with the vocabulary that governs them rather than scattering them into utility classes.

[ANTI_PATTERNS]:
- A boolean parameter selecting between two bodies is a policy inlined as a knob. The correct form collapses to a smart-enum row or union case: the case carries its own body via a constructor-injected delegate or an override of the abstract member; the caller passes the pre-constructed case. No `if`/`switch` re-derives the body at the dispatch site.
- A nullable flag tail â€” `T? optA = null, T? optB = null, bool doX = false` â€” fragments one optional context into multiple parameters. The collapsed form is one `Option<ContextRecord>` carrying all optional fields together; absence of the record means "default everything"; presence carries the override bundle. `Argument<T>` is the alternative when the factory must distinguish "caller provided this specific value" from "caller omitted, use domain default."
- A `params T[]` overload alongside a semantically identical `params ReadOnlySpan<T>` overload without `[OverloadResolutionPriority(1)]` on the span form means the compiler will prefer the array overload for collection expressions, producing a heap allocation. The span overload must carry priority `1`; the array overload may be kept for binary compatibility or removed in a greenfield context.
- Re-stating a default at every call site â€” `op.Exec(schedule: Schedule.recurs(3) | Schedule.exponential(10 * ms))` â€” is a non-derived default. The schedule belongs on the owning smart-enum row or policy record as a static field: `static readonly Schedule Default = Schedule.recurs(3) | Schedule.exponential(10 * ms)`. Callers that accept the default pass the field; callers that need a different schedule pass their own composed value.
