[ARGUMENT_COLLECTION_MATCH_SEMANTICS]:

[APPSTACK_RESOLUTION_CONTRACT]:
- `AppStack.resolve(attribute, override, fallback)` walks `overrides_stack` from newest to oldest, then walks the live `stack` from innermost frame outward, checking each app and its `_meta_parent` chain. The first non-`None` value wins. This means a per-invocation `app(tokens, print_error=False)` override takes priority over a stored app attribute, which takes priority over an inherited meta-parent attribute. A `fallback` is returned only when the entire resolution yields `None`. [cyclopts/app_stack.py lines 133ŌĆō158, 2026-06-09]
- `AppStack.overrides` collapses the full `overrides_stack` by iterating in reverse (newest first) and calling `setdefault` ŌĆö a value set by a parent frame cannot overwrite a value set by a closer child frame. This is the mechanism by which `app(tokens, console=my_console)` propagates `my_console` through nested meta dispatch without mutating any app state. [cyclopts/app_stack.py lines 93ŌĆō100, 2026-06-09]
- `AppStack(app)` initializes with `stack = [[app]]` ŌĆö the root app is always the bottom frame. Each `with app_stack(apps_for_context)` call pushes a new frame and pops on exit. The context manager handles the full meta-chain: for each app in the resolved list, its live `_meta` is pushed to a separate frame derived from the same `so_far` list prefix. Premature exit (exception) still pops because the `finally` block iterates the same `resolved_apps` list deterministically. [cyclopts/app_stack.py lines 17ŌĆō91, 2026-06-09]

[PARSE_KNOWN_ARGS_VS_PARSE_ARGS_SPLIT]:
- `parse_known_args` returns `(command, BoundArguments, unused_tokens, ignored)` without raising on leftover tokens; `parse_args` calls `_parse_known_args(raise_on_unused_tokens=True)` and raises `UnusedCliTokensError` or `UnknownOptionError` for any leftover. - `parse_commands` is called **twice** inside `_parse_known_args`: once with `include_parent_meta=True` to build `apps_for_context` (which sets up the `AppStack` frame), and once with `include_parent_meta=False` to build `execution_apps` (the actual dispatch chain).
