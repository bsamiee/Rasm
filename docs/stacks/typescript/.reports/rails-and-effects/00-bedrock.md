# rails-and-effects — bedrock (rails collapse algebra)

# Effect 3.21.2 — Rails & Effects Collapse-Depth Findings

All API names, signatures, and option shapes below verified against `node_modules/effect/dist/dts/` at effect 3.21.2. Each finding states the naive sprawl it deletes, the dense spelling, the growth axis, and verified traps.

## [ERROR_RAIL_COLLAPSE]

- **Tagged-error family + `catchTags` single site.** Naive form deleted: per-call-site `try/catch` islands, each with an `instanceof` ladder, plus an ad-hoc `Error1 | Error2 | ...` union hand-threaded through every signature. Dense spelling: declare each error once via `Data.TaggedError("HttpError")<{...}>` (`Data.d.ts:610`, returns `Cause.YieldableError` so it is directly `yield*`-able / `Effect.fail`-able), then recover at ONE site with `Effect.catchTags({ HttpError: h => ..., ValidationError: v => ... })` (`Effect.d.ts:6900`). The handler object keys are constrained to `Extract<E, {_tag}>["_tag"]` — exhaustiveness is type-checked, and each handled tag is subtracted from the residual `E`.
- **Growth axis (the load-bearing shape).** "Error alphabet grows as a CASE, recovery grows as a ROW." A new failure mode = one new `Data.TaggedError` class (a case in the union); a new recovery = one new key in the `catchTags` object literal (a row). The pipeline expression and every intermediate signature are untouched — `E` widens structurally, never by edit.
- **`mapError` vs `catchTag` placement law.** `mapError` (`Effect.d.ts:9525`) transforms `E → E2` *without* recovering (stays in failure channel, narrows/renames at a boundary); `catchTag`/`catchTags` (`6714`/`6900`) *removes* the handled tag from `E` and re-enters success. Rule: `mapError` at the boundary that re-labels foreign errors into the local alphabet; `catchTag` at the boundary that owns recovery. Never `catchAll` (`5582`) then re-`fail` to relabel — that erases tag exhaustiveness.
- **Defect forensics without widening E.** `Effect.sandbox` (`7295`, `Effect<A, Cause.Cause<E>, R>`) exposes the full `Cause` so you can inspect defects/interrupts, handle, then `Effect.unsandbox` (`8523`) back. Naive form deleted: catching `unknown`, manual `instanceof Error` defect sniffing, and polluting `E` with `Throwable`. Projections on the surfaced cause: `Cause.failures` (`Cause.d.ts:753`, `Chunk<E>`), `Cause.defects` (`766`, `Chunk<unknown>`), `Cause.failureOrCause` (`806`, `Either<Cause<never>, E>` — splits expected from defect), `Cause.prettyErrors`/`Cause.pretty` (`1888`) for rendering. `catchAllDefect` (`5877`) and `catchAllCause` (`5737`) recover the non-`E` channel without ever touching `E`. Trap: defects are NOT in `E` by design — only `sandbox` or `*Cause`/`*Defect` operators reach them; `catchTag` cannot.

## [POLICY_VALUE_COLLAPSE]

- **One composed `Schedule` value as data.** Naive form deleted: a hand-rolled retry loop with a counter, `setTimeout`-based backoff math, a jitter `Math.random()` sprinkle, and an `if (attempts > N) throw` guard — typically 15-30 lines scattered per call site. Dense spelling: build the policy once as a value — `Schedule.exponential("100 millis", 2)` (`Schedule.d.ts:1502`) `.pipe(Schedule.jittered)` (`1784`) `.pipe(Schedule.intersect(Schedule.recurs(5)))` (`1655`/`2390`) — then `Effect.retry(self, policy)` (`Effect.d.ts:7449`). The pipeline stays one expression; the policy lives in a vocabulary row.
- **Combinator algebra (all verified).** `union` (`2826`, recur if EITHER recurs → min-delay, longest-running), `intersect` (`1655`, recur if BOTH → max-delay, used to bound an infinite schedule by a count), `andThen`/`andThenEither` (`277`/`341`, run first to exhaustion then switch), `compose` (`733`, feed one schedule's output as the next's input), `jittered`/`jitteredWith` (`1784`/`1806`), `whileOutput`/`whileInput`/`untilOutput`/`untilInput` (`3311`/`3213`/`3069`/`2965`, predicate gates on the schedule's emitted value or fed input), `addDelay`/`modifyDelay`/`delayed` (`176`/`2019`/`1103`), `tapOutput` (`2760`, observe each decision), `check` (`524`, arbitrary recur predicate over `(in, out)`).
- **`Effect.retry` options-object shortcut.** Verified `Retry.Options<E>` (`Effect.d.ts:7323`): `{ schedule?, while?: (e:E)=>bool|Effect<bool>, until?, times?: number }`. So `Effect.retry(self, { times: 5, while: e => e._tag === "Transient" })` collapses the common case to one literal — no `Schedule` import. `Effect.repeat` has the symmetric `Repeat.Options<A>` (`20617`).
- **Growth axis.** Backoff strategy = swap one `Schedule` factory; add a ceiling = `.pipe(intersect(recurs(n)))`; add jitter = `.pipe(jittered)`; add a circuit-style "stop on fatal" = `.pipe(whileInput(isTransient))`. All are row-level edits to the policy value; zero pipeline change. Trap: `union` is "recur if either" (min delay) and `intersect` is "recur if both" (max delay) — confusing them inverts your retry budget; `intersect(recurs(n))` is the canonical "bounded backoff."

## [PIPELINE_SHAPE_COLLAPSE]

- **`Effect.gen` vs `pipe` — objective rule.** Use `Effect.gen` (`5029`) when later steps DEPEND on earlier bound values (sequential data-dependency, multiple bindings reused downstream) — it deletes the `flatMap`-nesting staircase and the `bind` chain. Use `pipe` when the value flows linearly through point-free transforms with no rebinding (`map`/`tap`/`mapError`/`retry` adornments). The rule is dependency topology, not taste: ≥2 reused intermediate bindings → `gen`; pure adornment chain → `pipe`.
- **`Effect.fn` — name + trace + span in one declaration.** Verified `Effect.fn` (`26939`): `fn(name, options?)` returns a generator-function wrapper that produces a *named, automatically-spanned, traced* effect-returning function in a single declaration. Naive form deleted: a plain function + manual `Effect.withSpan("name")` wrap + manual `Effect.annotateCurrentSpan`. `Effect.fnUntraced` (`26948`) is the perf-critical variant that drops tracing. This collapses the "function definition + observability scaffold" pair into one site.
- **Do-notation.** `Effect.Do` (`14836`) + `Effect.bind` (`14869`) + `Effect.bindTo` (`15071`) + `Effect.bindAll` (`14966`, binds an object of effects at once with the same concurrency options as `all`) — the pipe-friendly alternative to `gen` for accumulating a record without a generator body.
- **`Effect.all` / `forEach` concurrency-options collapse.** Verified `all` options (`Effect.d.ts:1044`): `{ concurrency?: Concurrency, batching?: boolean|"inherit", discard?: boolean, mode?: "default"|"validate"|"either", concurrentFinalizers? }`. Naive form deleted: `Promise.all` + manual `p-limit` pool + a `Promise.allSettled` reshape for partial-failure + a separate map to drop results. ONE option object replaces all of it: `concurrency: n` (or `"unbounded"`/`"inherit"`) is the pool, `discard: true` is the result-drop, `mode: "either"` returns `Array<Either>` (no short-circuit), `mode: "validate"` accumulates failures into `Array<Option<E>>`, `batching` toggles request batching. `forEach` (`2509`) carries the identical option set plus an index callback. Trap: `mode: "validate"` changes the success type to `Array<Option<E>>` and the error type accordingly — it does not short-circuit; default mode DOES short-circuit on first failure.

## [RESOURCE_COLLAPSE]

- **One scoped value owns acquire+use+release.** Naive form deleted: an `init()` call and a matching `cleanup()` call manually paired at every use site, with a `try/finally` to guarantee release and a leaked-handle bug whenever an exception escapes between them. Dense spelling: `Effect.acquireRelease(acquire, (res, exit) => release)` (`9752`) yields a `Effect<A, E, Scope>` — release is registered to the scope and runs on scope close regardless of success/failure/interrupt. `Effect.acquireUseRelease` (`10027`) fuses all three into one expression when the use is local. `Effect.addFinalizer` (`10278`) registers ad-hoc cleanup against the ambient scope.
- **`Scope` discharge.** `Effect.scoped` (`11039`, `Effect<A,E,R> → Effect<A,E,Exclude<R, Scope>>`) closes the scope at that boundary — the `Scope` requirement vanishes from `R`, proving at the type level that every acquired resource is released. `scopedWith` (`11025`) gives explicit scope access.
- **`Layer.scoped` — service-lifetime fusion.** `Layer.scoped(tag, scopedEffect)` (`Layer.d.ts:867`) builds a service whose construction is a scoped acquire+release, tied to the layer's (application's) lifetime. Naive form deleted: a service factory + a separate shutdown hook + ordering logic to tear down in reverse. The layer owns construction AND finalization; `Layer.mergeAll`/`provide` (`698`/`1081`) compose lifetimes with correct teardown order automatically.
- **Success/failure cleanup unification.** `Effect.ensuring` (`10349`, runs finalizer on ANY exit) collapses the `try { ... } catch { cleanup } finally { cleanup }` triple. `Effect.onExit` (`10786`, finalizer receives the `Exit`) and `Effect.onError`/`onInterrupt` (`10568`/`8756`) split by outcome when needed. Growth axis: a new resource = one more `acquireRelease` in the scoped block; teardown ordering is automatic (reverse acquisition).

## [VALIDATION_ACCUMULATION]

- **Error-accumulating decode replaces the check-and-throw ladder.** Naive form deleted: a sequential `if (!validA) throw; if (!validB) throw; ...` ladder that reports only the FIRST failure and loses the rest. Dense spelling: `Schema.decode(schema, { errors: "all" })` — verified `ParseOptions.errors?: "first" | "all"` (`SchemaAST.d.ts:1099`) — returns `Effect<A, ParseResult.ParseError, R>` (`Schema.d.ts:352`) whose `ParseError` carries the FULL tree of failures. One schema declaration owns shape + all field validators + accumulation.
- **`Either.all` partition.** `Either.all` (`Either.d.ts:1073`) over an array/record of `Either` short-circuits to the first `Left` OR collects all `Right`s; pair with `Either.getRight`/`getLeft` (`296`/`312`) for partitioning. For the effectful accumulation case, `Effect.all(effects, { mode: "validate" })` returns `Array<Option<E>>` (all failures), and `Effect.validateAll` (`Effect.d.ts:4188`) returns `Effect<Array<B>, NonEmptyArray<E>, R>` — verified return type is `RA.NonEmptyArray<E>` in the error channel, accumulating every element failure rather than stopping at the first.
- **Growth axis.** A new field constraint = one more property in the `Schema.Struct` (a case); accumulation behavior is a single option flag, not per-field plumbing. Trap: default `errors: "first"` short-circuits — accumulation is opt-in; and `decodeUnknown` (`337`) vs `decode` (`352`) differ only in input typing (`unknown` vs `I`), both accumulate identically.

## [SEQUENCING_OPERATOR_DEPTH]

The ladder that deletes manual loops/accumulators (all verified in `Effect.d.ts`): `forEach` (`2509`, the map-with-effect), `iterate` (`19954`, `while(cont(s)) s = body(s)` as a value), `loop` (`20175`, `for(s; cont; step)` collecting outputs), `reduce` (`3186`) / `reduceRight` (`3505`) / `reduceWhile` (`3343`) / `reduceEffect` (`3652`, effectful monoid fold), `mergeAll` (`2879`, fold over concurrent results), `partition` (`3023`, split into `[failures, successes]`), `filter` (`2097`) / `filterMap` (`2235`) / `filterOrFail` (`15816`), `takeWhile` (`1690`) / `dropWhile` (`1407`), `validateAll` (`4188`, accumulate-all) vs `validateFirst` (`4496`, first-success-of-validated), `firstSuccessOf` (`23732`, `Iterable<Effect> → first to succeed`), `raceAll` (`18024`) / `race` (`17628`) / `raceWith` (`18497`), `replicate`/`replicateEffect` (`3786`/`3910`), `whileLoop` (`21411`, low-level). Naive form deleted across the board: `for`/`while` with a mutable accumulator array and a `try` inside the loop body. Each operator is the dense, short-circuiting (or accumulating) value form. Trap: `validateFirst` returns the first SUCCESS (accumulating failures until then) while `firstSuccessOf` races for first success without accumulation guarantees — pick by whether you need the collected failures.

## [CACHING_MEMO_COLLAPSE]

- **One declaration owns memoization+TTL+lookup.** Naive form deleted: a module-level `Map<K,V>`, a `has`/`get`/`set` dance, a manual `Date.now()` expiry check, and a concurrency race when two callers miss simultaneously. Dense spelling tiers: `Effect.cached(self)` (`Effect.d.ts:725`, memoize a zero-arg effect, returns `Effect<Effect<A,E,R>>`), `Effect.cachedWithTTL(self, ttl)` (`309`), `Effect.cachedInvalidateWithTTL` (`517`, returns the cached effect + an invalidator), `Effect.cachedFunction(f, eq?)` (`782`, memoize a unary effectful function keyed by `Equivalence`), `Effect.once(self)` (`824`, run-at-most-once).
- **`Cache` module for keyed, bounded caches.** `Cache.make({ capacity, timeToLive, lookup })` (`Cache.d.ts:175`) returns `Effect<Cache<K,V,E>>` — one value owning capacity eviction + TTL + the lookup function. `Cache.makeWith` (`188`) lets TTL depend on the `Exit` (cache failures shorter than successes). The cache de-duplicates concurrent misses (single in-flight lookup per key) — the race the hand-rolled `Map` cannot.
- **Growth axis.** TTL/capacity policy = constructor fields; a new keyed lookup = one `Cache.make`. Trap: `cached` returns a NESTED effect (`Effect<Effect<...>>`) — you must run the outer once to get the memoized inner; forgetting this re-memoizes per call and defeats the cache.

## [STATE_COLLAPSE]

- **One cell replaces scattered mutable fields.** Naive form deleted: several `let` fields on an object, mutated in place, with read-modify-write races and no atomicity. Dense spelling: `Ref.make(initial)` (`Ref.d.ts:63`) for a single atomic cell; mutate via `Ref.update` (`185`) or read-and-write atomically via `Ref.modify(self, s => [result, newState])` (`121`) — ONE atomic step returning both the derived result and the next state, deleting the get/compute/set triple and its race window. `Ref.modifySome` (`137`) for partial updates.
- **Effectful and observable variants.** `SynchronizedRef` (`SynchronizedRef.d.ts:63`) adds `modifyEffect`/`updateEffect` (`169`/`265`) — atomic update where the transition itself is an `Effect` (serializes effectful transitions, no lock). `SubscriptionRef` (`SubscriptionRef.d.ts:159`) is a `Ref` whose `.changes` is a `Stream` of every value — collapses "mutable field + manual observer/event-emitter list" into one cell. `Ref.modify` is the canonical "compute-and-commit atomically"; `SubscriptionRef.modify` (`164`) does the same while publishing.
- **Growth axis.** A new piece of state = one more `Ref` (or one more field in a single record-typed `Ref`); concurrency-safety and atomicity come free. Observability = swap `Ref` → `SubscriptionRef` (same `modify` shape). Effectful transition = swap `Ref` → `SynchronizedRef` and use `modifyEffect`. Trap: `Ref.modify` is for pure transitions only — an effectful transition on a plain `Ref` cannot be atomic; you must use `SynchronizedRef.modifyEffect`, else interleaving corrupts state.

## Further Considerations

- **`mode: "either"` vs `validateAll` are not interchangeable at the type level.** `Effect.all(xs, { mode: "either" })` keeps the original element order with `Array<Either<E, A>>` (zero failures still typed as possibly-`Left`), whereas `validateAll` collapses to `Effect<Array<A>, NonEmptyArray<E>, R>` — the latter is the right collapse when "any failure fails the batch but report all," the former when downstream needs per-element success/failure positionally. Choosing wrong forces a re-partition that re-introduces the sprawl you deleted.

- **`Schedule` output type is load-bearing for `whileOutput`/`tapOutput` chaining.** `Schedule.exponential` emits `Duration`, `recurs` emits `number`, `fixed` emits `number` — when you `intersect`/`compose` heterogeneous schedules the output becomes a tuple, and `whileOutput` predicates must match that composed shape. The doctrine page should show the composed-output type explicitly (e.g. `Schedule<[Duration, number], ...>` after `exponential.pipe(intersect(recurs(n)))`) so readers don't write predicates against the wrong arm.

- **`Effect.fn` + `Effect.gen` is the single densest function form and should be the page's headline idiom.** `Effect.fn("name")(function*(args){ ... })` fuses: named function, generator body (the `gen` dependency-threading), an auto-span named `"name"`, and stack-trace capture — four concerns in one declaration. This is strictly denser than the common `const f = (a) => Effect.gen(...).pipe(Effect.withSpan("f"))` triple and is the clearest demonstration that "the unit of design is the dispatch surface, not the file."
