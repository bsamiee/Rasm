# shapes — bedrock (Effect shape-construct surface, effect@3.21.2)

# Effect 3.21.2 — Domain-Shape Ownership API Truth

`[SCHEMA_CLASS]:` ONE declaration that owns shape + codec + opaque type + structural identity simultaneously
- `Schema.Class<Self>(identifier: string)(fields, annotations?)` — the `<Self>` generic is mandatory; omitting it yields `MissingSelfGeneric<"Class">` (compile error). Pattern is `class User extends Schema.Class<User>("User")({...}) {}`.
- Runtime base is `Base: data_.Class` (`cjs/Schema.js:4971`), and `Data.Class = internal.Structural` (`cjs/Data.js:149`). So every `Schema.Class` instance inherits `StructuralPrototype` → automatic `Equal.equals` structural equality + `Hash.hash`. This is the free win: declare once, get an opaque nominal type (the `class` name), schema codec, AND Effect `Equal`/`Hash`.
- Constructor evidence: `new User(props, options?)` validates by default. `options` is `MakeOptions = boolean | { readonly disableValidation?: boolean }` — passing `true` or `{disableValidation:true}` skips validation. Static `User.make(...args)` is `ConstructorParameters`-typed (alternate construction path).
- Instance methods/getters defined in the class body are preserved (`Inherited`/`Proto` type params); `extend<Next>(id)(newFields)` adds fields + inherits prior class as `Inherited`.
- Exposed surface on the class value: `.ast` (an `AST.Transformation`), `.fields`, `.identifier`, `.annotations(...)`, `.extend(...)`. The class IS a `Schema<Self, Simplify<I>, R>` — directly usable in `Schema.decode*`, `Arbitrary.make`, `Pretty.make`, `JSONSchema.make`, `Schema.equivalence`.
- Optional fields: required-vs-void constructor arg is computed (`RequiredKeys<C> extends never ? void | C : C`) — a class with all-optional fields constructs via `new User()`.

`[SCHEMA_TAGGED_FAMILY]:` `_tag` discriminant baked in, share the `Class` machinery
- `Schema.TaggedClass<Self>(identifier?)(tag, fields, annotations?)` — injects `readonly _tag: tag<Tag>` literal into fields; constructor omits `_tag` from required args (`Struct.Constructor<Omit<Fields,"_tag">>`). Same `data_.Class` base → structural `Equal`/`Hash`.
- `Schema.TaggedError<Self>(identifier?)(tag, fields, annotations?)` → `TaggedErrorClass` which `extends Class<..., cause_.YieldableError>` (the `Proto` type param is `YieldableError`). This is the load-bearing error fact: a `Schema.TaggedError` instance IS a `Cause.YieldableError` → directly `yield*`-able inside `Effect.gen` (fails the effect with itself), carries a `.cause`/stack, and is a schema (admits encode/decode of the error across a wire). It additionally exposes `readonly _tag: Tag` at the type level.
- `Schema.TaggedRequest<Self>(identifier?)(tag, {payload, success, failure}, annotations?)` → `TaggedRequestClass extends Class<..., TaggedRequest<...>>`. Carries `.success` and `.failure` schemas + `_tag`. This is the only construct binding a request shape to typed success/failure schemas in one declaration; it is the substrate for `@effect/experimental` `Machine` procedures and for `RequestResolver`/RPC.

`[SCHEMA_COMBINATORS]:` structural composition, all return `Schema`/`PropertySignature`
- `Schema.Struct(fields)` → `Struct<Fields>` (a `Schema`), `.make(props, MakeOptions?)`, `.fields`, `.annotations(...)`, `.pick`/`.omit`. Structural type — NOT opaque, NOT a class, NO inherited `Equal`/`Hash` unless wrapped (decoded values are plain objects).
- `Schema.Union(...members)` → `Union<Members>` with `.members`; `Schema.Literal(...literals)` → `Literal` (single literal returns `SchemaClass<L>`); `Schema.Enums(obj)`; `Schema.TemplateLiteral(...)`.
- `Schema.brand(brand, annotations?)(self)` → `brand<S,B>` extends `BrandSchema<Type & Brand<B>, ...>` with `.make(unbranded, MakeOptions?)`. `Schema.fromBrand(constructor, annotations?)(self)` bridges an existing `Brand.Constructor` (from the `Brand` module) into a schema → `BrandSchema`. Branding is purely a TYPE-level nominal tag (`A & Brand<K>`); no runtime wrapper.
- Refinement/transform: `Schema.filter` (predicate), `Schema.filterEffect` (effectful predicate), `Schema.transform(from, to, {decode, encode, strict?})`, `Schema.transformOrFail(from, to, {decode, encode})` (decode/encode return `ParseResult`), `Schema.transformLiteral`, `Schema.attachPropertySignature(key, value)` (adds a constant discriminant property post-decode — canonical pattern for tagging untagged unions).
- Optional fields: `Schema.optional(self)` → `?:` with `| undefined`. `Schema.optionalWith(self, options)` where `OptionalOptions<A>` is a discriminated set: `{exact?, nullable?}` (presence semantics), `{default: LazyArg<A>, exact?, nullable?}` (decode-time default), or `{as: "Option", exact?, nullable?, onNoneEncoding?}` (decode to `Option<A>`). `Schema.withDefaults({constructor, decoding})` sets constructor-side + decode-side defaults on a property signature.
- Recursion: `Schema.suspend(() => Schema<A,I,R>)` → `suspend<A,I,R>` for self-referential schemas (must annotate `identifier` for JSONSchema/Arbitrary termination).

`[SCHEMA_ANNOTATIONS_DERIVATION]:` one schema, many derived artifacts
- `Annotations.Schema<A>` carries `identifier`, `title`, `description`, `examples`, `default`, `documentation`, `jsonSchema`, plus the derivation hooks: `arbitrary?: (...) => LazyArbitrary<A>`, `pretty?: (...) => Pretty<A>`, `equivalence?: (...) => Equivalence<A>`. These let you OVERRIDE the auto-derived instance per-schema.
- Derivation surfaces (each takes any `Schema<A,I,R>`): `Arbitrary.make(schema): FastCheck.Arbitrary<A>` (+ `Arbitrary.makeLazy`); `Pretty.make(schema): (a:A)=>string`; `Schema.equivalence(schema): Equivalence<A>`; `JSONSchema.make(schema, options?)`. `Schema.standardSchemaV1(schema, opts?): StandardSchemaV1<I,A> & SchemaClass<A,I,never>` — produces the cross-library Standard Schema interface (requires `R = never`).
- Net: a single `Schema.Class`/`Struct` declaration is the root from which arbitrary (PBT), pretty-printer, structural equivalence, JSON Schema, codec, and Standard-Schema all derive — eliminating the type→interface→schema→validator duplication.

`[DATA_MODULE]:` structural identity WITHOUT schema/codec overhead
- All Data values get `StructuralPrototype` → `Equal`/`Hash` structural equality (`Equal.equals(Data.struct({a:1}), Data.struct({a:1})) === true`; plain object comparison is `false`).
- `Data.struct(a)`, `Data.tuple(...)`, `Data.array(as)` (+ `unsafe*` variants skip prototype copy); `Data.case<T>()` returns a constructor; `Data.tagged<T>(tag)` constructor with fixed `_tag`.
- `Data.Class` and `Data.Structural` are constructable base classes (`new`). `Data.TaggedClass(tag)` → constructor injecting `_tag`, omitting `_tag` from args. NO schema, NO validation, NO codec — pure runtime shape + structural identity. This is what `Data` gives that `Schema.Class` does NOT need (Schema.Class already has it) but at ZERO schema/AST cost when you do not need wire encoding or validation.
- `Data.Error` → `new (...) => Cause.YieldableError & A`; `Data.TaggedError(tag)` → `Cause.YieldableError & {_tag} & A`. Yieldable in `Effect.gen`, structural `Equal`. Difference vs `Schema.TaggedError`: Data's error has NO schema (cannot encode/decode across a wire), NO `ParseResult` admission, NO annotations/HTTP-status hook — it is the in-process error rail; `Schema.TaggedError` is the serializable/boundary error rail.

`[DATA_TAGGED_ENUM]:` closed ADT in one statement with generated matchers
- `Data.taggedEnum<Union>()` returns `{ [Tag]: constructor } & GenericMatchers`. Each tag key is a constructor; `_tag` is auto-injected and omitted from constructor args.
- `GenericMatchers` provides `$is(tag)` → type guard `(u:unknown) => u is Extract<A,{_tag:tag}>`, and `$match` → exhaustive matcher accepting a handler record keyed by every `_tag`. `$match` has a curried form (`$match(handlers)(value)`) and a direct form.
- Generics: declare `interface Def extends Data.TaggedEnum.WithGenerics<N> { readonly taggedEnum: MyADT<this["A"], this["B"]> }` then `Data.taggedEnum<Def>()` — yields generic constructors (`Success<A>`, `Failure<E>`). Arities 1–4 verified (`WithGenerics<1..4>`).
- Net: `Data.TaggedEnum` + `$match` is the lightweight closed-ADT surface — structural equality, exhaustive dispatch, generic params, NO schema. Use when the ADT lives only in-process; reach for `Schema.Union(...TaggedClass)` when the ADT must serialize.

`[MATCH_MODULE]:` total dispatch over any value/ADT, schema-independent
- Builder roots: `Match.type<I>()` (match by type, defer input) and `Match.value(i)` (match a concrete value). Terminal-shortcut roots `Match.typeTags<I>()(handlers)` / `Match.valueTags(handlers)` build an exhaustive `_tag` dispatcher in one call returning `(input)=>Ret`.
- Pattern arms: `Match.when(pattern, fn)`, `Match.whenOr(...patterns, fn)`, `Match.whenAnd(...patterns, fn)`, `Match.not(pattern, fn)`. Tag arms: `Match.tag(...tagValues, fn)`, `Match.tagStartsWith(prefix, fn)`, `Match.tags(handlerRecord)` (partial), `Match.tagsExhaustive(handlerRecord)` (all `_tag` required). Discriminator on arbitrary field: `Match.discriminator(field)(...)`, `Match.discriminators(field)(record)`, `Match.discriminatorsExhaustive(field)(record)`.
- `Match.withReturnType<Ret>()` pins the return type across all arms (compile error if an arm diverges).
- Terminators: `Match.exhaustive` (requires remainder `never`, total — compile error on missing case), `Match.orElse(fn)` (fallback), `Match.orElseAbsurd`, `Match.option` (→ `Option<A>`), `Match.either` (→ `Either<A, unmatched>`). Refinement primitives: `Match.string/number/boolean/bigint/symbol/date/record/defined/nonEmptyString/is(...literals)/instanceOf/instanceOfUnsafe/any`.
- Net: `Match` is the dispatch surface that works on `Data.TaggedEnum`, `Schema` tagged unions, or raw discriminated unions equally — it does not require the ADT to be a Data/Schema construct. `Data.taggedEnum().$match` is the per-ADT generated equivalent for closed Data enums.

`[BRAND_MODULE]:` nominal/refined value tags, runtime-light
- `Brand<K>` is the phantom marker; `Branded<A,K> = A & Brand<K>`. `Brand.Constructor<A>` is callable `(unbranded) => A` (throws on invalid) plus `.option(x): Option<A>`, `.either(x): Either<A, BrandErrors>`, `.is(x): x is A` (refinement).
- `Brand.nominal<A>()` — no validation, pure type tag. `Brand.refined<A>(refinement|predicate, onFail)` — validating constructor. `Brand.all(...constructors)` — intersect multiple brand constructors into one (shared base enforced via `EnsureCommonBase`). `Brand.unbranded(x)` strips the tag.
- `Schema.fromBrand(constructor)` lifts a `Brand.Constructor` into a `BrandSchema` (so brand validation participates in decode); `Schema.brand(name)` creates the brand directly at the schema layer. Use `Brand` standalone for non-schema value objects; use `Schema.brand`/`fromBrand` when the branded value crosses a codec boundary.

`[SERVICE_OWNERSHIP]:` three tiers of dependency declaration
- `Effect.Service<Self>()(key, make)` where `make` is exactly ONE of `{effect}`, `{scoped}`, `{sync}`, `{succeed}` (+ optional `dependencies: ReadonlyArray<Layer>`, `accessors?: boolean`). Returns `Service.Class<Self, Key, Make>` which is simultaneously: a constructor (`new (impl)`), a `Context.Tag<Self,Self>`, a `.make(impl)` factory, a `.use(body)` accessor (auto-wraps Effect/Promise/value), `.key`, and a `.Default` layer (or `.DefaultWithoutDependencies` + dep-aware `.Default` when `dependencies` are supplied). With `accessors:true`, static method/constant proxies are generated (`Tag.Proxy`). Forbidden impl property names enumerated in `Service.ProhibitedType` (`make`, `key`, `use`, `context`, `_tag`, etc.). The `<Self>` generic is mandatory (`MissingSelfGeneric` otherwise). This is the densest service surface: one class = tag + implementation + default layer + accessors.
- `Context.Tag<Id,Value>` / `Context.GenericTag<Id,Service>(key)` — bare service identity, no default layer, no accessors. `class Foo extends Context.Tag("Foo")<Foo, Shape>() {}` for the class form (`TagClass`).
- `Context.Reference<Self>()(id, { defaultValue })` — a `Tag` with a `defaultValue: () => Value` so the service is ALWAYS available without a layer (FiberRef-backed default). Use for config-like always-present services.
- Net ladder: `Context.Reference` (always-defaulted) → `Context.Tag`/`GenericTag` (identity only) → `Effect.Service` (identity + impl + layer + accessors in one declaration).

`[STATE_MACHINE_OPTIONS]:` two surfaces, both present
- `@effect/experimental/Machine`: `Machine.make(initialize)` / `Machine.makeWith<State,Input>()(initialize)` where `initialize` returns a `ProcedureList<State, Public, Private, R>`. Procedures: `@effect/experimental/Machine/Procedure` — `Procedure.make<Requests,State>()<Req>()(tag, handler)` where `handler` returns `Effect<[response, nextState], Error, R>`; `Procedure.makeSerializable` binds a `Schema.TaggedRequest`. Public procedures are driven by `Schema.TaggedRequest`-typed requests. Run via `Machine.boot(machine, input?)` → `Actor`; `actor.send(request)` returns typed `Effect<Success, Error>`; `actor.join`; `Actor extends Subscribable<State>` (observable state). `Machine.makeSerializable({state, input}, init)` + `Machine.snapshot`/`Machine.restore` give persistable actors (`ParseResult.ParseError` channel). `Machine.retry(policy)` retries init. Actor-model: heavyweight, async, persistable, request/reply.
- Lightweight alternative (no new dep): `Data.TaggedEnum<State>` + `Match.tagsExhaustive` (or `$match`) for synchronous total transition functions `(state, event) => state`. No actor, no async mailbox, no serialization — pure fold-style state machine. Both exist; choose by need for actor semantics + persistence vs. pure transition algebra.

`[CROSS_CUTTING]:` opaque-vs-structural, yieldability, equality, construction
- OPAQUE nominal types: `Schema.Class`/`TaggedClass`/`TaggedError`/`TaggedRequest` (the `class` name is the type), `Data.Class`/`TaggedClass` (constructor identity), branded schemas (`A & Brand<K>` is nominally distinct). STRUCTURAL: `Schema.Struct`, `Data.struct`/`tuple`/`array`, raw object unions.
- Effect `Equal`/`Hash` integration (structural): ALL `Data.*` constructors and ALL `Schema.*Class`/`TaggedError`/`TaggedRequest` (because their runtime base is `Data.Class`/`StructuralPrototype`). NOT integrated: decoded `Schema.Struct` values (plain objects — reference equality only).
- Yieldable in `Effect.gen`: `Schema.TaggedError` (its class extends `Class<..., YieldableError>`) and `Data.TaggedError`/`Data.Error` (return `Cause.YieldableError`). `Schema.TaggedRequest` is a `Request`, sent via resolver/actor, not directly yielded as an error. Plain `Schema.Struct`/`Data.struct` are not yieldable.
- Constructor evidence: `new` on `Schema.Class` family VALIDATES by default; `new User(props, {disableValidation:true})` or `new User(props, true)` disables it; static `.make(...)` is the typed-args alternate. `Data.*` constructors NEVER validate (no schema). `Brand.Constructor` `(x)` THROWS on invalid; `.option`/`.either` are the safe constructors; `BrandSchema.make(x)` validates via the brand refinement.
- HTTP status (platform boundary): `@effect/platform/HttpApiSchema` carries the status annotation. `HttpApiSchema.annotations({ ..., status?: number })`, `HttpApiSchema.getStatusSuccess(schema)`, `HttpApiSchema.getStatusError(schema)`, `HttpApiSchema.Empty(status)`. `@effect/platform/HttpApiError` defines its errors as `Schema.TaggedErrorClass` instances (e.g. `HttpApiDecodeError`) — confirming the boundary error rail is `Schema.TaggedError` (serializable + status-annotatable), while `Data.TaggedError` stays in-process. The status lives as a schema annotation on the `Schema.TaggedError`, read at the HTTP boundary — `Data.TaggedError` has no such hook.

`[OWNERSHIP_DELTA_TABLE]:` precise capability difference where constructs overlap
- `Schema.Class` vs `Data.Class`: identical structural `Equal`/`Hash` + opaque type + class body. `Schema.Class` ADDS schema/codec/validation/annotations/derivation (arbitrary, pretty, equivalence, JSONSchema, StandardSchema) at the cost of an AST. `Data.Class` is the zero-codec subset.
- `Schema.TaggedError` vs `Data.TaggedError`: both yieldable + structural `Equal` + `_tag`. `Schema.TaggedError` ADDS schema admission (encode/decode across wire), annotations including HTTP `status` (via `HttpApiSchema`), and participation in `ParseResult`. `Data.TaggedError` is the in-process-only error.
- `Schema.Union(...TaggedClass)` vs `Data.taggedEnum`: both closed ADTs with exhaustive dispatch (`Match.tagsExhaustive` / `$match`). Schema union ADDS serialization + derivation; Data taggedEnum ADDS built-in `$match`/`$is` generated matchers + free generic-type support with zero AST.
- `Effect.Service` vs `Context.Tag` vs `Context.Reference`: all are `Tag`s. `Context.Tag` = identity only. `Context.Reference` ADDS an always-present `defaultValue` (no layer required). `Effect.Service` ADDS the implementation, the `.Default` layer, `.use` accessor, dependency wiring, and optional static accessor proxies in one declaration.
- `Schema.brand` vs `Brand.nominal`/`refined`: `Brand.*` is pure type-tag + runtime validating constructor with `.option`/`.either`/`.is`, no codec. `Schema.brand`/`fromBrand` lift the same brand into a `BrandSchema` so the tag enforces during decode/encode at a boundary.

Further Considerations
- `Schema.Class.make` has a subtle signature trap: it is `make<C>(this: C, ...args: ConstructorParameters<C>)` — calling `User.make({...})` works, but it does NOT accept the `MakeOptions` disable-validation flag the way `new User(props, {disableValidation})` does; the validation-skip escape hatch only exists on the `new` path, not the static `.make`. A doctrine that standardizes on `.make` loses the disable-validation lever.
- `Effect.Service` silently forks its layer surface on whether `dependencies` is present: with deps you get BOTH `.Default` (deps baked in) AND `.DefaultWithoutDependencies`; without deps you get only `.Default`. A doctrine template that always references `.DefaultWithoutDependencies` will fail to typecheck for dependency-free services — the ownership rule must branch on dependency presence.
- The `@effect/experimental/Machine` actor requires `Public` procedures to be `Schema.TaggedRequest` (serializable) but `Private` procedures may be the lighter `Procedure.TaggedRequest` (non-schema) — so a machine can mix a serializable public API with in-process-only private transitions. This is the bridge between the two state-machine surfaces: `Data.TaggedEnum`-style internal transitions can coexist as private procedures under a serializable public actor boundary.
