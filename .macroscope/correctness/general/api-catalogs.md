# [API_CATALOGS]

An `.api/` catalogue is an external-library capability index letting a design-page fence author call the surface with zero external lookups — judged as production law, never as documentation. `.claude/skills/docgen/templates/api-catalog.template.md` owns the schema; two-tier placement law (substrate vs folder) stays with the planning-corpus registry rules.

## [01]-[SCHEMA]

- A catalogue keeps the fixed ordered schema — package surface, public types, entrypoints, implementation law with explicit accept/reject bounds; a dropped section, a renamed column outside the closed vocabulary, or a catalogued member unverifiable against the real distribution is a finding.
- Signatures are abbreviated in-cell code spans carrying member name, argument shape, and load-bearing return — never a transcription fence (the full declaration lives on the composing design page) and never a run-on `- Members: a, b, c` bullet.

## [02]-[CURRENCY]

- A catalogue presents only the library's live current surface: an obsolete, deprecated, or superseded member, overload, or import path drops whole — the live replacement catalogued, never the dead form beside a pointer — and every replacement narration ("use Y instead", "replaced by", "formerly", "supersedes", "in older versions", "legacy") removes silently with no tombstone. Legacy anchoring hides in many shapes; hunt the semantics, not a keyword list.
- A version field or version number anywhere in a catalogue is a finding — the central manifest owns versions.
- A domain symbol genuinely named `Version`, `Legacy`, or `Obsolete` that IS the live current API is real surface, never flagged.

## [03]-[STACKING_DEPTH]

- `[STACKING]` grades at operator depth: each bullet names the exact member-level seam — this package's output flowing into the named sibling member with arguments (`to_builtins(struct, str_keys=True)` feeding `Span.set_attributes`), the sibling anchored by its catalogue path — and one within-library row naming the composing owner page or kernel.
- Shallow shapes demanding the member-level seam: a bare package-name row ("integrates with numpy"), a "pairs well with" phrase, a seam lacking a concrete member on either end, a sibling cited without its catalogue path, a missing within-library row, and `(none)` where the language-substrate tier carries an obvious composing sibling.
- A stacking seam verifies at both ends: a cited member the named sibling catalogue does not carry is a phantom seam — the author reads the substrate tier and each named sibling catalogue before the seam lands.
- `[RAIL_LAW]` `Reject` names the exact hand-rolled form THIS package forecloses ("a hand-rolled tokenizer where `parse_content_stream`/`TokenFilter` exist"); a Reject generic enough to paste unchanged between unrelated packages is the byte-copy defect. `[TOPOLOGY]` carries only behavioral law a signature cannot show — an int64 constraint ceiling, a mutating-vs-returning family split, a sentinel distinction — never signature restatement.
- Positive duty rides beside the negative: a catalogue spelling a canonical chained rail across sibling catalogues at member grain, or a Reject naming the specific reimplementation the surface deletes, is conformance worth naming.
