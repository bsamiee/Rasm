# protobuf-contracts bedrock

## message law

- Generated messages implement `IMessage<T>`, `IBufferMessage`, `IDeepCloneable<T>`, `IEquatable<T>`; the contract surface is `Parser` (static `MessageParser<T>`), `Descriptor` (static metadata), `WriteTo`, `MergeFrom`, `CalculateSize`, `Clone`. Handwritten byte DTOs beside generated messages are the rejected form — the generated type IS the wire vocabulary.
- `MergeFrom` is merge, not replace: scalar fields overwrite, singular message fields recursively merge, repeated fields APPEND, map entries overwrite per key. Parsing into a reused message object accumulates repeated-field content across parses — the non-obvious corruption source; `Parser.ParseFrom` always allocates fresh and is the default read spelling.
- `MessageParser<T>.ParseFrom` accepts `ReadOnlySequence<byte>` — fragmented pipe buffers parse without coalescing into one array; this is the read shape for framed IPC payloads arriving from a pipe reader.
- `CodedInputStream.CreateWithLimits(stream, sizeLimit, recursionLimit)` binds both parse bounds; the package default recursion limit is 100 — deeply nested recursive contracts (trees, expression graphs) hit it silently in production payloads that tests never reach; the limit is a declared policy value per contract family, not a constant to discover at failure time.
- `UnknownFieldSet` preserves unrecognized fields through parse-mutate-serialize round-trips: an old reader re-emitting a new writer's message does not strip fields it cannot name. This is the structural mechanism that makes additive evolution safe across mixed-version processes — receipt relays and store-and-forward hops keep foreign fields intact without understanding them.
- Proto3 presence: explicit `optional` fields generate `HasX` members and surface in `FieldDescriptor.HasPresence`; non-optional scalars conflate default-value-and-absent. Absence-bearing contract fields are declared `optional` at authoring time so the boundary can project absent-vs-default into the option carrier — retrofitting presence later is a wire-compatible but reader-behavior-changing edit.
- `Empty` is the unit payload; scalar wrapper messages (`google.protobuf.*Value`) are the nullable-scalar rows; both arrive as import protos shipped with the generator package alongside `any`, `timestamp`, `duration`, `field_mask`, `wrappers`.

## ByteString zero-copy

- `ByteString` is the immutable byte carrier with content-based equality (`==`, `Equals`, hash); read access is allocation-free via `Span` (`ReadOnlySpan<byte>`) and `Memory` (`ReadOnlyMemory<byte>`); `ToByteArray` always copies.
- `UnsafeByteOperations.UnsafeWrap(ReadOnlyMemory<byte>)` wraps caller memory with zero copy — the producer contract is that the wrapped memory is never mutated afterward; wrapping a pooled buffer that returns to the pool while the message is alive is the named use-after-free shape. Use it for large one-shot payloads (artifact bytes, tensor blocks) where the copy dominates; `CopyFrom(ReadOnlySpan<byte>)` is the safe default everywhere else.
- Ingest rows: `CopyFrom(byte[] / span / string+encoding)`, `CopyFromUtf8`, `FromBase64`, `FromStream`/`FromStreamAsync(stream, ct)`; egress rows: `WriteTo(Stream)`, `ToBase64`, `ToStringUtf8`. The base64 rows pair with text-constrained transports; the stream rows avoid intermediate arrays on both directions.
- Zero-copy composes the cap law: a `ByteString`-bearing field contributes its exact length to `CalculateSize`, so producer-side cap pre-checks (owned by the IPC lane) are precise even for wrapped foreign memory — no serialization probe needed.

## descriptor-diff drift law

- Every generated contract self-describes: `FileDescriptor.SerializedData` is the file's descriptor proto as bytes, `ToProto()` the structured form; `FileDescriptor.BuildFromByteStrings(descriptorData[, ExtensionRegistry])` reconstructs descriptor graphs from published bytes (dependency-ordered input). The drift gate therefore needs no build-step plumbing: the running process publishes `SerializedData` per contract file, the gate rebuilds both generations and diffs structurally. The build-time alternative (a descriptor-set emitted via extra generator arguments) has no first-class MSBuild switch and is the weaker route — runtime descriptors are always exactly what the binary carries.
- Diff algebra over `MessageDescriptor.ToProto()` (`DescriptorProto`) and `FieldDescriptor` rows (`FieldNumber`, `FieldType`, `IsRepeated`, `IsPacked`, `IsMap`, `HasPresence`, `JsonName`):
  - additive (tolerated): new message, new field with an unused number, new enum value, new service method, comment/order churn;
  - breaking (rejected): field-number reuse with different type or meaning, `FieldType` change on an existing number, singular<->repeated flip, packed-ness flip on repeated numerics, oneof membership change, field rename on a live number when any JSON projection exists (`JsonName` is part of the JSON contract even though binary ignores names);
  - removal (required ceremony): a removed field's number and name enter `DescriptorProto.ReservedRange`/reserved names; absence of the reservation classifies the removal as breaking because the number is re-claimable.
- The gate is an injected classifier value — `(oldDescriptorSet, newDescriptorSet) -> verdict` — supplied to the consumer seam (discovery-manifest checksum gate, store schema stamps), so transport owns the diff mechanics and consumers own only the verdict policy; reserved-range presence is the single signal distinguishing intentional removal from drift.
- Service-level drift walks `FileDescriptor.Services` -> `ServiceDescriptor.Methods`: a removed or re-typed method is breaking for callers (manifesting at runtime as unimplemented-method status); an added method is additive. `MessageDescriptor.Fields.InDeclarationOrder()` gives the stable iteration for canonical checksum computation — descriptor bytes are not canonical across generator versions, so the checksum hashes the diff-relevant projection, never raw `SerializedData`.
- `OriginalNameAttribute` preserves proto names where C# names diverge; reflection-driven tooling reads descriptor names, never CLR names, so renames in C# (via generator options) are not wire events.

## well-known types and JSON projection

- `Any.Pack(message)` embeds type URL plus bytes; `Any.Is(descriptor)` tests, `Any.Unpack<T>()` extracts. `Any` is the extension point for heterogeneous payload slots (fault details, attach-manifest extensions) — the closed alternative (oneof) is preferred when the case family is owned, `Any` only where foreign packages extend without editing the owner.
- JSON projection of `Any` and descriptor-driven formatting require `TypeRegistry`; `JsonFormatter.Settings` rows: `WithFormatDefaultValues` (emit zero-values), `WithTypeRegistry`, `WithFormatEnumsAsIntegers` (numbers instead of names), `WithPreserveProtoFieldNames` (snake_case instead of lowerCamel). `JsonParser.Settings` rows: `WithIgnoreUnknownFields` (forward-compatible read), `WithRecursionLimit`, `WithTypeRegistry`. Protobuf-JSON is an edge projection for inspection and foreign surfaces — binary remains the contract; round-tripping through JSON loses unknown fields, which silently defeats the additive-evolution mechanism.
- `FieldMask.FromString("a.b,c")` carries field paths for partial-update intents; the mask travels with the request and the applier resolves paths against descriptors — masks are the protobuf-native sparse-update vocabulary where patch semantics stay inside the binary contract (the JSON-document patch law lives at the server exposure seam).
- `Timestamp.FromDateTime` requires UTC kind; `Duration.FromTimeSpan` is the bridging row for BCL spans — both exist but the temporal law below routes domain time through the calendar-aware bridge instead.

## temporal bridges

- The suite temporal wire law: domain time crosses wire contracts as well-known and common-proto types, never as serialized temporal text. Outward projection is the extension surface on domain temporal values: `ToTimestamp` (instant), `ToProtobufDuration` (duration), `ToDate` (calendar date), `ToTimeOfDay` (clock time), `ToProtobufDayOfWeek`; inward projection mirrors: `ToInstant`, `ToNodaDuration`, `ToLocalDate`, `ToLocalTime`, `ToIsoDayOfWeek`.
- Verified range contracts are boundary faults to project onto typed rails at the call site (the package throws): instants before the common-era epoch floor reject; durations outside the ±315_576_000_000 s window reject; invalid wire payloads, leap-second and 24:00 time-of-day payloads reject; the unspecified day-of-week wire value maps to the none case rather than throwing — the one sentinel-to-vocabulary projection in the family.
- The STJ temporal bridge is one options mutation at contract composition: `ConfigureForNodaTime(options, provider)` or `ConfigureForNodaTime(options, NodaJsonSettings)`; `NodaJsonSettings` exposes sixteen converter slots (instant, interval, local date/time/datetime, annual date, year-month, date-interval, offset family, zone, duration, period, zoned) so per-suite overrides are slot writes, not converter re-registration. `WithIsoIntervalConverter`/`WithIsoDateIntervalConverter` swap interval representation from the structured object to the ISO-8601 interval string.
- Converter pattern facts that bind the wire contract: instants and local datetimes serialize extended-ISO; offset datetimes serialize RFC 3339; the default interval converter emits an object whose `Start`/`End` property names pass through the options' property-naming policy and whose instant values delegate to the registered instant converter — interval JSON therefore changes shape when the suite naming policy or instant slot changes, which is the non-obvious contract coupling to pin in golden bytes.
- The attribute route (`NodaTimeDefaultJsonConverterAttribute` per property) hard-pins the default zone provider and default converter choices and cannot carry suite overrides — it serves isolated DTOs only; suite contracts use the options route exclusively. Zone-bearing types (`ZonedDateTime`, `DateTimeZone`) require an explicit `IDateTimeZoneProvider`; serializing them without configuring the bridge is the failure that surfaces only at first write.
- Calendar guard: date-bearing converters validate ISO calendar and reject non-ISO calendar values at write — calendar variance is a domain concern that must be projected to ISO before the wire, never encoded ad hoc.

## one canonical wire vocabulary, N boundary projections

- One concept owns one wire message; every transport surface (binary RPC, JSON edge, patch document, tabular export) is a projection of that message, derived at the boundary — parallel DTOs per surface are the foreclosed form. The generated message is the canonical shape; domain owners admit from it and project into it at exactly one seam per direction.
- Enum drift across projections: binary carries numbers, JSON carries names (or numbers under the integer row); an enum value rename is therefore binary-additive but JSON-breaking — the classifier treats enum renames at JSON-exposed surfaces as breaking, interior-only enums as additive.
- Repeated and map fields surface as `RepeatedField<T>`/`MapField<K,V>` — generated-contract internals, mutated in place during construction and projected to immutable domain collections at admission; leaking them past the boundary leaks mutability into the interior.

## divergent

- Descriptor-diff as the one evolution law: the same classifier verdict gates three consumer seams — peer attach (manifest checksum compare), store open (schema stamp compare), replay decode (journal generation compare) — so contract evolution is legislated once and consumed as a verdict enum everywhere; per-seam ad hoc compatibility checks are the foreclosed family. The canonical-checksum projection (sorted field rows of number:type:label per message, method rows per service, reserved ranges) is itself a tiny protobuf message, making the gate's own contract evolvable under its own law.
- ByteString as the staging seam: `UnsafeWrap` over rented memory, `Span` reads on receive, and `CalculateSize`-driven cap pre-checks fuse into a zero-copy corridor where a payload is materialized exactly once between producer memory and the wire frame; every additional copy in that corridor is measurable waste at artifact scale, and the corridor's only safety obligation — wrapped memory outlives the message — is dischargeable by scoping the message's serialize-and-send inside the buffer lease.
- Temporal bridge as one conversion algebra: the protobuf pair (extension projections, throwing range contracts) and the STJ pair (converter slots, pattern-pinned text) are two codecs over one domain temporal vocabulary; the law that unifies them is direction-and-seam — outward conversion happens inside the wire-contract constructor, inward conversion inside the admission bridge, and no temporal value exists between seams in wire shape. Range rejections from both codecs land in the same typed fault family, so a duration overflow reads identically whether it failed at a binary or JSON edge.
