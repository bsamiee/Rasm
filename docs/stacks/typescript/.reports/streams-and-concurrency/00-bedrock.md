# streams-and-concurrency — bedrock (streams + concurrency density)

# Rails & Effects Core — Concurrency + Streams Slice (Effect 3.21.2, verified against `node_modules/effect/dist/dts/`)

Every operator below was read from the installed `.d.ts`. Signatures are data-last/curried unless noted. Each finding names the naive sprawl it deletes.

```
[STREAM_AS_LOOP]: A Stream pipeline replaces the while-loop + mutable buffer array + index counter triad. The loop body becomes an operator; the buffer becomes the chunk; the accumulator becomes carried state. Growth axis: pipeline length (each new concern is one more operator, not a new nested block).
```

- `mapAccum(s: S, f: (s, a) => readonly [S, A2])` — stateful fold that EMITS per element. Deletes: `let acc; for(...) { acc = step(acc, x); out.push(project(acc, x)); }`. Trap: step returns `[state, emit]` — **state first**, not `[emit, state]`.
- `scan(s, f: (s,a)=>S)` / `scanEffect` — running fold emitting every intermediate. Deletes the running-total accumulator. Trap: **emits the initial seed as the first element** (unlike `mapAccum`), so output is `n+1` elements.
- `grouped(chunkSize)` / `groupedWithin(chunkSize, duration)` — batching. Deletes the `buffer.push; if (buffer.length >= N || elapsed > T) flush()` scaffold + the `setInterval` timer. Trap: `groupedWithin` is **positional `(chunkSize, duration)`**, NOT an options object; `duration` is `DurationInput` (`"1.5 seconds"`).
- `throttle({ cost, units, duration, burst?, strategy? })` / `debounce(duration)` — rate control. Deletes hand-rolled token-bucket + `lastEmit` timestamp bookkeeping. Trap: `strategy` is the string union **`"enforce" | "shape"`** (default `"shape"`), not an enum; `cost` is `(chunk) => number` (per-chunk, not per-element); `debounce` is positional duration only.
- `aggregate(sink)` / `aggregateWithin(sink, schedule)` / `aggregateWithinEither(sink, schedule)` — windowed accumulation driven by a `Sink` + `Schedule<C, Option<B>, R>`. Deletes the windowing loop with dual triggers (count + time). Trap: `aggregateWithin` keeps only sink completions; `...Either` emits `Either<B, C>` exposing schedule ticks as `Left`-vs-`Right`.
- `paginate(s, f: (s) => readonly [A, Option<S>])` / `unfold(s, f: (s) => Option<readonly [A, S]>)` (+ `...Effect`, `...Chunk`) — cursor loops. Deletes `let cursor; while(cursor) { page = fetch(cursor); cursor = page.next; }`. Trap: **shapes differ** — `paginate` returns `[value, Option<next>]` (Option wraps the state); `unfold` returns `Option<[value, next]>` (Option wraps the whole pair). Pick `paginate` for "always emit, maybe continue" cursor APIs.
- `retry(schedule: Schedule<X, E, R2>)` (on Stream) — restarts the **entire stream** on failure under a policy. Deletes the `for (attempt = 0; attempt < max; attempt++) try {...} catch` retry wrapper. Trap: Schedule's IN-type is the error `E`.
- `share({ capacity, strategy?, replay?, idleTimeToLive? })` / `broadcast(n, maximumLag)` / `broadcastedQueues(n, maximumLag)` — one source, many consumers. Deletes N duplicated fetches/subscriptions of the same upstream. Trap: `share` is **overloaded** — `capacity: "unbounded"` branch has NO `strategy` field; `capacity: number` branch adds `strategy?: "sliding" | "dropping" | "suspend"`. There is no `replicateCount`. `broadcast` returns a scoped `Effect<TupleOf<N, Stream>>`; `broadcastedQueues` hands back `Dequeue<Take<A,E>>` (raw queues of `Take`, not streams).
- `partition(pred, { bufferSize? })` / `partitionEither(f: a => Effect<Either>)` — fan a stream into two by predicate. Deletes two filtered passes over one source. Trap: returns a **scoped** `Effect<[excluded, satisfying]>` (or `[left, right]`); both substreams must be consumed concurrently or the buffer (default size) backpressures.
- `zip` / `zipWith(f)` / `zipLatest` / `zipLatestWith` / `zipWithIndex` — pairwise/latest combination. Deletes manual index counters and "remember last value of B" state. Trap: `zipLatest` tracks latest **per-chunk** — intra-chunk intermediates are dropped; tuple order is `[left, right]`.
- `mergeAll({ concurrency, bufferSize? })(streams)` / `merge(that, { haltStrategy? })` / `mergeWithTag(struct, { concurrency })` — interleave concurrently. Deletes a worker-pool + `Promise.all`-with-limit shim. Trap: the field is **`concurrency`** (number | `"unbounded"`), not `concurrencyLimit`; `merge` takes `haltStrategy`, NOT concurrency.
- `tick(interval)` / `fromSchedule(schedule)` / `repeatEffectWithSchedule(effect, schedule)` / `fromAsyncIterable(iter, onError)` — clock/cursor sources. Deletes `setInterval` polling loops. Trap: `fromSchedule` emits the schedule's OUTPUT type; `fromAsyncIterable` requires an explicit `onError` mapper and has **no AbortSignal bridge** in its signature (cancellation rides the async-iterator protocol).

```
[SINK_ALGEBRA]: Terminal logic is a value, not a hand-coded consumer loop. A Sink is built once and reused; composing Sinks (zip/race) expresses simultaneous OR first-wins terminal conditions without rewriting the drain.
```

- `Sink.fold(s, contFn: Predicate<S>, f)` — fold with a **continuation predicate**: folding stops when `contFn(state)` is `false`. Deletes `while(cond(acc)) acc = step(acc, next)`. Sibling forms: `foldLeft(s, f)` (run to exhaustion, leftovers `never`), `foldUntil(s, max, f)` (count gate), `foldWeighted({ initial, maxCost, cost, body })` and `foldWeightedDecompose({...decompose})` (cost-budget windows — both take an OPTIONS object).
- `Sink.collectAll()` / `collectAllN(n)` / `collectAllToMap(key, merge)` / `collectAllToSet()` — materialize without a push loop. `collectAllToMap` requires a `merge` for key collisions (no silent overwrite).
- `Sink.forEach(f)` / `forEachChunk(f)` / `forEachWhile(f: a => Effect<boolean>)` — effectful drain; `forEachWhile` stops on `false` and leaves leftovers `In`.
- `Sink.zip(that, { concurrent? })` / `zipWith(that, f, { concurrent? })` / `zipLeft` / `zipRight` — run two terminal conditions on one stream. Deletes "tee the stream and run two consumers." Trap: **`concurrent` defaults to `false` (sequential)** — set `{ concurrent: true }` for true parallel terminals; this is the simultaneous-terminal idiom.
- `Sink.race(that)` → winner as union `A | A1`; `Sink.raceBoth(that, { capacity? })` → `Either<A1, A>` identifying which side terminated first. Deletes a `Promise.race` over two consumer loops.
- `Sink.head()` / `last()` / `count` / `sum` / `take(n)` / `drain` — prebuilt terminals (`count`/`sum`/`drain` are values, not calls).
- `Sink.map(f)` (output) / `mapInput(f)` (input) / `dimap({ onInput, onDone })` / `flatMap(f)` — Sink transformation. Trap: **`Sink.contramap` does NOT exist** — use `mapInput` for contravariant input adaptation.

```
[QUEUE_TOPOLOGY]: Backpressure POLICY is constructor data, not control flow. bounded/dropping/sliding select overflow behavior by name; the Enqueue/Dequeue interface split enforces the one-owner-writes / many-read shape at the type level. Deletes hand-rolled ring buffers + "is full?" checks + producer/consumer condition variables.
```

- `Queue.bounded(n)` / `unbounded()` / `dropping(n)` / `sliding(n)` → `Effect<Queue<A>>`. `bounded` blocks the producer on full; `dropping` discards the new offer; `sliding` evicts the oldest. The policy is the constructor — no `if (size >= cap)` branch anywhere.
- `offer(a)` / `offerAll(iter)` (on `Enqueue<A>`), `take` / `takeAll` / `takeUpTo(n)` / `takeBetween(min, max)` / `poll` (on `Dequeue<A>`). `Queue<A> extends Enqueue<A>, Dequeue<A>` — hand the producer the `Enqueue` view and consumers the `Dequeue` view; the type split is the access-control mechanism. Trap: `takeN` exists only as a standalone function, not a `Dequeue` member.
- `PubSub.bounded(capacity | { capacity, replay? })` (+ `unbounded`/`dropping`/`sliding`, all accepting the same `{ replay?: number }` object) — fan-out broadcast. `subscribe` returns **`Effect<Dequeue<A>, never, Scope.Scope>`** (Scope-required — each subscriber's queue is scope-bound; `publish(a)` / `publishAll(iter)`). Deletes a listeners array + manual notify loop with replay buffer. Trap: `replay: number` is the late-subscriber backlog count and lives in the **constructor**, not on `subscribe`.
- `Mailbox.make(capacity? | { capacity?, strategy?: "suspend" | "dropping" | "sliding" })` → `Effect<Mailbox<A, E>>` — **exists** (3.8+, experimental). A single owner that is producer AND completion-signaller: `offer`/`offerAll`/`fail(e)`/`failCause`/`end`/`done(exit)`/`shutdown` (write side) and `take`/`takeAll`/`takeN`/`await`/`clear` (read, on `ReadonlyMailbox`). Bridges via `Mailbox.toStream(ro)` and `Mailbox.fromStream(stream, opts)` (the latter scope-required). Deletes the callback-driven push source that needs a separate "stream is done / errored" channel — Mailbox carries success, failure, AND completion in one cell. Trap: Mailbox is **not** a `Queue`/`Dequeue`; its `takeAll`/`take` return `[Chunk<A>, done: boolean]` tuples, and the read side is `ReadonlyMailbox<A, E>`.

```
[FIBER_SUPERVISION]: One owner object replaces the manual Map<key, fiber> + interrupt-old-on-new bookkeeping. The lifecycle (run-latest / keyed-latest / run-set) is chosen by which owner type you make; all three are scoped (interrupt all on scope close) and auto-remove fibers on completion.
```

- `FiberHandle.make()` → `Effect<FiberHandle<A,E>, never, Scope>`. `FiberHandle.run(self, effect, { onlyIfMissing?, propagateInterruption? })` — **run-latest**: the second `run` interrupts the prior fiber by default. Deletes `if (current) current.interrupt(); current = fork(next)`. This is the canonical "only the newest matters" handler (debounced search, live preview).
- `FiberMap.make()` → scoped `FiberMap<K,A,E>`. `FiberMap.run(self, key, effect, { onlyIfMissing? })` — **keyed run-latest**: key collision interrupts the prior fiber at that key. Deletes `Map<K, Fiber>` + per-key interrupt choreography. `set`/`remove`/`get`(throws `NoSuchElementException`)/`has`/`size`/`join`/`awaitEmpty`.
- `FiberSet.make()` → scoped `FiberSet<A,E>`. `FiberSet.run(self, effect, { propagateInterruption? })` — **run-set**: ADDS (no replace, **no `onlyIfMissing`**), auto-removes on done. Deletes an array of in-flight fibers you manually splice. `FiberSet` is `Iterable<RuntimeFiber>`; `join` fails with the first error, `awaitEmpty` drains.
- All three expose `runtime()` → a curried `Effect<(effect, opts) => RuntimeFiber>` for forking into the owner from non-Effect callbacks (event handlers).
- `Deferred.make<A,E>()` → `Effect<Deferred<A,E>>` — one-shot handoff. `succeed`/`fail`/`done(exit)`/`complete(effect)`/`completeWith(effect)` all return `Effect<boolean>` (true = first completion). `await` suspends; `poll` → `Option<Effect<A,E>>`. Deletes `let resolve; new Promise(r => resolve = r)` callback wiring. Trap: `complete` memoizes the effect, `completeWith` does not (use the latter for cheaper repeated-await of a side-effecting producer); not scoped.
- `forkScoped` (inner-scope lifetime) / `forkIn(scope)` (caller-chosen scope) / `forkDaemon` (global, parent-independent) — structural lifetime binding vs manual "kill children on shutdown" lists.
- `Effect.makeLatch(open?)` → `Effect<Latch>` (also `unsafeMakeLatch`). `Latch` extends `Effect<void>`; surface: `open` (permanent), `close`, `release` (release waiters without changing state), `await`, **`whenOpen(effect)`** (gate an effect on open). Deletes a boolean flag + waiter array + manual flush. Default state is **closed**.

```
[STM_COMPOSITION]: STM<A,E,R> EXTENDS Effect — it is directly yieldable in Effect.gen (no explicit commit needed when used as an Effect; STM.commit(stm) materializes it). Multi-cell atomicity and blocking-retry-on-condition are the two things STM buys that SynchronizedRef cannot. Deletes lock acquire/release choreography and condition-variable wait loops.
```

- `STM.commit(stm)` → `Effect<A,E,R>` (NOT `STM.atomically` — that name does not exist). `STM` has its own `STM.gen`, `map`, `flatMap`. `STM.check(predicate)` → `STM<void>` is the **blocking retry**: the transaction re-runs automatically when a read `TRef`/`TMap` changes — this replaces a polling `while (!cond) sleep()` wait loop with declarative "retry until the world is right." `STM.cond(pred, error, result)` is the fail-or-succeed branch.
- `TRef.make(v)` → `STM<TRef<A>>`; `get`/`set`/`update`/`updateAndGet`/`modify(f: a => [B, A])` all return `STM`. `TMap.make/empty/fromIterable` → `STM<TMap<K,V>>`; `get` → `STM<Option<V>>`, plus `set`/`remove`/`has`/`size`/`updateWith`/`merge(key,val,combine)`/`retainIf`/`removeIf`. Composing several TRef/TMap ops in one `STM.gen` block commits **atomically** — deletes nested-lock ordering and partial-update rollback handling.
- `SynchronizedRef.make(v)` → `Effect<SynchronizedRef<A>>`; `updateEffect(f: a => Effect<A,E,R>)` / `modifyEffect` run an **Effect inside the update under serialization** (all return `Effect`, not STM).
- **Objective boundary**: SynchronizedRef wins for a SINGLE cell whose update must run an effect (IO/DB/async) — STM cannot run arbitrary effects inside a transaction. STM/TRef/TMap wins when (a) ≥2 cells must commit as one atomic unit, or (b) you need blocking `STM.check` retry on a condition. If neither holds, SynchronizedRef is strictly cheaper.

```
[INTERRUPTION_ALGEBRA]: Cancellation is composable structure, not AbortController threading. uninterruptibleMask gives a `restore` you apply to the regions that SHOULD stay cancellable; the AbortSignal bridge is automatic at both edges. Deletes manual signal plumbing through every call frame.
```

- `interruptible` / `uninterruptible` (region toggles); `interruptibleMask(f)` / `uninterruptibleMask(f)` where `f` receives `restore: <A,E,R>(Effect) => Effect`. Deletes the "make this whole block atomic except the one cancellable wait" hand-coding — wrap in `uninterruptibleMask`, apply `restore` only to the interruptible sub-effect. Trap: `restore` is reusable and re-entrant; it returns the effect in the ORIGINAL interruptibility, not unconditionally interruptible.
- `onInterrupt(cleanup: (fiberIds) => Effect)` / `onError(cleanup: cause => Effect)` / `onExit(cleanup: exit => Effect)` / `ensuring(finalizer)` / `acquireRelease(acquire, release)` / `acquireUseRelease(acquire, use, release)` / `addFinalizer(f: exit => Effect)`. Deletes `try/finally` + AbortController.signal checks. All finalizers are uninterruptible; `acquireRelease` requires `Scope`.
- **Inward bridge (host → Effect)**: `Effect.runPromise(effect, { signal?: AbortSignal })` and `runPromiseExit(effect, { signal? })` — passing a host `AbortSignal` interrupts the fiber on abort. Trap: **`runFork`/`runCallback` have NO `signal` option** (their `RunForkOptions` is scheduler/scope only) — the signal bridge exists only on the Promise runners.
- **Outward bridge (Effect → host)**: `Effect.async<A,E,R>((callback, signal) => ...)` — the register function receives an **`AbortSignal` as its second argument**, fired when the fiber is interrupted. This is THE canonical wrap for a callback-API: register on `signal`'s `abort` to cancel the underlying op. Same for `Effect.promise((signal) => PromiseLike)` and `Effect.tryPromise({ try: (signal) => ..., catch })`. Deletes threading an AbortController through your adapter.

```
[TIME_DRIVEN]: Schedule is the polling/retry/backoff algebra as a composable value; Clock is an injectable service so TestClock makes time deterministic. Deletes setInterval/setTimeout loops, ad-hoc exponential-backoff math, and real-time sleeps in tests.
```

- Type order is **`Schedule<Out, In, R>`** (output, input, requirements). Constructors: `spaced(d)` (gap after action), `fixed(d)` (fixed window, no pile-up), `exponential(base, factor?)` (emits a `Duration`, not a count), `recurs(n)`, `forever` (a constant, not a call). Trap: `Schedule.once` does NOT exist.
- Algebra: `intersect` (both must allow; LONGER delay; tuple output), `union` (either allows; SHORTER delay), `andThen` (sequential, merged output) / `andThenEither` (`Either`, Right = first), `compose(that)` (first's Out feeds second's In), `addDelay(f: out => DurationInput)`, `jittered`, `whileInput`/`whileOutput` (continue WHILE true), `recurWhile`/`recurUntil`, `tapOutput`, `collectAllOutputs` (→ `Chunk<Out>`), `passthrough`. This deletes hand-written backoff: `exponential("100 millis").pipe(jittered, intersect(recurs(5)))` replaces a manual `delay *= 2; if (++n > 5) throw` loop.
- `Effect.repeat(effect, schedule)` (Schedule IN-type = success `A`, OUT becomes result) vs `Effect.retry(effect, schedule)` (Schedule IN-type = error `E`). Both take the Schedule **positionally** (curried). `Effect.schedule` is the loosened variant (`In = A | undefined`). `Effect.sleep(DurationInput)`. Deletes the `setInterval` polling loop — model polling as `effect.pipe(Effect.repeat(Schedule.spaced("5 seconds")))`.
- `Clock` service: `currentTimeMillis: Effect<number>`, `currentTimeNanos: Effect<bigint>`, `sleep(Duration)`; access via `Clock.clockWith(f)` or the `Clock.Clock` tag. Trap: `Clock.sleep` takes parsed `Duration`, while `Effect.sleep` takes `DurationInput`.
- `TestClock.adjust(DurationInput)` advances virtual time; `setTime`, `sleeps`, `adjustWith`. Provided as a **Layer** (`TestClock.defaultTestClock`) that substitutes the live `Clock` — every `sleep`/`Schedule` delay resolves instantly under `adjust`. Deletes `vi.useFakeTimers()` + real-clock flakiness; one `adjust("1 hour")` deterministically drives an hour of scheduled polling.

```
[REACTIVE_CELL]: SubscriptionRef = one cell whose `.changes` is a derived Stream that replays the CURRENT value on subscribe, then every update. One value replaces (subject + listeners array + notify loop + "current value" getter) of an observer-pattern class.
```

- `SubscriptionRef.make(initial)` → `Effect<SubscriptionRef<A>>`. The type extends `SynchronizedRef<A>` (full Ref surface: `get`/`set`/`update`/`modify`/`updateEffect`/`modifyEffect`, all `Effect`) AND `Subscribable`, exposing `readonly changes: Stream<A>` (a **property**, not a method). Mutations via `set`/`update` auto-publish to all `.changes` subscribers — no explicit notify. Trap (load-bearing): subscribing to `.changes` **immediately emits the current value first**, then deltas — it is a hot, current-state-seeded stream, so consumers never need a separate "read initial" fetch.
- Derive views with stream operators: `ref.changes.pipe(Stream.map(...), Stream.changes)` — note `Stream.changes` (adjacent-dedup by `===`) is the operator that suppresses no-op re-emits, distinct from the `SubscriptionRef.changes` property. This collapses (computed/selector + memo + equality-check) into one pipeline.
- `TSubscriptionRef.make(v)` → `STM<TSubscriptionRef<A>>` is the STM-backed sibling: `.changes` is `STM<TDequeue<A>>` (not a Stream); convert via `TSubscriptionRef.changesStream(ref)` → `Stream<A>` or `changesScoped`. Use it only when the reactive cell must participate in a multi-cell atomic transaction; otherwise `SubscriptionRef` is cheaper.

## Further Considerations

- `share` and `PubSub`/`SubscriptionRef.changes` overlap but differ in seed semantics: `share` multicasts a live upstream with optional `replay: N` backlog and `idleTimeToLive` (upstream finalizes after the last consumer leaves, then restarts on the next) — it is a connection-sharing cache, NOT a state cell. Reach for `SubscriptionRef.changes` when subscribers need current-state-on-subscribe; `share`+`replay` when they need the last N events; they are not interchangeable.
- The `Sink.zip({ concurrent: true })` parallel-terminal idiom is the densest spelling for "compute multiple aggregates in one pass" (e.g., `Sink.sum` zipped concurrently with `Sink.count` to get mean) — but the DEFAULT is sequential, which silently double-drains intent; the `concurrent` flag is the load-bearing bit most reviewers miss.
- `FiberMap` + `runtime()` is the highest-leverage collapse for host event handlers: a keyed map of debounced per-entity work (e.g., per-document recompute) where re-firing a key interrupts the stale fiber, all auto-cleaned on scope close — this replaces the single most error-prone hand-rolled pattern (a `Map<id, {fiber, cancel}>` with manual interrupt-and-replace and leak-on-unmount).
