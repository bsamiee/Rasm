# [STACKS_TYPESCRIPT]

This folder is the TypeScript stack decision atlas. It routes language, derivation, value, shape, surface, rail, wiring, concurrency, stream, and boundary decisions to the concept page that owns the coding choice. The atlas builds one admission-first paradigm on an erased type system: raw ingress decodes exactly once through one Schema owner, the runtime value is the single source of truth and every type derives from it, decoded values travel `Effect` rails whose error and requirement channels are typed, capability arrives only through the Layer graph, and projections leave through encode at explicit boundaries. Zero `any`, zero `throw`, zero `enum`; absence is `Option`.

[IMPORTANT]: Pages carry no outside-source blocks, release narration, provenance, process state, project anchors, tool context, source-footnote blocks, or meta commentary. Pages name exact code, package, operator, combinator, and command spellings in code spans; verification happens before authoring, and the page states doctrine as fact.

## [02]-[DOCTRINE]

Sixteen laws in five groups govern every TypeScript decision in this stack. Concept pages instantiate them; no page restates them. The laws exist so correctness is structural rather than disciplinary: decode-once makes the interior total over valid values; closed tagged families convert change into compile-time pressure; vocabulary-as-values makes behavior recoverable from declarations alone; derivation makes every secondary surface provably consistent with its primary. TypeScript erases types at emit and checks them structurally, so the doctrine anchors truth in runtime values ŌĆö one vocabulary table, one Schema owner ŌĆö and treats the type level as their derived shadow: `typeof`, `keyof typeof`, indexed access, and Schema extraction compute every secondary surface, and a hand-written parallel is not duplication but a second, unverified source of truth. Density is the consequence, not the goal ŌĆö when one declaration carries the family, every remaining line is load-bearing. The atlas is sized for large systems: total lines and public surface grow sublinearly with capability because every owner is declared with the capacity to absorb the family it anchors ŌĆö growth lands as tags, rows, and policy values inside existing owners, never as new surfaces beside them. Enforcement is doctrine-first: the strictness flags, the sole lint rail, the exports map, and the gauge audits encode these laws ŌĆö the doctrine authors the gate, never the reverse. Findings against these laws are architecture pressure: fix the shape, not the diagnostic.

[FLOW]:
- `EXPRESSION_SPINE` ŌĆö all domain logic is expression-shaped on the `Effect<A, E, R>` carrier; dependent steps compose monadically through `Effect.flatMap` and `Effect.gen`, independent computations compose applicatively through `Effect.all` and `Effect.zipWith` ŌĆö dependence licenses sequence, independence licenses accumulation, and the carrier, never a flag, selects the combination algebra. A bind chain over independent operands reports only the first failure and discards the rest, so abort-versus-accumulate is a correctness decision fixed once at the boundary ŌĆö the `Effect.all` mode, `Effect.validateAll`, `Effect.partition`, decode with `errors: "all"`. Statements survive only inside measured kernels and platform-forced callbacks, and any page that shows one names the exemption. Composition runs through `pipe`, `Function.flow`, `Match` terminals, and `$match` on tagged families.
- `BOUNDARY_ADMISSION` ŌĆö raw material is decoded exactly once into an evidence-carrying owner: `Schema.decodeUnknown` admits or rejects at the edge, the `ParseError` lifts into the error channel at the same seam, and the interior never re-validates and never sees `null`, `undefined`, sentinels, or provider shapes. `Option` carries non-failing absence ŌĆö admitted by `Schema.optionalWith` ŌĆö the tagged error channel carries fallibility, and a `Promise` or thrown exception converts at the owning boundary through `Effect.tryPromise` and `Effect.try`. The lifecycle is `unknown -> Schema owner -> rail -> projection -> encode`.
- `CAPABILITY_CHANNEL` ŌĆö every dependency travels the requirement channel as a Tag and is satisfied only through the Layer graph: the service is one `Effect.Service` class owner, substitution is Layer provision ŌĆö never a mock framework or a module patch ŌĆö and requirement elimination to `never` at the composition root is the wiring proof. Module-level live singletons, ambient capability imports, and parameter-drilled dependencies are rejected; a definition names no engine, the root selects one.

[SHAPE]:
- `SCHEMA_AUTHORITY` ŌĆö the Schema owner is the single shape authority: the static type derives ŌĆö `Schema.Schema.Type`, the class itself on `Schema.Class` owners ŌĆö wire twins derive through `Schema.transform` and field encodings, variants and projections derive through `Schema.pick`, `Schema.omit`, `Schema.extend`, and `Schema.partialWith`, and derived surfaces ŌĆö arbitraries, JSON schema, equivalence ŌĆö flow from the same declaration. A hand-declared interface, type alias, or DTO beside a Schema owner is the named defect.
- `SHAPE_BUDGET` ŌĆö a concept owns exactly one runtime authority sized at roughly one fifth of naive TypeScript's shape count: one deep `Schema.Class` family with embedded sub-schemas, brand-in-field refinements, and class-carried methods replaces five or more loose interfaces, aliases, DTOs, and scattered branded scalars. Variants are tags inside one closed family ŌĆö `Schema.TaggedClass`, `Data.taggedEnum` ŌĆö never sibling schemas, and brands live inside rich owners as field refinements, never as free-floating exports. Three or more parallel shapes, sibling factories, repeated dispatch arms, or single-call helpers is the collapse trigger, not a style preference.
- `DEEP_SURFACES` ŌĆö prefer one rich polymorphic surface over many shallow ones: a module exposes one or two exports with a `_`-prefixed interior and no barrels, and each exported concept serves value and type under one name ŌĆö a class owner, a same-name `const`-plus-`type` pair, or declaration merging ŌĆö so a consumer takes a single import and never aliases, re-derives, or writes `typeof` at a call site. One deep owner that holds a full concern beats four fragments that scatter it.
- `MODAL_ARITY` ŌĆö one entrypoint owns every call modality: singular, plural, batch, and stream discriminate on the shape of the input value ŌĆö tag, structural pattern, non-empty tuple arity ŌĆö never on name suffixes or boolean knobs, and pipe-versus-direct is one `Function.dual` definition, never a parallel pair. A parallel parameter that re-describes the input is the knob smuggled back in, because the discriminant must be recoverable from the value itself; arity twins and suffix families are the named defect.
- `ANTICIPATORY_COLLAPSE` ŌĆö an owner is shaped for the family it will absorb, not the instance in hand: the moment a second tag, dimension, knob, or modality is conceivable, the shape generalizes so the next requirement lands as a tag, a vocabulary row, a policy value, or a carrier swap with zero new surface. The proof of a correct shape is the diff of the next feature ŌĆö one declaration inside the owner, every consumer untouched or broken loudly at compile time. Sizing for the instance and widening later is the rejected order: widening after call sites exist multiplies surfaces, widening before they exist is one polymorphic dispatch.

[DERIVATION]:
- `VOCABULARY_VALUES` ŌĆö one `as const satisfies` table is the single source of truth for a vocabulary: `satisfies` validates against the contract without widening the inferred literal shape, rows carry their own behavior ŌĆö status, schedule, policy, route ŌĆö and every secondary surface derives from the table. A hand-written union, `enum`, or parallel constant a table could derive is the named defect, and a `Match` or `switch` enumeration over a domain a vocabulary row already maps is the rejected dispatch ŌĆö keyed domains dispatch through lookup; `Match` owns structural and predicate dispatch on non-keyed shapes.
- `DERIVED_TYPES` ŌĆö types are computed from one value or type anchor so one declaration yields the family: `typeof` and `keyof typeof` derive from value tables, indexed access projects each axis, mapped, template-literal, and conditional types generate secondary surfaces, and `const` type parameters, `NoInfer`, and instantiation expressions pre-solve inference at the owner so consumers never re-instantiate generics or re-assert literals. A runtime literal that restates a correspondence the type level already carries is the rejected form ŌĆö derived literal types are the symbol currency, and names, paths, and discriminants travel as them.
- `SEMANTIC_NAMING` ŌĆö every file, module, class, function, parameter, field, tag, case, and region name uses the bounded context's canonical term, grammatical role, and tense. One semantic word is the default; two words are allowed when owner plus action, result, axis, or boundary is load-bearing; three words are the ceiling unless an external API or ambiguity proof requires more. Operations use action verbs, values and receipts use result nouns, vocabularies use stable noun or adjective rows, and boundary adapters preserve foreign names only at the seam. Renaming for variety, tense drift, abbreviations, prefix or suffix families, and parallel file or member labels for one concept are rejected.

[MATERIAL]:
- `LIBRARY_DEPTH` ŌĆö the Effect ecosystem is the standard library: `effect` owns rails, `Schema`, `Match`, collections, time, order, and equivalence; the `@effect/*` set owns platform, sql, rpc, cluster, workflow, ai, and telemetry. `HashMap`, `HashSet`, `Chunk`, `Option`, `Either`, `Duration`, `DateTime`, `Order`, and the `Array`/`Record`/`Struct` modules are the domain default; JS stdlib `Map`, `Set`, `Promise`, `Date`, and `null` survive only at the FFI seam. Use the deepest operator, combinator, or derived surface the package itself reaches for; wrappers, rename adapters, and stdlib-first reflexes are rejected.
- `INLINE_COMPOSITION` ŌĆö wrapping, injection, and decoration attach at the owner declaration: `pipe` composition at definition, decode and brand at the Schema field, telemetry and resilience as rail transformers at the `Effect.fn` definition seam ŌĆö `Effect.withSpan`, `Schedule` policy, `Effect.acquireRelease` lifetime ŌĆö and handler or method records `satisfies`-checked at the table. Loose intermediate consts, separate helper declarations, and consumer-side reassembly are rejected; an aspect materializes policy as values, and policy pushed inside the body or admission hoisted above the decode seam stops being recoverable from its declaration.

[INTEGRATION]:
- `ROOT_REBUILD` ŌĆö new capability is woven into the owning Schema, vocabulary, or service as if it had always existed; reshape the owner rather than appending beside it. No shims, compat aliases, deprecated re-exports, barrel indirection, or migration surfaces ŌĆö break the API when the collapse improves the system.
- `ONE_HOP_RESOLUTION` ŌĆö a name resolves to its semantics in one hop: no alias-to-type-to-const chains, no forwarding helpers, no helper or util shells, no convenience wrappers that rename or forward an external API. A `const` that re-exports a `_`-prefixed symbol, or a value that takes two jumps to trace, marks a layer to delete.
- `COMPOSED_IMPLEMENTATION` ŌĆö a feature of any complexity is implemented by composing the page owners ŌĆö Schema owners, typed rails, dispatch surfaces, Layer wiring, vocabulary policy ŌĆö before any new scaffold is named. The pages are one body, and the same concern resolves identically wherever it appears. A need with no composed spelling marks a missing case in an owning page's law: the law extends first, the feature lands second. Cards and snippets show this composition at full operator depth. Flat code ŌĆö logic below the operator depth the admitted packages reach ŌĆö is surface sprawl in time: it re-derives, line by line, what a deeper composed form states once.

## [04]-[RULE_ENFORCEMENT]

TypeScript erases types at emit and ships no totality checker for open shapes, so the doctrine's compiled form is a four-gate stack legislated harder than a checked language needs: the compiler under the full strictness set on a dual floor, one lint rail, the package exports map, and typed gauge audits. The manifest owns every pin, and the loop is one-directional ŌĆö a doctrine page legislates, a gate enforces; a rule never introduces law of its own.

- Compiler law: the strictness set is semantic law, not hygiene ŌĆö `strict`, `exactOptionalPropertyTypes`, `noUncheckedIndexedAccess`, `noPropertyAccessFromIndexSignature`, `noImplicitOverride`, `noFallthroughCasesInSwitch`, `noUncheckedSideEffectImports`, `strictBuiltinIteratorReturn`, `isolatedModules`, and `moduleDetection: "force"`. `erasableSyntaxOnly` makes `enum`, runtime `namespace`, and constructor parameter properties compile errors; `verbatimModuleSyntax` makes type-only traffic explicit at every import and export; `isolatedDeclarations` forces every export to carry its annotation so the public surface is readable without inference. Every check runs the dual floor ŌĆö `tsgo` and `tsc` ŌĆö and parity is the floor claim: a construct the two compilers disagree on is rewritten to the form both accept, never masked by a directive, and no page law assumes behavior only one compiler accepts.
- Mapping law: every doctrine law maps to its enforcing gate. Exhaustiveness rides `Match.exhaustive` and closed tagged families whose missing arm is a compile error; absence leakage rides `strict`, `exactOptionalPropertyTypes`, and `noUncheckedIndexedAccess`, with domain absence routed to `Option` by lint; `any` rides `noExplicitAny` at error; syntax legality rides `erasableSyntaxOnly`; import discipline rides `verbatimModuleSyntax`; shape proliferation, standalone brand exports, `throw` in domain flow, and stdlib leakage ride promoted lint rules; package and module boundaries ride the exports map and the gauges. A law with no mechanical gate is captured as a plugin rule or a gauge.
- Promotion law: Biome is the sole lint rail ŌĆö one configuration owns formatter, linter, and assist; no second linter and no boundary-lint layer exists. A doctrine-breaking anti-pattern that no compiler flag or shipped Biome rule rejects ŌĆö a `const`-plus-`type` restatement, a parallel schema beside a Schema owner, a standalone brand export, an arity twin, a barrel file, a thin rename wrapper ŌĆö is captured as a GritQL plugin rule and promoted to error. Style preferences and anything an existing gate already rejects are not new rules.
- Boundary law: the workspace package exports map is the structural gate ŌĆö an unexported interior module is physically unresolvable, and per-runtime subpaths keep server code unreachable from browser resolution; the permitted-edge ledger is enforced by module resolution, never by lint. Project tags are graph metadata that drive affected computation and never gate as lint. The `proof/gauge` audits own what resolution cannot express ŌĆö the edge-ledger import audit, the external-admission union, per-runtime subpath purity, banned-module scans ŌĆö and a gauge failure fails the build.
- Shape law: a rule describes the semantic shape of the anti-pattern ŌĆö trigger, predicate, exemption ŌĆö never a path or one-off symbol; every rule ships with positive spans that fire and valid dense code that must not.
- Finding law: a true positive is architecture pressure ŌĆö fix the shape, never the diagnostic; a false positive or a fix that adds ceremony without improving the system is rule pressure ŌĆö refine the rule. A `biome-ignore`, a `@ts-expect-error`, or an `as` cast that adds ceremony without improving correctness is itself the defect.

## [05]-[PAGE_CRAFT]

How pages in this folder are authored. The corpus is one body; these laws keep it coherent.

- Atlas law: this README owns doctrine and the routing table; each concept page owns one disjoint layer and is the only file barred from re-teaching a sibling's mechanic. A page names a sibling owner in a code span only to fix the shed boundary of its own layer ŌĆö the concern lives there, the consequence lands here ŌĆö and that bare ownership pointer is the page's whole reference to the sibling; cohesion comes from shared law, never from one page reconstructing another's surface. The README is the only file that carries a Markdown link.
- Zero meta: concept pages carry no provenance, source trace, release narration, process state, project, tool, or skill context.
- Page grammar: narrow index table, then family cards, then the snippet beside the rule it proves; the page ends at its last card section. Structure is identical across stack folders; content never is.
- Card fields are earned: `Use / Accept / Reject / Law / Boundary` lines appear on a card only where each one decides something; a field line that decides nothing is deleted, not filled.
- Snippet law: every snippet type-checks under the full strictness set on both compilers; identifiers are legal neutral names; placeholder strings such as `"<value-a>"` appear only inside literals; no project, host, or domain concept anchors a snippet.
- Snippet coverage: each snippet is doctrine-exemplary at full operator depth, roughly 3-4x denser than ordinary code, and exercises a surface region no other snippet in the corpus shows ŌĆö variety within the doctrine, zero duplicated demonstrations. The region is the snippet's spotlight demonstration; finalized surfaces composed as supporting material occupy no region and duplicate nothing.
- Scale fidelity: a snippet shows the form at the shape it takes in a large system ŌĆö decode, dispatch, rail, and vocabulary composed in one fence with the growth axis visible ŌĆö never an isolated minimum; a statement-bearing snippet sits beside the Exemption line naming its platform-forced seam.
- Code names before prose: every member a card or snippet names ŌĆö combinator, constructor, flag, derived surface ŌĆö is verified against the pinned package's shipped declarations before it is written; an unconfirmable claim is not authored, and a nameable surface spelled as prose is a defect ŌĆö the code span is the instruction.
- Card economy: cards are few, deep, and evidence-dense; near-peer cards merge until each retained card owns a decision cluster, and a card line carries exactly one decision. A thin card deciding one concept is a sibling line, not an owner.
- Snippet adherence: snippets obey the card they prove, compose finalized laws as supporting material, and reduce lines and surface through derivation, vocabulary dispatch, and package depth, never by deleting capability.
- Altitude routing: when two pages touch one fact, the ledger records the split ŌĆö mechanics at the owning page, consequence at the consumer; prose re-teaching an owned mechanic is repaired by routing to the owner, while composing owned surfaces inside a snippet is supporting material and owns nothing.
- Reject columns are load-bearing: every `Use` names the spelling, wrapper, or local pattern it deletes.
- Tables enumerate, cards legislate: rows stay atomic and narrow ŌĆö no prose cramming, no links inside cells; nuance moves to a card.
- Route scope is README-local: lookup rows and conflict rules live in README files; concept pages carry only the law they own.
- Manifest truth: package versions, the strict catalog, admitted packages, the exports map, tool settings, and tag metadata live in `pnpm-workspace.yaml`, the per-project manifests, `tsconfig.base.json`, and `biome.json`; no package-named pages; a package is named only where it changes the implementation choice.

## [06]-[CORPUS_LAW]

How the corpus accretes. The folder is one cohesive body, not isolated pages; atlas order is creation order and implementation order. The atlas `[STATE]` column is the law registry: a `finalized` page is binding law for every page authored after it; a `partial` page carries no authority and awaits rebuild; a `target` page exists only as roadmap scope. Finalization is a one-way gate ŌĆö a context-free cold grade of the full page and every snippet, converging to a zero-edit pass, flips the state; the producer's grade admits, the cold grade decides.

- Three-layer inheritance: every page is authored under the doctrine, under every finalized page ŌĆö adhered to, never restated, never referenced, never contradicted ŌĆö and from its own source reservoir.
- Prose consumes earlier layers as given: vocabulary, owners, rails, and policy values arrive settled, never re-taught, and the page spends its lines only on its own layer.
- Reservoir residue: a page's source workspace deliberately holds more than the page prints, and it outlives the page ŌĆö later enrichment and snippet work mine it without source-footnoting it; depth beyond one page's budget is corpus capital, never waste.
- Snippet stacking: code fully captures the card it proves, then composes on earlier layers' law at full doctrine depth ŌĆö the new surface in the spotlight, established surfaces as supporting material, every touched concern built at the standard its owning finalized page legislates.
- Region ledger: the workspace root keeps one ledger of owned regions ŌĆö page concerns at card altitude, snippet demonstrations at fence altitude. Snippet rows are written before code exists, page rows at finalization, and a duplicated region is repaired by routing to its owner, never by re-teaching ŌĆö the ledger is what makes ownership checkable without re-reading the corpus.
- Purpose: the corpus is loaded as the operative standard in place of weaker context; zero-meta, zero-anchor, and stated-as-fact are absolute because any provenance, hedge, or stale claim poisons every downstream generation that loads the page.
