# Static-Virtual Metadata Interfaces and the Self-Describing Correspondence Law

[STATIC_ABSTRACT_ADMISSION_CONTRACT]:
- Every generated owner implements a static-abstract admission interface rather than an instance one: `IObjectFactory<T, TValue, TError>` declares `static abstract TError? Validate(TValue?, IFormatProvider?, out T?)`, and the owner's generated body is the implementation — admission is a type-level operation, not a virtual on a live instance, so a value never has to exist before its own validation runs.
- The contract is tri-parameter by construction: `T` is the owner being admitted, `TValue` is the raw representation, and `TError` is constrained `where TError : class, IValidationError<TError>` — the failure vocabulary is itself a static-virtual interface whose single member is `static abstract T Create(string)`, so the entire admission stack from raw bytes to typed rejection is expressible without one instance method or one stringly message escaping the type system.
- `TValue` carries `notnull, allows ref struct`, so the raw representation slot legally accepts `ReadOnlySpan<char>`; a second `Validate` overload exists keyed exactly on `ReadOnlySpan<char>`, making span admission a distinct static-abstract member rather than a heap-string fallback — zero-allocation wire ingress is a first-class admission grammar, not a convenience.
- The marker base `IObjectFactory<TValue>` (member-less) is implemented in addition to the tri-parameter form, so a consumer constrains against the weak marker to mean "admits this raw shape" without naming the owner or error types, and against the strong form to actually invoke `Validate` — the same widening-to-marker discipline that lets carrier folds demand the weakest sufficient trait.

[CONSTRAINT_POLYMORPHIC_DISPATCH]:
- A smart-enum owner implements `ISmartEnum<TKey, T, TError>` whose static-abstract surface is `static abstract IReadOnlyList<T> Items`, `static abstract T? Get(TKey?)`, and `static abstract bool TryGet(TKey?, out T)` — the entire lookup and enumeration vocabulary is reachable through a constraint, so one generic method `where T : ISmartEnum<TKey, T, TError>` writes a parse-fold once and binds it to every smart-enum family with no per-family code.
- The static-virtual invocation requires only a thin generic relay because the language forbids calling a static-abstract member on a bare type parameter outside a constrained context; the package ships these relays (`StaticAbstractInvoker.Validate<T, TKey, TError>`, `ParseValue<TValue>`, `ValidationErrorCreator.CreateValidationError<TError>`) as `AggressiveInlining` statics whose whole body is `T.Validate(...)` / `TError.Create(...)` — a constraint-polymorphic admission entrypoint folds through the relay and the JIT specializes it per family with the static call inlined away.

```csharp
public static Fin<T> Admit<T, TKey, TError>(TKey key)
    where T : ISmartEnum<TKey, T, TError>
    where TKey : notnull
    where TError : class, IValidationError<TError> =>
    T.TryGet(key, out T item)
        ? FinSucc(item)
        : FinFail<T>(Error.New(TError.Create($"<unknown key: {key}>").ToString() ?? "<error>"));

public static B FoldFamily<T, TKey, TError, B>(B seed, Func<B, T, B> step)
    where T : ISmartEnum<TKey, T, TError>
    where TKey : notnull
    where TError : class, IValidationError<TError> =>
    T.Items.AsIterable().Fold(seed, step);
```

[METADATA_AS_SELF_DESCRIBING_UNION]:
- The closed-member list of every generated owner is published as a static-abstract property `static abstract Metadata Metadata` on `IMetadataOwner`, and `Metadata` is itself a closed `[Union]`: its cases are `RegularUnion` (carrying `IReadOnlyList<Type> TypeMembers`), `AdHocUnion` (carrying `IReadOnlyList<Type> MemberTypes`), `ComplexValueObject` (carrying `IReadOnlyList<MemberInfo> AssignableMembers`), `KeylessSmartEnum`, and a nested `[Union] Keyed` whose leaves are `SmartEnum` and `ValueObject` — the package describes the very shape taxonomy it generates as a union with the exact total-dispatch grammar it emits for user families.
- Because `Metadata` is a union, consuming it is the same named-argument total `Switch` the user writes for any family: a generic infrastructure adapter folds `owner.Metadata.Switch(regularUnion: ..., adHocUnion: ..., keyedSmartEnum: ..., keyedValueObject: ..., complexValueObject: ..., keylessSmartEnum: ...)`, and a new shape kind in the package surfaces as a compile break at that fold rather than a silently-skipped instance — the introspection seam inherits exhaustiveness from the same mechanism it inspects.
- The kind cases carry exactly the construction recipe each shape needs: `Keyed` adds `KeyType`, `ValidationErrorType`, a `Delegate ConvertToKey`, and a `LambdaExpression ConvertToKeyExpression`; `Keyed.SmartEnum` additionally carries a `Lazy<IReadOnlyList<SmartEnumItemMetadata>> Items` so item enumeration is deferred until the first introspection touch, and `Keyed.ValueObject` carries a `ConvertFromKeyExpressionViaConstructor` — the metadata distinguishes constructor-shaped admission from method-shaped admission so a persistence translator can compile the constructor expression where a `Validate` call is opaque.

[BASE_WALK_RESOLUTION_LAW]:
- The runtime lookup resolves a leaf type to its family metadata by one base-type walk for the explicitly-implemented static property — it `GetProperty("global::Thinktecture.Internal.IMetadataOwner.Metadata", Static | NonPublic)` up the base chain and caches the hit per queried type in a `ConcurrentDictionary` — so any infrastructure holding only a leaf instance and its runtime `Type` reaches the closed member list with no assembly scanning and no registration, and a regular-union case (a derived nested type) resolves to its owner's metadata through that same base walk.
- The candidate filter short-circuits before the interface check: primitives, arrays, enums, and pointers return null immediately, and `Nullable<T>` is unwrapped to its underlying type before lookup — the seam is safe to call speculatively on arbitrary `Type` values in a serializer or mapper hot path, and the per-type cache makes the steady-state cost a dictionary probe.
- Explicit interface implementation is load-bearing, not incidental: the static `Metadata` property is emitted as an explicit `IMetadataOwner.Metadata` so it never pollutes the owner's public surface, yet reflection finds it by its fully-qualified `global::`-prefixed name — a hand-rolled public static of the same name would not be discovered, so the resolution contract is the generator's private name, never a convention a user reproduces.

[DERIVED_CORRESPONDENCE_LAW]:
- The closed member list is the single primary correspondence from which every secondary map is folded, never enumerated: a wire-row assertion, a render-arm coverage check, a transition table, or a column registration becomes a fold over `Metadata.RegularUnion.TypeMembers` (or `AdHocUnion.MemberTypes`), so adding a case to the family is automatically present in every derived map and absent maps surface as a fold mismatch in a family-law test rather than a production decode failure.
- This converts library reshaping from a silent-drift hazard into a compile-or-test break by construction: a one-time fold `members.All(m => HasWireRow(m))` proves every case owns a wire contract, and the proof re-runs against the regenerated member list, so a case added without its wire row fails the law test — the published member list, not a hand-maintained registry, is the source of truth that keeps secondary maps in lockstep with the family.
- Direct consumption of the metadata surface outside generated code trips the package's own analyzer diagnostic, so the defensible zone is exactly family-law tests and infrastructure adapters; inside that zone the fold gives closed-family introspection — every case's wire, render, persistence, and transition obligation checked against one generated list — without the assembly-scanning, attribute-hunting reflection a non-self-describing library would force.

[OBJECT_FACTORY_ROUTE_ALGEBRA]:
- Multiple admission grammars on one owner are published as `IReadOnlyList<ObjectFactoryMetadata>` through `IObjectFactoryOwner.ObjectFactories` (a separate static-abstract property), each row carrying `ValueType`, `ValidationErrorType`, `SerializationFrameworks UseForSerialization`, `UseWithEntityFramework`, `UseForModelBinding`, and the optional constructor-shaped conversion expression — every downstream consumer partition (JSON, persistence, model binding) selects its route by filtering this row list, not by attribute re-discovery.
- Route selection is `LastOrDefault(filter)`, not first: when two factory rows both satisfy a consumer's filter the later-declared one wins, so factory-attribute declaration order is a behavior contract — appending a more-general factory after a specific one silently re-routes an existing consumer, and the correct discipline is to order rows general-to-specific and treat any reorder as a semantic change subject to the law test.
- Conversion resolution composes the two route sources: `FindMetadataForConversion` first asks `Find` for keyed metadata, applies the caller's `metadataFilter`, then folds the object-factory rows with the caller's `objectFactoryFilter`, and only falls back to the key-based `ConvertFromKeyExpression` when no factory row matches — the result is a single `ConversionMetadata` record carrying `Type`, `KeyType`, `ValidationErrorType`, and both a constructor-shaped and a method-shaped lambda, so a translator chooses the expression form its target understands (compiled constructor for query translation, delegate for in-memory) from one resolved value.

[DEFERRED_ITEM_LAW]:
- Smart-enum item metadata is `Lazy<IReadOnlyList<SmartEnumItemMetadata>>`, where each row is `{ object Key, object Item, string Identifier }` — the `Identifier` is the source-level field name captured at generation, so the case's program identity travels in the metadata and a discriminator or label derives from `Identifier` rather than a restated string literal, breaking compilation on rename instead of corrupting decode.
- Laziness is a static-initialization ordering guard: forcing the item list eagerly inside the metadata constructor would observe partially-initialized static readonly fields during the family's own class-construction, so the deferred list defers the read until the first introspection touch — any infrastructure that folds `Items` is therefore safe to call from a static constructor of an unrelated type without tripping a half-built family, and the cost is one `Lazy` barrier paid once per family.
- The key-lookup channel is a `TryGetFromKey` delegate `bool(object?, out object?, out object error)` carried in keyed metadata, so a fully type-erased lookup (key-as-`object` to item-as-`object` with an `object` error) is available to reflection-bound infrastructure that cannot name `TKey`, while the static-abstract `TryGet(TKey?, out T)` serves typed callers — one family publishes both an erased and a typed admission route, and the erased route's error channel is the same typed validation error boxed, not a separate failure vocabulary.
