[BEARTYPE_PROTOCOL_CACHE_INVALIDATION_FAILURE]:
- `beartype.typing.Protocol` installs a `_abc_inst_check_cache: dict[type, bool]` per protocol class at metaclass `__new__` time. The cache maps `type(obj) -> bool` and is a plain `dict` with no eviction or invalidation mechanism. Any object whose class acquires new methods after the first `isinstance` check against a protocol will return a stale cached `False` on all subsequent checks. This is the documented caveat: "objects with methods dynamically assembled at runtime" are explicitly unsupported. [beartype `typing/_typingpep544.py` lines 103â€“120, 258, 291â€“357, installed 2026-06-09]
- The cache is set on the protocol class object directly (`cls._abc_inst_check_cache = {}`), not on the metaclass. It is a plain Python `dict` â€” safe to read under GIL, but a data race under PEP 703 free-threading if multiple threads perform the first `isinstance` check for the same `obj_type` concurrently. The writes are idempotent (same result both times) but the redundant computation and dict mutation are unguarded. Use `beartype.typing.Protocol` only for type families stable after import.
