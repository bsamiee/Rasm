# [TS_REGION_MAP]

Ownership ledger for the `libs/typescript` five-domain FLAT planning branch — ONE branch ledger with five per-domain blocks plus the branch non-domain block, mirroring the C# suite ledger shape. Protocol and row shapes mirror the [suite standard](../../../csharp/.planning/README.md) ledger-protocol section, minus the signature-regions file: the design-page fences carry the signatures inline and the wire signature regions are owned by the C# ledger. Every owner name is unique within the TS branch and grepped clean against the suite owner-symbol registry; anchors are bare-domain-folder-qualified (`<domain>/page#CLUSTER`, the domain prefix a ledger qualification convention for the domain folder inside the one `@rasm/ts` package, never a published-package boundary — only `@rasm/ts` and its subpaths `./ui`/`./web`/`./node`/`./provisioning` are literal package surfaces). The per-domain DENSITY_BAR lives in each `<domain>/.planning/README.md`, never duplicated here — the single-state-surface law.

The browser stratum is two folders, not one: `ui` (the AppUi-analog UI/UX/components library, subpath `./ui`) and `platform` (the AppHost-analog SPA infrastructure entry, subpath `./web`) — the prior single `web` folder split by real concern. The five blocks are `interchange`/`projection` (stratum neutral), `ui`/`platform` (stratum browser), `services` (stratum node).
