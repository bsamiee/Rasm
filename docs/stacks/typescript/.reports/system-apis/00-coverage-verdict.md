### B. Is system-apis.md earning its core slot? ‚Äî KEEP as core page 7

**Decision: keep.** The C# corpus is decisive precedent ‚Äî its system-apis.md is finalized as core page 7 with seven authority cards (TEXT_AND_WIRE, COLLECTIONS_AND_IDENTITY, NUMERICS, RUNTIME_AND_OBSERVABILITY, IO_BUFFERS, BOUNDARY_AND_INTEROP) plus a SMELL_LOOKUP routing index. The page's value is **anti-reflex**: it intercepts the JS-native muscle-memory (`Date.now`, `JSON.parse`, `console.log`, `Math.random`, `crypto.randomUUID`, `Array.prototype.*`, `structuredClone`) that no other page would naturally trap, and routes each to its Effect owner (`Clock`, `Schema`, `Effect.log`, `Random`, `Iterable`/`Chunk`, `Equal`). Demoting it to a boundaries.md appendix fails: boundaries.md owns *foreign-form admission* (Promise/throwing/callback), a categorically different concern from *native-builtin replacement*. The two would cross-contaminate.

Steelman for demotion (rejected): "it's a cheap routing table authored last." True, but cheapness is the point ‚Äî it is the *load-bearing index* that makes pages 1-6 discoverable by smell rather than by knowing the answer already. The C# SMELL_LOOKUP table proves a routing-first entry surface earns its slot precisely because juniors search by symptom.

**Does a foundational concern lack a slot that system-apis displaces?** No. The gaps from (A) ‚Äî build-surface, signal-emission, GlobalValue, hot-path ‚Äî are all *smaller* than system-apis (each is a card or section, not a page), and three of the four route *into* system-apis or its neighbors. There is no foundational concern large enough to warrant the swap. **No swap.**

### C. Integration-order restatement traps ‚Äî splits to mandate

Three real altitude conflicts; ledger split decided per the global "mechanics at owner, consequence at consumer" rule:

1. **Cache/RateLimiter (rails #4) vs backpressure (streams #5).** Conflict: both govern flow-rate. Split: rails owns `RateLimiter`/`Cache` as *policy values over Effects* (token-bucket config, TTL, capacity); streams owns *backpressure as a Stream/Sink/Queue buffering strategy* (`Stream.buffer`, `Sink` chunking, `Mailbox`/`Queue` capacity policy). The seam: a `RateLimiter` *throttling a stream* is a stream-consumer applying a rails-owned value ‚Äî streams.md references, does not redefine, the limiter. State this cross-ref or streams will restate rate-limiting.

2. **Runtime roots (boundaries #6) vs Layer composition (rails #4).** Conflict: both touch `Layer`. Split: rails owns `Layer` *as an aspect value* (`Layer.effect`, `Layer.scoped`, merge/provide *composition law*); boundaries owns the *single runtime root* (`runMain`/`ManagedRuntime`, "one Layer graph per app", `Layer.launch`). Mechanics (how layers compose) at rails; consequence (where the one graph is materialized and run) at boundaries. This is the cleanest split in the set ‚Äî keep it explicit.

3. **`Scope`/`acquireRelease` (rails #4) vs fiber lifecycle + `Scope`-bound forking (streams #5).** Conflict: fiber forks are scope-bound; `Stream` is scoped. Split: rails owns `Scope`/`acquireRelease`/`RcRef` as the *resource-finalization rail*; streams owns *fiber-as-resource* (fork admission, `FiberHandle/Map/Set` registries, interruption). The seam: `Effect.forkScoped` is a streams-concern *using* a rails-owned `Scope`. Name it or both pages describe scope-bound finalization.

4. **Service ladder (shapes #2: Reference/Tag/Service) vs service surfaces+accessors (surfaces #3) vs Layer wrapping (surfaces #3 aspect / rails #4).** Three pages touch services. Split: shapes owns *which service construct* (Reference vs Tag vs `Effect.Service`); surfaces owns *the accessor/dual-signature surface* a service exposes; rails owns *Layer provision of* that service. Triple-altitude ‚Äî the most restatement-prone. Mandate: shapes states the construct choice and stops; surfaces never re-explains Tag-vs-Service; rails never re-explains accessor shape.

No ordering defects in the 1‚Üí7 sequence itself: language‚Üíshapes‚Üísurfaces‚Üírails‚Üístreams‚Üíboundaries‚Üísystem-apis is dependency-correct (each page consumes only earlier vocabulary; system-apis routes back to all six, correctly last).

### Decisions summary
- **A:** Add 2 cards to system-apis.md (`[SIGNAL_EMISSION]`, `[HOT_PATH]`); add `[BUILD_SURFACE]` section to language.md; own `GlobalValue` in boundaries.md `[HOST_SINGLETONS]`. No 8th page. Make 4 altitude-splits explicit (functional-utils, date/time, JSON, Array/Iterable/Chunk).
- **B:** Keep system-apis.md as core page 7. No demotion, no swap. Precedent and anti-reflex routing value confirmed.
- **C:** Mandate four ledger splits (Cache/RateLimiter‚Üîbackpressure; Layer‚Üîruntime-root; Scope‚Üîfiber-lifecycle; service triple-altitude). Integration order is sound.
- **API correction:** `Semaphore`/`Latch` are `Effect.make*` constructors, not modules ‚Äî fix the streams page label.

### Further Considerations
- The TS set elevates **streams-and-concurrency to core (#5)** where the proven C# corpus demotes its equivalent to a **domain lane** (`domain/concurrency.md`, #11). This is defensible ‚Äî Effect's fiber model is foundational to *all* non-specialized TS in a way C#'s `Task`/`Channel` is not ‚Äî but it asymmetrically inflates core from C#'s 7-with-concurrency-in-domain to TS's 7-with-concurrency-in-core, pushing `algorithms.md` (C# #6) out. Confirm there is genuinely no TS `algorithms.md` peer; if numeric-kernel doctrine emerges later it has no core slot and would force the 8th page B rejected.
- `Effect.fn` (surfaces #3) and `Effect.gen` (rails #4 gen-vs-pipe law) both define generator-based authoring; `Effect.fn` is the *named-span traced* variant. Route `Effect.fn`'s span-naming consequence to the future diagnostics lane, not surfaces, or surfaces silently owns tracing.
- `ManagedRuntime` (boundaries #6) and `ExecutionPlan` (rails #4) both shape *how effects run*; `ExecutionPlan` is a fallback/retry-across-providers policy that is itself *run by* a runtime root. Verify rails owns the plan *value* and boundaries owns its *execution*, or you get a fourth altitude conflict beyond the three in C.
