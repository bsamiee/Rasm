# [d3] — the headless math/data substrate under visx and Plot: scales, interpolation, array folds, path generators, geo math — never a charting surface, never a DOM owner

[PACKAGE_SURFACE]:
- package: `d3` · license `ISC`
- module: metapackage — 30 `d3-*` modules re-exported flat (`export * from "d3-<module>"` per dependency); ESM/UMD dual, tree-shaken per named import so the bundle cost is the modules actually touched.
- asset: types resolve per-module from the `@types/d3-*` family the bundle aggregates; every member below is a flat named export off `d3`.
- runtime: pure computation — number/array/string/date in, numbers/path-strings/GeoJSON-projected output; zero DOM requirement in the admitted scope.
- plane: `plane:runtime` (W4 `ui`, substrate under the chart owners).
- rail: `ui` data-viz substrate — the shared math both `@visx/*` and `@observablehq/plot` stand on.

`d3` is admitted as SUBSTRATE, not surface: the `@visx/*` set wraps `d3-scale`/`d3-shape` into React components and `@observablehq/plot` embeds most of the roster behind its grammar, so this catalog scopes d3 to the headless modules those owners lean on plus the folds a bespoke chart computes directly — scales, interpolation, array statistics/grouping, shape/path generation, format/time vocabularies, color-space math, geo projection math, and the layout kernels (hierarchy/force/delaunay/quadtree). The six DOM-coupled modules the bundle also carries (`d3-selection`, `d3-transition`, `d3-zoom`, `d3-drag`, `d3-brush`, `d3-axis`) are REJECTED wholesale — React owns the DOM, `system/act` owns gesture and transition, and an axis is a visx/Plot mark. A chart never "uses d3" as an authoring model here; it composes d3 math under a React-rendered or Plot-generated surface.

## [02]-[SCALE_AND_FOLD_CONTRACT]

```ts contract
// The scale is the one domain→range function family: construct, configure by chained policy, read as a function. visx re-exposes this exact object behind config-object factories.
const y = scaleLinear().domain(extent(rows, (r) => r.value) as [number, number]).range([height, 0]).nice()
const x = scaleBand<string>().domain(rows.map((r) => r.key)).range([0, width]).padding(0.2)   // bandwidth()/step() feed bar geometry
const heat = scaleSequential(interpolateViridis).domain([lo, hi])                              // data-value colormap — never a UI palette

// Array folds replace hand-rolled loops: group/rollup/bin are the reshape vocabulary every chart fold uses before a mark sees data.
const byKind = rollup(rows, (g) => sum(g, (r) => r.value), (r) => r.kind)                      // Map<kind, total>
const bins = bin<Row, number>().value((r) => r.score).thresholds(thresholdFreedmanDiaconis)(rows)

// Shape generators emit path data headlessly — the string lands on a React-rendered <path d>, never on a d3-selection.
const d = line<Point>().x((p) => x(p.t)!).y((p) => y(p.v)).curve(curveMonotoneX)(points)
```

## [03]-[INTEGRATION]

[STACK: under `@visx/*` (`.api/visx-scale.md`, `.api/visx-shape.md`, `.api/visx-axis.md`)] — visx IS this substrate with a React skin: `@visx/scale` wraps `d3-scale` factories into config objects, `@visx/shape` renders `d3-shape` generators as components, `@visx/axis` renders what `d3-axis` would have DOM-painted. Where visx exposes the wrapped capability, the visx spelling wins in a component body; raw d3 members appear in the FOLD that prepares data (`rollup`/`bin`/`extent`/`ticks`) — chart math in the fold, chart rendering in visx.

[STACK: under `@observablehq/plot` (`.api/observablehq-plot.md`)] — Plot embeds d3 (scales, shapes, projections, schemes) behind its grammar; consumers pass channel/scale OPTIONS, not d3 objects. Direct d3 shows up only around Plot: `d3-array` folds shaping rows before marks, `format`/`timeFormat` specifier strings Plot accepts verbatim, and named projections Plot resolves internally.

[BOUNDARY: `d3-scale-chromatic` vs `system/token`] — the token authority (OKLCH via `colorjs.io`, APCA-gated) owns every UI-surface color: series strokes, categorical chart palettes, axis/grid inks all resolve from tokens. `d3-scale-chromatic` is admitted ONLY as the data-value colormap family (`scaleSequential(interpolateViridis)` heat/terrain/density encodings) where the color IS the datum's value; a `scheme*` categorical array standing in for the token palette is the split-brain defect.

[BOUNDARY: DOM-coupled modules — rejected] — `d3-selection`/`d3-transition`/`d3-zoom`/`d3-drag`/`d3-brush`/`d3-axis` never appear: React reconciles all chart DOM, `system/act` owns gesture (`@use-gesture`) and transition rails, deck.gl/maplibre own map interaction, and an axis is a `@visx/axis` component or a Plot `axisX`/`axisY` mark. `d3-fetch` is likewise rejected (platform HTTP owns I/O); `d3-timer`/`d3-ease`/`d3-dispatch` are headless but redundant against the rAF clock, motion easings, and Effect pubsub already owned elsewhere.

[STACK: `d3-geo` beside the map stack (`.api/maplibre-gl.md`, `.api/deck.gl-core.md`, `.api/turf-turf.md`)] — the live basemap/camera is maplibre + deck.gl and analytic geodesy is turf; `d3-geo` serves the NON-map geo needs — projected thumbnails/minimaps through `geoPath` on offscreen canvas, graticule/circle generation, and spherical math where a chart (not a map) plots geography.

## [04]-[RAIL_LAW]

- Owns: headless chart math — scale construction, interpolation, array folds/binning/statistics, shape/path generation, number/time formatting vocabularies, color-space math, data-value colormaps, geo projection math, hierarchy/force/delaunay/quadtree layout kernels, deterministic random sources.
- Accept: d3 members in data folds and geometry preparation; visx/Plot as the rendering surfaces over this math; `piecewise`/`quantize`/`interpolateZoom` for value paths; `randomLcg`-seeded sources where reproducibility is asserted; `pathRound` for stable snapshot output; specifier strings (`format`, `timeFormat`) as data rows on axis/legend policy.
- Reject: any DOM-coupled module (`selection`/`transition`/`zoom`/`drag`/`brush`/`axis`); d3 as a charting surface beside visx/Plot; `scheme*` categorical arrays where the token palette rules; hand-rolled loops re-deriving `rollup`/`bin`/`extent`; `d3-fetch` at any boundary; a second easing/timer/dispatch vocabulary shadowing `system/act` and the Effect rails.
