# [PY_ARTIFACTS_DELTA]

The binary diff/patch arm of the package close — incremental artifact delta bundles keyed by a parent content key. Where `package/codec#CODEC` compresses one payload against nothing and `package/archive#ARCHIVE` folds many payloads into one container, `Delta` diffs ONE to-image payload against a parent from-image and stores only the compressed patch, so an artifact that revises a near-identical predecessor ships the delta, not the whole blob. It COMPOSES the `package/codec#CODEC` `Bundle`/`CompressionAlgo`/`CodecProfile`/`BundleEvidence` scaffold rather than re-owning it: `DeltaKnobs` (`from_image`, `parent_key`, `patch_type`, `algorithm`, `compression`) is the one `CodecProfile` knob struct the codec union carries for the `delta` case, and `_delta_in_process`/`_delta_unpack` are the worker arms the codec `_in_process`/`_unpack_in_process` dispatch delegates the `delta` tag to. The content key threads the `DELTA` parent-key per the ARCHITECTURE seam (`package/delta ← python:runtime [CONTENT_KEY]: DELTA parent-key from-image`) — the from-image is the only side-input the delta round-trip needs, so a content-addressed delta keyed by `parent_key` is the storage shape, never two patch entrypoints per kind. `detools` owns the superset diff engine (`bsdiff`/`hdiffpatch`/`suffix_array` native extensions) — a hand-rolled bsdiff or hdiffpatch suffix-array diff is the deleted form.

## [01]-[INDEX]

- [01]-[DELTA]: the `DELTA` (`detools` `create_patch`/`apply_patch`/`patch_info` against a parent from-image) binary-delta arm — the `DeltaKnobs` profile struct carrying the parent bytes and `parent_key`, the `DeltaPatchType`/`DeltaAlgorithm`/`DeltaCompression` `Literal` axes whose `(algorithm, patch_type)` combination selects the create kernel, the `Bundle.delta` parent-binding factory (declared at the `package/codec#CODEC` owner, summarized here), and the `_delta_in_process`/`_delta_unpack` worker pair the `package/codec#CODEC` dispatch delegates to, on the cp315 core band.

## [02]-[DELTA]

- Owner: `Delta` the binary diff/patch concern over the one `CompressionAlgo.DELTA` row, composing the `package/codec#CODEC` `Bundle` owner; `DeltaKnobs` the frozen delta policy struct (`from_image: bytes` the parent artifact bytes, `parent_key: ContentKey` the parent's runtime content key, `patch_type`, `algorithm`, `compression`); `DeltaPatchType`/`DeltaAlgorithm`/`DeltaCompression` the three closed `Literal` axes whose call-strings ride straight into the `detools` `create_patch` call rows. `Delta` owns no `Bundle`/`CompressionAlgo`/`CodecProfile`/`BundleEvidence`/`BundleManifest` type — those are the `package/codec#CODEC` scaffold this page extends through the one `CodecProfile` case and the two worker arms, so a delta bundle is one row on the one union, never a parallel delta owner.
- Cases: `CompressionAlgo.DELTA` binds `detools.create_patch`/`apply_patch` reading the `delta` `DeltaKnobs` profile case — matched by the codec `_in_process`/`_unpack_in_process` dispatch falling through to `_delta_in_process`/`_delta_unpack`, never an `if delta` branch. The `(algorithm, patch_type)` combination selects the create kernel (`bsdiff`×`sequential`/`in-place`/`bsdiff`, `hdiffpatch`×`hdiffpatch`, or `match-blocks`) through the `DeltaPatchType` (`"sequential"`/`"in-place"`/`"bsdiff"`/`"hdiffpatch"`), `DeltaAlgorithm` (`"bsdiff"`/`"hdiffpatch"`/`"match-blocks"`), and `DeltaCompression` (`"bz2"`/`"crle"`/`"lzma"`/`"zstd"`/`"lz4"`/`"heatshrink"`/`"none"`) `Literal` axes, never a per-mode patch type — the axes ARE the `detools` call-row strings, so the serialized `DeltaKnobs` carries no native patch-kind handle and the profile stays content-key-foldable.
- Modality: the `DELTA` arm is single-image diff/apply against a parent — `Bundle.delta(from_image, parent_key, payload, ...)` (the parent-binding factory the `package/codec#CODEC` owner declares) binds the parent at the call site so the delta keys by `parent_key`. `_delta_in_process` runs `detools.create_patch(BytesIO(from_image), BytesIO(payload), fpatch, patch_type=..., algorithm=..., compression=...)` diffing the parent against the to-image payload and writing the compressed patch the bundle stores; `_delta_unpack` reconstructs the to-image through `detools.apply_patch(BytesIO(from_image), BytesIO(patch), fto)` (the single reconstruction surface that peeks the header type and dispatches sequential vs hdiffpatch internally) and reads `detools.patch_info(BytesIO(patch))` for the patch-kind metadata the receipt carries; the parent `from_image` is the only side-input the delta round-trip needs, so a content-addressed delta keyed by `parent_key` is the storage shape, never two patch entrypoints per kind. The recovered to-image is one `(name, size, digest)` row keyed `f"from-{parent_key.hex}"` so the recovered member traces its parent, feeding the codec `BundleManifest.of` content-key fold.
- Compression: the `DeltaCompression` axis selects the patch-payload codec row at create time — `bz2`/`crle`/`lzma`/`zstd`/`lz4`/`heatshrink`/`none` is a `compression` row on `create_patch`, never a parallel patch container; `apply_patch` resolves the codec from the self-describing patch header, so `unpack` re-supplies neither the compression nor the algorithm, only the parent and the patch. The codec backends are the admitted siblings `lz4`/`zstandard`/`heatshrink2` and stdlib `bz2`/`lzma` that `detools` selects and frames — the delta arm never re-implements the codec, and the default `zstd` compression rides the same `zstandard` core the `package/codec#CODEC` `ZSTD` arm owns.
- Receipt: each delta pack contributes the `package/codec#CODEC` `BundleEvidence.measure` projection onto `core/receipt#RECEIPT` `ArtifactReceipt.Bundle` — the `frame_size` slot carries the to-image payload length (the reconstructed-content size, not the patch length), the `level` and `dict_id` slots stay zero (the patch compression is header-encoded, not a single integer level), and the `entries`/`verified` container slots stay zero (a delta owns no container-level CRC re-read). `_delta_in_process` returns the `(blobs, BundleEvidence)` pair the codec `_in_process` dispatch threads onto `_emit`, so the delta arm mints no receipt of its own; the `detools.patch_info` patch-kind metadata is interior evidence the round-trip captures, not a new receipt field. The shared `package/codec#CODEC` `BundleEvidence` value crosses no module boundary into the receipt owner — the codec `_emit` spreads its named fields onto the flat-scalar `Bundle` case.
- Packages: `detools` (`create_patch(ffrom, fto, fpatch, compression=, patch_type=, algorithm=, suffix_array_algorithm=, ...)`, `apply_patch(ffrom, fpatch, fto) -> int`, `patch_info(fpatch, fsize=None) -> (str, info)`, `Error` the single failure rail; the native `bsdiff`/`hdiffpatch`/`suffix_array` C extensions built from the sdist) on the cp315 core (locked `0.53.0`, the C extensions source-built); the patch-payload codec backends `lz4`/`zstandard`/`heatshrink2` and stdlib `bz2`/`lzma` are the `detools` runtime deps it selects through the `compression` row, never re-admitted by this page; the `package/codec#CODEC` `Bundle`/`CompressionAlgo`/`CodecProfile`/`BundleEvidence` scaffold and runtime (`content_identity.ContentKey` the `parent_key` carried in `DeltaKnobs`) consumed as the codec union's imports, never re-declared.
- Growth: a new patch type is one token on the `DeltaPatchType` `Literal` (and one `(algorithm, patch_type)` create-kernel combination `detools` already owns); a new diff algorithm is one token on `DeltaAlgorithm`; a new patch-payload codec is one token on `DeltaCompression` (the codec `detools` selects and frames); a new firmware data-format diff rides the `detools` `data_format`/`from_*`/`to_*` offset rows as additional `DeltaKnobs` fields, never a per-architecture patch surface; a new delta evidence fact rides the existing receipt slots. Zero new surface beside the two worker arms and the one profile struct.
- Boundary: a hand-rolled bsdiff/hdiffpatch suffix-array diff where `detools` owns the superset (subsuming `bsdiff4` and adding `hdiffpatch`, in-place segmentation, and firmware data-format awareness), a per-mode patch type per `patch_type`/`algorithm`/`compression` where the call-string axes combine, a second apply entrypoint per patch kind where `apply_patch` already dispatches on the self-describing header, a re-implemented `lz4`/`zstd`/`heatshrink` patch codec where `detools` selects it, a re-supplied compression/algorithm on `unpack` where the patch header carries them, a parallel `BundleManifest`-per-delta type, and a re-owned `Bundle`/`CompressionAlgo`/`BundleEvidence` where the `package/codec#CODEC` scaffold already owns them are the deleted forms; this page owns the one delta worker arm pair and the one profile struct composing the codec union, and re-mints no content key beyond the parent-traced member key. The arm resolves on the cp315 core (the `detools` C extensions source-build on cp315), so it never crosses the runtime subprocess lane the `package/codec#CODEC` gated `LZ4`/`BROTLI` arms ride.

```python signature
from io import BytesIO
from typing import Literal, assert_never

from msgspec import Struct

from rasm.runtime.content_identity import ContentKey

from artifacts.package.codec import BundleEvidence, CodecProfile, CompressionAlgo

type DeltaPatchType = Literal["sequential", "in-place", "bsdiff", "hdiffpatch"]
type DeltaAlgorithm = Literal["bsdiff", "hdiffpatch", "match-blocks"]
type DeltaCompression = Literal["bz2", "crle", "lzma", "zstd", "lz4", "heatshrink", "none"]


class DeltaKnobs(Struct, frozen=True):
    from_image: bytes
    parent_key: ContentKey
    patch_type: DeltaPatchType = "sequential"
    algorithm: DeltaAlgorithm = "bsdiff"
    compression: DeltaCompression = "zstd"


def _delta_in_process(payloads: tuple[bytes, ...], algo: CompressionAlgo, profile: CodecProfile) -> tuple[tuple[bytes, ...], BundleEvidence]:
    match profile:
        case CodecProfile(tag="delta", delta=DeltaKnobs() as k):
            import detools

            patch = BytesIO()
            detools.create_patch(BytesIO(k.from_image), BytesIO(payloads[0]), patch, patch_type=k.patch_type, algorithm=k.algorithm, compression=k.compression)
            blob = patch.getvalue()
            return (blob,), BundleEvidence.measure(algo, 0, 0, len(payloads[0]), 0, payloads, (blob,))
        case _:
            assert_never(profile)


def _delta_unpack(blob: bytes, algo: CompressionAlgo, profile: CodecProfile) -> tuple[tuple[str, int, bytes], ...]:
    match profile:
        case CodecProfile(tag="delta", delta=DeltaKnobs() as k):
            import detools

            sink = BytesIO()
            detools.apply_patch(BytesIO(k.from_image), BytesIO(blob), sink)
            payload = sink.getvalue()
            return ((f"from-{k.parent_key.hex}", len(payload), payload),)
        case _:
            assert_never(profile)
```
