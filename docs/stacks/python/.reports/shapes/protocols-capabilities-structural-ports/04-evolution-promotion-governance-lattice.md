# Evolution Promotion Governance Lattice

# Promotion Unit Atomicity

- A promotion unit is the smallest mergeable governance atom for one `port_symbol` — all catalog columns, bind records, mirror extractions, pipeline laws, and consumer changelog entries in the unit land together or revert together.
- Partial promotion — new `get_protocol_members` name without `negative-partial` defect row, or `tier_table_hash` rotation without `tier-traversal` law refresh — fails `frozenset_drift` and `orphan_port_symbol` CI gates.
- Multi-port promotion units are permitted only when a single backend SDK upgrade alters invoked surfaces on multiple mirror ports owned by one boundary package — unit lists every affected `port_symbol` and shared `promotion_unit_id`.
- Cross-context fan-out promotion units name obligated `consumer_contexts` from catalog rows — consumers upgrade anti-corruption imports in the same commit window as providing-context catalog diff or enter documented quarantine posture.
- Promotion unit metadata carries `promotion_unit_id`, `port_symbols`, `event_kind`, `rollback_eligible`, `forward_fix_exemption_ref`, and `proof_layers_to_replay` — undocumented promotion paths are merge blockers.

# Member Lifecycle Event Kinds

- **Required member addition** — breaking event; mirror extraction, usage-graph proof of zero stale call sites on retired paths, frozenset hash rotation, one `negative-partial` row per new name, bind factory signature updates, and every `register_law` witness block refresh in one unit.
- **Required member retirement** — breaking event; usage-graph diff proves zero production invocations; intermediate release may mark member documented-only with static checker warnings; removal requires frozenset hash rotation and retired slug documentation in changelog.
- **Optional capability via default member** — non-breaking when default body is pure and side-effect free; `default-member` slug proof re-runs; nominal mixin ABC collapse when defaults preserve third-party structural assignability.
- **Signature translation on mirror** — breaking when invoked parameter or return shape changes; static assignability matrix and bare structural assignment tests update without widening domain call sites; SDK methods not called by owning module do not expand mirror surface.
- **Scheduling model split** — breaking; `async_family` transitions to `dual_split`; async mirror protocol row and `async-shell-fanout` slug land; `sync_async_bridge` negative control must fail on existing sync port.
- **Refinement chain link insertion** — breaking when new middle link used in runtime gates; independent `@runtime_checkable` on every link in chain per GH-132604 policy; child frozenset must superset parent names with compatible semantics.

# Frozenset Hash Rotation Contract

- `member_frozenset_hash` fingerprints `frozenset(get_protocol_members(Port))` at unit close — rotation emits changelog row listing added and retired member names beside literal frozenset diff output.
- Hash rotation without paired `law_slugs` tuple diff fails catalog completeness review — added members require new negative defect ids; retired members remove obsolete defect rows unless static-only ports retain historical xfail modules under `transition_id`.
- Golden frozenset tests in CI block merge until promotion unit merges — unexpected diff on main without matching `promotion_unit_id` in changelog indicates silent SDK drift.
- Parameterized port keys remain unbound `type[Port]` in bind rows — hash rotation on generic `Port[T]` applies to unbound protocol class; call-site instantiation updates are static-layer proof only, not bind row key changes.
- Intersection join promotion rotates hash to union of parent member sets — join row decoration updates when `undecorated_join` defect id is in `negative_defect_ids` and runtime gate names the join type.

# Law Slug Diff Calculus

- Promotion unit declares `law_slugs_added` and `law_slugs_retired` tuples — harness registration smoke asserts `register_law(Port, slug)` entries match post-promotion `law_slugs` column exactly.
- Adding `typeis-semantic` slug requires named predicate symbols in `typeis_semantic` sub-row — lambdas forbidden; promotion unit includes predicate module diff.
- Adding `kleisli-bind` or `fault-severity-routing` slugs requires pipeline law module refresh — fault union variants added at a hop update seam projection rows in the same unit.
- Retiring `runtime-smoke` slug is permitted only when all runtime gates on the port collapse to `TypeIs` nominal predicates or delete — catalog `evidence_grades` column updates and `inherited_runtime_checkability` gate waived with explicit collapse `transition_id`.
- Adding `rebind-metamorphic` slug obligates metamorphic sub-rows with `handoff_stages` and `adapter_identity_must_differ=true` — promotion unit names egress participation rationale when slug was previously absent.

# Bind Row Co-Evolution

- `bind_row_id` factory signatures, `tier_table_hash`, and `scope_mutation` pattern update in the same unit as port member promotion — orphan bind record without catalog row or inverse fails `orphan_port_symbol` gate.
- `factory_symbol` return type remains `Result[Port, ConfigFault]` — credential binding and endpoint selection changes are bind promotion events, not domain interior edits.
- Tier reordering rotates `tier_table_hash` and refreshes `tier-traversal` law — property tests assert short-circuit supremum; weaker backends must not execute after first `Ok(Certified[C])`.
- Version band changes update `Map[Ver, Callable]` rows and `version-gate-ingress` law — closed `Ver` literal band edits forbid free-string interim migration paths.
- `ContextVar` absorb wrapper placement changes are bind-edge events — smart constructors and model validators remain forbidden from `ask(Port)`.

# Tier And Version Band Promotion

- `tier_policy` transitions — `none` to `ordered_short_circuit`, or addition of `version_gated` — are promotion units distinct from member addition when tier tables already exist but policy metadata was wrong.
- Tier promotion adds `tier-traversal` slug and `Certified[C]` provenance metadata on bind row — traversal tables hash separately from member frozenset hash.
- Version promotion adds companion `Versioned` metadata protocol or closes `Ver` literal band — `Unsupported` and `Sunsetted` fault shapes update at ingress Kleisli chain, not mid-interior.
- Sunset threshold edits are version promotion events — interior hops must not re-check sunset bands after root resolution; fault payload shapes at gate update in same unit.
- Combined `version_and_tier` policy promotion sequences version resolution before tier traversal in handoff graph — unit documents gate order witness in pipeline law module header.

# Async Family Split Promotion

- `async_family` transition from `sync` to `dual_split` mandates separate async mirror protocol, `@effect.async_result` generator sites at composition shell, and `async-shell-fanout` slug — sync call sites remain unchanged in the same unit.
- Bridging sync blocking IO through `async def` stubs on sync port methods is forbidden as promotion shortcut — `sync_async_bridge` negative defect must fail static proof before unit merges.
- `dual_split` promotion adds async catalog row fragment or `async_family` column update on same `port_symbol` when one mirror serves both families — law slugs union must list both pipeline shell types without duplicate `port_symbol` rows unless anti-corruption remap documents fork.
- Retiring async family — collapsing to `sync` only — requires proof that no async call sites remain in usage graph; `async-shell-fanout` slug retires with explicit `transition_id`.

# Cross-Context Consumer Choreography

- Providing context publishes promotion changelog tied to `member_frozenset_hash` diff and `law_slugs` delta — consuming contexts upgrade anti-corruption imports on obligated `consumer_contexts` list before integration smoke executes.
- Port symbol rename requires explicit anti-corruption remap at importing boundary module — silent re-exports of renamed ports fail consumer seam tests; remap row records `old_symbol`, `new_symbol`, `sunset_date`.
- Duplicate mirror ports across contexts collapse to one `port_symbol` owner in promotion unit — parallel identical mirrors in two `consumer_contexts` lists fail merge review unless fork documents sunset.
- Consumer-negative-import slug proofs re-run per context after bind factory signature change — vendor implementation modules must remain unreachable from consuming domain packages.
- Cross-context handoff continues transmitting canonical shapes and immutable configuration records — promotion units do not introduce pickled capability or shared mutable singleton backends across processes.

# Intersection Join And Refinement Promotion

- Splitting monolith port into intersection joins — `Readable[T]` × `Writable[T]` — is preferred promotion path over widening fat mirrors; unit adds join catalog rows, decoration on join when runtime gate names join, and pipeline ingress parameter updates.
- Collapsing join to monolith is permitted when single production implementer and no planned substitution — collapse `transition_id` retires join row and merges `law_slugs` into monolith row; harness collapse review documents rationale.
- Refinement chain insertion adds `refinement_link_ids` column entries — each link receives independent `@runtime_checkable` when `runtime_smoke` is in `evidence_grades`; partial re-decoration fails `inherited_runtime_checkability` gate.
- Incompatible refinement signatures split into separate `port_symbol` rows rather than coerced join promotion — promotion unit must not merge incompatible parent and child return types on one join type.

# Mirror Trim And Fat Surface Retirement

- Usage-graph diff proving zero invocations for mirror members triggers trim promotion — retired names rotate frozenset hash; `fat_mirror_unused` defect row proves negative control.
- Fat mirror widening without call-site need is rejected promotion — SDK surface growth does not expand mirror unless owning module invocation set grows in the same unit.
- Trim promotion may retire `generative` from `evidence_grades` when backend participation shrinks to smoke-only — harness layer binding updates.
- Boundary mirror modules co-evolve with catalog row — domain modules never import mirror definitions from vendor paths; trim is boundary-package-only edit set.

# Authorization Cap Band Promotion

- Adding capability token to gated port updates decoration-time `frozenset[Cap]` bands, gate negative-branch fault shapes, and authorization law rows in one unit — string comparison gates are not interim migration paths.
- Retiring capability token requires proof that no production gate still requires the token — frozenset subset algebra `req <= held` proofs update on representative decorator modules.
- Token renames are vocabulary promotion events on closed `StrEnum` bands — alias-only token edits preserve frozenset hash when scalar identity unchanged; new tokens rotate hash.
- Authorization promotion is orthogonal to member frozenset promotion — units may combine when one backend upgrade alters both invoked methods and advertised capability metadata.

# Rollback Versus Forward-Fix Arbitration

- Default promotion posture is rollback-first — partial unit landing on main without completed fan-out reverts entire `promotion_unit_id` per providing-context branch policy.
- Forward-fix permitted when rollback would orphan production adapters bound to retired SDK signatures and migration fold proves active read-path on obsolete port member set — `forward_fix_exemption_ref` documents store or adapter dependency proof.
- Forward-fix forbidden on frozenset collision or duplicate `port_symbol` catalog rows — vocabulary consolidate on port owner is sole legal path.
- Forward-fix forbidden when E1-style duplicate validation surfaces appear — two parallel mirror ports for same capability in one context after promotion indicates failed collapse review.
- Arbitration witness row — `promotion_unit_id`, `rollback_eligible`, `forward_fix_eligible`, `exemption_document_ref` — composition root owns row; contexts do not self-arbitrate promotion conflicts.

# Deprecation Intermediate Releases

- Breaking member retirement uses at least one intermediate release when production usage graph still references the member — static checker warnings and documented-only member status precede frozenset removal.
- `@runtime_checkable` deprecation on a link in refinement chain follows GH-132604 timeline — promotion unit schedules re-decoration before Python 3.20 hard `TypeError`.
- Changelog entries name `sunset_date` for retired `law_slugs` and `bind_row_id` factory replacements — consumers read changelog before upgrading anti-corruption imports.
- Intermediate release does not relax CI gates — `frozenset_drift` and `orphan_port_symbol` remain merge blockers; deprecation is usage-graph and documentation posture, not gate suspension.

# Promotion Gate Admission Rows

- Each promotion unit declares frozen admission row beside port owner — `promotion_unit_id`, `event_kind`, `port_symbols`, `pre_hash`, `post_hash`, `law_slugs_delta`, `bind_row_delta`, `consumer_obligations`, `proof_layers_to_replay`.
- `event_kind` draws from closed band — `{member_add, member_retire, default_member, mirror_trim, async_split, tier_reorder, version_band, join_split, join_collapse, cap_token, refinement_insert, factory_signature}` — free-string event labels fail harness discovery.
- `consumer_obligations` lists per-context actions — `{import_remap, factory_pin, negative_import_replay, changelog_ack}` — empty when providing context is sole consumer.
- `proof_layers_to_replay` subsets harness layers — static and frozenset layers replay before generative and metamorphic when unit touches pipeline or egress participation.
- Admission row without matching post-promotion catalog row state fails done criteria — row is the promotion intent; catalog row is the promotion result; drift between them blocks merge.

# CI Promotion Blockers

- Merge blocked when `pre_hash` to `post_hash` diff lacks matching admission row `promotion_unit_id` in changelog.
- Merge blocked when any `law_slugs_added` entry lacks `register_law` witness and parametrized pytest row.
- Merge blocked when `consumer_obligations` name context not yet running `consumer-negative-import` replay in same CI wave.
- Merge blocked when `async_split` event merges without `sync_async_bridge` negative control still failing on promoted sync port.
- Merge blocked when `interior_isinstance` grep regresses during promotion — port narrowing changes must not push conformance checks into domain interiors.
- Merge blocked when promotion adds ingress or wire `Protocol` field — seam altitude checklist.

# Failure Archaeology For Promotion Defects

- Post-merge runtime smoke failure after hash rotation — attribution targets incomplete `law_slugs_added` or undecorated refinement link in promotion unit, not backend brand first.
- Consumer import failure after providing-context merge — attribution targets missing `import_remap` obligation or `sunset_date` not yet reached on old `port_symbol`.
- Tier traversal order regression — attribution targets `tier_table_hash` rotation without `tier-traversal` property refresh in same unit.
- Version fault appearing mid-interior after promotion — attribution targets `version-gate-ingress` law omission or interior re-check introduced during promotion diff.
- Metamorphic re-bind failure after promotion — attribution targets missing `rebind-metamorphic` slug add when egress participation began in same unit.
- Diagnostic tags on promotion defects name `promotion_unit_id`, `event_kind`, and `port_symbol` — undifferentiated adapter errors indicate missing admission row metadata.

# Promotion Anti-Patterns

- Land mirror widening to match full vendor SDK when usage graph invokes subset — fat mirror promotion without load-bearing capability.
- Remove port member in same unit that production usage graph still references — silent assignability break until runtime smoke fails.
- Promote through nominal wrapper class duplicating every port member — structural injection or boundary pipeline preserves assignability.
- Add `@runtime_checkable` during promotion solely to feed `isinstance` where `TypeIs` is honest evidence grade — downgrades lattice discipline.
- Split sync and async by declaring `async def` on sync port method — forbidden shortcut indexed `sync_async_bridge`.
- Update golden frozenset without `negative-partial` rows for new required names — introspection green while behavior laws stale.
- Promote consumer context before providing-context catalog row merges — fan-out ordering inversion causes orphan import remap.
- Pickle port instances across processes as part of promotion-driven worker handoff change — settings cross; adapters rebuild.

# Harness Replay After Promotion

- Promotion unit close triggers ordered replay — static assignability and checker trinity parity, frozenset golden diff, runtime smoke with re-decoration audit, semantic `TypeIs` predicates, then generative and metamorphic layers declared in `proof_layers_to_replay`.
- Partial replay omitting static layer before generative is proof anti-pattern — catalog `evidence_grades` column governs subset but ordering is mandatory.
- Mutation testing reruns on rows where promotion touched `kleisli-bind`, `tier-traversal`, or `version-gate-ingress` slugs — interior folds on narrowed ports remain secondary surfaces.
- Promotion unit may declare `metamorphic_rebind` replay only on affected sub-rows — read-only observation ports omit write stages with documented rationale unchanged.
- Rollback unit replays `pre_hash` catalog state witnesses — git revert alone insufficient without admission row marking `rollback_eligible=true` and consumer obligation `changelog_ack`.

# Promotion Witness Row Linkage

- Each closed promotion unit emits one witness row beside the port owner — `promotion_unit_id`, `admission_row_hash`, `post_catalog_row_hash`, `changelog_revision`, `rollback_witness_ref` optional.
- `admission_row_hash` fingerprints the frozen admission row at unit open — post-merge catalog row hash must match `post_catalog_row_hash` column or `frozenset_drift` gate fires on next CI run.
- Witness rows bind to catalog rows by `port_symbol` foreign key — harness parametrizes promotion replay tests on `(port_symbol, promotion_unit_id)` axis, not pytest file paths.
- Rollback witness rows reference `pre_hash` catalog snapshot and `law_slugs` tuple before unit — git revert without witness row leaves harness discovery pointing at retired slugs.
- Changelog `revision` monotonically increases per `port_symbol` — consumers pin `changelog_revision` floor in anti-corruption modules until remap completes.

# Consumer Remap Sub-Schema

- Port symbol rename promotion carries remap row — `old_symbol`, `new_symbol`, `anti_corruption_module`, `sunset_date`, `import_paths_affected` tuple.
- `import_paths_affected` lists bounded-context import sites requiring alias shim or direct symbol swap — grep gate fails merge when listed path still imports `old_symbol` after `sunset_date`.
- Alias shim modules are boundary-local and time-boxed — `sunset_date` retires shim; consuming domain packages never re-export shim symbols in public `__all__`.
- Factory rename promotion updates `factory_symbol` in bind row and remap row — `resolve_*` public name stability is providing-context contract; internal factory symbol may change when export name is unchanged.
- Multi-consumer remap orders providing-context merge before any consumer obligation marked `import_remap` — CI wave enforces topological sort on `consumer_contexts` dependency graph.

# Representative Promotion Examples

- Single-method port adds optional default member — `event_kind=default_member`, `law_slugs_added=("default-member",)`, non-breaking hash rotation when no new required names, `evidence_grades` unchanged.
- Filesystem port tier reorder — `event_kind=tier_reorder`, `tier_table_hash` rotation, `law_slugs` retains `tier-traversal`, property tests vary candidate order in same unit.
- Codec port closes `Ver` literal band — `event_kind=version_band`, `version-gate-ingress` law refresh, `Unsupported` fault shape update, interior hops unchanged.
- Monolith split to `Readable` and `Writable` joins — `event_kind=join_split`, two new catalog rows plus monolith retirement row with `transition_id`, pipeline ingress parameters narrow per stage.
- SDK signature change on invoked mirror method — `event_kind=mirror_trim` or `factory_signature` when only bind factory translates; mirror member signatures update without fat surface growth.

# Assurance-Promotion Temporal Law

- Capability federation admits three non-overlapping temporal phases — **promotion in-flight**, **reconciliation in-flight**, and **assurance-between-promotions** — composition root owns phase transitions; contexts do not self-advance, self-certify, or mark steady state across phase boundaries.
- **Promotion in-flight wins** — when a promotion unit is open (`promotion_unit_id` pinned by admission row, partial fan-out witness gap on `consumer_obligations`, or port quarantine on promotion diff), promotion obligations from this document take precedence over sustain-phase assurance: mirror extraction, frozenset hash rotation, `law_slugs` diff, bind row co-evolution, and consumer `import_remap` must complete before steady-state invariants P1–P10 bind as certification truth; assurance must not widen mirror surface, hand-edit catalog rows, or patch bind factories to absorb partial evolution — `frozenset_drift` and `orphan_port_symbol` remain merge blockers, not cues to relax scheduled replay.
- **Reconciliation in-flight wins** — when catalog convergence fails, bind graph skew fires, scope map drift exceeds grace policy, or catalog reconcile choreography is incomplete on obligated contexts, reconciliation obligations take precedence over assurance certify and steady-state mark: catalog reconcile, bind co-evolution review, and scope re-bind at composition root must complete before post-promotion certify row; assurance must not patch domain interiors, relax `consumer-negative-import` proofs, or bypass anti-corruption remap to clear H-signals.
- **Assurance between promotions** — default sustain posture when no promotion unit is in flight, no catalog reconcile run is active, and no port quarantine blocks affected edges: P1–P10; scheduled replay, health tick, and deep assurance run on cadence; catalog, bind, scope, law, and consumer convergence morphisms assert logical revision stability across obligated `consumer_contexts` — not wall-clock synchronization.
- **Pre-merge is promotion-adjacent, not sustain** — pre-merge witness P6 plus fan-out obligation set runs on git diff before merge lands via change classifier; it blocks partial landing and replays `consumer-negative-import` on diff set — it does not substitute for post-promotion certify and does not discharge sustain-phase scheduled replay when no promotion diff is present.
- **Post-reconciliation certify bridges phases** — post-promotion certify row full P1–P10 runs once per committed promotion unit after catalog reconcile witness commit when promotion triggered convergence restoration, or after standalone reconcile when H-signal escalated during sustain; steady-state mark re-enters only after certify passes — capability equilibrium suspended from first edge-touching port edit or catalog skew signal through certify completion.
- **Assurance deferral under in-flight promotion or reconciliation** — scheduled replay, deep assurance, and convergence replay on `consumer_contexts` defer when promotion unit or catalog reconcile run is active on those contexts; boot witness P1–P3 and health tick P3, P5 on unaffected port edges may continue when root policy documents scoped degradation — assurance-paused degradation never defers pre-merge on promotion diffs, harness replay on `proof_layers_to_replay`, or post-reconciliation certify on the landing `promotion_unit_id`.
- **Substrate freeze law** — catalog rows, bind records, scope map registrations, mirror extraction modules, tier tables, and `register_law` witness tables specialize once per logical revision; promotion in-flight and catalog reconcile R1–R2 are the only lawful windows for port-edge mutation — assurance between promotions treats those artifacts as read-only witnesses except via classified edge-touching promotion per change classification.
- **Gate ordering** — boot before interior handoff; pre-merge before merge; harness replay before post-promotion certify; scheduled replay independent of request volume; health tick continuous during sustain; deep assurance last on release or mirror topology-change cadence — scheduled replay failure blocks deep assurance start; same static-before-generative ordering as harness layer binding.
- **Assurance versus reconciliation** — assurance loops are proactive on cadence, boot, post-promotion certify request, and health tick; reconciliation loops are reactive on H1–H6 health signals, frozenset revision lag, scope skew, promotion quiescence gap, and catalog reconcile cadence; assurance row failure on P1 or P2 escalates to H-signal first — repeated failure, grace expiry, or tick budget exceeded maps to catalog reconcile row and reconciliation phase — assurance does not patch mirror or bind factory to avoid reconcile.
- **Lifecycle phases** — design establishes catalog row and bind graph before cross-context port imports activate; promote suspends steady state until post-promotion certify; restore revokes steady-state mark until certify re-commits when H-signals fire during sustain; sustain holds assurance cadence between promotion events — composition root owns all phase transitions; contexts do not skip restore when catalog or bind skew signals fire.
- Temporal witness row — `phase` (`promotion_in_flight` | `reconciliation_in_flight` | `assurance_between_promotions`), `promotion_unit_id`, `reconciliation_run_id`, `quarantine_posture`, `deferred_assurance_rows`, `certify_pending` — composition root exports row; adapters read phase before port handoff acceptance; false steady state when phase is not `assurance_between_promotions` but certification row marks steady is equilibrium violation.

# Temporal Sunset Choreography

- `sunset_date` on remap rows and retired `law_slugs` is calendar-bound, not commit-bound — consumers may lag one release cycle when `changelog_ack` obligation is documented.
- Providing context maintains read-only `old_symbol` catalog row fragment until `sunset_date` — fragment rows carry `transition_id` and `successor_port_symbol` for harness discovery only, not for new bind registrations.
- Post-sunset CI job fails any import of `old_symbol` in `import_paths_affected` — job binds to promotion witness row, not ambient grep without row metadata.
- Trusted-replay pins update in version promotion units — encoder and factory module pins rotate with `version_band` events; replay chains skip canonical re-validation but re-bind ports from updated pins.

# Multi-Backend Coordination Rows

- When `tier_policy` is `ordered_short_circuit`, promotion units list every backend type in tier table — adding backend requires new tier row, traversal law refresh, and smoke row for semantic `TypeIs` predicate.
- Removing backend from tier table is breaking when production wiring still references factory bind row — usage-graph proof precedes tier row deletion.
- Primary backend promotion that changes attestation fields updates `typeis-semantic` sub-row and `Certified[C]` carrier shape — tier evidence in fault payloads remains stable vocabulary.
- Fallback tier promotion must not weaken short-circuit supremum — inserting weaker backend before stronger is tier reorder event with explicit traversal property replay.

# Catalog Diff Emission Contract

- Providing context exports machine-readable catalog diff per promotion unit — `port_symbol`, `pre_hash`, `post_hash`, `law_slugs_added`, `law_slugs_retired`, `bind_row_id` delta summary.
- Diff emission precedes consumer merge window — consuming contexts consume diff, not live providing-context source tree, when determining remap obligations.
- Diff without matching admission row `promotion_unit_id` is invalid emission — harness rejects orphan diffs at registration smoke.
- Combined multi-port diff lists shared `promotion_unit_id` once — per-port hash deltas appear as sub-rows keyed by `port_symbol`.
