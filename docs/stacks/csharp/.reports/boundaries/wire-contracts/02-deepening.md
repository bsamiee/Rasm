# Wire-Contracts: Deepening

[SERIALIZATION_LIFECYCLE_HOOKS_AND_CONTRACT_INVARIANTS]:
- `IJsonOnDeserializing` and `IJsonOnDeserialized` are interface-level contracts on any type in the object graph; implementing the interface is sufficient — no attribute involvement — and the runtime invokes them before and after property assignment respectively. The boundary pattern is to implement them on the attributed wire record, not the domain type: `IJsonOnDeserializing` initializes a scratch field such as a seen-field bitmask, and `IJsonOnDeserialized` asserts that all required fields were populated. The callback either produces a validated value via the factory or throws a `JsonException` captured and translated by the enclosing `liftIO` + `.Catch` rail.
- `JsonTypeInfo.OnDeserializing` and `JsonTypeInfo.OnDeserialized` (callback delegates on the metadata instance) are the equivalent modifier-based attachment points for types the caller does not own. Both fire synchronously on the deserialization thread; they cannot be async, cannot be re-entrant, and must not write to shared state. Assigning them via `WithAddedModifier` keeps the domain type clean while enforcing cross-cutting wire invariants — such as schema-version range checks on every envelope type — from one policy site.
- `JsonTypeInfo.OnSerializing` fires before property enumeration begins; at that point `JsonTypeInfo.Properties` is already frozen. A modifier that injects a `ShouldSerialize` predicate onto a property via `JsonPropertyInfo.ShouldSerialize` must run before `MakeReadOnly()`, not inside `OnSerializing`. An `OnSerializing` callback is the only valid pattern for dynamic predication that reads runtime state after freeze; property-level suppression before freeze is the correct pattern for static rules.

[WIRE_CONTRACT_PROPERTY_SHAPE_MECHANICS]:
- `JsonPropertyInfo.ShouldSerialize` is a `Func<object, object?, bool>` where the first argument is the parent object instance and the second is the property value. Returning `false` suppresses the property entirely — no key, no null. Role-based or capability-based field redaction at a boundary injects this predicate via a modifier on the source-generated context, holding a closure over an authorization snapshot; the domain record carries no serialization concerns.
- `JsonPropertyInfo.Get` and `JsonPropertyInfo.Set` can be replaced via modifier with arbitrary delegates that project through an additional transformation. Setting `Get` to a delegate that reads an `Option<T>` field and returns `null` when `IsNone` — combined with `[JsonIgnore(Condition = JsonIgnoreCondition.WhenWritingNull)]` — projects LanguageExt absence onto the wire as missing-key rather than explicit null, without adding a custom converter for the entire type.
- `JsonPropertyInfo.AssociatedParameter` exposes the constructor parameter linked to a property during `[JsonConstructor]`-attributed deserialization. When a modifier adds a new `JsonPropertyInfo` programmatically via `JsonTypeInfo.CreateJsonPropertyInfo`, that property will not have an `AssociatedParameter` and will be ignored during constructor-binding deserialization unless the type uses `JsonObjectCreationHandling.Populate`. Programmatically added properties on constructor-bound records must be on writable init-properties, or the type must use `Populate` mode.
- `JsonObjectCreationHandling.Populate` re-uses the existing instance on a property rather than replacing it, applying only to properties capable of instance-reuse: collections and classes with a parameterless constructor. For attributed payload records using positional constructors, `Populate` has no effect on those properties and the constructor-path binding dominates. This mode is correct for `[JsonExtensionData]`-backed overflow bags where an existing `Dictionary<string, JsonElement>` must accumulate unknown keys rather than be replaced.

[NULLABLE_AND_REQUIRED_CONTRACT_ENFORCEMENT]:
- `RespectNullableAnnotations = true` causes deserialization of a `null` JSON value into a non-nullable reference type property to throw `JsonException`. Enforcement happens before any `[JsonConstructor]`-attributed factory sees the value — the admission gate fires at the JSON token level, not at the factory. A wire type with `required string Field` where `null` arrives on the wire will throw before the domain factory is ever invoked, making the `[JsonRequired]` + `RespectNullableAnnotations` combination the outermost null-rejection layer.
- `RespectRequiredConstructorParameters = true` causes deserialization to throw when a JSON payload lacks a property that maps to a required constructor parameter. This fires even when the parameter has a default value — the presence of `required` on the parameter overrides the default. Without this flag, a missing JSON property silently binds the constructor default, which can produce domain objects in states the admission logic was never designed to reject.
- `[JsonNumberHandling(JsonNumberHandling.Strict)]` applied at the property level via attribute or via `JsonPropertyInfo.NumberHandling` set in a modifier is the correct posture for measurement fields: it prevents string-encoded numeric values — a common interop ambiguity — from silently binding to typed measurement fields. Physical-unit conversion fields must use `Strict` by default; `AllowReadingFromString` is opt-in only for protocols that document string-encoded numerics.

[WIRE_CODEC_COVERAGE_GAPS_AND_ALTERNATE_LOOKUP]:
- `FrozenDictionary<TKey, TValue>.GetAlternateLookup<TAlternateKey>()` returns an `AlternateLookup<TAlternateKey>` that performs lookups without allocating a `TKey` instance. When `TKey` is `string` and the comparer implements `IAlternateEqualityComparer<ReadOnlySpan<char>, string>`, the lookup accepts a `ReadOnlySpan<char>` directly from `Utf8JsonReader`'s span-based path. A single `static readonly FrozenDictionary<string, Action<...>>` initialized at startup with `GetAlternateLookup<ReadOnlySpan<char>>()` called once in the converter constructor eliminates both dictionary boxing and string allocation per property read.
- `Utf8JsonReader.HasValueSequence` must be checked before using `ValueSpan`; when `true`, the token value spans multiple `ReadOnlyMemory<byte>` segments and `ValueSequence` is the correct source. A custom converter that reads `ValueSpan` directly without checking `HasValueSequence` silently reads partial data on multi-segment pipeline inputs. `CopyString(Span<byte>)` normalizes this: it always returns the complete unescaped UTF-8 value regardless of segmentation at the cost of a stack allocation. `ValueIsEscaped` gates whether the span is already in raw form or requires unescaping.
- `Utf8JsonWriter.WriteRawValue(ReadOnlySpan<byte>, skipInputValidation: false)` validates the input as RFC 8259-compliant JSON before writing. At a pass-through boundary where a wire field carries an opaque JSON sub-document to be re-emitted verbatim, `skipInputValidation: true` is safe only when the input was received from a trusted internal source in the same request context. For externally-sourced payloads, leave validation enabled; the cost is proportional to the sub-document byte length.
- `JsonElement.GetRawText()` allocates a `string`. At high-throughput seams where the raw JSON sub-document must be compared or hashed but not decoded, `JsonElement.WriteTo(Utf8JsonWriter)` followed by `Utf8JsonWriter.BytesCommitted` is the allocation-minimizing path. `JsonElement.DeepEquals(a, b)` (static, no allocation) performs semantic JSON equality without producing text; use it for snapshot envelope checksum equivalence checks on the structured side before falling back to hash comparison.

[IO_MONAD_RESOURCE_AND_THREAD_SEAM_MECHANICS]:
- `IO<A>.Post` re-queues the IO computation onto the `SynchronizationContext` captured in `EnvIO` at the time the chain started. The captured context is the one present when `IO<A>.Run(EnvIO)` was called, not the context at the point `.Post` appears in the chain. A wire seam that calls `IO.Post` after a network awaitable marshals back to the caller's synchronization context only if `EnvIO` was created with a context present. When `EnvIO.New()` is called on a thread-pool thread with no synchronization context, `.Post` is a no-op and the continuation executes wherever the awaitable completes.
- `IO<A>.Local` creates an isolated cancellation scope whose `CancellationToken` child-links to the parent environment's token. Cancellation in the local scope signals only the child token; the parent scope continues. Wrapping the outbound call in `.Local` followed by `timeout(duration, ...)` produces a per-call cancellation boundary without cancelling the broader session. The local scope's resources are released when the scope exits regardless of cancellation.
- `IO<A>.WithEnv(Func<EnvIO, EnvIO>)` threads a modified `EnvIO` through a sub-computation and releases resources acquired within that sub-computation when it exits. `IO<A>.WithEnvFail(...)` releases only on failure, propagating the acquired resources upward on success. The correct pattern for a wire connection capsule is `WithEnv` (always-release at the seam boundary) combined with `bracketIO` at the inner call site for any sub-resources the connection acquires. Mixing `Bracket` and `WithEnv` on the same resource produces double-release if not structured carefully: only `WithEnv` should own connection teardown, and inner `bracketIO` should own stream and reader lifetimes.
- `IO<A>.Fork(Option<TimeSpan>)` queues the computation on the thread pool and returns an `IO<FiberIO<A>>` immediately. Resources acquired by the parent and passed into the fork are shared — documentation states they "should be treated with care." Wire boundary seams must not fork an `IO` that owns an undisposed connection handle acquired by the parent; the fork should receive a value-projected snapshot such as a deserialized message or typed receipt, with any resource acquisition inside the forked computation's own `bracketIO` scope.
- `Prelude.cancelToken` (the `IO<CancellationToken>` field) extracts the `CancellationToken` from the current `EnvIO`. Passing this token to `liftIO`-wrapped network calls integrates IO monad cancellation into BCL `CancellationToken`-accepting APIs without additional plumbing. The token reflects the state at the time `Run` was called, not at the time it is consumed; if `localCancel` creates a child scope after extraction, the extracted token is the parent's, not the child's. Extract `cancelToken` inside the computation, not before it is queued.

[RETRY_SEAM_LAW_AND_CONFLICT_EVIDENCE]:
- `retryWhile(schedule, computation, e => e.HasCode(ErrorCodes.TimedOut) || e.IsExceptional)` is the correct root-level retry predicate for transient wire failures: it retries on timeout and exceptional errors while propagating expected application-level errors — validation failures, authorization denials — immediately. The predicate is the only correct location to discriminate retriable from non-retriable errors; lower-layer operations must emit typed expected errors via `Error.New(message, innerException)` so the predicate can classify them by `.IsExpected` rather than by exception type inspection.
- `Schedule.exponential(initialDelay, factor) | Schedule.maxDelay(ceiling)` composed with `Schedule.jitter(factor)` produces a bounded exponential backoff with proportional jitter. The `|` operator on `Schedule` picks the minimum delay from two schedules at each step; `&` picks the maximum. For outbound retry at a wire seam, `Schedule.recurs(n) | Schedule.exponential(10*ms) | Schedule.maxDelay(5*seconds) + Schedule.jitter(0.25)` is a concrete five-attempt bounded jittered schedule expressed entirely as a `Schedule` value object with no imperative state.
- Exactly one retry owner exists per outbound hop, held at the composition root. Lower-layer operations that receive a `Fin<A>` from the wire layer emit `Fin.Fail(conflictEvidence)` where `conflictEvidence` is an `Error.New(message, underlyingException)` carrying the conflict reason. The root uses `retryWhile` to decide whether to retry; lower layers never call retry themselves. Lower layers use `liftIO(Func<Task<A>>).Catch(e => e.IsExceptional, e => IO.Fail<A>(translateToExpected(e)))`, producing typed expected errors, and the root predicate classifies those errors by code to determine retriability. This is the only structure that prevents retry loops from stacking at multiple layers.

[TEMPORAL_WIRE_VOCABULARY_DEPTH]:
- `Schedule.upto(duration, Func<DateTime>)` and `Schedule.fixedInterval(duration, Func<DateTime>)` both accept a `Func<DateTime>` clock delegate. At wire seams, this delegate must be `() => timeProvider.GetLocalNow().DateTime` (where `timeProvider` is the injected `TimeProvider`), never `DateTime.UtcNow`. This makes schedule-governed retry and repeat operations deterministic under test without requiring a separate abstraction layer.
- The `Func<DateTime>` accepted by wall-clock-aware schedule combinators uses `DateTime` internally, discarding UTC offset information. This has no observable effect on retry intervals, but audit receipts emitted inside a retry loop must capture `timeProvider.GetUtcNow()` — a `DateTimeOffset` with zero offset — as the timestamp, not the `DateTime` passed to the schedule, which loses offset provenance.
- `DateTimeOffset` wire fields with `[JsonIgnore(Condition = JsonIgnoreCondition.WhenWritingDefault)]` emit no key when the value is `DateTimeOffset.MinValue` (the default struct value). `DateTimeOffset.MinValue` must never reach a serialized field — the sentinel projection at the host boundary converts it to `None` before the domain model is constructed. The `WhenWritingDefault` guard is a second-line defense against uninitialized value-type fields escaping through code paths that bypass the boundary projection.

[TRACING_AND_CORRELATION_AT_WIRE_SEAMS]:
- `ActivitySource.HasListeners()` is a zero-allocation check that gates `Activity` span creation. At a wire seam producing typed receipt records, correlation fields — `TraceId`, `SpanId`, `TraceFlags` — must be captured only when listeners are active. The pattern: a `static readonly ActivitySource _source = new("boundaryName")` owned by the seam; receipt construction calls `_source.StartActivity(name, ActivityKind.Client)` guarded by `_source.HasListeners()`, captures `Activity.Current?.TraceId.ToHexString()` into the receipt's correlation slot, then disposes the activity. No `Activity` object appears on a public boundary surface.
- `DistributedContextPropagator.Current.Inject(activity, carrier, setter)` and `ExtractTraceIdAndState(carrier, getter, out traceId, out traceState)` are the vendor-neutral context propagation points for inbound and outbound wire calls. `CreateW3CPropagator()` produces the W3C `traceparent`/`tracestate` propagator. At an outbound seam, `Inject` populates header carrier objects before the wire call; at an inbound seam, `ExtractTraceIdAndState` reads the propagated context and `Activity.Current?.SetParentId(traceId, traceState)` links it. `DistributedContextPropagator.Current` is swappable at composition time via direct assignment, making the propagation strategy a composition policy.
- `Activity.AddException(Exception, ref TagList, DateTimeOffset)` records an exception event onto the active span using the OpenTelemetry semantic conventions. At a wire seam that catches an exception inside a `liftIO` lambda, call `Activity.Current?.AddException(ex)` before translating the exception into an `Error` rail value so the span carries the exception event even though the exception is not re-thrown. The `ref TagList` overload avoids allocating a `ReadOnlyCollection<Tag>` for the tag bag.

[SNAPSHOT_ENVELOPE_VERSIONING_AND_CODEC_STABILITY]:
- Snapshot envelopes carrying multiple domain payloads in a single wire transmission use `[JsonPolymorphic]` on the envelope's payload field rather than on the envelope itself. The envelope carries a `version` field (a `[SmartEnum<int>]` or `[SmartEnum<string>]` value object) and a `kind` discriminant field whose valid values are versioned per schema version. The polymorphism discriminator in the payload field is the runtime case; the version field selects which set of valid discriminator values is legal for that envelope. The admission modifier validates that the `kind` value is in the permitted set for the declared `version` before the domain factory sees it.
- `JsonTypeInfo.OriginatingResolver` identifies which resolver in the chain produced a given `JsonTypeInfo`. A modifier injecting redaction logic must gate on `OriginatingResolver` to avoid injecting onto types resolved by a foreign resolver in the chain that manages its own redaction: `if (typeInfo.OriginatingResolver == myContext) { inject modifier logic; }`.
- `JsonTypeInfoResolver.Combine(ReadOnlySpan<IJsonTypeInfoResolver>)` accepts a `ReadOnlySpan` argument, enabling stack-allocated resolver arrays for ephemeral contexts. At a seam that constructs a per-request `JsonSerializerOptions` with a request-scoped modifier carrying a per-request redaction predicate, combining via span avoids a heap allocation for the resolver array. The resulting options instance must call `MakeReadOnly()` before any serialization call; not calling it produces `InvalidOperationException` on the first mutation after use.

[BACKPRESSURE_AND_PIPELINE_CONTRACT]:
- `PipeOptions.UseSynchronizationContext = false` prevents `PipeReader.ReadAsync` continuations from marshaling back to a captured synchronization context. At a server-side or background-pipeline seam, this must always be `false`; defaulting to `true` on synchronization-context-bearing threads produces unnecessary thread switches between every read/process cycle.
- `PipeOptions.PauseWriterThreshold = 0` disables backpressure: `FlushAsync` never blocks regardless of how much data is buffered. The correct posture for boundary-facing seams is a non-zero threshold — 32 KB or a protocol-MTU multiple — so that the producer is back-pressured when the consumer is slow. Setting it to zero is safe only when the consumer is guaranteed to drain faster than the producer fills, which is never a safe assumption at a wire boundary with an adversarial remote.
- `PipeReader.ReadAtLeastAsync(minimumSize, CancellationToken)` blocks until at least `minimumSize` bytes are available or the pipe is completed. This is the correct primitive for framing protocols with fixed-length headers before variable-length bodies: read the header with `ReadAtLeastAsync(headerSize)`, parse the body-length field, then `ReadAtLeastAsync(bodyLength)`. Without `ReadAtLeastAsync`, a manual loop over `ReadAsync` with segment-position tracking is error-prone when the header straddles a segment boundary. The `examined` parameter in `AdvanceTo(consumed, examined)` must cover all bytes inspected but not fully consumed to avoid re-reading them.
- `Channel.CreateBounded<T>(options, itemDropped)` with `FullMode = BoundedChannelFullMode.DropOldest` and an `itemDropped` callback is the correct seam marker for wire event streams with bounded queue semantics. The callback is the single correct site for emitting a drop-receipt — a typed evidence record with the dropped item's `TraceId`, `Timestamp`, and `DropReason` — into a diagnostic side-channel. `ChannelOptions.AllowSynchronousContinuations = false` must be set on cross-seam channels to prevent the writer's call to `TryWrite` from synchronously executing a consumer continuation, which would re-enter the writer's synchronization context and invert the execution model.

[DOMAIN_VOCABULARY_WIRE_PROJECTION_MECHANICS]:
- `[SmartEnum<TKey>].DisableSpanBasedJsonConversion = false` (the default) causes the generated converter to use `Utf8JsonReader.CopyString(Span<char>)` to read the key, avoiding heap string allocation on the read path. The converter dispatches on `ReadOnlySpan<char>` directly via the generated key-comparison logic. This is the primary reason `[SmartEnum<string>]` should be preferred over raw `string` fields for bounded vocabulary values on wire types: the generated converter provides span-based zero-allocation admission for inbound messages while producing exact protocol string values on the write path.
- `[ObjectFactory(typeof(TFactory), UseForSerialization = SerializationFrameworks.SystemTextJson)]` on a factory method directs the Thinktecture source generator to use that factory as the `Read` implementation of the generated converter. This makes admission logic — range checks, sentinel rejection, context validation — part of the deserialization path rather than a separate post-deserialization step. The factory receives the key value already parsed from the wire; it must return a valid domain value or throw `ValidationException`, which the surrounding `JsonConverter.Read` catches and rethrows as `JsonException`. A `JsonException` at this site causes the enclosing `liftIO` + `.Catch(IsExceptional, ...)` to translate it into a typed expected error.
- `Thinktecture.ValueObjectAttribute.NullInFactoryMethodsYieldsNull = false` (the default) causes the generated converter to throw when the JSON value is `null` for a value object field. For `[ValueObject<T>]`-typed fields that should project `null` JSON to `Option<T>.None` at the wire boundary, a thin outer converter wraps the generated one: `Read` returns `Option.None<TVo>()` on null and delegates to the inner converter for non-null, isolating the null-projection policy at the boundary without modifying the generated type.

[PHYSICAL_UNIT_AND_STORAGE_NAME_BOUNDARIES]:
- Physical-unit measurement fields crossing a wire seam require two orthogonal concerns separated: the numeric value — unitless in the wire representation — and the unit discriminant, a `[SmartEnum<string>]` on the wire envelope. A `JsonNumberHandling.Strict` numeric field plus an adjacent unit enum field is the minimal protocol-correct shape. The domain receipt carries only the typed measurement after unit conversion; only the boundary converter reads both the raw numeric and the unit raw string together to perform the conversion and emit the typed domain value.
- Wire payloads that use string keys for physical units must never let those strings propagate into internal code as `string` fields. A `[SmartEnum<string>]` with `UseForSerialization` and span-based conversion is the correct admission type: the admission factory rejects unrecognized unit strings as expected errors, converting them to `Fin.Fail(Error.New(...))` before they reach the converter output. Domain code refers to the enum case; the protocol string appears only in the smart-enum key definition.
