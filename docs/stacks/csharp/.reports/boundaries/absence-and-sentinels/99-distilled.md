# Absence and Sentinels — Distillate

[SENTINEL_PROJECTION_LAW]:
- Every host sentinel — null, `IntPtr.Zero`, `nint(-1)`, `DateTime.MinValue`, `Guid.Empty`, a `0`-handle, a dedicated unset constant, a magic fallback — is foreign-language absence; project it to `Option<T>.None` at the exact foreign-call site: `Prelude.Optional(refOrNull)` for references, `Optional(nullable)` via `HasValue` for `Nullable<T>`, an explicit discriminant (`h.IsInvalid ? None : Some(h)`) for handles. Both `Optional` overloads are zero-allocation, and the implicit `operator Option<A>(A?)` invokes `Optional` and is byte-identical — so the explicit call costs nothing and marks the projection site.
- Value-type magic constants are never intercepted: the implicit conversion and the lenient admit fire only for reference types and `Nullable<T>`; a negative index, zero handle, or min-instant wraps as present — `Some(-1)`, a sentinel propagating inward wearing the absence type. These demand explicit discriminant expressions: `idx >= 0 ? Some(idx) : None`.
- `Nullable<T>` admission splits into two factories with opposite absence policy: the strict admit throws on no-value (absence is a contract violation); the lenient admit returns `None` (the sentinel projection proper). The reference-lenient form tests null, the value-lenient form tests the has-value flag; both land on one empty case, so a single consumer serves reference and value-type absence identically past the seam.
- `SafeHandle` is dual-sentinel: `IsClosed` is the always-accurate BCL closed bit while `IsInvalid` is abstract and may inspect only the handle value, so both test together — `handle.IsInvalid || handle.IsClosed ? None : Some(handle)` immediately after construction, with the raw handle never stored. `SetHandleAsInvalid()` sets the closed bit and suppresses the finalizer without releasing; after it, `DangerousAddRef` throws `ObjectDisposedException` with no safe retry. `DangerousAddRef(ref bool success)` never sets `success = false` on the non-throwing path — treating post-call `false` as a silent absence signal is wrong; projecting to `Option` before any ref-counted access removes the hazard entirely.
- A `ref struct` cannot inhabit `Option<T>`; conditional acquisition of a scope-shaped resource maps a boolean probe to a domain absence signal, and "currently held elsewhere" is a different absence from "does not exist" — contention handles explicitly, never folded into existence-absence.
- A sentinel never enters a persisted shape; `Option<T>` projects to a nullable wire field at the serialization seam and projects back on entry.

[NULL_REENTRY_THROUGH_PRESENCE]:
- The carrier's internal constructor sets the present bit unconditionally with no null check; `Map` and `Bind` route their present branch through it, so a null-returning projection manufactures `Some(null)` — presence wrapping a sentinel that re-entered wearing the absence type itself. The present-factory is equally unguarded: it is for values already proven non-null, and using it to admit a foreign field bypasses the projection the boundary exists to perform; the only null-rejecting admit is the boundary projection that tests against null and returns the empty case.
- `BiBind` routes both present and absent cases to new `Option<B>` values — the rail-preserving alternative to an intermediate `Match`-then-rewrap — but checks neither branch: `Some(null)` from the present branch yields a present carrier with null interior; a null return from the `None` thunk yields `default(Option<B>)` silently presenting as `None` with no diagnostic. `BiMap` is asymmetric — the `Func<Unit, B>` overload null-checks both branches, the `Func<B>` overload does not. Prefer `BiBind` to stay on the rail, and apply the defensive law to every present-branch and bi-directional projection: return the empty case explicitly when the target reference is absent; never let null flow into the present branch.
- The join cannot repair re-entry: `Bind(identity)` over `Some(Some(null))` yields `Some(null)` — the join collapses absent cases, never null values hiding inside present cases.

[NESTED_ABSENCE_JOIN]:
- `Option<Option<A>>` collapses by the monadic join — `flatten(ma)` is literally `ma.Bind(identity)`; outer-`None` and inner-`None` fold to one `None` and there is no third state: outer-present-inner-absent is indistinguishable from outer-absent after the join, so a boundary that must report which layer was absent discriminates before the flatten, never after.
- Every absence-returning step inside a bind chain is already a flatten: `Bind` short-circuits on absence and otherwise returns `f(Value)` as the result — never a nested optional needing a second collapse. An explicit `flatten` after a `Bind` signals the projection was written as `Map` returning `Option`; switching that `Map` to `Bind` fuses projection and join into one short-circuiting step.
- `SelectMany` is a two-level join in one method — receiver absence, bound-result absence, then projection — so a query comprehension over several optionals never materializes an intermediate nested optional and is the dense shape for an N-source admit where every source is independently optional: the first missing input collapses the whole `from … select` to `None`.
- `guard<Option>` does not compile — `Option` implements `Alternative` and `Choice` but not the `MonoidK` that `Prelude.guard` requires; absence filtering inside an `Option` comprehension is `Filter`/`Where` (both return `None` when the predicate fails on a present value), and typed-error absence guards over `Eff`/`Fin` use the error-bearing `guard(flag, Error.New(code, msg))` returning `Guard<Error, Unit>`.

[ABSENCE_TRANSFORMER_STACK]:
- The absence transformer is the standing flatten of effect-over-absence — a record over `K<M, Option<A>>` whose bind joins both layers on every step: bind the outer effect to reach the inner `Option`, route present into the continuation's inner run, route absent to the outer monad's pure-of-empty — one `K<M, Option<B>>`, never a stacked `M<Option<M<Option<B>>>>`. Hand-writing that bind is correct only at the single seam that constructs the transformer from a foreign effect-of-optional; everywhere inside, the transformer's bind owns the join.
- The two lift overloads carry opposite absence semantics: lifting a bare effect rewrites every produced value to present through the unguarded present-factory — it can never produce the empty case and resurrects the null-reentry hazard when the effect yields null; lifting an effect-of-optional preserves the inner cases unchanged. Choosing the wrong one silently fabricates or erases absence.
- The implicit-conversion family (bare carrier, pure value, typed failure, unit, effect, effect-of-value, effect-of-optional) absorbs sources without manual lifts — but the typed-failure and unit conversions both land on the empty case, so a foreign rail's failure flattens to absence and its reason is discarded at the conversion. A boundary that must preserve why the foreign step failed stays on the typed-error transformer.
- `OptionT` implements `Fallible<Unit, OptionT<M>>` — its `Catch` predicate receives `Unit`, not `Error`, so typed recovery (`HasCode`, `IsType<E>`) does not apply; recovery that discriminates absence causes uses `FinT`, which implements `Fallible<Error, FinT<M>>`. There is no `ToFin` on `OptionT<M, A>`: absence-to-failure runs `.Run()` to the inner `Option<A>`, `ToFin(Error.New(code, msg))` on it, then `FinT.lift`.
- A traversal over the stacked carrier threads both layers in one sweep, and the combinator choice fixes two policies simultaneously: the monadic traversal sequences each element's outer effect and short-circuits the absence layer at the first miss — later probes' effects never execute — while the applicative traversal runs every effect and combines the absence layer by its `Apply`. A hand-written loop tangles those two concerns into flags that drift independently.
- `Option<T>.ToIO()` promotes to `OptionT<IO, A>`; `OptionT.lift<M, A>` targets any other outer monad.

[ABSENCE_RAIL_REWRITE]:
- Moving absence onto another rail is a carrier rewrite — a total structural arrow dispatching on the case — never `Match`-then-rewrap and never `Map`: `Map` transforms values inside the carrier and cannot change the carrier, so it can never move absence onto the error rail. Each carrier publishes a fixed arrow set; the boundary picks the target rail by naming it, and the rewrite names the absence's error payload exactly once at the seam.
- `ToFin(Error)` and `ToEff(Error)` are the absence-to-failure arrows; the zero-arg overloads default to `Errors.None` — a `ManyErrors` with code `-2000000006` over an empty sequence, so `HasCode` always returns false and recovery sites cannot discriminate. Always supply a coded domain `Error` when the absence has a named cause.
- The transformer rewrite rides the inner run: the absence transformer rewrites to the typed-disjunction transformer by mapping its inner `Option`, supplying the left payload (value, thunk, or monoidal empty) for the empty case — one arrow threading through the outer effect untouched.
- `Fin`'s `BiMap` is in-rail only: the `Fail` branch re-codes the error but stays `Fail` and cannot escape to `B`; collapsing `Fin<A>` into a plain `B` with distinct success and failure materializations is `Match(Succ, Fail)`.
- Key-bounded lookups return `Option` via `Find`, never try-get-then-branch — a missing key is structural absence, and a state-mutating find-or-add must not masquerade as a default. The fallback `IfNone(value)` or `IfNone(thunk)` sits at the single point where a default is semantically correct; deeper placement creates invisible defaults that mask missing-key bugs. When no default is valid, `Find(key).ToFin(codedError)` converts the miss to a rail failure; `IfNone(() => throw …)` defeats the rail entirely.
- The boundary composition: `traverse` gathers N probes into one carrier of a structure, then one structural arrow moves that carrier onto the interior's rail — `Option<Seq<A>>` becomes `Fin<Seq<A>>` by the arrow, and the interior never sees the optional.

[PROBE_SWEEP_TRAVERSAL]:
- A boundary probing N foreign sources holds `K<T, K<F, A>>` — a structure `T` of absence carriers `F` — and hoists the carrier outward to `K<F, K<T, A>>` in one call (`Sequence`/`traverse`); both `T` and `F` are abstract, so one written expression serves every probe-collection shape and every absence carrier with no per-shape overload.
- The two collapse families are policies, not synonyms: `traverse`/`sequence` constrain `F` to `Applicative` and run every effect — a complete census of absence; `traverseM`/`sequenceM` constrain to `Monad` and sequence with `Bind`, short-circuiting the remainder at the first failing element — fail-fast. The constraint is the policy.
- The element transform feeds the carrier inside the fold — `traverse(probe, sources)` with `probe : A -> K<F, B>` — never a pre-pass `Map(probe)` then `sequence`; the pre-pass is byte-equivalent only for pure projections, and when the probe is effectful the fused `traverse` is the sole correct shape.
- A traversal never guarantees the element function runs: the absent branch returns `F.Pure` over the empty structure without invoking the transform — `Option.Traverse`/`TraverseM` on `None` return `F.Pure(None)` without calling `f`, and the transformer's conditional-effect form runs its action inside `M` only on present. A boundary obligation that must fire for every probe, including the absent ones, cannot live inside `traverse`: it sequences as an independent effect or splits the populations and emits over both; exactly-one-effect-path dispatch is `Match(Some: x => effect(x), None: () => alternateEffect())`.
- Per-element absence branches written before the sweep have already chosen short-circuit-or-accumulate implicitly and invisibly; projecting each probe to the carrier at its own seam and sweeping once makes that policy a named, typed decision at one site.

[ACCUMULATION_POLICY_BY_CARRIER]:
- Accumulate-versus-short-circuit is owned by the carrier's `Apply`, never by the sweep: `Option.Apply` and the monadic carriers discard surviving evidence at the first absence; `Validation<FAIL>.Apply` routes fail-with-fail through `FAIL`'s semigroup `Combine`, so an applicative sweep returns every failure reason in one value. Selecting the carrier selects whether the boundary reports one absence or all of them.
- The middle case: the tupled and chained applicative form is a left fold — once the first failing argument turns the accumulator into a failing function, every subsequent failing argument's error folds in (`Error +` builds a flattened `ManyErrors`) while subsequent successes contribute nothing. Treat it as accumulate-from-first-failure; when every reason must survive regardless of position, reach for the dedicated accumulating carrier.
- An N-source boundary combine is one tupled applicative call, the carrier inferred and the arity absorbed by the lifted function:

```csharp
static K<F, Session> OpenSession<F>(
    K<F, Endpoint> host, K<F, Credential> cred, K<F, Capability> caps)
    where F : Applicative<F> =>
    Applicative.apply((host, cred, caps), static (h, c, k) => new Session(h, c, k));
```

- One body, two boundary policies chosen at the call site by the carrier supplied: instantiated at `F = Validation<Error>` it returns every missing-input reason in a single `ManyErrors`; at `F = Option` it returns `None` on the first absence and costs nothing on the present path — no second method, no flag.

[POPULATION_PARTITION]:
- When the boundary must keep both survivors and casualties, the fold is a partition, not a traverse: `Partition` over `K<F, Fin<A>>` for any `Foldable<F>` yields `(Seq<Error> Fails, Seq<A> Succs)` in a single non-short-circuiting pass, so a capability census reports usable results and failure reasons simultaneously without two iterations.
- `partitionFallible` sequences each inner `K<M, A>` within the foldable, catches per-element failures through the `Fallible` trait, and accumulates `(Fails, Succs)` inside the outer rail without failing it — the dense surface for a batch where partial success is the expected steady state.
- For absence-typed probe results, the silent-drop collector (`Somes`) discards `None` with no record; when an absent probe warrants a metric, `Choose` with a labelled `None` branch replaces it — and `Seq.Choose` replaces any `Where`+`Select` pair that re-applies the predicate. Generalized: a partition that throws away the casualty population is the wrong fold whenever the boundary's health surface derives from why probes were absent.

[FALLBACK_CHOICE_FOLD]:
- The choice fold and the join fold are different folds over the same sources: choice (first-present) keeps one survivor and discards the rest; join (bind chain or applicative combine) requires all present and collapses at the first absent. "Any one of these foreign sources" is choice; "all of these together" is the chain — selecting the wrong fold inverts the absence semantics of the whole seam.
- `Option`'s `|` and `Combine` implement the first-`Some` semigroup: `primary | secondary | tertiary` evaluates left-to-right and short-circuits selection on the first present value — the correct surface for chaining fallback absence sources, replacing nested `IfNone` calls.
- Selection short-circuits; evaluation does not: the empty-or operator chooses between already-materialized carriers, so every losing alternative's construction cost and side effects are paid despite never being selected. An effectful or expensive fallback defers — a thunked empty-case factory, or the `Choice` trait over deferred carriers — or the fallback ladder becomes an unconditional fan-out.
- Error visibility on the error-rail chain: `Fin`'s `|` calls `Choose` and silently discards the lhs error; `+` accumulates — both-`Fail` yields `Fin.Fail(err1 + err2)` as a `ManyErrors`. Use `+` when a logging or health seam must inspect why each attempt failed; `|` only when the lhs error is genuinely irrelevant. `Error.Combine` flattens adjacent `ManyErrors` at combination time, so `HasCode`'s one-level scan detects a terminal code inside a `+`-combined failure — but a `ManyErrors` nested by direct construction is not traversed: rely on `+`-flattening, never on recursive detectability. `ManyErrors.Filter<E>()` extracts only the `IsType<E>` errors — the surface for splitting terminal from transient reasons inside a combined failure.

[INDETERMINATE_NONE_VERSUS_DEFINITE_STATE]:
- `Option.None` is structurally indistinguishable from "not yet known" to a downstream caller; whenever absence carries a cause, phase, or evidence, a closed family owns it — collapsing to `None` erases evidence at the consumer and forces re-derivation. The law instantiates identically across surfaces:
- Capability: unavailable, degraded, disabled, transitioning are cases of one `[Union]` family — at least `Available`, `Degraded`, `Unavailable`, each with its own payload — under total `Switch` with no default fallthrough. An `Unavailable` capability emits a definite `Fin.Fail` with a coded `Expected` error at every operation site, never an indeterminate `None`; a degraded outcome carries a usable result beside first-class typed degradation evidence the caller can inspect, route, and log without downcasting — never a nullable companion field, a swallowed exception, or a null result with a string message.
- Initial validation: the binding model carries an explicit `Unvalidated` case distinct from `Valid(T)` and `Invalid(reason)`, initial state always `Unvalidated`; binding `None` as initial and treating absence as "not yet invalid" renders falsely clean wherever a consumer folds `None` to a success path. The accumulating validation carrier requires a monoidal failure and cannot express `Unvalidated` — the union wraps it as a separate case rather than embedding the state where it cannot exist.
- Progress: declared-unknown progress is a discriminated family — `Unknown`, `Indeterminate`, `Known(fraction)` — never a nullable integer or bare float; the monotone non-decreasing law is enforced at the transition site, and a transient foreign error never regresses `Known` to `Unknown`: the seam clamps to `Indeterminate` or holds the prior `Known`, never passing the foreign value through.
- Probe streams: a false-returning non-blocking read conflates "no item yet" with "closed and empty" — transient versus terminal absence discriminates through the completion surface, never from the read result alone.
- Vocabulary-complete health: the absence of an unprobed key is itself evidence — the fold projects every key in the bounded vocabulary, mapping registry absence to an explicit `Unknown` case; folding only over present entries silently omits unprobed capabilities from the health picture.
- A definite `Disabled`/`Unavailable` emission at the single seam receiving the foreign state terminates propagation there; binding `None` into a downstream pipeline forces every consumer to individually defend against the indeterminate optional.

[CAPABILITY_PROVISION_AXIS]:
- A capability has two orthogonal absence axes that never collapse into one carrier: provision — whether the effect can reach the capability at all, a property of the monad-over-runtime pairing proved at compile time — and state — whether a reachable capability is currently usable, a value observed at runtime. Routing "not wired" and "degraded" through one `Option` or union erases the distinction the type system enforces for free.
- Provision is carried by `Has<M, VALUE>`: a trait on the effect monad with a single `static abstract K<M, VALUE> Ask`. A step constrains `where M : Has<M, Cap>` and calls `M.Ask` through static-virtual dispatch; a monad whose runtime lacks the capability never satisfies the constraint, so the gap is a build error at the composition root — absence that needs no sentinel because the absent value cannot be constructed.
- `Has<in M, VALUE>` is contravariant in `M`: a monad over a richer runtime substitutes wherever a less-capable one is demanded, so one concrete effect monad flows into every step whose capability subset it covers with no per-step adapter. Stacking several `Has` constraints on one step is the declarative capability manifest, proven all at once by the resolver; the accessor returns `K<M, VALUE>` — already lifted into the effect, composing with `Bind`, never escaping the rail as a bare handle.
- Provision-time absence is unrepresentable, so "might be wired" is a category error; the optional sits on the state axis inside the provided value: a genuinely dynamic host feature is provided as `Has<M, Option<Feature>>` or `Has<M, FeatureState>` over a closed family — the constraint stays satisfiable for every runtime, and the absence rides the bound value where total dispatch handles it. Making `Has` conditional pushes the absence back to the axis where it cannot be inspected.

[ENVIRONMENT_READ_SEAM]:
- The reach into the environment is a first-class value: `Ask<Env, A>(Func<Env, A>)` captures a pure projection that stays unapplied until a monad binds it, and one captured accessor binds polymorphically into `Reader<Env>`, `ReaderT<Env, M>`, `RWST<Env, W, S, M>`, and any `Readable<M, Env>` via `ToReader`, `ToReaderT<M>`, `ToRWST<W, S, M>`, `ToReadable<M>` — the same projection serves every stack carrying the environment, selected at the bind site, with no overload per stack.
- The environment read is the single authorized seam for foreign-sentinel projection because it is the only place the runtime's raw services are touched: every `Ask`/`asks` projection that reaches a nullable, an invalid handle, or a magic constant performs the `Option` projection inside its own body — `Ask(rt => Optional(rt.Service.MaybeHandle))` — so the bound value arrives domain-shaped, the projection site and the read site fuse into one value, and a projection appearing anywhere downstream means the raw runtime field leaked past the reader.
- Because the accessor is data, multi-provider fallback is the accessor semigroup, not a branch: capability projections chained by first-success collapse to one deferred value that resolves the environment once and yields the first reachable capability, the choice made by the carrier's `Choice` — never a hand-written ladder over the runtime.
- The canonical capability-bearing effect is a reader of the runtime stacked on the effect rail (`record Eff<RT, A>(ReaderT<RT, IO, A> effect)`); the runtime is the threaded read-only environment, never an injected service object — every access is a `Has` accessor bound into the rail or `runtime<RT>()` (the identity projection making the whole runtime the bound value) followed by a projection. The runtime-free and runtime-bearing effects are one family with a widening arrow applied implicitly by the disjunction and sequencing operators — a capability-agnostic step is the most general position and flows into every richer runtime for free.
- Capability scoping is environment narrowing: `local(Func<OuterRT, InnerRT>, innerEffect)` runs a minimal-runtime effect by projecting the outer runtime down to exactly the sub-capability the inner step needs, discarding the view at scope exit; the same-runtime `local(swap, ma)` runs under a transformed environment, is total over the transform, and carries no restore obligation — a sub-computation under a swapped capability is one `local`, never mutable set-then-reset. Narrow when delegating, widen when composing; both are environment maps over the one reader, and minimal-runtime authoring plus the two arrows replaces every hand-threaded context object.
- The capability read, its sentinel projection, and its absence-to-failure conversion fuse into one bound step — the runtime touched exactly once, no consumer able to reach a raw handle:

```csharp
static Eff<RT, Handle> handle<RT>() =>
    from cap in Readable.asks<Eff<RT>, RT, Option<Handle>>(rt => Optional(rt.RawHandle))
    from h   in cap.ToEff(Error.New(code, "capability unusable"))
    select h;
```

- When the capability is a closed state family instead of `Option`, the same body's `asks` yields the state value and total dispatch routes `Available` to the live handle, `Degraded` to a usable-but-flagged value carrying its evidence, and `Unavailable` to a terminal coded failure — all on the effect rail, the read and its classification a single inlined accessor.
