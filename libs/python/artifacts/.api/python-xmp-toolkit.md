# [PY_ARTIFACTS_API_PYTHON_XMP_TOOLKIT]

`python-xmp-toolkit` (import `libxmp`) is the XMP-packet read/author engine for the artifacts descriptive-metadata rail — the Adobe Extensible Metadata Platform RDF/XML block (Dublin Core, XMP-Basic, XMP-Rights, XMP Media Management, the Photoshop/IPTC-Core/IPTC-Ext schemas, PDF/A and PDF/X identification) that travels embedded in a JPEG/TIFF/PNG/PSD APP1 marker, an `Exif.Image.XMLPacket` IFD, or a PDF `/Metadata` stream — distinct from the IPTC-IIM `8BIM` block (`iptcinfo3`) and the binary EXIF IFD (`pyexiftool`/`pyvips`). It is a pure-Python `ctypes` binding (`libxmp` package, no compiled extension, no cp-gate) over the native **Exempi** C library (`libexempi`, the cross-platform reimplementation of Adobe's XMP Toolkit SDK), so every member is a thin `ctypes` thunk that resolves only after `ctypes.util.find_library('exempi')` (or the Homebrew/MacPorts dylib fallback) loads `libexempi` at first use. `XMPMeta` owns the in-memory property tree (parse RDF → typed get/set/append → serialize RDF), `XMPFiles` owns the smart-handler embedded-packet read/write inside a real file, `XMPIterator` walks the tree, and the `libxmp.utils` folds project a packet to a plain `dict`. The owning design page composes exactly `XMPMeta(xmp_str=...)` construction, the namespace-qualified `get_property`/`set_property` scalar surface keyed by the `libxmp.consts.XMP_NS_*` URI vocabulary, and `serialize_to_str(omit_packet_wrapper=True)` re-emit into the `exchange/metadata#METADATA` RASTER carrier's `_xmp`/`_xmp_packet` arms; it never re-implements the RDF/XML serializer, the smart-handler packet locator, or the schema-namespace registry the binding already owns, traps the binding's `XMPError`/`ExempiLoadError` as `(OSError, ValueError, RuntimeError)` with an `lxml` packet-parse fallback (the native library may be absent), and never lets the raw `XMPMeta`/`XMPFiles` handle cross the owner boundary — only the projected `dict[str, str]` logical fold does.

## [02]-[PUBLIC_TYPES]

[PUBLIC_TYPE_SCOPE]: the packet objects and error rail
- rail: metadata

`XMPMeta`, `XMPFiles`, and `XMPIterator` are the three public objects (`libxmp.__all__` re-exports `XMPMeta`/`XMPFiles` plus the two error classes and the `files`/`core` submodules). `XMPMeta` is the parse/get/set/serialize core with no knowledge of files; `XMPFiles` is the Exempi File Handler component (open a real file → `get_xmp`/`put_xmp` → close); `XMPIterator` is the property-tree walk Python's iterator protocol drives. The error hierarchy is two bare `Exception` subclasses — `XMPError` (every Exempi routine failure, mapped from the libexempi `xmp_get_error()` code through `ERROR_MESSAGE`) and `ExempiLoadError` (the native library could not be located/loaded). The owner traps both as `(OSError, ValueError, RuntimeError)` at the `_xmp`/`_xmp_packet` seam rather than a typed package rail, because XMP recovery is best-effort enrichment beside the `pyvips`/`pikepdf` decode and the native library may be unprovisioned.

## [03]-[ENTRYPOINTS]

[ENTRYPOINT_SCOPE]: `XMPMeta` — parse, read, write, serialize the packet
- rail: metadata

`XMPMeta(xmp_str=...)` parses on construction (an empty packet is `XMPMeta()`, parse-from-existing is the `xmp_str` kwarg, an internal-ref clone uses `_xmp_internal_ref`); `parse_from_str` re-parses into an existing object with an optional `<x:xmpmeta>` wrap. The read surface is one polymorphic accessor family keyed by `(schema_ns, prop_name)` where `schema_ns` is a URI constant and `prop_name` is a property path (the namespace prefix may instead lead the path): `get_property` returns the value as `str`, the typed siblings (`get_property_bool`/`_int`/`_long`/`_float`/`_datetime`) coerce, `get_array_item`/`count_array_items`/`does_array_item_exist` address ordered/bag/alt arrays by 1-based index, and `get_localized_text` reads one `xml:lang`-tagged alt-text entry by RFC-3066 generic+specific language. The write surface mirrors it (`set_property` + the typed `set_property_*`, `set_array_item`/`append_array_item` with an `array_options` keyword bundle, `set_localized_text`), every setter taking the `XMP_PROP_OPTIONS` `**kwargs`. `serialize_to_str`/`serialize_to_unicode`/`serialize_and_format` re-emit RDF (UTF-8) under the `XMP_SERIAL_OPTIONS` `**kwargs` — `omit_packet_wrapper=True` is the byte-blob form the metadata owner embeds. The three `@staticmethod` namespace functions (`register_namespace`/`get_prefix_for_namespace`/`get_namespace_for_prefix`) manage the global schema registry; `clone` deep-copies a packet. The owner composes `XMPMeta(xmp_str=blob.decode("utf-8","replace"))`, a `get_property(URI, local)` per `_FIELD_KEYS` qname, `set_property(URI, local, value)`, and `serialize_to_str(omit_packet_wrapper=True)` — never the typed accessor zoo per field (the one string accessor plus `msgspec.convert` coercion owns typing) and never the file surface (the raster carrier is `pyvips`-decoded bytes, not a path).

[ENTRYPOINT_SCOPE]: `XMPFiles` — embedded-packet I/O inside a real file
- rail: metadata

`XMPFiles` is the Exempi File Handler component: the smart-handler that knows where the XMP packet lives in a JPEG/TIFF/PNG/PSD/PDF/MP4/MOV/AVI/WAV/MP3 (and the packet-scanner fallback for unknown formats), so the owner never locates the packet by hand. The model is `open_file(path, **open_flags)` → `get_xmp()` (returns a fresh `XMPMeta` or `None`) → mutate → `can_put_xmp(meta)` guard → `put_xmp(meta)` → `close_file(close_flags)` (the write does NOT reach disk until close, optionally `XMP_CLOSE_SAFEUPDATE` for crash-safe temp-swap). `open_file`'s `**kwargs` are `XMP_OPEN_OPTIONS` — `open_read` for read-only, `open_forupdate` for read-write, `open_usesmarthandler`/`open_usepacketscanning` to force handler choice, `open_onlyxmp` for the space/time optimization. The metadata owner does NOT compose `XMPFiles` for the RASTER carrier (it operates on `pyvips`-decoded in-memory bytes, not a path, and re-embeds the serialized packet through `pyvips.Image.set("xmp-data", …)`); `XMPFiles` is the available native-handler path a future direct-file metadata lane (or a format `pyvips` cannot round-trip) would compose instead of the byte-blob `XMPMeta` path.

[ENTRYPOINT_SCOPE]: `XMPIterator` + the `libxmp.utils` packet→dict folds
- rail: metadata

`XMPIterator(xmp_obj, schema_ns=None, prop_name=None, **kwargs)` walks the property tree under Python's iterator protocol — each `__next__` yields `(schema, name, value, options_dict)` where the 16-key `options_dict` decodes the libexempi property-option bits (`VALUE_IS_ARRAY`/`ARRAY_IS_ORDERED`/`HAS_LANG`/`IS_SCHEMA`/`IS_INTERNAL`…) to booleans, and `skip(**kwargs)` prunes the remaining walk by `XMP_SKIP_OPTIONS` (`iter_skipsubtree`/`iter_skipsiblings`). The `**kwargs` are `XMP_ITERATOR_OPTIONS` (`iter_justleafnodes` to flatten, `iter_justchildren` for the immediate level, `iter_omitqualifiers`). `libxmp.utils.object_to_dict(xmp)` / `file_to_dict(path)` build on the iterator to fold a whole packet into `dict[str_namespace, list[(name, value, options)]]` — the bulk-projection form a "dump every property in this packet" read uses instead of an explicit `_FIELD_KEYS` qname loop. The metadata owner composes the targeted `get_property` loop (it reads a fixed logical set), so the iterator/`utils` folds are the available full-walk surface a provenance audit or an unknown-namespace recovery would compose. `terminate()` releases the global Exempi state — normally never called (a subsequent libxmp call crashes), present only for a deliberate teardown.

## [04]-[IMPLEMENTATION_LAW]

[INTEGRATION_STACK]:
- `exchange/metadata#METADATA` ownership: `python-xmp-toolkit`/`libxmp` is RETAINED per the roster lock but SUPERSEDED on the RASTER carrier — the rebuilt `Metadata` owner folds EXIF/IPTC/XMP/ICC in ONE cross-format pass over `pyexiftool` (standing) / `pyexiv2` (optional in-process), with `pikepdf` owning the PDF/XMP path, so the former four-provider split (`libxmp` XMP + `iptcinfo3` IIM + `pyvips` byte carrier + `exif`) is no longer composed and `libxmp` (additionally `unsupported` on cp315 without native Exempi) has no live `metadata.py` consumer; flagged for the final `pyproject` reconciliation (keep-vs-remove). The read/write folds below document the SUPERSEDED XMP integration as API reference, not a live path.
- read fold: `_xmp(blob)` constructs `XMPMeta(xmp_str=blob.decode("utf-8", "replace"))`, then for each `_FIELD_KEYS` row whose `.xmp` qname is set splits the `prefix:local` qname, resolves the prefix to its URI through `_XMP_NS[prefix]` (the local mirror of the `libxmp.consts.XMP_NS_*` constants), and reads `meta.get_property(uri, local)` into the one logical `dict[str, str]` — the same `dict` the EXIF and IPTC projections fold into the single `MetaFacts.from_logical` materialization. The whole block is wrapped `try/except (OSError, ValueError, RuntimeError)` returning the `lxml`-based `_xmp_xml(blob)` parse (one `etree.fromstring` over a `resolve_entities=False, no_network=True` parser, one namespaced `xpath` per qname) — the `libxmp` path is the primary, the `lxml` path the fallback when the native `libexempi` is absent (`ExempiLoadError`) or the packet trips the binding. The owner uses the single `get_property` `str` accessor (NOT the typed `get_property_int`/`_datetime` zoo), letting `msgspec.convert(strict=False)` in `from_logical` coerce the string values to each facet field's type — one accessor, one coercion site.
- write fold: `_xmp_packet(facts)` constructs an empty `XMPMeta()`, sets each writable `_FIELD_KEYS.xmp` qname from the `_flat(facts)` projection through `meta.set_property(_XMP_NS[prefix], local, str(value))`, and returns `meta.serialize_to_str(omit_packet_wrapper=True).encode("utf-8")` — the bare RDF blob (no `<?xpacket?>` PI) the raster writer embeds via `pyvips.Image.set("xmp-data", packet)` before `write_to_buffer` re-encodes. The same `(OSError, ValueError, RuntimeError)` trap falls through to an `lxml`-authored `rdf:RDF`/`rdf:Description` tree (including the `dc:subject` keyword `rdf:Bag`), so the writer degrades to a stdlib-authored packet on an unprovisioned host. The keyword array is the one place the typed array surface matters conceptually — `_flat` drops the `keywords` tuple, and the `libxmp` author path currently sets only scalar qnames (the `lxml` fallback hand-builds the `dc:subject` Bag); a future deepening authors the bag through `append_array_item(XMP_NS_DC, "subject", kw, array_options={"prop_array_is_ordered": False})` so the native author path carries keywords without the fallback.
- universal-tier stacking (`libs/python/.api`): the XMP fold rides the shared rails the owner already composes — `anyio.to_process.run_sync(limiter=WORKER_BAND)` carries the synchronous `XMPMeta` parse/serialize off the event loop onto the bounded worker band (the page's `partial(to_process.run_sync, limiter=WORKER_BAND)` `CarrierPolicy.lane`), which is doubly load-bearing here because the `ctypes`→`libexempi` call is a blocking native call AND because Exempi's global init is not free-threading-safe, so the single-process-per-worker band serializes it; `msgspec` (`convert(strict=False)` per facet, `structs.asdict` in `_flat`) materializes the XMP scalars into the `Descriptive`/`Rights`/`Capture`/`Place`/`History` facets; `expression.tagged_union` owns the two-case `Metadata` read/write discriminant the XMP arm folds under; and `rasm.runtime.faults.async_boundary` wraps the whole `metadata.raster.{read,write}` op so the XMP parse inherits the `structlog`+OpenTelemetry span and `RuntimeRail` typed-error envelope without a `libxmp`-specific `try/except` rail beyond the `lxml`-fallback trap. The `XMPError`→`(OSError, ValueError, RuntimeError)` mapping is the boundary translation that lets the `expression.Result` rail stay package-agnostic.
- namespace-constant seam: the owner's `_XMP_NS` `frozendict` is the local mirror of `libxmp.consts.XMP_NS_*`; a packet whose schema is not in `_XMP_NS` is simply not read (the owner reads a fixed logical set), and a new schema a field needs is one `_XMP_NS` entry keyed to the published constant plus one `_FIELD_KEYS` qname — never an invented URI. The PDF/A identification namespaces (`XMP_NS_PDFA_ID`/`_Schema`/`_Property`/`_Type`/`_Field`) are the constants `composition/sheet` needs for its PDF/A claim packet (see residual).
- vocabulary boundary: `libxmp` owns strictly the XMP RDF/XML packet — the editorial/rights/capture/location/identity scalar properties across the Adobe schema set. It does NOT own the IPTC-IIM `8BIM` block (`iptcinfo3`), the binary EXIF IFD (`pyexiftool`/`pyvips`), the C2PA signed-provenance manifest (`exchange/credential` `c2pa-python`), or the ICC profile header (`pillow` `ImageCms`); the `Iptc4xmpExt:DigitalSourceType` AI-provenance field IS an XMP property (it rides `_xmp` through the IPTC-Ext namespace), distinct from the signed credential assertion.

## [05]-[NATIVE_PROVISION]

- The binding is inert without a machine-provided **Exempi** shared library. `_load_exempi()` resolves it through `ctypes.util.find_library('exempi')`, then (macOS only) probes `/opt/homebrew/lib/libexempi.dylib` and `/opt/local/lib/libexempi.dylib`; failure raises `ExempiLoadError` on the FIRST member call (construction of a non-empty `XMPMeta`, any `get_property`/`set_property`, any `XMPFiles` op), never at `import libxmp`. The current development host has no `libexempi` installed (neither Homebrew nor MacPorts path present), so the Python surface above is reflected from source (complete and authoritative — every signature is verified against the installed `libxmp` 2.1.0 distribution) but no member EXECUTES until Exempi lands. This is precisely why the `exchange/metadata#METADATA` `_xmp`/`_xmp_packet` arms wrap every call in `(OSError, ValueError, RuntimeError)` with the `lxml` `_xmp_xml` fallback: the design is correct whether or not the host is provisioned.
- Provisioning is a Forge concern (`brew install exempi` on macOS, `apt-get install libexempi-dev` / `libexempi8` on Debian, the equivalent on the CI image), aligned with the `[03]` system-tool boundary convention (discovery-env → configured-path → bundled-fallback); it is never a design-page blocker. Once Exempi is present the native `libxmp` path becomes the primary and the `lxml` fallback the degraded path, with no design change. Exempi is itself permissively licensed (BSD-style), so it carries no copyleft obligation into the artifacts distribution.
