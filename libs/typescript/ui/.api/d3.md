# [TS_UI_API_D3]

[PACKAGE_SURFACE]:
- package: `d3` (ISC)
- module: metapackage re-exporting the `d3-*` modules flat (`export * from "d3-<module>"`); ESM/UMD dual, tree-shaken per named import.
- asset: per-module `@types/d3-*` declarations; every member is a flat named export off `d3`.
- runtime: pure computation — number/array/string/date in, numbers/path-strings/GeoJSON out; zero DOM in the admitted scope.
- plane: `plane:runtime` (W4 `ui`, substrate under the chart owners).
- rail: `ui` data-viz substrate — the math `@visx/*` and `@observablehq/plot` both stand on.

`d3` admits as headless-math SUBSTRATE, never a chart-authoring model: `@visx/*` wraps `d3-scale`/`d3-shape` into React components, `@observablehq/plot` embeds the roster behind its grammar, and a bespoke chart computes d3 folds directly under a React-rendered or Plot-generated surface. This catalog scopes d3 to its computation modules.

## [02]-[INTEGRATION]

[STACK: under `@visx/*` (`.api/visx-scale.md`, `.api/visx-shape.md`, `.api/visx-axis.md`)] — `@visx/scale` wraps `d3-scale` factories into config objects, `@visx/shape` renders `d3-shape` generators as components, and `@visx/axis` renders ticks from a d3 scale as React SVG, replacing `d3-axis` outright. Component bodies bind the visx spelling; raw d3 stays in the data FOLD (`rollup`/`bin`/`extent`/`ticks`) — chart math in the fold, rendering in visx.

[STACK: under `@observablehq/plot` (`.api/observablehq-plot.md`)] — Plot embeds d3 behind its grammar; consumers pass channel and scale OPTIONS, never d3 objects. Direct d3 rides only around Plot — `d3-array` folds rows before marks, and `format`/`timeFormat` specifier strings Plot accepts verbatim.

[STACK: `d3-geo` beside the map stack (`.api/maplibre-gl.md`, `.api/deck.gl-core.md`, `.api/turf-turf.md`)] — maplibre and deck.gl own the live basemap and camera, turf owns analytic geodesy; `d3-geo` serves NON-map geo — projected thumbnails through `geoPath` on offscreen canvas, graticule and circle generation, spherical math where a chart plots geography.

[BOUNDARY: `d3-scale-chromatic` vs `system/token`] — `system/token` (OKLCH, APCA-gated) owns every UI-surface color: series strokes, categorical palettes, axis and grid inks. `d3-scale-chromatic` admits ONLY as the data-value colormap family (`scaleSequential(interpolateViridis)`) where the color IS the datum; a `scheme*` categorical array standing in for the token palette is the split-brain defect.

[BOUNDARY: DOM and interaction owners] — React reconciles all chart DOM, `system/act` owns gesture (`@use-gesture`) and transition, deck.gl and maplibre own map interaction, and the rAF clock and Effect pubsub own timing and dispatch; the DOM-coupled and redundant d3 modules reject against these owners.

## [03]-[RAIL_LAW]

- Owns: headless chart math — scale construction, interpolation, array folds/binning/statistics, shape/path generation, number/time format vocabularies, color-space math, data-value colormaps, geo projection math, hierarchy/force/delaunay/quadtree kernels, seeded random sources.
- Accept: d3 members in data folds and geometry preparation; visx/Plot as the rendering surfaces over this math; `piecewise`/`quantize`/`interpolateZoom` for value paths; `randomLcg`-seeded sources where reproducibility is asserted; `pathRound` for stable snapshot output; `format`/`timeFormat` specifier strings as data rows on axis/legend policy.
- Reject: any DOM-coupled module (`selection`/`transition`/`zoom`/`drag`/`brush`/`axis`); d3 as a charting surface beside visx/Plot; `scheme*` categorical arrays where the token palette rules; hand-rolled loops re-deriving `rollup`/`bin`/`extent`; `d3-fetch` at any boundary; a second easing/timer/dispatch vocabulary shadowing `system/act` and the Effect rails.
