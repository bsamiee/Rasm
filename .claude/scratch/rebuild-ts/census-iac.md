## [3]-[CAPABILITY_MAP]

Genuinely owned today: the full `selfhosted-k8s` arm (bootstrap → secrets → object/fanout/postgres → observe/boards → workload → traffic) is realized end-to-end across the nine pages with concrete, compilable-shaped fences; the `Dispatch` equivalence map, `RunReceipt`/`DeployFault` vocabulary, `Guard`/`Drift` policy-and-drift projection, and the env/life/output seams are all fully authored on one owning page each with no duplication.

Claimed but not yet realized, named as mismatches:
- README's `[DOMAIN_PACKAGES]` table lists `@pulumi/eks`, `@pulumi/github`, `@pulumi/esc-sdk`, `@pulumiverse/acme`, `@pulumi/cloudinit`, `@pulumi/synced-folder` as folder dependencies on par with the consumed packages; all six have full `.api` catalogs but zero constructing fence anywhere in `.planning` — they are catalogued capability for arms/features the design pages describe only as future growth rows (ACME issuance, EKS promotion, static-distribution leg, pre-SSH cloud-init, GitHub Actions secret sync via the pulumi/github provider itself rather than Doppler's integration).
- `ARCHITECTURE.md`'s codemap presents every listed `.ts` file as "realized code" by its own stated convention, but no `src/` tree exists — the map is aspirational relative to disk state, consistent with the doctrine's own framing (`.planning` is authoring substrate) but worth flagging since `project.json`'s `sourceRoot` points at a directory that does not exist yet.
- `provider.md`'s `gcp` and `cloudflare` arm bodies are named "provider-seam-complete" in prose ([5]-[ARM_PROGRAMS] law) but their fences construct only one resource each (`gcp.container.Cluster`; `cloudflare.DnsRecord`) and return partial or empty output records — narrower than the full `_map` column each arm's row set promises (Cloud SQL, GCS, Secret Manager mirror, Cloud DNS for gcp; R2, Workers, Access, Total TLS for cloudflare are all `_map` cells with no realizer).
- `observe.md`'s `Boards` tier declares `_alertRows`/`_sloRows` as `declare const` signatures, not bodies — the page's own prose names this the "standing design item," so alerting/SLO realization is honestly deferred, not silently absent.
- `secret.md`'s `_MIRRORS` record carries only two of Doppler's documented sync targets (`awsSecretsManager`, `githubActions`); the page's own growth note names the remainder (parameter store, workspace variables, CI contexts, app platforms) as future rows, matching disk state.
- `IDEAS.md` and `TASKLOG.md` are both fully empty — no forward pool or open task exists to cross-check against the above gaps; every mismatch above is currently un-tracked as a card.
