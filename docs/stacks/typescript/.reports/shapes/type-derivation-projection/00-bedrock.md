# Type Derivation And Projection

[EXTRACTOR_READS_A_PHANTOM]:
- `Schema.Schema.Type<S>`, `Schema.Schema.Encoded<S>`, `Schema.Schema.Context<S>` are not `Schema`-interface lookups — each is `S extends Schema.Variance<infer A, infer I, infer R> ? slot : never`, a conditional `infer` over a phantom carrier `Variance<A, I, R>` whose body is `readonly [TypeId]: { _A: Invariant<A>; _I: Invariant<I>; _R: Covariant<R> }`; the triple is recovered from a structurally-encoded marker, so the extractor reaches every owner that extends the variance — `SchemaClass`, `Struct`, `Union`, `suspend`, `Class`, `transform` — without any of them re-stating the slots.
- `Type` is the decoded shape, `Encoded` the wire shape, `Context` the requirement set a transform pulls in (`never` for a pure codec); the static family is read off the runtime declaration, never restated as a parallel interface.
- The carrier slot variances are deliberately asymmetric: `_A` and `_I` are `Types.Invariant` (`(_: A) => A`), `_R` is `Types.Covariant` (`(_: never) => R`); extraction of `Type`/`Encoded` is therefore exact — a subtype owner does NOT widen its decoded type at the seam — while `Context` extracts up the requirement lattice, so an owner requiring `R1 & R2` surfaces `R1 | R2` in `Schema.Schema.Context`, the union the one `Layer` graph must satisfy, the requirement covariance the reason a transform reaching a service widens only `Context` and breaks no consumer's `Type`/`Encoded`.
- The three extractors are pure type-level — they erase at runtime; the value-level mirror of the same triple is `Schema.typeSchema`, `Schema.encodedSchema`, `Schema.encodedBoundSchema`, each returning a live `SchemaClass` that re-decodes.
- `Annotable.Self<S> = ReturnType<S["annotations"]>` extracts the precise post-annotation owner type off the `annotations` method's return rather than the variance — the one extractor that reads a method signature, not a phantom, because annotating must preserve `.fields`/`.members`/`.pick` and only the method's declared return carries them; a `Schema.annotations(owner, ...)` result is typed `Annotable.Self<typeof owner>`, the interface surviving the merge.

[PARALLEL_CARRIERS_COEXIST_ON_ONE_OWNER]:
- The variance phantom is one of FOUR independent extraction traits, each a distinct branded interface with its own `infer` namespace: `Schema.Variance<A, I, R>` (codec triple), `Serializable<A, I, R>` keyed on `symbolSerializable` (the self-serialization schema), `WithResult<SA, SI, FA, FI, R>` keyed on `symbolWithResult` (the success/failure schema pair), and `Context.Tag<Id, Value>` (the service identity) — an owner extending several carriers yields several disjoint families from one declaration, the trait the extractor names selecting which family it lifts.
- `Serializable.Type<T>`/`Serializable.Encoded<T>`/`Serializable.Context<T>` extract the serialization triple off the `symbolSerializable` slot, independent of the codec `Schema.Schema.Type` — a value whose wire form differs from its in-memory schema carries both, and `Schema.serialize(self)` / `Schema.deserialize(self, value)` run the `Serializable` schema, not the structural one, so the persisted projection is a second triple the owner declares once and never a parallel DTO.
- `WithResult.Success<T>`, `WithResult.SuccessEncoded<T>`, `WithResult.Failure<T>`, `WithResult.FailureEncoded<T>`, `WithResult.Context<T>` are five extractors off the one `{ success: Schema; failure: Schema }` slot — an operation owner yields its decoded success type, its wire success type, its decoded failure type, its wire failure type, and the requirement both schemas share, the bifurcated outcome a single declaration spells and `Schema.deserializeExit(self, value)` reconstructs as `Exit<SA, FA>`.
- `Context.Tag.Service<T>` lifts `T["Service"]` (or a `TagClassShape`'s second parameter) and `Context.Tag.Identifier<T>` lifts `T["Identifier"]` — the service-locating carrier extracts on the same phantom-`infer` discipline, so a `Context.Tag` and a `Schema.Class` answer to one extraction grammar: the type of what a dependency provides and the type of what a value decodes to are both read off their owner, neither restated.
- `SerializableWithResult<A, I, R, SA, SI, FA, FI, RR> extends Serializable<A, I, R>, WithResult<SA, SI, FA, FI, RR>` carries BOTH symbol slots, and its `Context<P> = SR | RR` fuses the serialization requirement with the result requirement into one union — the procedure owner's full requirement set is derived by `|`-folding the two carrier contexts, never conflated into a single `R` the composition root would under-provide.
- `Schema.TaggedRequest(tag)(payload, { success, failure })` is the apex fusion: the resulting `TaggedRequest<Tag, A, I, R, ST, SE, FT, FE, RR>` extends `Request.Request<ST, FT>` AND `SerializableWithResult`, so one declaration is at once a request value-constructor, the `_tag`-discriminated identity, the payload codec triple (`Serializable`), and the success/failure codec quadruple (`WithResult`) — a remote-procedure shape whose six derived types (`payload`, `payloadEncoded`, `success`, `successEncoded`, `failure`, `failureEncoded`) and fused requirement set are read off carriers, the hand-spelled request DTO + response DTO + error DTO triad deleted at one stroke.
- `TaggedRequestClass<Self, Tag, Payload, Success, Failure>` computes the fused `WithResult` parameters from the success/failure schemas (`Schema.Type<Success>`, `Schema.Encoded<Success>`, `Schema.Type<Failure>`, `Schema.Encoded<Failure>`) and the payload from `Struct.Encoded<Payload>`/`Struct.Context<Payload>` — every one of the request's nine type slots is a projection of three inputs, so adding a result field is one schema edit the whole carrier chain re-derives, the constructor input `Struct.Constructor<Omit<Payload, "_tag">>` already dropping the discriminant the class materializes.

```typescript
class Fault extends Schema.TaggedError<Fault>('Fault')('Fault', { key: Schema.String }) {}
class Op extends Schema.TaggedRequest<Op>('Op')('Op', {
    payload: { key: Schema.NonEmptyString, depth: Schema.Int.pipe(Schema.optionalWith({ default: () => 1 })) },
    success: Schema.Struct({ ids: Schema.Array(Schema.UUID) }),
    failure: Fault,
}) {}
type Payload = Schema.Serializable.Type<Op>               // { key; depth } — codec triple off symbolSerializable
type Ok = Schema.WithResult.Success<Op>                   // { ids } — success leg off symbolWithResult
type WireErr = Schema.WithResult.FailureEncoded<Op>       // { _tag: "Fault"; key } — wire failure leg
type NeedR = Schema.SerializableWithResult.Context<Op>    // SR | RR fused, one Layer at the root
const exit: Effect.Effect<Exit.Exit<Ok, Fault>, ParseResult.ParseError, NeedR> = Schema.deserializeExit(new Op({ key: '<value-a>' }), '<wire>')
```

[ENCODED_DECODED_ASYMMETRY]:
- `Type` and `Encoded` diverge by field modifier, and the divergence is the reason both extractors exist: `Schema.optionalWith(field, { default: () => v })` flips the Type token to required (`weight: number`) while the Encoded token stays optional (`weight?: number | undefined`) — the default materializes on decode, absent on the wire.
- `Schema.optionalWith(field, { as: 'Option' })` projects Type to `Option<A>` and Encoded to `A | undefined`; `{ exact: true }` removes the implicit `| undefined` from Encoded; `{ nullable: true }` adds `| null` to Encoded only — every knob reshapes one slot of the triple, never both symmetrically.
- `Schema.fromKey('wire_key')` renames the Encoded key while preserving the Type key, declaring the boundary key correspondence once on the field instead of in a separate rename map; it composes after `Schema.propertySignature` lifts a plain schema into a signature.
- `NullOr<S>` is `Union<[S, typeof Null]>`, `UndefinedOr<S>` is `Union<[S, typeof Undefined]>`, `NullishOr<S>` is `Union<[S, typeof Null, typeof Undefined]>` — nullability is a derived two- or three-member union over the owner, not a `| null` token spliced onto the type, so the codec validates the null branch and the static type carries it from the same declaration.

```typescript
const Row = Schema.Struct({
    stamp: Schema.DateTimeUtc.pipe(Schema.propertySignature, Schema.fromKey('stamp_key')),    // Type key `stamp`, Encoded key `stamp_key`
    weight: Schema.Number.pipe(Schema.optionalWith({ default: () => 1 })),                     // Type `weight: number`, Encoded `weight?: number | undefined`
    variant: Schema.Literal('<variant-a>', '<variant-b>').pipe(Schema.optionalWith({ as: 'Option' })),   // Type `Option<...>`, Encoded `variant?: ... | undefined`
})
const wire: Schema.Schema.Encoded<typeof Row> = Schema.encodeSync(Row)(Schema.decodeSync(Row)({ stamp_key: '<wire>', weight: 1, variant: '<variant-a>' }))
```

[CONSTRUCTOR_AS_FOURTH_SLOT]:
- A `Schema.Class<Self>` carries seven parameters `Class<Self, Fields, I, R, C, Inherited, Proto>` where `C` is the constructor-input shape — a fourth derived type distinct from `Self` (Type), `I` (Encoded), `R` (Context); `Schema.Struct.Constructor<Fields>` computes `C` by making default-bearing and Type-optional fields optional at the `new`/`.make` call site while the Type keeps them required.
- `Owner.make(props)` and `new Owner(props)` accept `C`, not `Type`; a field with `withConstructorDefault` is omittable in `C` and present in `Type`, so the constructor input is narrower than the value it produces — never type the `make` argument as `Schema.Schema.Type<typeof Owner>`.
- `make(props: RequiredKeys<Struct.Constructor<F>> extends never ? void | Simplify<Struct.Constructor<F>> : Simplify<Struct.Constructor<F>>, options?)` collapses the constructor argument to `void` when every field is defaultable — the `keyof ... extends never ? void` idiom computed off the derived `Constructor` projection, so `Owner.make()` is legal exactly when the field map admits it.
- `Owner.fields` exposes the field record at the type level for re-derivation (`keyof typeof Owner.fields`, indexed field access); the field map is the single source the pick/omit/extend family reads, so a sub-shape is computed from `Owner.fields`, never re-listed.
- `Owner.make(props, { disableValidation: true })` skips the decode pass for an already-trusted value; the `MakeOptions` boolean overload (`Owner.make(props, true)`) is the terse form — reserved for interior re-construction where the invariant already holds.

[PHASE_PROJECTION_CUTS_THE_CHAIN]:
- `Schema.typeSchema(owner)` yields `SchemaClass<A>` that validates the decoded interior — the schema a re-encode round-trips against — cutting the chain at the decoded end with all transforms applied; the type-level `Schema.Schema.Type<typeof owner>` and value-level `Schema.typeSchema(owner)` are the same projection at two phases, never a hand-written mirror type plus a hand-written validator.
- `Schema.encodedSchema(owner)` strips every refinement and transformation, yielding a `SchemaClass<I>` that validates raw wire shape only, cutting at the wire end with all transforms shed.
- `Schema.encodedBoundSchema(owner)` retains refinements up to the FIRST transformation point, so a refined-then-transformed field keeps its pre-transform predicate while shedding the transform; on a `compose`-stacked owner this yields the schema validating the input to the first transform — the partial-pipeline projection no manual chain-slicing reproduces.
- The three phase cuts re-select which AST node every secondary projector reads: `Arbitrary.make(Schema.encodedSchema(owner))` generates raw wire fixtures for a decode test, `Pretty.make(Schema.typeSchema(owner))` renders the decoded interior, and the projector composes after the phase-cut schema with no separate generator authored — the phase projection from the core triple and the family projection from the annotations stack in one expression.

[SUBSCHEMA_DERIVATION]:
- `Schema.pick(...keys)` and `Schema.omit(...keys)` derive a sub-schema whose `Type` and `Encoded` are both projected by the same key set (`Keys extends ReadonlyArray<keyof A & keyof I>`), returning a flattened `SchemaClass` — the sub-shape's codec, type, and validator descend from the one owner with zero re-declaration.
- A struct schema also exposes `.pick`/`.omit` as methods returning `Struct<...>` (not the flattened `SchemaClass`), preserving the struct interface — the method form keeps `.fields` and further `.pick` chaining; the free `Schema.pick` form collapses to `SchemaClass` and is terminal for struct methods.
- Trap: `Schema.pick`/`Schema.omit` on a schema carrying a transformation operate on the decoded `A` and encoded `I` only — they do not slice through a `transform`; pick a transformed schema and the projection sees the post-decode field set, so a key present only on the encoded side is unreachable.
- `Schema.partial(owner)` makes every field optional (`A[K] | undefined` on both slots); `Schema.partialWith(owner, { exact: true })` drops the `| undefined`, yielding `{ [K]?: A[K] }`; `Schema.required(owner)` is the inverse (`-?`), removing optionality from every field — three derivations off one owner replacing a hand-spelled optional/required variant pair.
- `Schema.extend(owner, that)` derives a super-schema with `Type<Self> & Type<That>` and `Encoded<Self> & Encoded<That>`; supported extensions are constrained — struct-with-struct only where overlapping fields themselves support extension, struct-with-index-signature, struct-with-union-of-supported, refinement/suspend of a struct, and struct-with-struct-transform only where from/to sides share no field with the target. A bare field collision on incompatible leaf schemas is rejected at the type level.
- `Schema.pluck(owner, key)` derives a single-field projection as a transformation: `Schema<A[K], { readonly [key]: I[K] }, R>` — it pulls one column out through a decode, so the result is a transform schema (not a struct), and `Schema.Array(pluck)` lifts a table column extraction; this is the derivation that replaces a `map(row => row.column)` after decode.
- `Schema.keyof(owner)` derives `SchemaClass<keyof A>` — the literal-union schema of the owner's keys, validating and branding key names; it is the runtime-checkable counterpart to the `keyof typeof owner.fields` type-level key set.

[CORRESPONDENCE_DECLARED_ONCE]:
- `Schema.transform(from, to, { strict: true, decode, encode })` declares the encoded↔decoded correspondence as one bidirectional pair of total functions — `decode: Encoded<From> -> Type<To>`, `encode: Type<To> -> Encoded<From>`; the resulting schema's `Encoded` is `Encoded<From>` and `Type` is `Type<To>`, fusing two shapes into one owner whose round-trip is the executable spec.
- `Schema.transformOrFail` is the fallible form: `decode`/`encode` return `Effect<_, ParseResult.ParseIssue, R>`, so a parse that can fail or reach a service is the same combinator with the failure and `R` channels live; `ParseResult.succeed`, `ParseResult.fail(new ParseResult.Type(ast, actual, message))`, and `ParseResult.fromOption(opt, onNone)` are the issue constructors — never a thrown error inside a transform body.
- `decode` receives `(fromA, options, ast, fromI)` — both the decoded source value and the original encoded input — so a transform can branch on the raw wire form; `encode` receives `(toI, options, ast, toA)` symmetrically. `strict: false` widens both directions to `unknown`, the escape hatch for shapes the checker cannot align, paid for with a lost type guarantee.
- `Schema.transformLiteral(fromLiteral, toLiteral)` declares one literal↔literal mapping (`transformLiteral<Type, Encoded>`); `Schema.transformLiterals([0, 'idle'], [1, 'busy'], [2, 'done'])` derives a `Union` of literal transforms in one call — the bidirectional enum-code table where adding a pair is one tuple, never a new switch arm in both a parse and a serialize function.
- `Schema.filter(predicate, annotations)` refines `Type` in place without changing `Encoded`; the predicate returns `FilterOutput` — `boolean | string | undefined | ParseResult.ParseIssue | FilterIssue` (or a `ReadonlyArray` thereof for multi-issue) where `{ path, message }` is the `FilterIssue` shape carrying the failure locus; `undefined`/`true` pass, a string/issue fails with that message.
- `Schema.filterEffect(f)` is the service-reaching refinement — `f` returns `Effect<FilterReturnType, never, FD>`, structurally a `transformOrFail<S, SchemaClass<Type<S>>, FD>` whose `To` re-states `Type<S>`, so a uniqueness or existence check that needs a layer threads `FD` into `Context` exactly as a transform leg does; refinement and transformation share one propagation rule, not two.

[COMPOSE_ALIGNMENT]:
- `Schema.compose(from, to)` fuses two transform schemas into one owner by aligning `Type<From>` with `Encoded<To>`, yielding `transform<From, To>` whose `Encoded` is `Encoded<From>` and `Type` is `Type<To>` — the intermediate shape is consumed at the seam and erased from the public triple, so a three-stage pipeline reads as one schema with two visible slots.
- The default overload demands `Type<From>` exactly equal `Encoded<To>`; the mismatch overloads accept a `From` whose `Type` is assignable to `Encoded<To>` (`B extends Encoded<To>`) or a `To` whose `Encoded` is assignable to `Type<From>` (`C extends Type<From>`), so subtype alignment composes without a bridging transform — the wider side is narrowed at the join, not adapted.
- `Schema.compose(from, to, { strict: false })` drops the alignment constraint and joins on `unknown`, the escape hatch where the intermediate shapes cannot be made to meet; the round-trip type guarantee at the seam is surrendered, so a strict-false compose is the reach-limit marker, never a default.
- Composition is associative on the codec but not on annotations: the composed schema carries an AST `Transformation` node whose kind is `ComposeTransformation` wrapping both legs, and the seam's own identifier/annotations are lost unless reattached on the composite — annotations live on the node, and `compose` builds a fresh one.
- `Schema.asSchema(owner)` widens a richly-typed composite (`extend`/`compose`/`Class` result) back to the plain `Schema<Type<S>, Encoded<S>, Context<S>>` interface by re-extracting all three through the same phantom (`Schema.AsSchema<S>` its type-level mirror) — the up-cast loses the construction-specific interface while keeping the triple, used only where a homogeneous collection cannot carry the precise shape, never as a habitual annotation.

[LAYERED_STRING_CODEC]:
- `Schema.parseJson(inner)` is `transform<SchemaClass<unknown, string>, S>` — the string→`unknown` `JSON.parse` leg pre-composed with the inner codec in one declaration, so a wire string decodes through `JSON.parse` then through `inner`'s field validation as a single owner; encode runs `inner` then `JSON.stringify`, and `ParseJsonOptions` configures both passes.
- `Schema.split(separator)` derives `transform<SchemaClass<string>, Array$<typeof String$>>` — a flat string Encoded becomes a `ReadonlyArray<string>` Type; composing `split('/')` with `Schema.Array(NumberFromString)` via `Schema.compose` derives a path-of-numbers codec where each segment re-decodes, the loose `string.split(...).map(Number)` collapsed into one round-trippable owner.
- `Schema.headOrElse(fallback?)` is `transform<S, SchemaClass<A[number]>>` over an array schema — it projects the first element on decode and re-wraps as a singleton on encode; chained after `split`, it derives a parse-first-segment codec, and the `fallback` lazy arg makes the empty-array case total instead of a separate guard.
- `Schema.NumberFromString`, `Schema.BigInt`, `Schema.DateTimeUtc` are themselves pre-composed `transformOrFail<SchemaClass<string,string>, ...>` owners — a string-leg codec is never hand-spelled with `parseFloat`/`new Date`; the leaf codec is admitted and composed, so a numeric column inside a JSON string is `parseJson(Struct({ n: NumberFromString }))`, two `JSON.parse`/`parseFloat` reflexes deleted at once.

[TEMPLATE_PARSE_DUALITY]:
- `Schema.TemplateLiteral(...params)` derives a single template-literal STRING type — `GetTemplateLiteralType` folds the param tuple into one `` `${A}${B}` `` literal so `Type` and `Encoded` are both that string; it is the validator for a structured key, not a parser, and adding a segment is one tuple entry the folded type absorbs.
- `Schema.TemplateLiteralParser(...params)` is the dual: `Encoded` is the folded string (`GetTemplateLiteralParserEncoded` joins each leg's `Encoded`), `Type` is the parsed TUPLE (`GetTemplateLiteralParserType` lifts each schema leg's `Type`), and `Context` is `Context<Params[number]>` — a flat `"id-42-open"` decodes to `readonly ["id", 42, "open"]` with each leg re-decoded, the hand-written regex-plus-destructure deleted, the encode leg re-joining the tuple into the string for free.
- A literal-value param contributes to both the Encoded string and the Type tuple as the literal itself (a separator is a fixed literal leg); a schema param contributes its `Type` to the tuple and its `Encoded` to the string — so embedding `Schema.NumberFromString` mid-template parses a numeric segment to `number` while the wire stays a digit run, the parse correspondence declared once across both directions.

[TRANSFORM_POINT_ORDERING]:
- `BaseClass.transformOrFail<Next>(id)(newFields, { decode, encode })` derives `Class<Next, Fields & NewFields, I, R | Struct.Context<NewFields> | R2 | R3, ...>` where `decode: Type<Fields> -> Effect<Type<Fields & NewFields>, ParseIssue, R2>` runs AFTER the base decodes — the transform sees the fully decoded base value and computes the added fields from it, the derived class inheriting base methods and prototype.
- `BaseClass.transformOrFailFrom<Next>(id)` is the same derivation with the transform point BEFORE base decode: `decode: I -> Effect<I & Encoded<NewFields>, ParseIssue, R2>` operates on raw Encoded, so a field synthesized from wire-level data (a checksum over the unparsed payload) attaches encoded-side, then the merged Encoded decodes through the base — the position of the seam in the decode pipeline is the only axis distinguishing the two, and choosing wrong reads pre-validation noise or post-validation values.
- Both forms thread `R2` (decode requirement) and `R3` (encode requirement) independently into the class `Context` alongside `Struct.Context<NewFields>` — a class-to-class transform reaching one service on decode and another on encode declares two requirement variables the composition root must satisfy, never a single conflated `R`.
- The encode leg is mandatory and total on the type level: `encode: Type<Fields & NewFields> -> Effect<Type<Fields>>` must reconstruct the base from the derived, so a transform adding a non-invertible field forces the encode to discard it explicitly — the codec refuses a one-way derivation, the asymmetry surfaced at declaration, not at the lost round-trip.

[CONTEXT_FAILURE_PROPAGATION]:
- `transformOrFail<From, To, R>` has `Context = Context<From> | Context<To> | R` — composing transforms unions every leg's requirement set, so a five-stage pipeline where one leg reaches a service carries that service in the composite `Context` and the whole chain is provided one `Layer` at the root, never a leg-local provide.
- A composite transform's failure is a `ParseResult.Composite` issue wrapping the leg's `ParseResult.Pointer` and `Transformation` AST nodes, so a parse error names which composition seam rejected — the failure locus is derived from the chain structure, not flattened to a single message; `ParseResult.succeed`/`ParseResult.fail` return `Either`, never a thrown error inside any leg.

[PROPERTY_SIGNATURE_TRANSFORM]:
- The `PropertySignature` carrier is the seven-slot derived type `PropertySignature<TypeToken, Type, Key, EncodedToken, Encoded, HasDefault, R>` — the two `Token` slots hold `":"`/`"?:"` independently per side and `Key` carries the renamed Encoded key, so every field-level optionality, default, and rename decision is one position read off the modifier, the field algebra data the owner carries rather than a parallel optional/required field pair.
- `Schema.optionalToRequired(from, to, { decode, encode })` derives `PropertySignature<":", TA, never, "?:", FI, false, FR | TR>` — an Encoded-optional field becomes a Type-required field through an `Option`-mediated transform: `decode: Option<FA> -> TI` materializes a value for the missing case, `encode: TI -> Option<FA>` where `none` omits the key from the wire, so a defaulted field's presence asymmetry is declared as a total bidirectional pair, not a post-decode patch.
- `Schema.requiredToOptional` inverts it (`PropertySignature<"?:", TA, never, ":", FI, ...>`): a wire-required field becomes Type-optional, `decode: FA -> Option<TI>` dropping it from the decoded shape when a sentinel matches; `Schema.optionalToOptional` keeps both sides optional with `Option -> Option` on both legs — the four-way optionality cross-product (`:`/`?:` × Type/Encoded) is three named field transforms, never a hand-spelled `K?` plus a runtime presence check.
- `Schema.attachPropertySignature(key, value)` derives `SchemaClass<A & { readonly [K]: V }, I, R>` — the discriminant lands on `Type` only, absent from `Encoded`, so a tag is materialized on decode and stripped on encode; attaching `('kind', '<variant-a>')` to each arm before a `Schema.Union` derives a discriminated union from tagless wire shapes, the discriminant computed by the codec rather than carried on the payload.

[FIELD_MAP_IS_THE_OWNER]:
- The owner the family extracts from is the field record `Fields = { readonly [x: PropertyKey]: Schema.All | PropertySignature.All }`; `Struct.Type<F>`, `Struct.Encoded<F>`, `Struct.Constructor<F>`, and `Struct.Context<F>` are independent mapped types over that one record, so the decoded shape, the wire shape, and the constructor-input shape are computed projections of the field map, never three parallel interfaces.
- `Struct.Type<F>` is `UnionToIntersection<{ [K in keyof F]: F[K] extends OptionalTypePropertySignature ? { readonly [H in K]?: Schema.Type<F[H]> } : { readonly [h in K]: Schema.Type<F[h]> } }[keyof F]>` — each field is mapped to a single-key object, the `[keyof F]` index distributes them into a union, and `UnionToIntersection` folds the union back into one record; optionality is decided per field by whether its `PropertySignature` Type-token slot is `"?:"`, the `?` modifier emitted only on that branch.
- `Struct.Encoded<F>` splits the key space in two and remaps each half: `{ [K in Exclude<keyof F, EncodedOptionalKeys<F>> as Key<F, K>]: Schema.Encoded<F[K]> } & { [K in EncodedOptionalKeys<F> as Key<F, K>]?: Schema.Encoded<F[K]> }` — the `as Key<F, K>` remap clause is where `Schema.fromKey` lands (the renamed Encoded key replaces the Type key), and the optional half is computed from the Encoded-side token, independent of the Type-side token the decoded projection reads.

[KEY_REMAP_FILTERS_THE_FAMILY]:
- `EncodedOptionalKeys<F> = { [K in keyof F]: F[K] extends OptionalEncodedPropertySignature ? K : never }[keyof F]` is the filter idiom — map each key to itself or `never` by a conditional, then index with `[keyof F]` to collapse the `never`s out, yielding exactly the subset of keys whose Encoded slot is optional; this is how a default-bearing field is required in `Type` yet optional in `Encoded` with no second key list authored.
- `Key<F, K> = [K] extends [never] ? never : F[K] extends PropertySignature.All<infer Key> ? [Key] extends [never] ? K : Key : K` resolves the Encoded key name off the `PropertySignature` carrier's `Key` slot, falling back to the declared key when no rename is present — the `[Key] extends [never]` guard distinguishes "no rename declared" from a renamed key, so the rename correspondence is read off the field modifier rather than tracked in a parallel map.
- The `as` key-remapping clause with a conditional producing `never` is the member-deletion primitive of the whole lane: a mapped type emitting `never` for a key removes that key from the result, so a family is filtered to a subset (`EncodedOptionalKeys`, `TaggedEnum.Args` dropping `_tag`, `ExtractFields` dropping an absent variant) by the same construct that renames keys, never by an explicit exclusion list.

[UNION_DISTRIBUTES_ITS_TRIPLE]:
- `Schema.Union<Members extends ReadonlyArray<Schema.All>>` extends `AnnotableClass<Union<Members>, Schema.Type<Members[number]>, Schema.Encoded<Members[number]>, Schema.Context<Members[number]>>` — `Members[number]` indexes the member tuple into a union of schemas and the extractor distributes over it, so the union's decoded type IS the distributed union of member decoded types, the variant family lifted from the tuple, never a hand-written `A | B | C` restated beside the schema.
- `Union(member)` with a single argument returns that member's type unchanged (`Union<Member extends Schema.All>(member: Member): Member`) — a one-member union collapses to the member with no `Union` wrapper, so a union built programmatically from a derived member list degrades cleanly at arity one.
- `.members: Readonly<Members>` exposes the tuple at the value level for re-derivation — a per-arm projection maps over `.members` and the result type is inferred from the tuple positions, the value-level mirror of `Members[number]`, so iterating the arms and lifting each arm's type share one source.

[TAG_ARITHMETIC_OFF_THE_UNION]:
- `Types.Tags<E> = E extends { _tag: string } ? E["_tag"] : never` distributes over a union and lifts the discriminant literal set; `Types.ExtractTag<E, K> = Extract<E, { _tag: K }>` selects one arm by tag and `Types.ExcludeTag<E, K> = Exclude<E, { _tag: K }>` removes it — the variant family is sliced by discriminant with built-in `Extract`/`Exclude` over the union, never an `is-*` predicate per arm.
- `Match.Tags<D extends string, P> = P extends Record<D, infer X> ? X : never` generalizes tag extraction to ANY discriminant field name `D`, not only `_tag` — a union keyed on a foreign discriminant (`type`, `kind`) yields its literal set through the same conditional-`infer`, so an externally-shaped tagged union dispatches without a rename adapter on the discriminant.
- `Match.ApplyFilters<I, A> = A extends Only<infer X> ? X : A extends Without<infer X> ? Exclude<I, X> : never` is the set-arithmetic the matcher threads as it narrows: `Only<X>` collapses the residual to `X`, `Without<X>` is `Exclude<I, X>`, and `AddWithout`/`AddOnly` accumulate the filters across arms so the post-match residual type is computed from the cases consumed, the exhaustiveness proof being the residual reaching `never`.

```typescript
const Variant = Schema.Union(
    Schema.Struct({ _tag: Schema.Literal('<variant-a>'), x: Schema.Number }),
    Schema.Struct({ _tag: Schema.Literal('<variant-b>'), x: Schema.Number, y: Schema.Number }),
)
type Variant = Schema.Schema.Type<typeof Variant>      // distributed off Members[number]
type Tag = Types.Tags<Variant>                         // "<variant-a>" | "<variant-b>", lifted
type B = Types.ExtractTag<Variant, '<variant-b>'>      // one arm sliced by discriminant
type Rest = Types.ExcludeTag<Variant, '<variant-a>'>   // residual family, no per-arm guard
const arms = Variant.members.map((m) => Schema.encodedSchema(m))   // per-arm projection over the tuple
const measure: (v: Variant) => number = Match.type<Variant>().pipe(
    Match.tag('<variant-a>', ({ x }) => x),
    Match.tag('<variant-b>', ({ x, y }) => x * y),
    Match.exhaustive,                                  // residual narrowed to never by ApplyFilters
)
```

[HKT_LIFTS_A_GENERIC_FAMILY]:
- A generic variant family is encoded as `interface F extends Data.TaggedEnum.WithGenerics<2> { readonly taggedEnum: MyUnion<this["A"], this["B"]> }` — the interface carries phantom slots `A`/`B`/`C`/`D` and `this["A"]` references them, so `TaggedEnum.Kind<F, X, Y>` re-applies the generics by intersecting `{ A: X; B: Y }` and indexing `["taggedEnum"]`, lifting the higher-kinded family the type system otherwise cannot abstract over.
- `Data.taggedEnum<F>()` over a `WithGenerics` definition returns a constructor record `{ [Tag in F["taggedEnum"]["_tag"]]: <A, B>(args) => Value }` plus `GenericMatchers<F>` whose `$is`/`$match` re-thread the generics per call — the whole constructor-and-matcher surface is one mapped type over the tag set, the family declared once and every arm derived.
- `TaggedEnum.Args<A, K> = { readonly [K in keyof E as K extends "_tag" ? never : K]: E[K] } extends infer T ? Types.VoidIfEmpty<T> : never` (with `E = Extract<A, { _tag: K }>`) drops `_tag` via the `as ... ? never` remap, then `VoidIfEmpty<T> = keyof S extends never ? void : S` collapses an empty payload to `void` — a nullary case constructor is callable with no argument, the arity recovered from the field map rather than a manual overload.

[VARIANCE_PHANTOM_ENCODES_THE_FAMILY_FOLD]:
- `Invariant<A> = (_: A) => A`, `Covariant<A> = (_: never) => A`, `Contravariant<A> = (_: A) => void` are the function-position phantom encodings that pin a type parameter's variance; `Invariant.Type<A> = A extends Invariant<infer U> ? U : never` recovers the carried type — the variance annotation rides a phantom field and the carried type is extracted by `infer`, so a parameter that must not vary is fenced structurally rather than by a runtime guard.
- `UnionToIntersection<T> = (T extends any ? (x: T) => any : never) extends (x: infer R) => any ? R : never` exploits contravariant inference position to fold a union into an intersection — the distributive arrow turns each union member into a parameter and `infer R` in argument position collects them as an intersection; this is the engine behind `Struct.Type`/`Constructor` folding the per-key object map, never a hand-merged intersection.

[VARIANT_SET_IS_THE_OUTER_OWNER]:
- `VariantSchema.make({ variants, defaultVariant })` declares the variant alphabet once and returns a closed toolkit (`Struct`, `Field`, `FieldOnly`, `FieldExcept`, `Class`, `Union`, `extract`, `fieldEvolve`, `fieldFromKey`) every member of which is parameterized by `Variants[number]`; the variant set is the second axis of the owner — beyond the field map, one declaration fixes which projections exist, so a new projection is a tuple entry the whole toolkit re-derives, never a new schema written beside the others.
- A persistence model owner reuses the six-variant toolkit pre-bound to `"select" | "insert" | "update" | "json" | "jsonCreate" | "jsonUpdate"`, so `Owner` itself decodes the `select` projection and `Owner.insert`, `Owner.update`, `Owner.json`, `Owner.jsonCreate`, `Owner.jsonUpdate` are five sibling `Schema.Struct` projections hanging off the one class — six codecs, six static types, one field map, the hand-spelled insert/update/select/json DTO quartet deleted.
- `defaultVariant` selects which projection the class instance IS: the `Class` self-type is `ClassFromFields<Self, Fields, ExtractFields<Default, Fields, true>>`, so the default variant's extracted fields define `Self`, its `Type`/`Encoded`/`Context`/constructor, while every other variant is reached as a named property; the default is not a seventh schema, it is the identity of the owner.

[FIELD_CARRIES_ITS_VARIANT_MEMBERSHIP]:
- A `Field<Config>` is a per-variant schema map — `Config = { readonly [variant: string]: Schema.Schema.All | Schema.PropertySignature.All | undefined }` — so a single field declares, in one record, which variants admit it and with what schema each one validates; the field, not an external column table, owns its participation across the whole family.
- `FieldOnly(...keys)(schema)` derives `Field<{ readonly [K in Keys[number]]: S }>` — the field appears in exactly the listed variants; `FieldExcept(...keys)(schema)` derives `Field<{ readonly [K in Exclude<Variants[number], Keys[number]>]: S }>` — present in every variant but the listed ones, the complement computed by `Exclude` over the closed alphabet, never a second key list; the two are the dual halves of one membership decision.
- `Field(config)` is the explicit per-variant form, and its argument is fenced by `config & { readonly [K in Exclude<keyof A, Variants[number]>]: never }` — a key outside the variant alphabet is typed `never` and rejected at declaration, so a misspelled variant name is a compile error on the field, not a silently dropped projection.
- `Struct.Validate<A, Variant>` walks the field map and forces `[keyof Config] extends [Variant]` for every `Field`, emitting the literal `"field must have valid variants"` as the field's type when a `Field` carries a variant the owner's alphabet does not contain — the cross-check that field membership is a subset of owner membership is a mapped-type assertion on the struct argument, the invariant the checker chases before any decode.

[EXTRACTION_DELETES_BY_REMAP_TO_NEVER]:
- `ExtractFields<V, Fields, IsDefault>` is the family-deletion engine: `{ readonly [K in keyof Fields as [Fields[K]] extends [Field<infer Config>] ? V extends keyof Config ? K : never : K]: ... }` — the `as` key-remap clause emits the key when the variant `V` is in the field's `Config`, and `never` when it is not, so a field absent from variant `V` is removed from `V`'s projection by the same remap-to-`never` primitive that filters any mapped family; the projection's field set is computed, never enumerated.
- A plain `Schema`/`PropertySignature` field (not wrapped in `Field`) is unconditional — its key never remaps to `never`, so it survives every variant projection; only `Field`-wrapped fields participate in variant filtering, making the wrapper the explicit opt-in to per-variant presence.
- The value side of the remap selects the schema: for a `Field` it indexes `Config[V]` (the variant-specific schema), for a nested `Struct` it recurses `Extract<V, Fields[K], IsDefault>`, for a bare schema it passes through — one mapped type resolves presence, per-variant schema, and nested-struct recursion in a single conditional chain.
- `Extract<V, A, IsDefault>` wraps the extracted field map: `Schema.Struct<Schema.Simplify<ExtractFields<V, Fields>>>` for a non-default variant, while `IsDefault extends true` short-circuits to `A` itself when `A` is already a `Schema.Schema.Any` — the default variant of an already-schema owner returns the owner rather than re-wrapping it, so the class's own decode path is not double-projected.

```typescript
const V = VariantSchema.make({
    variants: ['read', 'write', 'public'] as const,
    defaultVariant: 'read',
})
class Owner extends V.Class<Owner>('Owner')({
    id: V.FieldExcept('write')(Schema.Number.pipe(Schema.brand('Id'))),
    secret: V.FieldExcept('public')(Schema.Redacted(Schema.String)),
    label: V.Field({ read: Schema.String, write: Schema.String, public: Schema.String }),
}) {}
type Read = Schema.Schema.Type<typeof Owner>            // { id; secret; label } — default identity
type Write = Schema.Schema.Type<typeof Owner.write>     // { secret; label } — id remapped to never
type Public = Schema.Schema.Type<typeof Owner.public>   // { id; label } — secret remapped to never
```

[FIELD_KIND_IS_A_VARIANT_PROFILE]:
- A persistence field constructor IS its variant `Config` written once: `Generated(S)` is `Field<{ select: S; update: S; json: S }>` (database-supplied, absent from insert and the JSON-create paths), `GeneratedByApp(S)` adds `insert`, `Sensitive(S)` is `Field<{ select: S; insert: S; update: S }>` (every database variant, no JSON variant) — the column's lifecycle is the set of variant keys present, so reading the `Config` keys is reading the column policy, no separate annotation.
- `FieldOption(S)` projects asymmetrically by variant kind: `select`/`insert`/`update` become `Schema.OptionFromNullOr<S>` (a `transform<NullOr<S>, OptionFromSelf<...>>` decoding a nullable database cell to `Option`), while `json`/`jsonCreate`/`jsonUpdate` become `Schema.optionalWith<S, { as: 'Option' }>` and its `{ nullable: true }` variants (a missing key, not a null cell) — one field declaration emits two distinct absence encodings keyed by whether the variant is database or wire.
- `DateTimeInsert` is `Field<{ select: DateTimeUtc; insert: Overrideable<...>; json: DateTimeUtc }>` — present on insert as an overrideable current-time generator, present on select/json as a plain decoded `DateTime.Utc`, and absent from `update` by omission from the `Config`; `DateTimeUpdate` adds the `update` key with its own `Overrideable`, so insert-stamping versus update-stamping is the presence of one variant key, never a boolean.
- Idempotent on a `Field`: `FieldOption(existingField)` maps over the field's existing `Config` (`{ readonly [K in keyof S]: ... }`), routing database keys through `OptionFromNullOr` and the rest through nullable `optionalWith`, so wrapping an already-variant-profiled field re-projects each variant in place rather than rebuilding the profile.

[OVERRIDEABLE_IS_THE_GENERATED_SLOT]:
- `Overrideable<To, From, R>` is the seven-slot `PropertySignature<":", (To & Brand<"Override">) | undefined, never, ":", From, true, R>` — a Type of `(To & Brand<"Override">) | undefined` with the has-default slot `true`, so the constructor may omit the field and a layer-driven generator supplies it; the `Brand<"Override">` witnesses that an explicit caller value bypasses generation.
- `Overrideable(from, to, { generate, decode, constructorDefault })` declares the generation correspondence once: `generate: Option<ITo> -> Effect<From, ParseIssue, R>` produces the wire value from the optional caller override, threading `R` into the field's requirement set, so an insert timestamp or generated UUID reaches `Clock` or entropy through the same `R` the composite codec carries — the generated column's effect requirement is part of the field, surfaced on the variant that uses it.
- `Override(value)` brands a caller-supplied value `value & Brand<"Override">` so it survives the generator's `Option<ITo>` short-circuit — passing `Override(v)` at the insert call site forces the field to `v` instead of generating, the explicit-versus-generated decision carried in the value's brand, not a parallel flag argument.
- `UuidV4WithGenerate(brandedBytesSchema)` returns `Overrideable<Uint8Array & Brand<B>, Uint8Array>` — a binary UUID generated on insert keeps its brand `B` through generation, so the generated identifier is nominally distinct from a raw byte array at every consumer, the brand surviving the generator slot.

[CLASS_PROJECTIONS_FEED_DERIVED_OWNERS]:
- `Owner.json` is a `Schema.Struct`, so it is the input to a richer owner: `class OwnerJson extends Schema.Class<OwnerJson>('OwnerJson')(Owner.json) {}` lifts the json projection into a class with methods and getters, the wire-facing class derived from the persistence owner's json variant rather than declared as a parallel shape — the projection is the field source the dependent class reads.
- `extract(variant)(structOwner)` is the standalone projector matching the class's named properties — `extract('insert')(Owner)` yields the same `Schema.Struct` as `Owner.insert` for a bare `VariantSchema.Struct` that is not a class, so a union member or nested struct is projected to a variant without being promoted to a class; the dispatch is on the variant literal, total over the closed alphabet.
- `VariantSchema.Union(...members)` carries `Union.Variants<Members, Variants>` = `{ readonly [V in Variants]: Schema.Union<{ [K in keyof Members]: Extract<V, Members[K]> }> }` — every variant of a union of variant-structs is the union of that variant's projection of each member, so a discriminated entity whose arms each carry generated and sensitive columns projects arm-wise to a per-variant union, the variant filtering distributing across both the union arity and the field map in one type.
- `makeRepository(Owner, { tableName, spanPrefix, idColumn })` consumes the projections directly: `insert: (S['insert']['Type']) => Effect<S['Type'], never, S['Context'] | S['insert']['Context']>` types the insert argument as the insert projection's decoded type and the result as the default projection's, while `idColumn` is constrained `keyof S['Type'] & keyof S['update']['Type'] & keyof S['fields']` — the CRUD surface's argument and result types are read off the variant projections, the id column proven to exist in both select and update at compile time, no hand-written DTO between the model and the table.

[PROJECTOR_READS_THE_OWNER]:
- `Arbitrary.make(owner)` returns `FastCheck.Arbitrary<A>`, `Pretty.make(owner)` returns `(a: A) => string`, `JSONSchema.make(owner)` returns `JsonSchema7Root`, and `Schema.equivalence(owner)` returns `Equivalence.Equivalence<A>` — four secondary families lifted off the one schema owner by structural recursion over its AST, the test generator, renderer, wire-contract document, and structural equality derived from the same declaration that yields `Type`/`Encoded`/codec, never a hand-written `fc.record(...)`, `JSON.stringify` formatter, OpenAPI object, or field-by-field comparator.
- `Arbitrary.makeLazy(owner)` returns `LazyArbitrary<A>` — a `(fc: typeof FastCheck) => FastCheck.Arbitrary<A>` deferring the `FastCheck` binding so a recursive owner's generator references itself through the suspend point without eager construction; `Arbitrary.make` is `makeLazy` applied to the admitted `FastCheck` re-export, so the recursion-safe form is the primitive and the eager form the convenience.
- Of the four, only `Schema.equivalence` is exported on `Schema`; `Arbitrary.make`/`Pretty.make`/`JSONSchema.make` live in their own modules over the same AST — the projector that threads no extra context (equivalence) folds into the schema surface, the three that carry a generation context, a render compiler, or a target dialect stay module-owned, so the import shape itself encodes which projection needs configuration.
- A leaf codec carries its projectors already: `Schema.NumberFromString`, `Schema.DateTimeUtc`, `Schema.Redacted` each ship an `arbitrary` and `pretty` annotation so a struct built from them needs zero per-field generator wiring — `Arbitrary.make(Struct({ at: DateTimeUtc, n: NumberFromString }))` produces a valid-`DateTime`, valid-numeric-string generator with no annotation authored on the struct, the leaf annotation propagating up the AST fold.

[ANNOTATION_IS_THE_SINGLE_OVERRIDE_SLOT]:
- `owner.annotations({...})` accepts `Annotations.Schema<A>` carrying typed slots `arbitrary?: ArbitraryAnnotation<A>`, `pretty?: PrettyAnnotation<A>`, `equivalence?: EquivalenceAnnotation<A>`, `jsonSchema?: JSONSchemaAnnotation`, `examples?`, `default?`, `identifier?`, `message?`, `parseIssueTitle?`, `decodingFallback?` — every secondary-family override and every documentation fact rides one record on the declaration, so refining how a value generates, prints, or documents is one slot change at the owner, every projector re-reading it, never a parallel registration call per projector.
- `arbitrary: () => (fc) => fc.integer({ min: 1 })` overrides only the generator while `Type`, `Encoded`, codec, and equality stay derived — a slot reshapes exactly one projection, so a field whose default `fc` generator is too broad is narrowed without touching the decode path, the override scoped to the family it names.
- `decodingFallback: (issue) => Effect.succeed(<value-a>)` is the annotation that converts a `ParseIssue` into a recovered `A` during decode, so a non-fatal wire defect resolves to a substitute at the owning node rather than failing the whole decode — the recovery policy is a slot on the schema, not a `catchAll` wrapped around the boundary call.
- `identifier` on a referenced sub-owner makes `JSONSchema.make` emit a `$ref` into `$defs` instead of inlining, so a shape used in three places appears once in the document and is referenced thrice — the same identifier that names the AST node for error messages drives wire-contract deduplication, one slot serving the diagnostic and the document projection.
- `Schema.annotations(owner, {...})` and the `owner.annotations({...})` method are the merge form: new slots overwrite duplicates and absent slots inherit, returning `Annotable.Self<S>` — the precise owner interface (`.fields`, `.members`, further `.pick`) survives the annotation, so annotating never collapses a `Struct`/`Class` to the bare `Schema` interface the way `Schema.asSchema` does.

[CLASS_ANNOTATION_THREE_NODE_TARGET]:
- A `Schema.Class` is an AST `Transformation` (decode lifts the field record into the instance), so its annotation argument is `ClassAnnotations<Self, A> = Annotations.Schema<Self> | readonly [Annotations.Schema<Self> | undefined, (Annotations.Schema<Self> | undefined)?, Annotations.Schema<A>?]` — the tuple form places annotations on three distinct nodes: the transformation, the surrogate, and at position three `Annotations.Schema<A>` annotating the decoded TYPE node, so a `pretty`/`arbitrary` slot meant for the instance interior attaches at index three, not index one.
- The bare-record annotation form on a class targets only the transformation node, so an `examples` or `jsonSchema` slot passed as a plain record documents the codec-level view; the projector reaching for the decoded shape (`Schema.typeSchema(owner)` then `Arbitrary.make`) sees the position-three annotations — placing a generator override on the wrong tuple slot silently no-ops because the projector reads a different node.
- `Schema.TaggedClass`, `Schema.TaggedError`, `Schema.TaggedRequest` all thread the same `ClassAnnotations<Self, ...>` argument, so the discriminant-bearing owners carry the identical three-node annotation algebra — a tagged error's `pretty` for its message and `arbitrary` for property-test generation are slots on the one class declaration, never a sibling formatter and a sibling generator.

[GENERIC_CARRIER_THREADS_TYPE_PARAMETER_PROJECTORS]:
- `Annotations.Schema<A, TypeParameters extends ReadonlyArray<any>>` parameterizes the projector slots by the carrier's type parameters: `arbitrary?: ArbitraryAnnotation<A, TypeParameters>` is `(...arbitraries: { [K]: LazyArbitrary<TypeParameters[K]> }, ctx) => LazyArbitrary<A>`, `equivalence?: EquivalenceAnnotation<A, TypeParameters>` is `(...equivalences: { [K]: Equivalence<TypeParameters[K]> }) => Equivalence<A>`, `pretty?: PrettyAnnotation<A, TypeParameters>` symmetrically — a generic container's generator, comparator, and printer each RECEIVE the corresponding type-parameter's projector and return the container's, so the family composes through the parameter rather than re-deriving per instantiation.
- `Schema.declare(typeParameters, { decode, encode }, annotations)` is the custom-carrier owner where `annotations` is `Annotations.Schema<A, { [K in keyof P]: Schema.Type<P[K]> }>` — one `declare` fuses the codec (decode/encode each receiving the parameter schemas) with the parameterized projector slots, so a native carrier the schema AST does not know structurally still yields a correct generator and comparator for every element type, the projectors threaded the same way the codec is, in a single declaration.
- `R extends Schema.Context<P[number]>` on `declare` forces every requirement to enter through a type parameter, so `Schema.typeSchema`/`Schema.encodedSchema` of a declared carrier carry `never` context — the projector inputs and the codec inputs draw from one parameter list, and a service reached outside that list is a type error at declaration, not a `Context` leak discovered downstream.
- `ctx: ArbitraryGenerationContext` carries `maxDepth`, `depthIdentifier`, and `constraints` (a `StringConstraints`/`NumberConstraints`/`ArrayConstraints`/`DateConstraints`/`BigIntConstraints` tagged union) into the arbitrary annotation, so a recursive carrier's generator reads the depth budget and a refined leaf's generator reads the inferred numeric/length bounds from the same AST that holds the refinement — the constraint the codec validates is the constraint the generator obeys, propagated as context not duplicated as a generator parameter.

```typescript
interface Pair<A> extends Schema.Schema<readonly [A, A]> {}
const Pair = <S extends Schema.Schema.Any>(item: S) =>
    Schema.declare(
        [item],
        {
            decode: (inner) => (u, opts, ast) =>
                Array.isArray(u) && u.length === 2
                    ? ParseResult.succeed([u[0], u[1]] as const).pipe(
                          Effect.flatMap(([a, b]) =>
                              Effect.all([ParseResult.decodeUnknown(inner)(a), ParseResult.decodeUnknown(inner)(b)]),
                          ),
                      )
                    : ParseResult.fail(new ParseResult.Type(ast, u)),
            encode: (inner) => (pair) => Effect.all(pair.map(ParseResult.encodeUnknown(inner))),
        },
        {
            identifier: 'Pair',
            arbitrary: (itemArb) => (fc) => fc.tuple(itemArb(fc), itemArb(fc)),
            equivalence: (itemEq) => (x, y) => itemEq(x[0], y[0]) && itemEq(x[1], y[1]),
            pretty: (itemP) => (p) => `[${itemP(p[0])}, ${itemP(p[1])}]`,
        },
    )
const gen = Arbitrary.make(Pair(Schema.NumberFromString))   // tuple of valid numeric strings, item generator threaded
const eq = Schema.equivalence(Pair(Schema.String))           // pairwise String equivalence, no field-by-field comparator
```

[REFINEMENT_AND_TRANSFORM_REROUTE_THE_PROJECTORS]:
- A `Schema.filter` refinement narrows `Type` and, when its predicate carries a `jsonSchema` annotation or a recognized refinement schema-id, `JSONSchema.make` emits the matching keyword (`minLength`, `minimum`, `pattern`) into the node — a string length bound is one `Schema.minLength` in the codec and the same bound surfaces in the wire-contract document and the arbitrary's `StringConstraints`, three projections honoring one refinement.
- A `Schema.transformOrFail` interposes a `Transformation` node, so the projectors split by phase: `JSONSchema.make` on a transformed owner documents the ENCODED side (the wire shape consumers send), while `Arbitrary.make` generates the DECODED `A` — a numeric-string field documents as `{ type: "string" }` yet generates a `number`, so feeding the generator output back through `Schema.encode(owner)` is required to produce a document-conformant fixture, the phase mismatch a trap not a defect.
- `JSONSchema.make(owner, { target: "openApi3.1" })` retargets the dialect (`jsonSchema7` default, `jsonSchema2019-09`, `jsonSchema2020-12`), and `JSONSchema.fromAST(ast, { definitions, topLevelReferenceStrategy, additionalPropertiesStrategy })` is the lower form taking pre-seeded `$defs` and the `"skip"`/`"keep"` top-reference and `"strict"`/`"allow"` additional-properties policies — one owner emits every dialect by argument, never a per-dialect hand-maintained document.

[SURROGATE_AND_PARSE_TITLE_REDIRECT_THE_AST_FOLD]:
- `SurrogateAnnotation` (`type SurrogateAnnotation = AST`) overrides the AST a projector folds over: a `Schema.declare` carrier with no inner structure attaches a surrogate AST so `JSONSchema.make` and `Pretty.make` traverse the surrogate's shape instead of treating the carrier as opaque — the projection-visible structure is a slot, so a custom carrier presents a struct-shaped document while decoding through a native constructor.
- `parseIssueTitle: (issue: ParseIssue) => string | undefined` rides the same annotation record and feeds `ParseResult` rendering, so the decode-failure projection (the human-facing error string) is configured beside the success projections (generator, printer) on one owner — the failure family and the value families share the annotation surface, never a parallel error-formatter registry.
- `examples: readonly [A, ...A[]]` and `default: A` are documentation-only slots: `JSONSchema.make` emits them into `examples`/`default` of the node, and they do NOT seed `Arbitrary.make` — a hand-listed example is a wire-document fixture, the property-test generator stays derived from the codec structure, so the two never conflate and an example is not a substitute for a generator override.

[CONTRAVARIANT_IS_THE_ONLY_LIFT]:
- `Equivalence<in A>`, `Order<in A>`, and `Predicate<in A>` all declare their carried type in CONTRAVARIANT position, so the comparator/order/predicate family is a `Contravariant` functor with no covariant `map` — `mapInput(self, (b: B) => A): Equivalence<B>` is `contramap`, the sole re-basing operator, pre-composing a field accessor onto a leaf instance to retarget it at the owner that contains the field.
- One leaf instance re-bases onto every projection that yields its type: `Order.mapInput(Order.number, (x) => x.rank)` and `Equivalence.mapInput(Equivalence.string, (x) => x.label)` lift the primitive instance to the struct without restating the comparison — a per-field comparator is the leaf instance plus an accessor, never a hand-written `(a, b) => a.rank - b.rank`.
- The `Contravariant.imap` (`imap(to, from)`) collapses to `contramap(from)` because the forward leg is unused on an input-only carrier; `Semigroup` is the one instance whose `imap` needs BOTH legs (`Semigroup.imap(to, from)`) since `Semigroup<A>`'s `combine` carries `A` in output position too — the variance of the carrier dictates whether the lift is `mapInput` or a two-legged `imap`, read off the `in`/`out` annotation.
- A `Refinement<A, B>` narrows under `mapInput` only when the accessor preserves the narrowing evidence; `Predicate.mapInput` returns a plain `Predicate<B>` (refinement evidence is dropped through the input remap), so a type-narrowing predicate is re-based at the owner via `Predicate.struct`/`Predicate.tuple` (which thread refinement through), never via `mapInput` which loses the `is` guard.

[STRUCT_FOLD_IS_THE_PRODUCT_TYPECLASS]:
- `Equivalence.struct`, `Order.struct`, `Semigroup.struct`, `Monoid.struct`, and `Predicate.struct` are the `Product` typeclass's `struct` instantiated per carrier — each takes `Record<string, Instance<any>>` and folds it into one `Instance<{ readonly [K]: extracted }>`, the per-field instance map being the value-level mirror of the field-schema map the type extractors read; `Struct.getEquivalence`/`Struct.getOrder` are these two pre-pinned to the struct shape, deriving a struct-wide `Equivalence`/`Order` from per-field instances, never hand-written field-by-field.
- The fold's result type is computed by the same `[R[K]] extends [Instance<infer A>] ? A : never` conditional-`infer` across all five carriers, so the derived instance's domain type is LIFTED from the field-instance record, never annotated — supplying `{ rank: Order.number, label: Order.string }` yields `Order<{ rank: number; label: string }>` with the field types recovered, the struct comparator's domain proven to match the field instances at compile time.
- `tuple(...elements)` is the positional dual — `Order.tuple(Order.number, Order.string)` derives `Order<readonly [number, string]>` mapping `{ [I in keyof T]: ... }` over tuple positions; `product(self, that)` is the binary primitive (`Order<readonly [A, B]>`) that `tuple` and `struct` are built from, so a two-key composite is `product`, an N-key positional one is `tuple`, an N-key named one is `struct`, one `Product` algebra spanning all arities.
- `Predicate.struct`/`Predicate.tuple` flip the result to a narrowing `Refinement` exactly when any arm is a `Refinement.Any` (`[Extract<R[keyof R], Refinement.Any>] extends [never]` selects the plain-`Predicate` branch), so a record of refinements folds into a single struct-narrowing guard whose output type is `{ [K]: Refinement.Out }` — the field-wise type narrowing is derived from the per-field guards, never a hand-written `u is { name: string; age: number }` after N `typeof` checks.

[ORDER_COMBINE_IS_THE_ORDERING_MONOID]:
- `Order.combine(primary, secondary)` is backed by the `Ordering` `Monoid` whose `combine` returns the LEFT-MOST non-zero `Ordering` with `empty = 0` — so combining orders is the lexicographic tie-break chain, `secondary` consulted only where `primary` returns `0`; a sort precedence list is `Order.combineAll([byStatus, byRank, byLabel])`, the multi-key comparator declared as the ordered instance list, never a nested `if (a.status === b.status) { ... }` cascade.
- `Order.empty()` is the always-equal `Order` (the `Ordering` `Monoid`'s identity lifted), so `combineAll` over an empty precedence list degrades to total-equality cleanly and a precedence chain built programmatically from a derived key list is total at arity zero — `Order` carries the `empty` identity while `Equivalence` exposes `combine`/`combineMany`/`combineAll` (conjunctive AND) with no `empty`, so an `Order` fold seeds from a derivable identity and an `Equivalence` fold over a possibly-empty axis list needs a `make(() => true)` seed supplied at the call.
- `Equivalence.combine(a, b)` ANDs two equivalences (both must agree), `Equivalence.combineMany(self, collection)` ANDs a self plus a collection — the conjunctive fold is the value-level "equal on every projected axis", so equality across a subset of fields is `Equivalence.combineAll([eqOnId, eqOnVersion])` reading each `mapInput`-rebased leaf, the multi-field equality predicate composed from leaves rather than spelled field-by-field.
- `Order.reverse(O)` flips the comparator (descending), `Order.mapInput(O, accessor)` re-bases it, and the two stack: `Order.combine(Order.reverse(Order.mapInput(Order.number, (x) => x.priority)), Order.mapInput(Order.string, (x) => x.label))` is "priority descending, then label ascending" as one derived instance, the loose multi-field sort body collapsed to a combinator chain over leaf orders.

[ORDER_YIELDS_THE_PREDICATE_FAMILY]:
- One `Order<A>` derives a whole comparison-predicate family by application: `Order.lessThan(O)`, `Order.greaterThan(O)`, `Order.lessThanOrEqualTo(O)`, `Order.greaterThanOrEqualTo(O)`, `Order.between(O)`, and the selectors `Order.min(O)`, `Order.max(O)`, `Order.clamp(O)` — six predicate/selector spellings hand-written off `<`/`>` collapse to applications of the single derived order, so a domain comparator authored once yields its strict/non-strict/range/extremum surface with no per-relation function.
- `Order.clamp(O)({ minimum, maximum })` and `Order.between(O)({ minimum, maximum })` take the lattice bounds as a record, so a refined-range clamp is the struct order plus its two endpoints; the range check and the clamping projection share one `Order` instance, the bound consulted twice from one declaration rather than two parallel guards.
- `Bounded<A>` is `{ compare: Order<A>; maxBound: A; minBound: A }` — an `Order` fused with its lattice extrema — and `Monoid.min(B)`/`Monoid.max(B)` derive a `Monoid<A>` whose `combine` is min/max and whose `empty` is the opposing bound (`min`'s identity is `maxBound`), so a "minimum over a collection" fold is `Monoid.min(Bounded).combineAll(values)` with a total seed, never a `reduce` seeded with `Infinity`.
- `Semigroup.min(O)`/`Semigroup.max(O)` derive the same extremum combine from a bare `Order` WITHOUT bounds — usable where no identity element exists (`combineAll` is unavailable, only `combine`/`combineMany`), so a non-empty extremum fold over a domain with no representable bound is the `Semigroup`, a bounded one promotes to the `Monoid` — the presence of `Bounded` versus `Order` selects whether the fold has a seed.

[PREDICATE_LOGIC_IS_THE_SEMIGROUP_OF_GUARDS]:
- `Predicate.and`/`Predicate.or`/`Predicate.not`/`Predicate.xor`/`Predicate.eqv`/`Predicate.implies`/`Predicate.nand`/`Predicate.nor` are the full Boolean algebra over guards, and `Predicate.every(collection)`/`Predicate.some(collection)` are the conjunctive/disjunctive folds whose Boolean reduction `Boolean.SemigroupEvery`/`SemigroupSome` carry as `Semigroup` instances — so a multi-condition admission test is `Predicate.every([isPositive, isFinite, mapInput(below(cap), size)])`, the AND-chain declared as a guard list rather than a `&&` expression that re-derives short-circuit per call site.
- `Predicate.and`/`or` over two `Refinement`s widen the output type structurally (`B & C` for `and`, `B | C` for `or`), so composing refinements narrows to the intersection/union with no manual `is` re-spelling; `Predicate.compose(refinementA, predicateB)` chains a narrowing guard with a follow-on check on the narrowed type — the second predicate operates on `Refinement.Out` of the first, the two-stage narrowing declared once.
- `Predicate.mapInput(guard, accessor)` re-bases a leaf guard onto the owner's field exactly as the `Order`/`Equivalence` lift does, so the predicate family shares the contravariant re-basing grammar with the comparator family — one accessor lifts a leaf instance regardless of which carrier (`Order`, `Equivalence`, `Predicate`) it threads, the value-level projection vocabulary unified across the three input-position carriers.

[VALUE_FOLD_MIRRORS_THE_TYPE_FOLD]:
- The type-level `Struct.Type<F>` / `Struct.Encoded<F>` mapped fold over a field-SCHEMA record and the value-level `Equivalence.struct` / `Order.struct` fold over a field-INSTANCE record are the same projection at two phases — both read one per-key record and emit one composite, so the static shape and the comparator that operates on it descend from records keyed identically, the field key set never authored twice.
- `Schema.equivalence(owner)` folds the SAME structural recursion over the schema AST that `Equivalence.struct` folds over an explicit instance map — the codec yields the equivalence with zero per-field wiring, so `Equivalence.struct` is reached only where the comparison axis diverges from the codec structure (a case-insensitive field, a tolerance-bounded number) and the leaf `mapInput` injects the custom comparison; matching the codec exactly is `Schema.equivalence`, diverging from it is the explicit struct fold.
- `Struct.evolve(obj, { key: fn })` is the value-level field-wise transform whose result type is computed (`ReturnType` per evolved key, unevolved keys pass through) — the runtime mirror of a per-field `Schema.transform` over a struct, applied where the transform is a plain projection needing no codec; a struct re-shaped by mapping a subset of its keys is `evolve` with the projected shape inferred, never an object literal restating every key.
- `Struct.pick(...keys)`/`Struct.omit(...keys)`/`Struct.get(key)` are the runtime value projections matching the schema-level pick/omit/pluck — one vocabulary spans the type derivation and the value derivation, so a sub-shape and the function that materializes it carry the same key set with no second key list.
- `Struct.keys(obj)` returns `Array<keyof T & string>` excluding symbol keys; `Struct.entries(obj)` returns typed `[key, value]` pairs likewise string-keyed — the symbol-key exclusion is the trap: a struct with a `unique symbol` key loses that key from both projections, so symbol-keyed fields are invisible to value-level iteration.

```typescript
import { Equivalence, Order, Predicate, pipe } from 'effect'
interface Row {
    readonly variant: '<variant-a>' | '<variant-b>'
    readonly rank: number
    readonly label: string
}
const byRow: Order.Order<Row> = Order.combineAll([
    pipe(Order.mapInput(Order.number, (r: Row) => (r.variant === '<variant-a>' ? 0 : 1))),
    Order.reverse(Order.mapInput(Order.number, (r: Row) => r.rank)),
    Order.mapInput(Order.string, (r: Row) => r.label),
])
const sameRow: Equivalence.Equivalence<Row> = Equivalence.struct({
    variant: Equivalence.string,
    rank: Equivalence.number,
    label: Equivalence.mapInput(Equivalence.string, (s: string) => s.toLowerCase()),
})
const admissible: Predicate.Predicate<Row> = Predicate.every([
    Predicate.mapInput(Order.greaterThan(Order.number)(0), (r: Row) => r.rank),
    Predicate.mapInput((s: string) => s.length > 0, (r: Row) => r.label),
])
const top: (rs: ReadonlyArray<Row>) => Row | undefined = (rs) => rs.toSorted(byRow).at(0)
```

[RECURSION_IS_THE_ONE_HAND_WRITTEN_TRIPLE]:
- `Schema.suspend<A, I, R>(f: () => Schema<A, I, R>)` returns `suspend<A, I, R> extends AnnotableClass<..., A, I, R>` — `infer` cannot recover a type through a deferred thunk, so the suspend node is the SOLE point in the lane where `<A, I, R>` is supplied explicitly; the recursive owner's triple is named once at the suspend call and every non-recursive field still derives, the annotation a fence around the one position the extractor cannot fold.
- The recursive owner is declared by interface-merging the extracted type with itself: a `Struct` whose self-referential field is `Schema.suspend((): Schema.Schema<Node> => Node)` references the very `type Node = Schema.Schema.Type<typeof Node>` it produces — the merged interface closes the cycle, the explicit suspend annotation pins the slot the self-reference would otherwise leave `any`, and `Type`/`Encoded` may diverge across the suspend exactly as on any field (a recursive `NumberFromString` child decodes per level).
- `Schema.Class<Self>(id)(fields)` extends `Schema<Self, Simplify<I>, R>` with `ast: AST.Transformation`, so `Self` IS the Type slot and the class is its own recursion anchor: a self-typed field `children: Schema.Array(Schema.suspend((): Schema.Schema<Tree> => Tree))` references the class identifier mid-declaration, the `Self` parameter supplying the forward type the suspend thunk returns — the class fuses value-constructor, codec, and recursion fixpoint with no parallel `interface Tree` mirror.
- Mutually-recursive owners thread the cycle through paired suspends: each owner's cross-reference field is `Schema.suspend((): Schema.Schema<Other> => Other)` with `Other`'s triple named explicitly, so a two-node grammar closes as two declarations whose suspends point at each other — the family is the cycle, not a flattened union, and `Arbitrary.make`/`JSONSchema.make` read the suspend's `identifier` annotation to emit a bounded generator and a `$ref` rather than diverging on the recursion.

[REACH_LIMIT_FENCE]:
- A conditional, mapped, or template-literal type — and every value-level fold, projector, and carrier extraction — earns its place only by deriving a family off one owner and deleting the members it would otherwise spell by hand: a standalone `is-*` predicate, a union-arithmetic utility, a hand-merged intersection, a field-by-field comparator or guard, or a type that demonstrates the type system without removing real members is rejected regardless of correctness; the proof is the construct changing a projection the default fold cannot reach, the leaf instances proving insufficient, the variant property being decoded somewhere.
- A derived type, composed sub-codec, projection result, or value instance is materialized standalone only at three consumers; below that the extraction lives inline at its single use (`Schema.Schema.Type<typeof owner>` at the one call site, `Types.ExtractTag<Owner, "x">`, `WithResult.Success<typeof Req>` at the one handler, `Arbitrary.make(owner)` at the one test, `xs.toSorted(Order.mapInput(Order.number, (x) => x.rank))` at the one sort) — a named alias for a one-shot derivation is the alias-to-the-derivation hop the doctrine deletes, the owner re-read at each site, the extractor the only spelling.
- A computed type earns its place only by extracting a family off a carrier the owner genuinely extends; reaching for `WithResult.Success` on an owner with no `symbolWithResult` slot yields `never`, not an error, so a projection that silently collapses to `never` is the signal the carrier is absent — the extraction must name a trait the declaration actually carries, never a phantom the type system would fabricate.
- `Context` extracts covariantly while `Type`/`Encoded` extract invariantly, so a derived type that tries to widen the decoded shape through assignment of a supertype owner is rejected at the phantom — the invariant `_A`/`_I` slots are the structural fence, a "demonstrate variance" type that does not delete an enumerated family failing this carrier discipline before it fails the three-consumer threshold.
- The brand modifier `Schema.brand('Name')` adds nominal identity to `Type` (`A & Brand<'Name'>`) leaving `Encoded` plain — a derivation, not a parallel branded type; a brand with no runtime predicate behind it and no real nominal collision is deleted, the primitive standing on its own.
- A `compose` chain earns its place only by collapsing a multi-step wire transform into one round-trippable owner that deletes the intermediate map functions; a compose whose intermediate shape has no domain meaning and exists only to demonstrate the chain is the rejected register — the join must consume a real boundary stage.
- A transform whose `encode` cannot reconstruct the source is not composed — the codec demands a total inverse, so an extraction that genuinely discards information is a one-directional `Schema.transformOrFail` with an explicit `encode` that rebuilds a canonical witness, the lossy-projection temptation refused at the type level rather than at a broken round-trip later.
- A variant set earns its place only when its projections delete real hand-spelled shapes (the insert/update/select/wire DTO family a persisted entity carries); a `Field` whose `Config` lists every variant with the same schema is the bare schema written verbosely — drop the wrapper and the unconditional field survives every projection by default, so a `Field` is authored only where at least one variant differs in presence or schema, and a variant axis introduced without a downstream consumer of each projection is the rejected register.
- A projector override earns its slot only by correcting a derivation the AST fold gets structurally wrong (a refined range the default generator cannot infer, a native carrier with no structural shape); an `arbitrary`/`pretty`/`equivalence` slot restating what the codec already yields is the hand-written generator the projection deletes, rejected even when correct.
- A comparison, equality, generation, or rendering concern that reaches a service is NOT an instance fold or a projector slot: `Equivalence<A>`/`Order<A>`/`Predicate<A>`/`LazyArbitrary<A>`/`(a: A) => string` carry no `R`, so a uniqueness check against a store, an order requiring a fetched policy, or a generation needing a layer is a `Schema.filterEffect`/`transformOrFail` on the codec widening `Context`, never smuggled into a comparator or a projector slot — the value-instance algebra and the secondary families stay effect-free by carrier definition.
- A custom `Equivalence`/`Order` hand-written with `make` is admitted only where the comparison is genuinely non-structural (a domain tolerance, a canonical-form collapse); a `make` that re-implements field-by-field what `struct` folds from leaves is the fold spelled longhand, rejected — the proof is the leaf instances being insufficient, forcing `make`, not convenience.
