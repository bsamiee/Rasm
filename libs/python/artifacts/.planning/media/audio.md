# [PY_ARTIFACTS_MEDIA_AUDIO]

The temporal-artifact AUDIO-stream encode arm. `_encode_audio` encodes a sample-block sequence into a single-stream audio container over the FFmpeg floor `av` (PyAV) bundles in its wheel — `av.open(sink, mode="w", format=)` mints the `OutputContainer`, `MediaProfile.voiced` mints the typed `AudioStream`, `AudioFrame.from_ndarray(block, format="s16", layout=)` lifts each producer sample block, `AudioFifo.write`/`AudioFifo.read(frame_size)` rebuffers a variable-size producer block to the encoder's required fixed `codec_context.frame_size` (codecs such as AAC require exact frame sizes), and the shared `stream.encode(frame)` -> `container.mux(packets)` loop drives the sequence with the `stream.encode(None)` flush at end-of-stream. This page composes the `media/video#MEDIA` container owner — the shared `Media`/`MediaOp`/`MediaProfile`/`MediaEvidence`/`ContainerFormat` family, the `_drive`/`_flush`/`_probe` worker primitives, and the `_FRAME_FORMAT`/`_SAMPLE_FORMAT` vocabulary — and adds only the `AudioFrame`/`AudioFifo`/resample audio-stream encode worker plus the `MediaProfile.voiced` audio-stream-configure projection the `EncodeAudio` arm reads; it re-owns no container, profile, evidence, or content-key vocabulary. The `EncodeAudio` case is one `MediaOp` arm dispatched at `media/video#MEDIA`'s `_mux` `match`, worked here by the `_encode_audio` module-level function across the same `anyio.to_process.run_sync` subprocess lane, the muxer discriminated by the typed `ContainerFormat` value the profile carries. The whole encode runs cp315-core in-process: `av` is the abi3-cp311 markerless wheel that covers cp315 with delocate-vendored FFmpeg (no system binary), so the synchronous `add_stream`/`encode`/`mux` C loop is the one CPU-bound body the runtime drives over the subprocess lane, the finished container bytes the only payload crossing back. This arm contributes the single `ArtifactReceipt.Media` case the `media/video#MEDIA` container owner also contributes, never a parallel audio-receipt rail. This page closes the AUDIO half of the `MEDIA` idea.

```python signature
import io

import av
import numpy as np
from numpy.typing import NDArray

from artifacts.media.video import MediaEvidence, MediaProfile, _SAMPLE_FORMAT, _drive, _flush


def _encode_audio(blocks: tuple[NDArray[np.int16], ...], profile: MediaProfile) -> tuple[bytes, MediaEvidence]:
    sink = io.BytesIO()
    with av.open(sink, mode="w", format=profile.container.value) as container:
        stream = profile.voiced(container)
        fifo = av.AudioFifo()
        index = 0
        for block in blocks:
            fifo.write(av.AudioFrame.from_ndarray(block, format=_SAMPLE_FORMAT, layout=profile.layout))
            while (chunk := fifo.read(stream.codec_context.frame_size, partial=False)) is not None:
                _drive(container, stream, chunk, index, profile.rate)
                index += 1
        if (tail := fifo.read(partial=True)) is not None:
            _drive(container, stream, tail, index, profile.rate)
        _flush(container, stream)
    blob = sink.getvalue()
    return blob, MediaEvidence.measure(profile.container, profile.codec, float(len(blocks)), len(blocks), int(profile.bit_rate or 0), blob)
```
