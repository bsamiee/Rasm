# HAMT Physical Identity Layer

[FIVE_BIT_FRAGMENT_IS_THE_ROUTING_ATOM]:
- `SIZE = 5`, `MASK = BUCKET_SIZE - 1 = 31`, and `hashFragment(shift, hash) = hash >>> shift & MASK` slice one 5-bit window out of the hash per trie level — the descent reads the hash five bits at a time, low window first, so a value's bucket route is a base-32 numeral of its hash and the hash's job is to spread the LOW bits, never to be globally unique.
- `toBitmap(frag) = 1 << frag` turns a 5-bit fragment into a single set bit in a 32-wide bitmap, and `fromBitmap(mask, bit) = popcount(mask & (bit - 1))` is the dense-array index: the count of occupied slots below this fragment's bit. The trie stores only occupied children in a packed array and recovers each child's physical index by Hamming-weighting the bitmap prefix — so a 32-way branch costs one machine integer of mask plus a children array sized to live entries, never a 32-slot sparse vector.
- The routing is purely structural and never consults `Equal.equals`: `IndexedNode` descent is `mask & toBitmap(frag)` to test occupancy then `children[fromBitmap(mask, bit)]` to step, and only a terminal `LeafNode`/`CollisionNode` ever fires the comparator — so the cost of locating a key is `popcount` + array index per level, and `Equal.equals` is reached exactly once per lookup at the leaf, never per level.

[THE_GATE_RUNS_ONCE_AT_THE_LEAF_NOT_PER_LEVEL]:
- The bedrock gate `Hash.hash(self) === Hash.hash(that) && self[Equal.symbol](that)` describes the within-bucket settle; the physical layer reveals the bucket is a trie PATH, not a flat list — `getHash` walks `IndexedNode`/`ArrayNode` by fragment with zero comparisons, hits one `LeafNode`, and returns `Equal.equals(key, node.key) ? node.value : Option.none()`. A single mismatched leaf is the whole bucket: a hash collision in the low 5 bits that diverges at a deeper level routes the two keys to different leaves and they never meet a comparator.
- A `LeafNode` carries its own full `hash` field captured at insertion, but `getHash`'s `LeafNode` arm does NOT re-check `hash === node.hash` — it goes straight to `Equal.equals(key, node.key)`. The trie path already proved the fragments matched down to this leaf; the residual high-bit hash difference is irrelevant because a sole leaf in a slot means no other key routed there. The hash precondition the bedrock gate states is discharged STRUCTURALLY by the descent for the single-leaf case, and re-asserted explicitly only inside `CollisionNode`.
- This is why a fast-wrong `Equal` is a membership bug and a fast-wrong `Hash` is a routing bug with different blast radii: a wrong `Equal` corrupts exactly the leaf and collision arms (a stored key reports absent or a foreign key reports present), while a wrong `Hash` mis-ROUTES — the probe descends a different fragment path and lands on `EmptyNode`/`Option.none()` without the comparator ever running, so the value is unreachable rather than miscompared.

[THIRTY_TWO_BIT_CEILING_AND_THE_SHIFT_MODULO_TRAP]:
- The hash routes through seven `hashFragment` windows, not six: `shift` advances by `SIZE` per level over `0, 5, 10, 15, 20, 25, 30`, and the seventh window at `shift = 30` reads only the top two bits (`hash >>> 30` is a 0..3 value), so the trie addresses the full 32 bits as six clean 5-bit branches plus one degenerate 4-way branch. `Hash.optimize` is NOT a non-negativity clamp — it preserves bit 31 and duplicates it into bit 30 (`n & 0xbfffffff | n >>> 1 & 0x40000000`), so a fully-set hash optimizes to a fully-set hash; its role is the V8 SMI fast-integer tag, and every one of the 32 bits is genuine routing entropy because `hashFragment` reads through `>>>`, which is unsigned and indifferent to the sign bit.
- The engine masks every shift count to five bits, so a hypothetical `hash >>> 35` would silently become `hash >>> 3` and re-read already-consumed low bits — the descent provably never reaches that wrap because `mergeLeavesInner` converts depth exhaustion into a `CollisionNode` the instant two keys share a full 32-bit hash (`new CollisionNode(edit, h1, [n2, n1])` when `h1 === h2`), and two keys differing anywhere in the 32 bits diverge no later than the `shift = 30` window. The ceiling is therefore full-hash equality at depth seven, never a fragment overflow, so the modulo trap is structurally unreachable rather than guarded.
- Consequence for hash width: every bit of a 32-bit hash routes, but the top two bits route at coarse 4-way granularity from the seventh level alone, so discriminating power concentrated in bits 30-31 fragments far weaker than the same power in the low 30 bits — two keys agreeing on bits 0-29 share a six-level path and only split at the degenerate window, so a hand-rolled `[Hash.symbol]` whose entropy lives high produces deep shared prefixes and long packed arrays along the way. Spread the discriminating bits LOW, where the trie branches 32 ways, not high, where it branches four.

[COLLISION_DENSITY_IS_THE_REAL_COST_SURFACE_NOT_HASH_INEQUALITY]:
- `CollisionNode` holds `children: Array<LeafNode>` all sharing one full 32-bit hash, and both its read path (`getHash`'s `CollisionNode` arm, which re-asserts `hash === node.hash` before scanning) and its write path (`updateCollisionList`) are a LINEAR `Equal.equals` scan over that array — the moment two distinct-by-`Equal` keys produce the same hash, every subsequent `get`/`set`/`has` touching that bucket degrades from O(popcount-per-level) trie navigation to O(bucket-size) comparator sweeps. Collision count, not hash disagreement, is the term that dominates lookup cost.
- The law: a hash WIDER than its equality (hashing fields the comparator ignores) does not merely waste cycles — it changes the fragment path the probe routes down and reports a stored key absent (the bedrock scatter hazard, here located at the per-level routing step); a hash NARROWER than its equality (ignoring fields the comparator distinguishes) packs distinct keys into one `CollisionNode` and pays a full linear `Equal` scan on every bucket touch. The two failure modes are the two directions of the equal-implies-hash-equal obligation reflected onto the physical trie: scatter is a correctness break, over-collision is a complexity break, and the safe grain is a hash exactly as discriminating as the equality.
- Collision is not pathological input alone — it is the unavoidable consequence of a 32-bit address at scale: the birthday bound makes a shared 32-bit hash near-certain past tens of thousands of distinct keys regardless of hash quality, so a content-addressed structure that grows large WILL form `CollisionNode`s, and the design that survives it keeps the per-bucket `Equal.equals` cheap (a discriminant-first comparator that fails on the first field) rather than assuming collisions never happen. A `Data` value whose `[Equal.symbol]` walks every key before reaching the discriminating one turns each collision-bucket scan into a multiplied structural walk.

[NODE_PROMOTION_IS_A_DENSITY_THRESHOLD_NOT_A_DEPTH_ONE]:
- The trie swaps representations by occupancy, not by key count: an `IndexedNode` (bitmap + packed array) promotes to a dense `ArrayNode` (32-slot direct-indexed) the instant its children reach `MAX_INDEX_NODE = 16` via `expand`, and an `ArrayNode` packs back to an `IndexedNode` when removals drop it to `MIN_ARRAY_NODE = 8` via `pack` — a hysteresis band between 8 and 16 that prevents thrashing at the boundary. The bitmap+popcount indirection is paid only while a level is sparse; a half-full level abandons the bitmap and indexes the fragment directly.
- `ArrayNode` lookup is `children[hashFragment(shift, hash)]` — no `popcount`, no bitmap test, a single direct array read — so a hot dense level is faster per step than a sparse one, inverting the usual intuition that fuller is slower. The structure pays its routing cost where keys are sparse and spends a flat array where they are dense, so the per-level cost is bounded by `MAX_INDEX_NODE` regardless of total size.
- `IndexedNode` removal collapses a two-child node back to a bare sibling leaf (`children.length <= 2 && isLeafNode(children[indx ^ 1])` returns `children[indx ^ 1]`) and `CollisionNode` collapses a single-survivor list back to a `LeafNode` (`list.length > 1 ? new CollisionNode(...) : list[0]`) — the trie continuously re-flattens so that a key's depth tracks the LIVE contention on its fragment path, never the historical maximum. A bucket that once collided and then thinned returns to single-`Equal`-at-leaf cost with no rebuild.

[MERGELEAVES_BUILDS_THE_DESCENT_ONLY_TO_THE_DIVERGENCE_BIT]:
- Inserting a second key into an occupied leaf does not pre-allocate a full-depth path: `mergeLeaves` loops `mergeLeavesInner` extending the shift by `SIZE` per level only while the two hashes' fragments AGREE (`subH1 === subH2` returns a deferred `child => new IndexedNode(...)` thunk pushed on a stack), and the instant they diverge it builds one two-child `IndexedNode` and unwinds the stack of single-child wrappers from the divergence point back up. The trie depth for any pair is exactly the length of their shared fragment prefix plus one — minimal, never the depth-seven floor unless the hashes are genuinely equal.
- The ordering of the two leaves inside the divergence node is `subH1 < subH2 ? [n1, n2] : [n2, n1]` — children are stored in ascending-fragment order so `fromBitmap`'s `popcount` prefix-count yields the correct dense index, making the physical array order a function of the fragment value, not insertion order. Two keys inserted in either order produce the identical node shape, so the trie is a canonical structure of its key set and structural equality of two `HashMap`s built by different insertion sequences holds at the node level.
- The equal-hash terminal case is the ONLY place `mergeLeavesInner` produces a `CollisionNode` (`h1 === h2`), and it does so without any further fragment inspection — once two keys share the full 32-bit hash there is no bit left to fragment on, so the merge short-circuits straight to a comparator-keyed bucket. This is the structural witness that collisions are a hash-space property fixed at `optimize` time, never a depth the trie could descend its way out of.

```typescript
import { Data, Hash, HashSet } from 'effect';

// the leaf comparator the trie fires per bucket: discriminant first, then the cheapest
// distinguishing field — so a CollisionNode's linear Equal scan fails fast, never walking
// every key before reaching the one that differs
class Leaf extends Data.Class<{ readonly slot: number; readonly tag: string; readonly payload: string }> {}

// a hash whose entropy lives LOW, where the trie branches 32 ways: slot drives routing, tag breaks
// within-collision ties — reject a hash concentrating entropy in bits 30-31 (a coarse 4-way split
// only at depth seven) or one narrower than the Equal grain (forces CollisionNode linear scans)
const route = (l: Leaf): number => Hash.combine(Hash.hash(l.tag))(Hash.hash(l.slot));

const dense: HashSet.HashSet<Leaf> = HashSet.fromIterable(
    Array.from({ length: 24 }, (_, i) => new Leaf({ slot: i, tag: '<t>', payload: `<p-${i}>` })),
);
const distinct: number = HashSet.size(dense);
const present: boolean = HashSet.has(dense, new Leaf({ slot: 7, tag: '<t>', payload: '<p-7>' }));
const routed: number = route(new Leaf({ slot: 7, tag: '<t>', payload: '<p-7>' }));
```

[STRUCTURAL_REGION_COLLAPSES_THE_TRIE_TO_ONE_BUCKET]:
- The bedrock test-only `structuralRegion` forces `Hash.hash` to return `0` for every input; the physical consequence is total: every fragment at every shift is `0 >>> shift & 31 = 0`, so every key routes down the identical leftmost path and the trie degenerates to a single `CollisionNode` (or a single-leaf chain) where membership is a pure linear `Equal.equals` sweep. Inside the region a `HashSet` IS an unordered list with O(n) membership — correct, because the region's purpose is to compare raw decoded shapes by structure without honoring any hash, and the all-zero hash is exactly the routing-disabling switch that makes that uniform.
- This bounds the region's authority precisely and exposes a routing trap: `getHash`/`has` re-hash the PROBE on every call, so a lookup executed inside the region against a trie built outside hashes the probe to `0` and descends the leftmost fragment path — it locates the real bucket only if that bucket happened to route through fragment `0` at every level, otherwise it lands on `EmptyNode` and reports a stored key absent. The region neutralizes the routing layer for whoever is inside it, never for the persisted trie shape, which is why interior domain code never enters it: the all-zero hash is a test-only comparison regime, and crossing a real `HashSet` membership query through it silently mis-routes.

[CONTENT_ADDRESSING_REQUIRES_THE_FUSED_PROTOTYPE_AT_THE_LEAF]:
- The trie's correctness is entirely downstream of the value carrying `[Hash.symbol]` and `[Equal.symbol]`: `Hash.hash` falls back to `random(self)` — a per-reference `WeakMap` integer — for any non-`Hash` object, so a plain literal dropped into a `HashSet` routes by a STABLE-but-reference-keyed fragment path and its content-equal twin routes elsewhere, landing in a different leaf and reporting absent. The bedrock "fusion is the precondition" rule is, physically, the requirement that the leaf's stored `hash` field and the probe's hash be CONTENT-derived so they share a fragment path; reference-random hashing makes every distinct reference its own leaf and defeats content dedup at the routing layer, before `Equal` is ever consulted.
- `EffectPrototype` carriers (services, layers, streams) hash via `random` by design, so a `HashSet` of them is a reference-identity set whose trie routes each handle to its own leaf for its lifetime — content collapse is structurally impossible there not because the comparator refuses it but because two handles never share a fragment path. The structural-versus-reference identity choice the bedrock fixes at the prototype is, in trie terms, the choice between a content-canonical trie and a reference-canonical one, decided entirely by which `[Hash.symbol]` the leaf value carries.
