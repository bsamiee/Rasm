# [PY_ARTIFACTS_API_PSD_TOOLS]

`psd-tools` is the in-process PSD/PSB authority for the artifacts layered-raster rail: a single `PSDImage` document root that `open`s a `.psd`/`.psb` from any path/stream/bytes, `new`s or `frompil`s a blank canvas, exposes the whole layer tree through the `GroupMixin` container algebra (`descendants`/`find`/`findall`/`append`/`insert`), discriminates each node over the closed `Layer` family (`PixelLayer`/`Group`/`TypeLayer`/`ShapeLayer`/`SmartObjectLayer`/`AdjustmentLayer`/`FillLayer`), authors structure through `create_pixel_layer`/`create_group`/`PixelLayer.frompil`/`Group.new`, reads per-layer pixels/masks/effects/text through `topil`/`numpy`/`composite` + `Mask`/`Effects`/`SmartObject`, and `save`s the round-tripped document back to PSD bytes. The package owner composes `PSDImage`, the `Layer` polymorphic tree, the `Compression`/`BlendMode`/`ColorMode`/`ChannelID` constant vocabulary, and the `pil_io`/`numpy_io` exchange seam into the `export/layered` `.psd`-native channel-stack engine and the upstream inspect/extract path; it never re-implements the PSD record parser, the layer compositor, the RLE/ZIP channel codec, or the PIL/NumPy bridge psd-tools already owns, and it never re-authors the IFC/semantic models the C# boundary holds.

## [02]-[PUBLIC_TYPES]

[PUBLIC_TYPE_SCOPE]: document root + the closed layer family
- rail: layered

- rail: layered

[ENTRYPOINT_SCOPE]: pixel/mask/effects/text egress + the PIL/NumPy seam
- rail: layered

`composite` is the blend-faithful render: it walks the (sub)tree applying each layer's `BlendMode`/`opacity`/`clipping`/`mask`/`effects` and returns a flattened `PIL.Image.Image` (on `PSDImage` it is the whole document; on `Layer`/`Group` it is that node), with `viewport`/`layer_filter`/`apply_icc`/`force`/`ignore_preview` knobs — this is the authority over a naive layer-by-layer paste. `topil`/`numpy` extract a single node's stored channels (no blend), `channel`-selectable (`'color'`/`'shape'`/`'alpha'`/`'mask'` or a `ChannelID`). `thumbnail`/`has_thumbnail`/`has_preview` recover the embedded merged preview. `Mask.topil`, `Effects.find`/`items`, `TypeLayer.text`/`engine_dict`/`warp`, and `SmartObject.open`/`save`/`filetype` are the side-channel egress. The `pil_io`/`numpy_io` module functions are the lower seam the methods sit on, callable directly for batch conversion.

## [04]-[IMPLEMENTATION_LAW]

[LAYERED_PSD]:
- import: `from psd_tools import PSDImage` plus `from psd_tools.constants import BlendMode, Compression, ColorMode, ChannelID` and `from psd_tools.api.layers import Layer, Group, PixelLayer, TypeLayer, ShapeLayer, SmartObjectLayer` at boundary scope only; module-level import is banned by the manifest import policy. No libvips/JRE/native gate exists — the one abi3 RLE extension binds on cp315, so a missing-import here is a packaging fault, never a host-capability gate.
- open axis: `PSDImage.open(fp, max_alloc_bytes=...)` is the single decode factory across path/stream/bytes/`PathLike`; PSD vs PSB is discriminated internally and surfaced on `version`, never a per-format reader type. `max_alloc_bytes` is load-bearing for an untrusted blob — it caps the bomb allocation and is the gate the `PSDDecompressionWarning`/`PSDLargeImageWarning` ride; admit those warnings as a structured evidence signal, not a silent pass.
- tree axis: the layer tree is the `GroupMixin` algebra on `PSDImage`/`Group` — iterate/index for direct children, `descendants(include_clip=...)` for the recursive flatten, `find`/`findall` for name lookup; each node's concrete kind is read from `Layer.kind` and the seven `Layer` subclasses, never an `isinstance` ladder reconstructed per call and never a parallel per-kind reader. The container is also the mutation surface (`append`/`insert`/`extend`/`remove`/`pop`/`clear`), so re-parenting and re-ordering stay one algebra.
- author axis: `create_pixel_layer`/`create_group` mint-and-attach in one call, `PixelLayer.frompil`/`Group.new`/`group_layers` mint detached then attach, and `move_to_group`/`move_up`/`move_down`/`delete_layer`/`lock`/`unlock` own relocation/lifecycle — a manual `LayerRecord`/`ChannelDataList` construction is rejected where these factories exist. `opacity`/`blend_mode`/`compression`/`top`/`left` are constructor rows (the `BlendMode`/`Compression` enum members), never post-hoc record surgery.
- composite axis: `PSDImage.composite`/`Layer.composite` is the blend-faithful flatten — it honors each layer's `BlendMode`, `opacity`/`fill_opacity`, `clipping`/`clip_layers`, `mask`, and `effects`, with `viewport`/`layer_filter`/`apply_icc` knobs; this is the authority over a layer-by-layer `Image.paste` that discards blend and clipping. Rasterizing a vector `ShapeLayer` fill inside `composite` needs the optional `composite` extra (`aggdraw`/`scipy`/`scikit-image`); a pixel-only document composites without it.
- pixel axis: `topil(channel=...)`/`numpy(channel=...)` extract ONE node's STORED channels (no blend) keyed by `'color'`/`'shape'`/`'alpha'`/`'mask'` or a `ChannelID`; the `apply_icc` flag runs the embedded-`ICC_PROFILE` post-process, never a naive `convert('RGB')` that drops the profile. `thumbnail`/`has_thumbnail` recover the embedded merged preview as a cheap proxy when a full `composite` is not needed.
- side-channel axis: `Mask.topil`/`create_mask`/`update_mask`/`remove_mask` own layer masks; `Effects.find`/`items` enumerate the layer styles (`DropShadow`/`OuterGlow`/`ColorOverlay`/`Stroke`/`BevelEmboss`/`Satin`); `TypeLayer.text`/`engine_dict`/`warp`/`transform` expose live text + the type-engine descriptor; `SmartObject.open`/`save`/`filetype`/`resolution` extract or re-emit the embedded/linked asset (`open` is a context manager). Each is reached off its owning layer, never a re-parse of the raw tagged block.
- save axis: `PSDImage.save(fp)` re-emits the whole tree to PSD/PSB bytes; per-channel `Compression` (RAW/RLE/ZIP/ZIP_WITH_PREDICTION) is the author-time codec row, never a parallel writer. The `ZIP`/`ZIP_WITH_PREDICTION` channel codec is the seam where the universal compression substrate (`imagecodecs`/`zlib-ng`) can back the deflate path; RLE rides the bundled `_rle.abi3.so`.
- color-model axis: `ColorMode` (RGB/CMYK/LAB/GRAYSCALE/DUOTONE/MULTICHANNEL/INDEXED/BITMAP) is the document model the pub/print plane reads — a CMYK or LAB PSD round-trips and composites in its native space; the `pil_io.get_pil_mode(color_mode, alpha)` map is the canonical mode bridge, not an ad-hoc string guess.
- fault axis: parse/IO failure raises ordinary exceptions (attrs/struct/IO); the two `UserWarning` subclasses are non-fatal size signals. Lift both to the layered-export fault rail once at the boundary, never let a raw decode exception escape into domain logic.
- evidence: each PSD op captures the document `version` (PSD/PSB), `color_mode`, `depth`, `size`/`viewbox`, the layer count + per-layer kind/name/bbox/blend/opacity/visibility, ICC presence (`ICC_PROFILE`), thumbnail presence, the per-channel `Compression`, and output byte length as a layered receipt.
- boundary: psd-tools owns in-process PSD/PSB parse, inspect, blend-faithful composite, structural authoring, and round-trip — it is the `.psd`-native channel-stack authority for `export/layered`; `PhotoshopAPI` owns the native high-throughput PSD/PSB writer where raw write speed dominates (the categorical-best heavy writer), `psdtags`+`tifffile` own only the Photoshop-compatible layered-TIFF container, `pyvips`/`pillow` own the raster pixel edge feeding `frompil`/`create_pixel_layer`, and the C# `Rasm.Bim` graph owns IFC/semantic models — psd-tools never re-authors any of those.

[STACK_INTEGRATION]:
- universal `numpy` tier (`libs/python/.api/numpy.md`): the host pixel seam is `Layer.numpy(channel=...)`/`PSDImage.numpy()` out and PIL-via-`numpy` in — `numpy_io.get_array`/`get_layer_data` return canonical `uint8`/`uint16`/`float32` band arrays so a `pyvips`/`scikit-image`/`matplotlib`-produced raster becomes a layer (`Image.fromarray(arr)` -> `create_pixel_layer`) and an extracted PSD layer leaves to the array tier without a bespoke pixel struct; prefer `numpy()` over `topil().__array__()` for any real band math.
- universal `pillow` seam (folder `.api/pillow.md`): `topil`/`composite` egress and `frompil`/`create_pixel_layer`/`create_mask` ingress are PIL `Image.Image`, so Pillow is the exchange currency between psd-tools and every other raster owner — a `pyvips` pipeline result (`write_to_memory` -> `Image.frombytes`) becomes a PSD layer, and a composited PSD becomes a Pillow image for the document/figure owners, never a re-encoded file round-trip.
- compression tier (`libs/python/data/.api/numcodecs.md`, folder `.api/zlib-ng.md` / `.api/imagecodecs.md`): the `Compression.ZIP`/`ZIP_WITH_PREDICTION` per-channel codec is the deflate seam — the same managed compression substrate the layered/TIFF arm uses backs the channel ZIP path, so the PSD write codec and the TIFF/ORA container codec share one compression owner instead of each binding a private zlib.
- universal `expression` tier (`libs/python/.api/expression.md`): the boundary maps a parse/decode/save failure to `Result[LayeredReceipt, LayeredError]` — a malformed-PSD or bomb-exceeded failure is the typed failure arm, a successful `save`/`composite` yields the `Ok` receipt; the `try/except` over the psd-tools exceptions + the two `UserWarning`s (caught via `warnings.catch_warnings`) lives only in the boundary adapter, never in the domain pipeline.
- universal `beartype`/`msgspec` tier (`libs/python/.api/beartype.md` / `.api/msgspec.md`): the `BlendMode`/`Compression`/`ColorMode` enums and the layer-spec rows (`name`/`top`/`left`/`opacity`/`blend`/`compression`) are the typed authoring contract a `msgspec.Struct` layer descriptor carries and `beartype` guards at the boundary, so the `export/layered` `Layer` row decodes into a psd-tools `create_pixel_layer` call with no stringly-typed blend/codec.
- universal `anyio` tier (`libs/python/.api/anyio.md`): `composite`, `save`, and per-layer `topil`/`numpy` are CPU-bound native work, so the boundary owner drives one document per `anyio.to_thread.run_sync` worker (a `CapacityLimiter`-bounded fan over many PSDs), and only the finished `save` bytes / composited PIL image cross back to the async caller — never an unbounded thread pool over the GIL-releasing RLE/ZIP codec.
- universal `structlog`/`opentelemetry` tier (`libs/python/.api/structlog.md` / `.api/opentelemetry-api.md`): the per-op evidence (version, color_mode, depth, layer count + per-layer kind/blend/opacity, ICC/thumbnail presence, channel `Compression`, byte length) is the structured event/span payload; the `psd_tools.__version__` is read once at boundary init so it rides the receipt as a deployment fact.
