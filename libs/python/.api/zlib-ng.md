# [PY_DATA_API_ZLIB_NG]

`zlib-ng` (the `python-zlib-ng` project) binds the SIMD-accelerated zlib-ng C library as a drop-in replacement for the stdlib `zlib` and `gzip` modules, supplying the accelerated DEFLATE/gzip codec band of the data compression rail beside `zstandard`/`lz4`/`brotli`. It exposes three import surfaces: the `zlib_ng.zlib_ng` C extension (a byte-for-byte mirror of the stdlib `zlib` API — `compress`/`decompress`/`compressobj`/`decompressobj`/`adler32`/`crc32` — plus the modern `_ZlibDecompressor` bytes-feeding decompressor and the threading-internal `_ParallelCompress` block codec), the `gzip_ng` module (a drop-in for stdlib `gzip` with `GzipNGFile`/`open`/`compress`/`decompress`), and `gzip_ng_threaded` (a GIL-escaping multi-threaded gzip reader/writer that fans block compression across worker threads). The codec is the legacy-interop and gzip-container member of the payload-discriminated compression band: it owns the DEFLATE/zlib/gzip wire formats (RFC 1950/1951/1952) that downstream consumers and stdlib peers already speak, while `zstandard` owns archival, `brotli` owns `Content-Encoding: br` and WOFF2, `lz4` owns hot-path block compression, and `py7zr`/`stream-zip` own archive containers — one codec discriminant selects the rail, never parallel owners, and `zlib-ng` is chosen exactly when the bytes must round-trip with stdlib `zlib`/`gzip` at accelerated throughput. It never re-implements the DEFLATE codec, gzip framing, or the CRC/Adler checksums the native core already owns.

## [04]-[IMPLEMENTATION_LAW]

[COMPRESSION_ZLIB_NG]:
- import: `from zlib_ng import zlib_ng`/`gzip_ng`/`gzip_ng_threaded` at boundary scope only; module-level import is banned by the manifest import policy.
- drop-in law: `zlib_ng.zlib_ng` is a faithful mirror of stdlib `zlib` and `gzip_ng` of stdlib `gzip` — same function names, same signatures, same constants, same byte output. The substitution is `from zlib_ng import gzip_ng as gzip` / `from zlib_ng import zlib_ng as zlib`; existing `zlib`/`gzip` call sites accelerate with zero code change and the bytes round-trip with any stdlib peer. zlib-ng is chosen exactly when DEFLATE/zlib/gzip interop is required at accelerated throughput; a payload with no legacy-format constraint routes to `zstandard` (archival) or `lz4` (hot path) instead.
- framing axis: one DEFLATE codec, three wire framings selected by `wbits` — zlib (RFC 1950, `+MAX_WBITS`), raw DEFLATE (RFC 1951, `-MAX_WBITS`), and gzip (RFC 1952, `wbits=31` to encode, `+32` to auto-detect on decode). Never mint a per-format codec type; the `wbits` row discriminates. The compress-side framing is a boundary policy the decode side must match (or auto-detect for gzip), never assumed.
- modality axis: one-shot (`compress`/`decompress`), incremental zlib-style (`compressobj`/`decompressobj` minting `_Compress`/`_Decompress` with `flush(mode)`/`unused_data`/`unconsumed_tail`), and bytes-feeding decode (`_ZlibDecompressor` with `needs_input`/`eof`, the `BufferedReader`-friendly decoder) are all functions/factories on the one C extension, never parallel codec types. Pick the modality by input shape: a full buffer one-shots; a chunked stream drives `compressobj.compress`/`flush(Z_SYNC_FLUSH)`; a `BufferedReader`-fed decode uses `_ZlibDecompressor`.
- level axis: `level`/`compresslevel` is the standard zlib 0..9 range — `Z_BEST_SPEED` (1), `Z_BEST_COMPRESSION` (9), `Z_DEFAULT_COMPRESSION` (-1, library default). zlib-ng's acceleration is throughput at the same ratio band, NOT a reduced isal-style 0..3 range (the `GzipNGFile.__init__` docstring's "0 to 3" claim is a copy-paste artifact from a sibling project; the live `compress` docstring and `_COMPRESS_LEVEL_BEST = Z_BEST_COMPRESSION = 9` are authoritative). `strategy` (`Z_FILTERED`/`Z_HUFFMAN_ONLY`/`Z_RLE`/`Z_FIXED`) and `memLevel` tune the matcher on `compressobj`.
- checksum axis: `adler32`/`crc32` are rolling checksums (chain via the `value` arg across chunks) bit-identical to stdlib `zlib`; `crc32_combine(crc1, crc2, len2)` folds independently-computed CRCs of adjacent segments into the whole-stream CRC — the primitive that lets a parallel or chunked pipeline compute per-block CRCs and combine them without a serial re-scan. For content-identity/cache keys prefer the `xxhash` XXH3 digests (see `xxhash.md`); reserve `crc32` for gzip-trailer integrity and `zlib`-format interop.
- gzip-container axis: `gzip_ng` owns the gzip file format — `GzipNGFile` (binary file object, every `gzip.GzipFile` method except `truncate`), `open` (path/fileobj, binary or text), and one-shot `compress`/`decompress` with `mtime`. `BadGzipFile`/`FTEXT`/`FNAME`/`FHCRC`/`FEXTRA`/`FCOMMENT` are the re-exported stdlib gzip exception and header-flag bits. Use `gzip_ng` for any `.gz` file or gzip-framed stream; use bare `zlib_ng` for raw/zlib-framed DEFLATE without the gzip container.
- parallel axis: `gzip_ng_threaded.open(..., threads=N, block_size=B)` is the GIL-escaping path — the writer partitions the stream into `block_size` blocks and fans each through one of N `_ParallelCompress` workers (`compress_and_crc` returns block-bytes + CRC in a single GIL-released call), recombining per-block CRCs into the gzip trailer; the reader decodes single-threaded (gzip is not splittable). Choose threaded gzip for large streamed writes where the compressor is the bottleneck; `threads=0` transparently degrades to the single-threaded `gzip_ng.open`. Streaming only — no seek. Never hand-roll the block partition / CRC-combine / worker pool; `_ParallelCompress` + `crc32_combine` already own it.
- integration: lower a canonical `msgspec.msgpack.encode(...)` (or `msgspec.json.encode`) payload through `gzip_ng.compress(blob, compresslevel=...)` for a self-describing gzip artifact, or through `zlib_ng.compress(blob, wbits=-15)` for a raw block inside an outer envelope; stamp an `otel` span (see `opentelemetry-api.md`) with `level`, input/output byte lengths, `ZLIBNG_VERSION`, and the gzip `mtime`/CRC, and route the inbound leg `gzip_ng.decompress` → `msgspec.Decoder` under a `stamina` `retry_context` (see `stamina.md`) scoped to transport faults only — a `zlib_ng.error`/`BadGzipFile` corrupt-stream fault is terminal and never retried. For a large streamed sink, wrap the downstream writer in `gzip_ng_threaded.open(sink, "wb", threads=-1)` so block compression escapes the GIL and back-pressure stays in the `FlushableBufferedWriter`, not an in-RAM buffer; under `anyio` (see `anyio.md`) run the blocking threaded writer via `anyio.to_thread.run_sync` so the structured-concurrency scope owns cancellation. For many small records sharing structure, pass a preset `zdict=` on `compressobj`/`decompressobj` (pinned on both sides) rather than re-deriving per record.
- evidence: each codec call captures `level`, `wbits`/framing, input/output byte lengths, `crc32`/`adler32` (or the gzip trailer CRC), `threads` (for the parallel path), `ZLIBNG_VERSION`, and `mtime` as a compression receipt; the parallel writer additionally records `block_size` and the worker count.
- boundary: `zlib-ng` owns the accelerated DEFLATE/zlib/gzip codec (RFC 1950/1951/1952) and stdlib-`zlib`/`gzip` interop; Zstandard archival routes to `zstandard`; `Content-Encoding: br` and WOFF2 route to `brotli`; hot-path block compression to `lz4`; archive containers to `py7zr`/`stream-zip`; content-identity digests to `xxhash`; failures surface as `zlib_ng.error`/`BadGzipFile`, never as raw return codes; live UI stays outside this package. The underscore-prefixed runtime classes (`_Compress`/`_Decompress`/`_ZlibDecompressor`/`_ParallelCompress`/`_GzipReader`/`_ThreadedGzipWriter`) are minted by the public factory/open functions, never imported or constructed by name.

[RAIL_LAW]:
- Package: `zlib-ng`
- Owns: accelerated DEFLATE/zlib/gzip one-shot, incremental, bytes-feeding, and multi-threaded compression/decompression; Adler-32/CRC-32 checksums with `crc32_combine`; stdlib-`zlib`/`gzip` drop-in interop
- Accept: a level-and-framing-tuned codec feeding the compression owner — `zlib_ng` one-shot `compress`/`decompress` with explicit `level`/`wbits`, `compressobj`/`decompressobj` streaming with `strategy`/`zdict`/`flush(mode)`, `_ZlibDecompressor` bytes-feeding decode with `needs_input`/`eof`, `adler32`/`crc32`/`crc32_combine` checksums, `gzip_ng` drop-in (`GzipNGFile`/`open`/`compress`/`decompress` with `mtime`), and `gzip_ng_threaded.open(threads=, block_size=)` GIL-escaping parallel gzip — selecting modality by input shape and framing by `wbits`
- Reject: wrapper-renames of `compress`/`decompress`; a parallel codec type per framing where a `wbits` row suffices; an isal-style 0..3 level assumption (zlib-ng is standard 0..9); an in-RAM accumulation where `compressobj`/`gzip_ng_threaded` already streams; a hand-rolled block partition / CRC-combine / worker pool where `_ParallelCompress` + `crc32_combine` apply; `xxhash`-grade content-identity work pushed onto `crc32`; raw-DEFLATE bytes on a gzip channel or vice versa; an ambient/global preset dictionary instead of a both-sides-pinned `zdict`; direct import or construction of the underscore-prefixed runtime classes; routing this codec where a non-legacy payload should reach `zstandard`/`lz4`/`brotli`
