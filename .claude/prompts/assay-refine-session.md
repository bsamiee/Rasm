## [2] DOCTRINE

- **One shape per concept.** Extend tagged variants, enum members, or catalog rows instead of adding parallel types, aliases, factories, or dispatch arms.

## [3] SURFACE MAP

Map the current surface: definitions, structs, constants, literals, types, enums, external libraries, and the capabilities actually leveraged. Use that map as the substrate for all refinement.

## [4] DIMENSION I — CODE QUALITY

Targets:

- Collapse residual surface into denser polymorphic shapes.
- Replace weak or unbounded types with precise enums, annotated atoms, protocols, and typed absence.
- Move imperative leaks into folds, match expressions, or data-driven dispatch where it improves clarity.
- Root-shrink suppressions to the irreducible minimum.
- Strip dead vestiges and fix hidden fragile logic.

Judge output by fewer concepts, stronger types, deeper library leverage, zero new agent-facing surface, and preserved capability.

A new capability lands as one polymorphic surface and one touch per concern.

## [7] METHODOLOGY

Run each dimension as comprehension, critique, synthesis, implementation, adversarial review, and targeted fixing. The value filter is strict: collapse beats addition, inherent beats flag, one shape beats many, and a written reason kills bloat.
