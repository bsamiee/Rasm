# Carrier-Polymorphic Dispatch

[CARRIER_ARROW]:
- One arrow signed `K<F, A>` owns every monad modality at once: the same function body serves `Fin`, `Option`, `Eff`, `IO`, and `Validation` because the carrier `F` is a constrained type parameter, not a concrete return. A per-carrier copy of a pipeline (`Fin<T> ProcessFin(...)`, `Eff<RT,T> ProcessEff(...)`) is the rejected form — the carrier becomes a type parameter `where F : Monad<F>, Fallible<F>` and the four copies collapse to one definition.
- The trait stack is additive: each capability the body uses adds one constraint, and the constraint set is the precise capability contract. A pipeline that binds, fails, and recovers signs `where F : Monad<F>, Fallible<F>`; one that also chooses an alternative adds `Choice<F>`; one that aggregates adds `Alternative<F>`. The signature is the proof of which effects the body can perform — over-constraining (`where F : Monad<F>, Fallible<F>, Choice<F>, Alternative<F>, Traversable<F>`) on a body that only binds is dead surface that narrows the admitted carrier set for no reason.
- A `K<F, A>` result is consumed by `.As()` to recover the concrete carrier exactly once at the materialization boundary; interior composition stays in `K<F, _>`. Calling `.As()` mid-pipeline forces an early concretization that defeats the polymorphism — the cast belongs only where a concrete instance method or a non-trait consumer requires it.

[FALLIBLE_AXIS]:
- The error type is itself a carrier dimension. `Fallible<E, F>` declares `static abstract K<F, A> Fail<A>(E error)` and `static abstract K<F, A> Catch<A>(K<F, A> fa, Func<E, bool> Predicate, Func<E, K<F, A>> Fail)` — error type `E` and carrier `F` vary independently. `Fallible<F> : Fallible<Error, F>` is the fixed-`E` specialization for the standard error hierarchy; a domain carrier with a custom typed error implements `Fallible<MyError, F>` and inherits the same `|`/`Catch` algebra over `MyError` with no per-error wiring.
- The `|` operator family on `K<F, A>` is generated once for all `Fallible<F>` carriers: `operator |(K<F,A>, CatchM<E,F,A>)`, `operator |(K<F,A>, Fail<E>)`, and `operator |(K<F,A>, Error)`. Any carrier implementing `Fallible<F>` inherits the full recovery-chaining surface — a `static readonly CatchM<Error, F, A>` policy value applies via `|` against every fallible carrier without a monad-specific overload.
- `Catch` exists in three recovery shapes per selector: `Func<Error, K<F,A>>` (recover into the carrier), `Func<Error, Error>` (remap the error, still failing), and `Func<Error, A>` (recover to a pure value, gated `where F : Applicative<F>`). The selector itself has three forms — `Func<Error,bool>` predicate, `Error` value-match, and `int Code` — so error-code routing is a first-class arm: `fa.Catch(404, _ => fallback)` matches on `Error.Code` with no predicate lambda. The recovery-to-value form requires `Applicative<F>` because lifting `A` into `F` needs `Pure`; a `Monad`-only constraint cannot express it.

[COPRODUCT_DISPATCH]:
- The carrier-polymorphic discriminated-union dispatch primitive is `Coproduct<F>`, which derives its whole surface from one primary: `static abstract C Match<A,B,C>(Func<A,C> Left, Func<B,C> Right, K<F,A,B> fab)`. Every secondary arm — constant-left `Match(C, Func<B,C>, fab)`, constant-right `Match(Func<A,C>, C, fab)`, `IfLeft`, `IfRight` in both function and constant forms — is a `static virtual` default expressed through the one abstract `Match`. One implementation yields the entire eight-member dispatch family; re-authoring the constant overloads on an implementor is redundant.
- `CoproductK<F>` is the lifted twin: its `Match<A,B,C>` returns `K<F,A,C>` rather than naked `C`, keeping the result inside the carrier for further binding. The choice between `Coproduct` (eliminate to a value) and `CoproductK` (transform one branch, stay in the carrier) is the discriminant between terminal dispatch and continuing composition — both derive their default arms from the same single abstract `Match`.
- Construction is the dual minimal surface: `CoproductCons<F>` declares only `static abstract K<F,A,B> Left<A,B>(A)` and `static abstract K<F,A,B> Right<A,B>(B)`. Two introduction forms and one elimination form span the entire bifunctor algebra; `Partition<FF,A,B>(K<FF, K<F,A,B>>)` over a `Foldable` container is itself a `static virtual` default that folds each coproduct through `Match` into a `(Seq<A> Lefts, Seq<B> Rights)` split — bulk separation derives from the same primary with no bespoke loop.

[NATURAL_TRANSFORM]:
- A carrier change is one arrow, not a rebuild. `Natural<out F, in G>` declares `static abstract K<G, A> Transform<A>(K<F, A> fa)` — a structure-preserving map from carrier `F` to carrier `G` for every `A` simultaneously. Lowering `EitherT<L,M>` to `OptionT<M>`, or `Validation` to `Fin`, is a single `Transform` call rather than a per-element reconstruction; the payload `A` is untouched.
- `CoNatural<in F, out G>` is the reverse-variance arrow: `static abstract K<F, A> CoTransform<A>(K<G, A> fa)`. The variance flip means `Natural` lowers into a wider carrier and `CoNatural` lifts from a narrower one; pairing them on a carrier states an isomorphism witness. Hand-written `Match`-and-rebuild bridges between two carriers are the rejected form — the bridge is one of these two arrows.

[CHOICE_ALGEBRA]:
- Two distinct binary algebras over `K<F, A>` must not be conflated. `Choice<F>` declares `static abstract K<F,A> Choose<A>(K<F,A> fa, K<F,A> fb)` — left-biased first-success, surfaced as `|`. `SemigroupK<F>` declares `static abstract K<F,A> Combine<A>(K<F,A> lhs, K<F,A> rhs)` — monoidal concatenation. For `Option` the two coincide; for list-shaped carriers `Choose` returns the first non-empty while `Combine` appends both — selecting `Choose` where `Combine` was meant silently discards the right operand's content.
- The aggregate forms have different floors. `oneOf<F,A>(params K<F,A>[])` requires `Alternative<F>` — not merely `Choice<F>` — because the empty-array case must produce `Empty<A>()`, and only `Alternative<F> : Choice<F>, Applicative<F>` carries the `Empty` member. A varargs choice over a possibly-empty set is `Alternative`-floored; a strictly binary `a | b` needs only `Choice`. `MonoidK<F> : SemigroupK<F>` is the parallel split on the concatenation axis: `Combine` from the semigroup, `Empty` from the monoid.
- The second `Choose` overload, `Choose<A>(K<F,A> fa, Memo<F,A> fb)`, is the lazy alternative: the right branch is deferred. `Memo<F,A>` (built by `memoK<F,A>(Func<K<F,A>>)`) computes its value at most once and only when `fa` fails, giving genuine short-circuit semantics across the carrier. The eager `Choose(fa, fb)` evaluates both operands before choosing; reaching for the `Memo` overload is the only way to skip an expensive alternative when the primary succeeds.

```csharp
// One arrow, all fallible carriers; recovery is a value the operator threads.
static K<F, B> Run<F, A, B>(K<F, A> input, Func<A, K<F, B>> step, Memo<F, B> spare)
    where F : Monad<F>, Fallible<F>, Choice<F> =>
        input.Bind(step)
             .Catch(404, _ => F.Fail<B>(Error.New("absent")))   // code-routed arm
             .Choose(spare);                                      // lazy left-biased fallback

// Coproduct dispatch derives every arm from the one primary Match.
static K<F, C> Fold<F, A, B, C>(K<F, A, B> fab, Func<A, C> onL, C onR)
    where F : CoproductK<F> =>
        F.Match(onL, onR, fab);   // constant-right arm is a static-virtual default
```

[REQUEST_UNION_THREADING]:
- A verb family collapses to one request `[Union]` whose total `Switch` returns a single carrier-polymorphic arrow. The dispatch site signs `K<F, TResult> Dispatch<F>(Request req) where F : Monad<F>, Fallible<F>` and the union's generated `Switch` selects the per-case body; the carrier stays a free parameter so the same dispatcher serves the synchronous `Fin` consumer and the effectful `Eff` consumer. Returning a concrete `Fin<TResult>` from the dispatcher is the form that forces a parallel `Eff` dispatcher to exist — the collapse is incomplete until the carrier is a parameter.
- `[UnionSwitchMapOverloadAttribute(StopAt = ...)]` additively generates a `Switch` overload that treats a named sub-hierarchy as one leaf: the handler is a single `Action<FailureCase>`/`Func<FailureCase,T>` that does not descend into derived cases, alongside the always-generated exhaustive overload. A dispatch site that delegates an entire failure sub-tree to one carrier-polymorphic recovery arrow uses the stop-at overload so the leaf structure stays opaque to the caller; the exhaustive overload remains available where the full case set must be addressed.
- The discriminant is recovered from the value, never re-passed beside it. A frozen `FrozenDictionary<TKind, Func<Request, K<F, TResult>>>` keyed on the union's discriminant — built with `ProjectionEqualityComparer<Request, TKind>` so payload differences collapse to one row — is the table form of the same dispatch; the carrier-polymorphic arrow is the stored value and the lookup yields the complete per-case behavior. A parallel `params` array of case-correlated overrides passed beside the request is arity smuggled back into a knob and is rejected: the override belongs on the case, recoverable by `Switch`, not in a positional sidecar.
