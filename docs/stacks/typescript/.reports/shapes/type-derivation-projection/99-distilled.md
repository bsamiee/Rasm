# Type Derivation And Projection

[EXTRACTION_IS_INFER_OVER_PHANTOM_CARRIERS_THAT_COEXIST_ON_ONE_OWNER]:
- Every extractor is `S extends Carrier<infer ...> ? slot : never` over a phantom carrier whose body brands the slots, so it reaches every owner extending the carrier — `Class`, `Struct`, `Union`, `suspend`, `transform` — without one re-stating them; an owner extends SEVERAL independent branded carriers each with its own `infer` namespace, the trait the extractor names selecting which disjoint family it lifts: `Variance` (codec triple `Type`/`Encoded`/`Context`), `Serializable` keyed on `symbolSerializable` (self-serialization schema), `WithResult` keyed on `symbolWithResult` (success/failure pair), `Context.Tag` (service identity, `Service<T>` lifting `T['Service']`).
- The `Variance` slot variances are asymmetric: `_A`/`_I` are `Invariant` so `Type`/`Encoded` extract EXACTLY (a subtype owner never widens its decoded type at a seam), `_R` is `Covariant` so `Context` extracts UP the requirement lattice — an owner requiring `R1 & R2` surfaces `R1 | R2`, the union one `Layer` satisfies, the covariance the reason a transform reaching a service widens only `Context` and breaks no consumer's `Type`/`Encoded`.
- `Serializable.{Type,Encoded,Context}` drive `Schema.serialize`/`deserialize` (the persisted projection, never the structural codec, so a value whose wire form differs from memory carries both triples no parallel DTO); `WithResult.{Success,SuccessEncoded,Failure,FailureEncoded,Context}` are five extractors off one `{ success; failure }` slot `Schema.deserializeExit` reconstructs as `Exit<SA, FA>`, and `SerializableWithResult.Context<P> = SR | RR` `|`-folds the two carrier requirements, never conflating them into one `R` the root under-provides — the success/failure pair edited once re-derives the whole `Exit` reconstruction off `symbolWithResult`, no parallel response/error DTO authored.
- Accept: a derived type extracting a family off a carrier the owner genuinely extends. Reject: `WithResult.Success` on an owner with no `symbolWithResult` slot — it yields `never`, never an error, so a projection silently collapsing to `never` is the signal the carrier is absent; a "demonstrate variance" type widening the decoded shape through supertype assignment, fenced by the invariant `_A`/`_I` slots before the three-consumer threshold. Boundary: the type extractors erase, their value-level mirror `typeSchema`/`encodedSchema`/`encodedBoundSchema` each a live `SchemaClass` that re-decodes; materialize standalone only at three consumers, below that `Schema.Schema.Type<typeof owner>` lives inline, the owner re-read, the extractor the only spelling.

```typescript
class Fault extends Schema.TaggedError<Fault>('Fault')('Fault', { key: Schema.String }) {}
class Op extends Schema.TaggedRequest<Op>('Op')('Op', {
    payload: { key: Schema.NonEmptyString, depth: Schema.Int.pipe(Schema.optionalWith({ default: () => 1 })) },
    success: Schema.Struct({ ids: Schema.Array(Schema.UUID) }),
    failure: Fault,
}) {}
type Payload = Schema.Serializable.Type<Op>            // { key; depth } — codec triple off symbolSerializable
type WireErr = Schema.WithResult.FailureEncoded<Op>    // { _tag: 'Fault'; key } — wire failure leg off symbolWithResult
type NeedR = Schema.SerializableWithResult.Context<Op> // SR | RR fused, one Layer at the root
const exit: Effect.Effect<Exit.Exit<Schema.WithResult.Success<Op>, Fault>, ParseResult.ParseError, NeedR> =
    Schema.deserializeExit(new Op({ key: '<value-a>' }), '<wire>')
```

[ONE_FOLD_ENGINE_PARAMETERIZED_BY_ITS_CARRIER_IS_EVERY_PROJECTOR]:
- `Compiler<A> = (ast, path) => A` and `Match<A> = { [K in AST["_tag"]]: (node, compile, path) => A }` are the lane's whole structural-recursion machinery: `getCompiler(match)` ties the open recursion into a closed compiler and threads it back into every arm as `compile`, so each node folds its children by the same compiler, the fixpoint supplied, never re-opened per node.
- The codec, `Pretty.match` (`A = (a: A) => string`), the JSON-schema fold (`JsonSchema7`), the arbitrary fold (`LazyArbitrary<A>`), and `Schema.equivalence` (`Equivalence<A>`) are FIVE carriers over ONE `Match` shape and ONE `getCompiler` recursion — not parallel walkers — so adding a node kind extends every projector's obligation through the mapped-type key set; the arm map is keyed on `AST["_tag"]`, omitting an arm a missing-key error, the totality every projector inherits.
- A projector's PHASE is selected by which AST rewrite it folds, never a flag: the document fold runs over `encodedAST`, the generation fold over `typeAST`, so the document-versus-generation divergence is a structural consequence of disjoint sub-graphs — a numeric-string field documents `{ type: 'string' }` yet generates a `number`, so a document-conformant fixture requires feeding the generator output back through `Schema.encode(owner)`, the phase mismatch a trap not a defect.
- Accept: a bespoke `Match<ColumnSpec>` deriving a real new family (a SQL-column plan, a form descriptor, a binary layout) off the owner's AST, reusing every leaf annotation and inheriting totality and the `path` locus. Reject: a `Match` re-emitting what a shipped projector already yields, or a hand-walked `if (ast._tag === ...)` tree. Reject: reading or rebuilding `.ast` directly where `typeAST`/`encodedAST`/`encodedBoundAST`/`pick`/`omit`/`partial`/`mutable` already express the rewrite — descend to the AST to FOLD it, never to reconstruct a node a combinator constructs.

```typescript
import { Record, Schema, SchemaAST as AST } from 'effect'
const tags = ['Literal', 'StringKeyword', 'NumberKeyword', 'BooleanKeyword', 'Refinement', 'TupleType', 'Suspend', 'Transformation'] as const
const depth: AST.Compiler<number> = AST.getCompiler({
    ...Record.fromEntries(tags.map((t) => [t, () => 0] as const)),                                          // every leaf arm closed — total over AST['_tag']
    TypeLiteral: (node, compile, path) => 1 + node.propertySignatures.reduce((m, p) => Math.max(m, compile(p.type, [...path, p.name])), 0),
    Union: (node, compile, path) => 1 + node.types.reduce((m, t) => Math.max(m, compile(t, path)), 0),      // recursion threads compile, never re-dispatches
} as AST.Match<number>)
const owner = Schema.Struct({ tag: Schema.Literal('<variant-a>'), inner: Schema.Struct({ leaf: Schema.Number }) })
const nesting = depth(owner.ast, [])   // 2 — one Match table, path accumulated, no parallel walker
```

[THE_SEVEN_SLOT_PROPERTYSIGNATURE_IS_THE_FIELD_ALGEBRA_WHOSE_PRECONDITIONS_FIX_THE_STACK_ORDER]:
- The whole field-modifier vocabulary — `optionalWith`, `fromKey`, `propertySignature`, `withConstructorDefault`, `withDecodingDefault`, `withDefaults`, the four `optionalTo*` transforms — produces not parallel field kinds but WRITES into one carrier `PropertySignature<TypeToken, Type, Key, EncodedToken, Encoded, HasDefault, R>`: every modifier is `PropertySignature -> PropertySignature` (or a `Schema -> PropertySignature` lift) rewriting a named slot subset and returning the same carrier. The two `Token` slots are INDEPENDENT free bits, not one optionality flag, and three mapped folds read three different slots — `Struct.Type<F>` only `TypeToken`, `Struct.Encoded<F>` only `EncodedToken`, `Struct.Constructor<F>` reads `TypeToken` OR `HasDefault` — so a field's presence in the decoded record, the wire record, and the constructor input are three separate bit reads; `Key` is a three-state axis read positionally: `never` (lifted, un-renamed), a literal (renamed, locked), `PropertyKey` (the wildcard before any decision).
- `optionalWith<S, Options>` is the option set EVALUATED, never a code-path switch: `TypeToken = Has<Options, 'as' | 'default'> extends true ? ':' : '?:'`, `EncodedToken` unconditionally `'?:'`, the `Type` slot reshaped by `as`/`default`/`exact` and the `Encoded` slot by `nullable`/`exact` — the trap being each knob touches whichever side its semantics belongs to, never both symmetrically; `OptionalOptions` is a six-arm union rejecting `default` with `as` at the argument before any slot computes.
- Modifiers do not commute; their token preconditions force one legal order and two contradict exactly when those preconditions disagree: `withDecodingDefault` DEMANDS both tokens `'?:'` and `HasDefault` `false`, flips `TypeToken` to `':'`, so applying it to an already-required carrier (a `propertySignature` lift, an `optionalWith({ default })` result) is a compile rejection not a silent no-op; `withConstructorDefault` accepts any `HasDefault` and overwrites (idempotent, order-free, the safe terminal write) while `withDecodingDefault` accepts only `false` and refuses a second, the contradiction one-sided because a decode default flips a token that cannot flip twice. `withDefaults({ constructor, decoding })` fuses both in one rewrite, and `Exclude<Type, undefined>` in every default's output is the residue — the value materializes so `undefined` cannot survive the decoded side while `EncodedToken` stays `'?:'`. The legal stack is recoverable from the signatures: a raw schema is lifted before `fromKey` (which needs a `Key` slot), a token-flipping default precedes any modifier requiring the flipped state, the compiler walking the precondition chain the way the codec walks the decode.
- Reject: chaining `withDecodingDefault` onto `withConstructorDefault` — the rejection is always a TOKEN mismatch, never a `HasDefault` clash, the irreversibility encoded as a token flippable once. Reject: a `brand` modifier reaching for a `PropertySignature` slot — it is a `Schema -> Schema` combinator intersecting `Brand<B>` onto `Type` and leaving `Encoded` plain, so a branded field is branded FIRST then lifted (`Number.pipe(brand('Id'), propertySignature, fromKey('<wire>'))`), the nominal identity riding the value type the `Type` slot transports through every token flip; a brand with no runtime predicate and no real collision is deleted. Boundary: the four `optionalTo*` transforms carry a full `Schema -> Schema` so their VALUE slots may diverge where a default only flips the Type token — reached when the missing case must COMPUTE a value, the defaults when it is a constant.

```typescript
import { Schema } from 'effect'
const Field = Schema.String.pipe(
    Schema.propertySignature,                            // <':', string, never, ':', string, false, never> — both-required identity
    Schema.fromKey('<wire-key>'),                        // rewrites Key only — a second fromKey is rejected: a literal Key fails the PropertyKey wildcard precondition
    Schema.withConstructorDefault(() => '<value-a>'),    // rewrites HasDefault only — the one order-free modifier
)
const Row = Schema.Struct({ field: Field })
type Decoded = Schema.Schema.Type<typeof Row>            // { readonly field: string } — Type token ':' required
type Wire = Schema.Schema.Encoded<typeof Row>            // { readonly '<wire-key>': string } — renamed, Encoded token ':'
const made = Row.make({})                                // field omittable — HasDefault true the only slot the constructor read
```

[SUBSCHEMA_DERIVATION_FORMS_A_TWO_FLOOR_ONE_WAY_LATTICE]:
- The projection family is a lattice with two floors and an irreversible descent: the UPPER floor is the field-record owner (`Struct<Fields>`, carrying `.fields` and the method `.pick`/`.omit`), the LOWER the triple owner (`SchemaClass<A, I, R>`, codec only); a result re-stating the field RECORD stays upper and admits a next projection, a result projecting only the type maps has descended and cannot climb back, so the field map is capital the upper form conserves and the lower spends.
- The same verb sits on BOTH floors under one name decided by SYNTAX, not argument order: `self.pick(...)` (method, `Struct` result, chainable) versus `Schema.pick(...)(self)` (free, `SchemaClass` result, terminal) — so a chain that must continue past a `pick` is written method-form even where the free form reads terser. `partial`/`partialWith`/`required`/`keyof` have NO struct-method form — presence-flip is a token rewrite over the already-extracted map the field record cannot represent, so the missing method IS the type system encoding that flip exits the lattice; the order law follows: every field-set surgery that must chain is placed BEFORE every presence-flip and key-extraction.
- The enumerated descent map: `pick`/`omit` stay `Struct` via method, fall to `SchemaClass` via free form. `partial`/`required`/`keyof`/`attachPropertySignature` always lower. `pluck`/`getNumberIndexedAccess` fall BELOW the triple to a leaf — doubly terminal (no field record for a `pick`, a scalar `Type` for which `keyof` is meaningless) — the closure's absorbing state placed LAST (`Schema.Array(Owner.pipe(Schema.pluck('id')))`). `extend` returns the dedicated `extend<>` interface carrying NEITHER `.fields` NOR `.pick`, intersecting `Type`/`Encoded` while UNIONing `Context`, terminal and a left-fold over the addend list.
- `Schema.extend` is total at the type level (`extends Schema.Any`) but PARTIAL at construction — a struct-with-struct extend succeeds only where every overlapping field-name's leaf pair itself supports extension, the relation recursing into the overlap (two number refinements extend, two structs whose own overlaps extend), a leaf collision on incompatible primitives a thrown AST-merge error the type does not pre-flag; the supported catalog (struct-with-struct overlap-compatible, struct-with-index-signature, struct-with-union-of-supported, refinement/suspend of a struct, struct-with-struct-TRANSFORM only where the transform's `from`/`to` sides share NO field name with the target) is the merge's closure boundary, the no-overlap rule the sharpest because the merge cannot align a renamed/defaulted field with a same-named target without an ambiguous double-write, `fromKey` the escape moving the wire key out of collision.
- Reject: the free `Schema.pick` mid-chain then re-listing the field record by hand to re-enter the lattice — the alias-to-the-derivation the closure deletes; the method form conserves the record, spending it before the chain ends is the defect. Reject: `extend` to bolt one field onto a struct a `Struct.pick`-narrowed sibling already carries — the field added beside the owner, the lost `.fields` the price a single wider `Struct` declaration avoids. Boundary: re-entry into the rich class-owner runs ONLY through a record-conserving form — `Schema.Class(Owner.pick(...))` (a `Struct`) or `Schema.Class(Owner.pipe(Schema.filter(p)))` (a refinement carrying `RefineSchemaId` the `HasFields = Struct<Fields> | { [RefineSchemaId]: HasFields }` argument peels to), `Schema.Class(Schema.partial(Owner))` not type-checking; `Schema.asSchema` is the only widening re-flattening an `extend`/`compose` stack back to a plain inspectable triple, never back to a `Struct`.

```typescript
import { Schema } from 'effect'
const Owner = Schema.Struct({ id: Schema.UUID, label: Schema.NonEmptyString, secret: Schema.Redacted(Schema.String) })
const Public = Owner.omit('secret').pick('id', 'label')           // Struct -> Struct -> Struct: field-carrying, chain stays open
type Keys = keyof typeof Public.fields                            // 'id' | 'label' — .fields survives, the key set re-derives
class Sub extends Schema.Class<Sub>('Sub')(Owner.pick('id')) {}   // Struct.pick stays a Struct -> HasFields admits it
// class Dead extends Schema.Class<Dead>('Dead')(Schema.pick('id')(Owner)) {}   // REJECT: SchemaClass is neither Struct nor RefineSchemaId-bearing
```

[TRANSFORM_INVERTIBILITY_IS_A_THREE_RUNG_LATTICE_THE_TOTAL_ENCODE_ENFORCES]:
- `Schema.transform` declares the encoded↔decoded correspondence as one bidirectional total pair where the strict `decode` lands on `Encoded<To>` (NOT `Type<To>`): the composite is `to.decode ∘ userDecode ∘ from.decode`, the seam a RE-DECODE point — a malformed far-side value fails inside `to`, never at the seam, so the body is total over wire SHAPE and the refinement total over wire VALUE, the two obligations split. `decode` receives `(fromA, opts, ast, fromI)` — both decoded source and raw encoded input — so it branches on the wire form while computing from the decoded form; `transformOrFail` is the fallible form returning `Effect<_, ParseIssue, R>` with `ParseResult.succeed`/`fail`/`fromOption`, never a thrown error.
- Every transform sits on one rung the type cannot name but the codec enforces, recoverable from the SHAPE of `To`: ISO (mutual inverses — a key rename, a literal-pair table; flipping `from`/`to` yields the dual for free), SECTION (`decode ∘ encode = id` on a refined canonical subset, `encode ∘ decode` collapsing the equivalence class — `Trim` types `To` as `Trimmed`, `clamp` as the in-range filter, the refinement the encode-target witness making re-presentation total), and the REJECTED retraction-without-witness (a `decode` discarding information with no total `encode`). When the loss is ARITY, the witness is `Option` or a singleton rebuild: `head` is `transform<S, OptionFromSelf<...>>`, `pluck` rebuilds the single-key record — the smallest structure whose decode is the projection's left-inverse.
- Composite invertibility is the lattice MEET of the legs, never improved by the join: `ComposeTransformation` carries no functions, so iso∘iso stays iso, iso∘section drops to section, fidelity degrading monotonically; the SEAM carries its own rung independent of the legs and which `compose` overload type-checks is the witness — exact-equality alignment is iso, the subtype-narrowing overload (`B extends Encoded<To>`) a section down-casting the wider side at the join, `{ strict: false }` joining on `unknown` a seam with NO fidelity proof, the reach-limit marker never a default.
- `transformLiterals([0,'idle'],[1,'busy'],[2,'done'])` is the one TOTAL-ISO family by construction (each pair a closed finite bijection) — adding an enum code one tuple entry the `Union` absorbs and the inverse derives from the same pair, never a parse switch plus a serialize switch in lockstep, the arity-one `transformLiteral` the lone member; the iso holds only while BOTH projected literal columns are injective, two distinct `from` mapping to one `to` silently making the union a section. The layered string codecs compose the same way: `parseJson(inner)` is `transform<SchemaClass<unknown, string>, S>` pre-composing `JSON.parse` with `inner`, `split('/')` composed with `Schema.Array(NumberFromString)` a path-of-numbers codec re-decoding each segment, `NumberFromString`/`DateTimeUtc` themselves pre-composed `transformOrFail` owners — so a numeric column in a JSON string is `parseJson(Struct({ n: NumberFromString }))`, the `JSON.parse`/`parseFloat`/`string.split(...).map(Number)` reflexes deleted at once; `TemplateLiteralParser(...params)` is the dual of the `TemplateLiteral` validator whose `Type` is the parsed TUPLE (`'id-42-open'` to `readonly ['id', 42, 'open']`, the substrate compiling the node to a capturing `RegExp`).
- Reject: a `decode` lowercasing toward a bare `SchemaClass<string>` not `Lowercased` — the `encode` then has `string` on both sides with no canonical witness, so `decode ∘ encode` re-lowercases and an equality check diverges, the refinement on `To` the idempotence invariant. Reject: an `encode` that throws, returns an unrelated placeholder, or widens to `strict: false` to dodge the inverse — the codec's refusal of a one-way transform is the design forcing the author to NAME the witness, a genuinely lossy extraction either pushed onto `To` as a refinement (the section) or left as a post-decode projection with no round-trip claim.

```typescript
import { Schema } from 'effect'
const Slug = Schema.transform(
    Schema.String,
    Schema.Lowercased.pipe(Schema.pattern(/^[a-z0-9-]+$/)),    // To IS the canonical subset — the encode-target witness
    {
        decode: (s) => s.toLowerCase().replace(/[^a-z0-9]+/g, '-'),   // lossy: many inputs collapse to one slug
        encode: (slug) => slug,                                       // total + canonical: output already in the refined image
        strict: true,
    },
)
type Wire = Schema.Schema.Encoded<typeof Slug>   // string — every representable string
type Norm = Schema.Schema.Type<typeof Slug>      // the lowercased-pattern subset, the encode codomain
```

[REFINEMENT_AND_TRANSFORM_ARE_ONE_CONTEXT_PROPAGATION_RULE_NOT_TWO]:
- `Schema.filter(predicate, annotations)` refines `Type` in place without changing `Encoded`, the predicate returning `FilterOutput` — `boolean | string | undefined | ParseIssue | FilterIssue` (or an array for multi-issue) where `{ path, message }` carries the failure locus, `undefined`/`true` passing; `filterEffect(f)` is the service-reaching refinement, structurally a `transformOrFail<S, SchemaClass<Type<S>>, FD>` whose `To` re-states `Type<S>`, threading `FD` into `Context` exactly as a transform leg does — refinement and transformation share ONE propagation rule.
- `transformOrFail<From, To, R>` has `Context = Context<From> | Context<To> | R`, so a five-stage pipeline where one leg reaches a service carries it in the composite `Context`, provided one `Layer` at the root, never a leg-local provide; a composite failure is a `ParseResult.Composite` wrapping the leg's `Pointer`/`Transformation` AST nodes, naming which seam rejected, the locus derived from chain structure.
- A class-to-class transform threads `R2` (decode requirement) and `R3` (encode) INDEPENDENTLY into the class `Context` alongside `Struct.Context<NewFields>`: `BaseClass.transformOrFail<Next>(id)` runs the transform AFTER base decode (`decode: Type<Fields> -> Effect<Type<Fields & NewFields>, ParseIssue, R2>`), `transformOrFailFrom<Next>(id)` BEFORE (`decode: I -> Effect<I & Encoded<NewFields>, ...>`, a checksum over unparsed payload) — the seam position the only distinguishing axis, choosing wrong reading pre-validation noise or post-validation values.
- Boundary: a comparison, generation, or rendering concern that reaches a service is NOT a projector slot — `Equivalence<A>`/`(a: A) => string`/`LazyArbitrary<A>` carry no `R`, so a uniqueness check against a store is a `filterEffect`/`transformOrFail` on the CODEC widening `Context`, never smuggled into a comparator. The encode leg is mandatory and total — `encode: Type<Fields & NewFields> -> Effect<Type<Fields>>` must reconstruct the base, so a non-invertible added field forces an explicit discard, the asymmetry surfaced at declaration.

[VARIANT_SCHEMA_IS_THE_OUTER_OWNER_PROJECTING_BY_REMAP_TO_NEVER]:
- `VariantSchema.make({ variants, defaultVariant })` declares the variant alphabet once and returns a closed toolkit (`Struct`, `Field`, `FieldOnly`, `FieldExcept`, `Class`, `Union`, `extract`, `fieldEvolve`, `fieldFromKey`) parameterized by `Variants[number]` — the variant set the SECOND axis of the owner beyond the field map, a new projection a tuple entry the whole toolkit re-derives; `defaultVariant` is the IDENTITY of the owner (the class self-type is `ClassFromFields<Self, Fields, ExtractFields<Default, ...>>`), not a seventh schema, every other variant reached as a named property.
- `ExtractFields` is the family-deletion engine: `{ [K in keyof Fields as [Fields[K]] extends [Field<infer Config>] ? V extends keyof Config ? K : never : K]: ... }` — the `as` key-remap emits the key when `V` is in the field's `Config` and `never` when not, the same remap-to-`never` primitive that renames keys (`fromKey`), filters `EncodedOptionalKeys`, and drops `_tag` in `TaggedEnum.Args`; a plain (non-`Field`-wrapped) field is unconditional and survives every projection, the wrapper the explicit opt-in to per-variant presence.
- `Field<Config>` is a per-variant schema map declaring in one record which variants admit it and with what schema each validates; `FieldOnly(...keys)` and `FieldExcept(...keys)` are dual halves of one membership decision (the complement computed by `Exclude` over the closed alphabet), and `Field(config)` is fenced by `config & { [K in Exclude<keyof A, Variants[number]>]: never }` so a misspelled variant is a compile error; `Struct.Validate<A, Variant>` forces `[keyof Config] extends [Variant]` for every `Field`, emitting `'field must have valid variants'` when membership exceeds the alphabet. Reject: a `Field` whose `Config` lists every variant with the same schema — drop the wrapper, the unconditional field survives by default; a variant axis with no downstream consumer of each projection is the rejected register.

```typescript
import { VariantSchema } from '@effect/experimental'
import { Schema } from 'effect'
const V = VariantSchema.make({ variants: ['read', 'write', 'public'] as const, defaultVariant: 'read' })
class Owner extends V.Class<Owner>('Owner')({

type Read = Schema.Schema.Type<typeof Owner>            // { id; secret; label } — default identity
type Write = Schema.Schema.Type<typeof Owner.write>     // { secret; label } — id remapped to never

```
