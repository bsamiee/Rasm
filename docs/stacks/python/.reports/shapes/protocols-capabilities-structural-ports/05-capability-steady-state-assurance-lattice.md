# Capability Steady-State Assurance And Equilibrium Lattice

# Assurance-Promotion Temporal Law

- Capability assurance temporal law governs promotion in-flight and assurance-between-promotions phases — composition root owns transitions; sustain invariants P1–P10 bind only in `assurance_between_promotions` after post-promotion certify; promotion in-flight wins over scheduled replay and deep assurance on affected port edges; certify rows never substitute promotion admission or patch catalog rows to absorb partial evolution.
- **Promotion in-flight wins** — open `promotion_unit_id`, partial catalog landing, or port quarantine on promotion diff suspends steady-state mark; promotion obligations precede sustain scheduling on obligated `consumer_contexts` — P6 quiescence gap keeps phase `promotion_in_flight` until fan-out and harness replay complete.
- **Assurance between promotions** — no open promotion unit and no quarantine on affected replaceable-backend edges; P1–P10 and H1–H6 are binding; convergence morphisms assert logical catalog revision parity across contexts — not wall-clock synchronization.
- **Pre-merge is promotion-adjacent** — pre-merge witness P6 runs on git diff before merge; does not substitute post-promotion certify or discharge scheduled replay when no promotion diff is present.
- **Post-promotion certify bridges phases** — full P1–P10 after harness replay; steady-state re-entry only after certify row commits — equilibrium suspended from first edge-touching edit through certify completion.
- **Assurance deferral** — scheduled replay, deep assurance, and convergence replay defer on contexts with open promotion unit; boot P1–P3 and health tick P3, P5 may continue on unaffected edges under documented assurance-paused degradation — never defers pre-merge on promotion diffs or post-promotion certify on landing unit.
- **Substrate freeze** — witness catalog rows, bind records, frozenset hashes, tier tables, scope maps, and `law_slugs` tuples are read-only witnesses during sustain except via classified edge-touching promotion.
- Temporal witness row — `phase`, `promotion_unit_id`, `quarantine_posture`, `deferred_assurance_rows`, `certify_pending` — composition root exports row; false steady state when phase is `promotion_in_flight` but certification row marks steady is equilibrium violation.

# Convergence Proof Morphisms

- **Catalog convergence** — `member_frozenset_hash` at port owner equals hash recomputed from live `get_protocol_members(Port)` — mismatch triggers P1 failure and scheduled catalog reconcile row.
- **Bind convergence** — `tier_table_hash` and factory signatures at composition root match bind record exports — mismatch triggers P2 failure and bind co-evolution review.

# Assurance Cadence And Gate Binding

- Assurance gates bind to lifecycle hooks distinct from promotion enforcement — conflating hooks produces false steady-state confidence or duplicate proof spend; temporal phase from capability assurance temporal law in wave determines which gates are lawful at each hook.
- **Boot gate** — boot witness row P1–P3; runs after scope bind, before first canonical interior transform — failure equivalent to missing bind row block.
- **Pre-merge gate** — pre-merge witness row P6 plus fan-out obligation set — runs on git diff; independent of runtime promotion loop.
- **Post-promotion gate** — post-promotion certify row full P1–P10 — runs once per `promotion_unit_id` after harness replay; steady state re-enters only after certify row passes per capability assurance temporal law.
- **Scheduled gate** — scheduled replay row P1, P2, P4 — runs on root cadence policy; may overlap boot on cold start but never skips pre-merge on promotion diffs.
- **Health tick gate** — health tick row P3, P5 — runs during live operation; failure escalates to port quarantine row when tick budget exceeded.
- **Deep assurance gate** — deep assurance row static matrix through metamorphic re-bind — runs on release or mirror topology change cadence; scheduled gate failure blocks deep assurance start.
- Gate ordering — boot before interior handoff; pre-merge before merge; post-promotion before steady-state mark; scheduled independent of request volume; health tick continuous; deep assurance last on cadence.

# Change Classification Under Steady State

- Not every edit is a promotion event — steady-state change classification prevents unnecessary catalog fan-out while preserving equilibrium when port edges are untouched.
- **Interior-only change** — domain transform, fault routing match arm, or port method implementation within one context; no vocabulary, mirror, bind, or scope target change — assurance limited to context-local proof layers.
- **Witness-neutral change** — comment, diagnostic tag, or trace attribute without semantic effect — no assurance row update; arch rules still apply.
- **Edge-touching change** — any diff on port owner, mirror extraction, bind factory, tier table, scope map schema, or exported `Protocol` symbol — reclassifies as promotion event; post-promotion certify row mandatory.
- **Topology change** — bounded context add, split, merge, or new replaceable backend family — federation event; deep assurance row and full metamorphic re-bind mandatory before steady-state re-entry.
- Classification witness row — `change_class`, `assurance_rows_required`, `promotion_unit_required`, `fan_out_context_ids` — arch rule or git diff classifier exports row; misclassification is equilibrium violation when bind drift follows.

- Signal law — health signals map to assurance rows first; escalation to promotion blockers occurs when scheduled replay fails or health tick budget exceeded per root policy.

# Composition Root Assurance Loop

- Long-running processes admit a bounded assurance loop at composition root — distinct from promotion replay loop in ; runs during steady state and after certify row entry.

# Assurance Collapse Tests

- Collapse tests verify assurance remediation — extend promotion anti-patterns.
- **Skip certify after promotion** — merge lands promotion unit without post-promotion certify row — collapse by blocking steady-state mark and running full P1–P10.
- **Duplicate catalog rows** — test file re-declares `law_slugs` parallel to witness catalog table — collapse to single parametrized owner import.
- **Health tick in domain** — smart constructor or fold invokes periodic frozenset diff — collapse to root assurance loop and catalog witness import.
- **False steady state** — quarantine active or promotion unit open but certification row marks steady — collapse by coupling certify row to temporal witness `phase` and quarantine clearance.
- **Assurance during in-flight promotion** — scheduled replay or deep assurance mutates catalog row while `promotion_unit_id` open on affected contexts — collapse by deferring sustain gates until post-promotion certify per capability assurance temporal law.
- **Interior-only mislabel** — edge-touching mirror diff classified interior-only — collapse by reclassifying as promotion event and running fan-out obligation set.
- **Deep assurance before scheduled** — metamorphic re-bind runs while P1 catalog convergence fails — collapse by enforcing gate ordering; scheduled replay must pass first.
- **Runtime-only port steady mark** — port with `evidence_grades={runtime_smoke}` certified without static call-site collapse review — collapse to `TypeIs` nominal predicate or delete gate.
- Remediation order — classify change, read temporal phase, select assurance or promotion row, replay witnesses, certify or escalate — never disable assurance gate to absorb promotion debt.

# Assurance Versus Promotion Boundary

- Assurance and promotion loops are complementary — composition root schedules both with distinct triggers, phases, and proof budgets per capability assurance temporal law.
- **Assurance triggers** — cadence due, boot complete, post-promotion certify request, health tick — proactive; lawful only when temporal witness `phase` is `assurance_between_promotions`; may run when zero promotion units are in flight and no port quarantine posture is active on obligated contexts.
- **Promotion triggers** — `event_kind` from closed band, SDK signature drift, tier reorder, frozenset hash rotation — reactive to edge-touching diffs; blocks steady-state mark until certify row passes.
- **Escalation edge** — assurance row failure on P1 or P2 maps to H-signal first; repeated failure or tick budget exceeded maps to promotion quarantine row — assurance does not patch mirror or bind factory to avoid promotion.
- **Phase transition law** — edge-touching edit opens `promotion_in_flight`; partial fan-out or H-signal escalation during promote extends through harness replay to post-promotion certify; sustain holds only when temporal witness `phase` is `assurance_between_promotions` and port quarantine cleared.
- **Shared witness tables** — catalog rows and promotion admission rows– are inputs to assurance rows — certification commits catalog revision consumed by next scheduled replay.
- **Proof spend ordering** — health tick and scheduled replay before deep assurance before full metamorphic re-bind on release cadence — same static-before-generative ordering as harness layer binding.

# Steady-State Degradation And Read-Only Capability

- Steady-state degradation is explicit root posture — extends promotion quarantine.
- **Assurance-paused degradation** — deep assurance and scheduled replay deferred; health tick and boot witness remain active — documents maintenance window and resume precondition rows.
- **Port read-only steady state** — member promotion frozen; P1 and P2 still run on scheduled gate — frozenset parity on declared closure mandatory; extension default members capture absent optional capability.
- **Single-backend maintenance** — one adapter warm refresh while tier table unchanged — P3 scoped to maintaining bind row; interior transforms on unaffected ports continue.
- Posture change updates certification table and health tick escalation policy — contexts do not self-declare steady-state degradation.
- Resume from degradation requires post-promotion certify row or scheduled replay pass — same bar as promotion unit completion.
