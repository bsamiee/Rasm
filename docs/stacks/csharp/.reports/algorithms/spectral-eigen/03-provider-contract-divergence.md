# Provider Contract Divergence at the Eigen Boundary

[BINDING_CONTRACT_AND_ESCAPE_HATCH]:
- Every dense eigensolve resolves `Provider.EigenDecomp(...)` against whatever backend occupies the process-static provider slot at call time — there is no per-call override, no instance scoping, no ambient context — so the eigen route's numerical regime is decided by state no solve argument carries.
- The binding contract any substitute must satisfy is one method on the generic provider interface: `void EigenDecomp(bool isSymmetric, int order, T[] matrix, T[] matrixEv, Complex[] vectorEv, T[] matrixD)`. The parameter order is load-bearing and the carrier shapes are fixed: `matrix`, `matrixEv`, `matrixD` are length `order²` column-major, while `vectorEv` is always `System.Numerics.Complex[]` of length `order` regardless of `T`. A `float` or single-precision-complex native provider must up-convert its eigenvalues to double-precision `Complex` at this boundary — a precision asymmetry baked into the contract, not the algorithm.
- The dense factory is the only surface that touches `Provider.EigenDecomp`; the user-storage factory runs its own in-class tridiagonalize/diagonalize/Hessenberg/Schur pipeline and never consults the provider. Routing an eigensolve through user storage is the provider-immune escape hatch: it pins the managed numerical regime irrespective of which native binary the host has installed or which environment variable is set.
- The managed eigensolve kernels are serial regardless of the configured parallel degree: the degree knob gates other provider operations and re-verifies backends on mutation, but the symmetric QL sweep, the Hessenberg reduction, and the spectral-decomposition solve never partition. Raising the degree buys an eigen-heavy pipeline nothing on the managed path; only a native driver parallelizes the decomposition itself.
[CONTRACT_DIVERGENCE_AXES]:
- The managed path is classic Handbook/EISPACK lineage — Householder tridiagonalization then QL iteration for the symmetric case, orthogonal-Hessenberg reduction then real-Schur QR for the nonsymmetric case — with no workspace-query protocol, no info-return convention, and no LAPACK driver naming anywhere. A native provider substitutes a backward-stable LAPACK driver behind the same interface, and the two paths diverge on five contract axes that no carrier records: ordering, sign convention, convergence ceiling, error-rail shape, and last-bit determinism.
- Ordering divergence: the managed symmetric path applies an explicit ascending selection sort as its tail, swapping eigenvector columns in lockstep with eigenvalue entries; the managed nonsymmetric path emits eigenvalues in bottom-up Schur deflation order with no sort. A native symmetric driver also returns ascending values but a native nonsymmetric driver's ordering is its own deflation path. Any consumer relying on index correspondence between two solves of nearby operators is relying on provider-stable ordering that the contract does not guarantee.
- Sign and normalization divergence: eigenvectors are unique only up to sign (real) or unimodular phase (complex), and the managed Householder accumulation fixes a sign convention different from a native driver's largest-component-positive convention. Both bases are valid; their columns differ by sign or phase. A projection that assumes a fixed sign, or a regression baseline asserting eigenvector equality, breaks the instant the provider swaps even though both answers are correct.
- Convergence-ceiling and error-rail divergence: the managed symmetric QL throws a non-convergence signal at a fixed per-eigenvalue iteration count; the managed real-Schur QR throws at a ceiling proportional to order that resets on every deflation, so it bounds consecutive non-deflating steps since the last deflation, not total work. A native driver returns a nonzero info code on the same pathology rather than throwing — the error rail changes shape across the boundary from exception to status code, and a `Try`-based receipt that catches the managed exception sees nothing when a native provider sets an info code instead.
- Determinism divergence: the managed real-Schur QR's exceptional-shift escape substitutes fixed literal constants into the shift state, so its last-bit behavior on stalled bulge-chases is exactly reproducible — and that exact-determinism property is itself provider-specific, because a native driver's exceptional-shift heuristic produces different last bits. An eigenvalue-equality assertion tighter than the convergence band is therefore provider-coupled even when both providers are individually deterministic.
[BLOCK_DIAGONAL_PACKING_INVARIANT]:
- The block-diagonal carrier encodes a complex-conjugate pair as a 2×2 real-Schur block, and the packing is the same convention any native provider must emit for the dense consumer to read it: the real part lands on the diagonal at both pair indices, the positive imaginary part lands on the super-diagonal entry and the negative imaginary part on the corresponding sub-diagonal entry. A real eigenvalue writes only its diagonal entry with zero off-diagonals around its index. The eigenvalue vector and the block-diagonal carry the same imaginary information in two layouts, and a native provider that packs the off-diagonals with a different sign or transpose desynchronizes every consumer that reads rotation evidence from the block structure rather than the eigenvalue vector.
[PROVIDER_EVIDENCE_ON_THE_EIGEN_RECEIPT]:
- The eigen receipt must carry the provider identity as first-class evidence alongside the route variant, because the five-axis contract divergence means the same logical eigendecomposition has different sign, ordering, and last-bit content under different providers, and a downstream comparison of two receipts is only meaningful when both were produced by the same provider. Provider identity is to the eigen receipt what symmetry class is to the route: a discriminant that changes the meaning of every other field.
- Pinning the managed provider before first provider touch is the correct definition-time policy for any eigen proof receipt that must reproduce across hosts whose installed native binaries and ambient configuration differ; relying on ambient resolution delegates numerical reproducibility to host state the receipt cannot see. The pin is a one-time policy assertion at process start, matching the process-static shape of the slot it controls, never a per-solve parameter.
- The error rail must absorb both failure shapes: the managed boundary wraps a thrown non-convergence signal, but a native boundary that returns a nonzero info code reports the failure in-band, so a receipt taking only the throwing path silently mislabels a native non-convergence as success with meaningless vectors. The block Frobenius norm of the eigen-equation defect `‖A·V − V·D‖_F` is the only provider-agnostic correctness detector and is mandatory precisely because it is the one signal both rails surface identically.
