# Composition Law

[KIND_ERASURE_REANCHORING]:
- `K<F,A>` is a memberless phantom witness; every carrier implementing a trait also implements `K<M,A>` for its bound type, so one algorithm body over `K<M,A> where M : Monad<M>, Fallible<Error,M>` serves `Fin`, `Eff`, `Option`, `IO`, and transformer stacks uniformly — per-carrier pipeline duplication sharing identical structure is the rejected form this collapses.
- Failures raise through trait statics — `Fallible.fail<E,F,A>` and `Fallible.error<F,A>` — never through a concrete constructor such as `Fin.Fail` inside the polymorphic body, which silently pins the carrier and defeats the abstraction.
- The bridge back from the erased kind is per-carrier: the caller pins the returned kind with the carrier's own re-anchoring cast, and no universal erasure-removing cast exists. Omitting the pin is the most common carrier-polymorphic error — the value stays existentially kinded and refuses concrete-only operators. Nested stacks use a distinct lift/lower pair through the outer map for two-level values; the resilient lowering rebuilds via an identity map instead of throwing when a hand-constructed kind value is genuinely foreign.

[CONSTRAINT_CAPABILITY_CONTRACT]:
- The constraint set on a carrier-polymorphic arrow is its capability contract: functor grants mapping, applicative adds pure/apply/action, monad adds bind and recursion. Weakening a constraint is a provable statement about what the body cannot do — an arrow bound only to applicative cannot sequence-with-dependency and is therefore structurally accumulation-compatible and parallelizable.
- Dependencies enter as one-member capability witnesses, making the dependency set the type signature: an arrow constrained over exactly the capabilities it touches cannot smuggle unused capability past the signature, and adding a dependency is adding a constraint, never editing a container registration. The reader and state witnesses share one shape — environment ask/asks/local mirrors state get/gets/put/modify — with the whole-value form derived from the projecting form applied to identity, so a carrier implements one primitive per axis and inherits the family.
- The fallible family carries a typed error parameter — `Fallible<E,F>` with `Error`-pinned overloads as specializations — so capability constraints quantify over the failure vocabulary too; carriers with custom monoid error rails satisfy the same generic combinators.
- Mechanism: static-abstract trait members dispatch on the type argument at the call site with zero allocation and zero vtable indirection; a trait-constrained arrow monomorphizes per carrier at JIT specialization, and the carrier type itself is the trait-implementing witness, never an instance.

[NATURAL_TRANSFORM_HIERARCHY]:
- `Natural<F,G>` declares `static abstract K<G,A> Transform<A>(K<F,A>)`; `CoNatural<F,G>` is the dual `CoTransform`. `NaturalMono<out F,in G>` derives `CoTransform` from `F.Transform` — unidirectional promotion with zero manual implementation; `NaturalIso<F,G>` asserts a full roundtrip and is the wrong choice whenever one path loses failure information; `NaturalEpi<in F,out G>` implements `Natural<G,F>` via `F.CoTransform`, inverting the apparent direction of its type parameters.
- Constraint placement is load-bearing: `Natural.transform<F,G,A>` puts the obligation on the source, `CoNatural.transform<F,G,A>` on the target; in a transformer stack the call site goes downward via `CoNatural.transform<Outer,Inner,A>` and upward via `Natural.transform<Inner,Outer,A>`.
- Promotion is structural and domain-free: an algorithm returning `K<Fin,A>` promotes to `Eff`, `IO`, `Option`, `Try`, or `Either<Error>` at the instantiation site through `Natural.transform`, the explicit path when an implicit carrier operator is ambiguous. Transformer naturals are one-way where information is lost — `FinT<M>` promotes to `EitherT<Error,M>`, `OptionT<M>`, and `TryT<M>` with no inverse, because the narrowing is not an isomorphism.
- Memoized values cross naturals lazily: transforming a memo produces a fresh suspension that promotes the inner value on first read, so derived natural instances propagate memoization without forcing at transform time.

[APPLICATIVE_PRODUCT_ALGEBRA]:
- The applicative product ships curried and tuple-driven: the curried form peels one argument per apply across the arity ladder; the tuple form takes up to ten independent carriers plus an uncurried combiner in one call, currying once internally and threading apply across the elements. The monadic-result twin flattens a combiner that itself returns an effect — bridging independent-accumulate into a final dependent step without an explicit bind.
- The same call site changes failure policy purely by carrier type argument: an accumulating carrier collects all failures, a short-circuit carrier reports the first — every failure combination derives from the carrier's apply. Pre-built arithmetic lifts give element-wise numeric combination inside any applicative carrier with no manual operator application.
- Trap: query expressions desugar to bind — building an independent field product with from/select inside an accumulating carrier silently switches accumulation off with no compile-time indication; the applicative operator chain is the accumulating spelling, and the dependence/independence choice is invisible at the call site without this law.
- Apply and choice carry deferred-operand overloads taking a memoized effect forced behind a compare-and-swap gate, with an exception resetting the gate so the next read retries acquisition. Laziness pays only where the carrier's combination can decide from the left operand alone — a collection carrier with an empty function side never constructs the right operand, while failure-combining carriers force the memo unconditionally because accumulation needs both sides. Pay-for-reached versus pay-for-all is a property of the carrier's combination logic, never a guarantee of the memo wrapper.

[ALTERNATIVE_REPETITION_GRAMMAR]:
- The alternative trait — choice plus applicative with its own identity — is simultaneously the failure-fallback surface and a repetition grammar over any carrier with emptiness and choice: one-or-more, zero-or-more, separated and terminated repetition, and repeat-until-terminator with terminator-keeping twins are derived members expressed once against choice and pure, inherited by every alternative carrier. A backtracking parser, a non-deterministic search over a sequence carrier, and a first-success fallback over the absence carrier are the same code at three instantiations, and the repetition combinators stay stack-safe by threading the carrier's own trampoline.
- Combine and choose are distinct combination forms with carrier-specific law: on an accumulating transformer, semigroup combine runs both sides and merges failures while choice short-circuits on the first failure without combining — alternation is early-exit, combine is the accumulating form; on a failure-rail transformer, the monoidal combine is lazy alternation — a successful left side short-circuits and the right side is reached only through the inner monad's bind on failure.

[LIFTED_CONTROL_FLOW]:
- Conditional execution consumes predicates already in the rail: when/unless run an effect conditionally without materializing the boolean as a statement, and the lifted ternary returns an effect chosen by an effectful condition. The ternary's overload set spans the cross product of carrier-effect, pure-literal, and raw-IO branches, so a branch that is a constant or a bare effect lifts without explicit wrapping — the overload set absorbs the lift.
- The accumulate-while/until family replays a single effect, collecting results until the predicate flips, with monadic-suffix forms taking an effectful predicate — the rail-native do-while whose continuation condition is itself an effect; the unbounded repeat exists for daemon-shaped effects.

[RECUR_TRAMPOLINE]:
- The monad trait exposes a carrier-level trampoline whose step function returns an effect producing a two-case signal — loop with a new seed, or done with a result — so each iteration is itself an effect in the carrier: the loop body can fail, branch, or short-circuit through the rail while recursion stays bounded. The loop variable threads as data and the exit is data, never a break statement.
- Three sibling drivers form one family: the safe default, an unguarded variant for carriers already tail-safe, and a streaming variant — constrained to carriers with a lazy-enumerable natural-transformation pair — that collects every intermediate loop value into the carrier, turning the trampoline into a streaming unfold rather than a single-result fold.
- Transformers inherit the loop by adapting the signal through the inner payload: a failure-rail transformer maps an inner failure inside the step to a done-with-failure signal, so a domain loop over `FinT<IO,A>` is stack-safe, cancellable, and error-aborting with zero composition glue.

[ONE_INNER_BIND_DELEGATION]:
- Every transformer's bind is written exactly once, purely in terms of the immediately-inner monad's bind — the reader form runs the inner bind under a constant environment, the state form threads the value-state product through the continuation's run, the short-circuit forms match the inner case and either continue or collapse. A deep stack composes by one-level delegation — the spine is a chain of single hops, never a flattened reimplementation — so adding a transformer is additive at zero semantic cost to the layers below, and the cost of a stack is exactly one bind allocation per layer per step.
- The state layer is the only one whose inner bound value is not the bare value but a value-state product, and that asymmetry is the root cause of the capability law: a monad whose inner payload is a product cannot be projected back to a bare effect without discarding the state thread.

[CAPABILITY_BY_STACK_ORDER]:
- The entire composition-time effect surface — fork, await, timeout, bracket, repeat, retry, fold, post, local, uninterruptible — is declared once as default members on the unlift trait, each a one-line map over the projected inner effect, and the whole surface hinges on one abstract projection from the carrier to its bare effect. A transformer earns the entire algebra by supplying that projection; a transformer that cannot project — any product-payload layer — forfeits all of it.
- Stack order is therefore a capability decision, not a cosmetic one: a layer that needs retry, bracket, fork, or timeout must sit on the unliftable side of any product-shaped layer, and a state layer placed outside the effect boundary makes the outer effect un-bracketable. Order changes what the stack can do, not merely what it means.
- Unlift is a runtime contract, not a compile-time one: the unlift cascade resolves through an optional shadow surface, so a stack whose innermost monad never supplies the lift type-checks and then throws a wrapped exceptional at run time on first use. The optional sub-trait split is also the mechanism by which a pure carrier participates in every monadic arrow yet rejects effect lifting at the type level — capability is additive and à la carte, queried at the boundary, never all-or-nothing.
- The capability-constraint form is simultaneously the abstraction and the proof obligation: one body quantified over the conjunction of state, reader, unlift, and monad constraints replaces one algorithm per concrete stack, and a constraint set that includes unlift is unsatisfiable by any ordering that buries the effect-capable layer under a product-shaped one — the constraint that compiles asserts a legal ordering.
- Derived members are the canonical spellings: hand-recomposing fork-then-await re-describes the trait's own await member and forfeits its carrier fast-paths.

```csharp
static K<M, Report> Pipeline<M>(Input raw)
    where M : Stateful<M, Budget>, Readable<M, Config>, MonadUnliftIO<M>, Monad<M> =>
    from cfg   in Readable.asks<M, Config, Threshold>(static c => c.Limit)
    from spent in Stateful.get<M, Budget>()
    from _     in Stateful.modify<M, Budget>(b => b.Charge(raw.Cost))
    from step  in MonadUnliftIO.mapIO<M, Raw, Raw>(
                      io => io.Timeout(TimeSpan.FromSeconds(2)).Retry(Backoff),
                      Acquire(raw))
    from done  in Readable.local<M, Config, Report>(
                      c => c.Sandboxed(),
                      Finalize(step, cfg, spent))
    select done;
```

[STACK_ENTRY_LIFT_LAW]:
- Two lifts carry different semantics: layer-lift wraps an inner computation one level in the layer's constructor — the reader ignores its environment, the state layer pairs the value with unchanged state — and lifting across N layers is N nested one-level lifts; the effect-lift cascade threads a raw effect down through every intermediate layer until the effect base overrides. The distinction is decisive: layer-lift of a pure injection places a value-shaped placeholder at the current layer with no effect semantics, while the effect lift threads a real effect to the floor — conflating them silently substitutes a value where an effect was intended.
- The generic effect injection serves any monad in the IO hierarchy including transformer stacks — one IO-bound operation shared across effect flavors with no carrier-specific plumbing; the transformer's own lift member is the canonical path for an existing inner-monad computation.
- Each transformer overloads its query-expression bind per inner-fragment kind — environment asks, state gets/puts/modifies, pure literals, bare effects, and inner-monad steps each bind through the overload matching their shape — so one comprehension mixes layer-local effects with no explicit lift. A pipeline peppered with explicit single-layer lifts between domain steps is the rejected form: it re-describes a lift the overload set already performs.

[STACK_LOCAL_PROJECTION_AND_HOISTS]:
- Environment narrowing and environment rewriting are distinct combinators: narrowing changes the reader's environment type through a wide-to-narrow projection function — a sub-pipeline written against a narrow environment composes into a richer parent structurally, deleting marker capability interfaces — while rewriting preserves the type and maps the value for one scoped sub-effect, with the modification in scope for anything composed inside that scope.
- An environment-free effect promotes into a runtime-carrying context by running against the zero-allocation minimal runtime and lifting the result through the generic IO injection — the canonical embedding of pure-effect computations into runtime-carrying pipelines; the runtime itself projects into the bound value through an identity lift, available as data inside the pipeline with no accessor protocol.
- The two hoists are orthogonal: the value-hoist transforms the inner monad's bound computation while preserving both the layer and the monad type — the success value is extracted, transformed, and re-wrapped while the failure case short-circuits past the function; the type-hoist swaps the effect base under a fixed reader or state layer without unwinding the stack, with the target carrier constrained to supply the choice or alternative structure the hoist relies on.

[ACCUMULATING_TRANSFORMER_COMPOSITION]:
- A transformer's apply composes the outer monad's concurrency with the inner carrier's failure algebra: the failure-rail transformer's apply runs both sides through the outer applicative concurrently — parallelism when the base is IO — while preserving inner accumulation, whereas bind sequences; choosing bind where apply suffices forfeits the outer monad's parallelism and the inner carrier's accumulation at once.
- The accumulating validation transformer threads its monoid witness at run time as boxed delegates — empty and combine travel with the computation — so custom domain error types compose without per-step compiler-emitted trait dispatch. Its apply sequences both sides inside the inner monad before accumulating: under an IO base, field validation runs sequentially; parallel accumulation requires forking the sides manually and combining the resulting validations with the accumulating operator before re-entering the layer.
- The chronicle transformer is the third combination algebra — a log/abort duality over a three-case bifunctor: `dictate` lifts a bound value with no entry, `confess` lifts a fatal abort, `chronicle` lifts an entry alongside a value. Bind accumulates entries across steps without short-circuiting the bound value, and abort is typed in the chronicle vocabulary itself — the failure type is the log type, not a fixed error. It is correct exactly when log and abort share one vocabulary; recovery is consuming-edge and inner-monad-only: `memento` recovers a confession while preserving entries, `absolve` substitutes a default and continues, `condemn` converts any entry-with-value into an abort.
