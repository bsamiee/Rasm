# Parameter Algebra — Distilled

[POLICY_VALUE_LAW]:
- Configuration enters as one value that carries its own behavior: a smart-enum row with constructor-injected delegates, a union case, or a frozen policy row. A boolean parameter selecting between two bodies is policy inlined as a knob — the collapsed form passes the pre-constructed case and no `if`/`switch` re-derives the body at the dispatch site.
- A stateless union case (`T1IsStateless = true`) stores only the discriminator — the correct shape for policy entries with no configuration data (`None`, `Default`, `Absent`); use struct member types so accessors return structurally non-null defaults, and note `T1IsStateless` implies `T1IsNullableReferenceType` for reference members. Wrapping such a case in `Option<T>` stacks a second optionality layer on a discriminant that already models absence — `Option<T>` belongs where the entire union is absent, never inside one.
- Time-bearing policy values carry their own clock: schedule builders accept a current-time function bound at construction (`Schedule.minuteOfHour(0, () => clock.UtcNow)`), so the policy reads no global. The seeded-versus-random distinction in jitter travels as `Option<int>` presence, not a boolean. Composed schedules and named catch policies live in `static readonly` fields on the policy owner; no call site re-specifies backoff inside a retry loop, and callers needing different policy pass their own composed value.
- Behavioral near-twins are policy choices encoded in the value, not flags: `fixedInterval` suppresses delay on overrun and restarts the window from completion (load-leveling); `windowed` always emits residual time to the absolute boundary (cron alignment) — choosing one selects burst behavior at boundaries, and no parameter can repair a wrong choice downstream.

[OPTIONAL_CONTEXT]:
- `Option<T>` is the single admitted optional-parameter form. Declaration is `Option<T> x = default` (CS1736-safe for `[ValueObject]`/`[SmartEnum]` payloads); consumption is `IfNone(canonical)` where the canonical fallback comes from the policy owner — the static row or factory — so the default is derived once, never re-stated per call site.
- An optional runtime context with identity enters as one record: `T? ctx = null` is the boundary-only spelling, projected to `Option<T>` at admission so interior code never branches on null. The nullable flag tail (`T? a = null, T? b = null, bool x = false`) fragments one context into parallel parameters; the collapse is one `Option<ContextRecord>` carrying the override bundle.
- `Option<A>` compiles `||` because it defines `operator true` (`IsSome`) and `operator |` (left-biased choice) and deliberately omits `operator &` — so `&&` does not compile and all-present conjunction cannot be smuggled in as a short-circuit operator; it routes through `Bind` or `{ IsSome: true, Case: T v }` property-pattern destructuring. The eager `|` evaluates both operands; lazy short-circuit is a distinct `Choose` overload selected by the right operand's static type — a `Memo` (compute-at-most-once) right branch binds the lazy overload, a materialized value binds the eager one. Skipping an expensive alternative is therefore an overload selection driven by the argument's shape, not a flag.

[SUPPLIED_VERSUS_OMITTED]:
- `Argument<T>` is a `readonly ref struct` (`T : allows ref struct`) carrying `Value` + `IsSet`; implicit conversion from `T` sets `IsSet = true`, a default instance means omission. It distinguishes "caller supplied a value equal to the domain default" from "caller omitted" — the distinction `Option<T>` cannot make. It cannot be boxed, stored in a field, or cross `await`: its use is confined to one synchronous call frame, which bounds where the supplied-versus-omitted distinction can be threaded — never through deferred or asynchronous dispatch.
- The generator's canonical `Argument<T>` consumers are the partial-dispatch surfaces: each optional case slot of `MapPartially<TResult>` is `Argument<TResult>` — `IsSet` false returns the required `@default` value without invoking case logic. The generated `ValidateFactoryArguments` partial takes `ref T` per field, and `IObjectFactory<T,TValue,TError>.Validate` receives raw `TValue?` — expecting `Argument<T>` in either generated seam is a compile-time mismatch; it belongs only on bespoke factory overloads authored beside `[ObjectFactory<T>]`.

[RESOLUTION_PRESSURE_ON_PARAMETERS]:
- Among equally good candidates, the one whose every parameter received an argument beats the one that substituted an optional default — a real tie-breaker. A growing optional tail therefore loses resolution to a tighter sibling, silently redirecting existing call sites when both remain applicable; this is the resolution-level argument against the flag tail, because collapsing it into one `Option<ContextRecord>` removes the parallel candidate entirely. A trailing `params` does not count as an omitted optional, so an arity tail composes with an optional-context parameter without penalty.
- A named argument out of declaration order is legal only when no positional argument follows it (CS1738); a skipped-then-targeted optional is reachable only by name. Parameter identifiers are therefore load-bearing API surface: renaming a generated dispatch parameter — including via `SwitchMapStateParameterName` (default `"state"`) — breaks every named call site, a contract change, not a relabel. `NestedUnionParameterNameGeneration.Simple` shortens generated case-parameter names to the leaf and fails the build on leaf collisions; `Default` (type-prefixed) is the safe form whenever collision is possible.
- The generated total and partial dispatch surfaces coexist without ambiguity because the parameter algebra disambiguates, never a mode flag: the total `Switch` requires every case positionally (inapplicable once one is omitted); `SwitchPartially`'s action form makes the `@default` fallback optional while the func form requires it, so a call with no fallback can only bind the action form; a stop-at overload collapses a sub-hierarchy into one handler, lowering arity below the exhaustive form — the binding choice is recovered from argument arity and optionality shape.
- Caller-information optionals (caller-expression capture) are parameters the compiler fills; a priority-steered overload pair injects the captured default while the parameter stays invisible at call sites — supplying it by name defeats the capture.

[DEFAULTS_DERIVE_FROM_THE_OWNER]:
- Struct value-object defaults are owner-declared: `AllowDefaultStructs = false` (default) prohibits `default(T)` and parameterless construction and emits no empty instance; `true` permits the zero state and emits the static default instance named by `DefaultInstancePropertyName`. `Option<TVo> = default` at a call site yields `None` regardless; the domain default exists only when the owner enables it.
- `SkipFactoryMethods = true` is a cascade, not a knob: it suppresses `Create`/`Validate`/`TryCreate`, the type converter, both factory interfaces, the key-to-object implicit conversion, and all arithmetic operator generation permanently, and forces parsable and span-parsable off. A subsequent `[ObjectFactory<T>]` restores serialization, EF, and model-binding routes but never the arithmetic operators or parsable surfaces. `SkipIComparable` is independent of the cascade. Suppress-and-hand-author is the correct shape when admission spans multiple sources or must integrate a typed result rail.
- Absence policy is owner-declared and one-way coupled: `EmptyStringInFactoryMethodsYieldsNull = true` forces `NullInFactoryMethodsYieldsNull = true` (never the reverse), and it is the only way to make both null and empty yield null. `UnsafeConversionToKeyMemberType` (default explicit) generates a throwing key-extraction operator when default structs are allowed; `None` removes it so egress is explicit. `SkipKeyMember = true` hands the key member to the author: a property key must have an `init` setter; a field key may be `readonly`.
- Knob coupling is monotone and silent: ordering operators drag equality operators up to the same generation level and equality suppression collapses both; `SkipIParsable = true` forces span-parsable off (one-way — span-parsable can be skipped alone). One setting is the authority in each pair; setting both independently invites silent coercion.

[ADMISSION_FACTORY_ROUTING]:
- `[ObjectFactory<T>]` is allow-multiple and per-attribute routed: each declaration carries its own serialization-framework flags, EF-Core flag, and model-binding flag, so one owner routes JSON through a `Guid` factory and a binary format through a `string` factory, each with its own validation, no shared policy and no precedence question. `HasCorrespondingConstructor = true` adds a trusted direct-construction fast path that store materialization uses to bypass re-validation while wire input still validates — provenance-split admission as declared policy.
- The error type is type-level policy: a custom error implements the validation-error contract's `static abstract Create(string)` and is declared once via the validation-error attribute, propagating into every generated factory signature on that owner; per-factory override does not exist, and two owners needing different error types each declare their own.
- The span admission seam (`[ObjectFactory<ReadOnlySpan<char>>]`, `T : allows ref struct`) admits directly from character buffers, and the JSON deserializer routes through the same span factory for zero-copy parsing. Egress is asymmetric: the allocation-free reverse projection is the explicit `ToValue()` call — the generated implicit conversion targets the value type and silently selects the allocating path when the explicit call is omitted.

[POLICY_TABLE_CONSISTENCY]:
- A frozen policy table is the bounded-vocabulary surface for non-delegate behavior: one static field, one lookup returning the complete policy value — the caller never reconstructs behavior from key combinations. Span-based `Create` overloads admit collection-expression construction without intermediate allocation; `Keys`/`Items` return `ImmutableArray<T>`, the zero-allocation enumeration path (the dictionary-interface views allocate wrappers).
- The comparer is the policy definition, not infrastructure: a projection comparer keyed on a request's discriminant declares which dimension governs policy selection, collapsing payload differences onto one row. Comparer consistency is a law — the accessor governing a smart enum's identity and the comparer of any frozen table keyed on that enum must agree (ordinal versus ordinal-ignore-case mismatch produces silent lookup divergence), and string-keyed-object comparers exist precisely to delegate table equality to the enum's own key semantics.

[GENERATED_SURFACE_ASYMMETRIES]:
- Attribute asymmetry is the architecture signal: union case construction defaults open (`Public`) while value-object construction is always factory-gated (`Private`) — admission lives on value objects, dispatch lives on unions. On ad-hoc unions, privatizing the constructor also privatizes the implicit conversions in the same setting; `ConversionToValue = None` forces all member extraction through generated accessors, deleting ambiguous downcast paths. Regular `[Union]` exposes no constructor-access knob and no value-to-union conversion surface — its cases are subtypes, consumed by dispatch.
- `T1Name` renames every generated member for that slot (`Is{Name}`/`As{Name}` and dispatch parameter names) — a blast-radius rename, set before call sites exist. `DefaultStringComparison` (default ordinal-ignore-case) governs only generated hash/equality for string members of ad-hoc unions, never wire discriminators or parsing. `SkipEqualityComparison = true` deletes the whole equality surface for command/request unions where structural equality is meaningless, and makes backing-field layout irrelevant to correctness. Beyond five members, the ad-hoc forms end; the correct shape is a regular union with named case records.
