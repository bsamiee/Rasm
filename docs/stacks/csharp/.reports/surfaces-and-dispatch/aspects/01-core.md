# Aspect Architecture

[DEFINITION_TIME_WEAVING]:
- The attribute on a `partial` type declaration is the weave point; the generator emits a second partial file containing all cross-cutting members. No runtime proxy, no IL rewriting, no container interception.
- `[SmartEnum<TKey>]`, `[Union]`, `[ValueObject<T>]`, and `[ComplexValueObject]` declare the generated surface at definition time; the emitted partial file carries equality, conversion operators, `Switch`/`Map` dispatch, `ISmartEnum<TKey,T,TValidationError>`, `IParsable<T>`, `ISpanParsable<T>`, `IFormattable`, arithmetic operator interfaces, and serialization converters — all from a single declaration attribute.
- `SwitchMapMethodsGeneration.DefaultWithPartialOverloads` causes the generator to emit partial overloads of `Switch`/`Map` alongside the full-coverage overloads, giving the hand-written half a seam to override specific arms while generated dispatch covers the rest.
- `SwitchMapStateParameterName` (default `"state"`) on `[SmartEnum]`, `[SmartEnum<TKey>]`, and `[Union]` renames the thread-through parameter in every generated `Switch`/`Map` overload — aspect payload naming is owned entirely by the declaration site, not by downstream call sites.
- `NestedUnionParameterNameGeneration.Simple` vs. `Default` on `[Union]` controls whether intermediate type names appear in Switch/Map parameter names for hierarchical unions; `Default` produces `failureNotFound`, `Simple` produces `notFound`. The choice is load-bearing for callers: `Simple` insulates call sites from intermediate type renames, while `Default` guarantees uniqueness when leaf names collide across union families.
- `[ObjectFactory<T>]` (a C# 11 generic attribute) declares an additional factory route at definition time; `UseForSerialization`, `UseWithEntityFramework`, and `UseForModelBinding` each steer the corresponding cross-cutting integration aspect to route through that factory path rather than the primary key-based factory.
- `FactoryMethodGeneration.Default` on ad-hoc `[Union<T1,T2>]` derives the factory-method decision from type structure — triggers include type-parameter members, duplicate types, `object`, and interface arms — so the factory aspect's presence is determined at definition time by type topology, not by a manual flag.

[CONSTRUCTION_ADVICE_SEAMS]:
- `static partial void ValidateFactoryArguments(ref ValidationError? validationError, ref TKey value)` on `[ValueObject<T>]` and `[ComplexValueObject]` is the primary construction advice point. Both `ref` annotations are load-bearing: `ref validationError` sets the error without extra allocation, and `ref value` normalizes the raw input (casing, trimming, clamping) before the generated factory stores it. The hook executes inside the generator-produced `Create`/`TryCreate` body, before the type is instantiated.
- The validation error type is a typed aspect payload: `IValidationError<T>` requires `static abstract T Create(string message)`, constraining the error type as a static-abstract factory rather than a base class. A custom type implementing `IValidationError<TCustom>` where `TCustom : class, IValidationError<TCustom>` replaces the default `ValidationError` across the entire generated factory surface simultaneously by naming it in the `[ValueObject<T>]` declaration.
- `[UseDelegateFromConstructor]` on a `partial` method inside a `[SmartEnum]` class is a per-method aspect payload: the generator synthesizes a private delegate field with the method's matching signature, wires it through the constructor, and implements the `partial` method body as a delegate invocation. The `DelegateName` property forces a custom delegate type name, which matters when `ref` or `out` parameters would otherwise cause the generator to emit a `Func`/`Action` shape that cannot represent the exact signature.
- Partial constructors (C# 14) split a constructor into declaring and implementing parts. Only the implementing declaration may carry a constructor initializer (`: this(...)` or `: base(...)`), enforced by Roslyn diagnostic `ERR_PartialConstructorInitializer`. Partial constructors are the correct seam for source-generated initialization chains where the generator controls parameter admission and the hand-written half controls delegation to base.
- A partial property's definition part may not carry an auto-property body (`ERR_PartialPropertyDuplicateDefinition`), and any accessor declared in the definition part must be implemented in the implementing part (`ERR_PartialPropertyMissingAccessor`). Partial properties are strictly header/body splits — the definition owns the accessor contract, the implementation owns accessor bodies.
- A partial method with a non-void return type, `out` parameters, or `virtual`/`override` modifiers must carry an access modifier (enforced by `ERR_PartialMethodWithNonVoidReturnMustHaveAccessMods`), and any accessible partial method must have an implementing part. Legacy partial methods without accessibility modifiers may omit the implementing declaration and are silently dropped. Definition-time advice points for generators must use extended partial methods (with access modifier) when a non-void return or guaranteed execution is required.

[GENERIC_ATTRIBUTES_AS_TYPED_ASPECT_PAYLOAD]:
- `[SmartEnum<TKey>]` and `[ObjectFactory<T>]` are generic attributes (C# 11). The type argument is a verified compile-time type rather than a `Type` parameter, eliminating `typeof(T)` and enabling analyzer-side type checks in the source generator without runtime reflection.
- `[ObjectFactory<T>]` where `T` is `ReadOnlySpan<char>` routes the factory through span-based parsing — the zero-allocation admission path on .NET 9+. The generator emits `IObjectFactory<Owner, ReadOnlySpan<char>, TError>`; naming a non-span `T` produces a different generated interface. The `where T : allows ref struct` anti-constraint on `ObjectFactoryAttribute<T>` enables this: it permits `T` to be a ref struct without requiring it. Any custom `ref struct` input type is therefore also a valid factory discriminant.
- `[KeyMemberComparerAttribute<TComparer, TKey>]` and `[KeyMemberEqualityComparerAttribute<TComparer, TKey>]` attach a comparer type to the key member at definition time. The generator wires `TComparer` into the generated equality and comparison operator implementations; the comparer type implements `IComparerAccessor<T>` or `IEqualityComparerAccessor<T>`, providing a type-safe comparer injection path without any runtime constructor or property injection.

[SERIALIZATION_GRAMMAR_IDENTITY_ASPECTS]:
- Serialization is a definition-time aspect: `System.Text.Json`, Newtonsoft.Json, MessagePack, EF Core value converters, ASP.NET Core model binding, and Swashbuckle schema filters are generated as partial class members from the declaration attribute, conditioned by the presence of their respective companion packages. No runtime registration, no DI setup — converters are emitted into the type itself and discovered by framework reflection or source-generated registration.
- `SkipFactoryMethods = true` suppresses the generated serialization surface unless an `[ObjectFactory<T>]` with `UseForSerialization` is present. Without a factory route, no serializer can reconstruct the value object, so the aspect is withheld rather than generating broken converters.
- `ConversionOperatorsGeneration.Implicit` vs. `Explicit` vs. `None` on `[ValueObject<T>]` and `[SmartEnum<TKey>]` governs the generated `op_Implicit`/`op_Explicit` operators — the identity-at-boundary aspect. Implicit conversion makes the raw key type and the value-object type structurally interchangeable at call sites; explicit forces a cast and keeps the domain boundary visible.
- `OperatorsGeneration.DefaultWithKeyTypeOverloads` generates equality/comparison overloads that accept the raw key type alongside the value-object type. It allows comparisons between the primitive and the domain type without wrap/unwrap, at the cost of admitting unvalidated primitives into comparisons. Use `Default` (domain-type-only overloads) unless interop with unvalidated key values is explicitly required.
- `NullInFactoryMethodsYieldsNull` and `EmptyStringInFactoryMethodsYieldsNull` on `[ValueObject<string>]` convert null or empty-string input into `null` output from `Create`/`TryCreate` rather than a validation error. Setting `EmptyStringInFactoryMethodsYieldsNull = true` also forces `NullInFactoryMethodsYieldsNull = true` — the cascade is one-directional.

[COMPOSITION_TIME_POLICY]:
- `Schedule` is a lazy stream of `Duration` values. `|` is Union (minimum delay per step, length of the longer schedule); `&` is Intersect (maximum delay per step, length of the shorter); `+`/`Combine` is sequential append (all steps of the left, then all steps of the right). These are three algebraically distinct binary operations: `exponential(50*ms) | recurs(5)` terminates after 5 steps at exponential delays, while `exponential(50*ms) + recurs(5)` runs the exponential to exhaustion and then appends 5 zero-delay steps. `Schedule` is a `Semigroup<Schedule>` where `+` is `Combine`.
- `retryWhile(Schedule, IO<A>, Func<Error,bool>)` fires the predicate on every failure; returning `false` terminates the retry loop regardless of whether the schedule has remaining steps. `retryUntil` inverts the polarity — it stops when the predicate returns `true`.
- `CatchM<E, F, A>` is a predicate–handler pair. `@catch(Error, Func<Error, K<F, A>>)` constructs a `CatchM` keyed to a specific `Error` value (equality-matched); `@catch(Func<Error, bool>, ...)` is the predicate form. The `|` operator on `K<F, A>` with a right-hand `CatchM` is defined in `FallibleExtensions` via a C# 14 extension block constrained to `F : Fallible<E, F>` — the operator is not available on types that do not implement that trait. `catchOf<E, F, A>(Func<E, K<F, A>>)` matches all errors of subtype `E` and chains through matching errors via sequential `Bind` (requires `Monad<F>`, not `MonoidK<F>`).
- `K<F, A> | finally(K<F, B>)` executes `K<F, B>` regardless of whether the primary computation succeeds or fails, and returns the primary result. The `|` operator is defined in `FinalExtensions` for `F : Final<F>`, which every transformer stack satisfies. `finally` does not swallow errors — it re-propagates them after executing the cleanup side.
- The zero-argument `bracketIO(computation)` Prelude form calls `computation.Bracket()`, which calls `IO.WithEnv(env => env.LocalResources)` — a new `Resources` child scope with the existing cancellation source and sync context. Resources acquired via `use(...)` inside the computation are disposed when the scope exits. The three-argument form `bracketIO(acq, use, fin)` is implemented as `acq.Bind(x => use(x).Finally(fin(x)))` — a manual bracket without automatic resource tracking. The documentation recommends the zero-argument form when `use` registers resources with `use(...)`.
- Admission, identity, dispatch, serialization, and grammar attach at definition time because they are properties of the type itself — present at every call site, represented in the generated interface surface. Retry, recovery, and resource lifetime attach at composition time because they are properties of a specific call site — the same `IO<A>` can be retried with different schedules, and the type carries no trace of the policy.

[INTERCEPTORS_AND_MODULE_INITIALIZERS]:
- `[InterceptsLocation(version, data)]` on a static method routes invocations at the location encoded in `data` to the interceptor at compile time. `SemanticModel.GetInterceptableLocation(InvocationExpressionSyntax)` returns the opaque `InterceptableLocation` value; `CSharpExtensions.GetInterceptsLocationAttributeSyntax(InterceptableLocation)` produces the attribute syntax for the generated file. Interceptors are a source-generator-only weave surface — hand-written interceptors require manually correct location data, which is fragile.
- Interceptors are constrained to invocation expressions. A signature mismatch produces `ERR_InterceptorSignatureMismatch` (incompatible types) or a warning (compatible but not identical — for example, differing parameter names). Location encoding is versioned (`InterceptableLocation.Version`); the version integer is an argument to `[InterceptsLocation]` so format changes do not silently corrupt existing interceptors.
- `[ModuleInitializer]` marks a static, non-virtual, parameterless, void-returning method as a CLR module initializer — it runs once before any code in the module accesses types from that module. The method must not be generic, must not be in a generic type, and must not be a property accessor, constructor, or operator. It is the last-resort aspect for registration, codec installation, and process-static initialization that must be unconditional and before first use.
- `[ModuleInitializer]` ordering across assemblies is undefined; within a single assembly, module initializers run in source-file order, then in declaration order within files. Aspect registrations that depend on ordering must either consolidate into one method or use explicit dependency calls, not cross-method ordering across multiple `[ModuleInitializer]` declarations.

[STACKING_ORDER]:
- Definition-time aspects from Thinktecture emit into a single partial file per source-generator run. Multiple attribute types on one type produce independent generator passes; each pass targets disjoint member names, so no stacking conflict arises.
- Composition-time aspects stack left-to-right through the monadic pipeline. `computation.Catch(...) | @catch(...) | finally(cleanup)` evaluates in that order: the first `.Catch` handles errors from `computation`; `| @catch(...)` handles errors escaping the first handler; `| finally(cleanup)` runs unconditionally. `finally` must be the outermost `|` operand; a `@catch` between a `finally` and the computation is unreachable for errors that propagate through the catch handler, because `finally` re-propagates errors after cleanup.
- When a `[Union]` `Switch` dispatch carries `[UseDelegateFromConstructor]`-backed per-case behavior and a composition-time retry wraps the dispatch call site, the retry policy sees the entire dispatch as one atomic unit. Per-case retry granularity requires extracting the case arm into a separate `IO` computation before the switch. Definition-time case dispatch and composition-time error handling are orthogonal and do not compose at case granularity.
