## [2] DOCTRINE (non-negotiable — every change is judged against this)

- **SINGULAR advanced polymorphism — one shape per concept.** Unified rails; one `Detail`/`Report`/`Envelope`/`Completed`/`Fault`; one Engine; one routing surface. A new shape for an existing concept is REJECTED — extend a tagged-union variant / enum member / catalog row instead. NO rat-nesting (type-of-type alias, struct-in-struct that should be one, constant-of-constant chain), NO unnecessary differentiation (two near-identical types/params/factories for one concept), NO string proliferation (a new string lands ONLY in an enum member or a catalog row), NO dual paradigms (pick the one denser form and collapse the other into it).
- **Advanced ADT + AOT/aspect-style throughout.** Axis `StrEnum`s carry behavior payloads via `__new__` (`Runner.prefix`, `Input.flag/scoped`, `Language.strategy/suffixes`, `Mode.stream/writes`); one member instance serves the Cyclopts token, the `msgspec` wire value, and the `match` key at once. `msgspec` tagged unions with explicit short tags + `forbid_unknown_fields` for closed variant spaces. Cross-cutting concerns live ONLY as composable aspect `Layer`s (`checked ▷ logged ▷ traced ▷ retried`) at the two seams — never inline. Push behavior to decoration/composition (AOT), not duplicated method bodies.
- **Algorithmic, programmatic, universal bodies.** Every code body is data-driven: `match`/folds (`functools.reduce`)/comprehensions/`expression` Result rails/dispatch tables — ZERO `if`/`for`/`while`/`try` in domain logic (`try` ONLY at a marked codec/spawn/FS boundary seam). Variables and bounded vocabularies over raw literals; reusable projections over one-off branches. Logic is generated from the catalog/enums/discriminants, not hand-enumerated. Names are universal (`countBy`, not `countByIp`); narrow variants are rejected.
- **Deep external-lib leverage at FULL power (verify against `.venv/.../site-packages`, never training data).** `msgspec` (enc/dec hooks, `defstruct`, `Meta`, `convert`, `inspect`, `Raw`), `pydantic`/`pydantic-settings` (computed fields, `field_validator`/`model_validator`, `Annotated` constraints, alias choices, `model_copy`), `expression` (`map2`/`Block.collect`/`partition`/`sequence`/`pipe`/`@effect`), `anyio` (memory streams, `TaskGroup.start`, `to_thread`/`to_process`, `CapacityLimiter`, `Event`), `annotationlib` (`Format.STRING`, PEP 749), `beartype.vale.Is[]`, `opentelemetry` (Links/baggage), `fsspec`/`universal-pathlib`, `psutil`, `stamina`, `structlog`, `cyclopts`, `tree-sitter`, `importlib.metadata`/`inspect`, plus CLI `ast-grep`/`rg`/`fd`. Advanced typing: ultra-advanced `StrEnum`/`IntEnum` payloads, `Protocol` ports (structural DI, never `abc`), `NewType`/`Annotated` atoms, PEP 695 `**P`/`type` aliases, `assert_never` exhaustiveness.
- **No fragile logic, no hardcoding, no project coupling.** Point-and-go from any cwd/CI; no magic numbers (push to a named, validated setting or an enum payload); no path/namespace/one-off-symbol coupling (rules describe semantic shapes); sentinels projected to `Option`/typed absence at the boundary; weak/unbounded types replaced where the domain is precise. Resilience is INHERENT — a command cannot be mis-invoked; bad input degrades to a typed `Fault`/`EMPTY`, never a traceback.
- **No spam, no flat code, no dead surface.** No single-call thin helpers, no parallel near-duplicate factories, no vestigial fields/branches; root-shrink `type:ignore`/`# noqa` clusters rather than spreading them. Density is concept count, not byte count — NEVER delete functionality to hit a number; collapse it into a denser polymorphic surface in the SAME file.

## [4] DIMENSION I — CODE QUALITY (always first)

Workflow (line-by-line, adversarially verified — kill false positives, past audits produced false "P0 bugs"): map → critique → collapse → review-and-fix by INDEPENDENT agents → re-gate. Targets:
- **Collapse residual surface** into denser polymorphic shapes: merge mergeable pairs, fold single-use helpers into their caller, unify near-duplicate factories, stack loose defs that lego together. Target a real reduction where it preserves capability; report before/after concept counts.
- **Push ADT/AOP/typing further:** weak/unbounded types → precise enums/`Annotated`/`Protocol`; imperative leaks → folds/match; sentinels → `Option`; mutable accumulation → immutable folds; exception-as-control-flow → typed rails; cascading/nesting flattened; advanced `pydantic`/`msgspec`/`expression`/`annotationlib` features adopted where they densify.
- **Root-shrink suppressions** (`type:ignore`/`# noqa`) to the irreducible minimum (document each survivor's why). Strip dead vestiges, fix latent/hidden bugs, reduce fragility.
- Output is judged on: fewer concepts, denser polymorphism, stronger types, deeper lib leverage, zero new agent-facing surface, gates green cold + live-verified.

## [5] DIMENSION II — FUNCTIONALITY DEEPENING (inherent, never a flag)

Workflow (read all current functionality + the philosophy; propose; value-filter; implement; review-and-fix). Targets — improve WHAT the tool does without adding agent surface:
- **Integrate already-present libs MORE powerfully and UNIVERSALLY (not singleton methods):** `fsspec`/`universal-pathlib` (richer artifact-store behavior, backend-agnostic across every path, atomic write/rotate, glob/retention used everywhere a path is touched — one universal surface, not per-rail one-offs); `psutil` (deeper resource/liveness/fleet introspection woven into the lease + diagnostics + governor, universally); hardening (the resilience boundary made universal — every spawn/codec/FS seam degrades typed, every command foolproof); `pydantic`/`msgspec`/`expression`/`anyio`/`stamina`/`opentelemetry` pushed to their full capability where it adds real value.
- **Push each external lib further for value:** for every lib in the manifest, identify the capability you are NOT yet using that would densify a body, harden a seam, or enrich a `Match`/`Detail`/`notes` return — adopt it surgically.
- **Extend a CATEGORY of functionality with optimized/surgical changes:** richer per-domain evidence (more powerful typed `Detail`/`Match` WITHOUT a new shape), better auto-routing/auto-detection, deeper cross-language coverage (all five languages integrated through every applicable arm), faster hot paths (profile; the fastest correct primitive the ecosystem offers — `code` and every hot path approach Rust-level speed without dropped capability), richer auto-diagnostics on fault.
