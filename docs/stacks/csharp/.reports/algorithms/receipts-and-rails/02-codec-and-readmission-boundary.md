# Receipt Codec and Re-Admission Boundary Law

[KEYED_CODEC_IS_THE_RE_ADMISSION_GATE]:
- A single-keyed generated owner serializes as its bare key, not as a wrapping object: a `[ValueObject<double>]` whose key member is `_value` round-trips on the wire as the raw scalar, and the generated converter's deserialize path routes through the same `Validate(key, formatProvider, out item)` factory that admits a live value, so deserialization IS re-validation. There is no separate "parse then check" hop — the wire decode and the conservation check are one call. A receipt field typed as a generated owner therefore cannot be deserialized into an invalid state; a malformed persisted tolerance, rank, or condition number fails the factory at decode time and never reaches the receipt constructor.
- The keyed-codec contract is what makes a numeric receipt's stored fields tamper-evident at rest: a hand-edited persisted `Rank` that violates `0 <= Rank <= min(m,n)` is rejected by the value-object factory the instant the field is read back, before any cross-field conservation predicate runs. Storing a receipt field as a bare `int`/`double` rather than as a generated keyed owner discards this gate — the bare primitive deserializes unconditionally and the only validation that fires is whatever the receipt's own `Create` factory re-runs, which is one hop later and easy to omit.
- `SkipKeyMember` plus a manual key member, and `KeyMemberName` retargeting `_value`/`Value`, change the wire shape's source field but not the round-trip-through-factory law; `AllowDefaultStructs` is the one switch that breaks it — enabling default construction lets `default(TReceiptField)` bypass the factory entirely, so a struct-keyed receipt field with `AllowDefaultStructs = true` admits an unvalidated zero on both `default` construction and any deserializer that materializes structs field-by-field. For a receipt field the default-struct allowance is the rejected form: the zero singular value, zero step size, or zero tolerance must be an explicit factory-admitted value, never a silent default.
[SERIALIZATION_FRAMEWORK_AS_A_DEFINITION_TIME_ASPECT]:
- The serialization codec is woven at definition time by one attribute flag, not authored: the framework set is a bitfield (`SystemTextJson`, `NewtonsoftJson`, `MessagePack`, with `Json`, `All`, `None` as named combinations) selected on the owner attribute, and the generator emits a `JsonConverter`, a Newtonsoft converter, and a `MessagePackFormatter` per selected framework. A receipt that crosses a binary persistence boundary selects `MessagePack`; one that crosses a text/HTTP edge selects `SystemTextJson`; selecting `All` emits every codec whether or not the boundary uses it, which is dead generated surface unless the receipt genuinely traverses multiple transports. The flag is the policy value — the codec is derived from it, never hand-written, and a hand-rolled converter beside a generated owner is the anti-pattern the bitfield exists to delete.
- `ObjectFactoryAttribute(Type, UseForSerialization: ...)` declares an additional admission factory and binds it to a named framework set for serialization specifically, so a receipt can carry one validation path for in-process construction and a distinct, serialization-only factory for the wire — the deserialize path is contractually steered to the factory whose `Validate` re-checks the persisted payload. `UseWithEntityFramework` and `UseForModelBinding` are sibling routing flags on the same attribute: the same generated owner becomes the column codec and the boundary-binding codec without a parallel DTO, so a persisted receipt field is one type across the rail, the wire, the relational column, and the request boundary — no per-surface shape proliferation.
- `IObjectFactory<TKey>.Validate(key, IFormatProvider, out item)` threads an `IFormatProvider` through the deserialization admission, so a culture-sensitive numeric field (a tolerance parsed from a textual wire form) is admitted under an explicit, recorded format rather than ambient culture. A receipt codec that ignores the provider and parses with thread-default culture is non-deterministic across hosts; the provider is co-equal evidence the codec must pin, exactly as the algorithm's tolerance constants are.
[ERROR_RAIL_DOES_NOT_CROSS_THE_WIRE_AS_DATA]:
- The error carriers are records, not data contracts: an `Expected` error is `(Code:int, Message:string, Option<Error> Inner)` and an `Exceptional` error wraps a live `Exception`. Only the `Expected` shape is wire-faithful — its three fields are pure data and its EQUALITY IS BY `Code` ALONE, so two persisted expected errors with the same code compare equal regardless of message or inner. An `Exceptional` error is not cleanly serializable as data and its identity rides the captured exception, so a receipt that must persist a failure outcome converts every exceptional error to an expected error with a domain code at the boundary BEFORE serialization. Persisting an exceptional error is the rejected form: the deserialized side gets a degraded exception or a lost identity, and code-keyed re-admission dispatch silently breaks because the round-tripped error has no stable code.
- Because expected-error equality is code-keyed, the receipt's failure provenance survives the round trip exactly when the code is the discriminant and the message is treated as non-load-bearing display text. A re-admission check that dispatches on `error.HasCode(domainCode)` against a deserialized expected error is sound; one that pattern-matches on `Message` substring is not — the message is not part of identity and a codec is free to drop or localize it. The persisted receipt's typed-failure surface is therefore a bounded code vocabulary, never free text.
- `ManyErrors` is the monoid carrier (`Errors.None` is zero, `operator+` appends), and it serializes as the sequence of its constituent errors — but only if every constituent is expected. A receipt factory that accumulated several independent conservation violations into one `ManyErrors` must downgrade each exceptional member to an expected code before the aggregate is persisted, or the wire form is partially lossy. The accumulated-violation set survives the boundary as a code sequence whose order is the diagnostic order the factory produced; the codec preserves order, so the persisted aggregate replays the same left-to-right violation trail at re-admission.
[STORAGE_SHAPE_IS_THE_NUMERIC_WIRE_DISCRIMINANT]:
- A decomposition's matrix and vector evidence does not serialize through the matrix wrapper; it serializes through the underlying storage contract, and the storage CASE is the discriminant the codec persists. Dense storage is column-major flat data; sparse storage is compressed-row with separate value, column-index, and row-pointer arrays; diagonal storage carries only the diagonal band. A receipt that persists a factorization factor as a dense flat array when the live factor was sparse-compressed inflates the wire payload from O(nnz) to O(n²) and discards the sparsity structure that is itself evidence — the storage case records whether the algorithm produced a banded, sparse, or full factor, and collapsing every factor to dense erases that fact.
- The persisted numeric payload must therefore carry the storage discriminant as a named honesty marker alongside the values: a sparse factor round-trips as `(values, columnIndices, rowPointers, rowCount, columnCount)` and a re-admission check verifies the structural invariant `rowPointers.Length == rowCount + 1` and `columnIndices.Length == values.Length` before any numeric conservation predicate runs. A receipt that stores only a flattened value array without the index structure cannot be re-admitted into a sparse solver — the structural skeleton is unrecoverable, and the codec that drops it has discarded re-admission capability, not merely bytes.
- Singular-value and eigenvalue vectors persist as their storage payload, and the descending-order invariant (`S[0] >= S[1] >= ... >= 0`) is a structural assertion the deserialize-side re-admission re-checks; the order is not re-sortable at re-admission without invalidating every stored `ConditionNumber`, `Rank`, and `L2Norm` derived from index 0. The codec persists the vector in its produced order and the re-admission predicate asserts the order rather than restoring it — a re-sort on decode is data corruption masquerading as normalization.
[STATUS_AND_POLICY_ENUMS_AS_KEYED_WIRE_VALUES]:
- A behavior-carrying status or policy enumeration persists as its smart-enum key, and the generated codec round-trips through the same keyed factory, so a wire value outside the closed case set (a corrupted iteration-status code, an unknown regression-method tag) is rejected at decode rather than silently mapped to a default case. The keyed wire shape is what keeps the six-case iteration-status partition — converged, budget-exhausted, diverged, broke-down, cancelled, failed — intact across the boundary: the budget-exhausted case persists as a distinct key and re-admits as a distinct, retryable status, where a raw integer wire form would let a corrupted value alias into the failure case and destroy the retry distinction.
- The serialization framework bitfield on the smart enum defaults to all frameworks, and a persisted policy enum (the symmetry assertion, the QR method, the Fourier-convention bitfield, the regression method) is the densest possible wire evidence: one keyed value reconstitutes the full behavior row, including the constructor delegates that carry the case's algorithm branch. A receipt that persists a policy enum as a prose string rather than its key cannot re-admit the behavior — the prose is one hop from the vocabulary that selects the algorithm, and the keyed codec exists precisely to keep that hop at zero.
- A `[Flags]` convention enum (the spectral exponent-and-scaling bitfield) persists as its integer bitfield, and the re-admission test is a bitwise-equality check against the policy the downstream stage expects; persisting it as a decomposed pair of booleans or a name list splits one atomic discriminant into fields that can drift out of sync. The bitfield is the canonical wire form because the two orthogonal facts it encodes — exponent direction and scaling magnitude — are recoverable by mask, and a round-tripped bitfield re-admits both facts in one comparison.
[FACTORY_DUALITY_AS_A_CODEC_PAIR]:
- The construct-side `Create` factory and the deserialize-side re-admission factory are the two halves of one codec law, and they must run the same conservation predicates: `Create` validates a freshly computed decomposition before sealing the receipt; re-admission replays the identical structural checks against the persisted payload without re-running the O(n³) decomposition — it is O(fields). The asymmetry is the rejected form: a `Create` that asserts rank-nullity and descending order paired with a deserializer that trusts the bytes lets a tampered receipt enter the rail through the wire that could never have entered through construction. The single source of truth is one private validation routine both factories call, so the codec round trip is closed under the same invariant set as in-process construction.

```csharp
[ValueObject<int>(SkipIComparable = true)]
[ValidationError<RankError>]
public readonly partial struct Rank
{
    static partial void ValidateFactoryArguments(ref RankError? error, ref int value) =>
        error = value < 0 ? RankError.Create($"negative rank {value}") : null;
}

[Union]
public abstract partial record FactorReceipt
{
    public sealed partial record Sparse(double[] Values, int[] ColumnIndices, int[] RowPointers, int Rows, int Cols) : FactorReceipt;
    public sealed partial record Dense(double[] ColumnMajor, int Rows, int Cols) : FactorReceipt;

    public Validation<Error, FactorReceipt> ReadmitWire() =>
        Switch(
            Sparse: s => (Check(s.RowPointers.Length == s.Rows + 1, Codes.Skeleton),
                          Check(s.ColumnIndices.Length == s.Values.Length, Codes.Skeleton))
                             .Apply((_, _) => (FactorReceipt)s).As(),     // applicative: BOTH violations
            Dense:  d => Check(d.ColumnMajor.Length == d.Rows * d.Cols, Codes.Skeleton)
                             .Map(_ => (FactorReceipt)d));
}
```

- The persisted receipt's failure rail and its success payload converge on one carrier on decode: the deserializer yields `Validation<Error, TReceipt>` so an admitted receipt and the accumulated set of re-admission violations share the rail the in-process factory used, and the boundary conversion to the terminal `Fin` rail is one hop on the far side. A codec that returns a nullable receipt plus an out-parameter error list re-introduces the split-rail shape the carrier exists to collapse; the accumulating applicative on the validation side is what lets one round trip report every structural violation in the persisted payload at once rather than aborting on the first malformed field.
