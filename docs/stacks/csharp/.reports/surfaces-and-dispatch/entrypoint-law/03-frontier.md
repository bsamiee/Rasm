# Entrypoint Law — Frontier

[SCHEDULE_OPERATOR_TYPE_ALGEBRA]:
- `Schedule` and `ScheduleTransformer` are distinct types with a strict operator lattice. Methods returning `Schedule`: `spaced`, `linear`, `exponential`, `fibonacci`, `upto`, `fixedInterval`, `windowed`, and cron variants. Methods returning `ScheduleTransformer`: `recurs`, `maxDelay`, `maxCumulativeDelay`, `jitter`, `decorrelate`, `resetAfter`, `repeat`, `intersperse`; static fields `NoDelayOnFirst`, `RepeatForever`, `Identity` are also `ScheduleTransformer`.
- The `|` operator is defined on `Schedule` for three combinations: `(Schedule, Schedule)`, `(Schedule, ScheduleTransformer)`, and `(ScheduleTransformer, Schedule)` — all returning `Schedule`. `ScheduleTransformer` carries an implicit conversion to `Schedule`, so `transformer1 | transformer2` resolves via the implicit conversion of the left operand followed by the `(Schedule, ScheduleTransformer)` overload. The chain `Schedule.NoDelayOnFirst | Schedule.recurs(5) | Schedule.exponential(seed)` is valid: the left-associative evaluation produces `(NoDelayOnFirst | Schedule.recurs(5))` — `(ScheduleTransformer, ScheduleTransformer)` resolved via implicit conversion — then `(Schedule, Schedule)` for the final step.
- The `+` operator on two `ScheduleTransformer` values composes them as functions: `(t1 + t2).Apply(s)` is `t2.Apply(t1.Apply(s))`. Use `+` to combine multiple transformers into one before applying; use `|` to combine schedules and transformers into a running schedule. `Schedule + Schedule` is sequential append — when the first schedule exhausts its steps, the second continues.
- `Schedule | Schedule` is union: runs to the length of the longest, yields the minimum of both delays at each step. `Schedule & Schedule` is intersect: runs to the length of the shortest, yields the maximum of both delays. `Schedule + Schedule` is combine: sequential append — the second schedule begins when the first is exhausted. These three semantics are orthogonal and the symbols are not interchangeable.
- `ScheduleTransformer`'s implicit conversion to `Schedule` wraps the transformer in `Schedule.Forever | t`. An unwrapped `Schedule.recurs(5)` passed to `retry` is therefore equivalent to `retry(Schedule.Forever | Schedule.recurs(5), computation)`, which is finite — the implicit conversion is the mechanism by which bare transformers are valid retry-schedule arguments.

[EXTENSION_BLOCK_STRUCTURAL_CONSTRAINTS]:
- C# 14 extension blocks must be declared inside a top-level, non-generic, `static class`; placing an `extension(T)` block at top-level or inside a generic class produces CS9283. Multiple `extension(SameReceiver)` blocks in the same static class are permitted and additive; multiple static classes each containing an `extension(SameReceiver)` block are also additive across the compilation, enabling test-assembly probing methods in a separate extension block without modifying the production static class.
- The `extension(T)` block itself does not accept an access modifier (CS0106). Visibility is controlled per-member inside the block: `public`, `internal`, and `private` all compile and carry normal access semantics. A `private` member in an extension block is inaccessible from outside the static class containing the block — equivalent to a private static method on that class.
- Extension blocks cannot access `private` members of the receiver type (CS0122). The dispatch entrypoint written in an extension block must route entirely through the receiver's public API — the generated `Switch`/`Map` methods, public case properties, and explicit interface members.
- Extension blocks support property syntax: `public bool IsA => receiver is TypeA;` compiles without restriction, producing a non-virtual static method with a receiver parameter. This makes extension blocks suitable for attaching computed predicate properties to a union type without touching the generated partial.
- `[OverloadResolutionPriority]` applies to members inside extension blocks with the same semantics as on regular methods. A `ReadOnlySpan<T>`-based extension block member can be given priority over a `string`-based member to make the span path win at call sites that pass a literal.

[FROZENDICTIONARY_PRECISE_SURFACE]:
- `FrozenDictionary<TKey, TValue>.GetValueRefOrNullRef(TKey)` returns `ref readonly TValue`. Absence is detected with `Unsafe.IsNullRef(in slot)`. The dispatch idiom: `ref readonly var slot = ref table.GetValueRefOrNullRef(key); if (Unsafe.IsNullRef(in slot)) return Fin<Result>.Fail(...); return slot(request);` achieves zero-allocation lookup and invocation for `Func<Cmd, Fin<Result>>` values.
- `FrozenDictionary<TKey, TValue>` carries `[CollectionBuilder(typeof(FrozenDictionary), "Create")]`. A collection expression `FrozenDictionary<string, Handler> t = [(k1, h1), (k2, h2)];` lowers directly to `FrozenDictionary.Create<string, Handler>(span)` — no `new Dictionary<>()` intermediate, no `.ToFrozenDictionary()` call. The static field initializer `static readonly FrozenDictionary<string, Handler> _t = [(k1, h1), (k2, h2)];` is the canonical construction form.
- `FrozenDictionary<TKey, TValue>.AlternateLookup<TAlternateKey>` is a `readonly struct` with `ContainsKey(TAlternateKey)`, `TryGetValue(TAlternateKey, out TValue)`, and an indexer. The constraint `where TAlternateKey : notnull, allows ref struct` enables `ReadOnlySpan<char>` as the key type when the base comparer implements `IAlternateEqualityComparer<ReadOnlySpan<char>, string>`. Storing the `AlternateLookup` struct in a `static readonly` field avoids reconstructing the lookup delegate captured inside the struct on each call.
- `FrozenDictionary.Create<TKey, TValue>(IEqualityComparer<TKey>?, params ReadOnlySpan<KeyValuePair<TKey, TValue>>)` is the comparer-accepting overload. Passing the comparer at construction is the only way to configure alternate-key lookup; there is no post-construction comparer replacement.

[SMARTENUM_KEY_AND_SPAN_FACTORY]:
- The source generator produces `Key` as the default key property name for `[SmartEnum<TKey>]` types (or `_key` for private field access modifiers). `IConvertible<TKey>.ToValue()` is an explicit interface implementation delegating to `Key`. Direct use of `.Key` is the correct form at call sites and in table initialization; the `IConvertible<TKey>.ToValue()` path is available but adds unnecessary ceremony.
- On .NET 9+, the generator unconditionally adds `IConvertible<ReadOnlySpan<char>>` and `IObjectFactory<T, ReadOnlySpan<char>, TValidationError>` to every `[SmartEnum<string>]` type. Every string-keyed smart enum therefore supports span-based construction via `SmartEnum.Validate(ReadOnlySpan<char>, IFormatProvider?, out T?)` without any manual attribute. The full zero-alloc dispatch table pipeline: `Kind.Items.ToFrozenDictionary(k => k.Key, StringComparer.OrdinalIgnoreCase).GetAlternateLookup<ReadOnlySpan<char>>()` routes span-key lookups to the case's delegate behavior without allocation.
- `[ObjectFactory<ReadOnlySpan<char>>]` on a `[ValueObject<string>]` generates a span-accepting `Validate(ReadOnlySpan<char>, IFormatProvider?, out T?)` interface contract. The `ObjectFactory<T>` attribute is declared `where T : allows ref struct`, enabling this. Implementing this static method directly on the partial class provides a zero-alloc admission path from incoming wire buffers without converting to a heap-allocated string first.

[UNION_COARSE_DISPATCH_OVERLOAD]:
- `[UnionSwitchMapOverload(StopAt = [typeof(AbstractCase)])]` is an `AllowMultiple = true` attribute that generates an additional `Switch`/`Map` overload alongside the default full-coverage overload. The coarse overload treats `AbstractCase` and all its siblings as atomic — it does not descend into `AbstractCase`'s subtypes. A nested `[Union]` hierarchy with this attribute produces both the full leaf-level overload and a coarse overload that delivers `AbstractCase` without decomposing it.
- The coarse overload is architecturally distinct from `DefaultWithPartialOverloads`. A partial overload omits some leaf cases and has a fallback arm for unhandled cases. The `[UnionSwitchMapOverload]` coarse overload is exhaustive at the stop boundary — it has no fallback arm and covers all cases at the stop level. It is the correct form when some callers need full decomposition and others need grouped behavior, and both caller populations must be exhaustively covered.
- Multiple `[UnionSwitchMapOverload]` attributes on one union generate multiple additional overloads, each with distinct stop-at boundaries. A union with three nesting levels can expose three dispatch granularities simultaneously. The full-coverage overload from `SwitchMapMethodsGeneration.Default` is always the zeroth generated form; each attribute appends one more.

[PRIVATE_CONSTRUCTOR_SEALING_MECHANICS]:
- `ConstructorAccessModifier = UnionConstructorAccessModifier.Private` on `[Union]` seals the constructor and makes all generated implicit conversion operators private simultaneously. When `FactoryMethodGeneration = None`, case construction becomes impossible from outside the declaring class. The only surviving construction path is a `public static` factory method inside the partial class — accessible because it is inside the partial class, not outside it.
- `ConversionFromValue = ConversionOperatorsGeneration.None` removes the implicit conversion operators entirely, not just their accessibility. Pattern matching (`is CaseName c`) still works because pattern matching uses type testing, not conversion operators. `Switch` still works because it dispatches through the generated discriminator field, not through conversion. Setting `None` is the correct choice when the union must prevent accidental implicit upcasting in expression contexts while preserving the full dispatch surface.
- `ConversionToValue = ConversionOperatorsGeneration.None` removes explicit downcast operators entirely. With `ToValue = None`, the only ways to extract a case value are through the generated `Switch`/`Map` methods or the generated `.IsX`/`.AsX` properties. This eliminates the runtime `InvalidCastException` risk from downcasting the union to a case before switching, and is the correct form for a dispatch union whose callers should never hold a typed case reference.
- The combination `ConversionFromValue = None, ConstructorAccessModifier = Private, FactoryMethodGeneration = None` produces a union whose only construction surface is a hand-written static factory inside the partial class, with `Switch` and `Map` fully operative. This is the maximum-sealed entrypoint shape: input enters only through the one public factory, output routes only through the dispatch method, and no type-system backdoor exists.

[SCHEDULE_RETRY_RESOURCE_SCOPE_PRECISION]:
- `retry(Schedule, K<IO,A>)` releases resources registered via `use<A>` inside the computation on each failed attempt before the next retry fires. The docs state: "Any resources acquired within a retrying IO computation will automatically be released if the operation fails. So, successive retries will not grow the acquired resources on each retry iteration." A `bracketIO`-wrapped computation inside a `retry` loop acquires and releases cleanly per attempt.
- `retryWhile(Schedule, K<IO,A>, Func<Error, bool>)` and `retryUntil(Schedule, K<IO,A>, Func<Error, bool>)` combine schedule-based delay with a predicate guard. The predicate gates whether to retry at all; the schedule gates how long to wait before the next attempt. Both must permit a retry for it to proceed. `repeatWhile(Schedule, K<IO,A>, Func<A, bool>)` is the success-value counterpart: it inspects the successful result and re-runs the computation while the predicate holds, applying the schedule between re-runs.
