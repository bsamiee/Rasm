## THREE STRONGEST ATTACKS ON THE MIN POLE

**ATTACK 1 — "the `ls`-gate makes the doctrine read actually happen" is theater for the READ, real only for the LIST.** MIN will argue the enumerate-first `ls` immunizes the route. It immunizes the LIST against staleness — an agent cannot fake having run `ls docs/stacks/csharp/`, so the atlas can never go silently stale the way the four inline paraphrases did. That is real and I concede it. But the `ls` forces ZERO top-to-bottom INGESTION of the pages it lists. The panel's own adherence audit is dispositive (`infra[05]`, verbatim): *"READ ... in FULL — never a partial, skim, grep-jump, or section-sample for a fable agent holding a 4-page batch under a multi-thousand-token PRE preamble is aspirational; agents trim it."* So a PURE route trades a drift-prone-but-PRESENT inline law (mostly correct, physically in the prompt, unskippable) for an aspirational-and-ABSENT one (behind a tool call the agent satisfies with a 40-line `ls` and a skim). The drifted inline at least puts the law where the agent cannot skip it. Routing a law without a schema-forced successor that PROVES the read landed (`beyondMap`, `seamsTouched`) is a NET REMOVAL of a guard, not a modernization. MIN must pair every route with a successor check or concede the route weakens enforcement of the exact reads the quality bar depends on.

**ATTACK 2 — "folder-disjoint lanes give single-writer safety by construction" is disk-refuted.** MIN's threading position rests on `threading §2.3`: "Folder = the single-writer boundary, because the contended shared files (`ARCHITECTURE.md`/`README.md`) are folder-scoped." Disk says otherwise: `fd` confirms `libs/csharp/Rasm/ARCHITECTURE.md` and `README.md` live at the **PACKAGE root**, and NONE exist inside `.planning/`. Every sub-folder in the package (Parametric, Meshing, Drawing, Processing) shares that ONE index doc. The engine's own `folderOf` (`RB:541`) already returns the PACKAGE (`Rasm`), not the sub-folder — so either the lanes are package-granular (no intra-package parallelism gain) or they are sub-folder-granular and two concurrent lanes write the SAME `ARCHITECTURE.md`. The threading and execution-fit dossiers CONTRADICT here (F7 got it right: "the package root, one level ABOVE `.planning/`"); disk breaks the tie for F7. "Anchored-Edit-reapply" is — in the threading dossier's own words — "mandated in-prompt, not enforced by a lock." A retry-on-conflict convention is not a concurrency guarantee under 3+ contending writers; under a true race it livelocks or silently drops rows as each writer re-reads a different intermediate state. MIN's parallelism claim shrinks to "across PACKAGES freely"; within a package the index-doc write MUST serialize (F7's per-package index writer), and the safe partition is package-granular, not folder-granular.

**ATTACK 3 — "reconcile removal was right, in-pass repair for everything" is correct for seams and catastrophic for shared-central-file motion.** The reconcile-free law (`RB:4`, `campaign-method.md:41` "the agent who sees the defect owns the fix; deferral is a defect") is CORRECT for a uniquely-owned seam counterpart — the agent who disturbs a specific sibling page knows it and fixes it in-pass. It is CATASTROPHIC for a shared-central-file motion, and the DECISION proves it verbatim: `[10]:262` parked the GShark manifest drop, the `api-gshark.md` catalog delete, the `FAB`/`AppHost`/`Compute`/`Persistence` counterpart edits, and the four `[05]` acceptance dry-runs on *"the engine's terminal reconcile [with] whole-repository write authority; no post-leg residual pass exists"* — a stage the working tree DELETED (`RB:4`, composition ends at the review pool `RB:724-728`). A manifest pin cannot enter `deletePages` (it has no `capturedIn`, required by `PLAN_SCHEMA` `RB:45`); it has no single owning page (3 concurrent fable authors `RB:712-714` could each touch `Directory.Packages.props` → guaranteed race); and NOTHING verifies the drop.
