# Federation Charter Amendment Supersession And Closure Lattice

# Doctrine Family Invariants

- Amendment activation — new shape kind, federation edge type, violation signal, assurance invariant, propagation edge, or reconciliation row absent from prior federation witness tables — requires charter row before merge.
- Supersession activation — wave row contradicted by later wave without documented supersession witness — charter blocks merge until precedence stack reconciles.
- Closure activation — bounded context or federation graph reaches L5 readiness — charter emits closure commit row.
- Charter graph must remain acyclic — bidirectional amendment edges between waves indicate missing supersession witness or duplicate authority claim.
- Federation assurance temporal law governs runtime promotion, reconciliation, and sustain phases — charter evolve phase governs doctrine file edits only; amendment units do not substitute for post-reconciliation certify or bypass open E-signals on federation runtime handoffs.

- Seven charter invariants hold across the closed doctrine family — violation in any federation report without amendment row is charter defect even when runtime federation proof passes.
- **C1 single taxonomy** — stack layers, projection lattice roles, shape kinds, and federation edge types declared in integration doctrine and remain the sole taxonomy — later federation reports add obligations, not parallel names.
- **C2 report monotonicity** — higher-index federation reports bind later lifecycle phases only — promotion, restoration, assurance, charter — no report redefines interior stack altitude.
- **C3 witness unity** — propagation, reconciliation, assurance, and charter witness tables share vocabulary-owner import discipline — duplicate token or signal lists in report bodies are drift, not amendment.
- **C4 amendment atomicity** — doctrine edits land as promotion units with fan-out across obligated federation report sections and concept-axis binding paragraphs — partial report patch without charter unit is merge blocker.
- **C5 supersession explicitness** — retired row carries `superseded_by` witness pointer — silent deletion of enforcement or collapse-test bullets is charter violation.
- **C6 closure totality** — every federation lifecycle phase enforceable row across federation reports — orphan phase without witness is incomplete family.
- **C7 axis non-capture** — charter governs federation binding and report precedence — concept axes retain mechanics; charter never restates pydantic-core, replacement algebra, or capability intersection interiors.
- Charter restoration order — classify amendment, select lattice row, land atomic unit, replay cross-wave binding integrity, commit closure or supersession witness — never patch single bullet in isolation.

# Amendment Decision Lattice

- Doctrine changes classify through a finite chooser lattice — no free-form report edits without row binding.
- **Taxonomy extension row** — new shape kind, projection role, or federation edge type — requires meta-integration amendment section plus role-map row and charter witness fan-out — concept-axis mechanics land in separate axis promotion unit.
- **Obligation extension row** — new propagation edge, violation signal, assurance invariant, or enforcement catalog entry — extends witness tables only; taxonomy unchanged.
- **Precedence repair row** — conflicting bullets across federation reports — supersession witness names winning report, retired row id, and arbitration proof id — no merge until precedence stack passes.
- **Wording-neutral row** — clarification without semantic delta — no charter unit; arch diff classifier must confirm zero witness field change.
- **Retirement row** — row removal after sunset witness — `superseded_by` or `archived_with_assert_never` mandatory.
- **Closure certify row** — L5 readiness plus cross-wave integrity replay — emits doctrine-family certification stamp at composition root.
- Lattice law — each row names liable report owner, obligated report ids, concept-axis notifications, proof layers, and rollback unit id — undocumented amendment path blocks merge.

# Supersession And Precedence Stack

- Report precedence resolves conflicts without erasing history — stack order is binding authority, not file modification time.
- **Stack order** — charter supersedes assurance supersedes reconciliation supersedes propagation supersedes meta-integration on federation topology and vocabulary consensus — bounded-context integration doctrine wins on interior stack altitude, single validation gate, and federation assurance temporal law when federation reports address federation only.
- **Dual authority split** — federation-interior conflicts arbitrate per doctrine authority and precedence stack — integration doctrine wins on ownership; concept axes win on mechanics; charter records arbitration witness when federation and interior boundaries disagree.
- **Supersession morphism** — retired bullet remains in archive appendix with `superseded_by` pointer until migration fold on doctrine consumers proves zero reference — same law as schema_version sunset.
- **Non-supersession rule** — later report must not silently narrow earlier report obligation without retirement row — narrowing is amendment event with proof debt on all consumers.
- **Cross-family precedence** — shape-system-integration-doctrine wins on federation binding over concept-axis reports on mechanics — concept-axis proposal touching federation edge cites charter lattice row before axis merge.
- Precedence witness row — `retired_row_id`, `superseding_report`, `superseding_row_id`, `arbitration_id` when needed, `consumer_migration_proof_id`.

# Amendment Promotion Units

- Amendment unit is smallest merge-atomic doctrine change restoring charter invariants — partial units leave supersession pointers dangling.
- **Taxonomy unit** — role-map row, kind or edge row, charter amendment witness, and concept-axis notification stubs — rollback reverts entire unit.
- **Obligation unit** — witness table append in plus enforcement catalog linkage — taxonomy files unchanged unless lattice row says otherwise.

- **Closure unit** — cross-wave integrity replay, closure witness, and composition-root charter certification record — excludes interior domain code except boundary policy stubs.
- Atomicity test — after amendment unit, run integrity witness suite and affected assurance rows — failure triggers full unit rollback, not localized wave hotfix.
- Exemption rows document composition-root-only charter promotions — name liable owner, waves skipped, and proof layers still required; default is full atomic unit.

# Amendment Enforcement And CI Gate Binding

- Charter enforcement binds to lifecycle hooks distinct from assurance and reconciliation — conflating hooks produces false closure confidence.
- **Pre-merge gate** — P1 integrity and P2 witness parity on amendment diff set — failure blocks merge before wave publication.
- **Post-amend gate** — P4 federation regression subset when obligation extension touches active edges — failure triggers rollback unit.
- **Closure gate** — P6 and D4 certification — runs once per closure unit; production handoff charter attributes forbidden until pass.

- **Consumer gate** — composition-root config and concept-axis notification index assert `superseded_by` resolution — orphan retired row reference blocks boot.
- Gate ordering — pre-merge before post-amend before closure; scheduled integrity overlaps boot but never skips pre-merge on amendment diffs.

# Charter Versus Assurance Boundary

- Charter loop and assurance loop are complementary — charter runs on amendment events and scheduled integrity cadence; assurance runs on federation witness replay.
- **Charter triggers** — amendment unit merge request, supersession commit, closure certify request, scheduled integrity gate, precedence fault.
- **Assurance triggers** — cadence — A1–A6 replay; unchanged by wording-neutral edits.
- **Escalation edge** — integrity failure on C3 or C5 maps to `witness_drift` charter fault first; repeated failure blocks assurance certification until amendment or supersession unit lands.
- **Shared inputs** — assurance certification commits supply `certification_run_id` to closure witness — charter closure unit consumes latest assurance row before stamp.
- **Proof spend ordering** — P1 integrity before assurance scheduled replay before P4 federation regression before P6 closure — static-before-generative preserved.
