# EnvIO Scope Lattice And The Trampolined Effect Interpreter

[ENVIO_OWNERSHIP_BITMASK]:
- The effect environment is one class carrying four fields — a resource tracker, a cancellation token, the token's source, and an optional synchronization context — plus a two-bit `Own` mask recording which of source (bit 1) and resource tracker (bit 2) this instance allocated versus inherited. Disposal is bifurcated by the mask: `DisposeResources` releases the tracker only when bit 2 is set, `DisposeNonResources` disposes the source and the parent-token registration only when bit 1 is set; each guards a separate `Interlocked.CompareExchange` flag so a shared environment passed to a child can be half-disposed without tearing the parent's still-live half.
- Construction is the single seam where ownership is decided: the factory allocates a fresh source (setting bit 1) only when none is supplied, and when it does, it registers the inherited token to cancel the new source — wiring a one-way cancellation cascade from outer to inner. A supplied source is borrowed, mask stays clear, and the inner scope can be cancelled without disposing a resource it does not own. This is why "who allocated it" and "who may dispose it" are the same predicate.
- The four scope projections are not four behaviours but four mask-and-field permutations of one constructor. `Local` keeps the token, drops resources and source (forks a private tracker and a private source under the same deadline). `LocalResources` keeps source and tracker, forks nothing — a name binding for the same scope. `LocalCancel` keeps the tracker but forks a private source under the same token — a regional cancel boundary that cannot be taken down by a sibling's cancellation yet still propagates an upward cancel. `LocalSyncContext` re-captures the ambient context. Each combinator that looks like a distinct effect primitive is one of these permutations with a trailing continuation.

[BRACKET_FORK_LOCAL_TIMEOUT_AS_ONE_LATTICE]:
- Resource bracketing, forking, regional cancellation, deadlines, and interrupt-masking are all derived from the scope lattice, not from independent machinery. Plain bracket maps the environment through `env => env.LocalResources` and disposes both halves on exit; fail-only bracket maps the same way but its interpreter node disposes the resource half **only inside the catch arm** and the non-resource half always in `finally` — the success path leaves the resource subtree alive, which is the entire mechanism by which an acquiring computation can return a live handle to its caller. Retry re-enters fail-only bracketing per attempt so successful resources survive across the retry boundary; repeat re-enters full bracketing per step so each iteration's resources are reclaimed.
- A deadline is a cancellation scope with a clock: `Timeout` runs the body inside a child environment whose source is a time-limited cancellation source sharing the parent token, then disposes that child via `using`. It is structurally `LocalCancel` plus an expiry, so a timed region composes with an inner regional-cancel region by nesting two child sources, the inner strictly shorter-lived.
- Interrupt-masking substitutes `CancellationToken.None` and a fresh private source into a child environment **while keeping the parent's resource tracker** — so the masked region is uncancellable yet anything it acquires is still tracked in and reclaimed by the enclosing scope. The masked body must be the minimal atomic unit: any yield, fork, or async lift inside ignores the real token until the mask exits, but resource accounting does not leak.

```csharp
static IO<C> WithLeasedResource<A, B, C>(
    IO<A> acquire, Func<A, IO<C>> use, Func<A, IO<B>> release) =>
    acquire.BracketFail()                       // resources survive a successful use
           .Bind(a => use(a)
                       .Timeout(TimeSpan.FromSeconds(5))   // child cancel-source + deadline
                       .Bracket(use: x => IO.pure(x),
                                Catch: e => IO.fail<C>(e),
                                Fin: _ => release(a)))      // = Use.Catch(C).Finally(Fin)
           .Local();                            // regional cancel: a sibling abort cannot tear this down
```

[FORK_AS_CHILD_RESOURCE_SUBTREE]:
- A fork is the scope lattice projected onto a pool thread. The fork combinator allocates a private cancellation source, registers the parent token to cancel it (the same outer-to-inner cascade as environment construction), and creates a **child resource tracker whose parent is the forking scope's tracker** — resource lookup in the child first searches local keys, then walks to the parent, so a handle acquired before the fork is reachable and releasable from inside it, but a handle acquired inside is reclaimed by the child's own `finally` on the long-running task and never leaks into the parent. Cancellation is one-way down the chain and never up.
- The fork result is a pure value pair: an `IO<Unit>` that cancels by signalling the private source (swallowing already-cancelled and already-disposed races) and an `IO<A>` that awaits the task under the caller's environment. Because the handle is data, the awaiting can happen in a different scope, a different stage, even after the spawning region has unwound — the fork's lifetime is bounded by its private source, not by the spawner's stack frame. Re-implementing the await as `fork.Bind(f => f.Await)` defeats the carrier-polymorphic await's empty fast-path; the trait-level await is the canonical spelling.
- Parallel composition is two scope shapes over the fork primitive. Wait-all materialises every branch as a task under **one shared environment** and joins them — failures surface as the first faulted task, resources from all branches share the parent tracker. Wait-any runs every branch under a single forked-cancel child environment, races them, and on the first completion cancels the shared child source so the losers are torn down deterministically; the racing variant collects errors and only fails when every branch has failed, turning the race into a fallback ladder.

[TRAMPOLINED_CATCH_STACK_INTERPRETER]:
- The effect is a free structure of fused interpreter nodes — lift-sync, lift-async, bind, map, and pre-fused composites like bind-map, bind-bind-map, bind-bind-map-bind — walked by a trampoline that holds an explicit `Seq` of catch frames as its only mutable state. A catch node conses a handler frame and continues with its guarded operation; a catch-pop node drops the top frame; on an exception the loop pops the top handler and resumes with the `IO` that handler returns. There is no recursion and no per-bind stack frame, so an arbitrarily deep monadic chain runs in constant stack — the property that makes `Bind`-driven domain loops stack-safe without trampolining glue at the call site.
- The interpreter has a synchronous fast path that never instantiates the async state machine: it runs the trampoline while every node is sync, and only the moment it meets an async node does it hand the live catch-stack to the async continuation method, which resumes the same trampoline under `await`. A pipeline that happens to contain no async leaf completes with zero async-machinery allocation; one async leaf escalates the whole remaining chain exactly once. This is why mixing one async boundary into an otherwise-sync rail costs a single escalation, not per-step async overhead.
- Exception capture is the catch-stack, not a wrapper: the rail's failure semantics — error transform, recover, finally — are all catch frames pushed and popped by interpreter nodes, so `MapFail` is literally a catch that rethrows a transformed error, `Finally` is a node that runs cleanup on both arms, and bracket is `Use.Catch(handler).Finally(release)`. A raw `throw` inside a bind lambda is caught by the nearest frame and routed exactly as a rail failure would be — which is precisely why a throw that escapes all frames rethrows to the host and is **not** in the rail, and why the boundary must convert it before it enters.

[SYNC_CONTEXT_MARSHALING_SEAM]:
- Posting onto a host UI thread is a scope operation, not an ambient call. The post combinator reads the environment's captured synchronization context; with none present it degrades to a direct async run, and with one it marshals the body via the context, blocking on an auto-reset event that is itself released through the token — so a cancellation during the marshalled body unblocks the waiter and surfaces a cancellation rather than dangling. The captured context rides inside the environment and is forked or re-captured by the scope projections, so "which thread does cleanup run on" is answered by the environment, never by an implicit capture at the call site.
- Cleanup inherits the live environment, including its already-signalled token: a `finally` attached to a cancelled body runs with the token already cancelled, so any cancellable operation inside the cleanup short-circuits immediately. Cancel-safe teardown therefore wraps the cleanup in the interrupt mask so the release runs to completion under `CancellationToken.None` while still being accounted against the parent tracker — the one place where masking is mandatory rather than a minimal-unit optimisation.
