- Ordinary docs use GitHub alerts correctly in many tool/package READMEs, but also use inline `[IMPORTANT]` in `docs/usage.md:3`, `docs/host-libraries.md:5`, `docs/external-libs/README.md:3`, and `docs/system-api-map/README.md:3`.

Tables, records, matrices, diagrams, and field packets observed:
- Dense lookup/comparison tables: root README layout/build policy, `docs/system-api-map/packages.md`, external library API maps, testing-libs API maps, tool command surfaces, and package architecture matrices.
- Records and field packets: `tools/assay/README.md:9-16`, `PYTHON_QUALITY_TOOLING.md:15-21`, `tools/assay/README.md:183-203`, `tools/assay/README.md:304-348`.
- Diagrams: quality/assay/rhino bridge/package architecture Mermaid graphs; `libs/csharp/Rasm/Vectors/_ARCHITECTURE.md:121-200` is a larger architecture graph with renderer config and theme variables.
- Progress markers: no explicit progress bar found in the requested visible inventory; progress appears as status states and roadmap gate records rather than calculated bars.

## [RECOMMENDATIONS]

1. Promote the missing shapes into `information-structure.md` as named containers: domain-status record, command bundle, source-proof packet, topology-status bundle, type-shape block, package/API matrix, gate matrix, row-sidecar note, co-location triage record, and codemap tree.
2. Tighten `formatting.md` for real repo notation: type-local status markers, row-sidecar labels, ordinary-doc `[IMPORTANT]` replacement with GitHub alerts or prose modals, untagged fence rejection, codemap fence labels, and machine-consumed table exceptions.
3. Route proof-field semantics to `proof.md` before exact packet wording lands, especially `Source of truth`, `Evidence`, `Last verified`, `Review trigger`, and `Proof gap`.
4. Keep machine-consumed files as explicit exceptions; do not normalize analyzer release ledgers or parser-owned Markdown into ordinary documentation tables.
