# Parameter Algebra — Deepening

[CONVERSION_ASYMMETRY_AND_ACCESS_GATES]:
- Ad-hoc unions (`[Union<T1,T2>]`) inherit from `UnionAttributeBase`, which exposes both `ConversionFromValue` (default `Implicit`) and `ConversionToValue` (default `Explicit`). Regular `[Union]` exposes only `ConversionFromValue`. The asymmetry is intentional: ad-hoc unions are structurally typed composites where extracting a member case back to its naked type is an explicit operation, whereas regular unions rely on `Switch`/`Map` for case consumption. Setting `ConversionToValue = ConversionOperatorsGeneration.None` forces all member extraction through generated accessors, eliminating ambiguous implicit downcast paths.
- `UnionAttributeBase.ConstructorAccessModifier` defaults to `UnionConstructorAccessModifier.Public` for all union forms. `ValueObjectAttributeBase.ConstructorAccessModifier` defaults to `AccessModifier.Private`. This asymmetry is the architecture signal: union case construction is open by default; value-object construction is always factory-gated. Setting `ConstructorAccessModifier = Private` on a union privatizes both the constructor and the implicit conversion operators as one attribute property change — the constraint is not split across two settings.
- `UnionAttributeBase.UseSingleBackingField = true` forces an `object` backing field for all member cases, including struct cases. The struct case is boxed on storage and unboxed on access. This is a memory-reduction trade-off when the union carries cases whose sizes differ significantly; by default, each case gets its own typed backing field. Stateless struct cases (`T1IsStateless = true`) already store only the discriminator index with zero payload, making `UseSingleBackingField` irrelevant for those slots.
- `FactoryMethodGeneration` on `UnionAttributeBase` controls factory method generation for ad-hoc union members. `Default` detects triggers: type-parameter members, `interface` or `object` members, or duplicate member types cause generation. `None` suppresses generation globally. `Always` generates for every member regardless of triggers. `FactoryMethodGeneration = None` combined with `ConstructorAccessModifier = Private` produces a type with no public construction path; the custom factory must be authored manually.

[STATELESS_CASE_DISCIPLINE]:
- `T1IsStateless = true` on an ad-hoc union member instructs the generator to store only the discriminator index. The generated accessor `AsT1` returns `default(T1)`. For a class `T1`, `T1IsNullableReferenceType` is automatically coerced to `true` (because the accessor returns `null`), so stateless cases use struct marker types to make the null return structurally impossible.
- A stateless union case carries no information beyond the fact that the case was selected and is the correct shape for policy entries with no configuration data — a `None`, `Default`, or `Absent` case in a request union. The discriminant is the entire semantic content. Wrapping a stateless case in `Option<T>` adds a second optionality layer on top of the discriminant; `Option<T>` belongs on parameters where the entire union is absent, not inside a union that already models absence as a case.

[ARGUMENT_REF_STRUCT_CONSTRAINTS]:
- `Argument<T>` is `readonly ref struct where T : allows ref struct`. It cannot be boxed, stored in a field, used as a generic type argument, or held across an `await` boundary; its use is strictly within a synchronous call frame. The generated `ValidateFactoryArguments` partial does NOT use `Argument<T>` — it takes `ref T` for each declared field. `Argument<T>` is for bespoke factory overloads authored alongside `[ObjectFactoryAttribute<T>]`, where the factory needs to distinguish "caller supplied a value that equals the domain default" from "caller omitted the argument."
- When a custom factory implements `IObjectFactory<T, TValue, TValidationError>`, its `static abstract Validate(TValue? value, IFormatProvider? provider, out T? item)` method receives the raw `TValue?` — not `Argument<TValue>`. Expecting `Argument<T>` in a generated interface is a category error that produces a compile-time mismatch against the `IObjectFactory` static abstract signature.
- `IObjectFactory<TValue>` (one type parameter) is a marker interface with no members. `IObjectFactory<T, TValue, out TValidationError>` (three type parameters) inherits from it and adds `static abstract TValidationError? Validate(TValue?, IFormatProvider?, out T?)`. When `SkipFactoryMethods = true`, neither interface is implemented — `TypeConverter`, the implicit conversion operator, `SkipIParsable`, and all arithmetic operator generation are all suppressed. `[ObjectFactoryAttribute<T>]` restores serialization and EF Core routes but does NOT restore arithmetic operators or the `IComparable` path, which are permanently suppressed by `SkipFactoryMethods`.

[SCHEDULE_OPERATOR_SEMANTICS]:
- `Schedule + Schedule` calls `Combine`, which concatenates the two streams sequentially: the first schedule's durations run to exhaustion, then the second's continue. `Schedule | Schedule` calls `Union`, which runs both in parallel and emits the minimum duration at each step until the longer stream exhausts. `Schedule & Schedule` calls `Intersect`, emitting the maximum duration at each step until the shorter stream exhausts. These are three semantically distinct composition operators sharing no behavior; conflating `+` (sequential) with `|` (parallel minimum) produces schedules that run more iterations than intended.
- `ScheduleTransformer + ScheduleTransformer` composes two transformers left-to-right: `f + g` produces `x => g.Apply(f.Apply(x))`. The composition is not commutative: `NoDelayOnFirst + jitter(0.1)` removes the first delay then adds jitter to all remaining steps, while `jitter(0.1) + NoDelayOnFirst` adds jitter first and then replaces the first jittered duration with zero. Transformer composition order is the stacking order for policy mutation.
- `ScheduleTransformer` carries an implicit conversion to `Schedule` via `Schedule.Forever | transformer`. A bare transformer coerced to `Schedule` without an explicit base produces an infinite schedule bounded only by the transformer's own logic. Storing a transformer in a `static readonly Schedule` field triggers the coercion silently; the `Forever` base is invisible at the declaration site.

[FROZEN_TABLE_ALTERNATE_LOOKUP_CONSTRAINTS]:
- `FrozenDictionary<TKey, TValue>.TryGetAlternateLookup<TAlternateKey>` succeeds only when the dictionary's `Comparer` implements `IAlternateEqualityComparer<TAlternateKey, TKey>`. For string-keyed frozen dictionaries, the only accepted alternate key type is `ReadOnlySpan<char>`; no other alternate key type is accepted for `string` keys regardless of comparer capabilities. `GetAlternateLookup<TAlternateKey>()` throws `InvalidOperationException` when the constraint is unsatisfied; `TryGetAlternateLookup` returns `false`.
- `IAlternateEqualityComparer<TAlternate, T>` requires `bool Equals(TAlternate, T)`, `int GetHashCode(TAlternate)`, and `T Create(TAlternate)`. The `Create` method is used internally by the frozen table for certain operations that need to produce a canonical key from the alternate. Callers accessing policy rows by `ReadOnlySpan<char>` key pay no string allocation for the lookup path.
- `FrozenDictionary<TKey, TValue>.GetValueRefOrNullRef(TKey)` returns `ref readonly TValue`. Combined with `AlternateLookup`, a policy-table lookup on a `ReadOnlySpan<char>` key is allocation-free end-to-end. `FrozenSet<T>.AlternateLookup<TAlternate>` exposes `TryGetValue(TAlternate, out T)` rather than a `ref readonly` path, so membership vocabulary lookups with span keys do not share the ref-return optimization.
- `FrozenDictionary<TKey, TValue>.Empty` is a `static` property returning a singleton per `(TKey, TValue)` pair. Testing absence via `ReferenceEquals(table, FrozenDictionary<TKey, TValue>.Empty)` is the correct form for a policy table that may be entirely absent; it avoids the `Count == 0` property access.

[CATCHM_GENERIC_EXTENSION_BLOCK]:
- The `|` operator combining `K<F, A>` with `CatchM<E, F, A>` is defined in `FallibleExtensions` constrained to `where F : Fallible<E, F>`. Any monad implementing `Fallible<E, M>` inherits the operator through the extension; it is not per-type. A `static readonly CatchM<Error, Eff<RT>, A>` named catch policy field is directly applicable to `K<Eff<RT>, A>` values via `|` without monad-specific wiring.
- `CatchM<E, M, A>` checks `Match(error)`: if the predicate returns `false`, it returns the unchanged computation; if `true`, it invokes `Action(error)`. A chain `| namedCatch1 | namedCatch2` evaluates left-to-right; the first matching catch consumes the error. A broad predicate placed before a narrow one shadows the narrower catch silently.
- `bracketIO`'s `Catch: Func<Error, IO<C>>` parameter is a raw function, not a `CatchM`. A named `static readonly CatchM` cannot be passed directly; the bridge is `e => myCatch.Match(e) ? myCatch.Action(e).As() : IO.fail<C>(e)`. When the bracket catch needs the same recovery policy as a named catch, the recommended shape is a shared `static readonly Func<Error, IO<C>>` that the `CatchM`'s `Action` delegates to, keeping predicate-selection logic in the `CatchM` and the recovery body in the shared function.

[POLICY_VALUE_THREADING_FAILURE_SURFACES]:
- `SmartEnumAttribute<TKey>.ComparisonOperators` coerces `EqualityComparisonOperators` upward: when `ComparisonOperators` is set to `DefaultWithKeyTypeOverloads`, `EqualityComparisonOperators` is automatically promoted to the same level because comparison operators require equality operators. The reverse also holds: `ComparisonOperators` is forced to `None` when `EqualityComparisonOperators` is `None`. Setting the two independently produces a silent coercion; `ComparisonOperators` is the single authority, and `EqualityComparisonOperators` should not be set independently when `ComparisonOperators` is set.
- `SkipFactoryMethods = true` on a `ValueObjectAttribute<TKey>` cascades to: no `TypeConverter`, no `IObjectFactory` interfaces, no key-to-value implicit conversion, `SkipIParsable = true` (which forces `SkipISpanParsable = true`), and all arithmetic operator generation forced to `None`. An `[ObjectFactoryAttribute<T>]` on the same type restores serialization and EF Core routes but does NOT restore the arithmetic operators or `IComparable` path, which are permanently suppressed.
- `SmartEnumAttribute<TKey>.DisableSpanBasedJsonConversion = false` (default) causes the generated JSON converter to use a `ReadOnlySpan<char>`-keyed path during deserialization, avoiding a string allocation per JSON token. The flag has no effect when `KeyMemberType` is not `string`. Keeping this flag at its default across all string-keyed smart enums is the zero-configuration allocation reduction.

[OPTION_SEMIGROUP_SEMANTICS]:
- `Option<A>` implements `Semigroup<Option<A>>` via `Combine`: returns `this` if `IsSome`, `rhs` otherwise. The `|` operator delegates to `Choose`, calling the `Choice<Option>` trait implementation — both are semantically identical left-biased fallback. Neither computes a union of the contained values; the inner `A` type plays no role. A chain `optA | optB | optC` returns the first `Some` without evaluating later operands.
- `Option<A>.operator true` returns `IsSome`; `operator false` returns `IsNone`. These enable `if (opt)` and short-circuit `&&`/`||`. At the parameter-algebra layer, `option1 && option2` returns the second `Option<A>` if the first is `Some` (using the standard C# short-circuit table applied to `operator true`/`false`), enabling guard expressions that remain expression-form inside generated `Switch` arms returning `bool`.
