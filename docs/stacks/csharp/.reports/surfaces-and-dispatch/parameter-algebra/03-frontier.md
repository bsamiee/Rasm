# Parameter Algebra — Frontier

[SCHEDULE_MONAD_BIND]:
- `Schedule.Bind(Func<Duration, Schedule> f)` fans each emitted duration into a sub-schedule derived from it, then concatenates all sub-schedules. `SchBind` runs `Schedule.Run().Bind(d => f(d).Run())` — every duration in the outer schedule produces its own inner duration stream and the total emission is the flat concatenation. A duration-parameterized policy value — a schedule whose per-step spacing is a function of the previous duration — is expressed as a C# query expression: `from d in baseSchedule from d2 in Schedule.spaced(d * factor) select d + d2`.
- `Schedule.SelectMany(Func<Duration, Schedule> bind, Func<Duration, Duration, Duration> project)` runs `outer.Bind(x => bind(x).Run().Map(y => project(x, y)))`. The projection receives both the outer duration and the inner duration, enabling combined policy values such as capping sub-schedule output to the outer duration. `ScheduleTransformer` has no `SelectMany`, `Select`, or `Bind`; only `Schedule` supports LINQ query expressions.
- `Schedule | Schedule.spaced(cap)` clamps maximum delay by taking the minimum per step. `Schedule.Bind(d => Schedule.spaced(d.Min(cap)))` clamps by transforming each duration before producing the inner sub-schedule. Both produce the same clamped values but with different structural semantics — the bind form preserves duration-level provenance that the union form discards. The distinction is load-bearing when the policy derives the next sub-schedule from the previous duration rather than clamping independently.

[SCHEDULE_INTERLEAVE_AND_WALL_COMBINATORS]:
- `Schedule.Interleave(Schedule b)` zips both streams into pairs and emits both durations from each pair. The emitted count is `2 × min(a.Length, b.Length)`, terminating when the shorter stream exhausts. Using `Interleave` where union was intended doubles the retry count silently.
- `Schedule.maxDelay(Duration max)` is a `ScheduleTransformer` that maps every emitted duration through `Min(x, max)` — a per-emission cap with no effect on iteration count. `Schedule.maxCumulativeDelay(Duration max)` terminates the schedule when the running sum reaches `max`. These are orthogonal caps: `maxDelay` restricts spacing magnitude; `maxCumulativeDelay` restricts total wall-clock budget. A policy needing both uses `| maxDelay(2*s) | maxCumulativeDelay(30*s)` applied to the same base.
- `Schedule.windowed(Duration interval)` re-aligns each delay to the next boundary of the fixed interval (using the residual time to the next window). If an action runs longer than the interval, the next delay is zero. It is distinct from `Schedule.spaced(interval)`, which emits a constant `interval` regardless of when the action completes; `windowed` is the cron-alignment form and `fixedInterval` is the load-leveling form that suppresses delay on overrun but starts the next window from the completion time of the previous action.

[ATOM_POLICY_CELL_MECHANICS]:
- `Atom<A>` carries `implicit operator A(Atom<A> atom)` — the atom implicitly reads its current value when used as `A`. A `static readonly Atom<Schedule>` field initialized with a composed schedule can be passed directly to `retry(schedule, computation)` without `.Value`; the implicit conversion reads the current value at call time. Any `Swap` on the atom changes the policy for all subsequent retry calls without touching the call site.
- `Prelude.Atom<A>(value, Func<A, bool> validator)` returns `Option<Atom<A>>`, not `Atom<A>`. The validator runs on the initial value before the atom is returned; if the validator returns `false`, the result is `None` and the atom is never constructed in an invalid state. A `Schedule` atom that must always have at least one iteration encodes that constraint in the validator, forcing the caller to handle the admission failure at construction time rather than at first retry.
- `Atom<A>.SwapMaybe(Func<A, Option<A>> f)` aborts the swap and returns the current value unchanged when `f` returns `None`. The `Change` event is not fired on an aborted swap. `Swap(Func<A, A> f)` always proceeds unless the validator rejects it, which also suppresses `Change`. Every `Change` invocation therefore reflects state that was actually installed in the atom — the event is a reliable policy-change notification.
- `Atom<M, A>` (two-type-parameter form) carries read-only metadata `M` alongside state `A`; both `Swap` overloads receive `Func<M, A, A>` or `Func<M, A, Option<A>>`. The metadata is fixed at construction time and never changes; it is the immutable context for the swap function. The correct use: a policy atom needing access to a fixed configuration record without closing over it uses `Atom<Config, Schedule>` so the swap function can derive a new schedule from the configuration and current state together.

[CATCHOF_AGGREGATE_ERROR_WALKING]:
- `catchOf<E, M, A>(Func<E, K<M, A>> Fail)` calls `e.IsType<E>()` on the error, which recursively checks whether any sub-error in an aggregate is of type `E`, then folds matching sub-errors through `Fail` using `ForAllM` (fail-fast on first handler failure). A plain `@catch` with a predicate `e => e is E` does NOT perform this sub-error walk — it matches only the top-level error. The choice between `catchOf` and `@catch` is load-bearing when the error rail carries composite failures.
- `Error.Filter<E>()` returns an `Error` containing only sub-errors of type `E`, returning `Errors.None` when no sub-errors match. `Error.IsType<E>()` returns `true` if any sub-error satisfies the type predicate. When a custom named catch must discriminate on a structured aggregate, composing `@catch(e => e.IsType<MyError>(), handler)` with `e.Filter<MyError>().ForAllM(handler)` inside the handler body replicates `catchOf` behavior for that specific predicate without taking the `Monad` constraint that `catchOf` requires.

[FACTORY_REF_STRUCT_ADMISSION]:
- `ObjectFactoryAttribute<T>` declares `where T : allows ref struct`. A value object can declare `[ObjectFactoryAttribute<ReadOnlySpan<char>>]`, producing a factory seam accepting `ReadOnlySpan<char>` directly. This enables zero-allocation admission at parsing boundaries: a span from the wire layer is passed to `Create(ReadOnlySpan<char>)` and admitted into a fully typed value object without an intermediate `string` allocation. `UseForSerialization = SerializationFrameworks.SystemTextJson` additionally routes the `Utf8JsonReader`-based deserializer through the same span factory for zero-copy JSON parsing.
- `IConvertible<out T>` (where `T : allows ref struct`) enables a value object to implement `ToValue()` returning a `ReadOnlySpan<char>`. The generated `implicit operator T(VObj vo)` does not use this interface; callers invoke `ToValue()` explicitly to obtain the span without an allocation. This is the reverse path from the admission factory: span-factory admission in, `ToValue()` span projection out.

[PARTIAL_DISPATCH_OVERLOADS_FOR_DEEP_UNIONS]:
- `[UnionSwitchMapOverloadAttribute(StopAt = new[] { typeof(FailureCase) })]` applied to a multi-level regular union generates an additional `Switch` overload that handles `FailureCase` and its siblings at that level as a single parameter without descending into `FailureCase`'s derived cases. The generated overload receives `Action<FailureCase>` (or `Func<FailureCase, T>` for `Map`) rather than one parameter per leaf. The full exhaustive overload is always generated; the `StopAt` overload is additive. This is the correct shape when an entrypoint delegates handling of an entire sub-hierarchy to one handler without knowing its leaf structure.
- `SwitchMapMethodsGeneration.DefaultWithPartialOverloads` generates, alongside the exhaustive overload, a variant accepting `otherwise: Func<TState, TResult>` for unhandled cases. The state-threaded signature preserves the same state-threading contract as the full overload. This is the correct shape when a dispatch site handles a subset of known cases and delegates the rest to a single default policy value.

[IEQUALITY_COMPARER_ACCESSOR_CHAIN]:
- `IEqualityComparerAccessor<TKey>` declares `static abstract IEqualityComparer<TKey> EqualityComparer { get; }`. The generator calls `TComparerAccessor.EqualityComparer` at the type level when resolving the comparer for a smart enum's generated `GetHashCode` and `Equals`. `ComparerAccessors.StringOrdinal`, `StringOrdinalIgnoreCase`, and the culture variants implement both `IEqualityComparerAccessor<string>` and `IComparerAccessor<string>`. A custom comparer accessor is the mechanism for changing hash and equality semantics without mutating the key type.
- `[KeyMemberEqualityComparerAttribute<TComparerAccessor>]` and `[KeyMemberComparerAttribute<TComparerAccessor>]` are `allow-multiple = false`; only one comparer accessor governs the key member. A custom accessor must be consistent with the `FrozenDictionary` comparer used for any frozen policy table keyed on the same smart enum. Mismatching — ordinal on the smart enum but ordinal-ignore-case on the frozen table — produces silent lookup divergence. The correct form uses `ComparerAccessors.StringOrdinal` for both the smart-enum key attribute and the `FrozenDictionary.Create(StringComparer.Ordinal, ...)` call.

[ADHOC_UNION_STRUCTURAL_LIMITS]:
- Ad-hoc union `[Union<T1, T2, T3, T4>]` supports up to four type parameters; a fifth member requires `AdHocUnionAttribute` (non-generic, using `Type` constructor arguments) supporting up to five. Beyond five members, the correct shape is a regular `[Union]` with nested case records. The `T1Name` property overrides `Switch`/`Map` parameter names for that slot without changing the C# type name, enabling a type alias at the dispatch boundary while keeping the member type unchanged.
- `UnionAttributeBase.DefaultStringComparison` defaults to `StringComparison.OrdinalIgnoreCase` for string-keyed union discriminants. This governs the generated JSON discriminator comparison and the `ToString`/parsing path, not the structural equality of case instances. Setting it to `StringComparison.Ordinal` on a union participating in a case-sensitive wire protocol is required; the default ignore-case behavior produces silent mismatches when union case names share an ordinal-case distinction.

[VALIDATIONERROR_TYPE_WIRE]:
- `[ValidationErrorAttribute<TError>]` requires `TError : class, IValidationError<TError>` and registers the error type at the declaring type level, not at the factory method level. The generator invokes `TError.Create(string message)` when `ValidateFactoryArguments` sets a validation error, producing the fully typed error without a runtime cast. A single attribute propagates `TError` as the error type for all generated factory methods on that type — there is no per-factory override. When two value objects in a hierarchy need different error types, each declares its own `[ValidationErrorAttribute<T>]`.
- The default `ValidationError` type (a `sealed class` implementing `IValidationError<ValidationError>`) carries only a `Message: string` property. The typed custom error type replaces `ValidationError` entirely in generated factory signatures; generated methods return `TError?` for the validation error out-parameter when `[ValidationErrorAttribute<TCustom>]` is present and `SkipFactoryMethods = false`.

[FROZEN_DICTIONARY_KEYS_IMMUTABLEARRAY]:
- `FrozenDictionary<TKey, TValue>.Keys` returns `ImmutableArray<TKey>`. Iterating it via `foreach` never boxes and never allocates; the explicit-interface `IDictionary<TKey, TValue>.Keys` allocates a wrapper. When enumerating policy-table keys for validation or bulk projection, the `Keys` property is the zero-allocation path. `FrozenSet<T>` follows the same pattern via `Items` returning `ImmutableArray<T>`.
- `FrozenDictionary<TKey, TValue>.Empty` returns a singleton per `(TKey, TValue)` pair. `ReferenceEquals(table, FrozenDictionary<TKey, TValue>.Empty)` is the correct absence test for a policy table that may be entirely absent, avoiding the `Count == 0` property access.
