# [SUITE_FEATURES]

Higher-order feature atlas for the four app-grade packages. Each concept is a product the suite owns in concert across processes; mechanics live in the named planning pages, and a concept never adds a surface ΓÇõ it rides axis rows. Package-local atlases (`Rasm.<Pkg>/FEATURES.md`) carry the isolation concepts; this file carries the cross-package and multi-process topologies, and every concept names its owning page clusters across the four packages. Each concept carries a `[STATE]`: `FINALIZED` (every riding row is a transcription-complete fence), `PARTIAL` (the spine is fenced; a riding column/case sits on a research row), `MISSING` (the owner row is identified, the case is not yet a fence), `SPIKE` (gated on a bridge/native/server/app-root probe). A concept inherits its weakest riding row.

## [1]-[TOPOLOGY_VOCABULARY]

| [INDEX] | [TOPOLOGY] | [SHAPE]                                                                          |
| :-----: | :--------- | :------------------------------------------------------------------------------- |
|   [1]   | in-process | one host, all four packages composed in one ALC                                  |
|   [2]   | paired     | standalone discovers a running Rhino host over UDS; single-writer store lease    |
|   [3]   | companion  | host dials out to N spawned/attached service processes over UDS/HTTP2            |

The dual mandate binds every concept below: each is a universal standalone capability AND a first-class rhino/gh2 capability, distinguished by the host-modality row (`AppHost/host-profiles#PROFILE_AXIS`, `Persistence/store-profiles#PLACEMENT_MATRIX`, `Persistence/sync-collaboration#TRANSPORT_AXIS`, `AppUi/surface-hosts#HOST_AXIS`), never by a fork.

## [3]-[CROSS_PACKAGE_LAWS]

- One wire vocabulary: Compute owns the proto suite; every other package projects into it.
- One causal primitive: the `ReceiptSinkPort` HLC stamp orders cross-process evidence, deriving `SkewBound` at stamp time; receipts, bundles, and degradation stay process-local and correlate only through the stamp. Every cross-process product (sync op-log order, correlation join, tag invalidation, drill-down) is provable from this one primitive ΓÇõ siblings stamp, never mint.
- One content-address identity: the sealed snapshot `Hash` is the one `ContentAddress` and `XxHash128` `Append`/`GetCurrentHash` is the one incremental identity, feeding cache tag, blob lookup, diff `Closure`, model-result key, parquet rollup, and snapshot restore; no product mints a parallel key.
- One frame law: declared at `ArtifactSync`, consumed by `BlobRemote`.
- One canonical wire geometry: the proto geometry family; GeoJSON, NTS, and RhinoCommon shapes are boundary projections.
- One codec table: seam-keyed (config, snapshot, delta, gRPC, dashboard, deep-link) ΓÇõ STJ source-gen, MessagePack, protobuf each own named seams; no codec crosses into another's seam.
- One retry owner per hop, one cache owner, one clock seam, one drain conductor, one recency-horizon owner (`MODEL_RESULT_INDEX`) ΓÇõ the AppHost and Persistence ports; DB retry stays excluded from the outbound retry owner.
