# Iterative Convergence And Breakdown Algebra

[CRITERION_STACK_AS_ORDERED_FOLD]:
- The stop policy is a short-circuit fold over an ordered criterion list, not a set: the iterator walks the list in insertion order and returns the first non-`Continue` verdict, so the array index is a priority, not a label. The verdict an iteration receives is the highest-priority criterion that fires, and every lower-priority criterion that would also fire on that iteration is never consulted. Composing the stop policy is therefore choosing both the criterion vocabulary and its precedence in one ordered value — the list is the policy, and its order is load-bearing semantics.
- The default vocabulary built by the matrix builder is `[Failure, Divergence, IterationCount, Residual(τ)]` with `τ` carried per element type — `1e-12` for real and double-width complex, `1e-6` for single-width real and complex. The ordering fault is structural: `IterationCount` precedes `Residual`, so on the exact iteration where the count cap is reached the fold returns `StoppedWithoutConvergence` before the residual criterion is ever asked whether the iterate converged. A solve that lands inside tolerance precisely at the cap is reported as a non-convergence. The correct precedence is `[Failure, Divergence, Residual, IterationCount]` — NaN before divergence before convergence before the hard ceiling — so the convergence verdict wins the iteration it is true on, and the count cap is the genuine last resort.
- The cap default is a fixed constant independent of system size; it is not derived from `n`. A size-scaled cap is a policy value the boundary owner computes once — the natural unit is a multiple of the expected Krylov subspace dimension — and threads into the `IterationCount` criterion at construction. The size-blind default silently truncates large systems and over-spends on small ones; the cap belongs to the operator's dimension, not to a library constant.
- Constructing the iterator from an explicit ordered criterion array is the only way to fix both the precedence and the cap; the parameterless iterator clones the size-blind default stack, and passing a null iterator into the solve substitutes a fresh default-stacked iterator. The reuse shape is one criterion-stack policy value, built once at factorization time from the operator's dimension and the caller's tolerance, reused across every right-hand side that operator answers.

[PER_ITERATION_IDEMPOTENCY_CONTRACT]:
- Each criterion guards its body with a last-seen-iteration gate: if the iteration number it is handed does not exceed the number it last processed, it returns its cached verdict without recomputing. Re-invoking the fold with the same iteration number is therefore free and stable — the stop decision for iteration `k` is computed once and memoized inside each criterion. This makes the criterion stack safe to consult redundantly but means a criterion that tracks history advances its internal window only on a strictly increasing iteration number, and a solver that re-asks the same number twice gets the first answer, not a fresh evaluation.
- The convergence criterion's internal counter is the gap between the current iteration number and the iteration number at its previous in-threshold call, reset to a sentinel below zero on construction and on reset. On the first in-threshold call after a reset the gap is the iteration number plus one, which clears any required-sustain count immediately at any non-trivial iteration; under the normal per-iteration invocation pattern the gap collapses toward one because the previous-call marker advances every call. A sustain requirement above one is therefore unreachable under per-iteration polling and only becomes meaningful when the criterion is consulted every `k`-th iteration. The usable configuration is the degenerate sustain of zero — fire on the first in-threshold iteration — which is the only mode that behaves as a relative-residual gate under standard polling.
- The convergence test is a relative infinity-norm gate: the residual infinity norm against the source infinity norm scaled by the tolerance. The norm collapses to a real scalar regardless of the element type's precision, and the NaN guard compares against that real scalar directly. The criterion cannot be made generic over floating-point precision through generic-math constraints — the real-typed norm comparison is a fixed implementation detail, so the tolerance is a scalar policy value, never a type-parameterized one.

[NAN_DETECTION_IS_TRIPLY_REDUNDANT_AND_DIVERGENT]:
- Three separate criteria each detect non-finite state and each assign a different terminal verdict to it. The failure criterion flags a non-finite solution-or-residual norm as `Failure`; the divergence criterion flags a non-finite residual norm as `Diverged`; the convergence criterion flags a non-finite source-or-residual scaled norm as `Diverged`. Which verdict a NaN actually receives is decided purely by criterion order — the first NaN-aware criterion in the stack wins — so the same numerical event surfaces as `Failure` or `Diverged` depending only on precedence. Placing the failure criterion first makes NaN a distinct terminal class separable from honest divergence; demoting it folds NaN into the divergence bucket and erases the distinction.
- The fold's terminal verdicts partition into routing classes that are load-bearing for fallback. `Diverged` — a detected monotone residual blow-up — routes to a regularized retry or a direct solve, because more iterations will not help. `StoppedWithoutConvergence` — the count cap with no blow-up — routes to a direct solve optionally seeded with the partial iterate, because the trajectory was sound but slow. `Failure` and `Cancelled` route to reconstruction or abort. Collapsing these into one boolean convergence flag discards exactly the signal that selects the recovery path; the verdict enum is the recovery discriminant and must reach the boundary intact.

[DIVERGENCE_WINDOW_IS_A_STRICT_CONJUNCTION]:
- The divergence verdict is a strict all-pairs conjunction over a sliding window, not a trend test. Every adjacent pair in the window must satisfy both a non-decreasing condition and a strict growth-above-rate condition; a single pair that dips, or even one that grows but stays at or below the relative-increase rate, returns not-diverging for the whole window. The window length is the configured minimum-iteration count plus one, allocated lazily on first use and slid forward by a manual shift each iteration. The conjunction makes the criterion conservative by construction: it fires only on uninterrupted, super-threshold growth across the entire window.
- The window length is the single knob that trades false positives against detection latency, and the trade is opposite for the two solver families. On symmetric systems the residual descends near-monotonically, so the default long window detects genuine blow-up without spurious firing. On non-symmetric systems the residual oscillates through the warm-up phase, so any single dip inside the window suppresses the verdict — shrinking the window to its minimum to detect divergence sooner produces spurious `Diverged` signals during the normal oscillatory transient. The window length is thus a policy value keyed on the operator's symmetry class, carried beside the criterion stack, not a global default.

[ONE_BREAKDOWN_CONDITION_THREE_CONTROL_OUTCOMES]:
- The same mathematical breakdown — an inner product collapsing to a near-zero denominator in the bi-Lanczos recurrence, tested by a one-unit-in-last-place distance from zero — produces three incompatible control outcomes across the non-symmetric solver family, with no syntactic difference at the call site. One solver throws a numerical-breakdown exception; one cancels the iterator and exits the loop, leaving a cancelled verdict and the partial iterate; one substitutes one in place of the near-zero denominator and continues, never throwing and never cancelling. The breakdown is the same event; the recovery contract is a property of the solver kind, recoverable from the kind alone, and a request surface that swaps solvers swaps failure semantics silently.
- The unit-in-last-place test is a binary-representation distance, and it is asymmetric on non-finite input: a NaN denominator fails the near-zero test and is therefore not caught as a breakdown — it slips past the guard and propagates as a poisoned value into the next iterate, where only the separate NaN-aware stop criteria can catch it, one iteration later and as `Failure` or `Diverged` rather than as a breakdown. The breakdown guard catches the zero case, never the NaN case; the two pathologies need two independent rails.
- The unified rail is one outer boundary that catches the breakdown exception, treats a cancelled verdict as a partial result requiring verification, and treats a continued-through-substitution result identically — all three converge on the same obligation: recompute the true residual of the original operator against the returned iterate before trusting it. The substitution path is the most dangerous because it returns a normal verdict; the cancelled path is recoverable because the composite loop resets the verdict and continues; the exception path is loud. One catch combinator plus one residual witness collapses three control shapes into one failure-handling surface.

[PRECONDITIONER_INIT_ESCAPES_THE_VERDICT_RAIL]:
- The solve entrypoint returns the iterator's terminal verdict and wraps nothing in a catch, but the very first action inside the solver body — before the iteration loop begins — is to initialize the preconditioner against the operator. A preconditioner that rejects the storage shape, or hits a zero pivot during its incomplete factorization, throws out of that initialization, and the throw escapes the verdict-returning entrypoint entirely. The verdict rail and the exception rail are disjoint: a returned verdict means the loop ran; an escaped exception means it never started. Code that reads only the verdict treats a preconditioner-init failure as if no solve occurred, with stale output left in the result buffer.
- The modified-incomplete preconditioner requires the operator already be in compressed sparse storage and throws a typed argument exception otherwise, leaving its initialized flag false; a downstream apply on the un-initialized instance reads partially-assigned internal state and produces garbage or a null fault. The contract is: initialize the preconditioner outside the solve, gate the apply on the initialized flag, catch the zero-pivot breakdown and the storage-shape rejection at that boundary, and fall back to a diagonal preconditioner or a direct solve before entering the iterative path. The breakdown's reported row index is in the input's storage-iteration order, not in any fill-reducing permutation, so pre-permuting the operator both reduces zero-pivot probability and shifts the reported index — the caller, not the preconditioner, owns the permutation.

[COMPOSITE_SWALLOWS_AND_RESETS]:
- The composite solver sequences a solver list and routes on the verdict each produces, but it wraps every sub-solve in a catch-all that swallows any exception — breakdown, out-of-memory, preconditioner-init failure alike — restores the input and advances to the next solver. The result after exhausting the list carries the last non-convergence partial iterate with no record of which solvers threw versus which ran to a verdict. The composite's verdict routing is `Converged` returns; `StoppedWithoutConvergence` keeps the partial and continues; every other verdict discards and continues. Because it resets the iterator before each sub-solve, convergence and divergence history is fully independent per solver — no carry-over — and a cancelled verdict from one solver's breakdown handling is wiped before the next runs.
- The composite resolves each sub-solver's preconditioner from the solver setup, falling back to a unit preconditioner when the setup yields none, so the external preconditioner argument is reached only in the setup-yields-none case and is otherwise dead. The reset-per-solver discipline interacts with token-linked cancellation: resetting re-derives the linked cancellation source from the master token, so a master token already cancelled at the time of reset immediately re-cancels every subsequent solver on its first iteration — a pre-cancelled token poisons the entire composite chain. The composite's result therefore always requires an independent true-residual recomputation regardless of the verdict it returns, because the verdict alone cannot distinguish a genuine partial from a chain poisoned by a swallowed exception or a pre-cancelled token.

[THE_RESIDUAL_WITNESS_IS_PRECONDITIONER_DISTORTED]:
- The convergence verdict is computed against the preconditioned residual the loop carries, not against the true residual of the original operator. A left-preconditioned system has the loop measuring the residual of the transformed system, whose norm relationship to the true error is distorted by the preconditioner's conditioning. A `Converged` verdict means the preconditioned residual fell below tolerance; it does not certify that the true residual did. The independent recomputation of the true relative residual against the original operator is mandatory in the calling code and is the only correctness witness that survives preconditioning, breakdown substitution, cancellation, and composite swallowing alike.
- The witness collapses every failure mode into one acceptance gate. The verdict enum routes recovery; the true-residual scalar decides trust. A single boundary owns both: it runs the solve under a catch combinator that converts the breakdown exception and the preconditioner-init exception into the failure rail, reads the verdict for routing, and accepts the iterate only if the recomputed true relative residual is finite and within tolerance — so a substituted-through breakdown, a count-capped partial, and a preconditioned `Converged` all pass through the same finite-and-in-tolerance test before the iterate is admitted as a solution.

```csharp
// One criterion-stack policy value fixes precedence and a size-scaled cap; one solve
// boundary folds verdict-routing, breakdown-catching, and the true-residual witness into
// a single Fin rail. The criterion order is the policy; the witness is the trust gate.
static Iterator<double> Stack(int dim, double tol) =>
    new(new IIterationStopCriterion<double>[]
    {
        new FailureStopCriterion<double>(),          // NaN ⇒ Failure, first so it is distinct
        new DivergenceStopCriterion<double>(0.08, window: dim < 256 ? 3 : 10),
        new ResidualStopCriterion<double>(tol),      // convergence BEFORE the cap
        new IterationCountStopCriterion<double>(8 * dim),  // size-scaled ceiling, last resort
    });

static Fin<double[]> SolveIter(
    Matrix<double> a, Vector<double> b, IIterativeSolver<double> solver,
    IPreconditioner<double> pre, int dim, double tol) =>
    Try(() =>                                         // preconditioner-init + breakdown both escape
    {
        var x = Vector<double>.Build.Dense(dim);
        var verdict = a.TrySolveIterative(b, x, solver, Stack(dim, tol), pre);
        return (verdict, x);
    }).ToFin()                                        // exception rail ⇒ Fin.Fail, never a stale buffer
      .Bind(t => TrueRelResidual(a, b, t.x) is var r && double.IsFinite(r) && r <= tol
            ? FinSucc(t.x.AsArray())                  // witness gate: Converged/Cancelled/substituted
            : FinFail<double[]>(Error.New(            //   all pass ONLY through finite-and-in-tolerance
                $"residual witness failed: verdict={t.verdict}, r={r}")));
```
