# Nested Absence Flattening and the Transformer Join

[DOUBLE_ABSENCE_AS_JOIN]:
- Two absences in one value — `Option<Option<A>>` — collapse by the monadic join, not by a structural unwrap. `flatten(ma)` is literally `ma.Bind(identity)`: the outer `None` and the inner `None` are folded into a single `None` by the bind short-circuit, and only a doubly-present `Some(Some(a))` survives as `Some(a)`. There is no third state — "outer present, inner absent" is indistinguishable from "outer absent" after the join, so any boundary that must report *which* layer was absent has to discriminate before the flatten, never after.
- Every absence-returning step inside a bind chain is already a flatten. `Bind<B>(Func<A, Option<B>>)` short-circuits when `!isSome` and otherwise returns `f(Value)` *as the result* — the inner `Option<B>` is the chain's value, never an `Option<Option<B>>` requiring a second collapse. Reaching for an explicit `flatten` after a `Bind` is a sign the projection was written as `Map` returning `Option`, producing `Option<Option<B>>`; switching that `Map` to `Bind` fuses the projection and the join into one short-circuiting step.
- `SelectMany` is a two-level join in one method: it tests `IsNone` on the receiver, calls `bind(Value)`, tests `IsNone` on that result, and only then runs the projection — so a query-comprehension over several optionals never materializes an intermediate nested optional and short-circuits at the first absent generator. The comprehension form is the dense surface for an N-source boundary admit where every source is independently optional; the first missing input collapses the whole `from … select` to `None` without a nested type ever existing.

[NULL_REENTRY_THROUGH_THE_JOIN]:
- The carrier's internal constructor sets the present-bit unconditionally and performs no null check; the value field is a nullable slot. A projection that returns `null` from the present branch therefore manufactures a present-state carrier wrapping `null` — a foreign sentinel that re-entered the domain wearing the absence type itself. `Map` and `Bind` both route their present branch through this unchecked constructor, so a null-returning mapper inside a flatten chain produces `Some(null)`, and every downstream consumer sees presence while holding a sentinel.
- The companion present-factory does not guard either: it calls the unchecked constructor directly, so constructing presence from a possibly-null reference is unsafe at exactly the seam that should be safest. The only null-rejecting admit is the boundary projection that tests the reference against `null` and returns the empty case on null — the present-factory is for values already proven non-null, and using it to admit a foreign field bypasses the projection the boundary exists to perform.
- The defensive law for any bi-directional or present-branch projection is to return the empty case explicitly when the target reference is absent, never to let a null flow into the present branch. The flatten itself cannot repair a `Some(null)`: `Bind(identity)` over `Some(Some(null))` yields `Some(null)`, because the join only collapses absent *cases*, never null *values* hiding inside present cases.

[NULLABLE_ADMISSION_ASYMMETRY]:
- Admitting a value-type `Nullable<T>` splits into two factories with opposite absence policy, and conflating them is a boundary bug. The strict admit throws when the nullable has no value — it is for inputs whose absence is a contract violation. The lenient admit returns the empty case when the nullable has no value — it is the sentinel projection proper. The reference-type lenient admit tests the reference against `null`; the value-type lenient admit tests the structure's has-value flag; both land on the same empty case, so one written empty-case consumer serves reference and value-type absence identically once past the seam.
- The implicit conversion from a nullable reference fires only for reference types and value-type nullables; it never intercepts a value type carrying a magic constant. A negative index sentinel, a zero handle, or a min-instant must be projected by an explicit discriminant expression — the lenient admit will wrap the sentinel as present because, to it, the value is non-null. The implicit operator and the lenient admit are byte-identical, so naming the admit explicitly at the seam costs nothing and marks the projection site.

[TRANSFORMER_AS_FLATTENED_STACK]:
- The absence transformer is a record over `K<M, Option<A>>` — one outer effect, one inner absence, fused into a single rail so the interior never threads `effect-of-optional` by hand. Its bind joins both layers at once: it binds the outer effect to reach the inner `Option`, matches it, routes the present case into the continuation's own inner run, and routes the absent case to the outer monad's pure of the empty case. The result is one `K<M, Option<B>>`, never a stacked `M<Option<M<Option<B>>>>` — the transformer *is* the standing flatten of the two layers, recomputed on every bind.
- Lifting an ordinary effect into the transformer rewrites every produced value to present via the outer functor mapping over the present-factory; because that factory does not null-check, an effect that yields `null` becomes a present-state carrier of `null` inside the stack — the null-reentry hazard reappears at the lift seam. Lifting an effect that already yields the absence carrier preserves the inner cases unchanged. These are distinct overloads with opposite absence semantics: the bare-effect lift can never produce the empty case, the effect-of-optional lift can; choosing the wrong one silently erases or fabricates absence.
- The transformer absorbs a wide family of sources through implicit conversions — a bare absence carrier, a pure value, a typed-failure-as-empty marker, the unit-as-empty marker, an effect, an effect-of-value, and an effect-of-optional all collapse to the one transformer without a manual lift. The typed-failure and unit conversions both land on the empty case, so a failure on a foreign rail flattens to absence inside the transformer — which means the *reason* for the failure is discarded at the conversion; a boundary that must preserve why the foreign step failed cannot route it through the absence transformer and must keep it on the typed-error transformer instead.

[CARRIER_REWRITE_VERSUS_JOIN]:
- Collapsing two absences (join) is distinct from moving the absence onto another rail (carrier rewrite), and the transformer publishes the rewrites as total arrows. The absence transformer rewrites to the typed-disjunction transformer by mapping its inner run, supplying the left value (or a thunk, or a monoidal empty) for the empty case — one structural arrow, no match-then-rewrap, and the rewrite names the absent-side payload exactly once. The arrow is on the inner run, so it threads through the outer effect untouched.
- A bare effect-of-optional does not need the transformer to flatten: bind the outer effect, match the inner case, and re-pure into the outer — but that is the transformer's own bind written by hand, so the standing transformer is the collapse of that pattern into a value. The hand-written form is the correct shape only at the single seam that *constructs* the transformer from a foreign effect-of-optional; everywhere inside, the transformer's bind owns the join.

[BIASED_CHOICE_OVER_THE_JOIN]:
- First-present selection across fallback sources is the choice combinator, not nested empty-case handling. The empty-or operator returns the receiver when present and the alternative otherwise, so a chain of fallback projections evaluates left-to-right and stops at the first present value — the dense replacement for nested unwrap-or ladders. The carrier implements the choice trait but not the monoid-of-alternatives trait, so the guard combinator that needs the latter does not compile against it; absence filtering inside a comprehension uses the filter/where surface, which returns the empty case when the predicate fails on a present value.
- The choice fold and the join fold are different folds over the same sources. The choice fold (first-present) keeps one survivor and discards the rest; the join fold (bind chain) requires *all* present and collapses on the first absent. A boundary that wants "any one of these foreign sources" uses choice; a boundary that wants "all of these foreign sources together" uses the bind chain or the applicative combine — selecting the wrong fold inverts the absence semantics of the whole seam.
