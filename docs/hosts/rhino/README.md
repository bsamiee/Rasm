# [RHINO_HOST]

Rhino host behavior comes from local RhinoWIP XML, decompile confirmation when XML is silent, the API rail, owner-owned bridge scenarios, and the `Rasm.Rhino` boundary.

## [1]-[READ_ROUTE]

- Member availability: local RhinoWIP XML and the API rail.
- Missing XML semantics: decompile confirmation or runtime bridge scenario.
- Boundary design: `Rasm.Rhino` and architecture routes.
- Runtime confirmation: owner-owned typed `[RhinoScenario]` sources through the bridge rail.

## [2]-[BOUNDARIES]

Rhino docs choose confirmation routes; `Rasm.Rhino` owns public boundary shape. Keep native values, nullable returns, bool failures, resource lifetimes, UI dispatch, and document mutation inside typed boundary rails before exposing project concepts.

## [3]-[BOUNDARIES]

- RhinoCommon member catalogs stay in generated or API-owned surfaces.
- Native wrappers add domain behavior or disappear.
- Compatibility projection exists only when live Rhino documents can still emit the native value and the owning boundary maps it to a canonical project value.
