# readme — anticipatory density evidence + structural laws

# ANTICIPATORY-COLLAPSE EVIDENCE BASE — Effect 3.21.2 (verified against `node_modules/effect/dist/dts/`)

All doctrine-side APIs below were confirmed in the installed type definitions. Counts are stated as facts. "Artifact" = a distinct declared type, runtime object/value, exported symbol, or hand-written function a maintainer must author and keep in sync.

[VALIDATED_WIRE_CONCEPT]:

Equality and test-data generation are absent or hand-rolled (add 2 more: `userEquals`, `userArbitrary`). Type count 3, shape/object count 3, exported symbols 6-8.
Doctrine inventory (1 declaration): `class User extends Schema.Class<User>("User")({ id: Schema.UUID, name: NonEmptyString })` (`Schema.Class` verified `Schema.d.ts:4385`). From that ONE class, all of the following are *derived*, not authored: the type (`User` is the instance type), the bidirectional codec (`Schema.decodeUnknownSync`/`encodeSync`, verified), structural equality (instances are `Data`-backed — `Equal`/`Hash` built in), `Schema.equivalence(User)` (verified `Schema.d.ts:5069`), `Arbitrary.make(User)` for PBT (verified `Arbitrary.d.ts:45`), `Schema.standardSchemaV1(User)` for boundary validators (verified `Schema.d.ts:124`), and `JSONSchema.make(User)`. Type count 1, shape count 1, exported symbols 1.
Growth diff — "add `tier` field with default + new invariant": Naive edits 6-8 sites (interface, schema, DTO, both mapper directions, validator, factory, and the now-stale equality/arbitrary). Doctrine edits 1 line inside the field block: `tier: Schema.optionalWith(Tier, { default: () => "free" })` (verified `Schema.d.ts:1194`) plus an inline `Schema.filter` for the invariant. Cost: naive +0 types but ~8 touch-points; doctrine +0 types, 1 touch-point. The codec, equality, and arbitrary re-derive automatically.

[ASYNC_WORKFLOW]:
Naive inventory (~50 LOC, 0 reusable values): nested `try/catch`, a `setTimeout`-based timeout race, a `for` retry counter loop with manual `await sleep(2**i)`, a hand-built tracing span with `start`/`end` in a `finally`, and a fallback `catch` branch. None of the control policy is a value; all of it is inline statements duplicated at every call site.
Doctrine inventory (~10 LOC, policy is a value): one pipeline `task.pipe(Effect.timeout("5 seconds"), Effect.retry(policy), Effect.catchAll(fallback), Effect.withSpan("task"))` where `policy = Schedule.exponential("100 millis").pipe(Schedule.intersect(Schedule.recurs(5)))`. Verified: `Effect.timeout` (`Effect.d.ts:12599`), `Effect.retry` takes a `Schedule` policy (`Effect.d.ts:7449`, doc confirmed), `Effect.withSpan` (`Effect.d.ts:26084`), `Effect.catchAll` (`Effect.d.ts:5582`), `Schedule.exponential` (`Schedule.d.ts:1502`), `Schedule.intersect` (`Schedule.d.ts:1655`), `Schedule.recurs` (`Schedule.d.ts:2390`). The span is automatic on the effect; the timeout interrupts the fiber (no orphan timer). Reusable values: 1 (`policy`).
Growth diff — "add jitter + give up after 30s total": Naive rewrites the loop to inject `Math.random()` per iteration and adds a wall-clock check inside the loop reading a captured start time — touching the retry body and the timeout race. Doctrine wraps the existing value: `policy.pipe(Schedule.jittered, Schedule.upTo("30 seconds"))` (verified `Schedule.jittered` `Schedule.d.ts:1784`, `Schedule.upTo` `Schedule.d.ts:3170`). Cost: naive edits 2 control structures in-place; doctrine appends 2 combinators to one value, every consumer of `policy` inherits the change.

[ERROR_TAXONOMY]:
Naive inventory (~50 LOC): a base `AppError` class + N subclass errors + a `code` enum + N `isXError(e): e is XError` type guards + a `serializeError()` switch + a discrimination switch at each catch site. Type count N+1 classes + 1 enum; guards and serializer are parallel structures that drift from the class list.
Doctrine inventory: a `Schema.TaggedError` family (verified `Schema.d.ts:4439`). Each `class NotFound extends Schema.TaggedError<NotFound>()("NotFound", { id: Schema.UUID })` is simultaneously a throwable (`YieldableError`), a typed `E`-channel member, and serializable via its own codec — no separate guards (the `_tag` is the discriminant), no separate serializer (the schema is the codec), no code enum (the tag *is* the code). Recovery via `Effect.catchTags({ NotFound: …, Conflict: … })` (verified `Effect.d.ts:6900`) is exhaustive over the union.
Growth diff — "new failure mode + HTTP status": Naive +1 class +1 enum entry +1 guard +1 serializer arm +1 catch arm = 5 sites, with no compiler check that the catch sites are complete. Cost: naive +5 drift-prone artifacts; doctrine +1 artifact, exhaustiveness enforced.

[KEYED_CACHE]:
Naive inventory (~45 LOC): a `Map<K,V>` + a parallel `Map<K, timestamp>` for TTL + an `inFlight: Map<K, Promise<V>>` for single-flight + a `setInterval` cleanup sweeper + manual eviction on capacity. 4 mutable structures + 1 timer; concurrent dedup and eviction are hand-coded and race-prone.
Doctrine inventory (1 value): `Cache.make({ capacity: 1000, timeToLive: "5 minutes", lookup })` (verified `Cache.d.ts:175`). Single-flight is built in — the source doc states: "If multiple fibers attempt to get the same key the lookup function will only be computed once and the result will be returned to all fibers" (`Cache.d.ts:46-48`). LRU eviction at capacity and TTL expiry are intrinsic; no sweeper timer, no in-flight map, no timestamp map. 1 value, 0 timers.
Growth diff — "per-key TTL": Naive replaces the single TTL constant with a per-entry TTL map and rewrites every read/write to consult it. Doctrine switches the constructor to `Cache.makeWith({ capacity, lookup, timeToLive: (exit) => Exit.isSuccess(exit) ? "10 minutes" : "10 seconds" })` (verified `Cache.d.ts:188`, `timeToLive: (exit: Exit.Exit<Value, Error>) => DurationInput`) — TTL can even differ by success/failure of the lookup. Cost: naive rewrites the data model; doctrine swaps one constructor and supplies a function.

[BOUNDED_VOCABULARY]:

Doctrine inventory (1 policy table): one `as const` row table `const TIERS = { free: { price: 0, quota: 10, label: "Free" }, pro: {...} } as const` from which the union derives (`type Tier = keyof typeof TIERS`) and a single `Schema.Literal(...keys)` validates the wire form (verified `Schema.Literal` `Schema.d.ts:398`). [STRUCTURAL_LAWS]:

1. Derivation collapses artifact count multiplicatively, not additively. One `Schema.Class` replaces {type, codec, validator, equality, arbitrary, JSON-schema} — 6→1. The naive growth is N artifacts × M concepts; the doctrine growth is M declarations, each spawning N derivations for free. Proven by [VALIDATED_WIRE_CONCEPT] (6→1) and [ERROR_TAXONOMY] (class+enum+guard+serializer → one `TaggedError`).

2. Policy-as-value moves growth from edit-N-sites to add-one-input. A `Schedule`, a `Config`, a `Cache` config, or an `as const` row is a value composed once and consumed everywhere; new requirements append a combinator or a row instead of editing every call site. Proven by [ASYNC_WORKFLOW] (jitter+timeout = +2 combinators on one value) and [BOUNDED_VOCABULARY] (new tier = +1 row, 0 switch edits).

3. Discriminated families make exhaustiveness a compile error; parallel functions make omission silent. `TaggedRequest`/`TaggedError` + `catchTags`/`Match` force every new case to be handled at every dispatch; 4 loose functions + a switch compile fine with a missing arm. Growth cost shifts from "runtime bug surface" to "type error you cannot ignore." Proven by [VERB_FAMILY] and [ERROR_TAXONOMY] 5th-case growth.

4. Lifecycle ownership collapses into Scope; manual dispose multiplies coordination state. `Effect.forkScoped` + `SubscriptionRef` + `Stream` eliminate the {timer handle, listener array, dispose method, paused flag} cluster — teardown is structural, not authored. Each naive lifecycle concern (start, stop, pause, error) is a new mutable field; the doctrine threads them through one scoped pipeline. Proven by [STATEFUL_POLLER].

5. The wire and the domain are the same declaration. Naive code maintains interface↔DTO↔mapper as three drifting artifacts; `Schema.Class` makes encode/decode two directions of one codec, so the mapper layer (and its bidirectional drift surface) ceases to exist. Proven by [VALIDATED_WIRE_CONCEPT] (mapper pair → 0) and [CONFIGURATION_BLOCK] (env-read + validate + default → one `Config` value).

6. Concurrency invariants belong to the primitive, not the caller. `Cache` owns single-flight dedup and LRU/TTL eviction; the naive {in-flight map, timestamp map, sweeper interval} are re-implementations of guarantees the library already proves correct. New caching requirements (per-key TTL by Exit) are constructor parameters, not new data structures. Proven by [KEYED_CACHE].

7. Defaults, validation, and secrecy are facets of one value, not separate passes. `Config.withDefault`/`Config.redacted`/`optionalWith({default})` attach defaulting, redaction, and defaulting-with-validation to the field itself; naive code runs them as separate, individually-forgettable functions. Growth ("new secret per-env") is a field facet, not a new code path. Proven by [CONFIGURATION_BLOCK] and the field-default growth in [VALIDATED_WIRE_CONCEPT].

8. The denser surface has the cheaper derivative. Across all eight scenarios the naive next-requirement cost is 4-8 touch-points with silent-omission risk; the doctrine cost is 1 declaration/row/combinator with compiler-enforced completeness. Density is not measured at rest (LOC today) but at the margin (cost of the next field/verb/tier/failure) — the doctrine's marginal artifact count is O(1) where naive is O(N-sites). This is the bar: not "smaller now" but "constant-cost to grow."

Further Considerations
- `Schema.Class` instances are `Data`-backed, so `Equal.equals`/`Hash` work structurally without `Schema.equivalence` — the derived `equivalence` is for *encoded* comparison or custom `Equivalence`-consuming combinators; the README should distinguish instance equality (free) from schema-level equivalence (derived) to avoid implying two equality systems.
- `Effect.retry` accepts both a bare `Schedule` and an options object (`{ schedule, times, while, until }`); the densest doctrine form keeps the `Schedule` as a named reusable value (law 2) rather than inlining the options object, since only the value form is composable with `jittered`/`upTo`/`intersect` downstream.
- The 30.2KB MEMORY.md is over its 24.4KB limit and only partially loaded — several reference entries (notably the Effect v5 migration note `reference_languageext_v5.md` concerns LanguageExt, not this TS doctrine) may collide in vocabulary; when this doctrine lands, the TS shape-law cards should not reuse the C# `[Union]`/`Fin` rail names, since `TaggedRequest`/`catchTags`/`E`-channel are the TS-native equivalents and conflating them in shared memory will mislead future agents.
