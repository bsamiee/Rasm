# [PATTERNS]

Pattern law binding two or more language branches. A row admitted here binds every branch it names; a single-branch law routes to that branch's stack doctrine instead.

## [01]-[ROWS]

| [INDEX] | [LAW]                                                                                                           | [BINDS]             |
| :-----: | :-------------------------------------------------------------------------------------------------------------- | :------------------ |
|  [01]   | a derived artifact keys on the content hash of its source; cache validity is key equality, never path or mtime  | C#, Python, tooling |
|  [02]   | root and anchor discovery walks upward to a sentinel file, never a fixed `parents[N]` depth or a cwd assumption | Python, C#, tooling |
|  [03]   | an operational rail returns one typed envelope; failure rides the envelope, never sentinel values in data rows  | all branches        |
|  [04]   | a wire token admits only its exact emission spelling; a tolerant parse re-emitting normalized forks the key     | all branches        |
|  [05]   | a multi-field hash preimage length-frames every variable-width field and count-frames every adjacent collection | all branches        |
|  [06]   | a content key's preimage covers every identity-bearing member; a member outside it is declared derived on site  | all branches        |

C# spells the content key `XxHash128` on the wire, and cross-runtime peers compare wire tokens byte-wise — the emitting owner's spelling is the admission grammar. Row [05] rejects separator-joined concatenation outright: a separator character inside one value shifts two distinct field splits onto one digest, and fixed-width elements never identify the boundary between collections. Row [06] makes the key and the record one identity: a stored member the preimage omits is a split-brain whose stated re-word or re-order semantics is false unless the declaration names it derived-or-annotation.
