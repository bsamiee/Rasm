# [BIM_FAULTS]

`BimFault` is the closed `[Union]` fault band (band 2600) for the BIM-and-exchange failures every `Rasm.Bim` projector, codec, semantic, review, and planning entrypoint produces — `ModelRejected` (a foreign payload the semantic projection rejects, an IFC-legality violation, or a captured native-package exception funnelled at the import/export boundary), `UnmappedClass` (an IFC entity-type, classification code, zone, geo-feature, or predefined token the closed `IfcClass`/`Classification` vocabulary does not carry), `DanglingReference` (a `GlobalId`/`NodeId` a spatial host, type binding, edge endpoint, port, priced item, or commit names but the model graph never declares), `CodecReject` (a write-only/read-only capability miss, a catalogue-pending codec, an unresolved `Detect` format, or a degraded REST/geo service), and `CapabilityMiss` (an in-process evaluation the managed branch owns no kernel for — an IFC/STEP BRep tessellation, a CRS the transform algebra cannot reconcile, or a companion the rail cannot reach). The band is `Expected`-derived (`IValidationError<BimFault>`), so band 2600 IS the `Expected` `Code` and a bare typed case lifts DIRECTLY onto the `Fin<T>`/`Validation<Error,T>` rail through the LanguageExt implicit `Error` conversion — no `.ToError()` lowering hop — mirroring the sibling `Rasm.Materials/Construction/assembly#PLACEMENT_MODEL` `ConstructionFault` (2350) and the seam `Rasm.Element/Projection/fault#FAULT_BAND` `ElementFault` (2500) band shapes; each case carries the kernel `Op` operation context and a `Category` telemetry projection, and a recovery reads `error.IsType<BimFault.DanglingReference>()` or `error.HasCode(2600)`, never a message substring. The legacy per-case-code (`2601`–`2605`) + manual `Code`/`Message`/`ToError()` shape is the defect this owner closes: the typed case IS the `Error`.

The band is one of three on the in-process `Fin<T>` federation and never re-cases its neighbors. The kernel `Rasm` `GeometryFault` (the geometry-algorithm band) carries the shared degenerate-geometry failure a re-imported companion GLB raises, lowered through its OWN plain-union `.ToError()` because the kernel geometry band is not `Expected`-derived; the seam `Rasm.Element` `ElementFault` (2500) carries the structural-graph failures the seam's `GraphDelta`/`Bake`/`Assemble` total `Switch` produces — edge-endpoint resolution and `NodeId` uniqueness. So a projector's GeometryGym-side rejection is `BimFault` while the seam's structural-law rejection is `ElementFault`, and the IFC-SEMANTIC legality the `Projection/semantic#GRAPH_LEGALITY` `IfcLegality : IGraphConstraint` decides accumulates `BimFault.ModelRejected` applicatively over `Validation<Error,Unit>` [M3] — the `Expected` + `IValidationError` shape keeping the typed arm recoverable through `Error.Combine`, which a `.ToError()` hop would erase.

Wire posture: HOST-LOCAL. `BimFault` rides the rail every Bim entrypoint returns and never sits between wire and rail — the `Projection/semantic#SEMANTIC_PROJECTOR` `SemanticProjector.Project` / `#IFC_EGRESS` `Emit` / `#RELATION_ALGEBRA` `EdgeProjection.All`, the `Model/elements#IFC_CLASS` `IfcClass.Resolve`/`AdmitPredefined`, the `Semantics/classification#CLASSIFICATION_AXIS` `Classify`, the `Exchange/import#IMPORT_RAIL` `ImportGeometry`/`ImportIfc`/`ImportSpeckle` · `Exchange/export#EXPORT_RAIL` `Export`/`ExportIfc` · `Exchange/format#FORMAT_AXIS` `Detect` · `Exchange/tessellation#TESSELLATION_BRIDGE` `Plan`/`Resolve` exchange rail, the `Semantics/georeference#GEO_PROJECTION` / `Semantics/geospatial#GEOSPATIAL_SEAM` projections, the `Review/validation#IDS_FACETS` · `Review/issues#BCF_ARCHIVE` · `Review/diff#MODEL_DIFF` · `Review/versioning#VERSION_GRAPH` review rail, and the `Planning/cost#ESTIMATE` · `Planning/schedule#SCHEDULE` planning rail — while the `IfcLegality.Validate` legality gate alone returns `Validation<Error,Unit>` for applicative accumulation. `ElementSet.Query` (`Model/query#ELEMENT_SET`) is total and carries no fault rail. The folder carries no fault-band string-comparer accessor: `BimFault` dispatches through its generated total `Switch`, keys no `FrozenDictionary`, and a `[KeyMemberComparer]` on the fault is the deleted form.

```csharp signature
// --- [RUNTIME_PRELUDE] --------------------------------------------------------------------
using LanguageExt;
using Thinktecture;
using Expected = Rasm.Domain.Expected;
using Op = Rasm.Domain.Op;
using static LanguageExt.Prelude;

namespace Rasm.Bim;

// --- [ERRORS] -----------------------------------------------------------------------------
// The BIM-and-exchange fault band: Expected-derived so band 2600 IS the Expected Code and the typed case
// lifts bare onto Fin<T>/Validation<Error,T> with no .ToError() hop (mirrors ConstructionFault 2350,
// ElementFault 2500). [SkipUnionOps]: the band needs no generated per-case SelfOp because every case carries
// an explicit kernel Op operation context; arm-specificity is error.IsType<Case>(), telemetry error.Category().
// Construction is the nested-record ctor (new BimFault.Case(key, detail)) — [Union] generates the Switch/Map
// projection, not a per-case factory — and the Expected derivation makes the case an Error directly, so it lifts
// bare; IfcLegality.Validate accumulates ModelRejected over Validation via Error.Combine, which a .ToError() hop
// would erase. The kernel Rasm.Domain.Expected base ctor is PARAMETERLESS (Code a virtual Error member, Message
// abstract), so band 2600 IS the Code as an override and Message projects Detail.
[SkipUnionOps]
[Union]
public abstract partial record BimFault : Expected, IValidationError<BimFault> {
    private BimFault(Op key, string detail) { Key = key; Detail = detail; }

    public Op Key { get; }
    public string Detail { get; }
    public override int Code => 2600;
    public override string Message => Detail;

    // The IValidationError<BimFault>.Create string-only admission the generated converter bridge calls on a
    // deserialization reject; routes the unspecific case under a boundary-admission Op so the bridged case still
    // carries an operation context (never a default Op) and a raw message never escapes the typed family.
    private static readonly Op Admission = Op.Of(name: nameof(Admission));
    public static BimFault Create(string message) => new ModelRejected(Admission, message);

    public sealed record ModelRejected(Op Key, string Detail)     : BimFault(Key, Detail) { public override string Category => "ModelRejected"; }
    public sealed record UnmappedClass(Op Key, string Detail)     : BimFault(Key, Detail) { public override string Category => "UnmappedClass"; }
    public sealed record DanglingReference(Op Key, string Detail) : BimFault(Key, Detail) { public override string Category => "Reference"; }
    public sealed record CodecReject(Op Key, string Detail)       : BimFault(Key, Detail) { public override string Category => "Codec"; }
    public sealed record CapabilityMiss(Op Key, string Detail)    : BimFault(Key, Detail) { public override string Category => "Capability"; }
}
```

## [03]-[RESEARCH]

- [FAULT_BAND_SHAPE]: the `Expected`-derivation grounds against the rebuilt `Rasm.Materials` `ConstructionFault` (2350) and seam `Rasm.Element` `ElementFault` (2500) band shapes — both `: Expected, IValidationError<TSelf>` with a leading `Op Key`, a per-case `Category` override, the `Create` admission bridge, and bare-case lift — and the kernel `Rasm.Domain` `Op` `[ValueObject<string>]` (`Op.Of` caller-member-name), `Expected : Error` (parameterless `protected` ctor, `virtual Category`, and the inherited `virtual Error.Code`/`abstract Error.Message`), and the Thinktecture `IValidationError<TValidationError>` contract the `[Union]` generator emits (`.api/api-thinktecture-json`); band 2600 is a `Code => 2600` override (the kernel base carries no `Code`/`Message` ctor slot — the LanguageExt `Expected(string,int,Option)` ctor belongs to the OTHER `LanguageExt.Common.Expected`, disambiguated by the `using Expected = Rasm.Domain.Expected;` alias the production `UiFault` band uses; that production band also carries `[SkipUnionOps]`, the canonical fault-band annotation, because every case carries an explicit `Op Key` and needs no generated per-case `SelfOp` — distinct from the kernel `Fault` band's `[GenerateUnionOps]`), `Message => Detail`, arm-specificity is `error.IsType<BimFault.Case>()` (the `Error.IsType<E> where E : Error` probe — there is no `Error.Is<E>()`), and the `Category` string projects through the kernel `FaultExtensions.Category(this Error)` for telemetry. The legacy `[Union(ConversionFromValue = None)]` + manual `Code` switch + `Message` switch + `ToError()` shape is retired in favor of the `Expected` lift — the C# pattern law "fault = a closed `[Union]` deriving from `Expected`" the two rebuilt sibling bands already realize.
- [VALIDATION_ACCUMULATION]: the `Expected` + `IValidationError<BimFault>` shape is REQUIRED by the `Projection/semantic#GRAPH_LEGALITY` `IfcLegality : IGraphConstraint.Validate(GraphDelta, ElementGraph) → Validation<Error,Unit>` [M3] applicative accumulation — the `Apply`/`&` `Error.Combine` folds each independent `BimFault` into a `ManyErrors` whose `IsType`/`HasCode`/`Filter` overrides recurse over the inner collection, so the typed arm stays recoverable through the accumulated failure — `error.IsType<BimFault.ModelRejected>()` and `error.HasCode(2600)` survive the combine and `error.Filter<BimFault.ModelRejected>()` extracts the matching arms out of the `ManyErrors` — where a bare `Error.New(2600, message)` from a `.ToError()` hop would erase the arm into an untyped message; the seam's structural-graph `GraphDelta` switch (→ `ElementFault`) and Bim's IFC-semantic constraint (→ `BimFault` over `Validation`) are the disjoint two-interface split M3 mandates, the seam carrying no IFC vocabulary and Bim's constraint carrying no structural-graph mechanics.
- [GEOMETRY_FAULT_COMPOSE]: the kernel `Rasm` `GeometryFault.DegenerateInput` compose for the shared empty/non-finite re-imported-tessellation failure grounds against the `Exchange/tessellation#TESSELLATION_BRIDGE` companion re-import (`Fin<ImportedGeometry>`) and the kernel geometry fault band — `GeometryFault` is the kernel geometry-ALGORITHM band lowered through its OWN plain-union `.ToError()` (it is NOT `Expected`-derived, so its idiom differs from `BimFault`'s bare lift), and a re-imported companion GLB carrying a degenerate vertex set lowers `GeometryFault.DegenerateInput(...).ToError()` while a `BimFault` lifts bare — two `Error` values on one `Fin<ImportedGeometry>` rail, never a sixth Bim arm and never a synthesized `GeometryFault.ModelRejected` the kernel union never declares.
