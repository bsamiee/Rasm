# [PY_ARTIFACTS_API_PYTHON_MAGIC]

`python-magic` is the libmagic-backed worker-band detector the `exchange/detect#DETECT` owner composes BESIDE the default in-process `puremagic` sniffer — a thin `ctypes` binding (`magic/__init__.py`, no compiled extension of its own) over the host `libmagic.so`/`.dylib`/`.dll`, exposing a flag-pinned `Magic` cookie (`magic_open`/`magic_load`/`magic_buffer`/`magic_file`/`magic_descriptor`) plus stateless `from_buffer`/`from_file`/`from_descriptor` module rows that cook bytes/path/descriptor into a MIME type, human description, charset encoding, or `/`-separated extension list under one `MAGIC_*` bitmask. libmagic is a Forge-provisioned native dependency NOT on the runtime loader path, so the `Detect` owner reifies every detection across the `anyio.to_process.run_sync` subprocess seam — bounded by the shared `WORKER_BAND` `CapacityLimiter`, a transient `BrokenWorkerProcess` death recovered by a `stamina.AsyncRetryingCaller(...).on(BrokenWorkerProcess)` retry before `async_boundary` rails the exhausted failure — onto a worker that imports `magic` at boundary scope and constructs one `functools.cache`-memoised `Magic` cookie per `MagicFacet`, because libmagic returns one cooked string per call. Per the campaign brief this package is DEMOTED behind `puremagic` for the default path and RETAINED only where its broader compiled magic database recognizes a leaf signature `magic_data.json` lacks (the long tail of niche binary formats) — a strict-stronger-on-a-distinct-axis retention, never an overlap; the two never both own the default sniff. The owner composes the `Magic` cookie, the `setparam` `MAGIC_PARAM_*` recursion/ELF-bomb caps, the `version()` gate, and the `_handle509Bug` null-result quirk into the `DetectIdentity`/`MediaClass`/`Container`/`Trust` fold; it never re-implements the magic-pattern database, the compiled `.mgc` loader, or the per-flag bit math the native library already owns.

## [02]-[PUBLIC_TYPES]

[PUBLIC_TYPE_SCOPE]: cookie and fault
- rail: file control

The cookie carries a `threading.Lock` (`Magic.lock`) so a single cookie is reusable across calls under the worker's GIL; the `_handle509Bug` instance method returns `application/octet-stream` for the known libmagic 5.09 null-MIME quirk rather than raising. ## [03]-[ENTRYPOINTS]

[ENTRYPOINT_SCOPE]: stateless module detection
- rail: file control

The module functions own an internal `_instances`-cached default cookie keyed by the `mime` boolean (`_get_magic_type(mime)` builds `Magic(mime=mime)` once per boolean and reuses it); they expose ONLY the `mime` switch (description vs MIME-type). A richer flag policy — encoding, extension, compression look-through, keep-going, params — requires holding a `Magic` cookie, which is why the `Detect` owner constructs cookies rather than calling these rows.

[ENTRYPOINT_SCOPE]: configured cookie
- rail: file control

`Magic.__init__` is where every detection flag lives; the per-call methods take only the source and return the cooked string under the cookie's flags. One cookie per flag-policy is the canonical owner — the flags are NOT per-call arguments. The cookie auto-applies `setparam(MAGIC_PARAM_NAME_MAX, 64)` in `__init__` (the issue-190 fixed-internal-limit workaround) and tolerates a libmagic that rejects that call. The `Magic.from_descriptor` cookie method takes NO per-call `mime` override (unlike the module row), so the `Detect` worker excludes it — the gated `to_process` worker cannot resolve a parent-process file descriptor; only `Buffer`→`from_buffer` and `File`→`from_file` cross the seam. The `SURFACE` names drop the `Magic.` prefix; `extension=True` raises `NotImplementedError` when `version() < 524`, and `setparam` raises `MagicException` when libmagic lacks the param. The constructor is `Magic(mime=False, magic_file=None, mime_encoding=False, keep_going=False, uncompress=False, raw=False, extension=False)` (booleans in the flag table below).

| [INDEX] | [SURFACE]         | [CALL_SHAPE]                                                                               |
| :-----: | :---------------- | :----------------------------------------------------------------------------------------- |
|  [01]   | `from_buffer`     | `Magic.from_buffer(buf: bytes \| str) -> str` (thread-locked; `str` `utf-8`-encoded first) |
|  [02]   | `from_file`       | `Magic.from_file(filename: bytes \| str \| PathLike) -> str`                               |
|  [03]   | `from_descriptor` | `Magic.from_descriptor(fd: int) -> str` (excluded by the worker; fd does not cross seam)   |

[ENTRYPOINT_SCOPE]: flag and param vocabulary
- rail: file control

The constructor booleans set the underlying `MAGIC_*` bitmask; the `MAGIC_PARAM_*` ordinals address `setparam`/`getparam`. Compose the booleans, never the raw bits. The owner pins exactly ONE output boolean per facet — `Magic(mime=True)` then `Magic(mime_encoding=True)` as separate cookies — because `Magic(mime=True, mime_encoding=True)` returns the COMBINED `text/plain; charset=utf-8` form, not two facets; this is the `_FACET_FLAG` one-flag-per-cook law the `Detect` owner binds.

| [INDEX] | [FLAG_BOOLEAN]  | [MAGIC_BIT]                     | [EFFECT]                                                                           |
| :-----: | :-------------- | :------------------------------ | :--------------------------------------------------------------------------------- |
|  [01]   | `mime`          | `MAGIC_MIME_TYPE` (`0x10`)      | return the MIME type (`application/pdf`) instead of a description                  |
|  [02]   | `mime_encoding` | `MAGIC_MIME_ENCODING` (`0x400`) | return the charset (`utf-8`, `binary`)                                             |
|  [03]   | `extension`     | `MAGIC_EXTENSION` (`0x1000000`) | return a `/`-separated valid-extension list; requires libmagic `>= 524`            |
|  [04]   | `uncompress`    | `MAGIC_COMPRESS` (`0x4`)        | sniff inside gzip/bzip2/xz containers                                              |
|  [05]   | `keep_going`    | `MAGIC_CONTINUE` (`0x20`)       | return all matches separated by `n- `, not just the first                          |
|  [06]   | `raw`           | `MAGIC_RAW` (`0x100`)           | do not translate unprintable characters                                            |
|  [07]   | `magic_file=`   | (passed to `magic_load`)        | load a custom compiled `.mgc` or text magic database instead of the system default |

Two raw-bit families have no constructor boolean and compose directly:
- `MAGIC_NO_CHECK_*` (`COMPRESS`/`TAR`/`SOFT`/`APPTYPE`/`ELF`/`ASCII`/`TROFF`/`FORTRAN`/`TOKENS`): disable a libmagic test class to narrow the check set on an untrusted/latency-bounded ingest.
- `MAGIC_PARAM_*` (`INDIR_MAX`(0)/`NAME_MAX`(1)/`REGEX_MAX`(5)/`BYTES_MAX`(6) recursion/name/regex/byte caps plus `ELF_PHNUM_MAX`(2)/`ELF_SHNUM_MAX`(3)/`ELF_NOTES_MAX`(4) ELF-table caps): the `setparam`/`getparam` magic-bomb/ELF-bomb defense.

[ENTRYPOINT_SCOPE]: loader and module substrate (mechanics, not call surface)
- rail: file control

`magic.loader.load_lib` is the native-resolution mechanism the import guard depends on, not a surface the owner calls. It iterates `_lib_candidates()` (`ctypes.util.find_library('magic')`, then platform-specific paths: `/opt/local/lib`, `/usr/local/lib`, `/opt/homebrew/lib`, the Homebrew Cellar glob on darwin; `libmagic`/`magic1`/`cygmagic-1`/`libmagic-1`/`msys-magic-1` DLL prefixes on win32; `libmagic.so.1` on linux) and raises `ImportError('failed to find libmagic')` when none load — the provisioning fault the owner surfaces, distinct from a content fault. `maybe_decode` (`utf-8`/`backslashreplace`) and `coerce_filename` are the C-string boundary coercions every cooked return crosses.

| [INDEX] | [SURFACE]                                     | [CALL_SHAPE]                                             |
| :-----: | :-------------------------------------------- | :------------------------------------------------------- |
|  [01]   | `magic.loader.load_lib`                       | `load_lib() -> ctypes.CDLL`                              |
|  [02]   | `errorcheck_null` / `errorcheck_negative_one` | `(result, func, args) -> result`                         |
|  [03]   | `maybe_decode`                                | `maybe_decode(s: bytes \| str) -> str`                   |

- [03]: decode a libmagic C-string return as `utf-8`/`backslashreplace` (file-charset metadata never aborts the cook).
- [04]: the module-level `ctypes` binding (`restype=c_int`, `argtypes=[magic_t, c_int]`) applying a recomputed `MAGIC_*` bitmask onto a LIVE `Magic.cookie` after construction — the mechanism the `MAGIC_NO_CHECK_*` narrowing rides (no constructor boolean); `detect#DETECT._cookie` calls it to narrow the check set on an untrusted ingest, targeting the cookie's `Magic.cookie`/`Magic.flags`.

## [04]-[IMPLEMENTATION_LAW]

[CONTENT_IDENTITY]:
- import: `import magic` at boundary scope only — module-level import is banned by the manifest import policy, and the `Detect` page binds it as `lazy import magic` reified inside the `to_process` worker. Wrap the import in the libmagic-absent guard: an `ImportError` at import time means `magic.loader.load_lib` found no `libmagic`, which the owner surfaces as a Forge-provisioning fault, not a content fault.
- worker-band law: libmagic is a native C dependency OFF the runtime loader path, so EVERY detection crosses the `anyio.to_process.run_sync(_gated_detect, source, profile, policy, limiter=WORKER_BAND)` subprocess seam — never the in-process call the pure-Python `puremagic` peer rides. The `File` source row is read ON the worker side so the parent never pickles megabytes across the seam (the canonical reason `Source.File` exists beside `Source.Buffer`). A transient `BrokenWorkerProcess` death (OOM/signal kill of the worker) is recovered by a bounded `stamina.AsyncRetryingCaller(attempts=3, timeout=30.0).on(BrokenWorkerProcess)` retry; a deterministic crash is NOT retried (the `MagicParam` caps keep the magic-bomb defense, so a recursive-indirection bomb is a deterministic failure the boundary rails, not a retry target).
- detector axis: one `Magic` cookie per `MagicFacet` owns the flags, the `threading.Lock`, and the loaded database; the module-level `from_buffer`/`from_file`/`from_descriptor` are the stateless rows over the `_instances`-cached default cookie and expose ONLY `mime`. A multi-output identity (MIME + encoding + extensions in one pass) holds one `functools.cache`-memoised cookie per facet in the one worker crossing — the `_cookie(magic_file, facet_flag, flags, params)` build loads the compiled database once per config per worker and is reused across detections, never reloaded per facet per call.
- source axis: buffer and file are two call rows on the same cookie surface, never parallel detector types — `Source.Buffer`→`Magic.from_buffer`, `Source.File`→`Magic.from_file`, matched by one total `match` in `_cooked`. The in-memory-bytes row is canonical because admission already holds the payload; the file row reads on the worker side to avoid the parent pickling the payload. The descriptor row is excluded from the worker (a parent-process fd does not cross the process seam).
- output axis: `mime`/`mime_encoding`/`extension` are flag selections on one detection, never separate functions per output; the `_FACET_FLAG` law pins exactly one boolean per facet (`MIME`→`mime`, `ENCODING`→`mime_encoding`, `EXTENSION`→`extension`; `DESCRIPTION` cooks under no flag), so a full identity holds four cookies in one crossing and the worker folds the four cooked strings into the `DetectIdentity`. libmagic returns one cooked string per call, so the strongest match is `raw.split("\n- ", 1)[0]` cutting the `MAGIC_CONTINUE` separator.
- fault axis: a failed libmagic call raises `MagicException` (carrying `.message`); the cookie's `_handle509Bug` returns `application/octet-stream` for the known 5.09 null-result MIME quirk (a valid unknown-content MIME the owner classifies `MediaClass.UNKNOWN`, never an escaping exception). A genuinely broken magic database's `MagicException` is the worker raise the `async_boundary` `CLASSIFY` catch-all `boundary` case lands at the seam — lifted once at the boundary, never escaping as a bare exception into domain logic.
- evidence: each detection captures the MIME, description, charset encoding, detected-extension tuple, `keep_going` match set, input byte length, and the resolving `libmagic_version` as a content-identity receipt; the worker folds it into the frozen `DetectIdentity` `msgspec.Struct` and pickles that back across the process seam, never a re-minted identity. `version()` both gates the `EXTENSION` facet (`>= 524`) and rides onto `DetectIdentity.libmagic_version`.
- boundary: python-magic owns content sniffing only — it mints no `ContentKey` (the `evidence/identity#IDENTITY` content hash is a producing owner's concern) and contributes no `ArtifactReceipt`. Its resolved `DetectIdentity` is the admission gate the per-format readers dispatch on through the `MediaClass` discriminant — `python-docx`/`python-pptx`/`openpyxl` the OOXML branches, `odfpy` the ODF branch, `pymupdf`/`pypdf` the `application/pdf` and `EBOOK` branches, `pillow`/`pyvips` the image branches, `msoffcrypto-tool` the legacy/encrypted-Office branch, `av` the audio/video branches, the `scene/*` owners the `model/*` branch. The descriptive metadata INSIDE the identified container is the `exchange/metadata#METADATA` owner's concern, never re-read here. The `compat` shim functions (`detect_from_filename`/`detect_from_content`/`detect_from_fobj`/`open`/`magic_set`/`_create_filemagic`) are deprecation-wrapped and are NOT admitted — they collide with the upstream file-5.x binding; use the `Magic` cookie. Live UI stays outside this package.
