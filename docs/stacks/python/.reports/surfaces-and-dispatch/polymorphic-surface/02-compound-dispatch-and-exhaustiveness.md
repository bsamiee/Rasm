# [COMPOUND_DISPATCH_EXHAUSTIVENESS_DENSITY]

[COMPOUND_AND_NESTED_DISCRIMINANTS]:
- A single `@tagged_union` surface owning 5-8 correlated modalities scales to a compound axis by factoring one dimension into a nested `@tagged_union` type-parameter; the outer union dispatches on direction (e.g. `read`, `write`, `delete`) and the inner dispatches on scope (e.g. `one`, `batch`), each with `assert_never` guards at the leaf. This keeps two independent concerns independently foldable without multiplying case count at the outer level: 3 outer arms × 2 inner arms = 6 paths behind one `def handle(cmd: Cmd)` entrypoint. Verified live: `expression` 5.6.0, Python 3.15.0b1, 2026-06-09.
- The alternative — encoding both axes into a single flat `@tagged_union` with 6 case fields (`read_one`, `read_many`, `write_one`, `write_many`, `delete_one`, `delete_many`) and a two-level guard match — is concise for at most 4 combined paths. Beyond 4, declaration order is meaningful (it governs `_index`-based priority), and a prefix-guard `if t.startswith('read')` cluster is still syntactically valid but forces the checker to treat the guard as a runtime predicate rather than a structural discriminant, losing the static advantage. Nested `@tagged_union` wins on structural density when axes are independently variable. Verified live 2026-06-09.
- Pydantic 2.13.4 supports compound discriminants via `Discriminator(callable)` where the callable extracts a composite key from two or more fields: `Discriminator(lambda v: f"{v['direction']}_{v['scope']}")`. Each branch must be tagged with `Annotated[Model, Tag('key')]`; omitting `Tag` leaves the branch unreachable and pydantic raises `PydanticUserError`. The callable form is the only way to dispatch on two non-overlapping fields; Pydantic's `Discriminator(field_name: str)` accepts only a single field name. `TypeAdapter(Annotated[Union[A, B, C], Discriminator(fn)])` compiles without error; `TypeAdapter(Union[A, B, C])` alone with `Discriminator` on the raw union fails with `AttributeError: __pydantic_core_schema__`. Source: `pydantic/__init__.pyi`, pydantic 2.13.4 installed 2026-06-09.

[ENCODING_LIFECYCLE_TRADEOFFS]:
- `@tagged_union(frozen=True)` (expression 5.6.0): one class, O(1) init, `__hash__` derived from `(classname, tag, value)`, directly usable as dict key or in `heapq`. Lifecycle position: canonical in-process command/query ADT and closed domain variant. Static exhaustiveness: FAILS in both `ty` 0.0.44 and `mypy` 1.x — because the checker infers the residual match arm as `Shape` (the single class), not `Never`. `assert_never` fires at runtime on a missing arm but produces no static diagnostic. The mechanism: checkers narrow `A | B | C` type unions to `Never` after all arms; they do not narrow a single class with a string `tag` field even when all `Literal` tag values are covered. Verified: `ty` 0.0.44 reports `@Todo` (not `Never`) on `assert_never(never)` for all `@tagged_union` surfaces including `Result[T, E]` itself; annotating `tag: Literal['circle', 'rect']` does not change this. Source: `.venv/bin/ty check`, 2026-06-09.
- Standard type-union `A | B | C` (dataclasses with `@disjoint_base` or pydantic `BaseModel` with `Literal` discriminant): each variant is a distinct type; checkers narrow exhaustively to `Never` after all arms. `ty` 0.0.44 passes `All checks passed!` on a complete `Circle | Rect | Tri` match with `assert_never`. Lifecycle position: versioned API boundary, wire models, any surface where static exhaustiveness proof is a hard requirement. Penalty: new modality requires a new class, a new union alias update, and a new match arm — three edits vs one for `@tagged_union`. Source: verified with `ty check` on both forms, 2026-06-09.
- Pydantic discriminated union `Annotated[Union[A, B, C], Discriminator('kind')]`: same static exhaustiveness as the plain dataclass union because each branch is a distinct `BaseModel` subtype; adds coercion, validation, and schema generation at cost of ~5-8× parse overhead vs msgspec. Lifecycle position: rich ingress with field validation, settings parsing, or schema-bound API boundaries. Not suited for high-frequency in-process dispatch. Source: pydantic 2.13.4 installed 2026-06-09.
- msgspec 0.21.1 discriminated union with `tag=True` or `tag='value', tag_field='kind'`: encodes the discriminant into the wire payload (`{"type":"Circle","r":3.0}` or `{"kind":"circle","r":3.0}`); decodes back to the concrete struct type via `Union[A, B]` in `msgspec.json.decode`. Round-trips 10-30× faster than pydantic. Does not participate in in-process `match` dispatch — it is a wire codec, not a domain type. Pairing: msgspec structs own the wire representation; `@tagged_union` owns the domain representation; a boundary adapter converts between them. Source: msgspec 0.21.1 installed 2026-06-09.
- `@disjoint_base` (Python 3.15.0b1, `typing.disjoint_base`): a decorator that sets `__disjoint_base__ = True` on the class and all its subclasses; checkers can use this to prove that two hierarchies do not overlap. It does not seal the hierarchy (subclasses can still be added externally) and does not guarantee exhaustiveness by itself — the `A | B | C` union alias is still the exhaustiveness surface. Runtime behavior: purely a marker, no `__init__` or `__init_subclass__` override. Source: `python3.15 -c "import typing; help(typing.disjoint_base)"`, 2026-06-09.

[PARAMETRIC_CMD_RETURN_CORRELATION]:
- `@tagged_union(frozen=True) class Cmd(Generic[T])` with `write: tuple[str, T] = case()` flows `T` into the case field at definition time; `handle(cmd: Cmd[bytes]) -> Result[bytes | None, str]` is inferred correctly by `ty` 0.0.44 — `reveal_type(handle(Cmd(get='k')))` resolves to `Result[bytes | None, str]`. The return-type correlation across variants is NOT encoded in the type — all arms return the same `Result[bytes | None, str]` regardless of which tag is active. A variant-specific return type (e.g., `read` → `Result[bytes, str]`, `delete` → `Result[None, str]`) requires a separate typed overload surface or a per-variant handler function. Source: verified with `ty check`, 2026-06-09.
- The `@effect.result()` do-notation builder (`expression.effect.ResultBuilder`) composes `Cmd` dispatch in-line via `yield from handle(cmd)`: the generator yields through `Result.__iter__` which raises `EffectError` on `Error`, short-circuiting the builder's `while not done` loop and returning the `Error` directly. `@effect.result()` is a class instance (not a class), obtained by calling `effect.result()`. Chaining `yield from handle(Cmd(read=Scope(one='k1')))` followed by `yield from handle(Cmd(write=...))` composes two discriminated dispatch calls into one Kleisli chain without an intermediate `bind`. Source: `expression/core/builder.py`, `expression/effect/result.py`, 2026-06-09.
- `result.pipeline(*fns)` from `expression.extra.result` is a Kleisli `>=>` combinator reducing `fns` via `lambda acc, fn: lambda x: acc(x).bind(fn)` with `Ok` as the identity; it accepts 1-6 typed overloads with a variadic fallback. This is the correct surface for composing 3+ discrete dispatch-returning functions when the input flows left-to-right and each step produces `Result[T, E]`; `@effect.result()` is preferred when intermediate bindings are named or when the `yield from` notation is more readable than a pipeline of lambdas. Source: `expression/extra/result/pipeline.py`, 2026-06-09.

[EXHAUSTIVENESS_PROOF_MECHANICS]:
- The `@tagged_union` class is a single runtime class with a `tag: str` attribute; match patterns of the form `case Shape(tag='circle', circle=r)` match the class by identity and then check `tag == 'circle'` and bind `r = shape.circle`. The checker cannot prove that after all tag-valued arms the remaining object is `Never`, because `Shape` is one class and the residual is still `Shape`. This is a fundamental checker limitation on value-narrowing over a single-class tag field as of `ty` 0.0.44 and `mypy` — not a bug to work around but a known structural limit. The correct exhaustiveness tool for `@tagged_union` is `assert_never` as a RUNTIME guard only; the static proof is advisory (`@Todo`). Source: `ty check` outputs 2026-06-09.
- Static exhaustiveness proof requires a union type `A | B | C` where each member is a distinct Python type. After covering all N members in match arms, the checker narrows the residual to `Never` and `assert_never` passes with no diagnostics. `ty` 0.0.44 and `mypy` both prove this correctly for `BaseModel` unions and dataclass unions with `Literal` discriminant fields. Choosing `@tagged_union` over `A | B | C` trades static exhaustiveness for single-class density, O(1) construction, and `order=True` heapability. Source: verified 2026-06-09.
- The `assert_never(never)` call in a `@tagged_union` match arm serves as a runtime invariant sentinel: if a new case field is added to the union without a corresponding match arm, the `case _` arm fires and `assert_never` raises `AssertionError` with the unmatched instance. This is a post-deployment guard, not a pre-commit proof. Annotation: when using `@tagged_union`, every fold MUST include a `case _ as never: assert_never(never)` arm as the runtime equivalent of static exhaustiveness. Source: `typing.assert_never` Python 3.15.0b1 docs + live test 2026-06-09.
- `expression.extra.result.catch(exception=ExcType)` wraps a raising function at the boundary and returns `Result[T, ExcType]`; it is also a two-argument form `catch(fn, exception=ExcType)`. This is the correct boundary converter for wrapping third-party callables that raise. Source: `expression/extra/result/catch.py`, 2026-06-09.

[DENSE_5_TO_8_MODALITY_COLLAPSE]:
- An 8-case `@tagged_union(frozen=True, order=True)` (e.g. `scan`, `audit`, `sync`, `notify`, `write`, `delete`, `lock`, `unlock`) compresses 8 sibling functions — each with a separate docstring, type signature, and call site — into one class body (8 field declarations), one handler function (8 match arms), and one `heapq`-orderable surface via `_index` (1-based declaration order). Declaration order is semantic: earlier cases have lower `_index` and are "smaller" in `__lt__`. An 8-function pre-collapse typically spans ~80-120 LOC (signatures + docstrings + bodies); the collapsed form spans ~25-35 LOC. Source: verified construction + `heapq.heapify` test, 2026-06-09.
- Adding a 9th modality to the collapsed surface costs exactly: 1 case field declaration, 1 match arm in the handler, and zero new functions or call-site changes. The call site constructs `Task(new_case=payload)` directly with no factory or adapter. In a 8-function pre-collapse form, the same addition costs: 1 new function, 1 new docstring, 1 new type signature, possibly a new dispatch key. Source: derived from `tagged_union.py` anatomy, 2026-06-09.
- `frozen=True` adds `__hash__` as `hash((classname, tag, value))`; `order=True` adds `__lt__` comparing `(self._index, getattr(self, self.tag))` against another instance of the same class. Both are conditioned: neither is generated when the flag is `False`. The hash and ordering are consistent: two instances with the same tag and value are equal and have the same hash. This makes frozen-ordered `@tagged_union` suitable as a `heapq` element, a `frozenset` member, and a `dict` key simultaneously. Source: `expression/core/tagged_union.py` lines 83-119, 2026-06-09.
- `__init__` enforces exactly-one-active-case via `len(kwargs) != 1` after popping `tag`; the `tag` kwarg is optional and when provided must match the name of the single active case kwarg (enforced by `match tag or name, name: case str(tag), name if tag == name`). Providing a mismatched `tag` raises `TypeError: Tag X does not match case name Y`. The init then stores only the active case field plus `tag` in `__dataclass_fields__` so `dataclasses.asdict()` emits only the active case. Source: `expression/core/tagged_union.py` lines 57-78, 2026-06-09.
- The densest legal collapse pattern for 5-8 modalities sharing a request-response shape: one `@tagged_union` request ADT (`Cmd`), one polymorphic handler `def handle(cmd: Cmd[T]) -> Result[R, E]`, one `@effect.result()` do-notation consumer for sequencing, and `result.pipeline` for independent step chains. The four surfaces together own construction, discrimination, execution, and composition without any helper, factory, or wrapper function. Source: validated end-to-end runtime test, 2026-06-09.
