# Spectral-Eigen — Distillate

[ROUTE_AND_PROVIDER_BINDING]:
- `Matrix<T>.Evd(Symmetricity)` is the single generic eigensolve entry; `Symmetricity` is a four-case enum and passing the known case skips the O(n²) `IsSymmetric()` scan that `Unknown` pays on every call.
- Only `DenseMatrix` reaches `LinearAlgebraControl.Provider.EigenDecomp`; `SparseMatrix` and `DiagonalMatrix` route to the managed user-storage pipeline regardless of the active provider, so reaching a native driver requires materializing to `DenseMatrix` first — factorization products such as `QR.Q` are already dense clones and dispatch provider-backed without materialization.
- The provider slot is process-static with no per-call override, instance scoping, or ambient context: the solve's numerical regime is decided by state no argument carries. The user-storage route is the provider-immune escape hatch that pins the managed regime irrespective of installed native binaries; conversely, pinning the managed provider before first provider touch is the definition-time reproducibility policy — a one-time process-start assertion matching the slot's process-static shape, never a per-solve parameter.
- The managed eigensolve kernels are serial regardless of the configured parallel degree — the degree knob gates other provider operations only and re-verifies backends on mutation; raising it buys an eigen-heavy pipeline nothing on the managed path, and only a native driver parallelizes the decomposition itself.
- No sparse iterative eigensolver exists — `BiCgStab`, `GpBiCg`, `MlkBiCgStab`, `TFQMR` are linear-system solvers only; block-preconditioned eigeniteration is hand-assembled from dense `Evd` Rayleigh-Ritz subproblems plus sparse operator application.
- `evd.Solve` exists only for the symmetric spectral-decomposition path `V·Λ⁻¹·Vᵀ·b`; nonsymmetric `Solve` is a hard `ArgumentException`, not a degraded result — gate `Solve` exclusively to symmetric and Hermitian evidence.

[SYMMETRY_ROUTE_AS_FIVE_AXIS_CONTRACT]:
- The symmetry flag selects the entire output contract, not a performance path: orthonormal versus arbitrary-norm eigenvectors, real-diagonal versus block-diagonal `D`, single-column versus column-pair eigenvector encoding, ascending-sorted versus Schur-convergence-ordered eigenvalues, and a working eigenvector solve versus a norm-gated skip. A result union with distinct dense-symmetric and dense-general cases is the minimal honest shape; one collapsed "eigendecomposition result" forces every consumer to re-derive the five axes from the flag at each use site.
- The route is chosen by exact bit equality `_values[j*n+i] != _values[i*n+j]`: a one-ULP mirror difference silently acquires the full nonsymmetric contract — block-diagonal `D`, non-orthonormal columns, possible spurious complex pairs from rounding. The only route control is controlling the bits: `(s + s.Transpose()) * 0.5` before the call (three allocations, but both mirror entries land on the same IEEE 754 bit pattern so the exact-equality check passes unconditionally); the Hermitian form is `(s + s.ConjugateTranspose()) * 0.5`. `MapIndexedInplace` self-averaging is broken for this — it mutates the same backing array sequentially, so the mirrored entry has already been modified when read.
- The receipt's `IsSymmetric` is a `bool` that merges `Symmetric` and `Hermitian` (both map to `isSymmetric = true`); the original `Symmetricity` evidence is lost and must be carried separately when the distinction matters.
- Symmetric and Hermitian eigenvalues arrive ascending — a managed selection-sort tail (O(n²) comparisons, at most n−1 lockstep column swaps) that the LAPACK path replaces with the `dsyev`/`zheev` native ascending contract; nonsymmetric eigenvalues arrive in Schur-deflation order with no sort guarantee, and that order carries no spectral meaning — it is the bulge-chase deflation order, dependent on the Hessenberg pivot path, not on magnitude or real part.
- Re-sorting nonsymmetric eigenvalues requires carrying the permutation and reindexing the `EigenVectors` column-pair groups and the `D` 2×2 blocks in lockstep; reordering one carrier alone leaves each individually well-formed, so the desync is silent.

[REAL_SCHUR_COLUMN_PAIR_DECODE]:
- The real nonsymmetric route never produces a `Matrix<Complex>` carrier: a conjugate pair λ = a ± bi occupies two adjacent real columns — column j holds the shared real part u, column j+1 the imaginary part w, the true eigenvectors are u ± iw. The back-substitution emits `e[j] = +b` then `e[j+1] = −b`, so the positive-imaginary eigenvalue always indexes the +w column; reading `Column(j)` as a whole eigenvector silently discards the rotation half.
- The pair discriminant is fully derived from `EigenValues[j].Imaginary` — `> 0` opens a pair, `< 0` closes it, `== 0` is a single real column; no parallel pairing array exists, which is the load-bearing reason a result must carry `EigenValues` to interpret `EigenVectors`: the eigenvector matrix is not self-describing.
- `D` is the real-Schur block-diagonal, not a diagonal: a pair writes `[[a, b], [−b, a]]`, so `D` alone distinguishes a real eigenvalue (zero off-diagonals around its index) from a complex pair (anti-symmetric 2×2 block), and counting nonzero super-diagonal entries is the conjugate-pair census — `n − 2·pairs` real eigenvalues — a structural invariant and the cheapest spectral-type probe available before any eigenvector work.
- Lifting the packed form to the complex modal matrix is an index-first strided gather dispatched on the sign of the imaginary part, never a length check or try-cast:

```csharp
static Matrix<Complex> LiftSchurVectors(Matrix<double> packed, Vector<Complex> values) =>
    Matrix<Complex>.Build.DenseOfColumns(
        Enumerable.Range(0, values.Count).Select(j =>
            Math.Sign(values[j].Imaginary) switch
            {
                > 0 => packed.Column(j).Zip(packed.Column(j + 1),
                           (re, im) => new Complex(re, im)),
                < 0 => packed.Column(j - 1).Zip(packed.Column(j),
                           (re, im) => new Complex(re, -im)),
                _   => packed.Column(j).Select(re => new Complex(re, 0.0)),
            }));
```

[NONSYMMETRIC_EIGENVECTOR_QUALITY]:
- Eigenvector recovery is gated on one accumulated absolute-value norm over the upper Hessenberg band computed before the eigenvalue loop: when it is exactly zero the routine returns immediately, leaving `EigenVectors` as the accumulated similarity transform with no per-column solve — eigenvalues valid, eigenvector block structurally meaningless — and the boundary detects this only from the operator norm itself, never from the returned vectors.
- The quasi-triangular back-substitution guards complex division: when both denominators underflow it substitutes `ε_mach · ‖band‖`, injecting an O(ε_mach·‖A‖)-scaled spurious component into that column — the eigenvector near a defective eigenvalue carries a known-magnitude perturbation, and the residual `‖A·v − λ·v‖` is the only post-hoc detector.
- Recovered columns are raw triangular solutions back-transformed in place through the packed Householder accumulation: norms are arbitrary and differ per column, so any projection or modal weight assuming unit columns must `Column(j).Normalize(2)` first. The same `EigenVectors` property carries two normalization contracts — orthonormal on the symmetric route, non-orthonormal non-unit here — dispatched purely on the symmetry flag.
- Defectiveness never throws: the routine always emits n columns, and a Jordan block manifests as near-parallel columns — a rank-deficient `EigenVectors` — so detection is a rank or condition probe on the eigenvector matrix; eigenvalue multiplicity alone cannot distinguish a defective operator from a repeated-but-diagonalizable one.
- Per-eigenvalue conditioning is `1/|yᵢᴴ·xᵢ|` over left and right eigenvectors; the routine returns right eigenvectors only, so conditioning requires a second decomposition of the transpose — absent it, conditioning can be bounded but never reported.
- The exceptional-shift escape fires at fixed internal step counts and substitutes the literal `0.964` into the shift state on a stalled bulge-chase: the last bits of such eigenvalues are governed by the constant, not the matrix, so operators differing below the convergence band can return bit-identical eigenvalues — a determinism feature for regression baselines, a precision trap for any assertion tighter than the convergence band.

[CONVERGENCE_CEILINGS_AND_RESIDUAL_EVIDENCE]:
- The three convergence regimes are distinct: symmetric/Hermitian QL throws non-convergence at 1000 steps per eigenvalue; the real nonsymmetric Schur path throws at `iter >= 30 * order` where `iter` resets to zero on every 1×1 and 2×2 deflation — the ceiling bounds consecutive non-deflating steps since the last deflation, not total work; the complex nonsymmetric path has no outer bound at all (1×1 deflation only, exceptional shifts at internal counters 10 and 20 impose no ceiling) and can silently fail to terminate, making the block residual the only detection and unconditional for complex nonsymmetric inputs.
- Symmetric exhaustion uses the Wilkinson relative criterion (`|e[k]| ≤ precision × running spectral-radius estimate`; near-coincident eigenvalues are the pathological trigger) and leaves the eigenvector array partially updated: indices below the failing one are complete and orthonormal, the rest are meaningless intermediate Givens products, and the failing index is not exposed. The spectral-radius estimate `max(|d[k]| + |e[k]|)` must be captured before the call — it is unavailable post-throw — as the only near-degeneracy diagnostic the exhaustion evidence can carry.
- Exhaustion and argument violation (non-square input, non-SPD B) both surface as construction throws; the boundary must tag them into distinct typed failure cases before propagation so downstream discrimination never inspects exception text.
- No built-in eigen residual exists: per-pair `(A.Multiply(v) - v.Multiply(λ)).L2Norm()`, block `(A.Multiply(V) - V.Multiply(D)).FrobeniusNorm()` with `D = Matrix<T>.Build.DenseDiagonal(n, n, i => λ[i])`. One generic residual core parameterized on `Func<TM, Matrix<double>, Matrix<double>> applyA` replaces dense and sparse twins — the adapter's entire content is the `Multiply` dispatch for the representation, no domain logic.
- `Rank`, `IsFullRank`, and `Determinant` are unusable in domain logic: the zero test reduces to absolute `|λ| < ~3.33×10⁻⁸`, and `Determinant` short-circuits to exactly `0.0` the moment any eigenvalue crosses it — for ‖A‖₂ ~ 10⁶, numerically significant eigenvalues are misclassified. Recompute rank as `n − count(|λᵢ| < ε_rank × max|λⱼ|)` with a caller-supplied spectral-radius-relative `ε_rank`.
- Correctness evidence on the result: per-pair residual norms, the route variant (`dense-symmetric` | `dense-hermitian` | `dense-general` | `congruence-reduced`), and a sort-guarantee flag (ascending versus Schur-order, no guarantee).

[PROVIDER_CONTRACT_DIVERGENCE]:
- The binding contract any substitute satisfies is one method — `void EigenDecomp(bool isSymmetric, int order, T[] matrix, T[] matrixEv, Complex[] vectorEv, T[] matrixD)` — with parameter order load-bearing, the three square carriers length-`order²` column-major, and `vectorEv` always double-precision `Complex[]` regardless of `T`: a single-precision native provider up-converts its eigenvalues at this boundary, a precision asymmetry baked into the contract, not the algorithm.
- The managed path (Householder tridiagonalization + QL; orthogonal Hessenberg + real-Schur QR) has no workspace-query protocol and no info-return convention; a native driver behind the same interface diverges on five axes no carrier records: ordering (a native nonsymmetric driver's deflation order is its own — index correspondence between two solves of nearby operators is provider-stable only), sign/phase convention (managed Householder accumulation versus largest-component-positive; both bases valid, columns differ by sign or unimodular phase, so eigenvector-equality baselines break on provider swap), convergence ceiling, error-rail shape, and last-bit determinism (the managed fixed-literal exceptional shift is exactly reproducible; a native heuristic's last bits differ — equality assertions tighter than the convergence band are provider-coupled even when both providers are individually deterministic).
- The error rail changes shape across the seam: the managed path throws on non-convergence, a native driver returns a nonzero info code in-band — a catch-shaped boundary sees nothing and mislabels native non-convergence as success with meaningless vectors. The eigen-defect norm `‖A·V − V·D‖_F` is the one signal both rails surface identically, which is precisely why it is mandatory.
- The block-diagonal packing is itself a cross-provider invariant: real part on the diagonal at both pair indices, positive imaginary on the super-diagonal, negative on the sub-diagonal; a provider packing a different sign or transpose desynchronizes every consumer reading rotation evidence from `D` rather than the eigenvalue vector.
- Provider identity is first-class evidence beside the route variant: the same logical decomposition has different sign, ordering, and last-bit content under different providers, so two results compare only under the same provider — provider is to the result what symmetry class is to the route, a discriminant that changes the meaning of every other field.

[SOFT_LOCK_CARRIER_AND_TERMINATION]:
- The block iteration is one state-threaded fold over a single immutable carrier — full k-column `X` (converged and active alike), previous-direction block `P`, per-column residual norms, the active mask, the smallest aggregate residual ever seen, and the iteration index — each step a pure `Carrier → Carrier` transition driven by a bounded recursion schedule with early exit at zero active count.
- The mask is the discriminant, not a flag tail: `mask = mask.Zip(residualNorms, (m, r) => m && r > tol)` — AND-folded so a converged column never re-activates; the active count is the only quantity driving block-width contraction, and no parallel locked flag exists because the discriminant is recoverable from the residual vector itself.
- `X` stays full-width every iteration; only the working triple `[Xₐ | Rₐ | Pₐ]` contracts through the mask. Active-column extraction is `Matrix<T>.Build.DenseOfColumnVectors(block.EnumerateColumns().Where((_, j) => mask[j]))` — never `RemoveColumn` in a loop, which reallocates the backing array per removal at O(n·k²) pure copying. Full `X` rides the Rayleigh-Ritz substitution so locked columns keep refining; only their residual and direction work is skipped.
- The block residual is two matrix multiplies — `R = A.Multiply(X) − X.Multiply(Λ_diag)` (generalized: `− B·X·Λ_diag`) — then per-column `L2Norm` projects the convergence vector.
- The default tolerance is absolute in the column 2-norm, `sqrt(ε_mach)·n` — dimension-scaled, not magnitude- or operator-norm-scaled; the relative criterion `‖rⱼ‖ / (|λⱼ|·‖xⱼ‖ + ‖A·xⱼ‖)` is the correct override for spectra spanning orders of magnitude, supplied as a policy value on the request. Termination compares the worst active column (`residualNorms.AbsoluteMaximum()`) against the bar, never the mean; the mean aggregate is restart telemetry only.
- Termination has three sealed outcomes, cases not exceptions: `Converged` (active count zero), `Exhausted` (ceiling reached with active columns — the survivors are valid Ritz pairs at their last residual, recorded with that residual as evidence, not discarded), `Stalled` (orthonormalization factorization fails pre-convergence).
- Hard locking (freezing converged columns as constraints) caps each locked column's error floor at `tol` permanently; soft locking keeps converged columns in the substitution so they ride later refinement free. The result records which discipline produced each pair because the per-column error floor differs by construction.

[RAYLEIGH_RITZ_PENCIL_FOLD_AND_RESTART]:
- Near convergence, successive iterates approach linear dependence and `P` (which spans the same subspace as `[Xⁱ | Xⁱ⁻¹]` without holding `Xⁱ⁻¹`) goes ill-conditioned, so the `3k`-wide `[Xₐ | Rₐ | Pₐ]` Gram pencil loses rank: the transition catches the dense generalized-solve failure and folds to the leading `2k` principal block of the same Gram pair — one Rayleigh-Ritz body, two arities, dispatched on the prior solve's outcome:

```csharp
// One Rayleigh-Ritz body, two arities: the 3k pencil, folding to the leading 2k principal
// block of the same Gram pair when the wide pencil loses rank.
static (Vector<double> Values, Matrix<double> Vectors) RayleighRitz(
    Matrix<double> gramA, Matrix<double> gramB, int narrow) =>
    new Try<(Vector<double>, Matrix<double>)>(() =>
    {
        var e = Reduce(gramA, gramB).Evd(Symmetricity.Symmetric);
        return Fin.Succ((Vector<double>.Build.DenseOfEnumerable(
            e.EigenValues.Select(c => c.Real)), e.EigenVectors));
    }).Run()
      .IfFail(_ => RayleighRitz(gramA.SubMatrix(0, narrow, 0, narrow),
                                gramB.SubMatrix(0, narrow, 0, narrow), narrow));
```

- A forced restart fires independently of factorization failure: an aggregate-residual rebound past `2^restartControl × smallestResidualNorm` (default 20, so a ~10⁶ rebound) marks stale `P` poisoning the search direction; `smallestResidualNorm` updates as `Math.Min(prior, current)` each step — a monotone carrier field whose only purpose is this predicate. Both restart paths (Gram-rank failure, residual rebound) collapse to one `dropP` boolean consumed by the next subspace assembly.
- Iteration zero has empty `P` by construction, so `[Xₐ | Rₐ]` is the unconditional base — the narrow form is the base of the recurrence, not a special case; the `dropP`-true and iteration-zero branches share one assembler differing only in whether `Pₐ` appends.
- Reconstruction partitions the stacked Ritz eigenvector matrix row-wise by block width — `C_X = C[0:k]`, `C_R = C[k:k+p]`, `C_P = C[k+p:]` with `p` the surviving active count (a column dropping below the orthonormalization floor contracts the pencil from `3k` to `2k + p`): the new direction is `P⁺ = Rₐ·C_R + Pₐ·C_P` — the search-space component orthogonal to the current `X` span — and `X⁺ = X·C_X + P⁺`; `P⁺` is reused directly as next-iteration `Pₐ` after its own B-orthonormalization, never recomputed from `Xⁱ⁻¹`, the precision-preserving substitution that defines the method.
- Assembly precision: `TransposeThisAndMultiply` is Aᵀ·B, `TransposeAndMultiply` is A·Bᵀ, with conjugate forms for Aᴴ·B and A·Bᴴ — the Hermitian variant assembles `Wᴴ·A·W` via `ConjugateTransposeThisAndMultiply` and re-hermitizes the Gram before the Hermitian-flagged solve. Neither two-call assembly form exploits the Gram's symmetry, and the operator application allocates one n×3k intermediate. Sparse-times-dense multiply reads sparse row pointers directly against the dense backing at O(nnz·k) without densifying, so operator application stays sparse-priced. Pre-allocate one n×3k workspace and use the in-place `Append(right, result)` overload (sub-matrix copies into pre-allocated storage); chained `Append` allocates n×2k then n×3k intermediates.
- When the stacked basis W is orthonormal the subspace pencil reduces exactly to a standard symmetric solve on S = Wᵀ·A·W; under floating-point drift G = Wᵀ·W ≠ I the correct route is congruence through G's Cholesky factor — re-orthonormalize-then-standard is equivalent only when the orthonormality residual `‖Wᵀ·W − I‖_F < ε` is verified per iteration, and skipping that check accumulates silent Rayleigh-quotient error.

[B_ORTHONORMALIZATION_AND_CONGRUENCE_HALF_FACTOR]:
- B-orthonormalization `V ← V·(VᴴBV)^{−1/2}` runs the Cholesky route: Gram via `V.TransposeThisAndMultiply(B.Multiply(V))`, factor, then — because `V·R⁻¹ = (L⁻¹·Vᵀ)ᵀ` — solve through the LOWER factor: `chol.Factor.LU().Solve(V.Transpose()).Transpose()`. Routing through `chol.Factor.Transpose()` instead yields `V·L⁻¹` — the wrong half-factor, whose result is not B-orthonormal (`L⁻ᵀ·G·L⁻¹ ≠ I`) and whose defect is silent because the shapes agree.
- `Cholesky.Solve` never yields a half-factor — it runs forward and backward substitution in one pass, returning `(L·Lᵀ)⁻¹·B`; any `L⁻¹·A` step must go through `chol.Factor.LU()` directly, and `chol.Factor.Transpose().Solve(A)` gives `L⁻ᵀ·A`, the other wrong half. `Cholesky<T>.Factor` is the lower triangle.
- A Cholesky throw during the iteration discriminates by which block: on `Rₐ` every active residual collapsed — true convergence reached mid-step; on `Pₐ` it is the degeneracy that triggers the P-drop restart — identical exception type, different outcomes, tagged at the catch site before propagation.
- The generalized problem A·x = λ·B·x has no native route; with B = L·Lᵀ: (1) `Y = chol.Factor.LU().Solve(A)` gives L⁻¹·A; (2) `Z = chol.Factor.LU().Solve(Y.Transpose()).Transpose()` gives C = L⁻¹·A·L⁻ᵀ; solve C·y = λ·y; back-transform `x = L⁻ᵀ·y` via `chol.Factor.Transpose().LU().Solve(evd.EigenVectors)`. `Matrix<T>.Solve` has no triangular-aware route — always dense LU at O(n³) with no zero-skipping — so the congruence costs two O(n³) factorizations plus O(n²·k) back-substitution, with each `.LU()` and `.Transpose()` materializing n×n.
- Congruence asymmetry grows as O(n² × ε_mach × κ(L)): at κ(B) ~ 10¹² it reaches O(10⁻⁴), and un-re-symmetrized C routes through the nonsymmetric Schur path, returning complex eigenvalues for a real symmetric-definite problem — mirror-exact re-symmetrize, then pass the symmetric flag.
- Positive-definiteness detection is strictly `> 0.0`: exactly-singular B throws, near-singular B passes and produces an ill-conditioned factor — the guard is an ε·I shift before factorization; catching the throw addresses only the indefinite case. Non-dense input or result throws `NotSupportedException` at runtime — the abstract `Matrix<double>` signature hides it from the compiler.
- Three result states: indefinite-B (Cholesky throws — no eigenpairs exist), exhaustion (Cholesky succeeded, the solve on C exhausted — L is valid, and its condition evidence travels with the failure because it diagnoses ill-conditioned B versus a genuinely near-degenerate spectrum), success.

[DEFLATION_SEEDING_AND_ORTHONORMALIZATION_FLOORS]:
- An externally supplied invariant subspace `Y` deflates by B-orthogonal projection applied to every working block before each Rayleigh-Ritz: `V ← V − Y·(Yᴴ·B·V)`, collapsing to `V − Y·(Yᵀ·V)` at B = I; the projection is idempotent only when `Y` is itself B-orthonormal, so `Y` is B-orthonormalized once at setup and the per-step projection assumes that invariant.
- Constraints and soft locks are distinct axes that compose: constraints are external known directions projected out of `R` and `P` (search outside their span); soft locks are internal discovered convergence retained in the substitution. Conflating them — re-projecting converged internal columns as constraints — re-introduces the hard-locking accuracy ceiling by forbidding their refinement; the carrier holds `Y` separately from the mask.
- For a clustered near-zero spectrum, the known nullspace (the constant `1/√n` vector for a graph Laplacian) seeded as a frozen constraint removes the trivial λ₀ = 0 mode from the search: the active block converges on the spectral gap above it, and the iteration count drops by exactly the frozen-column count since each removes one direction of pencil rank.
- Start-block construction: seeded random n×k then thin-QR `Q` gives reproducible initialization for convergence regression; seeding the first column as `1/√n`, drawing the remainder randomly, and re-orthonormalizing via thin QR places the block in the relevant eigenspace rather than a random orientation, cutting iterations for clustered near-zero spectra.
- Orthonormalization floors: the Gram-Schmidt factorization throws only at an exactly-zero column norm — near-zero norms propagate silently, so column norms are checked against a domain floor before the call. Thin Householder QR is backward-stable where single-pass MGS loses orthogonality in near-dependent columns, at identical allocation (both clone the n×k input; neither writes back) and flop parity ≈ O(2nk²); conditional double-pass MGS pays its second O(2nk²) only when the R diagonal dips below floor. The near-dependence probe `qr.R.Diagonal().Any(v => Math.Abs(v) < floor)` lazily materializes R (a k×k copy) plus a length-k vector — not allocation-free.
- The diagonal-preconditioner correction `(λ − a_ii)⁻¹·rᵢ` falls back to identity when `|λ − a_ii| < ε`; the epsilon is algorithm policy supplied as a value, never machine precision.

[SCALAR_CARRIER_AND_COMPLEX_GENERIC_CEILING]:
- `EigenValues` is `Vector<Complex>` uniformly across all four scalar types — the carrier is not polymorphic by symmetry class; symmetric and Hermitian imaginary parts are exactly zero after the QL sweep zeroes the off-diagonal array, so result eigenvalue scalars are `double` for symmetric and Hermitian inputs even when the eigenvector matrix is `Matrix<Complex>`.
- Hermitian eigenvectors are genuinely complex: the conjugate untridiagonalization multiplies by conjugated phase factors to restore complex structure from the real rotations, so projecting eigenvectors to real parts is incorrect even though every eigenvalue is real; complex nonsymmetric eigenvalues carry nonzero imaginary parts and admit no real projection at all.
- Real-weight projection: `EigenValues.AsArray()` exposes the raw backing `Complex[]` without allocation, and the scalar `c => c.Real` loop is canonical — `MemoryMarshal.Cast<Complex, double>` yields a stride-2 span (even indices real), but no strided vectorized overload exists and the JIT does not auto-vectorize the stride-2 read, so the scalar loop is the floor unless spans are split manually.
- The generic ceiling: `Complex` implements only homogeneous `IMultiplyOperators<Complex, Complex, Complex>` — no `(Complex, double, Complex)` mixed form exists anywhere in the BCL — and lacks `INumber<Complex>` (no `IComparable`), so `INumberBase<T>` admits both `double` and `Complex` while `INumber<T>`/`IFloatingPoint<T>` exclude `Complex`. Generic real-weight scaling of complex columns cannot be expressed as `where T : IMultiplyOperators<T, TWt, T>`; the wide parameterization is an explicit delegate `Func<TM, double, TM> scale` supplied at construction (`(c, w) => new Complex(c.Real * w, c.Imaginary * w)`), since `Vector<T>.Multiply` requires a scalar of type `T`.
- Vectorized-primitive compile matrix for `Complex`: `Dot` and `MultiplyAdd` compile; `Norm` (no `IRootFunctions`) and `FusedMultiplyAdd` (no `IFloatingPointIeee754`) do not — real-domain hot loops vectorize directly, complex-domain equivalents are manual loops.
- The Rayleigh quotient maps `Complex → double` via `.Real`, breaking the generic chain regardless of constraint choice — the real scalar result is structural, not a convenience cast.

[FILTER_WEIGHTS_TRUNCATION_AND_DEGENERATE_SPREAD]:
- The eigenbasis filter `∑ᵢ f(λᵢ)·(vᵢᵀ·x)·vᵢ` owns its vocabulary as a closed union — heat `exp(−tλ)`, wave `cos(t√λ)`, biharmonic `λ²`, diffusion `1/(1+tλ)`, commute-time `1/√λ`, identity — with per-case `Invoke(double)`; a raw `Func<double, double>` bundle is open, breaks the closed-family invariant, and belongs only at the boundary where the weight vector materializes. Composition is a partial monoid: `Product(f₁, f₂)` invokes both (two virtual dispatches per eigenvalue, negligible at n ~ 500; inlining via pattern match would force the dispatch site to enumerate all cases), `Identity` is the unit and `Product(f, Identity)` folds to `f`; materialized weights compose via `PointwiseMultiply` at O(n), and the identity check `ForAll(w => w == 1.0)` short-circuits on first mismatch.
- Zero-mode law: Laplacian-derived λ₀ ≈ 0 diverges the `1/λ` and `1/√λ` filters; the weight is `λ < ε_zero ? 0.0 : f(λ)` — excluded, never clamped — with `ε_zero` a policy value. The spread floor is the minimum over eigenvalues above `ε_zero`, never the second eigenvalue index blindly, because that index may itself be sub-threshold under multiple near-zero modes; when the zeroth mode is structurally the zero mode, index-first exclusion via a `MapIndexed` guard applies, and `Zeros.Include` is required for zero-valued entries to reach the map function.
- The vectorized weight pipeline has exactly one non-vectorizable step: scalar `Complex → double` projection, then in-place vectorized scale by `−t` and in-place vectorized `Exp` (admitted by `double : IExponentialFunctions<double>`); zero-mode exclusion combines as one trailing scalar masked pass `MapIndexedInplace((i, w) => λᵢ < ε_zero ? 0.0 : w, Zeros.Include)` — the vector overload takes index and value, the matrix overload takes three.
- A weight sum below a bandwidth floor is a distinct exhaustion marker: zero weight sum — all modes excluded or underflowed — must not read as a valid zero-signal result, so the weight sum travels as evidence and the call site gates on it.
- Degenerate spread (`λ_max == λ_min` under spread normalization) is a sealed result case distinct from unusable pairs (finite diagnostics flagged structurally unreliable) and from success: all frequencies collapse into one bin and the output carries zero information while remaining indistinguishable from a well-conditioned result if clamped — fail, never clamp — and the condition is knowable only after the eigensolution is in hand, never from input properties.
- Truncation and cache law: the dense solve always computes all n eigenpairs; `SubMatrix(0, n, 0, k)` always allocates a fresh n×k backing array — column-major dense storage takes ownership, no zero-copy views. The symmetric working set is ≈ 2n² + O(n) doubles (~16 MB at n = 1000), nonsymmetric ≈ 3n² (one extra Hessenberg array). Retain the full eigenvector matrix for cached basis reuse: each filter call then pays one SubMatrix allocation, never the O(n³) solve. Cache evidence carries computed-versus-requested counts, truncated-from-cache versus fresh, and resident versus reconstructed (resident memory traded against recomputation); a hit requires matching n AND `Symmetricity` — a symmetry mismatch recomputes.
