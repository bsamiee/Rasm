# [CLOSER_DISPATCH]

Closer prompt for routed-row closure after all fix lanes land; closers run concurrently over disjoint territories. Sizing law: opus closes small single-file mechanical rows — consumer re-spells, one-case additions; ONE fable with fuller arming closes family, design, or capability work — multi-file moves, owner mints, miner-roster judgment.

- [CORPUS]: `<working-root>` is the territory and `<work-product>` the deliverable; never any git command in any form.
- [ARMING]: sized to weight — a small closer reads the 2-3 `docs/stacks/<language>/` pages owning the row's concern and each target page in full before its edit; a fable closer reads the full stack root set and the ARCHITECTURE strata map when rows cross folders.
- [DEMAND]: each routed row's COMPLETE text is the contract — inlined, or a pointer to the `<round-dir>/lane-*-report.json` `.routing[]` arrays with the instruction to grep them per target file before editing; a paraphrase drops constraints.
- [STANCE]: per row — rail-verify every named member first through `<truth-rail-command> --symbol <FQN>`; the rail is fuzzy-scored, `results[0].text` must equal the requested FQN exactly, and a miss means the member never lands. Judge the demand against the target page's own charter, then land or refuse — refusal takes exactly three shapes, each carrying its citation: charter-refused (the target's charter forbids the demand — cite the charter line), already-canonical (the demand duplicates or weakens the standing canonical owner — cite that owner), already-realized (current disk carries the capability under any spelling — cite the carrying lines); a refused row is never forced value. Fix at the root and exceed where the owner admits; extensions land on existing owners as rows, cases, or modalities, never parallel surfaces; zero functionality loss.
- [FRESHNESS]: re-read every target from current disk before its edit — concurrent agents land in parallel; a row disk already carries is the already-realized shape, reported with its evidence, never re-done.
- [VERIFICATION]: format gate once, batched over touched pages (`<format-gate-command>`); no builds or tests — proof follows the work product the corpus block declares.
- [RETURN]: JSON only — `{"rows": [{"id": "<id-or-file>", "verdict": "landed"|"refuted"|"already-landed"|"blocked", "note": ""}], "files": ["<path>"]}` — one row per routed demand; charter-refused and already-canonical return `refuted`, already-realized returns `already-landed`, each with its citation in the note.
