# Wave 1 Agent C - Core Layer + Analyzer Audit

## (i) Shape of the Core Layer

The Core layer occupies 912 LOC across six files and is structurally sound: it owns `GeometryContext` (sealed partial value-object record), `AnalysisRuntime` (runtime-record for `Eff<RT, T>`), the `OperationKey`/`OperationFault`/`SemanticFault`/`GeometryResult` quartet, the `Resilience.StandardPolicy` LanguageExt 5 `Schedule`, and two runtime modules - `RhinoMatch` (21-arm switch expression) and `FoldExtensions`. Boundary contract is well-formed: `[ValueObject<double>]` wrappers gate scalars at constructor, `Fin<T>` smart constructors enforce range invariants, `Validation<Error, T>` accumulates parallel failures at `GeometryContext.Create`, and the analyzer's scope classifier tags `Core.Domain`/`Core.Runtime` as Domain. Three odors: parallel-name proliferation (Validate/ValidatePair/ValidateFirst, OperationFault/SemanticFault, One/Many/Solved), bitmask abstraction overuse with zero callsite leverage, and one imperative regression in `FoldExtensions.StatsOf:39` that bypasses the discipline the same file exposes via `FoldFin`/`TraverseFin`.

## (ii) GeometryRequirement -> GeometryCheck DU Collapse

The empirical case is decisive. `GeometryRequirement` is a `[Flags]` enum with 8 bit-flags and 8 named composites. I enumerated 102 callsites across `libs/csharp/analysis`, tests, and Core itself: every one references a named composite (`Basic`, `CurveLength`, `AreaMass`, `VolumeMass`, `SurfaceEvaluation`, `MeshCheck`, `SolidTopology`, `StrictStructure`, `Strict`, `None`). Zero callsites perform `|` or `&` on bit-flags. The bitmask abstraction is unused externally. Internally, the 13 `GeometryCheck` static fields each carry a `requirement` field tested via `Includes` (`HasFlag`); the dispatcher folds across all 13 regardless. Right shape: `sealed abstract record GeometryRequirement` with one case per composite, each carrying an immutable `Seq<GeometryCheck>` listing applicable checks. The dispatcher then folds only `requirement.Checks`. Thinktecture `[Union]` is correct if external switch-style consumption is needed; otherwise hand-rolled abstract record with `Switch`/`Match` extensions aligns with CSP0702. Estimated reduction: -90 LOC (-10 removing `[Flags]` plumbing, -80 collapsing static-field check declarations into record cases). The collapse kills the CSP0707 risk in `Includes` and the CSP0001 statement-body risk in check lambdas.

## (iii) Analyzer Coverage Delta

The catalog ships 65 active CSP rules covering control flow, match discipline, type discipline, surface area, async, resources, performance, shape, DU shape, validation, regex, boundary metadata, reassignment, filter-map chains, Atom, telemetry, mutable accumulator, and governance. `AnalyzerDispatcher.cs` correctly wires these against Roslyn `IOperation` kinds. Three coverage gaps surfaced (C-14/15/16): no rule flags `[Flags]` enums whose external callsites never perform bitwise composition - the GeometryRequirement abuse is unflagged; no rule warns when `Aggregate` threads a `Fin<T>`/`Validation<E,T>` accumulator instead of `TraverseFin`/`TraverseValidation`; no rule detects ambient-state escape via boundary singletons such as `RhinoDoc.ActiveDoc` from Domain scope. The GeometryRequirement abuse could not have been caught by `pnpm check:cs` - the rule does not exist.

## (iv) What Is Good

The Core layer is overwhelmingly correct. `GeometryContext` is a textbook runtime-record: sealed partial, four `[ValueObject<double>]` wrappers with range gates, parallel-accumulating `Create` returning `Validation<Error, GeometryContext>`. `AnalysisRuntime` evolves immutably via `with`. `Resilience.StandardPolicy` uses canonical LanguageExt 5 algebra. Fault factories produce `Error.New` flowing through `Fin<T>`/`Validation<Error, T>` without throws. `RhinoMatch.RequireValid` is a clean switch expression. `GeometryValidation` composes the 13-check fold via applicative `Apply`. The layer respects file-scoped namespaces, explicit accessibility, sealed DUs, `static` lambdas. The CSP0001 violation in `FoldExtensions.StatsOf:39` is the lone exception.
