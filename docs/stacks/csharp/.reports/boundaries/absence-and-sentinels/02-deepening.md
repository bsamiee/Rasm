# Absence and Sentinels â€” Deepening

[SAFEHANDLE_DANGEROUSADDREF_SEMANTICS]:
- `DangerousAddRef(ref bool success)` never sets `success = false` in the non-exceptional path. It throws `ObjectDisposedException` when the closed bit is set, and reaches `success = true` only after a successful CAS loop. The canonical caller pattern initializes `bool success = false` before the call so that a thrown exception leaves `success` unset, enabling a `finally { if (success) handle.DangerousRelease(); }` guard. Treating `success == false` post-call as a silent absence signal is wrong; in the throwing path the `finally` block is the gate.
- `DangerousAddRef` atomically increments the reference count field, so the guard is not re-entrant-safe per handle instance. Projecting the handle to `Option<T>` before any ref-counted access eliminates the hazard entirely: the option is the scoped reference and the raw handle is never stored past the projection point.

[ATOM_CHANGE_EVENT_SUBSCRIPTION_IDENTITY]:
- `Atom<A>.Change` is a plain `delegate void AtomChangedEvent<in A>(A value)` event, not an observable or a disposable subscription. Deregistration requires `atom.Change -= handler` where `handler` is the exact same delegate instance used for `+=`. A new lambda at the `-=` site creates a delegate that compares unequal to the original, silently leaving the handler registered and firing indefinitely. The seam must capture the handler as a named field or local of type `AtomChangedEvent<A>` and store it for symmetric detachment.
- `AtomHashMap<K, V>.Change` fires `HashMapPatch<K, V>` events only when a listener is attached at mutation time; if no listener is attached, the internally tracked change log is never built. Subscribing to `Change` after a mutation never retroactively fires; the patch log is ephemeral. A seam that derives capability state from key transitions must subscribe before any write.

[OPTION_BIMAP_NULL_DIVERGENCE]:
- `Option<A>.BiMap<B>(Func<A, B> Some, Func<Unit, B> None)` passes its result through `Check.NullReturn`, which throws `ResultIsNullException` if either branch returns null. `Option<A>.BiMap<B>(Func<A, B> Some, Func<B> None)` does not call `Check.NullReturn` â€” if the `None` factory returns null, the result is `default(Option<B>)`, silently presenting as `None`. The two overloads differ only in the `None` parameter type but diverge on null safety. Prefer `BiBind`, which stays on the `Option` rail and never touches raw references.

[OPTION_TRAVERSE_EFFECT_SKIPPING]:
- `Option<A>.Traverse<F, B>(Func<A, K<F, B>> f)` when the option is `None` returns `F.Pure(Option<B>.None)` without calling `f`. `Option<A>.TraverseM<M, B>(Func<A, K<M, B>> f)` has identical `None` behavior. A seam that uses traversal to run a side-effecting validation or I/O lift over an optional value silently skips the effect when the option is absent. If the effect must always run for auditing or logging, it must be sequenced outside the traversal, not embedded in it.

[OPTION_SEMIGROUP_FIRST_SOME]:
- `Option<A>.Combine(Option<A> rhs)` returns `this` when `IsSome`; otherwise returns `rhs`. The `|` operator implements the same first-`Some` semigroup semantics. A chain `primary | secondary | tertiary` evaluates left-to-right and short-circuits on the first present value. When chaining fallback absence sources at a seam boundary, the semigroup is the correct surface rather than nested `IfNone` calls. `Seq.Choose<A, B>(Func<A, Option<B>> selector)` applies the selector and retains only `Some` values, replacing any `Where`+`Select` pair that re-applies the predicate.

[ATOMHASHMAP_SWAPKEY_OPTION_SEMANTICS]:
- `AtomHashMap<K, V>.SwapKey(K key, Func<Option<V>, Option<V>> swap)` atomically handles all four transitions: absentâ†’absent (no-op), absentâ†’present (add), presentâ†’absent (remove), and presentâ†’present (replace). This is the correct primitive for a capability registry where keys may materialize or disappear â€” a single `SwapKey` call replaces any read-then-write pair.
- The `Change` event fires only when a listener is attached at the time of the CAS loop body, not at the time of the outer `SwapKey` call. A subscriber added between the check and the `AnnounceChange` call will not receive the event for that specific mutation.

[OPTION_NATURAL_TO_FIN_COST]:
- `Option<A>.ToFin()` with no argument maps `None` to `Fin.Fail<A>(Errors.None)`. `Errors.None` is `ManyErrors(Seq.empty<Error>)` with code `-2000000006` and an empty error sequence. `HasCode` on `ManyErrors` delegates to `Errors.Exists(e => e.Code == code)` â€” an empty sequence always returns `false`. `ToFin()` therefore produces a failure that matches no code and no type predicate, making it impossible to discriminate at recovery sites. Always use `ToFin(Error.New(code, message))` with a non-zero domain code.

[CANCELLATION_NONE_TOKEN_REGISTRATION]:
- Registering on an already-cancelled token (`IsCancellationRequested == true`) invokes the callback synchronously before `Register` returns. `UnsafeRegister` skips `ExecutionContext` capture, so the synchronous invocation occurs directly on the calling thread. At a seam that transitions `Atom<CapabilityState>` from a cancellation callback, a `UnsafeRegister` callback that calls `Atom.SwapMaybe` must be reentrant relative to the registering thread. The guard is to check `IsCancellationRequested` before calling `Register` or `UnsafeRegister` and handle the already-cancelled state directly, bypassing registration.

[TERMINAL_BUILT_IN_CODES]:
- `Errors.SourceClosed` (code `-2000000013`, `IsExpected = true`) is the correct terminal receipt for a streaming or channel seam that has permanently exhausted its source, distinguishable by `error.HasCode(-2000000013)`. `Errors.Closed` (code `-2000000004`) covers general close states. Using these built-in singletons rather than custom domain codes avoids collision with the framework's own recovery machinery.
- `Errors.TimedOut` (code `-2000000002`) is the terminal receipt for a time-bounded absence detection window that expired. When `EnvIO.LocalCancelWithTimeout(TimeSpan)` times out, the resulting `Eff` failure carries this code. A recovery handler must distinguish a timed-out probe from a permanently-failed capability: a timeout is evidence of transient absence, not confirmed unavailability, and the capability cell should transition to `Degraded` rather than `Unavailable`.

[OPTION_GUARD_UNAVAILABILITY]:
- `Prelude.guard<F>(bool flag)` is constrained to `where F : MonoidK<F>, Applicative<F>`. `Option` implements `Alternative<Option>` and `Choice<Option>` but not `MonoidK<Option>`, so `guard<Option>(condition)` does not compile. Absence filtering inside an `Option` query expression must use `Option<A>.Filter(pred)` or `Option<A>.Where(pred)` instead â€” both return `None` when the predicate is false on a `Some` value and leave `None` unchanged, with no named error carrier.
- For typed-error absence guards over `Eff` or `Fin`, `guard(flag, Error.New(code, message))` is correct â€” the error-bearing overload returns `Guard<Error, Unit>` and binds into `Eff` query expressions. The absence is named and recoverable, not silently swallowed.

[LOCK_SCOPE_ABSENCE_INTERACTION]:
- `System.Threading.Lock.EnterScope()` returns a `ref struct Scope` whose `Dispose()` calls `Exit`. Because `Scope` is a `ref struct`, it cannot be stored in an `Option<Scope>`. A pattern that conditionally acquires a lock based on a domain-absent state must keep acquisition and scope structurally within the same stack frame. An async or effect-pipelined operation that needs conditional locking must instead use `Lock.TryEnter(timeout)` returning `bool` and map the `bool` to a domain absence signal. `Lock.TryEnter()` with zero timeout returns `false` immediately if contended; that is a boolean absence signal meaning "lock currently held," not "lock does not exist," and the caller must handle the contention case explicitly.

[CHANNEL_SEAM_ABSENCE_PROJECTION]:
- `ChannelReader<T>.ReadAsync(CancellationToken)` throws `ChannelClosedException` when the channel is completed and empty. At the seam boundary, capture via `Error.New(exception)` and map to `Errors.SourceClosed` or a domain-specific terminal receipt; the thrown exception is `InvalidOperationException`-derived and its `Message` is not stable â€” never match on string content.
- `ChannelReader<T>.TryRead(out T)` returns `false` for both "no item available" and "channel closed and empty" â€” the absence cause cannot be distinguished from `TryRead` alone. Check `Completion.IsCompleted` to distinguish transient absence from terminal absence.
- `ChannelWriter<T>.TryComplete(Exception? error)` embeds the exception into `ChannelReader<T>.Completion`, surfacing it from `ReadAsync` as a `ChannelClosedException` whose `InnerException` carries the original. A typed `Error` is lost through this path; terminal receipts that must survive the channel must be encoded in the element type, not in the completion signal.

[ATOMHASHMAP_CAPABILITY_REGISTRY_PATTERN]:
- A capability registry keyed by a bounded vocabulary is modeled as `AtomHashMap<CapabilityKey, CapabilityState>` where `CapabilityKey` is a `[SmartEnum<K>]`-generated key and `CapabilityState` is a closed union. `SwapKey(key, prev => Some(next))` transitions a single capability atomically. The health fold must project every key in the vocabulary â€” not just keys present in the map â€” using `smartEnum.Values.Map(key => (key, registry.Find(key).IfNone(CapabilityState.Unknown)))`. Folding only over `registry.AsIterable()` silently omits unprobed keys from the health picture.
