# Variant Unions and ADTs

[GROUP_OWNER_SELECTION]:

[OWNER_BY_SEAM_COST_NOT_CASE_COUNT]:
- Controlling rule: one owner is selected by `cost = codec_weight + scan_penalty + discriminant_coupling` per variant family, never by arm count — a two-arm family across a wire takes `Schema.Union` of tagged members, a ten-arm interior-only family takes `Data.taggedEnum`.
- `codec_weight` is the only seam-forced axis: a family never crossing `Schema.decodeUnknown` pays zero (tagged-class codec is dead surface), a wire crossing forces it regardless, collapsing the question to *which* tagged owner not *whether* to tag.
- Accept the bare inline `type T = A | B` floor only at one discriminant reader never serialized: it forfeits the whole derivation family (`Schema.equivalence`, `Arbitrary.make`, `Pretty.make`, `JSONSchema.make`, the `_tag` decode search tree), each forfeited artifact a future re-derivation — the floor admits exactly one `x.kind === "<lit>"` read before the minimum moves to `Data.taggedEnum`, whose `$is("<lit>")` is that guard derived once.
- Reject promoting on three near-identical arms: that is the trigger to *factor a shared field block* into a base `Schema.extend`-distributed across the union, not to choose tagged-versus-adhoc — the three seam axes alone move the owner.
- Reject re-wrapping a self-`_tag` shape through `Data.taggedEnum`: the structural gate `ChildrenAreTagged<A>` (mapping `"_tag" extends keyof A[K]`) resolves the constraint to the literal error `"…already has a `_tag` property."`, forcing the choice to *either* the hand-spelled inline tagged union *or* the generated owner — a pre-tagged member collides with the synthesized `{ _tag: Tag } & A[Tag]` field at the constructor's type position, never both layered.
- Boundary: the crossover is anticipatory — the moment a second reader, a wire crossing, or a reusable guard is conceivable, the minimum has already moved; widening an inline union after consumers exist re-derives the `_tag === "<lit>"` predicate and a fresh comparator at every site the tagged owner deletes with one declaration.

[DISCRIMINANT_COUPLING_TRICHOTOMY]:
- Controlling rule: three discriminant spellings partition on `discriminant_coupling` alone, every other axis equal — a bare `_tag: Schema.Literal` field and a `Schema.tag` defaulted signature both *store* the tag (wire-coupled) while `attachPropertySignature` *injects* it (computed on decode, stripped on encode, wire-decoupled), the trade coupled-versus-injected not stored-versus-defaulted.
- `Schema.tag(literal)` is `PropertySignature<…, true, never>` whose sixth slot `HasDefault = true` is load-bearing: the tag rides the signature as a constructor default so `Struct.Constructor` strips `_tag` to optional and `.make`/`new` never pass it — a bare `_tag: Schema.Literal("Open")` is instead a *required* argument, `Schema.tag` the only spelling decode-validated, encode-emitted, and constructor-elided at once, accepting `AST.LiteralValue` (string, number, boolean, bigint, `null`) so `Schema.tag(0)` discriminates on a number literal with identical elision.
- Accept a `symbol` discriminant for maximal decoupling: the injected key is unreachable by JSON yet live for interior `Match.discriminator` dispatch, the only spelling keeping the wire contract and the dispatch contract independently authored over one owner where a string key would collide.
- Reject a paired add-on-decode plus delete-on-encode `Schema.transform`: `attachPropertySignature` returns `SchemaClass<A & { readonly [k in K]: V }, I, R>`, one combinator owning both coupling directions.
- Boundary: the injected form pays the admission codec weight while paying zero wire coupling — the seam-cost minimum for a legacy payload whose tag must not serialize.

[GROUP_DISCRIMINANT_DISAMBIGUATION]:

[ADMISSION_GATE_SPANS_BOTH_SEAMS]:
- Controlling rule: `getLiterals` admits a property as a discriminant only when `AST.isLiteral(type) && !propertySignature.isOptional` — one required `Literal` AST node — so a `_tag` typed `Schema.Literal("a", "b")` (a *union* of literals, not one node) or an optional tag silently falls to `otherwise` and forfeits the bucket lookup with no type-level signal.
- The identical single-literal-required condition gates both seams: the decode search tree keys on it and `Match.discriminator(field)` keys on `Tags<D, R>` the same way — a family failing the gate at decode fails it at dispatch too, forcing a structural `Match.when` whose residual totality is `Refinement`-fragile where the gated family takes a key lookup.
- `scan_penalty` is set by member *kind* not count: one bare-`Struct` (`otherwise`) arm is appended to *every* discriminated bucket's candidate list, so `scan_penalty = bucket_size + otherwise_count`, minimized to `1` only when every member earns the gate — a wide or optional tag on a single arm pushes the whole family to undiscriminated cost.
- Boundary: the gate co-determines error locality — a strict-subset bucket localizes the failure diagnostic to that bucket's expected shape, so an `otherwise`-heavy or wide-tag union regresses decode cost, dispatch totality, and diagnostic blast radius with one shape decision; choosing the gated discriminant once amortizes a single key lookup across admission and every interior dispatch site.

[TWO_TREES_ONE_OWNER_DIRECTION_SELECTS_THE_LITERAL]:
- Controlling rule: `getSearchTree(ast.types, isDecoding)` builds two structurally distinct bucket maps from one `ast.types` — `isDecoding` is not a filter but selects *which literal each member contributes*, so the decode tree keys on each arm's encoded-side literal and the encode tree on its type-side literal, both recomputing when one arm is appended.
- A `Transformation` arm reads `getLiterals(isDecoding ? ast.from : ast.to, isDecoding)`, so a `transformLiteral<Type, Encoded>` decodes by its wire `from` code and encodes by its domain `to` name, two disjoint namespaces keying one owner, direction threaded down recursively never re-decided per nesting level.
- Accept the equality reuse: `Schema.equivalence` calls `getSearchTree(ast.types, true)` with decode direction hardcoded, so derived structural equality dispatches on the same encoded-side discriminant and the identical `String(a[name])` coercion — decode disambiguation, encode disambiguation, and equality are three reads of one search-tree algorithm, a clean tag namespace paying down all three at once and a collision regressing all three.
- Reject a `decode` switch on the numeric code beside an `encode` switch on the string name: the two-tree owner reads each direction's literal from the AST rather than assuming the field name is direction-stable, so a hand-written decoder keying on the domain name mis-routes every wire input on a renamed-field arm.
- Boundary: a `suspend` node forces its thunk per built tree — `getLiterals(ast.f(), isDecoding)` reaches through the cut point so a recursive tagged family decodes by single bucket lookup at every level (memoized over the AST node, per-input dispatch allocation-free), but an un-annotated thunk infers `any`, finds no `Literal` node, and silently falls to `otherwise`; the `Schema.Schema<…>` annotation on the thunk return is the load-bearing bit keeping the forced AST partitionable.

[ARM_SELECTION_IN_A_SHARED_BUCKET_IS_GATED_BY_LITERAL_HOMOGENEITY_AND_FIELD_EXACTNESS]:
- Controlling rule: the bucket index is `String(literal.literal)` under both build and lookup, so a numeric `0`, a string `"0"`, and a bigint `0n` hash to one key `"0"` and share a single bucket — a true bucket merge invisible to the type system.
- `kind: Schema.Literal(0)` and `kind: Schema.Literal("0")` in one union resolve both inputs to a two-candidate structural trial in declaration order, so the law is keep a discriminant family on *one* literal-value type.
- `Schema.TaggedClass`'s `Tag extends string` constraint is the surface guard for homogeneity (numeric tags spellable only off `Schema.tag`/`TaggedStruct`/bare `Literal`).
- The candidate set is accumulated by *union* of buckets while narrowed by *intersection* of presence — `candidates = candidates.concat(buckets[literal])` per populated namespace, no dedupe — so trial count is `Σ |bucket(namespace_i)|`, a member in two matched buckets tried twice, first structural success winning.
- A family sharing `kind: "shape"` but private on `form: "circle" | "square"` partitions on `form` while `kind` holds both, so collapsing the shared `kind` into a category arm removes the redundant namespace.
- `onExcessProperty: "error"` is evaluated *per admitted candidate* inside each TypeLiteral parse, so it is an arm-selection force not only a field-shape check — an input the discriminant correctly admitted fails because it carries a key that candidate's signature does not declare, the same input decoding cleanly under default `"ignore"` rejected under `"error"` as `Unexpected`.
- `"error"` runs each candidate's excess check against its *own* field set across a shared bucket, so a key belonging to sibling arm `B` present on an input meant for `A` makes `A` fail though `A` is the structurally-correct arm — selection gated by field exactness, not discriminant alone.
- Boundary: under `"ignore"` the excess loop is skipped entirely (`onExcessPropertyError || onExcessPropertyPreserve` guards the block), keeping selection on the discriminant axis the search tree owns; a union decoded under `"error"` must have *non-overlapping field sets* per shared bucket, or route the lenient policy at the boundary so arm selection stays a pure discriminant lookup — `attachPropertySignature` moves a bare struct out of the structural fallthrough into a singleton bucket without the wire carrying the tag.

[GROUP_MATCH_RESIDUAL]:

[RESIDUAL_IS_THE_MATCHER_CARRIER_FINALIZERS_ARE_ONE_READ]:
- Controlling rule: the residual is the matcher's own `value: Either<Provided, Remaining>` carrier the finalizer projects, not a fresh reconstruction — `Match.value(i)` seeds `Either.left(i)`, the first passing arm flips to `Either.right(evaluate)`, and once `Right` every later arm returns `this` unchanged (`if (this.value._tag === "Right") return this`), so first-match-wins is structural short-circuit on the carrier, not arm-ordering discipline.
- Five finalizers are one carrier read at three altitudes: `either` is primitive, `option` is `either` post-composed `{ onLeft: None, onRight: Some }`, `orElse(f)` the same read with `f(left)`, `orElseAbsurd` is `orElse(() => absurd)` — the residual a *value* the family forwards, reified at the *applied* `RemainingApplied = ApplyFilters<I, Filters>`, so `either` returns `Either<Unify<A>, R>` with the exact unhandled sub-union as the `Left` never the full `I`.
- Provenance, not a second entrypoint, selects function-versus-value: every finalizer branches `[Pr] extends [never]` — `Match.type<I>()` (`Provided = never`) finalizes to a reusable `(input: I) => …`, `Match.value(i)` (`Provided = I`) to the value directly, a `Match.value` chain baking one `i` into the carrier and spent after one run.
- Boundary: build a staged forward channel from `Match.type` and apply it per element; reserve `Match.value` for a one-shot residual whose only consumer is the immediately-following `Either` read.

[LEFTOVER_IS_THE_NEXT_STAGES_INPUT_AND_FIDELITY_IS_THE_RESIDUAL_FIDELITY]:
- Controlling rule: the channel composes through `Either.orElse(that: (left: E) => Either<A2, E2>)` — stage one's residual `E` is *literally* stage two's `Match.type<E>` input, the checker proving stage two is keyed on precisely the arms stage one declined, the result `Either<A | A2, E2>` carrying only stage two's leftover so staging narrows the residual monotonically `E → E2 → E3`; `Either.flatMap` is the dual axis, threading the *matched* value and *growing* the leftover as `E | E2`, the two combinators the two axes of the residual lattice selected by which channel the next stage consumes.
- The channel's whole value is the identity `R = true leftover`, so any arm matching at runtime without subtracting from `RemainingApplied` propagates a phantom domain forward — the break surfaces *loudly two stages downstream* where the stage-two `Match.type<R>` demands an arm for a case stage one already ran, the re-entry seam (not the offending arm) the diagnostic site; a non-subtracting arm is a refinement-less predicate (`PForExclude<P>` maps `Predicate.Refinement<A,B>` to `SafeRefinement<B,B>` subtracting `B`, but bare `Predicate.Predicate<A>` to `SafeRefinement<A,never>` subtracting nothing — `Match.nonEmptyString`/`instanceOf(C)` non-subtracting, `Match.string`/`number`/`boolean`/`instanceOfUnsafe(C)` subtracting), so route narrowing guards as `Refinement` (`(x): x is B`) when totality is the goal.
- Accept `Either.all([...stages])` for orthogonal-axis classification: independent partial matchers over the *same* input fold to one product `Either` whose `Right` is the tuple of matched results and whose `Left` is the union `E1 | E2 | …` of every axis's leftover — the applicative fold the sequential `orElse` chain cannot express, replacing parallel `if` ladders re-reading `_tag` whose joint failure would need a hand-merged sentinel.
- `option`/`Option.firstSomeOf` is the drop-dual axis: `either`/`orElse` reports *what* declined (the `Left` carries the re-enterable typed domain), `option`/`firstSomeOf` reports only *a* result (the leftover erased to `None`) — downgrading an `either` stage to `option` is the irreversible point past which the leftover can no longer drive a staged re-entry.
- Boundary: re-entry composition raises the precision bar a one-shot close never enforces — a matcher used only as terminal `exhaustive`/`orElseAbsurd` tolerates a phantom residual the empty close papers over, but the *same* matcher reused as a forward stage cannot; pairing the last stage's `Match.exhaustive` (residual `never`) collapses the final `Left` to `never`, turning a staged channel into a total dispatch assembled from independently-authored partial pieces.

[GROUP_TOTALITY_UNDER_WIDENING]:

[TWO_DETECTORS_ONE_OBLIGATION_AND_THE_DEFAULT_SWALLOW]:
- Controlling rule: totality is enforced by two structurally distinct detectors — *residual totality* gates `Match.exhaustive` on the recomputed `Remaining` being literally `never` (a one-sided subtraction proof, *nothing is left*), and *record-shape totality* gates `$match`/`tagsExhaustive`/`typeTags`/`discriminatorsExhaustive` on a record `P` whose keys must equal the tag alphabet (`[Tag in Tags<"_tag", R>]: Fn` demands every tag, `[Tag in Exclude<keyof P, Tags<…>>]: never` forbids foreign tags — a two-sided set-equality proof, *the handler set equals the tag set*).
- The detector decides how a new arm breaks and what the diagnostic says: the record detector is byte-stable across vectors (`Property '<NewTag>' is missing` at the dispatch site by *any* widening path, keying only on the resulting `Tags<"_tag", R>` alphabet) while residual totality names the *finalizer* with `Type '<NewArm>' is not assignable to type 'never'`, one declaration removed from the union edit that caused it; the subtraction proof cannot catch a stale arm (dropping a `Schema.Union` arm leaves a `Match.tag(removed)` ladder type-checking, `AddWithout` over an absent tag subtracting `never`, dead code accumulating invisibly) while the equality proof rejects the removed key as `never`.
- The optional-handler twins are the latent widen failure: `Match.tags(handlers)`/`Match.discriminators(field)(handlers)` type their record `[Tag in Tags<"_tag", R>]?: …` (every key optional) and thread `AddWithout<F, Extract<R, Record<"_tag", keyof P>>>`, so appending an arm type-checks *unchanged* and the new arm falls to the trailing `Match.orElse` whose handler is typed at the *recomputed* residual `RA` and silently widens to include it — no key demanded, the swallow surfacing only two edits downstream when the widened param loses a field every member no longer shares; the optional-record-plus-default is correct *only* for a deliberately-open family, a closed family taking `tagsExhaustive`.
- Boundary: a dispatch forwarding its leftover takes the residual detector (the leftover is a typed channel); a dispatch closing in place over a `_tag` family takes the record detector (no leftover, the most collapsed spelling, stale-arm rejection free) — `when`/`tag`-plus-`exhaustive` is admitted only for genuinely heterogeneous patterns (refinements, structural shapes) the record detector cannot key on.

[WIDENING_VECTOR_TRICHOTOMY]:
- Controlling rule: a family widens along three vectors, each re-opening totality at a different site — *arm-append re-declaration* (`Schema.Union(...existing, NewArm)`) widens `Schema.Type<Members[number]>` and re-opens *both* detector families at once (decode and dispatch), the loudest; *type-level alphabet widen* (a new `Data.taggedEnum` key) re-opens *only* the record detector (`$match`'s key gap — the decode-free owner has no `Schema.is` and no codec search tree); *input-position widen* (a matcher seeded over a union that absorbed an arm) re-opens `Remaining` silently — the filter sequence byte-identical, `Exclude<I_wide, consumed>` simply retaining the new member, breaking only at the finalizer.
- A tagless widen splits the detectors: `Schema.NullOr(Owner)` yields `Owner | null` whose `Tags<"_tag", I>` projection is *unchanged* (`null` contributes no `_tag`), so `tagsExhaustive` type-checks while `Remaining` retains `null` and `exhaustive` breaks with `Type 'null' is not assignable to type 'never'` — the record detector blind to any inhabitant *off* the discriminant axis; reconcile by admitting `null` as a `Schema.tag`-defaulted `Absent` arm so the widen *is* a `_tag` widen both detectors see.
- The `not` combinator can turn a widen into a *false* pass: `ApplyFilters<I, never>` is `never` for any `I`, so a poisoned `Filters` makes `exhaustive` accept unconditionally — a `not`-built matcher whose `Only` window fit the pre-widen `I` can, after arm-append, have a matched slice escape the window, collapsing `Filters` to `never` and satisfying `exhaustive` while the new arm runs unhandled; immunize by pairing each `not(pattern, f)` with one positive arm consuming `pattern`'s own slice (`AddWithout<Only<X>, X> = Only<never>`, honestly `never`), never two intersecting `not` windows a future `I` can escape.
- Return totality is a third axis a *placement* arms or disarms for future arms: `Match.withReturnType<Ret>()` placed *first* pins `Ret` while `Result` is still `never` (its `[Ret] extends [[A] extends [never] ? any : A]` constraint passing `any`) and constrains every *subsequent* arm's `ReturnType<Fn>`, so a widen returning the wrong type breaks twice; placed *late* it compares against a populated `A` and stops gating, silently admitting a divergent return `Unify` widens the whole `Result` to absorb — `withReturnType` is the first pipe step on any matcher whose return contract must survive arm growth.

[GROUP_PROJECTION_ALGEBRA]:

[PROJECTIONS_ARE_A_CLOSED_ALGEBRA_OVER_THE_OWNER]:
- Controlling rule: union projections form one algebra closed at the *value* and *typed-schema* level — carve a literal sub-vocabulary, carve a sub-union of arms, filter a heterogeneous stream to one arm, recover one arm's codec — each takes the owner (or a prior projection) and yields a value the same algebra accepts again, so a partition's residual sub-union feeds `Match.exhaustive`, a picked subset feeds an arm `Extract`, an arm codec feeds a selective decode; closure breaks the instant a step drops to `union.ast.types` (a stripped node carrying no codec and no static witness), so a typed-class projection indexes `members` (`Schema.decodeUnknownSync(members[0])` decoding precisely that arm) while a custom AST pass reads `types`.
- A literal sub-vocabulary and an arm-union projection are *the same projection one level apart*: `Schema.Literal(...values)` is the closed vocabulary owner exposing `literals` (typing `Literals[number]` — the legal set *is* the schema not a parallel `as const`), `pickLiteral(...subset)` carves the discriminant literals (`Literal<[...L]>`, every picked value checked an `A` of the parent), and `Extract<Schema.Type<typeof Union>, { _tag: Sub }>` carves the corresponding arms — `Match.whenOr(...Elevated.literals, f)` spreads the picked subset directly as patterns, widening `Elevated` by one `pickLiteral` argument widening both gate and routed arm with zero second edit; a `Schema.Literal('admin','owner')` hand-spelled beside the parent compiles identically yet carries *no* verified subset relationship, silently desyncing on a parent rename the `pickLiteral` projection breaks loudly on.
- The selective decode is a per-arm codec the union-wide codec cannot express: feeding `union.members[i]` to `Schema.decodeUnknown` admits exactly one variant (the *narrowing* decoder, the union the *admitting* decoder), surviving the effectful surface through `Stream.filterMapEffect`/`Stream.partitionEither` where each arm's decode failure rides the stream `E` channel; a union persisted as `Schema.Array(union)` recovers its per-element family through `Schema.getNumberIndexedAccess` (`SchemaClass<A[number], I[number], R>`, discriminant intact), never re-declaring the union past the array wrapper.
- Boundary: because each projection's output re-enters the algebra, a new arm appended to the owner widens every downstream projection (the residual sub-union, the subset's complement, a `filterMap`'s `$match` totality) at compile time with zero projection re-authored — the multiplicative collapse the per-site `x._tag === "<lit>"` partition forfeits.

[FILTER_PRESERVES_GROUPBY_ERASES_FILTERMAP_FUSES_SLOT_PIN_GOVERNS_GENERICS]:
- Controlling rule: `$is(tag)` composes into higher-order filtering by the *arity of the refinement parameter each surface accepts* — `Array.filter`'s `(a: NoInfer<A>, i: number) => a is B`, `Stream.filter`'s unary `Refinement<NoInfer<A>, B>`, `Filterable.filter`'s `(a: A) => a is B` all accept the unary monomorphic `$is` (the surplus `i` unread), `NoInfer<A>` the load-bearing bit blocking the guard's `unknown` input from widening `A` so `values.filter(Sig.$is("Open"))` resolves `A = Sig` from the carrier and `B = Extract<Sig, { _tag: "Open" }>` from the `u is` clause.
- `Array.groupBy(values, v => v._tag)` is the trap that *looks* like the per-tag partition and silently erases it: its return `Record<NonLiteralKey<K>, NonEmptyArray<A>>` widens the literal `_tag` to bare `string` and holds the **full union** per bucket (`groups["Open"][0].at` a compile error) — a key-returning selector erases the discriminant while a refinement-shaped predicate threads `Extract`/`Exclude`: `Array.partition($is("Open"))` yields `[Array<Exclude<A,B>>, Array<B>]` both halves exact, the excluded side the residual sub-union a downstream `Match.exhaustive` already knows `Open` is gone from.
- Accept `Array.filterMap` over a `$match`/`Match.option`-finalized matcher to fuse *partition by tag* with *transform per arm* in one *total* dispatch: every arm returns `Option.some(projection)` for kept tags and `Option.none()` for dropped, dispatch total (the checker demands every arm) while the result is partial (the `None` arms drop) — `Match.option` returns `Option<Unify<A>>` collapsing the residual to `None`, fusing three sibling passes (keep-filter, project-map, drop-filter) into one through the `Option` carrier.
- One law governs every generic-owner (`WithGenerics<N>`) projection: the slot must be concrete at the *site the projection reads it*, because the opaque `Kind<Z, A>` indirection is `infer`-inert — the generic `GenericMatchers.$is` *narrows only when directly applied* (its two-overload dual cannot solve the free `<T>` inside a `filter` callback, falls to the `unknown`-collapsing overload, and `forest.filter(Tree.$is("Leaf"))` yields `Tree<unknown>` arms with `.value` lost), so route a generic family through `Array.filterMap` with a data-first `Tree.$match(node, cases)` that infers the slot from the element.
- Boundary: the slot-pin law unifies three failures the surfaces present as unrelated — a higher-order `filter($is(tag))` (no caller value), a recursive constructor arm (the child contributes `unknown` before any annotation back-propagates), an `infer`-extraction over `Tree<A>` (the indexed access will not invert) — all one missing-concrete-binding fault resolved by routing through a concrete-value site (data-first `$match(value, cases)`, a pinned `Tree.Leaf<B>`, a pre-specialized `ReadonlyArray<Tree<number>>`); a deferred-callback projection drops a generic family to `unknown` arms no longer re-enterable, so `filter`-versus-`filterMap` over a generic union is the difference between exiting and remaining within the algebra.

[GROUP_WIRE_CODEC_PROJECTION]:

[STRIP_AND_JSON_PROJECTION_KEEP_THE_ARM_PARTITION_ON_THE_TAG]:
- Controlling rule: `typeSchema` and `encodedSchema` recurse the union by mapping the stripper over `ast.types`, so a tagged family's type-projection and wire-projection are each *themselves a union* of the same arm count — the wire view of an N-arm variant is an N-arm variant never a collapsed record, the `_tag` surviving every strip.
- Reject a parallel `WireUnion` restating each arm's encoded shape beside the domain union: `JSONSchema.make(encodedSchema(Owner))` is the wire schema and `Schema.is(typeSchema(Owner))` the interior guard, both keeping the arm partition, a new arm widening all three with one declaration.
- The `_tag` materializes on the JSON contract as `{ type: "string", enum: [literal] }` — a single-element `enum` *never* a JSON `const` — across every target (`jsonSchema7`, `2019-09`, `2020-12`, `openApi3.1`), so a consumer validates the discriminant by single-element-enum membership and a contract test asserting `const` mis-specifies the emitted shape.
- The union emits `anyOf` over its arm schemas (arity 0 → `constNever`, 1 → bare member, ≥2 → `{ anyOf }`), each arm an object schema carrying its singleton-enum `_tag`, structurally mirroring the runtime decoder's bucket partition.
- `compactUnion` fuses only *adjacent* bare single-element-enum members of the *same* type into one multi-value `enum`, so a `Schema.Literal('a','b','c')` vocabulary owner collapses to one closed enum while a `TaggedClass`/`TaggedStruct` family is structurally inert (its arms emit *objects*, `isCompactableLiteral` false) — the same `make` call yields two different shapes from a vocabulary owner versus a variant-family owner, a literal arm separated from its sibling by an object arm not merging (adjacency order-sensitive, the emitter walks declaration order once).
- A member's `identifier` annotation is the `$defs` cut point: the emitter replaces the arm inline with `{ $ref: "#/$defs/" + id }` and materializes `$defs[id]` by a *two-phase write* (assign the `$ref` placeholder *first*, then recurse the full arm), so a self-referential `suspend` arm finds its slot already occupied and terminates — `getIdentifierAnnotation` reaching *through* the suspend and a transform-to-declaration surrogate to find it, converting an infinite inline expansion into a finite cyclic contract.
- Boundary: a bigint or symbol discriminant has *no* JSON projection (the emitter throws a missing-annotation error for any non-string/number/boolean literal, `null` → `{ type: "null" }`), so `Schema.tag(0n)` forces an explicit `jsonSchema` annotation — the discriminant's wire-projectability narrower than its decode-admissibility; the encoded-side suspend re-threads its identifier with an `Encoded`/`EncodedBound` suffix so a recursive owner yields distinct domain-named and wire-named `$defs` graphs, `topLevelReferenceStrategy: "skip"` the one knob inlining the top-level union for an envelope that must not self-reference.

[GROUP_REQUEST_FAMILY]:

[TWO_SYMBOLS_ONE_ARM_FOUR_CORNERS_FROM_ONE_DECLARATION]:
- Controlling rule: `Schema.TaggedRequest<Self>()(tag, { payload, success, failure })` carries two orthogonal traits on one object keyed by two symbols never one merged record — `symbolSerializable` holds the *payload* `Schema<A, I, R>`, `symbolWithResult` holds `{ success; failure }` for the *outcome* — fusing six surfaces (tagged value-constructor, payload codec, structural `Equal`/`Hash`, runnable `Request.Request<SuccessType, FailureType>`, `SerializableWithResult` owner, `Schema.Union`-joinable nominal class) in one `class X extends … {}`, the result-pair carriage invisible to the decode search tree.
- The four contract corners recover by trait extractor never a mirror type: payload-decoded `Serializable.Type<typeof X>`, payload-wire `Serializable.Encoded<typeof X>`, outcome-success `WithResult.Success<typeof X>`, outcome-failure `WithResult.Failure<typeof X>` (plus encoded duals) — a function "what running this arm yields" is `WithResult.Success<typeof X>` never `Schema.Type<typeof X.success>` reaching through the live member; the up-casts `asSerializable`/`asWithResult`/`asSerializableWithResult` project each trait in isolation for a heterogeneous batch.
- The two slots split the arm's `R` into two distinct positions: `Struct.Context<Payload>` (payload-decode) and `Schema.Context<Success> | Schema.Context<Failure>` (result-decode), so `serialize(arm)` is `Effect<I, ParseError, R_payload>` while `serializeExit(arm, …)` is `Effect<…, R_result>` — an arm whose `success` schema pulls a service requirement carries it only on the outcome channel, the dispatch boundary and outcome boundary resolving against *different* layers, the requirement never dragged into the request-admission layer.
- The annotation axis is payload-scoped: the `annotations` argument is typed `ClassAnnotations<Self, Simplify<Struct.Type<{ _tag: tag<Tag> } & Payload>>>` (its message-scope sees the *payload* type only, never `success`/`failure`), so a per-outcome refinement (a `success` whose value carries `Schema.between`) rides the *outcome schema's* own annotation recovered through `successSchema(arm)` into `Arbitrary.make`/`JSONSchema.make` — the request value and its result two annotatable, two emittable surfaces, never one merged annotation conflating request and response.
- Boundary: the constructor elides two phantom slots by two composing mechanisms — `Struct.Constructor<Omit<Payload, "_tag">>` strips `_tag` (the `tag<Tag>` default supplies it) and `Request.Request.Constructor` omits `keyof Request.Request.Variance` (the covariant phantom) — so `X.make({ key })` passes neither discriminant nor variance witness; `Self` is mandatory (`MissingSelfGeneric<"TaggedRequest", …>` otherwise), the arm a first-class nominal class.

[FAMILY_DISPATCH_CHECKS_DISCRIMINANT_AND_PER_ARM_OUTCOME_AT_ONCE]:
- Controlling rule: a `Schema.Union` of `TaggedRequest` arms decodes by the same `_tag` search tree a plain `TaggedClass` union uses, so `Match.tagsExhaustive`/`$is`/`Schema.is` close over it identically — `Match.tag("QueryOne", req => …)` narrows `req` to the arm class making `req.success`/`req.failure` that arm's exact schemas and pinning the handler's return obligation, the dispatch and the result-type correlation one move.
- The handler's `Effect` is checked against the arm's *runnable* projection through `Request.completeEffect(arm, effect)` demanding `Effect<Request.Request.Success<arm>, Request.Request.Error<arm>, R>` and yielding `Effect<void, never, R>` — `Request.Request.Result<arm>` is `Exit<Success, Error>`, the *same* `Exit` shape `exitSchema` serializes, so the runnable outcome and the serializable outcome are one `Exit<SA, FA>` the trait pair derives from one declaration; a divergent handler `A`/`E` fails `completeEffect` at that single arm with the same locality `Match.tag` gives.
- `failure: Schema.Never` declares an infallible arm structurally not by a nullable error (`failureSchema(arm)` is `Schema.Never`, the runnable `E` channel `never`, the `ExitEncoded` failure branch uninhabited) — a heterogeneous batch over `TaggedRequest.All` (`Any | TaggedRequest<…, never, never, unknown>`, the trailing `never, never` the infallible arm) dispatching on the `WithResult` trait, each element's own outcome schemas threading through one loop with no per-arm fallibility branch.
- Boundary: pairing `Match.tagsExhaustive` (family discriminant coverage) with `Request.completeEffect` (per-arm runnable completion) checks both axes at once — a fourth request lands as one `TaggedRequest` class plus one matched-arm handler the checker demands; the rejected spelling is a hand-written `Schema.Exit({ success, failure, defect: Schema.Defect })` beside a separate `Effect<A, E>` signature, two contracts the next outcome desynchronizes.

[OUTCOME_NESTS_A_SIX_ARM_CAUSE_VARIANT_DEFECTS_ERASE_AND_THE_EXIT_CODEC_IS_PER_PROTO]:
- Controlling rule: the `failure` schema names only the *expected* `Fail.error` arm of a wider six-arm recursive cause variant the exit codec actually serializes — `ExitEncoded`'s `Failure` branch carries `CauseEncoded<E, D>` (`Empty`, `Fail`, `Die`, `Interrupt`, `Sequential`, `Parallel`), so an arm run as racing sub-effects serializes the *whole* composite cause tree through `serializeExit` never collapsing a parallel fault to whichever error won.
- A `Schema.TaggedError<Self>()(tag, fields)` arm placed in `failure` is the only fault both serializable through the result channel and directly `yield*`-able into the typed `E` channel without an `Effect.fail` wrapper, so appending a fault arm grows the runnable `Request.Request.Error<arm>` union and the wire `failureSchema(arm)` codec together with no second edit.
- The third `ExitEncoded` slot `D` is pinned to `unknown` by every accessor: `exitSchema` is built internally as `Exit({ success, failure, defect: Defect })` with the built-in `Schema.Defect` codec hardcoded into the `defect` position — an `unknown → unknown` transform decoding structured-error-shaped objects to `Error` and stringifying everything else — so a `Die`-thrown value the interior knew precisely re-enters the consumer at `Defect`'s lossy fidelity; a fault that must survive serialization typed belongs in the `failure` schema as a `TaggedError` arm, never left to surface as a defect.
- `exitSchema(arm)` is memoized in a process-global `WeakMap` keyed on the arm class's `proto`: the codec builds once per declared *class* not per value nor per serialization, so a resolver draining many values of one arm pays the `Exit({…})` build a single time and every element is the `encode`/`decode` walk alone — the rejected spelling constructing `Schema.Exit({…})` inline at the handler, re-allocating the transform every call and duplicating the cached codec.
- Boundary: the half-channel codecs see a different shape than the full exit by exactly the cause gap — `serializeFailure`/`deserializeFailure` read the bare `Fail.error` shape (no cause wrapper) so a fault-only notification crosses flat while `serializeExit` wraps it beside `Die`/`Interrupt`/composite arms; a `failure` union whose arms pull a service requirement places it only on `failureSchema`/`exitSchema`/`deserializeExit`, so the same caller decoding the returned `Exit` provides `R_result` while decoding the request payload provides only `R_payload`, the two `R` positions on the one proto keeping the fault-decode service at the channel that consumes it.
