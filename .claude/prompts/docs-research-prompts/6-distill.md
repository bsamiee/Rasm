Distillation turns each lane's full reservoir in `<WORKSPACE>` — the bedrock and the divergent series above it — into the single source the page is authored from: every candidate card the page might need, at the depth it needs, with zero duplication inside the lane and zero territory poached from sibling lanes. One agent per lane folder, plus an orchestrator ownership pass before any lane begins.

[DOCTRINE]:
- Every agent — the orchestrator and every distill agent — fully reads `<DOCTRINE>` before any other work: the laws, the collapse scan, the page craft, and what `<TARGET>` owns. That single read is the whole alignment requirement, and every selection, merge, and grade answers to it.

[OWNERSHIP] — orchestrator, before any lane runs:
- Free-chosen divergent facets drift across lanes; assume bleed until proven absent. Sweep every lane's file titles and group labels, assign each recurring theme to exactly ONE owning lane, and hand each distill agent its in-list (themes it owns, including variants sitting in other lanes' files) and out-list (themes it must drop because a sibling owns them). Routing moves material to its owner — a nuance is never deleted because it sat in the wrong folder.

[DISTILL] — one agent per lane:
- Read all of `00-bedrock.md` and every divergent file in the lane — the whole reservoir, nothing skipped — plus the routed-in material the orchestrator assigned from sibling lanes.
- Merge by nuance-union, never by averaging: when several files state near the same finding, state the shared spine once at full strength, then carry EVERY variant's unique nuance as qualifying clauses and edges attributed to conditions, not sources. The merged entry must be stronger than the strongest source; a summary that loses any variant's edge is a failed merge. When two findings genuinely conflict, verify cheaply if possible; otherwise prefer the more mechanistic claim over the more general, and when both hold under different conditions, state the condition split explicitly.
- Select and order the strongest material into one coherent body: group labels are candidate cards, ordered by what the page needs first; each candidate is shaped the way the page consumes it — controlling rule first, then accept and reject edges, boundary, the advanced interaction that earns it, and a snippet seed only where an advanced pattern justifies code. The divergent series sets the ceiling; the bedrock supplies the load-bearing foundations so every entry stands complete on its own.
- Cut what does not earn a card: table-stakes material, weaker duplicates of a stronger candidate, and findings whose value the doctrine's laws cannot place. Compression comes from dedup and density, never omission — a distillate near one-third to one-half of the reservoir's volume holding all of its distinct value is the expected shape.
- Write the result to `00-distilled.md` in the lane folder.

[REFINE] — the same agent, after writing:
- Cold re-read the distillate alone: any two entries sharing a spine merge into one; any entry that cannot stand without another file is completed in place.
- Source-coverage back-check: re-read the reservoir against the distillate asking one question — what distinct value exists in the sources that the distillate cannot reproduce? Everything found either enters the distillate or is consciously cut as table-stakes or weaker-duplicate. An accidental omission is a defect; a deliberate cut is a decision.
- Grade, rework, and repeat the two checks until a pass changes nothing.

[DOCTRINE_HARVEST] — orchestrator, after every lane clears:
- Read all distillates as one body. A principle that recurs across lanes and would change decisions on MULTIPLE pages is a doctrine-refinement candidate: report it with the exact law it sharpens and the proposed wording — principle-level only, universal across codebases, additive resolution on an existing law or a genuine gap, never a case rule. Page-level law stays in the distillates. The orchestrator adjudicates and applies accepted doctrine edits surgically; a doctrine edit mid-program triggers a targeted re-read of finalized pages against the changed law only.

[DISCIPLINE]:
- Findings and candidate cards only — logical category groups, with no meta-commentary, sourcing, versions, dates, cross-references, or introduction, routing, summary, or validation section. One H1, then `[GROUP_LABEL]:` groups of contiguous bullets with no blank line between items, and a blank line only around a fenced code block, with neutral placeholder names.

[GRADE]:
- The distill agent scores its own `00-distilled.md` one to ten on four axes — richness, veracity, density, advancement — and on relevance to the lane, against a bar of 9.2 on each. Grading is critical, harsh, but fair — not a 9.2 out of convenience, not low for nit-picking. Rework until it clears.
- The main agent then reads each `00-distilled.md` cold — giving no weight to the distill agent's pass — and grades and critiques it the same way, adding the ownership check: no sibling territory poached, no routed-in nuance dropped. It returns anything short for a targeted fix.

The distillate is the reservoir the page is mined from — more than the page will use, so the best can be chosen.
