# BCL System-Seam Replacements

[HANDLE-BASED FILE IO]:
- `File.OpenHandle(path, mode, access, share, options, preallocationSize)` returns `SafeFileHandle` without constructing a `FileStream`. The handle carries `IsAsync` state set at open time: only `FileOptions.Asynchronous` guarantees kernel-dispatched OS-async on Windows and io_uring dispatch on Linux — thread-pool emulation is the silent fallback for a synchronous handle passed to async calls.
- `RandomAccess` is a fully offset-addressed, stateless surface: `Read(handle, Span<byte>, long fileOffset)` / `Write(handle, ReadOnlySpan<byte>, long fileOffset)` for single-buffer; `Read(handle, IReadOnlyList<Memory<byte>>, long)` / `Write(handle, IReadOnlyList<ReadOnlyMemory<byte>>, long)` for vectored scatter/gather. Async variants accept `Memory<byte>` / `ReadOnlyMemory<byte>` (not `Span`) and return `ValueTask<int>` or `ValueTask<long>`. Position state is entirely caller-owned — no seek, no lock, safe for concurrent readers on distinct offsets.
- `RandomAccess.GetLength(handle)`, `SetLength(handle, long)`, and `FlushToDisk(handle)` complete the handle surface. Any shape that opens a `FileStream` solely to call `.Length`, `.SetLength()`, or `.Flush(true)` is the rejected form.
- `FileStreamOptions` is the only non-obsolete way to construct a `FileStream` when a stream interface is required; it consolidates `mode`, `access`, `share`, `options`, `bufferSize`, and `preallocationSize`. The positional `FileStream(path, mode, access, share, bufferSize, useAsync)` overload chain is the rejected form.
- `MemoryMarshal.Cast<TFrom, TTo>(Span<TFrom>)` reinterprets binary content in place — zero copy, no allocation. `MemoryMarshal.Read<T>(ReadOnlySpan<byte>)` and `TryRead<T>` read unmanaged blobs as structs; `Write<T>` / `TryWrite<T>` serialize structs to byte spans — eliminating `BinaryReader` / `BinaryWriter` for fixed-layout binary formats entirely.

[PATH ARITHMETIC]:
- `Path.Join(ReadOnlySpan<char>, ReadOnlySpan<char>[, ...])` and `Path.Join(params ReadOnlySpan<string?>)` are the allocation-minimizing forms; unlike `Path.Combine`, they never discard left segments when a right segment is rooted, which is the correct semantic for programmatic path construction. `Path.TryJoin` writes into a caller-supplied `Span<char>` with no allocation at all.
- `Path.Exists(string?)` unifies `File.Exists` and `Directory.Exists` into one predicate returning `true` for any filesystem entry — file, directory, or symlink target. The bifurcated check is the smell being replaced.
- `Path.GetRelativePath(relativeTo, path)` computes the relative path string; `Path.IsPathFullyQualified`, `EndsInDirectorySeparator`, and `TrimEndingDirectorySeparator` round out the non-allocating path predicate family. All accept `ReadOnlySpan<char>` overloads, avoiding round-trips through `string`.
- `Path.GetDirectoryName(ReadOnlySpan<char>)`, `GetFileName`, `GetExtension`, `GetFileNameWithoutExtension`, and `GetPathRoot` all accept and return `ReadOnlySpan<char>`, slicing the input with no heap allocation. The `string?` overloads exist for boundary cases only.
- `File.ReadLinesAsync(path, CancellationToken)` returns `IAsyncEnumerable<string>` — composable with `await foreach` and `System.Linq.AsyncEnumerable` operators without intermediate `Task<string[]>` buffering. Reject patterns that call `ReadAllLines` then iterate or that open a `StreamReader` to read lines.
- `File.AppendAllBytes(string, ReadOnlySpan<byte>)` and `AppendAllBytesAsync(string, ReadOnlyMemory<byte>, CancellationToken)` cover the binary-append case without stream construction.

[ONE-SHOT CRYPTOGRAPHY]:
- `CryptographicOperations.HashData(HashAlgorithmName, ReadOnlySpan<byte>, Span<byte>)` dispatches to any registered hash algorithm by name with no allocation. `TryHashData` writes into a pre-sized span; `HmacData` and `TryHmacData` follow the identical shape with an additional key span. `HashDataAsync(HashAlgorithmName, Stream, Memory<byte>, CancellationToken)` returning `ValueTask<int>` is the async streaming path — none of these surface `ICryptoTransform`. Reject patterns that construct `HMACSHA256` / `SHA256` instances, call `ComputeHash`, and discard the instance.
- `SHA256.HashData`, `SHA512.HashData`, `SHA3_256.HashData`, and `SHA3_512.HashData` are type-specific one-shots. `SHA3_256.IsSupported`, `SHA3_512.IsSupported`, and `SHAKE128.IsSupported` are static properties reflecting OS/hardware capability at runtime; calling any of these without a `IsSupported` guard is a latent `PlatformNotSupportedException` on downlevel hosts.
- `CryptographicOperations.FixedTimeEquals(ReadOnlySpan<byte>, ReadOnlySpan<byte>)` is the only correct constant-time comparison for secrets. A manual loop, `SequenceEqual`, or `==` is a timing oracle regardless of CPU branch-prediction behavior.
- `RandomNumberGenerator.Fill(Span<byte>)` fills a caller-owned buffer without allocation. `GetInt32(fromInclusive, toExclusive)` produces a uniformly distributed integer using rejection sampling internally. `GetItems<T>(ReadOnlySpan<T>, Span<T>)` samples with replacement and `Shuffle<T>(Span<T>)` shuffles in place — both cryptographically uniform.
- `AesGcm` offers both a key-only constructor and a `(key, tagSizeInBytes)` constructor. When constructed without `tagSizeInBytes`, `TagSizeInBytes` returns `null` and tag size is validated per-call rather than enforced at construction, widening the window for mismatched tag assumptions across call sites. Constructing with an explicit `tagSizeInBytes` locks the instance to one tag size. Nonce sizes are constrained to `NonceByteSizes` (12 bytes); tag sizes to `TagByteSizes` (12–16 bytes).
- `Base64Url` (`System.Buffers.Text`) encodes and decodes URL-safe base64 directly between `ReadOnlySpan<byte>` ↔ `Span<byte>` and `ReadOnlySpan<char>` ↔ `Span<byte>` with no padding ambiguity. `IsValid(ReadOnlySpan<byte>)` and `IsValid(ReadOnlySpan<char>)` check well-formedness with an optional `out int decodedLength`. Reject any hand-rolled base64url mapper or `Convert.ToBase64String` + string-replace chain.

[GENERATED REGEX]:
- `[GeneratedRegex(pattern, options, cultureName)]` targets `static partial` methods or `partial` properties returning `Regex` — `AttributeTargets.Method | AttributeTargets.Property`. The generator emits a `Regex`-derived type with state-machine match logic inlined as compiled C# — no `Regex` constructor call, no `RegexOptions.Compiled` JIT cost, no per-call lock on the shared cache.
- `Regex.EnumerateMatches(ReadOnlySpan<char>, pattern)` returns a `ValueMatchEnumerator` (ref struct, stack-allocated) that enumerates `ValueMatch` structs carrying `Index` and `Length` — no `Match` heap allocation, no `MatchCollection`. This is the correct path for scan-and-count or scan-and-slice workflows on spans.
- `Regex.Count(ReadOnlySpan<char>, pattern)` is the terminal aggregation; `IsMatch(ReadOnlySpan<char>, pattern)` is the presence test. Both accept `ReadOnlySpan<char>` directly.
- `[StringSyntax("Regex")]` marks `string` parameters with the regex syntax identifier; IDEs provide colorization and static analysis without runtime cost. Apply it to every method that accepts a pattern literal.
- `CompositeFormat.Parse(template)` pre-parses `{0}` / `{1:format}` templates once at construction; `string.Format(provider, compositeFormat, args)` accepts the result. `MinimumArgumentCount` is inspectable at construction time, moving format-string validation out of call-time and into startup. Reject runtime `string.Format(string, ...)` when the template is fixed.

[UTF-8 TEXT PIPELINES]:
- `"literal"u8` creates a `ReadOnlySpan<byte>` constant in the assembly's read-only data segment — UTF-8 encoded at compile time, zero runtime allocation, zero encoding call. Reject `Encoding.UTF8.GetBytes("...")` for string literals.
- `Utf8.TryWrite(Span<byte> destination, ref TryWriteInterpolatedStringHandler handler, out int bytesWritten)` formats an interpolated string directly as UTF-8 bytes with no intermediate string. The handler type is compiler-lowered from `$"..."` syntax. The overload accepting `IFormatProvider` forwards culture context into each `IUtf8SpanFormattable` hole.
- `Utf8.FromUtf16(ReadOnlySpan<char>, Span<byte>, out int charsRead, out int bytesWritten, bool replaceInvalidSequences, bool isFinalBlock)` returns `OperationStatus` for chunked conversion; `ToUtf16` is the reverse. These are the correct primitives for protocol transcoding — reject `Encoding.UTF8.GetBytes(chars)` for non-trivial volumes.
- `Encoding.TryGetBytes(ReadOnlySpan<char>, Span<byte>, out int bytesWritten)` and `TryGetChars(ReadOnlySpan<byte>, Span<char>, out int charsWritten)` are the non-throwing span variants on all `Encoding` instances. Prefer over `GetBytes(ReadOnlySpan<char>, Span<byte>)` when destination sizing is uncertain; they return `false` rather than throwing `ArgumentException`.
- `IUtf8SpanFormattable.TryFormat(Span<byte>, out int bytesWritten, ReadOnlySpan<char> format, IFormatProvider?)` is the UTF-8 output interface all numeric primitives implement via `INumber<T>`. `IUtf8SpanParsable<TSelf>.Parse(ReadOnlySpan<byte>, IFormatProvider?)` / `TryParse` is the symmetric input interface. Generic code constrained to `where T : IUtf8SpanFormattable` or `IUtf8SpanParsable<T>` handles any conforming type — all BCL numerics, `Guid`, `DateTimeOffset`, `TimeSpan`, and `Version`.

[GENERIC MATH AND PARSING INTERFACES]:
- `INumber<TSelf>` subsumes `ISpanParsable<TSelf>`, `ISpanFormattable`, `IUtf8SpanParsable<TSelf>`, `IUtf8SpanFormattable`, all arithmetic operator interfaces, and `IComparable<TSelf>`. A generic constrained to `INumber<T>` gets the full parse/format/arithmetic surface; constraining separately to `ISpanParsable<T>` when arithmetic is also required is a partial constraint that forecloses later operator use.
- `INumberBase<TSelf>.CreateChecked<TOther>(TOther)`, `CreateSaturating<TOther>`, and `CreateTruncating<TOther>` are `static virtual` default-interface methods for type-preserving numeric conversion — replacing per-type casts or `Convert.ToInt32(value)` chains. `CreateChecked` throws on overflow; `CreateSaturating` clamps; `CreateTruncating` wraps. The `TOther` constraint is `INumberBase<TOther>`, so the surface covers any BCL or domain numeric type without special-casing.
- `ISpanParsable<TSelf>.TryParse(ReadOnlySpan<char>, IFormatProvider?, out TSelf)` is the canonical parse entry for span input. A single generic call replaces per-type `int.TryParse(span, ...)`, `double.TryParse(span, ...)`, etc. `IUtf8SpanParsable<TSelf>` is independent from `ISpanParsable` — not a subtype — so UTF-8 byte paths require a separate generic constraint or a combined constraint alongside `ISpanParsable<T>`.
- `MemoryExtensions.Split<T>(ReadOnlySpan<T>, T)` and `SplitAny<T>(ReadOnlySpan<T>, SearchValues<T>)` return `SpanSplitEnumerator<T>` — a ref struct enumerator yielding `Range` values. `Split(ReadOnlySpan<char>, Span<Range>, char, StringSplitOptions)` writes ranges into a caller-supplied buffer and returns the count. All are allocation-free tokenizers replacing `string.Split()` for parsing over spans.
- `SearchValues<T>.Create(params ReadOnlySpan<T>)` produces a pre-built membership set used by `ContainsAny`, `IndexOfAny`, `IndexOfAnyExcept`, and `ContainsAnyExcept` on `ReadOnlySpan<T>`. `SearchValues<string>.Create(ReadOnlySpan<string>, StringComparison)` handles multi-string search. Construct once and store as `static readonly`; the internal representation selects the optimal strategy — bitmap, SIMD Teddy, or Aho-Corasick — based on the value set at construction time.

[CHANNELS AS PRODUCER-CONSUMER SEAM]:
- `Channel.CreateBounded<T>(BoundedChannelOptions)` and `CreateUnbounded<T>()` are the standard constructors. `BoundedChannelOptions` carries `Capacity` and `FullMode` (`Wait`, `DropNewest`, `DropOldest`, `DropWrite`); the `itemDropped: Action<T>` overload on `CreateBounded` receives dropped items for logging or metrics without polling. `Channel.CreateUnboundedPrioritized<T>(UnboundedPrioritizedChannelOptions<T>)` is a distinct constructor for priority-ordered consumption; the options type carries an `IComparer<T>`.
- `ChannelWriter<T>.TryWrite(T)` is synchronous and non-blocking; `WriteAsync(T, CancellationToken)` returns `ValueTask` and respects backpressure when `FullMode = Wait`. Calling `Complete(Exception?)` signals end-of-data; downstream `ReadAllAsync` terminates naturally. `ChannelReader<T>.ReadAllAsync(CancellationToken)` returns `IAsyncEnumerable<T>` composable with `System.Linq.AsyncEnumerable`; `TryRead(out T)` is the non-blocking pull. The channel replaces `BlockingCollection<T>` and manual `Task` / `SemaphoreSlim` producer-consumer loops.

[LINQ ADDITIONS]:
- `Enumerable.CountBy<TSource, TKey>(source, keySelector, comparer?)` returns `IEnumerable<KeyValuePair<TKey, int>>` — a streaming frequency map without materializing a dictionary. `AggregateBy<TSource, TKey, TAccumulate>(source, keySelector, seed|seedSelector, func, comparer?)` generalizes to arbitrary per-key accumulators. Reject `GroupBy(...).Select(g => (g.Key, g.Count()))` when these suffice.
- `Enumerable.Index<TSource>(source)` yields `(int Index, TSource Item)` tuples — canonical replacement for manual counter variables in `foreach`.
- `Enumerable.LeftJoin` and `RightJoin` are relational join overloads with nullable inner or outer result selectors; they replace the left-join idiom of `GroupJoin` + `SelectMany` + `DefaultIfEmpty`.
- `System.Linq.AsyncEnumerable` is part of the BCL — no external package required. It provides the full synchronous LINQ surface transposed to `IAsyncEnumerable<T>`: `Where`, `Select`, `OrderBy`, `AggregateBy`, `CountBy`, `Index`, and the rest use the same names as their synchronous counterparts. Each operator offers both a synchronous delegate overload (`Func<T, TResult>`) and an async overload (`Func<T, CancellationToken, ValueTask<TResult>>`); the async overload is preferred when the selector performs IO. The `WhereAwait` / `SelectAwait` naming belongs to the NuGet package `System.Linq.Async`, not the BCL assembly.

[DICTIONARY ALTERNATE LOOKUP]:
- `Dictionary<TKey, TValue>.GetAlternateLookup<TAlternateKey>()` returns an `AlternateLookup<TAlternateKey>` value type when the dictionary's comparer implements `IAlternateEqualityComparer<TAlternateKey, TKey>`; otherwise throws. `TryGetAlternateLookup<TAlternateKey>(out AlternateLookup<TAlternateKey>)` is the safe non-throwing form.
- `AlternateLookup<TAlternateKey>` exposes `TryGetValue`, `ContainsKey`, `TryAdd`, and `Remove` — all keyed by the alternate type. The `where TAlternateKey : allows ref struct` constraint means `ReadOnlySpan<char>` is a valid alternate key against a `Dictionary<string, T>` when the comparer is a `StringComparer.Ordinal`-family comparer; no string allocation occurs on lookup. `IAlternateEqualityComparer<TAlternate, T>` requires `Equals(TAlternate, T)`, `GetHashCode(TAlternate)`, and `Create(TAlternate)` — the `Create` method enables `TryAdd` to materialize the canonical key from the alternate, making the interface load-bearing beyond pure lookup.
- `HashSet<T>` carries the same `GetAlternateLookup<TAlternate>` / `TryGetAlternateLookup` surface with the same ref-struct allowance, providing allocation-free span-keyed membership tests against a string-keyed set.
