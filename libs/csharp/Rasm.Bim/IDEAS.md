# [RASM_BIM_IDEAS]

The forward pool of higher-order concepts for the host-neutral BIM-and-exchange engine. `[1]-[OPEN]` holds active ideas as cards; `[2]-[CLOSED]` records a finished or dropped idea with a one-line disposition.

## [1]-[OPEN]

[IFC_VOCABULARY_BREADTH]: the `model/elements#ELEMENT_MODEL` vocabulary is the canonical thin-subset defect on three axes of one concept — the IFC product type vocabulary — and the recovery is one owner widened on all three, never three parallel surfaces. (a) ENTITY-CLASS: the `IfcClass` `[SmartEnum<string>]` carries 12 entity rows (Wall/Slab/Column/Beam/Door/Window/Space/Proxy plus four infrastructure facilities) against the IFC4.3 schema's 800-plus rooted entities, so every real building/MEP/structural/infrastructure element projects onto `IfcBuildingElementProxy`; the breadth lands as one `IfcDomain`-discriminated row table — the architectural built-element family (`IfcCovering`/`IfcCurtainWall`/`IfcRailing`/`IfcRamp`/`IfcRoof`/`IfcStair`/`IfcPlate`/`IfcMember`/`IfcFooting`/`IfcPile`), the MEP distribution domain (`IfcFlowSegment`/`IfcFlowFitting`/`IfcFlowTerminal`/`IfcFlowController`/`IfcDistributionPort`), the structural-analysis domain (`IfcStructuralCurveMember`/`IfcStructuralSurfaceMember`/`IfcStructuralPointConnection`), and the IFC4.3 infrastructure/earthworks/geotechnics breadth the `.api/api-geometrygym-ifc` catalogue already enumerates (`IfcCourse`/`IfcEarthworksFill`/`IfcEarthworksCut`/`IfcGeotechnicalStratum`/`IfcBorehole`/`IfcRail`/`IfcAlignment`) — and the `IfcDomain` enum widens from three cases to the buildingSMART partition (Architecture/Structural/HvacFire/Electrical/Plumbing/Infrastructure/Geotechnical) so `query/element-set#ELEMENT_SET` discriminates by domain without a per-element type. (b) PREDEFINED-TYPE: the same thin-subset defect one level down — every IFC element class carries an `IfcXxxTypeEnum` `PredefinedType` (a wall is `STANDARD`/`SHEAR`/`PARTITIONING`/`SOLIDWALL`, a slab `FLOOR`/`ROOF`/`LANDING`/`BASESLAB`, a flow terminal `AIRTERMINAL`/`SANITARYTERMINAL`/`LIGHTFIXTURE`) and `BimElement` carries no predefined-type discriminant, so a "load-bearing wall" or "exterior door" query cannot filter what the schema models as first-class; a `PredefinedType` `[ValueObject<string>]` column on `BimElement` split through the catalogued `ParserIfc.IdentifyIfcClass(name, out predefinedConstant)`, discriminated by the `IfcClass` row's valid-predefined set with the `ELEMENTUSERDEFINED`/`OBJECTTYPE` user-defined fallback resolving the `ObjectType` string, so `query/element-set#ELEMENT_SET` gains a `ByPredefinedType` arm filtering within a class. (c) TYPE-OCCURRENCE: the `exchange/import-rail#IMPORT_RAIL` flattens every type object onto the thin `IfcSemanticModel.TypeRow(GlobalId, EntityType, Name)`, discarding the `IfcTypeProduct` representation-map library, the type's own `PredefinedType`, and the type-driven property/material inheritance the `properties/property-sets#PROPERTY_SETS` `QTO_TYPEDRIVENOVERRIDE` fold already half-reads; a `BimType` record carrying the `IfcElementType`/`IfcTypeProduct` discriminant, the type-bound `PropertySet`/material/`PredefinedType`, and the `IfcRepresentationMap` instanced-geometry library an `IfcMappedItem` occurrence references, so a 4000-identical-window model carries one `BimType` plus 4000 occurrence references rather than 4000 inlined geometries. FRONTIER RE-SEED — recovers the IFC product-type breadth the folder intent names as never-lose, one vocabulary owner on three axes, never three parallel cards.

A wall's layer build-up, a beam's section material, and a façade's constituent mix carry their real composition, never a single material name. FRONTIER RE-SEED + trickle-down realization of `csharp:OPEN_MATERIAL_STANDARD_WIRE`.

[GEODETIC_CRS_TRANSFORM]: the `georeferencing/coordinate-reference#GEO_REFERENCE` `ToTransform` hand-rolls the map-conversion as a `Math.Atan2` rotation plus translation plus uniform scale over the kernel transform algebra and carries the EPSG identity as a bare `ProjectedCrs` string with no actual datum reprojection — a model in EPSG:27700 (British National Grid) and a model in EPSG:25832 (ETRS89/UTM32N) federate onto one frame only by their map-conversion offsets, never by a true geodetic transform between datums. The georeferencing owner gains a real coordinate-transformation engine over the admitted `ProjNET` (catalogued at `.api/api-projnet`): the EPSG-keyed `ProjectedCrs` resolves a `CoordinateSystemServices.GetCoordinateSystem(srid)`, the `CoordinateSystemServices.CreateTransformation(sourceSrid, targetSrid)` builds the `ICoordinateTransformation`, and the `MathTransform.Transform` applies the Bursa-Wolf 7-parameter datum shift plus the projection so a multi-CRS federation reconciles every model onto the shared projected frame by its full geodetic transform (ellipsoid, datum shift, projection) rather than only the rigid map-conversion offset. The map-conversion stays the local-engineering-frame placement the kernel `Rasm` transform owns; the CRS reprojection is the new datum-bridging leg the IFC4.3 `IfcCoordinateOperation`/`IfcProjectedCRS` chain demands, composed once on the `GeoReference` so a model lands in the shared datum before the rigid offset applies. FRONTIER RE-SEED + ADMIT (`ProjNET` 2.1.0, MIT, the NetTopologySuite-ecosystem peer of the already-admitted `Npgsql.NetTopologySuite`) — the owner-to-extend-first is `georeferencing`, never a new transform owner.
