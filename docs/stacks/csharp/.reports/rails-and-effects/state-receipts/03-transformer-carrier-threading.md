# Carrier-Polymorphic Transformer Threading And Traversal Accumulation

[EFFECT_IS_A_TRANSFORMER]:
- The dependency-injecting effect monad is not a bespoke type — it is one record holding a single reader-transformer-over-`IO` field, `ReaderT<RT, IO, A>`. Every effect capability follows from that identity: the runtime `RT` is the reader environment, the captured side effect and exception lift come from the inner `IO`, and the effect's `Bind`/`Map`/`liftIO` are the transformer's. No state cell, no atom, no ambient slot participates in normal effect threading — the environment rides the reader function argument and never mutates.
- Because the effect is literally a reader transformer, lifting raw work into it is the transformer's `liftIO`: a plain `IO<A>` becomes `env => ioValue` — the environment is read and discarded, the inner effect runs unchanged. A runtime-reading effect is `env => M.Pure(f(env))`; a runtime-dependent async effect wraps a `Func<RT, Task<A>>` through the inner `IO`. The four constructor shapes (`Func<RT,A>`, `Func<RT,Fin<A>>`, `Func<RT,Either<E,A>>`, `Func<RT,Task<A>>`) all collapse to the same reader-of-`IO` projection; they are admission forms, not parallel monads.
- Reader scoping of the runtime is `local`: pre-compose an `RT -> RT` transform onto the wrapped run so the modified runtime is visible only inside the scoped sub-effect and reverts at the boundary. This is the scoping primitive for swapping a capability — clock, logger handle, cancellation budget — for one sub-tree without a new evaluation or a mutable cell; the transform is a pure environment rewrite, not a write.

[STATE_THREADS_THROUGH_THE_FUNCTION]:
- The state transformer wraps `Func<S, K<M, (A Value, S State)>>` — value first, state second in the returned tuple. State is never a field and never a cell; it is the argument threaded into the run and the second tuple element threaded out. `Bind` is the entire mechanism: run with `state`, pull `(value, State')` out of the inner monad, feed `State'` into the continuation's run.

```csharp
new StateT<S, M, B>(state =>
    M.Bind(runState(state),
        x => f(x.Value).runState(x.State)));   // State' carried forward, value handed to f
```

- This threading is strictly left-to-right and builds a left-nested tower of inner-`Bind` closures forced only at the final run, so a long fold accumulates one closure per step — the state-thread cost is closure nesting, not allocation of intermediate states. `Map` short-circuits the state untouched (`(f(x.Value), x.State)`); `Pure` injects through the inner monad and pairs the incoming state unchanged. A receipt accumulated as the threaded `S` therefore never lives in a mutable accumulator: each step's run returns the advanced receipt, and the consumer reads it as the second tuple element of the top-level run — purely functional accumulation with no atom and no spin loop.
- The transformer's state-effect vocabulary is a trait, not methods on the type: `Get` derives from `Gets(identity)`, while `Put`, `Modify`, and `Gets` are the abstract surface each carrier implements. One declared correspondence (`Gets`) yields `Get` for free — the read-the-whole-state form is the identity projection of the read-and-map form, never a second primitive.

[WRITER_IS_A_DIFFERENCE_LIST]:
- The log transformer wraps `Func<W, K<M, (A Value, W Output)>>` under a `Monoid<W>` constraint and threads the accumulator *in* as an argument rather than returning a fresh log per step. The run is seeded with the monoid identity; every `Bind` feeds the first computation's accumulated `Output` as the input accumulator of the second. This difference-list shape is why the writer does not suffer the quadratic left-nested combine of the naive log-returning writer — the combine is one threaded parameter, appended on the right.
- `tell` is `w => M.Pure((unit, w + item))`: the existing accumulator is the left operand, the emitted fact the right — append order is fixed and right-biased, so a fold of receipt emissions lands in emission order. `listen`/`pass`/`censor` each call the seeded run and re-prepend the prior accumulator, so nesting them inside a tight loop reconstructs the monoid combine per iteration — the cheap form is one outer `listen`/`censor` over the whole fold, never one per step. A receipt stream expressed as the writer's `W` is therefore a monoid append threaded through the effect with no cell and no observer subscription — the fact stream is the transformer output, projected by the consumer's fold over the final `Output`.

[THREE_CHANNELS_ONE_FUNCTION]:
- The fused reader-writer-state transformer collapses all three threadings into one wrapped function `Func<(R Env, W Output, S State), K<M, (A Value, W Output, S State)>>` under `Monoid<W>` and an inner monad. One `Bind` threads every channel in its correct discipline simultaneously — the environment held constant, the log and state advanced — proving the three are not three composed transformers but one closed dispatch over a triple.

```csharp
new RWST<R, W, S, M, B>(input =>
    ma.runRWST(input).Bind(x =>
        f(x.Value).runRWST((input.Env, x.Output, x.State))));   // Env constant, Output+State advanced
```

- The constant-`Env` / advanced-`(Output,State)` split is visible in that one continuation call: `input.Env` is re-supplied unchanged while `x.Output` and `x.State` carry forward. This is the canonical shape for an effect that reads a runtime, emits a receipt log, and threads accumulating algorithm state in one pass — replacing three parallel cells and three observer wirings with one function and one fold over the final triple. `Modify`/`Gets`/`Censor`/`Local` are case projections of this single carrier, not sibling surfaces.
- Stack-safe iteration is deliberately *not* `Bind` recursion: each transformer's recursion driver delegates to the inner monad's trampoline with a local step tagged `Loop`/`Done`, and an unexpected tag is a hard `NotSupportedException`, not a silent stall. A custom inner monad stacked underneath must honor that loop contract or deep transformer iteration surfaces the exception from inside the trampoline rather than overflowing the stack.

[TRAVERSE_SPLITS_ON_THE_CONSTRAINT]:
- One traversal trait carries both accumulating and short-circuiting forms, distinguished only by the carrier constraint: the applicative form requires `Applicative<F>` and composes via chained `Apply`; the monadic form requires the strictly stronger `Monad<M>` and composes via `Bind`. The applicative form visits and combines every element's effect; the monadic form sequences and stops at the first failure. Selecting between them is selecting the failure algebra of the whole traversal, encoded purely by which entry point is reached for.
- The split is real only where a carrier overrides both: the accumulating validation carrier's `Apply` combines two failures through the failure semigroup, while its `Bind` returns the first failure and discards the rest. Traversing it accumulates every error; binding it short-circuits. The trap is that the monadic-traverse *default* forwards to the applicative path — a carrier that wants genuine monadic short-circuit under the monadic-traverse name must override it, or the two entry points behave identically and accumulate. Confirm the override before assuming fail-fast.

```csharp
// accumulating: every failure combined through the semigroup
mf.ApplyI(ma, semigroup);                 // both-fail -> errors.Combine(other)
// short-circuit: first failure wins, rest discarded
ma is Fail x ? FailI<B>(x.Value) : f(s.Value);
```

- The applicative path runs through a single lift primitive, `F.Pure(f).Apply(fa).Apply(fb)` — never `Bind` — so the independence of the effects is structural: nothing in the applicative traversal can make a later element depend on an earlier element's value, which is exactly what licenses the parallel/accumulating reading. The monadic path's `Bind` is what introduces the dependency and therefore the short-circuit. This is the same accumulate-versus-fail-fast axis that governs validation combination, surfaced one level up at the traversal.

[K_ERASURE_AND_REANCHORING]:
- Every trait-level combinator — `Bind`, `Map`, `Apply`, `Traverse`, `Sequence` — returns the erased higher-kinded surface `K<F, A>`, because the language has no return-type higher-kinded polymorphism. The concrete carrier API (`Match`, deconstruction, case predicates) is unreachable on that erased value; a per-carrier `As()` re-anchors it by direct reference cast back to the nominal type. There is no universal erasure-removing cast and there cannot be — each carrier owns its own `As()`, and the cast is mandatory after any trait-surface composition, not optional sugar.
- `Sequence` is `Traverse(identity)` with the inner type fixed to the effectful element, so its applicative constraint, not the traversal body, is the first error a non-applicative inner carrier produces — an opaque constraint failure at the call before the traversal ever runs. The monadic-sequence form exists for inner carriers that genuinely lack a standalone applicative and accept sequential semantics; reaching for it is an explicit acceptance of fail-fast, not a fallback.
- Nested-carrier composition uses a distinct lift/lower pair beyond single-level `As()`: one combinator raises `K<M, child<A>>` to `K<M, K<child, A>>` through the outer `Map`, its inverse lowers back through the outer `Map`. These are what make `...T`-suffixed combinators thread a value through two stacked carriers without a mid-expression manual cast, and a resilient variant rebuilds via `Map(identity)` instead of throwing when a manually-constructed `K<>` value is genuinely foreign.

[SCHEDULE_IS_A_LAZY_DURATION_STREAM]:
- The retry/repeat policy is one abstract record whose sole core method yields a lazy `Iterable<Duration>` — there is no step-state object, no context, no fold record. Every policy is a deferred duration stream, and every backoff derives from one primary generator (an infinite stream of zero delays) mapped to a delay: spacing maps the zero to a constant, exponential maps it by index. The two policies whose term depends on the prior term — linear and fibonacci — are the only bespoke stateful iterators, because a pure index-map cannot express a recurrence. One correspondence generates the family; the recurrences are the named exception.
- Composition is stream algebra, and the operators form distinct lattices that must not be conflated: union takes the elementwise minimum and runs to the longer operand; intersection takes the elementwise maximum and truncates to the shorter; semigroup append concatenates. Bounding an infinite backoff is therefore an intersection with a finite count or a cumulative-delay gate — union will run to the longer stream and not bound it. The policy is a semigroup under append but carries no identity, so there is no empty policy; the never and forever values are convenience generators, not algebraic units.
- Decorators are transformers over the stream: a max-delay cap maps each duration through a clamp, jitter maps each through a random perturbation, a cumulative gate is a stateful iterator that stops once the running sum crosses a bound. These compose as function composition under a transformer monoid with the identity transform as unit, distinct from the policy's own append. A wall-clock deadline is the odd member — a standalone generator yielding zero delays while now-minus-start stays under the bound, not a decorator over an existing policy.
- The effect-rail driver prepends a zero delay so the first attempt fires immediately, then walks the lazy iterator as a cons-list: each step delays for the head then re-runs the bracketed effect, and stream exhaustion is the only non-error termination. A zero delay short-circuits to a no-op with no thread hop; a non-zero delay becomes a cancellation-observing async wait keyed on the effect's token, so a cancelled token cuts the loop between attempts without an explicit predicate. The recursion is the loop, trampolined inside the effect interpreter — an unbounded policy drives unbounded iteration without a stack overflow, which is exactly why an infinite backoff on the rail must be intersected with a bound rather than relied on to stop itself.
