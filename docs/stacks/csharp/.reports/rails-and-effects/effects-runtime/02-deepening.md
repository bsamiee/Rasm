# Effects-Runtime: Deepening

[IO_DSL_INTERPRETER]:
- The runtime is a trampoline, not a call stack. `IO<A>.RunAsync()` opens a synchronous loop over a `Seq<StackItem>` catch-handler stack; each iteration pattern-matches the current `IO<A>` node against `InvokeSync<A>`, `InvokeSyncIO<A>`, `InvokeAsyncIO<A>`, `IOCatch<A>`, `IOCatchPop<A>`, and `IOTail<A>`. Only when the node is `InvokeAsync<A>` ‚Äî carrying an unfinished `ValueTask` ‚Äî does the loop yield to `RunAsyncInternal`, which is an `async`/`await` state machine. This two-phase design avoids the state-machine allocation for fully-synchronous computation graphs.
- `IOTail<A>` is the trampoline step: it holds the next `IO<A>` and overrides `Map` and `Bind` with `NotSupportedException`. When the loop encounters `IOTail<A>` it replaces the current node with `iOTail.Tail` and continues without advancing a continuation; `.Map` and `.Bind` on a tail node throw at definition time, not at run time, making illegal chaining a build error, not a runtime surprise.
- `tail<A>(IO<A>)` from `Prelude` is the public constructor for `IOTail<A>`. `IOTail<A>.resolve<B,C>` is the select-partner: it unwraps a chained `IOTail<B>` into its `Tail` for the `B == C` case, and falls back to `Map` otherwise. The constraint `typeof(B) == typeof(C)` is a runtime equality check at the resolve site.
- `IOCatch<X,A>` embeds `Operation`, `Predicate`, `Failure`, `Final?`, and `Next` as one record node. The loop pushes a `StackItem(MakeHandler())` onto the catch stack before running `MakeOperation()`; `IOCatchPop<A>` is the matching tear-down node emitted after the happy path completes, which pops one handler off the stack without any try/finally around the operation itself.
- Extending `IO<A>` by subclassing directly causes `Errors.IODSLExtension` to be thrown at run time; the only legal extension points are `InvokeSync<A>`, `InvokeAsyncIO<A>`, `InvokeSyncIO<A>`, and `InvokeAsync<A>`. The `InvokeSyncIO<A>` variant returns `IO<A>` from `Invoke`, enabling the node to chain more DSL nodes as its return value; `InvokeSync<A>` returns the raw `A`, terminating that step.

[ENVIO_OWNERSHIP_AND_DISPOSAL]:
- `EnvIO` tracks two orthogonal ownership bits via the `Own` int: bit `1` for the `CancellationTokenSource` (owned when created internally), bit `2` for the `Resources` object (owned when created internally). `Dispose()` calls `DisposeResources()` then `DisposeNonResources()` in order. Each half uses `Interlocked.CompareExchange` to guarantee at-most-once disposal regardless of concurrent callers.
- `EnvIO.Local` is `New(null, Token, null, SynchronizationContext.Current)` ‚Äî both source and resources are null, so both are created fresh and both ownership bits are set. The new CTS links to the parent token via a registration. The sync context is captured from `SynchronizationContext.Current` at construction time, not at `Post()` call time. `EnvIO.LocalResources` is `New(null, Token, Source, SyncContext)` ‚Äî creates fresh resources (bit 2 owned) but reuses the parent source (bit 1 not owned); this is the variant used by `BracketFail()` so retry scopes share the same cancellation axis.
- `EnvIO.LocalCancelWithTimeout(TimeSpan)` is the mechanism behind `IO<A>.Timeout(TimeSpan)` via `IOTimeout<A,B>`: it creates a new CTS with the timeout, registers the parent token to cancel it, and the `using` on the local `EnvIO` in `IOTimeout.Invoke` disposes both the CTS and the registration once the operation completes.
- `EnvIO.LocalSyncContext` preserves all existing scope fields but re-captures `SynchronizationContext.Current` from the calling thread. Used to re-bind context after crossing a thread boundary before calling `IO.Post()`.
- When `EnvIO.New` receives a non-cancellable parent token, the registration is null and no linkage is established, eliminating the allocation. The `Own` bit for the source is still set, so `DisposeNonResources` cleans up the source regardless.

[RESOURCE_SCOPE_MECHANICS]:
- `Resources` holds an `AtomHashMap<object, TrackedResource>` (MVCC-safe). `Acquire<A>` calls `TryAdd` with the handle as the key ‚Äî if the same object reference is acquired twice, the second call is silently dropped, making acquire idempotent for identity-equal handles. `Release<A>` removes the key and runs the release IO; if not found locally it delegates to `parent`, enabling scope nesting where a child scope can release a handle registered in a parent.
- `IOLocal<X,A>` (the implementation of `IO<A>.Bracket()`) creates a local env via `MapEnvIO`, awaits the operation inside a `using` statement, then disposes the local env on exit ‚Äî both resource and non-resource cleanup run. `IOLocalOnFailOnly<X,A>` (the implementation of `IO<A>.BracketFail()`) only calls `local.DisposeResources()` in the `catch` block; on the success path, `local.DisposeNonResources()` runs in `finally` but the resource scope is left live, intentionally propagating acquired handles to the enclosing scope.
- `IOUseDisposable<X,A>.Invoke` checks whether the acquire `ValueTask` is already completed: synchronous acquires register into `env.Resources` immediately and return the next node; asynchronous acquires return `IOAcquireDisposableAsync<X,A>`, which registers the handle only after the `ValueTask` resolves. This eliminates the window between acquisition completion and registration where an exception would cause a leak.
- `Resources.DisposeU` creates a fresh CTS before iterating and uses `CancellationToken.None` in the disposal env, so resource release is uninterruptible by design ‚Äî cancellation of the outer scope does not abort cleanup.

[FINALLY_IMPLEMENTATION]:
- `IO<A>.Finally<X>(K<IO,X> finally)` constructs `IOFinal<X,A,A>`. `IOFinal.Invoke` sets a `flag = true` before starting the operation and clears it synchronously after `RunAsync` returns. If `RunAsync` throws before the flag clears, the `finally` block runs `Final.As().Run(envIO)` synchronously. If `RunAsync` returns a completed task, `Finally` is sequenced as `Final.Bind(_ => Next(result))` ‚Äî the finally IO runs before the continuation. If the task is incomplete, `IOFinalAsync` handles the await and similarly sequences finally before continuing.
- The `|` operator between `K<F,A>` and `Finally<F,X>` is defined in a C#-extension block: `(K<F,A> lhs, Finally<F,X> rhs) => F.Finally(lhs, rhs.Operation)`. This means `finalEffect | final(cleanup)` is the expression-spine spelling, avoiding method-call noise at the boundary.
- `IOCatch<X,A>` carries an optional `Final` field. When non-null, it is appended to both the success and failure paths via `Finally` inside `MakeHandler` and `MakeOperation`. The four-argument `Bracket(Use, Catch, Fin)` form on `IO<A>` assembles this in a single `Bind` expression rather than nesting two separate `IOCatch` nodes.

[UNINTERRUPTIBLE_SEMANTICS]:
- `IO<A>.Uninterruptible()` constructs `IOUninterruptible<A,A>`. `Invoke` creates a new `EnvIO` with `CancellationToken.None` ‚Äî the token is completely replaced, not linked. The source is `null` so a fresh CTS is created internally, making the scope unowned from a lifecycle perspective. Async completion falls through to `IOUninterruptibleAsync` which `await`s then disposes `LocalEnv` immediately after the `await`; only the resource scope survives.
- The uninterruptible scope does not isolate resource tracking: it passes `envIO.Resources` (from the parent) into the local env. This means resources acquired inside an uninterruptible block register in the parent scope and are released when the parent bracket exits, not when the uninterruptible scope does. Composing `uninterruptible` with `use` requires an enclosing `Bracket()` to own the resource lifetime.
- `uninterruptible<M,A>(K<M,A> ma)` on `MonadUnliftIO<M>` is implemented as `MapIO(ma, io => io.Uninterruptible())`. This means the uninterruptible wrapping operates at the `IO` layer regardless of the outer carrier ‚Äî `Eff<RT,A>` uses the same `IOUninterruptible` node.

[FORK_EXECUTION_MODEL]:
- `IO<A>.Fork(Option<TimeSpan> timeout)` launches a `Task.Factory.StartNew` with `TaskCreationOptions.LongRunning` ‚Äî it bypasses the thread pool and requests a dedicated thread from the OS. The forked computation runs inside a child `Resources` scope parented to `env.Resources`, so acquired handles in the fork register locally and are released when the fork completes. The parent resources are not affected by fork disposal.
- If a `timeout` is `Some`, a `CancellationTokenSource(timeout.Value)` is created; otherwise a plain CTS. A registration links the parent token to this source so parent cancellation propagates. Both the CTS and the registration are wrapped in a `CleanUp` disposable that is disposed in the `finally` of the forked task body.
- `ForkIO<A>` is a `readonly record struct` with two fields: `Cancel: IO<Unit>` and `Await: IO<A>`. `Cancel` fires `tsrc.Cancel()` swallowing `OperationCanceledException` and `ObjectDisposedException`. `Await` uses `AwaitAsync` which awaits the task while also observing the calling environment's cancellation token ‚Äî if the caller cancels before the fork finishes, `Await` propagates the caller's cancellation, not the fork's.
- `awaitAll(Seq<ForkIO<A>>)` uses `Task.WhenAll` over `f.Await.RunAsync(eio).AsTask()` projections ‚Äî each fork's `Await` IO runs with the outer `EnvIO`, meaning outer cancellation cancels all awaits simultaneously. `awaitAny` uses `Task.WhenAny`, picks the winner, then returns it; losers are neither cancelled nor awaited.

[SCHEDULE_INTERNALS]:
- `Schedule.Run()` returns `Iterable<Duration>`, not `IEnumerable<Duration>`. The `Iterable<Duration>` is a lazy pull sequence with a `GetIterator()` method; retry/repeat drives it via `schedule.PrependZero.Run().GetIterator()`, calling `MoveNext()` once per attempt. `PrependZero` injects a `Duration.Zero` as the first step so the initial attempt fires without delay.
- `Schedule.Never` is `SchNever.Default` ‚Äî its `Run()` returns an empty `Iterable<Duration>`. `Schedule.Once` is `Forever.Take(1)`. `Schedule.Forever` is `SchForever.Default` whose `Run()` yields `Duration.Zero` infinitely.
- `SchDecorrelate` iterates the base schedule and for each base duration emits `current + jitter‚ÇÅ` then `current - jitter‚ÇÇ`, where each jitter is an independent `SingletonRandom.Uniform(0, current * factor)` draw. The schedule produces twice as many steps as the base; `decorrelate` is an expansion transformer, not a per-step scalar.
- `Schedule.resetAfter(Duration max)` produces `SchResetAfter`, which drives `(s | maxCumulativeDelay(max)).Run()` in an outer `while (true)` loop ‚Äî once cumulative delay reaches `max`, the inner `Iterable<Duration>` exhausts and the outer loop restarts from the beginning of the original schedule. This produces an infinitely resetting bounded-window schedule.
- `Schedule.intersperse(Schedule inner)` is `Transform(s => s.Bind(inner.Prepend))` ‚Äî it inserts every duration from `inner` before each step of the outer schedule. The `params Duration[]` overload delegates to `intersperse(TimeSeries(durations))`, enabling inline duration literals as a variadic probe series between back-off steps.
- Calendar-aligned schedules ‚Äî `secondOfMinute(int)`, `minuteOfHour(int)`, `hourOfDay(int)`, `dayOfWeek(DayOfWeek)` ‚Äî accept an optional `Func<DateTime>? currentTimeFn` parameter. Passing a deterministic `currentTimeFn` makes these schedules testable without wall-clock coupling; the default is `() => DateTime.Now`.
- `Schedule.TimeSeries(params Duration[] durations)` constructs `SchItems` from an `Iterable<Duration>`. The `|` between two `Duration` values in a `Duration * factor` expression uses the `double` implicit conversion ‚Äî `Duration` stores milliseconds as `double` and all arithmetic operators work over `double`, converting back via `new Duration(double)`.

[DURATION_UNIT_SURFACE]:
- `Duration` is `readonly struct` over a `double Milliseconds` field. Implicit conversions exist from `double`, `TimeSpan`, and `Time`; explicit conversion from `Duration` to `TimeSpan`. Arithmetic operators (`+`, `-`, `*`, `/`, comparison) all operate on the underlying `double`.
- `Time` is `readonly struct` storing seconds as `double`. Extension methods `.Milliseconds()`, `.Seconds()`, `.Minutes()`, `.Hours()`, `.Days()` on `int`, `float`, and `double` produce `Time` values. Prelude constants `ms`, `seconds`, `minutes`, `hours`, `days` are pre-constructed `Time` values equal to `1` of each unit ‚Äî `10 * ms` produces `Time(0.010)` which converts implicitly to `Duration(10.0)`.
- `Duration.random(Duration min, Duration max, Option<int> seed)` is the public surface for `SingletonRandom.Uniform`. Seeding is optional; unseeded draws use `Random.Shared` internally. Using `Duration.random(...)` as the seed to a `Schedule.linear` de-correlates parallel retry storms without requiring `decorrelate()`.

[RUNTIME_CAPABILITY_AND_HAS]:
- `Has<M, VALUE>` is an `interface` with a single `static abstract K<M, VALUE> Ask { get; }`. The three-parameter static class `Has<M, Env, VALUE>` (where `Env : Has<M, VALUE>`) stores `Env.Ask` in a `public static readonly` field, amortizing static-virtual dispatch to a single initialization. Prefer `Has<Eff<RT>, RT, MyService>.ask` over calling `Env.Ask` at each bind site.
- `asks<Env,A>(Func<Env,A> f)` from `Prelude` constructs `Ask<Env,A>` ‚Äî a `readonly record struct` that exposes `ToReadable<M>()`, `ToReader()`, `ToReaderT<M>()`, and `SelectMany` for query-expression compatibility. Inside `Eff<RT,A>` pipelines, `asks<RT,Cap>(rt => rt.MyService)` is the idiomatic slot projection; the resulting `Ask<RT,Cap>` binds directly into `Eff<RT>` via `Readable<Eff<RT>, RT>.Asks`.

[MONAD_TRANSFORMER_IO_LIFT]:
- `MonadIO<M>.LiftIO<A>(IO<A> ma)` is `static abstract` ‚Äî every concrete `MonadIO` type must implement it. The default `LiftIO(K<IO,A>)` overload delegates to `LiftIO(IO<A>)` via `.As()`. For transformer stacks such as `OptionT<IO, A>`, `FinT<IO, A>`, and `EitherT<L, IO, A>`, `LiftIO` wraps the `IO` computation in the transformer's inner layer.
- `IO<A>.Try()` returns `FinT<IO, A>` ‚Äî it maps `Fin.Succ` over the result and catches via `Fin.Fail<A>`. The `FinT<IO, A>` wrapper is a lifted `IO<Fin<A>>`; running the inner `IO` returns `Fin<A>` directly without an outer `Fin` wrapping. Use `io.Try()` when the computation must run inside a transformer stack that accumulates outcomes without re-wrapping failure.
- `IO.SelectMany` extension block defines overloads from `K<IO,A>` into `OptionT<M,B>`, `EitherT<L,M,B>`, `FinT<M,B>`, and `ValidationT<F,M,B>` via `OptionT.liftIO<M,A>(ma).SelectMany(...)`. This enables mixed-carrier query expressions where an `IO` step binds directly into a transformer query without an explicit `.LiftIO` call.

[MAPIO_AND_TOID_USES]:
- `mapIO<M,A,B>(K<M,A> ma, Func<IO<A>, IO<B>> f)` where `M : MonadUnliftIO<M>` is the canonical slot for injecting raw `IO` transformations into any carrier. Every `Eff` or transformer that implements `MonadUnliftIO` exposes its `IO<A>` layer via `ToIO`, applies `f`, and re-wraps. This is how `uninterruptible`, `timeout`, `fork`, `localIO`, and `postIO` are all carrier-polymorphic: each calls `MapIO` with the appropriate `IO<A> ‚Üí IO<B>` wrapper.
- `localIO<M,A>(K<M,A> ma)` calls `M.LocalIO(ma)` which uses `MapIO(ma, io => io.Local())` ‚Äî it wraps the operation in a fresh cancel scope. `postIO<M,A>(K<M,A> ma)` calls `M.PostIO(ma)` which uses `MapIO(ma, io => io.Post())` ‚Äî it marshals execution onto the captured `SynchronizationContext`. Both are expressed purely through `MapIO` so they compose with any `MonadUnliftIO` stack.
- `ToIO<M,A>(K<M,A> ma)` returns `K<M, IO<A>>` ‚Äî it embeds the `IO<A>` layer as a bound value inside the carrier. This is used in `awaitAny(Seq<K<M,ForkIO<A>>>)` to extract the `IO<ForkIO<A>>` for each fork and run them independently.

[CANCELLATION_TOKEN_ACCESS]:
- `IO.token` is `new IOToken<CancellationToken>(pure)` ‚Äî a DSL node subclassing `InvokeSyncIO<CancellationToken>` whose `Invoke` calls `Next(env.Token)`. It participates in fusion: `Map` and `Bind` produce `IOTokenMap` and `IOTokenBind` nodes that carry the `Next` continuation, deferring token access until run time.
- `IO.source` is `IO.lift((EnvIO e) => e.Source)` ‚Äî a plain `InvokeSync` lift, not an `IOToken` variant. It does not participate in the token-bind fusion chain. `cancelToken` in `Prelude` is identically `IO.lift(e => e.Token)`, also a plain lift. Prefer `IO.token` over `cancelToken` in tight loops where the fusion-node chain can eliminate continuations; `IO.source` has no fusion-friendly alternative.
- `MonadIO<M>` exposes `Token` and `TokenSource` as `static virtual` properties defaulting to `M.LiftIOMaybe(IO.token)` and `M.LiftIOMaybe(IO.source)`. Any `MonadIO`-constrained generic function can read the token without breaking carrier abstraction.

[IOEMPTY_AND_MONOIDAL_CHOICE]:
- `IOEmpty<A>` is `InvokeSync<A>` whose `Invoke` throws `Errors.None` ‚Äî not `null`, but the `Error.None` sentinel wrapped as an exception. This makes empty propagate through `Catch` handlers that filter on `Errors.None`, so `io | IO.empty<A>()` is `io` on success and `IO.empty<A>()` on error, the exact `SemigroupK` zero semantics. `IOEmpty` fuses into `Map`, `Bind`, `FoldWhile`, and `FoldUntil` via `IOEmptyAsync` without executing continuations.
- `IO<A>.Combine(IO<A> rhs)` is `Catch((Error _) => true, (Error _) => rhs)` ‚Äî `MonoidK` addition is recovery-on-any-error. This means `io1 | io2` (the operator spelling of `Combine`) uses `Choose` which is itself built on `Catch`. Stacking three alternatives `io1 | io2 | io3` builds a left-linear chain of `IOCatch` nodes, each inspecting the previous failure.

[FOLD_SCHEDULE_OVERLOAD_MATRIX]:
- `IO<A>.FoldWhile<S>` has six overloads discriminating on schedule presence and predicate projection: `(Schedule, S, folder, Func<S,bool>)`, `(S, folder, Func<S,bool>)`, `(Schedule, S, folder, Func<A,bool>)`, `(S, folder, Func<A,bool>)`, `(Schedule, S, folder, Func<(S,A),bool>)`, and `(S, folder, Func<(S,A),bool>)`. The schedule-absent forms use `Schedule.Forever` as the default, producing an infinite fold bounded only by the predicate. All six delegate to `IOFoldWhile<S,A,S>` with the predicate normalized to `Func<(S State, A Value), bool>`.
- `FoldUntil` mirrors `FoldWhile` with negated predicate semantics ‚Äî the fold continues while predicate is false. Both families are also exposed via `FoldWhileIO<M,S,A>` and `FoldUntilIO<M,S,A>` on `MonadUnliftIO<M>` extension methods, enabling carrier-polymorphic stateful repetition. The `IOFoldWhile` node carries the schedule's `IEnumerator<Duration>` obtained via `schedule.PrependZero.Run().GetEnumerator()` and advances it per iteration.
