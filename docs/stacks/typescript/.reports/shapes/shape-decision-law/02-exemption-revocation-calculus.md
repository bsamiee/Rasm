# Exemption Revocation Calculus

[REVOCATION_IS_ONE_PREDICATE]:
- Each exemption from the generated owner is held by exactly one structural fact, and forfeited by exactly one axis whose appearance the exemption-tier value cannot carry — so revocation is not four rules but one predicate parameterized by tier: an exemption survives iff every demanded facet is covered by the tier's structural surface, and the first demanded facet the tier lacks promotes the value into the owner one rung up. The predicate is the inverse of the demotion direction read forward: demotion strips a dead capability from an over-tiered owner, revocation grants a demanded capability the exemption-tier owner has no slot to hold.
- The four exemptions are the four floor tiers — the `as const` table (no `ast`, no instance), the bare brand (`Brand.Constructor`, no `ast`), the native primitive (no `ast`, no nominal witness), and the statement inside a measured kernel (no rail) — and each has a single forfeiting axis: the table loses to equality-or-codec-or-identifier-or-dependency, the bare brand loses to codec, the native primitive loses to nominal-collision-or-refinement, the kernel statement loses to a domain value that must carry a typed failure, a codec admission, or a per-occurrence payload. The axis names the rung; the rung is never chosen by line count or perceived complexity.
- The cost asymmetry that governs promotion governs revocation identically: an exemption held one rung too long is a fan-out the moment the demanded axis appears — every `keyof typeof` read site, every bare-primitive comparison, every un-railed kernel value must be re-touched — while an exemption surrendered one rung too early is a single unused capability on an owner the value never exercises. Revocation is therefore a definition-time correction caught before consumers bind, never a refactor of a live owner, and the demanded axis is read off the concept signature before the floor tier is even named.

[TABLE_REVOKES_ON_FOUR_INDEPENDENT_AXES]:
- The `as const` table holds its exemption by being decode-free, dependency-free, instance-free, and namespace-free simultaneously, so four independent axes each revoke it alone — equality-demand, codec-demand, dependency-edge, identifier-need — and the table never extends to absorb any of them, because a value read by `keyof typeof` over a frozen literal has no `ast` to carry a codec, no `[Equal.symbol]` to carry structural comparison, no `R` to carry a dependency, and no identifier annotation to carry a `$defs` key. Each axis re-routes to a different rung, and reading which axis appeared is the whole decision.
- Equality-demand revokes to `Data.struct` (or `Schema.Data` if the value also decodes), because the table key is a bare string and structural membership in a `HashMap`/`HashSet` keys by `[Equal.symbol]`/`[Hash.symbol]`: a table row deduplicated by structure is impossible — the lookup compares strings, never the row's fields. The promotion is not an extension of the table; the table has no instance to compare, so the demand crosses owner families to the lowest tier that mints an `Equal`-bearing record.
- Codec-demand revokes to `Schema.Class` (or `Schema.Literal` if the keys carry no behavior column), because a value crossing a decode boundary needs an `ast` the table cannot mint — `Schema.decodeUnknown` consumes a schema's `ast`, and a frozen `as const` object exposes none. A table whose keys must round-trip through a wire channel is a `Schema.Literal` of the key set the moment the rows carry no behavior, and a `Schema.Class` the moment a row's fields are themselves admitted shapes.
- Dependency-edge revokes the table's *exemption* without revoking its *form*: the moment a row holds a closure, a `Schedule`, or a reference to a later-declared owner, the table is no longer the dependency-free vocabulary owner — it is a policy-value table that belongs with the vocabulary it dispatches, read by indexed access at the composition root, never a launchpad for a derived predicate built atop it. The form survives (it is still keyed dispatch); the exemption from being a vocabulary owner is what forfeits — a policy table is owned, not exempt.
- Identifier-need revokes to any owner carrying an `ast` annotation, because a table reused at three or more embed sites with its shape inlined explodes a JSON Schema instead of referencing one `$defs/<identifier>` definition, and the table has no identifier annotation to mint a `$ref` — a `ParseError` over a table-derived value produces an anonymous structural path no concept name can attribute. The missing-path is the diagnostic cost the exemption pays, and the third reference is the trigger that surrenders it.

```typescript
const Severity = {
    info: { rank: 0, retain: false },
    warn: { rank: 1, retain: true },
    error: { rank: 2, retain: true },
} as const satisfies Record<string, { readonly rank: number; readonly retain: boolean }>
const rank = (k: keyof typeof Severity): number => Severity[k].rank // exemption holds: keyed dispatch, no ast, no instance
// equality-demand appears -> the table key cannot dedupe by structure; re-route to a Data-bearing record:
const Marker = Schema.Data(Schema.Struct({ at: Schema.Number, level: Schema.Literal('info', 'warn', 'error') }))
const mark = (at: number, level: 'info' | 'warn' | 'error') => Data.struct({ at, level })
const seen = HashSet.make(mark(0, 'warn'), mark(0, 'warn')).pipe(HashSet.size) // 1 — keys by [Equal.symbol], never by string
```

[BARE_BRAND_REVOKES_ONLY_ON_CODEC]:
- The bare brand holds its exemption by being a `Brand.Constructor<A>` whose `make` validates without an `ast`, and exactly one axis revokes it: codec-demand. `Brand.nominal<A>()` (meaning-only) and `Brand.refined<A>(predicate, onFailure)` (a runtime invariant) both return a constructor with `.option`, `.either`, and `.is` modalities, sufficient for every in-memory admission — but neither participates in `Schema.decodeUnknown`, because the constructor is not a schema and carries no `ast` for the parse rail to consume. The value crossing a decode boundary is the sole signal that promotes the brand.
- The promotion is in-owner, never a re-spelling: `Schema.fromBrand(constructor)` lifts the existing `Brand.Constructor<C>` into a `BrandSchema<A & C, I, R>` reusing the constructor's own predicate as the schema's filter, so the invariant declared once for in-memory use participates in the parse rail at the boundary without restating it as a `Schema.filter`. The reject is a `Schema.brand("X")` over a re-spelled predicate declared beside the `Brand.refined` constructor — one invariant, two declarations that drift the moment the predicate tightens on one side only.
- The codec axis is the only revocation because the other axes the brand might face are already inside it: nominal distinction is the brand's reason to exist (it cannot revoke a property that defines the tier), and a multi-invariant primitive does not revoke to a schema but fuses inside the brand tier via `Brand.all(a, b, c)` under `Brand.EnsureCommonBase` — the constructor composes, accumulating every failed refinement in one `.either` pass, so the brand grows in invariant count without crossing into the schema family. The schema family is reached only when the value must be parsed, never when it gains a second invariant.

```typescript
type Handle = string & Brand.Brand<'Handle'>
const Handle = Brand.refined<Handle>(
    (s) => s.length > 0 && s.length <= 32,
    (s) => Brand.error(`<bad-handle:${s}>`),
) // exemption holds: in-memory admission via Handle(raw) / Handle.either(raw), no ast, no decode
const decoded = Handle.either('<raw>') // Either<Handle, Brand.BrandErrors> — recovery modality at the call site
// codec-demand appears (value arrives over a wire) -> lift the SAME constructor, never a parallel Schema.filter:
const HandleSchema = Schema.String.pipe(Schema.fromBrand(Handle)) // BrandSchema reusing Handle's predicate as the filter
```

[NATIVE_PRIMITIVE_REVOKES_ON_NOMINAL_OR_REFINEMENT]:
- The native primitive holds its exemption by carrying its invariant directly — a `number` that is its own count, a `string` that is its own label — and two axes revoke it, each one rung apart: nominal-collision promotes to `Brand.nominal` (two same-base values must never be confused by meaning), and refinement-invariant promotes to `Brand.refined`/`Schema.filter` (a runtime property must be proven). Neither is line count; both are the appearance of a fact the bare primitive structurally cannot witness — a primitive has no nominal tag and no validating `make`.
- The two axes are independent bits, not a ladder rung the primitive climbs once: a primitive can need nominal distinction without any predicate (`Brand.nominal`, zero runtime check, meaning-only), and can need a predicate without any nominal distinction — that second case revokes not to a brand but to `Schema.filter`, whose return type is `refine<C & B, From>`, narrowing the static type to the subtype while the runtime stays the base. The brand-vs-filter fork is whether two same-base values both satisfying the invariant must still be kept apart by meaning: if yes, the nominal axis fired (brand); if no, only the refinement axis fired (filter), and reaching for a brand there mints an identity that distinguishes nothing.
- The native-primitive exemption is the floor the brand-deletion law leaves standing, so its revocation is the deletion law read forward: a brand with no predicate and no nominal collision is ceremony the bare primitive replaces — and the primitive re-acquires the brand only when one of the two axes reappears. The exemption is monotone with the nominal ladder: bare primitive → `Brand.nominal` (meaning) → `Brand.refined` (predicate) → `Schema.fromBrand`/`Schema.brand` (codec), each rung reached only when its axis appears, and a value sitting one rung too high is the over-tier the demotion law strips back to the floor.

[KERNEL_STATEMENT_REVOKES_ON_A_RAILED_VALUE]:
- A statement survives only inside a measured kernel that names its exemption, and the single fact that revokes it is a domain value the statement produces that must carry a rail the statement cannot express — a typed failure, a codec admission, or a per-occurrence payload. A `for`/`while` accumulating a pure numeric scratch over a bounded index is the held exemption; the moment the loop body produces a value that can fail, must be decoded, or branches on a tag, the statement leaks an un-railed value into interior code that assumes the rail holds, and the kernel re-routes to the expression spine.
- The forfeiting axis names the spine member precisely: a fallible accumulation revokes to `Effect.reduce`/`Effect.forEach` (the rail threads the failure), a bounded fold over a state value revokes to `Effect.loop`/`Effect.iterate` (the state is the value, the predicate is the gate), and a tag-branch inside the body revokes to `Match.tagsExhaustive`/`Match.discriminatorsExhaustive` (the branch is the dispatch, total by construction). The statement form survives only where every produced value is pure, total, and tag-free — the three conditions the rail-bearing spine members each individually relax.
- The kernel exemption is asymmetric to the other three: the table, brand, and primitive revoke by *crossing an owner family*, but the kernel statement revokes by *re-expressing the same logic* in the spine combinator the rail demands — no new owner is minted, the loop body becomes the reducer, the mutable scratch becomes the threaded state, the `if`-ladder becomes the `Match` arms. The reject is the statement that produces a `ParseError`-capable value and then a separate post-loop validation step the interior must remember to call — the un-railed value already escaped the kernel before the check ran.

```typescript
// exemption holds: measured kernel, pure scratch, bounded index, no produced value can fail or decode
const checksum = (bytes: ReadonlyArray<number>): number => {
    let acc = 0
    for (const b of bytes) acc = (acc + b) & 0xffff
    return acc
}
// a fallible/decoding produced value appears -> the statement leaks an un-railed value; re-express in the spine:
const admitAll = (raws: ReadonlyArray<unknown>): Effect.Effect<ReadonlyArray<Handle>, ParseResult.ParseError> =>
    Effect.forEach(raws, (r) => Schema.decodeUnknown(HandleSchema)(r)) // the rail threads the failure the loop could not carry
```

[REVOCATION_AXIS_IS_READ_BEFORE_THE_TIER]:
- The forfeiting axis is a coordinate on the concept signature, not a property discovered after the floor tier is chosen, so the calculus runs the signature once and the exemption is granted or denied at that read: equality-demand and codec-demand are the two structural axes the floor tiers split on, dependency-edge and identifier-need are the two routability coordinates beside the shape, and the typed-rail demand is the channel-totality fact the kernel reads. A floor tier chosen against a signature that already carries the forfeiting axis is the misroute the calculus exists to catch before it locks in.
- The axes compose without precedence among the floor tiers, because each tier has a disjoint forfeiting set: the table forfeits to four axes, the brand to one, the primitive to two, the kernel to three, and no axis revokes two tiers to the *same* rung — equality revokes the table to `Data.struct` but never touches the brand (a brand's identity is its tag, not its fields), codec revokes both the table and the brand but to different owners (`Schema.Class` vs `BrandSchema`) because the table carries a struct and the brand carries a primitive. The destination is a pure function of (held tier, forfeiting axis), and that pair is the executable routing key.
- The calculus is total over the floor tiers: every floor exemption has a named forfeiting axis and a named destination rung, so a value at a floor tier whose signature later flips an axis re-routes by re-running the calculus, never by patching the floor owner in place. An exemption with no defined revocation is the gap that admits an un-promotable floor value into a context demanding a facet it cannot carry — the calculus closes that gap by pairing every floor tier with the exact axis that promotes it and the exact owner that receives it.
