# Definition-Time Generation Substrate

[PER_ASPECT_PROVIDER_FAN_OUT]:
- One owner declaration fans into independent incremental provider streams — core body, comparison, equality and arithmetic operators, parsing, span parsing, formatting, serializer surfaces — each gated by a predicate over the same extracted state and each emitting into the same partial. Aspects compose by addition of independent generators, never by one generator branching on flags: an owner that opts out of an aspect produces zero work in that aspect's stream rather than a skipped branch in a shared body.
- Every aspect stream is one parameterized five-stage skeleton — predicate filter, collect, distinct-by-type reduction, order-independent set comparer, re-expansion combined with global options into source output — where the predicate is the only aspect-specific line. The generation decision is data, not control flow: a boolean fold over the owner's own policy values (operator level non-none and the key carries the operators, an accessor, or string policy; span surface wanted and the key or a declared grammar is span-capable), evaluated once per owner per aspect.

[EQUATABLE_MODEL_AS_CACHE_KEY]:
- The extraction step is the firewall between the compilation's mutable symbol graph and the cache: candidate transforms read symbol data into frozen value models — strings, enums, flat capability booleans, immutable member arrays — and no symbol, syntax node, or location survives downstream, because symbol identity churns per keystroke and one retained symbol would bust the cache on every edit. Item equality keys on field name alone; whole-model equality is structural over every knob. The contract cuts both ways: anything the equality does not read cannot trigger regeneration, so everything emission depends on must be in the model — the equality definition is the regeneration specification.
- Cache granularity is the owner per aspect: flipping one operator level invalidates that owner's whole output, never a fragment, while two owners with identical policy differing only in name share a cached emission shape. Capability detection — which interfaces the key implements, which operators it declares — happens once at extraction into flat member-model fields, so emission is a pure function of the frozen model and never re-queries the type system.
- The collected batch is compared as a set, not a sequence: an order-independent comparer (mutual containment, count-only hash) makes the batch cache-stable under file-order and declaration-order permutation, so adding one owner invalidates only its own downstream slice. The set comparer's precondition is guaranteed by an upstream distinct keyed on fully-qualified type name — the two comparers are deliberately different relations on the same models, identity answering "same owner" for dedup (multi-partial declarations collapse to one) and structure answering "changed owner" for the cache.

[TOTAL_PIPELINE_DISCIPLINE]:
- Discovery is two-phase — a purely syntactic gate (class or struct declaration) rejects the overwhelming majority of nodes before any semantic model is touched, keeping per-keystroke cost proportional to changed declarations rather than compilation size — and every stage is wrapped in a fault barrier that converts exceptions into located diagnostics or a logged non-candidacy, rethrowing only cancellation. One malformed declaration degrades to non-generation of one owner with a diagnostic at its location; no stage can throw across the incremental boundary, so the pipeline is total by construction and fault-isolated per owner.
- Emission writes into leased pooled builders with a bounded pool that releases oversized buffers, and each emit threads the host cancellation token — steady-state regeneration reuses buffers warmed by the whole compilation and abandons long generations the instant the host cancels. Empty emission is suppressed structurally: the builder length is measured around the body and a file is emitted only when the body wrote past the prelude, with a logged warning otherwise — a misconfigured aspect is observable as a warning, never as an empty artifact in the compilation.

[OUTPUT_CHANNEL_SPLIT]:
- Domain surfaces — the owner body, admission triad, operators, diagnostics — register as source output and participate in semantic analysis, because downstream code must bind against the generated factory contract. Serializer codecs register as implementation source output: emitted late, invisible to semantic analysis and the editor's view of the type, never feeding another generator — terminal surfaces buy incremental throughput by leaving the analysis graph. One wire-facing owner therefore has its domain and codec surfaces generated through channels with different visibility and different recompute triggers.
- Codec generation is conditioned on the conjunction of the declared flag and the presence of the framework's integration assembly in the compilation — the serializer streams derive from the metadata-references provider, so a flag without the assembly produces the round-trip obligation diagnostics but no codec, and adding the reference retroactively materializes converters with no source edit.
- Deterministic hint names encode the owner's full position — namespace, containing-type chain with per-level generic arity, owner name and arity, then the aspect suffix — so same-named owners in different containers or at different arities never collide, and one owner's aspects land as separate files under a shared key: regenerating one aspect rewrites one file while sibling aspects' providers never recompute.

[DIAGNOSTICS_AS_PRECONDITION_GATE]:
- The structural preconditions of constructive unrevalidatability are error descriptors raised from the generation pipeline itself, before any body is generated: non-partial owners, reachable constructors, public initializers, mutable state, nullable keys, stacked shape attributes. The generated body never defends against these states because they cannot reach it — the no-double-validation guarantees are enforced ahead of generation, not assumed inside it. The same holds for contract obligations (the exact validate signature, the outbound projection when a wire is faced, the rehydration constructor when trusted rehydration is claimed) and for the wire partition (duplicate serialization, binding, or persistence claims are distinct errors), so a downstream codec can never face an ambiguous admission source.
- A self-referential constraint gates the whole aspect before it runs: the validation-error type bound on an owner is checked for satisfying its own static-create contract at definition time, so a fault-lattice member that fails to re-implement creation for its precise case fails the constraint and the owner never generates — the aspect that produces admission is itself guarded by an aspect over the fault type.

[CACHE_CONTRACT_OBSERVABILITY]:
- The opt-in emission counter watermarking generated files instruments the cache contract itself, not the output: under the value-model discipline a file regenerates only when its model changed, so a counter climbing on edits to unrelated declarations is direct evidence that some model over-reads — a retained symbol or location breaking the value-only equality. The build-property file logger is its complement: the counter proves the cache is churning, the log proves why a specific type fell out of the pipeline, with the rejected-state reason recorded only if the logger was enabled before the event.
