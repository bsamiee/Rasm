### [ONE_PRODUCER_CONTRACT]

`core/plan`'s `ArtifactWork(key, work, parents, admission, cost)` node is authoritative and UNCHANGED in shape; `Work[T] = Callable[[], Awaitable[RuntimeRail[T]]]`; `ArtifactPipeline.of(nodes, *, lane, warm, targets)` normalizes a lone node or any iterable and threads `warm` into `cache_seed` — the elision seed slot keyed admission probes (`plan.md:442-450,393`). Every producer page lands exactly this pair:

```python
def emit(self, /) -> ArtifactWork | Iterable[ArtifactWork]:
    return ArtifactWork(
        key=self._key,          # ContentIdentity.of over canonical(spec) ⊕ parent keys, CANONICAL_POLICY — minted PRE-RUN
        work=self._emit,        # the renamed private render thunk — Work[ArtifactReceipt]; the plan never invokes it
        parents=self._parents,  # upstream ContentKeys: bound-figure keys, member keys on a composite
        admission=Admission(keyed=None),
        cost=self._cost,        # render-time weight the CPM forward pass sums
    )

async def _emit(self, /) -> RuntimeRail[ArtifactReceipt]:
    ...  # threads self._key into the terminal receipt so receipt.slot == node.key; a failed production folds to the rail fault
```

- **KEY-OVER-INPUT law.** `ArtifactWork.key` mints over the producer's frozen request/spec owner's canonical bytes ⊕ its parent keys — deterministic and computable BEFORE `work` runs — so `keyed` admission probes the threaded `cache_seed` (`of(warm=)` is its seed slot — the `core/issue` construction threads it) and the runtime lane short-circuits BEFORE production. The current corpus mints over produced OUTPUT bytes (`font.md:274` computes `payload` then `ContentIdentity.of(f"font-{tag}", payload)`; `layered._emit` keys over `LayerFact.data`), so a key exists only after work runs and elision is impossible — the rejected order, corrected on every producer in its own rebuild per `[01]-[ENTRY_CONTRACT]`. `_emit` threads the same key into its terminal receipt (`receipt.slot == node.key` — the hit/miss match). A separate output content-address rides an evidence fact only where the store needs it (Gate #3); it is never the elision key.
- **Admission selection law.** `keyed` is the default (cache-eligible, the elision spine); `bare` is the forced-live non-addressable one-shot; `retried(RetryClass.OCCT)` is reserved for a whole-producer transient — a native render/press/raster crossing a worker that can die. The cases stand on disk (`plan.md:51-55`, `retried` carrying its MANDATORY `RetryClass`). A `keyed` node with an internal offload-retry keeps `keyed` and rides `lanes.offload(retry=...)` inside `_emit` — `ArtifactWork.lowered` (`plan.md:99`) is the lowering authority this rule composes, each node's own admission `match` onto the lane `Admit` arm.
- `_emit` is a bound zero-arg coroutine — it satisfies `Work[ArtifactReceipt]` directly, no lambda wrapper. The receipt carries its own key (`ArtifactReceipt.slot`), so `emit` never returns a `(key, receipt)` pair. Batch producers return `Iterable[ArtifactWork]` — one node per member with per-member keys, so per-member elision stays per-member (a re-issued set re-renders only changed members). A composite is ONE aggregate node whose `parents` are its member keys and whose receipt case carries the aggregate facts (`Register`/`Transmittal` the standing proof); receipts are terminal — incrementality is node granularity, never a partial receipt.

### [CONSTRUCTING_OWNER] — NEW `core/issue.md`; `ArtifactPipeline` becomes REAL

`core/plan` is the ENGINE, never the constructor. ONE NEW page owns pipeline construction for every modality, sited at the top of the dependency cone — it imports terminal producers downward; nothing in artifacts imports it (external hosts and sibling packages invoke it):

```python
@tagged_union(frozen=True)
class IssueRequest:  # the flagship discriminant — one closed vocabulary, one arity owns all modalities
    tag: Literal["sheet_set", "diagram_suite", "document_package", "single"] = tag()
    sheet_set: "TransmittalRecord" = case()              # terminal = delivery/transmittal (aggregate + members)
    diagram_suite: tuple["DiagramLayout", ...] = case()  # terminal = visualization/diagram/draw, one node per kind
    document_package: tuple["SpecSection", ...] = case() # terminal chain = section -> emit -> egress
    single: ArtifactWork = case()                        # any lone producer node


@dataclass(frozen=True, slots=True, kw_only=True)
class ArtifactIssue:
    lane: LanePolicy
    warm: Map[ContentKey, ArtifactReceipt] = Map.empty()  # the cache seed — in-session receipts now; Persistence fills it cross-run (Gate #3)

    async def issue(self, request: IssueRequest, /) -> RuntimeRail[Block[ArtifactReceipt]]:
        nodes, targets = self._nodes(request)  # total match: terminal.emit() -> Iterable[ArtifactWork]
        plan = await ArtifactPipeline.of(nodes, lane=self.lane, warm=self.warm, targets=targets).plan()
        return await self._driven(plan)        # folds the CPM fronts through the runtime LanePolicy.drain, front by front
```

- `issue()` is the composition root, not a producer — no receipt case. It calls the terminal producers' `emit()`, folds the nodes into `ArtifactPipeline.of(...)`, `.plan()`s, drains the fronts through the runtime `LanePolicy.drain` (`Block[Admit[T]]` per front — the landed lane surface), and returns the terminal receipts. Imports all point downward: `delivery/transmittal` (s3 within-stratum), `composition/sheet`/`document/emit`/`document/egress`/`visualization/diagram/draw` (s2), `core/plan` (s1), the runtime lane.
- The three flagship terminals stay PRODUCERS: `SheetSet.emit()` returns one node per sheet (cover, drawing-index, general-notes, drawing sheets); `transmittal.emit()` returns the member node set plus the ONE aggregate `Transmittal` node whose `parents` are the member keys; `document/emit` binds one `DocumentNode` context to N format nodes (absorbing `core/format`'s `TemplatePayload` admission and `bound()` fan-out as emit rows). The diagram-SUITE constructor gap closes as `issue(IssueRequest(diagram_suite=...))` over `draw.emit()` per kind.
- `core/plan`'s `[CONTENT_KEY_NODES]` producer roster refreshes to name `core/issue` as THE constructing owner; the plan imports no producer (the `Work` thunk carries the call), so no producer→plan→producer cycle forms. `core/` spans strata (receipt+plan the s1 spine, issue s3) — a stratum is a dependency position, never a folder grouping; `core/receipt` and `core/plan` settle before any producer entry binds and are never re-opened.
