# Entrypoint Law — Core

[MODAL_ARITY_MECHANICS]:
- `SwitchMapMethodsGeneration.Default` emits full-coverage `Switch`/`Map` overloads only; `DefaultWithPartialOverloads` additionally emits partial-coverage overloads with a fallback arm. The state-threaded form is always present: `Switch<TState>(TState state, Action<TState, CaseA> caseA, ...)` — zero-allocation because no closure forms around captured state.
- `SwitchMapStateParameterName` is a per-union string override (default `"state"`) that renames the threading parameter in generated signatures. When the same domain type hosts multiple nested `[Union]` families, distinct state-parameter names prevent generated-overload ambiguity at call sites that chain dispatchers.
- `NestedUnionParameterNameGeneration.Default` produces `failureNotFound`-style names (intermediate type name prefixed); `Simple` uses only the final leaf name and emits a compile error when two nested families share a leaf name — choice is load-bearing, not cosmetic.
- `FactoryMethodGeneration.Default` triggers factory generation only when a member forces it: type-parameter members, `object`, interface members, or duplicate types. `None` suppresses all factories unconditionally; `Always` generates for every member. Suppressing with `None` and providing a single static factory is the canonical way to implement a controlled entrypoint surface over a union whose cases the caller must not construct directly.
- `ConstructorAccessModifier.Private` on `[Union]` simultaneously seals the constructor and makes all generated implicit conversion operators private — callers cannot construct any case without going through the owning factory. This is the correct shape for a request union where the module owns the only valid entry path.

[AD_HOC_VS_REGULAR_UNION_DISPATCH_SELECTION]:
- `[Union<T1,T2,...>]` (generic attribute, up to five type parameters) and `[AdHocUnion(typeof(T1),typeof(T2),...)]` (non-generic attribute, same up to five) are semantically equivalent; the generic form requires C# 11+. Both produce an implicit conversion from each `Ti` to the union and an explicit conversion back; both emit the same `Switch`/`Map` pair. Choose ad-hoc when the payload types are already defined and the union is a positional sum without named cases. Choose regular `[Union]` when cases carry distinct named fields — because regular-union cases are declared as nested `sealed` or `abstract partial` subtypes and the generator emits a type-safe accessor for each named case rather than generic `AsT1`/`AsT2` accessors.
- `T1IsStateless = true` on an ad-hoc union member stores only the discriminator index, not the payload, eliminating the member's allocation for presence-only cases. The generated accessor for that member returns `default(Ti)`. Use struct types for stateless members to avoid null-handling complexity in the `Value` accessors.
- `UseSingleBackingField = true` forces the generator to use a single `object` backing field regardless of member types, including structs. This trades type safety at the backing-field level for a smaller generated surface and must not be chosen for performance-sensitive dispatch paths because it introduces boxing for struct members.

[PARAMS_COLLECTIONS_AND_ARITY_COLLAPSE]:
- `params ReadOnlySpan<T>` is the span-boundary absorber; it requires no allocation at the call site when the argument list fits on the stack. The BCL encodes this via `[System.Runtime.CompilerServices.ParamCollectionAttribute]` (distinct from the legacy `[ParamArrayAttribute]`). Declaring `static Fin<Result> Execute(params ReadOnlySpan<ItemCmd> cmds)` collapses singular, multi-item, and empty call sites into one entrypoint — the caller writes `Execute(a, b)` via a collection expression or `Execute([a, b])` and the compiler lowers the literal to a stack-allocated span when the count is statically known.
- `FrozenDictionary<TKey, TValue>.Create(params ReadOnlySpan<KeyValuePair<TKey, TValue>>)` enables inline dispatch-table initialization from a collection-expression literal without a heap-allocated intermediate. `FrozenDictionary<TKey, TValue>` carries `[CollectionBuilder(typeof(FrozenDictionary), "Create")]`, so the collection-expression syntax `FrozenDictionary<string, Handler> t = [(k1, h1), (k2, h2)];` lowers directly to `FrozenDictionary.Create<string, Handler>(span)` with no intermediate dictionary. This is the canonical construction form for bounded-vocabulary dispatch tables initialized at module load.
- `[OverloadResolutionPriorityAttribute(int)]` (`System.Runtime.CompilerServices`, .NET 9+) lets a higher-priority overload win without needing `params` as the tiebreaker. Applying a positive priority to the `ReadOnlySpan<T>`-based `params` overload over a legacy `T[]`-based overload ensures the span path is preferred when both are present.
- The input-redescribing-parameter anti-pattern: a `bool isBatch` or `int count` parameter beside the value re-encodes discriminant information the type system already provides. The discriminant must be recoverable from the value's type, case, or arity without a separate knob. When a count or mode is needed internally, derive it from `span.Length` or from pattern-matching the union case — never surface it as a parallel parameter.

[VERB_FAMILY_COLLAPSE_INTO_REQUEST_UNION]:
- The canonical collapse of a verb family (`Create`, `Update`, `Delete`, `Patch`) is a `[Union]` with one case per verb, `ConstructorAccessModifier = Private`, and one total dispatch method on the module. The dispatch method receives the union value and switches over it, eliminating three or more sibling entrypoints with shared preamble — each preamble becomes the constructor of its case; shared validation becomes the dispatch prologue.
- `DefaultWithPartialOverloads` on the request union emits an overload with a `_` fallback arm. When new cases are added, the full-coverage overload becomes a compile error at all switch sites; the partial overload lets call sites that care about a subset continue to compile while the dispatch internals use the full-coverage form. Full-coverage at dispatch internals maintains exhaustiveness.
- Nested `[Union]` families compose with `NestedUnionParameterNameGeneration.Simple` only when all leaf names across all nested families are distinct — otherwise use `Default` naming to preserve uniqueness. `Simple` mode is a compile-time contract that fails loudly on collision rather than silently shadowing.

[DEEP_SURFACE_AND_MODULE_INTERNALS]:
- The module's public surface is the single dispatch entrypoint plus the request union type. All case constructors are private via `ConstructorAccessModifier = Private`. The module's internal helpers, partial operations, and pipeline composition are kept `internal` or `private static`. The `[Union]` type may be `public` or `internal`; if the request union crosses an assembly boundary, use `ConstructorAccessModifier.Internal` so only trusted assemblies can construct cases.
- Extension blocks (C# 14) allow adding the dispatch entrypoint to the union type without modifying the generated partial. The pattern: `extension(RequestUnion req) { public Fin<Result> Execute(IDepA dep) => ... }`. This keeps the union declaration free of application-level logic while attaching the dispatch directly to the type it dispatches on. Extension blocks must be placed in the owning module's file, not in the generated file.
- `[UseDelegateFromConstructor]` on a `[SmartEnum]` partial method produces a per-item delegate field wired through the constructor. The generator produces a private delegate type whose signature matches the partial method; each static field passes a static method reference at construction. This is the mechanism by which a smart-enum item carries case-owned behavior without a switch arm — the behavior lives with the vocabulary, and the entrypoint method is the partial method shared across all items.

[COMPOSITION_TIME_ASPECTS_ON_ENTRYPOINTS]:
- `Schedule` is a lazy stream of `Duration` values; it does not execute until applied to `retry`, `repeat`, or `fold`. Composing schedules uses `|` (union: take min duration, run to length of longest) and `&` (intersect: take max duration, run to length of shortest). The canonical retry aspect: `retry(Schedule.recurs(5) | Schedule.exponential(10 * ms), computation)`.
- `@catch`, `catchOf<E>`, `expected`, `exceptional`, `expectedOf<E>`, `exceptionalOf<E>` are all `Prelude` methods returning `CatchM<Error, M, A>`. The `|` operator on `K<M, A>` takes a `CatchM` right-hand operand, making the catch an inline pipeline stage: `computation | @catch(err => fallback)`. Chaining `|` composes independent catch predicates left-to-right; the first match wins. The `final<F,X>(K<F,X>)` combinator produces a `Finally<F,X>` value; `computation | catchHandler | final(cleanup)` chains catch and cleanup in one expression, and the `finally` clause runs regardless of whether the catch matched.
- `bracketIO` has three distinct forms: single-arg (`bracketIO(computation)`), three-arg (`bracketIO(Acq, Use, Fin)`), and four-arg (`bracketIO(Acq, Use, Catch, Fin)`). Resources acquired via `IO.use<A>` inside the single-arg form are registered in a local scope and released when the scope exits regardless of success or failure — the preferred shape. The three-arg form handles resources that cannot implement `IDisposable` directly. The four-arg form adds a `Catch` arm for errors raised within the `Acq` computation; `Fin` always runs in all paths regardless.
- Resources acquired inside a `retry` loop are released if the attempt fails before the retry is exhausted — retry does not accumulate open resources across failed iterations. The XML docs state: "Any resources acquired within a retrying IO computation will automatically be released if the operation fails."

[OVERLOAD_RESOLUTION_ORDERING]:
- `OverloadResolutionPriorityAttribute` accepts any `int`; the compiler picks the highest priority among candidates that pass type-checking. Priority 0 is the implicit default. Applying priority 1 to a `params ReadOnlySpan<T>` overload and leaving the `params T[]` overload at 0 directs the compiler to the span path at all call sites that pass a collection expression or literals.
- When two overloads differ only in the nullability of a parameter (`T` vs `T?`), `OverloadResolutionPriority` is the only mechanism short of renaming that lets the non-nullable overload win at call sites where the argument is provably non-null. Applied to the non-null overload with a higher priority, it suppresses the ambiguity without a cast.
- The `DefaultWithPartialOverloads` partial-coverage overload has lower priority than the full-coverage overload in generated code. When both are callable, the full-coverage overload wins — consistent with the desired architecture: internal dispatch uses full coverage; external call sites that opt into partial coverage must call the partial overload explicitly.
