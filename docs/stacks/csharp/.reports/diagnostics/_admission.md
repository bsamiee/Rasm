# diagnostics ‚Äî admission answer

Question: does the doctrine need `Serilog.Extensions.Hosting`, and which sink set ‚Äî OTLP-only or more?

## verdict

`Serilog.Extensions.Hosting`: YES. Sink set: NOT OTLP ‚Äî admit a durable rolling-file sink (`Serilog.Sinks.File`) and an interactive console sink (`Serilog.Sinks.Console`); reject `Serilog.Sinks.OpenTelemetry`. The "likely OTLP-only" hypothesis inverts: OTLP is the one sink family Serilog must NOT own in this stack.

## evidence

1. The emission front must be the `Microsoft.Extensions.Logging` seam, not Serilog-first. The manifest's telemetry capabilities exist only on that seam: classified-tag redaction (`LoggerMessageState.RedactedTagArray` activated by `EnableRedaction`), enrichment rows (`ILogEnricher`/`IStaticLogEnricher` + `EnableEnrichment`), log sampling (`AddTraceBasedSampler`/`AddRandomProbabilisticSampler`), and global buffering (`AddGlobalBuffer`) are all `ILoggingBuilder`/service registrations consumed by the extended MEL logger. Serilog has no classification taxonomy and no redactor seam ‚Äî events emitted via `Log.*`/Serilog `ILogger` bypass the one enforcement seam entirely. Serilog therefore sits BEHIND the seam as a provider, never in front.
2. Sitting behind the seam requires the provider bridge plus host lifetime, which is exactly what `Serilog.Extensions.Hosting` packages: it depends on `Serilog.Extensions.Logging` (the MEL provider), and adds `AddSerilog(...)` service registration, `IDiagnosticContext`, `CreateBootstrapLogger()` returning a `ReloadableLogger` (two-stage init: bootstrap-window capture, then freeze when the host-configured pipeline is built), and close-and-flush on host shutdown. Admitting `Serilog.Extensions.Logging` alone forfeits the bootstrap window and the shutdown flush; admitting bare `Serilog` (current manifest state) provides no sink and no bridge ‚Äî it is inert without one of the two.
3. OTLP log export already has one owner: the OTel logger provider (`WithLogging` + `UseOtlpExporter`) exports MEL records sharing ONE `Resource` identity, one batch-processor governance, and one drain choreography with traces and metrics ‚Äî and `UseOtlpExporter` enforces single-ownership at build time. The Serilog OTLP sink (`Serilog.Sinks.OpenTelemetry`, `WriteTo.OpenTelemetry`: gRPC/http-protobuf, own `ResourceAttributes`, own `IncludedData` flags, own batching) would mint a SECOND OTLP route with divergent resource identity and unshared export-pressure governance ‚Äî split-brain export. Its one distinctive capability (LogEvent‚ÜíOTLP with trace/span fields) duplicates the admitted route.
4. What Serilog uniquely adds behind the seam ‚Äî and the MEL/OTel stack lacks ‚Äî is operator-grade local projection: audit sinks with caller-propagating delivery guarantees (`AuditTo`), declared fallback chains (`FallbackChain`), a typed pipeline-failure rail (`ILoggingFailureListener` with Temporary/Permanent/Final discrimination), runtime level switches (`LoggingLevelSwitch` + `MinimumLevel.Override`), sub-logger routing, and bounded batching with a retry ceiling (`BatchingOptions`). These earn Serilog its seat only if at least one real sink is admitted; the minimal set exercising them is file (durable local evidence surviving collector outage ‚Äî complements OTLP, which loses telemetry beyond queue+retry budget) plus console (interactive/desktop processes; OTLP presumes a collector that interactive tooling does not have).

## residency consequence (for the page, not the manifest)

- Admit: `Serilog.Extensions.Hosting` (brings `Serilog.Extensions.Logging` transitively), `Serilog.Sinks.File`, `Serilog.Sinks.Console`.
- Reject: `Serilog.Sinks.OpenTelemetry` (second OTLP owner), Serilog-first emission (bypasses redaction/enrichment/sampling/buffering), `Serilog.Extensions.Logging` as a standalone admission (loses bootstrap + flush ownership).
- Law shape the page legislates: one emission grammar (generated MEL), two projection backends as provider rows ‚Äî OTel logger provider owning the OTLP wire, Serilog provider owning operator-local sinks ‚Äî one enforcement seam upstream of both.
