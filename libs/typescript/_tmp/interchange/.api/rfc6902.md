# [API_CATALOGUE] rfc6902

`rfc6902` supplies the RFC 6902 JSON Patch and RFC 6901 JSON Pointer surface: the `Operation` discriminated union over the six patch verbs (`add | remove | replace | move | copy | test`), the error-accumulating `applyPatch` that returns one nullable error slot per operation rather than throwing, the `createPatch` diff generator, the `createTests` optimistic-concurrency guard builder, and the `Pointer` RFC 6901 path evaluator. It owns the recorded-intent partial-update vocabulary the C# wire crosses as `JsonPatchDocument<TModel>` operations, decoded on the TS side as the same six-verb union rather than a state-diff endpoint.

## [04]-[IMPLEMENTATION_LAW]

[PATCH_TOPOLOGY]:
- `applyPatch` is error-accumulating, never throwing: it returns one slot per operation, `null` for success and the typed error for failure, so a partial-failure patch reports every per-operation outcome in order rather than aborting at the first fault
- `applyPatch` mutates the target in place; the caller clones before applying when the pre-patch value must survive, because no immutable-apply variant exists
- `apply` is the single-operation form of `applyPatch`; its `op` discriminant selects the verb function and an unrecognized `op` returns `InvalidOperationError`
- `test` carries optimistic concurrency: a `TestOperation` whose `value` deep-inequals the pointer target returns `TestError(actual, expected)`, so a patch prefixed with `createTests(...)` self-guards against a replayed or stale apply
- operation order is significant — `move`/`copy` reference a `from` pointer evaluated against the post-prior-operation state, so reordering changes the result

[POINTER_LAW]:
- `Pointer.fromJSON` parses the RFC 6901 escape forms `~0` → `~` and `~1` → `/`; a raw `/`-joined string with unescaped tokens is the rejected path form
- `pointer.evaluate` returns `{ parent, key, value }` so a setter mutates `parent[key]` without re-walking; `get` returns `undefined` for an absent path and never throws
- the empty pointer `""` addresses the whole document root; `"/-"` addresses the array-append position on `add`

[LOCAL_ADMISSION]:
- decode the wire patch through `Schema.decodeUnknown` over the six-arm `Schema.Union` before `applyPatch`, so an unknown `op` faults at the decode boundary as `Additive`/`Breaking` drift rather than as an `InvalidOperationError` slot after a partial apply
- fold the `(error | null)[]` result into one typed `FaultDetail` rather than reading the array per index — a non-null slot is a `HopFault` carrying the failing operation index and `op` in evidence
- `createPatch` is a development-time diff generator; a contract patch is recorded intent decoded off the wire, never a re-derived diff at the consumer

[RAIL_LAW]:
- Package: `rfc6902`
- Owns: RFC 6902 JSON Patch apply/diff and RFC 6901 JSON Pointer evaluation
- Accept: a decoded six-verb `Operation[]`; a target object to mutate in place
- Reject: a state-diff endpoint re-deriving the patch at the consumer; throwing apply that loses per-operation outcome; an undecoded raw-wire patch applied before the `op` union admits it
