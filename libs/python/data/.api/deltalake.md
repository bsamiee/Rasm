# [PY_DATA_API_DELTALAKE]

`deltalake` supplies a native Rust-backed Delta Lake table reader and writer with ACID transactions, time travel, schema evolution, and change-data-feed access over object-store and local backends. `DeltaTable` owns lifecycle, reads, and maintenance; `write_deltalake` owns appends and overwrites; `QueryBuilder` runs DataFusion SQL over registered tables.

## [01]-[PACKAGE_SURFACE]

[PACKAGE_SURFACE]: `deltalake`
- package: `deltalake`
- import: `import deltalake`
- owner: `data`
- rail: columnar
- version: `1.6.0` (locked); license `Apache-2.0`
- wheel/ABI: `cp310-abi3` wheels for macOS x86_64/arm64, manylinux 2.17/2.28, musllinux 1.2 aarch64/x86_64, win_amd64; `_internal` is a Rust core, so the cp310 abi3 wheel is forward-compatible to cp315; runtime deps are `arro3-core` (Arrow C-data bridge for `QueryBuilder`/`load_cdf`/`deletion_vectors` egress) and `deprecated`
- capability: Delta Lake table exchange, ACID append/overwrite/merge/delete/update, time travel and version loading, change-data-feed reads, file compaction/z-order and vacuum/log-compaction, schema evolution and column-metadata mutation, OpenTelemetry tracing via `init_tracing`, and DataFusion SQL query over Arrow egress

## [02]-[CAPTURE]

[PUBLIC_TYPES]: `__all__` is the 21-name surface `DeltaTable`, `Metadata`, `write_deltalake`, `convert_to_deltalake`, `WriterProperties`, `ColumnProperties`, `BloomFilterProperties`, `Schema`, `Field`, `DataType`, `VariantType`, `Transaction`, `CommitProperties`, `PostCommitHookProperties`, `QueryBuilder`, `TableFeatures`, `init_tracing`, `rust_core_version`, `__version__` plus the `TableMerger`/`TableOptimizer`/`TableAlterer` accessor builders.
- `deltalake.DeltaTable` — managed Delta table; constructed from a `table_uri`; owns reads, writes, history, maintenance, and protocol; eager egress via `to_pyarrow_table`/`to_pandas` and lazy egress via `to_pyarrow_dataset`/`get_add_actions`; `load_cdf` and `deletion_vectors` return an `arro3.core.RecordBatchReader`; `optimize`/`alter` accessors expose `TableOptimizer`/`TableAlterer`.
- `deltalake.Schema` — Delta table schema; holds `Field` columns; `to_arrow(as_large_types=False) -> pa.Schema`, `from_arrow(type) -> Schema` (staticmethod), `to_json() -> str`, `from_json(json) -> Schema` (staticmethod).
- `deltalake.Field` — `Field(name: str, type: DataType, *, nullable: bool = True, metadata: dict | None = None)` typed column descriptor.
- `deltalake.DataType` — type-alias `Union[PrimitiveType, ArrayType, MapType, StructType, VariantType]` (NOT a factory); `PrimitiveType(data_type: str)` constructs a primitive from a type string (`PrimitiveType("integer")`), and `Field("col", "integer")` accepts the type string directly, so a `(name, type-string)` column-add tuple needs no separate type parser.
- `deltalake.Metadata` — table metadata: id, name, description, partition columns, configuration, and created time.
- `deltalake.Transaction` — application-level transaction descriptor with app id, version, and last-updated fields.
- `deltalake.QueryBuilder` — DataFusion SQL runner; `QueryBuilder().register(table_name: str, delta_table: DeltaTable) -> QueryBuilder` then `execute(sql: str) -> arro3.core.RecordBatchReader` (consume via `.read_all()`); register multiple tables to join across Delta tables in one SQL statement.
- `deltalake.WriterProperties(compression=, compression_level=, statistics_truncate_length=, default_column_properties=, column_properties=, data_page_size_limit=, dictionary_page_size_limit=, write_batch_size=, max_row_group_size=)` — Parquet writer tuning: compression, dictionary, statistics, bloom-filter, and per-column properties.
- `deltalake.ColumnProperties(dictionary_enabled=, statistics_enabled=, bloom_filter_properties=, encoding=)`, `deltalake.BloomFilterProperties(set_bloom_filter_enabled, fpp=, ndv=)` — per-column writer property carriers.
- `deltalake.CommitProperties(custom_metadata=, max_commit_retries=, app_transactions=)` — commit-time metadata: app transactions, custom metadata, and max commit retries.
- `deltalake.PostCommitHookProperties(create_checkpoint=, cleanup_expired_logs=)` — post-commit hook control for checkpoint creation and log cleanup.
- `deltalake.TableFeatures` — reader/writer feature enum: `ColumnMapping`, `DeletionVectors`, `TimestampWithoutTimezone`, `V2Checkpoint`, `AppendOnly`, `Invariants`, `CheckConstraints`, `ChangeDataFeed`, `GeneratedColumns`, `IdentityColumns`, `RowTracking`, `DomainMetadata`, `IcebergCompatV1`, `VariantType`, `VariantTypePreview` for protocol upgrades.
- `deltalake.VariantType` — Delta variant logical type.
- `deltalake.init_tracing(...)` — enables OpenTelemetry tracing for delta-rs operations with a configurable OTLP endpoint and automatic shutdown registration; the OTel span seam for write/merge/optimize evidence.

[EXCEPTIONS]:
- `deltalake.exceptions.DeltaError` — base native error; `DeltaProtocolError`, `TableNotFoundError`, `CommitFailedError`, `SchemaMismatchError` follow from invalid protocol, missing table, commit conflict, and schema drift.

[IMPLEMENTATION_LAW]:
- `write_deltalake` accepts any Arrow C-stream-exportable object (PyArrow, Polars, pandas via Arrow); `mode` discriminates `error`/`append`/`overwrite`/`ignore` and `schema_mode` controls merge versus overwrite evolution.
- `DeltaTable.merge` returns a `TableMerger` builder; chain `when_matched_update`/`when_not_matched_insert`/`when_not_matched_by_source_delete` then `execute()`, never separate per-clause commit calls.
- Reads are lazy through `to_pyarrow_dataset`; column projection and partition/predicate filters push down before materialisation, while `to_pyarrow_table`/`to_pandas` materialise eagerly.
- Time travel loads a prior version in place via `load_as_version(version)`; `restore` writes a new commit that reverts the table rather than mutating history.
- Writer tuning flows through `WriterProperties` with nested `ColumnProperties`/`BloomFilterProperties`; commit metadata and retries flow through `CommitProperties`.
- `delete`/`update`/`merge` return a metrics dict keyed `num_added_files`/`num_removed_files`/`num_updated_rows`/`num_copied_rows`/`num_deleted_rows`; `num_deleted_rows` is omitted from a predicate-less `delete`. `restore` returns the native restore metrics dict; the `num_removed_file`/`num_restored_file` key spellings originate in the Rust layer and are not surfaced in the Python docstring.
- `TableAlterer` evolves by addition and metadata only: `add_columns(fields)` appends columns, `set_column_metadata(column, metadata)` mutates per-column metadata, and `set_table_name`/`set_table_description`/`set_table_properties` mutate table-level metadata; the alterer exposes no column drop or rename, so a Delta schema rename or drop is unreachable and a portable `Evolve` binds only the `adds` clause on Delta.
- `deltalake.schema.Field(name, type, *, nullable=True, metadata=None)` and `deltalake.schema.PrimitiveType(type_string)` accept a primitive type string directly (`Field("col", "integer")`), so a `(name, type-string)` column-add tuple constructs a `Field` with no separate type parser; `DataType` is the `Union[PrimitiveType, ArrayType, MapType, StructType, VariantType]` alias, never a factory call.
- `QueryBuilder().register(name, table).execute(sql)` runs DataFusion SQL over one or more registered Delta tables and returns an `arro3.core.RecordBatchReader` (drain with `.read_all()`); it is the declarative SQL surface, distinct from the imperative `DeltaTable` reader, and registering multiple tables joins across Delta tables in one statement.
- `init_tracing` registers a delta-rs OpenTelemetry exporter once at process start; subsequent write/merge/optimize commits emit spans automatically, so the table-mutation evidence is collected at the Rust layer rather than wrapped per call.

[INTEGRATION_STACKING]:
- engine federation: `DeltaTable.to_pyarrow_dataset()` is a pushdown-capable `pa.dataset.Dataset` that a `datafusion` `SessionContext.register_table`/`read_table` or a `duckdb` `con.register`/`from_arrow` adopts directly, so a Delta table joins Parquet/CSV/object-store sources under one engine's SQL with predicate/column pruning pushing into the Delta scan — never a `to_pyarrow_table` eager copy fed to the engine.
- in-engine vs cross-engine SQL: `QueryBuilder` is the embedded DataFusion runner that returns an `arro3.core.RecordBatchReader` consumed zero-copy by polars/pyarrow; for cross-engine plan exchange the Delta dataset registers into the `datafusion`/`duckdb-substrait` Substrait spine instead, so SQL stays in-process while plans cross engines.
- write ingress stack: `write_deltalake`/`merge` accept any Arrow C-stream-exportable, so a `polars` `LazyFrame.collect()`, a `datafusion` `DataFrame.to_arrow_table()`, or a `duckdb` `fetch_record_batch()` writes straight into a Delta commit; `WriterProperties` carries Parquet tuning and `CommitProperties.app_transactions` carries the idempotent `Transaction` app-id under a `stamina` `retry_context` keyed by `max_commit_retries` for conflict resolution.
- observability stack: `init_tracing` and the `delete`/`update`/`merge`/`compact`/`z_order` metrics dict (`num_added_files`/`num_removed_files`/`num_updated_rows`/`num_copied_rows`/`num_deleted_rows`) compose one receipt — the OTel span carries the operation timing and the metrics dict carries the file/row deltas, never a hand-rolled commit log.

## [03]-[LOCAL_ADMISSION]

[RAIL_LAW]:
- Package: `deltalake`
- Owns: Delta Lake table read/write, ACID merge/delete/update, time travel and CDF, compaction/z-order/log-compaction and vacuum, schema evolution and column-metadata mutation, OpenTelemetry tracing, and embedded DataFusion SQL egress
- Accept: `DeltaTable(table_uri, ...)` as the lifecycle owner, `write_deltalake` for appends/overwrites, `merge(...).execute()` builder chains, Arrow-exportable inputs from any sibling frame/engine, `WriterProperties`/`CommitProperties` for tuning, `to_pyarrow_dataset()` registered into a `datafusion`/`duckdb` engine for cross-source federation, `init_tracing` for commit observability
- Reject: per-clause merge commits, eager `to_pyarrow_table` reads when predicate/column pushdown via `to_pyarrow_dataset` applies, hand-rolled Parquet log parsing, a `DataType` factory call where the type-string `Field`/`PrimitiveType` path applies, and duplicate per-frame write entry points outside `write_deltalake`
