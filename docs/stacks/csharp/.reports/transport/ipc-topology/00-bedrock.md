# ipc-topology bedrock

## UDS endpoint law

- `UnixDomainSocketEndPoint(string path)` validates against the platform's native socket-path budget at construction (sockaddr_un: ~104 bytes on BSD-derived systems, ~108 on Linux) and rejects oversize or empty paths with an argument-range failure — the budget is bytes after encoding, not characters, so endpoint directories stay short and ASCII; deep per-user temp trees are the path-budget trap that surfaces only on machines with long user names.
- The Linux abstract namespace (leading NUL in the path) binds no filesystem node: no file to unlink, but also no directory mode to enforce — abstract endpoints are the rejected row for credential-gated seams because the only access control is peer-credential checking after accept, and they are non-portable besides.
- Server bind: `KestrelServerOptions.ListenUnixSocket(socketPath, configure)` requires an absolute path and binds via the transport's `CreateBoundListenSocket`, which constructs `Socket(AddressFamily.Unix, SocketType.Stream, ProtocolType.IP)` and binds without unlinking an existing file — a pre-existing socket file surfaces as an address-in-use failure wrapped from the socket error, so pre-bind staleness resolution is the topology's job, never the listener's.
- Unlink-on-dispose is asymmetric and load-bearing: a socket records its bound file's full path at bind time, and disposing THAT socket best-effort-deletes the file (failures swallowed); accepted per-connection sockets carry an unbound copy of the endpoint, so disposing connections never unlinks the listener's file. Consequence: clean shutdown self-cleans, a killed process leaves a stale file, and the stale file then blocks the successor's bind — exactly the case the staleness probe owns.
- Client dial rides the handler connect seam: `SocketsHttpHandler.ConnectCallback` (`Func<SocketsHttpConnectionContext, CancellationToken, ValueTask<Stream>>`) constructs a Unix stream socket, connects to the endpoint, and returns `NetworkStream(socket, ownsSocket: true)`; the channel address is a placeholder authority — the context exposes the parsed `DnsEndPoint` and the callback ignores it. Failures inside the callback project as connect failures on the call, so the redial law sees one failure taxonomy for TCP and UDS rows.
- The Windows column of the same row axis is the named pipe: `ListenNamedPipe(pipeName)` on the server, and any connected duplex `Stream` returned from the same connect callback on the client — the row schema (path-like name, kernel-enforced local transport, manifest-published endpoint) is identical, so platform variance is one column value, not a second topology.

## peer credentials

- Call-level peer identity is structurally absent on local transports (the call context's peer string degrades to `"unknown"` off IP), so identity is connection-level kernel evidence: the accepted `Socket` surfaces through `IConnectionSocketFeature`, and a connection middleware row (`ListenOptions.Use`) verifies credentials once per connection BEFORE protocol negotiation proceeds — per-call re-verification re-derives a connection invariant.
- The probe is `Socket.GetRawSocketOption(int level, int name, Span<byte>)` with platform rows: Linux reads SOL_SOCKET(1)/SO_PEERCRED(17) into a 12-byte ucred (pid, uid, gid as native 32-bit values, captured at connect time so a later exec cannot launder identity); macOS reads SOL_LOCAL(0)/LOCAL_PEERCRED(1) into a 76-byte xucred (version word that must equal zero, effective uid, group count, sixteen group slots) and SOL_LOCAL(0)/LOCAL_PEERPID(2) for the 4-byte peer pid (LOCAL_PEEREPID(3) for the effective pid when the peer may be a translated process).
- Enforcement versus verification is a two-layer law: the endpoint DIRECTORY mode is the kernel-enforced perimeter — sockets live in a directory created with owner-only permissions, and connect requires traversal rights, so foreign-uid processes are denied before any byte flows; the peer-credential read is the verification receipt layered on top, comparing peer uid to the publisher uid carried in the discovery manifest and recording the pair as connection evidence. Relying on socket-FILE mode alone is the rejected form: file-mode enforcement on socket nodes is platform-inconsistent, directory traversal is not.
- A failed credential check tears down the connection with a typed rejection carrying (expected uid, observed ucred) — it never reaches the protocol layer, so a credential failure is distinguishable from every wire failure by its absence of any HTTP/2 evidence.

## discovery manifest law

- The manifest is the single attach contract: one small file per endpoint carrying socket path, publisher pid plus process-start stamp, epoch, contract checksum set, payload cap pair, and codec identifier — every field either routes the dial or gates it; a manifest field no gate reads is dead weight.
- Publication is atomic-by-rename: serialize to a temp file IN THE SAME DIRECTORY, then `File.Move(temp, final, overwrite: true)` — same-volume rename is atomic at the filesystem layer, so readers observe either the old manifest or the new one, never a torn write; writing the temp file in a different directory silently downgrades the move to copy-plus-delete and forfeits atomicity, which is the one way this protocol corrupts. `File.Replace` is the variant when the displaced generation must be retained as evidence.
- The credential boundary is established before any artifact exists: `Directory.CreateDirectory(path, UnixFileMode.UserRead | UnixFileMode.UserWrite | UnixFileMode.UserExecute)` creates the endpoint directory with owner-only mode atomically at creation — creating then chmodding leaves a window where the default mode admits foreign readers; `File.GetUnixFileMode` is the audit read, `File.SetUnixFileMode` the repair write.
- Staleness is a two-probe ladder: the advisory probe checks the manifest's pid (`Process.GetProcessById` failing with an argument failure means dead; alive demands the `StartTime` match the manifest's start stamp, or the pid was recycled); the authoritative probe is a connect attempt to the socket itself — connection-refused against an existing file is the definitive dead-listener signature, because the kernel answers for the socket regardless of process-table ambiguity. Probe order is advisory-first (cheap, no side effects) but only the connect probe licenses reclamation.
- Reclamation protocol: delete the stale socket file, delete or overwrite the manifest, then bind — a racing claimant loses at bind with address-in-use and resolves by re-reading the manifest rather than retrying the delete; the bind failure IS the mutual-exclusion primitive, no lock file needed.
- The contract gate is an injected classifier at attach: identical checksum short-circuits; on mismatch the attacher fetches both descriptor projections and the injected additive-vs-breaking verdict decides — additive attaches under older-reader rules, breaking refuses with a typed fault carrying both generations. The gate runs before the first RPC, so contract drift is an attach failure with evidence, never a mid-call decode failure. Diff algebra and verdict semantics arrive settled from the contract-evolution law; the manifest owns only when the gate runs and what refusal carries.

## framing and payload caps

- Raw-stream lanes (artifact corridors outside the RPC layer) frame as: fixed header — version byte, flags byte, 32-bit little-endian body length, 64-bit checksum — then body; the checksum covers length-plus-body so truncation and length corruption are both detected, and the fast-hash row (`XxHash3.HashToUInt64`) is the default with the CRC row reserved for foreign-interop framing; binary field access rides the little-endian primitives, never manual shifts.
- Receive order is a memory-amplification guard: `ReadExactlyAsync` the fixed header, validate the declared length against the manifest cap BEFORE any body allocation, then `ReadExactlyAsync` the body and verify the checksum BEFORE parsing — a malicious or corrupt length field costs a header read and a comparison, never an allocation; parse failures after a valid checksum indicate contract drift, checksum failures indicate transport corruption, and the two must land as distinct typed faults because their remediation differs (re-gate versus redial).
- The producer-side cap law: compute the exact wire size from the message's own size accounting BEFORE serializing, compare against the manifest cap, and reject oversize with a typed fault carrying (size, cap, frame kind) — the rejection costs a size computation, whereas post-serialization detection (the RPC layer's resource-exhausted path) has already paid allocation and encoding for a payload that was never sendable. The cap is ONE negotiated value published in the manifest and shared by both ends of the seam; asymmetric caps (sender permits what the receiver rejects) are a configuration defect the manifest's single cap field makes unrepresentable.
- Cap sizing is a policy row per frame kind, not one global number: control frames cap small (forcing chatty contracts to stay chatty), artifact frames cap at the corridor budget — a single undifferentiated cap either starves artifacts or lets control lanes balloon.

## epoch and rehandshake redial

- The epoch is a monotonic generation counter in the manifest, bumped per listener generation; the socket path embeds the epoch (one short suffix inside the path budget), so a successor never contends with its predecessor's file or its predecessor's lingering connections — the old generation drains on the old path while the new generation binds fresh, and stale-file reclamation becomes a cleanup concern instead of a liveness blocker.
- Redial discriminates on epoch: a wire failure triggers a manifest re-read; same epoch means transient — the hop's resilience owner handles retry and the topology does nothing; advanced epoch means the peer restarted — the channel to the old path is disposed, the contract gate RE-RUNS (a restart may carry a new binary with a new contract generation), credentials re-verify, and a fresh channel dials the new path. Skipping the re-gate on epoch advance is the named defect: it assumes binary identity across restarts, which is exactly what epochs exist to deny.
- Epoch advance with an unchanged checksum is the cheap common case (process bounce); the full rehandshake cost is paid only when the checksum also moved. In-flight work at epoch advance resolves by receipt: completed-and-receipted survives, unreceipted re-issues against the new generation under the invocation route's idempotency key — the redial law never re-sends bytes, it re-issues intents.
- Manifest watching is pull-on-failure, not push: re-reading the manifest on wire failure is sufficient because the only consumer of epoch change is a peer that just observed a failure; filesystem-event subscriptions add a liveness dependency on event delivery for a value that is only actionable at failure time.

## attach choreography

- Companion attach is a strictly ordered ceremony where the manifest is the commit point: (1) the owner pre-creates the owner-only endpoint directory; (2) the companion binds its epoch-suffixed socket and starts serving; (3) the companion publishes the manifest atomically LAST. Observers therefore never see a manifest pointing at an unbound socket — manifest presence implies a listener existed at publish time, and the staleness ladder covers everything after. Detach inverts the order: manifest first (un-publish), then listener teardown, then socket-file cleanup rides socket dispose — the manifest is the last artifact born and the first destroyed.
- Post-dial readiness is the wire health surface: the attacher watches the peer's health stream until the serving signal before routing traffic, and treats the shutdown-broadcast not-serving write as the drain edge — attach choreography consumes the health protocol, it never invents a parallel readiness ping.
- The embedded-foreign-surface case (a foreign runtime hosting a suite process, or a plugin seam between runtimes) uses the SAME manifest grammar with an attach-mode column: socket transport versus in-process handoff; the manifest is the only contract crossing the seam — no shared types, no linked assemblies, payloads opaque contract bytes until the owning contract decodes them. A foreign surface that imports suite types to "simplify" attach has converted a data contract into a binary coupling, which the next epoch bump breaks silently.
- One invocation route: remote commands, deep-links, and journal replay all enter through a single invocation surface sealing ONE receipt family; each invocation carries an origin discriminant (remote | deep-link | replay) and an idempotency key, so replay is re-presentation of the same invocations through the same gate — origin-specific entrances would fork the receipt family and make replay semantics diverge from live semantics by construction.
- The message bus is the named rejected form for suite-internal invocation: a broker introduces delivery-order ambiguity, at-least-once duplication that the idempotency key must then fight, an independent retry owner conflicting with the hop's one-retry-owner law, and an unGated path around the contract checksum gate — every property the manifest-plus-route design provides, the bus re-opens as a distributed-systems problem.

## divergent

- UDS-credentials-discovery as one capsule: directory creation with mode, bind, credential middleware, manifest publication, staleness ladder, and reclamation compose into a single endpoint-lifecycle owner with two public operations — publish (owner side) and attach (peer side) — each returning a receipt carrying every gate's evidence (mode verified, creds matched, epoch, checksum verdict, probe outcome). The capsule forecloses the scattered form: ad hoc socket paths at call sites, chmod-after-create windows, pid checks without start-stamp pairing, and attach code that dials before gating. Everything between "directory exists" and "channel ready" is one fold over gate rows, so adding a gate (a new manifest field) is one row plus one receipt field.
- Framing-caps-epochs fusion — the corridor invariant: a frame is only ever constructed below the cap (producer pre-check), only ever admitted below the cap (pre-allocation length gate), only ever parsed after integrity (checksum-then-parse), and only ever interpreted inside a generation (epoch in the manifest that set the cap). The four checks are one ordered predicate chain whose failure taxonomy is disjoint by construction — oversize, truncated, corrupt, drifted — and each maps to exactly one remediation: re-chunk, re-read, redial, re-gate. A corridor implementing any subset re-discovers the missing failure class in production as the ambiguous one.
- Attach choreography as ordering algebra: publish-last and destroy-first make the manifest a linearization point, turning multi-artifact state (directory, socket, listener, manifest) into a single observable bit with well-defined edges; the staleness ladder is then exactly the recovery for the one history publish-last cannot rule out — death after publish. Every alternative ordering admits an observable lie (manifest without listener, socket without manifest owner), and each lie costs a bespoke probe; the canonical order needs precisely one. The same algebra gives crash-consistent UPGRADE for free: a successor publishes its manifest only after its own gate-ready listener exists, so even a mid-upgrade crash leaves peers attached to a coherent generation or cleanly refused — never half-attached.
