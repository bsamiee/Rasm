# Key Union Totality As Proof Obligation

[THE_KEY_UNION_IS_A_CARRIED_PROOF]:
- The closed key union `keyof typeof Table` is not a convenience type — it is a proof object the table carries: the proposition "every value of this key type names a present row" is true by construction and inhabited by the literal union itself, so a parameter typed `keyof typeof Table` is a value that already holds the membership proof, never a value to be re-checked.
- The indexed access `Table[k]` is the proof's CONSUMER: the indexed-access type rule yields `Row` (never `Row | undefined`) exactly because `k`'s type is provably a subset of the table's own key set, so the access discharges the obligation at the type level — the totality of the read is the key type, not a runtime guard, a length check, or a `?? throw`.
- The discharge survives the strictest checker posture: under no-unchecked-indexed-access an access `Obj[s: string]` is `Row | undefined`, but `Table[k: keyof typeof Table]` stays `Row` because the key is not an arbitrary index but a member of the literal union the object's own type declares — the flag forces `undefined` only on indices the type cannot prove present, and `keyof typeof` is the proof that closes that gap.
- The proof is single-source and self-renewing: the same literal that seeds the rows seeds the key union, so adding a row both extends the proof's domain and breaks every consumer that enumerated the old domain — the obligation is re-discharged automatically wherever a key flows from `keyof typeof`, and re-incurred loudly wherever a key was hardcoded against the old union.

[INDEXED_ACCESS_IS_TOTAL_ONLY_FROM_THE_UNION]:
- Totality of `Table[k]` holds only when `k` ARRIVES carrying the proof: a `k` typed `keyof typeof Table` reads total, a `k` typed `string` reads `Row | undefined` (or lies as `Row` without the strict flag), so the read's totality is a property of the key's PROVENANCE, never of the table — the same bracket access is total or partial depending on what proof its key carries.
- The boundary between the two reads is exactly the carrier: a key still inside the closed union indexes directly `Table[k]`, a key whose membership is unproven reads through `Record.get(Table, key): Option<Row>` — and `Record.get`'s key parameter is `NoInfer<keyof typeof Table>`, so a statically-literal foreign key is rejected at compile time and only a genuinely runtime-decided key reaches the `Option` rail.
- A bare `Table[foreignKey]` typed `Row` where `foreignKey: string` is the totality LIE: it asserts a proof the key does not carry, producing `undefined` at runtime under a `Row`-typed binding the checker believes total — the defeat is not a widened table but an unproven key indexing a proven one, and the repair is to carry absence as `Option` through `Record.get`, never to widen the access.

[ERASURE_IS_STRUCTURAL_BECAUSE_THE_PROOF_IS_NOT_RECONSTRUCTIBLE]:
- The proof cannot be rebuilt from a runtime structure because the language has no dependent path from an array's element VALUES back to a key UNION: `Record.fromEntries(entries): Record<NonLiteralKey<Entry[0]>, Entry[1]>` widens the key not as a library choice but as the only sound type, since the checker cannot prove the reconstructed key set equals the original literal union — the erasure encodes a structural impossibility, which is precisely why it is silent.
- `NonLiteralKey<K> = K extends string ? IsFiniteString<K> extends true ? string : K : symbol` is the proof-erasure operator: a finite literal key is recognized as carrying a closed proof and is therefore the very thing widened to `string` (the proof is discarded), while a symbol key collapses to `symbol` — the operator names exactly the cases whose membership proof the reconstruction destroys.
- `IsFiniteString` is the proof-finiteness test walking the key character by character: it returns `true` for `""` and recurses, returning `false` the instant a segment is `string` or `` `${number}` ``, so a template-pattern key `` `v${number}` `` is proof-LESS before any transform — its key union is `string` at construction because an infinite alphabet carries no closed-membership proof to begin with, and reconstruction merely confirms what synthesis never proved.
- The erasure is invisible to a value-equality audit and visible only to a type audit: `Record.fromEntries(Record.toEntries(Table))` is a runtime identity yet a type widening, so the proof is gone with the bytes unchanged — the audit that catches it is whether `keyof typeof Result` is the literal union or `string`, never any runtime comparison, because the loss lives entirely in the discarded type.

[CARRIER_FLOW_DECIDES_PRESERVATION]:
- The carrier-flow law subsumes the per-combinator preserve/erase split: a combinator keeps the key proof iff `K` reaches only the CALLBACK position and the result key set is a union the checker can name, and discharges it iff `K` is asked to flow back into an OUTPUT KEY position whose membership the reconstruction cannot prove — so the question is never the combinator's name but whether the proof is consumed (read) or re-asserted (rebuilt).
- `Record.collect((key, value) => ...): Array<B>` is the law's clean exit: the callback receives the precise `K` so per-key totality is live INSIDE the fold, the result is intentionally keyless, and the proof is held exactly as long as it is read and dropped where it stops mattering — distinct from `Record.fromEntries`, which asks the proof back into a rebuilt key position and so discharges it to `string` while pretending the record is unchanged.
- `Record.reduce(zero, (acc, value, key: K) => ...)` folds the table to a single carrier with the precise `K` live in the accumulator step, so an aggregate over the vocabulary stays proof-aware per key without ever materializing a rebuilt record — the fold is the proof-preserving collapse where `fromEntries`-then-reduce is the proof-erasing one over identical runtime values.
- `Record.singleton(key, value): Record<K, A>` PRESERVES the literal key while `Record.empty(): Record<NonLiteralKey<K>, V>` widens — the asymmetry is the proof itself: a singleton's key is the literal argument, provably present, so a one-row table built by `singleton` indexes total, while one accumulated into `empty` carries no proof because an empty record's surviving key set is `never` widened to `string` and every later insertion adds a key the type cannot prove.

[INTERSECTION_IS_A_GUARDED_ERASURE]:
- `Record.intersection(a, b): Record<IntersectKeys<K0, K1>, C>` does not unconditionally erase: `IntersectKeys<K1, K2> = [string] extends [K1 | K2] ? NonLiteralKey<K1> & NonLiteralKey<K2> : K1 & K2`, so intersecting two literal-keyed tables PRESERVES the proof as `K0 & K1` and only erases to `NonLiteralKey<K0> & NonLiteralKey<K1>` once either side has already leaked `string` into its key union.
- The guard makes intersection the one merge whose proof-preservation is CONDITIONAL on its inputs: a clean two-table intersection keeps the closed intersection union, a single already-widened operand poisons the result, so a vocabulary that survives `intersection` proves both operands kept their proofs — the combine surfaces a prior erasure as a now-erased output rather than hiding it.
- `Record.union(a, b): Record<K0 | K1, A | B | C>` and `Record.difference: Record<K0 | K1, A | B>` preserve unconditionally because the result key set is the literal UNION of two known sets, which the checker CAN compute, while `intersection`'s result is the unknown overlap, which it cannot — so merge-by-union keeps the proof, merge-by-overlap keeps it only when neither input was already open.

[ERASED_PROOF_CASCADES_THROUGH_RESIDUAL_SUBTRACTION]:
- `Match.exhaustive(self: Matcher<I, F, never, A, ...>)` is total only when the matcher's RESIDUAL (third parameter) has subtracted to `never`; the residual is the input union minus every matched arm computed by `Types.AddWithout`/`ApplyFilters`, so exhaustiveness is the proposition "the input union is exhausted" and the key proof is what makes the input union finite enough to exhaust.
- The cascade is residual arithmetic: a matcher over an intact `keyof typeof Table` subtracts a finite literal union to `never` and `Match.exhaustive` compiles; a matcher over an ERASED key reads input `string`, and `Exclude<string, finitely-many-literals>` is still `string ≠ never`, so the exhaustive finalizer that proved totality upstream now refuses to compile downstream — the erasure does not corrupt the proof, it makes the totality claim unprovable, and the residual is the term that records exactly which members the type system still believes uncovered.
- The defeat is silent not because the build stays green but because erasure forces the consumer to ABANDON the proof: facing a residual that will not reach `never`, the call site swaps `Match.exhaustive` for `Match.orElse(f)`, which accepts any residual and returns total over the open input — the obligation is deleted by the substitution the erasure compelled, so the build is green at the cost of an opted-out proof, and a future missing arm vanishes into the catch-all instead of breaking.
- The indexed-read site is where erasure passes truly silently: `Table[k]` over a `k` widened to `string` types `Row` (without the strict flag) or `Row | undefined` (with it) but never refuses to compile, so a read keyed on an erased union lies about totality where a `Match.exhaustive` keyed on it would at least fail loud — the read trusts the proof the key no longer carries, which is why a key-erasing transform endangers indexed access more than it endangers dispatch.

[ABSURD_IS_A_PROOF_OF_IMPOSSIBLE_RESIDUE]:
- `Match.exhaustive` and `Match.orElseAbsurd` are two discharge forms of the same residual obligation: `exhaustive` requires the residual `Remaining` to be statically `never` (the checker proved exhaustion), `orElseAbsurd` returns the matched union ASSERTING the residue is unreachable at runtime when the checker cannot narrow it to `never` — so a vocabulary whose totality the type system cannot prove but the program guarantees discharges through `orElseAbsurd`, never a fabricated fallback member.
- The fallback terminals are the residual carriers: `Match.option` puts the unmatched residue in `Option.None`, `Match.either` carries the residual type itself in `Either.Left` (the `Left` IS the uncovered union members), and `Match.orElse(f)` consumes the residue as a function argument typed to the exact uncovered complement — so a partial dispatch over a vocabulary surfaces precisely which members remain at the type level, the residual being the dispatch dual of `Exclude<keyof typeof Table, Covered>`.
- A designated catch-all member uses `Match.orElse`, a forbidden-unhandled vocabulary uses `Match.exhaustive`, a structurally-impossible residue uses `Match.orElseAbsurd` — the three are not interchangeable styles but distinct claims about the residual: resolve it, prohibit it, or assert it unreachable, and choosing `orElse` where `exhaustive` would compile is silently opting out of the proof a closed key union already carries.

[THE_OBLIGATION_HAS_A_BOUNDARY_DUAL]:
- The compile-time proof a structural table discharges is RE-INCURRED at every boundary where a foreign value names a key, and its runtime carrier is `Schema.keyof(struct): SchemaClass<keyof A>` — the codec whose admitted alphabet is exactly the struct's key set, so a boundary string naming a column discharges the membership obligation ONCE through decode and travels inward as a typed key the interior reads total with no re-proof.
- `Schema.keyof` derives the proof carrier from the struct in one hop, so adding a field widens the admitted key alphabet with zero codec edits — the runtime proof is recomputed from the same single source the compile-time `keyof typeof` reads, never a parallel `Schema.Literal` of column names drifting from the struct it claims to describe.
- The structural proof and the codec proof answer different questions over one key set: `Record.has(Table, key)` is the structural presence probe (boolean) and `Record.get` the structural narrowing read (`Option`), while `Schema.is(Vocab)` is the codec guard (typed refinement) and decode the codec narrowing read — the structural table cannot synthesize a runtime guard because its proof is a compile-time fact with no value, so a value of unknown provenance MUST cross the codec, after which it indexes the structural table total.
- `Struct.keys(o): Array<(keyof T) & string>` and `Record.keys` are the runtime enumerations that carry the proof out as a typed array, the array-shaped mirror of `keyof typeof`, so iterating the vocabulary at runtime stays keyed by the closed union — the enumeration is the one runtime value the structural table CAN produce that still holds the membership proof, distinct from `fromEntries` of the same entries which discards it.

[THE_PROOF_IS_THE_ONLY_THING_DELETING_THE_FALLBACK]:
- Indexing a closed function table by its own `keyof typeof` is total without a fallback arm precisely because the key union IS the case set — the proof the key carries makes `Table[k].run(arg)` exhaust every case, so a `default:`, an `orElse`, or a `?? throw` at the call site is the absent-case rail the table already proved unnecessary and is deleted, not because the cases happen to be covered but because the type forbids an uncovered key from arriving.
- The fallback returns the moment the proof is erased: a dispatch keyed on a widened `string` cannot prove the case set exhausted, so the call site re-grows the `default`/`orElse` the closed key deleted — the presence of a fallback arm on a vocabulary dispatch is itself the diagnostic that the key proof was lost somewhere upstream, the dispatch having degraded from total-by-type to partial-by-runtime.
- One proof discharges two obligations at once — case-exhaustion and pre-access membership — so a `keyof typeof`-keyed read needs neither a dispatch fallback nor a membership probe before access, while a foreign-keyed read re-incurs BOTH: it must resolve absence (the membership obligation, carried by `Option` through `Record.get`) before it can dispatch, the two checks that reappear together being the single deletion the carried proof performed once.
