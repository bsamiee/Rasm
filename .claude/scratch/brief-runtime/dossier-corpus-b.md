Verdicts sit high (8-9) because the burden-of-proof bar is met on capability; the campaign-defining rulings are structural (layering + missing owner), not per-page rebuilds.

## VERDICT CANDIDATES (strongest structural rulings, most-severe first)

1. **RUNTIME MODULE GRAPH CONTAINS HARD IMPORT CYCLES.** As-designed module-level imports produce at least: `metrics ↔ lanes` (metrics.md L40 needs `DrainReceipt` for `ZERO_DRAIN` at load; lanes imports `Metrics`), `receipts ↔ lanes` (receipts.md L53,61,112 bind `DrainReceipt` at load; lanes imports `Receipt`/`Signals`/`Redaction`), and `resilience → metrics → lanes → resilience` (resilience.md L303-307 CALLS `Metrics.retry_hook()` in the module-level `RETRY_HOOKS` tuple). Realized verbatim, these raise `ImportError: cannot import name … from partially initialized module`. ROOT CAUSE: the drain vocabulary (`DrainOutcome`/`DRAIN_COLUMNS`/`DrainReceipt`, lanes.md L60/79/94) is OWNED by execution/lanes but CONSUMED by observability (metrics+receipts), while lanes consumes observability (Metrics.observe, Signals.emit via `@drained`) — inverting the intended "observability is the local-evidence leaf everything feeds" layering. CHARTER AS SHOULD BE: hoist the drain taxonomy (at minimum `DrainOutcome`/`DRAIN_COLUMNS`, likely `DrainReceipt`) into a lower shared owner both strata depend UP on (faults, or a new leaf `vocab`/`shapes` module), OR defer lanes' observability imports to function scope. The design currently does neither and never acknowledges the cycle.

2. **THE PYTHON WIRE VOCABULARY (`rasm.runtime.shapes`) IS UNAUTHORED.** wire.md L116-120 + serve.md (via `WireProtoCodec[S,M]`) are generic over 16 canonical msgspec Structs + `FaultDetail` imported from `rasm.runtime.shapes`, which has no design page and no class definitions anywhere in the corpus (verified grep). The codec, the `_PROTO_VOCABULARY` registry, the servicer dispatch, and the CausalFrame-bearing `FaultDetail` all reference shapes that do not exist. This is the single largest COVERAGE gap in corpus-b and the natural home for the `PY_WIRE_ALIGNMENT` count-prefix canonical-writer law (RASM-COMPONENT-PARADIGM-DECISION [AMENDMENTS]) the queued campaign must build against. CHARTER AS SHOULD BE: a `transport/shapes.md` (or `wire`-owned `[SHAPES]` region) authoring the 16 canonical Structs as the Python mirror of the C# wire vocabulary, single-mint-decoded, with the FaultDetail two-half HLC carrier and the count-prefixed canonical-byte contract recorded as the wire law.

3. **CODEMAP-VS-REALITY LAYOUT DRIFT.** ARCHITECTURE.md maps nested sub-domain source (`observability/receipts.py`, `transport/wire.py`) but all 13 pages import flat `rasm.runtime.<module>`; the realized package is flat and the codemap omits the two real modules the code depends on (`shapes`, `content_identity`). Ruling: either flatten the codemap to the true module set (adding shapes + content_identity) or commit to nested packages and fix every import — the current split is a standing seam-map lie.

4. **TRACE-CONTEXT DOUBLE-HANDLING AT THE SERVE INGRESS.** serve.md wires `aio_server_interceptor` (native W3C extract + SERVER-span parenting, L111-112) AND manually re-extracts via `Signals.continue_inbound` + re-activates via `Signals.attach(parent)` (L119,167). If the interceptor already parents/activates the SERVER span, the manual `attach(remote-parent)` re-roots handler spans to the remote parent (siblings of SERVER), fracturing the local tree; at minimum it is redundant extraction. receipts.md/telemetry.md justify the extract/attach SPLIT but never reconcile it with the interceptor's own extraction. Ruling: define the interceptor as span-emission-only (extraction disabled) with `continue_inbound`/`attach` owning parent linkage, or drop the manual pair and let the interceptor own both — pick one context authority.

5. **`measured`/synchronous-instrument OWNER SELF-DUPLICATION (metrics).** The synchronous-recorder definition is restated across `SyncInstruments` fields, `_seed_sync` literals, `_HIST_SLOT`, and `INSTRUMENTS` rows (metrics.md L134-196); `_seed_sync` re-implements the install fold; the "one row + one slot" growth claim (L142) undercounts the real 3-4 edit sites. Ruling: seed `_sync` by folding `INSTRUMENTS` against the no-op API meter, deleting `_seed_sync` and the name/unit restatement so the table is the single source.

6. **`latched` OWNERSHIP MISPLACEMENT.** A generic one-shot idempotent-init aspect (telemetry.md L228-243) lives in the OTLP-install owner and is imported by metrics. Ruling: relocate to the lowest owner (`faults`), making it a first-class runtime primitive rather than a telemetry export.

7. **[RESEARCH] PAGE-CRAFT INCONSISTENCY.** Full / absent / empty-stub across the eight pages (wire.md L379-380 the empty instance). Ruling: normalize — settled pages either all carry a research appendix or fold research into the family cards; delete the wire stub regardless.

8. **DRAIN-RAIL EVIDENCE LOSS (telemetry).** The host-drain fault subject is the constant `"resource"` for every signal (telemetry.md L212) and CLASSIFY drops the `TimeoutError(signal.value)` message, so a wedged exporter cannot be attributed to traces-vs-logs-vs-metrics in the accumulated `BoundaryFault`. Ruling: thread `subject=kv[0].value` so the drain rail keeps the per-signal identity it claims to.
