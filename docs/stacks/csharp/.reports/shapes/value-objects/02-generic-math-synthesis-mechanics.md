# Generic-Math Operator Synthesis Mechanics

[DUAL_PATH_OPERATOR_DETECTION]:
- Operator capability on the key is computed once into a per-key flag record and drives every arithmetic and relational satellite; the record is filled by two independent passes that never re-probe. The interface pass walks the key's implemented interfaces and, on finding a homogeneous generic-math operator interface, grants the entire operator family in one assignment — both the unchecked and checked flag of each arithmetic family, all four relational flags for comparison — with no per-operator method inspection. The method pass then sets flags individually from the presence of each corresponding operator method, only for families the interface pass left dark. A key either lights up a whole family through an interface or accretes operators one method at a time.
- The interface pass demands exact self-self operand symmetry: the framework numerics interface by name and namespace, three type arguments, the first two both equal to the key. The result argument is unconstrained — a key whose binary operator yields a third type still triggers full-family generation — while a key implementing only the mixed-operand form (key against a foreign right operand) fails the second-argument equality and is invisible to the interface pass entirely. Operand symmetry, not result-type identity, is the admission gate.
- The method pass classifies an operator as arithmetic only when it is static, takes exactly two by-value parameters, and has return type and both parameter types all equal to the key type. A hand-written key operator returning a delta type, or mixing operand types, satisfies neither pass and produces no owner operator — the owner's generated arithmetic is strictly the homogeneous closed-operation subset of the key's algebra; richer key algebra surfaces only as hand-written domain operations on the owner.
- Checked detection is a separate flag bit fed by a separate operator-method name: a key carrying only the unchecked operator yields an owner with no checked variant, and a key carrying only the checked operator yields the checked operator with no generic-math interface — the interface emits only when the unchecked bit is set, because the checked member lives inside the same interface and is meaningless without the default one. The owner mirrors a partially checked-capable key exactly rather than fabricating the missing mode.

[INTERFACE_EMISSION_GATING]:
- The equality-operator interface is the unconditional floor — always emitted with self-self type arguments because it derives from the owner's own equality, not from any key operator. Arithmetic and ordering participation are earned from the key; equality participation in generic math is reached by every owner.
- A partial relational-operator set on the key produces the matching partial operator subset on the owner and suppresses the comparison interface, whose declaration is gated on full four-operator presence — a silent capability split where source-level operator use compiles while generic-constraint dispatch rejects the owner. The string-key and comparer-accessor arms of the comparison gate manufacture the interface over keys with no relational operators at all, so the lift to full interface citizenship is policy-driven where the key's own algebra cannot reach it.

[RESULT_TYPE_NARROWING_CAST]:
- Every arithmetic body wraps the raw key arithmetic in a cast back to the key type exactly when the key is one of the integral types the language promotes under binary arithmetic — the byte, signed-byte, short, unsigned-short, and char family — computed from the key's special type, never from operator inspection. The cast target is the key type, not whatever result type the key's operator interface declares, so a key designed to widen under arithmetic has that widening erased at the owner boundary: the owner's arithmetic is closed over the key by force.
- The narrowing cast sits inside the checked wrapper for the checked variant and outside it for the unchecked variant, so one narrow-key operator pair carries two different failure surfaces: unchecked promotes, computes, truncates silently, then re-admits the wrapped value — an out-of-range sum validates its modular residue with no signal under a permissive invariant — while checked computes under overflow detection on the wide type and throws before any narrowing. Identical operands, divergent failure species, selected by the call site's checked context.
- Mixed-operand bodies null-guard their two operands asymmetrically and structurally: each operand's guard is computed independently from that operand's own nullability, so a class owner with a non-nullable struct key produces a mixed operator that throws on a null owner and can never throw on the key — the asymmetry is per-operand origin, not a single owner-side rule.

[OWNER_AGAINST_KEY_EQUALITY_ROUTING]:
- The key-overload equality operator routes through the owner's configured equality comparer, not default key equality — `owner == rawKey` tests under the same case or accessor policy that governs owner-to-owner equality. It is a policy-aware membership test against the owner's value space, never a proof the raw operand is admissible: a raw value the factory would reject simply returns unequal, indistinguishable from an admissible-but-different value. The clamp between the two operator-generation settings keeps the mixed-operand equality interface co-present whenever the mixed-operand comparison interface is requested, so the heterogeneous pair never splits even where the source attribute spells only one.

[ALGEBRAIC_CEILING]:
- No identity-element or aggregate numeric interface is ever synthesized — no additive or multiplicative identity, no min-max carrier, no full number interface — because an identity element is a constant minted outside admission, an unvalidated owner value the generator refuses to manufacture. The owner's advertised structure under each operator is a magma, never a monoid: the unit is unrepresentable without bypassing the factory, so every seed, zero, or bound a generic kernel needs enters as an admitted parameter, and every generated operator body routes its result back through the factory — generated arithmetic on a range-invariant owner is a validation site at every application, by construction.
