# [PY_COMPUTE_GRADUATION]

Graduation records move useful Python evidence into the C# owner system. A graduation receipt describes handoff; it does not mint C# receipts or authorize product runtime behavior.

## [1]-[GRADUATION_OWNER]

[GRADUATION_RECEIPT]:
- Owns: source package, target C# owner, subject, evidence bundle, residual limits, and decision route.
- Input: study receipt, artifact bundle, data exchange bundle, or model asset manifest.
- Output: C# handoff record for `Rasm.Compute`, `Rasm`, or another named owner.
- Boundary: no C# `ComputeReceipt`, no benchmark claim, no source generation.

[HANDOFF_AXIS]:
- Owns: handoff kind values for solver, symbolic derivation, model asset, array layout, unit law, and uncertainty law.
- Output: one handoff axis consumed by graduation receipt.
- Boundary: no duplicated C# owner taxonomy.

## [2]-[CROSS_OWNER_RULES]

- Solver evidence routes to `Rasm.Compute` unless it belongs in the RhinoCommon-aware kernel.
- Symbolic derivation routes to C# source owners only after the derivation is stable and reproducible.
- Model asset evidence routes to C# model lanes only after artifact digest and IO metadata exist.
- Unit and uncertainty evidence routes to C# quantity/unit owners only as policy rows.

## [3]-[RED_TEAM]

- Reject handoff records that claim production readiness.
- Reject graduation without data/artifact evidence references.
- Reject Python-only benchmark conclusions.
- Reject C# source-shape claims absent from the C# owner planning pages.
