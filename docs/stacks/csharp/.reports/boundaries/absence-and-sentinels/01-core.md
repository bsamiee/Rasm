# Absence and Sentinels — Boundary Law

[SENTINEL_PROJECTION]:
- Every host sentinel — null, `IntPtr.Zero`, `nint(-1)`, `DateTime.MinValue`, `Guid.Empty`, `0`-handle, a dedicated "unset" constant, or a magic fallback value — is a foreign-language absence signal. It must be projected to `Option<T>.None` at the exact foreign-call boundary using `Prelude.Optional(nullableRef)` (reference types), `Prelude.Optional(nullable)` (value `Nullable<T>` via `HasValue`), or an explicit discriminant expression (`h.IsInvalid ? None : Some(h)`) for handle types. `Optional(A? value)` calls the reference overload; `Optional(A? value) where A : struct` calls the struct overload — verified: both are zero-allocation inline conversions.
- `Optional` returns `default(Option<A>)` — the `Option<A>.None` static field — for null and for `Nullable<T>` where `!HasValue`. The implicit `operator Option<A>(A? a)` on `Option<A>` invokes `Prelude.Optional` directly, so an explicit `Optional(...)` call and a cast-implicit are byte-identical; the explicit form is preferred because it names the projection site.
- `SafeHandle.IsInvalid` and `SafeHandle.IsClosed` are the dual-sentinel surface for native handles. Project to `Option<T>` immediately after construction: `handle.IsInvalid || handle.IsClosed ? None : Some(handle)`. Once inside the domain, the `Option` carries lifetime; the raw handle is never stored. `DangerousAddRef` must be called with `ref bool success` inside the projection site if the handle is held across a suspension point — the `success` flag is itself a boolean absence signal that must be checked before the ref-counted access proceeds.
- A sentinel value must not enter a persisted shape. `Option<A>` implements `ISerializable` (verified on `readonly struct Option<A>`) and its serialization ctor reads `IsSome` + `Value`; a persisted field of type `Option<T>` round-trips correctly. The corresponding pattern for aggregates is a nullable-valued field projected back to `Option` in the domain read-path — never a raw nullable persisted into a domain record.
- When a foreign API returns `0`-or-negative for an integer sentinel (e.g., `UnsetIntIndex = -1`), the projection must be explicit: `idx >= 0 ? Some(idx) : None`. The implicit `operator Option<A>(A? a)` only fires for reference and `Nullable<T>` types; it does not intercept value types with magic constants. Missing this produces a `Some(-1)` — a sentinel propagating inward wearing a typed wrapper.

[TYPED_ABSENCE_CARRIERS]:
- `Option<T>` is the sole carrier for presence/absence at domain boundaries. It is a `readonly struct` so no heap allocation for the `None` case; `default(Option<T>)` is `None`. Pattern matching uses `{ IsSome: true, Case: T value }` property patterns (the `Case` property returns `null` for `None` and boxes the value for `Some`) to avoid the `Match`-mid-pipeline prohibition.
- `Option<T>.ToFin(Error fail)` converts absence to a typed failure for use on the error rail: `opt.ToFin(Error.New(code, "absent"))`. `Option<T>.ToFin()` (no arg) uses `Errors.None` — the empty error — which carries no diagnostic content; always supply a domain-specific `Error` when the absence has a named cause.
- `Option<T>.ToEff(Error fail)` lifts absence into an `Eff<T>` computation; the fail overload is required — the zero-arg overload uses `Errors.None` which likewise loses context. `Option<T>.ToIO()` promotes to `OptionT<IO, A>` via `OptionT.lift<IO, A>(this)` for threading absence through `IO` pipelines.
- `OptionT<M, A>` is the monad transformer for threading absence through an outer effect `M`. The two construction paths are `OptionT.lift<M, A>(Option<A>)` for a known value and `OptionT.lift<M, A>(K<M, A>)` for lifting a successful `M` computation. `OptionT.None<M, A>()` constructs a known absent state inside the outer monad. The transformer is `Natural<Option, OptionT<IO>>` — the `Natural.Transform` from `Option` to `OptionT<IO>` is verified on the `Option` trait class.
- `HashMap<K, V>.Find(key)` returns `Option<V>` — verified: `public Option<V> Find(K key)`. All key-bounded lookups in domain structures must use `Find`, never a try-get-then-branch pattern. Missing-key is structural absence; a fallback over a missing key (`FindOrAdd`) is a state-mutation, not an absence, and must not masquerade as a default.

[CAPABILITY_ABSENCE_AS_TYPED_STATES]:
- An unavailable capability is not `Option.None`; it is a discriminated case in a closed capability family. `Option<T>` models a value that might be absent; a capability state is a richer fact — unavailable, degraded, disabled, or transitioning — and each case carries distinct fields. Collapsing all capability states to `Option.None` erases that evidence at the consumer and forces re-derivation.
- A degraded outcome is `Either<(TResult result, TDegradation evidence), TResult>` or a dedicated union where the right case carries a usable result alongside first-class degradation evidence — never a nullable companion field, never a swallowed exception, never a null result with a string message. The degradation evidence must be a typed value the caller can inspect, route, and log without downcasting.
- Capability-absent states with mandatory consumer handling are best expressed as a `[Union]`-generated closed family with at least `Available`, `Degraded`, and `Unavailable` cases. Each case carries its own payload. The total `Switch` dispatcher enforces that every consumer handles all three modalities; there is no default fallthrough. The absence of a compiler-enforced total dispatch on this family is a design hole.
- A capability in `Unavailable` state must emit a definite `Fin.Fail(terminalError)` at every operation site, not an indeterminate `Option.None`. `Option.None` is structurally indistinguishable from "not yet known" to a downstream caller — it provides no evidence about why the operation could not proceed. `Fin.Fail` with a coded `Expected` error is definite: it names the condition and stops propagation.

[INITIAL_VALIDATION_STATE]:
- A screen or form must never transiently render in a valid state before user interaction. The domain model for a UI binding must carry an explicit `Unvalidated` state distinct from `Valid` and `Invalid`. A three-case union — `Unvalidated`, `Valid(T value)`, `Invalid(Error reason)` — owns this surface. The initial state is always `Unvalidated`. Binding `Option.None` as the initial value and treating absence as "not yet invalid" is the anti-pattern: any consumer that folds `None` to a success path will render falsely clean.
- `Validation<F, A>` requires `F : Monoid<F>` for applicative accumulation — verified: `Empty<F, A>()` calls `F.Empty`. A validation-state closed union that wraps `Validation<Error, T>` for the `Valid`/`Invalid` cases and adds `Unvalidated` as a separate case is the correct dense surface; do not embed `Validation` where the `Unvalidated` state cannot be expressed.

[PROGRESS_AS_DISCRIMINATED_VALUE]:
- Declared-unknown progress is a discriminated value, not a nullable integer or a float in `[0, 1]`. A `[Union]`-generated progress family with cases `Unknown`, `Indeterminate`, and `Known(double fraction)` enforces monotone non-decreasing law: any transition from `Known(f1)` to `Known(f2)` where `f2 < f1` is a violation, detectable at the swap site with `Atom<ProgressState>.SwapMaybe(prev => prev is Known(var f) && next < f ? None : Some(next))`. `SwapMaybe` returns the prior value when `f` returns `None`, making the rejection observable as a snapshot comparison.
- An `Unknown` → `Indeterminate` → `Known` monotone progression must never regress to `Unknown` on a transient error. The boundary seam that feeds progress from a foreign source must detect any such backward transition and clamp it to `Indeterminate` or hold the prior `Known` value; it must not simply pass the foreign value through.

[DISABLED_STATE_DEFINITENESS]:
- A host state that is absent must emit a definite `Disabled` or `Unavailable` signal to consumers — never bind an `Option.None` into a UI or reactive pipeline and rely on downstream `IsNone` checks. The distinction is that `Option.None` at a subscription boundary creates an indeterminate optional that consumers must individually defend; a definite `Disabled` emission from the capability cell terminates propagation at the single seam where the foreign state is received.
- `Atom<CapabilityState>` with `CapabilityState` as the closed union owns this boundary cell. The initial value is `Unavailable` (or the domain's "before-init" case), and the first successful probe transitions it to `Available`. `SwapMaybe` is the only mutation API because it makes the failed-transition case observable. The `Change` event on `Atom<A>` fires on every successful swap — verified in the `Atom.Swap` source, which invokes `this.Change?.Invoke(val2)` after `CompareExchange` succeeds. Absent subscription deregistration, this event fires indefinitely; the seam must dispose its registration.

[TERMINAL_TYPED_RECEIPTS]:
- Native-library unavailability, store corruption, and schema-version rejection each require a terminal typed receipt — a `sealed record` deriving `Error` (using the `Expected` or `Exceptional` hierarchy) with a domain-specific code (non-zero `int` verified via `Error.HasCode(int code)`) and no internal retry path. `Error.IsType<E>()` (verified: `return this is E`) is the discriminant at recovery sites. `Error.Is(Error error)` performs code-equality for `Expected` (code != 0) or message-equality (code == 0) — prefer a non-zero code for all terminal receipts to prevent accidental equality with string-matched errors.
- A terminal receipt must not silently re-enter the system. The `Catch<F, A>(K<F, A> fa, Func<Error, bool> Predicate, Func<Error, K<F, A>> Fail)` API on `FallibleExtensions` routes recovery through a predicate; a predicate that matches the terminal receipt's code and returns a fallback is a hidden retry path. The law is: terminal receipts are either propagated unchanged or surfaced as a Fin/Eff failure; they are never transformed into a default result.
- Schema-version rejection specifically must produce a receipt that carries the stored version and the expected version as typed fields — not a formatted string. A consumer that must display or log the version mismatch must not parse a string; it reads the typed fields from the receipt.

[MISSING_ASSET_KEY_FALLBACKS]:
- An unhandled lookup failure on a key-bounded vocabulary must not throw or return null. The correct surface is `HashMap<K, V>.Find(key)` returning `Option<V>`, followed by `IfNone(defaultValue)` or `IfNone(() => deriveDefault())` at the single point where a fallback is semantically correct. Placing the fallback deeper creates invisible defaults that obscure missing-key bugs.
- When no default is semantically valid — a missing vocabulary entry is a programming error, not an expected absence — the projection at the boundary must be `Find(key).ToFin(Error.New(code, "missing key"))`, converting the lookup miss into a domain error that propagates on the rail. Calling `.IfNone(() => throw ...)` defeats the rail entirely and is forbidden under `EXPRESSION_SPINE`.

[ATOM_AND_OPTION_GATED_CELLS]:
- `Atom<A>(A value, Func<A, bool> validator)` returns `Option<Atom<A>>` — verified: `internal static Option<Atom<A>> New(A value, Func<A, bool> validator)` returns `None` when the initial value fails the predicate. This is the option-gated construction pattern: the cell itself is absent when its initial state is invalid. `atomIO<A>(A value, Func<A, bool> validator)` lifts this into `IO<Atom<A>>` and raises `Errors.ValidationFailed` (a coded `Expected` error) on construction failure.
- `Atom<Option<T>>` is the standard cell for singleton token ownership: the `None` state represents unowned/released, and `Some(token)` represents held. `SwapMaybe(current => current.IsNone ? Some(newToken) : None)` attempts a take; returning `None` from the swap function leaves the cell unchanged, and the prior `Some(token)` snapshot signals that another owner holds the token. No explicit lock is required; the CAS loop inside `SwapMaybe` provides the acquire-side ordering guarantee.
- `Atom.SwapMaybe(Func<A, Option<A>> f)` returns the post-swap value: the new value on success, the unchanged value on guard-rejected swap. Distinguishing success from rejection requires comparing the return against the known prior value or encoding the intent in the swap function's structure — the return is not a discriminated result. This is a known gap: callers that need to know whether the swap was accepted must read the atom value before and after, or encode a sentinel in the state that makes the transition observable.

[CANCELLATION_AS_ABSENCE_BOUNDARY]:
- `CancellationToken` is a first-class absence signal for time-bounded and scope-bounded computations. It is not an optional parameter with a `default` — a function that silently ignores cancellation at a foreign boundary violates the seam contract. Every foreign call that blocks must receive the token, and every suspension must check `IsCancellationRequested`.
- `CancellationToken.UnsafeRegister(Action<object?> callback, object? state)` skips `ExecutionContext` capture and `SynchronizationContext` flow — verified in source. Use `UnsafeRegister` at high-frequency projection sites where the callback is a simple state CAS or an `Atom.Swap`. Use `Register` (full capture) when the callback needs to marshal onto a specific context. The two-arg `UnsafeRegister(Action<object?, CancellationToken> callback, object? state)` passes the token itself as the second parameter, avoiding a closure capture of the token.
- `CancellationTokenSource.CreateLinkedTokenSource(ReadOnlySpan<CancellationToken>)` is the .NET 10 params-span overload — verified. A seam that must honor both an outer session token and an inner operation token creates a linked source; the linked source owns the cancellation lifetime and must be disposed at scope exit. Disposing the linked source does not cancel the originating tokens; it only releases the registration.
- `EnvIO` (the `IO` monad environment) exposes `Token`, `Source`, and `LocalCancel` — verified source. `LocalCancel` creates a new `EnvIO` with a fresh `CancellationTokenSource` while inheriting `Resources` and `SyncContext`. Use `LocalCancelWithTimeout(TimeSpan)` to scope a time-bounded absence detection window inside an `IO` computation without propagating the timeout to the outer environment.

[HEALTH_FROM_TYPED_STATES]:
- System health must be derived from typed capability states and `DateTimeOffset` instants, never from status strings or boolean flags. A health surface is a fold over a `HashMap<CapabilityKey, CapabilityState>` that produces a typed aggregate: `Healthy`, `Degraded(Seq<CapabilityKey> impaired)`, `Unhealthy(Seq<CapabilityKey> failed)`. The three-state aggregate is itself a closed union.
- `TimeProvider.GetUtcNow()` is the injectable time source for all instants stored in health records — verified: `TimeProvider.System` is the production singleton, and override in tests replaces it with `FakeTimeProvider`. A health record that stores `DateTimeOffset.UtcNow` directly is untestable and violates the seam boundary for time.
- `TimeProvider.GetElapsedTime(long startingTimestamp)` returns elapsed `TimeSpan` from a prior `GetTimestamp()` call without a second argument — verified. This is the correct surface for measuring time-since-last-probe without holding a second timestamp field.

[SERIALIZATION_SHAPE]:
- `Option<T>` is `[Serializable]` and implements `ISerializable` with `GetObjectData` writing `"IsSome"` and conditionally `"Value"` — verified source. Wire contracts that must carry optional domain values may embed `Option<T>` directly only when the serializer understands it. For foreign wire protocols (JSON, Protobuf, MessagePack), project `Option<T>` to a nullable wire field at the serialization boundary and project back to `Option<T>` on deserialization; the domain model never carries the wire representation.
- The serialization ctor on `Option<A>` reads `info.GetValue("IsSome", typeof(bool))` and `info.GetValue("Value", typeof(A))` — throwing `SerializationException` if `IsSome` is true but `Value` cannot be cast. A domain record that stores `Option<T>` as a field and is serialized with a formatter that does not support the `ISerializable` contract (e.g., `System.Text.Json` without a custom converter) will produce a broken round-trip; the boundary law is to project before serializing.
