# Owner Composition Algebra

[NESTING_CLOSURE]:
- The six owner kinds are closed under nesting: a keyed value object is a complex value object's member, a value object or smart enum is a union case's payload, a smart-enum item carries another owner as an instance column, and a frozen table keys on a tuple of owners. Composition never escapes the family — every member of a generated owner is either a primitive the language closes or another generated owner that closes itself — so a domain graph assembled entirely from owners has one provable totality story, not a patchwork of validated and unvalidated seams.
- The enclosing owner inherits its members' generated capabilities by structural composition, never by re-declaration. A complex value object whose member is a keyed owner gets that member's admission, equality regime, comparer policy, and default-safety for free, because the generated factory initializes through the member's already-validated instance and the generated equality delegates to the member's own `IEquatable<T>`. Re-validating, re-comparing, or re-checking the nested member at the outer factory duplicates capability the inner owner already proved — the outer owner validates only the cross-member invariants the inner owners cannot see.
- Admission is single-entry per owner and composes depth-first: constructing the outer owner forces every inner owner through its own factory first, so an aggregate of N nested owners admits raw material at exactly N seams, each owning its slice of the invariant, with zero re-validation between layers. The trust gradient is monotonic inward — once a value is the outer owner it carries the conjunction of every inner proof, and interior code that re-checks any layer is operating below the gradient.

[DEFAULT_SAFETY_PROPAGATION]:
- Default-hostility propagates as an interface fact, not a per-owner declaration: an owner that disallows `default` implements the marker interface, and the enclosing struct's generator walks the full interface set of every member transitively to decide whether the product can legally have a `default`. One default-hostile member at any nesting depth poisons the entire product struct upward through the closure — the analyzer computes this from the member graph, so the outer owner never restates the constraint and can never silently regain a `default` its members forbid.
- The propagation is the structural reason a class owner and a struct owner are not interchangeable under nesting: a struct value object whose key is a reference type, or a complex struct any of whose members disallow defaults, is itself forced default-hostile, which makes its own `default` inexpressible and removes it from optional-parameter and array-hydration positions. A nested aggregate therefore either stays entirely struct-and-default-safe down its spine or takes the class form at the first default-hostile member — there is no mixed lattice where the outer struct allows `default` while an inner member forbids it.
- `default`-via-array is the one breach the closure cannot police, and it inherits down the nesting: a struct owner hydrated from pooled or array storage materializes the zero pattern of every nested struct member at once, so an aggregate of struct owners needs the liveness gate at the outermost storage seam, or every layer takes the class form. The decision is made once at the storage boundary for the whole spine, not per layer.

[EQUALITY_REGIME_COMPOSITION]:
- The three identity regimes do not blend across nesting — they stack, and the outer owner's regime is fixed by its own kind regardless of its members'. A complex value object (structural identity) holding a keyed value object member compares that member through the member's own key-delegated equality, so structural identity at the outer layer dispatches into key identity at the inner layer: the regimes compose without conflict because each owner owns exactly its own level. A regular union (case identity) whose case payloads are value objects compares cases by runtime type first, then defers to the payload's structural or key equality only within the matching case.
- The pervasive case-insensitive string default is the one regime fact that leaks across layers silently: it is declared per owner and does not inherit, so an outer owner with an ordinal-sensitive string invariant composed over an inner string-keyed member that left the default in place yields an aggregate whose two layers disagree on what equal means — the outer compares ordinally, the inner case-insensitively, and two aggregates differing only in an inner member's case are unequal at the outer comparison yet collide at the inner hash. Ordinal sensitivity must be declared at every string-bearing layer of the spine independently; there is no inherited string-comparison stance.
- Equality and ordering comparers are a mandatory pair at every layer, and the pairing obligation does not telescope: a nested member whose comparer policy declares equality must also declare ordering, or hashing and sorting disagree at that layer alone while the outer layers stay consistent — the inconsistency is local to the under-specified member and invisible until that member participates in a sorted container. Identity-without-order is a legal per-layer stance only when declared explicitly at that layer, and the outer owner cannot grant it to an inner member.

[CONVERSION_DIRECTION_ALGEBRA]:
- Conversion direction encodes evidence flow and the directions compose into a typed gradient: owner-to-key is implicit because erasing evidence is free, key-to-owner is explicit because creating evidence validates, and nesting chains these so that projecting an aggregate down to its innermost primitive is a sequence of implicit erasures while reconstructing it upward is a sequence of explicit admissions. A value travels outward-to-primitive frictionlessly and inward-to-aggregate only through validation at each layer — the asymmetry is the type system encoding the direction of trust, and a use site that reconstructs without the explicit cast is rejected.
- Ad-hoc union conversion mirrors the keyed direction — value-to-union implicit, union-to-value explicit — so an ad-hoc union nested as a member of a complex value object admits its raw alternatives implicitly at construction and projects out explicitly, keeping the ingress-implicit/egress-explicit polarity consistent across the structural and union layers of one aggregate. Mixing the polarity between layers (implicit egress at one, explicit at another) is the readability defect the unified default prevents.
- A single accessibility knob token-gates the entire ingress surface of an owner at once — constructors, conversion operators, and factory verbs move together — and under nesting this means the outer owner can hand-write a factory that is the only ingress while its members each independently token-gate their own ingress. The aggregate has one composed admission funnel whose every layer is private except the deliberately exposed verb; no layer leaks a public constructor that would let interior code synthesize an inner owner past its invariant.

[DISPATCH_COMPOSITION]:
- Nested union dispatch is granularity-polymorphic on the static type of the reference, not the runtime value: a branch-typed reference dispatches over that branch's cases while a root-typed reference dispatches over the root arms with the intermediate appearing as a single arm. An intermediate node carrying its own union attribute generates its own dispatch that hides the root's, so the same runtime instance is matched at coarse or fine granularity depending only on the declared type at the call site — one tree exposes a lattice of dispatch granularities selected by reference typing, and the stop-at overload declares the coarse grouping at the owner rather than at each call site.
- Exhaustiveness composes multiplicatively under nesting and the compile-break surface is the product of the layers' growth signatures: adding a root case breaks every root-granularity site, adding a branch case breaks every branch-granularity site, and the two break sets are disjoint because they bind by named argument at distinct dispatch surfaces. Choosing the dispatch granularity at each consumer is therefore choosing which growth axis that consumer is coupled to — a branch-granularity consumer is immune to sibling-branch growth and a root-granularity consumer is immune to within-branch growth.
- State-threaded dispatch threads `allows ref struct` state and result through every nesting level, so a span can be the dispatch state of an outer union whose arm re-dispatches an inner union with the same span, capture-free, the whole way down — a catamorphism over a nested owner graph runs with zero allocation and zero boxing because every generated dispatch surface in the spine declares the ref-struct-tolerant signature. A recursive owner folds two ways simultaneously on one declaration: case-owned recursion through overridden abstract members on the base, and call-site recursion through dispatch re-entry, and the two interact without interference because the abstract-member route never changes a call site while the dispatch route breaks every call site on growth.

[METADATA_BRIDGE]:
- The owner taxonomy is itself a generated `[Union]` — its cases are the six kinds, it dispatches through its own exhaustive state-threaded `Switch`/`Map`, and infrastructure that must discriminate owner kind rides that dispatch instead of bespoke attribute reflection. A runtime type resolves to its metadata case in one hop through a cached lookup that walks base types to find the static-abstract metadata property, so a mapper, codec registry, or schema emitter is written once against the taxonomy union and inherits a compile break if the taxonomy itself ever grows a kind.
- The keyed metadata case carries the key projection redundantly as both a compiled `Delegate` and a `LambdaExpression`, which is the mechanism that keeps a private-key owner query-translatable: an expression-translating consumer composes key access from the lambda tree without the owner ever exposing a public key member, so encapsulation of the key and participation in expression-based query translation are not in tension. The keyed case additionally carries the inverse `ConvertToKey` delegate and a `TryGetFromKey` validator, so the full admit-and-project pair for any keyed owner is reachable from its runtime type alone.
- A static-abstract invoker bridges from a runtime `Type` to the static factory contract, calling the owner's static `Validate` through the constrained generic without the caller naming the owner type at compile time — the fully dynamic edge where an aggregate is reconstructed from wire or storage runs admission through this invoker per layer, so even reflection-driven hydration passes through every nested owner's real validation rather than bypassing it. The metadata namespace is declared unstable, which is the design statement that this bridge belongs to infrastructure adapters alone: domain code holds the typed owner and never reaches for the mirror, and any domain-layer use of the lookup is a missing typed surface.

[COMPOSITION_SPAM]:
- Re-validating a nested owner at the enclosing factory is the subtle composition defect: it reads as defensive but it duplicates a proof the type already carries, widens the outer factory with checks that can never fail, and couples the outer invariant to the inner owner's internal rule so that tightening the inner rule silently changes the outer factory's behavior. The outer factory validates only the relations between members that no single member can see; everything else is already admitted.
- Flattening a nested aggregate into one wide complex value object with all the leaf primitives as direct members discards every inner owner's admission, comparer, and default-safety and re-scatters those invariants as ad-hoc checks across the outer factory — the inverse of collapse. The pressure to flatten usually traces to a serialization shape, and the correct move is one object factory projecting the aggregate to its wire grammar while the interior stays the composed owner graph, never collapsing the domain spine to match the wire.
- An aggregate whose every layer re-declares the same string-comparison or comparer policy that an outer policy owner could fix once is policy duplication across the spine; the correspondence belongs to the vocabulary or policy owner that selects the members, derived once, not restated at each nesting level where it can drift out of agreement with the others.
