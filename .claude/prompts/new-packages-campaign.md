[SESSION_ORCHESTRATION_DISCIPLINE] The mechanics of running a session, distinct from the campaign law:
- Commit by explicit pathspec only: `git commit -m <msg> -- <enumerated planning paths>` of the gate's files, never `git add -A` and never a foreign path. The orchestrator is the sole committer and the sole ledger writer; workflow agents author content and return it, and never run git. Keep two or fewer; prefer resuming a near-done run from cache (`Workflow({scriptPath, resumeFromRunId})`, cached prefix instant) over launching new, and `TaskStop` a still-running workflow before resuming it.
