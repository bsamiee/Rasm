## [1]-[OPEN]

[NEURAL_FIELD_SURROGATE]: Promote the solver `Surrogate` reduced-order duality from a GP/POD column into a neural-field surrogate row evaluated on the model lane.
- The `solver#OPTIMIZER_LANE` `Surrogate` already carries the reduced-order duality (Gaussian-process covariance Cholesky, POD/SVD reduced basis) behind the solve contract; a neural-field row trains a compact implicit-field network (a coordinate-MLP or Fourier-feature field) on prior solve results and evaluates it through the `models#INFERENCE_MODES` OrtValue run so a parametric design point returns a sub-millisecond field prediction the optimizer treats as an error-bounded surrogate of the full solve.
- Unlocks interactive design-space exploration: the `solver#SWEEP_AND_BUDGET` N-dim DOE sweep evaluates thousands of parametric points against the learned field instead of the full FEA/CFD solve, the `FrameBudget` governor returns the surrogate field within a frame and refines the seed points async, and the surrogate error bound gates fall-through to the true solve — a wrong fast prediction is poisoned by the same residual witness the dense lane already imposes.
- Draws on the 2025-2026 maturation of neural-operator and implicit-neural-field surrogates for PDEs (DeepONet/FNO-class operators, coordinate networks) and the existing one-content-keyed-artifact identity: the surrogate weights are one content-addressed ONNX artifact keyed by the parametric-family digest, so a trained field reuses the model-lane session capsule, the warm-start blob, and the result cache without a parallel surrogate store.

[NEURAL_RESIDUAL_FIELD_CODEC]: Add a learned residual-prediction row to the `interchange#FIELD_RESULT_CODEC` so a field chunk encodes as a neural prediction plus a small bounded residual.
- The field codec quantizes each chunk to a bit budget under a relative error bound today; a residual row predicts each chunk from its grid neighbours through a tiny ONNX field model and stores only the residual against the prediction, so a smooth simulation field (the common case for converged solves) compresses far past the per-chunk quantizer at the same error bound while the lossless fall-through stays the exact terminal.
- Unlocks order-of-magnitude smaller solve-result blobs for streamable viz: the `interchange#TILE_PARTITION` octree streams residual-coded chunks that the viewport decodes by re-running the same predictor, the achieved residual against the bound stays auditable on the `Cache` receipt, and a chunk whose residual exceeds the bound falls back to the existing lossless deflate with no new failure mode.
- Draws on learned-compression research for scientific arrays (neural and autoregressive residual coders for floating-point field data) and the existing zero-copy chunk handoff: the predictor is one model-lane ONNX session shared across chunks, so the codec composes the model lane it already sits beside rather than minting a second inference path.

[TILES_METADATA_LAYER]: Deepen `interchange#TILE_PARTITION` from a pure geometry octree into a 3D-Tiles metadata layer carrying per-feature semantic and analysis attributes.
- The tile partition emits a streamable-LOD octree over the content-keyed geometry today; a metadata layer attaches the 3D-Tiles `EXT_structural_metadata` schema to each tile so a feature carries its BIM classification (read from the `Rasm.Bim` IFC semantic graph at the same content-key) and its per-element solver field values (read from the `solver#DISCRETIZATION_MESH` `FieldSpace`) as queryable property tables, joining the two projections of the one content-keyed IFC artifact at the tile boundary.
- Unlocks attribute-driven web visualization with no second store: the AppUi viewer styles and filters tiles by IFC class or by a solved field band (stress, daylight, temperature) directly from the tile metadata, a feature-id texture maps a picked pixel back to its element, and the same content-key that addresses the geometry addresses its metadata so a re-tessellation at a new deflection re-keys both together.
- Draws on the 3D-Tiles 1.1 `EXT_structural_metadata` / `EXT_mesh_features` glTF extensions (now broadly supported across web globe and viewer runtimes) and the existing SharpGLTF tile-emit path: the metadata is one schema column on the leaf-tile content emit, never a parallel attribute store or a second tiling owner.

[COMPILED_EP_CONTEXT]: Generalize the `models#SESSION_CAPSULE` EP-context warm-start into a fleet-shared compiled-context distribution across the substrate axis.
- The session capsule reads a compiled `ep.context_*` blob content-addressed by the model checksum and degrades to a fresh compile on an incompatible-device verdict today; the generalization treats the compiled context as a first-class wire artifact — the host that first compiles a model for a device publishes the context blob through the Persistence blob lane keyed by `(checksum, OrtEpDevice fingerprint)`, and a cold companion or farm node on matching hardware warms from that blob over the existing `remote#ARTIFACT_FRAMES` frame leg instead of recompiling.
- Unlocks near-zero cold-start for the compute farm: the `intent#SUBSTRATE_AXIS` warm-affinity column already reorders the eligible chain toward the node holding the matching context blob, so a fleet of identical-hardware nodes compiles a model once and every peer warm-starts — the compatibility verdict from `OrtEnv.GetModelCompatibilityForEpDevices` is the gate, and a mismatched fingerprint cleanly falls back to a fresh compile.
- Draws on the ONNX Runtime EPContext compiled-model design (the `OrtCompileApi`/`EpContext` node format that serializes a device-compiled subgraph) and the existing content-address seed: the device-keyed context is one artifact-index row family, so the warm-start route stays the one blob-lane owner the capsule already writes rather than a second EP-cache.

[REACTIVE_SOLVE_GRAPH]: Deepen the `scheduling#JOB_GRAPH` dependency-DAG scheduler into a content-keyed reactive recompute engine where a changed input invalidates only its downstream dirty subgraph and every clean node replays its cached receipt.
- The job-graph scheduler already layers speculative, preemptible, and spill-to-store orchestration over the bounded lanes with a dirty-subgraph re-solve mention but no incremental-identity binding; the deepening keys every graph node on the `interchange#CONTENT_ADDRESSING` digest of its inputs so a re-run diffs the new input digests against the prior graph, marks only the nodes whose key moved dirty, schedules the transitive dirty closure on its declared `WorkLane`, and serves every clean node from the `models#RESULT_CACHE` receipt without re-execution — the whole package becomes one incremental compute engine over the existing identity and receipt rails.
- Unlocks interactive parametric editing across the AEC pipeline: a Grasshopper-style design tweak that nudges one parameter re-runs only the affected tessellate/solve/encode chain while every unchanged branch returns instantly from cache, the `FrameBudget` governor bounds the dirty closure to the frame and refines the rest async, and the AppUi twin and Persistence federation receive only the moved nodes' receipts so a coupled geometry-and-field iteration transmits its genuine delta with no second versioning scheme.
- Draws on demand-driven incremental-computation research (self-adjusting computation, content-addressed build graphs) and the existing single content-address seed plus the deterministic result cache: the node key is the one `XxHash128` law and the clean-node short-circuit is the existing cache hit, so the engine composes the identity and receipt owners it already sits beside rather than minting a second memoization or a parallel dependency tracker.
