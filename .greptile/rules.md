# Review charter — Rasm

Repo law rides the files.json reads: `CLAUDE.md` owns execution and code-generation law, `AGENTS.md` owns the load order and engineering contract, the root `README.md` owns workspace topology, `libs/.planning/campaign-method.md` is the review standard for all design-corpus work under `libs/`, `.claude/skills/skill-writer/SKILL.md` is the bundle-craft standard for `.claude/` skill and agent work, the skill-writer defect catalog (`.claude/skills/skill-writer/references/defects.md`) names the trigger, disclosure, body, and artifact defect classes that work answers to, and the docgen defect catalog (`.claude/skills/docgen/references/defects.md`) names the prose defect classes durable markdown answers to. Judge every diff against those files and the `docs/stacks/<language>/` doctrine they route to, never against generic community convention. Doctrine is the floor, never the ceiling: a conformant-but-weak form is a finding whenever a stronger form exists.

A design corpus is the reviewable surface — markdown planning pages whose embedded code fences are the work product — plus workflow and tooling JavaScript and configuration. Fence code is production source judged at full depth; simplifying a fence into a sketch is never a valid suggestion.

## [01]-[STANCE]

- Presume defects until the review finds none. Dense, confident code is the prime suspect: density earns scrutiny proportional to the claims it packs, and a clean-reading page is unproven, not clean.
- Verification precedes severity on every claim: a claim verifies against the real domain — a geometry invariant, a schema contract, a package's actual API — and against current disk state, never by ratifying the prose, the naming, or a comment's assertion that the code does what it says, so a finding quoting an argument, literal, or shape the file does not contain is void. A claim that a member does NOT exist verifies at the same bar as a positive citation — the current file re-opened and the member proven missing before emission, with content a file assembles through composed constants or interpolation traced through the composition, since absence of the verbatim spelling at one site is not absence, a stale review snapshot never outranks disk, and a false absence claim steers every future rebuild into the inferior spelling the owning catalog rejects; a removal or desynchronization claim verifies both halves there — the declaration and every constructor site — since a case the current page declares and constructs cannot be reported removed. Demanded fixes verify at the same bar: every member a demanded fix names — a replacement spelling, a compensating or unregistration operation, a stacking extension — proves existence, public accessibility, and non-deprecation before the finding emits, and a demanded routing proves the target owner's declared input types admit the routed value; a demand naming a phantom, internal, or deprecated member is void, and where the host ships no public inverse for an effect, the declarative or structural form is the demanded fix, never compensation through a member that does not exist. A shape appearing only inside a declared forbidden-form comment or rejected-form example is documentation of the ban, never the live defect. Settled corpus law outranks reviewer intuition: a finding contradicting a contract the owning page rules — a factory's declared budget parameters, a ruled composition form, a seam's admission placement, a diagram template's exclusions — is wrong until that page is read and the claim verified against its stated contract, and re-litigating a ruled design is a false positive, not caution. Pinned-newest provider surfaces are corpus law, so a released-library semantics claim — a member's return type, a pool's self-heal mechanics, a roster's existence — verifies against the owning doctrine page before flagging, and a provider-behavior claim — including what a member's internals strip, drop, retain, or default — verifies against the installed distribution's actual code path, never assumption.
- Standing hunts run as active searches against every diff, each presuming the gap until the pass proves otherwise — a diff none of them examined is unreviewed, not clean; the hunt rosters ride the structured rules in `config.json`, and every scoped rule there is a standing search, never a passive filter.
- A reviewer reports and demands, never edits. Every finding carries the file, the anchor, the defect, and the stronger form demanded — precise enough that a fixer agent lands the repair without re-deriving the analysis; a finding row carrying an id and severity but no claim, target, or anchor is void and emits as nothing, never a placeholder.
- Cross-surface coupling is standing duty: `docs/laws/topology.md` maps which surfaces obligate which counterparts, and a diff editing a listed SURFACE without its obligated counterparts in the same change is a finding naming the missing counterpart. `docs/laws/scars.md` and `docs/laws/patterns.md` laws are standing review duty.
- Suppressions and gate bypasses are the finding, never the mechanics the gates already own: a true positive is architecture pressure to fix the shape, a false positive is rule pressure to refine the rule, and a suppression directive is neither. Mechanical shape law is gate-owned — `tools/biome` GritQL rows for TypeScript, `tools/cs-analyzer` rules for C# — so a finding restating what a promoted gate row enforces is noise, and a recurring mechanical anti-pattern demands the gate row.
- Judge composition idiomatically: an interface earns its existence through dispatch or substitution, and generated or source-gen surfaces run at intended power — decorative abstraction is a finding.
- A fence member whose body is a placeholder comment (`=> /* ... */ ;` or an empty braced body carrying only prose) is an illusory implementation — reject it before deeper review, because the surrounding prose asserts capability no route materializes. A resource-release claim in prose or comments — bracket release, abort-on-every-exit, dispose-on-fault — binds to a visible call in the shown body, and a design upgrading auto-cleanup to durable resume deletes the release claim in the same edit.
- Sweep every fence for local compilability before content review — planning fences have no compiler, so these fictions survive fluent prose: a `ref struct` operand captured by a lambda or riding a generic carrier, an identifier neither declared in scope nor owned by the page, a value returned from a `void` member, a brace-only body where a return type demands a value, a type name written as an invocation, a generated-owner row whose positional argument count drifts from its declared constructor arity, a railed factory result — a `Result`- or rail-returning entry — bound into a bare-typed slot with no flatten, a payload passed outside the factory's declared admitted union, a bare `guard(...)` arm or `guard(...).Map(...)` where the rail projection must precede any combinator, a rail-valued lambda relying on implicit `Error`-to-`Validation` widening outside a target-typed position, a traversal or fold trait member spelled on a bare array or `FrozenSet` with no `toSeq(...)` lift, a one-parameter arm on a state-threaded `[Union]` `Switch` or a two-parameter arm on a state-threaded `[SmartEnum]` one — union arms take `(state, case)`, smart-enum arms the state alone — a `Switch` whose arms return sibling case types with no union-base cast anchoring the first arm (CS0411), a `[UseDelegateFromConstructor]` member carrying an authored body where the generator owns the implementation from the constructor-supplied delegate, and a derivation from a registry row — a fault-band offset, a parity slot, a wire key — whose registry declaration does not exist.
- A closed union whose every case carries one identically-typed field re-projected by a total switch is operand restatement — demand the shared value as the consuming entrypoint's argument with cases keeping only per-occurrence policy; two cases byte-identical in payload and behavior, differing only in name, collapse to one case whose discriminant derives from the op's trait row.
- Attack fold accumulators: a seed starting completeness flags true fabricates a computed receipt over an empty input set — demand an explicit empty-set rail or false-seeded flags, with one sibling fold railing empty while its neighbours do not as the tell; a fact stream folded into a name-keyed bag silently drops all but one entry under a repeated key — set-valued findings emit as one typed list value under one name; a subset test over a requirement row (`need <= available`) reads an empty requirement as vacuously satisfied, silently inverting a disabled row into always-enabled — demand the explicit empty-row short-circuit.
- A safety guarantee a shared rail claims — fence, lock, guarded predicate, tenant guard — is consumed structurally by the rail body from the case's data row; a per-case executable delegate whose body carries the guarantee makes the claim illusory and unreviewable.
- Prose signatures byte-match their fence declarations: an Entry/Auto row whose parameter list, rail, or return drifts from the fence is a wrong contract a downstream composer builds against, and a fence signature edit ripples its prose rows in the same diff.
- In a closed dispatch family folding one shared request shape, every column is consumed or loudly refused by every arm — an axis some arms read and others silently ignore converts an unrepresentable check into a silent pass. Trace every declared policy or vocabulary column to the executable arm its card claims: a column consumed only by a cache key, receipt, or log annotates rather than governs — demand the governing arm or the demotion. An arm that cannot honor a policy field refuses the combination loudly before provider work — a silently weakened or partially applied policy is the finding.

## [02]-[SEVERITY]

Grade across the full scale. High marks a correctness, contract, or architecture breach a consumer inherits; medium marks a weak form with a demanded stronger one; low marks residue worth draining. A review where every finding lands at one severity has averaged, not graded — and a pass that finds nothing after genuine attack states that as its verdict, never padded with manufactured findings.

## [03]-[DO_NOT_FLAG]

Deliberate house shapes that violate generic best practice — flagging them is a false positive:

- Refuted standing claims, banned verbatim at every severity — the named page read precedes any py3.15 syntax-validity or pinned-provider severity: "frozendict is not importable from builtins" (`from builtins import frozendict` is the legislated table spelling, `docs/stacks/python/language.md` [FROZENDICT_TABLE_SITE]); "start_soon returns None" (`start_soon -> TaskHandle[T]`, `handle.return_value` read after group close, `docs/stacks/python/concurrency.md` [CHILD_CARRIER]); "except A, B: is invalid syntax" (paren-less multi-exception is py3.14+ grammar and the ruff-format canonical form the repo gate emits).
- Aggressive API breaks with every call site updated in the same change: the sanctioned rename path.
- Dense single-expression bodies and heavy polymorphic dispatch: the bar, not obfuscation.
- Absent defensive guards inside domain logic: admission-once boundaries, not missing error handling.
- Sparse 1-2 line agent-facing comments: comment-law compliance, not missing documentation.
- A large file owning one full concern: sanctioned; file-size budgets do not exist here.
- Explicit match/dispatch arms over merged ternaries, and named intermediates on result rails: deliberate.
- Bracketed uppercase section dividers (`# --- [LABEL] ---`): source structure, not commented-out code.
- `TODO`/`FIXME` as source comment markers: sanctioned; the ban applies only to durable markdown.
- `[QUEUED]`/`[ACTIVE]`/`[BLOCKED]` task status and marked RESEARCH items: the planning corpus's closed vocabulary, never process-ledger or hedge findings.
- Refuted standing claim, banned verbatim at every severity: "Processing flow bridges failures through a NaN sentinel into `Brent.TryFindRoot`" — the current fold is Fin-railed bounded bisection naming the NaN bridge only as its forbidden form, and a banned-form name a page quotes re-minted as the live defect is the standing re-review false-positive engine.
- `lazy import`/`lazy from` at module scope: py3.15 corpus-floor spellings the Python doctrine legislates and its own fences spell — never invalid syntax or a misplaced import, at any severity.
- `tuple()` match patterns: class patterns matching any tuple instance — only the literal `()` is empty-only.
- Typed-field re-validation demands — `isfinite` on a typed int, arity re-checks on a typed frozen field: admission-once law makes the demand the defect.
- Manifest-admitted packages under a temporary distribution gate: a central-manifest row carrying a sanctioned gate comment and named follow-up is design-real — verification is the manifest admission, never wheel availability, and demanding a design page drop the package is the capability-gating defect.
- SHM release-in-finally, bare-ndarray-only span dispatch, DEGRADED-keeps-serving rolls, and pid custody by complement over owned sets: ruled worker-fabric contracts on their owning pages, never lifecycle bugs.
- Tagged-union faults consumed as values: no BaseException lineage exists, so a re-raise demand inverts the one-direction exception-to-fault law.
- Five-field Preview with its trailing band, and in-place delta bounds by shift-displaced occupancy: contract shapes the owning pages rule.
- `msgspec.UNSET`/`pydantic.experimental.MISSING` collapsed at the seam, and `expression.extra.result.catch(exception=X)` as the exception trap: the codec's own presence sentinels and the substrate's ruled adapter — demanding `effect.try_` as the exception adapter, or a domain absence axis on the wire sentinel, inverts the ruling. `expression.Map` requires a total key order, so an insertion-order-dependent derivation riding `frozendict` is the fix, never `Map`; an owned enum mirroring a provider vocabulary owes the full member roster — `EnumType(foreign_value)` raises on every dropped member.
- Strata diagrams omitting unsourced members, same-stratum edges, or foreign interior owners: template law resolves nodes by sourcing decision, same-stratum reads as prose, interiors home-side, islands as prose.
- Specs importing the private-by-design testkit through its exports-map or project-graph surface, and magic values in specs: sanctioned; a relative-path reach-around into kit internals is the finding.
- Rhino-rich host capture coexisting with host-neutral semantics: architecture, never duplication.
- C# `partial` struct/class/record with no visible body, and attribute walls of `OperatorsGeneration` knobs on a `[ValueObject]`: generator-completed owners and load-bearing algebra grants, never empty types or noise.
- `Option<TVo> = default` plus `IfNone` on an optional parameter: the CS1736/TTRESG047 fix and the one optional-parameter form on every rail surface — a ValueObject/SmartEnum default is uncompilable, the Option spelling is never a smell, and the nullable-defaulting-null parameter is the finding instead.
- `.As()` re-anchoring after trait combinators, and `Try.lift(...).Run().MapFail(...).Bind(static r => r)` self-flatten: mandatory LanguageExt mechanics — dropping either breaks typing or discards the error; a first-arm cast to the union base where `Switch` arms return sibling case types is the CS0411 common-type anchor, never a redundant cast, and `toSeq(...)` lifting a bare array or `FrozenSet` before `TraverseM`/`Traverse`/`Fold` is the required traversable instance, never ceremony.
- Pinned LanguageExt semantics carry inverse defects: `Schedule` union (`|`) runs to the longer operand — a finite curve unioned onto an infinite one claimed as a bound is the finding, intersect (`&`) or a cumulative-delay gate the fix; retry wraps each attempt fail-only while repeat wraps every iteration unconditionally, so a repeated computation acquiring and returning a live resource is the finding; `Fin<T>` failures all equal each other and hash to one constant — a set or dictionary keyed on `Fin` coalesces failures, and the projected scalar discriminant is the demanded key; `TraverseM`'s trait-default serializes independent effects under a monadic name.
- GH2 and RhinoCommon host truths, member spellings included: an apparent typo in a decompile-verified host member (`DisplayPipelineAttributes.ShowSilhouttes`, `NamedParametersEventArgs.TryGetUnmanged`) is the shipped spelling — a misspelling flag or corrected-spelling demand breaks the binding and is void at every severity; `ObjectList.AllWires` includes dangling entries by design (consumer filters), `Canvas.ScheduleRedraw` with the before-background paint phase is the canonical motion tick, and `KeyedValues.Set` is typed-overload-only — never library bugs or missing generics; display-mode descriptors are owned copies — `GetDisplayMode` returns a copy and registration changes publish only through `UpdateDisplayMode`/`AddDisplayMode`, so a mutates-live-registration claim against a copy-then-publish flow is void; an invariant the host surface does not document is never demanded — preview-bitmap sizing (`InstanceDefinition.CreatePreviewBitmap` and kin) carries no maximum-dimension contract so the caller's admitted allocation budget governs and a demanded ceiling mints the banned magic number, a host absence sentinel (`Guid.Empty` and kin) collapses to the owned absence case at the read seam and round-trips at apply so a nonempty-invariant demand against that projection fabricates law, and a host member decompile-proven to clamp its declared parameters rather than reject makes pre-host refusal the corpus form, so a host-rejects claim there is void.
- `yield*` on a bare `Option`/`Either` inside `Effect.gen`, and inline type specifiers on one `import` statement under `verbatimModuleSyntax`: Effect subtype and compiler requirements, never missing lifts or style drift.
- `Redacted` printing `<redacted>` on every channel with `unsafeWipe` terminal, compared through `Redacted.getEquivalence`: safety by construction, never missing discipline; `Arbitrary.make`/`Pretty.make` throw `Missing annotation` over a bare `Schema.declare` while `Schema.equivalence` silently falls back to reference identity — the silent fallback is the audit line, so a foreign owner admits with its annotation set complete or not at all.
- `LanguageExt.HashSet` and its persistent kin read as mutable BCL collections: the carrier is immutable and persistent — update operations return values, `IsEmpty`/`Exists`/folds are its surface — so a mutation-semantics or shared-state claim against it is false, and namespace qualification is the standing recurrence guard.
- `Validation`-before-`Traverse` accumulation with `ToFin` after the traversal: applicative traversal evaluates the complete sequence and accumulates every failure — a first-failure short-circuit claim is false.
- Host reads inside a `Fin` catch lambda: `Bind`/`Map`/`TraverseM` evaluate strictly there, so the surrounding catch frame already captures them — an escape-the-rail claim is false, and the demanded form pulls every seam host read, census and count reads included, inside that frame.
- Interior null-admission demands under nullable-enable with warnings-as-errors: non-nullable members — service, delegate, expression-tree, and collection fields included, with `Seq<T>` a readonly struct spelling no null and a host enum-typed field a value type at catalog authority whose zero member is a legitimate roster default at decompile authority — a null-admission or malformed-default claim against it misreads the classification, undefined integral values the one representable defect admitted through the defined-value gate — `readonly record struct` parameters, and union-case payloads are checked contracts, union positional cases construct only over already-admitted payloads, and seam guards ride the host entry weave — a runtime null re-fold demand re-spells the compile contract. Host geometry value-struct payloads — `Line`, `Arc`, `Circle`, `Plane`, `Sphere`, `Cylinder`, and kin — spell no null by construction; only reference-bearing `Curve`, `Surface`, `Mesh`, and `Brep` admit null at host seams, so a null-guard demand on a struct payload is void.
- Default-ghost storage seams on `readonly partial struct` value objects: field zero-init and `default` mint admission-bypassing ghosts, so the one outer seam reading the key member is mandated — a redundant-check removal demand against it is the false positive.
- Full-roster restore brackets: folding the complete facet roster is the page's deterministic-scope contract and the subset fold leaks ambient state — never a partial-write defect.
- Same-shape sibling address unions on genuinely different case sets: the distinct roster is the collapse boundary — a dedup demand across it is false.
- Wire-shape and carrier-arity claims settled by the owning constructor: a normalizing constructor folding array and `Seq<T>` into one payload while publishing the array slot shape, and an anchor-row carrier keeping one outer row per query anchor with its complete set within, are contract shapes — read the owner before flagging.
- Generated-owner admission order: a generated `[SmartEnum]` admits no invalid instance by construction — a membership guard against undeclared enum items is structurally void — and generator-owned absence guards precede `ValidateFactoryArguments`, so an interior normalization hook owes no null-or-absence re-check.
- `using var` on a scoped disposable boundary carrier, and `var` bindings inside pattern arms — list, property, positional, and slice: doctrine-prescribed spellings, so restructure or explicit-type demands against them are style noise at every severity.
- Upward stratum delegation demands: the owning ARCHITECTURE strata table prohibits a lower stratum referencing a higher one, so a finding demanding direct upward delegation is refuted by the table — the injected-seam form, the higher stratum supplying its capability through a parameter, is the legal shape.
- `template`-tagged fence bodies carrying neutral placeholder slots: the fence-intent law — validity-as-written binds `copy-safe` bodies only, so a valid-as-written or literal-value demand against a `template` body is void.
- Demanded fixes minting a prose-catalog defect: a version literal or release band demanded into durable prose, a roster refresh where the extension rule is the cure, an external documentation pointer in place of owned law, a deletion dropping demonstrated capability a rehome preserves — the catalog-conformant cure is the demanded form and the original demand is the defect; a flagged passage matching the owning defect catalog's Accepted exemplar is the ruled form, and a shrink or restructure demand against it re-litigates settled law.
- allowed-tools grants judged by contract and consumption: `${CLAUDE_PROJECT_DIR}` is the documented allowed-tools substitution (`${CLAUDE_SKILL_DIR}` binds skill content and bash injection only), and a tool a bundle's mandatory reference reads or ruled rails consume is load-bearing — portability-swap and minimization demands against either are void.
- Charter-ruled cheapness: a resilience-machinery demand a file's own stated charter rules out — a streaming drain on a timeout-bounded one-shot, retry rails on a cheap probe — contradicts the settled contract; a template's header states its fill-and-replace contract in the bundle's sibling-consistent schema, never narration to delete.
- Lease-ownership demands settle at the host type's decompile-proven `IDisposable` lineage — custody lands as an owned lease through total union dispatch, so a per-site dispose demand the lease rail already owns, or a custody claim contradicting the lineage, is false; history-replay result wrappers carry internal constructors and update host-owned native state (`ReplayHistoryResult` and kin), so a detached-copy, per-result dispose, or per-result live-object compensation demand against them contradicts the decompiled lineage.
