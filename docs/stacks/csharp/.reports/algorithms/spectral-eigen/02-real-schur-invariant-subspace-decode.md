# Real-Schur Invariant-Subspace Decode

[COLUMN_PAIR_EIGENVECTOR_ENCODING]:
- The real nonsymmetric route never produces a `Matrix<Complex>` eigenvector carrier. `EigenVectors` is `Matrix<double>`, and a complex-conjugate eigenpair `λ = a ± bi` occupies two adjacent real columns: column `j` holds the real part `u`, column `j+1` holds the imaginary part `w`, so the two genuine eigenvectors are `u + iw` (paired with `a + bi`) and `u − iw` (paired with `a − bi`). The conjugate-eigenvalue ordering is fixed: the back-substitution emits `e[j] = +b` then `e[j+1] = −b`, so the positive-imaginary eigenvalue always indexes the column carrying `+w`. A consumer that reads `EigenVectors.Column(j)` as a single eigenvector for a complex eigenvalue gets only the real part and silently discards the rotation half.
- The discriminant for "this column starts a complex pair" is recoverable from the eigenvalue vector alone — `EigenValues[j].Imaginary > 0` opens a pair, `< 0` closes it, `== 0` is a real eigenvector in its own single column. No parallel pairing array is stored; the encoding is fully derived from the sign of the imaginary part, which is the load-bearing reason a receipt must carry `EigenValues` to interpret `EigenVectors` rather than treating the eigenvector matrix as self-describing.
- Reconstructing the true complex modal matrix from the packed real form is a strided gather, not a copy. For each `j` with positive imaginary part, the pair `(Column(j), Column(j+1))` becomes two complex columns; for each real `j`, `Column(j)` lifts with zero imaginary part. The reconstruction is index-first dispatched on the sign of `EigenValues[j].Imaginary`, never on a length check or a try-cast:

```csharp
static Matrix<Complex> LiftSchurVectors(Matrix<double> packed, Vector<Complex> values) =>
    Matrix<Complex>.Build.DenseOfColumns(
        Enumerable.Range(0, values.Count).Select(j =>
            Math.Sign(values[j].Imaginary) switch
            {
                > 0 => packed.Column(j).Map2(packed.Column(j + 1),
                           (re, im) => new Complex(re, im)),
                < 0 => packed.Column(j - 1).Map2(packed.Column(j),
                           (re, im) => new Complex(re, -im)),
                _   => packed.Column(j).Map(re => new Complex(re, 0.0))
            }));
```

[BLOCK_DIAGONAL_D_AS_QUASI_TRIANGULAR_EVIDENCE]:
- `D` is not diagonal for the real nonsymmetric route — it is the real Schur block-diagonal. A conjugate pair writes a 2×2 block `[[a, b], [−b, a]]`: the diagonal carries `a` twice, `D[j, j+1] = +b` (the super-diagonal), and `D[j+1, j] = −b` (the sub-diagonal). The off-diagonal magnitude `|b|` is the imaginary part recovered directly from `D`, so `D` alone distinguishes a real eigenvalue (zero off-diagonals around its index) from the real part of a complex pair (a nonzero anti-symmetric 2×2 block). A receipt that records `D` as merely "the eigenvalue diagonal" loses the rotation evidence that proves the spectrum has a complex pair at that index.
- The block structure is the only in-band signal of how many real invariant subspaces versus rotation-planes the operator carries. Counting nonzero super-diagonal entries of `D` yields the number of conjugate pairs without touching the complex eigenvalue vector; `n − 2·(pair count)` is the count of real eigenvalues. This is a structural invariant of the Schur form, not a heuristic, and it is the cheapest spectral-type census available before any eigenvector work.

[BACK_SUBSTITUTION_NORM_GATE]:
- Eigenvector recovery is gated on a single accumulated norm computed before the eigenvalue loop: the running sum of absolute values over the upper Hessenberg band. When that accumulator is exactly zero — the operator reduced to a strictly lower or all-zero band — the routine returns immediately and leaves the eigenvector matrix as the accumulated similarity transform with no per-column solve performed. A degenerate all-zero or nilpotent-shaped operator therefore yields eigenvalues but a structurally meaningless eigenvector block, and the boundary distinguishes this case only by checking the operator norm itself, never by inspecting the returned vectors.
- The quasi-triangular back-substitution solves the upper-triangular system column by column from the bottom, and for a complex pair it routes through a guarded complex division: when both real and imaginary denominators underflow, the denominator is replaced by `ε_mach · ‖band‖` — a deliberate regularization that prevents a divide-by-zero but injects an `O(ε_mach·‖A‖)`-scaled spurious component into that eigenvector column. The eigenvector for a near-defective eigenvalue is thus not "wrong" but carries a known-magnitude perturbation tied to the operator norm; the residual `‖A·v − λ·v‖` is the only post-hoc detector and is mandatory for any eigenvector consumed near a defective spectrum.
- The recovered eigenvectors are not normalized. After back-substitution each column is the raw solution of the triangular system, then back-transformed through the accumulated Householder similarity by an in-place right-multiply `EigenVectors ← EigenVectors · H_packed`. Column norms after this step are arbitrary and differ across columns; any downstream projection `vᵀ·x` or modal weight that assumes unit columns must explicitly `Column(j).Normalize(2)` first. The symmetric route returns orthonormal columns; the nonsymmetric route returns a non-orthonormal, non-unit basis spanning each invariant subspace — the same `EigenVectors` property carries two different normalization contracts dispatched purely on the symmetry flag.

[DEFECTIVE_AND_NEAR_DEFECTIVE_SURFACE]:
- A defective operator (algebraic multiplicity exceeding geometric multiplicity) has no full eigenvector basis, yet the routine never throws for defectiveness — it always emits `n` columns. For a true Jordan block the back-substitution produces near-parallel columns (the eigenvector repeated with the norm-gate regularization injected), so defectiveness manifests as a rank-deficient `EigenVectors` matrix, not an error. Detection requires a condition or rank probe on `EigenVectors` itself; the eigenvalue multiplicity alone cannot distinguish a defective operator from one with a genuine repeated-but-diagonalizable eigenvalue.
- Eigenvector conditioning is per-pair and asymmetric: the sensitivity of eigenvalue `λᵢ` is `1/|yᵢᴴ·xᵢ|` where `yᵢ` and `xᵢ` are the left and right eigenvectors. The routine returns only right eigenvectors, so the conditioning number is not computable from its output alone — recovering left eigenvectors requires a second decomposition of the transpose. A receipt that must report per-eigenvalue conditioning for a nonsymmetric operator carries the transpose-route cost as a structural consequence of the one-sided eigenvector contract, and absent it can only bound conditioning, never report it.
- The exceptional-shift escape fires at fixed internal step counts and injects a deterministic ad-hoc shift to break a stalled bulge-chase; one of these escapes substitutes the literal constant `0.964` into the shift state when the quadratic shift estimate degenerates. An operator whose Schur convergence repeatedly triggers this escape produces eigenvalues whose last bits are governed by that constant rather than by the matrix, so two operators differing below the convergence band can return bit-identical eigenvalues. This is a determinism feature for regression baselines and a precision trap for any test asserting eigenvalue agreement tighter than the convergence band.

[ROUTE_DISPATCH_AS_DERIVED_CONTRACT]:
- The symmetry flag is not merely a performance switch — it selects the entire output contract: orthonormal versus arbitrary-norm eigenvectors, real-diagonal versus block-diagonal `D`, single-column versus column-pair eigenvector encoding, ascending-sorted versus Schur-convergence-ordered eigenvalues, and a working eigenvector solve versus a norm-gated skip. All five contract axes are derived from one boolean, so a receipt union with a `dense-symmetric` case and a `dense-general` case is the minimal honest shape; collapsing them to one "eigendecomposition result" forces every consumer to re-derive the five-way contract from the symmetry flag at each use site.
- The Schur convergence order carries no spectral meaning — it is the deflation order of the bulge-chase, which depends on the Hessenberg reduction's pivot path, not on eigenvalue magnitude or real part. A consumer that needs eigenvalues ordered by real part, magnitude, or proximity to a shift target must sort explicitly and carry the permutation to reindex both `EigenVectors` columns and `D` blocks in lockstep; reordering eigenvalues without simultaneously permuting the column-pair groups in `EigenVectors` and the 2×2 blocks in `D` desynchronizes the three carriers, and the desync is silent because each carrier remains individually well-formed.
- Reaching this route requires a genuinely nonsymmetric dense operator: the symmetric exact-bit-equality check routes any operator whose mirror entries differ by even one ULP through this nonsymmetric path, so an operator that is symmetric in intent but assembled through non-mirror-exact arithmetic silently acquires the entire nonsymmetric contract — block-diagonal `D`, non-orthonormal columns, possible spurious complex pairs from rounding. The defense is mirror-exact re-symmetrization at assembly so the operator is bit-symmetric before the route decision, not after; the route is chosen by bit equality, and the only way to control the route is to control the bits.
