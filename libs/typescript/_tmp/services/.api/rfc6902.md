# [API_CATALOGUE] rfc6902

`rfc6902` is a complete, dependency-free RFC 6902 JSON Patch + RFC 6901 JSON Pointer engine over plain JSON-serializable values. Three layers stack: the barrel (`applyPatch`/`createPatch`/`createTests` + the `Operation` discriminated union + `Patch`) is the document-delta surface; the `patch` submodule exposes the per-op appliers (`add`/`remove`/`replace`/`move`/`copy`/`test`) plus the single `apply` that dispatches on `operation.op`, returning a typed error rather than throwing; the `diff` submodule exposes the Levenshtein-optimal diff engine (`diffAny`/`diffArrays`/`diffObjects`) and the pluggable `Diff`/`VoidableDiff` override seam so a custom subtree comparator falls back to `diffAny`. The `pointer` submodule is the RFC 6901 `Pointer` class (parse, evaluate, get, set, immutable `add`/`parent`, escape round-trip), and `util.clone` is the structural-copy primitive that makes `applyPatch` non-mutating. For the services tier it is the wire-neutral document-delta format: the `messaging/entity#ENTITY` collaborative-session op-log, `messaging/rpc#INTERNAL_RPC` partial-update payloads, and persistence state-diffing all express a change as a `Patch` (`Operation[]`) decoded/encoded at the boundary rather than a bespoke per-domain op union.

- package: `rfc6902`
- version: `5.2.0`
- license: `MIT`
- tier: `node` (services messaging) — pure isomorphic JSON manipulation, zero runtime built-ins, safe on any tier including browser; legacy CJS with no `exports` gate, so deep subpath imports (`rfc6902/pointer`, `rfc6902/patch`, `rfc6902/diff`, `rfc6902/util`) are open beside the named-export barrel.
- rail: json-patch

## [02]-[OPERATION_ADT]

`Operation` is a six-arm discriminated union keyed on `op`; `Patch = Operation[]`. `add`/`replace`/`test` carry `value`; `move`/`copy` carry a `from` pointer; `remove` carries only `path`. This is the wire shape — decode it at the boundary through a `Schema.Union` tagged on `op` and the arms are exhaustive.

| [INDEX] | [SYMBOL]           | [SHAPE]                            | [SEMANTICS]                                             |
| :-----: | :----------------- | :-------------------------------- | :----------------------------------------------------- |
|  [01]   | `AddOperation`     | `{ op: 'add'; path; value }`      | insert at array index / add-or-replace object member   |
|  [02]   | `RemoveOperation`  | `{ op: 'remove'; path }`          | delete at `path`; `path` MUST exist                    |
|  [03]   | `ReplaceOperation` | `{ op: 'replace'; path; value }`  | remove-then-add at `path`; `path` MUST exist           |
|  [04]   | `MoveOperation`    | `{ op: 'move'; from; path }`      | remove at `from`, add at `path`; `from` ⊄ `path`       |
|  [05]   | `CopyOperation`    | `{ op: 'copy'; from; path }`      | add value read at `from` to `path`; `from` MUST exist  |
|  [06]   | `TestOperation`    | `{ op: 'test'; path; value }`     | assert deep-equality at `path`; the guard/precondition |
|  [07]   | `Operation`        | union of `[01]`–`[06]`            | discriminated on `op`                                  |
|  [08]   | `Patch`            | `Operation[]`                     | ordered operation list, applied left-to-right          |

```ts contract
// rfc6902/diff — the six-arm union (all `path`/`from` are RFC 6901 pointer strings)
export interface AddOperation     { op: 'add';     path: string; value: any }
export interface RemoveOperation  { op: 'remove';  path: string }
export interface ReplaceOperation { op: 'replace'; path: string; value: any }
export interface MoveOperation    { op: 'move';    from: string; path: string }
export interface CopyOperation    { op: 'copy';    from: string; path: string }
export interface TestOperation    { op: 'test';    path: string; value: any }
export type Operation = AddOperation | RemoveOperation | ReplaceOperation | MoveOperation | CopyOperation | TestOperation
export type Patch = Operation[]
```

## [03]-[APPLY_AND_GENERATE]

The barrel drives whole-`Patch` application and generation; the `patch` submodule exposes the per-op appliers and the single `op`-dispatching `apply` beneath them. Neither throws for a failed operation — each returns a typed error instance (or `null`), so failures fold as data, never exceptions.

| [INDEX] | [SURFACE]                                                                       | [MODULE]        | [RESULT]                                                                          |
| :-----: | :------------------------------------------------------------------------------ | :-------------- | :------------------------------------------------------------------------------- |
|  [01]   | `applyPatch(object, patch: Operation[], options?): (MissingError \| TestError \| InvalidOperationError \| null)[]` | `rfc6902`       | one result per op, index-aligned; `null` = success, else the failing op's error  |
|  [02]   | `createPatch(input, output, diff?: VoidableDiff): Operation[]`                   | `rfc6902`       | minimal Levenshtein-optimal patch from `input` to `output`                       |
|  [03]   | `createTests(input, patch: Operation[]): TestOperation[]`                        | `rfc6902`       | `test` ops capturing current values at each patched path — the optimistic-lock guard |
|  [04]   | `apply(object, operation: Operation, options?): MissingError \| InvalidOperationError \| TestError \| null` | `rfc6902/patch` | single-op switch on `operation.op`; unknown `op` → `InvalidOperationError`        |
|  [05]   | `add` / `remove` / `replace` / `move` / `copy` (object, op, options?): `MissingError \| null` | `rfc6902/patch` | granular mutators; the arms `apply` dispatches to                                |
|  [06]   | `test(object, op: TestOperation, options?): TestError \| null`                   | `rfc6902/patch` | equality assertion; `TestError` carries `.actual` / `.expected`                  |

```ts contract
// rfc6902/patch — typed error rail (no throw for a failed operation) + apply option
export declare class MissingError extends Error { path: string; constructor(path: string) }
export declare class TestError extends Error { actual: any; expected: any; constructor(actual: any, expected: any) }
export declare class InvalidOperationError extends Error { operation: Operation; constructor(operation: Operation) }
export interface Options { implicitArrayCreation?: boolean }  // "add /-" auto-creates a leaf array when the parent is absent
```

## [04]-[DIFF_ENGINE]

`createPatch` delegates to `diffAny`, which routes to `diffArrays` (dynamic-programming Levenshtein edit script), `diffObjects` (key `subtract`/`intersection` over own-enumerable non-`undefined` keys), or an equality short-circuit. The `diff?` parameter is the override seam: supply a `VoidableDiff` that returns operations for the subtrees it wants to own and `void` to fall back to the built-in `diffAny` for everything else — the mechanism for domain-aware minimal diffs (e.g. move-detection on keyed arrays) without reimplementing the engine.

| [INDEX] | [SYMBOL]                                              | [MODULE]       | [ROLE]                                                             |
| :-----: | :---------------------------------------------------- | :------------- | :---------------------------------------------------------------- |
|  [01]   | `Diff = (input, output, ptr: Pointer) => Operation[]` | `rfc6902/diff` | total subtree comparator signature                                |
|  [02]   | `VoidableDiff = (input, output, ptr) => Operation[] \| void` | `rfc6902/diff` | partial override; `void` falls back to `diffAny` — the `createPatch` `diff?` type |
|  [03]   | `diffAny(input, output, ptr, diff?): Operation[]`     | `rfc6902/diff` | top dispatch: array vs object vs leaf; the default fallback       |
|  [04]   | `diffArrays<T>(input, output, ptr, diff?): Operation[]` | `rfc6902/diff` | Levenshtein add/remove/replace edit script over arrays            |
|  [05]   | `diffObjects(input, output, ptr, diff?): Operation[]` | `rfc6902/diff` | member add/remove/replace over object keys                        |
|  [06]   | `isDestructive({ op }: Operation): boolean`           | `rfc6902/diff` | true for `remove`/`replace`/`move` — classifies an op's destructiveness for op-log gating |
|  [07]   | `subtract(minuend, subtrahend): string[]`             | `rfc6902/diff` | own-key set difference (JSON semantics: drops `undefined`)        |
|  [08]   | `intersection(objects: ArrayLike<object>): string[]`  | `rfc6902/diff` | own-keys shared by every object                                   |

## [05]-[POINTER]

`Pointer` is the RFC 6901 path as a `tokens` array. `fromJSON` parses an escaped string; `evaluate` returns `{ parent, key, value }` (with `parent: null` for the empty root pointer); `get`/`set` read/write at the path; `push` mutates in place while `add`/`parent` return new pointers (shallow-immutable). `escapeToken`/`unescapeToken` are the standalone `~0`↔`~`, `~1`↔`/` round-trip the token layer uses.

| [INDEX] | [SURFACE]                                     | [MODULE]         | [RESULT]                                                        |
| :-----: | :-------------------------------------------- | :--------------- | :------------------------------------------------------------- |
|  [01]   | `new Pointer(tokens?: string[])`              | `rfc6902/pointer`| construct from raw tokens                                      |
|  [02]   | `Pointer.fromJSON(path: string): Pointer`     | `rfc6902/pointer`| parse a properly-escaped pointer string (static)               |
|  [03]   | `pointer.tokens: string[]`                    | `rfc6902/pointer`| the decoded path segments                                     |
|  [04]   | `pointer.toString(): string`                  | `rfc6902/pointer`| re-serialize to an escaped pointer string                     |
|  [05]   | `pointer.evaluate(object): PointerEvaluation` | `rfc6902/pointer`| `{ parent, key, value }`; `parent: null` at the empty root      |
|  [06]   | `pointer.get(object): any`                    | `rfc6902/pointer`| value at the path                                             |
|  [07]   | `pointer.set(object, value): void`            | `rfc6902/pointer`| write at the path (mutating)                                  |
|  [08]   | `pointer.push(token: string): void`           | `rfc6902/pointer`| append a token in place (mutating)                            |
|  [09]   | `pointer.add(token: string): Pointer`         | `rfc6902/pointer`| new pointer with the token appended (immutable)               |
|  [10]   | `pointer.parent(): Pointer`                   | `rfc6902/pointer`| new pointer for the parent path (immutable; root's parent = root) |
|  [11]   | `escapeToken(token)` / `unescapeToken(token)` | `rfc6902/pointer`| `~0`/`~1` RFC 6901 escape round-trip (standalone functions)    |
|  [12]   | `clone<T>(source: T): T`                       | `rfc6902/util`   | recursive structural copy — the pre-`applyPatch` immutability primitive |
|  [13]   | `objectType(object): 'array' \| 'object' \| 'null' \| 'number' \| ...` | `rfc6902/util` | JSON-aware type discriminant the diff engine routes on          |

## [06]-[IMPLEMENTATION_LAW]

[TOPOLOGY]:
- `applyPatch` MUTATES `object` in place and returns the index-aligned result array; a failed op leaves the target in a partially-applied state. For immutable semantics, `clone(object)` (from `rfc6902/util`) first, then apply to the copy.
- The result array is `null`-or-error per operation index; the errors (`MissingError`/`TestError`/`InvalidOperationError`) are `instanceof`-discriminable — `TestError` additionally carries `.actual`/`.expected` for optimistic-lock audit.
- `createPatch` produces the minimal edit script; `Options.implicitArrayCreation` lets `add "…/-"` create a leaf array when the parent property is absent. Only the leaf array is inferred — missing intermediate objects still error.
- `createTests(input, patch)` snapshots current values as `test` ops: prepend them to a patch to make application optimistic-concurrency-safe (the patch aborts at the first `TestError` if the base drifted).

[EFFECT_STACK]: how the services rebuild composes this onto the `effect` universal-tier rails (`libs/typescript/.api/effect.md`) instead of raw mutation.
- Decode at the boundary: model the wire delta as `Schema.Array(Operation)` where `Operation` is a `Schema.Union` of six `Schema.Struct`s tagged on `op` — the same `parse-don't-validate` law every services boundary uses — so an inbound patch is validated to the ADT before it touches state, and an unknown `op` fails decode rather than reaching `InvalidOperationError` at apply time.
- Apply as an effect over a clone: `Effect.sync(() => clone(state))` then `Effect.map` the `applyPatch` result array; fold the array with `Array.filterMap`/`Option.fromNullable` into a `Data.TaggedError` when any entry is non-`null`, so a partial-apply failure is a typed rail value, never a thrown exception or a silently corrupted document.
- Generate under a span: `createPatch(prev, next)` wrapped in `Effect.sync` and traced through the `@effect/opentelemetry` tier emits the delta size as a span attribute; `createTests(prev, patch)` prepended yields the optimistic-concurrency guard the durable op-log needs.

[SIBLING_STACK]:
- `messaging/entity#ENTITY`: the collaborative-`SessionEntity` `Apply` procedure is the first-class consumer — express the C# CRDT op-log payload decoded off the `interchange` boundary as a `Patch` and fold it into the per-key `SessionState.document` via `applyPatch(clone(state), patch)`, replacing the bespoke `Insert`/`Delete` slice union with the standard RFC 6902 delta; `isDestructive` gates whether an op requires a revision bump, and `createTests` on the prior revision makes the fold reject a stale op exactly-once.
- `messaging/rpc#INTERNAL_RPC`: a partial-update procedure carries a `Patch` as its `@effect/rpc` request payload (Schema-serialized against the same `Operation` union), so a mutation RPC ships the minimal delta rather than the whole entity — `createPatch` on the caller, `applyPatch` on the handler.
- `persistence`: `createPatch(previousRow, nextRow)` diffs an entity revision into a journalled delta; `Pointer.get`/`evaluate` reads a single property-path out of a stored document without deserializing the whole tree.

[RAIL_LAW]:
- Package: `rfc6902`
- Owns: RFC 6902 JSON Patch (generation, whole-patch and per-op application, test-op guards) and RFC 6901 JSON Pointer navigation/escape over plain JSON values.
- Accept: `Schema`-decoded `Operation[]` as the wire delta; `clone`-then-`applyPatch` for immutable application; `createTests` for optimistic-concurrency guards; a `VoidableDiff` override for domain-aware minimal diffs.
- Reject: a bespoke per-domain op union where the RFC 6902 `Operation` ADT is the portable wire format; `applyPatch` on shared state without a preceding `clone`; treating the result array as a boolean — a non-`null` entry is a typed failure to fold; a hand-rolled pointer parser where `Pointer.fromJSON`/`escapeToken` round-trip.
