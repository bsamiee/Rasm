# Struct-Class Owner — Distilled

[OWNER_FUSION]:
- `class Self extends Schema.Class<Self>("Id")({ ...fields })` is one declaration yielding six fused surfaces — nominal class type, runtime value-constructor (`new Self`), `Schema<Self, Encoded, R>` codec, structural `Equal`/`Hash` (via the `Data.Class` prototype base), an `Arbitrary`, and a `Pretty` — collapsing the chain `interface X` + `const codecX` + `type XT = typeof` + `class XImpl`.
- The `<Self>` self-generic is mandatory: omitting it resolves the return type to the `MissingSelfGeneric` template-literal string, surfacing the error at the class header rather than at a use site.
- `Schema.Class` is curried — `Class<Self>(identifier)` returns the fields-taking function; the identifier is positional and required, distinct from the trailing `annotations` argument.
- The instance type is `Struct.Type<Fields> & Inherited & Proto` over three orthogonal slots: `Fields` is the decoded record, `Inherited` is the parent on `.extend`, `Proto` is prototype behavior (`{}` for a plain class, `cause_.YieldableError` for `TaggedError`) — class-body methods and getters ride `Proto` and ARE part of the instance type while the codec `I = Struct.Encoded<Fields>` never serializes them, so a getter changes the dispatch surface with zero wire-shape change and the round-trip reconstructs it because decode runs `new this`.
- `Self.fields` re-exposes the original `PropertySignature` objects (not resolved schemas) as `{ [K in keyof Fields]: Fields[K] }`, so spreading carries each field's `optionalWith`/`withConstructorDefault`/`fromKey`/brand modifiers intact — `class Sub extends Schema.Class<Sub>("Sub")({ ...Parent.fields, extra })` derives a sibling owner whose shared keys reshape on a parent modifier change where a transcribed record drifts.

[CONSTRUCTOR_ADMISSION_SEAM]:
- `new Self(props)` and `Self.make(props)` run one fixed four-step transform with no branch on field shape: shallow-copy `props` (the immutability seam — the literal stays unaliased and reusable), `delete props["_tag"]` (gated `kind !== "Class"`), `lazilyMergeDefaults`, then the validation gate; all four owner kinds route through one `makeClass` body differing only by `kind` string and `Base`.
- `_tag` is deleted-then-regenerated from the tag field's own `withConstructorDefault(() => tag)`, so `new Member({ _tag: "Wrong", ...fields })` cannot forge identity — the supplied tag is stripped before merge and the canonical literal re-supplied; the runtime `delete` and the type-level `Struct.Constructor<Omit<Payload, "_tag">>` enforce input-forbidden from both sides.
- `lazilyMergeDefaults` fills `out[key]` only when `out[key] === undefined && isPropertySignature(field)`, iterating `Reflect.ownKeys` (string AND symbol), so a bare `Schema.String` field never receives a construction default; it reads the default off the type-side node by tag (`PropertySignatureDeclaration.defaultValue` else `ast.to.defaultValue`), so a `fromKey`-renamed defaulted field still merges because the default rode the type-side `to`.
- `make` is the `this`-polymorphic static `make<C extends new (...args) => any>(this: C, ...): InstanceType<C>` forwarding varargs to `new this(...args)` — identical validation path, pipe-friendly, never a bypass; `ConstructorParameters<C>` resolves against the static receiver so `Sub.make` reads `Sub`'s own widened parameters with zero per-subclass `make` re-declaration.
- The constructor validates against `constructorSchema = schema.annotations(...)` — the **type side** — through `ParseResult.validateSync = getSync(AST.typeAST(schema.ast), true)`: `typeAST` collapses every `Transformation` to its `.to` face (erasing encode/decode functions) but **preserves every `Refinement`**, so construction checks field refinements and cross-field invariants while running no representation conversion.
- This splits the two admission events of one owner: `new`/`make` validate the **decoded type plus refinements**; `Schema.decodeUnknown(Self)` validates the **encoded type, runs every transform, then the refinements** — a `Schema.NumberFromString` field accepts a `number` at the constructor (the `string → number` transform erased to its `number` target) and a `string` at decode, so `new Self({ count: "5" })` is a constructor type error while decode admits it.
- `new Self(props, { disableValidation: true })` (bare `true` is the `MakeOptions` shorthand) skips ONLY the `validateSync` call; the shallow copy, `_tag` strip-and-regenerate, and `lazilyMergeDefaults` all run before the gate, so a trusted instance is well-shaped (tag-correct, defaults-filled) but invariant-unverified — trust suppresses the predicate, never the structural normalization.
- `disableValidation` is reachable only at the in-process constructor seam; `Schema.decodeUnknown` takes `ParseOptions` and has no such knob, and the asymmetry is structural: the decode transform's `from` (the refined encoded struct) parses one node earlier, so its inner `decode: i => new this(i, true)` is safe — a wire value can never bypass the refinement, a caller granting trust at `new Self` removes the only place it ever runs.

[FIELD_MODIFIER_ALGEBRA]:
- A field value is `Schema.All | PropertySignature.All`; every modifier returns a `PropertySignature`, so they compose by `.pipe` — `Schema.String.pipe(Schema.propertySignature, Schema.fromKey("source_key"))` — because `PropertySignature` is `Pipeable`.
- `Schema.optional(s)` is the unconfigured optional (absent-or-present with `| undefined` on both sides); `Schema.optionalWith(s, opts)` is the configured optional whose `OptionalOptions` is a CLOSED discriminated union — `{ exact?; nullable? }`, `{ default: LazyArg<A>; exact?; nullable? }`, four `{ as: "Option"; ... }` rows — so illegal mixes (`default` with `as`) are type-level rejected; `{ default }` flips the type-side token `?: → :` (required-with-default on the instance, optional on the encoded side), `{ as: "Option" }` lifts to `Option<A>`, `{ exact: true }` removes `| undefined` from the admitted set.
- `Schema.fromKey(encodedKey)` remaps only the *encoded* key, leaving the instance name canonical — the boundary rename rides the field inline, composes after `propertySignature`, and preserves the `HasDefault` flag through the rewrite.
- The three default seams are three distinct AST writes, not one knob: `withConstructorDefault(() => v)` writes the type-side `defaultValue` slot `lazilyMergeDefaults` reads (fires at `new`/`make`, invisible to decode); `withDecodingDefault(() => v)` writes a `PropertySignatureTransformation` running in the decode parse (fires at decode, invisible to construction); `withDefaults({ constructor, decoding })` stacks both writes.
- Both seams share one missing-value predicate — an explicit `{ key: undefined }` triggers the same default as omission — yet diverge at the type level decisively: `withConstructorDefault` flips `_HasDefault` to `true` (becomes constructor-optional), while a *pure* `withDecodingDefault` flips the type token `?: → :` keeping `_HasDefault` false, so the field stays constructor-**required** — a decoding default never relieves the constructor of demanding the key.

[THREE_AST_CODEC_TOPOLOGY]:
- `Self.ast` materializes lazily (cached per class in a process-global `astCache` `WeakMap`) as `transform(encodedSide, declaration, { decode: i => new this(i, true), encode: identity })`: the `from` is the encoded-side struct AST, the `to` is an opaque `AST.Declaration` (a nominal node, never the field-record `TypeLiteral`) carrying an `instanceof`-keyed decode — so the three faces partition the work, `from` validating representation+refinements, the transform reconstructing, `to` asserting nominal identity.
- `typeAST(Self.ast)` collapses the `Transformation` to `typeAST(ast.to)` (the nominal declaration, not the field record) while preserving EXACTLY six annotation ids onto the surviving node — examples, default, jsonSchema, arbitrary, pretty, equivalence — dropping every other annotation on the transformation slot, the silent trap behind misrouting a `title`/`message` to the wrong annotation slot; `encodedAST(Self.ast)` walks `ast.from`, recursing each field's own transform to its encoded side, so a `NumberFromString` field contributes its own `from`/`to` and the topology nests to fully-primitive wire forms.
- The opaque `to` `Declaration` exposes no property signatures, so the class plants the type-side struct as `SurrogateAnnotationId` on the declaration and a merged surrogate on the Transformation node: JSON-Schema generation checks `getSurrogateAnnotation` first and redirects the whole walk through the surrogate struct (the sole reason a class has a structural JSON Schema), and union decode dispatch reads `getLiterals` off the surrogate to harvest each member's `_tag` — the surrogate is why a `Schema.Union` of class owners dispatches `decodeUnknown` by discriminant with no manual switch.
- The `fallbackInstanceOf` arm (`hasProperty(u, classSymbol) && is(typeSide)(u)`) admits a value bearing the per-class symbol and matching the type-side struct even when not `instanceof this`, so re-deserialized or cross-realm instances re-admit structurally where raw `instanceof` would reject.

[ANNOTATION_TUPLE_TARGETING]:
- `ClassAnnotations<Self, A>` is `Annotations.Schema<Self>` OR the three-slot tuple `[typeAnnotations, transformationAnnotations?, encodedAnnotations?]`; the object form is sugar for the one-element tuple targeting the type face alone, so reaching the encoded or transformation face REQUIRES the array brackets — each face is a separately-annotatable AST, never one map for the whole codec.
- The type-face slot seeds FOUR derivations — `declarationSurrogate` (the projection-recoverable struct), `typeSide` (drives `fallbackInstanceOf`), `constructorSchema` (the validator `new`/`make` run), and the merged surrogate — so a `parseOptions` placed there governs the constructor's own validation AND every type-face decode at once, one policy value reaching four schemas the loose spelling threads per call site; the encoded slot annotates only the `from` struct so a `jsonSchema` override shapes the wire projection without touching the validator.
- The aspect split is by whether the aspect's type names `Self`: `parseOptions`/`parseIssueTitle`/`title`/`message`/`decodingFallback` are typed against the issue or encoded value and sit inline at the class header; `equivalence`/`arbitrary`/`pretty` are typed `Equivalence<Self>`/`LazyArbitrary<Self>`/`Pretty<Self>` and trip `TS2310: 'Self' recursively references itself as a base type` at the header — these ride the `fieldsOr` struct via `Schema.annotations({ equivalence, pretty })` over the field-record type before the class wraps it.
- Bolting `Self.annotations(a)` on AFTER declaration returns a `SchemaClass<Self, I, R>` stripped of `new`/`make` and the instance base — the trap that splits the aspect from the owner; the trailing tuple argument keeps the invariant, the failure rendering, the recovery, and the comparison on the one declaration that also yields the constructor, and all of them survive `.extend`/`.transformOrFail`.

[OWNER_WIDENING_LAW]:
- `Self.extend<Sub>("SubId")({ ...newFields })` produces `Class<Sub, Fields & NewFields, I & Struct.Encoded<NewFields>, R | Struct.Context<NewFields>, C & Struct.Constructor<NewFields>, Self, Proto>` — the parent becomes `Inherited`, the prototype chain and `Proto` survive (a `TaggedError` subclass stays yieldable), `instanceof Parent` holds; it is structural widening, not re-declaration.
- The constructor obligation `C` accrues by the SAME `C & Struct.Constructor<NewFields>` intersection across every widening verb (`.extend`, `.transformOrFail`, `.transformOrFailFrom`) — a left-fold of per-step intersections, not a re-derivation from `Fields & NewFields`, because each step only HAS `NewFields` in scope; the intersection is why a parent's optionality survives untouched (`{ readonly K?: T } & X` keeps `K` optional, so no widening silently re-requires a relieved key).
- Void-ability is read off `RequiredKeys<C> extends never`, where a key is required exactly when `{}` is not assignable to its single-key pick (neither `?:`-optional nor `_HasDefault`-backed): the set is monotone-up under accrual (a widening can only add keys), so a defaulted addition leaves every call site untouched while the first bare addition flips THIS owner to props-demanding and breaks every `new Sub()`/`Sub.make()` site at compile time — locally, never reaching up the chain, the parent's `new Parent()` still type-checking.
- The modifier algebra on the new field, read off `C` alone, IS the anticipatory proof: a `withConstructorDefault`/`optional`/`optionalWith({default})` addition contributes an optional factor (silent-compatible), a bare addition a required factor (loud break) — sizing an owner all-defaulted absorbs additions with zero churn; the rejected order is sizing with a required field then relieving it, which leaves existing `new Self(props)` over-supplying a now-optional key while the type permits omission.
- A parent-versus-new key collision is a hard `throw` inside `extendFields(a, b)` (`for (const key of Reflect.ownKeys(b)) if (key in a) throw`) which runs BEFORE `makeClass` for all three verbs — so a colliding subclass crashes at module-load (definition time), never at first `new Sub`; widening is total over the full own-key set (string AND symbol) or it fails loudly. The top-level `Schema.extend(that)` has broader reach (struct⋈struct, ⋈`Schema.Record` index, ⋈union, ⋈refinement/suspend) and admits overlap where the overlap supports extension, where `Class.extend` throws on any name collision.
- The two effectful verbs differ in ONE expression — the schema each hands `makeClass`: `.transformOrFail` builds `transformOrFail(parentCodec, typeSchema(Struct(merged)), …)` (decode sees `Struct.Type<Fields>`, the DECODED parent), `.transformOrFailFrom` builds `transformOrFail(encodedSchema(parentCodec), Struct(merged), …)` (decode sees `Simplify<I>`, the parent's ENCODED record) — the suffix is a `from`-side binding decision, and the `decode` input type the checker enforces is the read-out of which AST face the inner `from` walked to.
- The position is the law a computed field's source obeys under one fixed parse order (`from(i)` → `decode` → `to`): a field that is a function of DECODED parent values (a sum of parsed numbers) is a `transformOrFail` row, a field that is a function of RAW encoded bytes (a checksum over the wire string, a discriminant sliced from a not-yet-parsed value) is a `transformOrFailFrom` row — the wrong suffix feeds the derivation a representation it cannot compute from, surfacing as a type error on the `decode` input parameter, not a runtime drift.
- Both verbs leave the encoded face `I` UNCHANGED — unlike `.extend` which widens it to `I & Struct.Encoded<NewFields>` — because `encodedAST` on a `Transformation` recurses the `from` exclusively and the new fields sit on the inner `to`: a computed field accrues into `C` (constructible) and the instance type yet stays wire-invisible (reconstructed every decode, dropped on encode), so a memoized aggregate or parse-time index lands as one transform row instead of a forgeable `.extend` wire field plus a strip-before-encode discipline.
- Both transform verbs union the function requirements `R2`/`R3` onto the owner's `R` (where `.extend` accrues only `R | Struct.Context<NewFields>`), so a context-dependent derivation — a computed field needing an injected service — is reachable ONLY through the transform verbs, surfacing on the owner's own channel, provided once at the composition root, firing at decode while in-process `new` supplies the field as a plain value; the requirement growth is itself a discriminant of which verb widened. Both pass through the identical `makeClass` body, so each is a FULL owner (`new`/`make`/methods/`Equal`/`Hash`/`Self.fields`) where a top-level `Schema.transformOrFail` over a class would demote to a bare codec view.

[SUSPEND_RECURSION_CYCLE]:
- A bare `Self` inside the field record reads a temporal-dead-zone binding and fails twice as one cause — `ReferenceError` at module load, `TS2506 'Self' referenced in its own base expression` plus `TS7024` implicit-any on the thunk; `Schema.suspend(() => Self)` (`= make(new AST.Suspend(() => f().ast))`) is the only legal forward reference because the thunk body is unevaluated at record-construction, so the field record completes and the `class Self` binding lands before any code reads `f`.
- The explicit `Schema.suspend((): Schema.Schema<Self> => Self)` return annotation is mandatory exactly when recursion makes the return type self-referential — the type-level analogue of the runtime deferral; the single-parameter `Schema.Schema<Self>` form holds ONLY while the encoded face equals the instance type, and a class method/getter (riding `Proto`, absent from the encoded record) breaks that equality with `TS2322 'Encoded' … missing`, forcing a hand-declared encoded `interface` as the `I` argument — the same encoded-face cut a mutual cycle imposes, here triggered by the `Proto`-versus-codec divergence.
- The cycle is broken at two `memoizeThunk` layers each firing once: the `AST.Suspend` constructor wraps its `f` (freezing one node identity), and the parser's `Suspend` arm builds `const get = memoizeThunk(() => goMemo(ast.f(), isDecoding))`; stable node identity is the precondition for the second cache because `goMemo` is `WeakMap`-keyed on the node and `AST.equals` for a `Suspend` is reference identity (`self === that`) — a structural compare would itself recurse forever. A static-face fold re-wraps rather than resolves (`typeAST` returns `new Suspend(() => typeAST(ast.f()))`, `encodedAST` minting a derived `${id}Encoded` JSON identifier so the type-side and wire-side cycles resolve to distinct `$ref`s), so projecting a recursive owner yields another recursive schema with the cycle intact, and the parser, `Schema.equivalence`, and `Pretty` all close it with the one shared `memoizeThunk(() => go(ast.f()))` idiom.
- The derived `Arbitrary` terminates by a DIFFERENT cycle-break — a two-pass reference graph, not the parser's single deferral: a `getDescription` pass converts a re-encountered `Suspend` into a `{ _tag: "Ref", id }` via an `idMemoMap`, so the recursion is materialized as an explicit `letrec`-style graph whose `id` is the `depthIdentifier` fast-check decays against, and a suspended array field generates via `fc.oneof({ maxDepth: 2, depthIdentifier }, fc.constant([]), fc.array(...))` biasing toward the empty arm as depth accrues; the recursive context clamps `maxLength` to `Math.max(2, minLength ?? 0)` REGARDLESS of a wider `Schema.maxItems` annotation — the depth-decay overrides the breadth constraint because unclamped recursive breadth multiplies into unbounded trees, the annotation governing only the leaf shape.
- JSON-Schema generation ALWAYS demands an identifier for a `Suspend` (forcing identifier handling even under `topLevelReferenceStrategy === "skip"`, throwing `requires an "identifier" annotation` at the suspended path otherwise) and breaks the cycle by pre-registering `$defs[id] = { $ref }` BEFORE recursing the body — a `Schema.Class` supplies its constructor identifier as the JSON identifier automatically, the named class being the collapsed form an anonymous recursive struct forces a manual `.annotations({ identifier })` to match.
- A mutual cycle closes when each owner's suspend thunk names the OTHER's eventual binding (`A → Suspend → B → Suspend → A`), forces an explicit encoded `interface` on at least one node (inference cannot close an encoded type naming itself — `Schema.suspend((): Schema.Schema<Folder, FolderEncoded> => Folder)`), and leaves every member a full owner: a recursive union member that is a `TaggedClass` keeps its discriminant and body behavior, the suspension wrapping the reference, never demoting it.

[PARSE_FAILURE_OWNERSHIP]:
- The eight-arm union `Type | Missing | Unexpected | Forbidden | Pointer | Refinement | Transformation | Composite` is three structural roles, not eight peer faults: `Composite` is the only branch (`issues: SingleOrNonEmpty<ParseIssue>`), `Pointer` the only path edge (`{ path: Path; actual; issue }`, one child plus a `PropertyKey` step), `Refinement`/`Transformation` single-child kind-tagged wrappers, and `Type`/`Missing`/`Unexpected`/`Forbidden` the terminating leaves — a consumer folds on branch/edge/wrapper/leaf keyed by `_tag`, never eight arms.
- Only `Pointer` carries `path`; a flat per-field address is the concatenation of every `Pointer` along the root-to-leaf spine, never a field on the leaf — so a leaf relocated under a different key needs no change, its address living in the edges above it, and `Path = SingleOrNonEmpty<PropertyKey>` lets one edge carry a multi-level re-key. `Missing.actual` is fixed `undefined` (absence has no value) while `Missing.ast` recovers the property signature whose key never arrived.
- `errors: "all"` versus default `"first"` switches the tree topology at every struct node via one repeated `if (allErrors)` arm — push `[stepKey, new Pointer(...)]` onto a local `es` array and `continue`, or return `Composite` immediately — and `Composite` always carries `output`, the partially-decoded record of fields that passed, so a wide failing decode retains every well-formed field beside the issue list. Baked as `parseOptions: { errors: "all" }` on the header it rides `Self.ast` (merged by `goMemo` via `mergeInternalOptions`), so every `decodeUnknown(Self)`, every nested decode of `Self` as a member, and the `new`/`make` validator inherit the wide fan with no per-seam argument.
- Three unrelated fault origins converge on the identical `Composite`/`Pointer` topology so one `ParseError` handler walks them uniformly: a nested field decode failure, a cross-field `Schema.filter` invariant returning a path-targeted issue, and a refinement returning a structured `ParseIssue` all terminate at the same leaf algebra — a width-violation on `upper` lands at the exact `upper` address a mistyped `upper` would, the consumer never branching on origin. `Refinement.kind: "From" | "Predicate"` splits "the fields were malformed" from "the fields were valid but the cross-field invariant failed", and under `errors: "all"` a `From`-failing stable filter emits BOTH children at once.
- `Forbidden` is the lone execution-mode-mismatch leaf (not a shape fault): `handleForbidden` forks an `Effect`-producing parser on a `SyncScheduler`, and if `unsafePoll()` returns nothing (async work performed) emits `Forbidden(ast, actual, "cannot be resolved synchronously")`; a synchronous DIE (unprovided dependency) becomes `Forbidden(ast, actual, Cause.pretty(cause))` while a real `ParseIssue` propagates unchanged — `options.isEffectAllowed` is the single disarming gate, so an owner with a `filterEffect` invariant or a `Context.Tag`-bearing field decodes cleanly through `decodeUnknown` (effect-allowed) yet produces a `Forbidden` leaf through `decodeUnknownSync`/`Either`, the requirement surfacing on the issue tree when the entrypoint erased `R`.
- `parseOptions` bakes per-owner parse policy at definition — `errors: "first" | "all"`, `onExcessProperty: "ignore" | "error" | "preserve"`, `propertyOrder`, `exact: true` — so an owner that must report every field failure or reject excess keys declares it once, never threaded as a decode-call override.
- `decodingFallback: (issue) => Effect<A, ParseIssue>` is wired at `goMemo` as an `orElse` arm baked onto the node's own parser, firing automatically at every decode of the owner (never a per-consumer `catch`) and ONLY on the decode path (never encode — trust is inbound) — the recovery `Effect` can itself fail with a refined issue, and being wrapped in `handleForbidden` it inherits the sync boundary, so a context-dependent fallback is admissible only through an effect-allowed entrypoint.
- `message: (issue) => string | Effect<string> | { message; override }` resolves by `getMessage`, the tree's subtlest policy: it WALKS toward the inner issue under `useInnerMessage = !override && (isComposite || (isRefinement && kind === "From") || (isTransformation && kind !== "Transformation"))`, so a leaf-precise field message survives bubbling up through a class owner unless the owner sets `{ override: true }` to replace the whole subtree — the precedence is data on the annotation, never call-site order; an `Effect<string>` return makes the message a fallible context-readable computation resolved inside the parse. A `Missing` leaf carries its message on a SEPARATE field-level `missingMessage: () => string | Effect<string>` nullary thunk (absence has no `actual` for a `(issue) => …` message to inspect), riding the specific property signature so an owner with several required fields names each omission distinctly, never one header annotation that cannot tell which key vanished. `parseIssueTitle: (issue) => string | undefined` reshapes a node's own title keyed on its AST (so `Unexpected`, the lone leaf without an `ast`, can never carry one), discriminating a `Composite` invariant title from each `Type` leaf by `issue._tag`.
- `ParseError` is the `YieldableError` wrapper holding `{ issue }`, so the whole tree travels in the typed `E` channel as one value a boundary `Effect.catchTag('ParseError', …)` folds once — and `Composite.output` makes the fold non-destructive (render the subtree AND read salvaged fields from one node), deleting the loose spelling of a second best-effort decoder beside the strict one.

[REFINED_OWNER_INVARIANT]:
- `Schema.Class`/`TaggedClass`/`TaggedError` take `fieldsOr: Fields | HasFields<Fields>` where `HasFields<F> = Struct<F> | { readonly [RefineSchemaId]: HasFields<F> }`, so `Schema.Struct({...}).pipe(Schema.filter(pred))` is a legal first argument — the owner is born from a refined struct, never a plain record plus a separate post-construction check; the `refine` result declares `[RefineSchemaId]: From` precisely to satisfy that arm, and the constructor unwraps it so `Self.fields` still re-exposes each `PropertySignature` after the invariant wraps them.
- The cross-field invariant rides the owner as a definition-time aspect gating EVERY admission event — `new`/`make` (the refinement is a `Refinement` node `typeAST` preserves) and `decodeUnknown` alike — so a refined struct whose every field is defaulted yields a `void`-able `new Self()` that still runs the refinement over the defaulted instance, the invariant never skipped for the empty-argument form; this collapses `class X` + free `assertValid(x)` + `if (!ok) throw` + the call-site discipline that remembers to invoke it into one declaration.
- The `Schema.filter` predicate returns `FilterReturnType = FilterOutput | ReadonlyArray<FilterOutput>` over `undefined | boolean | string | ParseResult.ParseIssue | FilterIssue`: `undefined`/`true` pass, `false`/`string` report the annotation/that message, `FilterIssue = { path; message }` targets a specific field, the ARRAY form emits multiple path-targeted issues in one pass (the one-pass multi-field report under `errors: "all"`, never chained sibling filters), and a returned structured `ParseIssue` injects a `Pointer`/`Composite` subtree identical to a nested decode failure.
- A predicate that is a type guard (`(a): a is B`) narrows the owner's type to `C & B` on every instance and projection, so the narrowing invariant and the gating invariant share one `Schema.filter` surface; `Schema.filterEffect(f)` (where `f: → Effect<FilterReturnType, never, FD>`, extending `transformOrFail<S, SchemaClass<Type<S>>, FD>`) joins the requirement `FD` onto the owner's `R` for a dependency-injected invariant, and `Schema.filter` annotations carry `schemaId`/`jsonSchema` so the gate projects into the JSON Schema and is registry-recoverable, never opaque to derived surfaces.

[TAGGED_OWNER_FAMILY]:
- `Schema.TaggedClass`/`Schema.TaggedStruct`/`Schema.TaggedError` are one `Proto`-parameterized owner family over the shared `makeClass` body, differing only by the `Proto` slot and whether a class base is minted: `TaggedClass<Self>()("Tag", fields)` injects `_tag: tag<"Tag">` as the first field (a `withConstructorDefault`-backed literal exposed as a static readonly discriminant) with `Proto = {}`, `TaggedError` is the same shape with `Proto = cause_.YieldableError`, and `TaggedStruct("Tag", fields)` is the value-only sibling — a `Struct<{ _tag } & Fields>` with no class/`Equal`-base/`new`, `Self.make` omitting `_tag` — reached when no instance methods or nominal identity are needed.
- The `_tag` travels as a compiler-checked symbol (`Self._tag` static, `instance._tag` literal type), so exhaustive dispatch keys off it with no restated string; the identical `_tag` field is a data record in a `TaggedClass` and a typed failure-channel member in a `TaggedError`, the difference being exactly the `Proto = YieldableError` intersection the seventh type parameter supplies, so promoting a tagged-class owner to a failure type is a one-slot change, never a re-declaration.
- The `Proto = cause_.YieldableError` slot is a four-channel polymorphic membership pinning the failure type to `this`: `[EffectTypeId]`, `[StreamTypeId]`, `[SinkTypeId]`, `[ChannelTypeId]` variance structs all bind the error to the same `this`, so one declaration makes the instance a member of the `E` alphabet of an `Effect`, `Stream`, `Sink`, AND `Channel` simultaneously.
- The owner is BOTH yieldable AND value-keyed because of one load-bearing prototype-spread order: `StructuralCommitPrototype = { ...CommitPrototype, ...StructuralPrototype }` where `CommitPrototype` supplies the four variance slots / iterator / `pipe` / `_op: OP_COMMIT` and `StructuralPrototype` OVERRIDES the `[Equal.symbol]`/`[Hash.symbol]` only — replacing `EffectPrototype`'s reference identity with value-structural comparison surgically, every other slot surviving — so two equal-field error instances satisfy `Equal.equals` and key one `HashSet` bucket while remaining failure-channel members.
- `yield* error` is the value itself, not sugar over `yield* Effect.fail(e)`: the instance bears `_op: OP_COMMIT`, the fiber's evaluation table arm `[OP_COMMIT](op) => internalCall(() => op.commit())` calls the base `commit() { return fail(this); }`, and the single-shot iterator (`new SingleShotGen(new YieldWrap(this))`) drives the gen — so the error IS an `Effect` node whose evaluation is its own failure, the `fail` lift happening inside `commit` at fiber-step time, needing no `.toEffect()`/`.fail()` and no `Effect.fail` wrap at the raise site.

[PROMOTION_AND_COLLAPSE]:
- The promotion ladder is monotone, and sizing below the needed rung splits a codec from its value across two declarations: `Schema.Struct` (no method/tag/`Equal`-consumer) → `Schema.Class` (the moment a getter, tag, or structural-equality consumer appears); a `Data.taggedEnum` (value-only, no codec) → a `Schema.Union` of `TaggedClass` members (the moment a member needs a validating constructor, wire codec, `Equal`/`Hash`, or `instanceof`); a `Schema.Class` → a `TaggedRequest` (the moment a success/failure result codec attaches to the same concept) → a `Model.Class` (the moment a second persistence/wire variant attaches). Sizing for the instance and bolting capability on later is the rejected order; the moment a second case is conceivable, the owner generalizes.
- A projection that must stay instantiable derives field-side from `Self.fields` into a new `Schema.Class`/`Schema.Struct` (retaining `new`/`make`/`Equal`-base/`_tag`/methods); a projection used only as a transient decode/encode contract uses a top-level combinator (`Schema.pick`/`omit`/`pluck`/`partial`/`required` over a `Class` all return a bare `SchemaClass` — `Schema` plus `make` only, the value-constructor and instance base gone) — the choice is owner-retention versus throwaway view, never style. `Schema.Struct(fields)` keeps its `make`/`.fields`/`.pick`/`.omit` instance methods (struct-level projection preserves the struct), but `Schema.Struct(fields, indexSignature)` returns a `TypeLiteral` with no `.pick`/`.omit` — admitting one index signature forfeits struct-level projection, so a struct that must stay projectable keeps open keys as a `Schema.Record` field rather than an index signature; `Schema.asSchema(self)` is a viewing cast at a `Schema.Any` registry boundary, never a step before `new`/`make`.
- The collapse triggers are concept-density signals, each naming the exact loose spelling the owner deletes: ≥3 parallel record shapes or sibling struct factories for one concept fold into one `Schema.Class`/`TaggedClass` union member whose field modifiers carry per-variant optionality; ≥3 sibling tagged classes consumed together (one wire field, one switch, one column) fuse into one `Schema.Union` (deleting the per-member `is`-guard zoo and the `A | B | C` restatement); ≥3 co-traveling shapes for one operation (payload, result, error, hand-written `{encode, decode}`) fold into one `TaggedRequest` (deleting the standalone `Request.tagged` plus separate response schema); ≥3 sites hand-rolling a `Map<string, Owner>` + an `eq`-by-id helper + a comparator collapse onto the owner directly — the `Data.Class` prototype base makes a class owner a value-keyed `HashSet`/`HashMap` key with no key function (deduplication is `HashSet.fromIterable`, a multi-key sort is one `Order.struct` read off `Self.fields`), the structural `Equal`/`Hash` mechanics living at greater depth elsewhere. A `type X = Schema.Schema.Type<typeof Self>` restating what the class already is, a second `const` rebuilding the shape, or a standalone `new XImpl()` after declaration are the exact spellings the class-as-both-type-and-value owner deletes.
