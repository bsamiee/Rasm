# [API_CATALOGUE] rfc6902

`rfc6902` supplies the RFC 6902 JSON Patch and RFC 6901 JSON Pointer surface the recorded-intent partial-update rail composes: the six-verb `Operation` discriminated union (`add | remove | replace | move | copy | test`), the error-accumulating `applyPatch` that returns one nullable error slot per operation rather than throwing, the single-op `apply` and its six per-verb functions, the `createPatch` Levenshtein diff generator, the `createTests` optimistic-concurrency guard builder, the `Pointer` RFC 6901 evaluator with its `escapeToken`/`unescapeToken` primitives, and the recursive `clone`. It is the decode-side owner of the C# `JsonPatchDocument<TModel>`/`FieldMask` partial-update wire the `Codec/patch.md` `PatchRail` folds — the six-verb union decoded off the wire, never a state-diff re-derived at the consumer. The load-bearing subtlety is the module split: the barrel re-exports only eight symbols; the error classes and per-verb functions the fault fold pattern-matches on require deep-module imports.

## [04]-[IMPLEMENTATION_LAW]

[PATCH_TOPOLOGY]:
- `applyPatch` is error-accumulating, never throwing: one slot per op, `null` on success and the typed error on failure, so a partial-failure patch reports every per-op outcome in order rather than aborting at the first fault — the slot index is the op index, resolving the exact failing op.
- `applyPatch` mutates the target in place and there is no immutable-apply variant: the consumer snapshots first. The `Codec/patch.md` rail pre-clones with native `structuredClone(value)` and applies to the clone, so the pre-patch value survives a mid-patch fault; the package's own `rfc6902/util` `clone` is the recursive-copy alternative where `structuredClone` cannot reach the value shape.
- `test` carries optimistic concurrency: a `TestOperation` whose `value` deep-inequals the pointer target returns `TestError(actual, expected)`, so a patch built as `[...createTests(clone, patch), ...patch]` self-guards a replayed or stale apply — the guard prefix occupies the leading `guard.length` result slots.
- op order is significant — `move`/`copy` reference a `from` pointer evaluated against the post-prior-op state, so reordering changes the result.

[POINTER_LAW]:
- `Pointer.fromJSON` consumes a properly-escaped string; `escapeToken`/`unescapeToken` are the RFC 6901 primitives (`~1`→`/` decoded before `~0`→`~`, so `~01` correctly resolves to `~1` not `/`). A `fieldMaskLower` building pointers from dotted field paths escapes each token through `escapeToken` rather than a bare `.replace`.
- `pointer.evaluate` returns `{ parent, key, value }` so a setter mutates `parent[key]` without re-walking; `get` returns `undefined` for an absent path and never throws.
- the empty pointer `""` addresses the document root (`{ parent: null, key: "", value: object }`); `"/-"` addresses the array-append slot on `add`, and `Options.implicitArrayCreation` lets `add` at `/-` infer a missing leaf array.

[STACKS_WITH]:
- `effect` (`.api/effect.md`): decode the raw wire patch through `Schema.decodeUnknown` over the six-arm `PatchOpWire` `Schema.Union` BEFORE `applyPatch`, so an unknown `op` faults at the decode boundary as `Additive`/`Breaking` drift, never as a post-apply `InvalidOperationError` slot, and the decoded `ReadonlyArray<Operation>` is what `applyPatch`/`createTests` receive; the `(error | null)[]` result then folds into one typed fault — scan the first non-null slot, resolve its op by slot index against the `[...guard, ...patch]` array, and classify by `instanceof TestError` (concurrency mismatch) vs `MissingError`/`InvalidOperationError` (apply failure) into the `Ingress/fault.md` `Data.TaggedEnum` dispatched by `Match.tagsExhaustive` — the `instanceof` check is why the error classes deep-import from `rfc6902/patch`
- `@bufbuild/protobuf` (`.api/bufbuild-protobuf.md`): the `Codec/patch.md` `fieldMaskLower` projects a C# `@bufbuild/protobuf/wkt` `FieldMask` sparse-update message into `replace` operations — each masked dotted path escaped through `escapeToken` into an RFC 6901 pointer, its value resolved out of the update message via `Pointer.fromJSON(pointer).get(message)`; `@bufbuild/protobuf` `isFieldSet`/`clearField` supply the `remove`/presence arms and `clone(schema, message)` snapshots the pre-apply message
- `Codec/patch.md` recorded-intent rail: `createPatch` is a dev-time diff generator only — a contract patch is recorded intent decoded off the wire, never a diff re-derived at the consumer against a local before/after

[RAIL_LAW]:
- Package: `rfc6902`
- Owns: RFC 6902 JSON Patch apply/single-dispatch/per-verb + diff generation, and RFC 6901 JSON Pointer evaluation with its escape primitives and recursive clone
- Accept: a `Schema`-decoded six-verb `Operation[]`; a pre-cloned target to mutate in place; the barrel for `applyPatch`/`createTests`/`Pointer`, deep `rfc6902/patch` for the error classes the fault fold matches
- Reject: a state-diff endpoint re-deriving the patch at the consumer; a throwing apply that loses per-op outcome; an undecoded raw-wire patch applied before the `op` `Schema.Union` admits it; a barrel import of the error classes (they are deep-only, and the `instanceof` check silently fails)
