# Definition-Time Generation Substrate

[PER_ASPECT_PROVIDER_FAN_OUT]:
- The admission owner is not generated by one monolithic pass; each cross-cutting capability — formatting, comparison, parsing, span-parsing, comparison operators, equality operators, arithmetic operators, the serializer surfaces, and the core owner body — is an independent incremental provider keyed off the same upstream attribute discovery. One owner declaration fans into a dozen parallel `IncrementalValuesProvider` streams, each `Where`-gated by a predicate over the same extracted state, so an owner that opts out of an aspect produces zero work in that aspect's stream rather than a skipped branch in a shared body. The aspects compose by addition of independent generators emitting into one partial, never by one generator branching on flags.
- Each aspect stream is the identical five-stage skeleton: `Where(predicate over state)` selects the owners that need it, `Collect` materializes them into one array, a `Select` reduces the array to `Distinct` by type, `WithComparer` installs an order-independent set comparer, and `SelectMany` re-expands to per-owner outputs that `Combine` with the global options and feed `RegisterSourceOutput`. The predicate is the only aspect-specific line; the collect-dedup-recompare-expand spine is shared, so the pipeline is one parameterized shape instantiated per capability, not N hand-written pipelines.
- The aspect predicate encodes the generation decision as data, not control flow: the comparison-operators stream admits an owner only when its operator-generation level is non-none and the key member carries comparison operators, owns a comparer accessor, or is a string; the span-parsable stream admits only when the key member is span-parsable or the owner declares a span-of-char validate grammar and the parse surface is not skipped. The whole "should this surface exist" question is a boolean fold over the extracted settings, evaluated once per owner per aspect, with the owner's own policy values as the inputs.

[EQUATABLE_MODEL_IS_THE_CACHE_KEY]:
- Every extracted state flowing through the pipeline implements `IEquatable<TSelf>` and `IHashCodeComputable`, and the whole-model equality is structural over every setting the owner carries — key kind, factory method names, the absent-as-null knobs, each operator level, the conversion levels, the comparer accessors, member declaration order. The incremental engine compares the new model against the cached one by that structural equality; an edit that does not change any extracted setting produces an equal model and short-circuits regeneration of the entire owner, while flipping one operator level produces an unequal model and invalidates the owner's whole output, never a fragment. The cache granularity is the owner, and the cache key is the model's structural identity.
- The extraction step is the firewall between the compilation's mutable symbol graph and the cache: the candidate transform reads `ISymbol` data into a frozen value model and the symbol is never carried downstream. A model that referenced a symbol would compare unequal on every keystroke because the symbol identity churns; the discipline is that the model holds only equatable value data — strings, enums, immutable arrays of member states — so two compilations producing the same admission shape produce equal models and reuse the emission.
- Member states model the key member's full capability surface as flat booleans and enums — is-reference-type, is-nullable-struct, is-formattable, is-comparable, is-parsable, is-span-parsable, the nullable annotation, the implemented comparison and arithmetic operator sets — so the generated body decides which interfaces to implement by reading the member model, never by re-querying the type system at emit time. The capability detection happens once during extraction; emission is a pure function of the frozen model.

[SET_EQUALITY_OVER_THE_COLLECTED_BATCH]:
- After `Collect` the aspect's owners live in an immutable array whose element order tracks declaration discovery order and is therefore unstable across edits; comparing that array by sequence equality would bust the cache whenever an unrelated owner moved. The pipeline installs an order-independent set comparer that hashes both arrays into a pooled hash set and tests mutual containment, so the collected batch is treated as a set — adding or moving one owner invalidates only that owner's downstream slice, leaving the rest of the batch cached.
- The set comparer's correctness rests on a precondition the dedup stage guarantees: the array must contain no duplicates, enforced by a `Distinct` keyed on the owner's fully-qualified type name immediately upstream. Two attributes on one owner, or two owners differing only in name, would otherwise produce redundant entries that the set comparison cannot disambiguate; type-name dedup collapses them to one before the set comparer ever runs, so the same owner never generates the same aspect twice.
- The dedup comparer keys on fully-qualified type name alone, not structural model equality — it answers "is this the same owner" so a single owner reached through multiple discovery paths emits one aspect file, while the structural model equality one stage later answers "did this owner change" for the cache. The two comparers are deliberately different relations on the same models: identity for dedup, structure for caching.

[SYNTACTIC_PREFILTER_BEFORE_SEMANTICS]:
- Attribute discovery runs through an attribute-metadata-name provider whose first argument is a purely syntactic predicate — the candidate is admitted iff its node is a class or struct declaration — evaluated before any semantic model is touched. The cheap syntactic gate rejects the overwhelming majority of nodes without binding symbols; only survivors reach the expensive transform that reads the semantic model and builds the equatable state. The two-phase shape (syntactic accept, then semantic extract) is what keeps per-keystroke discovery proportional to changed declarations, not to compilation size.
- The candidate predicate is wrapped in a fault barrier that logs and returns "not a candidate" on any exception, so a malformed declaration cannot crash discovery for the whole compilation — one broken node degrades to non-generation of one owner, never a generator-wide failure. The same fault barrier wraps the semantic transform and the emit step, each catching to a reported diagnostic, so the generation pipeline is total: every stage either produces its value or surfaces a diagnostic, and no stage can throw across the incremental boundary except a cancellation, which is rethrown to honor the host's cancellation token.

[SOURCE_VERSUS_IMPLEMENTATION_OUTPUT_SPLIT]:
- The owner body, its admission triple, its operators, and its diagnostics register as source output — they participate in semantic analysis and other generators can see them. The serializer surfaces register as implementation source output — emitted late, invisible to semantic analysis, never feeding another generator. The split is load-bearing: admission shape must be visible so downstream code binds against the generated factory contract, while wire codecs are terminal and gain incremental throughput by being excluded from the analysis graph. An owner facing a wire therefore has its domain surface and its codec surface generated through two different output channels with different visibility and different recompute triggers.
- The serializer streams additionally branch on which serialization assembly is referenced — the presence of a specific framework assembly in the compilation selects which codec generator runs — so wire codec generation is conditioned on transitive package admission, not on the attribute alone. Declaring a serialization flag without the framework assembly present produces the round-trip obligation diagnostic but no codec; the codec materializes only when both the flag and the assembly coexist.

[DETERMINISTIC_HINT_NAME_ENCODING]:
- The generated file's hint name is a total function of the owner's location in the type graph: namespace, then the chain of containing types each suffixed with a backtick-and-arity when generic, then the owner name with its own backtick-arity, then the aspect's file-name suffix, then the generated extension. Two nested owners with the same simple name in different containers, or the same name at different generic arities, never collide because the encoding injects the full containing path and every arity. The hint name is the emission's identity to the host, and encoding the full nesting and arity is what lets one owner's many aspect files coexist and lets distinct owners never overwrite each other.
- The aspect suffix is what partitions one owner's output across files — the comparable surface, the parsable surface, the operators surface, and the core body each carry a distinct suffix into the same name-and-arity prefix — so the per-aspect provider fan-out lands as per-aspect files under a shared owner key. Regenerating one aspect rewrites one hint-named file; the others are untouched because their providers did not recompute.
- Empty emission is suppressed structurally: the generator measures the builder length before and after the body runs and emits a file only when the body actually wrote past the prelude, logging a warning otherwise. An aspect whose body produces nothing for a given owner adds no empty file to the compilation, so the file set is exactly the non-trivial surfaces, and a misconfigured aspect that silently produces no code is observable as a warning rather than an empty artifact.

[DIAGNOSTICS_ARE_THE_DEFINITION_TIME_GATE]:
- The malformed-owner failures are not runtime exceptions but compile-time diagnostics raised from the same pipeline that generates: a non-partial owner, a non-private constructor, a primary constructor on an owner whose constructor cannot be made private, a public `init` accessor on a complex-owner member, a mutable field on the owner or its base, a nullable key, more than one shape attribute on one type. Each is the structural precondition of constructive unrevalidatability stated as an error descriptor, so the no-double-validation guarantees the generated body relies on are enforced before the body is generated, not assumed.
- The fault-family and factory-contract obligations are themselves descriptors: an owner declaring the factory attribute must implement the static validate method with the exact three-out signature or it is an error naming the required signature; an owner facing a wire must implement the outbound projection or it is an error; a trusted-rehydration flag without a matching constructor is an error; an enumeration carrying the rehydration flag is an error. The round-trip law, the validate-spine obligation, and the no-bypass-on-enumerations rule are compile-visible the moment the attribute is placed.
- The multi-attribute partitioning is a cluster of descriptors that make wire routing total at definition time: two factory attributes claiming the same serialization framework, two claiming model binding, two claiming the persistence integration are each distinct errors. Each framework concern resolves to exactly one admission grammar by construction, so a downstream codec can never face an ambiguous source, and the ambiguity is caught at the attribute, not at serialization.
- The comparer and operator descriptors enforce the relational lattice as coupling, not as free flags: a string-keyed owner without an equality comparer is an error demanding one, a complex owner with string members without a default string comparison is a warning demanding one, an owner with a comparer but no equality comparer (or the reverse) is a warning, and a comparison-operator level that diverges from the equality-operator level is an error demanding they match. The "comparison strength may not exceed equality strength" rule and the "equality and ordering must share one policy" rule are the descriptors themselves, so the generated body never has to reconcile a contradictory relational request.
- A self-referential definition-time constraint gates the whole aspect: the validation-error type bound on the owner is checked for satisfying its own static-create contract before generation begins, so a fault lattice member that fails to re-implement create for its precise case fails the constraint check and the owner never generates. The aspect that produces admission is guarded by an aspect over the fault type, and a broken fault lattice is a definition-time error rather than a runtime surprise inside a generated factory.

[EMISSION_IS_POOLED_AND_CANCELLABLE]:
- The body is written into a leased, pooled, reused builder rather than a fresh allocation per owner, with a bounded pool that returns oversized builders to the heap instead of retaining them — emission throughput is a memory-discipline concern the generator owns, because incremental regeneration runs emission on every relevant keystroke. The builder lease is per-emit and the pool is shared across owners, so steady-state regeneration of one owner reuses buffers warmed by the whole compilation.
- Every aspect's generate step threads the host cancellation token into the body and rethrows only the cancellation, catching every other exception to a reported error keyed at the owner's location. A long generation for one owner is abandoned the instant the host cancels, and a throwing generation surfaces as a located diagnostic on that owner while the rest of the compilation's owners generate unaffected — the emission stage is independently cancellable and independently fault-isolated per owner.
