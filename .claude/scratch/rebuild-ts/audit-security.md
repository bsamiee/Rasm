# AUDIT — libs/typescript/security

Line-by-line doctrine audit of the nine `security` design pages against `docs/stacks/typescript`
composed in full. Findings only. Fence locations are `<page>.md:<line>` or `<page>#CLUSTER`.

## [0]-[THE SIXTEEN LAWS — STATED AND SCOPED]

FLOW
- `EXPRESSION_SPINE` — all domain logic is expression-shaped on `Effect<A,E,R>`; dependence licenses sequence, independence licenses `Effect.all`/`zipWith`; statements survive only in measured kernels. BEARS: the folder is `Effect.gen`-heavy with `cond ? Effect.void : Effect.fail` guard-statements the rail owns as `filterOrFail`; no applicative accumulation where independent operands exist (`Material.ring`, `Material.jwks`).
- `BOUNDARY_ADMISSION` — raw material decodes exactly once through a Schema owner; `ParseError` lifts at the same seam; the interior never re-validates or sees `null`/`as`. BEARS: `sign#KEY_MATERIAL` casts `JSON.parse(pem) as JWK` and re-parses the same PEM twice inside `_admit`; header grammars in `verify` are hand-split.
- `CAPABILITY_CHANNEL` — every dependency is a Tag satisfied by the Layer graph; no module-level live instance or ambient capability. BEARS: `crypto.randomUUID()`/`crypto.getRandomValues` are ambient global reads in domain flow (session/apikey/webauthn/Shredder); `WebAuthnTrust` mutates process-global `SettingsService`/`MetadataService` singletons.

SHAPE
- `SCHEMA_AUTHORITY` — the Schema owner is the single shape authority; a hand-declared interface/alias/DTO beside an owner is the defect. BEARS: `verify#VERIFY_FOLD` `PublicKey` is a bare `type` discriminated union with a string `scheme` field where a tagged family belongs; `Ring`/`Keyset`/`Ceremony`/`Creds`/`Principal`/`FramedCookie` are loose `type` products.
- `SHAPE_BUDGET` — one deep owner per concept at ~1/5 naive shape count; three-plus parallel shapes or repeated dispatch arms is the collapse trigger. BEARS: nine fault classes share one shape but each re-declares the same `_reasons`+`_faults`+guard triple; `PublicKey` union + `_verifyAsym` if-ladder restate a case family.
- `DEEP_SURFACES` — one or two rich exports per module, `_`-prefixed interior, one terminal `[EXPORTS]` block. BEARS: mostly honored (single `[EXPORTS]` tails, `_`-interiors); `Crypto` exposes 11 members incl. three constant-time compare surfaces (`compare`/`matches`/`same`).
- `MODAL_ARITY` — one entrypoint owns every call modality; discriminate on input value, never name suffixes. BEARS: `Jwt.verify`/`Jwt.verifyExternal` is a suffix twin (local vs remote-issuer) that one overloaded `verify` discriminating on an issuer descriptor collapses.
- `ANTICIPATORY_COLLAPSE` — the owner is shaped for the family it absorbs; the next feature is a row, not a new surface. BEARS: fault/`KeyAlg`/`Role` tables carry only the forward guard, so a dead-row drift is unspellably prevented in one direction only; `Role` closure is recomputed per request rather than derived once.

DERIVATION
- `VOCABULARY_VALUES` — one contract-checked `as const` table is the single source of a vocabulary; every secondary surface derives; a `Match`/`switch` over a keyed domain is the rejected dispatch. BEARS: strong overall (`_faults`, `_dialects`, `_rows`, `RolePermission`, `_codeReason`); the converse guard the `verify` dialect table carries (`_Kinds`) is absent on every fault/`KeyAlg`/`Role` table.
- `DERIVED_TYPES` — types compute from one anchor; a runtime literal restating a type correspondence is rejected. BEARS: honored (`keyof typeof`, indexed access throughout).
- `SEMANTIC_NAMING` — canonical term, role, tense; no suffix families. BEARS: `verify`/`verifyExternal`, `compare`/`matches`/`same` brush the suffix-family line.

MATERIAL
- `LIBRARY_DEPTH` — the Effect ecosystem is the stdlib; use the deepest primitive the package reaches; stdlib survives only at the FFI seam. BEARS: no `Effect.fn` seam, no `Metric`/`withSpan`/`Schedule`/`Semaphore`/breaker anywhere — the resilience and observability substrate the doctrine legislates is entirely unexploited on crypto/auth/egress surfaces.
- `INLINE_COMPOSITION` — wrapping/telemetry/resilience attach at the owner declaration seam as policy values. BEARS: zero aspect seams; resilience and telemetry that should ride each service's `Effect.fn` pipeline are absent, so no consumer hook exists (campaign laws #3/#6).

INTEGRATION
- `ROOT_REBUILD` — new capability woven into the owner as if always there; no shims. BEARS: honored (in-folder composition through `crypt/sign`, ports).
- `ONE_HOP_RESOLUTION` — a name resolves to its semantics in one hop. BEARS: honored.
- `COMPOSED_IMPLEMENTATION` — features compose page owners before any new scaffold; a machine/statechart is a data-carried owner, never a switch ladder. BEARS: the OAuth authorization-code, WebAuthn challenge-response, and session-rotation ceremonies are imperative `Effect.gen` flows with string-keyed external stashes — not data-carried machines / typestate, the single deepest structural gap.

## [1]-[crypt/sign.md] — the crypto authority

- `sign.md:40-95` `SignFault` **[MEDIUM, derivation]** — `_reasons` tuple and `_faults` record are two anchors for one vocabulary, guarded only forward. `_Rows` proves `_faults` row shape; `Schema.Literal(..._reasons)` + `get class(){ return _faults[this.reason].class }` proves `_reasons ⊆ keyof _faults` (the getter fails to compile on an extra reason). But NO guard proves `keyof _faults ⊆ _reasons`, so a `_faults` key absent from `_reasons` is an undetected dead row. `verify.md`'s dialect table already carries the bidirectional pair (`_Keys`+`_Kinds`); the fault tables must mirror it. FIX: add the converse guard `type _Reasons<R extends (typeof _reasons)[number] = Reason> = R` to every fault namespace (SignFault, VerifyFault, SecretFault, SessionFault, CsrfFault, CredentialFault, OAuthFault, WebAuthnFault, ClaimFault, PolicyFault). RECURS across all nine pages — this is one cross-cutting fix.
- `sign.md:64-82` `KeyAlg` **[MEDIUM, derivation]** — same gap, functionally riskier: `_Keys<K extends Kind = (typeof _algs)[number]>` proves `_algs ⊆ keyof KeyAlg` only. `Schema.Literal(..._algs)` (in `Material.ring`) is the runtime gate; a `KeyAlg` row missing from `_algs` silently makes a valid scheme undecodable — a live key refused, not a compile error. FIX: add `type _Algs<K extends (typeof _algs)[number] = Kind> = K`.
- `sign.md:127-138` `_imported` **[MEDIUM, boundaries]** — `importJWK(JSON.parse(pem) as JWK, alg)` casts untyped JSON to `JWK`; `_admit` (`sign.md:152`) re-parses the SAME pem again for `Predicate.hasProperty(JSON.parse(pem), "d")`. Two `JSON.parse` of one string plus an `as` cast where `BOUNDARY_ADMISSION` mandates one decode producing evidence. FIX: one `Schema.parseJson(JwkShape)` decode at the seam yielding a typed JWK; the private-key discriminant (`"d"` presence) becomes a field of the decoded owner, not a re-parse.
- `sign.md:188-198` `Material.ring` verify build **[MEDIUM, rails-accumulate/resilience]** — `Effect.forEach(published.keys, ...)` is first-failure abort: one malformed published JWK collapses the entire verify set, so a rotated-in bad key breaks verification for every good key. `rails-and-effects.md` ACCUMULATION [04] `Effect.partition` keeps good keys and quarantines the bad; a JWKS is exactly the partition case. FIX: `Effect.partition` over `published.keys`, admit the satisfying half, route the excluded to a diagnostic tap.
- `sign.md:188-195` `Material.ring` **[LOW-MED, parameterization]** — a 10-year verify window is a hardcoded `Duration.days(3650)` buried in the fold, and the synthetic carrier is `kind: "signing"` for a Verify key (admitted only via `filterOrFail` to `Verify`). `zero hardcoded values`: the horizon is a config/policy value; the carrier kind should read `verify`.
- `sign.md:239-282` `Crypto` **[HIGH, resilience — campaign #6]** — `digest`/`verify`/`derive` run argon2id unbounded-concurrent (login=19MB, apiKey=12MB, kek=64MB memoryCost). A login storm spawns unbounded 19–64MB hashes with no bulkhead → OOM. `concurrency.md` [16] `Effect.makeSemaphore`+`withPermits` (or `withPermitsIfAvailable` for load-shed) is the cross-site bound the campaign's resilience law demands on a memory-bound surface. FIX: construct one `Semaphore` at `Crypto` build, bracket every argon2 call under a config-sized permit budget; the `kek` derive takes `withPermits(k)` at a higher weight.
- `sign.md:239-282` `Crypto` **[LOW, deep-surfaces]** — three constant-time compare surfaces (`compare` HMAC-over-body, `matches` SHA-fingerprint, `same` raw-string). Distinct inputs justify distinct entries, but they are candidates for one polymorphic `verified(input)` discriminating on the compared shape.
- `sign.md:268-271` `matches`/`same`/`fingerprint` **[LOW, boundaries]** — pure sync helpers call `decodeHex` (throwing) and `sha256` outside any `Effect.try`; a structurally malformed stored hash throws an uncaught defect into `gen` domain flow. Inputs are folder-controlled, but the throw path is unguarded where the sibling `compare` wraps `Effect.try`.
- `sign.md:416-484` `Jwt.verify` / `Jwt.verifyExternal` **[MEDIUM, MODAL_ARITY]** — a `verify`/`verifyExternal` suffix twin: one concept (verify a JWT) with two call shapes (own local JWKS vs a remote per-issuer descriptor) and two returns (`AccessClaims` vs raw `JWTPayload`). `surfaces-and-dispatch.md` OVERLOAD_COLLAPSE: one overloaded `verify(token)` / `verify(token, issuer)` with a shape-following return. FIX: collapse to one entrypoint discriminating on the presence of the issuer descriptor.
- `sign.md:446-462` `Jwt.verifyExternal` **[MEDIUM, resilience]** — the remote `jwtVerify` rides `Effect.tryPromise` with jose's internal cooldown/timeout only; no Effect-level `Effect.timeout`, transient-gated `Effect.retry`, or circuit breaker despite `jwks` being a retryable-classed fault. Campaign #6: egress rides resilience internally. FIX: wrap the remote verify in `Effect.fn("jwt.verifyExternal")(body, Effect.timeoutFail(...), Effect.retry({schedule, while: transient}))`.
- `sign.md` folder-wide **[MED-HIGH, observability]** — no `Effect.fn` span seam, no `Metric.frequency` over the reason vocabulary, no `annotateLogs` on any crypto surface. Token mint/verify latency and verify-failure-by-reason are the canonical auth signals; the fault families already expose bounded `reason`/`class` axes perfect for `Metric.frequency`. `surfaces-and-dispatch.md` ASPECT_SEAM + `rails-and-effects.md` OBSERVABILITY are entirely unused. FIX: every service method becomes `Effect.fn("<span>")(body, ...pipeline)` carrying span, reason-metric, and resilience.

## [10]-[FOLDER VERDICT — DEEPEST WEAKNESSES BY LEVERAGE]

1. **Ceremonies as imperative gen, not machines (campaign #4).** OAuth authorization-code, WebAuthn challenge-response, and session rotation are the three richest protocol/lifecycle flows in the folder, and all three are `Effect.gen` with string-keyed external stashes and manual position checks. The campaign's machine law and `derivation.md` typestate both point here: data-carried statecharts (Data.taggedEnum transition families; the `@effect/experimental` `Machine` with snapshot/restore for the durable OAuth redirect flow), making illegal transitions unspellable and protocol position type-witnessed. Highest structural leverage — it reshapes three pages.

2. **Resilience substrate entirely unexploited (campaign #6).** No bulkhead on unbounded-concurrent argon2 (19–64MB hashes → OOM under a login storm), no Effect-level retry/timeout/breaker on any remote egress (`Jwt.verifyExternal`, `Secret`/Doppler — whose retry the prose claims but the fence omits, OAuth arctic legs), no `Schedule` policy value anywhere. The doctrine legislates `Semaphore`, `Schedule`, `Effect.timeout`, `ExecutionPlan` for exactly these surfaces. Every store/backend/egress owner must ride resilience internally at its `Effect.fn` seam.

3. **Observability substrate absent folder-wide (`ASPECT_SEAM`/`OBSERVABILITY`, campaign #3).** Zero `Effect.fn` spans, zero `Metric`, zero `annotateLogs` — on auth surfaces whose fault families already carry bounded `reason`/`class` axes ideal for `Metric.frequency`. Because there is no aspect seam, there is also no place for the resilience pipeline of #2 or the per-tenant/per-consumer telemetry hooks the campaign wants. Converting every service method to `Effect.fn("<span>")(body, ...pipeline)` is the single move that unlocks #2 and #3 together.

4. **Shape/derivation rigor inconsistent with the folder's own exemplar.** Nine fault tables plus `KeyAlg` and `Role` carry only the forward vocabulary guard, while `verify.md`'s dialect table carries the bidirectional pair; a `verify`/`verifyExternal` suffix twin and a `PublicKey` string-discriminant union + `_verifyAsym` if-ladder sit beside otherwise-clean vocabulary tables. Uniform bidirectional guards, one collapsed `verify` overload, and a tagged `PublicKey` family close the gap.

5. **Boundary and capability leaks.** `JSON.parse(pem) as JWK` (twice) in `sign._admit` violates decode-once; `crypto.randomUUID()`/`crypto.getRandomValues` are ambient global reads in domain flow (session/apikey/webauthn/Shredder) that bypass the folder's own entropy port and defeat deterministic testing; `WebAuthnTrust` mutates process-global attestation state that breaks per-tenant soundness.
