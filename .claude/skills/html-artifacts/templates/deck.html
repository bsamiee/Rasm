Collapsing the Shape Surface

One polymorphic owner replaces five parallel exports.

Set the frame: this is a density decision, not a rewrite. The audience is the reviewers who own the call sites.

Context

The Shape concept ships five entry points — get_primary, get_many, list, find, search — each re-deriving the same routing and boundary mapping.

  • Every new modality has added a sibling export, not a row.
  • Boundary translation is duplicated across three of the five.
  • Callers pick an entry by name, so arity leaks into the call site.
Anchor the cost in the reviewers' own diffs — the last two modalities each touched all five sites.

Evidence

Arity, filter, and provider move into the Request shape; the owner dispatches on a closed vocabulary.

The −214 is surface, not capability — every modality still reachable, just through one door. Have the branch open if asked.

Decision

[DECIDED] Merge the five exports into Shape.dispatch(req), discriminating on req.arity against a bounded vocabulary.

[BOUNDARY] PRIMARY maps once at ingress; internals read canonical names only.

Name what is explicitly out of scope: the wire format is unchanged, so no downstream service moves this quarter.

Asks

  • Approve the single-entry dispatch shape by Friday.
  • One reviewer to own the call-site migration audit.
  • Confirm the arity vocabulary is closed — no runtime-supplied kinds.
Close on the Friday date and the one named owner — leave with a commitment, not a discussion.
