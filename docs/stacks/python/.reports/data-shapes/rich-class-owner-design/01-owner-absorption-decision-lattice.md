# Promotion Gate And Behavior-Density Quorum

- Rich class owner requires ≥1 durable algorithm family beyond passive field storage: validated transition, context-bound projection, service fabrication, vocabulary homomorphism, or fallible parse rail returning `Self | Fault` / `Result[Self, E]`.
- Fields-only records without algorithm families absorb into parent owner fields, msgspec catalog rows, or inline frozen tuple collapse — method absence is the primary demotion signal, not LOC.
- Single-scalar coercion absorbs into `Annotated[..., BeforeValidator]` or pydantic `field_validator(mode="before")` on the enclosing owner — never a sibling class for one-axis ingress law.
- Cross-field storage invariants provable from stored fields alone promote pydantic `model_validator(mode="after")` on the settings owner — external guard functions are absorption defects.
- Concept-density trigger for collapse before promotion: ≥3 parallel types, sibling factories, repeated `match` arms, or single-caller private helpers for the same concept — fold into one polymorphic owner first, then decide base family.
- Open extension axes (plugins, unbounded backends) absorb into `singledispatch` or registry bind rows at the composition root — sealed families promote nominal `@disjoint_base` variants, not string routing tables.
- Transition methods that can fail promote `Result[Self, E]` or `Self | Fault` on the same owner — bare `raise` at decorator parse boundaries is anti-promotion unless pydantic construction law owns the rail as `ValidationError`.
- Assurance-promotion ordering: behavior-density quorum decides artifact class only after concept collapse — owner proof row binding replace kernel (Tier S shallow replace | Tier D child isolation | Tier V full `model_validate` replay) precedes steady-state certification; algorithm family without proof-row minima is absorption defect, not deferred promotion debt.
- Post-promotion certify runs only after promotion unit commits — pre-merge gates witness triage outcome; boot and handoff gates run before interior enrichment consumes materialized owners; field snapshot green without replace-kernel negative controls does not discharge Tier V promotion.

# Ordered Absorption Sequence

- Step 1 — test behavior density: absent durable algorithm family → demote immediately; do not proceed to base-family triage on fields-only shapes.
- Step 2 — run concept collapse: ≥3 parallel shapes for one concept → merge to one polymorphic owner before choosing msgspec, pydantic, or dataclass base.
- Step 3 — bind base family from admitted packages using ingress, wire, CLI, and transition duties — never infer family from LOC or import convenience.
- Step 4 — assign projection tier on pydantic owners: field-closed method, interior `@property`, context-bound method, or `@computed_field` export — tier mismatch absorbs into lower tier.
- Step 5 — publish owner proof row with replace kernel, validator replay expectation, and enrichment epoch gates — promotion without row binding absorbs even when methods exist.
- Step 6 — run post-promotion proof bundle on the row before marking owner ready for registry bind or worker handoff — necessary bundle blocks merge; sufficient bundle discharges rich-owner quorum.

# Absorption Targets When Rich Ownership Fails

- Closed vocabulary with wire token, payload slots, alias law, and scalar factory (`from_returncode`) → payload-carrying `StrEnum`; absorb into plain `Literal` field when no algebra, iteration, or factory homomorphism is load-bearing.
- Closed in-process fault/result envelope without settings egress, computed export, or env sources → `expression.@tagged_union(frozen=True)`; absorb into pydantic only when an arm needs `computed_field`, `settings_customise_sources`, or Tier V `model_validate` transitions.
- Capability surface with no stored domain axes → `@runtime_checkable` `Protocol` slice + `TypeIs` predicate; algorithm bodies stay on the materializing production owner — protocols do not accumulate method families.
- Registry/catalog wire row (`Tool`, `Check`, `Bind`) with typed enum axes and param subclass reference → msgspec struct row; handler `match` on routed param products owns dispatch — row methods allowed only for pure projection from stored fields.
- Ephemeral fabrication opts validated once at service constructor → nested frozen sub-owner parameter on fabrication method; absorb sub-owner entirely when backend admits no schema-worthy opts.
- Opaque scalar with one construction gate → `NewType` + parent validator; not a rich class.
- Enrichment delta keys without new stored shape → PEP 728 `TypedDict` with `closed=True`, `NotRequired[...]`, and `Unpack[ConfigurationDelta]` on transition signature — not `{Concept}Patch` siblings.
- Collection transforms over materialized members → `expression.Block`/`Map` fold on the owner type — not a wrapper class per aggregation step.
- Interior-only subprocess/env folds with no schema or settings export duty → stdlib `@property` on pydantic owner absorbs — `@computed_field` promotion requires export-tier authorization on the owner proof row.
- Context bag parameters (`dict[str, object]`, bare `Path`, untyped env maps) absorb into typed settings owner or structural port slice with `TypeIs` gate — never promote a `{Context}Carrier` class.
- Worker or subinterpreter context parameters absorb into encoded settings snapshot plus local fabrication at worker entry — never promote `{Worker}Scope`, `{Process}Handle`, or port-typed carrier classes across interpreter boundaries.

# Base-Family Triage Under Admitted Packages

- Cyclopts CLI/param binding without load-bearing env validation → `@dataclass(frozen=True, slots=True, kw_only=True)`; cyclopts reads dataclass field metadata only — param promotion to pydantic requires explicit seam `model_validate`, never implicit tier collapse.
- Ingress, settings, env parsing, alias routing, computed egress, nested env partial merge at construction, or Tier V enrichment transitions → pydantic `BaseModel`/`BaseSettings` with `frozen=True`, `extra="forbid"`; settings own configuration transitions via full snapshot `model_validate`.
- High-volume wire/decode/report/evidence and hash-keyed catalog rows → msgspec `Struct(frozen=True)` with `forbid_unknown_fields`, `omit_defaults`, optional `gc=False` and `cache_hash=True` when acyclic and key-duty hot; relocate behavior off struct when operations need context parameters or validator replay.
- attrs/cattrs/namedtuple are not admitted shape engines — stdlib `@dataclass` behind `@dataclass_transform(frozen_default=True, slots_default=True, kw_only_default=True)` factory is the sole lightweight record admission path.
- msgspec forbids overriding `__init__`/`__new__` — alternate constructors are `classmethod` factories; construction law on structs stays in `__post_init__`, `Annotated[..., msgspec.Meta(...)]`, or pydantic ingress seam — not parallel init subclasses.
- pydantic owns validator replay, computed export, and settings source ordering — msgspec owns hot encode/decode and catalog key stability — dataclass owns cyclopts param capsules and trusted `copy.replace` without re-validation; mixing duties on one base absorbs into explicit seam conversion, not dual inheritance.
- expression `Result`/`Option`/`pipe`/`@effect.result` own fallible composition across owners — rail interiors do not mirror the same variant set as a pydantic hierarchy in one bounded context.
- Trusted same-type pydantic swap → `copy.replace` / `model_copy(update=...)`; untrusted, computed, or nested-axis deltas → full snapshot `model_validate` — base-family triage binds replace kernel in owner proof row (Tier S | Tier D | Tier V), not ad hoc per call site.
- Partial nested `model_copy(update={"nested": {...}})` never owns typed nested transitions — explicit child snapshot assembly + `model_validate` promotes to parent transition method.

# Nominal Class Versus Tagged Generic Versus Enum Versus Service

- Distinct runtime classes with disjoint method sets per variant → `@disjoint_base` nested `_Seal` under outer owner; field-only discriminant differences absorb into `class Shape[T: Tag]` PEP 695 generic with one runtime class.
- Vocabulary semigroup/homomorphism (`join`, `fold`, `from_returncode`) → enum module or enum `@classmethod` when `cls` polymorphism adds nothing; instance methods on unrelated owners (`Report.rail_fold`) are seam defects.
- Service fabrication constructing third-party products (`ArtifactStore`, port-typed filesystem) → separate service owner with port-typed dependencies; settings owner owns `store(...)` fabrication orchestration, not vendor logic inside pydantic validators.
- Param-owner `bound(verb) -> Self | Fault` with protected `_arity` hook → one polymorphic surface across param subclasses; static fault constructors stay on base param owner — subclasses override hook only, never parallel `bound` implementations.
- Payload enum cases are owners for wire/CLI/routing metadata — bare string tables outside the enum duplicate ownership and break `match` exhaustiveness.
- Wire tagged msgspec unions decode first, then `match` on domain owner — promote parallel wire/domain class trees only when wire shape is policy-canonical; otherwise domain owner plus egress projection absorbs wire siblings.
- Pydantic discriminated unions own ingress when foreign schema is hierarchical and validation-heavy; in-process `@tagged_union` absorbs when no computed export or settings source attaches — pick one union owner per bounded context.

# Python 3.15 Layout, Disjointness, And Introspection Gates

- PEP 800: classes with non-empty `__slots__` (including `@dataclass(slots=True)` and pydantic `@dataclass_transform` synthesized layouts) are disjoint bases at type-check time — explicit `@disjoint_base` on nested `_Seal` still required when variant siblings share a non-slot mixin base, when exhaustiveness must be nominal beyond slot inference, or when stubtest parity cannot rely on heuristic slot inference alone.
- Rich owners cannot multiply-inherit unrelated disjoint bases — nested frozen sub-owners compose; dual msgspec/pydantic inheritance absorbs into explicit seam conversion, not promotion.
- Python 3.15 allows `__dict__` and `__weakref__` entries in `__slots__` on any class — pydantic `@cached_property` on frozen owners may use slotted `__dict__` only when owner proof row documents cache tier; default absorb into `@computed_field` without cache or snapshot `model_validate` recompute.
- `annotationlib.get_annotations(owner, format=Format.VALUE)` is mandatory for metadata walkers on runtime-evaluated bases — direct `__annotations__` fails cyclopts, pydantic schema compile, and beartype field admission under PEP 649/749 deferred evaluation.
- `@runtime_checkable` must be re-declared on every `Protocol` subclass used in `isinstance` — inherited checkability without re-decoration deprecates in 3.15, raises `TypeError` in 3.20; port slices ship re-decoration in owner proof rows and 3.20-readiness fixtures.
- `@enum.verify(EnumCheck.UNIQUE | EnumCheck.CONTINUOUS | ...)` (≥3.15) owns vocabulary law at class body — post-hoc alias repair absorbs into `_add_value_alias_` at member construction, not free functions.
- Free-threaded builds (PEP 779): post-materialization frozen instance reads need no locks — class-scoped memo tables promote only with import-published `frozendict`/`frozenset`, `ContextVar` lane, or explicit lock; absorb mutable class attributes into composition-root publish.
- `inspect.getmembers_static(owner)` owns catalog walks — live `getmembers` on settings owners executes `@computed_field`/`@property` and can IO; promote static twin before any new introspection-driven operation on rich owners.

# Projection Tier Triage On One Owner

- Stored fields only → field-closed methods or enum/variant `match` arms — no ambient parameters.
- Stored fields + must appear in schema/settings export → `@computed_field` stacked above `@property` when fold uses stored axes only.
- Stored fields + subprocess/rail interior only → stdlib `@property` absorbs — exporting widens settings schema without policy review.
- Stored fields + anchored paths/sibling settings axes → context-bound method with typed settings or port slice parameter — never `@computed_field` when normalization executes IO or depends on parent-only siblings at construction time.
- Parameters beyond `self` on folds → method family, not computed field — fabrication and path algebra promote here.
- `exclude_computed_fields` is egress knob only — Tier V transitions must not assume dumps omit computed evidence validators re-validate against.

# Proof-Spine Non-Sufficiency And Field-Only Demotion

- Field snapshot equality alone does not prove equivalent enrichment law when replace kernel, validator replay tier, morphism identity, or schema-hash slice drifted — audit replay keyed on dumps without hop spine witnesses absorbs into negative-control territory, not promotion evidence.
- Tier V promotion requires predecessor `is not` successor at each validated transition link — shallow `model_copy` green on final fields while skipping validator replay demotes to Tier S absorption target on the same method surface.
- Owner proof row must name replace kernel and validator replay expectation per transition method — methods sharing a concept but diverging on tier binding absorb into one polymorphic entry with explicit tier column, not parallel classes per kernel.
- Alias-only vocabulary edits must leave enrichment hop spine unchanged except documented hash slices — routing-stable alias perturbation with spine drift signals promotion unit incompleteness, not a new rich owner.
- Proving metamorphic chains from field equality without transition-method replay list is insufficient when computed exports, nested child snapshots, or morphism arms participate — collapse proof tables to name methods invoked, not only terminal field tuples.

# Fallible Envelope And Morphism Absorption

- Param-owner parse faults and rail `@tagged_union` faults are parallel families — promote one canonical fault product per boundary; morphism at composition root absorbs cross-family re-wraps, not per-handler string adapters.
- `TypeIs` on morphism output promotes before rail `match` — bool guards and `cast` on enrichment seams absorb into policy violation.
- Settings construction defects stay `ValidationError` via pydantic validators — runtime projection preconditions (`local_root` for subprocess) stay `@property` raise on materialized owner; unifying into `Result` on construction absorbs when pydantic owns construction rail.
- Registry `_bound` handlers `match` on `Self | Fault` without re-wrapping owner-emitted faults — param owner does not catch its own `bound` faults.
- In-process enrichment lifecycle encodes as phantom `Record[Stage: Literal["canonical", "enriched"]]` on the owner generic — reject `{Concept}Draft` siblings unless wire persistence stores a materially different pre-canonical shape than enriched fields.
- Wire/CLI/cyclopts edges accept canonical field snapshots or adapter-projected structs only — method products (`ArtifactStore`, port handles) are seam-local; stage parameter binding on owner generic makes stage mismatch a static defect.
- Composition root materializes settings once — env sources and `settings_customise_sources` do not re-run inside `with_configuration`, `store`, or nested override transitions; env re-read inside enrichment absorbs into composition-root defect, not a new owner class.
- `ContextVar` tokens absorb parallel-task override lanes — not a substitute for validated `model_validate` transitions on the settings owner.
- Enrichment return values rebind local names — identity keyed on `id(settings)` absorbs into value equality or explicit snapshot keys documented in owner proof row.

# Worker And Subinterpreter Closure Absorption

- Multiprocessing and `concurrent.interpreters` workers absorb port-typed fields, live service handles, and parent `ContextVar` tokens — transmit encoded settings snapshot bytes plus schema-hash slice only; worker entry rebuilds stores through fabrication methods on received snapshot.
- Pickle of port-typed or service-typed fields on frozen owners is negative fixture — absorption target is snapshot encode/decode row on owner proof table, not a `{Worker}Adapter` rich class carrying handles.
- Parent enrichment after worker spawn does not retroactively upgrade in-flight worker snapshot without explicit reload protocol documented on owner proof row — absorb `{Reload}Coordinator` classes into composition-root handoff policy.
- Subinterpreter and multiprocessing share handoff schema — `worker_kind` literal records process model; forbidden-field attestation belongs in handoff proof row, not a parallel worker owner hierarchy.
- CPU-bound folds on owner-derived evidence may run in isolated interpreters — shared class-level memo tables on rich owners race unless import-published immutable or lock-guarded; absorb process-wide mutable memo into composition-root publish, not worker-local rich owner.

# Decorator And Runtime Admission In Triage

- Class-body validators and computed fields are the default behavior admission path on pydantic owners — promoting standalone pre-construction functions absorbs when `field_validator`/`model_validator` can own the law.
- `@beartype` on exported transition and fabrication entrypoints promotes when static proof ends at the seam — decorating field bags or interior private kernels absorbs as noise; stack obeys `trace > authorize > validate > cache > govern > retry > operation` outward-in.
- `TYPE_CHECKING`-only field imports on runtime-evaluated bases reject promotion to rich owner until unconditional imports restore — shape owner cannot be rich if decorators cannot see real field types at class body.
- Cyclopts `@Parameter` + frozen dataclass promotes param rich owner; adding pydantic to the same param promotes only when env/settings validation is load-bearing and seam `model_validate` is explicit in owner proof row.

# Mutable Payload And Nested Reference Triage

- Parent `frozen=True` with nested mutable `dict`/`list` fields promotes child-reference swap via `copy.replace` or child `model_validate` snapshot — in-place nested mutation absorbs into doctrine violation, not a mutable-wrapper owner.
- `frozendict`/`frozenset` policy tables on settings promote when allowlists participate in equality or enrichment transitions — plain `dict` fields demote to ingress-only projection targets until promoted to immutable publish at materialization.
- msgspec `cache_hash=True` rows promote as registry keys only when equality axes are stable post-`structs.replace` — stringly catalog fields demote to passive wire evidence without key duty.
