# Absence and Sentinels ā€” Boundary Law

[SENTINEL_PROJECTION]:
- Every host sentinel ā€” null, `IntPtr.Zero`, `nint(-1)`, `DateTime.MinValue`, `Guid.Empty`, a `0`-handle, a dedicated "unset" constant, or a magic fallback value ā€” is a foreign-language absence signal. Project it to `Option<T>.None` at the exact foreign-call boundary using `Prelude.Optional(nullableRef)` for reference types, `Prelude.Optional(nullable)` for `Nullable<T>` via `HasValue`, or an explicit discriminant expression (`h.IsInvalid ? None : Some(h)`) for handle types. Both `Optional` overloads are zero-allocation inline conversions. The explicit call is preferred over the implicit cast because it names the projection site.
- `Optional` returns `default(Option<A>)` ā€” the `Option<A>.None` static field ā€” for null and for `Nullable<T>` where `!HasValue`. The implicit `operator Option<A>(A? a)` invokes `Prelude.Optional` directly; the two are byte-identical.
- When a foreign API returns a negative integer sentinel (e.g., a dedicated "not found" constant such as `-1`), the projection must be explicit: `idx >= 0 ? Some(idx) : None`. The implicit operator fires only for reference types and `Nullable<T>`; it does not intercept value types with magic constants. Missing this produces `Some(-1)` ā€” a sentinel propagating inward wearing a typed wrapper.
- A sentinel value must not enter a persisted shape. `Option<A>` implements `ISerializable`; its serialization constructor reads `IsSome` and conditionally `Value`, so a field of type `Option<T>` round-trips only under legacy `ISerializable`-aware formatters ā€” a surface current .NET no longer ships; the interface is inert in practice. For foreign wire protocols, project `Option<T>` to a nullable wire field at the serialization boundary and project back on deserialization; `System.Text.Json` without a custom converter does not understand `ISerializable` and will produce a broken round-trip.

[SAFEHANDLE_PROJECTION]:
- `SafeHandle.IsInvalid` and `SafeHandle.IsClosed` are the dual-sentinel surface for native handles. Project to `Option<T>` immediately after construction: `handle.IsInvalid || handle.IsClosed ? None : Some(handle)`. Once inside the domain, the `Option` carries lifetime; the raw handle is never stored.
- `SafeHandle.SetHandleAsInvalid()` sets the `IsClosed` bit via `Interlocked.Or` and suppresses the finalizer without calling `ReleaseHandle` ā€” it marks the handle as closed-but-not-released when the OS has already reclaimed the resource. Because `IsClosed` is a BCL field check (always accurate) while `IsInvalid` is abstract and may inspect only the handle value without reading the closed bit, both flags must be tested together. After `SetHandleAsInvalid`, `DangerousAddRef` throws `ObjectDisposedException` immediately; there is no safe retry path once either flag is set.
- `DangerousAddRef(ref bool success)` never sets `success = false` in the non-exceptional path; it throws `ObjectDisposedException` when the closed bit is set, and reaches `success = true` only after a successful CAS loop. The canonical caller pattern initializes `bool success = false` before the call so that a thrown exception leaves `success` unset, enabling a `finally { if (success) handle.DangerousRelease(); }` guard. Treating `success == false` post-call as a silent absence signal is wrong ā€” in the throwing path, the `finally` block is the gate. Projecting the handle to `Option<T>` before any ref-counted access eliminates the hazard entirely; the atomic reference-count increment means nested `DangerousAddRef`/`DangerousRelease` pairs on the same handle are safe so long as each pair is balanced, but the canonical projection removes the need to reason about nesting at all.

[TYPED_ABSENCE_CARRIERS]:
- `Option<T>` is the sole carrier for presence/absence at domain boundaries. It is a `readonly struct`, so the `None` case allocates nothing; `default(Option<T>)` is `None`. Pattern matching uses `{ IsSome: true, Case: T value }` property patterns to avoid the `Match`-mid-pipeline prohibition.
- `Option<T>.ToFin(Error fail)` converts absence to a typed failure for use on the error rail. `Option<T>.ToFin()` with no argument uses `Errors.None` ā€” a `ManyErrors` with code `-2000000006` and an empty error sequence. `HasCode` on `ManyErrors` delegates to `Errors.Exists(e => e.Code == code)`; an empty sequence always returns `false`, making it impossible to discriminate at recovery sites. Always supply a domain-specific `Error` with a non-zero code when the absence has a named cause.
- `Option<T>.ToEff(Error fail)` lifts absence into an `Eff<T>` computation; the zero-arg overload likewise uses `Errors.None` and loses diagnostic context. `Option<T>.ToIO()` promotes to `OptionT<IO, A>` ā€” use `OptionT.lift<M, A>(Option<A>)` when the target outer monad is not `IO`.
- `HashMap<K, V>.Find(key)` returns `Option<V>`. All key-bounded lookups in domain structures must use `Find`, never a try-get-then-branch pattern. A missing key is structural absence; `FindOrAdd` is a state mutation and must not masquerade as a default.
- `Option<A>.Traverse<F, B>(Func<A, K<F, B>> f)` and `TraverseM` when the option is `None` return `F.Pure(Option<B>.None)` without calling `f`. `OptionT<M, A>.IfSome(Action<A> f)` runs the action inside `M` only when `Some` ā€” identical silent-skip behavior. A side-effecting audit at a boundary that must fire unconditionally cannot use either form; it must be sequenced as a separate operation outside the traversal or conditional. Use `Match(Some: x => effect(x), None: () => alternateEffect())` to ensure exactly one effect path fires.
- `Option<A>.Combine(Option<A> rhs)` returns `this` when `IsSome`; otherwise returns `rhs`. The `|` operator implements the same first-`Some` semigroup semantics. A chain `primary | secondary | tertiary` evaluates left-to-right and short-circuits on the first present value; this is the correct surface for chaining fallback absence sources at a seam boundary rather than nested `IfNone` calls. `Seq.Choose<A, B>(Func<A, Option<B>> selector)` retains only `Some` values, replacing any `Where`+`Select` pair that re-applies the predicate.
- `Prelude.guard<F>(bool flag)` is constrained to `where F : MonoidK<F>, Applicative<F>`. `Option` implements `Alternative<Option>` and `Choice<Option>` but not `MonoidK<Option>`, so `guard<Option>(condition)` does not compile. Absence filtering inside an `Option` query expression must use `Option<A>.Filter(pred)` or `Option<A>.Where(pred)` ā€” both return `None` when the predicate is false on a `Some` value. For typed-error absence guards over `Eff` or `Fin`, use `guard(flag, Error.New(code, message))` ā€” the error-bearing overload returns `Guard<Error, Unit>` and binds into `Eff` query expressions.
- `Option<A>.BiBind<B>(Func<A, Option<B>> Some, Func<Option<B>> None)` routes both present and absent cases to new `Option<B>` values, keeping the pipeline monadic without an intermediate `Match`-then-rewrap. `BiBind` does not call `Check.NullReturn` on either branch: a `Some(null)` return from the `Some` branch produces an `Option<B>` with `isSome = true` and a null interior ā€” a sentinel propagating inward wearing a typed wrapper ā€” while a null return from the `None` thunk produces `default(Option<B>)` silently presenting as `None` with no diagnostic. Neither branch may return null; the canonical defense is to return `None` explicitly when the target reference is absent.
- `Option<A>.BiMap<B>(Func<A, B> Some, Func<Unit, B> None)` calls `Check.NullReturn` on both branches; `Option<A>.BiMap<B>(Func<A, B> Some, Func<B> None)` does not ā€” a null-returning `None` factory silently produces `default(Option<B>)` presenting as `None`. `Fin<A>.BiMap<B>(Func<A, B> Succ, Func<Error, Error> Fail)` is an in-rail transform: the `Fail` branch re-codes the error but stays in `Fail` state and cannot escape to `B`. A seam that needs to collapse `Fin<A>` into a plain `B` with distinct success and failure materializations must use `Match(Succ: ..., Fail: ...)`, not `BiMap`. Prefer `BiBind` over `BiMap` on `Option` to remain on the rail and avoid null-safety asymmetry entirely.
- `OptionT<M>` implements `Fallible<Unit, OptionT<M>>`; its `Catch` predicate receives `Unit`, not `Error`. The `@catch` infrastructure with typed error codes ā€” `error.HasCode(...)`, `error.IsType<E>()` ā€” does not apply. A seam that needs typed recovery distinguishing timeout-absence from resource-unavailability-absence must use `FinT<M>`, which implements `Fallible<Error, FinT<M>>` with full typed predicates. Absence-to-failure conversion on `OptionT<M, A>` is performed by extracting the inner `Option<A>` via `.Run()`, calling `.ToFin(Error.New(code, msg))` on that `Option<A>`, then lifting into `FinT` via `FinT.lift<M, A>(fin)`; there is no `ToFin` surface on `OptionT<M, A>` itself.
- `FinExtensions.Partition<F, A>()` operates on `K<F, Fin<A>>` for any `F : Foldable<F>` and yields `(Seq<Error> Fails, Seq<A> Succs)` in a single fold without short-circuiting. A boundary that sweeps a collection of capability probes as `Seq<Fin<CapabilityResult>>` and calls `Partition()` extracts both typed failures and usable results simultaneously. For `Option`-typed probe results, `OptionExtensions.Somes<A>` drops absent values silently; when the `None` case warrants a log or metric, use `Choose` with a labeled `None` branch.
- `partitionFallible<F, M, A>(K<F, K<M, A>> fma)` sequences each inner `K<M, A>` within the foldable, catches failures from `Fallible<M>`, and accumulates `(Seq<Error> Fails, Seq<A> Succs)` inside the outer `K<M, ...>` without failing the outer monad on individual inner failures. This is the dense surface for batch absence projection where the outer rail must remain live even when some probes fail.
- The `|` operator on `Fin<A>` calls `Choose(rhs)` ā€” the lhs error is silently discarded when lhs is `Fail`. The `+` operator (semigroup combine) accumulates: when both are `Fail`, it returns `Fin.Fail(err1 + err2)` where `+` produces a `ManyErrors` containing both. A fallback chain expressed as `primary | secondary | terminal` discards all intermediate errors. Use `+` when a logging or health seam must inspect why each attempt failed; use `|` only when the lhs error is genuinely irrelevant to callers. `ManyErrors.Filter<E>()` returns a new `ManyErrors` (or `Errors.None`) containing only errors where `IsType<E>()` holds ā€” the correct surface for partial recovery: extract terminal typed errors from a combined failure, emit them to the health seam, and separately attempt recovery on the remaining transient errors. `ManyErrors.HasCode(int code)` delegates to `Errors.Exists(e => e.Code == code)` ā€” a one-level scan over direct `Code` values of the contained sequence; it does not recurse into nested `ManyErrors` elements. This scan holds for flat combinations produced by `Error.Combine` (the `+` operator flattens adjacent `ManyErrors` into a single sequence at combination time), so a terminal code is detectable inside a combined `ManyErrors` produced by `Fin` applicative or semigroup combination. A `ManyErrors` nested as a single element inside another `ManyErrors` ā€” possible via direct construction ā€” is not traversed; rely on `+`-combination flattening rather than assuming recursive detectability.

[CAPABILITY_ABSENCE_AS_TYPED_STATES]:
- An unavailable capability is not `Option.None`; it is a discriminated case in a closed capability family. `Option<T>` models a value that might be absent; a capability state is a richer fact ā€” unavailable, degraded, disabled, or transitioning ā€” each case carrying distinct fields. Collapsing all capability states to `Option.None` erases that evidence at the consumer and forces re-derivation.
- A degraded outcome must carry a usable result alongside first-class degradation evidence ā€” never a nullable companion field, never a swallowed exception, never a null result with a string message. The degradation evidence must be a typed value the caller can inspect, route, and log without downcasting.
- Capability-absent states with mandatory consumer handling are expressed as a `[Union]`-generated closed family with at least `Available`, `Degraded`, and `Unavailable` cases, each with its own payload. The total `Switch` dispatcher enforces that every consumer handles all modalities; there is no default fallthrough.
- A capability in `Unavailable` state must emit a definite `Fin.Fail(terminalError)` at every operation site, not an indeterminate `Option.None`. `Option.None` is structurally indistinguishable from "not yet known" to a downstream caller. `Fin.Fail` with a coded `Expected` error is definite: it names the condition and stops propagation.
- A capability registry keyed by a bounded vocabulary is modeled as `AtomHashMap<CapabilityKey, CapabilityState>` where `CapabilityKey` is a `[SmartEnum<K>]`-generated key and `CapabilityState` is a closed union. `SwapKey(key, prev => Some(next))` transitions a single capability atomically. The health fold must project every key in the vocabulary ā€” not just keys present in the map ā€” using `SmartEnumType.Items.Map(key => (key, registry.Find(key).IfNone(CapabilityState.Unknown)))`. Folding only over `registry.AsIterable()` silently omits unprobed keys from the health picture.

[ATOM_AND_OPTION_GATED_CELLS]:
- `atomIO<A>(value, validator)` wraps `Atom.New(value, validator)` in `IO.lift`; when the initial value fails the validator, it raises `Errors.ValidationFailed` (code `-2000000011`, `IsExpected = true`) as a failed `IO<Atom<A>>`. A recovery site that needs to distinguish construction failure from other errors should match against `error.HasCode(-2000000011)` or `error.Is(Errors.ValidationFailed)`. The non-IO `Prelude.Atom<A>(value, validator)` factory returns `Option<Atom<A>>.None` on failure without throwing; the two construction paths have different error surfaces and must not be treated interchangeably.
- `Atom<Option<T>>` is the standard cell for singleton token ownership: `None` represents unowned, `Some(token)` represents held. `SwapMaybe(current => current.IsNone ? Some(newToken) : None)` attempts a take; returning `None` from the swap function leaves the cell unchanged. `Atom<A>.SwapMaybe(Func<A, Option<A>> f)` returns type `A` ā€” the raw current value after the operation. When the swap succeeded it is the new value; when the guard rejected it is the unchanged prior value. These two cases are indistinguishable from the return value alone; there is no discriminated result type. For `Atom<Option<Token>>`, the caller must hold a reference to the prior value before the swap ā€” matching the returned value against the newly constructed token indicates success; matching the prior held token indicates rejection. There is no post-hoc way to determine which branch executed from the return value alone.
- `Atom<A>.Change` is a plain `delegate void AtomChangedEvent<in A>(A value)` event, not an observable or a disposable subscription. Deregistration requires `atom.Change -= handler` where `handler` is the exact same delegate instance used for `+=`. A new lambda at the `-=` site creates a delegate that compares unequal to the original, silently leaving the handler registered and firing indefinitely. The seam must capture the handler as a named field or local of type `AtomChangedEvent<A>` and store it for symmetric detachment.
- `AtomHashMap<K, V>.Change` fires `HashMapPatch<K, V>` events only when a listener is attached at mutation time; if no listener is attached, the internally tracked change log is never built. The `Change` event fires only when a listener is attached at the time of the CAS loop body, not at the time of the outer `SwapKey` call. A seam that derives capability state from key transitions must subscribe before any write; subscribing after a mutation never retroactively fires.
- `AtomHashMap<K, V>.SwapKey(K key, Func<Option<V>, Option<V>> swap)` atomically handles all four transitions: absentā†’absent (no-op), absentā†’present (add), presentā†’absent (remove), and presentā†’present (replace). This is the correct primitive for a capability registry where keys may materialize or disappear ā€” a single `SwapKey` call replaces any read-then-write pair.

[DISABLED_STATE_DEFINITENESS]:
- A host state that is absent must emit a definite `Disabled` or `Unavailable` signal to consumers ā€” never bind an `Option.None` into a UI or reactive pipeline and rely on downstream `IsNone` checks. `Option.None` at a subscription boundary creates an indeterminate optional that consumers must individually defend; a definite `Disabled` emission from the capability cell terminates propagation at the single seam where the foreign state is received.
- `Atom<CapabilityState>` with `CapabilityState` as the closed union owns this boundary cell. The initial value is `Unavailable`, and the first successful probe transitions it to `Available`. `SwapMaybe` is the only mutation API because it makes the failed-transition case observable.

[INITIAL_VALIDATION_STATE]:
- A screen or form must never transiently render in a valid state before user interaction. The domain model for a UI binding must carry an explicit `Unvalidated` state distinct from `Valid` and `Invalid`. A three-case union ā€” `Unvalidated`, `Valid(T value)`, `Invalid(Error reason)` ā€” owns this surface. The initial state is always `Unvalidated`. Binding `Option.None` as the initial value and treating absence as "not yet invalid" is the anti-pattern: any consumer that folds `None` to a success path will render falsely clean.
- `Validation<F, A>` requires `F : Monoid<F>` for applicative accumulation. A validation-state closed union that wraps `Validation<Error, T>` for the `Valid` and `Invalid` cases and adds `Unvalidated` as a separate case is the correct dense surface; do not embed `Validation` where the `Unvalidated` state cannot be expressed.
- `Validation<Error, A>.ToFin()` is an extension method constrained to `Validation<Error, A>`. A seam using a custom `Validation<TError, A>` where `TError` is not `Error` cannot use this extension; it must manually match `Success` and `Fail` cases. The canonical projection: match the `Fail` case through `MyError.ToError()` returning `Error.New(myError.Code, myError.Message)`, then call `ToFin()` on the converted `Validation<Error, A>`. This projection belongs inline at the seam, not inside the validation type itself, to keep the canonical `Error` rail clean.

[PROGRESS_AS_DISCRIMINATED_VALUE]:
- Declared-unknown progress is a discriminated value, not a nullable integer or a float in `[0, 1]`. A `[Union]`-generated progress family with cases `Unknown`, `Indeterminate`, and `Known(double fraction)` enforces the monotone non-decreasing law: any transition from `Known(f1)` to `Known(f2)` where `f2 < f1` is a violation, detectable at the swap site with `Atom<ProgressState>.SwapMaybe(prev => prev is Known(var f) && next < f ? None : Some(next))`.
- An `Unknown` ā†’ `Indeterminate` ā†’ `Known` progression must never regress to `Unknown` on a transient error. The boundary seam that feeds progress from a foreign source must detect any backward transition and clamp to `Indeterminate` or hold the prior `Known` value; it must not pass the foreign value through.

[TERMINAL_TYPED_RECEIPTS]:
- Native-library unavailability, store corruption, and schema-version rejection each require a terminal typed receipt ā€” a `sealed record` deriving `Error` with a domain-specific non-zero `int` code ā€” and no internal retry path. `Error.IsType<E>()` is the discriminant at recovery sites. `Error.Is(Error error)` performs code-equality for `Expected` (non-zero code) or message-equality (zero code); always use a non-zero code for terminal receipts to prevent accidental equality with string-matched errors.
- A terminal receipt must not silently re-enter the system. `Catch<F, A>(K<F, A> fa, Func<Error, bool> Predicate, Func<Error, K<F, A>> Fail)` routes recovery through a predicate; a predicate that matches the terminal code and returns a fallback is a hidden retry path. Terminal receipts are either propagated unchanged or surfaced as a `Fin`/`Eff` failure; they are never transformed into a default result.
- Schema-version rejection must produce a receipt carrying the stored version and the expected version as typed fields ā€” not a formatted string. A consumer that displays or logs the mismatch reads typed fields, never parses text.
- `Errors.SourceClosed` (code `-2000000013`, `IsExpected = true`) is the correct terminal receipt for a streaming or channel seam that has permanently exhausted its source, distinguishable by `error.HasCode(-2000000013)`. `Errors.Closed` (code `-2000000004`) covers general close states. Using these built-in singletons rather than custom domain codes avoids collision with the framework's own recovery machinery.
- `Errors.TimedOut` (code `-2000000002`) is the intended terminal receipt for a time-bounded absence detection window, but a `LocalCancelWithTimeout`-expired computation surfaces `Errors.Cancelled` (code `-2000000001`, `Is(TimedOut) = false`) because the underlying `TaskCanceledException` is mapped by `Error.New(exception)` via `Expected` code `-2000000001`. A recovery handler keyed exclusively to `error.HasCode(-2000000002)` will never fire for a `LocalCancelWithTimeout` expiry; the correct discriminant is `error.HasCode(-2000000001)` combined with context ā€” the absence-detection scope that owns the timeout is the only site that can classify this cancellation as transient-probe rather than session-abort. Both codes warrant `Degraded` rather than `Unavailable` transitions: they are evidence of transient absence, not confirmed unavailability.
- `retryUntil(schedule, ma, e => e.HasCode(terminalCode))` is the correct surface for re-probing a transiently absent capability: the predicate stops retry when the error is terminal and allows retry on transient codes. Each iteration wraps its attempt in `BracketFail()`, which calls `WithEnvFail((EnvIO env) => env.LocalResources)` creating an isolated resource scope per attempt so acquired-and-failed resources are released on that attempt's failure while outer cancellation still terminates the loop. The recursion is trampolined on the `IO` data structure, not the stack; the interpreter handles the chain iteratively and the stack does not grow per attempt. A terminal receipt must still not re-enter the system ā€” the predicate that matches a terminal code must propagate unchanged via `IO.fail(e)`, never route to a silent fallback.
- `Error.Inner` is `Option<Error>`. `Expected` records declare an `Inner` primary-constructor parameter of type `Option<Error>`; `Exceptional` derives its `Inner` by testing `Value?.InnerException != null` and lifting via `Error.New(Value.InnerException)`, otherwise `None`. A seam capturing a chained exception should construct `Error.New(code, message, inner: Error.New(innerException))` so that `Inner` carries the nested cause as a typed `Option<Error>`, not as a string field. Recovering callers traverse the chain via `error.Inner.Bind(i => i.Inner)` without parsing text.
- `IO.lift<A>(Func<Fin<A>> f)` internally calls `f().ThrowIfFail()`. `Fin<A>.ThrowIfFail()` calls `FailValue.Throw()` which dispatches to `Error.Throw<A>()`. On `Expected`, this throws `WrappedErrorExpectedException` carrying the code and message ā€” a lossless round-trip. On `Exceptional`, it re-throws the original exception if `Value` is non-null, or throws `ExceptionalException` if `Value` is null. A seam that receives `Fin<A>` from a deserialized or remoted source and lifts it via `IO.lift(fin)` must verify that exceptional errors carry live `Exception` objects before the lift if stack-trace preservation is a boundary requirement.

[MISSING_ASSET_KEY_FALLBACKS]:
- An unhandled lookup failure on a key-bounded vocabulary must not throw or return null. Use `HashMap<K, V>.Find(key)` returning `Option<V>`, followed by `IfNone(defaultValue)` or `IfNone(() => deriveDefault())` at the single point where a fallback is semantically correct. Placing the fallback deeper creates invisible defaults that obscure missing-key bugs.
- When no default is semantically valid ā€” a missing vocabulary entry is a programming error, not an expected absence ā€” project at the boundary as `Find(key).ToFin(Error.New(code, "missing key"))`, converting the lookup miss into a domain error that propagates on the rail. `IfNone(() => throw ...)` defeats the rail entirely; throwing inside a functional pipeline violates the expression-spine invariant that all domain logic composes monadically.

[CANCELLATION_AS_ABSENCE_BOUNDARY]:
- `CancellationToken` is a first-class absence signal for time-bounded and scope-bounded computations, not an optional parameter with a `default`. A function that silently ignores cancellation at a foreign boundary violates the seam contract.
- `CancellationToken.UnsafeRegister(Action<object?> callback, object? state)` skips `ExecutionContext` capture and `SynchronizationContext` flow. Use `UnsafeRegister` at high-frequency projection sites where the callback is a simple state CAS or an `Atom.Swap`. The two-arg overload `UnsafeRegister(Action<object?, CancellationToken> callback, object? state)` passes the token itself as the second parameter, avoiding a closure capture.
- `CancellationToken.None` has a null `_source`; both `Register` and `UnsafeRegister` short-circuit and return `default(CancellationTokenRegistration)` without registering the callback. A seam that conditionally registers on a token that may be `CancellationToken.None` must treat the default as a never-cancelled signal and operate without a cancellation gate for that duration.
- Registering on an already-cancelled token (`IsCancellationRequested == true`) invokes the callback synchronously on the calling thread before `Register` returns. A `UnsafeRegister` callback that calls `Atom.SwapMaybe` creates a reentrancy hazard: if the swap body calls back into code that itself calls `Register` on the same already-cancelled token, the inline synchronous invocation re-enters the swap body on the same thread. Guard by checking `IsCancellationRequested` before calling `Register` or `UnsafeRegister` and handling the already-cancelled state directly, bypassing registration entirely.
- `CancellationTokenSource.Cancel()` invokes `ExecuteCallbackHandlers` synchronously on the calling thread. `CancellationTokenSource.CancelAsync()` schedules `ExecuteCallbackHandlers` via `Task.Factory.StartNew(..., TaskScheduler.Default)` when callbacks exist; when no callbacks exist it returns `Task.CompletedTask` immediately. A seam that transitions `Atom<CapabilityState>` via a `UnsafeRegister` callback will have that transition run synchronously under `Cancel()` but asynchronously on a thread-pool thread under `CancelAsync()`. Code that reads `atom.Value` after calling `CancelAsync()` without `await`ing the returned task may observe the pre-cancelled capability state. `CancelAsync()` returns `Task.FromException(new ObjectDisposedException(...))` when the source is already disposed; awaiting it without a catch propagates the disposal exception.
- `CancellationTokenSource.CreateLinkedTokenSource(ReadOnlySpan<CancellationToken>)` accepts a params-span of tokens. A seam that must honor both an outer session token and an inner operation token creates a linked source; the linked source owns the cancellation lifetime and must be disposed at scope exit. Disposing the linked source does not cancel the originating tokens.
- `EnvIO` exposes `Token`, `Source`, and `LocalCancel`. `LocalCancel` creates a new `EnvIO` with a fresh `CancellationTokenSource` while inheriting `Resources` and `SyncContext`. `LocalCancelWithTimeout(TimeSpan)` scopes a time-bounded absence detection window inside an `IO` computation without propagating the timeout to the outer environment. Both `LocalCancel` and `LocalCancelWithTimeout` inherit the outer `Resources` tracker, so resources acquired inside either scope are tracked in the shared outer `Resources` and survive an inner timeout. `BracketFail()` creates an isolated `LocalResources` scope and releases it on failure regardless of whether it wraps or is wrapped by `Timeout`; both orderings (`ma.BracketFail().Timeout(timeout)` and `ma.Timeout(timeout).BracketFail()`) release the bracket scope's resources on expiry. The distinction that does matter is bare `ma.Timeout(timeout)` without any `BracketFail`: resources acquired inside the timed body remain in the outer `Resources` and are deferred to outer-scope disposal rather than released immediately on expiry.
- `EnvIO.Local` creates a fresh `CancellationTokenSource` and, when `Token.CanBeCanceled`, registers `Token.Register(source.Cancel)` ā€” storing the registration in `Registration` and disposing it when the `EnvIO` is disposed. This side-channel registration fires synchronously on `Cancel()` and asynchronously on `CancelAsync()`, matching the asymmetry documented in the `CancellationTokenSource.Cancel` vs `CancelAsync` bullet above. `EnvIO.LocalResources` by contrast passes the same `Source` as the outer `EnvIO` without creating a new source; `Cancel()` on the parent source fires immediately inside the child scope. Disposing the child via `DisposeNonResources()` does not dispose the shared source because the ownership bit is not set. A timed-out `BracketFail` probe scoped via `LocalCancelWithTimeout` creates a new source that IS owned and disposed by the child, isolating the timeout from the outer cancellation chain.
- `MonadIO.syncContext<M>()` returns `K<M, Option<SynchronizationContext>>`; called as `MonadIO.syncContext<IO>()` it returns `IO<Option<SynchronizationContext>>`. The `SyncContext` field on `EnvIO` is nullable ā€” `Optional(e.SyncContext)` lifts it to `Option`. The absence of a synchronization context is a first-class optional: `Some(ctx)` in UI-thread environments and `None` in background-thread or console environments. A seam that conditionally marshals based on context presence must use `MonadIO.syncContext<IO>().Bind(ctx => ctx.Match(Some: c => marshalOn(c), None: () => runDirect()))` rather than reading `SynchronizationContext.Current` directly ā€” the direct read bypasses the `EnvIO` boundary and is untestable.

[CHANNEL_SEAM_ABSENCE_PROJECTION]:
- `ChannelReader<T>.ReadAsync(CancellationToken)` throws `ChannelClosedException` when the channel is completed and empty. At the seam boundary, capture via `Error.New(exception)` and map to `Errors.SourceClosed` or a domain-specific terminal receipt; the thrown exception is `InvalidOperationException`-derived and its `Message` is not stable ā€” never match on string content.
- `ChannelReader<T>.TryRead(out T)` returns `false` for both "no item available" and "channel closed and empty" ā€” the absence cause cannot be distinguished from `TryRead` alone. Check `Completion.IsCompleted` to distinguish transient absence from terminal absence.
- `ChannelWriter<T>.TryComplete(Exception? error)` embeds the exception into `ChannelReader<T>.Completion`, surfacing it from `ReadAsync` as a `ChannelClosedException` whose `InnerException` carries the original. A typed `Error` is lost through this path; terminal receipts that must survive the channel must be encoded in the element type, not in the completion signal.

[LOCK_SCOPE_ABSENCE_INTERACTION]:
- `System.Threading.Lock.EnterScope()` returns a `ref struct Scope` whose `Dispose()` calls `Exit`. Because `Scope` is a `ref struct`, it cannot be stored in an `Option<Scope>`. A pattern that conditionally acquires a lock based on a domain-absent state must keep acquisition and scope structurally within the same stack frame. An async or effect-pipelined operation that needs conditional locking must instead use `Lock.TryEnter(timeout)` returning `bool` and map the `bool` to a domain absence signal. `Lock.TryEnter()` with zero timeout returns `false` immediately if contended; that is a boolean absence signal meaning "lock currently held," not "lock does not exist," and the caller must handle the contention case explicitly.

[HEALTH_FROM_TYPED_STATES]:
- System health must be derived from typed capability states and `DateTimeOffset` instants, never from status strings or boolean flags. A health surface is a fold over a `HashMap<CapabilityKey, CapabilityState>` that produces a typed aggregate: `Healthy`, `Degraded(Seq<CapabilityKey> impaired)`, `Unhealthy(Seq<CapabilityKey> failed)`. The aggregate is itself a closed union.
- `TimeProvider.GetUtcNow()` is the injectable time source for all instants stored in health records; `TimeProvider.System` is the production singleton. A health record that calls `DateTimeOffset.UtcNow` directly is untestable and violates the seam boundary for time. `TimeProvider.GetElapsedTime(long startingTimestamp)` returns elapsed `TimeSpan` from a prior `GetTimestamp()` call without a second argument ā€” the correct surface for measuring time-since-last-probe without holding a second timestamp field.
- `HashMap<Key>` is a non-static trait class implementing `Foldable<HashMap<Key>>`, `Functor<HashMap<Key>>`, `MonoidK<HashMap<Key>>`, and `SemigroupK<HashMap<Key>>`. Code that folds a capability map via `.AsIterable().Fold(...)` or LINQ `.Aggregate(...)` bypasses the `Foldable` trait and is not polymorphic ā€” it cannot be lifted into an `IO` fold that checks cancellation per element. For a vocabulary-complete health aggregate over `AtomHashMap<CapabilityKey, CapabilityState>`, call `AtomHashMap.Find(key)` per vocabulary key and map absence to `CapabilityState.Unknown`; do not fold over `AtomHashMap.AsIterable()`, which silently omits capabilities not yet probed.
