# Execution Standard

Every writing and review stage of a workflow holds one demand level, independent of the project and the artifact kind — real code, a design corpus, configuration, durable prose all take the same aggression. Project artifacts (a quality doctrine, capability catalogs, an architecture law) sharpen this standard where they exist; their absence never lowers it. This reference owns the stage-prompt CONTENT bar; topology, dataflow, and handoff integrity are the patterns reference.

## [01]-[STANCE]

Every substantive stage — author, critique, red-team, fixer alike — holds the work hostile:

- Every artifact under review was authored by ANOTHER engineer and is naive, shallow, or illusory until it survives a real attack; the burden of proof is on the work, never the reviewer. Dense, confident, idiom-fluent output is the PRIME suspect for hollowness — a name promising capability the body omits, decorative density carrying nothing, a stub dressed as finished work.
- Naivety is a defect on two orthogonal axes, both intolerable. COVERAGE — the owner models a thin slice of its concept: the obvious three fields where the domain carries fifteen, a two-case family for a twenty-case domain. APPROACH — enumerated hardcoded instances standing where one parameterized, algorithmic owner generates the space: a fixed roster of variants is seed DATA feeding one generator over named parameters, never the mechanism itself.
- Every enumerated checklist, collapse-signal list, and attack list in a stage prompt is a FLOOR hunted past, never the complete set — and every EXEMPLAR riding them is load-bearing steering data, never illustration: agents reproduce a prompt exemplar as output with the force of the law it rides, so an exemplar must be archetypal, domain-true, and pointed at the work's intended direction. That floor clause bounds a list's completeness, never a wrong exemplar — a stale or misdirected one seeds wrong output at scale, unrefused because it arrived as law — so audit every exemplar as a seed the run will grow, with the rigor of the law sentences around it.
- NO CHURN: an edit requires a named violated law or invariant and the concrete case that breaks it; a clean verdict earned by an attack that finds nothing is a first-class result, proven by adding nothing — and a reviewer forced to edit invents defects.

This stance is SUBSTANCE, not register: burden of proof on the work, both naivety axes, and the cold pass bind every stage regardless of model, but the PHRASING is model-shaped. A Claude-executed stage carries the hostile register verbatim — Claude reads the intensifiers as calibration. A codex-executed stage takes the same demands de-conflicted and neutral: intensifiers that sharpen a Claude lane make a codex lane over-use tools and probe out of territory, so a codex-routed prompt is built codex-shaped, never the Claude-register builder (codex-lanes reference, the prompt-shape law).

## [02]-[WRITER_BAR]

Implement stage authors ground-up, never patches:

- New capability is woven into the owning shape as if it had always existed — a case, row, field, operation, or policy value INSIDE an existing owner before any new surface appears; the surface narrows by absorption, never omission. Shims, compat aliases, migration layers, and append-beside are rejected; the API breaks when the collapse improves the system.
- Capability is conserved absolutely: densify, never delete; zero current consumers never lowers the bar, because the owner is shaped for the demand the domain implies. Anticipate the family — the proof of a correct shape is the diff of the next feature: one declaration inside the owner, every consumer untouched or loudly broken.
- Material depth: every admitted dependency is mined to the operator depth it ships; logic below that depth re-derives line-by-line what a deeper composed form states once, and a hand-rolled reimplementation of shipped capability is a defect. A cited external member is verified against its source of truth before it is written; an unverifiable member is never authored.
- Variation lives in input shape, policy values, and table rows — one polymorphic entrypoint per concern discriminating on the value itself, never name-suffix siblings, boolean knobs, or parallel exports. Anything that can vary is a parameterized value, never a hardcode.
- Where the project carries a quality doctrine, the writer reads it AT SOURCE in full and treats it as the floor to push past; where none exists, this register still binds in the language's own strongest idiom.
- Each writer's own blind pass is the primary product: its cold defect list and design rulings precede any recon consumption, and the majority of its diff comes from its own attack — the own-pass-first ladder and its tripwire are the mapping-lanes section's contract.

## [03]-[REVIEWER_BAR]

Critique and red-team are REBUILDERS with the writer's full authority, never annotators:

- COLD PASS first: the reviewer derives its own defect list from the artifact on disk before consulting any prior claim; producer output is strictly a refutation target (handoff law: patterns reference). Scope rows bound where the reviewer looks first — never what it may fix.
- Beyond fixing: a reviewer that only repairs findings underperforms its mandate. Types, constants, classes, and functions sharing a discriminant COLLAPSE into fundamentally stronger owners; thin owners EXTEND to the full domain their concept carries; a fundamentally stronger design, once seen, is BUILT — never defended against. That work ends objectively denser and more capable than the prior stage left it, or the reviewer proves the strongest form is already present by finding nothing.
- Distinct objectives per stage, never two identical reviews. CRITIQUE is predicate-positive: the clause-by-clause conformance and capability-completeness audit — verify each required law holds, cite the clause, repair every miss in place. RED-TEAM is predicate-negative: the pre-mortem — counterfactual on the core owner/algebra/dispatch, diff-of-the-next-feature, long-tail and failure-mode attack (empty/singular/plural/stream/malformed/concurrent/cancelled/partial-failure), boundary and ownership integrity, sprawl and phantom members — then one full cold re-review of every conformance dimension by name.
- Residuals drain forward, never pool: every defect found is fixed at its root in the same pass regardless of scope; deferral is reserved for territory a LIVE sibling owns, and deferred rows drain at the nearest subsequent stage — a terminal mega-reconcile is the priced-in failure of every stage before it.

## [04]-[MAPPING_LANES]

One burden-reduction contract: writers write, cheap lanes read.

- Cheap-model mapping lanes precede every substantive writing stage, absorbing the deep reading — full-file territory reads, dependency-surface enumeration to operator depth, seam and consumer tracing, current-state census, defect hunting — so the writer arrives pre-mapped instead of spending context on reconnaissance. That legwork licenses scale: pre-paid navigation raises the scope an expensive writer holds; deepening the cheap lanes shrinks the expensive fan, never the reverse.
- Mapping fans at the WORK-UNIT granularity the writing batches will own — one lane set per unit (a sub-folder, a module, a coherence boundary), each lane's scope congruent with the batch seam that consumes it. One lane over a whole corpus or package dilutes per-file depth in proportion to the unit count and starves every batch equally; mappers outnumbering writers is the correct shape, never an inversion.
- Each map PRODUCT is a consumer-scoped contract, never a dump: every stratum's dependency-catalog roster classified used / underutilized / unused — shallow usage of a composed member counts as underutilized (one call where the surface carries a family); a location map anchoring each concern to its owning file and line; seam and ripple endpoints both ends named; entries scoped to their consuming stage.
- Cross-territory relevance rides the same map — what sibling units hold that this unit composes or plainly touches — so the writer navigates by trust-then-verify at the anchor instead of exploring; the evidence comes from surgical probes at named symbols, never full foreign reads, under the same verification duty.
- Where a run fans concurrent writers over one corpus, ideation between mapping and writing runs as TWO lanes with disjoint charters, never one merged log — merged, the verified-small entries crowd out ambition and the product regresses to a fixlog.
- CORRECTIONS CENSUS consolidates the map-surfaced defects — drift, phantoms, spelling repairs, seam mismatches, wiring gaps — deduped and disk-verified: the non-binding fix addendum writers land where it intersects their scope and a terminal stage drains where it does not.
- AMBITION WORKLIST carries only NEW capability the domain admits — dimensions, modalities, families, operations — under a value bar routing any correction-shaped entry back to the census; each idea names an owner, an admitted member or domain concept, a disk anchor, and why, with cross-territory ideas naming both ends and their seam. Ideas are ambition and information, never a prescription or a ceiling; the second-pass cull deletes false and small entries, never ambitious ones.
- Maps carry INFORMATION, never prescriptions: anchored facts, verified member spellings, seam endpoints, capability the concept admits but nothing exploits. A map entry that tells the writer what to build — a fence sketch, a prescribed shape, a pre-ruled design — anchors and weakens the writer and is the lane's defect; findings name the law and the defect, never the resulting code.
- Consumption runs an OWN-PASS-FIRST ladder, binding on writers and reviewers alike: (1) the stage's own blind pass — defect list, collapse targets, and design rulings derived from disk BEFORE any recon product opens; (2) the maps as grounding to verify and exceed; (3) the census as addendum; (4) the worklist as ambition.
- Anchoring is procedural before it is tonal: a consumption protocol that orders report-reading first pre-commits the writer's budget regardless of what the prose disclaims. That tripwire makes the ladder checkable — a stage whose diff maps one-to-one onto the recon rows has failed its mandate, because the recon covers a minority of what the work demands.
- Delivery is part of the contract: a map produced but not routed is spend without effect. Each consuming stage's prompt names the exact report paths, orders the read per the ladder, and instructs full-read, spot-verify, and exceed — the writer never relitigates what a map settled and never treats it as a ceiling.
- One read is never delegated: the writer consumes its quality doctrine at source itself — doctrine conformance cannot ride a cheap lane's summary. Everything else — inventory, seams, member verification, staleness — is mapped for it.

## [05]-[DURABLE_LEARNING]

One loop converts run findings into standing law, active wherever the repo carries a maintenance-law corpus (a laws router such as `docs/laws/README.md` that owns the corpus admission law):

- Every substantive stage reads the corpus at source — it stays small by its own admission law, so the read is cheap — and an edit touching a topology surface lands the obligated counterparts in the same pass.
- Writer and reviewer schemas carry a required-but-usually-empty `harvest` attestation: nominations of generalizable lessons only — a reusable collapse pattern, an unnamed naivety class, a review rule that catches the defect class before review, a hard-won coupling — each citing the exact existing clause it hardens, or proving absence across the searched surfaces. A stage-local fix never nominates.
- ONE terminal doctrine stage per run adjudicates the pooled nominations refutation-first against the live surfaces under the corpus admission law — harden an existing clause, then extend an existing file, then mint a new file — with the repo's durable-prose skills loaded before any edit. Landing nothing is a first-class verdict; a landed row states law and carries no run provenance; a rule a formatter, analyzer, or gate can enforce routes to that gate owner, never to reviewer prose. That lander prompt ORDERS a read of the corpus router and never restates its routing or bar — a workflow that inlines the routing forks it from the owner and drifts on the next landing.
- Terminal residual drains run as a fixpoint loop: one serial closer per round takes the full pooled residual set, verifies every row against live disk before acting (rows a later stage already resolved are culled with proof), fixes the rest at root, and re-feeds the still-open remainder; a round cap and a no-shrinkage progress gate bound the loop, and the genuinely blocked remainder — each row naming its blocker and owner — rides the run return.
