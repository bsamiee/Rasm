# Absence and Sentinels — Boundary Law

[SENTINEL_PROJECTION]:
- Every host sentinel — null, `IntPtr.Zero`, `nint(-1)`, `DateTime.MinValue`, `Guid.Empty`, a `0`-handle, a dedicated "unset" constant, or a magic fallback value — is a foreign-language absence signal. Project it to `Option<T>.None` at the exact foreign-call boundary using `Prelude.Optional(nullableRef)` for reference types, `Prelude.Optional(nullable)` for `Nullable<T>` via `HasValue`, or an explicit discriminant expression (`h.IsInvalid ? None : Some(h)`) for handle types. Both `Optional` overloads are zero-allocation inline conversions.
- `Optional` returns `default(Option<A>)` — the `Option<A>.None` static field — for null and for `Nullable<T>` where `!HasValue`. The implicit `operator Option<A>(A? a)` invokes `Prelude.Optional` directly, so an explicit call and an implicit cast are byte-identical; the explicit form is preferred because it names the projection site.
- `SafeHandle.IsInvalid` and `SafeHandle.IsClosed` are the dual-sentinel surface for native handles. Project to `Option<T>` immediately after construction: `handle.IsInvalid || handle.IsClosed ? None : Some(handle)`. Once inside the domain, the `Option` carries lifetime; the raw handle is never stored. `DangerousAddRef` must be called with `ref bool success` inside the projection site if the handle is held across a suspension point — the `success` flag is itself a boolean absence signal that must be checked before the ref-counted access proceeds.
- A sentinel value must not enter a persisted shape. `Option<A>` implements `ISerializable`; its serialization constructor reads `IsSome` and conditionally `Value`, so a field of type `Option<T>` round-trips correctly under `ISerializable`-aware formatters. For foreign wire protocols — JSON, Protobuf, MessagePack — project `Option<T>` to a nullable wire field at the serialization boundary and project back on deserialization; the domain model never carries the wire representation. The `System.Text.Json` serializer without a custom converter does not understand `ISerializable` and will produce a broken round-trip.
- When a foreign API returns a negative integer sentinel (e.g., `UnsetIntIndex = -1`), the projection must be explicit: `idx >= 0 ? Some(idx) : None`. The implicit `operator Option<A>(A? a)` fires only for reference types and `Nullable<T>`; it does not intercept value types with magic constants. Missing this produces `Some(-1)` — a sentinel propagating inward wearing a typed wrapper.

[TYPED_ABSENCE_CARRIERS]:
- `Option<T>` is the sole carrier for presence/absence at domain boundaries. It is a `readonly struct`, so the `None` case allocates nothing; `default(Option<T>)` is `None`. Pattern matching uses `{ IsSome: true, Case: T value }` property patterns to avoid the `Match`-mid-pipeline prohibition.
- `Option<T>.ToFin(Error fail)` converts absence to a typed failure for use on the error rail. `Option<T>.ToFin()` with no argument uses `Errors.None` — a `ManyErrors` with code `-2000000006` and an empty error sequence — which matches no `HasCode` predicate and no `IsType<E>()` predicate at recovery sites. Always supply a domain-specific `Error` with a non-zero code when the absence has a named cause.
- `Option<T>.ToEff(Error fail)` lifts absence into an `Eff<T>` computation; the zero-arg overload likewise uses `Errors.None` and loses diagnostic context. `Option<T>.ToIO()` promotes to `OptionT<IO, A>` — use `OptionT.lift<M, A>(Option<A>)` when the target outer monad is not `IO`.
- `OptionT<M, A>` is the monad transformer for threading absence through an outer effect `M`. `OptionT.lift<M, A>(Option<A>)` constructs from a known value; `OptionT.lift<M, A>(K<M, A>)` lifts a successful `M` computation. `Natural<Option, OptionT<IO>>` is implemented via a `Natural.Transform` on the trait class.
- `HashMap<K, V>.Find(key)` returns `Option<V>`. All key-bounded lookups in domain structures must use `Find`, never a try-get-then-branch pattern. A missing key is structural absence; `FindOrAdd` is a state mutation and must not masquerade as a default.

[CAPABILITY_ABSENCE_AS_TYPED_STATES]:
- An unavailable capability is not `Option.None`; it is a discriminated case in a closed capability family. `Option<T>` models a value that might be absent; a capability state is a richer fact — unavailable, degraded, disabled, or transitioning — each case carrying distinct fields. Collapsing all capability states to `Option.None` erases that evidence at the consumer and forces re-derivation.
- A degraded outcome must carry a usable result alongside first-class degradation evidence — never a nullable companion field, never a swallowed exception, never a null result with a string message. The degradation evidence must be a typed value the caller can inspect, route, and log without downcasting.
- Capability-absent states with mandatory consumer handling are expressed as a `[Union]`-generated closed family with at least `Available`, `Degraded`, and `Unavailable` cases, each with its own payload. The total `Switch` dispatcher enforces that every consumer handles all modalities; there is no default fallthrough.
- A capability in `Unavailable` state must emit a definite `Fin.Fail(terminalError)` at every operation site, not an indeterminate `Option.None`. `Option.None` is structurally indistinguishable from "not yet known" to a downstream caller. `Fin.Fail` with a coded `Expected` error is definite: it names the condition and stops propagation.

[INITIAL_VALIDATION_STATE]:
- A screen or form must never transiently render in a valid state before user interaction. The domain model for a UI binding must carry an explicit `Unvalidated` state distinct from `Valid` and `Invalid`. A three-case union — `Unvalidated`, `Valid(T value)`, `Invalid(Error reason)` — owns this surface. The initial state is always `Unvalidated`. Binding `Option.None` as the initial value and treating absence as "not yet invalid" is the anti-pattern: any consumer that folds `None` to a success path will render falsely clean.
- `Validation<F, A>` requires `F : Monoid<F>` for applicative accumulation. A validation-state closed union that wraps `Validation<Error, T>` for the `Valid` and `Invalid` cases and adds `Unvalidated` as a separate case is the correct dense surface; do not embed `Validation` where the `Unvalidated` state cannot be expressed.

[PROGRESS_AS_DISCRIMINATED_VALUE]:
- Declared-unknown progress is a discriminated value, not a nullable integer or a float in `[0, 1]`. A `[Union]`-generated progress family with cases `Unknown`, `Indeterminate`, and `Known(double fraction)` enforces the monotone non-decreasing law: any transition from `Known(f1)` to `Known(f2)` where `f2 < f1` is a violation, detectable at the swap site with `Atom<ProgressState>.SwapMaybe(prev => prev is Known(var f) && next < f ? None : Some(next))`.
- An `Unknown` → `Indeterminate` → `Known` progression must never regress to `Unknown` on a transient error. The boundary seam that feeds progress from a foreign source must detect any backward transition and clamp to `Indeterminate` or hold the prior `Known` value; it must not pass the foreign value through.

[DISABLED_STATE_DEFINITENESS]:
- A host state that is absent must emit a definite `Disabled` or `Unavailable` signal to consumers — never bind an `Option.None` into a UI or reactive pipeline and rely on downstream `IsNone` checks. `Option.None` at a subscription boundary creates an indeterminate optional that consumers must individually defend; a definite `Disabled` emission from the capability cell terminates propagation at the single seam where the foreign state is received.
- `Atom<CapabilityState>` with `CapabilityState` as the closed union owns this boundary cell. The initial value is `Unavailable`, and the first successful probe transitions it to `Available`. `SwapMaybe` is the only mutation API because it makes the failed-transition case observable. The `Change` event on `Atom<A>` fires on every successful swap; absent subscription deregistration, this event fires indefinitely and the seam must dispose its registration.

[TERMINAL_TYPED_RECEIPTS]:
- Native-library unavailability, store corruption, and schema-version rejection each require a terminal typed receipt — a `sealed record` deriving `Error` with a domain-specific non-zero `int` code — and no internal retry path. `Error.IsType<E>()` is the discriminant at recovery sites. `Error.Is(Error error)` performs code-equality for `Expected` (non-zero code) or message-equality (zero code); always use a non-zero code for terminal receipts to prevent accidental equality with string-matched errors.
- A terminal receipt must not silently re-enter the system. `Catch<F, A>(K<F, A> fa, Func<Error, bool> Predicate, Func<Error, K<F, A>> Fail)` routes recovery through a predicate; a predicate that matches the terminal code and returns a fallback is a hidden retry path. Terminal receipts are either propagated unchanged or surfaced as a `Fin`/`Eff` failure; they are never transformed into a default result.
- Schema-version rejection must produce a receipt carrying the stored version and the expected version as typed fields — not a formatted string. A consumer that displays or logs the mismatch reads typed fields, never parses text.

[MISSING_ASSET_KEY_FALLBACKS]:
- An unhandled lookup failure on a key-bounded vocabulary must not throw or return null. Use `HashMap<K, V>.Find(key)` returning `Option<V>`, followed by `IfNone(defaultValue)` or `IfNone(() => deriveDefault())` at the single point where a fallback is semantically correct. Placing the fallback deeper creates invisible defaults that obscure missing-key bugs.
- When no default is semantically valid — a missing vocabulary entry is a programming error, not an expected absence — project at the boundary as `Find(key).ToFin(Error.New(code, "missing key"))`, converting the lookup miss into a domain error that propagates on the rail. `IfNone(() => throw ...)` defeats the rail entirely and is forbidden under `EXPRESSION_SPINE`.

[ATOM_AND_OPTION_GATED_CELLS]:
- `atomIO<A>(value, validator)` wraps `Atom.New(value, validator)` in `IO.lift`; when the initial value fails the validator, it raises `Errors.ValidationFailed` (code `-2000000011`, `IsExpected = true`) as a failed `IO<Atom<A>>`. A recovery site that needs to distinguish construction failure from other errors should match against `error.HasCode(-2000000011)` or `error.Is(Errors.ValidationFailed)`. The non-IO `atom(value, validator)` Prelude function returns `Option<Atom<A>>.None` on failure without throwing; the two construction paths have different error surfaces and must not be treated interchangeably.
- `Atom<Option<T>>` is the standard cell for singleton token ownership: `None` represents unowned, `Some(token)` represents held. `SwapMaybe(current => current.IsNone ? Some(newToken) : None)` attempts a take; returning `None` from the swap function leaves the cell unchanged. The post-swap value returned by `SwapMaybe` is the current state — the new value on success, or the unchanged value if the guard rejected. Distinguishing success from rejection requires the caller to hold a reference to the prior value and compare it to the result, because there is no discriminated return type.

[CANCELLATION_AS_ABSENCE_BOUNDARY]:
- `CancellationToken` is a first-class absence signal for time-bounded and scope-bounded computations, not an optional parameter with a `default`. A function that silently ignores cancellation at a foreign boundary violates the seam contract.
- `CancellationToken.UnsafeRegister(Action<object?> callback, object? state)` skips `ExecutionContext` capture and `SynchronizationContext` flow. Use `UnsafeRegister` at high-frequency projection sites where the callback is a simple state CAS or an `Atom.Swap`. The two-arg overload `UnsafeRegister(Action<object?, CancellationToken> callback, object? state)` passes the token itself as the second parameter, avoiding a closure capture.
- `CancellationToken.None` has a null `_source`; both `Register` and `UnsafeRegister` short-circuit and return `default(CancellationTokenRegistration)` without registering the callback. A seam that conditionally registers on a token that may be `CancellationToken.None` must treat the default as a never-cancelled signal and operate without a cancellation gate for that duration.
- `CancellationTokenSource.CreateLinkedTokenSource(ReadOnlySpan<CancellationToken>)` accepts a params-span of tokens. A seam that must honor both an outer session token and an inner operation token creates a linked source; the linked source owns the cancellation lifetime and must be disposed at scope exit. Disposing the linked source does not cancel the originating tokens.
- `EnvIO` exposes `Token`, `Source`, and `LocalCancel`. `LocalCancel` creates a new `EnvIO` with a fresh `CancellationTokenSource` while inheriting `Resources` and `SyncContext`. `LocalCancelWithTimeout(TimeSpan)` scopes a time-bounded absence detection window inside an `IO` computation without propagating the timeout to the outer environment.

[HEALTH_FROM_TYPED_STATES]:
- System health must be derived from typed capability states and `DateTimeOffset` instants, never from status strings or boolean flags. A health surface is a fold over a `HashMap<CapabilityKey, CapabilityState>` that produces a typed aggregate: `Healthy`, `Degraded(Seq<CapabilityKey> impaired)`, `Unhealthy(Seq<CapabilityKey> failed)`. The aggregate is itself a closed union.
- `TimeProvider.GetUtcNow()` is the injectable time source for all instants stored in health records; `TimeProvider.System` is the production singleton. A health record that calls `DateTimeOffset.UtcNow` directly is untestable and violates the seam boundary for time.
- `TimeProvider.GetElapsedTime(long startingTimestamp)` returns elapsed `TimeSpan` from a prior `GetTimestamp()` call without a second argument. This is the correct surface for measuring time-since-last-probe without holding a second timestamp field.
