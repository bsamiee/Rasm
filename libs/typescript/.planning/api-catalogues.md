# [TS_API_CATALOGUES]

The branch evidence protocol over the `.api/` catalogue set — the single decompile-grounded truth surface every fence's API spelling resolves against. Catalogues are reflected from installed `node_modules/<pkg>/dist/dts/*.d.ts` (the same declaration surface `assay api` reflects for npm packages); a member, knob, or behavior not traceable to a catalogue page or a completed decompile probe is never written into a fence — it becomes a RESEARCH item on its owning page. The named defect class is the phantom API: a documented member that does not exist in the installed `.d.ts`. This page carries no API member tables — those live in the per-library catalogue files; it carries the protocol, the catalogue-to-domain map, and the verified gap posture.

## [1]-[EVIDENCE_PROTOCOL]

- Source of truth: the installed `.d.ts` under `node_modules`, member-by-member, at the version the workspace catalog pins. `assay api query --key <pkg> --symbol <Name>` is the resolver; where the resolver mis-fuzzes across same-named exports (the `effect` 176-module case) or is unavailable on a non-interactive PATH, direct `.d.ts` grep is the authoritative fallback and is equivalent or stronger.
- Completeness bar: every documented symbol exists with exact spelling, count=1; zero phantom APIs; no load-bearing surface missing. Overload sets render as the single canonical curried/data-last arm; large inferred conditional types render as the load-bearing call shape with the variant set spelled out.
- Scope discipline: a catalogue is non-exhaustive by charter where a package ships hundreds of modules — it scopes to the load-bearing surfaces the five charters consume and records the deliberately-elided modules as scope decisions, never gaps.
- Tier tag: where no Nx `project.json` tag exists yet, the catalogue records the stratum by import-graph inference (`node:*` imports → node; browser-only peers → browser; effect+platform only → neutral) and flags it as inference until the project graph materializes.

## [3]-[VERIFIED_GAP_POSTURE]

- effect: 176-module surface scoped to the load-bearing core; the five red-team repairs landed (`Effect.serviceOption` not `Effect.service`; `Layer.effect` not `Layer.fromEffect`; `Schema.Record` options-bag arity; `Schema.optional` return; `Schema.Array` `Array$` token). - @effect/platform, @effect/platform-browser, @effect/platform-node, @effect/rpc, @effect/cluster, @effect/workflow, @effect/experimental, @effect/cli, @effect/sql, @effect/sql-pg, the @effect/ai set, @effect/opentelemetry, @effect/vitest, @effect-atom/atom + atom-react: verified clean or repaired against the installed `.d.ts`; the per-library `gaps` notes carry the residual scope decisions and the irreducible Nx-tag inference flags.

- @effect/language-service, @effect/tsgo, @effect/eslint-plugin: CLI/plugin surfaces with no importable value contract; catalogued from package.json + `--help` self-report + schema.json, recording the no-value-import posture.

## [4]-[REFINEMENT_HORIZON]

A catalogue is re-verified every loop against the installed `.d.ts` at the pinned version; a version bump re-runs the reflection and folds any member-shape divergence back into the dependent fence. The capability-descriptor codegen row (a typed effect-classed capability SDK + MCP client generated from the C# `capability-registry#CAPABILITY_CATALOG` descriptor) is the one catalogue-side horizon entry, gated on the upstream C# fence and recorded as a TS-downstream DAG seam; until it lands the `@effect/ai` McpServer/McpSchema module-level surface stays catalogued at module level only.
