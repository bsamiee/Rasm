# surfaces-and-dispatch — bedrock (entrypoint + dispatch collapse)

`[ENTRYPOINT_MODAL_ARITY]`: ONE value-level entrypoint discriminates on input shape via verified guards — `Predicate.isIterable` `(input: unknown) => input is Iterable<unknown>`, `Array.isArray` (dual: refines `Array<A>` and asserts `ReadonlyArray`), `Option.isOption` `(input: unknown) => input is Option<unknown>`, `Predicate.isRecord`/`isObject`/`isTagged(tag)`. The typed result correlates via overload pair (`{ (x: K): V; (x: K[]): V[] }`) or conditional return `T extends readonly K[] ? V[] : V`. Deletes the `getById`/`getMany`/`getByIds` triad: `const resolve: { (k: Id): Eff<V>; (ks: readonly Id[]): Eff<V[]> } = (k) => Array.isArray(k) ? Effect.forEach(k, one) : one(k)`. Growth axis = new input modality is a new guard arm, not a new exported verb. TRAP: order guards most-specific-first — `isRecord` swallows `isOption` (Options are records). The boolean-mode param (`get(id, { many: true })`) and name-suffix family are the rejected forms; both fail the 3-4x bar by forcing every consumer to know the suffix vocabulary.

`[DUAL_LAW]`: `Function.dual` collapses data-first + data-last into ONE impl — the LAW for any operator a custom owner exposes both pipe-able and applied. Verified two arg forms: `dual(arity, impl)` (integer arity, the common case) and `dual((args) => args.length === N, impl)` (predicate form, mandatory when trailing args are optional so arity is ambiguous). Type the overload pair explicitly: `const op: { (that: T): (self: S) => R; (self: S, that: T): R } = dual(2, (self, that) => ...)`. Deletes the parallel `opPiped`/`opApplied` sibling pair and every `(self) => (that) =>` manual curry. Growth = the impl body; both call styles ride free. TRAP: predicate form is REQUIRED when an optional param makes the data-first arg count collide with data-last — arity-number form silently misroutes there.

`[SATISFIES_DISPATCH_TABLE]`: `const T = { ... } as const satisfies Record<Tag, Handler>` is the law for pure value dispatch over a finite union. Under `noUncheckedIndexedAccess` (confirmed active), indexing `T[tag]` where `tag: Tag` (finite union, not `string`) yields the handler type WITHOUT `| undefined` — the table is total by construction, so no `?? fallback` ceremony. Rows carry POLICY VALUES not bare fns: `{ retry: Schedule; timeout: Duration; render: (x) => View }` per row, read field-wise. Deletes the `switch (tag) { case "a": ...; case "b": ... }` ladder and the N parallel `aHandler`/`bHandler` consts. Derive secondary tables from the primary via mapped type — declare ONE correspondence, compute the rest: `type Inverse = { [K in keyof T as T[K]]: K }`. Growth = one row; `satisfies` makes a missing/excess key a compile error at the table, every consumer untouched. TRAP: `as const` BEFORE `satisfies` (order matters — `satisfies` widens otherwise); template-literal-derived keys (`` `${Verb}_${Noun}` ``) need the union pre-materialized or the exhaustiveness check erodes.

`[MATCH_EFFECTFUL_DISPATCH]`: when arms return Effects or need predicate/structural matching (not bare `_tag` lookup), `Match.typeTags<U>()({ A: (a) => eff, B: (b) => eff })` is the one-call dispatcher — verified signature returns `(input: U) => Ret` and enforces exhaustiveness (excess keys typed `never`). `Match.valueTags` for value-level matchers, `Match.tags`/`Match.tagsExhaustive` for pipe-composed matchers, `Match.tag("A", "B", fn)` for collapsing multiple tags to one arm. Rows returning Effects compose: `Match.typeTags<Req>()({ Read: (r) => repo.get(r.id), Write: (r) => repo.put(r) })` — output `Effect<...>` unifies across arms. Deletes the effectful `switch` with `Effect.succeed` in every arm. TRAP: `satisfies`-table beats `Match` for PURE value dispatch (no Effect, no predicate) — `Match` adds a matcher allocation; reserve it for effectful/predicate/nested-discriminant arms. `Match.withReturnType<R>()` pins the arm return type when inference widens.

`[TAGGEDENUM_DOLLAR_MATCH]`: `Data.taggedEnum<U>()` yields a constructor record PLUS `$match` and `$is` derived from the union — verified: `$is: <Tag>(tag) => (u) => u is Extract<A, {_tag: Tag}>` and `$match` (curried matcher). One declaration produces constructors + total matcher + per-tag refinements. Deletes hand-written `makeA`/`makeB` factories AND the separate `isA`/`isB` guards AND the match switch — three families collapse to one `taggedEnum` call. Growth = one case in the union type; constructors, `$is`, `$match` all extend automatically. For recursive/generic enums use the `TaggedEnum.WithGenerics` interface form. TRAP: `$match` is curried-pattern style; if you need a one-shot value matcher prefer `Match.valueTags`. `Data.taggedEnum` instances get structural `Equal`/`Hash` free — relevant when used as map keys.

`[VERB_FAMILY_COLLAPSE]`: a request union — `Schema.TaggedRequest<Self>()(tag, { payload, success, failure })` family (verified: payload is `Struct.Fields`, success/failure are schemas) or `Data.taggedEnum` — plus ONE total handler is the entire API surface. N exported functions (`createUser`, `deleteUser`, `listUsers`) collapse to `handle: (req: Req) => Effect<Resp, Err>` dispatched by `Match.typeTags`. The request shape carries its own success/failure schema, so the handler's typed result rail is per-case-correct without overloads. Deletes the N-function module export AND N ad-hoc error types (each `TaggedRequest` owns its `failure`). Growth = new request case + new `Match` arm; every consumer (client, server, queue) keeps calling the ONE `handle`. TRAP: `TaggedRequest` requires a `Self` generic (`MissingSelfGeneric` error otherwise) — `class Get extends Schema.TaggedRequest<Get>()("Get", {...})`. Pairs with `@effect/rpc` for the wire boundary without re-declaring shapes.

`[SERVICE_SURFACE_COLLAPSE]`: ONE `Effect.Service<Self>()(key, { effect | scoped | sync | succeed, dependencies, accessors: true })` owns a concern's COMPLETE method family as a closed record — verified the four make-forms and `accessors?: boolean`. `accessors: true` generates static proxies so `MyService.method(x)` works without `Effect.flatMap(MyService, s => s.method(x))` — deletes the per-method `serviceFunctions` boilerplate. Sibling services (`UserReader` + `UserWriter` + `UserValidator` over one aggregate) collapse to one `User` service whose record holds all methods. The method-count law: when ≥3 methods share a discriminable input shape, merge them into ONE polymorphic method dispatching internally (see `[ENTRYPOINT_MODAL_ARITY]`) rather than exposing siblings. `Context.Reference<Self>()(id, { defaultValue })` for ambient policy (verified: requires `defaultValue: () => Service`) — deletes threading a config param through every call; it has a default so consumers needn't provide it. TRAP: `dependencies` on the Service builds the layer graph in place — no separate `Layer.provide` wiring file. `accessors: true` proxies lose `this`-bound state; fine for pure method records.

`[ASPECT_COMPOSITION]`: pure-Effect middleware is a value of shape `<A,E,R>(self: Effect<A,E,R>) => Effect<A,E,R'>`. `withSpan`, `retry`, `timeout`, `annotateLogs`, `tapErrorCause` all expose this data-last overload (verified `withSpan(name, opts)` returns `(self) => Effect<...>`). Compose ONE reusable aspect by `flow`-ing them: `const instrumented = flow(Effect.retry(policy), Effect.timeout(d), Effect.withSpan("op"), Effect.annotateLogs({svc}))` — a single value applied by `pipe(self, instrumented)`. Deletes copy-pasting the same `.pipe(retry).pipe(timeout).pipe(withSpan)` chain at every call site. `Effect.fn("name")(function*(){...})` is the definition-time aspect — wraps a generator-bodied function with a span AND gives stack-trace capture for free (verified `Effect.fn` accepts `(name, options?)`). Layer-level: `Layer.tap`/`Layer.tapErrorCause`/`Layer.map`/`Layer.fresh` (verified) wrap layer construction. Schema-annotation-driven derivation (annotations on a `Schema` → derived `Arbitrary`/`Equivalence`/`Pretty`) is the definition-time aspect for data. TRAP: aspect ORDER is semantic — `retry` outside `timeout` retries the whole timeout window; `timeout` outside `retry` bounds total. `withSpan` removes `Tracer.ParentSpan` from `R` (verified `Exclude<R, Tracer.ParentSpan>`) — outermost span owns the trace root.

`[PIPEABLE_OWNER]`: custom owners inherit free `.pipe()` via `Pipeable.Class()` (verified `Pipeable.Class: () => PipeableConstructor` and `<TBase>(klass) => TBase & PipeableConstructor`) or `extends Effectable.Class<A,E,R>` (verified abstract class) when the owner IS an Effect. The class gets `pipe(...fns)` threading itself through — deletes the bespoke `.map()/.flatMap()` fluent method wall and lets the owner reuse the entire `flow`/`pipe` combinator ecosystem. `Pipeable.pipeArguments(self, arguments)` is the manual impl if not extending. Growth = the owner gains any new combinator for free; none added to the class. TRAP: a fluent BUILDER (`.withX().withY().build()`) is REJECTED for a single options-object + `satisfies` — the builder fragments one config into N setter methods (N sibling surfaces) where `make(opts satisfies Config)` is one. Reserve `Pipeable` for genuine value-threading owners (parsers, streams, custom monads), not config assembly.

`[VARIADIC_TUPLE_ABSORPTION]`: variadic tuple signatures absorb arity into ONE signature — `pipe`/`flow` (verified overload chain `pipe<A,B,C,...>`) and any owner using `<A extends ReadonlyArray<unknown>>(...a: A)`. Labeled tuples (`[id: Id, opts?: Opts]`) name positional args without an object. Optional trailing policy params collapse to a single options object with defaults via plain destructuring-with-defaults (`({ retries = 3, timeout = "5s" }: Opts = {}) => ...`) or `Schema.optionalWith(s, { default: () => v })` at the schema boundary. Deletes the `fn(a)`, `fn(a,b)`, `fn(a,b,c)` overload ladder AND positional-arg-creep where the 4th boolean param means something. Growth = the tuple type extends; the signature is unchanged. TRAP: under `exactOptionalPropertyTypes` (confirmed active), an optional options field cannot be passed `undefined` explicitly — use `Schema.optionalWith` or omit the key entirely; `{ timeout: undefined }` is a type error where `{}` is valid.

`[INDEXING_TOTALITY_LAW]`: the cross-cutting invariant enabling all table dispatch — under `noUncheckedIndexedAccess` (confirmed), `Record<FiniteUnion, V>[key: FiniteUnion]` is `V` (total, no `| undefined`), but `Record<string, V>[key: string]` is `V | undefined`. KEY all dispatch tables by the finite union, never widen the key to `string`. This is what makes `as const satisfies Record<Tag, Handler>` undefined-free and deletes every defensive `?? throw new Error("unknown tag")` — the type system proves totality, so the runtime guard is dead code. TRAP: a template-literal key type (`` `${A}.${B}` ``) is finite only if A and B are finite unions; one `string` factor poisons it back to partial indexing.

Further Considerations
- `Schema.TaggedRequest` + `@effect/rpc` lets the SAME request union be the in-process handler, the wire protocol, AND the typed client — the verb-family collapse extends across the network boundary with zero shape re-declaration, which most treat as three separate surfaces (DTO, client SDK, server router).
- `Context.Reference` (verified, requires only `defaultValue`) is strictly denser than a `Service` for ambient read-only policy because it needs no Layer to run — but it cannot carry effectful construction; the moment a policy needs IO to build, it must promote to `Effect.Service` with `scoped`/`effect`, and that promotion is API-breaking for `accessors`. Choose `Reference` only for genuinely pure ambient values.
- `Match.withReturnType<R>()` placed before the arms turns the matcher into a contravariant check that REJECTS any arm not returning `R` — using it converts a dispatch table into a compile-time exhaustiveness-AND-uniformity proof, which catches the common bug where one arm accidentally returns a wider/narrower type than its siblings.
