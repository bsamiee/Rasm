# Dispatch Forms — Distilled

[TRAIT_DEFAULT_DERIVATION]:
- A carrier trait declares the smallest closed abstract core and derives the operation family as static-virtual defaults: the reducing trait declares two predicate-gated folds and derives roughly fifty receiver operations from them — count as a fold incrementing an integer, emptiness as a seeded fold halting on the first element, partition as one fold threading a pair; the applicative trait declares lift plus apply and derives the whole effectful-sequencing surface; the alternative trait adds empty plus choose and derives the repetition suite. Satisfying a trait costs only its abstract core; the derivation is literal, and trait inheritance stacks surfaces — the most-derived trait a carrier satisfies selects its richest dispatch surface, every ancestor surface arriving with it.
- Override only where a structurally cheaper primitive exists — a carrier with an O(1) size field overrides count and inherits the other forty-nine — and an override never alters the law the default encodes, only its cost: the trait default is the executable specification a candidate override is verified against, and a carrier with no faster primitive correctly ships zero overrides.
- The deferred-second-argument overloads on choose and apply are mandatory infrastructure, not optimization: the recursive repetition combinators pass their recursive arm as the thunk, and an eager-only carrier forces the recursion at construction and overflows before work begins. A custom carrier that omits the lazy overload silently loses the entire derived repetition surface.

[FORM_SELECTION_LAW]:
- Five structurally distinct forms, selected by where ownership lives, never by call-site convenience: state-threaded generated `Switch` when the consumer owns the logic but a closed vocabulary owns exhaustiveness; case-owned delegate rows when the vocabulary item IS the behavior and every consumer wants the same effect; frozen lookup tables when the key is a value and the result is static data derivable at type-load; extension blocks when the receiver is foreign but the behavior belongs to the consuming module; structural pattern dispatch when the input's shape — not its nominal type — discriminates.
- Hard triggers: three or more consumers writing identical full-coverage `Switch` arms over one vocabulary is the delegate-row collapse signal — completeness becomes a constructor invariant instead of a distributed call-site discipline. Frozen tables beat delegate rows only for pure static mappings and for cross-vocabulary joins where neither vocabulary owns the other's keys; a frozen table keyed by a vocabulary that restates its own delegate rows is a duplicate-entry maintenance burden with a silent missed-new-item failure mode. Structural patterns are chosen precisely when the input vocabulary is open; generated `Switch` when closed — structural dispatch over a closed family trades compile-time totality for a silent `_` arm.
- Mixing forms at one call site signals an unresolved ownership boundary — except the one valid composition: a `Switch` arm probing a frozen table is dispatch plus data retrieval, not two dispatch mechanisms. Extension blocks compose with generated dispatch at foreign-union boundaries: the block owns the module-to-domain translation surface, the generated `Switch` inside it owns per-case projection, and no helper sits between.

[STATE_THREADED_SWITCH]:
- `Switch<TState>(TState state, ...)` and `Switch<TState, TResult>` carry `allows ref struct` on both `TState` and `TResult`, so spans and ref-struct accumulators traverse dispatch unboxed. Closure-freedom is analyzer-enforced (non-static lambda draws the info diagnostic): every capture routes through the state parameter; multi-field context enters as a value tuple or record struct. Named arguments are mandatory at `Switch`/`Map` sites (analyzer-enforced), which makes the state parameter's name API surface: it defaults to `state` and collides only when a case type camel-cases to exactly `state` — resolved per-owner via the state-parameter-name option.
- `Map` is eager value dispatch — raw values, not functions; every branch value is computed before dispatch runs. It is the constant verdict-table form only; a computed expression in a `Map` argument is pre-paid on every call regardless of the live case. `SwitchMethods` and `MapMethods` are independent generation axes: a constant-verdict owner keeps `Map` and drops `Switch`; a data-carriage owner suppresses both and is consumed only by structural patterns — suppression leaves the backing field, conversions, and equality untouched.
- Ad-hoc unions dispatch through a private integer discriminator (`_valueIndex` switch, not an `is`-chain); separate typed fields per member by default, one boxed `object` field under the single-backing-field option. Stateless members store no field — the generated arm keeps the standard two-parameter shape and receives `default(Tn)`, so the lambda is `static (s, _) => ...`. Member renames via `TnName` are total (factory, parameters, accessors, conversions) — the resolution for two members sharing a simple type name. Ad-hoc unions reject `allows ref struct` on their own type parameters (generator error); regular `[Union]` nested cases emit guarded constraints, so ref-struct state threading routes through regular unions over named cases.
- Generic ad-hoc unions: any factory-triggering member (type parameter, interface, `object`, duplicate type) flips the whole union to factory-only construction — mixed construction is categorically impossible; `FactoryMethodGeneration.Always` makes the uniformity explicit so a later member addition cannot silently flip the surface. `SkipEqualityComparison` suppresses equality but never dispatch — the ephemeral-dispatch-carrier form (delegates, handles) that must not become a dictionary key.
- Generated totality versus authored: the exhaustive `Switch` has no `@default` parameter at all, so a new case breaks every call site at compile time; an authored `switch` with `_ => throw` compiles and silently misses the new case at runtime. The canonical shape is an abstract root, private constructors, sealed nested case records; nested hierarchies need class nodes at non-leaf levels. Nested-case parameter naming: `Simple` produces leaf-only names and fails the build (duplicate parameter) when sibling families share a leaf name; `Default` is collision-safe by construction.

```csharp
[Union] public partial record Shape
{
    public sealed record Circle(double Radius) : Shape;
    public sealed record Rect(double W, double H) : Shape;
}
// Generated totality: adding a case breaks this site at compile time.
double area = shape.Switch(
    circle: static (_, c) => Math.PI * c.Radius * c.Radius,
    rect:   static (_, r) => r.W * r.H,
    state: unit);
```

[DELEGATE_ROWS]:
- A delegate-from-constructor partial method becomes a per-item column: each row injects its implementation as a trailing constructor argument, dispatch cost is one field read plus invoke, and a missing argument on a new item is a compile-time arity failure — the column is enforced, not conventional. A custom named delegate type is generated only for `ref`/`in`/`out` parameters or an explicit name; otherwise `Func`/`Action` back the field. Declared access is preserved — private behavior rows expose no delegate-typed public state.
- N parallel behaviors over the same inputs collapse into one delegate returning a value tuple once N exceeds two — the cost model is one field per column per item plus redundant marshaling across separate invocations. Abstract members are the most expensive tier (derived nested case types, intermediate constructors, case-typed declaration sites) and are justified only for per-instance state, never per-item policy a static function can carry.

[FROZEN_TABLES]:
- Duplicate-key behavior is per-construction-form: the LINQ `ToFrozenDictionary(keySelector)` path throws on the first duplicate; span-fed `Create` is last-wins (later entries overwrite silently); `ToFrozenSet` deduplicates silently. The enforcement for `Create` is a structurally non-duplicating source — generated `Items` is deduplicated by definition. Construction selects an internal strategy once (empty singleton; small linear-scan variants; dense-integral; ordinal-string substring-hashing; perfect-hash fallback): smart-enum keys are reference types and route small tables to linear scan with no hash — an authored switch expression over a small vocabulary is not faster.
- `GetValueRefOrNullRef` returns `ref readonly TValue` into internal storage (test with null-ref check) — the single-probe path for read-heavy kernels, valid for the table instance's lifetime, existing only on the main-key surface. Alternate lookups (`ReadOnlySpan<char>` over string keys with an ordinal-class comparer) expose `TryGetValue`/`ContainsKey`/indexer only — no ref path; the safe acquisition is the try-form, and a `false` result is a hard startup failure, because the fallback instance silently null-refs every probe. Cache the lookup struct in a static field; it wraps the table reference with no state.
- A derived index over a vocabulary lives in a `Lazy<FrozenDictionary<...>>` with execution-and-publication semantics: lazy because item materialization must precede projection — an eager static initializer on the vocabulary type races item initialization. Frozen tables are rejected when the key set is open, when results derive from per-call context (that context belongs in the state-threaded arm), or when the table restates a vocabulary's own delegate rows.

[KEY_ADMISSION_SEAM]:
- One generated keyed declaration IS three dispatch forms at once: the key-to-case lookup table, the all-items sweep, and the boundary-admission factory are three static-abstract views of the same type — the "table or vocabulary or admission" selection dissolves, a call site reaches for whichever member its problem names, and because admission and dispatch share one error payload and one validate signature, admitting a batch of keys and dispatching over the resolved cases is one carrier-polymorphic pass, never an admit step feeding a separate dispatch step.
- A keyed vocabulary's generated `Validate(TKey?, IFormatProvider?, out TItem)` is the one-hop bridge from raw external key to dispatch-ready item — its body is the allocation-free `TryGet` probe plus a typed no-such-item error. Openness and closedness sit on opposite sides of one seam: any external key may arrive (open), a validation error or a vocabulary item leaves (closed), and the subsequent `Switch` is compile-time-total. A hand-written name-lookup followed by a separate dispatch is two hops where the generated seam is one. The span-keyed `Validate(ReadOnlySpan<char>, ...)` overload admits protocol text with zero string materialization.
- The comparer accessor is the single arbiter of key equality everywhere: a static-abstract accessor bound at definition time governs the owner's `Equals`/`GetHashCode`, and every frozen table, set, or index keyed on the owner resolves the same comparer — case-insensitive table dispatch becomes correct by construction, and because the accessor is a compile-time constant per owner, the frozen strategy analysis sees a known comparer and no call site can supply a divergent one. A generic index builder constrained on the accessor reads it directly, making table-comparer agreement provable rather than remembered.
- Conversion posture decides raw-key interoperability: implicit owner-to-key lets owner-valued code probe raw-key tables with no projection ceremony; explicit key-to-owner keeps admission visible; `None` on both severs the owner from raw-key dispatch entirely — the deliberate strictest posture. Mixed owner-versus-key relational operators (key-type overloads generation) must agree with the accessor's ordering for range-guard dispatch to be lawful.

[CARRIER_POLYMORPHIC_FORM]:
- The dispatch form selects which arm runs; the carrier the arms return selects how results combine — orthogonal axes. An arm returning `K<F, A>` makes the entrypoint simultaneously the `Fin`, `Eff`, `IO`, and `Validation` dispatcher: the carrier is a constrained type parameter bound at the call site, and the per-carrier sibling family (`RunFin`, `RunEff`, ...) is the rejected form. The trait stack is the precise capability contract — each constraint admits one combinator family (`Monad` binds, `Fallible<E,F>` fails/catches, `Choice` falls back, `Alternative` adds `Empty`, `Traversable` sweeps) — and over-constraining beyond what the body uses is dead surface that narrows the admitted carrier set for nothing. `.As()` recovers the concrete carrier exactly once at the materialization boundary; mid-pipeline concretization defeats the polymorphism.
- Dependencies ride the carrier, not the signature: capability projection (`Has`/ask) and state threading (`Stateful` get/modify/local, with save-restore inside the combinator) keep the dispatch signature at exactly the request; a trailing dependency tail is the smuggling form. Capability dispatch keys off the environment type parameter through accessor traits — the reader trait projects and locally rewrites the environment, the stateful trait puts/modifies, the mutation trait hands back a lock-free atomic cell of the inner environment — so adding a capability is adding a trait the environment implements, never a method on a service, and a call site widens its requirements by intersecting constraints, never by depending on a wider interface. The typed-error axis varies independently: pinning `Fallible<E, M>` to a domain error union makes the failure surface a closed family — the raise spelling is the typed case, recovery predicates pattern-match it, and the catch family (predicate, value-match, code-routed; recover-to-carrier, remap-error, recover-to-pure-value under `Applicative`) reads the typed error directly. The recovery operator family is generated once against the trait, so a named catch policy applies to every fallible carrier via `|` with no per-monad wiring and no pre-cast.
- Independent arms combine applicatively; dependent steps bind monadically — and the choice is load-bearing, not stylistic: a `Bind` chain over independent arms silently discards all failures after the first. The tuple applicative join (`(K<F,A>, ..., K<F,J>).Apply(func)`, arities 2-10) runs each slot — a `Switch`, a lifted table probe, a delegate-row invocation — under the carrier's `Apply`: with an accumulating carrier (failure type constrained to a monoid, so the append is free at every join) it reports the union of all failures; with an effect carrier it short-circuits. Same call site, same join — the carrier alone switches the semantics. `Traverse`/`TraverseM` are the same split at collection arity (`TraverseM` returns `K<M, Iterable<B>>` — the `.As()` lands at the call site). Accumulate-versus-abort is two trait routes through ONE carrier: its `Apply` appends failures, its `Bind` cannot.
- The join owns no logic: its only legitimate content is total construction over already-dispatched values — a branch inside the join is a fourth dispatch smuggled into combination and lifts into its own carrier-returning arm. All slots share one `F` by construction, so failure semantics are decided once; arms with heterogeneous effect needs narrow into the common carrier inside the arm. Expensive slots enter as `Memo` (lazy, compute-at-most-once, resettable): a short-circuiting carrier leaves later memo slots unforced, and the lazy `Choose` overload — selected by the right operand's static type — is the only way fallback dispatch skips a costly alternative.
- Operator algebra is three distinct semantics never conflated: `|` is `Choose` (first-success fallback, no error inspection; `Pure` terminus is constant recovery); `+` is `Combine` (monoidal merge of success structures); the predicate-guarded catch composes left-associatively as handler wrapping — inner handlers see errors before outer ones, so a broad catch before a narrow one shadows it silently. `oneOf` over a possibly-empty candidate set requires the `Alternative` floor (only it carries `Empty`); binary fallback needs only `Choice` — and `Choice` versus `SemigroupK` diverge on list-shaped carriers (first-non-empty versus concatenation), so selecting `Choose` where `Combine` was meant silently discards content.
- Looping dependent dispatch is `Recur(seed, step)` — the tail-recursive bind where the step returns continue-or-done as a closed two-case family: the live case picks the successor, termination is an arm, and the stack never grows regardless of length. Because the dispatch body is pure description until the carrier is interpreted, the same generated arms run under a deterministic test carrier and the production effect carrier — a totality proof under the cheap carrier is simultaneously a proof of the production entrypoint. Carrier changes are one structure-preserving arrow (natural transformation, with the co-variant twin for lifting), never a match-and-rebuild bridge; the bifunctor dispatch family derives every constant and side-specific arm from one abstract `Match` as static-virtual defaults.

```csharp
// Three independent dispatches joined once; the carrier alone decides
// accumulate-all (Validation) versus abort-first (Eff) — no flag, no branch.
static K<F, Order> Assemble<F>(Region r, Tier t, Channel c)
    where F : Applicative<F> =>
    (r.Switch(state: unit, domestic: static (_, d) => F.Pure(d.Zone),
                           foreign:  static (_, f) => F.Pure(f.Zone)),
     t.Switch(state: unit, gold:     static (_, _) => F.Pure(Priority.High),
                           standard: static (_, _) => F.Pure(Priority.Normal)),
     c.Switch(state: unit, web:      static (_, w) => F.Pure(w.Route),
                           partner:  static (_, p) => F.Pure(p.Route)))
    .Apply(static (zone, prio, route) => new Order(zone, prio, route));
```

[TYPE_LEVEL_STATIC_DISPATCH]:
- The self-constrained factory contract (`where TOwner : IObjectFactory<TOwner, TValue, TError>`, sole member a static abstract `Validate`) is the inversion of case dispatch: `Switch` is closed over one owner's cases; the constrained generic is open over the unbounded family of all generated owners and closed at each instantiation — one boundary entrypoint admits every present and future owner with no arm per owner, resolved by the JIT with no instance, no vtable, no reflection. The two-channel outcome (nullable typed error, out item) is the type-level validation cell one rail bridge lifts uniformly; the error vocabulary stays owner-chosen because the error type itself carries a static abstract `Create`, so a generic bridge synthesizes context-bearing errors without naming any concrete class.
- Span-keyed admission is a constraint instantiation, never a conversion: the span IS the `TValue` (the constraint carries `allows ref struct`), the owner generates a span-consuming parse body, and the generic entrypoint inlines aggressively — protocol text decodes into the closed vocabulary with zero heap traffic on both legs, since the outbound instance projection mirrors inbound admission through the same family. The generic type-converter written once against the contract pair and instantiated per owner is the proof shape: one deep parameterized surface owning conversion for an unbounded owner set, the inverse of one shallow converter per owner.
- Static-virtual dispatch has a hard reach limit: an unresolved static abstract member is not a first-class value — no open delegate, no method group, no expression tree. The boundary law: type-level static dispatch serves boundaries whose owner is resolved at compile time at the call site; runtime-discovered owners dispatch over the generated metadata surface instead — the static abstract metadata list reifies every declared admission front door (each with its boundary-routing flags) as a typed vocabulary a generic codec folds over, keeping the runtime-discovered case reflection-free. Choosing the static-virtual form for runtime discovery reintroduces reflection through closed-generic invocation — the exact cost the contract exists to delete.

```csharp
// One boundary admits every generated owner; the failure vocabulary stays owner-defined.
static Fin<TOwner> Admit<TOwner, TValue, TError>(TValue value, IFormatProvider? culture = null)
    where TOwner : IObjectFactory<TOwner, TValue, TError>
    where TValue : notnull, allows ref struct
    where TError : class, IValidationError<TError> =>
    TOwner.Validate(value, culture, out var item) is { } error
        ? FinFail<TOwner>(Error.New(error.ToString() ?? "rejected"))
        : FinSucc(item!);
```

[EXTENSION_BLOCKS]:
- Extension blocks are pure projection and behavioral enrichment on a receiver the module does not own — no state, no `abstract`/`virtual`/`override`/`sealed`/`partial`/`protected` members, no participation in inheritance dispatch; a mixin architecture falls back to interfaces. State a projection needs enters via a frozen table keyed on receiver identity, a thread-local slot, or a parameter. Blocks require a top-level non-generic static class; multiple blocks on one receiver merge into one declaration space within a class and are additive across classes — which is also the test-assembly seam for probe members.
- Receiver modifiers follow four compiler rules: `ref` receivers require a value type or `where T : struct`; `ref readonly`/`in` receivers require a concrete named value type (the generic struct constraint is rejected); instance operators on struct receivers require `ref`; static-class receivers admit no operators. Non-method members (properties, operators) must satisfy inferrability — every block type parameter appears in the receiver-plus-member parameter set; body-only type parameters are errors.
- Two hard egress limits: expression trees reject extension property access and extension-defined `&&`/`||` operators unconditionally (methods remain legal as static-call rewriting) — surfaces feeding query translators fall back to classic static extension methods; and block members lower to static methods carrying a marker attribute, which disqualifies them as interceptors. Members are barred from receiver privates, which union dispatch never needs (the generated surface is public); but a `private` member in one block is reachable from a block on a different receiver in the same static class — shared helpers belong at file scope or on the containing class, never inside a block.
- Priority steering groups by containing static class only: the preferred and demoted overloads must be co-declared or the prune never engages; cross-class same-name extensions compare by ordinary betterness regardless of priority.

[STRUCTURAL_PATTERNS]:
- The open-vocabulary form: list/slice patterns over spans, positional and property-relational combinations, constant content — correct where any caller may extend the input space (protocol decoding, arity detection, composite conditions). Its totality is runtime-only: a missing arm is a warning plus a runtime exception, never a compile error; a `_` arm that proves totality by comment is the diagnostic to eliminate, and where totality matters the closed generated surface owns the dispatch.
- The two compose as layers: generated `Switch` owns closed case dispatch at the union boundary; structural patterns destructure the statically-typed payload inside the arm — two layers of exhaustiveness, no `is` re-check, positional patterns binding directly off primary-constructor cases. `when` guards are legitimate only for constraints patterns cannot express (range guards, derived equality); a guard restating an expressible property pattern collapses.

[INTERCEPTORS]:
- Three constraints bound interceptors as a weaving mechanism: the interceptor's containing type must be non-generic (eliminating substitution of `Switch` call sites on generic request types); only ordinary member methods qualify (extension-block members and classic extension methods are disqualified by their lowered shapes); and only the checksum-based interceptable-location API is supported. Interceptors therefore weave definition-time policy at concrete, non-generic, module-scoped call sites only — never across a generic dispatch surface.

[PARTIAL_DISPATCH_MECHANICS]:
- Stop-at overloads coarsen a named sub-hierarchy to one atomic arm: the generator walks the flat case set, recursively removes the stop type's descendants, and seats the stop type as a leaf — exhaustive at the boundary, no fallback. A leaf stop type is a no-op; a new leaf under a stop type does NOT break stop-at call sites — the deliberate gap total dispatch would catch, which is why the form carries the semantic claim that the subtree is intentionally opaque to this caller. The separately named partial dispatchers express the different claim "I handle some cases, the rest hit `@default`" — required-versus-optional fallback distinguishes their func and action forms, and per-case optionality travels as set/unset argument slots, so the binding choice between total, partial, and stop-at is recovered entirely from argument arity and optionality shape.
